## 1. 问题

### 痛点

网站在移动端（375px-428px）无法正常使用：首页、文章详情、归档页的双栏布局严重挤压，文字溢出视口，侧栏叠在正文上。导航栏 6 个项在小屏上挤成一行无法操作。暗色模式下 about 页和 archive 页存在硬编码亮色值（inline style），切换后出现白色文字块和背景异常。关于页全部是占位符文本，访客无法了解作者身份。

### 影响范围

移动端流量占博客访问的主要部分（估算: >50%）。当前移动端体验等于放弃一半以上的读者。暗色模式是已上线功能但体验残缺，比没有更糟——偏好暗色主题的开发者群体占技术内容站读者的高比例。关于页是建立作者信任感的核心页面，空白状态直接影响读者是否愿意继续阅读。

### 为什么现在做

v1 roadmap 的第一个里程碑。后续 M2（阅读体验）和 M3（细节打磨）都依赖 M1 的基础可用性。移动端布局破损属于 P0 级缺陷——不修复则网站实质上不可用。

## 2. 目标用户

| 角色 | 场景 | Before | After |
|------|------|--------|-------|
| 使用 375px-428px 设备的手机读者 | 通过社交平台分享链接进入网站阅读文章 | 双栏挤压、文字溢出视口、无法正常阅读，需手动缩放 | 单栏堆叠布局，文字占满宽度，无水平滚动条 |
| 偏好暗色主题的开发者读者 | 点击主题切换按钮使用暗色模式阅读 | 部分页面仍为亮色背景/文字，视觉割裂 | 全站暗色一致，无颜色异常区块 |
| 通过搜索引擎首次到达的访客 | 浏览关于页了解作者和网站定位 | 看到占位符文本，无法判断作者可信度 | 看到真实作者简介、联系方式和写作动机 |

## 3. 核心假设

- **假设**: 修复移动端布局 + 暗色模式 + 关于页内容后 → 读者在任何设备上都能无障碍地阅读全部内容
- **验证方式**: 所有页面在 375px/768px/1440px 三断点下无布局溢出；`grep` 搜索 inline style 中 `#` 色值结果为 0；关于页有真实作者名和至少一个可点击联系方式

## 4. 方案

- **Before**: 手机上打开首页，双栏挤压成两条窄带，文字无法阅读 → **After**: 768px 以下自动切换单栏堆叠，特稿在上、侧栏文章列表在下
- **Before**: 手机上打开文章详情，侧栏标签/主题叠在正文上 → **After**: 侧栏移到正文下方，正文占满宽度
- **Before**: 手机上导航栏 6 个项挤成一行 → **After**: 导航项自动换行，所有项可操作
- **Before**: 切换暗色模式后 about 页联系方式区域仍为白色背景 → **After**: 所有颜色通过 CSS 变量响应主题切换
- **Before**: 关于页显示"我是一个写代码的人"占位文本 → **After**: 显示真实作者简介、写作动机和联系方式

## 5. 验收标准

- 用 375px 宽度浏览器打开首页 → 应看到单栏布局，特稿标题和正文可正常阅读，无水平滚动条
- 用 375px 宽度打开文章详情页 → 应看到正文占满宽度，侧栏标签在正文下方显示
- 用 375px 宽度打开归档页 → 应看到年月列表和统计侧栏改为堆叠，筛选按钮可点击
- 用 375px 宽度打开关于页 → 应看到主内容和侧栏堆叠显示，联系方式可读
- 导航栏在 375px 下 → 应看到所有项不溢出，均可点击
- 切换暗色模式 → 应看到全站无白色闪烁，无颜色异常区块
- 在项目中搜索 `style="` 包含 `#` 开头色值的 inline style → 应看到结果为 0
- 打开关于页 → 应看到真实作者名（非占位符）
- 关于页 → 应看到至少一个可点击的联系方式链接

## 6. 排除项

- 不做汉堡菜单（导航项少于 6 个，换行即可满足需求）
- 不做平板专属布局（768px 以上复用桌面双栏，无需单独适配）
- 不做暗色模式自动跟随系统偏好的实时监听（当前初始检测已有，满足使用场景）
- 不做关于页头像上传功能（保持 emoji 占位，避免引入图片处理依赖）
- prose 样式的暗色适配推迟到 M2（与 TOC、代码块增强一起做，减少重复改动）
