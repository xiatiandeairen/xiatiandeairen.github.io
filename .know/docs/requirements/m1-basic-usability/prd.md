# M1 基础可用性

## 1. 问题

网站目前在移动端无法正常使用。首页、文章详情、归档页的双栏布局在 768px 以下宽度的设备上严重挤压，文字溢出、侧栏叠在正文上。导航栏的 · 分隔项在小屏上挤成一团。暗色模式下，about 页和 archive 页存在硬编码的亮色值（inline style），切换暗色后出现白色文字块和背景异常。关于页全部是占位符内容，访客无法了解作者身份。

移动端流量占博客访问量的主要部分。当前状态等于放弃了一半以上的读者。暗色模式是已上线功能但体验残缺，比没有更糟。关于页是建立作者信任感的核心页面，空白状态直接影响读者是否愿意继续阅读。

v1 roadmap 将 M1 定义为第一个里程碑——后续 M2（阅读体验）和 M3（细节打磨）都依赖 M1 的基础可用性。

## 2. 目标用户

**手机用户**：通过社交平台分享链接进入网站的读者，使用 375px-428px 宽度的 iOS/Android 设备。当前体验：双栏挤压、文字溢出、无法正常阅读。

**暗色模式用户**：偏好暗色主题的开发者群体（技术内容站的高占比人群）。当前体验：点击切换按钮后部分区域仍为亮色，视觉割裂。

**首次访客**：通过搜索引擎或文章分享第一次来到网站的读者。当前体验：关于页是占位符，无法了解作者是谁、为什么写这些内容。

## 3. 核心假设

**修复移动端布局 + 暗色模式 + 关于页内容后，读者在任何设备上都能无障碍地阅读全部内容。**

验证方式：所有页面在 375px / 768px / 1440px 三个断点下无布局溢出；暗色模式下全站无硬编码颜色残留（grep 检查 inline style 中的 `#` 色值为 0）；关于页有真实的作者名、简介和至少一个联系方式。

## 4. 方案

- **Before**: 手机上打开首页，双栏挤压成两条窄带，文字无法阅读 → **After**: 自动切换为单栏堆叠布局，特稿在上、侧栏文章列表在下
- **Before**: 手机上打开文章详情，侧栏标签/主题叠在正文上 → **After**: 侧栏移到正文下方，正文占满宽度
- **Before**: 手机上导航栏 6 个项挤成一行 → **After**: 导航项自动换行或折叠
- **Before**: 切换暗色模式后 about 页的联系方式区域仍为白色背景 → **After**: 所有颜色通过 CSS 变量响应主题切换
- **Before**: 关于页显示"我是一个写代码的人"占位文本 → **After**: 显示作者真实简介和联系方式

## 5. 验收标准

- 用 375px 宽度浏览器打开首页 → 看到单栏布局，特稿标题和正文可正常阅读，无水平滚动条
- 用 375px 宽度打开文章详情页 → 正文占满宽度，侧栏标签在正文下方显示
- 用 375px 宽度打开归档页 → 年月列表和统计侧栏改为堆叠，筛选按钮可点击
- 用 375px 宽度打开关于页 → 主内容和侧栏堆叠显示，联系方式可读
- 导航栏在 375px 下不溢出，所有项可访问
- 切换暗色模式 → 全站无白色闪烁，无颜色异常区块
- 在项目中搜索 `style="` 包含 `#` 开头色值的 inline style → 结果为 0（所有颜色走 CSS 变量）
- 关于页显示真实作者名（非"我是一个写代码的人"占位符）
- 关于页至少包含一个可点击的联系方式链接

## 6. 排除项

- 不做汉堡菜单（导航项少，换行即可）
- 不做平板专属布局（768px 以上复用桌面双栏）
- 不做暗色模式自动跟随系统偏好的实时监听（当前已有初始检测，够用）
- 不做关于页的头像上传功能（保持 emoji 占位）
- prose 样式的暗色适配放 M2（与 TOC、代码块增强一起做）
