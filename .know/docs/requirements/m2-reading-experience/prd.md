# M2 阅读体验提升

## 1. 问题

M1 解决了基础可用性，但读者在网站上的停留深度不足。长文章（2000-4000 字）缺少导航手段，读者无法快速跳转到感兴趣的章节。7 篇 AI Skills 系列文章之间没有串联入口，读者不知道阅读顺序。读完一篇后没有引导继续阅读。技术文章的代码片段无法一键复制。

这些问题的共同点是：读者来了、读了一篇，但缺少"继续读下去"的动力和便利性。

## 2. 目标用户

**深度阅读者**：对文章主题感兴趣，愿意花 10-15 分钟阅读长文的开发者。需要目录导航快速定位，需要相关推荐发现更多内容。

**系列读者**：从某一篇 AI Skills 文章入口进来，想了解整个系列的读者。需要知道共几篇、当前读到第几篇、下一篇是什么。

**实践者**：想把文章中的代码片段用到自己项目的开发者。需要一键复制代码块。

## 3. 核心假设

**增加文章内导航和文章间引导后，单次访问的平均浏览页数从 1 提升到 2+。**

验证方式：文章详情页有可点击的 TOC 且滚动高亮；系列页面有阅读顺序导航；文章底部有相关推荐；代码块有复制按钮且点击后有成功反馈。

## 4. 方案

- **Before**: 长文章只能线性滚动阅读 → **After**: 侧栏显示自动生成的目录，点击跳转，滚动时高亮当前章节
- **Before**: 7 篇 AI Skills 文章散落在文章列表中 → **After**: `/series` 页面展示系列入口，文章详情页显示系列导航条（上一篇/下一篇）
- **Before**: 读完一篇没有引导 → **After**: 文章底部基于标签重合度推荐 2-3 篇相关文章
- **Before**: 代码块只能手动选中复制 → **After**: 代码块右上角显示语言标签和复制按钮，点击后显示"已复制"反馈

### 任务

| 任务 | 文档 | 进度 |

| 代码块增强 | — | 1/1 |

## 5. 验收标准

- 打开任意长文章（如 ai-agent-operating-system）→ 侧栏显示所有 h2/h3 标题的目录
- 滚动文章 → 目录中对应标题高亮
- 点击目录项 → 页面平滑滚动到对应标题
- 移动端 → 目录隐藏（侧栏已折叠到底部，TOC 不显示）
- 打开 `/series` → 看到 AI Skills 系列，7 篇文章按阅读顺序列出
- 打开系列中的任一篇文章 → 顶部或底部有系列导航条，显示当前篇数和上下篇
- 打开任意文章 → 底部（上下篇导航之前）显示 2-3 篇相关文章，标题可点击
- 相关文章基于标签重合度排序，不推荐当前文章自身
- 文章中有代码块 → 右上角显示语言名称和复制按钮
- 点击复制按钮 → 代码内容写入剪贴板，按钮文字变为"已复制"并在 2 秒后恢复

## 6. 排除项

- 不做 TOC 的移动端浮动展开（移动端直接不显示 TOC）
- 不做系列文章的自动检测（手动在 frontmatter 中标记系列和顺序）
- 不做代码块的行号显示
- 不做代码块的 diff 高亮
