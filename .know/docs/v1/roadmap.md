# 夏天的爱人 产品路线图

## 1. 产品愿景

| | |
|---|---|
| **解决什么问题** | 需要一个个人内容发布平台，承载技术写作、阅读笔记和生活随想 |
| **给谁用** | 作者本人（发布）+ 对 AI 工程化/技术写作感兴趣的开发者（阅读） |
| **核心差异** | 报纸/杂志排版风格的静态站，强调阅读体验而非社交互动 |

## 2. 版本规划

### v1 — 打磨至成熟

| 维度 | 标准 |
|------|------|
| **功能** | 所有已实现页面（首页、文章、归档、关于、搜索、标签、主题、RSS）功能完整无缺陷 |
| **质量** | Lighthouse Performance ≥ 90, Accessibility ≥ 90；所有页面 375px/768px/1440px 三断点无布局问题；暗色模式无硬编码颜色残留 |
| **用户** | 作者可以独立发布文章并在所有设备上正常阅读 |

#### 里程碑

| # | 里程碑 | 验证点 | 进度 | 需求 |
|---|--------|--------|------|------|
| M1 | **修复基础可用性** — 响应式适配、暗色模式修复、关于页真实内容 | 手机浏览器打开所有页面无布局溢出；切换暗色模式无白色闪烁或颜色异常 | 1/1 | [m1-basic-usability](../requirements/m1-basic-usability/prd.md) |
| M2 | **提升阅读体验** — TOC 目录、系列文章、相关推荐、代码块增强 | 文章详情页侧栏显示可点击目录；AI Skills 系列有独立入口页和顺序导航；代码块有复制按钮 | 0/4 | — |
| M3 | **细节打磨** — 搜索优化、样式对齐、归档 URL 参数、清理示例数据、OG 图片 | 搜索结果有关键词高亮；标签/主题页使用统一排版；归档筛选状态可通过 URL 分享 | 0/5 | — |

## 3. 当前版本

### 包含

| 能力 | 解决什么问题 |
|------|-------------|
| **全站响应式** | 移动端双栏布局挤压导致无法正常阅读 → 768px 以下自动切换单栏 |
| **暗色模式完整适配** | 部分页面 inline style 硬编码亮色值，暗色下出现白色文字/背景异常 → 所有颜色走 CSS 变量 |
| **关于页真实内容** | 当前全是占位符，访客无法了解作者 → 填入真实个人信息和联系方式 |
| **文章目录导航** | 长文章（2000+ 字）无法快速跳转到特定章节 → 侧栏自动生成 TOC 并高亮当前位置 |
| **系列文章入口** | 7 篇 AI Skills 文章无阅读顺序指引 → 独立系列页面串联全部文章 |
| **相关文章推荐** | 读完一篇后没有引导继续阅读 → 基于标签匹配推荐 2-3 篇相关文章 |
| **代码块增强** | 技术文章的代码片段无法一键复制 → 添加语言标签和复制按钮 |
| **搜索体验优化** | 搜索结果无高亮、样式粗糙 → 关键词高亮 + 空状态引导 + 样式对齐 |
| **标签/主题页样式对齐** | 这两个页面排版层级和交互模式与新设计语言不一致 → 统一 |
| **归档页 URL 参数** | 分类筛选状态无法分享 → 筛选写入 URL query string |
| **清理示例数据** | 11 篇原始示例笔记稀释真实内容 → 删除或替换 |
| **OG 分享图片** | 文章分享到社交平台无预览图 → 自动生成标题+站名卡片图 |

### 排除

- 评论系统（v2 考虑）
- 多语言支持
- CMS 后台管理界面
- 自定义域名配置（当前使用 github.io）
- 站内统计/分析
- Newsletter 订阅

## 4. 风险与依赖

| 风险 | 影响 | 缓解 |
|------|------|------|
| 关于页内容依赖用户提供真实信息 | M1 无法关闭 | 先用最小版本（姓名+一句话介绍+GitHub 链接），后续迭代丰富 |
| 示例数据删除可能影响标签/主题页面的路由生成 | M3 清理示例数据时页面报错 | 删除前检查哪些标签/主题只被示例数据使用，确认无残留路由 |
| OG 图片自动生成需要额外依赖（canvas/sharp） | M3 引入新 npm 依赖增加构建复杂度 | 评估 Astro 生态现有集成方案，优先用无额外依赖的方案 |
