> 训练一个模型,从线性回归到几千亿参数的 transformer,本质是同一个三步循环:**算一次损失(loss,"错得多离谱"的一个数),问微积分哪边是下坡(梯度),朝下坡走一步**。Adam、warmup、layer norm、梯度裁剪,全是这个循环上的装饰。这章把循环拆到没有黑盒——梯度就是"每个权重动一点点,损失变多少",反向传播就是链式法则加上"中间结果别重算"。配套代码 `examples/math-for-builders-from-scratch/src/stage04-optim.ts` 把每个机制都跑了一遍,而且每个数字都是算出来的、不是断言的。

读法提示:这章的钩子是两个亲手实验。一是 ✦ 亲手推一次反向传播,让你永远记住它不是魔法;二是 ⚡ 亲手把学习率调到发散,看 loss 怎么从 6.6 爆成 9.5×10¹⁷。两个实验的真实输出都贴在下面,你可以自己跑 `npx tsx src/stage04-optim.ts` 对拍。

## 一、导数与梯度:下坡方向从哪来

### 1. 导数是"动一点点,变多少"

先把术语剥到最朴素。**导数(derivative)就是"输入动一点点,输出变多少"的比率**。函数 `f(x)=x²` 在 `x=3` 处的导数是 6,意思是:你把 x 从 3 推到 3.001,f 大约涨 0.006(=6×0.001)。仅此而已。

**梯度(gradient)是导数在多变量下的推广**:一个函数有 N 个输入(模型有 N 个参数),梯度就是一个 N 维向量,第 i 个分量是"只动第 i 个输入、其它不动时,输出的变化率"。它指向**函数上升最快的方向**——所以下坡方向就是 `-梯度`。这是整个训练的全部直觉:梯度告诉你每个参数往哪个方向、动多大幅度,损失下降最快。

为什么训练要用梯度而不是别的?因为它是**局部信息**——只需要在当前这一点算一次,就知道往哪走。模型有几十亿参数,你不可能枚举所有方向试,但你能一次性算出所有方向的"敏感度",这正是梯度。

### 2. 解析梯度 vs 数值梯度:谁给谁当裁判

工程上有两种拿到梯度的方式:

- **解析梯度(analytic gradient)**:手推或自动微分(autodiff)出公式,直接算。快、精确,但**容易推错**——一个符号错、一个系数漏,整个训练悄无声息地朝错方向走。
- **数值梯度(numeric gradient)**:用有限差分(finite differences,"前后各挪一点点看差值")逼近。慢(每个参数要算两次前向),但几乎不会推错。

实战的标准做法不是二选一,而是**让数值梯度给解析梯度当裁判**——这就是每个框架测试里都有的"梯度检查(gradient check)"。stage 的 ① 就在做这件事。看代码 `src/stage04-optim.ts` 里的有限差分实现:

```typescript
function numericGradient(f: (x: number[]) => number, x: readonly number[], h = 1e-5): number[] {
  const g = new Array(x.length).fill(0);
  for (let i = 0; i < x.length; i++) {
    const plus = x.slice();
    const minus = x.slice();
    plus[i] += h;
    minus[i] -= h;
    g[i] = (f(plus) - f(minus)) / (2 * h);  // 中心差分,误差 O(h²)
  }
  return g;
}
```

这里有两个被新手忽略、却实打实影响正确性的工程决策,值得停下来看:

**为什么用中心差分 `(f(x+h)-f(x-h))/2h` 而不是前向差分 `(f(x+h)-f(x))/h`?** 中心差分的误差是 O(h²),前向是 O(h)。同样的 h,中心差分大约精确 h 倍。当你拿它当裁判时,裁判本身得足够准,否则你分不清是解析梯度错了还是裁判不准。

**为什么 `h=1e-5` 而不是更小?** 直觉会说 h 越小越逼近真导数。错。h 太小时,`f(x+h)` 和 `f(x-h)` 几乎相等,两个相近的浮点数相减会发生**灾难性抵消(catastrophic cancellation)**——有效位数被吃掉,结果是数值噪声。h 太大又留下截断误差。float64 里 1e-5 到 1e-6 是甜点区。这个 trade-off 本身就是"为什么不能无脑追求精度"的活教材,第 06 章数值稳定性会把它讲透。

stage ① 在 `f(x,y,z)=x²+3xy+sin(z)` 这个非平凡函数上对拍,真实输出是:

```
① 数值梯度 vs 解析梯度（有限差分对拍）
   解析梯度 = [0.500000, 3.900000, 0.877583]
   数值梯度 = [0.500000, 3.900000, 0.877583]
   最大分量误差 = 2.03e-11  → 接近 0，解析公式正确
```

`2.03e-11` 这个数字是关键。它不是 0,因为有限差分本来就有 O(h²) 的近似误差;但它小到 10⁻¹¹ 量级,远低于任何会影响训练的阈值。**这就是"梯度检查通过"的标准长相**——不是等于零,而是小到你信得过解析公式。实战里设阈值通常 1e-5 到 1e-7,你写的任何手推梯度,上线前都该这么对拍一次。

> 这里 `f(x,y,z)` 是个玩具函数,选它纯粹因为三个分量分别覆盖平方项、交叉项、三角函数,能把解析推导的常见出错点都暴露。可迁移的不是这三个数,是"梯度检查"这个习惯。

## 二、梯度下降:把循环跑起来

### 1. 损失曲面与那个三步循环

有了梯度,训练就是一句话:**`参数 ← 参数 − 学习率 × 梯度`,循环到收敛**。学习率(learning rate)是你每步沿下坡走多远的步长。这一行就是梯度下降(gradient descent, GD)的全部。

stage 用它拟合一条带噪声的直线 `y = 2x − 1 + N(0, 0.3)`(N 是高斯噪声)。这里要诚实标注:**这是 seeded 合成数据**——已知真参数 slope=2、intercept=−1,加上固定随机种子的高斯噪声。为什么用玩具问题?因为有了已知答案,才能**验证优化器真的在工作**,而不是 benchmark 它的速度。能迁移到真实训练的是机制和曲线形状(收敛/震荡/发散),不是绝对的 loss 数值。

GD 的核心循环在 `src/stage04-optim.ts`:

```typescript
function gradientDescent(params0, lr, steps, grad, loss): GdRun {
  let params = params0.slice();
  const losses: number[] = [];
  let diverged = false;
  for (let i = 0; i < steps; i++) {
    const l = loss(params);
    losses.push(l);
    if (!Number.isFinite(l)) {  // loss 变成 NaN/Inf 就停止记录噪声
      diverged = true;
      break;
    }
    params = add(params, scale(grad(params), -lr));  // p ← p − lr·grad
  }
  return { params, losses, diverged };
}
```

注意一个**刻意的设计**:它不自动裁剪、不自动早停,只记录 `diverged` 标志然后让调用方把爆炸画出来。`backward` / `gradientDescent` 这类纯函数把时钟、IO、随机源都隔在外面(种子从 `main` 注入,绘图交给调用方),这是"纯核+命令壳"的写法,也让 loss 序列能被原样画成曲线。藏起发散就毁了这章的课。真实训练器会在这里做梯度裁剪或降 lr——但你得先亲眼见过它怎么坏。

stage ② 的真实输出,80 步拟合:

### 2. ⚡ 学习率:最反直觉、最折磨人的旋钮

这是本章的开放钩子,也是每个训练过模型的人都栽过的坑:**学习率太大会发散,太小爬不动,而"合适"的窗口可能很窄,且没有通用公式告诉你它在哪。**

为什么反直觉?新手以为学习率就是"快慢旋钮",调大=训练快。错。学习率不是线性的快慢,它有一个**相变阈值**:跨过去,行为从"收敛"突变成"发散",不是慢慢变差,是质变。

对一个二次型 loss(均方误差正好是参数的精确二次函数),这个阈值是可以**精确算出来**的,等于 `2/L`,其中 L 是损失曲面的曲率——具体说是 Hessian 矩阵(二阶导数矩阵,描述曲面弯曲程度)的最大特征值。stage 没有猜这个阈值,而是闭式算了出来。看 `src/stage04-optim.ts` 里 MSE Hessian 最大特征值的闭式解:

```typescript
function topHessianEigenvalue(data: LinearData): number {
  const n = data.xs.length;
  const sxx = data.xs.reduce((s, x) => s + x * x, 0);
  const sx = data.xs.reduce((s, x) => s + x, 0);
  const a = (2 * sxx) / n;
  const b = (2 * sx) / n;
  const c = 2;  // (2/n)·Σ1 = 2
  // 对称 2x2 矩阵 [[a,b],[b,c]] 的最大特征值,取 '+' 根:
  // 稳定性由最陡方向(最大曲率)决定,只要某一个方向步子超界就发散
  return (a + c + Math.sqrt((a - c) ** 2 + 4 * b * b)) / 2;
}
```

为什么取最大特征值而不是平均?因为 GD 一旦在**最陡的那个方向**步子超界就会发散,哪怕其它所有方向都好好的。稳定性是被最坏方向卡死的。

stage ③ 在同一个问题上扫了三档学习率,真实输出:

看清楚阈值的数字:`L=2.757`,`2/L=0.725`。lr=0.870 只是阈值的 1.2 倍——**仅仅超界 20%,行为就从收敛翻转成发散**。这不是"调大一点训练快一点",是掉下悬崖。这就是为什么调学习率这么折磨:好与坏之间没有缓冲带。

⚡ **开放问题:学习率没有通用解。** `2/L` 只对二次型 loss 精确成立,而真实神经网络的 loss 高度非凸、Hessian 随训练动态变化、还分不同层不同尺度——你算不出一个全局的 L,也就没有公式给你最优 lr。学界至今靠的是:warmup(先小后大)、cosine/线性衰减调度、Adam/AdamW 这类自适应方法各维度自己估步长,以及 LR range test 之类的经验扫描。这些都是**经验启发式,不是解析解**。"给定网络和数据,最优学习率调度是什么"目前无通用解、仍在活跃研究(μP/maximal update parametrization 这类工作试图让最优 lr 跨规模可迁移,但远未通用)。

### 3. 失败模式:loss 真的爆成 Infinity

stage 最后把学习率推到 5 倍阈值,让它真的溢出。这不是吓唬人的描述,是跑出来的:

```
--- 失败模式：学习率远超 2/L，loss 真爆成 Infinity ---
   lr=3.627 (=5×阈值) 前几步 loss: 6.60e+0 → 4.83e+2 → 3.74e+4 → 2.96e+6 → 2.37e+8 → 1.91e+10
   loss 在第 162 步变为 Infinity（检测到非有限值，提前停止）
```

每步 loss 大约乘以 ~77 倍(6.6→483→37400→...),这是**几何级数增长**。根因很干净:每步残差被放大 `|1 − lr·L| > 1` 倍,当这个倍数超过 1,误差就指数膨胀,直到撞穿 float64 的上限(~1.8×10³⁰⁸)变成 Infinity,再算一步就是 NaN,从此整个网络的参数全废。

为什么要跑到第 162 步才 Infinity?因为从 6.6 涨到 10³⁰⁸ 需要足够多步的几何累积。stage 故意选了能**真撞到非有限值**的 lr 和步数,展示的是真 Infinity,不是"一个大数字然后我手一挥说它早晚 NaN"。

实战对策三选(代码注释里也列了):**调小 lr、梯度裁剪(gradient clipping,把过大的梯度截断)、或用自适应步长(如 Adam)**。再强调一次诚实边界:这里的绝对数值是 toy 数据,偏乐观;能迁移到真实训练的是"**越界即发散**"这个相变行为,以及"loss 突然变 NaN ≈ 八成是学习率/数值问题"这个排障直觉。你真实训练里见到 loss 变 NaN,第一反应应该就是查这里。

## 三、链式法则与反向传播:为什么它不是魔法

### 1. ✦ 亲手推一次,让你永远记住

反向传播(backpropagation, backprop)被神化得太厉害,以至于很多人以为它是某种深奥算法。它不是。**反向传播 = 链式法则 + 复用中间结果。** 链式法则你高中就学过:复合函数 `f(g(x))` 的导数是 `f'(g(x))·g'(x)`——一层层导数相乘。反向传播就是把这个规则在计算图上从输出端往输入端、逐层应用一遍,顺手把前向算过的中间结果存下来别重算。就这两点。

stage 搭了个最小的两层网络让你看全过程:`x(2维) → [W1, b1] → tanh → [W2, b2] → 标量 yhat`,loss = `0.5·(yhat − target)²`。那个 0.5 是故意的——求导后正好抵消平方的 2,让 `dL/dyhat = (yhat − target)` 干净利落,没有多余系数。

前向传播存下反向要用的中间激活,看 `src/stage04-optim.ts`:

```typescript
function forward(net, x) {
  const z1 = [dot(net.W1[0], x) + net.b1[0], dot(net.W1[1], x) + net.b1[1]];
  const a1 = z1.map(Math.tanh);          // 隐层激活,反向要复用
  const yhat = dot(net.W2, a1) + net.b2; // 输出,回归头无激活
  return { yhat, z1, a1 };
}
```

反向传播逐层应用链式法则,从输出端往回:

```typescript
function backward(net, x, target): ToyGrads {
  const { yhat, a1 } = forward(net, x);
  const dyhat = yhat - target;                    // dL/dyhat,0.5 抵消了平方的 2

  // 输出层
  const dW2 = a1.map((a) => dyhat * a);           // dL/dW2_j = dyhat · a1_j
  const db2 = dyhat;

  // 回传到隐层激活,再穿过 tanh 到 z1
  const da1 = net.W2.map((w) => dyhat * w);       // dL/da1_j = dyhat · W2_j
  const dz1 = da1.map((d, j) => d * (1 - a1[j] * a1[j])); // 过 tanh:乘 (1−a1²)

  // 隐层参数:dz1 是上游信号,x 是本地输入
  const dW1 = dz1.map((d) => x.map((xi) => d * xi)); // 外积 dz1 ⊗ x
  const db1 = dz1.slice();
  return { dW1, db1, dW2, db2 };
}
```

把这段读三遍。每个梯度都是 **(上游传来的信号) × (本地导数)** 这一个模式:`dW2 = dyhat × a1`,`dz1 = da1 × tanh的导数`,`dW1 = dz1 × x`。从右往左,每往前一层,上游信号就是上一层算好的那个 d。这就是链式法则被"乘"出来的样子——你亲手写一遍,以后看任何框架的 autograd 都知道它在干嘛。

### 2. 为什么存 a1:复用是反向传播能 scale 的全部原因

`forward` 为什么要把 `a1` 返回出来?因为反向时每个本地导数都是前向激活的函数(`dW2` 要用 `a1`,`dz1` 要用 `a1²`)。**把前向激活缓存下来,一次反向传播的代价就是 O(一次前向);不缓存、每个参数单独用有限差分去求,代价是 O(参数量 × 一次前向)。**

这个差距就是反向传播能训千亿参数、而有限差分只配当裁判的原因。一个 10 亿参数的网络,有限差分要做 20 亿次前向才能求一次梯度;反向传播一次前向+一次反向就够。stage 里有限差分只用来对拍验证,正是因为它太慢,只配当 oracle(裁判)。

stage ④ 的真实输出,而且做了梯度检查:

```
④ 两层 toy 网络一次前向 + 反向传播  x=[1,-2] target=0.5
   预测 yhat = 1.186559   loss = 0.235682
   dW1 = [[0.1466, -0.2932], [-0.0620, 0.1240]]
   db1 = [0.1466, -0.0620]
   dW2 = [0.5724, -0.6328]   db2 = 0.6866
   反向传播 vs 有限差分 最大误差 = 1.23e-11  → backprop 正确
```

`1.23e-11`——又是 10⁻¹¹ 量级,和 ① 一个数量级。这就是"backprop 推对了"的证据:把整个网络的解析梯度(手推链式法则)和数值梯度(有限差分裁判)逐分量对拍,最大差异小到忽略。**你写的任何自定义层、任何手推梯度,上线前都该跑这一步。** 这不是学术洁癖,是省下你 debug "训练不收敛但不知道哪错了"的几天时间——梯度推错是最隐蔽的 bug,因为它不报错,只是悄悄朝错方向优化。

> 失败模式提醒:反向传播本身不会"坏",但它放大上游的数值问题。如果某层激活饱和(比如 tanh 输入太大,导数 `1−a1²` 趋近 0),梯度乘过去就接近 0——这就是梯度消失(vanishing gradient)。反过来,层数深、权重大,梯度连乘会爆炸。这两个是深网训练的经典病,layer norm、残差连接、合适的初始化都是为了治它。这章的两层玩具网络太浅看不到,但机制就藏在那个连乘里。

## 四、凸性与拉格朗日乘子:什么时候能信梯度、约束怎么办

### 1. 凸性:为什么线性回归好训、神经网络难训

stage ② 的拟合为什么稳稳收敛、③ 的"合适 lr"为什么一调就对?因为**均方误差关于参数是凸函数(convex,碗形,只有一个最低点)**。凸函数的关键性质:任何局部最低点就是全局最低点,梯度为零的地方就是答案,GD 只要步长合规就一定收敛到它。

这正是线性/逻辑回归这类模型好训的根本原因——loss 是凸的,你几乎不可能训坏。而**神经网络的 loss 高度非凸**:有无数局部极小、鞍点(saddle point,某些方向是谷某些方向是峰)、平原。GD 在非凸曲面上没有"一定到全局最优"的保证,你能拿到的只是"一个还不错的局部解"。

这件事的实践含义很大:**别指望神经网络训练有线性回归那样的确定性。** 同样的数据、同样的代码,换个随机种子(不同初始化)可能收敛到不同的解、不同的 loss。这不是 bug,是非凸的本性。"为什么深度网络的 SGD 经常能找到泛化很好的解,尽管理论上非凸无保证"——这本身就是个**没有完全解答的开放问题**(loss landscape 几何、过参数化为何有帮助,都是活跃研究方向)。你做工程时记住一条:凸=可信可复现,非凸=经验+多跑几次。

### 2. 拉格朗日乘子:带约束的最优化

到目前为止都是无约束优化——参数随便取。但很多真实问题有约束:"在预算 ≤ B 的前提下最大化收益","在权重范数 = 1 的约束下找最优方向"。这时候 GD 不能直接用,因为它会走出约束之外。

**拉格朗日乘子(Lagrange multiplier)**是处理等式约束的经典工具。核心直觉一句话:**在约束曲面上的最优点,目标函数的梯度必须和约束的梯度平行**——否则你还能沿着约束面挪一点让目标更优。平行就意味着 `∇f = λ∇g`,那个比例系数 λ 就是拉格朗日乘子。

stage ⑤ 解了个最小例子:`min x²+y² s.t. x+y=1`(求原点到直线 x+y=1 最近的点)。看 `src/stage04-optim.ts`:

```typescript
function solveLagrange(): { x: number; y: number; lambda: number } {
  // ∇f = (2x, 2y),∇g = (1, 1)  →  2x=λ, 2y=λ  →  x=y;代入 x+y=1  →  x=y=1/2, λ=1
  const x = 0.5;
  const y = 0.5;
  const lambda = 2 * x;  // 由 2x=λ
  return { x, y, lambda };
}
```

真实输出,而且数值验证了 KKT 条件(Karush-Kuhn-Tucker,带约束最优的判据):

```
⑤ 拉格朗日乘子：min x²+y²  s.t. x+y=1
   解 (x, y) = (0.5, 0.5)   λ = 1   f* = 0.5000
   ∇f - λ∇g 残差 = 0.00e+0   约束 g 残差 = 0.00e+0  → KKT 条件成立
   几何校验：原点到直线距离² = 0.5000 = f*  ✓
```

两个残差都是 `0.00e+0`(精确零,因为这是闭式解,不是迭代逼近),再用"原点到直线 x+y=1 的距离 = 1/√2,平方 = 0.5"几何对拍——三种独立路径(代数解、KKT 残差、几何)互相印证,这才叫"验证过",不是"我相信对"。

> 为什么这里用闭式解而不是又来一个 GD 循环?因为拉格朗日乘子的全部价值就在于:约束最优满足一个**精确的代数条件**(梯度在约束面上平行)。展示这个条件然后数值验证它,比把它埋进另一个迭代循环里诚实得多。你改改约束 `g`、重推一遍,就能体会这个方法的通用性。

拉格朗日乘子在 ML 里不是摆设:SVM 的对偶形式、带 KL 散度约束的策略优化(PPO 那一类,见第 11 章及强化学习相关章节)、正则化的约束视角,底层都是它。λ 还有个漂亮的解释——它是**约束的"影子价格"**:约束放松一个单位,最优值变化多少。这在资源分配类问题里直接就是经济含义。

## 五、把这章钉进直觉

回到开头那个三步循环,现在你应该能把每一步背后的数学都说清楚:

1. **算 loss**——一个标量,衡量错得多离谱。
2. **求梯度(反向传播)**——链式法则逐层相乘 + 复用前向中间结果,代价 O(一次前向)。上线前用有限差分对拍,误差到 10⁻¹¹ 才信。
3. **走一步**——`参数 ← 参数 − lr × 梯度`。lr 有个 `2/L` 量级的发散阈值,超界 20% 就从收敛翻转成发散,且真实网络上没有通用公式给你这个值。

四条带走的硬判断:

- **梯度检查不是可选项。** 任何手推梯度/自定义层,跑一次有限差分对拍,看误差到不到 10⁻¹¹ 量级。这一步省下的是你 debug "不收敛但不知哪错"的好几天。
- **loss 变 NaN,八成是学习率或数值问题。** stage 里 5× 阈值的 lr 第 162 步爆 Infinity 就是范本——几何级数膨胀撞穿 float 上限。对策:降 lr、梯度裁剪、Adam。
- **凸=可信可复现,非凸=经验+多跑几次。** 线性回归你训不坏;神经网络换个种子就换个解,这是本性不是 bug。
- **反向传播不是魔法。** 你已经亲手推过一遍了——它就是链式法则加上"中间结果别重算"。

这章是优化这条路的"沿梯度下坡"主干。还有一条完全不同的路——**不求导数,靠随机采样去逼近**(MCMC、模拟退火那一类)。当 loss 不可导、或维度高到梯度也救不了时,那条路才上场。第 11 章统计力学会接着这里讲采样优化,你可以把两条路对照着看:一条要梯度、走得快但要曲面光滑;一条不要梯度、走得慢但啃得动硬骨头。
