先把这本书要打的靶子摆出来:你已经会用 PyTorch / numpy / 一堆向量数据库,代码跑得通、库调得顺,但凡模型一旦不工作——loss 不降、检索召回烂、A/B 实验看不出结论、量化后精度崩——你只能换个 learning rate、换个 embedding 模型、换个随机种子,瞎试。这不是数学难,是你从没把数学接到你真在做的东西上。**会调库不懂原理,是当代开发者最贵的错觉**:happy path 上它零成本,一进 failure mode 就让你按小时烧钱瞎调参。这本书每一块知识都绑一个能跑的 demo,治的就是这个病。

这一章不讲任何具体技术,只画地图:哪几块数学回报最高(先学)、计算机权重下该砍掉什么(别浪费时间),以及后面 12 个 stage 各在地图哪个坐标。读完你应该能决定:**跳着读哪几章、按什么顺序、每章读完拿什么 demo 当自己学懂了的判据。**

## 反面教材:AI 批量生成的 survey 稿长什么样

这本书的写法有个明确的反面参照——你随手让大模型生成一篇《程序员需要的数学》,它会给你:一段"数学是计算机科学的基石"开场,然后线代、概率、微积分、离散数学各来一节,每节列定义、列公式、列"应用场景:机器学习 / 密码学 / 图形学",结尾"持续学习很重要"。

这种稿子的问题不是错,是**没有一句话是可证伪的,也没有一行代码能跑**。它告诉你"PCA 用于降维",但不告诉你 PCA 在什么数据上会给你垃圾结果、为什么、代价是什么。它把"梯度下降会收敛"当事实陈述,却从不让你亲眼看一次它不收敛。survey 稿的信息密度趋近于零,因为它绕开了所有"具体到会疼"的地方。

本书的每一章反着来:**一个具体论点 + 一段真能跑的代码 + 至少一个故意触发的失败模式 + 这机制的代价**。配套代码仓在 `examples/math-for-builders-from-scratch/`,纯 TypeScript、零运行时依赖、离线 CPU 可跑、固定随机种子——同一台机器每次跑出逐字节相同的数。本章后面引用的所有量化,都来自这个仓真实运行的输出,不是我编的、也不是从教科书抄的。

## 实用回报最高的四块:为什么是这四块

数学很大,你这辈子用不完。但对一个在写应用 / 做 ML / 搞后端的开发者,ROI(投入产出比)排序其实相当清楚。我把它们按"两边通吃程度"排:**一块知识同时喂养你最多的日常任务,它的 ROI 就最高。**

### 第一梯队:概率统计(对应第 01–02 章)

为什么排第一:你每天产出的"X% 置信""这个实验显著""模型 91% 准确",**全是概率陈述,而且你大概率在误读它们**。最贵的单点 bug 是忽略基率(base rate,事件本身有多常见)。看一个真数字,来自第 01 章配套代码 `src/stage01-bayes.ts` 跑出的疾病检测算例:

```
先验 (基率) P(病)      = 0.001
灵敏度 P(阳|病)         = 0.99
特异度 P(阴|健康)       = 0.99

解析 PPV  P(病|阳)      = 9.02%
蒙特卡洛 PPV (2e6 抽样) = 9.17%
直觉答案约 99%,真实 PPV 约 9% —— 这 90 个百分点的差就是基率陷阱。
```

一个 99% 准确的检测,在万分之十患病率(prevalence,人群中真有病的比例)下,阳性结果里真有病的只有 **9%**。你的直觉说 99%,真相是 9%,差 90 个百分点。这不是脑筋急转弯——它是你每次看到"反垃圾模型 99% 准确"却被误报淹没时的同一件事。第 01 章把这个用解析式和 200 万次蒙特卡洛(Monte-Carlo,靠大量随机抽样估计)交叉验证,两者差 0.154 个百分点,证明那 9% 不是公式推导失误。

统计这块(第 02 章 `src/stage02-stats.ts`)的核心论点更扎心:**最危险的不是算错 p 值,是算对了 p 值然后误读成"我们发现了什么"**。那一章用从零实现的双样本 t 检验,把"零假设为真时 p 值在 [0,1] 上近似均匀分布""95% 置信区间约 95% 的时候盖住真值"这两条**定义**,用一万次重跑画出来给你看——你看完就再也不会把 p<0.05 当成"有 95% 把握"了。

### 第二梯队:线性代数(对应第 03、10 章)

为什么排第二:你碰的每个 ML pipeline,拆到底就是三个线代操作——PCA(找数据变化最大的方向)、SVD(把矩阵压到本质秩)、余弦检索(按夹角给向量排序)。第 03 章 `src/stage03-linalg.ts` 的明确目标是:这三个**全部从 `core/linalg.ts` 的 dot / matmul / normalize 拼出来,不调任何 BLAS**(线代加速库),让你看清"没有魔法,只有协方差矩阵的特征向量加几次矩阵乘"。

线代的回报还有个隐藏的高 ROI 出口在第 10 章傅里叶:`src/stage10-fourier.ts` 的第一条论点就是"**DFT 本质是矩阵乘法**"——它显式构造 N×N 的 DFT 矩阵,用 `matVec` 跑一遍,再写个 FFT 跟矩阵版对拍到浮点精度,并 wall-clock 测出 O(n²) 和 O(n log n) 的真实差距。当你意识到傅里叶变换就是换一组基底,线代的复利就显出来了。

### 第三梯队:离散数学与逻辑(对应第 05、09 章)

为什么它在第三梯队而不是更低:离散数学是**每个算法的骨架**——图、组合计数、归纳、复杂度。但它对日常应用开发的"每天都用"程度略低于概率和线代,所以排第三。第 05 章 `src/stage05-discrete.ts` 的做法很对路:不光印公式,而是**暴力枚举 vs 闭式解对拍**——∑1..n 的闭式 n(n+1)/2 对每个 n 都跟循环求和比一遍,把"归纳法"变成可证伪的代码;再 wall-clock 跑出 O(n) / O(n log n) / O(n²) / O(2ⁿ) 的真实时间表。

它的高价值兄弟是第 09 章可计算性 `src/stage09-computability.ts`:DFA / 图灵机 / 停机问题的对角线论证 / SAT→子集和的 NP 归约,**全做成能跑的代码**。停机问题不再是"听过的哲学",而是一段具体的代码矛盾;NP 难不再是术语,而是 n=20 能跑、n=60 没救的那张实测表。一个对这三条边界(能判定 / 不可判定 / 可判定但难解)有体感的开发者,会停止问那些注定无解的问题。

### 第四梯队:微积分与最优化(对应第 04 章)

为什么微积分排第四而不是更前——对纯应用开发它不像概率那样天天用,但对碰 ML 训练的人它是发动机:**任何基于梯度的模型,从线性拟合到 transformer,都是同一个循环**——量 loss、问微积分哪边是下坡(梯度)、走一步。第 04 章 `src/stage04-optim.ts` 把这个循环手写出来,而且做了每个框架内部都在做的那个检查:**数值梯度(有限差分)≈ 解析梯度**。当你亲眼看到这俩对得上,你就懂了"梯度"不是玄学,只是"每个权重变一单位,loss 变多少"。

这四块的 ROI 排序就是本书前半的推荐阅读顺序。它们"两边通吃":概率喂养你的实验 / 评估 / 风控直觉,线代喂养你所有 ML 数据操作,离散喂养你的算法与系统设计,微积分喂养你的训练。**先学这四块,剩下的按需点。**

## 计算机权重下,数学和物理各砍掉哪一刀

这是本书第二个核心判断:作为开发者重学数学物理,**你不能按数学系 / 物理系的方式学,得砍**。砍的原则是统一的——**砍掉纯严格性,留下"用得上 + 会疼"的部分**。

### 数学砍掉的那一刀:ε-δ 严格性

数学系花在 ε-δ 证明、测度论基础、定理的完备性证明上的时间,对你 ROI 极低。你不需要会证明实数完备性,你需要知道**浮点数根本不是实数,这个 gap 会咬你**。这正是第 06 章数值计算 `src/stage06-numerics.ts` 的整章主题:IEEE-754 双精度是一个**有限**的二进位有理数集合,不是 ℝ。它会让一个"理应为真"的 == 比较返回 false,让一个教科书上正确的公式丢光有效数字,让一百万个小数的求和漂移。那一章把每个 gap 对着高精度参照**量成一个具体的数字**,而不是留在民间传说里。

换句话说:数学系教你"为什么实数是连续的",这本书教你"为什么你的代码里它不连续、什么时候会爆、爆了赔多少"。前者是严格性,后者是工程。**你要后者。**

### 物理砍掉的那一刀:解析求解与第一性推导

物理系大量时间花在解析求解微分方程、从第一性原理推导守恒律。对开发者,物理的高 ROI 出口不在"解方程",而在**两类可迁移的计算机制**:

一是**蒙特卡洛与统计力学采样**(第 11 章 `src/stage11-statmech.ts`)。它的核心论点漂亮:每个现代生成式采样器——扩散模型、能量模型、MCMC 推断、Gibbs 采样——都是**同一个机制**:提议一个随机移动,用玻尔兹曼因子 exp(-ΔE/T) 决定接受概率,于是你不用算归一化常数就能收敛到目标分布。那一章把这套东西用 Metropolis MCMC、模拟退火、Ising 相变跑三遍,让你看清"这是一个机制,不是三个魔术"。物理的"配分函数"被翻译成了你训练扩散模型时绕不开的那件事。

二是**信息论与热力学的物理下界**(第 07 章信息论 `src/stage07-infotheory.ts`、第 12 章硬件 `src/stage12-hardware.ts`)。第 07 章告诉你"这个 log / JSON / 模型权重最小能压到多大"就是它的熵,句号;交叉熵 loss 为什么是对的目标,因为最小化它等于最小化你模型相对真实分布浪费的比特(=KL 散度)。第 12 章则从一个 NAND 门搭出整个逻辑栈,最后印出 Landauer 极限——擦除一个比特的物理能量代价 kT·ln2。这是物理给计算划的硬底线,不是哲学。

砍完之后剩下的物理,对开发者就两句话:**会采样(因为生成模型都是采样),知道物理下界(因为它决定你能优化到哪)。** 其余的解析功夫,留给物理系。

### 一个被两边共享的出口:密码学数论(第 08 章)

第 08 章 `src/stage08-crypto.ts` 是离散数学 / 数论的应用出口,但值得单独点名,因为它把"难"这个概念做成了可跑的:每次 TLS 握手、SSH 登录、签名的 JWT,都压在**一个不对称性**上——两个素数相乘很便宜,把乘积分解回去(被认为)很难。那一章用扩展欧几里得、快速模幂、Miller-Rabin 素性测试拼出 toy RSA,然后**真的把它分解破解掉**,证明安全性纯粹是素数大小的函数。所有数字用 BigInt 精确算,不是估计。

### 诚实标注:本书数字的三种来源

这件事必须开篇说清,因为它是本书区别于 survey 稿的命根:本书引用的每个数字属于以下三类之一,我会随处标注,绝不混淆——

- **代码真算出**:如本章的 PPV 9.02%、两法之差 0.154 个百分点,来自 `stage01-bayes.ts` 当场运行。
- **wall-clock 真测出**:如各 stage 的 O(n²) vs O(n log n) 耗时对比,是机器实测的墙钟时间。
- **toy / 合成数据,标注 (est.)**:很多 demo 用的是手写小语料 / 合成点云。比如第 01 章那个 100% 准确率的垃圾邮件分类器(6/6 全对),**它的绝对准确率毫无意义**——语料是我手编的 10 封训练 / 6 封测试。能迁移的从来不是绝对值,而是机制(先验×似然、log 域求和、Laplace 平滑)和**失败模式**:关掉平滑后,一个训练里没见过的词("lottery")就把 spam 和 ham 两边的 log 概率都打成 `-Infinity`,argmax 直接退化失判。这种"会疼的具体"才是 survey 稿永远给不了你的。

⚡ **开放问题:什么样的数学才算"这个开发者真懂了"——目前无通用解。** 本书用的判据是"能从零写出可跑 demo + 能制造并解释它的失败模式"。这是个**可操作但不完备**的代理指标:它显然强过"会背公式 / 会调库",因为它逼你触碰失败模式;但它也漏掉了真正的专家能力——在一个**没有现成 demo、问题定义本身都模糊**的新场景里,判断该用哪块数学、要不要用、用了的代价值不值。把数学能力机械量化(写得出 demo = 懂)和把它当玄学(说不清但我感觉懂了)都是错的,中间那条线学界和工程界都还没画清楚,目前没有公认的、可迁移的"数学成熟度"评测。这本书能做的,是把判据从"背得出"硬推到"做得出 + 坏得明白",剩下那一截留给你在真实问题里自己长。

下一章我们就从 ROI 第一梯队的第一块开始:`stage01-bayes.ts`,把"X% 置信"这个你每天都在说、却大概率在误读的数字,拆到代码里。
