先把一个会让大多数工程师不舒服的数字摆上桌：一个灵敏度 99%、特异度 99% 的检测，对一个万分之十（0.1%）患病率的病，**当它报"阳性"时，真有病的概率只有 9%**。不是 99%，是 9%。这个数字是配套仓 `examples/math-for-builders-from-scratch/src/stage01-bayes.ts` 跑出来的解析值 9.02%，并被一次 200 万人的蒙特卡洛模拟交叉验证到 9.17%——两者差 0.154 个百分点，落在采样噪声里。直觉给你 99，贝叶斯给你 9，**这 90 个百分点的鸿沟就是本章要讲透的东西**。它不是脑筋急转弯，它是安检、医疗筛查、风控反欺诈、垃圾邮件过滤每天都在踩的同一个坑。

本章不堆公式。我们只要一行——`后验 ∝ 似然 × 先验`——然后用三段能 `npm run stage01` 直接跑的代码把它锤进去:疾病检测（基率陷阱）、从零的朴素贝叶斯垃圾邮件分类器、蒙特卡洛估 π。每一段都印它真算出来的数字，每一段都附一个**故意写坏的失败模式**，告诉你这机制在什么条件下崩、以及它当初为什么要那么设计。术语首现给一句大白话，跑题的不讲。

## 一行公式:后验 ∝ 似然 × 先验

贝叶斯定理（Bayes' theorem，用新证据更新信念的公式）的完整形态是:

```
P(H|E) = P(E|H) · P(H) / P(E)
```

`H` 是假设（hypothesis，比如"这封邮件是垃圾"），`E` 是证据（evidence，比如"邮件里有 free 这个词"）。但工程上你几乎从不需要分母 `P(E)`——它对所有竞争假设都一样，只是个归一化常数。砍掉它，剩下的就是那句能装进脑子的话:

```
后验 ∝ 似然 × 先验
P(H|E) ∝ P(E|H) · P(H)
```

- **先验**（prior，看到证据前的信念）`P(H)`:这病多罕见？这词在垃圾邮件里多常见？
- **似然**（likelihood,假设成立时证据出现的概率）`P(E|H)`:如果真是垃圾邮件，出现这个词的概率多大？
- **后验**（posterior,看到证据后更新的信念）`P(H|E)`:看到这个证据后，假设成立的概率。

这一行同时是三个系统的引擎:垃圾邮件分类器对每封邮件算 `P(垃圾|词们) ∝ P(词们|垃圾)·P(垃圾)`，取较大的那个类；医疗诊断算 `P(病|阳性)`；A/B 实验的贝叶斯版算 `P(B 比 A 好|观测数据)`。同一根杠杆，撬三块石头。先看它在医疗诊断上怎么把直觉撬翻车。

## Part 1:基率陷阱——为什么 99% 的检测大概率在骗你

核心机制是把贝叶斯定理的分母用全概率公式（law of total probability，把一个事件按所有互斥情况拆开求和）展开。阳性的来源有两种:真病人测出阳性（真阳），健康人误报阳性（假阳）。stage01 里 `posteriorPositive` 就是这一行:

```typescript
// examples/math-for-builders-from-scratch/src/stage01-bayes.ts
function posteriorPositive(prior: number, sens: number, spec: number): number {
  const fpr = 1 - spec; // false-positive rate P(positive | healthy)
  const pPos = sens * prior + fpr * (1 - prior); // law of total probability
  if (pPos === 0) throw new Error("test never returns positive; posterior undefined");
  return (sens * prior) / pPos;
}
```

`pPos` 是分母——**所有阳性**，包括真阳 `sens·prior` 和假阳 `fpr·(1-prior)`。返回值是真阳占全部阳性的比例，也就是 PPV（positive predictive value，阳性预测值——"报阳里真有病的比例"）。代入 stage01 的真实参数 `prior=0.001, sens=0.99, spec=0.99`:

```
先验 (基率) P(病)      = 0.001
灵敏度 P(阳|病)         = 0.99
特异度 P(阴|健康)       = 0.99
解析 PPV  P(病|阳)      = 9.02%
蒙特卡洛 PPV (2e6 抽样) = 9.17%
两者之差               = 0.154 个百分点
```

为什么 99% 的检测只给出 9% 的 PPV？算一下绝对人数就懂了。10 万人里:有病的 100 人（0.1%），检测抓住其中 99 人（真阳）;健康的 99900 人，1% 误报，产生约 999 个假阳。报阳的总共 99+999≈1098 人，里头真有病的只占 99/1098≈9%。**假阳的绝对数量碾压真阳**，因为健康人的基数太大了。这就是基率谬误（base rate fallacy）的全部:直觉只盯着"检测多准"（似然），忘了"这事本来多罕见"（先验/基率）。

`后验 ∝ 似然 × 先验` 在这里现身——后验被那个 0.001 的先验死死压住。stage01 把这件事可视化得很直白:

```
直觉(误) │████████████████████████████████████████████████│ 0.9900
真实PPV │████············································│ 0.09016
  灵敏度 │████████████████████████████████████████████████│ 0.9900
```

灵敏度和直觉一样高，真实 PPV 趴在地板上。更关键的是,**PPV 主要由基率决定，不是由检测精度决定**。stage01 固定灵敏度/特异度都在 99%，扫患病率，得到这条曲线的锚点:

```
  患病率  0.1% → PPV 9.0%
  患病率  1.0% → PPV 50.0%
  患病率  5.0% → PPV 83.9%
  患病率 50.0% → PPV 99.0%
```

同一个 99% 的检测，病从万分之十变成一半，PPV 从 9% 飙到 99%。这就是为什么医生不会拿三甲检测去给无症状人群做普筛——基率太低时阳性几乎全是假阳；也是为什么安检、风控的"命中"必须人工复核——罕见事件的报警，先验上就该不信。**工程落地的判断**:任何"分类器/检测器报了正例"的场景，先问一句"正例本来占多少？"。在 0.1% 正例率上跑一个 99% 精度的模型，你的告警队列会被假阳淹没——这不是模型烂，是基率在说话。

### 失败模式 A:小样本下蒙特卡洛会骗你

上面解析值 9.02% 和 200 万抽样的 9.17% 对得很齐，容易让人以为"模拟随便跑跑就行"。stage01 故意把样本砍到 500 打脸:

```
--- 失败模式 A：样本太小时蒙特卡洛不可信 ---
n=500 时的 PPV 估计     = 14.29%
```

为什么坏:患病率 0.1% 意味着 500 人里期望只有 0.5 个病人。一次模拟里要么一个阳性都没采到（PPV 直接是 NaN，代码里 `simulatePosterior` 对此返回 `NaN`），要么采到一两个，估计值剧烈抖动——这次 14.29%，换个种子可能是 0% 或 33%。**罕见事件用小样本估计，结果不是噪声大，是根本不可信。** 设计上的应对:罕见事件优先用解析式（闭式解没有采样方差），逼不得已模拟就把 n 拉到能采到几百个目标事件的量级。这也是 stage01 为什么默认用 200 万抽样——它要的是 0.154 个百分点的吻合度来证明闭式解没写错，500 给不了这个。

## Part 2:从零朴素贝叶斯——把那行公式写成分类器

医疗诊断只有一个证据（阳性）。垃圾邮件有几十个证据（每个词）。朴素贝叶斯（naive Bayes，假设各证据条件独立的贝叶斯分类器）的"朴素"就是那个独立假设:假装词与词之间互不相关，于是似然能拆成连乘:

```
P(垃圾|词们) ∝ P(垃圾) · ∏ P(词ᵢ|垃圾)
```

这个独立假设对真实语言**是错的**（"free"和"prize"明显相关）。但分类器照样能用，因为我们只要 argmax 落在正确的类上，不需要校准的概率值。stage01 的 `classify` 把那行公式写成了对数域求和:

```typescript
// examples/math-for-builders-from-scratch/src/stage01-bayes.ts
function classify(model: NaiveBayesModel, text: string): { label: "spam" | "ham"; scores: Record<"spam" | "ham", number> } {
  const scores: Record<"spam" | "ham", number> = { spam: model.logPrior.spam, ham: model.logPrior.ham };
  for (const w of tokenize(text)) {
    for (const cls of ["spam", "ham"] as const) {
      scores[cls] += model.logLikelihood[cls].get(w) ?? model.logUnseen[cls];
    }
  }
  return { label: scores.spam > scores.ham ? "spam" : "ham", scores };
}
```

两个工程决策埋在这几行里,都不是随意的:

**为什么对数域**——一封邮件几十个词，每个 `P(词|类)` 都是零点几，几十个零点几连乘会下溢（underflow，浮点数小到被舍成 0）到 float64 的 0，于是所有类都得 0，argmax 失效。取对数把连乘变连加,`log(a·b)=log a+log b`，加法不下溢。所以 `scores` 从 `logPrior` 起步，一路 `+=` 各词的 `logLikelihood`,全程不做指数还原——只比大小，不需要还原成真概率。

**`?? model.logUnseen[cls]`**——训练时没见过的词，用一个兜底的对数概率，而不是直接当 0。为什么必须兜底,看失败模式 B。

跑 stage01 的 toy 语料（训练 10 封/测试 6 封）:

```
预测明细：
  [✓] 真实=spam 预测=spam logΔ(spam-ham)=   5.39  "claim your free prize now"
  [✓] 真实=spam 预测=spam logΔ(spam-ham)=   2.51  "win cash click the offer"
  [✓] 真实=ham  预测=ham  logΔ(spam-ham)=  -2.48  "are we still meeting tomorrow"
  [✓] 真实=ham  预测=ham  logΔ(spam-ham)=  -3.22  "please review the report"
  [✓] 真实=spam 预测=spam logΔ(spam-ham)=   3.19  "free shipping on your order"
  [✓] 真实=ham  预测=ham  logΔ(spam-ham)=  -3.40  "lunch at noon friday"
训练集 10 封 / 测试集 6 封，分类准确率 = 100.0% (6/6)
```

那个 `logΔ(spam-ham)` 是两个类对数分的差——正数偏垃圾、负数偏正常，绝对值是"判得多有底气"。**诚实标注**:这是手写的 toy 语料，6/6 的 100% 准确率没有任何 benchmark 意义,绝对值是凑出来的。可迁移的不是这个数,是这套机制——`先验×似然`、对数域求和、下面要讲的拉普拉斯平滑。换成真实邮件、上万词表，机制一字不改，数字才有意义。

### 失败模式 B:不平滑,一个生词就把整个判断打成 0

朴素贝叶斯最经典的坑:训练集没出现过的词,它的 `P(词|类)` 是 `0/总数 = 0`，连乘里一个 0 湮灭整个乘积。stage01 关掉拉普拉斯平滑（Laplace smoothing,给每个计数加一个小常数避免零概率）来演示:

```
--- 失败模式 B：关掉 Laplace 平滑，未见词把概率打成 0 ---
  文本 "lottery jackpot tomorrow"
  log P(spam) = -Infinity   log P(ham) = -Infinity
  两个类都得到 -Infinity（log 0）：未见词 P(word|class)=0 湮灭了整个乘积 → argmax 退化、无法判别。
```

"lottery"和"jackpot"训练集里没有,无平滑时 `P(lottery|spam)=0`，取对数是 `-Infinity`，两个类都被打成 `-Infinity`，argmax 在两个负无穷之间退化,无法判别。修复就是训练时的那个 `+smoothing`:

```typescript
// examples/math-for-builders-from-scratch/src/stage01-bayes.ts — trainNaiveBayes
const denom = totalWords[cls] + smoothing * (v + 1);
for (const w of vocab) {
  const c = counts[cls].get(w) ?? 0;
  logLikelihood[cls].set(w, Math.log((c + smoothing) / denom));
}
logUnseen[cls] = Math.log(smoothing / denom);
```

分子 `+smoothing`(取 1 就是 add-one),分母补 `smoothing·(v+1)` 保持它仍是合法概率分布（所有词概率加生词兜底和为 1）。代价上很直白——平滑等价于给每个词、包括没见过的词,塞了一点点先验概率质量，相当于一个"假装见过一次"的极弱先验。这又是 `后验 ∝ 似然 × 先验` 的影子:你不能让似然里出现严格的 0，否则它会把先验和其他所有证据一起归零。**工程默认**:任何基于频次估概率的地方（n-gram 语言模型、类目计数、CTR 估计），都得平滑,否则迟早被一个零计数干翻。

## Part 3:蒙特卡洛估 π——期望、方差与 1/√n 的代价

前两部分用贝叶斯算后验。但很多后验/期望根本没有闭式解，这时只能采样——蒙特卡洛（Monte Carlo,用随机采样估计数值）。它的数学根基是期望和方差:把"圆内点的比例"看成一个随机变量,它的**期望**等于 π/4，样本均值是它的无偏估计;它的**方差**决定估计抖多厉害,而样本均值的标准差按 `1/√n` 收缩。stage01 往单位正方形里扔飞镖,数落在四分之一圆内的比例:

```typescript
// examples/math-for-builders-from-scratch/src/stage01-bayes.ts — estimatePi
for (let i = 1; i <= maxN; i++) {
  const x = rng();
  const y = rng();
  if (x * x + y * y <= 1) inside++;
  // ... 在 checkpoint 处记录 est = 4 * inside / i ...
}
```

`P(落在圆内) = 圆面积/方形面积 = (π/4)/1`,所以 `π ≈ 4·比例`。seeded（用固定种子,可复现）跑出来:

```
样本数 n        π 估计        绝对误差      理论误差≈1/√n
         100   3.480000    0.338407    0.100000
        1000   3.096000    0.045593    0.031623
       10000   3.132800    0.008793    0.010000
      100000   3.141480    0.000113    0.003162
     1000000   3.142492    0.000899    0.001000
    10000000   3.141812    0.000220    0.000316
真值 π = 3.141593
```

看绝对误差那列怎么缩:100 个样本误差 0.34,到 1000 万降到 0.0002 量级。但**收敛慢得令人沮丧**——误差按 `1/√n` 走,意味着**每多要一位准确数字,要约 100 倍样本**（误差降 10 倍需要 √ 关系下的 100 倍 n）。这不是实现没优化，是蒙特卡洛的根本代价,写在方差里。注意 n=100 万那行误差(0.0009)反而比 10 万那行(0.0001)大——`1/√n` 是**期望趋势**不是单调保证，单条采样轨迹会在趋势线附近上下浮动，这恰恰是方差的体现，不是 bug。这也是为什么严肃的蒙特卡洛要配方差缩减技术（重要性采样、对偶变量、分层采样）——直接加样本性价比太低。

### 失败模式 C:破坏独立性,估计器稳稳收敛到错的数

蒙特卡洛假设各维度的随机数相互独立。stage01 故意把 y 设成 x 的拷贝:

```
--- 失败模式 C：x 与 y 用同一个随机数（坐标相关）→ 收敛到错误常数 ---
  错误估计 (x=y, n=1000000) = 2.827912  (应为 3.141593)
```

为什么坏、为什么危险:`y=x` 时所有飞镖落在对角线上,"在圆内"退化成 `2x²≤1` 即 `x≤1/√2≈0.707`,于是估计值收敛到 `4·0.707≈2.83`，不是 π。**致命的是它不报错**——估计器不崩、不抛异常，只是稳稳地、自信地收敛到一个错的常数。这比崩溃可怕得多:崩溃你立刻知道，悄悄给错数你可能一直信着它。这也呼应 `core/rng.ts` 里 PRNG 的设计警告——一个 Rng 实例如果被两条逻辑独立的流共享,会耦合彼此,重排调用顺序就悄悄改变"无关"结果。**工程教训**:蒙特卡洛里维度间独立性是隐式契约,违反了没有编译器帮你,只能靠对结果的 sanity check（这数收敛到的值合理吗？）和已知 case 的回归测试兜底。

## ⚡ 开放问题:高维空间里采样,目前没有通用解

本章三段都在低维玩——疾病检测是 1 维证据,π 是 2 维。一旦维度上去,贝叶斯推断和蒙特卡洛都撞上**维度灾难**(curse of dimensionality,高维空间里数据指数级稀疏)这堵墙,而**它至今没有通用解**。

具体地:贝叶斯后验 `P(参数|数据)` 在高维参数空间里那个归一化分母(全空间积分)几乎永远算不出闭式解,只能采样近似。但简单的"扔飞镖"式采样在高维彻底失效——你撒的点几乎全落在后验概率近乎为 0 的地方,有效样本趋近于零。现代做法是 MCMC（马尔可夫链蒙特卡洛,用相关样本链探索分布,如 Metropolis-Hastings / HMC / NUTS）和变分推断（variational inference,用简单分布近似复杂后验),但它们各有死穴:MCMC 在多峰（multimodal,有多个概率高峰）后验上会卡在一个峰里出不来,而且**何时算"收敛"了至今没有可靠的通用判据**——`R̂` 之类的诊断只能证伪不能证真,你永远不能完全确定链已经充分混合;变分推断快但系统性低估方差,给你过度自信的后验。

这不是工程没跟上,是数学上的开放难题。贝叶斯深度学习里对神经网络权重做完整后验推断,2024–2025 仍是活跃研究——没有哪个方法能在高维、多峰、大数据上同时做到准确、快、有收敛保证。下一章[统计推断]接着这块讲:当后验算不动时,频率派(frequentist)的置信区间和假设检验是另一条路,它绕开了"先验从哪来"和"后验怎么积分"这两个贝叶斯的硬骨头——代价是它回答的是另一个问题。两条路各有取舍,下一章掰开讲。

> **术语首现备查**:贝叶斯定理=用新证据更新信念;先验=看证据前的信念;似然=假设成立时证据出现的概率;后验=看证据后更新的信念;PPV=报阳里真有病的比例;基率谬误=只看准确率忘了事件本身多罕见;朴素贝叶斯=假设证据相互独立的贝叶斯分类器;拉普拉斯平滑=给计数加常数避免零概率;蒙特卡洛=用随机采样估数值;维度灾难=高维空间数据指数级稀疏。
