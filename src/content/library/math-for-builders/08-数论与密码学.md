每次你按下浏览器地址栏的小锁、SSH 登一台服务器、验一个签名 JWT,你都在赌同一个东西:把两个素数乘起来很便宜,把它们的乘积拆回去(believed to be)很贵。整套公钥密码——TLS 握手、签名、密钥协商——不依赖任何"加密算法很巧妙",它依赖这一条**不对称**(asymmetry,正向反向代价差很多)在数学上站得住。这一章不讲库怎么调,而是用三个原语从零把 RSA 和 Diffie–Hellman 拼出来,跑通,然后**亲手把 RSA 掰断**——让你看清它的安全性其实是素数大小的一个函数,如此而已。

代码全在 `examples/math-for-builders-from-scratch/src/stage08-crypto.ts`,用 BigInt(任意精度整数,没有浮点舍入)逐行算,下面引用的每个数字都是它跑出来的真值。

## 单向函数:这章的整个赌注

✦ **公钥密码建立在"正向易、反向难"的单向函数(one-way function,正着算快、反着算到目前为止没人能算快)上。** 单向函数不是一个具体函数,是一类押注:存在某个 `f`,给 `x` 算 `f(x)` 便宜,但给 `f(x)` 反推 `x` 没有已知的便宜办法。RSA 押的是**大数分解**(把 `n` 拆成 `p·q`),Diffie–Hellman 押的是**离散对数**(已知 `g`、`g^a mod p` 求 `a`)。

注意我一直在说"believed to be"、"没有已知办法"。这是诚实的:**没人证明过分解一定很难**。如果明天有人发表多项式时间的整数分解算法,今天全世界的 TLS 一夜之间裸奔。我们押的是"几十年最聪明的人没找到捷径"这个经验事实,不是数学定理。这一点会在第 9 章(可计算性,P vs NP)里展开——分解到底属不属于 NP-intermediate、量子的 Shor 算法为什么能破它,都是那一章的料。

那个具体的"正向"运算,就是下面第二个原语:模幂 `base^exp mod m`。它算起来 O(log exp) 次乘法,便宜得很;但没有私钥想反过来,就得分解或解离散对数。**这个便宜的正向 + 昂贵的反向,就是整章的发动机。** 先把它和它依赖的另外两个原语建起来。

## 原语一:扩展欧几里得,拿到模逆元

RSA 生成私钥时要算 `e^-1 mod φ(n)`——模逆元(modular inverse,模意义下的"倒数"):满足 `e·d ≡ 1 (mod φ)` 的那个 `d`。怎么算?用扩展欧几里得(extended Euclid),它在求 gcd 的同时顺便吐出贝祖系数(Bézout coefficient,满足 `a·x + b·y = gcd` 的那对 `x,y`)。当 `gcd(e, φ) = 1` 时,那个 `x` 就是逆元。

```ts
// src/stage08-crypto.ts
function egcd(a: bigint, b: bigint): [bigint, bigint, bigint] {
  if (b === 0n) return [a, 1n, 0n];
  const [g, x1, y1] = egcd(b, a % b);
  return [g, y1, x1 - (a / b) * y1];
}

function modInverse(a: bigint, m: bigint): bigint {
  const [g, x] = egcd(((a % m) + m) % m, m);
  if (g !== 1n) throw new Error(`no inverse: gcd(${a}, ${m}) = ${g} ≠ 1`);
  return ((x % m) + m) % m; // egcd 的 x 可能为负,归一化进 [0, m)
}
```

跑出来 `17^-1 mod 3120 = 2753`,校验 `17 × 2753 mod 3120 = 1`,对上了。

**失败模式:逆元不存在时千万别返回假值。** 逆元只在 `gcd(a, m) = 1` 时存在。代码这里**主动 throw**(`if (g !== 1n)`),而不是返回个 0 或随便一个数。为什么这么设计:如果在 RSA 密钥生成时悄悄返回一个错的 `d`,你会得到一把"加密能跑、解密出垃圾"的密钥,而且要等到第一条消息解密失败才暴露——那时密钥可能已经发出去了。在密码学代码里,**早崩比晚错好一万倍**,这是贯穿这一章的设计原则。

## 原语二:快速模幂,就是那个单向函数

```ts
// src/stage08-crypto.ts
function modpow(base: bigint, exp: bigint, m: bigint): bigint {
  if (m === 1n) return 0n; // 退化环:模 1 一切都是 0,先挡掉
  let result = 1n;
  base %= m;
  while (exp > 0n) {
    if (exp & 1n) result = (result * base) % m; // 该位为 1 → 把当前平方折进来
    base = (base * base) % m; // 为下一个 bit 平方
    exp >>= 1n;
  }
  return result;
}
```

这就是 square-and-multiply(平方-乘,按指数的二进制位逐位处理)。跑 `7^256 mod 13 = 9`。256 的指数本来要乘 255 次,这里只要 ~8 次(log₂256)。

两个工程细节别忽略:

1. **每步都 `% m`。** 不取模的话 `base ** exp` 会膨胀成天文数字——一个 2048-bit 的 `m^e` 直接算出来要占几个 GB,程序当场爆。取模让中间结果永远 ≤ m²,把 BigInt 钉在可控大小。这不是优化,是能不能跑的问题。
2. **正向 O(log exp),反向无捷径。** 加密 `m^e mod n` 便宜;但拦截者拿到密文 `c`,想反推 `m`(即求 `c` 的 e 次方根 mod n)没有私钥 `d` 就抓瞎。这个不对称就是单向函数的肉身。

## 原语三:Miller–Rabin,怎么在不试除的情况下找素数

RSA 要两个大素数。1024-bit 的素数你不可能"从 2 试除到 √n"——那是 2^512 次除法。实际做法是概率素性测试(probabilistic primality test):随便挑几个"证人"数 `a`,看 `n` 在它们面前露不露馅。一个合数挺过一轮的概率 ≤ 1/4,所以 k 轮后假阳率 ≤ 4^-k。

```ts
// src/stage08-crypto.ts (节选核心循环)
witness: for (let i = 0; i < rounds; i++) {
  const a = 2n + BigInt(Math.floor(rng() * Number(n - 4n)));
  let x = modpow(a, d, n);
  if (x === 1n || x === n - 1n) continue; // a^d 已是 1 的平凡平方根
  for (let j = 0n; j < r - 1n; j++) {
    x = (x * x) % n;
    if (x === n - 1n) continue witness; // 找到非平凡 -1,a 不是证人
  }
  return false; // 平方根链里没有 -1 → a 证明 n 是合数
}
return true; // 挺过所有轮:素数概率 ≥ 1 - 4^-rounds
```

代码里用 20 轮见证找到 `p = 64381, q = 70199`,假阳率 ≤ 4^-20 ≈ 10^-12。

**失败模式:固定/弱的证人集会被 Carmichael 数骗过。** 有一类合数(Carmichael 数)对所有与它互素的底数都"表现得像素数"。如果你偷懒只用固定的几个证人(比如永远用 2、3、5),针对性构造的合数能整个骗过去。**随机抽证人**才是那个 4^-k 上界成立的前提。这里有个教学权衡:代码用 `core/rng`(可复现的伪随机)选证人,是为了让教学输出每次一样;但**真实实现必须从 CSPRNG**(密码学安全随机源)抽证人,理由下面失败模式 1 会血淋淋地讲。

## 把 RSA 拼出来,跑一个 round-trip

三个原语齐了,组装 RSA:

```ts
// src/stage08-crypto.ts
function genRsa(p: bigint, q: bigint): RsaKeypair {
  if (p === q) throw new Error("p and q must differ");
  const n = p * q;
  const phi = (p - 1n) * (q - 1n); // 欧拉函数 φ:小于 n 且与 n 互素的整数个数
  let e = 65537n;
  if (e >= phi || egcd(e, phi)[0] !== 1n) e = 3n; // 玩具模数太小时退回 3
  const d = modInverse(e, phi);
  return { n, e, d, p, q };
}
const rsaEncrypt = (m, n, e) => modpow(m, e, n);
const rsaDecrypt = (c, n, d) => modpow(c, d, n);
```

这里藏着几个**为什么**,不是随便选的:

- **`p ≠ q`**:若相等,`n = p²` 是完全平方数,开个平方根就分解了,等于没加密。代码开头就 throw。
- **`e = 65537`**:它是素数(保证容易和 φ 互素),且二进制 `0x10001` 只有两个 1,所以模幂时乘法次数少、加密快。这是工业界的标准选择,不是巧合。
- **私钥 `d = e^-1 mod φ`**:这就是原语一的用武之地。知道 φ 就能算 d,而算 φ 需要知道 p、q——所以**分解 n 等价于拿到私钥**。

跑一个完整 round-trip(用真实 stage 输出):

```
随机素数 p = 64381, q = 70199  (Miller–Rabin 找到,20 轮见证)
公钥 n = p*q = 4519481819
公钥指数 e = 65537
私钥指数 d = 854053673  (= e^-1 mod φ(n),绝密)
明文 "MATH" 打包为整数 m = 1296127048
密文 c = m^e mod n = 1998076648
解密 c^d mod n = 1296127048 → 还原文本 "MATH"
round-trip 成功 ✓
```

**这是 textbook RSA(教科书 RSA),生产里它本身就是个漏洞。** 它是确定性的:同样明文永远出同样密文,泄漏"两条消息是否相同";而且可塑(malleable),攻击者能在不解密的情况下篡改密文对应的明文。真实 RSA 必须套 OAEP/PSS 填充(给明文加随机性 + 完整性)。这里裸跑只为给你看清数学骨架——这正好引出本章的 ⚡。

## Diffie–Hellman:同一个单向函数,换个协议

RSA 是"拿公钥加密"。Diffie–Hellman 解的是另一个问题:Alice 和 Bob 从没见过面,怎么在一条**所有人都能监听**的线路上协商出一把只有他俩知道的密钥?

用的还是 `modpow`。公开参数 `(p, g)`,各自挑一个私密指数,交换 `g^secret`,然后都算 `g^(a·b)`——而 `a`、`b` 从不上线传输。

```ts
// src/stage08-crypto.ts
function diffieHellman(p, g, secretA, secretB) {
  const publicA = modpow(g, secretA, p); // Alice → Bob
  const publicB = modpow(g, secretB, p); // Bob → Alice
  const sharedFromA = modpow(publicB, secretA, p); // (g^b)^a mod p
  const sharedFromB = modpow(publicA, secretB, p); // (g^a)^b mod p
  return { publicA, publicB, sharedFromA, sharedFromB };
}
```

跑出来:

```
公开参数 p = 924967, g = 5
Alice 私密 a = 82663  → 公开 g^a mod p = 866906
Bob   私密 b = 56081  → 公开 g^b mod p = 754048
Alice 算 (g^b)^a = 912951
Bob   算 (g^a)^b = 912951
双方得到同一密钥 ✓ (a,b 从未上线传输)
```

监听者看得到 `p, g, g^a, g^b`,要还原共享密钥就得从 `g^a` 反推 `a`——这就是**离散对数问题**,RSA 押分解,DH 押它,但都是同一类"反向难"的赌注。

**失败模式:这里的 DH 没做认证,裸跑挡不住中间人(MITM)。** 监听者只能看不能改时 DH 安全;但如果攻击者能**篡改**线路,他可以分别和 Alice、Bob 各做一次 DH,夹在中间转发。DH 解决的是机密性,**不**解决"对面到底是不是 Bob"。真实 TLS 里 DH 永远配着证书/签名做身份认证。代码里 `g = 5` 还有个偷懒处:注释明说没验证 5 是不是该素数的原根(primitive root,能生成整个乘法群的元素);不是原根的话共享密钥会落在一个小子群里,熵骤降。教学可以,生产不行。

## 亲手把它掰断:安全 = 素数有多大

现在最爽的部分。只拿公钥 `(n, e)`,用最蠢的办法——从 2 试除到 √n——分解 `n`,拿到 `p、q`,重算 φ,重建私钥 `d`:

```ts
// src/stage08-crypto.ts
function breakRsa(n: bigint, e: bigint): { p; q; d; trials } {
  let trials = 0;
  for (let f = 2n; f * f <= n; f++) {
    trials++;
    if (n % f === 0n) {
      const p = f, q = n / f;
      const phi = (p - 1n) * (q - 1n);
      return { p, q, d: modInverse(e, phi), trials }; // d 从偷来的 φ 重建
    }
  }
  throw new Error("n is prime — not an RSA modulus");
}
```

结果:

```
分解出 p = 64381, q = 70199  (64380 次试除)
重建私钥 d = 854053673  → 与真私钥 一致 ✓ 密钥已破解
实测耗时 = 1.396 ms (本机 wall-clock)
```

**1.396 毫秒,密钥没了。** 重建的 `d = 854053673` 和真私钥一模一样。这把"RSA 安全 = 分解难"从口号变成了你眼睛能看到的事实:小素数 = 秒破。

为什么真实 RSA 还活着?把这个试除法外推到 1024-bit 模数(同样的算法,~2^512 次除法):

```
同样的试除法分解一个 1024-bit RSA 模数约需 9.22e+138 年 (est.,宇宙年龄约 1.4e10 年)
```

**这个 9.22e+138 年是估算,我标清楚。** 它有两处不诚实之处,我都摊开:(1) 它假设每次除法耗时恒定,按玩具机器的速率线性外推;(2) 真实攻击者**绝不**用试除,而用 GNFS(通用数域筛,目前最快的通用分解算法),比试除快出几十个数量级。所以这个数字的**绝对值毫无意义**——它唯一传递的是**形状**:分解代价随较小素数的位数指数级爆炸。真实结论是经验性的:2048-bit RSA 在公开记录里至今没被分解过,但这是"还没人做到",不是"数学证明做不到"。

### ⚡ 别自己造生产密码学:数学对 ≠ 实现安全

到这里你可能觉得:我把数学搞对了,RSA 不就安全了?**错得离谱。** 上面所有代码数学都正确,但作为生产密码学全是垃圾。这一章压轴的开放问题正是:**数学正确性和实现安全性是两回事,而后者没有"照着课本写"这条捷径。** stage 演示了两种"数学完全正确、实现却被秒破"的方式。

**失败模式 1:弱随机源 → 密钥可预测。**

```
两方用同一 PRNG 种子(42) 生成素数: 50627 与 50627 → 完全相同 ✗
```

两方用同一个伪随机种子,生成出**完全相同**的素数。`core/rng` 用的 mulberry32 是 32-bit 状态——攻击者最多枚举 2^32 ≈ 43 亿个种子就能复现你的"随机"密钥,几分钟的事。**密钥的安全性 = 它的熵**(回到第 7 章:熵就是不可预测性的度量)。一个数学上完美的 RSA,配一个可枚举的随机源,等于把私钥贴在门上。真实密钥必须用 CSPRNG(`crypto.randomBytes` / `getrandom`),32-bit PRNG 永远不许碰密钥材料。

**失败模式 2:两把密钥共用一个素数 → 一次 gcd 全破。**

```
n1 = 6481781021, n2 = 6561962063
gcd(n1, n2) = 80021 → 立刻得到公因子 80021 (无需试除),两把私钥同时沦陷 ✗
```

如果两个不同的 RSA 模数**碰巧共享一个素因子**(典型成因:设备用低熵随机源批量生成密钥),你**不用分解任何东西**——直接对两个公开的 `n` 求 gcd,O(log n) 的欧几里得算法立刻吐出公因子,两把私钥同时完蛋。这不是理论:2012 年一次互联网级扫描真的在野外的 TLS 证书里发现了大量共享素因子的密钥,根因就是嵌入式设备启动早期熵不足。

这就是为什么 ⚡ 标"目前无通用解":**没有任何办法让"业余手写密码学"变安全。** 课本算法对、单测过、round-trip 成功——这些都不构成安全。真实威胁是侧信道(side-channel,从耗时/功耗/缓存命中泄漏密钥)、弱随机、padding oracle、实现 bug,它们每一个都能绕过"数学正确"。这个领域的工程铁律只有一条:**用经过审计、经过实战的库**(OpenSSL / libsodium / BoringSSL),它们的常量时间比较、抗侧信道实现、CSPRNG 接入是几十年攻防打磨出来的。stage08 的全部代码,标题里就写着"仅教学"——它存在的意义之一,就是让你亲眼看到照课本写的东西能被多快地破掉,从而**永远打消自己上生产的念头**。

## 一页纸总结

- 公钥密码 = 押"正向易、反向难"的单向函数:RSA 押分解,DH 押离散对数。这是经验性押注,不是数学定理(第 9 章继续)。
- 三个原语:扩展欧几里得(拿模逆元)、快速模幂(那个单向函数本体)、Miller–Rabin(不试除找素数)。逆元不存在/合数混入时**早崩**,别返回假值。
- RSA 安全 = 较小素数的位数。玩具素数 1.396 ms 被秒破;1024-bit 试除需 9.22e+138 年(est.,但真实攻击用 GNFS,远快于此,2048-bit 仍是经验上安全的下限)。
- ⚡ 数学对 ≠ 实现安全,且没有通用解。弱随机让密钥可枚举,共享素因子让一次 gcd 全破,侧信道绕过一切数学正确性。**生产一律用审计过的库,永不自造。**
- 承上(第 7 章信息论):密钥强度本质是熵;启下(第 9 章可计算性):"反向到底有多难"是 P vs NP 和量子 Shor 算法的疆域。
