如果你只从这一章带走一句话:**统计物理学里"对玻尔兹曼分布采样"和机器学习里"从模型生成/做后验推断",是字面意义上同一套数学**。不是类比,是同一个 acceptance 规则、同一个能量函数、同一个绕开归一化常数的技巧。扩散模型反向过程、能量模型(EBM)的 Langevin 采样、贝叶斯后验的 MCMC、组合优化的模拟退火——剥到底都是 Metropolis。本章用配套仓 `examples/math-for-builders-from-scratch/src/stage11-statmech.ts` 把这台机器**造三遍**,让你亲眼看到它是一个机制不是三个 trick:① 采一个一维双峰分布,② 退火解 TSP(旅行商问题,找最短闭合路径),③ 伊辛模型(一格磁铁自旋的格子模型)扫温度看相变。

先说清楚术语债。**配分函数 Z(partition function,所有状态概率之和,做归一化用)**——这是统计力学和 ML 共同的噩梦:它是一个高维积分,几乎永远算不出来。本章的核心爽点就是:**Metropolis 采样根本不需要 Z**。

## 一台机器:Metropolis 与玻尔兹曼分布

玻尔兹曼分布说:一个状态 $x$ 出现的概率正比于 $e^{-E(x)/T}$,$E$ 是能量,$T$ 是温度。低能量状态概率高,温度越高分布越平。要采它,你得知道归一化常数 $Z = \sum_x e^{-E(x)/T}$——而这是个天文数字的求和。

Metropolis 算法的精妙在于:**它只看概率的比值**。从状态 $x$ 提议一个新状态 $x'$,接受概率是 $\min(1, e^{-(E(x')-E(x))/T})$。比值里 $Z$ 上下一约就没了。看配套代码 `src/stage11-statmech.ts` 的 `metropolisSample`,核心就三行:

```typescript
const xProp = x + stepStd * gaussian(rng);
const logpProp = logDensity(xProp);
// 在 log 空间接受:accept iff log(u) < Δlogp
if (Math.log(rng() + 1e-300) < logpProp - logp) {
  x = xProp;
  logp = logpProp;
}
```

两个工程细节不是装饰,是会坏的地方。**第一,在 log 空间算**。代码注释写得很直白:双峰高斯混合在尾部密度 ~1e-300,直接乘除会下溢成 0,accept 比值变 0/0。换成 `logpProp - logp` 就稳了。这不是统计学家的洁癖——你写任何真实的 EBM/扩散采样器,能量都在 log 空间,原因一模一样。**第二,提议必须对称**。代码用对称高斯噪声 `x + N(0, stepStd²)`,所以 Hastings 修正项是 1,接受规则退化成裸的 `min(1, ratio)`。提议不对称时(比如带漂移的提议),你漏掉修正项,采出来的就是错分布——而且错得很隐蔽,直方图看着像那么回事,峰位置却是偏的。

跑出来的真实数字(种子 1101,40000 步去掉 2000 burn-in):

```
样本量 (去 burn-in)     = 38000
落在左峰 (x<0.5) 的比例 = 53.3%  (目标混合权重 = 60%)
样本均值                = 0.341  (理论 0.6·(-2)+0.4·3 = 0.000)
```

这里我要诚实:**53.3% vs 目标 60%、均值 0.341 vs 理论 0.000,这俩数字其实不算特别好**。为什么?因为这个目标分布的两个峰离得远(中心 -2 和 +3),中间是低概率的"势垒"。链要从左峰跳到右峰,必须连续走过一串低概率点,步长 1.5 让这种跨峰跳跃很少发生——这就是 MCMC 最经典的失败模式:**模式间混合慢(poor mixing)**。链在每个峰内部采得很准,但两峰之间的相对配重要采准就需要海量步数。代码注释明确点了这个 failure mode:步长太小链跨不过去,太大几乎全被拒。这不是 bug,这是 MCMC 诊断的核心难点——**你看一条链收敛了,不代表它真的遍历了整个分布**,它可能只是在一个模式里待得很舒服。

但形状是对的。把经验直方图和解析密度并排画(都用 `asciiBar`,各自 autoscale),峰位和相对高度一致:

```
经验直方图 (MCMC 样本计数):           目标密度 (解析):
-1.9 │████████████████████████████████████████│ 5337     -1.9 │████████████████████████████████████████│ 0.5973
 2.8 │████████████████████████████████████████│ 3215     2.8 │██████████████████████████████████████████│ 0.3952
```

两个峰、相对高度比、整体轮廓全对上了。这就是那句话的证据:**没算 Z,只靠样本,就还原了正确分布的形状**。这正是扩散模型能工作的全部理由——它从来没算过图像分布的归一化常数,它只学会了"往高概率方向挪一步"。

⚡ **这里埋一个前沿/开放问题:多模态分布的高效采样,目前没有通用解。** 上面 53.3% 的偏差就是缩影。当目标分布有许多被高势垒隔开的模式(真实世界的后验、复杂能量地形几乎都是这样),标准 MCMC 会卡在某个模式里指数级长的时间出不来。这就是 ML 里反复出现的 **mode collapse(模式坍缩)** 的物理根源。学界有一堆缓解手段——并行回火(parallel tempering)、哈密顿蒙特卡洛(HMC)、归一化流辅助提议——但"如何在任意多模态分布上保证高效遍历"仍是开放的活跃研究方向,没有一招通吃的方法。你在生产里跑 MCMC/采样,**收敛诊断(R-hat、有效样本量)是必须的,不是可选的**,因为这个问题没被解决。

## 同一台机器,温度趋零:模拟退火

模拟退火就是 Metropolis 加一条:让温度 $T$ 随时间往 0 降。物理直觉很干净——高温时几乎什么 move 都接受,搜索自由乱走,能爬出局部最优;$T \to 0$ 时只有改善的 move 能活下来,系统沉进一个极小值。`src/stage11-statmech.ts` 的 `annealTsp` 把这用在 TSP 上,能量 = 路径长度:

```typescript
const dE = candLen - len; // 能量 = 路径长度,要最小化
// 改善的 move 总接受;变差的 move 以 exp(-ΔE/T) 接受——
// 高温时这些"上坡"move 正是爬出局部最优的关键
if (dE < 0 || rng() < Math.exp(-dE / Math.max(t, 1e-9))) {
  tour = cand; len = candLen;
  if (len < bestLen) { bestLen = len; best = tour.slice(); }
}
t *= cooling;
```

**整个算法的艺术全在 `cooling` 这一个数上**,这也是它最容易坏的地方。降温太快,你在第一个掉进去的坑里就冻住了。配套 stage 专门把这个失败模式做成了对照实验:同一个 30 城实例、同样 60000 步、同一组 5 个种子,**只改降温率**,慢的(×0.9999)和快的(×0.99)直接 PK。基准用 nearest-neighbor + 2-opt 启发式(一个强基线,注意:不是证明最优——随机实例没有闭式最优解,谎称"距最优 X%"是不诚实的,所以代码诚实地拿强启发式当参照)。真实数字:

```
基准 (NN + 2-opt 启发式)  = 4.6565  (强基线,非证明最优)
慢降温 (×0.9999) 5 种子均值 = 4.6288  (相对基线 -0.6%)
快降温 (×0.99)   5 种子均值 = 4.8222  (相对基线 3.6%)
两者差距 = 4.2%（快降温更差）
```

慢降温甚至略微超过了强基线(-0.6%),快降温落后 4.2%。收敛曲线(seed=202)把差别画得一目了然:

```
慢降温:从乱序(14.8)平滑逼近 4.63,一路下探
快降温:几步内冲到 4.89 就贴底冻住,再也下不去
```

这里有三个工程判断值得你抄走。**第一,为什么用 5 个种子取均值而不是一次?** 退火是随机的,单个种子可能走运或踩雷,翻转结论。"慢的稳赢"这种 robust 主张必须能扛住跨种子平均——这是上一章和全书反复强调的诚实数字原则。**第二,为什么不用"在圆上的点"这种玩具实例?** 代码注释点破了:点在圆上的 TSP 太简单,怎么降温都能找到最优,反而**藏住了我们想暴露的失败模式**。要演示退火怎么坏,你得用一个真有许多局部最优的崎岖地形(单位正方形里 30 个随机点)。**第三个反直觉的点**,代码注释里也写了:比 0.9999 **更慢**地降温同样会变差——因为在固定步数预算内 $T$ 根本没冷下来,系统一直在乱走从不沉淀。所以真正的教训不是"越慢越好",而是 **schedule 必须匹配你的步数预算**。这跟深度学习的 learning rate schedule 是同一个道理:warmup、cosine decay,本质都是在调一条"探索→收敛"的温度曲线。

退火 vs 梯度下降的分工也值得点一句:梯度下降需要可微的能量面,退火不需要——它只要你能算 $\Delta E$ 和提议一个邻居 move。所以组合优化(TSP、调度、布线)这类离散、不可微的问题是退火的主场,而连续可微的损失面是梯度的主场。两者都是"在能量面上下山",区别只是用不用导数信息。

## 相变:伊辛模型与"涌现"的物理直觉

现在到本章最有 ML 共鸣的部分。**伊辛模型**:一个 $L \times L$ 的格子,每个点是一个自旋 $\pm 1$,相邻自旋同向时能量低(倾向对齐)。在这个系统上跑 Metropolis(同一台机器第三遍),扫描温度,测**磁化** $|M|$(全部自旋的平均绝对值,衡量整体有多"对齐"的序参量)。`src/stage11-statmech.ts` 的 `isingMagnetization` 核心是单自旋翻转的局部能量变化:

```typescript
const localEnergyChange = (r, c) => {
  const s = spin[idx(r, c)];
  const neighborSum = spin[idx(r-1,c)] + spin[idx(r+1,c)]
                    + spin[idx(r,c-1)] + spin[idx(r,c+1)];
  // 翻转 s 的代价 ΔE = 2·s·Σneighbors (J=1)
  return 2 * s * neighborSum;
};
```

两个设计选择有讲究。**为什么用 $|M|$ 而不是带符号的 $M$?** 低温时格子会任选两个对称基态之一(全上或全下),每次跑随机选一个;带符号的 $M$ 跨多次运行会平均到 0,**把序给抹掉了**。绝对磁化才是有限格子上诚实的序参量。**为什么用周期边界(环面)?** 让每个自旋都有 4 个邻居,没有特殊的边界 case。

扫温度的真实输出(16×16=256 自旋,每点 600 平衡 + 300 测量 sweep):

```
Onsager 解析临界温度 T_c = 2.269
温度 → 平均 |磁化| (序参量):
   T=1 │████████████████████████████████████████│ 0.9996
   T=2 │████████████████████████████████████····│ 0.9089
T=2.27 │█████████████████████████████···········│ 0.7339
 T=2.4 │█████████████████████···················│ 0.5327
 T=2.7 │████████································│ 0.1911
   T=4 │████····································│ 0.1102
|磁化| 跌破 0.5 处 T ≈ 2.43  (有限格子,偏离 2.27 是预期的有限尺寸效应)
```

看这条曲线。低温(T=1)几乎完全对齐 $|M|≈1$,高温(T=4)几乎完全无序 $|M|≈0.11$,**而在 $T_c≈2.27$ 附近,磁化骤然崩塌**。这就是**相变**:一个参数(温度)连续变化,系统的宏观性质在某个临界点**不连续地突变**。不是平滑过渡,是质变。Onsager 在 1944 年给出了二维伊辛精确临界温度 $T_c = 2/\ln(1+\sqrt2) ≈ 2.269$——历史上第一个被严格求解的相变,代码里直接算了这个解析值对照。

我们测到的跌破 0.5 点在 $T≈2.43$,偏离 2.27。**这个偏离不是 bug,是物理**:有限尺寸效应。16×16 太小,真正尖锐的相变只在无限大格子上出现,小格子上转变被抹圆、临界点被推高。这是个诚实的 toy:形状和定性结论(低温有序、高温无序、临界点骤变)是真的,绝对数字带尺寸偏差。

**为什么这跟大模型"涌现(emergence)"有关?** "涌现"指模型规模/训练量越过某个阈值后,某种能力(多步推理、in-context learning)突然出现,而不是平滑爬升。这条曲线给了它最干净的物理直觉:**系统在某个控制参数处发生相变,宏观行为质变**。注意我用词谨慎——这是**直觉来源/类比**,不是说大模型涌现就是字面意义的热力学相变。学界对"涌现是不是真相变、还是度量选择造成的假象"本身就有争论(比如有论文论证某些涌现曲线是因为用了非连续的 metric)。但伊辛模型让你亲手摸到"参数过临界点→性质突变"这件事是真实存在的物理现象,不是玄学。

### 失败模式:平衡不足会伪造相变

最后这个失败模式,任何跑过蒙特卡洛模拟的人都该刻进 DNA。`isingMagnetization` 的不变量是:**`equilSweeps` 必须足够大,让你测的是平衡态系综,不是还在演化的瞬态**。代码专门做了对照——T=2(应该明显有序),平衡 sweep 给够 vs 给不够:

```
T=2, 5 次独立运行:
  平衡=2   sweep: 均值 |M| = 0.638  抖动 std = 0.070
  平衡=400 sweep: 均值 |M| = 0.909  抖动 std = 0.011
```

平衡只给 2 个 sweep 时,$|M|$ 被严重低估(0.638 vs 真值 0.909),而且 run 间剧烈抖动(std 0.070,是充分平衡的 6 倍多)。后果很阴险:**你会误以为相变发生得更早、更软**——因为本该 $|M|≈0.9$ 的低温点读成了 0.64,整条曲线被压平,临界点看着往左挪。你没测系统真实性质,你测的是"系统还没想清楚自己要去哪"的中间状态。

为什么平衡不足这么坑?因为**临界点附近有"临界慢化"(critical slowing down)**:关联时间发散,链需要极长时间才能 decorrelate。代码注释里 600 个平衡 sweep 就是为此选的——刚好够让 16×16 的曲线保持单调。这条教训直接迁移到 ML:你跑 MCMC 后验推断、跑扩散采样,**burn-in/采样步数不够时,你拿到的不是分布的样本,是初始化残留的偏差**。等平衡再测——这是采样方法不可谈判的纪律。

## 收束:三个 demo,一台机器

回到开篇那句话。① 采双峰分布、② 退火解 TSP、③ 伊辛扫温度,你以为是三个不同的算法,其实是同一台机器换了三层皮:

| 角色 | ① MCMC 采样 | ② 模拟退火 | ③ 伊辛模型 |
|------|------------|-----------|-----------|
| 能量 $E$ | $-\log p(x)$ | 路径长度 | 自旋耦合 $-\sum s_i s_j$ |
| 提议 move | 高斯扰动 | 翻转路径段 | 翻单个自旋 |
| 接受规则 | $\min(1, e^{\Delta\log p})$ | $e^{-\Delta E/T}$ | $e^{-\Delta E/T}$ |
| 温度 $T$ | 固定 | 降到 0 | 扫描(看相变) |
| 绕开了什么 | 配分函数 $Z$ | 全局搜索的组合爆炸 | $Z$ |

而这台机器正是现代生成式 AI 的底层:扩散模型的反向去噪是带学习能量梯度的 Langevin 采样,能量模型用对比散度(本质是短链 MCMC)训练,贝叶斯神经网络用 HMC/SGLD 采后验。理解了"提议-接受-不需要 Z"这一个循环,你就理解了它们共同的数学骨架。

下一章(第 12 章「计算机为什么能算」)会把视角从"算法对不对"切到"机器凭什么能算":从一颗 NAND 门搭出全部逻辑、算术与记忆,再算出擦除 1 bit 的能量物理下界(Landauer 极限)——这些采样和退火循环里的 `exp`、随机数、格子遍历,最终都要落到晶体管这层物质基础上才能发生。本章给了你物理直觉,下一章给你执行它的物理硬件。
