如果你只记住这章一句话:**DFT(离散傅里叶变换)就是第 3 章那个矩阵-向量乘法,只不过矩阵的每个元素是一个复数旋转因子。** 这不是类比,是字面意义上的相等。我们配套代码里的 `dftViaMatrix` 直接复用了第 3 章的 `matVec`,跑出来跟教科书里那个吓人的求和公式逐元素误差 `1.47e-14`——浮点舍入级别的零。所以傅里叶不是一门新数学,它是线性代数换了一组基:从"逐时刻采样值"这组基,换到"各个频率成分"这组基。

为什么值得专门花一章讲一个"换基"?因为这次换基有个魔法性质:**很多在时域(time domain,信号随时间起伏的原始波形)里极难的操作,换到频域(frequency domain,信号由哪些频率叠成)后变成一行乘法。** 最典型的是卷积——两个信号做卷积在时域是 O(N²) 的嵌套求和,在频域只是逐点相乘。滤波、相关、模糊、回声,全是卷积的马甲。这就是为什么 JPEG、MP3、5G、Shazam、CT 重建、甚至 Transformer 里的某些长卷积加速,底层都站在这同一台引擎上。

这章四个机制,每个先讲怎么坏、为什么这么设计,最后一个失败模式(混叠)我们会让信号在你眼前"变脸"。

## 一、正弦基:为什么任意信号都能拆成纯音的叠加

### 直觉:傅里叶在赌什么

傅里叶级数的核心断言是:**任何周期信号,无论多丑,都可以写成一堆不同频率、不同振幅、不同相位的正弦波之和。** 一个方波?是无穷多个奇次谐波正弦的叠加。一段人声?是基频加一串泛音。这听起来像信仰,但它是有严格证明的——正弦函数族 `{cos(2πkn/N), sin(2πkn/N)}` 在 `[0, N)` 上两两正交(内积为零),它们构成了 N 维信号空间的一组**正交基**。

正交是关键。回忆第 3 章:在正交基下求坐标不需要解方程组,只需要拿信号跟每个基向量做内积。DFT 的第 k 个输出 `X[k]`,字面意思就是"原信号在第 k 个频率正弦上的投影分量"。这就是为什么频谱能告诉你"信号里含多少 5 Hz 成分"——它在算投影。

### 代码:DFT 矩阵长什么样

配套代码 `examples/math-for-builders-from-scratch/src/stage10-fourier.ts` 里,我们没有藏在复数类后面,而是把 DFT 矩阵显式建出来:

```ts
function dftMatrix(N: number): { cos: number[][]; negSin: number[][] } {
  const cos: number[][] = [];
  const negSin: number[][] = [];
  for (let k = 0; k < N; k++) {
    const cRow: number[] = [];
    const sRow: number[] = [];
    for (let n = 0; n < N; n++) {
      const angle = (-2 * Math.PI * k * n) / N;
      cRow.push(Math.cos(angle));
      sRow.push(Math.sin(angle));
    }
    cos.push(cRow);
    negSin.push(sRow);
  }
  return { cos, negSin };
}
```

矩阵第 `[k][n]` 个元素是 `exp(-2πi·kn/N) = cos(2πkn/N) - i·sin(2πkn/N)`。因为本书 core 库只在 `number[]` 上工作(无运行时依赖),复数被拆成两个平行的实数数组(re/im),DFT 矩阵就成了两个实矩阵(cos 部分和 -sin 部分)。这是个刻意的可读性取舍:你能看见每个旋转因子,没有任何东西藏在 `Complex` 类里。然后 `dftViaMatrix` 就是两次矩阵-向量乘法:

```ts
function dftViaMatrix(signal: readonly number[]): Complex {
  const N = signal.length;
  const { cos, negSin } = dftMatrix(N);
  return { re: matVec(cos, signal), im: matVec(negSin, signal) };
}
```

`matVec` 是从第 3 章直接 import 的。这一行就是"DFT 即线性代数"的全部证据。

### 失败模式:频谱泄漏(spectral leakage)

正弦基的正交性有个前置条件经常被忽略:**信号必须在窗口里是整周期的。** DFT 默认你给的 N 个样本是无限周期信号的一个完整周期。如果你截下来的一段刚好不是整周期(比如频率落在两个 bin 中间),DFT 会把这个"不合群"的频率的能量,涂抹到周围一大片 bin 上——这叫频谱泄漏。本来应该是一根尖峰,结果变成一座小山包,旁边还拖着旁瓣。

这不是 bug,是 DFT 对"非整周期信号"的诚实反应:它只会用它手里那组整周期正弦去拟合你的信号,凑不出来的部分就摊开。工程上的对策是**加窗(windowing)**:在做 DFT 前给信号乘一个两端衰减到零的窗函数(Hann、Hamming、Blackman),强行让边界连续,代价是主峰变宽一点、换取旁瓣大幅压低。下一节的 stage 演示里,我们刻意把三个频率都放在整数 bin 上(2、5、11),就是为了避开泄漏、让峰干净——这是 toy 设定,真实信号你几乎总要加窗。

## 二、频谱识别:把混在一起的频率分离出来

这是傅里叶最像魔术的一幕。我们合成一个把三个频率(bin 2、5、11)叠在一起的信号,振幅各不相同,肉眼看波形就是一团乱麻。喂进 FFT,频谱直接把三根尖峰戳在正确的位置上:

```ts
const planted = [
  { bin: 2, amp: 1.0 },
  { bin: 5, amp: 0.6 },
  { bin: 11, amp: 0.3 },
];
const signal = Array.from({ length: N }, (_, n) =>
  planted.reduce((s, t) => s + t.amp * Math.cos((2 * Math.PI * t.bin * n) / N), 0),
);
const spectrum = magnitude(fft(signal, new Array(N).fill(0)));
const half = spectrum.slice(0, N / 2 + 1);
```

stage 真实输出的半谱:

```
 k=2 │████████████████████████████████████████████████│ 16.00
 k=5 │█████████████████████████████···················│ 9.600
k=11 │██████████████··································│ 4.800
其余 │················································│ ~1e-15(浮点零)
代码自检: 最大的 3 个 bin = [2, 5, 11]  (与种入完全一致 ✓)
```

注意非峰值 bin 不是真的零,是 `2e-15` 到 `8e-15` 量级——这是浮点舍入的地板,不是信号。三根峰的高度比 `16 : 9.6 : 4.8 = 1 : 0.6 : 0.3`,精确对应种入的振幅比。这个"自检"行不是装饰:代码真的把最大的 3 个 bin 取出来跟种入值比对,一致才打勾。这是本书的硬规矩——claim 要在代码里成真,不靠你眼睛看图。

### 为什么只看半谱

代码只取了前 `N/2+1` 个 bin。原因是:**实数信号的频谱是共轭对称的**——bin k 和 bin N-k 互为复共轭,幅值一样。上半谱是下半谱的镜像,不携带新信息。这不是优化,是数学事实:实信号只有 `N/2+1` 个独立的频率自由度。这个对称性后面滤波时还会咬你一口:你置零高频,必须连它的镜像一起置零,否则反变换出来的信号带虚部、不再是实数。

### 失败模式:bin 不等于 Hz

新手最常踩的坑:**把 bin 索引当成 Hz。** bin k 对应的真实频率是 `k · fs / N`(fs 是采样率)。只有当采样窗口恰好是 1 秒时,bin 才碰巧等于 Hz——下一节混叠演示就是故意这么设的,为了读数干净。换个采样率或窗长,bin 5 可能是 5 Hz、也可能是 215 Hz。Shazam 之类的音频指纹系统如果搞错这个换算,整个频率坐标系就全错位了。记住口诀:**频率分辨率 = fs / N**,想分辨更细的频率,要么降采样率、要么加长窗口,没有免费午餐。

## 三、FFT:把 O(N²) 砍成 O(N log N) 的那个分治

DFT 是矩阵乘法,N×N 矩阵乘 N 维向量,是 O(N²)。N=1024 时这是一百万次乘加,N=百万(一秒立体声音频)时是一万亿次——实时处理直接死。FFT(快速傅里叶变换)不是另一种变换,**它算的是一模一样的 DFT,只是用分治把重复计算消掉了。**

### 机制:偶/奇拆分共享子结果

Cooley-Tukey 基-2 FFT 的核心观察:一个长度 N 的 DFT,可以拆成两个长度 N/2 的 DFT——一个管偶数下标样本,一个管奇数下标样本——然后用旋转因子(twiddle factor)把它们拼起来。关键在于,这两个半长 DFT 的结果**被上下两半输出共享**,所以从不重算:

```ts
function fft(re: readonly number[], im: readonly number[]): Complex {
  const N = re.length;
  if (N === 1) return { re: [re[0]], im: [im[0]] };
  if (N % 2 !== 0) throw new Error(`[fft] length ${N} is not a power of two`);
  // ...split into even/odd, recurse...
  const E = fft(evenRe, evenIm);
  const O = fft(oddRe, oddIm);
  for (let k = 0; k < N / 2; k++) {
    const ang = (-2 * Math.PI * k) / N;
    const tw = { re: Math.cos(ang), im: Math.sin(ang) };
    const tRe = tw.re * O.re[k] - tw.im * O.im[k];
    const tIm = tw.re * O.im[k] + tw.im * O.re[k];
    outRe[k] = E.re[k] + tRe;             // 蝶形:上半用 +twiddle·odd
    outIm[k] = E.im[k] + tIm;
    outRe[k + N / 2] = E.re[k] - tRe;     // 下半用 -twiddle·odd
    outIm[k + N / 2] = E.im[k] - tIm;
  }
  return { re: outRe, im: outIm };
}
```

那个 `+twiddle·odd` / `-twiddle·odd` 的成对结构叫**蝶形运算(butterfly)**——FFT 的整个计算图就是层层叠叠的蝶形。递归深度 log₂N,每层 O(N) 工作,合起来 O(N log N)。

### 实测:加速比随 N 增长

光说复杂度没用,我们 wall-clock(墙钟时间)实测。stage 真实输出:

```
N=1024, 20 次平均 (本机 wall-clock):
  O(N^2) 矩阵 DFT = 24.341 ms/次
  O(N log N) FFT  = 0.180 ms/次
  加速比 = 135.5x  (校验和 -5.45e-12)
  理论上界 N/log2(N) = 102.4x (est., 不含常数因子)
```

实测 135.5 倍,比理论上界 102.4 倍还高。为什么会超过理论?因为理论上界 `N/log₂N` 只数了浮点运算次数,没算常数因子:矩阵 DFT 每次调用都要重建那个 1024×1024 的矩阵(`dftMatrix`),那是 O(N²) 的内存分配和写入,而 FFT 完全不碰这块。这恰好是 FFT 省掉的真实成本之一。**注意"本机 wall-clock"这个标注**:绝对毫秒数换台机器就变,可迁移的是那个比值,而且比值随 N 增大而增大——这才是渐进复杂度在现实里的样子。

### 为什么必须是 2 的幂(以及为什么这是个真坑)

代码里有一行 `throw`:长度不是 2 的幂直接报错。这不是偷懒。基-2 FFT 靠把 N 一分为二递归,N 不是 2 的幂就分不均,递归逻辑直接错——**不是慢,是结果静默错误**,这是教学代码库里最坏的一种 bug。所以我们宁可大声 throw 也不返回垃圾。

真实世界的 FFT 库(FFTW、numpy.fft)对任意长度都能跑,靠的是混合基算法:把 N 因式分解(Cooley-Tukey 处理合数因子,Bluestein 算法处理大质数因子)。但有个广为人知的性能悬崖:**N 是大质数时,FFT 退化到接近 O(N²)。** 所以信号处理实践里大家习惯把信号 zero-pad(补零)到下一个 2 的幂或 5-smooth 数,宁可多算点也别撞上质数长度。这是真实 codebase 里的隐性约定,新手常被坑。

## 四、频域滤波:为什么降噪是"置零再变回来"

现在兑现开头那个承诺——时域很难、频域一行。降噪在时域要设计滤波器系数、做卷积,麻烦。在频域?**信号能量集中在低频,噪声铺满所有频率,那就把高频 bin 全置零,再反变换回去。** 就这么粗暴。

```ts
const cutoff = 12;
const F = fft(noisy, new Array(N).fill(0));
const filtRe = F.re.slice();
const filtIm = F.im.slice();
for (let k = 0; k <= N / 2; k++) {
  if (k > cutoff) {
    filtRe[k] = 0;
    filtIm[k] = 0;
    const mirror = (N - k) % N;   // 共轭镜像必须一起置零,否则反变换带虚部
    filtRe[mirror] = 0;
    filtIm[mirror] = 0;
  }
}
const recovered = ifft(filtRe, filtIm).re;
```

stage 真实输出:

```
噪声 std=0.8, 截止 bin=12 (信号能量集中在 bin 3 与 7)
滤波前 SNR = 0.80 dB
滤波后 SNR = 11.82 dB
改善 = 11.02 dB
```

SNR(信噪比)从 0.80 dB 拉到 11.82 dB,改善 11 dB——一个数量级以上。这就是卷积=频域相乘的威力:一个理想低通滤波器在时域是 sinc 函数,跟信号做卷积是 O(N²) 的活;在频域它就是一个"低频留、高频砍"的 0/1 掩码,逐点相乘,O(N)。

### 三个必须诚实交代的点

**第一,这个 11 dB 偏乐观,别拿去吹。** stage 输出自己标了:这是 toy 合成信号(干净正弦 + 已知截止频率 + 种子固定的高斯噪声)。真实音频里信号和噪声没这么干净分带。**可迁移的结论是:当信号和噪声占据分离的频带时,低通能恢复 SNR;当它们重叠时,频域滤波无能为力**——你砍掉噪声的同时一定砍掉等量信号。这条边界比那个具体 dB 数重要一百倍。

**第二,那个 mirror(镜像)置零不是可选项。** 前面说过实信号频谱共轭对称。你只置零 `k > cutoff` 的上半 bin、不动它们在 `N-k` 的镜像伙伴,反变换出来的信号就带非零虚部,不再是合法的实信号。这是一个特别隐蔽的 bug:代码能跑、SNR 也像是改善了,但你的"实信号"悄悄成了复数。代码里那行 `const mirror = (N - k) % N` 就是在维护这个不变量。

**第三,这是 brick-wall(砖墙式)滤波,频域硬切。** 频域一刀切,在时域对应的是 sinc 卷积核——它有无限长的振铃拖尾。后果是滤波后的信号会出现 Gibbs 振铃(吉布斯现象):在原信号有突变的地方,旁边冒出过冲和振荡。真实滤波器(Butterworth、Chebyshev)用平滑的过渡带换取无振铃,代价是过渡带里频率衰减不彻底。又是一个没有免费午餐的取舍:陡峭度 vs 振铃,你只能选一头。

## 五、混叠:采样不够快,频谱会当面对你撒谎

这是本章的失败模式压轴,也是整个数字信号处理里最反直觉、最致命的一个陷阱。**奈奎斯特采样定理(Nyquist):要无失真还原一个最高频率为 f 的信号,采样率必须严格大于 2f。** 低于这条线,高频信号不会消失,它会**伪装成一个不存在的低频**——而且在离散数据里,你没有任何办法分辨真假。

我们让一个 30 Hz 的真信号,用 32 Hz 的采样率去采(奈奎斯特要求 > 60 Hz,这里公然违反):

```ts
const fsHz = 32;       // 采样率,对 30 Hz 信号远远不够
const trueHz = 30;
const N = fsHz * 1;    // 1 秒窗口 => bin 索引 == Hz
const sampled = Array.from({ length: N }, (_, n) =>
  Math.cos((2 * Math.PI * trueHz * n) / fsHz),
);
const half = magnitude(fft(sampled, new Array(N).fill(0))).slice(0, N / 2 + 1);
const peakBin = half.reduce((best, v, k) => (v > half[best] ? k : best), 0);
const aliasHz = Math.abs(trueHz - fsHz);   // 折叠公式
```

stage 真实输出:

```
真实频率 = 30 Hz, 采样率 fs = 32 Hz (奈奎斯特要求 fs > 60 Hz, 本例违反)
 2Hz │████████████████████████████████████████████████│ 16.00
其余 │················································│ ~1e-14
频谱峰值出现在 2 Hz, 而非真实的 30 Hz
折叠公式预测的混叠频率 = |30 - 32| = 2 Hz (与实测峰值一致 ✓)
教训: 采样率不足时, 30 Hz 的真信号与 2 Hz 的假信号在离散数据里无法区分。
```

信号当着你的面**变脸**了:输入 30 Hz,频谱坚称是 2 Hz,而且峰值高度 16.00 一点不含糊——它没报错、没警告、没给任何"我不确定"的信号。这才是混叠最可怕的地方:**它不会失败,它会自信地给你一个错误答案。** 折叠公式 `|trueHz - fs| = |30 - 32| = 2 Hz` 精确预测了这个假频率,代码自检对上了。

直觉上为什么会这样?想象拍摄高速旋转的车轮——快门(采样)跟不上轮子转速时,你会看见轮子慢慢倒转或静止。这就是视觉版的混叠,叫 wagon-wheel effect。30 Hz 的波在 32 Hz 采样下,每个采样点正好错过一点点,连起来就画出了一条 2 Hz 的慢波。

### 为什么这事不能靠"采完再补救"

混叠是**不可逆的信息丢失**。一旦欠采样,30 Hz 和 2 Hz 在你的离散数据里就是同一串数字,任何后处理都救不回来——信息在采样那一刻就永久没了。所以工程上的铁律是:**抗混叠滤波(anti-aliasing filter)必须在采样之前、用模拟电路做。** ADC(模数转换器)前面那个低通模拟滤波器,作用就是在采样前把超过 fs/2 的频率成分物理地砍掉,确保进入采样器的信号本身就满足奈奎斯特。等数字化之后再想滤,黄花菜都凉了。这是软件工程师最容易忽略的一环:有些约束必须在边界之外、在数据进入你的系统之前就满足。

## 六、⚡ 开放问题:亚奈奎斯特采样能否绕过这条铁律

奈奎斯特定理给了一条看似不可逾越的下界:采样率必须 > 2× 最高频率。但过去十几年有一个活跃的研究方向在追问:**如果信号是"稀疏"的——频域里只有少数几个非零成分(就像我们 stage 里那种只有 3 根峰的信号)——能不能用远低于奈奎斯特的采样率把它精确还原?**

这就是**压缩感知(compressed sensing)**和它的衍生分支。核心思路是:奈奎斯特定理保的是"任意"带限信号的最坏情况,但现实信号往往高度稀疏,稀疏性本身是额外的先验信息,理论上可以拿来换采样率。在医学成像(加速 MRI 扫描)、宽带频谱感知等场景,这个方向已经做出了能落地的系统。

但要清楚地标注:**这不是一把通用钥匙,奈奎斯特对一般信号依然是铁律。** 压缩感知只在信号确实稀疏、且你能设计出合适的随机测量矩阵时才成立;它对噪声敏感,重建是个优化问题(求解 L1 最小化)而非一次变换,代价高且有失败模式。"亚奈奎斯特采样在什么样的信号类和噪声水平下能可靠工作、重建保证有多强"——这一类问题**目前没有覆盖所有场景的通用解,仍在研究**。把它当成"奈奎斯特在特定先验下的可松弛边界"来理解,而不是"奈奎斯特被推翻了"。

---

收个尾。这章从头到尾在讲同一件事:**傅里叶变换是一次换基,把信号从时域搬到频域,而这次换基让卷积——也就是滤波、相关、模糊这一大类操作——从 O(N²) 的求和坍缩成一行逐点乘法。** FFT 让这次换基本身也变快(stage 实测 N=1024 时 135.5 倍)。频域给你的是上帝视角:你能看见信号由哪些频率构成、能精确地砍掉某段频带、能解释为什么音频压缩能丢掉一半数据而人耳听不出来。

但频域也有它的诚实边界,这章每一节都钉了一个:正弦基要整周期否则泄漏;bin 不是 Hz;FFT 怕质数长度;频域滤波只在频带分离时有效、硬切会振铃;最致命的是采样不够快时混叠会当面撒谎、且不可逆。这些"何时坏、为什么坏、代价是什么",才是把傅里叶从一个数学公式变成你能在生产里靠它做决策的工程工具的部分。下一台引擎——如果你接着往下读统计力学那章——会从另一个角度逼近同一种"宏观规律从微观涨落里涌现"的思路。
