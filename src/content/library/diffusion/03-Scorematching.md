第 02 章搭好了反向过程的骨架:从 `N(0,I)`(标准正态噪声)出发,一步步往回走,理论上能走回数据。但那一章一直把核心的那个网络当黑箱——它每一步吐出一个"该往哪个方向去噪"的向量,我们没说清那个向量**到底是什么量**。如果你以为它在学密度 `p(x)`(每个点有多大概率是真实数据),那就走进了 GAN 和 VAE 当年正面硬刚、头破血流的那条路。这章告诉你 diffusion 绕开了它:网络学的不是 `p(x)`,是 `∇_x log p(x)`——对数密度的**梯度**,一个在每个坐标点都告诉你"往哪走密度更高"的向量场。这个量有个名字叫 score function(评分函数,大白话:密度的上坡方向)。

为什么这个区别是 diffusion 全部优雅的来源,而不是一个数学小聪明,是这章要讲透的事。本章代码在 `examples/diffusion-from-scratch/src/stage03-score.ts`,共享数值底座在 `src/core/`(`nn.ts` 的 MLP、`optim.ts` 的 Adam、`data.ts` 的数据生成、`plot.ts` 的 ASCII 可视化)。

> **诚实声明(必读)**:本章配套的 `stage03-score.ts` 当前**没有跑通**——`npm run typecheck` 在该文件报两处错(`evaluateScoreField` 的返回值缺 `cells` 字段;`countOffShellMispoint` 是个返回 `-1` 的占位 stub,真实现 `offShellMispointRate` 还没写)。所以本章**不引用任何"实际运行输出"的数字**。我引用的所有数字分两类:(1) 源码里**硬编码的超参数常量**(`SIGMA_GOOD`、`TRAIN_STEPS` 等),这些是确定的;(2) 源码里**断言的目标阈值**(`>85%`、`~50%`、`<1e-3`),这些是代码作者设定的验收线,**不是实测结果**,我会逐处标注。机制和曲线形状可信,绝对数字等 stage 跑通后以真实输出为准。

## 为什么不能直接学 p(x):归一化常数这堵墙

先说清楚要绕开的是什么。任何概率密度都长这样:

```
p(x) = exp(-E(x)) / Z,   Z = ∫ exp(-E(x)) dx
```

`E(x)` 是能量(energy,大白话:某点有多"不像"数据,越大越不像),这部分你可以用一个神经网络随便参数化。麻烦在分母 `Z`,叫归一化常数,也叫配分函数(partition function)——它是能量在**整个空间**上的积分,保证 `p(x)` 加起来等于 1。

`Z` 是 energy-based model(基于能量的模型)几十年没真正解决的世纪难题。对 2D toy 数据你也许能数值积分硬算,但对一张 256×256 图像,`x` 是 196608 维,`∫...dx` 是十九万维空间上的积分,**没有任何办法精确算**。MCMC(马尔可夫链蒙特卡洛)估计要跑到收敛、方差巨大;变分下界(VAE 那一路)是在 `Z` 上套近似;GAN 干脆放弃显式密度、改用判别器隐式逼近。三条路都是在跟 `Z` 较劲。

✦ **这章最反直觉、也最优雅的一点**:你永远不需要知道 `p(x)` 是多少,只需要知道它的**方向**。看 score 的定义——对数密度的梯度:

```
∇_x log p(x) = ∇_x [ -E(x) - log Z ] = -∇_x E(x) - ∇_x log Z
```

`Z` 是个常数(不依赖 `x`),它对 `x` 的梯度**恒等于 0**。于是:

```
∇_x log p(x) = -∇_x E(x)
```

那堵几十年没翻过去的墙,在求梯度的瞬间被微分掉了。你不再需要 `Z`,也就不再需要积分整个空间。这是为什么 diffusion / score-based model 能绕开似然问题,而 GAN / VAE 必须正面处理它。代价是:你失去了直接评估 `p(x)` 的能力(知道方向不等于知道高度)。但对**生成**这件事,方向恰恰就够了——给定每个点的上坡方向,你就能把一个随机起点沿着向量场往高密度区推,这就是采样。

## 但 score 本身也不知道,怎么学?denoising score matching

问题变成:`p(x)` 我们没有解析式(只有一堆数据点),怎么训练一个网络去逼近 `∇_x log p(x)`?直接的 score matching(Hyvärinen 2005)需要算 score 的散度(divergence),对高维数据成本爆炸。实践里用的是 Vincent 2011 的 **denoising score matching**(去噪 score matching),它把"学 score"换成"学去噪",而后者有现成的回归目标。

核心是一个闭式结果。给一个干净点 `x0`,加固定尺度的高斯噪声:

```
x̃ = x0 + σ·ε,   ε ~ N(0, I)
```

对加噪后的 σ-平滑密度 `p_σ`,它的 score 有解析形式(见 `stage03-score.ts` 文件头第 9-14 行注释):

```
∇ log p_σ(x̃) = E[ -(x̃ - x0)/σ² | x̃ ] = E[ -ε/σ | x̃ ]
```

读这个式子:**给定一个噪点 `x̃`,它的 score 就是"指回原始干净点的方向",再除以 σ²**。所以一个网络如果能在看到 `x̃` 时预测出当初加进去的那个 `ε`,它就(差一个 `-1/σ` 的常数)预测出了 score。这就把"学梯度场"翻译成了"学去噪",而去噪是个标准的 L2 回归。

`stage03-score.ts` 的训练循环(`trainScoreNet`,第 157-189 行)是这套理论最直白的代码落地:

```ts
for (let i = 0; i < batchSize; i++) {
  const r = rng.choice(n);
  const e0 = rng.gaussian();
  const e1 = rng.gaussian();
  // x̃ = x0 + σ·ε ; 把 ε 存为回归 TARGET(它就是 score,差 -1/σ)
  xs[i * 2] = data.data[r * 2] + sigma * e0;
  xs[i * 2 + 1] = data.data[r * 2 + 1] + sigma * e1;
  eps[i * 2] = e0;
  eps[i * 2 + 1] = e1;
}
// ...
const pred = predictNoise(net, embed, xNoisy);
// L2 去噪损失:它的最小解是 E[ε|x̃],正是 score 方向
const diff = pred.sub(target);
const loss = diff.mul(diff).mean();
```

网络是个普通 MLP,`[2 + EMBED_DIM, 64, 64, 2]`(`makeNet`,第 143-146 行):输入是 2 维坐标拼一个时间嵌入,两层 64 宽,输出 2 维预测 ε。注意它**已经**带了时间嵌入(`FIXED_TIMESTEP = 1`,第 141 行),即便这章只有单一噪声尺度——这是故意让网络形状和第 04 章 DDPM 的去噪器**完全一致**,这章每个样本共用同一个 timestep,到第 04 章 timestep 才真正随机取。这种"形状提前对齐"省掉了后面重写网络的麻烦。

### 失败模式:loss 为什么不会到 0,而且这不是 bug

新手第一次跑去噪 score matching 常被吓到:loss 降到某个值就**不动了**,远不到 0。容易当成没收敛、学习率不对、网络太小。都不是。

原因在回归目标本身。逐样本的 `ε` 是随机抽的——同一个噪点 `x̃` 可能由很多不同的 `(x0, ε)` 组合产生。网络看到的输入是 `x̃`,但目标 `ε` 带着不可预测的随机性。L2 损失的最优解是**条件期望** `E[ε | x̃]`,而不是某一个具体的 `ε`。所以 loss 的下限是 `Var(ε | x̃)`(给定 `x̃` 时 `ε` 的条件方差),这是个**不可约**(irreducible)的常数,网络再大也压不下去。

这恰恰是 score matching 能成立的关键,不是缺陷:网络被逼着去学"平均方向",而那个平均方向 `E[ε | x̃]` 正好就是(缩放后的)score。源码作者把这点写进了运行时输出(`main`,第 338-340 行),提前堵住误读:

```ts
console.log(
  `NOTE: loss 不会到 0。L2 最优解是 E[ε|x̃],逐样本 ε 的条件方差是不可约的下限`,
);
```

`TRAIN_STEPS = 800`(第 53 行)、Adam 学习率 `3e-3`(第 158 行)、batch 128(第 161 行)是源码定的训练配置。"loss 降到约 0.1 左右"是文件头第 21 行的**描述性预期**,不是实测值——stage 跑通前不要把它当数字引用。可迁移的是**形状**:陡降后在一个正的地板上趋平,地板高度由数据噪声本身决定。

## 验证 score ↔ ε 的等价:一行代码就是第 04 章的桥

✦ 面试常问"diffusion 和 score-based model 是什么关系"。标准答案:**它们是同一个东西的两种语言**。DDPM(Ho et al. 2020)说网络预测噪声 `ε`;score-based model(Song & Ermon)说网络预测 score `s`。两者差一个确定的缩放常数。在本章的单一尺度 σ 下,这个关系就是:

```
s(x̃) = -ε_pred(x̃) / σ
```

`evaluateScoreField`(第 238-239 行)在 query 向量场时就是这么把 ε 转成 score 的:

```ts
// score = -ε_pred / σ
const sx = -epsPred.data[g * 2] / sigma;
const sy = -epsPred.data[g * 2 + 1] / sigma;
```

源码还专门写了 `verifyScoreEpsIdentity`(第 295-310 行)用两条独立代码路径算同一个 score,断言它们的逐元素误差 `< 1e-3`(第 348 行)。注意这个断言**不是学习结果,是一致性检查**——score = -ε/σ 是定义,两条路径只该差浮点舍入,误差本就该接近 0。它存在的意义是个接线保险:如果第 04、05 章的采样器把 score 接到了错的量上,这里会先炸。`1e-3` 是源码设的阈值,不是实测误差。

**为什么这座桥这么重要**:它让你可以用 DDPM 的 ε-prediction 训练(目标是 unit-variance 的 `ε`,跨噪声尺度都是 `N(0,I)`,回归尺度天然稳定),却在采样时随时把它读成 score 来做 Langevin / 反向扩散。第 04 章文件头(`stage04-ddpm-train.ts` 第 11-15 行)把这点说得更狠:ε 是"干净的中间量",因为 `x0`-prediction 在小噪声时尺度爆炸,而 score `∇log q` 需要把 `1/√(1-ᾱ_t)` 那个因子折进去。换句话说,**本章单尺度下的 `-1/σ`,到了第 04 章多尺度 DDPM 里就变成 `-1/√(1-ᾱ_t)`**(`ᾱ_t` 是第 01 章定义的累积保留比例,大白话:到第 t 步原始信号还剩多少)。同一个等式,只是 σ 换成了随 t 变化的 `√(1-ᾱ_t)`。本章用 2D 把这个缩放关系钉死,第 04 章直接拿去用。

## 把 score 画出来:向量场该指向数据

理论说网络学到的是"指向高密度区的向量场"。怎么肉眼验证?`evaluateScoreField`(第 208-259 行)在一个 21×21 的网格(`gridN = 21`,范围 `[-2.5, 2.5]²`)上 query 学到的 score,用 8 方向 ASCII 箭头画出来,再统计**有多大比例的格点 score 指向最近的数据模式**(夹角 < 90°,即点积为正)。

这里有个必须说清的事实修正:**本章配套数据不是双月,是 3 个高斯混合**(`mixtureOfGaussians`,3 个中心 `[-2,-2] [2,-2] [0,2.5]`,600 个点,见第 47-52 行)。全书大部分 stage(01、04 等)用的是 two moons(双月),但 `stage03-score.ts` 这一章用的是三模高斯混合——因为要验证"向量场指向最近模式",离散的几个团比连续月牙更容易定义"最近模式在哪"。如果你读到别处说"箭头指向双月",那是书的统一叙事,**这一 stage 的真实数据是三个团**。

判定逻辑是几何点积(第 240-249 行):

```ts
const mode = nearestMode(x, y, modes);
const tx = mode[0] - x; // 从格点指向最近模式的方向
const ty = mode[1] - y;
const dot = sx * tx + sy * ty;
// ...
if (!onMode && dot > 0) towardCount++;
```

源码断言"好"的尺度下 `>85%` 的格点 score 指向最近模式(第 364 行)。再强调一次:**`85%` 是源码设的验收阈值,不是实测命中率**——stage 没跑通,我不知道真实值。可迁移的判断是:一个学好的 score 场,在数据团之间的空地上也该有明确的指向,把过路的样本往最近的团里拉。

## ⚡ 单一噪声尺度学不好,这正是 DDPM 要 T 步的根因

现在到本章最值钱的那个失败模式——它直接解释了为什么 diffusion 要走几百上千步,而不是一步到位。

`stage03-score.ts` 的 `④ FAILURE` 部分(第 366-383 行)做了个对照实验:把训练噪声尺度从 `SIGMA_GOOD = 0.5` 换成 `SIGMA_TINY = 0.01`(第 54-55 行),重新训一个网络,在**同一个宽网格**上再测一次"指向最近模式"的比例。

为什么小 σ 会坏?看加噪公式 `x̃ = x0 + σ·ε`:σ 极小时,噪点 `x̃` 几乎贴着原始数据点。网络在整个训练过程中**只在数据壳(data shell)附近见过样本**——离数据团稍远的空地,它一个训练样本都没见过。于是远离数据的格点上,它的 score 是纯粹未定义的外推,乱指。源码注释(第 27-30 行)把这叫 low-density-region problem(低密度区问题):

```
④ FAILURE retrain with a TINY σ (0.01). The net only ever saw points hugging the data,
   so far from the data its score is uninformed garbage: the "points toward nearest
   mode" rate on the SAME wide grid collapses toward chance (~50%).
```

`~50%` 是源码预期的塌陷目标(趋近随机瞎猜),不是实测。但这个失败的**逻辑**是硬的,无关具体数字:真实采样从纯噪声 `N(0,I)` 出发,起点必然落在远离数据的低密度区。如果你只有一个小尺度,**第一步就走偏,而且永远回不来**——网络在那片区域根本没有正确的方向信号。

这就是 Song & Ermon 那个关键洞见的全部动机:**用多个噪声尺度,从大到小退火**。大尺度的 σ 把噪声铺满整个空间,给低密度区也提供了正确的"往数据方向走"的粗方向;小尺度的 σ 在贴近数据时提供精度。一个尺度负责一段距离的"往回拉"。把这串离散尺度连起来,就是 DDPM 的 T 步前向/反向过程;再把步数推到无穷、变成连续时间,就是 score SDE(随机微分方程形式)。第 04 章的多步 DDPM,本质上就是在自动化这章手动演示的"多尺度退火"。

**这里有个真正开放、目前无通用解的问题**:噪声尺度数(对应采样步数 T)和采样成本之间怎么取舍,仍是活跃研究。尺度太少,低密度区覆盖不够,样本质量塌;尺度太多,采样要跑几百上千次网络前向,慢到没法用。DDIM(第 05 章会讲)、蒸馏(distillation)、consistency model 这些方向都在攻"少步数也能采好"的问题,但**没有一个在所有数据分布上通用最优**——每种方法都在质量、多样性、速度的三角里选了不同的妥协点。如果有人告诉你某个采样器"全面碾压",那基本是在特定 benchmark 上的过拟合结论,换个分布就未必。这个 trade-off 至今没有银弹。

## 小结:这章给第 04 章留下了什么

退一步看这章建立的链条:

- **不学 `p(x)`,学 `∇log p(x)`**——归一化常数 `Z` 在求梯度时被微分掉,这是 diffusion 绕开似然难题的根。
- **denoising score matching** 把"学梯度场"翻译成"学去噪",`x̃ = x0 + σ·ε` 让回归目标 `ε` 现成可得,而 L2 最优解恰好是 score 方向。
- **score = -ε_pred / σ**——单尺度下 score 和噪声差一个常数缩放,这个等式到第 04 章变成 `-1/√(1-ᾱ_t)`,是 ε-prediction(DDPM 语言)和 score(Song 语言)互通的桥。
- **单一小尺度学不好低密度区**,而采样必然从低密度区(纯噪声)出发——这逼出了多尺度退火,正是第 04 章 DDPM 那 T 步存在的根本理由。

第 04 章把这章手动的"单尺度去噪 + 读出 score"折叠成一行 MSE:随机抽 t、用第 01 章的闭式 `q(x_t|x_0)` 一次性加噪、让网络预测那个 `ε`,跑满 T 步反向链,看 `N(0,I)` 重建出数据形状。这章建立的 score↔ε 等价,就是那一行 MSE 背后真正学到的东西。
