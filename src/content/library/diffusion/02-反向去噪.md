先把结论钉死,省得你被「反向去噪」这个词唬住:**撤销一步前向加噪,有闭式解**。第 01 章我们把双月数据一步步加噪推到纯高斯;现在反过来,已知 `x_t` 想推 `x_{t-1}`,这个反向后验 `q(x_{t-1}|x_t,x_0)` 是个高斯,均值和方差全是 schedule (噪声调度,每步加多少噪的时间表) 的算术,不含任何待学参数。唯一的麻烦在均值里那个 `x_0`——干净样本——你采样时没有它,它就是你要生成的东西。**网络要学的,只是「在不知道 x_0 时估出这个均值」,本质是回归,不是生成**。理解这一句,你就懂了为什么 DDPM 的训练损失是一行 MSE 而不是对抗损失。

这一章我先把 `x_0` 当成已知(用一个能解析算出最优 `x_0` 估计的「作弊」去噪器),证明反向链确实能把 `N(0,I)` 噪声搬回双月流形;再把作弊拿掉,让你看见缺口在哪。对应代码是 `examples/diffusion-from-scratch/src/stage02-reverse.ts`,固定种子 `RNG(1337)`,纯离线、不连网、不调任何 LLM,同一台机器逐字节复现。下面所有数字都来自它的真实输出。

## ✦ 反向后验在已知 x_0 时是闭式高斯,网络只是来估均值的

很多人对 diffusion 的第一印象是「网络凭空学会了从噪声里捞出图像,这事一定很玄」。不玄。把前向和反向摊开看:

前向(第 01 章)是 `x_t = √ᾱ_t·x_0 + √(1-ᾱ_t)·ε`,`ᾱ_t` 是累积保留比例,`ε` 是标准高斯噪声。这是个线性高斯过程。线性高斯过程有个性质:**给定两端,中间那一步的条件分布也是高斯,而且系数能直接解出来**。这就是 Ho et al. 2020 (DDPM 原论文) 的 eq. 6-7:

```
μ̃_t = coefX0·x_0 + coefXt·x_t
coefX0 = √ᾱ_{t-1}·β_t / (1-ᾱ_t)
coefXt = √α_t·(1-ᾱ_{t-1}) / (1-ᾱ_t)
β̃_t   = (1-ᾱ_{t-1})/(1-ᾱ_t)·β_t      (各向同性的后验方差)
```

`β_t` 是第 t 步加的噪声量,`α_t = 1-β_t`。注意均值 `μ̃_t` 是 `x_0` 和 `x_t` 的**凸组合**——两个已知点的加权平均,权重纯 schedule 算术。方差 `β̃_t` 干脆是个跟数据无关的常数。`stage02-reverse.ts` 里 `computePosterior` 把这几行直接照抄成代码:

```typescript
const coefX0 = (Math.sqrt(alphaBarPrev) * betaT) / oneMinusAlphaBarT;
const coefXt = (Math.sqrt(alphaT) * (1 - alphaBarPrev)) / oneMinusAlphaBarT;
const variance = oneMinusAlphaBarT === 0 ? 0 : ((1 - alphaBarPrev) / oneMinusAlphaBarT) * betaT;
```

跑出来的系数表(cosine schedule, T=200)长这样:

```
  t  |   coefX0   coefXt   posterior_var
  199 | 0.007784 0.031621   9.989e-1
  150 | 0.016608 0.974804   3.742e-2
  100 | 0.021572 0.977041   1.554e-2
   50 | 0.038728 0.961134   6.438e-3
   10 | 0.155137 0.844862   1.245e-3
    1 | 0.596179 0.403821   1.520e-4
    0 | 1.000000 0.000000   0.000e+0
```

读这张表能读出整个反向过程的脾气。`t=199`(起点附近)时 `coefX0=0.0078`,几乎不信任 `x_0` 估计,主要靠 `x_t` 微调,而且方差 `≈1.0` 极大——早期一步几乎是纯随机游走。越往 `t=0` 走 `coefX0` 越大:`t=1` 时 `0.596`,`t=0` 时 `1.0` 且方差归零,直接返回 `x_0` 估计。这条曲线就是「越靠终点越信任你对干净样本的判断」。

**关键:这张表里没有一个数需要训练**。它全是 schedule 决定的。整个反向采样器里,唯一不是固定算术的东西,是 `μ̃_t` 公式里的那个 `x_0`。网络存在的全部理由,就是在你没有 `x_0` 时把它估出来。所以 DDPM 训练目标是「让网络的 `x_0`(或等价地 `ε`)估计逼近真值」的 MSE,一个标准回归。没有判别器、没有对抗博弈、没有 mode-seeking 的不稳定——GAN 那套训练地狱,diffusion 用一个闭式后验加一个回归网络绕开了。这是 diffusion 比 GAN 好训练的根因,不是工程调参的运气。

### 为什么不能直接退化成「把真 x_0 塞回去」

你可能想:既然均值要 `x_0`,演示时我就把每个点自己的真 `x_0` 塞回去不就完了?**这是个会骗过你的陷阱**。看系数表 `t=0` 那行:`coefX0=1, coefXt=0`。最后一步直接返回 `x_0`。如果每步都喂真 `x_0`,那「还原结果」字面上就等于输入,加不加噪声看起来都一样——你证明的是 `x_0 = x_0`,一个同义反复,什么机制都没暴露。

所以 `stage02` 用的不是真 `x_0`,而是**有限数据集上的解析最优 `x_0` 估计**:`E[x_0|x_t] = Σ_i w_i·x_0^(i)`,权重 `w_i ∝ exp(-‖x_t - √ᾱ_t·x_0^(i)‖² / (2(1-ᾱ_t)))`。直觉:`x_t` 是某个数据点加噪来的,问「最可能是哪个点生的」,对所有候选点按似然做 softmax 加权平均。这正是贝叶斯最优去噪器——**一个完美训练的网络收敛到的就是它**。我们能精确算出来,只因为这里的「分布」是 200 个点的有限集;真实数据上算不了,才需要网络去逼近。`estimateX0` 实现了它,代价 `O(batch·n·2)`,在 n=200 时无所谓,真模型把这整个函数换成一次网络前向。

有个边界要小心:`t=0` 时 `1-ᾱ_t≈0`,softmax 的温度趋零会让 logits 爆成 `Inf`。代码里 floor 了一下:

```typescript
const twoVar = Math.max(2 * (1 - sched.alphaBars[t]), 1e-12);
```

不 floor 就是 `NaN` 满屏。这是数值实现里反复出现的坑:解析公式在端点除零,工程实现必须兜底。

## §2 用最优去噪器跑反向链:噪声团真能收拢成双月

把 `estimateX0` + `computePosterior` 串起来,从 `x_T ~ N(0,I)` 出发,每步 `reverseStep` 估 `x_0`、算后验、按方差采样 `x_{t-1}`,一路走到 `t=0`。为什么从纯高斯出发合理?第 01 章那条 cosine 链跑到头 `ᾱ_T = 6.072e-8 ≈ 0`,意味着 `x_T` 里原始信号几乎清零、剩下全是噪声,跟 `N(0,I)` 没区别。反向采样器就该从这个先验起步。

中间快照(ASCII 散点)能看见噪声团逐步收拢:`t=199` 是一团弥散的噪声,`t=100`、`t=50` 慢慢出现结构,`t=0` 已经能认出两道弯月。量化用 Chamfer 距离(两个点云互相找最近邻的平均平方距离,越小越像):

```
还原 vs 原数据    : 0.005171
纯噪声 vs 原数据  : 0.246165   (基线: 啥也不做)
改善倍数          : 47.6x
断言 0.005171 < 0.05 -> PASS
```

为什么用 Chamfer 而不是逐点误差?因为 `x_T` 是新抽的噪声,生成的点跟原数据**不是同一批身份**,逐点配对没意义。我们只关心**支撑集是否重叠**:每个生成点都靠近某个真数据点,且每个真数据点附近都有生成点。两个方向都小,才说明没丢 mode、没冒出多余的质量。47.6 倍的改善说明反向链确实把噪声搬回了流形。

**这里必须诚实标注**:`0.005171` 这种绝对值是乐观的——2D toy、n=200,本来就好看。可迁移的事实不是这个绝对数,而是 §2 和下面 §3 之间的**相对差距**。别拿这个数去对标真实数据集。

## §3 失败模式:均值完全正确,但去掉随机项 z,点云塌缩成一坨

现在做个对照实验,这是本章最有教育意义的部分。**同一个最优去噪器,均值每一步都算对,只把后验方差强行置 0**——也就是反向步里不再注入随机噪声 `z`,直接取后验均值:

```typescript
const std = injectNoise && t > 0 ? Math.sqrt(variance) : 0;
const out = mean.data.slice();
if (std > 0) for (let i = 0; i < out.length; i++) out[i] += std * rng.gaussian();
```

`injectNoise=false` 时 `std=0`,`x_{t-1}` 退化成确定性的后验均值。结果:

```
健康还原 (有 z)   : 79 个簇, Chamfer 0.005171
塌缩 (var=0)      : 1 个簇,  Chamfer 2.217584
簇数收缩          : 79 -> 1   (79.0x 减少)
Chamfer 恶化      : 0.005171 -> 2.217584   (428.9x 变差)
```

79 个簇塌成 1 个,Chamfer 恶化 428.9 倍。ASCII 图里所有点涌成一条斜线挤向中心。**注意:均值公式没动,每一步的 `μ̃_t` 跟 §2 一模一样**。坏掉的唯一原因是去掉了 `z`。

机制要讲透。回头看系数表:早期 `t≈199` 时 `ᾱ_t≈0`,`estimateX0` 里所有候选点的 softmax 权重几乎相等(因为 `√ᾱ_t·x_0^(i)` 全被压到 0 附近,谁也不比谁更像),于是 `x_0` 估计退化成**数据均值**——一个点,落在两道弯月中间的空隙里。健康版靠每步注入的 `z` 把不同轨迹推向不同方向,打破这个对称;去掉 `z` 后,所有轨迹被同一个「数据均值」吸过去,没有任何东西能把它们分开,于是一起涌向中心。**这是 mode collapse(模式塌缩,生成器只覆盖少数模式) 的微缩版**。

所以反向随机性 `z` 不是「凑出来的噪声」,它是**维持样本多样性、覆盖整个流形的承重墙**。DDPM 论文里那个 `+σ_t·z` 不是装饰。这条对照实验也顺手回答了一个常见困惑:「反向过程不就是去噪吗,为什么还要加噪声」——因为去噪的均值是收缩的,不加噪就所有样本收缩到同一点。

### ⚡ 反向方差到底固定还是可学?随机还是确定?这是设计选择,不是定论

§3 可能让你得出「`z` 是必须的」的结论。**别急着下定论,这恰恰是 diffusion 里仍在演化的开放设计空间**。

第一层:**方差固定还是可学**。DDPM 原版把后验方差固定成 `β̃_t`(就是上面系数表那列),不学。Improved DDPM (Nichol & Dhariwal 2021) 让网络也预测方差——具体是学一个插值系数,在 `β_t` 和 `β̃_t` 两个边界之间插 log-variance。代价是多一个输出头,收益是**小步数采样质量明显更好**。固定方差在步数砍到很少时会偏差累积,可学方差能补一部分。

第二层更激进:**反向到底该不该有随机性**。§3 证明的是「在 DDPM 这套均值估计下,去掉 `z` 会塌」,**不是「确定性采样一定塌」**。把整个反向过程看成 probability-flow ODE(概率流常微分方程,一条确定性轨迹)——无随机项——同样能从噪声走回数据。这就是第 05 章 DDIM 的理论根。DDIM 也去掉 `z`,但它不塌,靠的是每步用更准、更具信息量的 `x_0` 估计让轨迹天然分离,不需要随机踢一脚。**确定性却不丢多样性**,这跟 §3 的塌缩看似矛盾,其实是两种不同的反向公式。

所以「反向该不该有随机性」目前没有通用最优解:DDPM(随机)采样质量上限高但步数多,DDIM/ODE(确定性)步数少、可做潜变量插值和精确似然但有自己的取舍,可学方差又是另一条路。哪种在什么 step budget、什么数据上最优,**仍是活跃研究**,不是已经盖棺的结论。本章只把这个问题埋下,DDIM 的确定性反向链留到第 05 章实现。

## 这一章建立了什么,接下来缺什么

钉死三件事:(1) 撤销一步前向加噪有**闭式高斯后验**,反向过程客观存在,不是网络硬凑;(2) 这个后验的均值是 `x_0` 和 `x_t` 的凸组合,**唯一的未知是 `x_0`**;(3) 反向随机性 `z` 承重——去掉它(在 DDPM 均值下)会塌成 mode collapse 的微缩版。

缺口也明确:`x_0` 在采样时拿不到。本章用「有限数据集上的解析最优 `E[x_0|x_t]`」作弊补上了它,但真实数据上这个解析解算不出来(你没法对整个数据分布做 softmax)。所以下一步必须用一个网络去逼近它。第 03 章会告诉你,逼近 `x_0`、逼近 `ε`(噪声)、逼近 score(对数概率密度的梯度) 这三者其实是同一个量的线性变换,DDPM 选了估 `ε`——第 04 章就训练这个 `ε` 网络,把本章的解析作弊器换成真正学出来的东西。
