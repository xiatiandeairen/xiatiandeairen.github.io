到第 05 章为止,我们的模型只会一件事:从 `N(0, I)` 出发,去噪成训练分布里的**某个**样本。混合高斯有三个簇,它就在三个簇里随机落一个。但你想要的从来不是「随便给我一个」,而是「给我类别 0 的」——文生图里这叫 prompt,这里叫 condition(条件)。本章把无条件模型升级成条件模型,然后量化一个所有人都会撞上的代价:让它越听话,它就越没创意。

这是个真会坏的权衡,不是修辞。我先把它跑出来。配套代码 `examples/diffusion-from-scratch/src/stage06-cfg.ts`,固定目标类别 0,用 DDIM 采 300 个点,扫 guidance scale `w = 0, 1, 3, 8`,实测(seed=1337,两次运行逐位一致):

```
  w     :        0       1       3       8
  保真度 :     1.3%   74.7%  100.0%  100.0%
  多样性 :   3.8433  0.0387  0.0001  0.0000
```

保真度 = 落进目标簇 3σ 范围的样本比例(越高越听话),多样性 = 样本云的平均逐坐标方差(越高越发散)。`w=0` 时几乎没人落在目标簇(1.3%),样本云铺满整个数据范围;`w=3` 时 100% 听话,但多样性已经塌到 `0.0001`——所有点几乎挤成一坨。这条曲线就是本章的全部:**guidance scale 是「听话 vs 多样」之间唯一的旋钮,而且它没有免费档位**。下面拆开讲它怎么来的、为什么这么设计、以及推到极端时怎么彻底坏掉。

## 先说为什么不能直接「训一个条件模型就完了」

最朴素的想法:训练时把类别 label 喂进网络,采样时指定 label,完事。这确实能做(代码里的网络本来就吃 label),但你会发现一个问题——**它太「温和」了**。

原因在于训练目标。DDPM 的 ε-prediction(预测加进去的噪声)是个回归任务,网络学的是「给定 `x_t` 和类别 c,数据的条件期望长什么样」。期望是平均,平均是保守的。当三个簇在 2D 上离得近、或者噪声大时,条件期望会被邻近簇拉偏,网络给出的 `ε_cond` 只是「稍微偏向类别 c」,而不是「斩钉截铁就是 c」。`w=1`(纯条件,不外推)那一档的实测就是证据:保真度只有 74.7%,还有四分之一的样本跑偏了。

所以问题不是「网络不会条件生成」,而是「条件信号太弱」。怎么放大?这就引出 guidance。

## Classifier guidance:借分类器的梯度,笨重的前身

2021 年 Dhariwal & Nichol 的做法(论文里叫 "Diffusion Models Beat GANs"):**另外训一个分类器** `p(c | x_t)`,这个分类器要能在**带噪的** `x_t` 上判类别(注意不是干净图,是每个噪声水平都得会判)。采样时,在原本的去噪方向上,额外加一项分类器对输入的梯度:

```
ε_guided = ε_uncond − w · √(1−ᾱ_t) · ∇_{x_t} log p(c | x_t)
```

直觉是对的:`∇log p(c | x_t)` 指向「让分类器更确信这是类别 c」的方向,沿它走就把样本推得更像 c。`w` 控制推多狠。这其实就是 score(分数,即 `∇log p(x)`,见第 03 章)的贝叶斯分解——条件分数 = 无条件分数 + 类别似然的梯度。

但它有三个让人头疼的工程代价,正是后来被取代的原因:

1. **要额外训一个模型**。而且这个分类器必须在所有噪声水平上鲁棒——拿个现成的 ImageNet 分类器不行,它没见过 `x_t` 这种被加了一半噪声的图,判出来的梯度是垃圾。等于把训练成本翻倍。
2. **梯度质量直接决定生成质量**。分类器在高噪声段本来就判不准(图都快成纯噪声了,凭什么知道类别),它给的梯度噪声大,会把采样带歪。
3. **类别空间被钉死**。分类器是按固定类别集训的,想加新类别、想用连续条件(比如文本 embedding),整个分类器要重训。

第 3 点对文生图是致命的:你不可能为「一只穿西装的柴犬」训一个分类器。所以这条路在生产里基本没活下来。

## Classifier-free guidance:「free」指免去那个分类器

Ho & Salimans 2022 的洞察,一句话:**不需要单独的分类器,一个网络自己就能扮演两个角色**。

代价分解 `∇log p(c | x_t) ∝ ε_cond − ε_uncond`——也就是说,只要我手上同时有条件预测 `ε_cond` 和无条件预测 `ε_uncond`,它俩的差就是分类器梯度的等价物,根本不用真去训分类器。于是采样公式变成纯外推:

```
ε_guided = ε_uncond + w · (ε_cond − ε_uncond)
```

`w=0` 退化成纯无条件(忽略 label),`w=1` 是纯条件,`w>1` 沿「条件比无条件更像目标」的方向**加大步子**。本质就是「往听话的方向多走几步」。代码里就是这一行(`stage06-cfg.ts` 的 `sampleCFG`):

```ts
// CFG extrapolation: w=0 -> uncond, w=1 -> cond, w>1 -> push past cond.
const eg = epsUncond[i] + w * (epsCond[i] - epsUncond[i]);
const x0hat = (x[i] - somaT * eg) / saT;
next[i] = saPrev * x0hat + somaPrev * eg;
```

那「同一个网络怎么同时会条件和无条件」?靠训练时的一行——**随机丢弃条件**。每个样本以概率 `p_drop` 把类别置空(喂一个全零的 null token),逼网络在没有 label 时也得给出合理的去噪。`stage06-cfg.ts` 的训练循环:

```ts
// CFG label dropout: null -> the model sees the zero token, learning ε(x_t, t, ∅).
classIds.push(rng.uniform() < P_DROP ? null : data.labels[idx]);
```

代码里 `P_DROP = 0.15`,即 15% 的训练步走无条件分支。这个 null token 不是 embedding 表里的一行,而是**全零向量**——这样「丢弃 label」和「请求无条件预测」在数学上是同一个操作,采样时只要把 class id 设成 `null` 就拿到 `ε_uncond`。代码注释把这条不变量钉得很死:

```ts
// INVARIANT: classEmbed.parameters() must include this table or the labels never learn.
```

这就是「classifier-**free**」的全部含义:免掉那个外挂分类器,把条件和无条件揉进一个网络的一次训练里。代价?采样时每一步要跑**两次**前向(一次条件、一次无条件),推理算力 ×2。文生图里这笔账你天天在付——这也是后面 ⚡ 那一节「能不能去掉 CFG」研究的直接动机。

### 一个容易踩的实现坑:null token 必须真的「无信息」

如果你把 null token 也做成 embedding 表里的一行(而不是全零),训练时它会学到一个**非零的、带类别倾向的**向量——因为 dropout 是随机的,某些类别被丢得多,这行 embedding 会沾上那些类别的统计。结果 `ε_uncond` 不再是真正的「类别无关」基线,外推方向 `ε_cond − ε_uncond` 被污染,`w` 的语义就漂了。代码用全零向量 + 在反向里跳过 null 行(`if (cid === null) continue`)来保证这点:null token 没有参数,无从被带偏。这是个不写注释就会半年后自己踩的坑。

## 旋钮转起来:从 1.3% 到 100% 的代价

回到开头那张表,现在逐档解读它**为什么**长这样。`stage06-cfg.ts` 用了一个关键设计保证对比公平:**所有 w 共享同一份起始噪声 `x_T`**。DDIM 在 η=0 下是确定性的(见第 05 章),所以唯一的变量就是 `w`,曲线的走向可以干净地归因到 guidance 本身,不掺采样噪声。

```ts
// Shared starting noise across ALL guidance scales: this is what makes the comparison
// honest — same x_T, only w differs, so trend is causally w's doing.
```

- **w=0(纯无条件,保真度 1.3% / 多样性 3.8433)**:模型完全无视 label,从三个簇里随便采。1.3% 是「碰巧落进类别 0」的概率(三个簇,加上判定半径,蒙中约 1/3 再打折)。这一档证明:**不外推时,条件信号弱到几乎不存在**——因为 `w=0` 把条件项整个抹掉了。

- **w=1(纯条件,保真度 74.7% / 多样性 0.0387)**:这是「老老实实用条件预测、不放大」的档位。保真度跳到 74.7%,多样性从 3.84 暴跌到 0.039——条件项一旦参与,样本立刻向目标簇收拢。但还有 25% 跑偏,印证了前面说的「条件期望太温和」。

- **w=3(保真度 100% / 多样性 0.0001)**:外推 3 倍,全部落进目标簇。但注意多样性已经 `0.0001`——样本云塌成一个点附近的小团。**保真度的最后那 25 个百分点,是用几乎全部多样性换来的。**

- **w=8(保真度 100% / 多样性 0.0000)**:多样性已经低于打印精度。300 个不同的起始噪声,被 guidance 全压到同一个落点。这不是「类别 0 的多样生成」,而是「类别 0 最典型的那一个点」复制 300 份。

权衡的物理在这里:guidance 把概率质量从「类别 c 的真实分布」挤向「最像类别 c 的单点(条件分布的众数/mode)」。`w` 越大,挤得越狠。**保真度和多样性不是两个独立旋钮,是同一个旋钮的两端**,你不可能同时拉满。文生图里这就是为什么 CFG scale 拉高后,同一个 prompt 反复生成会越来越像、构图越来越套路——业界默认值停在 7~8,正是这条曲线上「够听话但还没把多样性榨干」的折中点。本例是它的 2D 微缩证据。

## 失败模式:over-saturation 有两副面孔

「guidance 过强」不是一种坏法,是**两种**——而且第二种反直觉,连最初写这个 demo 的人都猜错过。`stage06-cfg.ts` 把两种都跑出来了。

**(A) 塌成一点(多样性死)。** 这一种已经在可用区间内发生了。回看上面的扫描:`w=3` 时多样性 `0.0001`、保真度 100%——点云塌进簇心附近的一个点,多样性归零但还安全地待在目标簇里。代码用一个布尔断言钉住它:

```ts
const collapsedToPoint = divCollapsePoint < 0.01 && fidCollapsePoint > 0.95;
```

实测 `w=3` 多样性 `0.000121`(< 0.01)、保真度 100%(> 0.95),`-> PASS (确认塌缩)`。

**(B) 冲出分布(保真度死)。** 这一种是惊喜。按 (A) 的趋势你会自然预期:`w` 再大,样本只会塌得更死。但把 `w` 推到 16,实测:

```
(B) 冲出分布: 把 w 推到极端 (w=16), 外推过度 -> 采样轨迹失稳, 点云冲出目标簇
    w=16: 保真度 0.0% (从 100% 崩到 ~0), 多样性 0.044029
    对比 w=8 (簇内塌点, 保真 100.0%): w=16 把质量挤出了真实簇 —— 不是更紧的好样本, 而是 off-distribution 的坏样本。
```

注意两个数:**保真度从 100% 掉到 0%**,多样性从 `0.0000` 反弹到 `0.044`。样本没有「塌得更紧」,而是**整团飞出了目标簇**(0% 落在 3σ 内),散落到簇外的空地上(ASCII 散点里点云跑到右上角和边缘,簇中心反而空了)。代码的诚实之处在这条注释里——spec 原本假设 `w=16` 会塌成更紧的点,但测出来不是,作者把真实结果写进了断言:

```ts
// NOTE (honesty): the spec hypothesized w=16 would collapse to an EVEN TIGHTER point than w=8.
//   The measured numbers say otherwise — collapse to a point already completes by w=3, so by
//   w=16 there is nothing left to tighten and the over-extrapolation instead pushes samples
//   OFF the cluster (fidelity 100% → ~0%).
```

所以失败模式断言被写成了正确的形态——「极端 w 摧毁保真度」,实测 `-> PASS (确认冲出分布)`,程序 `EXIT=0`:

```ts
const offDistribution = fidBlown < 0.5 && fidBlown < fidSeries[3];
```

(B) 是本章最重要的一课:**CFG 的线性外推在大 w 下会冲出数据流形(data manifold,真实数据在高维空间里实际占据的那层「皮」)**。机制是这样的——外推公式 `ε_uncond + w·(ε_cond − ε_uncond)` 假设「条件方向」在任意步长上都成立。但 `ε_cond − ε_uncond` 是在数据附近估出来的局部方向;当 `w` 大到把 `x_0_hat` 推出训练数据见过的区域,网络在那片没数据的空地上的预测纯属外插瞎猜,去噪轨迹失控,样本被甩到簇外。**它不是塌进众数,而是冲过了头,落到流形外的无人区。**

两种失败合起来才是完整图景:guidance 既能把质量挤成一点(A,多样性死),也能把质量挤出边界(B,保真度死)。**两者都是「过度听话」——把全部概率质量从目标类的真实分布上挪走,要么挤成一点,要么挤出流形。** 这正是真实文生图 over-saturation(过饱和)的本质:CFG scale 拉太高(典型 >15),图像出现颜色爆掉、对比度失真、结构崩坏——因为采样轨迹离开了模型真正学过的自然图像流形。我们的 2D toy 用「飞出目标簇」这个最朴素的形态,把同一个物理量到手测到了。

> 一个诚实标注:spec 最初猜 `w=16` 会塌得更紧,这个预期是错的——多样性的崩塌在 `w=3` 就已经完成,`w=16` 没东西可塌,过度外推转而把样本推出了流形。代码没有掩盖这个偏差,而是把断言改写成正确的刻画并通过(`EXIT=0`,seed=1337 可复现)。把这段经过留在这里,因为它恰好示范了:**连写这个 demo 的人都对大 w 的行为预期错了**,而这正是下一节要讲的开放问题。

## 为什么是「外推」这种笨办法,而它真的对吗

退一步看 CFG 的设计哲学。它本质是一个**无原理的线性假设**:把 `ε_cond − ε_uncond` 当成一个可以任意缩放的「方向向量」,认定走得越远越像目标。在 `w` 不太大时,这个假设在数据流形附近近似成立,所以 CFG 简单、便宜(就改两行)、效果好,统治了文生图三年。

但它从一开始就没有理论保证大 `w` 安全。我们刚刚在 `w=16` 上亲眼看到了它的崩溃边界。这把我们带到本章的前沿问题。

## ⚡ 前沿:CFG 的「无脑线性外推」正在被换掉,但换成什么没有定论

CFG 好用,但它的两个硬伤——**大 w 偏离流形**(我们的 `w=16` 实测)和**推理算力 ×2**(每步两次前向)——催生了一整片仍在研究的方向。截至 2024,没有一个公认的通用替代:

- **Dynamic thresholding(动态截断,Imagen 2022)**:每步把 `x_0_hat` 的极端值按分位数裁回合理范围,强行把样本拽回流形。是个有效的补丁,但本质是「事后救火」,没解决外推方向本身错的问题。
- **CFG++(2024)**:把外推重新参数化,让大 w 下的轨迹更贴流形,缓解过饱和。改进明确,但仍是在「线性外推」框架内打补丁。
- **自适应 / 区间 guidance**:不同噪声水平用不同的 `w`(高噪声段弱、低噪声段强),甚至只在中间一段时间步开 guidance。说明「单一 w 全程恒定」本身就是次优——但最优调度随模型和任务变,没有通用公式。
- **去掉 CFG**:guidance distillation(把「两次前向的外推结果」蒸馏进一个网络的单次前向,推理省一半),以及直接训条件模型 + 别的手段拿听话度。能省算力,但通常要牺牲一点可控性或质量。

这些方向都在抢同一个目标:**既听话又不失真,最好还便宜**。连「CFG scale 到底该全程恒定还是分段」这种基础问题都没有定论——你刚才看到连这个 toy demo 的作者都预期错了 `w=16` 的行为。**最优 guidance 策略随模型/任务变化,目前无通用解,正在研究。** 这是个你接手就能立刻动手实验的开放题:在我们的 `stage06-cfg.ts` 上,把恒定 `w` 换成「随时间步衰减的 w」,看看能不能在保住高保真度的同时,把 `w=16` 那种流形外崩溃推迟甚至消除。

## 收口:condition 不只是 label

本章用「类别 0/1/2」当条件,因为它最容易可视化和量化。但 CFG 的外推公式对**条件是什么**毫不关心——把 `ε_cond` 换成「以文本 embedding 为条件的预测」,同一套 `ε_uncond + w·(ε_cond − ε_uncond)` 原封不动,就是 Stable Diffusion 的 guidance。这就是为什么 CFG 是文生图「prompt 听话程度」滑块的底层物理。

第 07 章 latent diffusion 会把整套扩散搬到一个压缩后的 latent 空间(先把数据编码成低维表示,在那里跑 DDPM,再解码)。CFG 在那里**一字不改地复用**——它作用在去噪网络的 `ε` 输出上,不关心 `x` 是像素还是 latent。我们这一章量到的权衡曲线和流形外崩溃,会原样出现在 latent 空间里。带着这条曲线进下一章。
