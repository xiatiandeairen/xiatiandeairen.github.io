## 同一张网,换个采样公式就能加速——这是 diffusion 最值钱的解耦

把第 04 章训好的去噪网络拿过来,**一个权重不动**,只把采样的更新公式从 DDPM 换成 DDIM,采样步数就能从上千步压到几十步。这不是两个模型,是同一个 ε-predictor(预测噪声的网络)上两套反向积分法。理解这一点,你就理解了 diffusion 工程里最反复被问的那个面试题——「DDPM 和 DDIM 要不要分别训练?」——答案是不用,而这个「不用」正是 DDIM(Song et al. 2021)最大的实用价值。

为什么值钱:训练一次几小时到几天,采样却是你上线后每次出图都要付的钱。如果加速采样要重训,那加速本身就太贵了。DDIM 把训练和采样彻底解耦——训一次,采样策略随便换、随时换,甚至同一个 checkpoint 你可以给批量离线任务用 250 步求质量、给交互式 demo 用 20 步求速度。第 00 章我留过一句伏笔:「diffusion 是用采样时间换质量的模型」。本章就是把那句话变成一条你能盯着看的曲线。

本章对应配套仓里的 `examples/diffusion-from-scratch/src/stage05-samplers.ts`,它做四件事:inline 训一个小 ε-predictor(seed 1337,确定性,短训到足够学出 two-moons 形状)、跑 DDPM/DDIM 步数扫描、验 DDIM 确定性、做噪声插值,最后用 S=2 触发崩溃。**下面所有数字都来自这个文件的真实运行,我连一个反直觉的结果都没替它圆场。**

> 一句诚实话先放这:这是 2D toy、几万参数、1500 步短训。绝对 Chamfer 偏乐观,绝对步数也偏少(真实图像扩散同等质量常需 50–250 步)。能迁移的是**机制**(DDPM 注随机噪声、DDIM 是确定性 ODE)和**曲线形状**(存在一个步数膝点,之下质量崩)。绝对值别拿去对标生产。

## 先看 DDPM:为什么它「天生」要走完上千步

DDPM 的反向是一条**马尔可夫链**(每步只依赖上一步、且每步注入新随机噪声)。配套仓 `sampleDDPM` 的核心更新,逐行对应 Ho et al. 的后验公式:

```ts
// ε̂ = net(x_t, t)
const eps = predictNoise(net, x, t);
// x̂_0 = clip( (x_t - √(1-ᾱ_t)·ε̂) / √ᾱ_t )           先估「干净点」并夹回数据支撑
const x0Hat = clipX0((x.data[k] - sqrtOneMinusAbT * eps.data[k]) / sqrtAbT);
// μ = coefX0·x̂_0 + coefXt·x_t                          x̂_0 形式的后验均值
const mean = tPrev < 0 ? x0Hat : coefX0 * x0Hat + coefXt * x.data[k];
// x_{tPrev} = μ + σ·z,  z~N(0,I),  σ = √β_eff
newBuf[k] = mean + sigma * (sigma > 0 ? rng.gaussian() : 0);
```

这里 `sampleDDPM` 没用课本最常见的 `μ=(1/√α)(x−β/√(1−ᾱ)·ε̂)` 那一种写法,而是先显式重建「干净点」`x̂_0` 再代入后验均值——两种写法在 `x̂_0` 不夹时数学等价,但这种形式能让 DDPM 和 DDIM **用同一个 `clipX0` 夹子**,把对比隔离到「更新规则」本身、而非夹不夹的差异(这一步是本章所有数字成立的关键,后面专门讲)。

注意最后一行那个 `sigma * z`:**每一步都往结果里掺一勺新的高斯噪声**。这是 DDPM 的定义性特征,也是它慢的根因。后验均值 μ 只是「最可能的去噪方向」,真实的 `x_{tPrev}` 还要叠一个方差 σ 的随机扰动——这是为了让反向链真的从数据分布里采样,而不是塌到单一众数(mode)。代价是:这勺噪声要靠后续**很多步**才能被均值项一点点抹平。步数砍到几十,噪声还没来得及平均掉,你拿到的就是「去噪一半还带毛刺」的样本——这正是高 T 图像扩散上 DDPM 少步退化显著的根因(本章 toy 因为链短把它 under-state 了)。

代码里有个工程细节值得停一下。DDPM 本来只定义在全链上,但为了和 DDIM 在 NFE(number of function evaluations,网络前向次数——这是诚实的成本轴,不是 wall-clock)上做苹果对苹果的比较,`sampleDDPM` 也接受 DDIM 的「respaced」稀疏步表,用等效 α 跨步跳:

```ts
// α_eff = ᾱ_t / ᾱ_{tPrev}, β_eff = 1 - α_eff —— 让 respaced 跳步保持自洽
const alphaEff = abT / abPrev;
const betaEff = 1 - alphaEff;
```

相邻步时这恰好退化回课本里的 α_t、β_t。这么写不是为了让 DDPM 变好,恰恰相反——**它让你看清:让 DDPM 跳步,等于让它在更稀的网格上注入同样规模的噪声却没有足够步数去平均,这正是它低步数变差的机制。**

最后一步(t=0)`sigma` 强制设 0:不去掉这勺噪声,你的输出永远是「干净结果 + 一层噪声」,永远不干净。这种 off-by-one 的边界处理,是手写采样器最容易翻车、又最不容易在 ASCII 图上一眼看出的地方。

## DDIM:把马尔可夫链改写成确定性 ODE

> ✦ **专家钩子**。DDIM 的精妙全在这:它把 DDPM 的随机马尔可夫反向,改写成一条**非马尔可夫的确定性过程**,数学上对应 diffusion 的 probability-flow ODE(概率流常微分方程——同一个 marginal 分布对应的无随机性轨迹)。同一个训练目标(预测 ε)既能用 DDPM 的随机 SDE 解,也能用 DDIM 的确定性 ODE 解。这就是训练/采样解耦的数学根:**网络学的是分数场(score),采样只是选一个积分器去走这个场**。

看配套仓 `sampleDDIM` 的核心,和上面 DDPM 对照着读:

```ts
// x̂_0 = (x_t - √(1-ᾱ_t)·ε̂) / √ᾱ_t        先用当前噪声估计「干净点」
const x0Hat = (x.data[k] - sqrtOneMinusAbT * eps.data[k]) / sqrtAbT;
// x_{tPrev} = √ᾱ_{tPrev}·x̂_0 + √(1-ᾱ_{tPrev})·ε̂   再 re-noise 回前一档,无 z
newBuf[k] = sqrtAbPrev * x0Hat + sqrtOneMinusAbPrev * eps.data[k];
```

两个关键差异:

1. **没有 `rng.gaussian()`**。整段没抽一个随机数。给定初始噪声 `x_T`,整条采样路径是一个确定性函数 `x_T → 样本`。这就是 η=0 的 DDIM。
2. **它显式预测 `x̂_0`**(网络眼里此刻的「干净点」),再把它 re-noise 到前一个噪声档位。正因为更新只依赖 `(ᾱ_t, ᾱ_{tPrev})` 这一对常数、不依赖中间档,你可以**任意稀疏地选时间网格**大步跳——这就是 DDIM 能 20 步出图的结构原因。`buildStepIndices(S)` 就在干这件事:从 T 个档里等距挑 S 个(含首尾),倒序成严格递减的跳步轨迹。

确定性带来两个白送的副产品,后面两节我用真实运行证给你看:**字节级可复现**,和**latent 插值**(两个噪声种子之间平滑过渡)。这两个都只在「采样是确定性映射」时才成立——DDPM 因为每步注噪,做不到。

## 权衡曲线:NFE 1000→10,质量怎么衰减(以及一个反直觉结果)

这是本章的核心产出。配置:`T=200, cosine schedule, two-moons N=400, seed=1337`。先报训练真实输出:

```
ᾱ_0=9.997e-1  ᾱ_{T-1}=6.072e-8 (≈0 -> x_T≈N(0,I))
训练 1500 steps: loss 0.9756 -> 0.4750 (2.1x 下降)
```

`ᾱ_{T-1}≈6e-8` 这个数很重要(第 01 章讲过):它说明 cosine schedule 把信号保留因子压到了几乎 0,所以从 `N(0,I)` 起步采样这个前提是成立的——如果 ᾱ_T 还残留信号,采样器从纯噪声起步时会撞到它训练时从没见过的分布,样本必崩。这里用 cosine 而非 linear,正是因为小 T 下 linear 会在 t=T 留残差。

现在是步数扫描的真实 Chamfer(Chamfer 距离——生成点云到真实点云的双向最近邻平均距离,越低越好;反向项惩罚「覆盖不全/mode collapse」):

```
权衡曲线 (Chamfer vs S):
   S     DDPM      DDIM    DDIM/best  DDPM/best
 1000  0.2314  0.3471     1.60x      1.13x
  100  0.2056  0.2581     1.19x      1.00x
   50  0.2286  0.2348     1.08x      1.11x
   20  0.2272  0.2176     1.00x      1.10x
   10  0.2480  0.2205     1.01x      1.21x
```

**这张表的头条是 DDIM 那一列从上往下走的方向:它越跳越好,直到 S=20 触底(0.2176),正好是它自己的最优。** 也就是说——

```
断言验证 @S=20: DDIM 质量比 = 1.00x(接近 1 = 接近自身最优), NFE 20 vs 全链 200 (省 10x 算力)。
  -> DDIM 在 S=20 仍接近全链质量(Chamfer 0.2176 vs 全链 0.3471),用 ~1/10 步数换到近乎不变的质量 [已数值验证]
```

把全链 200 步压到 20 步,DDIM 的质量不但没掉,反而比走完全链(0.3471)还略好。这就是 DDIM 的招牌结论被这个 toy 如实复现:**确定性 ODE 积分器在稀疏网格上跳几十步,就能把质量逼到全链水平,NFE 省一个数量级。** 注意 DDIM 的全链(S=1000)反而是它最差的一行(1.60x)——这不矛盾:全链时每一步的 ᾱ 间距极小,re-noise 项里那点积分残差被乘了几百次慢慢累积;稀疏跳步反而让每步的 x̂_0 预测主导、残差累积更少。**这是 DDIM 一个容易被忽略的细节:对 ODE 积分器,「步数多」不等于「更准」,它有一个甜点区。**

**但这里有个常被博客省略的前置条件,我必须讲透,否则你拿去对标生产会被打脸。** 标准叙事是「低步数 DDIM 优于 DDPM」,潜台词是「DDPM 少步会崩」。看这张表的 DDPM 那一列:从 S=1000 到 S=10,它几乎纹丝不动(0.23 上下浮动),压根没崩。代码里的诚实标注点破了原因:

```
-> 对照 DDPM @S=20 = 0.2272 (其全链 = 0.2314, 比 1.10x): 短链 T=200 上 DDPM 注入的方差仍能在
   20 步内被平均掉,故未明显劣化(诚实标注: 真实高 T 图像扩散上 DDPM 少步退化会显著得多)。
```

为什么?DDPM 慢的根因是每步注入的那勺随机噪声 σ·z 要靠很多后续步去平均。但这个 toy 的链只有 T=200,而且数据是 2D——respaced 到 20 步时,跨步跳的等效 β 仍然小到让方差在 20 步内基本被均值项抹平。**真实图像扩散的链是 T=1000、数据是几十万维,同样压到 20 步,DDPM 的噪声远远来不及平均,样本会明显带毛刺**——那才是「DDIM 低步数完胜 DDPM」的战场。这个 200 步的 toy under-state 了 DDPM 的少步退化。

所以这张表的可迁移结论分两层:

- **稳的结论(机制级,任何规模都成立)**:DDIM 用 ~1/10 的 NFE 守住质量,这是确定性 ODE 没有随机噪声要平均的结构性红利。
- **不稳的部分(规模相关,toy 偏乐观)**:「DDIM 在 20 步完胜 DDPM」的对比强度,取决于链长 T 和数据维度。短链小数据上 DDPM 也不崩,所以这里 DDIM 是「持平略胜」;高 T 高维上 DDPM 的少步退化才显著,DDIM 的优势才拉开。**博客只抄「DDIM 更省步」,不告诉你这条对比的强度是 T 和维度的函数——这正是 survey 稿和能跑的实验之间的差距。**

那「步数膝点」呢?它对 DDIM 确实存在,只是在这个收敛良好、流形简单的 toy 上,膝点被推到了 S<20:从 1000 到 20 一路改善,要再往下压到 S=2 才崩(下一节)。**「存在一个步数之下质量崩」是稳的结论;「那个步数是几」依赖网络收敛度、数据流形曲率和 solver 阶数。**

## DDIM 确定性:不是「看起来一样」,是字节级一样

确定性这个词容易被当口号。配套仓直接做了字节比较来证明,不靠肉眼看散点:

```
DDIM 确定性验证 (同噪声两次采样应字节级一致):
  两次 DDIM 采样字节级一致: PASS (确定性成立)
  (对照: DDPM 注入随机 z,同种子不同 rng 调用顺序即不一致 — 这是它随机的代价)
```

实现上靠 `bytesEqual` 逐元素比 `Float64Array`——没有 epsilon 容差,要求**完全相同的 IEEE-754 位**。能 PASS 是因为 DDIM 全程零随机数:同一个 `x_T` 喂进去,每步 `x0Hat`/re-noise 都是纯算术,顺序固定,结果必然逐位重现。

这个性质在工程上不是花活:它意味着 DDIM 采样可缓存、可做回归测试(「改了 schedule 代码,样本变了吗?」字节 diff 一下就知道)、可在分布式里复现同一张图。DDPM 想复现得连同 rng 的**调用顺序**一起固定,任何并行化打乱抽样顺序就崩——这是随机性的隐性税。

## Latent 插值:确定性映射的免费副产品

因为 `x_T → 样本` 是确定性映射,你可以在两个种子噪声之间线性插值,对每个插值点采样,得到一条**平滑的样本轨迹**。这就是图像 diffusion 里「两张脸之间渐变」的 2D 版本。配套仓在 `α∈{0.0, 0.5, 1.0}` 上各跑了一次真实 DDIM 采样,α=0.5 的散点是 α=0 与 α=1 两端形态的中间过渡(三张完整 ASCII 散点见 `npx tsx` 运行输出),代码末尾这句对插值结果的断言:

```
-> 插值 α 从 0 到 1,样本平滑地从一个噪声决定的样本族过渡到另一个(确定性映射的体现)。
```

关键不在「散点长什么样」,而在**为什么能平滑过渡**:DDIM 把每个 `x_T` 唯一映到一个样本,α 连续变化时 `x_T` 连续变化,样本也跟着连续变化。DDPM 做不到——它每步注随机噪声,同一个 `x_T` 两次采样都不一样,谈不上「`x_T` 决定样本」,自然没有可插值的潜空间。

代码里有个**我特意留的诚实小坑**,你照搬到生产前必须知道:

```ts
// Spherical-ish blend kept linear for clarity; 对单位方差噪声,a=0.5 时范数会略缩
for (let i = 0; i < blended.length; i++) blended[i] = (1 - a) * noiseA[i] + a * noiseB[i];
```

线性插值两个 `N(0,I)` 噪声,中点 `0.5·A+0.5·B` 的范数期望会**缩小**(两个独立单位向量的均值范数约 0.71 倍),于是 α=0.5 处的 `x_T` 偏离了「半径 √D 的高斯壳层」——网络训练时从没见过这么短的噪声,中间帧质量会比两端差。生产里正确做法是 **slerp(球面线性插值)**,沿高斯壳层走、保范数。这里为讲清楚保留了 lerp,但代码注释明确标了缩范数问题。**手写 diffusion 插值翻车,九成是栽在「lerp 高斯噪声破坏了范数」这一步上**,而不是采样器本身。

## 失败模式:DDIM 压到 2 步,ODE 离散化误差爆炸

把 DDIM 步数压到极端的 S=2,触发 ODE 离散化误差崩溃。真实输出:

```
失败模式: DDIM 步数压到 S=2 (ODE 离散化误差过大):
  steps=[199,0]  NFE=2  Chamfer=5.0050
  (对照 DDIM S=20 Chamfer=0.2176; S=2 暴涨 23.0x)
```

`steps=[199,0]` 意思是:直接从 t=199(纯噪声)一步预测 `x̂_0`,再一步落地。Chamfer 从 S=20 的 0.2176 暴涨到 **5.0050(23.0x)**——点云不是「略糊」,是直接飞出数据范围、退化成几个稀疏离散簇,两条弯月一条都没盖住。机制清楚:

```
-> 2 步无法积分出双月曲线,点云退化成稀疏离散簇、覆盖不住两个弯 -> Chamfer 暴涨。
   这是 ODE 离散化误差:步太大,每步的 x̂_0 预测被外推到远处,样本质量崩。
```

把 ODE 求解器类比成开车:DDIM 每步是「看一眼当前方向(ε̂),沿直线开一段」。步子越大,直线越偏离真实的弯曲轨迹。two-moons 的反向轨迹是弯的,你用 2 段直线去拟合两条弯月,几何上不可能——`x̂_0` 在第一步就被外推到一个离任何真实点都很远的位置,Chamfer 自然炸。

这恰好划出了上一节那个「膝点」:S=20 时 DDIM 处于甜点、0.2176,S=2 时离散化误差压垮一切、5.0050。膝点就在这两者之间。**这也点明 DDIM 的命门——误差随大步确定性累积,没有随机噪声帮你「搅匀」。在这个 toy 上网络训得够好、流形够简单,所以膝点很低(S<20 都稳);换成训不透的网络或更弯的流形,膝点会大幅上移,这就是为什么真实图像扩散同等质量常需 50–250 步、还要配高阶 solver。**

## ⚡ 前沿:DDIM 的 50 步还是嫌慢——「逼近 1 步且不掉质量」是当前圣杯

DDIM 把上千步压到几十步,但几十步在交互场景(实时生成、端侧)还是太慢。如何逼近 1–4 步出图,是当前 diffusion 加速最热的开放问题,**目前没有免费午餐**:

- **一致性模型(Consistency Models,Song 2023)**:不学「逐步去噪」,直接训练一个映射「ODE 轨迹上任意噪声点 → 同一个 x_0」,1–4 步出图。代价是训练更难(要么从头训、要么从预训练 diffusion 蒸馏),且单步质量仍逊于多步。
- **rectified flow / flow matching**:把 probability-flow 的轨迹**拉直**——轨迹越直,ODE 越好积分,越少步数就够。本章 two-moons 的弯曲轨迹之所以难,正是因为路径不直;flow matching 试图从训练阶段就让路径接近直线。
- **蒸馏(progressive / adversarial distillation)**:把一个多步「老师」压成少步「学生」,渐进式每轮把步数减半,或用对抗损失逼学生单步追上老师。

共同的硬约束:**步数越少,越吃训练技巧和数据质量。** 本章的实验已经给了一个 toy 级别的预演——网络只训 1500 步,DDIM 大步跳就崩;真要 1 步出图,网络必须把整条 ODE 轨迹「记进」一次前向里,这对训练的要求远高于普通 diffusion。「逼近 1 步且不掉质量」目前**无通用解、正在被一致性模型/flow matching/蒸馏三条路线同时攻**,且没有一条是把现成 checkpoint 一插就提速——都要额外训练或蒸馏。这也是为什么本章开头那个解耦(训一次、采样随便换)在 1 步加速这件事上**部分失效**:DDPM↔DDIM 之间确实零成本切换,但要再快一个数量级,你又得回到训练里去。

## 小结:这条曲线该怎么用,以及交给第 06 章什么

本章没重训一个权重,只换采样公式,就把 diffusion 的核心取舍量化成了一张表。带走三件事:

1. **DDPM/DDIM 共享同一个 ε-predictor,采样策略可热切换**——这是面试标准答案,也是工程上你该利用的解耦。
2. **「DDIM 低步数完胜 DDPM」有前置条件**(高 T、高维、网络收敛)。本章短链 toy 上这些条件不全,DDPM 少步没崩,DDIM 只是「持平略胜」——别把它当无条件加速开关。注意一个反直觉细节:DDIM 反而是全链 S=1000 那行最差(0.3471),稀疏跳步到 S=20 才触底(0.2176),对 ODE 积分器「步数多 ≠ 更准」。可迁移的真结论是:采样质量是「采样器 × 网络收敛度 × 流形曲率 × 步数 × 数据维度」的联合函数。
3. **确定性是 DDIM 的结构性红利**:字节级可复现 + latent 插值,但插值要用 slerp 不是 lerp。

第 06 章 classifier-free guidance 同样**只在采样阶段动手脚**——它在 `predictNoise` 的输出上做条件/无条件两次前向再线性组合。本章 `sampleDDIM`/`sampleDDPM` 的步进循环和 `predictNoise` 接口,就是 guidance 要插进去的那个钩子:guidance 不改采样器的步进逻辑,只改每步喂给步进的那个 ε̂。把采样器接口理清楚,第 06 章就只是「换一个 ε̂ 的来源」。
