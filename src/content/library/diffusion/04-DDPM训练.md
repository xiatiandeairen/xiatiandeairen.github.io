第 03 章的结论是一个等式:score(数据对数密度的梯度,大白话「往高密度方向的箭头」)和加进去的噪声 ε,在任一噪声水平上只差一个 −1/√(1−ᾱ_t) 的标量因子。当时这看着像个数学小技巧。本章要兑现它的工程价值:既然 score 等价于 ε,那训练 score 模型就等价于训练一个「猜噪声的回归器」——而回归器的损失,就是你大一学过的均方误差(MSE)。

把这句话写成代码,DDPM 的整个训练循环只有六行实质逻辑(`examples/diffusion-from-scratch/src/stage04-ddpm-train.ts`):

```ts
const x0 = /* 一个 batch 的真实数据点 */;
const { xt, eps } = addNoise(x0, timesteps, sched, rng); // 闭式加噪,顺手记下 ε
const pred = predictEps(d, xt, timesteps);               // 网络猜这个 ε
const diff = pred.sub(eps);
const loss = diff.mul(diff).mean();                       // ‖ε − ε_θ‖² 一行 MSE
loss.backward(); opt.step();
```

没有判别器,没有变分下界,没有 Langevin 链跑在训练里。就是回归。这是 diffusion「训练稳」名声的全部来源,本章会把「为什么这么稳」拆到机制层,然后真训一个网络验证它,最后用一个对照实验证明这个配方哪一环抽掉就塌。

## 训练目标:闭式加噪「免费」给你回归标签

监督学习要标签。GAN 没有显式标签,靠判别器现造一个移动的目标;score matching 的标签是 ∇log q,你根本拿不到真值。DDPM 的聪明之处在于:它的标签是你自己刚刚画出来的那个噪声。

回到第 01 章的前向闭式公式(forward process,大白话「一步把干净数据加噪到任意浑浊度」):

```
x_t = √ᾱ_t · x_0 + √(1−ᾱ_t) · ε,   ε ~ N(0, I)
```

注意这里的 ε 是**我们自己采的**。代码里 `addNoise` 一边算 x_t 一边把这个 ε 存下来当回归目标(`stage04-ddpm-train.ts` 第 97–119 行):

```ts
function addNoise(x0, timesteps, sched, rng) {
  // ...
  const e = rng.gaussian();
  eps[r * dim + c] = e;                       // 记下来,这就是标签
  xt[r * dim + c] = sa * x0.data[...] + soma * e;
  // ...
  return { xt, eps };  // 同时返回浑浊数据和制造它的噪声
}
```

这就是「免费标签」的来源:前向过程是我们设计的、有闭式解的,所以对任意 (x_0, t),我们都知道精确的 ε。网络的任务就是反推——给它 x_t 和 t,猜出当初加进去的 ε 是什么。损失天然是 MSE,因为 ε 是连续向量。

为什么这能逼近 score?第 03 章已证:最优的 ε_θ 满足 ε_θ(x_t,t) = −√(1−ᾱ_t) · ∇log q(x_t)。所以训练好的 ε 网络,缩放一下就是 score 网络。你训的是噪声预测器,白拿一个 score 估计器。

## ✦ DDPM 真正的贡献是「简化」,不是公式

如果你只记一件事:**Ho et al. 2020 最大的工程贡献不是推出某个公式,而是把理论推出来的复杂东西删掉,发现删完反而更好。** 这是「理论给方向、工程做减法」的教科书案例,值得拆开讲,因为它直接影响你调参时的默认选择。

**减法一:预测 ε,不预测 x_0,也不预测均值。** 同一个网络可以有三种等价的输出参数化(parameterization,大白话「让网络吐哪个量」):直接吐去噪后的 x_0、吐反向高斯的均值 μ、或吐噪声 ε。三者数学上可互推,但训练表现差很远。原因在标签的尺度稳定性:ε 在每个噪声水平都是单位方差的 N(0,I),回归目标的量纲恒定,损失天然良态(well-conditioned),不需要按 t 做逐步加权。代码注释把这点说得很直(第 11–15 行):

```
// WHY ε-prediction (not x_0-prediction, not score): predicting ε is the Ho et al. 2020
//   parameterization. ε is unit-variance N(0,I) at every t, so the regression target has a
//   stable scale across all noise levels ... (Predicting x_0 directly has target variance that
//   collapses as t→T; predicting the score ∇log q needs the 1/√(1−ᾱ) factor folded in.)
```

预测 x_0 的麻烦在 t→T 时:那时 x_t 几乎是纯噪声,x_0 的信息基本被抹平,但你还要求网络精确吐出 x_0,等于让它在一片噪声里硬猜原图,标签方差大到训练不稳。预测 ε 反过来:t→T 时 ε 就是 x_t 本身的主成分,好猜;t→0 时 ε 难猜但本来损失贡献也小。ε 是三者里量纲最干净的中间量。

**减法二:损失权重直接设成 1。** 把变分下界(ELBO)严格展开,每个 t 的损失项前面有一个跟信噪比(SNR)挂钩的复杂权重。Ho et al. 试了之后发现:**把这个理论权重全扔掉、所有 t 一视同仁,效果反而更好。** 这就是著名的「simplified objective」L_simple。我们的代码就是这个简化版——`train` 里对一个 batch 直接 `.mean()`,没有任何逐 t 加权(第 144–147 行):

```ts
const pred = predictEps(d, xt, timesteps);
const diff = pred.sub(eps);
const loss = diff.mul(diff).mean();  // 所有 t 平权,权重恒为 1
```

理论权重会过度强调高噪声步(那里 SNR 低、权重大),但高噪声步本身信息少、对最终画质贡献有限;平权等于隐式地把训练算力挪给中低噪声步,那里才决定细节。这不是「偷懒」,是一个被实证打赢的设计选择。

**面试高频题:为什么 DDPM 训练比 GAN 稳?** 答案落在监督信号的性质上。GAN 是两个网络互相追逐的不动点博弈(生成器和判别器各自移动对方的目标),没有静止的损失地形,所以会模式崩溃、训练震荡、需要各种 trick 续命。DDPM 没有第二个网络:每个 (x_0, t, ε) 三元组都是一条**独立、静止**的监督信号,损失是固定的 MSE,标签是采样时就钉死的常数。本质上 diffusion 把生成问题降维成了一堆独立的监督回归子任务——而监督回归是深度学习里最稳的那类问题。这就是为什么你几乎不用调 DDPM 的训练稳定性,只需要调采样质量(第 05 章)和条件强度(第 06 章)。

## 网络结构:为什么必须告诉它「现在第几步」

去噪网络要吃两个输入:浑浊数据 x_t 和当前步 t。前者显然,后者是初学者最容易漏的——很多人第一反应是「网络看 x_t 的浑浊程度不就知道几步了吗」。不行,而且这事关本章的失败模式。

我们的网络很小,一个带时间嵌入的 MLP(`stage04-ddpm-train.ts` 第 67–90 行):

```ts
function makeDenoiser(rng, useTime) {
  const emb = new SinusoidalEmbedding(EMB_DIM);          // t → 16 维正弦向量
  const sizes = [2 + EMB_DIM, ...HIDDEN, 2];             // 输入拼接后 18 维
  const mlp = new MLP(sizes, "silu", rng);
  return { mlp, emb, useTime };
}

function predictEps(d, x, timesteps) {
  const timeBlock = d.useTime
    ? d.emb.forward(timesteps)                            // 正常:正弦时间嵌入
    : new Tensor(new Float64Array(batch * EMB_DIM), ...); // 失败模式:全零
  const input = Tensor.concatCols([x, timeBlock]);       // [x(2) ⊕ t-emb(16)] = 18 维
  return d.mlp.forward(input);
}
```

跑起来打印的结构(实测输出):

```
去噪网络 ε_θ: 输入 [x(2) ⊕ sin-emb(16)] → MLP 18→128→128→2 (silu) → ε(2),共 19202 个参数
```

时间用**正弦嵌入**(sinusoidal embedding,大白话「把整数 t 编成一串不同频率的 sin/cos」)而不是直接喂标量 t,理由跟 Transformer 的位置编码同源:标量 t 在 0..999 量纲太大且单调,网络难以让相邻步产生足够不同的响应;正弦嵌入把 t 摊到多个频率上,既保留相对距离又有丰富的高频区分度。注意它**无参数**——是固定特征映射,所有权重都在 MLP 里,这也是代码把 emb 和 mlp 并排放而不是做成子类的原因(第 56–59 行注释)。

为什么网络必须知道 t?因为**同一个 x_t,在不同 t 下正确的 ε 完全不同**。一个坐标值 0.8 的点,如果它来自 t=10(几乎干净,只掺了一丁点噪声),那 ε 很小;如果来自 t=990(几乎纯噪声),那 ε 几乎就是 0.8 本身。没有 t,网络面对同一个 x_t 不知道该吐大 ε 还是小 ε,只能输出一个跨所有噪声水平的「平均 ε」。这个平均值在反向链每一步都用错幅度去噪,误差会沿 T 步累积——下面的对照实验会把这个塌缩演示出来。

## 真训一个:Chamfer 460 → 0.23

配方和结构都齐了,跑。训练前先量一条基线:用一个**没训练**的随机初始化网络去采样,看它有多差。这一步不是装饰,是为了让「训练有没有用」有个可证伪的对照——代码里它是一条会真的抛异常的断言(第 307–313 行),如果训练链路有 bug(比如忘了 zeroGrad、adjoint 写反),trained 打不过 baseline,程序直接报错而不是输出一张「看着还行」的图。

衡量好坏用 Chamfer 距离(大白话「两堆点云互相找最近邻的平均距离」,越小越像)。先看未训练基线的实测输出:

```

未训练网络的采样 (应是无结构噪声):

  .    : .:. : -   .- --  -::+   :      .
.     ..  : ... .::..+-*:- .
```

一团没结构的散点,Chamfer 460。然后训练 2000 步,实测:


注意 loss 收在 0.24 附近、不是 0。这是对的,不是没训好:ε 是 N(0,I),即便网络完美预测了**可预测的那部分**,每个样本里仍有不可约的随机分量(因为给定 x_t,真实的 ε 本身有条件方差)。MSE 的下限是这个条件方差,不是 0。初学者看到 loss 不归零容易慌,记住:**diffusion 的 loss 绝对值没有意义,有意义的是它降下来并趋稳。** loss 曲线实测确实是降到一条平线:

```

         |      **      *** ** * *** ***    ** * **** ***  *
         |                    *         ***   *      *   ** ******
```

现在用训练好的网络,跑第 02 章的反向链采样。反向链本身没变,只是每步的 ε 从「公式」换成「网络预测」(`sample`,第 163–193 行):

```ts
const eps = predictEps(d, xtTensor, timesteps).data;   // 唯一改动:ε 来自网络
const mean = invSqrtAlpha * (x[idx] - (beta / soma) * eps[idx]);
const z = t > 0 ? rng.gaussian() : 0;                   // 最后一步 t=0 不再加噪
x[idx] = mean + sigma * z;
```

实测结果:


从纯高斯噪声出发,跑完 1000 步反向链,还原出两个弯月——这就是 diffusion 的全部魔法,而训练它只用了一行 MSE。

**诚实标注(这点必须说清,否则就成了我们在批判的那种 survey 稿):** 上面的 0.24、0.23、2010x 这些**绝对值只对这个 2-D toy 有意义,而且偏乐观**——2-D 太简单了,图像空间(几万维)的 loss、Chamfer、收敛步数完全不是这个量级。可以迁移到真实系统的,只有三件**机制事实**:(a) loss 降下来并趋稳;(b) 训练好的 ε 网络能还原数据分布的形状,而未训练 / 缺关键组件的网络不能;(c) 下面要演示的「抽掉时间条件就塌」。代码第 17–25 行的 HONESTY 块和第 316–319 行的运行时打印都在反复强调这一点,这是本书的底线:数字标清适用边界,不让 toy 的乐观值伪装成普适结论。

## 失败模式:抽掉时间嵌入,loss 只升一点,采样却塌成一团

现在做本章最有教学价值的对照实验:重训一个**完全相同**的网络——相同初始化种子、相同训练种子、相同 19202 参数量——唯一的改动是把时间嵌入那一块换成全零(`useTime=false`,见上面 `predictEps`)。这是一个干净的控制变量实验:任何质量差异都只能归因于「网络瞎了,不知道现在第几步」,排除了随机性的干扰(代码第 329–331 行专门为此用了独立但对齐的种子)。

先看 loss,实测:

```
t-blind 训练后 loss(后10步均值)=0.3175 (有时间嵌入时为 0.2441)
```

**loss 只从 0.2441 升到 0.3175,看着没差多少。** 一个只盯 loss 的人会觉得「也还行嘛,稍微差一点」。再看采样:

```
t-blind 采样 300 点 → Chamfer = 0.3379 (有时间嵌入时为 0.2291)
t-blind 网络的采样 (应糊成一团,不成双月):
                               .
                 . ..  .    .  .
     .            ..  .:  :. . :
       .     ..  @:.:  .:.... .:. .  . .
   .      .   :=.     .   . .  :*: .   .
.        . . @  . . ==.:=* .:=.:  :.: .   .   .      .
```

散点糊成中心一团,不成双月。这里的关键矛盾值得盯死:

> **loss 只差一点点(升 30%),Chamfer 却差一截(0.34 vs 0.23,差 1.48x),采样图直接塌。**

为什么 loss 的小差异会被采样放大成质变?因为 loss 是把**所有 1000 个噪声水平平均**后的一个数。t-blind 网络对每个噪声水平都输出那个「跨水平的平均 ε」——这个平均值对每个具体的 t 都只是「稍微错」,所以平均下来的 loss 只升 30%。但反向链不是只走一步,它要走 1000 步,**每一步都用这个稍微错的幅度去噪,误差不会抵消而是沿链累积**,1000 步复利下来就从「稍微错」滚成「彻底塌」。代码的结论打印把这句话钉死了(第 351–353 行):

```
注意 loss 只差一点点、Chamfer 却差一截:这正是关键 —— 平均掉所有噪声水平的 ε
只让训练 loss 略升,但反向链每步都用错误幅度去噪,误差沿 1000 步累积,最终塌成模糊团。
时间条件不是可选项: 网络必须知道当前噪声水平,才能对不同步预测不同的 ε。
```

这个失败模式有普适的方法论价值,远超 diffusion 本身:**训练损失和最终任务质量之间隔着一条多步推理链时,loss 的小退化会被链长放大成大退化。** 你不能只看 loss 选模型,必须跑下游任务量端到端指标。这也解释了第 05 章为什么花大力气研究采样器——同一个训练好的网络,采样链怎么跑(步数、噪声调度、求解器)直接决定你能不能把这个网络的能力兑现出来。

## ⚡ 前沿:预测什么、怎么加权,至今没有统一最优

本章一直在讲 ε-prediction 的好,但要诚实说一句:**ε-prediction 不是终点,「预测什么、损失怎么加权」是 diffusion 调参至今没有收敛的核心战场,目前无通用最优解,仍在活跃研究中。**

几条主线,各有适用场景,没有谁全面碾压:

- **v-prediction**(Salimans & Ho)。让网络预测一个 v = √ᾱ_t·ε − √(1−ᾱ_t)·x_0 的混合量。它在高噪声端比 ε 稳、低噪声端比 x_0 稳,是 ε 和 x_0 的几何插值。在**蒸馏**(distillation,把多步模型压成少步)和高分辨率场景里,v-prediction 往往比纯 ε 更好训。
- **x_0-prediction**。前面说它在 t→T 难训,但在**少步采样**和某些条件生成里反而有优势,因为它直接预测目标、误差不经过 1/√(1−ᾱ) 放大。
- **EDM(Karras et al. 2022)**。这是迄今最系统的一次重整理:它把「预测目标、噪声调度、损失加权、网络预条件(preconditioning)」全部重新参数化成一组可解耦的设计旋钮,并给出在小步数下大幅提质的具体配方。EDM 的贡献不是发明新目标,而是揭示了 ε/x_0/v 都只是同一组旋钮的不同取值,你应该按噪声水平动态选择最优组合,而不是全程锁死一种。

为什么至今没统一?因为最优参数化依赖于**噪声水平、数据模态、采样步数预算、是否蒸馏**这几个轴的交互,不同组合下赢家不同。Ho et al. 的 L_simple 是个极好的、稳的默认起点(本章用的就是它),但当你要榨少步数下的画质、或做蒸馏时,「换个预测目标 + 换个加权」常常是单点收益最大的一刀。这块没有银弹,知道有哪些旋钮、各自什么时候用,就是当前的工程水平。

## 小结:本章给后面留下了什么

把本章压成几句:DDPM 训练 = 闭式加噪拿免费 ε 标签 + 一行 MSE 回归 + 时间条件让网络区分噪声水平。它稳,是因为没有第二个网络在追逐,每个三元组都是独立静止的监督信号。它的工程精髓是 Ho et al. 的两刀减法:预测 ε(量纲最干净)、权重设 1(实证打赢理论)。它的命门是时间条件——抽掉它 loss 只升一点但采样塌,因为误差沿反向链累积。

更重要的是,本章产出了一个**公共资产**:一个训练好的去噪网络 ε_θ。第 05 章会拿它来研究采样器——同一个网络,反向链怎么跑能从 1000 步压到几十步还不掉画质(这正是上面 ⚡ 里 EDM 那条线的工程落地)。第 06 章会给它加条件输入,做 classifier-free guidance(无分类器引导),让它不只采「某个双月」而是采「我指定的那一类」。训练这关过了,后面全是怎么用好这个网络的故事。
