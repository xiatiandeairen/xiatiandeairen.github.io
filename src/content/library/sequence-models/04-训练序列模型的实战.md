第 03 章那个 LSTM cell,我们只在 toy 的「记住第一个字符到序列末尾」分类任务上验过它能跨步搬信息。那是 feasibility 证明,不是真活儿。真活儿是这样的:给模型一段文本,让它逐字预测下一个字符,训完之后让它自己往下写——这就是字符级语言模型(character-level language model,逐字符建模的语言模型)。这一章把第 03 章的 cell 接上 embedding 和 readout,在一段 567 字符的伊索寓言上训 40 个 epoch,然后做两件事:量训练曲线、让它生成文本。重点不是「它能学会」(它能),重点是**它在哪儿坏、为什么这么坏**——而那个坏,正是理解第 05 章注意力为什么碾压 RNN 的入口。

配套代码 `examples/sequence-models-from-scratch/src/stage04-char-lm.ts`,跑 `npx tsx src/stage04-char-lm.ts`。它自带第 03 章的 `LSTMCell`(不 import stage03,免得跑它的 main),复用 `core/data.ts` 的 `charSeq`、`core/metrics.ts` 的 `perplexity`/`timeit`。本章所有数字都是这个文件的真实输出,wall-clock 是 `timeit` 量的,参数量是按 shape 数的,绝对值在 toy 语料上偏乐观——这点我会反复点出,因为可迁移的从来是相对趋势,不是绝对数。

## 任务搭建:窗口、词表、和那个必须先记住的下界

先把数据摆清楚。`charSeq` 把 567 字符的语料切成长 24、步长 24 的窗口(stride=seqLen 表示窗口不重叠),得到 23 个窗口,词表 26 个字符。按尾部 20% 留出验证集:train 19 窗口、val 4 窗口。不 shuffle,所以 val 是语料结尾那段模型没训过的真实文本——这点对后面「val 先降后升」的解读很关键,如果 shuffle 了,val 就只是 train 的同分布抽样,过拟合信号会被冲淡。

```ts
// stage04-char-lm.ts
const SEQ_LEN = 24;
const data = charSeq({ seqLen: SEQ_LEN, stride: SEQ_LEN });
// ...
const nVal = Math.max(1, Math.floor(seqs.length * 0.2));
const trainSeqs = seqs.slice(0, seqs.length - nVal);
const valSeqs = seqs.slice(seqs.length - nVal);
```

模型是 `embed[26x24] -> LSTM(hidden=64) -> readout[64x26]`,真实参数量 25098。在看任何困惑度数字之前,你必须先把**均匀下界**钉进脑子:词表 26 个字符,完全瞎猜(每个字符等概率)的困惑度就是 26。

困惑度(perplexity,模型对下一个字符「有多犹豫」的指标)= `exp(平均交叉熵)`。直觉:困惑度 = N 意味着模型平均把概率摊在 N 个候选上。困惑度 1 = 每步都笃定预测对;困惑度 26 = 跟掷 26 面骰子一样。

为什么先讲下界?因为没有下界,「困惑度 6.70」是个无意义的数。在一篇 AI 批量生成的 survey 里你会读到「模型达到了 6.70 的困惑度」然后没了——读者根本不知道这是好是坏。6.70 的意义全在于它对比的是 26:模型把犹豫从 26 个候选压到了不到 7 个。这就是为什么训练前先打印一行 `训练前 train 困惑度: 25.97 (接近均匀下界 26)`——25.97 ≈ 26 是模型「尚未学习」的铁证,初始权重就是在瞎猜。

## teacher forcing:训练时永远喂正确答案

现在到这章的核心机制。训练时,我们怎么给模型喂序列?答案是 **teacher forcing**(教师强制:每一步都喂真实的历史字符,而不是模型自己上一步预测的字符)。

看 `forwardTeacherForced`:它拿到整条真实 id 序列,逐步把**真实**字符 embed 进去,cell 走一步,readout 出 logits,收集每步 logits 拼成 `[T, vocab]`,一次性算交叉熵。

```ts
// stage04-char-lm.ts
forwardTeacherForced(ids: number[]): { logits: Tensor; h: Tensor; c: Tensor } {
  let h = Tensor.zeros([1, this.hidden]);
  let c = Tensor.zeros([1, this.hidden]);
  const rows: Tensor[] = [];
  for (let t = 0; t < ids.length; t++) {
    const x = this.embed.forward([ids[t]]); // 喂的是真实 ids[t]
    const out = this.cell.step(x, h, c);
    h = out.h; c = out.c;
    rows.push(this.readout.forward(h));
  }
  return { logits: Tensor.concat(rows, 0), h, c };
}
```

注意第 t 步喂的是 `ids[t]`——真实字符,不管模型上一步预测对没对。这是个深思熟虑的设计,不是偷懒:

1. **可并行 / 稳定**。每步的输入都是确定的真实序列,所以整条序列的 logits 可以一口气算完(这里因为 batch=1 还是串行,但在真正的 GPU 实现里 teacher forcing 让你能把整个时间维度并行展开)。如果训练时喂模型自己的预测,每步输入依赖上一步随机采样结果,梯度路径会变成在自己的噪声上反传,训练几乎无法收敛。
2. **梯度信号干净**。损失直接对比「给定正确历史,模型预测的下一字符」和真实下一字符,误差不会被前面步骤的错误污染。

训练循环就是标准三件套:teacher-forced 前向、`backward`、`clipGradNorm(params, 5)`(梯度裁剪,防止某步梯度爆炸把权重冲飞)、Adam `step`。

```ts
// stage04-char-lm.ts
const { logits } = model.forwardTeacherForced(s.x);
const loss = logits.crossEntropy(s.y);
loss.backward();
lastGradNorm = clipGradNorm(params, 5);
opt.step();
```

实测数字:单 epoch 真实耗时 86.5 ms(wall-clock,warmup 丢弃),末次裁剪前梯度范数 0.153,远低于裁剪阈值 5——说明训练后期早已平稳,裁剪在这个 toy 尺度上几乎没触发,它是为训练早期或更大模型准备的安全带。

train 困惑度从 17.1 一路掉到 1.06。1.06 意味着模型在训练集上几乎每个字符都预测对了。听起来很棒——直到你看 val。

## val 困惑度先降后升:小语料上的过拟合是教学素材不是 bug

val 困惑度的轨迹是这一节的全部价值:

```
val 困惑度随 epoch (注意先降后升 = 过拟合):
  ▇▆▅▃▂▁▁▁▁▁▁▁▁▁▂▂▂▂▂▃▃▄▄▄▅▄▅▄▆▄▆▅▇▆▇█████   18.1 -> 最低 6.70 (epoch 7) -> 20.38
```

读这条线:18.1 起步,epoch 7 触底到 **6.70**,然后一路反弹到 20.38。最终态是 `train 困惑度=1.06 / val 困惑度=20.38`。

逐层拆开它在说什么:

- **6.70 远低于均匀下界 26**(代码里用 `bestVal < uniformPpl * 0.5` 判定):模型在它**没见过**的语料尾部把犹豫从 26 压到了 6.70。这是「真学到了字符分布、不是 noise」的硬证据。N=1 的 toy 任务我不敢说「模型理解了语言」,但「val 远低于均匀下界」这一条是站得住的——它是相对下界的对比,不是绝对断言。
- **train 降到 1.06 而 val 反弹到 20.38**:经典过拟合。25098 个参数面对 567 字符的语料,容量远超数据,模型不再学规律而是开始**背训练集**。背得越熟(train→1.06),对没背过的尾部越没辙(val→20.38,几乎退回均匀瞎猜)。
- **epoch 7 是 early-stopping 点**:最好的模型是 epoch 7 那个,不是最后那个。训练曲线告诉你的不只是「学了多少」,还有「该在哪儿停手」。

这里要诚实划线:**绝对的过拟合幅度是 toy 语料造的**。567 字符上任何有点容量的模型都会背下来。真实大语料(几 GB 文本)下,train 和 val 会**一起**持续下降,不会这么早张开。可迁移的结论是那个**形状**:teacher forcing 训练下,train 和 val 早期同步下降、后期 train 持续下探而 val 触底反弹——这个 gap 张开的过程,就是「模型从学规律切换到背数据」的可观测信号。任何序列模型训练你都该盯着这条 gap。

代码刻意每个 epoch 都记录 train+val 困惑度,就是怕采样太粗错过 epoch 7 那个 val 谷底:

```ts
// stage04-char-lm.ts
// 注释原文:on a tiny corpus val ppl dips early then rises as the model
// overfits — we must not miss the dip by sampling too coarsely
const recordCurve = (epoch: number): void => {
  const tp = evalPerplexity(model, trainSeqs);
  const vp = evalPerplexity(model, valSeqs);
  // ... track bestVal / bestValEpoch
};
```

## 自回归采样:推理时,模型只能喂自己

训练完,生成文本走的是完全不同的路径——**自回归采样**(autoregressive sampling:逐步预测一个字符,再把它当输入预测下一个)。这里有个根本的不对称:训练时 `forwardTeacherForced` 每步喂真实字符,推理时模型没有真实字符可喂,只能把**自己刚预测出的字符**喂回去。

看 `generate`:先用 seed(prompt)暖一下状态(这段还是 teacher-forced,因为 prompt 是已知的),然后进入循环——采样一个字符、push、把它 embed 回去喂给 cell、再出下一个 logits。

```ts
// stage04-char-lm.ts
for (let t = 0; t < n; t++) {
  const next = sampleFromLogits(cur.data, temperature, rng);
  generated.push(next);
  const x = model.embed.forward([next]); // 喂回模型自己的预测
  const out = model.cell.step(x, h, c);
  h = out.h; c = out.c;
  cur = model.readout.forward(h);
}
```

`喂回模型自己的预测` 这一行,就是整章 ✦ 钩子的物理位置。下一节会看到它的代价。

先说 temperature(温度:采样前对 logits 缩放的旋钮)。`sampleFromLogits` 把 logits 除以温度再做 softmax:温度 < 1 让分布更尖(更贪心、更确定),温度 > 1 让分布更平(更随机),温度 → 0 退化成 argmax。

```ts
// stage04-char-lm.ts
const e = Math.exp((logitsRow[i] - max) / T); // 除以温度后再 exp
```

实测两个温度(用固定 seed 保证 bit 级可复现):

```
temperature=0.5  unique字符比=0.20  熵=3.63bit
  "the north wind and the sud he closely did cloaely did closely did closely did cloade"
temperature=1.0  unique字符比=0.23  熵=3.80bit
  "the north wind and the sut the more closely did closely did closely did cloade cloak"
```

两件事值得看:(1) 文本从乱码长出了**局部英文结构**——`the north wind`、`closely`、`cloak` 都是语料里的真词,模型确实学到了字符级拼写规律。(2) 低温 0.5(unique 0.20、熵 3.63bit)比高温 1.0(unique 0.23、熵 3.80bit)更重复、更确定——温度旋钮如预期工作。但注意,**两个温度都开始打转**:`closely did closely did closely did`。这不是温度的锅,温度只控制随机性强弱;打转的根因在下一节。

## 失败模式:把生成拉长,亲眼看它塌缩

现在做这章最该做的实验:用近贪心采样(温度 0.01,把采样随机性几乎归零,好让我们隔离出**纯粹的 exposure bias**而不是采样噪声)连续生成 400 字符——训练窗口才 24 字符,我们要看生成长度远超训练窗口时质量怎么变。代码按 80 字符分段,每段量 unique 字符比和字符熵:

```
[  0-80] unique=0.20 熵=3.60bit  "north wind and the sud he closely did closely did closely did closely did closel"
[ 80-160] unique=0.11 熵=3.07bit  "y did closely did closely did closely did closely did closely did closely did cl"
[160-240] unique=0.11 熵=3.10bit  "osely did closely did closely did closely did closely did closely did closely di"
[320-400] unique=0.11 熵=3.07bit  "y did closely did closely did closely did closely did closely did closely did cl"
```

退化曲线:

```
unique字符比 随生成位置: █▁▁▁▁   0.20 -> 0.11   (下降 44%)
字符熵(bit) 随生成位置:   █▁▁▁▁   3.60 -> 3.07
```

首段 unique=0.20,到末段塌到 0.11,下降 44%;熵从 3.60bit 降到 3.07bit——分布塌缩到少数几个字符。再看那个对照实验最狠:**前 24 字符(=训练窗口长度)unique=0.54,远段 unique=0.11**。长度内 0.54、长度外 0.11,这个差就是 exposure bias 的代价,直接量出来了。

为什么会这样?两个原因叠加,缺一不可:

**(1) Exposure bias(曝光偏差:训练只见正确历史、推理要吃自己的错误,二者分布不一致)。** ✦ 这是这章的核心钩子,也是很多人调不好 RNN 生成的真因。teacher forcing 训练时,模型每一步的输入都是**正确历史**——它从未见过「自己上一步出了个错字、现在要在这个错的基础上继续」是什么场景。推理时它必须吃自己的输出。一旦某步走偏(哪怕只错一个字符),模型就被推到了训练时从未到过的状态空间(off the training manifold),它在那里没有任何经验,于是更容易再错,误差像滚雪球一样累积。训练和推理的输入分布根本不是一回事——这是个结构性的训练-推理不一致,不是调学习率能修的。

**(2) 固定宽度状态的记忆衰减。** LSTM 的全部记忆就是那个 hidden state,64 个浮点数(代码里 `HIDDEN = 64`)。无论你让它生成多长,它能携带的上下文就这么多 bit。生成到第 200 个字符时,它早记不清第 1 个字符是什么了,只能退回到最强的短程习惯——而语料里 `closely did` 这种高频片段就是最强的局部 attractor(吸引子:模型反复落回的稳定循环)。于是文本塌缩成 `closely did closely did` 的死循环。

这两个原因为什么必须叠加讲?因为单独任何一个都不足以解释这个具体的塌缩形态。光有 exposure bias,模型会越走越离谱(各种乱);光有有限状态,在 teacher forcing 下也不至于死循环(每步有真实字符把它拉回正轨)。是「误差累积把模型推离 manifold」+「有限状态记不住长上下文、只能抓短程 attractor」一起,才产生了「逐渐塌缩为重复」这个特定的退化曲线。

再划一次诚实线:**绝对的退化速度被 toy 语料夸大了**——567 字符上模型很快就没有新东西可生成。可迁移的是那条**单调下降趋势**:生成长度 >> 训练窗口时,质量单调下降。这个趋势在任何规模的 RNN 生成上都成立,只是大模型把崩溃点推得更远。

## ⚡ 前沿:现代 LLM 仍用 teacher forcing,为什么没崩

这里是这章留给你的开放问题。你可能会想:既然 teacher forcing 有 exposure bias 这个结构性缺陷,现代 LLM 应该早就抛弃它了吧?

恰恰相反——**GPT、Llama、Claude 这些模型,预训练仍然是纯 teacher forcing**(术语叫 next-token prediction,本质就是本章的 `forwardTeacherForced` 放大到万亿 token)。exposure bias 在理论上一天都没消失。那它们为什么能生成几千字不崩、而我们这个 LSTM 200 字符就塌?

业界目前的理解(注意:**这是缓解,不是根治,没有公认的通用解**)有两条腿:

1. **海量数据 + 巨大容量把「off-manifold」的区域填满了。** 我们的模型在 567 字符上一走偏就到了训练时从未见过的状态;而一个在数 TB 文本上训练的模型,它见过的「带错误的、奇怪的、半通顺的」上下文实在太多,所谓「训练 manifold」覆盖面极广,模型走偏后大概率仍落在它见过的某种分布里,自我纠错能力强得多。
2. **Transformer 的全局注意力替换了固定宽度状态。** 这是第 05 章的主题,也是为什么本章是它的前置。LSTM 把整段历史压进 64 维 hidden state,信息瓶颈是死的;注意力让每一步都能**直接回看**前面所有字符,不经过有损压缩。本章失败模式的第二条腿(固定状态记不住长上下文)在注意力那里得到了**第一次结构性缓解**——不是靠更大的状态,而是靠根本不压缩。

但请记住「缓解」二字。exposure bias 的累积本质还在:这就是为什么超长生成仍会跑题、为什么需要 RLHF / DPO 这类后训练去对齐生成行为、为什么 scheduled sampling(训练时按概率混入模型自己的预测)、professor forcing 这类「让训练更像推理」的方法被反复提出又因为破坏训练稳定性而难以大规模落地。**「如何在不牺牲 teacher forcing 训练稳定性的前提下消除训练-推理不一致」,到今天仍是开放问题,没有通用解。** 你在这个 567 字符的 toy 上亲手摸到的塌缩曲线,就是这个问题最干净的形状。

下一章我们拆注意力,你会看到同样这段「记忆衰减」在结构上被绕过——但你也会带着本章的判断力去看:它绕过的是哪条腿,没绕过的又是哪条。
