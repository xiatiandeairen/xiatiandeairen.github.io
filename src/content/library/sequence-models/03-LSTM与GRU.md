第 02 章的梯度尸检给了一个干脆的结论：vanilla RNN 在长依赖上不是「训得不够」，而是「梯度根本到不了」——反向路径每经过一步就乘一个 `(1 - h²) < 1` 的 tanh 导数，乘上二十几步，从答案传回到要记住的那个 token 的梯度已经地板式归零。本章不换优化器、不加层、不调学习率，只动一件事：**把递归的代数结构换掉**。换完之后，同一个 copyTask、同一个种子、同一个 Adam，LSTM 的准确率从 RNN 的 0.570 跳到 1.000。

这不是魔法，是一条被刻意修出来的「梯度高速公路」。这一章的全部内容就是把这条公路指给你看——它具体是哪一行代码、为什么有效、以及它修好了什么、又没修好什么。

配套代码在 `examples/sequence-models-from-scratch/src/stage03-lstm-gru.ts`，三个 cell（`RNNCell` / `LSTMCell` / `GRUCell`）都定义在这个文件里并 export 给后续章节复用。本章所有数字来自 `npx tsx src/stage03-lstm-gru.ts` 的真实运行（train 时间是真实 wall-clock），copyTask 的实现在 `core/data.ts`。

## 同一个任务，三个 cell，唯一的变量是递归代数

先把对照实验的纪律说清楚，否则数字不可信。本章训练三个模型——vanilla RNN、LSTM、GRU——跑在**完全相同**的任务上：copyTask，发射 1 个随机 payload token，接 20 步空白，再接一个分隔符，最后要求模型把开头那个 token 吐回来。依赖跨度（dependency span，即答案位置到它依赖的输入之间隔了多少步）是 `k + delay + 1 = 22` 步。任务的自描述直接打在 header 里：

```
copyTask: 256 seqs, T=23 (k=1 payload + delay=20 + delim + k answer), vocab=6, dependency span=22 steps
随机基线准确率 = 1/4 = 0.250（瞎猜的下界）
```

三个模型共享同一个数据种子（`makeRng(7)` 造数据、`makeRng(1234)` 决定 batch 顺序）、同一个初始化种子（`makeRng(42)`），同一组超参（隐藏维 24、300 步、batch 32、lr 0.01）。换句话说，**唯一变量是 cell 内部那几行递归算式**。这是整章可信度的地基：如果数据顺序、初始化、优化器任何一个跟着架构变了，"LSTM 比 RNN 强" 就可能是噪声而非信号。

为什么 payload 长度 `k=1`（只记一个 token）？代码注释里写了取舍：`k=2` 会把「跨越间隔记住」和「按顺序吐出两个 token」两件事混在一起，门控 cell 会卡在约 0.65（两个位置对一个），分不清是记忆机制不行还是输出顺序不行。`k=1` 是第 01/02 章那个干净探针，能给出最锐利的「RNN 失败 / 门控解出」的分裂。这是**反 survey 的做法**：survey 会说「LSTM 缓解长依赖」，工程书要告诉你为了让这句话变成一个可证伪的实验，连 payload 长度都得为它服务。

结果：

```
[vanilla RNN] final acc=0.570  params=894  train=656ms  final loss=0.8502
[LSTM       ] final acc=1.000  params=3126  train=3596ms  final loss=0.0006
[GRU        ] final acc=1.000  params=2382  train=3216ms  final loss=0.0008
```

RNN 卡在 0.570——比随机基线 0.250 好，因为 span=22 还没到完全无解，它学到了一点局部统计，但离 copy 出正确 token 还差得远。LSTM 和 GRU 都打到 1.000，loss 掉到 1e-3 量级（基本是数值地板）。代价也明白写着：LSTM 3126 参数 / 3596ms，是 RNN（894 / 656ms）的约 3.5 倍参数、5.5 倍墙钟时间。**门控不是免费的**，它用算力换梯度——这笔账后面第 06 章 SSM 想用线性递归重新算一遍。

## 加性更新：那一行就是整条高速公路

LSTM 的全部秘密在 `LSTMCell.step` 里。先看代码（`stage03-lstm-gru.ts`）：

```ts
step(xt: Tensor, state: Tensor[]): Tensor[] {
  const [hPrev, cPrev] = state;
  const H = this.hiddenDim;
  const z = this.gates.forward(Tensor.concat([xt, hPrev], 1)); // [B, 4H]
  const i = z.slice(1, 0, H).sigmoid();          // 输入门：写多少新信息
  const f = z.slice(1, H, 2 * H).sigmoid();      // 遗忘门：留多少旧 cell state
  const o = z.slice(1, 2 * H, 3 * H).sigmoid();  // 输出门：暴露多少 cell state 当 h
  const g = z.slice(1, 3 * H, 4 * H).tanh();     // 候选：新信息本身
  const c = f.mul(cPrev).add(i.mul(g));          // 加性 cell 更新——这就是高速公路
  const h = o.mul(c.tanh());
  return [h, c];
}
```

LSTM（长短期记忆网络，Hochreiter & Schmidhuber 1997）多出一条 `c`（cell state，可以理解成一条贯穿全序列的传送带）。三个门都是 sigmoid（输出 0~1，当成阀门开度），候选 `g` 是 tanh。**关键的不是有三个门，而是 `c = f * c_prev + i * g` 这一步是加法。**

为什么这一行就是高速公路？做一下反向。第 02 章里 vanilla RNN 的麻烦是 `dh_t/dh_{t-1}` 要过 tanh 导数，恒小于 1，连乘必然消失。而 cell state 这条路上：

```
dc_t / dc_{t-1} = f_t   （逐元素）
```

如果网络学会对某个要长期保存的值把遗忘门 `f` 开到约等于 1，那么梯度从 `c_t` 传回 `c_{t-1}` 就是**乘以约等于 1**，而不是乘以一个 tanh 导数。连乘约等于 1 还是约等于 1——没有几何衰减。这就是 Hochreiter 当年起名 "constant error carousel"（恒定误差旋转木马）的东西：误差信号可以在这条传送带上原地打转、不衰减地往回传。

这里有一个**容易被略过但极关键的设计细节**，代码里也专门处理了——遗忘门的偏置初始化成 `+1`：

```ts
const b = this.gates.b!;
for (let c = H; c < 2 * H; c++) b.data[c] = 1.0; // f 门那段 bias 初始为 +1
```

为什么？如果不动 bias，初始化时门约等于 `sigmoid(0) = 0.5`，意味着 cell state 每步默认衰减一半——这本身就是一种衰减，网络得先学会「不要忘」才能开始记。把 f 门 bias 推到 +1，初始 `f ≈ sigmoid(1) ≈ 0.73`，记忆**默认持久**，网络只需要学「何时该忘」。这是 Jozefowicz 等人验证过的标准 trick，去掉它收敛肉眼可见地变慢。注释里点明：这是整个初始化里唯一不对称的地方，且是故意的。这种「为什么这么设计、不这么设计会怎样」的细节，正是 AI 批量生成的 survey 永远讲不到的——它们会列出三个门的名字，但不会告诉你 f 门 bias 必须偏置。

## ✦ 把 LSTM 砍到只剩高速公路，它往往还能跑

这是本章最该带走的一句话：**LSTM 能记长依赖的关键不是「门多」，而是 cell state 那条不反复过 tanh 挤压的加性通道。**

顺着这个判断往下推：三个门里，输出门 `o` 只决定「暴露多少给 h」，输入门 `i` 决定「写多少」，真正护住梯度的是遗忘门 `f` 维持的那条加性路径。所以如果你把 LSTM 砍到只留最小的门控加性结构——业界有个东西就叫 minimal gated unit（最小门控单元），把三门并成一门——它在很多任务上还能工作。这反过来证明了一件事：**门控的本质是「保护梯度通道」，不是「实现复杂记忆」。** 复杂记忆是副产品，护梯度才是主线。

GRU（门控循环单元，Cho 等人 2014）就是这个判断的一个现成证据。看代码：

```ts
step(xt: Tensor, state: Tensor[]): Tensor[] {
  const hPrev = state[0];
  const z = this.ru.forward(Tensor.concat([xt, hPrev], 1)); // [B, 2H]
  const r = z.slice(1, 0, H).sigmoid();         // 重置门：过去状态多少喂给候选
  const u = z.slice(1, H, 2 * H).sigmoid();     // 更新门：旧 h 与候选之间的插值权重
  const n = this.inToN.forward(xt).add(this.hToN.forward(r.mul(hPrev))).tanh();
  const ones = Tensor.ones([hPrev.shape[0], H]);
  const h = ones.sub(u).mul(n).add(u.mul(hPrev)); // h = (1-u)*n + u*h_prev，漏积分更新
  return [h, c];
}
```

GRU 把 LSTM 的 cell 和 hidden 合成一个状态、砍掉输出门、只留更新门和重置门。它的高速公路是 `h = (1-u)*n + u*h_prev`——一个凸插值（leaky integrator，漏积分器）：当更新门 `u ≈ 1` 时 `h ≈ h_prev`，状态原样传下去，梯度同样乘约等于 1。**同一条加性高速公路，换了个写法，参数少了约 1/4。**

实验印证：GRU 用 2382 参数（vs LSTM 3126）达到完全相同的 1.000。一个实现陷阱注释里特意标了——重置门 `r` 必须在候选的递归 matmul **之前**乘进 `h`（所以 `n` 用了 `inToN` / `hToN` 两个独立 Linear 而非一个 fused concat），这一步顺序是大家最容易写错的地方。GRU ≈ LSTM、参数更少这个结果，就是「高速公路而非门数」这个判断的可跑证据。

## 把高速公路画成数字：梯度沿时间的范数

「梯度不消失」不能只靠嘴说，得测出来。代码在训练最后一步捕获每个时间步的 `||dL/dstate_t||`（state 梯度的 L2 范数），重画第 02 章那条消失曲线，RNN 对 LSTM 叠在一起：

```
梯度沿时间的范数 ||dL/dstate_t||（t=0 是最早一步，离要记住的输入最近）：
    0.125 |                                                              **
    0.014 |                                                           ***
    0.000 |oooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooooo
           *=RNN d(h_t)  o=LSTM d(h_t)
```

注意这张图的尺度有点骗人——它是按 RNN 末几步的大梯度定的纵轴，所以 LSTM 的曲线看着贴地。真正的对比在下面两个数字里：

```
早期步平均梯度范数: RNN=1.55e-13  LSTM=2.72e-7  => LSTM 在最早步保留的梯度约为 RNN 的 1752883.9x
跨整段衰减比 (末步范数 / 首步范数，越大越平): RNN=6.61e+11  LSTM=1.30e+3  (>1 说明早期步的梯度反而更小=衰减；LSTM 的高速路让该比值更接近 1)
```

第一行是核心证据。`t=0` 是最早那一步，离要记住的 payload 最近，也正是第 01 章 RNN 失败的地方。RNN 在这里的梯度范数是 `1.55e-13`——已经underflow 到接近 float64 地板，这就是「梯度到不了」的字面意思。LSTM 是 `2.72e-7`，**约是 RNN 的 175 万倍**。175 万这个数字本身别太当真（它取决于具体 batch 和尺度），可迁移的是数量级：消失梯度和不消失梯度之间差了六七个数量级，这是定性的鸿沟，不是调参能补的差距。

第二行是衰减比（末步范数 / 首步范数，越接近 1 说明越「平」）。RNN 是 `6.61e+11`——意思是末步梯度比首步大十一个数量级，也就是首步梯度相对地被压没了。LSTM 是 `1.30e+3`，仍有衰减但比 RNN 平了八个数量级。高速公路没让曲线完全水平（毕竟还有输出门、tanh 在 h 上的挤压），但它把斜率从「悬崖」拉成了「缓坡」。**这就是修好的样子，写成了数字。**

## 失败模式：门控修了梯度，没修容量

现在讲门控**没**修好的东西——这是本章最该被记住的边界，也是 survey 最爱含糊带过的地方。门控解决的是梯度消失，**不是**无限记忆。LSTM 仍然把整段历史压进一个固定宽度（这里是 24 维）的向量。把依赖跨度推得远超这个向量能缓存的量，LSTM 也会塌——但这次塌的原因是容量，不是梯度。

代码用同一个 LSTM 架构、同样训练预算，只把 delay 翻倍再翻倍：

```
失败模式：把序列拉长，固定隐状态的容量上限暴露出来
（同一 LSTM 架构、同样训练预算，只增大 delay；这次失败是容量，不是梯度）
  delay=20 (span=22): LSTM final acc=1.000  loss=0.0006
  delay=40 (span=42): LSTM final acc=0.281  loss=1.3795
  delay=80 (span=82): LSTM final acc=0.281  loss=1.3793
```

span=22 满分；span=42 直接掉到 0.281——离随机基线 0.250 只剩一点点；span=82 还是 0.281。注意它不是缓慢下滑，而是过了某个点**断崖**：一旦任务要求超过固定状态能可靠维持的量，准确率塌回随机附近，再加长也不会更糟（已经在地板了）。

这里要诚实区分两种失败，否则会得出错误结论。第 01/02 章 RNN 在 span=22 的失败是**梯度消失**——信号传不回去，学不到。本章 LSTM 在 span=82 的失败是**容量瓶颈**——梯度传得回去，但一个 24 维向量物理上装不下需要无损复现的全部信息。同样是「准确率掉」，根因完全不同，修法也完全不同：前者靠改递归代数（本章做的），后者靠……不再用固定向量当唯一记忆。

这正是第 05 章注意力的动机。注意力让每一步**直接读取所有历史**，而不是把历史挤进一个向量——它从根上拆掉了这个容量瓶颈。本章的伏笔代码注释写得很直白：「要让模型在任意距离上『直接访问历史而非压进一个向量』，需要 attention（第 05 章）。」

还有一个本章没碰但同样遗留的问题：LSTM/GRU 的训练是严格串行的 O(n)——`step` 必须一步接一步算，t 步依赖 t-1 步的 state，没法在序列维度上并行（这也是上面 LSTM 墙钟 3596ms 比 RNN 慢的部分原因，更是它在 GPU 时代输给 Transformer 的根本原因）。注意力靠一次性算全矩阵换来训练并行，第 06 章的 SSM（状态空间模型）则试图用线性递归同时拿到「线性复杂度」和「并行训练」。这两个遗留问题——容量和串行——是后面三章的全部驱动力。

## ⚡ 选择性门控：这条血脉一直没断

把镜头拉到 2024。一个反 survey 的判断：**「门控思想」从来没有过时，它只是换了载体。**

Mamba（一种选择性状态空间模型）的核心创新被官方叫做 "selective" SSM——它让 SSM 的状态转移参数变成**输入相关**（input-dependent），也就是说模型能根据当前 token 决定「这一步要不要把信息写进状态、要不要清空状态」。把这句话翻译成本章的语言：这就是一个 input-dependent 的门。它的直系祖先，就是你这一章亲手实现的 LSTM 输入门 `i = sigmoid(...)`——一个根据当前输入决定「写多少进记忆」的阀门。区别只在于：LSTM 的门作用在一个非线性递归的固定状态上，Mamba 的门作用在一个线性递归的、可并行扫描的状态上。门的**思想**——让数据自己决定记忆的读写——一字未改。

2024 年的 xLSTM（扩展 LSTM）更是把这层关系挑明了：它直接回到 LSTM，把那两个遗留问题逐个改——用指数门控和矩阵记忆扩状态容量（对应本章的容量瓶颈），用可并行的变体重构递归（对应串行问题），明摆着是冲着「LSTM 当年输给 Transformer 的两个点」去的。

我要诚实标注**这里没有定论**：⚡ 选择性 SSM / 线性注意力这条线，目前**没有公认会取代 Transformer 的结论**，仍在活跃研究中——绝大多数生产级大模型 2026 年初仍以注意力为主干，Mamba 系多见于混合架构（部分层换成 SSM）和长序列特化场景。它是不是「下一个主干」，还是「注意力的有益补充」，目前无通用解。但有一件事是确定的、也是本章想让你看清的：当你读到 Mamba 的 selective gate、xLSTM 的 exponential gating 时，你不该把它们当全新发明——你这一章手写的那个 `i = sigmoid(...)` 输入门，就是它们的祖宗。**理解了这条加性高速公路加一个数据相关阀门，你就握住了从 1997 年到 2024 年这条没断过的血脉。**

## 小结：修好了什么，没修好什么

一句话收束：本章只改了递归的代数结构——把 RNN 的 `h_t = tanh(...)` 换成 LSTM 的加性 `c_t = f·c_{t-1} + i·g`（或 GRU 的凸插值）——就在第 01 章 RNN 失败（acc 0.570）的同一任务上做到了 1.000，梯度在最早步的留存量是 RNN 的六七个数量级之多。这条加性高速公路，而非门的数量，是真正起作用的东西（GRU 用更少参数同样满分、minimal gated 还能跑，都是证据）。

但门控只是修了梯度这一条路：固定宽度的隐状态仍是硬瓶颈，span 翻两倍 LSTM 就断崖回随机（0.281）；串行 O(n) 训练在 GPU 时代是另一笔还不上的账。这两个遗留问题分别交给第 05 章（注意力：直接访问历史、并行训练）和第 06 章（SSM：线性递归）。而你这章亲手写的输入门，会在第 06 章 Mamba 的选择性门控里再次和你见面——同一个思想，新的载体。
