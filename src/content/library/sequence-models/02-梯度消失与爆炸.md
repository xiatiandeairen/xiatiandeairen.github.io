先把结论钉死:第 01 章那个 vanilla RNN 学不会长依赖,不是因为容量不够、训练不够久、学习率没调好——是因为**反向传播的梯度在沿时间往回走的途中,被同一个矩阵反复乘到了零**。这不是工程问题,是数学问题。你换更大的 hidden、跑更多 epoch、加更多数据,都救不了它。本章的任务是把这句抽象论断变成你能在终端里看见的曲线:对一次真实的 forward+backward,打印每个时间步 `||dL/dh_t||` 的范数,看着它从健康值一路跌到 `1e-17`。

这章对应的可跑代码是 `examples/sequence-models-from-scratch/src/stage02-vanishing-grad.ts`,纯离线、固定种子、`time npx tsx` 跑完 **1.7s**。下面所有数字都是它的真实输出,我会标清楚哪些是实测、哪些是 toy 警告。

## 梯度为什么是「一连串 Jacobian 的连乘」

vanilla RNN 一步的更新(第 01 章的 cell,这章原样重建)是:

```
h_t = tanh(W_hh · h_{t-1} + W_xh · x_t + b)
```

对应代码里 `SimpleRnnCell.step`:

```ts
// stage02-vanishing-grad.ts:61
step(x: Tensor, hPrev: Tensor): Tensor {
  const recur = hPrev.matmul(this.Whh); // [batch, hidden]
  const inp = this.inProj.forward(x);   // W_xh · x_t + bias
  return recur.add(inp).tanh();
}
```

注意一个刻意的设计:这章把 `W_hh`(递归矩阵,把上一步隐状态映射到这一步)单独拎出来当一个裸 `Tensor`,而不是塞进 `Linear` 里。原因是本章全部论证都围着 `W_hh` 的**谱半径**(spectral radius,矩阵最大特征值的模,记作 ρ)转,我需要直接读它、改它的 ρ。`W_xh + b` 这种输入投影才放进 `Linear`。

现在做反向传播。损失 `L` 只挂在最后一步 `h_T` 上(代码里 `loss` 只从 `readout.forward(h)` 算,`h` 是最后一个隐状态)。要算 `L` 对早期某个 `h_t` 的梯度,链式法则要求把中间每一步的雅可比(Jacobian,一个向量函数对其输入的偏导矩阵)乘起来:

```
dL/dh_t = dL/dh_T · (∂h_T/∂h_{T-1}) · (∂h_{T-1}/∂h_{T-2}) · ... · (∂h_{t+1}/∂h_t)
```

而单步雅可比是固定形式:

```
∂h_{k}/∂h_{k-1} = diag(tanh'(·)) · W_hh
```

`diag(tanh')` 是逐元素的 tanh 导数(`tanh' ≤ 1`,在饱和区趋近 0),`W_hh` 是那个被反复乘的矩阵。**关键在「反复」二字**:回溯 L 步,你就乘了 L 个长得一模一样的雅可比。一个矩阵自乘 L 次,它的命运由谱半径 ρ 决定:

- **ρ < 1**:`ρ^L → 0`,梯度指数衰减——**消失**。远端时间步的梯度趋近零,网络学不到任何长程关系。
- **ρ > 1**:`ρ^L → ∞`,梯度指数膨胀——**爆炸**。梯度冲到 `1e3`、`1e5`,再一步就是 NaN。

`tanh' ≤ 1` 这个因子只会让有效乘子比 ρ 更小,所以即便 ρ 略大于 1,实际是消失还是爆炸要看 `ρ · E[tanh']` 这个**有效乘子**跨没跨过 1。这就是为什么纯 ρ 不能单独定生死,得实测有效乘子。

## 把 ρ 当旋钮:实测两个 regime

光说定理没用。代码里 `setSpectralRadius` 用幂迭代(power iteration,反复 `v ← Mv/||Mv||`,收敛到主特征向量)估出当前 ρ,再整体缩放 `W_hh` 把 ρ 钉到指定值:

```ts
// stage02-vanishing-grad.ts:107
function setSpectralRadius(cell: SimpleRnnCell, target: number): number {
  const current = estimateSpectralRadius(cell.Whh);
  const factor = target / current;
  for (let i = 0; i < cell.Whh.size; i++) cell.Whh.data[i] *= factor;
  return estimateSpectralRadius(cell.Whh); // 实测达成的 ρ
}
```

这样两个 regime 是**控制出来的,不是抽签抽到的**——这点很重要,N=1 的运气命中说明不了问题,所以代码打印实测达成的 ρ 让你核对。配置:`hidden=24`,展开 `T=40`,损失只在最后一步取(让梯度回传全程)。

### ρ = 0.5:梯度消失

实测达成 `ρ=0.5000`。回溯一步,梯度缩小到 **0.3817 倍**(这就是实测有效乘子 `ρ · mean(tanh') < 1`,比纯 ρ=0.5 还小,因为 tanh' 又压了一刀)。结果:

```
最旧步 ||dL/dh_0|| = 2.093e-17   vs   最新步 ||dL/dh_39|| = 3.668e-1
衰减倍数(最新/最旧) = 1.75e+16 倍
```

差了 **16 个数量级**。最旧那步的梯度是 `2e-17`,已经掉进 float64 的噪声地板了。范数曲线(左=靠近 loss 的最新步,右=最旧步):

```
0.367 |*
0.122 | **
0.041 |   *
0.000 |    ********************************************************
       +------------------------------------------------------------
        *=||dL/dh|| (ρ<1)
```

读法:只有最靠近 loss 的两三步梯度是健康的,回溯不到十步就贴地了。**这就是第 01 章长依赖失败的物理根因**——跨度 90 步的依赖,梯度信号到那一步只剩 `1e-17`,等于没有梯度,那一步的权重永远学不动。

这里有个实现陷阱值得讲:你不能用「最新/最旧」的两点比值去估衰减率。最旧步已经下溢到 `2e-17` 的浮点垃圾,两点比值会被噪声主导,甚至给出 >1 的荒谬结果(明明在衰减)。代码里 `perStepRate` 改用**对数线性最小二乘拟合**,且只在高于噪声地板(`max · 1e-12`)的区间拟合,这样恢复出来的每步乘子才稳定追踪 ρ:

```ts
// stage02-vanishing-grad.ts:191 — 为什么用拟合不用两点比值
const floor = maxNorm * 1e-12;  // 丢掉下溢的尾巴
// ... 对 log(norm) vs t 做 OLS, rate = exp(slope)
```

这是个真实的数值工程教训:**消失场景下,你的测量工具本身会被下溢污染**,不处理就会得出错误的衰减率。

### ρ = 3.0:梯度爆炸

实测 `ρ=3.0000`。这次回溯一步梯度**放大到 1.2091 倍**(有效乘子 >1,越往回越大):

```
最旧步 ||dL/dh_0|| = 5.008e+2   vs   最新步 ||dL/dh_39|| = 3.649e-1
放大倍数(最旧/最新) = 1.37e+3 倍
全参数梯度 global L2 范数 = 1.886e+3
```

曲线方向反过来了——越回溯越高:

```
559.414 |                                                         **
248.831 |                                                   ***
 62.481 |                                          ******
  0.365 |******************************************
         +------------------------------------------------------------
          *=||dL/dh|| (ρ>1)
```

注意 ρ=3 但每步只放大 1.21 倍,远小于 3。这正是 `tanh'` 那刀的体现:tanh 把大激活压进饱和区,`tanh'` 接近 0,硬生生把有效乘子从 3 拉到了 1.21。所以爆炸比纯线性 `ρ^L` 预测的温和得多——但 40 步累积仍然冲到 `5e2`,再深就是 NaN。

爆炸的解法很直接:梯度裁剪。`clipGradNorm` 计算全参数梯度的 global L2 范数,超过阈值就**整体等比缩放**(方向不变):

```ts
// core/optim.ts:117
if (norm > maxNorm && norm > 0) {
  const scale = maxNorm / norm;
  for (const p of params) for (let j = 0; j < p.size; j++) p.grad[j] *= scale;
}
```

实测契约:裁剪前 `norm=1.886e+3`,若 `maxNorm=1.0`,所有梯度按 `5.30e-4` 缩放。爆炸是**数值问题**,clip 把范数拉回阈值就救命了——这是个真能修的问题。

## ✦ 专家钩子:clip 只治爆炸,不治消失

这是新手最常踩的坑,我见过太多人把 `clip_grad_norm_` 当成「梯度问题」的万能膏药贴上去就完事。**爆炸和消失根因不同、解法不同,clip 只对其中一半有效。**

- 爆炸是**数值问题**:梯度太大要溢出。clip 缩放大小即可,信息没丢,只是把步长拉回安全区。
- 消失是**信息问题**:远端梯度本身就是 `1e-17`,信息已经在连乘里被乘没了。你 clip 一个接近零的梯度——除以一个很大的数再乘,或者干脆不动——它还是接近零。**clip 改变不了梯度的相对结构,只改全局尺度。**

代码的【失败模式】实验把这点钉死了:训练一个 vanilla RNN(健康初始 ρ + 全程 clipGradNorm)去解 `copyTask`(把序列前段的几个符号原样复述到后段)。

**短跨度(依赖 4 步):**
```
最终 loss=0.0060, 答案区准确率=100.0%, 随机基线=25.0%
触发裁剪步数=12/240  (clip 在工作,没让梯度炸成 NaN)
```

**长跨度(依赖 93 步):**
```
最终 loss=1.3789, 答案区准确率=28.9%, 随机基线=25.0%
触发裁剪步数=47/240  (clip 同样在工作)
```

量化对比:

```
短跨度高于随机:+75.0pp(学会了)
长跨度高于随机:+3.9pp (≈停在随机)
跨度 4 → 93 步,准确率提升从 +75.0pp 坍缩到 +3.9pp
```

两次训练都触发了梯度裁剪(短 12/240,长 47/240),证明 clip 确实在工作、确实把爆炸治住了、确实没让训练炸成 NaN——**但长依赖照样学不会**。根因一句话:`clip 缩放梯度大小,改变不了 ρ^L 衰减带来的『远端梯度 ≈ 0』`。clip 能让你的训练「不崩」,但不能让它「学会」长程关系。把 clip 当长依赖的解法,是对消失和爆炸根因的混淆。

> 诚实警告:`copyTask` 是合成数据,绝对准确率偏乐观。可迁移的不是「93 步准确率 28.9%」这个具体数,而是**相对趋势**:跨度↑ → vanilla RNN 准确率坍缩到随机,且 clip 无法挽救。换数据集绝对值会变,这个坍缩曲线不会变。

## 出路:给梯度一条不被反复乘的高速公路

既然根因是「同一个 `diag(tanh')·W_hh` 被连乘 L 次」,出路就显然了:**别让梯度走那条反复相乘的路**。如果隐状态的更新里有一条**加性的、近似恒等的直通路径**,那么这条路上的雅可比就接近单位矩阵 `I`,连乘 `I·I·...·I = I` 不衰减不膨胀——梯度有了一条「高速公路」。

这正是第 03 章 LSTM / GRU 的 cell state 要干的事:`c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t`,当遗忘门 `f_t ≈ 1` 时,`∂c_t/∂c_{t-1} ≈ I`,梯度可以几乎无损地流过几百步。**换架构不是优化、不是 nice-to-have,是消失问题的唯一解法**——本章的连乘视角就是为了让你在第 03 章看见门控时,立刻明白它在数学上修的是哪一项。

## ⚡ 前沿:SSM 如何从数学上控制这个连乘

门控是 1997 年(LSTM)的答案。2021 年后的 SSM(state space model,状态空间模型)复兴给出了一个更深的答案,而**本章的 Jacobian 连乘视角正是理解它的钥匙**。

SSM 的递归形如 `h_t = A · h_{t-1} + B · x_t`——注意这里没有逐步的 `tanh` 非线性挤压,递归是线性的,转移矩阵 `A` 在所有时间步**共享且固定**。于是「连乘 L 个雅可比」退化成了「`A` 的 L 次幂 `A^L`」,一个可以精确分析、精确控制的对象。S4 这类工作的做法是:把 `A` **对角化**,这样 `A^L` 就是对角线上每个特征值各自的 L 次幂,梯度沿时间的行为由你**显式选定的特征值谱**决定;再配合特定的初始化(如 HiPPO 理论给出的结构化 `A`,它让状态成为输入历史的最优多项式投影),从数学上保证长程梯度既不消失也不爆炸。

换句话说,vanilla RNN 是「祈祷随机初始化的 `W_hh` 谱半径恰好在 1 附近」,LSTM 是「用门控造一条 `≈I` 的旁路绕开连乘」,而 SSM 是「直接设计连乘本身的谱,让 `A^L` 可控」。这是 SSM 复兴的理论支点。

需要明确标注:**「如何为任意任务自动选出最优的状态转移谱」目前没有通用解,仍在研究中。** HiPPO/S4 的结构化初始化在长序列 benchmark(如 Long Range Arena)上很强,但它是为「记住整段历史」这类任务设计的;面对需要选择性遗忘、内容相关路由的任务,固定的 `A` 又成了瓶颈——这直接催生了第 07 章要讲的 Mamba(让 `A`、`B` 随输入动态变化的 selective SSM)。如何在「可分析的固定谱」和「表达力强的输入依赖动态」之间取得最优平衡,是这个方向当下最活跃的开放问题。

## 小结

- vanilla RNN 的长依赖失败是数学必然:BPTT 让梯度乘上 L 个相同雅可比 `diag(tanh')·W_hh`,命运由谱半径 ρ 决定。
- 实测可见:ρ=0.5 时 40 步衰减 `1.75e16` 倍,最旧步梯度 `2e-17`(消失);ρ=3.0 时膨胀到 `5e2`、global norm `1.9e3`(爆炸)。
- 梯度裁剪治爆炸(数值问题)不治消失(信息问题):clip 全程工作、防住了 NaN,长跨度 93 步准确率仍坍缩到 +3.9pp 的随机水平。
- 解法是给隐状态加加性直通路径(门控,第 03 章)或直接设计转移矩阵的谱(SSM,第 06-07 章)。两者都在修同一项:那个被连乘的雅可比。
