> 第 05 章把长依赖问题用 attention(注意力,让每个位置直接看全序列)解决了,但你已经看到账单:序列长到 4K、8K,那张 n×n 的注意力矩阵就是一道墙,显存和算力都按 O(n²) 涨。这章给第三条路——既要 attention 的长依赖,又要 RNN 的 O(n)。核心赌注只有一句话:**把 RNN 里那个 tanh 拿掉**。去掉非线性后递归变成线性,而线性递归有一个 RNN 永远做不到的性质——它等于一个卷积,可以在训练时一次性并行展开。
>
> 替你解决的问题:让你不再把「O(n) 高效」和「长依赖能记住」当成必须二选一的对立面。SSM(状态空间模型)证明了在「线性 + 结构化 A」这对组合下,二者可以同时拿到。这也是为什么 2021 年 S4 在长序列基准 Long Range Arena 上第一次把 Transformer 按在地上——非注意力路线第一次被证明可行。

读法提示:这章每个论点都绑死配套仓 `examples/sequence-models-from-scratch/src/stage06-ssm.ts` 的真实运行输出,不是推导出来骗你信的,是跑出来给你看的。重点啃三处:`[1]` 递归==卷积的机制自证(误差 < 1e-8)、`[4]` O(n) vs O(n²) 的实测 wall-clock、`[5]` |A|>1 时状态真的溢出成 Infinity。这三处分别对应 SSM 的「双面孔」「标度优势」「稳定性是前提不是可选」。

## 一句话先把 SSM 钉住:它就是去掉 tanh 的 RNN

回忆第 01 章的 RNN 单元:`h_t = tanh(W·h_{t-1} + U·x_t)`。那个 `tanh` 是它的命门——它带来表达力,也带来第 02 章讲透的连乘灾难:梯度沿时间反传时要连乘 N 个雅可比,tanh 的导数 <1 让梯度指数消失,长依赖学不动。

SSM 的第一个动作粗暴得近乎挑衅:**把 tanh 删了**。

```
h_t = A · h_{t-1} + B · x_t      (状态更新,纯线性)
y_t = C · h_t                     (读出)
```

`A` 是状态转移(state transition,旧记忆衰减多少),`B` 是输入投影(新信息进多少),`C` 是读出投影(从状态读出什么)。没有激活函数夹在中间。直觉上你会觉得这是退步——非线性不是深度学习的灵魂吗?去掉它表达力不就废了?

这正是 SSM 反直觉的地方,也是配套代码 `stage06-ssm.ts` 头注释里写死的章节论点(第 3-9 行):**正因为线性,同一个计算可以有两副面孔**。这不是为了省非线性,是为了换来一个 RNN 结构上拿不到的能力。下面我们一副面孔一副面孔拆。

实现上,这章的 `SSMBlock` 给**每个特征通道一组独立的标量 (A,B,C)**(`stage06-ssm.ts:63-106`)。一个 `dim` 宽的 SSM = `dim` 个独立的标量 SSM 并排跑。这叫对角 SSM(diagonal SSM),是 S4D / Mamba 的诚实内核去掉「选择性」后剩下的东西。为什么用对角而不是稠密矩阵 A?因为对角让「递归==卷积」这条恒等式精确且可检视;稠密 A 会把它埋在矩阵乘法里看不清(代码注释 `stage06-ssm.ts:11-15`)。

```typescript
// stage06-ssm.ts:91-101 —— 递归前向,显式沿时间轴扫描
forward(inputs: Tensor[], batch: number): Tensor[] {
  const A = this.transition(); // [1,dim], A = sigmoid(aLogit) ∈ (0,1)
  let h = Tensor.zeros([batch, this.dim]); // 初始状态 h_0 = 0
  const outputs: Tensor[] = [];
  for (let t = 0; t < inputs.length; t++) {
    // h_t = A⊙h_{t-1} + B⊙u_t  (逐元素;A/B/C [1,dim] 沿 batch 轴广播)
    h = A.mul(h).add(this.B.mul(inputs[t]));
    outputs.push(this.C.mul(h)); // y_t = C⊙h_t
  }
  return outputs;
}
```

注意这里 `A/B/C` 在每个时间步是**同一组共享权重**——和第 01 章 RNN 一样靠 BPTT(沿时间反传)训练,引擎的「梯度累加不覆盖」规则保证了共享权重的 BPTT 正确(注释 `stage06-ssm.ts:54-56`)。

## 第一副面孔:线性递归 == 卷积(并行训练的根)

把递归展开。`h_0 = B·x_0`,`h_1 = A·h_0 + B·x_1 = A·B·x_0 + B·x_1`,继续推:

```
h_t = Σ_{k=0}^{t} A^k · B · x_{t-k}
y_t = C · h_t = Σ_{k=0}^{t} (C · A^k · B) · x_{t-k}
```

看那个括号 `C·A^k·B`——它只依赖 k,不依赖具体输入。把它记成 `K[k] = C·A^k·B`,上式就是一个**因果卷积**:`y_t = Σ_k K[k]·x_{t-k}`。这个核 `K` 就是 SSM 的脉冲响应(impulse response,给一个单位输入系统怎么衰减)。

这意味着什么?**递归形式是 O(n) 内存、严格串行**(第 t 步等第 t-1 步算完);**卷积形式是 O(n) 工作量、但沿时间并行**(所有 y_t 可以一起算,FFT 还能更快)。训练时用卷积形式吃满 GPU 并行,这就是 SSM 摆脱第 01 章「RNN 串行反传慢」的关键。两副面孔是**同一个函数**——这是数学恒等式,不是近似。

但「数学上相等」和「代码里相等」是两回事(浮点、累加顺序、实现 bug 都可能让它们偏)。所以这章不口头断言,而是数值自证。代码 `stage06-ssm.ts:117-151` 写了两个独立实现:

```typescript
// stage06-ssm.ts:117-125 —— 递归实现
function scanRecurrent(u, A, B, C) {
  let h = 0;
  for (let t = 0; t < u.length; t++) { h = A*h + B*u[t]; y[t] = C*h; }
  return y;
}
// stage06-ssm.ts:133-151 —— 卷积实现(核 K[k]=C·A^k·B)
function ssmKernel(A, B, C, length) {
  let aPow = 1; for (let k=0;k<length;k++){ K[k]=C*aPow*B; aPow*=A; } return K;
}
```

跑 5 组随机 (A,B,C) 和随机输入,比对两种前向的逐点最大误差(`[1]` 实测输出):

```
trial 0: A=0.307 B=2.334 C=-0.839 max|y_rec - y_conv| = 1.78e-15
trial 1: A=0.380 B=-0.187 C=-1.228 max|y_rec - y_conv| = 1.67e-16
trial 2: A=0.675 B=0.392 C=-0.108 max|y_rec - y_conv| = 3.47e-17
trial 3: A=0.766 B=-0.031 C=0.288 max|y_rec - y_conv| = 3.47e-18
trial 4: A=0.535 B=0.052 C=-0.149 max|y_rec - y_conv| = 6.94e-18
最大误差 1.78e-15 < 1e-8 ✓ (两种前向数值等价,机制为真)
```

最大误差 1.78e-15,贴着 float64 机器精度(约 2.2e-16)的地板。这不是「差不多相等」,是「相等到浮点数能表达的极限」。**双面孔为真**。

这里要诚实说一个工程细节:这个 toy 用朴素的 `O(t)` 累加算每个 `y_t`,所以卷积形式在这章的代码里其实是 `O(n²)` 的双层循环(`stage06-ssm.ts:143-150`)——它是用来**证明恒等式**的,不是性能实现。真正的 S4 用 FFT 把卷积做到 `O(n log n)`,或者用结构化矩阵的快速算法。这章的标度优势(下面 `[4]`)测的是**递归扫描** vs **注意力**,那才是 SSM 推理时真正跑的 O(n) 路径。

## 第二副面孔:推理时退回 O(1) 状态递归

训练完用卷积并行,推理(自回归生成)时切回递归形式:每来一个新 token,只需 `h = A·h + B·x; y = C·h`——状态 `h` 是固定大小,**与已生成长度无关**。

对比第 05 章的 attention:每生成一个 token 要回看前面所有 token 的 KV cache,cache 随序列线性增长,单步计算随序列线性增长,总生成是 O(n²)。SSM 推理是**每步 O(1)、总 O(n)**,状态就那一个固定大小的 `h`。这是 SSM 在长上下文推理上对 Transformer 的结构性优势:不存 KV cache,显存不随上下文涨。

一个 RNN 工程师到这里会皱眉:O(1) 状态递归不就是 RNN 吗?RNN 早就 O(1) 推理了,问题从来是它**训练慢(串行)且学不会长依赖**。对——SSM 的赢面不在「O(1) 推理」这一项(那 RNN 也有),而在「**同时**拿到 RNN 的 O(1) 推理 + 卷积的并行训练 + 下面要讲的可控长依赖」。三者同时成立,靠的就是「线性」这一刀。

## 长依赖到底解没解?在 copyTask 上实测

光说不练是 survey 的毛病。这章把 SSM 扔到第 05 章用过的长依赖压力测试 **copyTask** 上:序列里先给 k 个 payload 符号,隔 delay 步,再要求模型把它们原样吐出来。delay 拉到 12,依赖跨度 = k + delay + 1 = 15 步——一个无记忆模型必死,逼模型真把早期信息携带过长间隔。

```
copyTask: 64 seqs, T=17 (k=2 payload + delay=12 + delim + k answer), vocab=6, dependency span=15 steps
参数量 = 486  (A/B/C 各 32 + embed 192 + readout 198)
loss: 1.7750 -> 0.8378
答案区准确率 = 65.6%  (随机基线 = 25.0% over 4 payload symbols; span=15 steps 长依赖已解)
```

486 个参数的玩具 SSM,在 15 步依赖跨度上把答案区准确率从随机基线 25% 拉到 65.6%。**长依赖确实被解了**——这是去掉 tanh、改用结构化 A 之后,第 02 章那个「梯度消失学不会长依赖」的老毛病被绕开的直接证据。

诚实标注:这是 toy 数据(词表小、信号干净、训练集上测的记忆探针),65.6% 这个绝对数字是乐观的,别拿去当 S4 的真实水平(代码注释 `stage06-ssm.ts:25-28` 也明说了)。能迁移的是**相对结论**:同样跨度,第 01 章纯 RNN 学不动,这里的线性 SSM 学得动。机制为真,绝对值仅供方向参考。

它怎么做到的?看状态演化的 heatmap(`[3]`,行=时间步,列=前 12 个 state 通道,亮=该通道 |state| 大):

```
::  .  -.. .       (早期 payload 进来)
:   .  #:: =
=      #--.+
+.     #==.*
*: .   #+*:*
#: .   #**:#
%- .   #*#:#
@- ..  %**:=       (delim 在 t=14)
@= .   %*#:+       (答案区)
```

看第 1 列和中间那几列——某些通道从早期一路保持高亮到答案区。这就是**高-A 通道**在干活:`A` 接近 1 的通道衰减慢,把早期 payload 的信息一路「拖」到答案区不丢。不同通道有不同的 `A`,等于不同的记忆时间常数,有的管短期有的管长期。这正是初始化时故意让 `A` 偏高(`aLogit` 平均偏正,`sigmoid(1)≈0.73`)的原因(`stage06-ssm.ts:72-77`)——copyTask 需要长记忆,就给它配长记忆通道。

## 标度对比:O(n) vs O(n²),实测 wall-clock

前面都是定性。这章直接拿秒表测:同样 modelDim=64,序列从 64 拉到 512,真实跑 SSM 递归扫描和稠密注意力 QK^T·V 的前向耗时(`[4]`,真 wall-clock,reps=5):

```
n     SSM(O(n))    Attn(O(n²))   attn/ssm    MAC attn/ssm
64       0.0201       0.4516       22.4x        128x
128      0.0127       1.5712      123.6x        256x
256      0.0203       6.0095      296.8x        512x
512      0.0386      23.6558      613.5x       1024x
n 增大 8x: SSM 耗时 x1.9 (理想线性 8x), Attn 耗时 x52.4 (理想平方 64x)
```

读这张表别只看「快多少倍」,看**斜率**:n 翻 8 倍,SSM 耗时只涨 1.9 倍(理想线性是 8 倍——比线性还好,小 n 时常数项和 cache 占主导),注意力涨 52.4 倍(理想平方是 64 倍)。倍数差从 n=64 的 22 倍拉到 n=512 的 613 倍——**这个比值随 n 单调发散,就是 O(n²)/O(n) = O(n) 的直接观测**。序列越长,SSM 的优势越夸张。

```
attn/ssm n=64  | █ 22.4
attn/ssm n=128 | ████████ 123.6
attn/ssm n=256 | ███████████████████ 296.8
attn/ssm n=512 | ████████████████████████████████████████ 613.5
```

工程上要诚实:wall-clock 受常数项、JIT、cache 行为污染(注意 n=64 比 n=128 还慢一点,就是 JIT 预热没稳),所以绝对数字别太当真。但 MAC(乘加次数)计数是纯理论的,attn/ssm 从 128x 到 1024x 严格按 n 翻倍——这条是干净的,相对趋势铁定:**SSM 近线性,Attn 近平方**。

## 失败模式:为什么 SSM 必须「结构化 A」

到这里 SSM 看起来太美好了:O(n)、长依赖、并行训练。代价在哪?代价在**稳定性,而且它不是可选项,是可用的前提**。

回到展开式 `h_t ≈ A^t · (早期输入)`。状态随时间按 `A^t` 演化。这是把双刃剑:

- `|A| < 1`:`A^t` 衰减,旧输入的影响有界地淡出 → 可用的有限记忆。
- `|A| > 1`:`A^t` 指数增长,状态范数爆炸,几十步内输出冲到 Inf/NaN。

这章用脉冲输入(只有 x_0=1,其余全 0,这样状态恰好是纯 `A^t`,隔离掉输入噪声,让你看清稳定性**只**取决于 |A|)实测(`[5]`):

```
A=0.90 稳定 (|A|<1): 脉冲响应衰减
  |state| t=0 -> t=39: 1.00e+0 -> 1.64e-2  (A^39 = 1.64e-2)
A=1.00 临界 (|A|=1): 不衰减
  |state| t=0 -> t=39: 1.00e+0 -> 1.00e+0  (A^39 = 1.00e+0)
A=1.05 不稳定 (|A|>1): 指数爆炸
  |state| t=0 -> t=39: 1.00e+0 -> 6.70e+0  (A^39 = 6.70e+0)
A=1.10, T=8000, 脉冲: 末步 |state| = Infinity (t=7448 起 |state| 溢出为 Infinity — 数值真的崩了,不是比喻)
```

`A=1.10` 在 T=8000 的序列上,t≈7448 处状态**真的溢出成 Infinity**——不是「梯度有点大」这种修辞,是 float64 装不下、数值字面崩掉。这就是为什么不能让优化器自由地把 A 推到任意值。

解法是**用参数化把 A 钉死在稳定域**。这章的 `SSMBlock` 不直接学 A,而是学一个无约束的 `aLogit`,然后 `A = sigmoid(aLogit) ∈ (0,1)`(`stage06-ssm.ts:65, 82-85`)。优化器爱把 `aLogit` 推到实数轴哪儿都行,`sigmoid` 保证 A 永远落在 (0,1) 稳定域——**结构性稳定,不靠运气,不靠 clip,不靠手调**。训练后实测验证:

```
对照:训练后 SSM 的 A=sigmoid(aLogit) 全部落在 (0,1):min=0.655 max=0.925 (全部稳定 ✓)
```

这就是「结构化 A」的本意——不是某种高深矩阵技巧,而是**用构造方式从根上排除掉不稳定的可能**。

现在把本章的专家钩子说透:**✦ SSM 能同时拿到 O(n) 和长依赖,靠的是「线性 + 结构化 A」这对组合,缺一不可**。

- 去掉「线性」(把 tanh 加回去):递归不再等于卷积,训练退回第 01 章的串行 BPTT,并行优势没了——退回普通 RNN。
- 去掉「结构化 A」(让 A 随机、无约束):`|A|>1` 的通道按 `A^t` 爆炸成 Inf/NaN(就是上面 `[5]` 演示的),`|A|≪1` 的通道又瞬间忘光学不到长依赖——退回 RNN 那套「不爆就消失」的两难。

两个条件是 SSM 这台机器的两根支柱,抽掉任一根它就塌回 RNN 的老毛病。这也是为什么你不能把 SSM 简单理解成「线性 RNN」就完事——「线性」给了并行,「结构化」给了可控,合起来才是 SSM。

## ⚡ 前沿:HiPPO——长依赖能力的数学根,以及它还没被讲透的部分

这章的 toy 用 `A=sigmoid(aLogit)` 把 A 钉在 (0,1),够演示稳定性,但它有个软肋:**A 的初始值是随机的**,长记忆通道是训练「碰」出来的(靠 `aLogit` 偏正这个先验)。真正的 S4 不靠碰——它用一个叫 **HiPPO**(High-order Polynomial Projection Operators,高阶多项式投影算子)的特定矩阵来初始化 A。

HiPPO 的思想是:让状态 `h` 在每个时刻都是「用一组正交多项式对**至今为止的整段历史**做最优逼近」的系数。换句话说,状态不是随便记点东西,而是数学上**最优地压缩历史**——老信息以一种「最小化重构误差」的方式被保留。这是 SSM 长依赖能力的数学根:不是「希望它能记住」,而是「初始化就让它以最优方式记住」。2021 年正是带 HiPPO 初始化的 S4 在 Long Range Arena(专测超长序列的基准)上**大幅超过 Transformer**,第一次证明非注意力路线在长序列上不仅可行,而且能赢。这是序列建模史上的一个转折点——在此之前「长序列就上 Transformer」近乎默认,S4 把这个默认捅破了。

为什么说这是前沿而非定论?**目前对 HiPPO「为什么这么好用」还没有完全干净的理论闭环。** 几个仍在研究的开放点:HiPPO 给的是连续时间的最优逼近理论,但实际跑的是离散化版本,离散化方式(双线性 / 零阶保持)如何影响逼近最优性,边界还在厘清;对角化近似(S4D 把 HiPPO 矩阵近似成对角,就是本章这种结构)在什么条件下保住逼近能力、什么条件下退化,没有通用判据;以及——第 07 章的核心——HiPPO 是**输入无关**的固定记忆策略,而 Mamba 证明了让 A/B/C **随输入变化**(选择性)能进一步大涨,但「选择性」一加进来,「线性递归=卷积」这条恒等式就**破了**(核不再固定),只能退回扫描——于是又催生了硬件感知的并行扫描算法。**记忆该固定还是该选择、选择性和并行化怎么兼得,目前没有通用解,正是这条路线最活跃的研究前沿。**

## 小结与承上启下

```
小结:递归=卷积 (机制自证 <1e-8) | 长依赖可解 | O(n) 胜 O(n²) | 稳定 A 是前提
```

把这章压成四句你该带走的判断:

1. **SSM = 去掉 tanh 的 RNN**,线性这一刀换来「递归==卷积」的双面孔(误差贴机器精度,`[1]`)。
2. **双面孔 = 训练并行(卷积)+ 推理 O(1) 状态(递归)**,同时拿到 RNN 和 Transformer 各自的好处。
3. **长依赖靠结构化 A**:对角 + 稳定域参数化,让不同通道有不同记忆时间常数,15 步跨度可解(`[2][3]`)。
4. **稳定性是前提不是优化**:|A|>1 真的溢出成 Infinity(`[5]`),所以 A 必须用构造方式钉在稳定域。

承上:第 01 章的 RNN 给了 O(1) 推理但串行+梯度消失,第 02 章解释了为什么消失,第 05 章用 attention 解决长依赖但赔上 O(n²)。这章用「线性+结构化」一并拿下三者。启下:第 07 章 Mamba 会指出这个 toy SSM 的最后一块短板——它的 A/B/C 对所有输入一视同仁,记不住「该重点记哪个 token」。Mamba 的「选择性」让参数随输入变化来补这一刀,代价是亲手打破本章引以为豪的「递归=卷积」恒等式,逼出全新的并行扫描工程。带着「线性给了什么、结构化给了什么」这两个问题进第 07 章,你会看得很清楚 Mamba 到底动了哪根支柱、又付了什么代价。
