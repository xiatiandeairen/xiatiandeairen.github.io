induction head（归纳头，机制可解释性最有名的一个发现）经常被人误解成「注意力对角线」——你打开一张 attention 图，看到第二份拷贝的 query 沿对角带亮起来，就宣布「找到 induction head 了」。这是错的，而且错得有代价。对角图案是结果不是机制。真正的 induction head 是**两个头串成的电路**：第 0 层一个「前位头」（previous-token head，把上一个 token 的身份搬进当前位置），第 1 层一个「归纳头」据此「找到我上次出现的地方的下一格」并把那个 token 复制出来。缺任何一头，电路就断，对角图案也不会出现。本章不停在画图，而是用第 05 章学的 patching 把这两头逐个敲掉，看准确率塌不塌——这才是 induction circuit 的定义性证据。

这一章是全书前五章方法的合流：第 02 章教你读注意力矩阵，第 05 章教你用干预做因果判定。我们在 `examples/interpretability-from-scratch/src/stage06-induction-head.ts` 里新训一个 2 层玩具模型，把所有方法连起来跑一遍。所有数字都来自这个 stage 的真实运行输出，估算的地方会标出来。

## 一、为什么是变偏移复制任务，不是普通 copy

先说任务，因为任务设计本身就是一个机制陷阱，踩进去整章结论全是假的。

最自然的复制任务是「固定偏移」：序列前半是块 S，后半在固定距离外重复 S，模型要预测后半。问题在于——**这个任务一个纯位置头就能作弊解掉**。所谓纯位置头，就是「无论内容是什么，永远往回看 k 格」。它不需要内容匹配，不需要第二层，一个 1 层模型就能 100% 解。如果我们用固定偏移任务,然后宣称「1 层学不会、必须 2 层」，那是撒谎——`stage06` 的注释里明确记了我们验证过这个陷阱。

所以这章用的是 `gapRepeat` 任务（见 `stage06-induction-head.ts` 的 `gapRepeat` 函数），结构是：

```
[随机长度前缀] [块 S] [随机填充] [SEP] [S 重复]
```

关键在那个**随机长度前缀**。看代码：

```typescript
const pre = Math.floor(rng() * (gapMax + 1)); // 0..gapMax filler BEFORE S => variable offset
const seq: number[] = [];
for (let i = 0; i < pre; i++) seq.push(Math.floor(rng() * vocab));
seq.push(...S);
```

前缀长度每个样本都不同，所以块 S 的首次出现位置是浮动的，从第二份拷贝某一格回看到它的源位置的距离**每个样本都变**。这一下就废掉了「往回看固定 k 格」的位置作弊：模型被逼着按内容做 induction——「我现在这个 token 是 X，上次 X 出现在哪？它后面跟的是什么？」。这正是 2 层和 1 层的分水岭。

实际跑的配置：`gapRepeat(8, 4, 4)`，词表 9（8 个内容 token + 1 个 SEP 分隔符），序列长 13（`blk + gapMax + 1 + blk = 4+4+1+4`）。词表选 8 是为了让填充 token 撞上块 S 内容的概率低到「源位置通常无歧义」——这是个刻意的 toy 取舍，后面诚实边界会说它的代价。

⚠ 顺便说一个设计细节，它解释了为什么很多人复现 induction 时训不动：core 里有个标准的 `inductionTask` 只有单个可评分位置，梯度信号稀疏，优化器会「饿死」。`gapRepeat` 让第二份拷贝的**每一个位置**都可评分（见 `scorablePositions`），梯度密集得多，这才训得起来。任务设计直接决定能不能训，不是调 lr 能补救的。

## 二、先确认它真学会了，而不是碰运气

拆电路之前必须先证明这模型确实学会了 in-context 复制，否则后面拆出来的全是噪声。`stage06` 的 step 1 训了一个 2 层模型（`dModel=32, nHeads=4, nLayers=2`，约 1 万参数），训了 2000 步，对照两条基线：

```
in-context 准确率 (第二份拷贝各位置)
 trained 2L | ############################## 0.741
 random-init | ###### 0.139
 chance(1/V) | #### 0.111
```

最终 loss = 0.5751，wall-clock 41387 ms。

读这三个数：随机初始化的模型 0.139，几乎就是瞎猜基线 0.111（1/词表）；训练后的 2 层模型 0.741。**远超随机即「学会了」**。这里要诚实——它没到 100%，原因后面诚实边界讲，但 74.1% 对一个万参数模型在变偏移匹配任务上已经是明确的「学会」信号，不是过拟合噪声（评测用的是另一套随机种子生成的样本）。

注意这里我们没看 loss 绝对值就下结论。0.5751 这个 loss 单独看没意义——你不知道这任务的 loss 下限在哪。**用基线对照而不是看绝对数**，这是贯穿全书的判定纪律（第 01 章训玩具模型时就强调过）。

## 三、按电路理论给每个头打分，定位两类头

现在进正题。我们有 2 层 × 4 头 = 8 个头，要找出哪个是前位头、哪个是归纳头。新手做法是肉眼翻 8 张注意力图——慢、主观、不可复现。`stage06` 用的是**按电路预测的源位置给注意力打分**（`scoreHeads` 函数），把「像不像」变成一个数。

电路理论给出两个可测量的预测（每个头在 300 个样本上平均）：

- **前位头**应该：在第一份拷贝内部，位置 `j` 的 query 把注意力打到 `j-1`（前一个 token）。这正是它「把上一个 token 搬进来」的动作。打分 = `attn(j -> j-1)`。
- **归纳头**应该：第二份拷贝里 token `S[j]` 的 query，把注意力打到「`S[j]` 首次出现处的下一格」`sStart+j+1`。打分 = `attn(query -> 源+1)`。

代码里这两个目标是直接从任务的 ground-truth plant 里读出来的（`plant.sStart` / `plant.sepIdx`），不是事后猜的：

```typescript
const q = plant.sepIdx + 1 + j; // second-copy query for token S[j]
const key = plant.sStart + j + 1; // slot right after the source of S[j]
if (key <= q) {
  induction[l][h] += pat[q * seq + key];
}
```

跑出来的结果，前位头打分（预期集中在 L0）：

```
L0H0 | 0.138    L0H1 | 0.430    L0H2 | 0.495    L0H3 | 0.590
L1H0 | 0.216    L1H1 | 0.206    L1H2 | 0.223    L1H3 | 0.252
```

归纳头打分（预期集中在 L1）：

```
L0H0 | 0.111    L0H1 | 0.124    L0H2 | 0.169    L0H3 | 0.137
L1H0 | 0.480    L1H1 | 0.516    L1H2 | 0.480    L1H3 | 0.158
```

判定一目了然：**前位头 = L0H3（0.590），归纳头 = L1H1（0.516）**。前位头打分在 L0 那一排整体高（L0H1/H2/H3 都 ≥0.43），归纳头打分在 L1 那一排整体高（L1H0/H1/H2 都 ≥0.48）。这是电路形状的第一个证据：**前位头在更靠下的层（L0），归纳头在更靠上的层（L1）**——符合「先搬运、再匹配」的串联顺序。如果是反的（归纳头在 L0），那这模型学的根本不是 induction，`stage06` 的形状检查会直接报警 `⚠ 层序不符预期`。

把归纳头 L1H1 在一个固定样本上的注意力矩阵打出来（行=query，列=key），就是那张著名的「对角带」图：

```
样本 tokens=[5,6,1,4,4,5,3,7,8,5,6,1,4]  S 首次出现于 [0..3], SEP@8, 第二份拷贝 @9..
   0123456789111
 9  *    -
10   @
11    %
12    .#.
  scale: ' .:-=+*#%@'  vmin=0.000 vmax=1.000
```

看第 9~12 行（第二份拷贝的四个位置）：它们的亮格分别落在第 1、2、3、4 列附近——也就是块 S 首次出现处（0~3）往后偏一格的位置。这就是归纳头在干的活：「我是 token 5，5 上次在位置 0，0 后面跟的是 6，输出 6」。**但记住开头那句话**：这张漂亮的对角图只是表象。它之所以能出现，是因为 L0H3 已经在每个位置悄悄写进了「我跟在谁后面」的信息，L1H1 才有 key 可匹配。下一节用 ablation 把这个因果链钉死。

## 四、逐头 ablation：证明两层是串联，缺一不可

打分定位是**相关性**——「这个头的注意力落在理论预测的位置上」。但相关不等于因果。也许这个头打了那个注意力但对输出毫无贡献。要证因果，得把它**敲掉**看准确率塌不塌。这就是第 05 章的 activation patching 在这章的应用。

`stage06` 用 **mean-ablation**（均值消融）而不是直接置零。为什么这个选择重要——看 `meanAblate` 函数的注释：head 的输出会喂给输出投影 + 下游 LayerNorm，**直接置零会把残差流推到训练分布之外**，LayerNorm 对偏离分布的输入反应剧烈，会高估这个头的重要性（这是第 05 章 patching 章节就埋的坑）。用这条序列上各位置的均值替换，是一个「在分布内」的廉价替身，只抹掉这个头的**内容贡献**而不破坏整体激活的统计量。

```typescript
hooks[point] = (_name, value) => {
  const [rows, cols] = value.shape;
  const mean = new Float64Array(cols);
  for (let r = 0; r < rows; r++) for (let c = 0; c < cols; c++) mean[c] += value.data[r * cols + c];
  for (let c = 0; c < cols; c++) mean[c] /= rows;
  // ...每行都替换成均值
};
```

逐头敲掉的准确率下降（clean = 0.741）：

```
abl L0H0 | 0.206    abl L0H1 | 0.152    abl L0H2 | 0.142    abl L0H3 | 0.166
abl L1H0 | 0.034    abl L1H1 | 0.047    abl L1H2 | 0.043    abl L1H3 | 0.029
```

这里数字是「准确率**下降量**」。敲掉任何一个 L1 头都让准确率几乎不动（下降 0.03~0.05，因为冗余）……等等，先别误读——这些是单头下降量，看着小是因为同层其他头能顶上。真正干净的串联判据是**整层敲掉**：

```
敲掉整个 L0 (前位头层): 准确率 0.741 -> 0.318  (掉 0.423)
敲掉整个 L1 (归纳头层): 准确率 0.741 -> 0.360  (掉 0.381)
```

✦ 这就是 induction circuit 的定义性证据，也是本章最该记住的一条：**两层各自敲掉都让准确率显著塌（各掉 0.4 左右），说明它们是串联（series）不是并联。** 如果是两个独立并联的解法，敲掉其中一层另一层还能撑住，准确率不会塌这么多。L0 损伤比 L1 还大（0.423 > 0.381），方向也对——L1 的归纳依赖 L0 搬过来的 key，L0 一断，L1 的 query 无处可匹配，整个电路失效。这把第三节的相关性升级成了因果：**对角图案的因，是 L0 前位头在每个位置写入的「我跟在谁后面」信号**。

注意逐头消融揭示的另一件事：**每层的电路是冗余分布在多个头上的**，没有单个头独占。敲掉单个 L1 头只掉 0.03~0.05，因为同层另外两个归纳头能补上。这一点在第七节诚实边界里很重要——真实模型的冗余只会更多。

## 五、相变：电路是「啪」一下成形的，不是匀速长出来

induction head 之所以在可解释性圈子里地位特殊，是因为 Anthropic 在《In-context Learning and Induction Heads》里把它和训练过程中的一个**相变**对应起来：训练到某个点，loss 曲线上会有一个小台阶（in-context 能力突然出现），同时 induction head 在这个点成形。换句话说，「模型突然学会从上下文里学」这件事，机制上对应「induction 电路突然接通」。

我们在 toy 规模上复现这个同步相变。`stage06` step 4 用**同一架构、同一随机种子**，只改训练步数，每个点都是一个真实训练出来的 checkpoint，测它的 in-context 准确率：

```
in-context 准确率 vs 训练步数
  50 steps | 0.338
 150 steps | 0.426
 400 steps | 0.448
 800 steps | 0.506
1400 steps | 0.699
2000 steps | 0.733
```

**最大单段跃升 = +0.193，出现在 ~1400 步**。从 800 步到 1400 步，准确率从 0.506 跳到 0.699，是其他相邻段（每段 +0.02~0.06）的好几倍。曲线不是匀速爬坡，而是有一个明显的拐点——电路是较突然接通的。固定种子是关键：让「训练步数」成为唯一变量，跃升才能归因到电路成形而不是初始化运气。

⚡ **诚实标注一个前沿差异**：真实大模型的 induction 相变机制更复杂，且尚无完整理论。Anthropic 那篇把相变和「双 token 重复的 in-context loss」专门指标的骤降对应起来，背后涉及多个 induction head 的协同成形、与其他电路的竞争、以及具体在哪一步成形对超参/数据分布的敏感依赖——**这些为什么会发生、能不能预测相变点，目前没有通用解，仍在研究**。我们这个 toy 复现的是相变的**现象学**（曲线有拐点），不是它在真模型里的**机制成因**。把 toy 的 +0.193 跃升等同于真模型的相变机制，是过度外推。可迁移的是「固定种子扫步数、看曲线有没有拐点」这个方法，不是这个具体数字。

## 六、失败模式：1 层模型为什么机制上注定学不会

前面说过变偏移任务逼出内容匹配。现在用它来做最干净的对照实验：把同样的任务给一个**1 层模型**训，而且故意多给步数（2500 步）。如果它也能学会，那「必须两层串联」的整套论证就垮了。

```
in-context 准确率: 2 层 vs 1 层
   2L (可串联) | 0.741
  1L (无法串联) | 0.449
 chance(1/V) | 0.111
```

1 层最终 loss = 1.1125（远高于 2 层的 0.5751），准确率 44.9% vs 2 层 74.1%。1 层确实比瞎猜（0.111）强——它能学到一些位置统计和填充模式的捷径——但**撞在一个天花板上下不来**。

这个天花板不是调参能突破的，是**机制上的硬约束**。回到电路定义：归纳头的 query 要匹配的是「我紧跟在 X 后面」这个 key。这个 key **不是输入里现成就有的**——输入每个位置只有「我是谁」，没有「我跟在谁后面」。这个 key 必须由**一个更早的注意力层**把前一个 token 的身份搬过来才会存在。1 层模型没有「更早的层」，所以根本没有可匹配的 key，**归纳电路在结构上无法构造**。再多训练步数也只是在没有归纳能力的前提下榨干位置捷径，撞天花板。

这是本章最值得带走的一条判断方法：**当你怀疑某个能力需要某种结构时，去训一个机制上「残废」的对照模型**，看它撞不撞天花板。能力上的差距（74.1% vs 44.9%）配上机制上的解释（无更早层 → 无可匹配 key），比任何注意力图都更有说服力。这比单看「2 层能、所以需要 2 层」强——后者可能只是 2 层参数多。1 层我们给了更多步数，排除了「只是没训够」。

## 七、诚实边界：哪些是真的，哪些别外推

这是一个 toy，照搬数字到真实模型会出错。明确划界：

**会迁移的（方法和形状）**：

- 方法四件套——按电路预测的源位置给注意力打分定位头 → ablation 验证必要性 → 对照机制上残废的基线（1 层） → 扫步数看相变。这套流程在任何规模都成立。
- 电路形状——前位层在下、归纳层在上、两层串联缺一不可。这是 induction circuit 的结构定义，真模型也是这个形状。

**不会迁移的（具体数字和「干净度」）**：

- **冗余**：本 toy 里前位头集中在 L0、归纳头集中在 L1，但每层都冗余分布在多个头上（敲任一 L0 头都掉准确率，没有单个头独占电路）。真实模型的冗余只会更多——很多个 induction head、模糊（fuzzy）匹配的头、以及一头多职（一个头同时干好几件事）。「找到那个 induction head」在真模型里通常是个伪命题。
- **74.1% 这个绝对数**：没到 100% 有两个具体原因——变偏移匹配对一个万参数模型本就难；填充 token 偶尔和块 S 内容碰撞，造成源位置歧义（词表才 8 个）。这是 toy 规模的产物，不是 induction 机制的固有上限。
- **+0.193 的相变幅度**：见第五节 ⚡，真模型相变机制更复杂且无通用解。

一句话总结这章的位置：我们把第 02 章的注意力图和第 05 章的 patching 合到一起，从权重里拆出了 induction 电路并因果验证了它的串联结构。但全程我们拆的是**电路（circuit）级**的东西——「哪些头串起来实现了这个行为」。下一章（第 07 章）会换一个视角问：能不能不看头与头的连接，而是直接找出残差流里被激活的**特征（feature）**？电路级和特征级是两套互补的解释语言，各有各坏的地方——这正是第 07 章要对照的。
