先把一个常见误会掐掉：在 transformer 上画一张注意力热图，看到某个头在「=」位置死死盯着第一个操作数，然后说「这个头负责了加法」——这不是机制可解释性，这是看图说话。问题不在图画得好不好，在于**注意力图是相关性证据**：它告诉你一个头往哪儿看（attention pattern，即每个 query 位置对各 key 位置的权重分布），从不告诉你模型的答案是否真的依赖它往那儿看到的东西。本书第 02 章会直接演这个坑：挑出全模型里 pattern 最尖锐、最像「电路」的那个头，把它消融（ablation，把某组件的激活置零或换成均值看答案掉多少），结果答案几乎不动——图骗了你。

机制可解释性（mechanistic interpretability，简称 mech interp，意思是「把网络当成可拆的机器逆向出它的算法」）要做的是另一件事：对一个**已经训练好**的网络，找出哪些具体权重和激活组合在一起，执行了某个具体行为——这个组合叫**电路**（circuit，意思是「跨层协作完成一件事的若干组件」）。判据是可证伪：每个「我发现了 X 电路」的论断，都要能用一次干预实验推翻或证实，而不是用一张漂亮的图说服你。

这就是本书和「AI 批量生成的可解释性 survey」的根本分野。survey 会告诉你有 probing、有 logit lens、有 attention 分析、有 activation patching、有 SAE，列成五个并列的 bullet。本书的第一个判断是：**这五样不是五个并列方法，是同一件事的不同切面**——在网络的命名激活点上读取或干预。把这一点讲透，比把五个名词各介绍一遍有用得多。

## 研究对象：为什么是几千参数的玩具，而不是 GPT

要让「我发现了电路」可证伪，你需要一个有**精确地面真值**的研究对象。本书训一个几千参数的 `TinyTransformer`（1-2 层、1-4 头、d_model 16-32）去解合成任务——模加法 / 复制 / induction / skip-trigram，每个任务的正确答案能用一行代码精确算出（task oracle，即「不靠模型、直接算出标准答案的函数」）。

研究对象的定义在 `examples/interpretability-from-scratch/src/core/nn.ts`，注释把尺寸选择的理由写死了：

```ts
// SIZE BY DESIGN: 1-2 layers, 1-4 heads, dModel 16-32, a few thousand params. Small enough
//   to train on CPU in seconds AND small enough that a human can hold the whole circuit in
//   their head — which is the only regime where "we fully reverse-engineered it" is honest.
```

「小到一个人能把整个电路装进脑子」——这是「我们完全逆向了它」这句话唯一能诚实成立的尺度。在 GPT 规模上，没人能声称完全逆向，所有结论都是局部、概率性的。玩具模型用尺寸换来了可证伪性。

代价必须当场摊开，否则就是另一种 survey 式的过度承诺：**toy 模型的绝对数字偏乐观，不外推**。探针打到 100%、patch 恢复 100% 的 logit 差距，在一个干净的合成电路上是正常的，绝不意味着真实模型这么干净。`core/nn.ts` 把这条诚实边界钉进了源码注释，并要求每个 stage 重复一遍：

```ts
// HONESTY (repeated in every stage): absolute numbers from this model (loss, probe acc,
//   patch recovery, SAE feature count) are OPTIMISTIC because the task is synthetic and the
//   model tiny. What transfers to real models is the MECHANISM and the SHAPE of curves, not
//   the magnitudes.
```

可迁移的是**机制和曲线形状**，不是数值：induction head 的 QK 对角化模式、logit lens 逐层逼近答案、patching 把因果责任集中到少数组件、SAE 在叠加下拆出可命名特征。这些关系在任何尺度成立；绝对值不成立。

## 统一框架：一个 hook，五个动词

把五样工具捏成一件事的，是 `core/nn.ts` 里的 **hook 系统**——一个让你在网络计算的命名点上插手的机制。每层的 `forward` 接一个可选的 `hooks` 对象，在固定命名点（`resid_pre / attn_out / resid_mid / mlp_out / resid_post / head_z`）调用一个钩子，钩子能**观察**激活（拿去做缓存、logit lens、probing、SAE），也能**替换**激活（返回一个新张量，用于 patching、ablation）。

hook 的契约只有一句话——纯函数：拿一个张量，返回一个张量。返回原张量就是观察，返回新张量就是干预。源码注释把这条不变量和它防的失败模式写在一起：

```ts
// A hook receives the activation at a named point and returns what should flow onward.
// Return the same tensor to observe-only; return a new tensor to intervene.
export type Hook = (name: string, value: Tensor) => Tensor;

// FAILURE MODE this guards against: a hook that mutates the activation in place instead of
//   returning a replacement — that would corrupt the cached "clean" copy patching relies
//   on. Hooks MUST be pure: take a Tensor, return a Tensor (the same one to observe-only).
```

为什么坚持一个统一钩子，而不是每个实验各搞各的探针？因为跨技术交叉印证是这门学科的核心动作——「patch 落在 lens 指出的那一层了吗？」这个问题只有在「lens 说的第 1 层」和「patch 改的第 1 层」是同一个东西时才有意义。`core/nn.ts` 的注释把这点说死了：

```ts
// WHY a unified hook point instead of ad-hoc instrumentation per experiment: if logit lens
//   read the residual stream one way and patching grabbed it another way, the two could
//   disagree about WHAT "the residual stream at layer 1" even is, and cross-experiment
//   corroboration (the whole point — does the patch land where the lens says?) would be
//   meaningless. One hook taxonomy = one ground truth for "where" in the network.
```

工具箱本体在 `core/interp.ts`。把它的开头注释翻成动词表，五样工具立刻塌缩成三种 hook 用法：

- **缓存**（`runWithCache`）= 观察并存下来的 hook。一次前向，把每个命名激活点 detach（脱离计算图，防止后续分析把梯度写回来）存一份，这是其它所有工具的原料。
- **probing**（`linearProbe`）/ **logit lens**（`logitLens`）/ **SAE**（`trainSAE`）= 在缓存的激活上**读取**。读什么不同，读这件事相同。
- **patching**（`patch`）= 观察并替换成另一次运行的激活的 hook。
- **ablation**（`ablate`）= 观察并替换成零 / 均值的 hook。

`core/interp.ts` 的设计原则注释，一句话把它们焊死：

```ts
// DESIGN PRINCIPLE: every tool here is built on the ONE hook mechanism in nn.ts. Caching is
//   "hook that observes and stores". Patching is "hook that returns a stored activation from
//   another run". Ablation is "hook that returns a zeroed/mean-replaced activation". Because
//   they share the hook taxonomy, a head named in one tool means the same thing in another —
//   that consistency is what lets findings corroborate across techniques.
```

「在一个工具里命名的头，在另一个工具里指的是同一个东西。」这就是统一框架的全部价值。

### 残差流：为什么 logit lens 几乎是免费的

有一个细节值得单独拎出来，因为它解释了为什么这套工具能成立——**预归一化残差流的可加性**。`core/nn.ts` 的 `TransformerBlock` 用 pre-norm（先归一化再进子层，子层原始输出直接加回残差流），注释讲了原因：

```ts
// PRE-NORM (norm before the sublayer, add the raw sublayer output to the residual): this
//   keeps the residual stream a clean linear accumulator — each component WRITES to it
//   additively. That additivity is exactly what logit lens and patching exploit: the
//   stream at any point is a sum of contributions you can read or swap.
```

残差流（residual stream，意思是「贯穿各层、各组件往里累加输出的那条主干向量」）是个干净的线性累加器。因为它是累加的，你可以借用最终的 LayerNorm + unembedding，套到一个**更早的、部分的**和上，问「如果模型在这儿就交卷，它会说什么」。这就是 logit lens（第 04 章）——不用训练，逐层看答案如何逐渐清晰。`core/interp.ts::logitLens` 一共就两行，但它也诚实标注了 caveat：早层的残差流对最终 LN 的统计量是离分布的，所以早层 lens 的概率是近似的，可信的是**单调逼近这个形状**，不是早层的精确数值。

## 暗线一：叠加，特征比维度多

整本书有两条贯穿的暗线，第一条是**叠加**（superposition，意思是「网络把比维度数更多的特征挤进同一组神经元里」）。它直接决定了「读神经元」这条路为什么走不通。

一个训练好的 transformer 里的 MLP 神经元，很少只表示一件事——它会对几个不相关的输入都激活。所以「看这个神经元在干什么」是死路。第 07 章的工具是稀疏自编码器（SAE，sparse autoencoder，意思是「学一个比维度更宽的字典，把激活重写成少数字典原子的稀疏和」）。`core/interp.ts::trainSAE` 的注释把动机和诚实边界放在一起：

```ts
// WHY: a neuron in superposition encodes several features at once, so reading neurons
//   directly is misleading. An SAE learns an OVERCOMPLETE dictionary (more features than
//   dims) with an L1 sparsity penalty, so each activation is reconstructed as a sparse sum
//   of dictionary atoms — and those atoms are often monosemantic ("nameable").
// HONESTY: on a tiny model with few real features the SAE easily finds clean atoms; at scale
//   features are messier and dead-feature / shrinkage problems dominate.
```

⚡ **前沿/开放问题——叠加的可操作化，目前无通用解，正在研究**。Anthropic 在 2023-2024 的 SAE 路线（"Towards Monosemanticity" 及后续的 dictionary learning 工作），把「叠加」从一个理论猜想变成了能动手的工具：用稀疏自编码器在过完备字典里拆出**单义特征**（monosemantic feature，意思是「只对一类语义输入激活的特征」）。本书第 07 章在 toy 模型上复现这套拆解，并诚实标注玩具和 Claude 规模的差距。但要清楚标注：这远不是解决了的问题。**SAE 本身有一堆未解的病**——死特征（dead feature，训练后从不激活的字典原子）、特征收缩（shrinkage，L1 惩罚把激活值系统性压小）、字典宽度该取多大没有原则性答案、不同 seed 训出的字典不完全对得上、以及最根本的——拆出来的特征到底「真」还是只是数据集的统计伪影，没有公认判据。第 07 章会故意把 L1 惩罚拧到极端，演字典坍缩这个失败模式。把 SAE 当成「叠加问题已解」是错的；它是目前最好的可操作工具，不是终点。

## 暗线二：相关不等于负责

第二条暗线，也是这门学科的成人礼：**相关 ≠ 因果**。

✦ 这里是本书最该被记住的一句判断：**可解释性的核心不是「可视化」，而是「因果」**。一张漂亮的注意力图、一个高分探针、一条逼近答案的 logit lens 曲线——全是相关性证据。注意力图说一个头往哪看，探针说信息在那一层**线性可读**，lens 说答案在某一深度**浮现**。没有一个能证明那个组件被模型**用到了**。只有 activation patching（激活修补，又叫 causal tracing，意思是「把一次运行的某个激活换成另一次的，看行为是否随之改变」）这种干预实验能。从相关到因果的跨越，是这门学科的成人礼，本书第 05 章兑现。

patching 的逻辑，`core/interp.ts::patch` 的注释讲得最干净：

```ts
// THE CAUSAL EXPERIMENT: run the model on a CLEAN input (gives the right answer) and a
//   CORRUPT input (breaks it). Then re-run CLEAN but OVERWRITE one activation with the value
//   it had in the CORRUPT run. If that single substitution destroys the clean answer, that
//   activation is CAUSALLY responsible. ... This is correlation-proof evidence,
//   unlike a probe (which only shows information is present, not that it's USED).
```

恢复率（recovery）量化「这一次替换重现了多少 clean→corrupt 的 logit 落差」：接近 1 = 这个组件单独就解释了行为，接近 0 = 它在这里无关。**恢复集中在少数组件 = 局部电路；恢复弥散 = 分布式计算**。这是 probe 给不了的——probe 只说信息在场，不说信息被用。

第 05 章还会强调一个大家容易做错的设计点：clean 和 corrupt 必须**结构完全相同、只差一个语义因子**，恢复的 logit 差距才能干净地隔离出那个因子的电路。`stage05` 的注释里，clean=[a,b,=] 和 corrupt=[a',b,=] 只在第一个操作数上不同——能恢复差距的组件，就是携带操作数 `a` 的那个。corrupt 设计错了，整个因果结论就废了。

ablation 是因果家族的另一半，问的是**必要性**（去掉它答案掉多少）。但它自带一个失败模式，`core/interp.ts::ablate` 标得很明确：零消融一个被 LayerNorm 过的流，可能因为破坏了归一化而看起来灾难性，跟电路本身无关——所以均值消融是更诚实的默认值。这类「工具本身怎么骗你」的细节，正是本书相对 survey 的密度差所在。

## 全书地图：每章在同一个研究对象上验一种方法

把工具链摊在一张表上，配上它属于哪种证据、坑在哪、代码在哪：

| 章 | 方法 | 证据类型 | 核心失败模式 | 代码 |
|---|---|---|---|---|
| 01 | 训研究对象 + grokking | —（前提） | train 100% ≠ 学会（记忆 vs 泛化） | `src/stage01-train-toy.ts` |
| 02 | 注意力可视化 | 相关 | 尖锐 ≠ 重要（salience ≠ importance） | `src/stage02-attention-viz.ts` |
| 03 | 线性探针 | 相关 | 没有 shuffled-label 基线的探针在撒谎 | `src/stage03-probing.ts` |
| 04 | logit lens | 相关（准因果深度图） | 早层离分布，看形状不看数值 | `src/stage04-logit-lens.ts` |
| 05 | activation patching | **因果** | corrupt 设计错则结论全废 | `src/stage05-activation-patching.ts` |
| 06 | induction head | 因果 + 机制 | 1 层模型机制上不可能做到 | `src/stage06-induction-head.ts` |
| 07 | 稀疏自编码器 | 读取（拆叠加） | L1 过大 → 字典坍缩 / 死特征 | `src/stage07-sae.ts` |
| 08 | 定位流水线 | 相关 × 因果交叉印证 | 全绿相关 + 因果为零的组件 | `src/stage08-pipeline.ts` |

两个串联点值得提前讲，因为它们是全书叙事的脊椎。

**第 01 章先把研究对象训出来，而且坚持要它真正泛化。** 这不是凑个起点——`src/stage01-train-toy.ts` 的注释把「为什么必须先证明泛化」讲死了：

```ts
// (2) the model actually GENERALIZED the planted structure rather than memorizing the training
//   pairs (else we'd reverse-engineer a lookup table, not an algorithm).
```

如果模型只是背下了训练对，你逆向出来的「电路」就是一张查找表的碎片，不是算法。所以 stage01 用真正的 train/test 划分演 grokking（顿悟，意思是「训练准确率早早满分后，测试准确率延迟很久才突然跳起来」）：在 modAdd(23) 这个 529 对的玩具任务上（chance = 1/23 ≈ 0.043，泛化门槛设在 0.9），有 weight decay 时 train/test 曲线先分离后重合（记忆先于泛化），无 weight decay 时持续分离（只记忆不泛化）。注释把数值的诚实边界也写清了——相变发生的**步号**随种子抖动且偏乐观，不可迁移；可迁移的是这个**曲线形状**。所以本章不在这里引用任何「相变发生在第 N 步」的具体数字，那正是不该外推的东西。

**第 08 章是收口**，把所有工具按固定顺序跑在同一个行为上，拼一张证据矩阵，再只用因果工具（patch 恢复 × ablation 必要性）算一个忠实度分数。`src/stage08-pipeline.ts` 故意制造的失败模式，是全书论点的浓缩：

```ts
// THE FAILURE MODE THIS STAGE EXISTS TO SHOW: we deliberately surface a component where the
//   CORRELATIONAL evidence is all green (attention high AND probe high) but the CAUSAL evidence
```

一个组件，注意力高、探针高、相关证据全绿——但因果证据为零。这个组件就是整本书的反面教材：如果你停在相关层，你会拍着胸脯说它是电路；只有干预实验拦住你。这就是为什么忠实度（faithfulness，意思是「解释能否预测干预后的行为」）只能从因果工具算，不能从「故事自洽」算。

## 怎么读这本书

如果你是来要结论的——「注意力头 N 负责了任务 X」——这本书会让你失望，因为它的全部努力是教你**不要相信没经过干预实验的结论**，包括它自己印出来的。每个 stage 都离线 CPU 几秒跑完，打印真算 / 真测的数字，并且**必演一个失败模式**，不只走 happy path。

往下读的顺序就是 stage 顺序。第 01 章把研究对象训出来——没有一个真正泛化了的模型，后面所有电路都是在拆查找表。读完第 01 章，你会第一次拿到一个「值得逆向」的对象。
