第 02 章的注意力图能让你看出「=」位在汇聚两个操作数，但它有个硬上限：注意力权重只描述 query 看了哪些 key，不描述残差流（residual stream，每个 transformer 组件都读写的那条共享数据总线）里实际躺着什么数。一个头权重 0.5 0.5 分给两个操作数，它搬过去的是 `a` 和 `b` 本身、是 `a+b`、还是某个旋转后的傅里叶相位？看权重永远看不出来。本章换一把更定量的尺子：线性探针（linear probe，在冻结激活上训一个线性分类器，问「这个量在这里能不能被线性读出」)。

先把这把尺子的刻度对准。`examples/interpretability-from-scratch/src/stage03-probing.ts` 复用第 01 章那个 `modAdd(7)`（7 以内模加法）的共享 checkpoint——同任务、同默认配置、同缓存权重，所以本章的所有结论都是关于第 01 章验证过的那同一个对象，`finalLoss=0.0005`。它跑出来的开场：

```
=== Stage 03: 线性探针 — 各层残差里编码了什么 ===
研究对象: modAdd(7)  (复用 stage01 共享 checkpoint, finalLoss=0.0005)
样本数=300  类别数=7  纯随机基线(1/类别)=0.143
```

7 个类别，纯随机猜中概率 `1/7 = 0.143`。记住这个 0.143,本章三分之二的论点都是在跟它和另一个更阴险的基线较劲。

## 探针在问什么，不在问什么

探针的全部语义就一句话：**在网络的点 P，特征 F 能不能被一个线性读取器提取出来。** 高探针准确率意味着「一个下游的线性组件如果想用 F，它在 P 这里读得到」。注意这是个关于「可读性」的命题，不是关于「模型实不实际用」的命题——这条区分是本章末尾整个争议的根。

为什么是**线性**探针而不是塞个两层 MLP 上去?因为探针越强,它越能自己把答案算出来,而不是把模型已经算好的答案读出来。一个深 MLP 探针在「操作数 a 在 pos0」上拿满分,你分不清这是「a 已经线性可读」还是「MLP 自己学会了从 a 的 embedding 里恢复 a」。线性探针把探针自己的算力压到最低,逼出来的高准确率才更可能是模型的功劳。这是 probing 这套方法论的隐含契约:探针的弱,是它作为测量工具可信的前提。

代码里只探**残差流上的点**,跳过单个 head / mlp_out 的私有输出:

```ts
function residualPoints(model: TinyTransformer): string[] {
  const pts = ["embed"];
  for (let l = 0; l < model.cfg.nLayers; l++) {
    pts.push(`blocks.${l}.resid_pre`, `blocks.${l}.resid_mid`, `blocks.${l}.resid_post`);
  }
  return pts;
}
```

理由写在源码注释里,值得抄过来:残差流是每个组件都读写的共享总线,所以它是诚实地问「F 在这个深度可读吗」的地方,不会把它和某单一组件的私有输出混为一谈。`resid_pre / resid_mid / resid_post` 分别是一个 block 进入前、attention 写回后、MLP 写回后的三个采样点——这样我们能看「答案」沿深度逐步组装的过程。

## 答案在哪一层突然变得可读

第一个探针:逐层问「残差流能不能线性解码答案 `(a+b)%7`」。每层打印两条——`probe` 是用真标签训出来的准确率,`shuf`(后面细说)是打乱标签的对照。

```
answer-probe accuracy
  embed probe | ###### 0.167
   embed shuf | ###### 0.167
 b0.pre probe | ###### 0.167
  b0.pre shuf | ###### 0.167
 b0.mid probe | #################################### 1.000
  b0.mid shuf | ############ 0.320
b0.post probe | #################################### 1.000
 b0.post shuf | ############# 0.357
 b1.pre probe | #################################### 1.000
  b1.pre shuf | ############ 0.337
 b1.mid probe | #################################### 1.000
  b1.mid shuf | ############ 0.340
b1.post probe | #################################### 1.000
 b1.post shuf | ############ 0.323
  最强层: blocks.0.resid_mid  probe=1.000  shuf=0.320  gap=0.680
```

读这张图要看「突变点」。`embed` 和 `b0.pre` 处答案探针都是 0.167——基本贴着随机线,意思是这两层的残差流里没有任何线性可读的答案信息,合理:模型还没开始算。然后到 `blocks.0.resid_mid`,也就是第 0 层 attention 刚写回残差之后,探针**直接跳到 1.000**。答案在这一步被组装出来并写进了「=」位的残差流。

这个台阶式的可读性曲线,正是 probing 作为定位工具的价值:它把「计算发生在哪」从一句定性的话变成了一个能指到具体层的箭头。源码的解读跟这一致——「答案在『=』位的残差流里越深越线性可读,与『算完才写回残差』一致」。注意诚实边界:`probe=1.000` 是 toy 任务的乐观绝对值,真实模型上同一个探针的准确率会随探针容量和样本数剧烈漂移。可迁移的不是这个 1.000,而是**形状**:某个特定层出现可读性突变。

## 信息是位置绑定的,探针必须指定位置

第二个探针把一个容易被忽略的事钉死:探针读的不是「模型整体」,而是「某个位置上某个量」。代码对操作数 `a` 在两个不同位置各训一个探针——pos0(token a 实际所在的位置)和「=」位:

```
operand-a @ embed (two positions)
a @ pos0 probe | #################################### 1.000
 a @ pos0 shuf | ####### 0.207
 a @ "=" probe | ###### 0.167
  a @ "=" shuf | ###### 0.167
        chance | ##### 0.143
```

同一个量 `a`,同一个 embed 层,只是问的位置不同,探针结果从满分 `1.000` 掉到约等于随机的 `0.167`。原因在源码注释里讲透了:token `a` 住在 pos0,所以那里的 embed 直接解码出 `a`;而「=」位的 embed 是那个常量 EQ token——在 attention 把 `a` 的值搬过来之前,它根本无从知道 `a` 是几。

这给一个实操教训:**一个忽略位置、把所有位置的激活混在一起训的探针,会报告一个误导性的中间数。** 它会把 pos0 的满分和「=」位的随机平均成一个不上不下的值,然后你对着这个值得出「a 的信息半可读」的错误结论。正确做法是结构按「层 + 位置」二维定位,不是给模型整体报一个数。

## 失败模式:没有对照的探针准确率会骗人 ✦

现在讲本章的专家钩子,也是 probing 整套方法里最容易被新手跳过、被审稿人第一个揪住的点:**裸报一个探针准确率是无意义的,它必须显著超过同一个探针在打乱标签上的准确率才算证据。**

为什么?因为线性探针虽然弱,但只要它有足够容量、样本量又有限,它就能靠死记硬背 `(激活 → 标签)` 这张表,把**随机标签**也拟合到高于随机的程度。`shuf` 那条线就是干这个的:把标签和激活的对应关系彻底打乱后再训探针,此时激活里**没有任何关于这个乱标签的真信息**,探针能拿到的全部准确率都来自纯记忆。代码用 `interp.linearProbe` 同时算出真标签和打乱标签两个数,把这个陷阱直接打印出来:

```
[3] 失败模式: 没有对照的探针准确率会骗人
    打乱标签后最高基线: blocks.0.resid_post shuf=0.357 (纯随机应为 0.143)
    超出纯随机 +0.214 = 纯记忆带来的虚假准确率。
    教训: 报告 probe=0.9 而不报 shuf 基线, 等于把记忆当成"信息存在"的证据。
```

看清这个数:`blocks.0.resid_post` 的打乱标签探针拿到了 `0.357`,而纯随机应该是 `0.143`。中间这 `+0.214` 是凭空冒出来的——激活里没有这个乱标签的信息,这 0.214 纯粹是探针把 300 个样本里一部分 `(激活, 乱标签)` 对背了下来。换句话说,如果你在某个嘈杂的真实层上看到 `probe=0.36`,不报基线,你会兴奋地宣布「这一层编码了答案」;但它可能跟这里的 `0.357` 一样,全是记忆,零真信息。

这就是为什么源码注释把这条对照叫 "THE NON-NEGOTIABLE CONTROL"。判据不是 `probe` 的绝对高度,而是 **`probe` 和 `shuf` 之间的 gap**。回看上一节那个最强层:`probe=1.000  shuf=0.320  gap=0.680`——0.68 的 gap 才是「答案真的在这层线性可读」的证据,而不是那个 1.000。把这两个数分开看,你才不会被记忆抬升骗过去。

这也顺带给了一个量化的「探针容量警告」:在这个干净 toy 模型、300 样本下,记忆就能把基线从 0.14 抬到 0.36。样本越少、探针越强、类别越多,这个虚假抬升越凶。真实模型上探针动辄上千维输入、几万样本,基线漂移更难肉眼判断——所以基线不是可选项,是探针报告的必填字段。

## PCA 投影:模运算的环,看见了一点,但别夸大

最后一个探针换个角度。`modAdd` 已知的代数解法之一是把数字嵌到一个圆环上,加法变成旋转(这是 Nanda 等人 grokking 模加法工作里的经典发现)。如果这个 toy 模型也学到了这个结构,那操作数 `a` 的表征投影到二维平面上,应该按 `0,1,2,...,6` 的顺序绕成一个环。代码对每个操作数取一份 embed 激活,做 top-2 PCA(主成分分析,把高维压到方差最大的两维)投影:

```
embed PCA (label = operand a)
|                     1  2                     |
|3                    6                        |
|                         4                    |
|                                             5|
|              0                               |
  x:[-0.17,0.19] y:[-0.13,0.14]
  环状一致性: 3/7 个相邻角度对是模 p 相邻 (=0.43; 随机期望≈0.29, 完美环=1.0)
```

诚实地讲:这不是一个教科书式的干净圆环。代码用一个真实的几何判据量化了「像不像环」——把所有点按绕中心的角度排序,数有多少对角度相邻的点其操作数也恰好模 7 相邻(±1)。结果是 `3/7 = 0.43`,高于随机期望的 `0.29`,但远低于完美环的 `1.0`。

关键是源码对这个数的态度,值得整个团队学:**它没有只凭散点图断言「学到了环」或「没学到环」。** 注释明说,top-2 PCA 把 `d=32` 维压成 2 维会丢掉真实环面所在的高维子空间,所以「看不见干净环」既可能是模型没学到、也可能是投影损失把它压没了——两种可能在这张图上无法区分。这个 2 层模型在 400 步只学到了部分旋转结构,是诚实的判断;把它说成「模型学会了傅里叶环」是过度解读。

这一节的方法论价值不在那个 0.43,而在这种克制:**降维可视化是假设生成器,不是证明。** 散点图能让你产生「这里可能有环结构」的假设,但要证实,你得用能保留高维信息的工具(比如直接探傅里叶频率),不能靠肉眼看二维散点。

## 探针读出了 ≠ 模型用了:交给第 05 章的裂缝 ⚡

现在把本章最大的一条裂缝撑开。我们证明了答案在 `blocks.0.resid_mid` 线性可读、操作数位置绑定、gap 远超基线。一切看起来很扎实。但有个问题探针**结构上回答不了**:模型在前向计算里,到底有没有**用**这个可读的信息?

这不是抬杠。一个量在某层线性可读,完全可能是计算的副产品——它躺在残差流里,但下游组件可能根本没读它,真正驱动输出的是另一条路径。探针只证明了「信息在那」(correlation),没证明「信息被用了」(causation)。这又是一次相关≠因果,跟第 02 章注意力图的局限是同一个病根:观察性证据看到的是「同时出现」,推不出「因为所以」。

这条裂缝不是我编的,是 probing 文献里争了多年的核心议题。Hewitt 和 Liang 提出 **control task**(对照任务):给每个词随机分配一个固定的假标签,如果探针在这个毫无语言学意义的假任务上也拿高分,说明探针测的是自己的容量而不是表征里的真结构——本章的 `shuf` 基线就是这个思路的一个简化版实例。Belinkov 在他对 probing 的系统性批判里把话说得更直接:探针准确率高,只能说明信息**可被解码**,不能说明这信息**对模型的行为有因果作用**。

正是这条「能读出 ≠ 在用」的批判,直接催生了**干预式探针**和因果中介分析(causal mediation analysis)这一支:不要只是被动读激活,而是主动去**改**激活,看模型输出变不变。如果你把 `blocks.0.resid_mid` 里那个可读的答案信息抹掉或替换,模型的预测随之改变,你才拿到了「模型确实在用它」的因果证据。

**这个问题目前没有通用解。** 因果中介、干预式探针、activation patching 都是在不同假设下逼近因果,但「如何严格证明一个内部表征对模型输出有因果贡献、且不被其他冗余路径补偿」仍是开放研究——尤其当模型有多条冗余计算路径时,抹掉一条另一条会顶上来,让单点干预的结论失真。本章只负责把这条裂缝撑开、量化到能看见的程度;闭合它的第一把工具——activation patching(激活补丁,把一次运行的某段激活换成另一次的,看输出怎么变)——是第 05 章的主题。

带走两条能迁移到任何真实模型的形状,不是那些 toy 的乐观绝对值:

1. **探针准确率必须显著超过它自己的打乱标签基线才算证据。** 本章 `0.143 → 0.357` 的记忆抬升就是反例——不报基线,你会把纯记忆当成「信息存在」。判据是 gap,不是绝对高度。
2. **内部结构按「层 + 位置」定位,不是给模型整体报一个数。** 同一个量在 pos0 满分、在「=」位随机;答案在 `b0.mid` 突然可读——结构是有坐标的。

下一章我们换一把不需要训练探针的尺子——logit lens(把中间层激活直接投到输出词表上看它「现在想说什么」),它和探针抓的维度互补;第 05 章再用 patching 把本章撑开的因果裂缝正式闭合。
