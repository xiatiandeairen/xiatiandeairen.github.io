注意力图是整个 transformer 里最容易画、也最容易骗人的东西。第 01 章我们已经把「先确证模型真学会、再开始解剖」这条纪律立住了——那一章训的是模加法。本章换任务：stage02 自己训一个 2 层 4 头、`dModel=32` 的玩具模型，跑 copy 任务（序列形如 `[t0..t4][SEP][t0..t4]`，分隔符后每位要复现前半对应 token），训到 `finalLoss=0.0003`、准确率 100.0%（实测 wall-clock 11633 ms）。为什么不沿用第 01 章那个模加法 checkpoint？因为模加法序列只有长度 3，注意力图是张 3×3 的小方块根本没法看；copy 任务长度 11，给出信息丰富的 11×11 热图——这是本章后面 03/04/05/07 章重新回到模加法 checkpoint 的一个例外。模型确实学会了一个真电路——这点很重要，因为**给没学会的模型画注意力图等于解读噪声**，那时候每个头都退化成 attention sink（注意力塌到第 0 位的默认 no-op 行为）。所以本章一上来先做一件事：把这个已经学会的电路里每个头「在看哪」摊开给你看。然后我用本章后半把这张图的可信度按在地上摩擦。

本章对应代码 `examples/interpretability-from-scratch/src/stage02-attention-viz.ts`,它用 `trainToyModel(copyTask(4,5), defaultTrainConfig(task, { steps: 600, seed: 20250219 }))` 现训这个 copy 模型(deterministic 且按 config key 缓存,重跑不会重训)。

## 注意力矩阵从哪来：lastPatterns 这个 side channel

要画图,先得拿到每个头 softmax 之后的注意力权重。`MultiHeadSelfAttention` 在每次前向时把每个头的 pattern 缓存进 `lastPatterns`(一个扁平的 `seq*seq` `Float64Array`)。`runWithCache` 跑一次前向,跑完后这个缓存就反映**那一个输入**的注意力。`stage02-attention-viz.ts` 把它 reshape 成 `(seq, seq)` 矩阵:

```ts
function patternMatrix(model: TinyTransformer, layer: number, head: number): number[][] {
  const flat = model.blocks[layer].attn.lastPatterns[head];
  const seq = Math.round(Math.sqrt(flat.length));
  const m: number[][] = [];
  for (let i = 0; i < seq; i++) {
    const row: number[] = [];
    for (let j = 0; j < seq; j++) row.push(flat[i * seq + j]);
    m.push(row);
  }
  return m;
}
```

读法钉死:**行 i = query 位置**(第 i 个 token 在发问「我该看谁」),**列 j = key 位置**(它在看谁),值越大越关注。因果掩码(causal mask,只允许看自己和前面、不许偷看未来)保证上三角 `j>i` 近似全空——这是一个结构不变量,画出来第一眼就该检查:**如果上三角不空,要么 mask 写错了,要么你在解读一个坏掉的模型**。这是 ASCII 热图为数不多的、真能当 sanity check 用的地方。

为什么用 copy 任务而不是 modAdd 或别的?三个工程理由,代码注释里写得很直白:(1) `seqLen=11` 给出视觉信息丰富的 11×11 图,modAdd 的 3×3 图根本没法看;(2) copy 的解是一个干净的「回看 n+1 格」的 copy head——正是本章要讲的那种**可命名**模式;(3) 它真能秒级训到 100%,我们解读的是真电路不是噪声。

## 把每个头画出来:对角、自指、回看

下面是单个固定输入 `tokens = [0 0 3 2 2 4 0 0 3 2 2]`(SEP=4 在 pos5)上,8 个头的真实注意力图(stage02 实测输出,scale `' .:-=+*#%@'`,越靠右越深表示越关注):

```
  00322400322

0   @        
3    @       
2     @      
2    #.  .   
  scale: ' .:-=+*#%@'  vmin=0.000 vmax=1.000
```

`L0H2` 是全场最像「电路」的头。注意 pos7/8/9 这几行的 `@`(满格关注)排成一条往左下偏的斜线——query 在 pos7 看 pos2、pos8 看 pos3、pos9 看 pos4,稳定回看 5 格。而 copy 任务的几何要求正是「后半 query 回看 n+1 = 6 格」找到自己的前半孪生 token……等一下,代码自动分类给它打的标签是 `back-offset:5` 而不是 6。这个 off-by-one 不是 bug,是真信号:模型学到的回看偏移和你纸上推的几何**差一格**,因为它还要叠加 SEP 的位置错位。**这正是注意力图的价值所在——它让你看见模型实际几何,哪怕和你的先验不符**。但记住这只是「它在看哪」,不是「它有没有用」。

对比一个 `self` 头(query 大量看自己,主对角线)和一个 `diffuse` 头(没有一致目标、注意力摊开):

```
  L1H0  [self, focus=60%, entropy=1.46 bits]        L0H0  [diffuse, focus=30%, entropy=2.16 bits]
  00322400322                                          00322400322
0 @                                                  0 @          
0 -*                                                 0 =+         
3 -+.                                                3 *:.        
2    %                                               2  .*:       
2    .%                                              2 ..=:.      
4 .-.  =                                             4 = .::.     
0 .:.  -:                                            0  :. ..-    
0    =.  -                                           0  .: .-:    
3     :   *                                          3 .....:...  
2     .   -+                                         2   ..:  ..: 
2     .   :-:                                        2   ::.  ....
```

`L1H0` 在前半段贴着对角线(看自己),`L0H0` 则糊成一片、最深也只到 `@`(第 0 位 sink)加零散小点。这套分类用的是一组**故意粗糙**的几何规则——对每个 query 取因果合法 key 上的 argmax,统计落在哪个固定回看偏移最多:偏移 0 = self、偏移 1 = prev-token(前一词,bigram-copy 的前驱)、某个一致的大偏移 = copy/induction 签名、没有一致目标 = diffuse。代码作者在注释里写得毫不含糊:

> 这个分类器是 HEURISTIC。它会给那些根本不真正属于某一类的头也硬贴一个标签——而这恰恰是本章的全部教训:**标签是假设,不是结论(a label is a hypothesis, not a finding)**。

## 熵:量化「看起来有多笃定」

肉眼看图会累,而且会骗。我们要一个数:**注意力熵**(attention entropy,衡量一个 query 的注意力摊得多开)。对一个 query 行(一个在 key 上的概率分布)算 Shannon 熵(bits):0 = 全部质量压在一个 key 上(最尖锐),`log2(n)` = 在 n 个 key 上均匀(最弥散)。熵越低 = 越尖锐 = 越「像关键头」。

这里有个容易踩的坑,代码用一行注释钉死了:

```ts
function meanAttentionEntropyBits(pattern: number[][]): number {
  let sum = 0;
  let counted = 0;
  for (let i = 0; i < pattern.length; i++) {
    const legalKeys = i + 1; // causal: query i may attend to keys 0..i
    if (legalKeys < 2) continue; // a single legal key has no choice -> skip, don't dilute
    // ... 对 j in 0..i 累加 -p*log2(p)
  }
  return counted === 0 ? 0 : sum / counted;
}
```

**为什么跳过只有 1 个合法 key 的 query**(pos0)?因为因果掩码下 pos0 只能看 pos0,熵恒为 0——它没得选,不是它「笃定」。把这种零熵行算进平均会人为压低均值,让每个头看起来都比实际更尖锐。这是个典型的「指标被结构污染」陷阱:你以为在量选择性(selectivity),其实量的是掩码。剔掉无选择的行,数字才反映真选择性。

跑出来的熵排名(越低越尖锐):

```
entropy(bits)
L0H2(back-offset:5) | ############# 0.871
      L0H3(diffuse) | ################## 1.219
         L1H0(self) | ###################### 1.459
         L1H1(self) | ############################ 1.898
      L0H1(diffuse) | ############################## 2.052
      L1H2(diffuse) | ############################### 2.072
      L1H3(diffuse) | ############################### 2.101
      L0H0(diffuse) | ################################ 2.161
```

`L0H2` 以 0.871 bits 一骑绝尘。任何看这张排名的工程师,十有八九会一指点上去:「就是它,这是 copy 电路的核心头。」我们现在就把这个判断送上法庭。

## ✦ 失败模式:salience ≠ importance,2019 年那场争论的核心

这是本章真正的目的,也是 2019 年「Attention is not Explanation」那场争论的核心:**注意力权重显示一个头「在看哪」(salience,显眼),从不告诉你模型「需不需要」它在那看到的东西(importance,必需)**。一个头可以注意力 100% 压在某个 token 上,却对最终 logit 毫无因果贡献——因为下游把它的输出忽略了,或者别的头冗余地做了同样的事。

唯一能回答「必不必需」的,是**因果干预**:把这个头消融(ablate,把它的输出替换掉看模型崩不崩),量行为掉多少。stage02 对最尖锐的 `L0H2` 在 200 例 held-out 上做了消融,而且故意做了两种约定对照:

```
  (a) 最尖锐头 L0H2 (back-offset:5, entropy=0.87 — 人眼会一口咬定"这就是电路"):
      mean-ablation: answer-logit 跌幅=0.561, 准确率 100.0% -> 77.8%
      zero-ablation: answer-logit 跌幅=0.538, 准确率 100.0% -> 83.9%
      => 同一个头, 两种约定给出不同"重要性"; 仅凭注意力图谁都无法预测这两个数字。
```

两件事一起暴露:

**第一,量级你猜不到。** 这个头消融后准确率从 100% 掉到 77.8%(mean)——它确实重要,但「重要」这个程度,注意力图里**没有任何信息能让你提前算出 0.561 这个 logit 跌幅**。0.871 bits 这个熵值和「掉 22 个百分点」之间没有可推导的桥。如果你只有图,你能说的最多是「它看得很专注」,不能说「拿掉它模型废一半」。

**第二,连因果测试本身都不是一个数。** 同一个头,mean-ablation 跌 0.561、zero-ablation 跌 0.538;准确率 mean 掉到 77.8%、zero 掉到 83.9%——**反过来了**。为什么?消融约定是有代价的设计选择。看 `interp.ts::ablate` 的实现:

```ts
const replacement =
  mode === "zero"
    ? Tensor.zeros(ref.shape)
    : meanValue ?? meanRows(ref); // mean over positions if no reference mean supplied
```

zero-ablation 把激活直接置 0,会把残差流(residual stream,各层叠加的主干信息流)推到模型训练时从没见过的分布外区域(off-distribution),可能因为破坏了 LayerNorm 的归一化假设而**高估**重要性——掉得「看起来更惨」未必是这个头更关键,而是你把流弄坏了。mean-ablation 用跨位置均值替换,留下「平均贡献」,off-distribution 更轻,是更诚实的默认。`interp.ts` 注释直说了这是 zero-ablation 的「documented ablation failure mode」。

教训冷酷:**不声明消融约定的「重要性」结论是没有意义的**。一个头是不是关键,取决于你怎么问。

## ⚡ 逐头扫描:7/8 个头是因果惰性的

把消融扩到全部 8 个头(mean-ablation,准确率跌幅,越大越必需):

```
accuracy drop when ablated
      L0H0(diffuse) |  0.000
      L0H1(diffuse) |  0.000
L0H2(back-offset:5) | ################################ 0.222
      L0H3(diffuse) | # 0.004
         L1H0(self) |  0.000
         L1H1(self) |  0.000
      L1H2(diffuse) |  0.000
      L1H3(diffuse) |  0.000
```

**8 个头里 7 个消融后准确率几乎不动(|跌幅|<0.5%)**,尽管它们每一个在前面的图里都有「看着像某种功能」的图案——self、diffuse、各种斜纹。本章训出来的这个 copy 电路,真正承重的就 `L0H2` 一个头。其余 7 个头要么冗余、要么下游根本不读它们的输出、要么它们的「self / diffuse」模式只是 attention sink 的变体,没参与决策。

这里有个诚实但关键的细节:这次,因果最关键的头(`L0H2`)**恰好**就是注意力最尖锐的头。看上去 salience 和 importance 对上了。**但这是 (b) 的因果测试确认的,不是 (a) 的图能保证的**——在更杂的真实模型里,二者经常分道扬镳:最尖锐的头可能是个 attention sink(看着专注其实是 no-op),而真正承重的头注意力图糊成一团。这个 toy 上它俩对齐,是任务太干净的运气,不是规律。可迁移的不是「尖锐头就是关键头」,而是**「先命名假设,再用因果干预证伪,且报告约定」这条流程**。

## 这道坎通往哪:patching、logit lens、QK/OV

到这里,注意力图的角色应该清楚了:它是**生成假设的廉价工具**,不是**验证机制的证据**。它能让你看见模型的真实几何(那个 off-by-one 的回看偏移)、快速 sanity check(上三角空不空)、缩小嫌疑范围。但它给的是相关性,不是因果。

这给后面三章埋了三条线:

- **第 05 章 activation patching**:本章的消融是「拿掉一个头看崩不崩」,patching 更进一步——把干净输入的某个激活搬到 corrupt 输入上,看能不能**修复**行为,直接定位某个组件对某个行为的因果贡献。`interp.ts` 里 `ablate` 旁边那个返回 `recovery`(单组件能复现多少 clean→corrupt 跌幅)的函数就是为它准备的。本章的「salience ≠ importance」是它的动机。
- **第 04 章 logit lens**:把中间层激活直接投影到词表看「模型现在偏向哪个 token」。注意它和注意力图同病——**它也是「读」不是「证」**。读到某层已经偏向正确答案,不等于这层因果上导致了正确答案;同样需要干预来确认。
- **第 06 章 QK/OV 电路分解**:现代电路分析已经基本抛弃「光看注意力图」,转向把一个头拆成两条独立电路——**QK 电路**决定「看哪」(query-key 交互算出注意力分布),**OV 电路**决定「搬什么」(output-value 把看到的 token 信息搬进残差流)。本章只给了 QK 侧的视角(注意力图本质就是 QK 的产物),而**一个头的因果作用主要藏在 OV 侧**:`L0H2` 那 7/8 头里的某些「看着在看东西」却惰性,很可能就是 OV 把它看到的东西映射成了近似零向量。第 06 章会在 induction head 上把 QK/OV 完整拆开,那时你会明白为什么「光看 QK 图」从原理上就漏掉了一半电路。

诚实边界再说一遍(stage02 自己也印了):toy 任务上模式可命名性偏强、本例最尖锐头恰好确实关键,这些都是乐观的。真模型里头更杂——多义(polysemantic,一个头干多件事)、叠加(superposition,多个特征挤在同一维度)。可迁移的是方法,不是这些漂亮数字。下一章我们换一把「读」的工具(logit lens),你会看到它踩进和本章一模一样的坑。
