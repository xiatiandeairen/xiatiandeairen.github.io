LoRA 把可训练参数压到 0.5%，这话只有在你能指着一个「100% 是多少」的基座时才有意义。第 02 章起的每个 PEFT 方法都定义成「冻结 base、加一小撮增量」，可如果 base 本身没被稳定训过，你冻的就是一堆没收敛的垃圾权重，后面所有「增量补上 A→B 任务差距」的对照实验全部失真。所以本章的活很窄也很重：产出**一份可复现的基座 checkpoint**，证明它真学会了任务 A，并且把「让它学得会」的那个前提——数值正确的前/反向 + 稳定的优化——一次性钉死。

代码在 `examples/peft-from-scratch/src/stage01-base.ts`，core 引擎在同目录 `core/`。先跑一遍，本章所有数字都来自这次真实运行，不是引用论文。

## 先证引擎，再谈收敛

训练前第一件事不是训练，是 grad-check。理由很实在：一个写错的 adjoint（反向传播里某算子对应的梯度规则）会产出「看着在收敛、其实梯度方向是错的」曲线，你能对着它调一小时找不到根因。有限差分（finite difference，用 `(f(x+h)-f(x-h))/2h` 近似导数）当场就能抓出来。

`stage01-base.ts` 第 [1] 段对四个算子各跑一次 `numericalGradCheck`：

```
[1] gradcheck max-rel-err (must < 1e-4):
    matmul=4.11e-11  softmax=1.65e-10  layernorm=8.50e-11  crossEntropy=1.37e-10
    ✓ 引擎 adjoint 可信, 训练曲线下降才有意义
```

四个值都在 `1e-10` 量级，离 `1e-4` 的断言阈值有六个数量级余量。这个余量不是凑的：`core/tensor.ts` 的 `numericalGradCheck` 用中心差分（central difference），对 f64 配 `h=1e-5` 精度到 `~1e-6`，所以阈值卡在 `1e-4` 是故意留宽，让真 bug 一定越界、数值噪声一定不越界。它的契约也很硬——要求 `x.requires_grad = true`，否则解析梯度根本没填进去：

```ts
export function numericalGradCheck(
  f: (x: Tensor) => Tensor,
  x: Tensor,
  h = 1e-5,
): number {
  if (!x.requires_grad) throw new Error("numericalGradCheck: x.requires_grad must be true");
```

这里有两个一上来就会坑你的点，stage01 的注释里都标了，值得抄进你自己的脑子。

第一个：**LayerNorm 不能直接 `sum()` 做 grad-check**。LayerNorm 每行被构造成均值为零，所以 `sum(LN(x))` 几乎不随 `x` 变，它的解析梯度约等于 0。相对误差的分母是 `|analytic| + |numeric|`，两个都接近 0 时相对误差会爆炸——这是 near-zero / near-zero 的数值假象，不是 bug。stage01 的解法是把输出耦合回输入：

```ts
const errLayerNorm = numericalGradCheck((t) => ln.forward(t).mul(t).sum(), gcInput);
```

`.mul(t)` 让标量重新依赖输入，grad-check 干净地落到 `8.50e-11`。softmax 用了同一招（`.softmax().mul(t).sum()`）。**判断准则**：grad-check 的探针函数（probe）必须让输出对输入有非退化的依赖，否则你测的是浮点噪声不是梯度。

第二个：crossEntropy 是本 stage 现写的**融合节点**（fused node），必须单独验。引擎里没有 `log` 算子，就算有，把交叉熵写成 `-log(softmax)` 两步也是经典数值陷阱——softmax 可能下溢到 0，`log(0) = -Inf` 直接毒死整个 batch。`stage01-base.ts` 把 log-sum-exp 折进一步算，并在每行减去最大值：

```ts
let mx = -Infinity;
for (let c = 0; c < vocab; c++) mx = Math.max(mx, logits.data[i * vocab + c]);
// ...
const logsumexp = mx + Math.log(sumExp);
total += logsumexp - logits.data[i * vocab + targets[i]];
```

它的反向就是教科书形式 `softmax(z) - onehot(target)`，有限且良态。但「教科书形式」不等于「我没敲错」——所以 `errCrossEntropy=1.37e-10` 这个数才是它能被信任的唯一理由。

✦ 这就是本章的第一个专家钩子：**一个稳定收敛、grad-check 通过的基座长什么样**，是后面所有 PEFT 章公平对照的前提。你后面比较 LoRA r=4 vs r=8，比较 Adapter vs prefix，所有结论都建立在「base 这一层没有隐藏的数值病」之上。这一段不通过，后面所有数字都不用看。

## 任务 A：让它真学会「记住 token」

基座任务故意选了最朴素的：**copy**，target 完全等于 input。`stage01-base.ts` 配 `vocab=12, seqLen=8`，模型是 `Embedding → 单个 TransformerBlock → Linear head`，按 token 分类训练。为什么选 copy？因为第 02 章的微调任务是 copy→reverse（把序列倒过来）：基座学会「记住 token」，微调只需补上「反序」这一个增量能力。基座任务和微调任务共享底层表征，A→B 的差距才小到能被几百个 LoRA 参数补上——这是后面对照实验成立的设计前提，不是随手选的。

第 [2] 段训练 40 个 epoch，Adam（自适应学习率优化器）`lr=0.005`：

```
[2] 训练基座 (Adam lr=0.005, 40 epochs):
    理论初始 CE (uniform over vocab=12) ≈ ln(12) = 2.4849 | 实测训练前 loss = 2.8159 (随机初始化 ≈ 均匀分布)
    实测 loss: 2.8159 -> 0.0000
    val token-accuracy: 14.1% -> 100.0% (held-out)
```

注意它先打了一个理论锚点：均匀分布下每个位置的交叉熵 ≈ `ln(vocab) = ln(12) = 2.4849`。实测训练前 loss 是 `2.8159`，比理论值略高（随机初始化不是完美均匀，略偏）。**为什么要打这个锚**：没有它，你说「loss 下降了」可能只是数值漂移；有了它，`2.8159 → 0.0000` 才能确证「从接近瞎猜下降到学会了」。验证集 token 准确率 `14.1% → 100.0%`——14.1% 接近 `1/12 ≈ 8.3%` 量级的瞎猜基线，说明训练前确实在乱猜；100% 是在**训练从未见过的 32 条 held-out 序列**上测的，不是背了训练集。

stage01 把这些都写成断言，不是打印完就算：

```ts
assert(preTrainLoss > uniformCE * 0.8, "pre-train loss far from uniform entropy (init suspicious)");
assert(
  lossHistory[lossHistory.length - 1] < lossHistory[0] * 0.2,
  "base did not converge (loss drop < 80%)",
);
assert(valAcc1 > valAcc0 + 0.3, "base did not generalize (val accuracy barely moved)");
```

三条断言分别守三件事：初始化没坏（起点接近均匀熵）、真收敛了（loss 掉了 80% 以上）、真泛化了（验证准确率显著上升）。`copy` 任务在 toy 规模能干到 100% / loss 归零，这是任务太简单+模型够用的结果。

⚠ 别被绝对值骗了。这是 toy scale——`dModel=32`、`vocab=12`、`seqLen=8`，CPU 上秒级跑完，loss 能真打到 0.0000。真实规模的基座绝不会 loss 归零、准确率 100%。**可迁移的不是绝对值，是曲线形状**：起点贴着 `ln(vocab)`、一个清晰的收敛 elbow（拐点）、然后平台。你在真模型上看到的也该是这个形状，只是平台不在 0。

## 100% 可训练，这就是要被压缩的基线

第 [3] 段从 live model 真数参数构成（不是引用论文数字）：

```
[3] 参数构成 (从 live model 真数, 非引用论文):
emb   │██······································│ 384  embedding table
block │████████████████████████████████████████│ 8,544  transformer block
head  │██······································│ 396  output projection
    total=9324 trainable=9324 (100.0%)
    NOTE: 此处 100% 可训练 = full fine-tuning 基线; 后续 LoRA/Adapter 会把这个比例压到个位数百分比。
```

9324 个参数全部可训练。**这就是全书的分母**。第 02 章 full fine-tuning 会原样改这 9324 个；第 03 章起 LoRA / Adapter 把可训练比例压到个位数百分比，而压缩的对照基准就是这里这个 100%。`core/nn.ts` 的 `Module.numParams({trainableOnly})` 让每个 stage 都能诚实地打出「X% 可训练」——它读的是 `requires_grad` 标志，冻结一个参数它就不计入 trainable，所以后面那些「0.5%」不是嘴上说的，是从同一个计数器数出来的。注意 block 占了 8544（绝大头），emb 和 head 各几百——这个分布也是后面「LoRA 该插在哪」讨论的伏笔：增量预算花在 attention/FFN 的大矩阵上才划算。

## Snapshot：为什么不每章重训

第 [4] 段把模型 dump 成 checkpoint：

```
[4] checkpoint dumped:
    params=19 floats=9324 | deep-copy verified (mutating model left buffer intact)
```

19 个参数张量、9324 个 float，和上面对得上。关键是**为什么要 snapshot 而不是每章重训**：PEFT 必须建立在一个**固定不动**的 base 上。如果每章都重训基座，PRNG 状态、浮点累加顺序的任何差异都会让 base 漂移，于是你比较 LoRA r=4 和 r=8 时，底下的 base 已经不是同一个了——对照实验失去意义。所以后续 stage 一律 `loadBase`（用同种子重训出的 bit-identical 基座再加载），不重训。

stage01 还顺手验证 dump 是**深拷贝**而不是共享引用——改原模型不能动到已存的 buffer：

```ts
const before = ckpt.buffers[0][0];
model.emb.table.data[0] += 999;
const after = ckpt.buffers[0][0];
assert(before === after, "checkpoint is not a deep copy (mutating model changed buffer)");
```

这个断言看着像小题大做，但浅拷贝是新手 checkpoint 代码最常见的 bug：你以为存了一份快照，其实存的是引用，后续训练一改权重快照跟着变，「冻结的 base」根本没冻住。`coding.md` 里讲过「写两处必须同源否则一处派生」——checkpoint 是 base 的派生快照，深拷贝是它和 live model 解耦的硬保证。

## 失败模式:抽掉 LayerNorm，残差流爆给你看

到这里 happy path 都通了。但本书每章都要讲「怎么坏、为什么这么设计」，基座这章要坏的东西是 **LayerNorm**（层归一化，把每行激活重新拉回零均值单位方差）。

很多人把 LayerNorm 当成「加上去训得稳一点」的玄学。stage01 的第 [5] 段做了个干净的对照实验把它的真实作用钉死：同种子、同数据、同优化器、同学习率，**唯一变量是有没有 LayerNorm**。学习率推到激进的 `0.04`：

```
[5] 失败模式: 去掉 LayerNorm (激进 lr=0.04, 同种子/同数据, 唯一变量=归一化):
    有 LayerNorm: 1.4740 -> 0.0000  (稳定下降)
    无 LayerNorm: 2.5544 -> 586.3229  (峰值 8402.3814)

    NO-LayerNorm CE loss (发散)
8402.381 │                              ••
   1.596 │••••• ••••• ••••••••••••••••••    • • ••
```

有 LayerNorm 的模型在 `lr=0.04` 下仍单调下降到 0；抽掉它，同样的步长把 loss 从 2.55 推到峰值 **8402**，最后停在 586，彻底发散。为什么?pre-LN 的残差流（residual stream，每层把输出加回输入的那条主干）在 attention 和 FFN 两次相加里尺度会逐层放大，gelu 和无界 matmul 继续放大它；没有归一化把尺度钉回来，同一个 Adam 步长就 overshoot，loss 震荡爆到天文数字。

实现上 stage01 为了公平，专门写了个 `UnnormalizedToyLM`，借用 `TransformerBlock` 的子层构造器拿到**一模一样的初始化**，只把 LayerNorm 换成恒等映射：

```ts
forwardIds(ids: number[]): Tensor {
  const h = this.emb.lookup(ids);
  // pre-LN replaced by IDENTITY (this is the neutered normalization):
  const a = this.attn.forward(h).add(h); // residual, but no LN to keep scale in range
  const f = this.ff2.forward(this.ff1.forward(a).gelu()).add(a); // residual, no LN
  return this.head.forward(f);
}
```

注意它不是「调坏初始化来演示发散」——attention、FFN、残差、init、lr、seed、data 全部一致，唯一动的就是那两个 LayerNorm。所以发散只能归因于归一化的缺失，不是 demo 作弊。**真正的 lesson**：LayerNorm 的作用是**拓宽可用学习率区间**。在安全 lr（0.005）下两者都能训，看不出差别；把 lr 推激进，差别才暴露。

这直接接回本章的使命。后续微调冻结的是 base 权重——如果 base 是在一个没有 LayerNorm、靠运气没爆的不稳定优化里训出来的，你冻的就是垃圾权重，LoRA 建在上面再怎么调也补不回来。所以本书每章都要求 **base 先收敛、grad-check 过、再 snapshot**，这不是仪式，是 PEFT 对照实验成立的物理前提。

## ⚡ 前沿钩子:grad-check 是你调新算子的第一道防线

最后留一个会一直跟着你的工具。`numericalGradCheck` 在本章是训练前的入场券，但它的真正价值在更远处：**你以后每写一个自定义算子，它就是第一道防线，工业代码里它就是单测。**

这不是 toy 教学的专利。DoRA（把权重拆成幅度和方向分别更新）、PiSSA（用 base 权重 SVD 的主成分初始化 LoRA）这类新方法，核心都是**新的初始化或新的前/反向路径**。你手写它们的 backward 时,大概率第一版就是错的——忘了某个 broadcast 的反向求和、transpose 在 matmul 反向里方向反了，这些 bug 产出的梯度「看着合理但是错的」，会让你对着一条缓慢收敛或诡异震荡的 loss 曲线浪费几小时。`core/tensor.ts` 的注释把这件事说得很直白：

```
* FAILURE MODE it surfaces: if you wire LoRA's B/A grads wrong, this fires immediately
*   instead of you debugging a wrong-but-converging loss curve for an hour.
```

⚡ 这里有个目前**没有通用解、仍在研究**的开放问题：grad-check 只能验证**你已经写出来的反向**对不对，它不能告诉你这个反向**该不该存在**——即一个新算子有没有更数值稳定的等价形式（就像 crossEntropy 的 log-sum-exp 重写之于朴素 `-log(softmax)`）。哪些算子组合在大模型规模下会因为浮点累加、低精度（QLoRA 的 4-bit 量化路径正是这一类）而悄悄失真，至今没有自动化工具能在你写之前就告警；目前业界只能靠 grad-check 事后兜底 + 人工推导数值稳定形式。第 07 章 QLoRA 的反量化路径会再撞上这个问题——到时候你会感谢现在就把 grad-check 当反射。

下一章我们把这份基座 `loadBase` 进来，做 full fine-tuning（全参微调）——把这 9324 个参数全部解冻、在 reverse 任务上重训，量出「全参微调」的质量上限和代价。它是后面所有 PEFT 方法要逼近的天花板，也是要省掉的成本。
