第 03 章我们把 LoRA 跑通了,`y = x(W₀ + BA·α/r)`,`B` 零初始化、`A` 小高斯。但有个旋钮我故意没碰:`r` 选多少。这是任何人上手 LoRA 的第一个实操问题,网上的答案通常是「8 起步,不够就翻倍」——这是经验法则,不是理解。本章不背法则,我们做一件能复现的事:把 `r` 从 1 扫到 16 (退化成全秩) 训同一个任务,看准确率、参数量、收敛步数怎么动,然后对学出来的增量矩阵做 SVD (奇异值分解,把矩阵拆成几个独立方向 + 各自强度),直接看它「真正用了几个方向」。

结论先说:`r` 不是越大越好,准确率对 `r` 的曲线是个曲棍球杆 (hockey stick) ——爬到任务的「有效秩」就饱和,再往上加纯属浪费。但这话空口说没用,得有数。

## 把「有效秩」从玄学变成已知量

要证明「微调需要的更新是低秩的」,最大的难处是:在真实模型上你不知道 ground truth 的秩是多少,只能事后看 SVD 然后说「你看衰减得很快」——这是循环论证。所以本章的代码做了一件更狠的事:**自己种一个秩已知的更新矩阵**,再让 LoRA 去复原它。这样我们能提前断言准确率该在哪儿饱和,而不是观察完再编故事。

看 `examples/peft-from-scratch/src/stage04-rank.ts` 里 `buildPlantedDelta()` 怎么种这个矩阵:

```typescript
function buildPlantedDelta(): { deltaStar: Tensor; trueSpectrum: number[] } {
  const data = new Float64Array(D_OUT * D_IN);
  // Span the full dimension with random rank-1 terms but kill the tail with DECAY^k, so the
  //   tail terms (k >= TRUE_RANK) are present but ~0 — that is "effective" vs "exact" rank.
  for (let k = 0; k < D_OUT; k++) {
    const sigma = Math.pow(DECAY, k); // strictly decaying
    if (k < TRUE_RANK) spectrum.push(sigma);
    const u = Array.from({ length: D_OUT }, () => normal(0, 1));
    const v = Array.from({ length: D_IN }, () => normal(0, 1));
    const un = l2norm(u), vn = l2norm(v);
    for (let i = 0; i < D_OUT; i++)
      for (let j = 0; j < D_IN; j++) data[i * D_IN + j] += sigma * (u[i] / un) * (v[j] / vn);
  }
  return { deltaStar: Tensor.from(Array.from(data), [D_OUT, D_IN], false), trueSpectrum: spectrum };
}
```

关键在 `DECAY = 0.35`。这是个 16×16 的矩阵,我们用 16 个秩 1 外积 (rank-1 outer product,两个向量相乘成的最简单矩阵) 把它填满——所以它在数学意义上是**满秩**的。但每个外积的权重是 `0.35^k`:第 0 个是 1.0,第 1 个 0.35,第 2 个 0.12,第 3 个已经只有 0.015。技术上 16 个方向都在,可第 4 个往后小到约等于噪声。这就是**有效秩 (effective rank)** 和精确秩的区别——精确秩是 16,有效秩约等于 3。

为什么要刻意造一个「满秩但有效低秩」的矩阵,而不是直接造个干净的秩 3 矩阵?因为真实的微调更新就长这样:它从不是数学意义上的纯低秩,而是能量高度集中在头几个方向、长尾衰减到忽略不计。造一个纯秩 3 矩阵会把题目做得太干净,掩盖「低秩是近似不是精确」这个最重要的事实。

任务本身是个冻结的线性映射:`y = x(W₀ + ΔW*)ᵀ`,`W₀` 冻住,LoRA 要补的正好是 `ΔW*`。运行时先把种进去的谱打出来:

```
planted ΔW*: shape 16×16, planted effective rank ≈ 3 (σ decay = 0.35); first σ's = [1.000, 0.350, 0.122]
planted ΔW* singular spectrum (top 8):
  0.993  0.350  0.115  0.037  0.009  0.004  0.001  0.000
  sparkline: █▃▁       (fast decay => low effective rank)
```

注意 SVD 算出来的实测谱 `0.993 / 0.350 / 0.115` 和我们种的权重 `1.000 / 0.350 / 0.122` 几乎吻合——因为随机向量近似正交,外积权重就近似是奇异值。第 4 个奇异值 0.037 已经是头部的 3.7%,第 5 个 0.009 进入噪声。**有效秩约 3,这是我们提前知道的答案。** 现在让 LoRA 去逼近它,看它在 `r=3` 附近是不是真的饱和。

(关于 SVD 本身:代码里 `singularValues()` 手写了一个单边 Jacobi 迭代,因为我们的 mini 框架 core 里没有 SVD,而我们只需要一个 16×16 矩阵的奇异值。算法细节这里不展开,它是确定性的、固定 30 轮扫描,够稳。)

## 扫 r:曲棍球杆曲线

`trainRank()` 对每个 `r` 重新播种再训,关键的实验卫生在这一行注释里说清楚了——同一 seed-family,唯一变量是 rank:

```typescript
function trainRank(r: number, W0: Tensor, train: Sample[], val: Sample[]) {
  // Re-seed per rank so every rank sees the SAME data-order randomness and SAME init draws
  //   up to the rank-dependent param count — isolates r as the only varying factor.
  seed(4040 + r);
  const model = new LoRALinear(W0, r, /*alpha*/ 8);
  // ...
}
```

如果不固定这个,你扫出来的曲线里混着数据顺序的随机性、初始化的随机性,根本分不清是 `r` 的功劳还是运气。**扫超参的第一纪律:把除被扫变量外的一切钉死。** 扫出来的对照表:

```
r | trainable params | %ofΔW(256) | train MSE | val MSE  | val acc | conv@step
--+------------------+------------+-----------+----------+---------+----------
 1|               32 |      12.5% | 3.95e-3 | 4.67e-3 |   88.3% |   —
 2|               64 |      25.0% | 7.66e-4 | 7.65e-4 |  100.0% |     4
 3|               96 |      37.5% | 3.44e-4 | 3.62e-4 |  100.0% |     3
 4|              128 |      50.0% | 7.89e-5 | 8.27e-5 |  100.0% |     3
 6|              192 |      75.0% | 2.49e-6 | 2.43e-6 |  100.0% |     2
 8|              256 |     100.0% | 2.75e-5 | 3.21e-5 |  100.0% |     2
16|              512 |     200.0% | 1.64e-4 | 1.73e-4 |  100.0% |     2
```

逐列读,这张表把整章的论点全装下了:

**val acc 这一列是曲棍球杆。** `r=1` 时只有 88.3%——一个秩 1 的增量装不下三个方向的更新,差的就是欠拟合 (underfit,容量不够)。`r=2` 跳到 100%,`r=3` 之后全是 100%。代码把这条断言钉死了:

```typescript
assert(accSat > accLow + 0.05, "accuracy did not climb from r=1 to effective rank — task too easy/hard");
assert(Math.abs(accHigh - accSat) < 0.07, "accuracy did NOT saturate past effective rank — claim broken");
```

运行时打出来:

```
saturation check: acc(r=1)=88.3%  acc(r=3)=100.0%  acc(r=8)=100.0%
  ✓ acc climbs r=1 → r=3, then SATURATES r=3 → r=8 (|Δ| < 7pt).
```

爬升段 (`r=1→3`) 是「秩不够,加秩有用」;饱和段 (`r=3→8`) 是「秩够了,加秩白搭」。拐点就在我们种的有效秩 3 附近——这不是巧合,是我们提前算好的答案被实验复现了。✦ **这就是「r 选多少」的可复现方法论:不要拍脑袋,扫一遍找拐点,拐点就是这个任务的有效秩。**

**%ofΔW 这一列是代价。** 全量 ΔW 是 256 个参数 (16×16)。`r=3` 只用 96 个 (37.5%) 就拿到满分;`r=8` 用满 256 个,`r=16` 用到 512 个——是全量的两倍。为什么 `r=16` 会超过 100%?因为 LoRA 的参数量是 `r×(d+k) = r×32`,当 `r` 大到接近 `min(d,k)`,两个矩阵 `B(16×r)` 和 `A(r×16)` 加起来反而比直接存一个 16×16 还多。这就引出失败模式。

**train MSE 比 val MSE 更值得盯。** `r=6` 的 train MSE 是 2.49e-6,`r=16` 反而退到 1.64e-4。MSE 在 `r=6` 见底后又往上,不是越大越好——这是优化噪声叠加过拟合的影子,后面细说。

## 失败模式:r = min(d,k) 退化成全秩 LoRA

把 `r` 推到 `min(d_out, d_in) = 16`,LoRA 就退化了。这时 `BA` 能表达任意 16×16 矩阵,LoRA 的「低秩约束」彻底消失,你得到的是一个绕了个弯、参数还更多的全量微调。代码专门把这个失败模式打出来,不回避:

```
失败模式 — r = min(d,k) = 16 (退化为近全秩 LoRA):
  trainable params = 512 = 200.0% of full ΔW (vs r=3: 96 params, 37.5%)
  val acc r=16: 100.0%  vs  r=3: 100.0%  → 5.3x the params for 0.0pt accuracy.
```

**5.3 倍的参数,换 0 个百分点的准确率。** 这是过参数化最干净的证据。你以为「秩越大表达力越强,效果该更好」,但表达力早在 `r=3` 就够用了,多出来的容量无处可去。

那多出来的容量去哪了?看代码诚实标注的过拟合代理 (overfit proxy):

```typescript
// We do NOT assert overfit: on this clean synthetic task accuracy is already 100% at the
//   effective rank, so extra capacity has almost no MSE to chase either way. The robust,
//   demonstrable failure is the PARAM BLOWUP for zero accuracy gain; the train/val GAP is the
//   honest overfit proxy (train fits noise the val does not).
```

运行输出:

```
  best val MSE: r=16 2.28e-6 vs r=3 5.80e-5 (both already < 0.001; the marginal MSE difference is noise, not a real win).
  train→val MSE gap (overfit proxy): r=16 8.44e-6 vs r=3 1.79e-5  (larger/positive gap at high r = capacity fitting train noise that does not generalize).
```

这里我要点出代码作者的克制:它**没有**断言「高 `r` 一定过拟合」。为什么?因为这是个干净的合成任务,准确率在有效秩处已经 100%,额外容量几乎没有 MSE 可追,过拟合的信号很弱。`r=16` 的 best val MSE 2.28e-6 看着比 `r=3` 的 5.80e-5 小,但代码明说了「这点差异是噪声,不是真赢」。能站得住的失败是参数爆炸换零收益;train→val gap 只是个方向性的代理,让符号自己说话。这种「区分能机械证明的 vs 只是看着像的」的诚实,正是这本书要做的反面——不是 AI 那种「高秩导致过拟合所以要小心」的空泛断言,而是「在这个 setup 下过拟合信号弱,我能证明的只有参数浪费」。

参数代价的可视化更直观:

```
r=1  │███·····································│ 32  params  (acc 88%)
r=3  │████████································│ 96  params  (acc 100%)
r=8  │████████████████████····················│ 256  params  (acc 100%)
r=16 │████████████████████████████████████████│ 512  params  (acc 100%)
```

`r=3` 那条短杠拿到了和 `r=16` 那条满杠一样的准确率。你为长杠多付的每一格,都是白付。

## 给 ΔW 做体检:学出来的谱回声了种进去的谱

前面是黑箱看输入输出。现在打开黑箱,直接 SVD 学到的增量矩阵。我们用 `r=8` 的模型——它有 8 个方向的容量,远超有效秩 3,**如果它真用满 8 个方向,说明任务需要 8;如果它只集中在头几个,说明低秩假设成立**。看 `LoRALinear.learnedDelta()`:

```typescript
/** The learned update ΔW = (B@A)·(α/r), as a detached matrix (for SVD / heatmap). */
learnedDelta(): Tensor {
  return this.B.matmul(this.A).scale(this.alphaOverR);
}
```

对它做 SVD:

```
learned ΔW (from r=8 LoRA) singular spectrum (top 8):
  0.992  0.359  0.120  0.037  0.005  0.002  0.001  0.000
  sparkline: █▃▁       (learner concentrates energy in few directions, like the planted ΔW*)
  effective rank (σ² energy reaching 90%) = 2  (planted ≈ 3)
```

把它和最开头种进去的谱并排:

```
planted:  0.993  0.350  0.115  0.037  0.009 ...
learned:  0.992  0.359  0.120  0.037  0.005 ...
```

**几乎一模一样。** 给了 `r=8` 八个方向的预算,优化器自发地只在头 2-3 个方向上堆能量,后面的奇异值衰减到噪声——和我们种进去的低秩结构吻合。它没有「因为有 8 个槽就用满 8 个」,而是学出了任务真正需要的形状。代码用「σ² 能量累积到 90% 需要几个奇异值」来量化有效秩,算出来是 2 (种的是约 3,差异是 90% 阈值的边界效应)。

这就是「为什么低秩够用」从信念变成观测的那一刻:不是因为论文这么说,是因为你给优化器足够的秩,它自己选择了低秩。`learnedDelta()` 末尾的 `.scale(alphaOverR)` 别忽略——`α/r` 是第 03 章引入的缩放,它让你调 `r` 时不必重调学习率,这也是为什么本章能干净地只扫 `r` 一个变量。

代码还打了张 heatmap,`|value|` 映射到灰阶,低秩矩阵会显出条纹状结构而非雪花噪声——肉眼也能看出它「有结构」。这里不贴 ASCII 图了,跑一遍 `stage04-rank.ts` 就能看到。

最后,代码自己标了 toy-scale 的诚实边界,这条必须原样转达:

```
⚠ toy-scale: 有效秩的绝对值取决于我们手工植入的 ΔW* (σ 衰减 = 0.35);
  可迁移的是两条形状: (1) 奇异值快速衰减, (2) 准确率-vs-r 曲线有拐点、过拐点后饱和。
```

别把「有效秩 ≈ 3」当成放之四海的常数背走——那是我们用 `DECAY=0.35` 种出来的。能迁移到真实模型的是两条**形状**:真实微调更新的奇异值快速衰减,以及准确率对 `r` 的曲线有拐点、过拐点就饱和。LoRA 原论文在 GPT-3 175B 上测过,`r=1` 在多数任务上就已经很能打,`r=8` 普遍够用——衰减得比你想的还快。

## ⚡ 前沿:每层有效秩不同,固定 r 是浪费

本章默认所有层用同一个 `r`,但奇异值谱暴露了一件事:不同权重矩阵的有效秩不一样。注意力的 query/value 投影、不同深度的层、MLP 的上下投影,各自需要的秩不同。给所有层都设 `r=8`,等于在「有效秩 2 的层」上浪费了 4 倍预算,又可能在「有效秩 12 的层」上欠拟合。

**AdaLoRA** (Adaptive LoRA,2023) 正是冲着这个观察去的:它不预设 `r`,而是给每层一个秩**预算**,训练中按每个奇异值方向的重要性动态裁剪——把不重要方向的秩匀给重要的层。本章这张奇异值谱图,就是 AdaLoRA 的动机图:既然能量分布因层而异,固定 `r` 在信息论上就是次优的。

但这是个仍在研究的方向,**没有通用解**。AdaLoRA 引入了新的超参 (总预算、裁剪调度、重要性度量),把「调一个 `r`」变成了「调一组旋钮」,在不少任务上相对固定秩 LoRA 的提升是边际的,代价是实现和调参复杂度上升。「每层最优秩到底怎么定」目前没有不依赖训练动态的先验答案——你还是得训起来看。换句话说,本章给你的可复现扫描方法论,在多层模型上的正确推广形态,学界仍在摸索。

## 小结与接下

我们把第 03 章那个悬而未决的 `r` 旋钮拧明白了:准确率对 `r` 是曲棍球杆,拐点在任务的有效秩处,过拐点就饱和;把 `r` 推到 `min(d,k)` 是退化成参数更多的全量微调,5.3 倍参数换 0 收益;给优化器富余的秩,它自发学出低秩——SVD 谱回声了种进去的结构。「低秩够用」不再是信念。

`r` 是 LoRA 的「秩」旋钮。但增量不只有「秩」一个自由度:第 03/04 章我们一直把 `BA` 插在线性层的权重上,可微调的增量还能插在**别的位置**(注意力前后、FFN 中间)、用**别的结构**(瓶颈 MLP 而非低秩矩阵)。第 05 章就去拆 Adapter——另一族 PEFT 方法,看看「在哪插、插什么」这两个自由度怎么换来不同的速度/质量权衡。
