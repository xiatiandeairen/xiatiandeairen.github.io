> **机制先行** —— 这一章不教"怎么调出好看的界面",教"为什么有的界面一眼就高级、有的一眼就廉价"。高级感不是玄学、不是天赋,是 6 条可以拆解、可以复刻的视知觉与认知规律。看懂它们,你不用学会画图,也能让自己做的界面从"程序员风"变成"看着像个正经产品"。
> **这是核心章**:6 条机制逐条讲透,每条配 Apple / Google / 字节 的真实界面深拆,并给你带具体数值(色值、间距、字重)的可执行做法。结尾给你一套"5 分钟让界面变高级"的清单。

---

## 为什么资深工程师做的界面,常常一眼廉价

你后端架构能扛百万并发,但自己写个落地页 / 后台 / 工具站,出来的东西总透着一股"开发者 demo"的味道。不是丑——配色没大错、控件也都对齐了——就是**不高级**。投资人一看就觉得不靠谱,用户一看就不想付费。

很多人把这归结为"我没有审美天赋"。错了。**高级感的绝大部分,不来自天赋,来自对几条视知觉规律的遵守。** 你觉得 Apple 官网高级,不是因为设计师有神来之笔,而是因为它在层次、留白、对齐、克制、一致、精度这 6 件事上,**每一处都没有出错**。

反过来,廉价感几乎总是同样几个错误的叠加:层次扁平(全都一样重)、不留白(塞满)、不对齐(随手摆)、太花(字体颜色一堆)、不一致(每页一个样)、不精确(差几个像素没人管)。

好消息是:**这 6 件事全部可以用规则和具体数值来执行,不需要你"有感觉"。** 你是工程师,规则和数值正是你最擅长的东西。下面逐条拆。

## 机制一:视觉层次 —— 大脑按对比自动分配注意力,层次清晰 = 不费脑 = 高级

人看一个界面的第一眼,不是逐字阅读,是**大脑在 50 毫秒内自动扫描,按"对比 / 大小 / 位置 / 颜色"把元素排出一个重要性顺序**,决定眼睛先看哪、再看哪。

**为什么它排第一**:如果界面上所有元素都一样大、一样重、一样颜色,大脑就**无法自动排序**,它只能逐个去看、自己费力判断"哪个重要"。这种"什么都一样重"的界面,逼用户动脑,累——而**累,就是廉价感的核心来源**。反过来,层次分明的界面,大脑一眼就知道"先看这个大标题,再看这行小字",毫不费力——**毫不费力,就被感知成"高级、专业、为我考虑过"**。

**深拆 · Apple 产品页的"一句话顶天立地"**
打开任意一个 iPhone 产品页,最顶上往往只有一句话,比如某代的 "Pro. Beyond."——**字号巨大(桌面端常在 60–80px 甚至更大)、字重很粗,几乎占满第一屏**。它和下方正文(可能只有 17px)形成了**悬殊的对比**。这种悬殊不是为了好看,是为了**强行替你的大脑做好排序**:你一眼就知道"这页在讲什么",根本不用读。这就是层次的极致用法——**用巨大的尺寸对比,制造一个无可争议的视觉焦点。**

而新手做落地页,标题 24px、正文 16px,差距太小,**大脑分不出主次**,整页"温吞水",没有焦点,于是显得平庸。

**深拆 · Google 搜索结果页的三级灰度层次**
一条搜索结果,Google 用**三个层级**清晰区分:标题(蓝色、较大、可点击)、网址(深灰小字)、摘要(黑灰正文)。**三者用颜色 + 大小拉开了明确的层次**,你的眼睛会自动先抓蓝色标题。如果三行全是同一个黑色、同一个字号,你就得逐行读才能找到要点——那就是早期很多搜索引擎结果页"密密麻麻看着累"的原因。Google 用最小的视觉手段(就是颜色和字号的差异),把"扫一眼就能挑结果"这件事做到了零负荷。

**深拆 · iOS 设置页的"分组 + 弱化次要信息"**
iOS 设置里,每一项的主标题是黑色,而右侧的状态值(比如"已开启""中文")是**浅灰色**。**主信息强、次信息弱**,这个层次让你扫的时候只看黑色项就行,灰色的细节"需要时再看"。新手常犯的错是**把所有文字都用纯黑**,主次不分,信息一拥而上。

**给你的可执行做法(带值)**:
- 建立**至少 3 级文字层次**,用字号 + 字重 + 颜色三者叠加拉开,别只靠一个维度。一个能用的桌面端 scale:

| 层级 | 字号 | 字重 | 颜色 |
|---|---|---|---|
| 大标题 H1 | 32–48px | 700 | `#1A1A1A` |
| 小标题 H2 | 20–24px | 600 | `#1A1A1A` |
| 正文 | 16px | 400 | `#3C3C43` |
| 次要 / 说明 | 13–14px | 400 | `#8E8E93` |

- 一屏**只允许一个最强焦点**(通常是主标题或主 CTA 按钮),其余一律退让。问自己:"我眯起眼,第一个看到的是不是我希望用户先看的那个?"
- 次要信息(时间戳、状态、辅助说明)**主动调灰、调小**,别让它和主信息抢。

## 机制二:留白与呼吸感 —— 留白是稀缺信号,稀缺 = 贵

留白(negative space,也叫负空间)就是元素之间、元素和边界之间"什么都没有"的地方。新手最反直觉的一点是:**留白不是浪费,留白本身就是设计,而且是高级感最强的单一信号。**

**为什么留白 = 高级**:这背后有一条朴素的认知联想——**信息密度高 = 廉价,信息密度低 = 昂贵**。想想线下:十元店把货架塞到爆,奢侈品店一个偌大的空间只摆三个包。报纸分类广告挤满版面(便宜),时尚大刊一整页只放一张图配一行字(贵)。**大脑把"敢于留白、敢于不塞满"解读为"它不缺地方、不缺自信、东西值钱"。** 此外,留白还**降低认知负荷**(机制一的延伸):周围越空,中间那个元素越突出、越好聚焦。

**深拆 · Apple 官网的"巨幅留白"**
Apple 官网的产品图周围,永远留着**大片的空白**。一个产品、一句话、大量呼吸空间,绝不在一屏里塞两个卖点。这种"奢侈的空"传递的潜台词是:"我只想让你专注看这一个东西,它值得你全部的注意力。"**对比某些电商首页**,首屏塞满 banner、优惠券、秒杀、分类入口——信息量是大,但廉价感扑面而来,因为它在喊"快看!全都便宜!"。同样卖东西,留白多寡直接决定了气质是"奢侈品"还是"地摊"。

**深拆 · Google 首页的极致空**
Google 首页几乎全白,正中一个搜索框。这片巨大的留白,既把认知负荷压到零(机制一),也让 Google 显得**极度自信和克制**——"我不需要用花哨的东西证明自己"。这种"空"本身就是品牌气质。

**深拆 · iOS 系统的"行间呼吸"**
iOS 的列表、卡片之间,行高和内边距都给得很充裕,绝不把文字怼到边框上。**文字和容器边缘之间的内边距(padding),系统级常用 16px 起步**;卡片和卡片之间也留着明确的间隙。这种"不挤"让每个元素都有自己的呼吸空间,扫起来轻松。安卓上很多第三方 App 把信息怼得很满,行距很紧,对比之下就显得局促、廉价。

**给你的可执行做法(带值)**:
- **大胆增加间距。** 工程师做的界面 80% 的廉价感来自"太挤"。把你觉得"差不多"的间距**翻倍试试**,通常立刻就高级了。
- 元素内边距别吝啬:按钮内边距至少 `12px 24px`(上下 12、左右 24);卡片内边距 `16px` 起,舒展点用 `24px`。
- 给内容区设**最大宽度**,别让正文在大屏上拉满一整行。正文阅读宽度控制在 **640–720px**(约每行 60–75 个英文字符 / 30–40 个汉字)最舒服,两侧自动留白。
- **段落、区块之间留足垂直间距**:大区块之间 `64px` 甚至 `96px` 都不嫌多。

**一句话**:当你犹豫"是不是太空了",答案几乎总是"还不够空"。

## 机制三:Gestalt 格式塔 —— 大脑自动把"靠近 / 对齐 / 相似"的东西归为一组

格式塔(Gestalt)是一组心理学原理,描述大脑如何**自动把零散元素组织成有意义的整体**。对 UI 最有用的有四条:

- **接近性(Proximity)**:离得近的元素,被自动看成一组。
- **对齐 / 连续性(Alignment)**:排在一条线上的元素,被看成有关联、有秩序。
- **相似性(Similarity)**:长得像(同色 / 同形 / 同字号)的元素,被看成同一类。
- **闭合性(Closure)**:大脑会自动脑补缺失的部分,把不完整的形状看成完整的。

**为什么它决定"整洁 vs 凌乱"**:界面凌乱,本质不是元素太多,而是**大脑无法把它们正确分组**——该在一起的散开了,不相关的却挨着。而**整洁、专业的观感,几乎等于"分组清晰 + 对齐严格"**。其中**对齐是性价比最高的一条**:把所有元素的左边缘对齐到同一条线上,界面立刻"齐整、professional";哪怕只差几个像素的错位,大脑都能隐约察觉到"乱",虽然说不出哪里乱。

**深拆 · iOS 设置页 = 接近性的教科书**
iOS 设置把功能项**分成一组组带圆角的卡片**:Wi-Fi / 蓝牙 / 蜂窝在一组,通知 / 声音 / 专注在另一组。**组内行挨得近,组与组之间留明显间隙**——这就是纯粹的接近性原理:**用"距离"而不是"分割线"来表达分组**。你不用读,一眼就知道哪些是一类。对比一个糟糕的设置页:几十项平铺、等距排列、靠细线硬分割,大脑得自己费力归类,累。

**深拆 · Material Design 的 8dp 网格 = 对齐的工程化**
Google 的 Material Design 规定所有元素都吸附到 **8dp 的网格**(图标等小元素可用 4dp)。这意味着每个元素的位置、每段间距都是 8 的倍数,**天然全部对齐**。为什么 Google 要立这条铁律?因为**对齐不能靠"目测大概齐",目测必然出现几像素的错位,而错位的累积就是"乱"**。用一个固定网格把所有东西吸附上去,对齐就从"靠手感"变成了"靠规则"——这正是工程师该有的解法。

**深拆 · Instagram 九宫格 = 相似性 + 对齐**
Instagram 个人主页的九宫格,**每张图同样大小、同样间距、严格对齐成网格**。正是这种高度的"相似 + 对齐",让一堆内容各异的照片在视觉上成为一个整洁、统一的整体,而不是杂乱的一堆图。很多内容创作者还会刻意让九宫格配色统一(强化相似性),整个主页就更像一件"作品"。

**给你的可执行做法(带值)**:
- **对齐压倒一切。** 给所有内容定一条**左边缘基准线**,标题、正文、按钮全部左对齐到它。能消灭 80% 的"乱"。
- **用一个间距网格(spacing scale)**,所有间距都从里面取值,不许随手填:`4 / 8 / 12 / 16 / 24 / 32 / 48 / 64`(px)。这是大厂的通用做法(8 的倍数体系)。
- **用"距离"表达分组**:相关的元素靠近(间距 `8px`),不相关的拉开(间距 `32px+`)。**让"组内间距明显小于组间间距"**,分组就不言自明,常常都不需要分割线。
- **同类元素长一个样**:所有主按钮同一种样式,所有卡片同一个圆角和阴影。相似性会帮你建立秩序感。

## 机制四:克制 —— amateur 用得多,pro 用得少

克制(restraint)指的是:**有意识地限制你使用的字体数量、颜色数量、字重数量、间距取值**。这是区分业余和专业最锋利的一刀。

**为什么少 = 高级**:每多一种字体、多一个颜色、多一种圆角,都是在界面里**引入一个新的"视觉变量"**,而变量越多,大脑越难找到秩序,观感就越乱、越"花"。**业余的本能是"我要丰富一点、活泼一点",于是不断做加法;专业的本能是"我要统一、要克制",不断做减法。** 一个只用 1 种字体、2 个主色、一套间距 scale 的界面,天然就和谐——因为变量少,想乱都乱不起来。**"花"和"廉价"几乎是同义词。**

**深拆 · Apple 全线只用一种字体家族(SF)**
Apple 的系统、官网、产品,几乎全部使用自家的 **San Francisco(SF)字体家族**——通过同一家族里的不同字重(Regular / Medium / Semibold / Bold)和字号来制造层次,而**绝不混用第二种风格迥异的字体**。这种"一种字体打天下"带来高度的统一与高级。**反面教材**是新手页面:标题用一个艺术字体、正文用宋体、按钮又来个圆体——三种字体一上来,廉价感拉满。**记住:一个界面,正文 + 标题用同一种(最多两种、且角色分明)字体就够了。**

**深拆 · Material Design 的"一个主色 + 一个强调色"**
Material Design 的配色体系核心是**一个 primary color(主色)+ 一个 secondary/accent color(强调色)**,再加上一套中性的灰阶。颜色被严格约束,所以无论怎么搭,整体都协调。Google 不是"颜色用得少所以朴素",而是**用极少的颜色构建了清晰的功能含义**(主色 = 品牌 / 主操作,强调色 = 需要被注意的动作)。**颜色一旦泛滥,它就丧失了"指示重点"的能力**——满屏都是重点,就等于没有重点。

**深拆 · 字节系产品的克制配色**
抖音、剪映这类字节产品,界面主体是**大面积的深色 / 中性灰底**,把鲜艳的颜色(品牌色、点赞红)**只留给最关键的操作和品牌点**。正因为底色克制,那一点品牌色才跳得出来、才有力量。如果整个界面到处都是高饱和色,反而什么都突出不了。

**给你的可执行做法(带值)**:
- **字体:一种为主**(系统字体最稳:中文 `PingFang SC` / 英文 `SF` 或 `Inter`),最多再加一种角色完全不同的(比如代码用等宽)。**永远不要三种以上。**
- **颜色:一个主色 + 中性灰阶 + 至多一个强调色。** 状态色(成功 / 警告 / 错误)各一个就够。一个干净的起手配色:

```css
:root {
  --primary:   #2563EB;   /* 主色:品牌 / 主操作 */
  --accent:    #F59E0B;   /* 强调:个别需要拉注意的点,慎用 */
  --success:   #16A34A;
  --danger:    #DC2626;
  /* 中性灰阶(见机制六,带轻微色相,不是纯灰) */
  --text-strong: #1A1A1A;
  --text-weak:   #6B7280;
  --border:      #E5E7EB;
  --bg:          #FFFFFF;
  --bg-subtle:   #F9FAFB;
}
```

- **字重:用 2–3 档就够**(400 正文 / 600 强调 / 700 标题)。别一个界面冒出 300/400/500/600/700 五种字重。
- **圆角:全站统一 1–2 个值**(比如卡片 `12px`、按钮 `8px`),别每个组件一个圆角。

**心法**:每加一个新变量前,问"我能不能复用已有的?"——能复用就别新增。**克制不是限制创意,是创意的纪律。**

## 机制五:一致性 —— 可预测 = 信任 = 专业

一致性指的是:**相同的东西在任何地方都长一个样、行为一致**。同一种按钮在每一页都是同样的颜色、圆角、内边距;同一类操作在任何地方点击的反馈都相同;同样的间距规则贯穿全站。

**为什么一致 = 信任**:大脑天生是个"模式预测机器"。当界面一致,用户**只需要学习一次,就能预测所有地方的行为**——"哦,蓝色圆角的就是可点的主按钮",这个认知在全站通用,用起来毫不费力。而当界面不一致(这页按钮是蓝的圆角、那页是绿的方角),用户**每到一个新页面都要重新判断**,这种"猜不准"会累积成一种潜意识的不安:"这产品是不是没做好?是不是不靠谱?"**可预测带来掌控感,掌控感带来信任,而信任正是"专业"的内核。** 反过来,不一致是廉价感里最隐蔽、最伤的一种——它甚至比单页面丑更伤,因为它动摇的是信任。

**深拆 · Apple HIG = 把一致性写成法律**
Apple 的 **Human Interface Guidelines(人机界面指南,HIG)** 详尽规定了 iOS 上每一种控件应该长什么样、放在哪、怎么响应:返回按钮永远在左上、Tab Bar 永远在底部、轻扫返回的手势在所有 App 里都成立。**正因为有 HIG,你下载任何一个陌生的 iOS App,几乎不用学就会用**——你过去在别的 App 攒下的所有肌肉记忆都迁移过来了。这种"全平台可预测"是 iOS 体验高级感的根基之一,而它来自一套被严格执行的一致性规范。

**深拆 · Material Design = 一致性的可复用化**
Google 把一致性做成了**一整套可复用的组件系统(Material Components)**:按钮、卡片、对话框、输入框都有标准定义和行为。开发者直接用这套组件,产品天然就一致,而且和数十亿台安卓设备上的其它应用观感统一。**Material 的本质,是把"一致性"从设计师的自觉,变成了工程上可安装、可复用的库**——这对没有设计师的独立开发者是巨大的福音。

**深拆 · 设计系统(Design System)为什么是大厂标配**
无论 Apple、Google、字节,都维护着自己的**设计系统 / 组件库**(颜色、字体、间距、组件都有唯一定义)。原因正是一致性无法靠"每次手动对齐"维持——团队一大、页面一多,必然走样。**把规则沉淀成 token 和组件,一致性才能规模化。** 这对你是直接启发:**别每个页面现调样式,一开始就抽出一套 token(颜色 / 间距 / 字号变量)和几个基础组件,全站复用。**

**给你的可执行做法(带值)**:
- **第一件事就是建 token**,而不是逐页面调样式。把颜色、间距、字号、圆角、阴影全部定义成 CSS 变量 / 主题常量(像机制四那段 `:root`),全站只引用变量。改一处,全站统一变。
- **同类组件只做一个版本**:一种主按钮、一种次按钮、一种卡片、一种输入框。需要变体时从基础组件派生,而不是重写。
- **直接站在巨人肩上**:没有设计师就**用成熟设计系统 / 组件库**(Web 端如 shadcn/ui、Material UI、Ant Design;移动端遵循各自平台规范)。它们已经把一致性、间距、状态都处理好了,你白捡。
- **交互反馈也要一致**:所有可点元素 hover / 按下 / loading 的反馈方式统一。

## 机制六:细节与精度 —— 像素级对齐、成体系的间距、克制的阴影与动效 = "用心" = 高级

细节指的是那些**用户说不出、但能感觉到**的精确度:像素是否对齐、阴影是否自然、动效是否顺滑克制、图标是否同一套线宽。

**为什么细节决定"最后一公里"**:前五条让界面"不出错",而**细节让界面从"还行"跨到"高级"**。原理是一种潜意识的归因——**当一个界面处处精确(没有半像素错位、阴影柔和自然、动画恰到好处),大脑会无意识地判断"做这个东西的人很用心、很专业",并把这份信任投射到整个产品**。反之,一个差 2px 的错位、一道生硬的黑色阴影、一个突兀的弹跳动画,会被解读成"糙、不讲究",哪怕用户说不清哪里不对。**细节是高级感的"质感层",它解释了为什么两个看起来差不多的界面,一个显贵、一个显廉价。**

**深拆 · Apple 的阴影:柔和、低饱和、像真实光**
Apple 界面里的阴影(以及 macOS 窗口、iOS 卡片的投影)**永远是柔和、大面积扩散、低不透明度**的,模拟真实世界里柔和的环境光,绝不是一道生硬的深色边。**廉价界面的典型错误**是用 `box-shadow: 0 0 5px #000`——纯黑、不透明、小范围,看起来像"贴了一层脏边",非常假。Apple 的逻辑是:**阴影要像物理世界的光,才自然、才高级。**

**深拆 · 苹果 / 抖音的动效:有阻尼、有缓动,绝不线性**
iOS 的滑动、剪映 / 抖音的转场,动效都带**缓动曲线(ease-in-out)**和恰当的时长,符合物理直觉——东西加速启动、减速停下,像有质量、有惯性。**线性动画(匀速)和过长 / 过快的动画都显廉价**,因为现实里没有匀速突停的运动。同时它们都很**克制**:动效是为了让状态转换更易理解,**点到为止,绝不为炫技而做满屏花哨动画**(那恰恰是廉价的信号)。

**深拆 · Material 用"高度 + 阴影"系统化地表达层次**
回到扁平化与拟物的那段历史:早期 iOS 是**拟物(skeuomorphism)**——按钮带高光、阴影、材质,模仿现实物体;iOS 7 起转向**扁平化(flat)**,去掉多余装饰、强调内容本身和清爽的色块。而 Google 的 **Material Design** 走了第三条路:它不堆装饰,但用**"高度(elevation)+ 阴影"这一套系统化的规则**来表达层次——越靠上的元素阴影越明显,像纸片叠在不同高度。**注意它的克制:阴影是有规范、成体系的(每个 elevation 等级对应固定的阴影参数),而不是随手加。** 这正是"细节成体系"的范例——**同样是用阴影,成体系的是高级,随手加的是廉价。**

**给你的可执行做法(带值)**:
- **阴影要柔、要淡、要带一点点颜色**,别用纯黑硬阴影。好的卡片阴影:

```css
/* 廉价:生硬纯黑小阴影 */
box-shadow: 0 0 5px #000;

/* 高级:多层、柔和、低透明、轻微偏暖/冷 */
box-shadow:
  0 1px 2px rgba(16, 24, 40, 0.06),
  0 4px 12px rgba(16, 24, 40, 0.08);
```

- **动效:时长 `150–300ms`,曲线用 `ease` / `ease-in-out`(或 `cubic-bezier(0.4, 0, 0.2, 1)`,即 Material 的标准曲线),绝不用 `linear`。** 只给"状态转换"加动效,不为炫技乱加。
- **图标用同一套**(同一线宽、同一风格,如全用 Lucide / SF Symbols),别东拼西凑混搭风格。
- **较起真来对齐**:善用浏览器的标尺 / 设计稿的对齐线,把那些"差一两个像素"的错位修掉。用户意识不到对齐了,但能感觉到"齐整"。

## 廉价感的具体来源:逐条给你反例

把上面 6 条反过来,就是廉价感的清单。这几个错误,工程师几乎人人都犯过——逐条记住,优先消灭:

**1. 纯黑 `#000` 和纯白边界 —— 自然界里没有纯黑**
现实世界里你几乎看不到绝对的纯黑,光线让最暗的地方也带一点环境色。用 `#000` 做文字 / 阴影,大脑会觉得"刺、假、生硬"。**高级做法:用一个带轻微色相的深灰当"黑"**,比如 `#1A1A1A` 或带点蓝灰的 `#0F172A`。同理,纯黑的边框 `#000` 改成浅灰 `#E5E7EB`。

**2. 满饱和原色 —— 屏幕能显示不代表你该用**
`#FF0000` 纯红、`#00FF00` 纯绿、`#0000FF` 纯蓝,这些满饱和的原色刺眼、扎眼、显廉价(像早期网页和廉价警示标)。**高级做法:用稍微降低饱和度、调整明度的颜色**,比如红用 `#DC2626`、蓝用 `#2563EB`。大厂配色几乎找不到满饱和原色。

**3. 随手填的间距 —— 没有节奏就是乱**
按钮内边距拍脑袋写 `7px`、卡片间距 `13px`、这里 `15px` 那里 `18px`——这些"随手值"之间没有数学关系,**大脑感受不到节奏,只感受到混乱**。**高级做法:所有间距都从一个 scale 里取(`4/8/12/16/24/32/48/64`),全站只用这几个数。** 间距有了公约数,界面自然有韵律。

**4. 太多字体 —— 三种字体起步就廉价**
标题艺术体、正文宋体、按钮圆体,凑一起像拼贴报。**高级做法:全站一种字体家族,用字重和字号拉层次;最多两种且角色分明。**

**5. 长文本居中 —— 每行起点都在跳**
把一大段文字居中对齐,**每一行的左起点都不一样**,眼睛读完一行找下一行起点时会"找不着北",非常累、非常业余。**高级做法:正文一律左对齐**(中英文都是);居中只用于极短的标题 / 单行文案。

**6. 信息怼满、不留白**:见机制二,最高频的廉价来源。
**7. 每页一个样、组件不统一**:见机制五,最伤信任的廉价来源。

## 高级 vs 廉价:一张对照表

| 维度 | 廉价做法 | 高级做法 |
|---|---|---|
| "黑"色 | `#000` 纯黑 | `#1A1A1A` / `#0F172A`(带色相的深灰) |
| 主色饱和度 | `#FF0000` 满饱和原色 | `#DC2626` 降饱和、调明度 |
| 灰阶 | 纯灰 `#808080`(无色相,显脏) | 带轻微色相 `#6B7280`(冷灰) |
| 间距 | `7/13/15/18` 随手填 | `4/8/16/24/32` 统一 scale |
| 字体数 | 3 种以上混用 | 1 种为主,字重拉层次 |
| 字重对比 | 标题 24 / 正文 16(差太小) | 标题 40·700 / 正文 16·400(悬殊) |
| 正文对齐 | 居中 | 左对齐 |
| 阴影 | `0 0 5px #000`(硬黑) | 多层柔和低透明 |
| 动效曲线 | `linear` 匀速 | `ease-in-out`,150–300ms |
| 一致性 | 每页按钮各异 | token + 组件全站统一 |
| 信息密度 | 塞满一屏 | 大留白,一屏一焦点 |

## 反模式(工程师高发)

- **用"加东西"解决"不高级"**:加渐变、加阴影、加颜色、加图标——通常该做的是**减**(留白、克制),而不是加。
- **纯黑纯原色直接上**:`#000` 和 `#FF0000` 是廉价感最快的来源,改成带色相的深灰和降饱和色,五秒见效。
- **间距凭手感**:没有 scale 就没有节奏,界面必乱。
- **追求"丰富 / 活泼"**:多字体、多颜色 = 花 = 廉价。专业的方向是统一。
- **逐页面手调样式**:必然不一致。先抽 token 和组件。
- **细节不较真**:差 2px 不修、用硬黑阴影、用匀速动画——这些"用户看不出"的地方,恰恰是高级感的分水岭。

## 动手:这一章怎么练

1. **拆解练习**:打开 Apple 官网、Google 搜索结果、iOS 设置页,对照本章 6 条机制,**逐条写出"它在这里用了哪条、怎么用的"**。练到你看任何界面都能说出高级 / 廉价在哪。
2. **建一套 token**:照机制四 / 五的 `:root` 例子,给你正在做的产品定义一套颜色 + 间距 + 字号 + 圆角 + 阴影变量,**全站只引用变量,不写魔法数值**。
3. **改造练习**:拿你自己做过的一个"程序员风"界面,**只做减法和对齐**——把纯黑改深灰、间距归到 scale、文字左对齐、加大留白、统一字体,**不加任何新东西**,看它能高级多少。
4. **白嫖设计系统**:给一个项目接入一套成熟组件库(shadcn/ui 等),感受"一致性 + 细节"被现成处理好是什么体验。

## 附:5 分钟让界面变高级的清单(solo 急救包)

来不及系统重做时,**按顺序做这 7 件事,几乎立刻提升一个档次**:

1. **纯黑 → 深灰**:所有 `#000` 文字改 `#1A1A1A`,纯黑边框改 `#E5E7EB`。
2. **饱和度砍一刀**:刺眼的原色全部降饱和(红→`#DC2626`,蓝→`#2563EB`)。
3. **间距归位**:所有间距改成 `4/8/16/24/32` 里的值,删掉随手数。
4. **拉开层次**:主标题字号 / 字重显著加大(`32px+`、`700`),次要文字调灰调小。
5. **加留白**:把主要区块间距和卡片内边距**翻倍**;给正文设 `max-width: 680px`。
6. **左对齐 + 统一字体**:正文全左对齐,字体收到一种为主。
7. **修阴影**:把硬黑阴影换成柔和多层的(见机制六 CSS),动效曲线改 `ease`。

**做完这 7 条,你不需要任何审美天赋,界面也会从"开发者 demo"变成"像个正经产品"。** 高级感是规则,不是天赋——而规则,正是你最擅长执行的东西。
