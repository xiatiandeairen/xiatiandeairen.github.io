> **这一章解决一个独立开发者最容易踩的双重坑**:分发上不知道 AI 时代有个新入口;变现上用老思路定价,结果重度用户越多、亏得越多。读完你会知道冷启动期怎么借 AI 工具目录起量,以及如何用计量模型把 token 成本风险扛住。

---

## AI 产品为什么有两个特殊点

做普通 SaaS,分发和变现的老逻辑基本够用:找渠道、定价、收订阅、看 churn。但 AI 产品有两处和传统 SaaS 的本质不同,如果用旧框架套上去,会在两个方向同时出错。

**分发上多了一个新流量洼地。** AI 工具目录这个物种三年前不存在。用户今天想找一个能帮自己做某件事的 AI 工具,第一反应不是去 Google 搜,而是去 AI 工具目录上筛。这是 AI 这波浪潮打开的新流量入口,且目前竞争程度远低于 Google SEO——早进去的人先吃红利。

**变现上有一个结构性成本陷阱。** 传统 SaaS 的成本结构是"固定成本 + 很低的边际成本":服务器成本在用户量到一定规模后几乎可以忽略,所以固定订阅很合理。AI 产品不一样:**每次调用都要消耗 token,token 成本随用量线性上涨。** 一个每天重度使用的用户,产生的 API 成本可以轻松超过他每月付给你的订阅费。固定收入封顶、成本随用量飙升,这个剪刀差是 AI 产品变现的死亡陷阱。

下面拆六条打法,每条钉一个真实案例。

---

## 打法一:冷启动期逐个铺 AI 工具目录

**There's An AI For That(TAAFT)** 是这类目录里用户量较大的一个。它的逻辑很简单:用户来这里的诉求是"我想找一个能帮我做 X 的 AI 工具",然后按类别、功能、价格筛选。一个产品只要上了目录、信息写清楚,就能被动地被有明确需求的用户发现。

对于 solo 开发者,AI 工具目录的价值在于**你不需要有受众、不需要有社交账号粉丝,就能获得精准流量**。用户来目录是带着意图的——他想找某类工具,发现你,试用转化率自然比泛流量高。

具体怎么做:上线第一周就铺,不要等产品"完善"。TAAFT、Futurepedia、AI Tools Directory、Product Hunt 的 AI 专区、Toolify.ai……这些地方的提交都是免费的,一个个去做。每个目录的信息填写要针对平台风格:有的重视一句话 pitch,有的看截图,有的看标签分类。花半天时间批量提交一轮,后面就是被动曝光。

**和豆包 / Gemini 比什么?** 这是关键认知要建立清楚。字节的豆包有抖音、头条、微信(渠道合作)做分发背书,Google 的 Gemini 内嵌在搜索、Android 系统里。这些母平台分发是**结构性优势,solo 学不了、也争不了**——你在这个维度上永远输。但这些巨头在 AI 工具目录上的布局反而很弱,因为目录是小工具的主场,不是通用 AI 助手的主场。**你的战场就是它们顾不上或看不上的垂直细分场景。**

## 打法二:把 token 成本算进定价,别在发布后才发现亏损

在做定价之前,先把这道数学题算清楚。

假设你的产品是一个 AI 写作助手:

- 用户每次生成一篇文章:输入 ~1000 tokens + 输出 ~1500 tokens = ~2500 tokens
- 使用 GPT-4o 的成本:输入 $2.5/M tokens + 输出 $10/M tokens(「待核」,价格随时变动)
- 一次生成约 $0.017
- 一个普通用户一个月生成 50 次:成本约 $0.85
- 一个重度用户一个月生成 500 次:成本约 $8.5

如果你的固定订阅是 $9.9/月,看起来普通用户能赚钱,但**重度用户一个月就把你亏进去了**。而且 SaaS 产品的规律是:真正付费、长留存的往往是重度用户,他们在你这里找到了真实价值,所以用得最多——这恰恰是"被最忠实的用户拖亏"的结构。

**这道题在产品上线之前就该算,而不是等营收数据出来再补救。**

## 打法三:按量计费 / 积分制,把成本波动转移出去

解法是**把"用量"从固定成本里剥离出来,让用户的使用量和他的付费直接挂钩**。

**积分制是最顺手的实现方式**。举个真实结构:你买 100 积分 = $1,每次 AI 调用消耗 N 个积分。积分买断,不过期或慢速过期。这样做的好处:

1. 你的每次 token 消耗都有积分覆盖,成本可控;
2. 用户感知的是"用完再充",不会产生"我付了月费却没用完"的亏感;
3. 轻度用户花的少,重度用户按量付,你不再补贴任何人。

另一个常见变体是**套餐分层**:基础版每月 $9、含 50 次生成;专业版每月 $29、含 500 次;团队版每月 $99、含 2000 次。这种模式让重度用户自动升层,保证覆盖成本。

**Cursor 的案例值得参考。** Cursor 是一个 AI 代码编辑器。它的定价不是"无限 AI 调用 $X/月",而是明确告知每个套餐的"快速请求"次数(「待核」具体数字),超出后降速或按量计费。这个模式确保了重度用户不会无底洞拖累单位经济。

## 打法四:固定订阅必须设使用上限 / 公平使用条款

如果你出于转化考虑,一定要做"无限使用"的固定订阅,那有一条必须加的条款:**公平使用政策(Fair Use Policy)**。

逻辑是:绝大多数用户是轻度到中度使用者,他们不会超出某个用量阈值;极少数重度用户(常见的说法是 1-5% 的用户消耗了大多数资源,「待核」具体比例)会把成本推高。公平使用条款的作用是**给自己留一个合理的兜底机制**:超出某个用量,你有权降速、提示升级,或者限制单日调用。

这不是"欺骗用户"——它是行业标准。Cloudflare、很多邮件服务商都有类似条款。关键是**把政策写清楚、放在显眼位置**,用户买之前就知道规则,后续不会有纠纷。

**一个反面教训的结构**:有开发者做了"固定 $5/月、无限调用"的 AI 摘要工具。前三个月营收看着不错,但随着用户规模上去,有一批重度用户每天摘要几十篇文章,单月 API 成本开始超过订阅收入。到这时候再改价或加限制,用户体验很差、会引发退款投诉。**定价结构是上线前就要定死的事,不是用户增长后再调整的事。**

## 打法五:把单位经济当一等大事盯

**单位经济(Unit Economics)** 说的是:一个用户,你赚多少、花多少?两个核心指标:

- **CAC(获客成本)**:拉来一个付费用户,你花了多少钱 / 时间?
- **LTV(用户生命周期价值)**:这个用户在整个留存期内,给你贡献多少净收入?

健康的 SaaS 通常说 LTV:CAC ≥ 3(「待核」,不同阶段标准不同)。对 AI 产品来说,还有第三个变量要进计算:**每个用户的 API 成本**。正确的公式应该是:

**净 LTV = 付费收入 - 服务成本(含 API token)**

如果算下来净 LTV 是负的,增长越快死得越快——这不是比喻,是很多 AI 产品冷启动期的真实状态。

怎么盯?建一张简单的表格(不需要复杂系统):

| 指标 | 本月数据 |
|---|---|
| 付费用户数 | |
| 总订阅收入 | |
| 总 API token 费用 | |
| 毛利 = 收入 - API 成本 | |
| 每付费用户平均 API 成本 | |
| LTV(按平均留存月数估算) | |

每个月更新一次,异常立刻看原因。这张表的意义是:**你要在财务崩盘之前发现问题**,而不是等到银行账户见底才知道模式有问题。

## 打法六:母平台分发你争不到,但内容分发你能做

豆包靠字节的流量矩阵、Gemini 靠 Google 的搜索和 Android 入口——这类母平台分发根本不是 solo 的竞争维度。但有一类分发方式是人人可做的:**内容分发**。

形式包括:写一篇"我用 AI 做了什么"的 build in public 文章、在 Twitter / X 发使用案例、在 Reddit 的垂直 subreddit 解答问题并顺带提产品、在 YouTube 发使用教程……

**内容分发对 AI 产品尤其有优势**,因为"AI 工具怎么用"本身就是高搜索量的话题。你做一个 AI 写作工具,同时写一系列"AI 写作技巧"的内容,流量是双向的——用户找内容,顺路发现你的产品。

这是 Pieter Levels 多年来验证的模型:**build in public + 内容记录 = 持续的被动曝光**。他的 nomadlist.com、remote.com 等项目的流量,有相当大比例来自他持续几年在 Twitter 上的公开记录(「待核」具体流量拆分)。

关键不是"写爆款",而是**持续、真实、专注在一个细分话题**。算法和搜索引擎都偏好持续更新、主题聚焦的内容源。

## 新手 vs 资深

| | 新手 | 资深 |
|---|---|---|
| 冷启动 | 发完 Product Hunt 等流量 | 同时铺 AI 工具目录,持续被动曝光 |
| 和巨头竞争 | 想在母平台分发上竞争 | 认清巨头的结构优势,找他们顾不上的垂直场景 |
| AI 产品定价 | 固定低价订阅,不限用量 | 积分 / 套餐分层,或固定订阅加公平使用条款 |
| 成本认知 | 看收入,不看 API 成本 | 盯净 LTV,收入 - token 成本才是真正盈利能力 |
| 重度用户 | 看用量高当成留存好 | 先算这个用户有没有正净贡献,再庆祝 |
| 内容分发 | 发布日冲一波,不再管 | 持续输出垂直内容,做被动流量的复利资产 |

## 反模式(工程师高发)

- **"我的产品不是 AI 工具目录的菜"**:很多开发者觉得自己的产品太小众或太复杂,不值得上目录。恰恰相反,目录用户就是在找小众工具,这是你最准确的受众匹配场景。
- **固定低价 + 无限调用**:看起来有竞争力,实际上是在承诺一个你承担不起的成本。上线初期用量低感觉还好,规模一上来立刻崩盘。
- **"先起量再调价"**:定价结构改变是一次危机,老用户会闹,新老用户产生不公平感。**不要把上线后的调价当成退路,一开始就设计对。**
- **看到 API 账单就慌,随机涨价**:涨价的姿势比涨价本身更重要。提前告知、给老用户锁价期、说清楚原因——这是可以做到的。随机涨价、没有说明,才是真正的用户流失根源。
- **不算 token 成本,只盯 MRR**:MRR 增长是假繁荣,如果毛利为负,增速越快越糟糕。

## 动手:这一章怎么练

1. **现在就铺 AI 工具目录**:列出 5 个 AI 工具目录(TAAFT、Futurepedia、Toolify、Product Hunt AI 专区、AI Tools Directory),每个平台花 20 分钟提交你的产品信息。如果产品还没上线,先把信息模板准备好,上线当天立刻批量提交。
2. **把单位经济算一遍**:拿你现在或计划中的产品,算清楚"一个平均用户每月 API 成本是多少"、"现在定价覆盖了吗"、"重度用户的情况怎样"。这道题不超过一小时,但是分生死的一小时。
3. **设计你的用量防护机制**:决定用积分制、套餐分层还是固定订阅加公平使用条款。把规则写进落地页和用户协议。**在发布之前**定稿,不要留到后来再改。
4. **建一张月度单位经济表**:哪怕只有五行,把收入、API 成本、付费用户数、每用户净贡献记下来。每月更新一次。
