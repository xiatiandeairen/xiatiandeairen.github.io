> 会用 Figma 和**用 Figma 做出东西**是两回事。这章不教软件操作手册——教你怎么用临摹建立视觉判断力、用工具体系把迭代速度拉起来、最终做到自己开一个 Figma 文件就能把界面拖到可交付状态。

---

## 为什么要先临摹,不要先"原创"

许多工程师转型做设计会踩同一个坑:打开 Figma,想"我来设计一个界面",然后盯着空白画布半小时,要么交出一堆方块,要么堆出一个"工程风"——什么都在,什么都难看。

原因不是没有审美,而是**视觉判断力需要大量输入才能建立**。你能写出好代码,是因为你读过几万行代码,知道"这段写法不地道";同理,你能出好界面,前提是你的眼睛喂过足够多优质界面的细节——间距怎么分层、字重怎么区分信息层级、颜色怎么引导视线。这些东西不是天赋,是通过大量拆解积累的模式库。

**逆向拆解远比凭空想象高效。** 一张高质量的界面里,经验设计师的每一个决定都是有理由的。你临摹一遍,不是简单复制,而是在追问:这里的间距为什么是这个值?字号为什么要换?这个颜色处理为什么显得高级?追问出来的答案,才是你真正的收获——它会固化成你下次做自己产品时可以直接调用的判断。

**工具熟练度直接决定迭代速度。** Figma 里手慢一步是所有人都会遇到的瓶颈。一个想法从脑子里到屏幕上,中间隔着五个不熟悉的操作,你就会不自觉地减少迭代——因为太累了。工具熟了,改一版只要三分钟,你自然愿意多试几个方向。**Figma 的核心用法不多,但要练到肌肉记忆级别。**

## 打法一:Figma 五个核心机制,要练到不用想

Figma 不是全功能都要精通。你需要把下面这五条练到"不用想":

### Frame 和约束(Constraint)

Frame 是 Figma 里的基础容器,比 Group 更接近真实的 HTML 盒模型。**每一个界面区块都应该是一个 Frame,而不是用 Group 凑合。** Frame 可以设置 Clip content(截掉溢出内容),可以设置圆角,最关键的是它的子元素可以设置约束(Constraint)——Left/Right/Center/Scale,水平方向;Top/Bottom/Center/Scale,垂直方向。

约束的价值在于:你把一个按钮的 Constraint 设成 Right + Bottom,它就永远钉在右下角——不管父 Frame 拉多宽,按钮不会乱跑。这是从"做固定稿"到"做响应式稿"的分水岭。

**Apple 设置页的拆解练习**:iOS 设置页最值得临摹约束体系。每一行的文字、分隔线、箭头图标,在不同尺寸(iPhone SE 到 Pro Max)上的对齐方式都是算过的。临摹时强迫自己给每个元素设 Constraint,而不是把位置写死——做完之后把 Frame 宽度拖一拖,如果没乱,说明你设对了。

### Auto Layout

Auto Layout 是 Figma 里最接近 CSS Flexbox 的机制。选中一个 Frame,加 Auto Layout,子元素就会自动排布:方向(水平/垂直)、间距(Gap)、内边距(Padding)。

最重要的实践规则:**凡是列表、按钮、导航栏、卡片,一律套 Auto Layout,不要手动定位。** 手动定位的代价是:改一次内容,所有间距都要手动调——一张稿子里有三十个这样的地方,改一版就要花四十分钟。Auto Layout 改文字长度,间距自动撑开,省掉九成的调整时间。

这里给一套间距 scale 供参考(基于 4px grid):

```
4px  — 同一元素内部极小间距(图标与标签贴合处)
8px  — 紧相关元素间距(标签与输入框)
12px — 中等相关(列表项内部上下 padding)
16px — 标准块间距(卡片内 padding、section 间距)
24px — 较大分隔(不同内容组之间)
32px — 大块留白(页面顶部状态栏到标题)
48px — 超大留白(全屏hero区块)
```

Figma 的间距如果不在这个 scale 上(比如 13px、17px),出来的稿子通常会有"哪里不对但说不清"的廉价感——不是颜色问题,是间距的节奏乱了。

**临摹 Material 3 组件库的价值**:Google Material 3 有官方 Figma 组件库可以直接复制到自己文件里。临摹之前先拆开看 Auto Layout 是怎么搭的——每个按钮、每个卡片、每个 Chip 都是 Auto Layout 嵌套,间距规律一眼可见。这是学 Auto Layout 最快的方式:不要自己摸索,直接拆 Google 的稿子。

### 组件与变体(Component & Variant)

一个元素用了两次,就应该做成组件(Component)。三种状态(默认/Hover/Pressed),用变体(Variant)管理。

**为什么这是实操档位里最省时间的一步**:按钮颜色要改——改一个 Master Component,所有地方自动同步;状态切换——在 Variant Panel 点一下,不用复制粘贴再手动改。一个产品界面里按钮、输入框、导航栏、Tab Bar、列表项,全部组件化之后,改一版设计稿的时间从一个小时缩到十分钟。

做组件时有一个实用原则:组件越小越好。不要把整个卡片做成一个组件,而是把"卡片里的标签"、"卡片里的头像"分别做成组件,再组合。这样组合出来的组件最灵活。

**剪映案例拆解**:剪映工具栏里的每一个图标+文字组合(剪辑/音频/文字/贴纸)都是同一个模式——上方图标、下方标签、选中时底色高亮。这是教科书级的 Variant 场景:一个组件,两个 Variant(默认/选中),四种内容靠属性填入。临摹剪映工具栏,你会理解为什么"内容与样式分离"是组件设计的核心——你只维护一套样式,内容随便换。

### 样式系统(Styles)

颜色和文字要抽成 Styles,不要在每个元素上写裸值。

实操做法:在 Figma 里建一个"Styles"页面,定义:

- **Primary** `#0071E3`(Apple 蓝的示例值,使用时按实际品牌色替换)
- **Text/Primary** `#1D1D1F`
- **Text/Secondary** `#6E6E73`
- **Background/Primary** `#FFFFFF`
- **Background/Secondary** `#F5F5F7`
- **Separator** `#D2D2D7`

文字样式:

```
Title/Large  — SF Pro / 34px / Bold / -0.4px letter-spacing
Title/Medium — SF Pro / 28px / Bold / -0.3px
Body         — SF Pro / 17px / Regular / 0px
Body/Bold    — SF Pro / 17px / Semibold / 0px
Caption      — SF Pro / 13px / Regular / 0px
Caption/Bold — SF Pro / 13px / Semibold / 0px
```

(以上以 Apple HIG 规范为参照,独立项目按需调整)

**为什么不定义 Styles 就容易出问题**:不用样式,主色改一次要找几十个地方逐一修改;字号稍有出入,界面的层级感就会模糊。定义了 Styles 之后,整个产品的视觉一致性是机制保证的,不是靠人肉看齐的。

## 打法二:高质量临摹起步——不是描图,是拆解

临摹不是截图拖进来放大了描边。**有价值的临摹是:看 → 猜规律 → 按自己理解重建 → 对比原图找差距。**

### 临摹 Apple 设置页:学间距与层级

**为什么选 iOS 设置页**:这是苹果视觉语言最"教科书"的地方——没有花哨的品牌色,规律极其纯粹,全部注意力都在层级和间距上。

做法:
1. 截一张 iOS 设置主页的全屏(包含状态栏)
2. 拖入 Figma,缩小到半透明放在图层最下面,**不要描**,盖掉它
3. 从零开始用 Frame + Auto Layout 重建每一行、每一个 section
4. 建完再把原图叠上来对比:间距对了吗?字重对了吗?颜色灰度对了吗?

**从这次临摹里你会学到三件事**:

第一,**分组逻辑**。设置页的 section 之间是 35px 左右的留白(「待核」,实测为准),section 内部每行 44pt 高(这是 Apple HIG 定义的最小可触摸目标高度)。这个 44pt 不是随意的——太小了点不准,太大了浪费空间,44pt 是"不用瞄准"的最小值。

第二,**颜色的节制**。整页几乎只有两个颜色:主文字深灰 `#1D1D1F` 和副文字灰 `#6E6E73`。唯一的蓝色 `#0071E3` 只出现在可交互元素上(开关、箭头、可点击标签)。**颜色少,控制精准,每个颜色有意义**——这是设置页显得"干净"的根源。

第三,**分隔线的重量**。设置页的 separator 是 `#C6C6C8`,1px(逻辑像素),只出现在 section 内部——不是每行之间都有,而是视觉上有需要的地方才有。工程师自己做常犯的错是:到处加 border,结果界面像表格,显得廉价。苹果的分隔线是"留白做了分隔,分隔线只是辅助"。

### 临摹抖音信息流卡片:学排版密度和视觉重量

**为什么选抖音信息流**:抖音的每一帧都处理着超高信息密度——视频画面 + 标题 + 话题 + 用户名 + 互动数据 + 音乐信息,挤在一块竖屏里没有丝毫廉价感。这是极难的排版课题。

做法:截一张抖音播放页(非全屏广告那种),临摹右侧的互动区(点赞/评论/分享的纵向按钮组)和底部的标题+音乐条。

**从这次临摹里学到的**:

第一,**右侧互动区的间距**。点赞图标 + 数字 + 评论图标 + 数字,纵向排列,图标约 44px,图标与数字间距约 4px,每组之间约 20-24px(「待核」,实测为准)。这个间距比你直觉想的要大——因为这是用大拇指单手盲操作区域,间距小了误触率高,体验立刻差。

第二,**文字在深色背景上的处理**。标题文字是纯白 `#FFFFFF`,但下面带了一层渐变黑色蒙层(从底部透明到约 `rgba(0,0,0,0.5)`),保证白字在任何视频封面上都可读。这个渐变蒙层是视频类产品的标准解法——不是每个封面都做两版字色,而是用蒙层制造可预测的对比度。临摹时要把这个蒙层做出来,不能偷懒。

第三,**字号的极端克制**。抖音底部标题区字号大约 14-15px,比你以为的小——因为视频本身才是主体,文字是补充。文字字号一旦大了,视觉重心移到文字上,视频的沉浸感就破了。这是内容驱动产品里的字号哲学:文字是配角,不要抢主角的戏。

### 临摹 Material 3 组件:学组件化思维

Google Material 3 在 Figma Community 里有官方文件,直接复制到自己文件里。

**重点拆解三个组件**:

**Card**:Material 的 Card 有 Elevated / Filled / Outlined 三个变体。打开 Auto Layout 查看:内 padding 是 16px,子元素纵向 gap 是 8px(内容区)。颜色差别很微妙——Elevated 是白色底加阴影,Filled 是浅色填充无阴影,Outlined 是白色加细边框。**三种变体在视觉重量上是递减的**,用哪种取决于卡片在页面里的优先级。

**Button**:Material 3 的 Button 也有三档视觉重量:Filled(最重,主操作)、Tonal(中,次操作)、Outlined(最轻,辅助)。字体用 Label Large,14px Medium。左右 padding 24px,上下 padding 10px,圆角 100px(做成药丸形)。**这三档的存在本身就是设计决策**:你的界面里同一个位置出现了几个按钮?它们的优先级对应 Material 的哪一档?大多数工程师直觉会全做成 Filled,结果页面里全是蓝色按钮,用户不知道先点哪个。

**Navigation Bar**:底部导航栏是 72px 高,图标 24px,标签 12px,active indicator 是一个 64×32px 的胶囊形色块。图标与标签之间 4px。**这里最值得学的是 indicator 的设计**——不是简单的"选中变色",而是加了一块背景色来增加点击面积感知,让用户更清楚"我在哪里"。这个 indicator 的概念是 Material You(动态主题)的一部分:颜色来自用户壁纸提取的 seed color,组件外观随设备主题变化。这背后是 Google 把"系统感"作为信任建立方式的产品逻辑。

## 打法三:建自己的 Swipe File(灵感库)

Swipe file 是广告行业的术语:把你看到的好的东西截下来存着,需要时翻。设计里同理——你见过的优质界面是你的素材库,没见过的东西你很难"想出来"。

**操作方案**:

在 Figma 里建一个专门的文件叫 "Inspiration"。页面按类型分:导航模式、空状态、加载动画、表单、卡片、底部弹窗(Bottom Sheet)、Toast 通知……

遇到好的 App 界面,截图,拖进对应页面。**不只是存图——旁边加一条文字注释:"为什么这个好?"** 强迫自己说出原因,这个过程才真正把"见过"变成"内化"。

推荐几个高质量来源:
- **Mobbin**(mobbin.com):专门收集真实 App 截图,按 UI 模式分类,iOS/Android 都有。临摹素材首选。
- **Figma Community**:大量真实产品的设计文件开放,可以直接进去翻图层。
- **Linear**、**Vercel**、**Raycast**:这三款 SaaS 的 Web 端在设计细节上属于独立开发者的参照标杆,极简、高密度信息、视觉高级感。截图存下来,仔细看它们的字号 scale、颜色使用、空白处理。

**Swipe file 的核心价值不是"好看的截图",是你标注过原因的截图。** 三个月后你再翻,每一张都是一条设计判断,不是一张不知道为什么存的图。

## 打法四:拆开真实产品,追问"为什么这么排"

光临摹还不够——你需要**主动带着问题去看**。推荐一个习惯:每次打开一个 App,给自己三分钟,问五个问题:

1. **这个页面的视觉重心在哪里?** 眼睛第一秒落在什么地方,为什么?
2. **字号用了几级?** 标题、正文、辅助信息分别是多大?它们的差距够区分层级吗?
3. **颜色用了几个?** 品牌色出现在哪?灰色用了几档?
4. **留白有没有节奏?** 间距是否在一个规律的 scale 上,还是显得随意?
5. **哪一个细节显得"贵"?** 找一个让你觉得"这地方处理得不错"的点,具体说出是什么。

这五个问题用在任何 App 上都适用。三个月坚持下来,你会发现"第一眼感觉"开始有了具体解释——不再是"这个好看那个不好看",而是"这个好看因为 X,那个不好看因为 Y"。**这就是视觉判断力。**

**两个高价值对比练习**:

**Linear vs Jira**:同样是项目管理工具,Linear 的列表视图和 Jira 的列表视图放在一起看。Linear 的字号 scale 是 12/14/16,颜色只用灰色系 + 一个强调色,行高精确控制;Jira(尤其是早期版本)的信息密度同样高,但颜色更多、视觉重心不稳定、按钮层级模糊。差距不是功能上的,是视觉语言上的——**信息密度相同时,越"克制"越显得高级**。

**Apple.com vs 普通科技公司官网**:苹果官网的产品页有一个规律:**一屏只讲一件事**,文字极少(核心词三个字以内),大图或视频全屏。对比很多科技公司的官网:一屏放六个功能点、三个按钮、两段介绍文字,每样东西都在争夺注意力,结果每样都弱。苹果这么做的底层逻辑是峰终定律——每一屏都要制造一个清晰的视觉印象,而不是用堆砌换得"信息完整"。

## 差做法 vs 好做法对比

| 维度 | 差做法 | 好做法 |
|------|--------|--------|
| 间距 | 随手拖,13/17/23px 都有 | 严守 4px grid:8/12/16/24 |
| 颜色 | 多色(品牌蓝+警告橙+成功绿全在首页) | 主色 1 个,中性色 2-3 档,语义色分开 |
| 字号层级 | 全页字号差不多(14-16px 模糊) | 明确三级:标题/正文/辅助,最大最小差 ≥4px |
| 按钮 | 全部 Primary 蓝色按钮 | 按操作优先级分 Primary / Secondary / Ghost |
| 分隔线 | 每行都有 border | 只在需要视觉分隔的地方加,其余用留白 |
| 组件 | 每处手动复制,改一次要找 30 个地方 | Master Component + Variant,改一处全改 |
| 临摹 | 凭感觉拖个差不多 | 精确还原 + 追问每个值的原因 |
| 圆角 | 随便写 4 或 8 | 全局统一:小元素 4/6px,卡片 12/16px,按钮 8/100px |

## 新手 vs 资深

| | 新手 | 资深 |
|---|---|---|
| 起点 | 打开 Figma 凭感觉摆元素 | 先看参照,明确间距 scale 和颜色系统 |
| 间距 | 随意,看着差不多就行 | 全在 4px grid 上,有意识区分层级 |
| 组件 | 用一次复制一次 | 凡用两次就抽 Component,状态用 Variant |
| 颜色 | 看着好看多加几个 | 极度克制:主色 1 个,灰色 2-3 档 |
| 临摹 | 截图描一遍就算完 | 拆 Auto Layout、追问每个值,找差距 |
| 迭代速度 | 改一处要手动改 30 处 | Auto Layout + Component,改一次秒同步 |
| 评判标准 | "好看/不好看" | "对比度、层级、节奏对不对" |

## 反模式(工程师高发)

- **用 Group 代替 Frame**:Group 没有约束,没有 Clip,拖动后子元素位置乱跑。凡是容器就用 Frame。

- **间距不在 scale 上**:13px、17px、21px 出现在同一张稿子里,界面节奏乱。规则是:只用 4 的倍数,偶尔用 6 的倍数。

- **颜色用 HEX 裸值、不抽 Style**:主色改一次要找几十个元素。第一天就把颜色抽成 Style,永远不用裸值。

- **临摹只看"像不像",不追问"为什么"**:临摹完没有学到任何判断力。每个值都要能说出理由。

- **组件不做,靠眼睛对齐**:眼睛对不齐,人会累,稿子也会有微小但可见的错位。

- **一上来就想原创**:视觉判断力不够时原创只会产生"有想法但难看"的结果。先临摹 10 张优质界面再谈原创。

- **忽视字重**:把字重当装饰而不是层级信号。正文用 Regular,标题用 Semibold/Bold,辅助信息用 Regular 但颜色变浅——这套组合是层级感的来源。

## 动手:这一章怎么练

1. **复刻一个真实 App 界面(完整路径)**:选 iOS 设置页或 Linear 的 Issue 列表。截图→拖入 Figma→盖掉→从零重建。要求:全部用 Frame+Auto Layout,不手动定位;间距全在 4px grid 上;颜色全部抽 Style。完成后叠回原图对比,记录三处"我的跟原版哪里不一样,为什么"。

2. **建你自己的 Swipe File**:新建 Figma 文件 "Inspiration",第一周往里存 20 张截图,每张旁边写一句"这里好在哪"。不用多,20 张认真标注比 200 张什么都不写有价值得多。

3. **拆 Material 3 官方组件库**:把 Google 的 Material 3 Design Kit 复制到自己文件里,打开每一个组件看 Auto Layout 是怎么嵌套的,间距是哪几个值。不要自己猜,直接读别人写好的答案。

4. **五问练习**:这周用的每一个 App,打开之后给自己三分钟问五个问题(视觉重心/字号级数/颜色数量/留白节奏/哪一个细节显得贵)。坚持两周,你会发现看界面的视角开始变。

## 自测清单

- [ ] 我能区分 Frame 和 Group,并且在所有容器上用 Frame
- [ ] 我的间距全在 4px grid 上,同一张稿子里没有出现 13/17/21px
- [ ] 我定义了颜色和文字 Styles,没有任何裸 HEX 值散落在元素上
- [ ] 我临摹过至少一张优质界面,能说出三处"为什么这么做"
- [ ] 我有一个 Swipe File,并且每一张图旁边都有文字注释
- [ ] 我的按钮用了 Component + Variant 管理,不是一个一个手动复制
- [ ] 我能在五分钟内说清楚一个陌生 App 的颜色用了几个、字号分几级、视觉重心在哪

## 真实参照

- **Figma Community — Material 3 Design Kit**(Google 官方):拆组件学 Auto Layout 的最佳素材,直接可用
- **Apple Human Interface Guidelines**(developer.apple.com/design/human-interface-guidelines):iOS 组件规范、间距、字号、颜色的原始出处,免费在线读
- **Mobbin**(mobbin.com):真实 App 截图库,按模式分类,临摹素材库首选
- **Refactoring UI**(Adam Wathan & Steve Schoger):工程师转型设计必读,专门讲"为什么你的界面看起来像个原型而不是产品",有大量 before/after 对比
- **Linear**、**Vercel**、**Raycast**:三款独立开发者圈公认的高设计水准产品,直接截图拆解
- **Design+Code**(designcode.io):Figma 实操教程,偏 iOS/SwiftUI 方向,有免费章节「待核」是否持续更新

---

*注:Apple HIG 中 44pt 最小触摸目标、4px/8px 间距 grid 为公开设计规范;具体 App 内部 px 值(抖音信息流间距、iOS 设置页 section spacing 等)为公开界面实测估算值,标「待核」处请以实测为准;Figma Community 上的 Material 3 设计文件为 Google 官方发布的公开资源。*
