训练 loss 从 4.160 掉到 1.689,这说明不了任何事。loss 只告诉你模型在拟合训练对(center 预测 context)这个代理任务上越来越好;它不告诉你向量里有没有语义。你完全可以把 loss 压到很低,而向量空间是一团各向异性的浆糊——所有向量挤在一个窄锥里,cosine 全是 0.9+,"相似"这个词失去意义。所以训完第一件事不是庆祝 loss 下降,是换一把跟 loss 无关的尺子去量它。这一章就是那把尺子,而且是一整套——因为没有任何单一指标能独立证明"向量懂语义",每一个都有它的盲区和作弊位。

配套代码在 `examples/embeddings-from-scratch/src/stage07-eval-analogy.ts`。它干了一件刻意的事:在同一个文件里用三种方法(PPMI 计数、softmax skip-gram、负采样 skip-gram)各训一套向量,然后用**同一套评估协议**量它们。这样"哪个方法更好"才有意义——不同尺子量不同向量得出的排名是垃圾。文件头有一段坦白值得先读:

```typescript
// HONESTY ON ABSOLUTE NUMBERS: the corpus is ~40 words of templated toy text and
// training is seconds, not hours. Absolute accuracies here are OPTIMISTIC versus
// real corpora (the structure is dense and noise-free). What transfers is the
// MECHANISM (vector arithmetic recovers analogies at all) and the RELATIVE trend
// (learned word2vec vectors beat raw count vectors; excluded < not-excluded).
```

记住这句:下面所有绝对数字对 toy 语料偏乐观,能迁移的是**机制**和**相对趋势**,不是 0.80 这个具体值。真实大语料上的类比准确率是百分之几十,不是 80%。

## 最近邻:最便宜也最容易自欺的嗅探

最近邻(nearest neighbor,给一个词找向量空间里 cosine 最接近的几个词)是你训完该跑的第一个测试,因为它零成本、肉眼可读、能立刻抓出灾难性 bug。给 `king` 找 top-3,如果回来的是 `queen / prince / princess`,你知道训练至少没崩;如果回来的是 `the / a / um`,你知道空间塌了,不用再往下看。

stage07 用 [04] 负采样向量跑出来的定性表:

```
king     → queen(0.96)  princess(0.94)  prince(0.94)
queen    → king(0.96)  prince(0.92)  princess(0.91)
dog      → bone(0.89)  um(0.89)  cat(0.73)
fish     → he(0.88)  bread(0.78)  ate(0.71)
north    → west(0.95)  south(0.92)  east(0.89)
um       → dog(0.89)  bone(0.81)  mouse(0.81)
```

`king` 和 `north` 那两行很漂亮——皇室聚一起、方位聚一起,这是你想看到的。但**真正值得讲的是难看的行**。`dog` 的第二近邻是 `um`(一个口头填充词,语料里几乎不携带结构),`fish` 的最近邻是 `he`。这不是 bug,是诚实——`um` 在这个 toy 语料里出现得几乎随机,它没有稳定的上下文,所以学不出有意义的向量,它的最近邻必然是噪声。代码注释把这一点点破:

```typescript
// 注: 'um' 是 filler 词 (语料中几乎不携带结构), 其最近邻应当杂乱无意义 —
//     这正是'数据里没有的结构学不出来'的失败模式演示。
```

### 为什么这是个失败模式而不是缺陷

这是嵌入的根本约束:**向量只能编码语料统计里存在的结构**。`um` 没有稳定共现 pattern,模型对它的向量就是初始化噪声被少量梯度轻推后的残留。如果你的真实系统里某个重要实体的最近邻一团乱,先别怀疑模型——去查它在语料里出现几次、上下文稳不稳定。十有八九是数据问题,不是算法问题。

最近邻的盲区也正在这里:**它只能定性,不能定量**。`king → queen` 好看,但你没法用一张人工读的表去比较"softmax 和负采样哪个整体更好"。人工嗅探十几个词就累了,而且你会无意识挑那些好看的词去看(确认偏误)。所以需要第二把尺子。

## king - man + woman ≈ queen:不是魔术,是训练目标的副产品

✦ 这是整本书最有名的一行,也最被误解。`king - man + woman ≈ queen` 经常被当成嵌入"涌现"出的神奇智能。它不是。它是一个可以从训练目标推导出来的几何后果,而且**只在向量足够好时才成立**——所以它恰恰是检验表示质量的试金石,不是炫技。

机制是这样的。skip-gram 的目标是让共现的词点积大。如果语料里 `king` 和 `man` 共享一批上下文(都跟 `royal`、`he`、`throne` 等出现),`queen` 和 `woman` 共享另一批(跟 `she`、`royal`、`throne`),那么"性别"这个区分维度,在 `king` 和 `man` 上的投影会接近,在 `queen` 和 `woman` 上也接近。于是向量差 `king - man` 和 `queen - woman` 在性别那个方向上指向同一处——这就是一条近似平行的"性别轴"。`king - man + woman` 等于"拿 king 减掉它的男性分量再加上女性分量",几何上自然落到 `queen` 附近。

关键词是**近似平行**和**线性**。类比算术 `b - a + c ≈ d` 暗含一个强假设:语义关系在向量空间里是**线性平移**,而且不同词对的同一关系是**平行向量**。这个假设在 word2vec 这类浅层共现模型上经验地成立得不错,但它不是定理,是巧合性的几何规律。关系越抽象(比如"首都-国家"比"单复数"难)、词频越低,平行性越差,类比就越解不出。这是第一个盲区:**类比准确率低,可能是向量差,也可能是这个关系本来就不线性**——指标分不开这两者。

stage07 把类比题集做成沿三条已植入语义轴枚举,不是手挑好看的题:

```typescript
const specs: Array<[string, string, string, string]> = [
  // gender axis — the canonical king-man+woman family
  ["king", "queen", "man", "woman"],
  ["man", "woman", "king", "queen"],
  ...
  // royalty-rank axis (king:prince :: queen:princess — "the junior of")
  ["king", "prince", "queen", "princess"],
  ...
  // direction axis (opposite-of structure)
  ["north", "south", "east", "west"],
];
```

## 作弊位:不排除输入词,准确率全是假的

⚡ 现在讲类比评估那个著名的、至今没有"一劳永逸通用解"的坑。解类比 `b - a + c ≈ ?` 时,你算出目标向量后要在词表里找最近的词当答案。问题是:**算出的目标向量,几何上往往离 b 和 c 自身最近**。`king - man + woman` 这个向量,跟 `king`、`woman` 的 cosine 经常比跟 `queen` 还高。如果你不把 a、b、c 三个输入词从候选池里排除,模型会直接"答"出一个输入词——而你测的根本不是它学没学会关系,是"答案是不是输入之一"这个跟语义无关的几何偏置。

这是评估协议里最容易踩、踩了数字还特别好看的坑。Mikolov 原始 word2vec 论文的标准协议就是排除三个输入词;后来无数复现因为漏了这一步,报出虚高的准确率。代码里两个协议并排实现,`analogyAccuracyNoExclude` 故意传空排除集来复现错误做法:

```typescript
function analogyAccuracyNoExclude(
  matrix: number[][],
  questions: AnalogyQuestion[],
): { accuracy: number; correct: number; total: number } {
  let correct = 0;
  for (const q of questions) {
    const got = analogySolve(matrix, matrix[q.a], matrix[q.b], matrix[q.c], []); // no exclusion
    if (got === q.expected) correct++;
  }
  ...
}
```

注意第 4 个参数那个空数组 `[]`——正确协议传的是 `[q.a, q.b, q.c]`。stage07 把排除/不排除两种协议的准确率并排打出来:

```
[02] PPMI      排除输入: 8/10 = 0.80   |  不排除: 2/10 = 0.20   |  落差 -0.60   |  不排除时 7/10 题答成了输入词
[03] softmax   排除输入: 7/10 = 0.70   |  不排除: 1/10 = 0.10   |  落差 -0.60   |  不排除时 8/10 题答成了输入词
[04] neg-samp  排除输入: 8/10 = 0.80   |  不排除: 1/10 = 0.10   |  落差 -0.70   |  不排除时 9/10 题答成了输入词
```

三法合计 **24/30 题**,不排除时的"答案"其实是某个输入词本身。看 [04] 的逐题对比最直观:

```
king:queen :: man:woman      排除→woman    ✓  不排除→man      ✗ (答成了输入词!)
king:queen :: prince:princess 排除→princess ✓  不排除→prince   ✗ (答成了输入词!)
north:south :: east:west     排除→west     ✓  不排除→east     ✗ (答成了输入词!)
east:west :: north:south     排除→south    ✓  不排除→north     ✗ (答成了输入词!)
```

同一套向量,同一道题,只因为候选池里多留了三个输入词,正确答案就被输入词挤掉了。

### 这里有个反直觉点:这套题里"不排除"反而让分数*下降*

你可能注意到落差是负的(-0.60、-0.70),分数变低了。这跟"作弊刷高分"的直觉相反,得讲清楚,不然你会以为不排除是安全的。原因在于**这套题集里 expected 答案永远不等于任何输入词**(`king:queen::man:?` 的答案是 `woman`,不是 king/queen/man 里任何一个)。所以不排除时,输入词占据了最近位,把正确答案挤出去,准确率下降。

但"作弊"的方向需要 expected 恰好就是某个输入词才会显现。代码专门构造了一组退化题来复现这个方向——把 expected 故意设成输入词 `b`:

```typescript
const degenerate: AnalogyQuestion[] = questions.map((x) => ({
  a: x.q.a,
  b: x.q.b,
  c: x.q.c,
  expected: x.q.b, // expected = b 本身 (输入词)
}));
```

跑出来:

```
退化演示 (expected 故意设成输入词, 复现'不排除虚高'方向, 用 [04] 向量):
  排除输入: 0/10 = 0.00   不排除: 2/10 = 0.20   虚高 +0.20
```

同一套向量,仅因不排除,准确率从 0.00 升到 0.20。升幅有限(因为 `b-a+c` 有时落到 c 而非 b),但**这 +0.20 全是输入偏置白拿的假分**——expected 本来就是输入词,命中毫无语义意义。

把两个方向合起来看,结论是:**不排除时算出的数字,无论是涨是跌,测的都是 `b-a+c` 的输入偏置,不是习得结构**。它跟向量质量没有单调关系,所以它本身就是个无意义的数,必须排除输入词才能让这个指标重新有定义。这就是为什么 `analogyAccuracy`(正确协议)是默认导出、`analogyAccuracyNoExclude` 只是个演示用的本地函数。

### 为什么这个坑至今没有"通用解"

排除三个输入词解决了最低级的作弊,但类比评估更深的问题还开放着:**类比题集本身怎么算公平?** 排除了输入词,模型还能靠词频先验(高频词更容易被选中)、靠词表大小(候选越少越容易蒙对)刷分。更根本的,前面说的"语义关系=线性平移"这个假设,在不同关系类型上成立程度天差地别,所以一个综合类比准确率把"复数"和"国家首都"混在一起平均,数字几乎没有可解释性。社区后来的反应是干脆少用类比、转向下游任务评估(把嵌入喂进真实任务看效果),但下游评估又把"嵌入质量"和"下游模型质量"耦合在一起,分不开。**"如何内在地、不依赖下游任务地度量一个嵌入空间好不好",目前没有公认的通用答案,仍是开放问题。** 你能做的是:别信任何单一指标,多个正交指标交叉看,并且永远报告你用的协议(排不排除、题集怎么来的)。

## 分离度:绕开类比的几何假设

类比指标的盲区是它绑死了"线性平移"假设。第三把尺子绕开它:不问"关系是不是平行向量",只问一个更弱、更稳的问题——**同类词的 cosine 是不是显著高于随机词对**。如果连这个都不成立,空间就没聚类,什么都别谈。

`clusterSeparation` 算两组 cosine 的均值:同簇内所有词对(应该高)vs 随机抽的词对(应该接近 0),报告两者之差:

```typescript
const clusters: string[][] = [
  ["king", "queen", "prince", "princess"],
  ["dog", "cat", "mouse", "bird"],
  ["bread", "cheese", "rice", "fish"],
  ["north", "south", "east", "west"],
];
```

随机对的抽样有个易错点代码专门处理了:抽到的随机对如果碰巧同簇,要丢弃重抽,否则随机基线被污染、分离度虚高。还有个跨模型公平性细节——三个模型必须看**同一批**随机对,所以每个模型都用 `new Rng(999)` 同种子重置:

```typescript
// fresh rng (same seed) per model so all three see the SAME random pairs —
// otherwise the random-pair baseline would differ and the comparison is unfair.
const sep = clusterSeparation(m.vec, vocab, new Rng(999));
```

跑出来的分布分离:

```
[02] PPMI      同类均值 0.539  随机均值 0.083  分离度 0.456
[03] softmax   同类均值 0.684  随机均值 0.058  分离度 0.626
[04] neg-samp  同类均值 0.755  随机均值 0.297  分离度 0.457
```

这里藏着比类比准确率更细的信息。神经向量(03/04)的同类均值明显高于 PPMI——它们把同类词聚得更紧。但注意 [04] 的随机均值 **0.297** 偏高(PPMI 才 0.083)。这是个轻微的警告信号:负采样向量有点各向异性倾向(所有向量略微同向,连随机对都有正 cosine),这正是本章开头说的"浆糊"风险的苗头。softmax 反而随机均值最低(0.058)、分离度最高(0.626)——它的空间更"干净"。

这就是为什么单看类比准确率([04] 和 [02] 都是 0.80,打平)会漏掉真相:**分离度告诉你 [02] 的 0.80 是靠稀疏高维侥幸得来,而 [04] 的随机基线偏高暗示它有各向异性问题**。两个指标正交,各抓一面。这也是 MEMORY 里那条教训的具体化——机械的单一打分会饱和、会骗你,得用多个角度交叉。

## 三法横向对比:别从 toy 数字得出错误结论

最后把所有指标汇总到一张表,用同一口径(类比一律用正确的排除协议)比三种方法:

```
方法            类比准确率   同类-随机分离度   维度
[02] PPMI          0.80            0.456     43
[03] softmax       0.70            0.626     24
[04] neg-samp      0.80            0.457     24
```

PPMI 零训练就拿到 0.80 的类比分,跟训练了 40 epoch 的负采样打平。**这里是最容易得出错误结论的地方**:你会想说"那训练没用啊,计数就够了"。错。代码注释把这个陷阱标得很清楚:

```typescript
// - 在这个稠密 toy 语料上, PPMI 零训练就拿到不俗的类比分 —— 别据此宣称'计数=学习':
//   真实大语料里 PPMI 是 V 万维稀疏向量, 类比算术与最近邻都远不如低维稠密向量,
//   且 O(V^2) 内存不可行 (见 stage02 注释)。toy 规模掩盖了这一差距, 这正是 honesty 要点。
```

为什么 toy 上 PPMI 这么强?因为这个语料只有 43 个词、结构稠密无噪声,PPMI 的 43 维向量信息密度足够。真实语料词表是几万到几十万,PPMI 矩阵是 `V × V` 的稀疏巨阵,`O(V²)` 内存直接不可行,而且高维稀疏向量的 cosine 被大量 0 分量稀释(看上面 PPMI 同类均值才 0.539,神经向量 0.755)。低维稠密向量(D=24)用几十维就压缩了同样的结构,这才是 word2vec 的价值——**toy 规模恰好掩盖了维度灾难,让计数法看起来不输训练法**。这是 N=1 玩具实验最危险的外推陷阱:相对趋势(神经向量分离度更高)能迁移,绝对排名(PPMI 打平)不能。

维度那一栏顺便回答了一个工程问题。softmax 和负采样都用 D=24 拿到了跟 PPMI 的 43 维相当甚至更好的分离度——**稠密低维不只是省内存,它本身就是一种有损压缩,逼模型把语义塞进有限维度,反而过滤掉了稀疏共现里的噪声**。这跟第 8 章要讲的降维(把 D=24 再压到 2 维肉眼看)是同一个直觉的延续:好的表示在低维里依然保持结构。

效率上,负采样以 `O(K)` 代价(K=5 个负样本)逼近 softmax 的 `O(V)` 全词表归一化,本次运行快了约 **7.3 倍(2597ms vs 357ms)**——在 V=43 的 toy 上就有 7 倍,真实 V=几万时这个比值会放大成"能不能训"的差别。这也印证了为什么第 4、5、6 章要费那么大劲讲负采样和 InfoNCE:不是为了更准,是为了让训练在大词表上**根本可行**。

## 把这章当成一套检查单

回到开头那句:没有单一指标能证明向量懂语义。这一章给你的是一套互补的尺子,每把都有明确的盲区,合起来才有判断力:

- **最近邻**——零成本嗅探,抓灾难性 bug;盲区是只能定性、易确认偏误。先跑它,难看就别往下查算法,去查数据。
- **类比准确率**——量化关系结构;盲区是绑死线性假设、且**必须排除输入词**否则数字无意义。报告时永远写清协议。
- **分离度**——绕开线性假设的更弱更稳的检验;能抓出类比准确率漏掉的各向异性问题。
- **横向对比**——但 toy 绝对值会骗你,只信相对趋势,警惕从"PPMI 打平"外推到"训练没用"。

这套判据不是 stage07 专属——它就是前面第 2-4 章那几个 stage 训练时该用的共用尺子,也是第 8 章可视化之前最后一道定量关卡。训完一套向量,先用最近邻嗅一下,再跑排除输入词的类比,再看分离度,三个都过得去,才轮到把它降维画出来给人看。
