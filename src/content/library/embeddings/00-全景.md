先说一个我见过太多次的场景:有人 `pip install sentence-transformers`,`model.encode(texts)`,余弦相似度排个序,系统就上线了。它能用——直到换了个语料(中文换法律文书、英文换医疗记录),召回突然变差;或者有人把相似度阈值写死成 0.7,换个模型后这个数字毫无意义。问题不在 API 用错,而在于**他们把 embedding 当成一个黑盒常量,而它其实是被一个具体训练目标、在一个具体语料上、用一组具体负样本拟合出来的参数。** 训练目标决定了"什么算近",换语料就是换数据分布,阈值是分布的函数不是物理常数。这本书的全部目的,就是把这个黑盒拆开:embedding 是怎么从一堆离散符号被训成稠密向量的。懂了训练机制,你才知道换语料该重训还是微调、阈值该怎么定、向量"看着稠密"为什么可能根本没语义。

配套代码仓在 `examples/embeddings-from-scratch/`,纯 TypeScript、离线、CPU 可跑,不依赖 PyTorch / 任何 ML 框架,自带一个标量自动微分(autograd,自动求梯度)引擎(`src/core/autograd.ts`),不需要 LLM / API key / 网络。每章对应一个 stage 文件,可以 `npx tsx src/stageNN-*.ts` 单独跑,loss 下降、最近邻、类比准确率全是代码真算真打印的。所有随机来自种子 PRNG(`src/core/rng.ts`,全程禁用 `Math.random`),同 seed bit-for-bit 可复现。这章本身没有独立 stage——它的职责是把后面 8 个 stage 摆进同一张地图,并讲清楚这张地图为什么长这样。

> ⚡ 关于数字的诚实声明,贯穿全书:配套语料是 `src/core/text.ts` 生成的 toy 语料(约 40 词、模板化、语义聚类是刻意埋进去的)。**绝对数字一律偏乐观**(40 词词表、结构密集无噪声),后面每章打印的 loss / 准确率 / 相似度的绝对值都不要拿去对标真实系统。可迁移的是**机制**(loss 真的靠真实梯度在降)和**相对趋势**(单步成本随词表线性涨、PPMI 比原始计数干净、excluded 准确率低于 not-excluded)。每处我都标注。

## 表示学习要解决什么:把"语义近"变成"几何近"

机器学习模型吃的是数字,但语言、用户、商品这些东西本质是离散符号。最朴素的编码是 **one-hot**(独热,一个词占一维、那维是 1 其余全 0)。它的问题不是"维度高"这种工程小事,而是一个结构性绝症。

stage01 (`src/stage01-onehot.ts`) 把这个绝症测成了数字。one-hot 编码一个词表,本质就是把单位矩阵的每一行拿来用——任意两个不同词的 one-hot 向量,余弦相似度**恰好是 0**。代码里这样验证:

```ts
// stage01-onehot.ts — 扫遍所有不同词对,报告见过的最大余弦
let worst = 0;
for (let i = 0; i < vocab.size; i++) {
  for (let j = i + 1; j < vocab.size; j++) {
    worst = Math.max(worst, Math.abs(cosineSimilarity(oneHot[i], oneHot[j])));
  }
}
// measured max |cos| over all distinct pairs = 0.0000
```

这句话翻译成人话:`king` 和 `queen` 的相似度,跟 `king` 和 `fish` 的相似度,**一模一样,都是 0**。几何里压根没有"更像一点"的容身之处。表示学习要解决的第一件事就是这个——造一个稠密(每维都带值)、低维的空间,让"语义近"能被表达成"几何近"(余弦大 / 距离小)。

stage01 还顺手演示了高维空间的第二个坑:**维度灾难**(curse of dimensionality)。它对 `d = 2/10/100/1000` 各撒一堆高斯随机点,测"最近点距离 / 最远点距离"的比值:

```ts
// stage01-onehot.ts
const dims = [2, 10, 100, 1000];
// d 越大,minDist/maxDist 越趋近 1:最近的点几乎和最远的点一样远
const ratio = maxDist === 0 ? 1 : minDist / maxDist;
```

维度越高这个比值越逼近 1——最近邻和最远邻几乎一样近,"最近"这个词在高维随机数据里逐渐失去意义。one-hot 恰好活在 `d = 词表大小` 的空间里(真实里 10^5 以上),早就泡在这个对比度塌缩的区域。这正是下游《向量检索引擎》那本书开篇要处理的同一个魔鬼(那里讲怎么在这种空间里**检索**;这里讲怎么先把向量**训出来**,让结构能扛住降维)。

那解药长什么样?stage01 第三段手摆了一组稠密向量当预览:同主题的词(king / prince)放在同一根"royalty 轴"上,哪怕它俩没有任何字面重叠,余弦也非零。一个刻意设计的诚实细节是,demo 里每个主题的两篇文档**共享零个字面词**(royalty-A 用 king/queen,royalty-B 用 prince/princess):

```ts
// stage01-onehot.ts — 同主题文档刻意不共享任何字面词,
// 否则 bag-of-words 会靠字面重叠"蒙对",悄悄推翻本章论点
{ label: "royalty-A", words: ["king", "queen", "rules"] },
{ label: "royalty-B", words: ["prince", "princess", "kingdom"] },
```

这样 one-hot 词袋在同主题检索上退化到约等于随机(距离只由字面词重叠决定,跟语义无关),而手摆的稠密码能把同主题文档聚到一起。**注意:stage01 的稠密向量是人手摆的,不是学出来的。** 怎么从数据里自动学出这些轴,是 stage03 之后的事。这就引出了全书的主线坐标。

## 全书地图:一条从"统计"到"学习"的坐标轴

把 8 个 stage 摆在一根轴上,左端是"纯数手指头数出来的统计量",右端是"用梯度下降在预测任务上训出来的参数"。整本书就是沿着这根轴往右走,每往右一步换来什么、代价是什么,都用代码量出来。

**stage01 — one-hot 与维度灾难(轴最左,连统计都还没开始)。** 上一节讲完了:确立"为什么需要稠密表示",并把维度灾难、正交性、词袋检索失败这三件事测成数字。这章的稠密向量是手摆的,是一张"目标长什么样"的预览图。

**stage02 — 共现矩阵与 PMI(`src/stage02-cooccur-pmi.ts`,第一个能学出语义的 count-based 表示)。** 这章的论点很挑衅:**不需要梯度下降,光靠数共现 + 一个信息论重加权 + 经典线性代数,就能得到带真实语义结构的词向量。** 流程是:数共现矩阵 → 算 PPMI(positive pointwise mutual information,正的点互信息,衡量"两个词一起出现是不是比偶然更频繁")→ 截断 SVD 降维。这章故意不 import autograd——"无训练"就是它的全部卖点。它埋了一个漂亮的失败模式:**原始计数的"关联强度"被频率主导**,每个查询的最近邻都是同一批高频词(the / a / is),因为高频词给所有人的点积都贡献一个大项:

```ts
// stage02-cooccur-pmi.ts — 未归一化的关联强度(原始计数行的点积)
// 是失败模式:高频词 j 给很多词对贡献 cooc[q][j]*cooc[i][j] 两个大因子,
// 淹没每个查询的 top 列表
```

PPMI 把每个词的频率基线除掉,同样的数据、频率折掉之后,几何结构就干净了。这章还会把 word2vec(下一章)预先 reframe 成"对这个 PMI 矩阵的隐式、可扩展的分解"(Levy & Goldberg 2014),让神经版本不再神秘。

**stage03 — skip-gram + 全 softmax(`src/stage03-skipgram.ts`,全书的转折点)。** 这是 embedding **第一次真的被训练**——从"你算出来的统计量"变成"被梯度下降优化的参数"。模型是教科书形态的 word2vec skip-gram:两张嵌入表(中心词表 W_in、上下文表 W_out),`p(context | center) = softmax(对全词表的点积)`,损失是负对数似然。它故意用最朴素的全 softmax,因为这样机制最透明——也因为它**不可扩展**:softmax 的分母要对整个词表求和,每个训练对都付 O(V) 的代价。stage03 直接把这个墙测出来:

```ts
// stage03-skipgram.ts — ③ 词表规模扫描,单步 forward+backward 耗时随 V 增长
const vocabSizes = [50, 200, 800];
const stepMs = vocabSizes.map((v) => measureStepMs(sweepRng, v, DIM, 7));
// 结论:每个训练对都要对全词表做 V 次点积,成本随 V 线性涨
```

这章还演示了三个失败模式,每个都讲了"为什么这么设计":学习率过大 → loss 发散(但 stable softmax 兜住不爆 NaN);朴素 softmax 去掉 max 平移 + 大 lr → `exp` 溢出真爆 NaN(这正是 log-sum-exp 技巧存在的理由);初始化方差过大 → softmax 饱和、梯度消失、几乎学不动。三个看着像、根因完全不同的坑,分开演示。

**stage04 — 负采样(`src/stage04-negative-sampling.ts`,砍掉 O(V) 那堵墙)。** Skip-Gram with Negative Sampling(SGNS,Mikolov 2013)把"一次 V 路分类"换成"(1+k) 次独立的二分类":这个 (center, context) 是真的吗?这些 (center, negative) 是假的吗?每步成本变成约 (1+k) 次点积,**基本与词表大小无关**——这一个改动让 word2vec 能扩到百万词表。这章扫 k=1/5/20 暴露成本↔质量权衡,并对比三种负采样分布(uniform / unigram / unigram^0.75),解释 word2vec 为什么挑 0.75 次幂。失败模式:原始 unigram 采样会让几乎每个负样本都是高频填充词(the / a),模型把容量浪费在推开它本来就忽略的停用词上 → 退化向量。

**stage05 — InfoNCE / 对比学习(`src/stage05-infonce.ts`,负采样的现代化身)。** 这章的论点是:skip-gram 负采样和现代对比学习(SimCLR / MoCo / 句向量)**是同一个想法穿了不同衣服**——"把正样本对拉近,把其余一切推开"。InfoNCE 把"其余一切"做成字面意义上的 batch 内其他成员:每个 anchor 有一个正样本(自己的增强视图),同 batch 的其他成员当负样本,不需要单独的负采样器。它会打印训练前后的 **alignment(正样本有多近,越低越好)和 uniformity(在球面上铺得多开,越负越好)**,这对 Wang-Isola 诊断指标证明空间真的分开了。失败模式直接把**表示坍缩**(representation collapse,所有向量挤成一坨)测成一个数:把温度调到接近 0 且只用正样本(分母没有负样本)→ uniformity 退化到约等于 0。

**stage06 — 温度与负样本数(`src/stage06-temperature-negatives.ts`,对比学习的两个旋钮)。** 这章把 stage05 留下的两个超参拧到底。**温度 τ** 是唯一决定"loss 多尖锐地聚焦于最难的那个负样本 vs 把压力摊到所有负样本上"的旋钮:太小 → softmax 变成硬 max、梯度集中在一个负样本上、训练抖、空间过度分离几个方向;太大 → softmax 摊平趋于均匀、正负样本不再可分、空间坍缩。**负样本数**越多互信息下界越紧、uniformity 越好,但**急剧边际递减**(对数式)。还有**硬负样本**(语义上接近 anchor 但不是真正样本):适量注入更强梯度、锐化空间,过量则混入太多假负样本(本该接近的东西被推开)、分离反而回退——一条先升后降的非单调曲线,不是"越多越好"。这章每条曲线都靠真实 Adam 训练算出来,可迁移的是曲线**形状**(τ 有内部最优、负样本数递减、硬负样本先升后降),不是绝对值。

**stage07 — 评估:最近邻 + 类比题(`src/stage07-eval-analogy.ts`,判一个空间好不好)。** 前面在造向量,这章问唯一重要的问题:它们到底好不好,你怎么知道?手段是一组几何测试——最近邻嗅探、著名的 king - man + woman 类比算术、相似度分布分离度——施加在用三种方法(PPMI 计数 / skip-gram / SGNS)训出的向量上,放到同一把尺子上排名。本章必须演示的失败模式是:**类比准确率必须把三个输入词从候选池里排除**,否则 b - a + c 通常最接近 b 或 c 自己,你测的是"答案是不是输入之一",报出一个漂亮但毫无意义的数。代码两种都算并打印差距,让虚高可见:

```ts
// stage07-eval-analogy.ts — load-bearing 教学点:
// analogy accuracy is meaningless unless you EXCLUDE the three input words
// from the candidate pool. 不排除 = 测"答案是不是输入之一",虚高。
```

**stage08 — 降维可视化:PCA 与 t-SNE(`src/stage08-pca-tsne.ts`,把 D 维空间画给人看)。** 训出的 embedding 活在 D 维(配套里 D=16),人只能看 2 维。每张 2D 图都是有损投影,**投影方法决定哪些结构存活、哪些被毁**。PCA 是线性投影(到方差最大的两个方向,有唯一全局解);t-SNE 是非线性、保局部邻域(用梯度下降匹配高维高斯邻居分布和低维 Student-t 分布)。然后一个 perplexity(困惑度)扫描(5/15/40)对同一份数据产出三张明显不同的 t-SNE 图——本章的硬教训:**一张 t-SNE 图是超参依赖的漫画,不是 ground truth。** 别拿一张漂亮的 t-SNE 截图当作"我的 embedding 学得很好"的证据。

把这根轴收一下:**stage01-02 是"统计"(数出来,不训练),stage03 是分水岭(开始训练),stage04-06 是"学习"的主体(怎么高效训、怎么用对比目标训、两个旋钮怎么拧),stage07-08 是"训完怎么判好坏、怎么看"。** 每往右一步,你买到了什么(可扩展性 / 更好的几何 / 统一视角)、付了什么代价(计算 / 超参敏感 / 可解释性),都在对应 stage 里量过。

## 它在哪条链路上:embedding 是检索的上游

书库里有一本《向量检索引擎》,讲的是 HNSW / IVF-PQ / Hybrid / recall 怎么测——**那本书假设向量已经存在了**,它的输入是一堆 embedding,问题是"怎么在百万级向量里快速找最近邻"。这本书是它的**上游**:embedding 本身怎么被训出来。两本书在 `vector-search/00` 讲的维度灾难是同一个,但分工干净——上游(这本)负责让向量带语义、扛得住降维;下游(那本)负责在这些向量上建索引、快速检索。

这个分工有个实际后果值得现在就记住:**检索质量的天花板由 embedding 质量决定,索引只能逼近它,不能超过它。** 索引算法再好,recall 上限也是"暴力扫描这批向量能召回的相关结果";如果这批向量的语义本来就糊(训练目标错、负样本设计差),你在下游怎么调 HNSW 参数都救不回来。这就是为什么我反对"换个更快的向量库"当作召回变差的第一反应——先回到上游问:这些向量是用什么目标、什么语料、什么负样本训的?

## ⚡ "embedding 是免费午餐"是个错觉

最后点一个会贯穿全书、目前没有通用解的开放问题。很多人默认 embedding 是"免费的语义"——`encode` 一下就拿到一个稠密向量,稠密=有信息=有语义。**这是错觉。表示的好坏完全取决于训练目标和负样本设计,目标错了,向量看着稠密其实没语义。**

这不是一句口号,是后面每章用真实数字戳破的:stage05/06 的坍缩 demo 会让你看到,目标设计差时所有向量挤成一坨,余弦相似度算出来还都挺高(因为大家方向一致),但 uniformity 接近 0——它**看着能用,其实把所有东西都判成相似**;stage02 会让你看到,关联度量不归一化时,"最近邻"全是高频停用词,稠密归稠密,语义全是频率噪声;stage04 会让你看到,负采样分布选错(原始 unigram),模型把容量全花在推开停用词上,留给真实语义的容量所剩无几。

为什么说它"目前无通用解"?**因为"什么训练目标 / 什么负样本能产出好表示"在很大程度上是任务相关的、靠经验和评测试出来的,没有一个理论能在训练前告诉你"这个目标会不会给你想要的语义"。** 对比学习里"难负样本挖到多硬最优"、句向量里"用什么增强构造正样本对"、检索里"in-batch 负样本够不够 vs 要不要 hard negative mining"——这些都还在活跃研究,2024 年起一批工作仍在用更大 batch、更巧的负样本构造、更对齐下游的训练目标去刷,没有收敛到一个"照着做就对"的配方。alignment / uniformity 这类诊断指标(stage05)能在训练**后**告诉你空间塌没塌,但不能在训练**前**替你选目标。

所以读这本书的正确姿势不是"学会几个模型",而是:**学会判断一个 embedding 空间是不是真有语义、坏会怎么坏、坏了往哪查。** 调 sentence-transformers 的人很多,知道向量是怎么来的、换语料为什么会废、阈值为什么不能瞎设的人不多。这本书想让你成为后者。下一章 stage01,我们从那个余弦恰好为 0 的绝症开始,把"为什么必须有稠密表示"测成数字。
