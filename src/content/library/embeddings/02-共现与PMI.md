先把这章要证伪的命题摆上桌:**得到带真实语义结构的词向量,不需要梯度下降。** 数一数词在哪些词旁边出现,做一次信息论意义上的重加权,再做一次经典线性代数的降维——`king` 和 `queen` 就已经聚到一起,`the` / `a` 被甩开。第 3 章的 word2vec 不是魔法,它是把本章这件事换成了一个可大规模并行的隐式矩阵分解(Levy & Goldberg 2014 证明 skip-gram 负采样本质在分解一个偏移的 PMI 矩阵)。**你先看懂数出来的版本,神经版本就从「神秘」变成「同一件事的另一种算法」。**

本章对应可跑代码 `examples/embeddings-from-scratch/src/stage02-cooccur-pmi.ts`,离线、固定种子(语料 seed=42,SVD 幂迭代 seed=7),同机逐字节可复现。下面引用的数字全来自它的真实输出。一句话提醒边界:语料是约 40 个模板词的 toy(`vocab=43`),**绝对的邻居质量是乐观的**;能迁移到真实语料的是机制,不是具体的相似度数值。下文凡引用绝对数都按这个口径读。

## 分布假说:意义藏在邻居里

> "You shall know a word by the company it keeps." —— J.R. Firth, 1957

这句话不是修辞,是一个可执行的工程假设:**一个词的意义,可以由它经常出现在哪些词旁边来近似。** 近义词倾向有相似的共现邻居——`dog` 和 `cat` 都跟 `run` / `bone` / `fish` 这类词同框,`king` 和 `queen` 都填进 royalty 模板的同一批槽位。反过来,如果两个词的「邻居分布」很像,我们就有理由说它们意义相近。这叫分布假说 (distributional hypothesis,意义由分布近似)。

第 1 章我们诉求稠密向量,但卡在 one-hot 的正交牢笼里——每个词跟其他词的相似度恒为 0,向量里没有任何语义。分布假说给出第一条出路:**别去「学」一个向量,直接「数」出来。** 把每个词表示成「它跟词表里每个词共同出现了多少次」这一行计数,这一行本身就是一个向量。`vocab=43`,所以每个词先得到一个 43 维向量。

怎么算共现?滑一个窗口扫过句子,窗口内的词两两记一次。代码里窗口大小是 2(中心词左右各 2 个):

```ts
const windowSize = 2;
const cooc = cooccurrenceMatrix(corpus.sentences, vocab, windowSize);
```

跑出来的原始共现矩阵长这样(取频率最高的 14 个词做热力图,深色 = 共现多):

```
the         %%--==      ****  ==%%
a           %%%%  @@  **..::==
is      %%%%      ::  ..  ::==
and     --%%    ==::  ::::        ::
ate     ==    ==        ..        ..
man       @@::::            ::
...
  scale " .:-=+*#%@" → [0.000 .. 117.000]
```

第一眼就该看出毛病:`the` / `a` / `is` 这几行**到处都深**。它们不是因为跟谁有语义关系才共现多,而是因为它们本来就无处不在。这正是下一节要打的靶子。

术语对一下:这张 `V×V` 的方阵叫共现矩阵 (co-occurrence matrix);词表里词的编号(id)是按频率降序分配的,所以 `id=0..4` 是 `the, a, is, and, ate`——「id 小 = 高频」这个约定后面量化「污染」时要用到。

## 失败模式:原始计数被高频词主导

把共现矩阵当向量之后,最自然的相似度是两行的点积——`king` 和 `i` 这两行逐位相乘求和,值越大越「相关」。代码里这就是 `rawDotNeighbors`,故意**不归一化**:

```ts
function rawDotNeighbors(cooc: number[][], qid: number, k: number): number[] {
  const scored: Array<{ i: number; s: number }> = [];
  for (let i = 0; i < cooc.length; i++) {
    if (i === qid) continue;
    let s = 0;
    for (let j = 0; j < cooc.length; j++) s += cooc[qid][j] * cooc[i][j];
    scored.push({ i, s });
  }
  scored.sort((a, b) => b.s - a.s || a.i - b.i);
  return scored.slice(0, k).map((x) => x.i);
}
```

拿三个语义毫不相干的词去查最近邻——`king`(王室)、`dog`(动物)、`north`(方向)。如果点积真在测语义,这三个查询的结果应该完全不同。实际跑出来:

```
raw-dot('king')  → is, rules, queen, cat, dog
raw-dot('dog')   → is, rules, cat, king, ate
raw-dot('north') → then, east, the, a, west
```

`is` 和 `rules` 在 `king` 和 `dog` 的结果里**同时排在最前**。`king` 和 `dog` 八竿子打不着,却共享榜首邻居。`north` 那行也混进了 `the` / `a`。这不是语义,是频率污染。

**为什么必然这样?** 点积 `Σ_j cooc[q][j] * cooc[i][j]` 里,一个高频词 `j`(比如 `is`)给几乎每一对 `(q, i)` 都贡献一个「大 × 大」的项——因为它跟谁都共现得多。于是任何查询的点积排名,都被这几个高频列顶上去,跟查询本身的意义无关。**这是设计缺陷,不是数据噪声:用未归一化的关联强度做相似度,数学上就会偏向高频词。**

这里有个常被教科书说错、本章代码刻意避开的陷阱:很多书说「原始计数的最近邻全是停用词」。在这个 toy 语料上,如果你用**余弦**相似度(cosine,夹角余弦,先把每行除以自己的模长)去查,这个现象**不复现**——因为余弦已经把每个词的模长归一化掉了,等于免费替你做了 PMI 一半的活。所以代码不去打那个假靶子。真正可复现的频率污染,活在**未归一化**的点积里——而那恰恰是 PMI 要修的量。

把判断量化一下,别靠肉眼看三条样本。代码用 8 个内容词查询,统计「最近邻里有多少比例的槽位被高频词(频率前 1/4,即 `id < 11`)占据」,占比越低越干净:

```
raw dot  │██████████████████████████████│ 0.675
PPMI cos │██████████░░░░░░░░░░░░░░░░░░░░│ 0.225
      raw=68%  PPMI=23%
```

原始点积有 **68%** 的邻居槽位是高频词——七成的「最相关词」其实只是「最常见词」。这套向量基本不可用。

## PMI:把频次换成信息量

修法的核心一句话:**比较两个词关联强度之前,先把各自单独出现的概率除掉。** 留下的不是「它们共现了几次」(频次),而是「它们共现得比偶然该有的多多少」(信息量)。这就是点互信息 (PMI, pointwise mutual information):

$$\text{PMI}(i,j) = \log \frac{P(i,j)}{P(i)\,P(j)}$$

直觉:分母 `P(i)P(j)` 是「假如 i、j 互相独立,本该共现的概率」。`the` 跟谁的实际共现都约等于这个基线(因为 `the` 到处都是),所以 `PMI(the, *) ≈ 0`,被自动压平。`king` 跟 `queen` 因为共享模板槽位,实际共现远超基线,`PMI` 很大。**频率本身被分母约掉了,这正是上一节缺的那步。**

代码里的实现,关键是用「计数 × 总数 / (行和 × 列和)」这个等价形式算,避免小概率连乘下溢:

```ts
const pmi = Math.log((cij * total) / (rowSum[i] * colSum[j]));
ppmi[i][j] = pmi > 0 ? pmi : 0; // the PPMI clamp
```

最后那行 `pmi > 0 ? pmi : 0` 是把负 PMI 截到 0,得到 PPMI(positive PMI,正点互信息)。**为什么截?** 两个失败模式:

1. **`log(0)`**:绝大多数词对从没共现过,`P(i,j)=0` → `PMI = log(0) = -∞`。代码对 `cij === 0` 直接 `continue` 跳过(留 0),不让 `-∞` / `NaN` 污染整行。
2. **负 PMI 不可靠**:稀疏计数下,「这两个词共现得比偶然还少」这种负信号几乎全是采样噪声,不是真信号。Bullinaria & Levy(2007)的实证结论是 PPMI 这个变体产出最可用的向量。所以截掉负的——这一行 clamp 是「为什么几何结构干净」的最关键一笔。

⚡ 顺带钉一个**仍无通用解的开放问题**:PMI 对低频词对系统性偏高——一个只出现一两次的稀有词,只要碰巧跟某词共现一次,`P(i)P(j)` 极小,算出的 PMI 就虚高。业界有一堆补丁:context distribution smoothing(把 `P(j)` 抬到 0.75 次幂,word2vec 负采样里同款)、shifted PPMI(`max(PMI - log k, 0)`)、加 Laplace 平滑……但**没有一个在所有语料/任务上都最优**,选哪个仍是经验调参。低频词的统计估计本质上是数据稀疏问题,counting 这条路绕不过去——这也是后面神经方法用海量数据「软化」这个问题的动机之一。本章 toy 语料词频集中,这个偏差被掩盖了,真实语料上它是头号坑。

换上 PPMI 行、用余弦,同样三个查询:

```
ppmi('king')  → queen(1.00)  kingdom(0.89)  prince(0.61)  princess(0.61)  dog(0.43)
ppmi('dog')   → bone(0.85)   queen(0.43)    king(0.43)    fish(0.36)      prince(0.34)
ppmi('north') → south(0.99)  west(0.98)     east(0.97)    then(0.30)      go(0.23)
```

对比上一节同样的三行:`king` 的邻居从 `is, rules` 变成了 `queen, kingdom, prince, princess`——全是王室词;`north` 的邻居是 `south, west, east`——全是方向词。**同一份数据,只是把频率基线除掉,几何结构发生了质变。** 量化指标上,高频词占据的邻居槽位从 **68% 降到 23%**——PPMI 把频率污染砍掉了三分之二。

这就是本章那句 Firth 名言的兑现:纯计数 + 一次重加权,就拿到了能用的向量。注意此刻**还没有任何训练**——没有 optimizer,没有 autograd,没有一次梯度更新。

## 截断 SVD:压成稠密向量

PPMI 行能用了,但它还是个 43 维的**稀疏**向量(大量 0)。第 1 章诉求的是稠密低维向量,真实语料里 `V` 是几万几十万,直接拿 `V` 维稀疏行又大又含噪。降维的经典工具是奇异值分解 (SVD, singular value decomposition,把矩阵拆成几个主方向的叠加),只取前 `d` 个最强方向就叫截断 SVD。

本章的窗口对称,共现矩阵和 PPMI 矩阵都是对称阵,SVD 退化成对称矩阵的特征分解,代码用幂迭代 + deflation(找一个特征向量就从矩阵里减掉它,再找下一个)手写了一个,约 50 行全可审计。词向量取「`sqrt(特征值) × 特征向量`」——按奇异值加权,强方向权重更大,经验上比裸特征向量的相似度几何更好:

```ts
const scale = Math.sqrt(Math.max(eigvals[k], 0));
for (let i = 0; i < V; i++) {
  emb[i][k] = eigvecs[k][i] * scale;
}
```

`Math.max(eigvals[k], 0)` 是又一处防御:toy 的 deflation 会累积浮点误差,产出小的负特征值,`sqrt` 负数会得 `NaN` 毒化所有余弦。截到 0——一个近零特征值本就没携带真信号。**这是 toy 求解器:幂迭代对相近特征值收敛慢,deflation 误差在头几个分量之后累积,`d≤8`、40×40 矩阵上够用,真实矩阵上必须换 LAPACK,别拿它当生产用。** 这里学的是机制,不是数值。

降到 8 维稠密向量后,语义簇还在不在:

```
svd nearest('king'):  queen(1.00)  rules(0.95)  the(0.95)
svd nearest('dog'):   bone(0.96)   cat(0.93)    run(0.85)
svd nearest('bread'): she(0.99)    they(0.99)   rice(0.98)
svd nearest('north'): south(1.00)  west(1.00)   not(1.00)
```

`king→queen`、`dog→bone/cat/run`、`north→south/west` 都活下来了——royalty / animal / direction 三个簇被压进 8 维仍然成立。这就是基于计数的词向量,**全程零梯度**。(toy 语料下 `king` 的邻居还混进 `the`,`bread` 跑偏到 `she/they`,绝对质量乐观,前面已声明口径。)

关键对照:把同一套 SVD 用在**原始计数**矩阵(不做 PPMI),压完仍然脏:

```
svd-raw nearest('king'): queen(1.00)  the(0.94)  um(0.93)
```

`king` 的邻居又冒出 `the` / `um` 这种高频词。**对垃圾做低秩压缩,得到的是低秩的垃圾**——SVD 不会替你修频率污染,PPMI 那步省不掉。这反过来说明 PPMI 是这条管线里不可替代的一环,不是可选的美化。

最后一个最小 sanity,把「语义 > 频率」钉成一个数:

```
cos(king, queen) = 0.995
cos(king, "the") = 0.945
king is CLOSER to queen than to 'the' ✓ (semantic > frequency)
```

`king` 离 `queen`(0.995)比离 `the`(0.945)更近。这个 gap 在 toy 上偏小、绝对值偏乐观,但方向对了:语义压过了频率。

## 接住第 3 章:同一个矩阵的两种解法

回到开头那句证伪命题——它成立了。整条管线:**数共现 → PPMI 重加权 → 截断 SVD**,产出的向量里 `king/queen/kingdom` 聚成王室簇、`north/south/west` 聚成方向簇,而这一切「No optimizer, no autograd」(代码末行原话)。

把本章三步排一下账,顺手记住每步是在修哪个失败模式:

| 步骤 | 输入问题 | 这步干了什么 | 没这步会怎样 |
|------|----------|--------------|--------------|
| 共现计数 | one-hot 无语义 | 用「邻居分布」表示词 | 向量间相似度恒为 0 |
| PPMI | 点积被高频词主导(68%) | 除掉频率基线,留信息量 | 邻居全是 `is/the/a` 噪声 |
| 截断 SVD | 稀疏高维 | 压成 8 维稠密向量 | 维度大、含噪、难下游用 |

为什么要在跳进神经网络之前先花一章数数?因为第 3 章的 word2vec(skip-gram)会让你觉得它在「学习语义」,很神秘。但 Levy & Goldberg(2014)证明:**skip-gram 负采样优化的目标,等价于隐式分解一个偏移的 PMI 矩阵**——本章你显式构造、显式分解的那个矩阵,word2vec 用梯度下降隐式地、可扩展地做了一遍。两者在数学上是同一个对象的两种解法:本章是「显式构造 + 经典 SVD」,第 3 章是「隐式逼近 + 随机梯度」。

那 word2vec 凭什么赢?不是因为它语义更对,而是因为**它不用把 `V×V` 的矩阵在内存里建出来再分解**——真实语料 `V` 几十万,那个矩阵建不起来。神经版本逐样本在线更新,天然可扩展到大语料,还顺手把上一节 PMI 低频偏差的补丁(那个 0.75 次幂的平滑)做进了负采样分布里。你带着「它在分解 PMI 矩阵」这把尺子去看第 3 章,就不会被「神经网络学到了词义」这种说法唬住——它学到的,是本章你已经数出来的东西。
