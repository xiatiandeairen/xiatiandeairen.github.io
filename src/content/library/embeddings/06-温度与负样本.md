第 05 章结束时,你有一个能收敛的 InfoNCE(把"拉近正样本、推开负样本"压成一个 softmax 损失)训练回路。它跑通了——但"跑通"和"学出能用的表示"之间隔着两个超参,而这两个超参恰好是新手最容易当成无关紧要的旋钮随手设掉的:温度 τ 和负样本。

这一章我要论证的具体命题是:**这两个旋钮不是性能微调,它们直接决定你学出的几何长什么样。** τ 决定损失把梯度压在哪一个负样本上、表示有多"尖";负样本数量决定空间铺得多开(且收益是 log 式递减,不是线性);难负样本决定分离速度——但越过某个点会把语义近邻当负样本推开,反伤结构。我会用 `examples/embeddings-from-scratch/src/stage06-temperature-negatives.ts` 的真实 sweep 把每条都钉死,包括那些 toy 数据上**没**显现、必须靠真实数据才看得到的曲线形状——后者我会明确标注,不强行叙事成"已确认"。

先说清楚 toy 的边界:语料 `vocab=43`、`positive pairs=5768`、`dim=16`、`epochs=8`、每 epoch 采 120 个 anchor,整套 sweep 实测 `wall-clock=5146 ms`。绝对数字(alignment 能到 0.24、clusterAcc 能到 0.56)是被 toy 的密集人造结构灌出来的,别拿去对标真实任务。**能迁移的是每条曲线的形状**,不是数值本身。

## τ 到底在 softmax 里干什么

InfoNCE 对单个正样本对的损失,在 stage06 里是这样的(`infoNceLoss`):

```ts
function infoNceLoss(anchor: Vec, positive: Vec, negatives: Vec[], tau: number): Value {
  const invTau = 1 / tau;
  const posLogit = cosSim(anchor, positive).mul(invTau);
  const negLogits = negatives.map((n) => cosSim(anchor, n).mul(invTau));
  const allLogits = [posLogit, ...negLogits];

  // Stable softmax: subtract max (detached constant)
  let maxData = -Infinity;
  for (const l of allLogits) if (l.data > maxData) maxData = l.data;
  const expTerms = allLogits.map((l) => l.sub(maxData).exp());
  let denom = new Value(0);
  for (const e of expTerms) denom = denom.add(e);
  const pPos = expTerms[0].div(denom);
  return pPos.log().mul(-1);
}
```

τ 进来的方式只有一处:每个余弦相似度(范围约 [-1,1])在进 softmax 之前先除以 τ。就一个除法。但这个除法控制的是 logits 的**动态范围**。余弦差 0.2,τ=1 时 logit 差 0.2,softmax 几乎拉不开;τ=0.05 时 logit 差变成 4,`e^4 ≈ 55` 倍的概率差——softmax 被逼向 argmax。

为什么用余弦而不是裸点积?这是个有意的设计,代码里 `cosSim` 先把两个向量 L2 归一化再点积。原因:τ 只有在它除的是**有界**相似度时才有稳定、可解释的含义。裸点积上,τ 的效果会和向量模长纠缠,而 Adam 会让模长自由膨胀——你以为在调 τ,实际在和模长打架。SimCLR / CLIP 都在 projection head 输出后做 L2 归一化,就是这个理由。归一化放在 autograd 图里(`normalize` 函数,`sqrt` 下加了 `1e-8` 的 eps 防止早期近零向量把梯度炸成 NaN——归一化对比损失的经典 NaN 来源)。

### ✦ τ 越小,梯度越集中在最难的负样本上

这是本章最反直觉、也最值得记住的一点:**τ 不是均匀缩放,它决定损失对"最难区分的负样本"的关注度。**

机制在 softmax 的形状里。损失 `-log p+`,其中 `p+` 是正样本在所有 logits 上的 softmax 概率。反向传播时,每个负样本拿到的梯度正比于它的 softmax 概率 `p_k`。τ 越小,logits 被放得越大,softmax 越尖——概率几乎全堆在相似度最高的那一个负样本(也就是"最难区分"的 hard negative)上。于是梯度也几乎全压在它身上。τ 越大,logits 被压扁,softmax 趋于均匀,梯度平摊到所有负样本。

换句话说:**小 τ 让损失只盯着当前最像正样本的那个负样本死磕;大 τ 让损失对所有负样本一视同仁。** 这直接决定了学出的表示有多"尖"——小 τ 把少数几个方向使劲撑开,大 τ 把压力摊平、空间更圆。

看实验①(负样本固定=16,无难负样本)扫 τ 的实测:

```
  τ        finalLoss   alignment↓  uniformity↓  clusterAcc↑   margin↑
   0.05    2.3056     0.8581    -2.9228     0.3125    0.3410
   0.10    1.8645     0.9078    -2.9558     0.2500    0.4569
   0.20    1.7640     0.9074    -2.9841     0.2500    0.5618
   0.50    2.0066     0.4518    -2.5332     0.4375    0.8400
   1.00    2.3048     0.2455    -2.2166     0.5625    0.9335
```

读法先说清这三个指标(它们来自 `core/eval.ts`,alignment/uniformity 是 Wang & Isola 2020 的诊断对):

- **alignment↓**:人为植入的正样本对在空间里有多近(L2 距离均方),越小越好。
- **uniformity↓**:整个空间铺得多开 vs 塌缩,越负越好(越接近 0 = 越塌)。
- **margin↑**:同簇平均余弦 − 异簇平均余弦,本章驱动曲线用的连续分离度指标。代码里特意用 margin 而不是 clusterAcc,因为 16 个词的 clusterAcc 只能 1/16 跳着走,太粗,看不出内部峰;margin 是连续的,旋钮的效果才读得出来。

注意一个有意思的撕裂:小 τ(0.05)的 uniformity 最好(-2.92,空间铺得最开),但 alignment 最差(0.86,正样本对反而被推远了)。这正是"梯度全压 hard negative"的副作用——小 τ 把空间使劲撑开撑出了好的 uniformity,但代价是连正样本对都被卷进这场撑开里推散了。大 τ(1.0)反过来:alignment 极好(0.24),margin 也最高(0.93)。

### 两端各自怎么坏:失败模式实测

stage06 单独跑了两个极端 τ 把病灶量化出来(`失败模式` 块):

```
  τ=0.02 (过小): loss曲线波动 std=2.1578  vs τ=1 的 std=0.1018
         → 过小 τ 把 softmax 逼成 argmax, 梯度几乎全压在单个最近负样本上, 更新抖
  τ=10 (过大): clusterAcc=0.5000  uniformity=-1.9645 (越接近0=越塌)
         softmax(正样本)=0.1207 ≈ 均匀基线 1/(K+1)=0.0588 → 几乎不区分正负
```

**过小 τ 的病:训练抖。** 用每 epoch 损失曲线的标准差当更新方差代理,τ=0.02 的 `std=2.1578`,τ=1 的 `std=0.1018`——抖了 21 倍。为什么?softmax 被逼成 argmax,backward 几乎全部梯度只走那个相似度最高的负样本。每一步都在跟当前最难的那个对手死磕,而"当前最难"每步都在换,于是更新方向反复横跳。这也是 stage06 必须用数值稳定 softmax(减最大 logit 再 exp)的原因:τ=0.05 时 `s/τ` 接近 20,`e^20` 还好,但多项相加再加更宽的相似度分布,不减 max 的话求和会溢出——这是小 τ 产生 Inf/NaN 的头号原因。减 max 在数学上是恒等变换(在比值里约掉),但让每个 exp 都待在安全范围。

**过大 τ 的病:退化成近均匀 softmax,几乎不区分正负。** τ=10 时正样本的 softmax 概率算出来是 `0.1207`,而完全不区分的均匀基线是 `1/(K+1)=0.0588`。已经很接近"瞎猜"了。clusterAcc 掉到 0.50,uniformity 只有 -1.96(越接近 0 越塌)。logits 被压得太平,正负样本在损失眼里几乎一样,梯度推不动,空间塌缩。

### ⚡ 这个 toy 没显现的:τ-margin 的内部峰(无通用解)

诚实地说:实验① 的 margin 单调升到 τ=1(端点),`最优 τ=1`,**没有出现内部峰**。stage06 的输出自己标了这点:`本次峰值落在端点 τ=1 (toy 数据 + 少 epoch 下未现内部峰)`。

但理论和真实数据上,τ-margin 应该是**先升后降**——既不是越小越好,也不是越大越好,而是中间某个值最优。这里 toy 没显现,因为人造簇过密 + epoch 太少,τ 还没大到让空间塌缩,曲线就到端点了。失败模式块的 τ=10 实际已经摸到了"过大塌缩"的下坡,只是没在 [0.05, 1.0] 这个 sweep 区间里。

**而"最优 τ 是多少"目前没有通用解,正在研究。** SimCLR 经验值 0.1、CLIP 把 τ 设成可学习参数(初始约 0.07,带上限裁剪防塌)。为什么不能一劳永逸?因为最优 τ 取决于 batch 里 hard negative 的分布、embedding 维度、数据本身的语义密度——这些每个任务都不同。"温度调度"(训练中动态改 τ)和 CLIP 式可学习 τ 都是当前活跃方向,没有一个公式能从数据先验直接算出最优 τ。把它当成必须扫的超参,不要拍脑袋设 0.07 就完事。

## 负样本数量:为什么它等价于 batch size,且收益递减

第二个旋钮:每个正样本对配多少个负样本。

先讲清"等价于 batch size"这件事。stage06 这里是显式采样负样本(`sampleNegatives`,从 unigram^0.75 频率表里抽,拒绝采到 anchor 和真正的正样本——把正样本采成负样本会同时拉近又推开同一对,梯度自相矛盾)。但生产级对比学习(SimCLR / CLIP)用的是 **in-batch negatives**:一个 batch 里,每个样本的正样本是它自己的增强视图,**其他所有样本都当它的负样本**。于是 batch size = N,每个样本就有 N−1 个负样本。**"负样本数量"和"batch size"在 in-batch 方案里是同一个旋钮。** 这也是为什么 SimCLR 论文要堆到 batch 4096、CLIP 堆到 32768——它们不是在堆 batch,是在堆负样本数。

为什么更多负样本更好?InfoNCE 是互信息(两个变量共享多少信息)的一个下界,而**负样本数 K 越大,这个下界越紧**。直觉:对比任务是"从 K+1 个候选里认出正样本",K 越大题越难,逼模型学出更可区分的表示,uniformity 更好。

但收益是 log 式的——这是关键。看实验②(τ 固定=最优=1):

```
  #neg     finalLoss   alignment↓  uniformity↓  clusterAcc↑   margin↑
     4     1.2254     0.4867    -2.2129     0.3125    0.8701
    16     2.3048     0.2455    -2.2166     0.5625    0.9335
    64     3.6328     0.2661    -2.1215     0.3750    0.8968

  margin 增量 (每档 ×4 负样本): 4→16 = 0.0635,  16→64 = -0.0367
  后段/前段增益比 = -0.58 (<1 → 边际递减)
```

每档负样本数翻 4 倍。如果收益是线性的,两次翻倍应该带来差不多的 margin 增量。实际:4→16 涨了 0.0635,16→64 反而跌了 0.0367,**后段/前段增益比 -0.58 < 1**,边际递减得非常清楚。这和 InfoNCE 的 log 式下界一致:互信息下界随 K 的增长是 `log K` 量级,不是线性,翻倍 K 只带来固定增量,且 toy 数据上后段直接被噪声吃成负值。

这有直接的工程含义:**别无脑堆负样本/batch。** 从 256 堆到 4096 可能值,从 4096 堆到 65536 带来的 margin 增量会小到不值那笔显存和算力——而 batch size 在对比学习里是显存大头。真实任务上你该扫的是"边际收益跌到不值成本"的拐点,不是越大越好。(题外话:toy 上 16→64 的 alignment/clusterAcc 都退化了,这是 toy 簇过密 + epoch 少的噪声,真实数据上递减更平滑、不至于倒退;但"递减"这个形状本身是稳的。)

## ⚡ 难负样本:双刃剑,分离度先升后降

第三个旋钮最微妙,也是本章的第二个开放问题。

难负样本(hard negative,当前在表示空间里离 anchor 很近、但不是它正样本的词——模型当下的"最容易认错的对手")。stage06 的 `sampleHardNegatives` 这样近似:用**脱离计算图的当前权重**(eval 时读一次,不扩展 autograd 图)给所有候选按到 anchor 的余弦排序,从最近的约 25% 这个"硬带"里采。`hardRatio` 控制多大比例的负样本来自硬带,其余仍按频率均匀采。

为什么适量难负样本能加速分离?均匀采来的负样本大多是"显然不像"的词,损失对它们的梯度接近 0(softmax 概率几乎为 0)——白采。难负样本是模型当前正在犯错的近邻,梯度信息量大得多,直接推动空间把它们和正样本掰开。

但代码注释里就埋了双刃剑的机制(`sampleHardNegatives`):硬带"deliberately overlaps the anchor's true semantic cluster"——硬带刻意和 anchor 的真·语义簇重叠。`hardRatio` 一旦太高,硬带里开始混进**假负样本**(false negative,本来就该离 anchor 近、属于同一语义簇的词)。把它们当负样本推开,等于亲手拆掉刚学出的簇结构。

**于是 margin 对 hardRatio 应该是先升后降的非单调曲线**——少量难负样本帮忙,过量反伤几何。看实验③(τ + #neg 固定):

```
  hardRatio  finalLoss   alignment↓  uniformity↓  clusterAcc↑   margin↑
   0.00       2.3048     0.2455    -2.2166     0.5625    0.9335
   0.25       2.4561     0.4447    -2.5574     0.3125    0.7341
   0.50       2.5790     0.7366    -2.8355     0.3750    0.5948
   0.75       2.5989     0.6834    -2.9086     0.3125    0.5669
   1.00       2.6307     0.6674    -2.8732     0.1875    0.5293
```

这里有个清晰可读的 trade-off,即使没显现内部峰:随 hardRatio 升,**uniformity 一路变好**(0→1.0:-2.22 → -2.87,空间被难负样本撑得更开),但**alignment 一路变差**(0.25 → 0.67,正样本对被推散)。这正是"难负样本把空间撑开,代价是把本该靠近的正对也推开"的实测——`spread↑ 但正对被推开` 的取舍肉眼可见。margin 作为净指标,从 0.93 单调跌到 0.53。

### 这个 toy 没显现内部峰——为什么,以及真实数据上怎样

实验③ 的 margin 峰值落在 `hardRatio=0`(端点),没有先升后降。stage06 自己标注了:`本次落在端点 — toy 簇过密, 难负样本几乎都是假负样本; 真实数据上 25%~50% 常呈内部峰`。

原因要害:toy 的簇太密了,任意两个"近邻"基本都是同簇,硬带里几乎**全是假负样本**——所以哪怕一点点 hardRatio 都在拆结构,曲线从 0 就开始往下掉,峰退化到端点。真实语料里语义空间稀疏得多,离 anchor 近但不同簇的"真·难负样本"大量存在,25%~50% 的 hardRatio 通常能在拆掉真邻居之前先吃到分离加速,于是内部峰出现。

**而"难度调到多硬、比例调到多少"目前没有通用解,正在研究。** 难点的根:你没有 ground-truth 标签来区分"真·难负样本"和"假负样本"——硬带是按当前表示的余弦近邻定义的,而当前表示本身可能就错了。这是个鸡生蛋问题。debiased contrastive learning(估计并扣掉假负样本的影响)、hard negative mixing(在表示空间合成难负样本而非采真样本)都是这个方向的活跃尝试,但没有一个能在任意任务上自动定出最优难度和比例。实践建议:把 hardRatio 当超参从一个保守值(如 0.1~0.3)开始扫,**同时盯 alignment 和 margin 两个指标**——只看 uniformity 会被"空间撑开了"骗过去,而 alignment 正在偷偷崩。

## 三个旋钮怎么一起拧

把 stage06 收尾的诚实总结翻译成工程动作:

1. **τ 先扫。** 它影响最大、最反直觉。记住小 τ→尖、聚焦 hard negative、易抖(实测 std 2.16 vs 0.10);大 τ→圆、近均匀、易塌(实测正样本 softmax 0.12 ≈ 均匀基线 0.06)。SimCLR 0.1 / CLIP 可学习 τ 是参照不是答案。τ 锁定再动别的。

2. **负样本数/batch 扫到边际拐点为止。** 收益 log 式递减(实测后/前增益比 -0.58),不是越大越好。in-batch 方案里它就是 batch size,堆它前先算这笔显存值不值那点 margin。

3. **难负样本最后引入,小比例起步,双指标监控。** 适量加速分离,过量把真邻居当负样本推开(实测 hardRatio↑ 时 uniformity↑ 但 alignment 从 0.25 崩到 0.67)。没有 ground truth 区分真假难负样本,这是开放问题。

一个收尾的诚实声明,也是这本书想立的标杆:stage06 在 toy 上**确实显现了**的是——① τ 两端的真实病灶(抖 + 塌)都被量化;② 负样本边际递减;③ 难负样本的 spread/alignment 取舍。**没显现**的是 τ-margin 的内部峰和难负样本的非单调峰,因为 toy 簇过密 + epoch 少而落在端点。我把这两条标成"需真实数据",而不是硬凑数字假装确认——这正是 AI 批量生成的 survey 稿最爱犯的错:把"理论上应该"写成"实测证明"。曲线的**形状**和**机制**是真的,某些峰在这个 toy 上**就是没跑出来**,两者都得说。
