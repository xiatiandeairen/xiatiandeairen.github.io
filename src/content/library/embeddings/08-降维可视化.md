第 07 章结束时,你手里是一堆数字:某个词的 top-k 近邻、analogy 命中率、cosine 相似度。数字诚实,但它有个盲区——它一次只回答一个局部问题(`king` 的邻居是谁),你看不到整片空间长什么样:有几个簇?簇之间什么关系?有没有一个词诡异地落在了不该落的地方?16 维(本章嵌入维度)的空间人脑想象不出来,于是工程师的标准动作是:把它压到 2 维,画成散点图,用视觉皮层一眼扫过去。

这一步看着像"画个图而已",其实是这本书最容易出错、也最容易自欺的一章。**任何把 16 维压到 2 维的投影都是 lossy(有损)的——它必然丢掉结构,问题只是丢哪部分。** 你选的投影方法,等于替你决定了"哪部分结构活下来给你看,哪部分被碾平"。PCA 和 t-SNE 这两个最常用的方法,碾平的是完全相反的东西。用错了,你会对着一张图得出一个高维空间里根本不成立的结论,而且毫无察觉——图太有说服力了。

配套代码在 `examples/embeddings-from-scratch/src/stage08-pca-tsne.ts`,跑 `npx tsx src/stage08-pca-tsne.ts`。它不 import 别的 stage(import 一个 stageNN 文件会执行它的 `main()`),而是自己用第 03 章那套 skip-gram 重新训一份真嵌入——因为**你没法可视化一个你没学出来的空间**。下面所有解释方差比、KL 损失、近邻保留率都是从这次真实运行里数出来的,不是估的。

## 先确认:我们画的是结构,不是噪声

可视化最阴险的失败模式,是你给一堆**随机噪声**降维,t-SNE 照样能画出几个看着像模像样的簇,然后你对着噪声编故事。所以画图前第一件事是确认底层向量真的学到了东西。代码先训 150 epoch 的全 softmax skip-gram:

```ts
// stage08-pca-tsne.ts — 全 softmax skip-gram 损失(真实归一化似然,不用 negative sampling 近似)
function negLogSoftmax(center: Vec, outEmb: Mat, contextId: number): Value {
  const scores = outEmb.map((o) => dot(center, o)); // V 个分数
  // softmax 分母 = sum_j exp(score_j); -log p(context) = log Z - score_ctx.
  const expSum = sumValues(scores.map((s) => s.exp()));
  return expSum.log().sub(scores[contextId]);
}
```

这次运行的损失从 **3.860 降到 1.897(下降 1.962)**。损失真在掉,几何就是真学出来的,不是随机初始化的残留。这个前提钉死之后,后面两张图才有资格被认真解读。toy 规模(43 词词表、16 维)下簇的清晰度比真实语料偏乐观,这点要诚实标注;能迁移到真实场景的不是"簇有多干净"这个绝对值,而是**相对的故事**:PCA 和 t-SNE 谁保住了什么、perplexity 怎么重塑地图。

## PCA:朝方差最大的方向看过去

PCA(Principal Component Analysis,主成分分析)的思路朴素到近乎暴力:在 16 维空间里,数据点云朝哪个方向铺得最开(方差最大),就把那个方向当第一根坐标轴(PC1);在垂直于它的方向里再找铺得最开的当第二根(PC2);然后把所有点投到这两根轴张成的平面上。它是**线性**投影——新坐标是原坐标的线性组合,没有任何弯曲。

代码里它是纯闭式线性代数,不需要梯度:中心化 → 算协方差矩阵 → 用幂迭代(power iteration,反复拿向量乘矩阵,会收敛到最大特征向量)取前两个特征向量 → 投影。我特意不给 PCA 套 autograd,因为那会掩盖一个关键事实:**PCA 有唯一的全局解**。同样的数据,你跑一百遍 PCA 得到同一张图(顶多整体翻转,特征向量符号不定)。这跟下面的 t-SNE 形成最重要的对比。

```ts
// stage08-pca-tsne.ts — 幂迭代取最大特征向量
function topEigenvector(C: number[][], rng: Rng): [number[], number] {
  const d = C.length;
  let v = normalize(Array.from({ length: d }, () => gaussian(rng, 0, 1)));
  for (let iter = 0; iter < 200; iter++) {
    const Cv = matVec(C, v);
    v = normalize(Cv); // C^k v 会沿最大|特征值|方向放大最快 → 收敛到主特征向量
  }
  const lambda = dotNum(v, matVec(C, v)); // Rayleigh 商给出对应特征值
  return [v, lambda];
}
```

这次运行的关键数字:**PC1 解释方差 22.7%,PC2 解释方差 16.8%,两轴合计 39.5%**。这个数字是 PCA 最大的诚实之处——它直接告诉你**这张 2D 图代表了原始空间多少结构**。剩下 60.5% 的方差分布在你看不见的 14 根轴上,在这张图里被彻底压平了。如果两个词在图上看着挨得很近,它们完全可能在那 14 个隐藏维度上离得很远。PCA 不掩饰这一点,它把"我只给你看了 39.5%"明明白白印出来。

PCA 的近邻保留率(高维里每个词的 3 个最近邻,有多少还是 2D 图里的最近邻)是 **31.8%**。不高。原因正是它的设计取向:PCA 优化的是**全局方差**,它在乎"整片点云的大致摊开形状对不对",不在乎"`king` 旁边那几个词有没有被拆散"。要看全局骨架,PCA 很合适;要看局部簇,它就力不从心。

### 怎么坏:不中心化,第一根主轴会被浪费掉

PCA 有一条不能省的前置步骤:**先减去均值(中心化)**。代码注释里我把它标成 CRITICAL INVARIANT。为什么?"方差最大的方向"这个定义,只有在点云以原点为中心时才指向"结构";如果点云整体偏离原点(均值向量非零),那"离原点最远的方向"会被**点云的偏移**主导,而不是被点之间的差异主导——你会把第一根、也是信息量最大的坐标轴,浪费在描述"所有点共有的那个常量偏移"上。

代码里 stage 专门跑了一次不中心化的 PCA 当反面教材。这次运行:嵌入均值向量的模长 = **1.164**(非零,说明点云确实偏离原点)。然后测 PC1 跟"均值方向"的夹角余弦绝对值:

- **不中心化:|cos| = 0.881** —— PC1 几乎直接指向均值方向,第一根主轴基本在描述"点云在哪",不是"点怎么散开"。
- **正确中心化:|cos| = 0.056** —— PC1 跟均值方向几乎正交,它在描述真正的结构差异。

这个 0.881 vs 0.056 的对比,是"忘了中心化"这个 bug 永远可见的症状。注意一个诚实的细节:这次 toy 数据上,两种做法的近邻保留率只差一点点(中心化 31.8% vs 不中心化 31.0%)——因为这份数据均值模长不算大。但**偏移越大、维度越高,浪费一整根主轴的代价越严重**。别用 toy 数据上指标差距小,去给"中心化可以省"背书,这是典型的拿小样本噪声当结论。

## t-SNE:不要全局,只把每个点的邻居关系搬到 2D

t-SNE(t-distributed Stochastic Neighbor Embedding,t 分布随机近邻嵌入)的取向跟 PCA 正好相反。它不在乎全局方差,它只问一件事:**每个点在高维里的邻居,在 2D 里还是不是它的邻居?** 它愿意为了把局部簇画紧,任意扭曲全局——这是非线性投影。

机制分三步,值得讲透,因为后面的误读陷阱全长在这套机制上:

**第一步,高维近邻概率 P。** 对每个点 i,定义"j 是 i 的邻居"的概率正比于 `exp(-||xi-xj||² / 2σ_i²)`——距离越近概率越高,是个以 i 为中心的高斯。这里有个关键的 per-point 带宽 σ_i:稠密区的点要用小 σ(邻居多,得收紧),稀疏区的点要用大 σ。σ 由 **perplexity(困惑度,可粗暴理解为"每个点算几个有效邻居")** 这个超参反推出来——代码用二分搜索调 σ,让每个点的邻居分布熵恰好等于 `log(perplexity)`:

```ts
// stage08-pca-tsne.ts — 二分搜索 σ,让每行的邻居分布熵命中目标 perplexity
function conditionalRowForPerplexity(d2: number[], selfIdx: number, targetEntropy: number): number[] {
  let betaLo = 1e-8, betaHi = 1e8, beta = 1.0; // beta = 1/(2σ²),精度
  for (let iter = 0; iter < 50; iter++) {
    // ...用当前 beta 算邻居分布,算它的香农熵...
    const diff = entropy - targetEntropy;
    if (Math.abs(diff) < 1e-5) break;
    // beta 越大(σ 越小)→ 熵越低(有效邻居越少)
    if (diff > 0) { betaLo = beta; beta = betaHi === 1e8 ? beta * 2 : (beta + betaHi) / 2; }
    else { betaHi = beta; beta = (beta + betaLo) / 2; }
  }
  return row;
}
```

记住这一步:**perplexity 决定了每个点"看多远"的邻居。** 它不是一个无关紧要的训练细节,它是整张图的形状旋钮。

**第二步,低维近邻概率 Q。** 在要学的 2D 坐标 Y 上,定义 `q_ij` 正比于 `(1 + ||yi-yj||²)^-1`——这是 Student-t 分布(自由度 1,即柯西分布),不是高斯。这个**重尾**是 t-SNE 名字里那个"t"的来历,也是它解决"crowding problem(拥挤问题)"的关键:重尾让远的点可以在 2D 里待得很远而不产生爆炸的梯度,否则高维里中等距离的一大堆点会在 2D 里全挤成一团。

**第三步,最小化 KL(P‖Q)。** 让 2D 的邻居分布 Q 去逼近高维的邻居分布 P,用的是 KL 散度(衡量两个分布差多少)。而且——这是本书的一条主线——这个最小化**就是第 03 章那套 backprop**,不是什么 t-SNE 专用的封闭公式。代码把 KL 损失搭成 Value 计算图,直接 `loss.backward()`:

> ✦ **这就是本章最该带走的判断:PCA 看全局结构,t-SNE 看局部簇,两者回答的是不同问题。** PCA 告诉你"整片空间的主轴朝哪、这张图代表了多少结构(39.5%)";t-SNE 告诉你"谁跟谁是一伙的(近邻保留 45.0%)"。拿 t-SNE 去问全局问题(两个簇的相对位置)、或拿 PCA 去抠局部簇结构,都是用错工具,会得出错误结论。它们不是"哪个更好",是分工。

## ⚡ t-SNE 的图极易误读:漂亮可能纯属调参

现在到了这一章——也是这本书——最需要你警惕的地方。t-SNE 的图太好看了,簇分得清清楚楚,你会忍不住读出三种结论:① 这两个簇离得远,所以语义无关;② 这个簇比那个簇大,所以它更"分散";③ 这张图里它们分得这么开,数据本身就是这么分的。**这三种读法基本都是错的。** 簇间距、簇大小、乃至"分得开不开",在 t-SNE 里几乎都不是数据的客观属性,而是 perplexity 这个旋钮的产物。

stage 用同一份向量、只改 perplexity 扫了 5 / 15 / 40 三个值,画出三张差异极大的图。最直接的证据是坐标轴尺度——同一份数据:

- **perplexity=5:** x∈[-19.068, 16.138],近邻保留率 51.2%
- **perplexity=15:** x∈[-6.335, 6.604],近邻保留率 45.0%
- **perplexity=40:** x∈[-0.782, 0.596],近邻保留率 36.4%

坐标范围从 ±19 一路缩到 ±0.6,缩了三十多倍。簇与簇拉开的"距离感"完全是 perplexity 给的。小 perplexity 每个点只看很近的几个邻居,于是图碎成很多小块、摊得很开;大 perplexity 每个点看一大片邻居,于是所有东西被拉拢到一起、挤成一小坨。**"看起来分得很开"完全可能只是你选了 perplexity=5。** 换个值,同一份数据就"挤成一团"了。

注意近邻保留率也跟着变(51.2% → 45.0% → 36.4%),这次运行里小 perplexity 反而保留得更好。但这不是"perplexity 越小越好"的通用规律——它跟数据的簇粒度有关,真实数据上常常是中间某个值最好。要点是:**没有"正确的" perplexity,只有"为了回答某个问题而选的" perplexity,而且你必须扫几个值看图稳不稳,不能拿一张图当真相。** 这是 t-SNE 用户最常跳过、也最致命的一步。

### 怎么坏:照着 t-SNE 簇间距下语义结论

把上面的警告坐实成数字,stage 跑了第二个反面教材。取 royalty 簇(king/queen/prince/princess)和 animals 簇(dog/cat/mouse/bird),先量它们在**原始 16 维**里的关系——两簇质心的 cosine = **0.442**,这是个固定不变的客观事实。然后量同一对簇在三张不同 perplexity 的 2D 图里,质心之间的欧氏距离:

```
原始 16 维: royalty 簇质心 与 animals 簇质心 cosine = 0.442 (固定不变)
  perplexity=5:  2D 簇间距 = 20.352
  perplexity=15: 2D 簇间距 = 6.392
  perplexity=40: 2D 簇间距 = 0.496
```

高维关系**恒定**(0.442),2D 簇间距却从 20.352 暴跌到 0.496,差了四十倍。如果你拿 perplexity=5 那张图,说"royalty 和 animals 隔得这么远,所以语义上毫不相干",再拿 perplexity=40 那张图,说"它们贴得这么近,所以高度相关"——你会从同一份数据得出两个相反的结论,而高维真相自始至终没变过。

所以这条规矩是硬的:**任何关于"哪两个簇更近/更远"的结论,必须回到原始高维空间用 cosine 验证,绝不能照 t-SNE 图上的间距读。** t-SNE 的图只可信一件事:谁和谁聚成了一簇(局部邻居)。簇内是可信的,簇间的距离、簇的大小、簇的形状,都不可信。

⚡ 这不是某个 toy 实现的毛病,而是 t-SNE 的固有性质,**目前没有通用解**:它的损失函数只约束"邻居关系",从不约束"簇间几何",所以簇间距离本就是欠定的。学术界提出过一些缓解方向——UMAP(另一种降维方法)号称比 t-SNE 更保全局结构、对超参没那么敏感,但它的全局距离也不是定量可信的,只是"没那么离谱";也有人主张同时看 PCA(可信全局)+ t-SNE(可信局部)互相参照,或干脆放弃用单一 2D 图下定量结论。如何在二维里同时如实保住局部和全局结构,仍是活跃的研究问题,没有一个能让你"放心照图读距离"的标准答案。

## 收束:从 one-hot 的死路,到一张能看见的地图

退一步看整本书走过的路。第 01 章我们从 one-hot 出发,那是一条死路:每个词是一根孤立的坐标轴,任意两个词的相似度恒等于 0,`king` 和 `queen` 在数学上跟 `king` 和 `桌子` 一样无关——空间里**没有结构可言**,也就没什么可画的。然后我们一步步往里灌结构:共现与 PMI(第 02 章)、skip-gram 把预测上下文的任务压进低维向量(第 03 章)、negative sampling 和 InfoNCE 让训练在大词表上可行(第 04–06 章)、再用 analogy 和近邻把质量量成数字(第 07 章)。

这一章是终点也是验收:我们把那个学出来的 16 维空间**投影到一张你能用眼睛看的地图上**——royalty 聚一块,animals 聚一块,方位词聚一块。从"任意两词相似度恒为 0 的死空间",到"语义相近的词在图上挨在一起的活地图",这就是表示学习(representation learning)整件事的意义:**把人类的语义,变成了机器能算、也能画出来给人看的几何。**

但这张地图同时是一份警告。它既是这本书的成果,也是最容易自欺的产物。PCA 和 t-SNE 都是 lossy 投影:PCA 只给你 39.5% 的方差并诚实地告诉你这一点,t-SNE 给你漂亮的局部簇但簇间几何随 perplexity 乱跳。**最后一条原则,适用于这本书讲过的每一个嵌入:可视化是用来生成假设的,不是用来下定论的。看到图上有意思的东西,回到原始 16 维空间用 cosine、用近邻、用 analogy 去验证。** 图会骗你,数字不会——只要你记得数字来自哪个空间。
