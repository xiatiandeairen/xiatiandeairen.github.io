第 02 章我们用 PMI(点互信息,衡量两词共现是否超出偶然)矩阵算出过一组向量,`king` 和 `queen` 的余弦相似度确实不低。但那组向量是"读"出来的:扫一遍语料数共现次数,做个矩阵分解,结束。没有一个参数被梯度推动过,没有任何"训练"发生。本章要做的事,从一句话开始就和前两章分道扬镳:**我们不再统计共现,而是把它变成一个监督学习任务——给定中心词,预测它周围会出现哪些词。** 同样的语料,同样的 `king`/`queen`,但这一次向量是被一个预测损失通过反向传播训练出来的参数。

这个转变看着像换了个算法,实际上是换了一套世界观。下面先讲清楚 skip-gram 到底把"共现"重写成了什么,再用配套仓库 `examples/embeddings-from-scratch/src/stage03-skipgram.ts` 的真实运行结果验证它确实学到了东西,最后——也是本章存在的真正理由——用实测数字把全 softmax 的成本墙砸给你看,那堵墙就是第 04 章要拆的。

## ✦ 核心洞察:把"数共现"改写成"预测上下文"

第 02 章的做法可以一句话概括:`king` 和 `queen` 向量像,是因为我们**显式数过**它们旁边出现的词高度重叠,然后把这个重叠塞进了矩阵分解。相似性是统计的副产品。

skip-gram 的洞察是把因果关系倒过来:**与其统计共现,不如定义一个"用中心词预测上下文"的预测任务,然后让梯度下降去优化向量,使预测尽量准。** 当模型为了"预测 `king` 周围出现 `queen`/`prince`/`kingdom`"而调整 `king` 的向量时,它会被迫把 `king` 推到一个能同时预测好这些词的位置;而 `queen` 因为要预测几乎同一批上下文,也会被推到附近。相关词向量自动靠拢,不是因为我们数了共现,而是因为**它们要解同一道预测题**。共现信息还是那个共现信息,但它从"被统计的对象"变成了"被预测的目标",学习的压力由反向传播自动施加。

这就是为什么 word2vec 当年是个观念突破而不只是工程优化:它证明了语义结构可以作为一个预测任务的**涌现产物**,而不需要任何人手工设计"相似"的定义。下游所有的 sentence embedding、双塔检索、对比学习,本质上都是这一招的放大——把"什么和什么该靠近"写成一个可微的预测/对比目标,剩下交给梯度。

### 模型:两张表 + 全 softmax

具体到 skip-gram 的最朴素形式(原始论文形态,本章不加任何加速技巧),模型就两样东西,定义在 `stage03-skipgram.ts` 的 `SkipGram` 接口里:

```ts
interface SkipGram {
  win: Mat;  // V x D, center-word vectors (the embedding we keep)
  wout: Mat; // V x D, context-word vectors (auxiliary, discarded after training)
  vocabSize: number;
  dim: number;
}
```

注意是**两张** V×D 的嵌入表,不是一张。`win[center]` 是一个词"作为中心词"时用的向量,`wout[w]` 是它"作为被预测的上下文"时用的向量。为什么同一个词要有两个向量?因为一个词扮演"我是谁"和"我能预测谁"这两个角色时,在向量空间里需要的位置是不同的——把两张表绑成一张(weight tying)在这个尺度上会互相打架、还会把机制搅浑。训练完我们只保留 `win` 当作最终嵌入,`wout` 直接丢掉(这也是原论文的约定)。

打分和损失也极其直白,看 `pairLoss` 这个函数(精简后):

```ts
function pairLoss(model: SkipGram, pair: Pair, stable = true): Value {
  const center = model.win[pair.center];
  // score_w = <W_in[center], W_out[w]> for every w. This O(V) loop is the cost.
  const scores: Value[] = [];
  for (let w = 0; w < model.vocabSize; w++) {
    scores.push(dot(center, model.wout[w]));
  }
  // ... softmax over scores, NLL on the true context word ...
}
```

逐行翻译:对中心词向量 `center`,和词表里**每一个**词的输出向量做点积,得到 V 个分数;过 softmax 变成"给定中心词,每个词是上下文的概率分布";损失就是真实上下文词的负对数概率(NLL,negative log-likelihood,即"模型对正确答案给的概率越低,惩罚越大")。点积大 = 模型认为这俩词容易共现,这正是我们想让 `king·queen` 学大的量。

这里的 `dot` 和 `Mat` 不是 numpy,是配套仓库自己手写的标量 autograd(`examples/embeddings-from-scratch/src/core/autograd.ts`)——和本系列姊妹书《tiny-dl》同一个核。每个 `dot` 都在动态计算图上挂好了局部导数,`loss.backward()` 一调就能把梯度回灌到 `win`/`wout` 的每个元素。训练循环的不变量写死在 `trainSkipGram` 里:

```ts
opt.zeroGrad();              // 清掉上一步的梯度(autograd 是累加的,不清会串)
const loss = batchLoss(model, batch, stable);
loss.backward();            // 反向传播,梯度灌进 win/wout
opt.step();                 // Adam 更新参数
```

`zeroGrad → forward → backward → step` 这个顺序是契约,顺序错或漏掉 `zeroGrad`(忘清梯度,导致梯度跨步累加)是 autograd 最经典的发散 bug。代码特意把 `zeroGrad` 挂在优化器上(`core/optim.ts` 的 `Adam`),让你想忘都忘不掉清哪些参数。

### 它真的学到了吗:loss 下降 + 语义聚类

机制讲完,得拿真数字证明它不是在自欺。这是 toy 语料(400 句、2042 token、词表 V=43、维度 D=16、5768 个训练对),绝对数字偏乐观,**能迁移的是机制(loss 真降)和相对趋势,不是绝对值**——这点 stage 输出第一行就老实标注了。

正常训练(Adam,小随机初始化,lr=0.05,120 epochs)的损失曲线:

```
loss 3.761 → 1.819 (降幅 51.6%, 120 epochs)
均匀分布基线 loss = ln(V) = 3.761 (训练后应低于此)
```

这里有个值得记住的判据:**均匀分布的 NLL 恰好等于 ln(V)**。如果模型啥也没学、对每个词都猜等概率,损失就停在 ln(43)=3.761。而我们的初始损失 3.761 几乎正好踩在这个基线上(小初始化让初始 softmax 接近均匀,符合预期),训练后降到 1.819——**显著低于基线,这才证明模型学到了一个非平凡的条件分布,而不是噪声**。下次你训任何分类/预测模型,第一件事就该算 ln(类别数) 看看 loss 有没有真的破基线,这比盯着 loss 绝对值有意义得多。

学到的语义结构(用 `win`,余弦相似度):

```
king  → queen(0.98)  princess(0.88)  prince(0.88)  kingdom(0.84)
north → east(0.99)   south(0.98)     west(0.97)    then(0.45)
cat   → dog(0.69)    mouse(0.67)     bone(0.61)    rice(0.53)
cos(king, queen) = 0.985
cos(um,   king)  = 0.563 (filler 词无信号, 接近噪声)
```

`king`/`queen` 互为最近邻、方位词聚成一簇,这些都是梯度推出来的,不是抄共现矩阵。同样值得看的是 **`um` 这个 filler 词(语料里随机撒的、不携带任何上下文信号)和 `king` 的相似度只有 0.563,接近初始噪声水平**——这是一个诚实的负样本:你学不到数据里不存在的结构。一个会对所有词都报高相似度的嵌入是坏的,这个 0.563 恰好说明空间没有塌缩。

## ⚡ 成本墙:全 softmax 的分母正比于词表大小

到这里 skip-gram 看着很美:机制干净、loss 真降、语义出来了。那为什么真实的 word2vec 从来不用这个朴素版本?**因为它根本跑不起来。**

回头看 `pairLoss` 里那个被注释标了 `// This O(V) loop is the cost` 的循环:对**每一个**训练对,softmax 的分母都要对**整个词表**求和——也就是中心词向量要和全部 V 个输出向量各做一次点积。一对的成本 ∝ V。一个 epoch 几千对、几十个 epoch、再乘上每步 V 次点积,总成本里那个 V 是甩不掉的乘数。

这不是纸上推导,stage ③ 直接量出来了。`measureStepMs` 在合成词表上把 V 拨到不同值,实测单步 forward+backward 的墙钟耗时(取 7 次中位数,真测量不是估算):

```
V=50  │██░░░░░░░░░░░░░░░░░░░░░░░░░░░░│ 0.177
V=200 │███████░░░░░░░░░░░░░░░░░░░░░░░│ 0.766
V=800 │██████████████████████████████│ 3.081  (ms, median of 7)
   V 50→200 (×4.0): 时间 ×4.32 (理想线性应≈×4.0)
   V 200→800 (×4.0): 时间 ×4.02 (理想线性应≈×4.0)
```

词表翻 4 倍,单步耗时也几乎正好翻 4 倍(×4.32 和 ×4.02,贴着理想线性)。曲线是直的,O(V) 是真的。

现在把这条直线外推到真实场景。word2vec 原始论文用的是 Google News 语料,词表约 70 万词。按这条 O(V) 直线,每个训练对要做 70 万次点积才能算一次 softmax 分母——而语料有几十亿词、几十亿个训练对。**toy 上 V=800 单步 3 毫秒看着无所谓,放到 V=70 万、再乘上几十亿对,训练时间直接膨胀到不可行的量级。** 这个外推是估算(基于实测斜率),但斜率本身是测出来的硬数据,结论方向不会错:全 softmax 的分母让训练成本和词表大小死死绑定,词表一大就崩。

这就是 stage 小结那句"全 softmax 让机制透明但成本 O(V)"的全部分量。我们在本章**故意**付这个全 softmax 的代价,就是为了让你先亲手摸到这堵墙,再去看第 04 章怎么拆。

**⚡ 开放面:这个瓶颈到今天也没有"免费"的通用解。** 第 04 章会讲的两条经典路线——negative sampling(负采样,每步只和少数随机负样本比,把分母近似掉)和 hierarchical softmax(层次 softmax,把 V 路分类拆成 log V 次二分类)——都是用近似换速度,各有偏差和工程代价(负采样的采样分布要调、层次 softmax 的树结构影响精度)。再往后到大语言模型时代,输出词表是几万到十几万的 BPE token,最终那个 softmax 分母**至今仍是推理和训练的真实开销大头**,工业界靠的是 fused kernel、混合精度、各种 sampling 技巧硬扛,而不是有了某个一劳永逸消除分母的算法。"如何在不牺牲精度的前提下绕开归一化分母"这个问题,从 2013 年的 word2vec 到今天的 LLM,仍然是**没有通用最优解、还在持续被研究的**开放方向(noise-contrastive estimation、各种 self-normalizing 的尝试都只在特定条件下成立)。本章的 O(V) 实测,就是这条研究线最朴素的起点。

## 三种把训练搞崩的方式(以及它们为什么崩)

知道一个机制"正常时怎么跑"只是一半,资深工程师值钱的另一半是知道**它怎么坏、为什么这么设计**。stage03 特意内置了三个失败演示,每个都对应一个真实会踩的坑。

### 失败 ①:学习率过大 → 发散,但不爆 NaN

把 lr 从 0.05 调到 5.0(其余不变):

```
loss 3.762 → 27.091 (不降反升; 均匀基线 ln(V)=3.761, 正常 lr=0.05 收敛到 1.819)
```

损失不降反升,最后停在远高于均匀基线 3.761 的地方——**比瞎猜还差,模型学崩了**。原因:Adam 每步的有效步长大致就是 lr,lr 太大时每一步都跨过损失的极小点来回弹,参数越推越远,损失发散。

这里有个**反直觉的诚实点**:很多人以为 lr 太大就会 NaN。不会。只要 softmax 是数值稳定版的,lr 过大只会**发散**(loss 爆到 27 但仍是有限数),不会 NaN。NaN 是另一回事,看失败 ②。

### 失败 ②:朴素 softmax(去掉 max 平移)→ 真正爆 NaN

同样 lr=5.0,但这次把 `pairLoss` 的 `stable` 关掉——也就是 softmax 不再减去最大分数:

```
已爆 NaN: 第 3 epoch loss=NaN (对比 ① 同 lr 但 stable softmax 不爆, 只发散)
```

第 3 个 epoch 就 NaN 了。链条是:lr 大 → 分数变大 → `exp(score)` 溢出成 `Inf` → `Inf/Inf = NaN` → NaN 顺着反向传播污染**所有**参数,全盘报废。

`pairLoss` 默认开 `stable=true`,做的就是经典的 **log-sum-exp trick**:exp 之前先把每个分数都减去当前的最大分数。最大值是 detach 的常数(不参与求导),它给每一项都平移同样的量,**数学上不改变 softmax 结果、不改变梯度,只改变数值**——但它保证了最大的那个 exp 的指数 ≤ 0,`exp(≤0) ≤ 1`,永远不会溢出。这就是为什么所有正经的 softmax 实现(PyTorch/TF 的 `log_softmax`、`cross_entropy`)都内建这个平移。代码把 `stable=false` 单独暴露出来,纯粹是为了让你亲眼看见不平移会怎么死——平时**永远别关**。

注意 ① 和 ② 的区分:**同样的大 lr,稳定 softmax 只发散、朴素 softmax 才 NaN**。这是两个不同的失败模式,排查时分清楚——看到 NaN 先怀疑数值溢出(softmax/log/除零),看到 loss 飞但还是有限数则怀疑 lr/梯度。

### 失败 ③:初始化方差过大 → softmax 饱和,梯度消失

这个最隐蔽。lr 正常(0.05),只把初始化标准差从 0.1 调到 5.0:

```
loss 26.869 → 25.771 (降幅仅 4.1%, 对比小初始化降幅 51.6%)
"king" 中心词 softmax 最大概率 ≈ 1.000 (接近 1 = 饱和, 几乎无梯度可学)
```

降幅只有 4.1%(正常是 51.6%),模型基本没学。不发散、不 NaN,就是**学不动**。原因:初始化太大 → 初始点积量级巨大 → softmax 一个峰独大(`king` 的最大概率直接 ≈ 1.000)→ 进入饱和区 → `∂loss/∂score ≈ 0` → 梯度消失 → 参数推不动。

这正是为什么 `buildSkipGram` 的 `initStd` 是**载荷参数**,默认必须给小值(代码里 0.1):向量要从接近原点起步,初始点积才接近零,第一个 softmax 才接近均匀,梯度才有正常的量级可用。stage 把 `initStd` 暴露成可调,就是为了把这个"看不见的崩法"显式化——它不报错,只是悄悄不学,是最容易被误诊成"模型不行"的那一类。

把三个失败连起来看,它们共享一个主题:**softmax 既是 skip-gram 的心脏,也是它最脆弱的地方**——太大的 lr 让它发散,无平移让它溢出,太大的初始化让它饱和。理解这一个组件的三种死法,比记住任何超参表都管用。

## 小结与承接

本章完成了全书最关键的一次观念切换:**embedding 从"直接读出来的统计量"(第 01/02 章)变成了"被梯度下降训练的参数"(本章)。** skip-gram 用一个"中心词预测上下文"的监督任务,让反向传播自动把相关词向量拉近;我们用 toy 语料验证了 loss 真破 ln(V) 基线、语义聚类真出现、filler 词真没信号;又用三个内置失败演示把 softmax 的三种死法讲透。

但我们也亲手撞上了那堵墙:全 softmax 的分母正比于词表大小,实测单步耗时随 V 线性增长,真实词表一大就不可行。这堵墙就是下一章的全部动机——**第 04 章会用 negative sampling 把"对全词表归一化"换成"只和少数负样本对比",在保住语义质量的前提下砍掉 O(V) 这个乘数**,顺带引出后面 InfoNCE、温度、对比学习这一整条现代表示学习主线。
