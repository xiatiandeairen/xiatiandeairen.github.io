第 03 章那个全 softmax 的 skip-gram,每走一步训练都做同一件昂贵的事:拿中心词向量去和**整个词表**的每个输出向量点积一遍,求 exp,加起来当分母。词表 V=40 的玩具语料上你没感觉,但这个分母的成本严格正比于 V。真语料词表轻松 10^5,大模型 tokenizer 到 10^5~10^6,意味着每一个 (中心词, 上下文) 训练样本——而一遍语料有上亿个这样的样本——都要做一次百万级的求和加百万级的反向传播。这不是慢一点的问题,是根本跑不完。

负采样(negative sampling,SGNS = Skip-Gram with Negative Sampling,Mikolov 2013)解决的就是这一个分母。它的赌注很激进:**根本不算那个分母**。把"在 V 个词里挑出正确上下文"这道多分类题,换成一道完全不同的题——"给你一个 (中心词, 上下文) 对,判断它是真的还是我编的"。真对要判 yes,几个随机编的假对要判 no。一道 V 路 softmax 就这样塌成了 1+k 个独立的二分类,每个二分类只是一次点积加一个 sigmoid。

配套代码在 `examples/embeddings-from-scratch/src/stage04-negative-sampling.ts`,跑 `npx tsx src/stage04-negative-sampling.ts`。下面所有数字——单步墙钟、loss、最近邻、采样概率——都是这个文件真跑出来的,不是估的;玩具尺度(V=43)上不可信的地方我会明确标出来,可迁移的是趋势。

## 为什么 softmax 必须算完整分母,而二分类不用

先把第 03 章的成本钉死。skip-gram 的目标是最大化 `P(上下文 | 中心词) = exp(c·o_ctx) / Σ_v exp(c·o_v)`。分子只碰一个输出向量,但那个归一化分母 `Σ_v` 要遍历全词表。代码里这就是一个对 `outEmb` 所有行的循环:

```ts
// softmaxStepLoss: 第 03 章的全 softmax 损失项, 这章拿来计时对比
function softmaxStepLoss(inEmb: Mat, outEmb: Mat, center: number, context: number): Value {
  const c = inEmb[center];
  const scores: Value[] = [];
  for (let v = 0; v < outEmb.length; v++) scores.push(dot(c, outEmb[v]).exp());
  let denom = new Value(0);
  for (const s of scores) denom = denom.add(s);          // ← 这条 V 路求和是全部成本
  const numer = scores[context];
  return numer.div(denom).log().mul(-1);
}
```

注意这个 `denom` 不只是前向慢——它在自动微分图里是 V 个 exp 节点的和,反向传播要把梯度灌回**每一个**输出向量。每步更新整张 `outEmb`。这是绕不开的:softmax 的概率定义本身就要求归一化,归一化就要求看到所有竞争者。

负采样换了目标函数,于是甩掉了这个约束。它不再问"上下文是哪个词",而是对每个采样负样本 n_j 单独问"(c, n_j) 是真对吗"。损失变成:

```
L = -[ logσ(c·o_ctx) + Σ_{j=1..k} logσ(-c·n_j) ]
```

代码里就是 `pairLoss`,逐项把 logσ 累加再取负:

```ts
function pairLoss(model: Sgns, center: number, context: number, negatives: number[]): Value {
  const c = model.inEmb[center];
  const posTerm = logSigmoid(dot(c, model.outEmb[context]));  // 真对: 推高 c·o_ctx
  let loss = posTerm;
  for (const n of negatives) {
    // logσ(-c·n): 注意是给分数取负, 不是给整项取负 → 推低 c·n (σ(-x)→1 即 x→-∞)
    loss = loss.add(logSigmoid(dot(c, model.outEmb[n]).mul(-1)));
  }
  return loss.mul(-1);  // 对数似然 → 要最小化的损失
}
```

关键在循环边界:`negatives` 只有 k 个(默认 5)。整个损失图只碰 `1 + k` 个输出向量,反向传播只更新这 `1 + k` 行。**与 V 无关**。这就是全部的魔法——softmax 的成本写在 `for (v < outEmb.length)` 里,SGNS 的成本写在 `for (n of negatives)` 里,前者随词表涨,后者是常数。

### 一个容易被略过的设计决策:两套向量矩阵

代码里 `Sgns` 有 `inEmb`(词当中心)和 `outEmb`(词当上下文/负样本)两个矩阵,不是共享一个。为什么?如果共享,一个词当中心去预测它自己当上下文时,`w·w` 就是向量模长平方,天然很大,sigmoid 直接饱和到 1——模型学到"每个词最像它自己"这种废话,几何结构塌成各自为政。两套矩阵是 word2vec 的标准做法,让"作为中心"和"作为上下文"是两个可以不同的角色。这是个 1 行代码的决策,但选错了整个空间学不出结构。

### ✦ 负采样的精妙:repulsion 才是防塌的唯一力量

把 `pairLoss` 拆开看力的方向。正项 `logσ(c·o_ctx)` 推高真对的点积——真上下文互相吸引。负项 `logσ(-c·n)` 推低假对的点积——采样到的负样本被推开。

这里有个反直觉的点:**如果只有正项会怎样?** 所有词互相吸引、没有任何排斥力,最优解是把所有向量塌到同一个点——任意两个向量点积都最大,正项 loss 全局最小。空间退化成一个点,什么都没学到。负采样里那 k 个被推开的负样本,是**唯一**阻止坍缩的力。

这也直接推出了为什么"采样哪些词当负样本"这么要命——后面第三个实验整章在讲这个。你推开的词决定了空间被撑开的方向。推错了(全推开虚词),空间就在没意义的维度上被撑开。这个"正样本拉近、采样负样本推开"的骨架,正是第 05 章对比学习(InfoNCE)的母题;SGNS 可以看成只有一个正样本、k 个负样本、用 sigmoid 而非 softmax-over-negatives 的对比学习特例。

### 实测:softmax 随 V 线性涨,SGNS 几乎不动

代码用同一个自动微分引擎,在几个合成词表尺寸上各计时一步前向+反向(DIM 固定,预热后取稳态,见 `measureSoftmaxStepMs` / `measureSgnsStepMs`)。真测结果:

```
  V      softmax(ms/步)   SGNS k=5(ms/步)   softmax/SGNS 倍数
  40            0.161           0.063              2.6x
  200           1.808           0.146             12.4x
  1000          7.689           0.342             22.5x
  4000         26.235           1.883             13.9x
```

读法:softmax 从 V=40 的 0.161ms 涨到 V=4000 的 26.235ms——约 163 倍,词表大了 100 倍,成本大了百倍量级,基本线性。SGNS 同区间从 0.063ms 到 1.883ms,涨幅小得多,曲线接近水平(末行的轻微抬头和 4000x4000 矩阵的内存/缓存效应、玩具引擎的常数开销有关,不是算法本身随 V 涨)。

这张表就是整章的全部论点。玩具尺度的绝对值偏小、倍数列也不平滑(玩具引擎噪声大),**可迁移的不是具体倍数,是两条曲线的形状**:softmax ∝ V,SGNS ≈ 常数。把 V 外推到 10^6,softmax 单步要做百万次点积,SGNS 还是 6 次(1+5)。word2vec 能在维基百科全量语料、百万词表上训出来,就是因为它走的是下面那条平的线。

同一次训练(k=5,unigram^0.75)的收敛也是真的:loss 从 3.402 降到 2.940,2400 步,训练墙钟 125.0ms,平均 0.052ms/步。学出来的最近邻 `king → queen(0.87) prince(0.86) walk(0.63)`——语料是程序生成的有 planted 结构的玩具,所以 king/queen/prince 该近的确实近了。

## k:负样本数,一个成本换质量的旋钮

k 是每个正对配几个负样本。第二个实验扫 k=1/5/20,同语料同种子,只动 k:

```
  k    训练墙钟(ms)   ms/步     planted-pair 平均cos   king 最近邻
  1           47.5    0.020                0.355   princess(0.75)  kingdom(0.67)
  5          112.4    0.047                0.542   queen(0.87)  prince(0.86)
  20         330.9    0.138                0.566   prince(0.86)  queen(0.81)
```

成本侧很干净:每步要算 (1+k) 次点积,k 从 1 到 20,ms/步 从 0.020 涨到 0.138,近似线性——你多买几个负样本就多付几次点积。

质量侧(planted-pair cos,越高表示该近的词越近)是先升后饱和:k=1 只有 0.355,k=5 跳到 0.542,k=20 才 0.566。从 5 到 20,质量只多了 0.024,成本却涨了快 3 倍(112ms → 331ms)。最近邻列也印证:k=1 时 king 的头号邻居是 princess 而非 queen,信号还没稳;k=5 已经 queen/prince 排前;k=20 几乎没改善还略有抖动。

为什么先升后饱和?k=1 时每步只有一个负样本提供排斥力,梯度方差大、撑开空间的方向太随机,所以质量差。k 变大,每步的排斥力来自多个方向、更接近真实噪声分布的期望,空间撑得更稳。但一旦负样本数足够刻画"哪些方向该推开",再加就是边际递减——你已经在正确方向上推了,多推不会更对。这跟 word2vec 论文的经验吻合:小语料 k 取 5-20,大语料 2-5 就够(大语料每个词出现次数多,跨样本累积的负样本天然多,单步不必配那么多)。玩具上的甜点是 k=5。

这里要给一句诚实话:planted-pair cos 这个标量在玩具上能区分 k=1 和 k=5,但 k=5 到 k=20 的 0.542→0.566 已经接近这个指标在小词表上的天花板,不要把它当精密刻度。判断甜点更该看"成本曲线在哪开始陡涨而质量曲线在哪开始平"——这两条曲线的交叉点比单个数字可靠。

## ⚡ 噪声分布:0.75 次方是个没有通用解的经验魔法

负样本从哪个分布里抽?这是负采样最微妙、也最缺乏理论的地方。代码给了三个选择(`buildNegWeights`):

```ts
case "uniform":      return vocab.counts.map(() => 1);              // 每词等概率
case "unigram":      return vocab.counts.slice();                   // 按词频原样
case "unigram075":   return vocab.counts.map((c) => Math.pow(c, 0.75)); // 词频^0.75
```

先看这三个分布实际会抽中什么。最高频词是 "the"(出现 418 次),单次负采样抽中它的概率:

```
    uniform    : 2.3%
    unigram    : 16.3%
    unigram075 : 10.8%
```

这三个数把问题讲透了。raw unigram(按词频原样采)把 **16.3% 的负样本砸在 "the" 这一个词上**——你的负样本预算六分之一花在反复告诉模型"别把 the 当成某词的特定上下文"。但 the 几乎是所有词的上下文,模型本来就学不出"the 是谁的专属上下文",这个信号是空的。内容词之间(king vs dog vs bread)拿不到区分压力,因为负样本都被虚词占了。这就是 raw unigram 的失败模式:**负样本被高频虚词淹没,排斥力浪费在无信息的方向上**。

另一头,uniform(等概率)把 the 的采样概率压到 2.3%——但这是因为玩具词表里罕见词多、虚词天然占比低。词表一大就翻车:真语料 V=10^5 时,uniform 下每个词概率 ~10^-5,你几乎永远抽不到高频词,负样本全是只出现过一两次的长尾词。长尾词跟谁都几乎不共现,把它推开等于没推(它本来就远),排斥力同样是空的。

0.75 次方是 Mikolov 团队试出来的折中:`count^0.75` 压平了头部(the^0.75 还是比 rare^0.75 大,但比值被压缩了),把虚词从 16.3% 降到 10.8%,同时又给中频内容词留了足够的采样质量。两头的坑——uniform 抽不到信息量、raw unigram 被虚词淹没——它都绕开了。

为什么是 0.75 而不是 0.7 或 0.8?**没有推导。** 这是网格搜索在他们的语料上跑出来的数,论文里就是直接报了这个值。后续工作(如 Word2Bits、各种 GloVe 变体的对比实验)反复确认 0.75 附近确实好,但"为什么恰好是 3/4 次幂"至今没有令人信服的第一性原理解释。它对语料的频率分布(Zipf 系数)、领域、词表大小理论上都该敏感,但实践中 0.75 鲁棒得离谱。**这是个目前无通用解、仍属经验调参的开放问题**:最优噪声分布应随数据分布自适应,但没人给出可证明最优的自适应规则,生产里大家就抄 0.75。

### 玩具尺度的诚实读法:别信饱和标量,信负样本组成

代码用三个分布各训一次,报了 planted-pair cos 和"虚词侵入近邻率":

```
  [uniform]    planted-pair cos=0.615 · 虚词侵入近邻率=0.095
  [unigram]    planted-pair cos=0.541 · 虚词侵入近邻率=0.095
  [unigram075] planted-pair cos=0.542 · 虚词侵入近邻率=0.095
```

如果你只看这两个标量,会得出"uniform 最好、0.75 没优势"的结论——这是**错的**,而且代码注释里专门拦了这个坑。原因:V=43 的玩具词表上 planted 结构太强、指标饱和,三个分布的标量几乎打平,排名还随种子抖动。在这种尺度上拿单个饱和标量给分布排序,就是把噪声当信号。

uniform 在玩具上"看起来不差"纯属伪优势:玩具里罕见词多、虚词占比低,uniform 没怎么被虚词坑到。这个优势在真语料(虚词占比高、长尾极长)直接消失——前面分析的两个坑都是 uniform 在大词表上才暴露。

那玩具上什么是可信的?**负样本组成那三个百分比**(2.3% / 16.3% / 10.8%),因为它们是从权重表精确算出来的,跟词表规模无关、跟训练随机性无关。raw unigram 把 16.3% 负样本砸在 the 上、0.75 降到 10.8%——这个差异是确定的、可迁移的。质性上也看得到失败模式的影子:raw unigram 训出来 `king → ...and`、`bread → rules/not`,把虚词和跨簇词拉进了近邻;0.75 下 king 的近邻干净得多。

这一条值得单独记:**小词表 / 小数据上,别靠单个会饱和的聚合标量给方案排名,去找那个跟规模无关、能精确算出的中间量**(这里是采样组成)。很多"我们的方法在 benchmark 上 +0.3%"的对比,在数据量不够、指标已饱和时,报的就是种子噪声。

## 小结与下一章的接口

把这章的账算清:SGNS 用 `1+k` 个二分类(sigmoid)替换了第 03 章的 V 路 softmax,单步成本从正比于 V 变成常数(实验①真测,V 从 40 到 4000 softmax 涨 163 倍而 SGNS 基本平)——这是 word2vec 能 scale 到百万词表的唯一原因。k 是成本换质量的旋钮,质量先升后饱和,玩具甜点 k=5(实验②)。负采样分布决定排斥力花在哪:raw unigram 把 16.3% 负样本浪费在虚词上(失败模式),uniform 在大词表抽不到信息,unigram^0.75 折中——而 0.75 这个数本身是没有通用解的经验魔法(实验③)。

第 05 章会把"正样本拉近、负样本推开"这个骨架正式抽象成对比学习,用 InfoNCE 损失把这章的 k 个负样本变成一个 softmax-over-negatives(注意:不是 over-vocab,所以仍然便宜),并引入温度参数控制推开的力度。你会发现 SGNS 其实就是对比学习家族里最早、最朴素的那个成员——这章理解的每一处权衡,在那里都会以更一般的形式重新出现。
