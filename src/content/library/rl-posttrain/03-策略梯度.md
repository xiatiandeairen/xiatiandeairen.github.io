前两章你学的是「价值」：第 01 章铺 MDP（马尔可夫决策过程，状态-动作-奖励的循环），第 02 章在 gridworld 上把每个格子的状态价值 V(s) 迭代出来，再 argmax 导出策略。那条路有个隐含假设——**你能把价值表算准，策略就是价值的附属品**。这章把这个假设反过来：跳过价值，直接对策略本身求梯度做上升。这就是 REINFORCE（最朴素的策略梯度算法），也是 PPO / GRPO / RLHF 这一整条后训练（post-training，预训练之后用 RL 对齐模型）流水线的真正起点——因为语言模型的动作空间是几万个 token，你没法像 gridworld 那样把 V 表枚举出来，只能直接学策略。

但我要先把结论摆在前面，免得你被「能不能学会」这个伪问题带偏：在本章的 toy 环境里，策略梯度**几乎一定能学会**。真正决定它在大模型上死活的，是**方差**。这章 80% 的篇幅在讲一件事——这个梯度估计是无偏的（unbiased，期望意义下指向正确方向），但方差大到能让你在真实任务上完全学不动。baseline、奖励归一化、advantage 这三件解药，没有一件改变无偏性，全都只是在压方差 / 稳步长。把这点吃透，你才能看懂第 04 章 PPO 的 clip（裁剪步长）和第 08 章 GRPO（组相对策略优化）为什么敢扔掉整个 critic 网络。

本章配套代码全在 `examples/rl-posttrain-from-scratch/src/stage03-reinforce.ts`，依赖的数学原语在 `examples/rl-posttrain-from-scratch/src/core/probability.ts`。下面的所有数字都来自 `npx tsx src/stage03-reinforce.ts` 的真实输出，环境是 8 状态 × 4 动作的 contextual bandit（情境赌博机：每步给你一个状态，你选一个动作，立刻拿一个带噪奖励就结束）。**toy 的绝对数字是乐观的**——可迁移的是相对趋势和机制，不是具体命中率，这点我会反复标注。

## 一、从「学价值」到「学策略」：softmax 策略与 ∇logπ

### 策略本身就是一张可微的表

第 02 章的策略是「查 V 表 → argmax」，argmax 不可微，没法直接对它求梯度。策略梯度的第一步是把策略变成一个**可微的概率分布**。最朴素的做法：给每个 (状态, 动作) 一个 logit（未归一化的打分）θ[s][a]，然后 softmax 成概率。

看配套代码里策略表的初始化（`stage03-reinforce.ts`）：

```ts
// Linear logits θ[s][a]; policy is softmax(θ[s]). Init at 0 → uniform start, so
// entropy starts at log(nActions) and we can WATCH it (not) collapse.
const theta = Array.from({ length: env.nStates }, () => new Array<number>(env.nActions).fill(0));
```

全部初始化为 0，意味着每个状态下 softmax 出来是均匀分布——4 个动作各 0.25。这不是随手填的：均匀起点让熵（entropy，分布的不确定性）从最大值 log(4)≈1.386 开始，后面我们就能**看着它塌**（这是失败模式 A 的关键诊断信号，见第三节）。

softmax 的实现别自己拍脑袋写，数值稳定性是正确性而非优化。看 `core/probability.ts`：

```ts
// softmax: logits -> probability distribution. The max-subtraction is not an
// optimization, it is correctness: exp(1000) is +Infinity, and Infinity/Infinity
// is NaN. Subtracting max(logits) leaves the distribution identical (softmax is
// shift-invariant) but keeps every exp argument <= 0, hence in (0, 1].
```

减去 max 这一步——softmax 平移不变（shift-invariant，给所有 logit 加常数结果不变）——是后面一个反直觉结论的伏笔：第四节会看到，正因为有它，奖励放大 1 万倍也不会真的算出 NaN，伤害是另一种形式。记住这点。

### 策略梯度定理：为什么是 R · ∇logπ

REINFORCE 的更新规则一行就能写：朝着「让高回报动作的对数概率上升」的方向走。数学上是策略梯度定理（policy gradient theorem）：

```
∇θ E[R] = E[ R · ∇θ log π(a|s) ]
```

大白话：**想让期望回报上升，就把每个采到的动作的 log 概率，按它拿到的回报加权，做梯度上升**。回报高的动作，log 概率往上推得多；回报低（甚至负）的，往下压。注意这是个期望——你拿不到真期望，只能用采样的单个轨迹去估它,这正是方差的源头。

对 softmax 策略，∇logπ 有个干净的闭式解。看代码里的核心更新（`stage03-reinforce.ts`）：

```ts
// ∇ log π(a|s) for softmax = (1_a − π). θ[s] += lr · advantage · ∇logπ.
for (let k = 0; k < env.nActions; k++) {
  const grad = (k === a ? 1 : 0) - probs[k];
  theta[s][k] += cfg.lr * advantage * grad;
}
```

`(k === a ? 1 : 0) - probs[k]` 就是 ∇logπ（对数似然梯度）：被选中的动作 a 那一项是 `1 - π(a)`（正，往上推），没被选中的是 `-π(k)`（负，往下压）。这就是 softmax 策略梯度的全部数学。这三个零件——`log π`、`∇logπ`、以及马上要讲的 `advantage`——是 PPO 和 GRPO 共用的地基。PPO 的 ratio 是 `exp(logπ_new − logπ_old)`，DPO（第 07 章）的 loss 是两个 logπ 之差，全都站在这个 `logProb` 上（见 `core/probability.ts::logProb`）。

这里先埋一个钩子：注意 `grad` 里有个 `(1_a − π)` 项。当某个动作的 π 被推到接近 1（one-hot，独热，几乎所有概率压在一个动作上），它的 `(1 − π) → 0`，其它动作的 `−π → 0`——**整个梯度趋近 0，策略卡死**。这是后面两个失败模式的共同机制。

## 二、为什么这个估计无偏但高方差 + baseline 的本质

### 无偏，但方差是真正的敌人

跑一下四配置对照（baseline 开关 × 归一化开关），真实输出：

```
[1] 四配置对照（8 状态×4 动作, lr=0.1, 8000 步, 随机基线命中率≈0.25, 本 toy 奖励∈[0,1] 近零均值）
  无baseline 无归一 | 命中率=1.000 | 早期 advantage 方差=2.05e-1 | 奖励曲线 ▁▁▅▆▇███████████████
  有baseline 无归一 | 命中率=1.000 | 早期 advantage 方差=1.77e-1 | 奖励曲线 ▁▁▄▆▇███████████████
  无baseline 有归一 | 命中率=1.000 | 早期 advantage 方差=1.74e-1 | 奖励曲线 ▁▃▆█████████████████
  有baseline 有归一 | 命中率=1.000 | 早期 advantage 方差=1.44e-1 | 奖励曲线 ▁▃▇█▇███████████████
```

四个配置全部收敛到命中率 1.000。**老实讲：在这个 toy 上 baseline 的收益本就小。** 任务简单 + 奖励近零均值（means 落在 [0,1]，optimal=1.0 其余在 [0,0.6)，见 `core/envs.ts::makeContextualBandit`），baseline 该减的偏移本来就接近 0。我把这个诚实地放在这里，是因为很多教程会在 toy 上夸大 baseline 的作用——你跑出来发现「开不开都 1.0」，反而怀疑自己理解错了。没错，baseline 在零均值小任务上确实近乎无用。

但 advantage 方差那一列已经在透露信号：从 2.05e-1（无 baseline 无归一）单调降到 1.44e-1（双开）。**方差降了，命中率没变**——因为任务太简单，方差还没大到吃掉信号。要看 baseline 真正发力，得把奖励推到真实 RLHF 的形态。

### baseline 为什么不引入偏差，只降方差 ✦

这是判断一个人是否真懂策略梯度的试金石。先看 baseline 怎么算（`stage03-reinforce.ts`，per-state 运行均值）：

```ts
// Per-state running baseline b(s) = mean reward in state s. Subtracting it is
// unbiased (E[b·∇logπ] = b·0 = 0) but recenters the advantage near 0, shrinking
// its variance — the variance-reduction trick we measure directly below.
const baseline = new Array<number>(env.nStates).fill(0);
```

代码注释把证明压成了一行：`E[b·∇logπ] = b·0 = 0`。展开讲——

我们把更新从 `R·∇logπ` 换成 `(R − b)·∇logπ`。多减的那一项是 `b·∇logπ`，它对梯度的期望贡献是：

```
E_a[ b · ∇logπ(a|s) ] = b · E_a[∇logπ(a|s)] = b · Σ_a π(a)·∇logπ(a) = b · Σ_a ∇π(a) = b · ∇(Σ_a π(a)) = b · ∇(1) = 0
```

关键在 `Σ_a π(a)·∇logπ(a) = Σ_a ∇π(a) = ∇(Σπ) = ∇1 = 0`：**概率必须和为 1，对常数求梯度是 0**。所以只要 b 不依赖于动作 a（per-state baseline 满足，它只依赖 s），减掉它对梯度的**期望**毫无影响——估计仍然无偏。

那它有什么用？它改变的是**方差**，不是期望。`Var(R−b) < Var(R)` 当 b 接近 R 的均值时。直觉：原始更新里，一个全为正的回报会让**所有**被采到的动作都往上推（只是程度不同），梯度向量整体偏置、方向噪声大；减掉均值后，回报有正有负，好动作真往上、坏动作真往下，每一步的修正方向更干净，跨样本的抖动更小。

**一句话记牢：baseline 是个「期望意义下的零」——加它不改变你要去的方向，只让你少走冤枉路。** 这个洞察直接通向本章的前沿钩子（第五节）。

### baseline 的真正价值随奖励偏移增长

toy 上 baseline 没用，是因为奖励近零均值。真实 RLHF 的 reward model 打分**全是正的**（一个 0~10 或经过 sigmoid 的标量），这是一个巨大的正偏移。配套代码给奖励加常数 offset 模拟这种形态，实测：

```
[2] baseline 的真正价值随奖励偏移增长（真实 RLHF 奖励全为正 = 大偏移）
  offset= 0 | 无baseline: adv方差=2.05e-1 命中率=1.000 | 有baseline: adv方差=1.77e-1 命中率=1.000 | 方差比=1.16×
  offset= 5 | 无baseline: adv方差=2.45e-1 命中率=0.875 | 有baseline: adv方差=1.77e-1 命中率=1.000 | 方差比=1.38×
  offset=20 | 无baseline: adv方差=1.68e-1 命中率=0.125 | 有baseline: adv方差=1.77e-1 命中率=1.000 | 方差比=0.95×
```

读这张表要小心，有个陷阱：

- **offset=0**：方差比 1.16×，baseline 几乎无用——印证上一节。
- **offset=5**：方差比升到 1.38×，更重要的是**命中率被救回**：无 baseline 跌到 0.875，有 baseline 稳在 1.000。偏移一上来，不减基线的策略就开始学废了。
- **offset=20**：方差比看着是 0.95×（≤1），像是 baseline 帮了倒忙——**这是假象**。无 baseline 此时已经塌成 one-hot 卡死（命中率仅 0.125），分布根本没散开，advantage 自然抖不起来，所以方差**反而低**。低方差不是好事，是「已经死了不会动」。真正的信号是命中率：**0.125 → 1.000**。大偏移下不减基线直接学废，有 baseline 稳稳满分。

为什么大偏移会学废?回到 ∇logπ 更新：`θ += lr·(R−b)·∇logπ`。当 R 被偏移项主导（比如 R≈20，但好坏动作的真实差异只有 1.0），步长被那个无意义的 +20 撑大，第一次更新就把策略一把推成 one-hot——锁死机制（第三、四节细讲）。减掉 baseline，offset 被消掉，步长重新由真实的动作差异决定。

**这就是为什么 PPO / GRPO 必带 baseline / advantage。** 真实奖励全为正是常态，不减基线 = 步长被偏移项绑架 = 学废。这不是「锦上添花」，是「不做就崩」。

### advantage = return − baseline

把上面两件事合起来,就得到了 advantage（优势函数）的定义：

```
A(s,a) = R(s,a) − b(s)
```

「这个动作比该状态的平均水平好多少」。代码里就是 `reward - b`（`stage03-reinforce.ts`）：

```ts
let advantage = reward - b;
if (t < earlyCutoff) advVar.push(advantage); // measure BEFORE normalization rescales it
```

在这个单步 bandit 里，return 就是单步 reward（没有未来回报要折现）。在 PPO 里 advantage 会升级成 GAE（generalized advantage estimation，用 critic 估的多步优势），但**本质不变：用某个 baseline 把绝对回报变成相对优势**。第 04 章你会看到 PPO 把 b(s) 换成一个学出来的 critic 网络 V(s)；第 08 章 GRPO 会把它换成「同一道题采样一组答案的组内平均」。三者是同一个零件的三种实现。

## 三、失败模式 A：学习率过大 → 熵塌缩

光讲解药不讲怎么坏，等于没讲。第一个失败模式是 entropy collapse（熵塌缩）：学习率过大，策略在**探索之前**就塌成 one-hot，之后再也动不了。

代码用一个夸张的 lr=8 触发它（健康值 lr=0.1），并打印熵曲线作诊断（`stage03-reinforce.ts` 每隔约 1/25 步采样一次平均熵）：

```
[3] 失败模式 A — 学习率过大 → 熵塌缩到 one-hot 后学不动（entropy collapse）
  健康 (lr=0.1)  最终平均熵=0.037 (满熵=1.386) 命中率=1.000
     熵曲线 █▆▃▂▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁  ← 平滑下降到近 0（学完才确定性）
  塌缩 (lr=8)  最终平均熵=0.003 命中率=0.375  ← 熵≈0=已 one-hot, ∇logπ→0, 被锁死
     熵曲线 █▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁  ← 一步砸到地板（探索前就锁死）
```

两条熵曲线对比就是整个故事。健康的 lr=0.1：熵从满熵 1.386 **平滑下降**到 0.037——策略边探索边收敛，等它对每个状态都试过几次、确认了最优动作，才慢慢变确定。命中率 1.000。

塌缩的 lr=8：熵**一步砸到地板**（0.003）。第一次更新就因为步长太大把某个动作的概率推到接近 1，于是 `(1−π)→0`、`−π→0`，梯度归零——回到第一节埋的钩子。一旦锁死，哪怕锁的是**错动作**，也再没有梯度能把它掰回来。命中率从 1.000 崩到 0.375（8 个状态里有 5 个锁在了错动作上）。

为什么是「探索前」这三个字最致命：策略梯度是 on-policy（同策略，只能从当前策略采到的动作里学）。如果你在试过所有动作之前就把概率压到某一个上，那些没试过的动作以后被采样的概率趋近 0，你**永远不会知道它们更好**。这跟监督学习的 lr 过大完全不同——监督学习 lr 大了是震荡 / 发散，调小就好；策略梯度 lr 大了是**信息性死亡**，你把探索的可能性本身杀掉了，调小也救不回已经锁死的那次运行。

这也是为什么 PPO 要用 clip 把每步策略更新的幅度硬限制住——它本质上是在防止这种「一步跨太大导致不可逆锁死」。clip 不是为了精度，是为了不让策略在一步之内跳出可恢复的范围。第 04 章细讲。

## 四、失败模式 B：奖励尺度大 + 不归一化 → 步长被尺度绑架

第二个失败模式和 A 同根：步长正比于奖励量级。奖励尺度一大，单步就过冲，锁死。但触发器从「lr 太大」换成了「奖励数值太大」。

固定 lr=0.5，扫奖励缩放倍数 rewardScale，对比归一化开 / 关，实测：

```
[4] 失败模式 B — 奖励尺度大 + 不归一化 → 单步过冲锁死（步长∝奖励量级）
  rewardScale=    1 | 不归一 命中率=1.000 | 归一 命中率=0.875
  rewardScale=   10 | 不归一 命中率=0.750 | 归一 命中率=0.875
  rewardScale=  100 | 不归一 命中率=0.250 | 归一 命中率=0.750
  rewardScale= 1000 | 不归一 命中率=0.250 | 归一 命中率=0.750
  rewardScale=10000 | 不归一 命中率=0.250 | 归一 命中率=0.750
```

不归一化那列：命中率随尺度**单调垮塌**，1.000 → 0.250（0.25 就是 4 动作随机猜的水平，等于完全没学）。机制还是锁死——`θ += lr·advantage·∇logπ`，advantage 里带着被放大的奖励，scale=100 时第一步就把策略推成 one-hot。

归一化那列：命中率对尺度**几乎不变**（0.875 / 0.875 / 0.750 / 0.750 / 0.750）。这就是 scale-invariant（尺度不变性）——奖励放大 1 万倍，训练行为基本不变。看归一化怎么做到的（`stage03-reinforce.ts`）：

```ts
if (cfg.normalizeAdv) {
  advStats.push(advantage);
  // Divide by running std (+ε for cold-start zero). This makes the step
  // scale-invariant: scaling all rewards scales advantage and std together, so
  // the ratio — and the step — is unchanged.
  const std = advStats.std;
  if (std > 1e-8) advantage = advantage / std;
}
```

道理很简单：advantage 除以它自己的运行标准差。你把所有奖励乘 100，advantage 和它的 std 同时乘 100，**比值不变**——所以有效步长不变。这就是归一化在难任务上不可省的原因：你通常不知道 reward model 输出的数值尺度（不同 RM、不同 prompt 分布尺度都不同），归一化让你不必为每个尺度重调 lr。注意它和 baseline 是**正交**的两件事：baseline 管「减偏移」（第二节），归一化管「除尺度」。PPO 里两者都在（advantage 减 critic baseline + batch 内归一化），不是二选一。

这里要做一个**诚实更正**——很多人（包括我早期）会以为大奖励会让 θ 溢出成 NaN。不会。回到第一节 softmax 的 max-subtraction：它保证 exp 的参数永远 ≤0，θ 再大也不会算术爆炸。

> **诚实更正**：softmax 的减 max 让 θ 不会真的溢出成 NaN；伤害是「过早锁死」而非「算术爆炸」。命中率垮塌不是因为出现了 NaN，是因为大尺度让首步把策略一把推成 one-hot。如果你只防 NaN（加个 clip 防溢出）而不归一化,问题照样存在——这是个容易因为「程序没崩」就误判已修复的坑。

A 和 B 共享同一机制：**单步过大 → 探索前锁成 one-hot → 锁错就学不动**。A 的过大来自 lr，B 来自奖励尺度。PPO 的 clip 治 A 那一侧（限步长），advantage 归一化治 B 那一侧（去尺度依赖），两者一起构成了策略梯度能在真实任务上稳定跑的地基。

## 五、前沿钩子：GRPO 为什么敢扔掉整个 critic ⚡

回到第二节那个试金石：baseline 是个「期望意义下的零」,**任何不依赖动作 a 的量都能当 baseline，都不引入偏差**。这句话里藏着一个激进的工程选择。

PPO 的 baseline 是一个**学出来的 critic 网络** V(s)——和 policy 同等规模的第二个神经网络，要额外的前向 / 反向 / 显存 / 调参。在 LLM 后训练里这意味着你同时扛着两个百亿参数模型，成本几乎翻倍，而且 critic 本身难训（value 估不准时 advantage 带噪，反而坑 policy）。

GRPO（Group Relative Policy Optimization，DeepSeek 在数学 / 推理 RL 里用的方法，第 08 章详解）的洞察就是直接把第二节那个证明用到极致：**既然任何不依赖动作的量都是合法 baseline，那我何必学一个 critic？同一道题（同一个 prompt）采样一组 G 个答案，用这组答案奖励的组内平均当 baseline 不就行了？** 对第 i 个答案：

```
A_i = (r_i − mean(r_1..r_G)) / std(r_1..r_G)
```

这个组内平均不依赖于「这一条具体选了哪个 token」（它对整组求平均），所以是合法 baseline，**不引入偏差**——这正是第二节那个 `E[b·∇logπ]=0` 的直接推论。而你注意到没有：它同时也是第四节的归一化（除以组内 std）！GRPO 把 baseline 和归一化用**同一组样本**一次性搞定，代价是每个 prompt 要多采几个答案（G 通常 4~16）——但采样比训练一整个 critic 网络便宜太多。理解了 baseline 的本质，你就能看懂 GRPO 为什么敢这么省：它不是发明了新数学，是把「baseline 只需不依赖动作」这个老定理的自由度榨干了。

那为什么这不是免费午餐、为什么还没有「通用解」⚡——**这是目前正在研究、无通用结论的开放问题**：

- **组内 baseline 的方差比 critic 大还是小,没有定论。** critic 用一个网络对所有 prompt 摊销了估计，组内平均只用当前这 G 个样本，G 小的时候 baseline 自己就很抖。到底哪个方差小，取决于任务、G、reward 形态——没有一刀切答案。
- **G 取多少是经验值。** G 太小 baseline 不准，G 太大采样成本线性涨。最优 G 随任务变，目前靠扫。
- **奖励全对 / 全错时组内 std→0,advantage 退化。** 一道题如果采样的 G 个答案奖励全一样（推理任务里「全对」或「全错」很常见），组内 std 接近 0，归一化要么爆要么把 advantage 抹平——这个退化的稳健处理目前各家实现不一,没有公认最优方案。

换句话说：「扔掉 critic 用组基线」是一个被验证有效、但远未被理论闭合的方向。它能成立，根基就是你这章学的——baseline 不引入偏差只降方差。能不能在所有任务上都比 PPO 的 critic 更好,**目前无通用解,正在研究**。

## 小结：你这章造好的三个零件

把这章拆成可迁移的事实（不是 toy 的具体命中率）：

1. **策略梯度无偏但高方差。** `∇E[R]=E[R·∇logπ]` 期望方向永远对，但单样本估计的方差是它在真实任务上死活的决定因素。baseline / 归一化都不改无偏性,只压方差 / 稳步长。
2. **baseline 的收益随奖励偏移增长。** toy 零均值上近乎无用（方差比 1.16×），真实 RLHF 全正奖励是大偏移，不减基线步长被偏移项主导 → 学废（实测命中率 0.125 vs 1.000）。这就是 baseline 在后训练里必需的原因。
3. **归一化保证 scale-invariant。** 奖励放大 1 万倍命中率几乎不变，让你不必为每个 reward 尺度重调 lr。和 baseline 正交，PPO / GRPO 两者都要。
4. **两个失败模式同根：** 单步过大 → 探索前锁成 one-hot → 锁错就学不动。lr 触发的是 A，尺度触发的是 B。这正是 PPO 用 clip 限步长、用归一化去尺度的动机。

你现在手里有了 `log π`、`∇logπ`、`advantage` 三个零件,以及对「方差才是敌人」的肌肉记忆。第 04 章 PPO 会把这三个零件装进一个真正能在大模型上稳定跑的更新里——核心就是用 clip 解决本章两个失败模式暴露的「步长不可控」问题。第 08 章 GRPO 则会回到本章第五节,把 critic 整个扔掉。本章是这一切的地基。

> 诚实标注：本章所有命中率 / 方差数字来自 8 状态 × 4 动作的单步 bandit toy（`core/envs.ts::makeContextualBandit`，lr / steps / seed 固定保证「唯一变量是开关」）。toy 绝对值是乐观的——可迁移的是相对趋势（方差比随 offset 增长、命中率随尺度垮塌）与机制（锁死、scale-invariance），不是具体数值。在真实 LLM 后训练上这些效应只会更剧烈。
