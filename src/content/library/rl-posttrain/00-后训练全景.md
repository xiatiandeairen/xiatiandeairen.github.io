先把一句话钉死：后训练（post-training，预训练之后让模型对齐人类意图的所有手段）不是一堆并列算法，而是同一个动作的七种变体——**用奖励改进策略**。区别只在两处：奖励从哪来、策略被什么拴住。你要是把 SFT、RM、RLHF、DPO、GRPO 当成五门要分别背的课，面试和实战都会被它们的失败模式打穿；把它们当成「奖励来源 × 约束方式」这张二维表上的几个点，整条线就塌缩成一句话。

这一点不是我编的，是配套代码 `examples/rl-posttrain-from-scratch/src/stage00-landscape.ts` 第 0 节直接打印出来的统一框架：

```text
bandit      : 奖励=环境真值      策略=动作分布   无信用分配
REINFORCE   : 奖励=环境真值      策略梯度        高方差
PPO         : 奖励=环境真值      裁剪比率 + 价值基线
RLHF        : 奖励=学到的 RM     PPO + KL 拴住参考策略
DPO         : 奖励=隐式(偏好对)  无 RM、无采样、闭式损失
GRPO        : 奖励=RM/可验证     组内相对优势、去掉价值网络
RLVR        : 奖励=可验证(对/错) 推理任务、奖励无噪声
```

读这张表的正确方式：从上往下，奖励信号一步步「变脏又变省」——从环境给的干净真值（bandit/PPO），到要花钱训出来的代理 RM（RLHF），到干脆不训 RM 把偏好对直接塞进损失（DPO），最后回到推理任务里那种对/错二值、零噪声的可验证奖励（RLVR）。这条曲线就是 2024 年以来对齐主力从 PPO 漂向 DPO 系与 GRPO 系的物理动因，本章末尾会算这笔成本账。

## 为什么预训练完还要 RL：模仿的天花板

预训练给你的是一个会续写互联网的概率机器，它最大化的是「下一个 token 的似然」。这个目标和「给出人类想要的回答」之间有一道鸿沟：互联网上既有好答案也有烂答案,似然最大化对两者一视同仁。SFT（supervised fine-tuning，监督微调，拿人写的黄金答案做模仿学习）能把分布拉向「好答案」，但它的天花板是死的——**模仿学习只能逼近示范数据,无法超过示范者**。示范里没有的好回答,SFT 学不到;示范里偶尔出现的烂模式,SFT 照单全收。

RL 解决的正是这个：它不要求你给出黄金答案,只要求你能**对两个答案说出哪个更好**。判别比生成容易得多——你未必写得出最好的代码,但你能看出两段代码哪段更对。后训练这条线的全部价值,就是把「人类的判别能力」灌进模型,让模型可以探索到示范数据之外、但人类会更喜欢的区域。

代价是:一旦从「模仿固定答案」转向「优化某个奖励」,你立刻引入两个会反复咬你的问题——奖励本身可能是错的(reward hacking),以及优化得太猛会把模型推到参考策略从没去过的胡言乱语区(KL 失控)。这两条是全书暗线,下面用真实数字把它们摆上台面。

### ✦ 后训练岗面的不是「你会不会调 PPO」

说句行话:后训练面试真正区分人的问题,从来不是「PPO 的 clip 系数设多少」,而是「RM、DPO、GRPO 各自怎么坏、边界在哪」。会调参的人很多,能讲清每个机制的失败模式与适用边界的人少。本书的结构就是冲着这个设计的——**每一章钉死一个失败模式**:第 5 章讲 RM 怎么被标注噪声封顶,第 6 章讲 RLHF 怎么 reward hacking、KL 怎么救,第 7 章讲 DPO 在分布偏移下怎么崩,第 8 章讲 GRPO 去掉价值网络后组内优势怎么塌。这一章先把判定它们好坏的三把尺子立起来。

## 三把尺子:胜率、噪声敏感度、KL

全书反复用三个量判定一个后训练方法到底有没有用。关键是——stage00 把这三个量**真算了一遍**,不是 assert 一个写死的数字。先看为什么必须真算:如果连「会选 vs 乱选」都区分不出,后面任何漂亮的训练曲线都不可信。

### 度量 (a) 胜率:有没有真的变好

胜率(win-rate)的定义:在一个隐藏的真值判官(`trueRewardFn`,代码里扮演「上帝级人类」)下,新策略 vs 参考策略,谁的回答被判更好的比例。代码用三个 chooser 在同一组候选上比,确保比的是「会不会选」而不是「谁抽到的候选运气好」:

```ts
// stage00-landscape.ts, evalWinRate()
const refScore    = (e) => e.trueScores[0];                 // 参考:永远取候选 #0
const randomScore = (e) => e.trueScores[sampleCategorical(...)]; // 随机:均匀乱选
const goldenScore = (e) => Math.max(...e.trueScores);       // oracle:选真最优
```

stage00 实测输出(N=4 候选、1000 prompt、seed 99):

```text
随机策略 vs 参考 胜率 : 0.475  (不学习 ⇒ 必须≈0.500，平局算0.5)
黄金策略 vs 参考 胜率 : 0.857  (oracle 选真最优 ⇒ 远高于0.5)
```

读法:不学习的随机策略落在 **0.475**(≈0.5,噪声范围内),这是地板——任何后训练方法跑出来低于 0.5 就是负优化。oracle 落在 **0.857**(不是 1.0,因为很多 prompt 上候选本来就差不多,黄金和参考的真值差为 0,算平局),这是天花板。**0.475 到 0.857 之间这段空间,就是后训练能赚到的全部改进**。后续每一个 stage 都要在这条线上证明自己 > 0.5。

为什么不直接用「平均奖励」?因为绝对奖励值在合成世界和真实世界不可比,而「相对参考策略的胜率」是可迁移的——这也是真实 RLHF 用 pairwise 人类评测、而非绝对打分做主评的原因。

### 度量 (b) 噪声敏感度:RM 的天花板,optimizer 救不回

这是全书最反直觉的一条,也是 ✦ 失败模式系列的第一块基石。所有基于奖励的方法(RM/RLHF/DPO/GRPO)都被一个东西封顶:**偏好标签本身有多干净**。stage00 直接量「可分性」——标签里 `trueReward(chosen) > trueReward(rejected)` 的比例,不训任何 RM,纯量标签质量:

```text
0%  注入翻转 : 可分性=0.763  实际翻转占比=0.000  (<1.0：BT 抽样在近似对上会标反)
30% 注入翻转 : 可分性=0.610  实际翻转占比=0.301  (噪声单调砸可分性)
可分性下降幅度 : 0.154  ← 这就是 RM 准确率的天花板，optimizer 再强也救不回标签丢掉的信号。
```

两个细节值得停下来想。第一,**0% 注入噪声时可分性也只有 0.763,不是 1.0**。为什么?看偏好世界的标注代码(`src/core/preference.ts`):

```ts
// generatePairs(): Bradley–Terry labeling
const pAOverB = 1 / (1 + Math.exp(-(ra - rb)));  // a 被偏好的概率 = σ(rewardA - rewardB)
let aIsChosen = rng() < pAOverB;                  // 不是永远选高的那个!
```

它没有「永远把真值高的标为 chosen」,而是按 Bradley–Terry 模型(用奖励差的 sigmoid 当偏好概率,模拟人类对接近的两项会标反)采样。这是**故意的**——如果标签零噪声,RM 准确率会是无意义的 100%。真实人类对接近的两个回答本来就会分歧,这个 0.763 才是诚实的起点。

第二,30% 注入翻转后可分性掉到 **0.610**,意味着约 1/3 的监督信号方向是反的。这就是失败模式:**下游的 RLHF/DPO 会照着错标签认真学,而且 optimizer 越强,学错得越彻底**。这条规律在第 5 章(RM)和第 7 章(DPO)会用真实训练曲线复现——你会看到准确率怎么随噪声单调下滑,以及为什么「换更大的模型/更猛的优化器」完全救不回标签丢掉的信号。garbage in, garbage out 在这里不是口号,是 0.763 → 0.610 这个可量化的塌陷。

### 度量 (c) KL-to-ref:拴住策略的那根绳

KL 散度(Kullback–Leibler divergence,衡量两个概率分布差多远的信息距离)在后训练里扮演「拴绳」:KL(π_new ‖ π_ref) 越大,新策略离预训练/SFT 得到的参考策略越远。RLHF/DPO 都在损失里加这一项,防止策略为了刷奖励漂到胡言乱语区。stage00 在三个手搓策略上算真值:

```text
参考策略 π_ref          : [0.402, 0.329, 0.269]
小漂移 π_A  KL(A‖ref)   = 0.1083   策略=[0.629, 0.231, 0.140]
大漂移 π_B  KL(B‖ref)   = 0.7320   策略=[0.960, 0.029, 0.011]
退化   π_C  KL(C‖ref)   = 0.9119         策略=[1.000, 0.000, 0.000] ← 在 ref 有概率处给了硬0
方向不对称：KL(ref‖C)   = ∞ (灾难漂移)    (反方向有限)
```

注意 π_C:它坍缩成「只说一句话」、把其他动作的概率全杀成**硬 0**。RLHF 用的 KL(C‖ref) 在这个方向上仍是有限值(0.9119,因为 C 的零概率项不贡献),但只要再大它一点就把策略钉死成单点了——这正是 KL 拴绳要在它彻底退化前拉住的灾难。π_A(0.108)和 π_B(0.732)的对比则给你直觉:同样往动作 0 集中,B 集中得多,KL 就大得多。

还有一个工程上要命的细节:**KL 是非对称的**。代码里 `klCategorical(polC, ref)`(即 KL(C‖ref))是有限的 0.9119,但反方向 `klCategorical(ref, polC)`(即 KL(ref‖C))是 ∞——因为 ref 在 C 给了硬 0 的动作上仍有概率,`log(ref/0)` 炸成无穷。RLHF 用的是 KL(new ‖ ref) 这个方向:它重罚「新策略往参考的零概率区漂」(new 在 ref≈0 处堆概率→KL 飙升),却不会因为新策略主动收窄支撑集就炸成 ∞。如果用反方向 KL(ref ‖ new),惩罚结构完全不同——任何收窄都会被罚到无穷。这个不对称性在第 6 章实现 PPO 的 KL 惩罚项、第 7 章推导 DPO 闭式损失时都会再出现,届时还会撞上「k1 vs k3 KL 估计量」的方差问题(`src/core/probability.ts::klFromLogprobs`,这里 stage00 已经埋了一行 `KLk1=-0.8573 KLk3=0.4995`——单样本下两个估计量差异巨大,只有取期望才等于真 KL)。

## 实验台的设计哲学:为什么离线 + 合成 + 确定性

讲到这要解释全书 `core/` 实验台为什么长这样,因为这决定了你能信哪些数字、不能信哪些。三条设计原则:

**1. 离线 + 合成偏好世界,但有 ground truth。** 真实 RLHF 的根本痛点是:你永远不知道「学到的」离「真实的」有多远——因为真实人类偏好不可计算。stage00 的偏好世界反过来:它定义了一个可计算的 `trueRewardFn` 当上帝,所有偏好标签都从它派生(还带可调噪声)。于是「学到的 RM vs 真实奖励」「DPO 策略 vs 真最优」永远可量化。这是合成世界唯一、但决定性的好处。代价我后面会诚实交代。

**2. 确定性 + 逐位可复现。** 整个实验台由种子 PRNG(伪随机数生成器 `src/core/rng.ts::mulberry32`)驱动,任何地方碰了 `Math.random` 都会破坏「同种子同曲线」的保证。stage00 把同一个 corePass 跑两遍来当烟雾测试:

```text
seed 42 (1st): gauss=-0.9562,-0.2730,-1.8416 | ... | KLexact=0.4770 | KLk1=-0.8573 KLk3=0.4995
seed 42 (2nd): gauss=-0.9562,-0.2730,-1.8416 | ... | KLexact=0.4770 | KLk1=-0.8573 KLk3=0.4995
逐位相同      : ✅ 是
seed 43       : gauss=-0.0032,1.0509,-0.0824 | ...
换种子即不同  : ✅ 是
```

为什么对教学这么重要?因为 RL 训练曲线天生抖,如果你换种子就得到不同结论,你根本分不清「这是算法的真效果」还是「这次种子运气好」。逐位可复现 = 你看到的每条曲线差异都是算法差异,不是随机噪声。这也是我对 N=1 实验的态度——单次跑出漂亮结果不算数,本书每个数字都标了种子,你可以自己换种子复跑看趋势稳不稳。

**3. 实验台是真的,不是摆设。** stage00 第 2 节拿一个随机策略在 5 臂 bandit(多臂老虎机,最简单的 RL 环境,只有动作没有状态)上跑 200 步,真测累计 regret(遗憾值,相对已知最优臂少赚的总和):

```text
真臂均值     : [2.76, -0.07, -0.95, 0.08, -0.14]   最优臂 #0 (均值 2.76)
随机策略均奖 : 0.3543    累计 regret : 483.95  (随机策略≈线性增长)
```

随机策略的 regret 线性增长(sparkline 是一条直上的斜线),因为它从不利用学到的信息。这条线是后面所有「会学习」的策略要打败的基线——第 1 章的 bandit、第 3 章的 REINFORCE 都从这里起跑。

## ⚡ 前沿:对齐主力正在从 PPO 漂走,这笔成本账是本书暗线

回到开头那张表。2024 年起,前沿模型的对齐主力肉眼可见地从 RLHF-with-PPO 漂向两个方向:DPO 系(直接偏好优化,跳过 RM 和在线采样)和 GRPO 系(组内相对优势 + 可验证奖励,去掉价值网络)。理解这次迁移的成本账,是读这本书的暗线。

成本账大致是这样:RLHF-with-PPO 要同时维护**四个模型**(策略、参考、奖励模型、价值网络)在线滚动采样,工程复杂、显存吃紧、训练不稳。DPO 把奖励模型用数学闭式吸收进损失,省掉 RM 训练和在线采样这「后训练之昂贵中段」;GRPO 用组内多个采样的相对排名当优势,省掉价值网络。在可验证奖励(RLVR,推理任务里答案对/错二值、零噪声)的场景下,GRPO 系尤其香——奖励干净到根本不需要训 RM。

但这里必须标注:**这次迁移没有通用解,谁在什么场景胜出仍在研究中**。DPO 省掉的在线采样,在分布偏移大的任务上会反咬(策略漂到训练偏好对没覆盖的区域就失准,第 7 章会撞这个);GRPO 去掉价值网络换来的方差,在奖励稀疏时会放大;而 PPO 那套虽然贵,在需要细粒度信用分配(credit assignment,把最终奖励正确归因到中间步骤)的长程任务上仍有它的位置。「什么时候该用哪个」目前没有干净的判定规则——这恰恰是后训练这个方向还活着、还值得深入的原因。本书不卖「DPO 取代一切」这种 survey 式的干净叙事,而是带你把每个方法的失败模式亲手跑坏一遍,让你自己长出判断力。

## 诚实声明:这些数字偏乐观,可迁移的是趋势

最后泼一盆冷水,这盆水全书统一适用。stage00 的合成偏好世界远比真实 RLHF 简单:奖励可计算、噪声可控、动作空间是个位数。所以**绝对数值偏乐观**——真实世界里 0.857 的 oracle 胜率、0.763 的干净可分性都达不到。

可迁移的不是绝对值,是相对趋势,这三条贯穿全书:

- 胜率 > 0.5 ⇒ 后训练真的带来改进(0.5 是不学习的下界)
- 标注噪声 ↑ ⇒ 偏好可分性 ↓ ⇒ RM 上限 ↓ ⇒ 下游照着噪声学
- KL(new ‖ ref) ↑ ⇒ 策略漂移越远;在参考有概率处给硬 0 → ∞ 是要拴住的灾难

这三个量在 stage00 里全是实时算出来的,没有一个硬编码 assert——你换种子复跑,数字会变,但这三条不等式不会翻。带着这张地图和这三把尺子,我们从第 1 章的多臂 bandit 开始,一步步把 SFT → RM → RLHF → DPO → GRPO 这条线,连同每一步的失败模式,亲手搭起来。
