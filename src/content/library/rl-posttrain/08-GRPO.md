第 4 章你给 PPO 配了两个网络：一个 policy，一个 critic（价值网络，预测「从这个状态往后大概能拿多少奖励」）。优势 = 奖励 - critic 的预测，critic 估得准，优势就低方差，更新就稳。代价是这个 critic 在真实 LLM 里跟策略同量级——你要再喂一个跟主模型一样大的模型进显存，还要单独训它、单独调它的学习率、单独担心它训崩。GRPO（Group Relative Policy Optimization，组相对策略优化，DeepSeek 提出）的一句话主张是：**这个网络可以不要**。它不学 baseline，它现采一组答案，用这组自己的均值/标准差当 baseline。

把这件事钉死成一个判据，看配套仓 `examples/rl-posttrain-from-scratch/src/stage08-grpo.ts` 跑出来的真实头部输出：

```
menu(真奖励): #0[len6,kw1]=-0.80  #1[len12,kw3]=1.73  #2[len24,kw3]=-5.47  #3[len40,kw2]=-37.79
真最优=#1（golden length=12）。优势 = (奖励 - 基线)，PPO 基线=学到的 critic V(s)，GRPO 基线=组内均值/标准差。
[1] GRPO vs PPO（同种子 / 同 400 迭代 / 每步采样预算=8）
  PPO  网络数=2(policy+critic) 参数=5  选中#1 概率=1.000
  GRPO 网络数=1(policy)        参数=4  选中#1 概率=0.999
```

同一个种子、同样 400 次迭代、每步同样 8 个采样预算，两个方法都把概率几乎全压到 `#1`（真最优答案，长度 12、命中 3 个关键词）。结果一样，但 GRPO 网络数从 2 降到 1、参数从 5 降到 4。这个 toy 里只省 20%（critic 在这里退化成一个标量），意义在相对趋势：**真实 LLM 里 critic 跟策略同量级，省的是一整个大模型的显存和训练开销**。这是 GRPO 在 2024–2025 成为推理模型训练主力的第一现实理由——不是它学得更好，是它学得一样好却便宜一半。

## 只换一行：GRPO 是「换了 baseline 的 PPO」

先破除一个误解：GRPO 不是新的优化器，它跟 PPO 共用完全相同的 clipped surrogate（裁剪过的目标函数）更新。`stage08-grpo.ts` 里 PPO 和 GRPO 调的是同一个 `clippedStep`，差别只在传进去的 `advantage` 是怎么算的：

```ts
// clippedStep — PPO 与 GRPO 共用的更新，唯一区别是传进来的 advantage 怎么来
const ratio = Math.exp(logpNew - logpOld);
const unclipped = ratio * advantage;
const clipped = Math.max(1 - clip, Math.min(1 + clip, ratio)) * advantage;
const clipActive = clip > 0 && clipped < unclipped;
const gradScale = clipActive ? 0 : advantage * ratio;
```

PPO 分支算优势：

```ts
// PPO: critic 是学出来的标量 V(s)，advantage = reward - V(s)
const advs = rolls.map((x) => x.r - critic);
// ...更新策略后，再把 critic 回归到这批的平均奖励
critic += criticLr * (batchMeanR - critic);
```

GRPO 分支算优势——这是整章的核心，也是唯一真正不同的地方：

```ts
// GRPO: 没有 critic。mean & std 只在「这一组」上算
const mean = group.reduce((s, x) => s + x.r, 0) / groupSize;
const variance = group.reduce((s, x) => s + (x.r - mean) ** 2, 0) / groupSize;
const std = Math.sqrt(variance);
const eps = 1e-6;
const advs = group.map((x) => (x.r - mean) / (std + eps));
```

PPO 那行 `critic += ...` 的回归更新，在 GRPO 里整个消失了，连带消失的是 critic 这个网络、它的学习率 `criticLr`、它自己的训练不稳定性。GRPO 的 baseline 是**当场算的、零参数的蒙特卡洛估计**：这一组答案的平均分。

### ✦ 这就是第 3 章 REINFORCE-with-baseline 的小批量版

如果你读过第 3 章会立刻认出来：REINFORCE 的梯度是 `(奖励 - baseline) × ∇logπ`，加 baseline 纯粹为了降方差、不改变期望梯度的方向。第 3 章那个 baseline 是「历史平均奖励」的滑动估计。GRPO 把「历史平均」换成「此刻这一组的平均」——**它就是 REINFORCE-with-baseline，只不过 baseline 从一个全局滑动量变成了每步现采一组、组内现算**。PPO 的 critic 是「学一个函数来预测 baseline」，GRPO 是「干脆采几个样本直接平均出 baseline」。一个是参数化估计，一个是蒙特卡洛估计，估的是同一个东西。把握住这条线，你就知道 GRPO 不是凭空冒出来的新算法，它是 RL 算法谱系里「baseline 怎么来」这一维度上的一个朴素选择。

这也顺带回答了一个常被问的问题：**为什么 GRPO 对「只有结果奖励」的场景特别合适。** critic 要学的是「从中间状态往后能拿多少奖励」（value function），可你在 LLM 生成里根本没有可靠的中间奖励信号——奖励只在整条回答生成完、给奖励模型或验证器打分时才出现一次。让 critic 去拟合一个只在末尾有信号的 value function 本就吃力。GRPO 绕开了这个难题：它不需要估中间价值，它只需要「同一个 prompt 的几个完整回答里，哪个相对好」。结果奖励场景下，GRPO 是更自然的形状。

## 组内归一化为何抗 reward 尺度漂移

看 `stage08-grpo.ts` 那行优势公式：`(x.r - mean) / (std + eps)`。分子减掉组均值，分母除以组标准差——这是标准的 z-score 归一化。它带来一个 PPO 没有的好性质：**优势的量纲恒定在 ±1 附近，跟奖励本身的绝对尺度无关**。

为什么这要命？奖励模型（RM）的输出在训练过程中尺度会漂。今天 RM 给好答案打 2.0、坏答案打 -1.0；过几千步重新校准后变成好答案 200、坏答案 -100。对 PPO 来说，优势 = 奖励 - V(s) 的绝对值跟着放大 100 倍，等于偷偷把学习率放大 100 倍，更新步长失控。你得反复调 critic 学习率、调 reward 的 clip 范围去压它。GRPO 因为除了 std，无论 RM 输出是 ±1 还是 ±200，归一化后优势都回到 ±1 量级——**奖励整体平移或缩放，对组内 z-score 完全没影响**。这把「奖励尺度调参」这件烦事从训练里删掉了。

代价藏在分母里：std 一旦塌成 0，整个机制就崩。这是下一节的失败模式。

## 组大小 G：新冒出来的旋钮

天下没有免费的 baseline。PPO 用一个学出来的 critic 给所有样本提供 baseline，每步采几个样本是另一回事。GRPO 把 baseline 和采样绑死了：**baseline 的质量直接由组大小 G 决定**——组就那么几个样本，均值和标准差就靠它们估。G 太小，baseline 是噪声；G 太大，每步采样预算翻倍。这是 GRPO 引入的新旋钮，PPO 没有。

`stage08-grpo.ts` 扫了 G ∈ {2,4,8,16}，同种子：

```
[2] 组大小 G 对优势方差与收敛的影响（GRPO，同种子）
  组G | 优势方差(均) | 选中项 | 该项概率 | 末期期望真奖励
   2  |    0.120     |   #1   |  0.991   |    1.617
   4  |    0.087     |   #1   |  0.998   |    1.706
   8  |    0.065     |   #1   |  0.999   |    1.718
  16  |    0.050     |   #1   |  1.000   |    1.728
```

趋势是单调的：G 从 2 涨到 16，优势方差从 0.120 降到 0.050，末期选中真最优 `#1` 的概率从 0.991 爬到 1.000，期望真奖励从 1.617 逼近上限 1.73。读这个表要注意一个陷阱：**优势方差降低不等于「学得少了」**。因为 std 归一化已经把优势量纲钉死在 ±1，这里的方差降低反映的是 baseline 估计本身的噪声在下降——组越大，组均值/标准差越接近策略的真实分布，baseline 越准。方差降的是估计误差，不是学习信号。

实践含义：G 不是越大越好，是「够用就行」。组大一倍，每步生成成本就翻倍——在真实 LLM 里每个样本是一整段自回归生成，这是实打实的算力。DeepSeek 实践里 G 取十几到几十量级，是「baseline 够稳」和「采样预算可承受」之间的折中。这个旋钮你躲不掉，得自己掂量。

## 失败模式一：G=2，符号被运气翻转

把 G 推到极端小，失败模式就裸露出来。`stage08-grpo.ts` 直接对比了 G=2 和 G=16：

```
[3a] 失败模式：G=2 双样本基线噪声大
  G=2  选中#1 概率=0.991 优势方差=0.120
  G=16 选中#1 概率=1.000 优势方差=0.050
```

为什么 G=2 危险，得看机制而不只是看数字。G=2 时 baseline 只由两个样本估计。假设这一组恰好抽到两个都不怎么样的答案——比如 `#0`（-0.80）和 `#2`（-5.47）——那么组均值是 -3.1，于是 `#0` 相对组均值是「正优势」，策略会被推着**更倾向那个其实并不好的答案**。换句话说，单次抽样的运气可以让一个好答案被算成负优势、一个差答案被算成正优势，**优势的符号被运气翻转**。G 大时这种翻转被平均掉，G=2 时它频繁发生，于是收敛更慢、末期置信度更低（0.991 vs 1.000）。这不是 bug，是双样本估计的固有噪声。它告诉你 G 有个下界，低于它 baseline 就不可信。

## 失败模式二：组内奖励全相等，优势归零，策略冻结

这是 GRPO 结构性的阿喀琉斯之踵，也是 `stage08-grpo.ts` 里我没有藏起来、专门用 `degenerate` 分支暴露的崩溃：

```
[3b] 失败模式：组内奖励全相等 → 优势归零 → 不学习
  组内全选同一动作(无区分度): 起始期望真奖励=-10.5804 → 末期=-10.5804  位移=2.19e-7
  末期策略 #0:0.25 #1:0.25 #2:0.25 #3:0.25（仍是初始均匀 0.25，完全没动）
```

400 次迭代，策略一动没动——四个动作还是初始的均匀 0.25，期望真奖励位移只有 2.19e-7（数值噪声级别）。根因在那行优势公式里看得一清二楚：当组内每个答案拿到**完全相同**的奖励时，组均值就等于这个奖励本身，组标准差 std = 0。于是每个优势 = `(r - mean) / (std + eps)` = `0 / 1e-6` = 0。所有优势归零，梯度归零，策略冻结。

注意代码里 `eps = 1e-6` 是用来防 0/0 崩溃的，**但它不会把 std 抬起来掩盖问题**——分子本来就是 0，eps 再小，0 除以任何正数还是 0。这是我故意不去 clamp std 的原因：这个塌缩本身就是要讲的课。GRPO 的学习信号 **100% 来自「组内有奖励差异」**。组里大家分数一样高（或一样低），它就没有任何东西可学。PPO 在这里不会完全冻死——它的 critic 提供的是一个跨步学习的绝对 baseline，即便单批没有内部差异，critic 和奖励之间仍可能有差值。GRPO 把 baseline 锁死在「组内」，就把命运也锁死在「组内必须有对比」上。

这条限制不是抽象的。想象你训一个数学推理模型，某个 prompt 太简单，采样 16 个回答全做对、全拿满分——这一组对 GRPO 是零信息，白采。或者反过来太难，全做错全 0 分，同样零梯度。**GRPO 真正在学的，是那些「组内有的对有的错」的 prompt**。这直接决定了你的奖励信号必须能真正区分回答好坏。

## ⚡ 前沿：可验证奖励是 GRPO 的事实标配，但「奖励从哪来」仍无通用解

把上一节的限制推到底，就到了 2024–2025 这波推理模型训练的核心配方。GRPO 是 DeepSeek-R1 等模型的训练主力，它配的不是传统奖励模型，而是 **RLVR（可验证奖励的 RL，Reinforcement Learning with Verifiable Rewards）**：答案能用程序自动判对错（数学题对答案、代码过单测），就用这个 0/1 信号当奖励。原因正是上一节：GRPO 要靠组内差异学习，而可验证奖励恰恰能在一组回答里干净地分出对错——有的过测试有的没过，组内 std 非零，baseline 有东西可算。这套「GRPO + 可验证奖励」组合在这两年成了事实标配，本章这个 `stage08-grpo.ts` 就是它最小的可跑实现。下一章会把奖励来源从合成 RM 换成真正的验证器，正面处理这个交接。

但要诚实：这个组合的前沿恰恰卡在「可验证」三个字上。数学和代码有自动判分，所以 RLVR 在这两个域跑得通。可绝大多数你真正想让模型变好的能力——写作质量、对话有用性、推理是否真的严谨而非碰巧答对——**没有可靠的自动验证器**。退回去用奖励模型，又会把奖励 hacking、reward 尺度漂移这些老问题带回来，而且 RM 自己也会被刷分。「如何为不可验证的任务构造既不能被刷分、又能在组内提供真实差异的奖励信号」，目前没有通用解，是正在被密集研究的开放问题。GRPO 把优化这一环做得又便宜又稳，反而让全部压力压到了「奖励从哪来」——而那一环，至今没人解决。

## 三方对比：PPO / GRPO / DPO 各自放弃了什么

收尾把第 4 章 PPO、第 7 章 DPO、本章 GRPO 摆在一起。三者都是「让策略偏向更好的回答」，区别在于它们各自砍掉了什么：

- **PPO**：留全套——奖励模型 + critic + 在线采样。最通用，能处理任意标量奖励，但最重：两个大模型 + critic 的训练不稳定 + 奖励尺度调参。
- **GRPO**：砍掉 critic，保留奖励模型/验证器 + 在线采样。用组内 z-score 当 baseline，省一个大模型、自动抗奖励尺度漂移。代价是每步要采一组（G 成了新旋钮）、组内无差异就学不动。
- **DPO**：连在线采样和奖励模型一起砍掉——见第 7 章，它把 RL 目标用数学等价改写成一个直接在「偏好对」上跑的监督式损失，离线、无采样、无 RM。代价是它吃的是固定的偏好数据集，没法像 GRPO 那样在训练中持续探索策略当前生成的新样本。

一句话记忆：PPO 是「学一个 critic 给 baseline」，GRPO 是「采一组样本现算 baseline」，DPO 是「把 baseline 和 RL 一起用代数消掉」。GRPO 卡在 PPO 的在线探索能力和 DPO 的轻量之间——保留了在线采样这个 PPO 的核心优势，又甩掉了 critic 这个 PPO 最重的包袱。这就是它在推理模型训练里胜出的位置。

最后照例一句诚实声明，照搬 `stage08-grpo.ts` 末尾那行：本章的奖励是合成 RM，绝对值是乐观的；**可迁移的是相对趋势——GRPO 省一个网络、优势方差随 G 下降、组内无差异即学不动**。这三条在真实 LLM 上同样成立，第 9 章我们用真验证器把它们再验一遍。
