先把这章要你记住的一句话钉死：**探索不是"偶尔随机一下"的小聪明，它是一笔你不交就永远拿不到最优解、交多了又白白浪费预算的税；唯一能告诉你这笔税交得值不值的，是 regret 曲线的形状，而且必须看多 seed 平均的形状，不是单次。** 你要是把 bandit 当成"概率论小练习"跳过去，到第 8 章 GRPO 看到一组 response 里探索塌缩、模型只会复读同一个套路答案时，就会发现那是这一章 F1 失败模式的 1:1 放大，只是那时候你已经烧了几千刀算力才看出来。

这一章只处理**最小的 RL 问题**：k 臂老虎机 (k-armed bandit，k 台老虎机你每次拉一台，每台吐钱的均值不同但你不知道)。它把 RL 砍到只剩一个决策——没有状态、没有"这一步影响下一步"、没有 credit assignment (功劳分配，把最终回报摊回到中间每一步)。RL 里所有让人头疼的东西在这里全消失了，**唯一剩下能搞砸的就是探索-利用的权衡**。你在这里建立的直觉——"不去试就永远不知道，这是个永久成本"——会原封不动地出现在一次 70B 模型的 RLHF 训练里。

本章对应的可跑代码是 `examples/rl-posttrain-from-scratch/src/stage01-bandit.ts`，它编译后纯离线运行，不连网不调任何 LLM，固定种子。下面引用的所有数字都来自它的真实输出（`npx tsx src/stage01-bandit.ts`）。

## 为什么 regret 是唯一诚实的尺

第一个要破除的直觉：用"总收益"衡量策略好坏是错的。总收益依赖每台老虎机的钱多钱少，把同一个策略放到吐钱多的 bandit 上数字就大，跟策略本身好不好没关系。

正确的尺是 **regret (后悔值)**：每一步，事后最优臂的均值减去你实际拉的那个臂的均值，累加起来。它衡量的不是"你赚了多少"，而是"**你因为不知道哪个臂最优而白白损失了多少**"。看 `src/core/metrics.ts` 里的定义：

```typescript
export function regret(bandit: Bandit, pulledArms: number[]): number[] {
  const best = bandit.trueMeans[bandit.optimalArm];
  const cumulative: number[] = [];
  let acc = 0;
  for (const arm of pulledArms) {
    acc += best - bandit.trueMeans[arm]; // per-pull regret 是均值差，不是带噪声的那次采样
    cumulative.push(acc);
  }
  return cumulative;
}
```

注意第 6 行那条注释：每一步的 regret 用的是**真均值的差**，不是这次实际吐出来的带噪声的钱。这是关键——如果你用实际收益算，运气好的一次随机拉中也会让 regret 看起来下降，曲线就成了噪声。用真均值差，曲线只反映"你的决策有多接近最优"，运气被剥离掉了。

而这条尺能成立，全靠一个 toy 环境才有的特权：**我们知道答案**。`src/core/envs.ts` 构造 bandit 时，每个臂的真均值是固定下来的，最优臂是哪个我们清清楚楚——但 agent 永远看不到（代码里那句 `Do not peek in agents!` 的注释不是玩笑，agent 一旦偷看真均值，整章的教训就成了骗局）。真实 RLHF 里你没有这个特权，没有"事后最优"可以减，所以 regret 退化成各种 proxy（reward model 分数、win-rate）。但 proxy 会被 hack（第 5 章细讲），而这里的 regret 不会——**这是你这辈子能拿到的最干净的一把尺，珍惜它，理解曲线形状该长什么样**。

### regret 曲线的两种形状：线性 vs 次线性

记住这个区分，它是判断一个探索策略生死的标准：

- **线性增长**：regret 每一步都在涨同样多。意味着这个策略**永远在交探索税**，或者干脆锁死在次优臂上交利用税。怎么都收不住。
- **次线性 (sub-linear)**：regret 涨着涨着曲线变缓。意味着策略**找到最优臂之后基本不再后悔**，探索成本是一次性付清的。

好策略和坏策略的分水岭就是这条曲线后半段是直的还是弯的。下面四个策略，你盯着它们的 ASCII 曲线看这一件事就够了。

## 四种策略同台

本章在**同一个 10 臂 bandit** 上跑四种策略，跨 24 个独立 seed 平均。同一个 bandit 是故意的——所有策略面对完全一样的题目，比较的就纯粹是策略本身，不是题目难易。这个 bandit 的真均值是：

```
10 臂真均值 [0.79, 1.12, -0.73, 0.63, -1.55, 1.56, -0.81, 0.49, -0.55, -0.74]
最优臂 #5(1.565)，与次优差距 0.448（差距越小越难分辨）
预算 1000 步 / arm；跨 24 个独立 seed 平均
```

差距 0.448 是刻意挑的（构造种子 3）：差距太大谁都能轻松找到最优臂，所有策略看起来都很棒，区分不出好坏；差距小到 0.448、又被单位方差的噪声盖住，策略之间才真正拉开档次。这是 toy 实验设计的一个普适原则——**题目要难到能暴露差异，但别难到谁都做不出来**。

四种策略的实测结果（直接来自 stage 输出）：


先别看哪个均 regret 最低，先看曲线形状。**ε-greedy 那条线全程匀速爬升，近乎一条直线**；Thompson 那条前段陡涨然后明显压平——它在前面快速付清探索成本，后面几乎不再后悔。这就是上面说的线性 vs 次线性，肉眼可辨。下面逐个拆。

### 策略一：ε-greedy —— 固定探索是永久的税

ε-greedy 是最朴素的想法：绝大多数时候拉当前估计最好的臂（利用），但以概率 ε 随机拉一个臂（探索）。看 `stage01-bandit.ts` 的实现，干净到只剩选臂逻辑：

```typescript
function epsilonGreedy(epsilon: number): Agent {
  return (bandit, steps, rng) => {
    const est = makeEstimates(bandit.nActions, 0);
    return runWith(bandit, steps, rng, est, () => {
      // explore 分支两次 rng draw(掷币 + 选臂)，exploit 分支一次。每个分支抽数固定，是 seeded 可复现的关键
      if (rng() < epsilon) return Math.floor(rng() * bandit.nActions);
      return argmax(est.Q); // 取当前 Q 最大的臂
    });
  };
}
```

`est.Q[a]` 是 agent 对臂 a 均值的运行估计，用增量更新 `Q += (r - Q)/n`（`recordPull` 函数）。这个 O(1) 内存的写法不是炫技——真实 RL 训练步数太多，存下每一次 reward 不现实，所以从 bandit 开始就用增量均值，跟你后面在 PPO 里见到的 running statistics 是同一招。

它的命门写在均 regret 上：**270.6，四种里最高，而且曲线全程近线性**。为什么？因为 ε 是**常数**。哪怕它早就找到了最优臂 #5，每一步仍然有 10% 的概率去随机拉一个臂，其中大概率是次优的。这部分损失永远不会停。regret 因此以 ~ε·(平均差距) 的速率线性增长，永远收不住。这就是教科书里"**固定探索是一笔永久的税**"的经典结论。

调小 ε 能减税，但减得越狠探索越不足，找到最优臂的概率越低——你只是在两种亏法之间挪。真正的解法是让探索率**随时间和把握程度自动衰减**，这正是后面 UCB 和 Thompson 做的事。

### 策略二：optimistic 初始化 —— 不用随机也能逼出探索（但靠初值续命）

第二个策略很聪明：还是纯贪心（ε=0，永远拉 Q 最大的），但把每个臂的初始 Q **故意设得很高**。看实现：

```typescript
function optimisticGreedy(initialValue: number): Agent {
  return (bandit, steps, rng) => {
    const est = makeEstimates(bandit.nActions, initialValue);
    return runWith(bandit, steps, rng, est, () => argmax(est.Q));
  };
}
```

机制很巧：真均值都在 0 附近，初值设成 5（远高于任何臂）。贪心总拉 Q 最高的臂，但每拉一次，真实 reward 会把那个臂的 Q **往下拽**向它的真均值。于是"还没试过的臂"看起来永远最高，agent 被逼着把每个臂都扫一遍，扫完再收敛。**探索是从高初值里自然涌现的，策略里一点随机都没有。**

它的实测均 regret 158.2，比 ε-greedy 好不少，曲线后段也明显压得比 ε-greedy 平——收敛后几乎不再 regret。但你看那个 **±253.7 的巨大 std**，这是个警报，下面 F2 会解释它来自哪里。先记住：optimistic 的全部魔力都寄生在"初值 > 真均值"这一个前提上。前提一破，它瞬间退化成最危险的纯贪心。

### 策略三：UCB1 —— 对不确定的臂给奖励，确定了就收手

UCB1 (Upper Confidence Bound，置信上界) 是第一个把"不确定性"显式量化进决策的策略。它每步选的臂是：

$$Q[a] + c \cdot \sqrt{\frac{\ln t}{N[a]}}$$

前一项是均值估计（利用），后一项是**置信半径 (confidence radius)**——拉得越少的臂（N 小）这项越大，拉得越多越收缩。看代码：

```typescript
function ucb1(c: number): Agent {
  return (bandit, steps, rng) => {
    const est = makeEstimates(bandit.nActions, 0);
    return runWith(bandit, steps, rng, est, (t) => {
      if (t < bandit.nActions) return t; // 种子阶段：每臂先拉一次，让 ln(t)/N 有定义
      const tt = t + 1;
      let best = 0, bestScore = -Infinity;
      for (let a = 0; a < bandit.nActions; a++) {
        const bonus = c * Math.sqrt(Math.log(tt) / est.counts[a]);
        const score = est.Q[a] + bonus;
        if (score > bestScore) { bestScore = score; best = a; }
      }
      return best;
    });
  };
}
```

这叫"**乐观面对不确定 (optimism under uncertainty)**"：我不靠掷币随机探索，而是**确定性地去试我最没把握的那个臂**。一个臂被拉得够多、置信半径缩到很小后，UCB 就不再浪费预算去碰它了。这正是它能拿到次线性（理论上对数级）regret 的原因——探索是定向的、会自动收敛的，而不是 ε-greedy 那种永不停止的盲掷。

实测均 regret 157.9，跟 optimistic 几乎一样低，但有个决定性差别：**±std 只有 20.8，最优臂率 93%**。它不光平均表现好，而且**稳定**——24 个 seed 跑出来的结果都差不多。这种稳定性比平均值更值钱，下一节讲为什么。

一个诚实的脚注：UCB1 的对数 regret 界限假设 reward 有界在 [0,1]，而我们的 bandit 是无界高斯。所以这里 UCB 是当**启发式**用的，c 是手调来匹配 reward 量级的，常数不是教科书那个。但定性结论——靠不确定性定向探索拿到次线性 regret——成立。代码注释里专门标了这点，这种"算法假设和实际环境不符"的诚实在真实 RL 里更要紧，因为大多数论文的理论保证在你的环境里都不严格成立。

### 策略四：Thompson 采样 —— 让贝叶斯后验替你掷币

Thompson 采样换了一套语言：给每个臂的均值维护一个**后验分布 (posterior，看过数据后对均值的概率信念)**，每步从每个臂的后验里**采一个样**，拉采样值最高的臂。看实现：

```typescript
function thompson(priorVar: number): Agent {
  return (bandit, steps, rng) => {
    const est = makeEstimates(bandit.nActions, 0);
    const priorPrecision = 1 / priorVar;
    return runWith(bandit, steps, rng, est, () => {
      let best = 0, bestSample = -Infinity;
      for (let a = 0; a < bandit.nActions; a++) {
        const n = est.counts[a];
        const postPrecision = n + priorPrecision; // 已知 σ²=1，每个观测加 precision 1
        const postMean = (n * est.Q[a]) / postPrecision;
        const postStd = Math.sqrt(1 / postPrecision);
        const sample = gaussian(rng, postMean, postStd);
        if (sample > bestSample) { bestSample = sample; best = a; }
      }
      return best;
    });
  };
}
```

机制：拉得少的臂后验宽（不确定），偶尔会采出一个很高的样而被选中——**探索是贝叶斯不确定性的自然副产物，没有任何 bonus 项要调**。这里用了高斯-高斯共轭 (conjugacy，先验和后验同族，更新有闭式解)：reward 是已知方差 σ²=1 的高斯，配高斯先验，后验还是高斯，后验精度 = n + 先验精度，闭式更新一行搞定。

实测它是四个里的**绝对赢家：均 regret 73.4，最优臂率 99%，±std 仅 18.3**。曲线前段陡涨后段几乎水平——付清探索成本后近乎零 regret。在大量实证里 Thompson 经常是简单 bandit 的最优选择，这次也不例外。

## ✦ UCB 和 Thompson 是同一件事的两种语言

这是面试能不能讲清楚的分水岭，也是这章最值得带走的洞见：**UCB 的置信半径和 Thompson 的后验采样，是同一件事的频率派 vs 贝叶斯派两种说法。**

两者干的是完全相同的事——**对不确定的臂给探索倾斜，对确定的臂收手**——只是数学语言不同：

| | UCB1（频率派） | Thompson（贝叶斯） |
|---|---|---|
| 不确定性的载体 | 置信半径 `c·√(ln t / N[a])` | 后验分布的标准差 `√(1/postPrecision)` |
| 探索怎么发生 | 给不确定的臂**加一个确定的 bonus** | 从不确定的臂的**宽分布里采样**，偶尔采出高值 |
| 不确定性怎么消 | N 变大 → 半径 √(1/N) 收缩 | n 变大 → 后验方差 1/(n+prior) 收缩 |
| 决策方式 | 确定性：每步选 argmax(Q+bonus) | 随机性：每步 argmax(后验采样) |

盯着代码看：UCB 的 `c * Math.sqrt(Math.log(tt) / est.counts[a])` 和 Thompson 的 `postStd = Math.sqrt(1 / postPrecision)`，**分母都是某种 N，都随采样次数收缩**。一个把这个宽度当确定的加分，一个把它当采样的散布。UCB 用"上界"做乐观，Thompson 用"采样可能采到高值"做乐观——乐观面对不确定，是同一个原则的两个面孔。

能在面试里把这层对应关系讲出来，说明你理解的是**探索的本质**（不确定性驱动），而不是背了两个公式。

## 为什么 RL 评测必须重复多次：单 seed 是骗局

现在讲这章给全书立的第一根、也是最重要的一根支柱。看上面表里 ε-greedy 的 **±137.9**——它在告诉你一件吓人的事。把同一个 ε-greedy 策略在 24 个不同 seed 上各跑一遍，最终 regret 的分布是：

```
ε-greedy(0.1) 跨 24 seed：最小 136.4 / 最大 578.5 / 均值 270.6 ± 137.9
单 seed 极差 442.1（≈均值的 163%）
```

读一遍这个数字：**最幸运的那个 seed regret 136.4，最倒霉的 578.5，差了 4 倍多。极差是均值的 163%。** 这意味着如果你只跑一个 seed，你完全可能挑到最好的那个（得意洋洋宣布"我的策略 regret 才 136"），也可能挑到最差的（沮丧地以为策略坏了）——**两个结论都是错的，都是运气**。

`src/core/rng.ts` 顶部那段注释把这件事说死了：RL 是整个 ML 里**最难复现的子领域**。一条"work 了"的策略曲线可能纯是 seed 运气；同样的代码换个 seed 就崩。teaching repo 唯一的防御是逐字节可复现 + 多 seed 平均。代码里的 `aggregate` 函数就是干这个的——把每个 seed 的 regret 曲线累加平均，同时**保留每个 seed 的最终 regret 来报 std**。那个 std 是单 seed 图会彻底藏起来的东西，而它恰恰是这章方法论上的核心。

**给全书的规矩：以后看任何 reward / win-rate / KL 曲线，先问"几个 seed？有没有 std？"** 单 seed 的 100% 命中是噪声，不是信号。这个教训在 LLM 后训练里被放大到极致——一次 RLHF 跑几天烧几千刀，你没法跑 24 个 seed，于是单次的好结果极易让人误判。bandit 在这里几乎是免费地让你把这个教训刻进直觉。

## 失败模式：探索是怎么死的

讲两个失败模式。它们不是反例陈列，是**这章最贵的部分**——因为它们在真实 RL 里反复要你的命，只是那时候没有真均值让你一眼看穿。

### F1：纯贪心（ε=0）锁死在次优臂

把 ε 设成 0，就是永远拉当前估计最好的臂，从不探索。直觉上"总拉最好的"听着没错，实测是灾难：

```
ε=0 均regret 598.8 ± 693.8，最优臂率 38%
24 seed 中 15 个完全锁死在次优臂（末段 200 步 0 次碰最优）
```

**±693.8——std 比均值还大，这本身就是"锁死"的指纹。** 为什么？纯贪心的命运由早期的几次随机采样决定：哪个臂第一次恰好吐了个偏高的噪声样本，它的 Q 就被顶上去，贪心从此只拉它，再也不去碰别的——哪怕真正的最优臂就在隔壁。运气好（第一个偏高的恰好是最优臂）regret 近乎 0；运气坏（偏高的是次优臂）regret 线性暴涨。**24 个 seed 里 15 个灾难性锁死，末段 200 步一次都没碰最优臂。** 巨大的 std 正是这两极分化的产物。

这里有个代码上的精巧设计值得指出。F1 用的是 `evaluatePerSeedBandit`——**每个 seed 用一个全新的 bandit**，而不是前面四策略共享的那一个。看注释里的理由：贪心的锁死病在固定 bandit 上可能看不见，因为如果最优臂恰好是 argmax 平局时优先返回的 0 号臂，贪心一上来就拉对了。只有让最优臂在 seed 间散落到不同下标（落在 0 号的概率只有 ~1/k），真正的锁死才暴露出来。**这是个测量方法论的教训：你的实验设置本身可能掩盖你想观测的病。**

把这个 1:1 映射到 LLM：第 8 章 GRPO 里，如果一组采样的 response 早期都收敛到同一个套路，模型就会锁死在那个套路上不再探索别的解法——表面上 reward 在涨，实际上它只是把一个次优策略练得更熟。那就是 F1，只是没有真均值让你当场识破。**防 F1 的根本办法就是保证持续探索**，这也是为什么 PPO/GRPO 要在目标里加熵正则 (entropy bonus，鼓励策略别太确定)。

### F2：optimistic 初值设太低 → 退化成 F1

回到策略二那个 ±253.7 的警报。optimistic 的探索完全寄生于"初值 > 真均值"。把初值往下调，看会发生什么：

```
Q0=5    均regret 71.4      最优臂率 75%   ← 远高于真均值：逼出全臂扫描
Q0=1    均regret 151.8     最优臂率 75%   ← 临界：仅部分臂被逼探索
Q0=0    均regret 386.2     最优臂率 54%   ← 不高于真均值：退化贪心，锁死
Q0=-2   均regret 1413.4    最优臂率 4%    ← 不高于真均值：退化贪心，锁死
```

看这个单调崩坏：初值从 5 降到 -2，均 regret 从 71 涨到 1413（**20 倍**），最优臂率从 75% 塌到 4%。Q0=0 已经不再高于真均值（真均值在 0 附近），高初值逼出全臂扫描的机制失效，optimistic **悄悄变回了 F1 的纯贪心**——第一个偏高的臂锁死全场。Q0=-2 时几乎所有 seed 都锁死，最优臂率 4% 形同瞎拉。

F2 的杀伤力在于它的隐蔽：你以为你用的是一个"会探索"的策略，代码里也确实没写贪心，但**探索是个假象，前提一破它就裸成贪心**。这映射到真实系统里一类极阴险的 bug——你的探索机制依赖某个隐含假设（学习率、温度、初始化、KL 系数），假设悄悄不成立后，系统表面还在"跑"，探索却已经死了，而你看 reward 曲线根本看不出来，因为锁死在次优臂上 reward 也能慢慢爬。

**F1 和 F2 合起来的教训：探索不会大声宣布它死了。** 它死得无声无息，只有当你有一把诚实的尺（这里是 regret + 多 seed std）时才看得见。这就是为什么这章先建尺、再讲策略。

## ⚡ 前沿：LLM 在线对齐就是放大版的 bandit

最后把这章接到全书的真实目标，也接到一个**目前没有通用解、正在研究**的开放问题。

LLM 的在线对齐 (online alignment，训练中实时采样 response 拿反馈再更新) 本质就是一个被放大到极致的 bandit 问题，而且每个维度都更难：

- **臂的数量是天文级。** bandit 这里 10 个臂。一个 prompt 的候选 response 是整个语言空间，"臂"无穷多，你连枚举都做不到，更别说每个臂拉够次数估准均值。UCB 那个"每臂先拉一次"的种子阶段在这里直接破产。
- **reward 是要花钱标的，探索预算是真金白银。** bandit 里拉一次臂免费。LLM 里"探一条 response 值不值"要么烧人工标注，要么烧一个会被 hack 的 reward model（第 5 章）。**哪条 response 值得花标注预算去探**——这就是 bandit 的探索-利用决策，只是每次探索的成本高到要进财务报表。
- **best-of-n 采样本质是一次性 bandit。** 推理时从模型采 n 条 response 选最优那条（best-of-n / rejection sampling），就是"一次性把 n 个臂全拉一遍选最好"——一个没有"利用"阶段、把全部预算砸进一轮探索的退化 bandit。n 设多大？这就是探索预算分配问题，跟本章 ε 怎么调是同一个权衡的另一种长相。

开放问题在这儿：**在 response 空间无穷大、reward 带噪声且会被 hack、探索预算按真金白银计的情况下，怎么做 principled 的探索？** 本章四个策略没一个能直接搬过去——UCB 的置信半径在无穷臂上没法算，Thompson 在语言空间上的后验没有闭式解。学界有一堆尝试（把 RLHF 当 contextual / neural bandit、active learning 挑标注样本、各种探索 bonus 设计），但**没有一个公认的通用解**——这是当前对齐研究里活跃的开放方向。你在本章对探索-利用建立的全部直觉，是你看懂这些前沿工作的入场券；但要清醒：这些直觉指方向，不给现成答案。

## 带走什么

- **regret 曲线的形状是唯一诚实的尺**：次线性（弯）= 探索一次性付清，线性（直）= 永久交税或锁死。看形状，不看绝对数。
- **必须多 seed 平均 + 报 std**：单 seed 的好结果可能是 4 倍极差里挑出来的运气。这是全书评测的铁律，到 LLM 后训练只会更重要。
- **UCB 和 Thompson 是频率 vs 贝叶斯讲同一件事**：不确定性驱动探索，确定了就收手。讲清这层对应是理解探索本质的标志。
- **探索死得无声无息**（F1 锁死、F2 假探索退化），只有诚实的尺看得见。这正是 PPO/GRPO 要加熵正则、要盯探索指标的原因。

诚实声明：上面的绝对 regret 依赖这套 seed 和这个 toy bandit 的具体真均值，偏乐观。可迁移的是**相对趋势**——UCB/Thompson 次线性 < ε-greedy 线性 < 纯贪心锁死风险，且必须多 seed 才看得清。下一章我们给环境加上状态，进入 gridworld，"探索"的难度会因为 credit assignment 的加入再上一个台阶。
