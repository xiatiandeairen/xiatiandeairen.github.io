先把结论摆在桌上。在我们这章配套代码 `examples/rl-posttrain-from-scratch/src/stage07-dpo.ts` 跑出来的干净标签对照里，同一个参考策略、同一份菜单、同一个评测集，RLHF（β=0.5）的真奖励是 **-26.884**，DPO（β=0.5）是 **1.138**；DPO 选中那个真正好的答案 `#1` 的概率 0.979，RLHF 反而被自己的奖励模型推到了 `#3`（概率 0.695）；而 DPO 的前向次数（成本代理）只有 RLHF 的约 **1/1.97**。一句话：在这个 toy 世界里，DPO 又便宜又稳还更准。

但这个数字会骗人，且骗法很有教育意义——它不是因为 DPO 算法天生更聪明，而是因为 RLHF 被迫优化一个会偏好长答案的代理奖励模型，DPO 直接吃真判官标的偏好。把这层看穿，你才不会被「DPO > RLHF」这种空话带跑。本章干三件事：手写 DPO 损失并推导它的隐式奖励、用 β 扫描证明它和 RLHF 的 KL 系数是同一根缰绳、再用三个失败模式说清它什么时候自己崩。

> 数字声明：下面所有带具体小数的数字都来自 `stage07-dpo` 的真实运行输出（seeded，可复现）。合成环境的绝对值偏乐观，可迁移的是相对趋势，不是绝对值——这点文末再强调一次。

## 从 RLHF 的最优解推出 DPO 的损失

DPO（Direct Preference Optimization，直接偏好优化，意思是不训奖励模型直接拿偏好对优化策略）不是凭空冒出来的损失函数。它是 RLHF 那个优化目标的解析解，硬解出来的。

第 6 章里 RLHF 的目标是：最大化奖励，同时别离参考策略（SFT 模型，监督微调后的初始策略）太远。写成式子，对每个 prompt $x$：

```text
max_π  E_{y~π}[ r(x,y) ]  -  β · KL( π(·|x) || π_ref(·|x) )
```

这里 $\beta$ 是 KL 惩罚系数——也就是缰绳的松紧。这个目标有闭式最优解（标准的「带 KL 正则的 reward maximization」结论）：

```text
π*(y|x) = (1/Z(x)) · π_ref(y|x) · exp( r(x,y) / β )
```

关键一步在这里。把这个式子反解出 $r$：

```text
r(x,y) = β · log( π*(y|x) / π_ref(y|x) )  +  β·log Z(x)
```

也就是说，**任何一个奖励函数，都对应一个最优策略；反过来，任何一个策略，都隐式定义了一个奖励**——这个隐式奖励就是 $\beta \cdot \log(\pi/\pi_{\text{ref}})$，加上一个只跟 $x$ 有关、跟 $y$ 无关的配分项 $\beta\log Z(x)$。

DPO 的杀招是：偏好建模（Bradley-Terry 模型，把「A 比 B 好的概率」写成两者奖励之差过 sigmoid）只关心**同一个 prompt 下两个回答的奖励之差**。而那个 $\beta\log Z(x)$ 只跟 $x$ 有关，在 $r(x,y_w) - r(x,y_l)$ 里直接消掉。于是偏好概率变成：

```text
P(y_w ≻ y_l) = σ( β·[ log(π(y_w)/π_ref(y_w)) - log(π(y_l)/π_ref(y_l)) ] )
```

`σ` 是 sigmoid。右边没有 $r$、没有 $Z$、没有奖励模型——只有策略 $\pi$ 和参考 $\pi_{\text{ref}}$。最大化这个偏好的对数似然，就是 DPO 损失：

```text
L_DPO = -log σ( β·[ (logπ(y_w)-logπ_ref(y_w)) - (logπ(y_l)-logπ_ref(y_l)) ] )
```

### ✦ 这就是「隐式奖励」钩子，别只会调包

如果你只记住一件事：**DPO 里那个 $\beta\log(\pi/\pi_{\text{ref}})$ 就是奖励**。RLHF 显式训一个 RM 来打这个分，DPO 把它折叠进策略自己。能从「带 KL 约束的 RLHF 最优解」一路推到这个分类损失、并解释为什么 $Z(x)$ 消掉，是把 DPO 学懂的标志；只会 `pip install trl` 调 `DPOTrainer` 的人讲不出这一步,也就看不懂后面那一堆变体到底在动哪一项。

代码里这个损失就是字面实现。看 `stage07-dpo.ts` 的 `runDPOFrom`：

```ts
const margin = beta * ((lp[yw] - refLogp(yw)) - (lp[yl] - refLogp(yl)));
// g = 1 - σ(margin): the logistic loss gradient scale.
const g = 1 / (1 + Math.exp(margin));
for (let k = 0; k < logits.length; k++) {
  const dYw = (k === yw ? 1 : 0) - probs[k];
  const dYl = (k === yl ? 1 : 0) - probs[k];
  logits[k] += lr * beta * g * (dYw - dYl);
}
```

`margin` 就是上面方括号里那一坨乘 $\beta$；`g = 1 - σ(margin)`（代码里 `1/(1+exp(margin))` 等价于 `1-σ(margin)`）是 logistic 损失的梯度尺度。注意这里没有 `rmScore` 的影子——`rmScore` 那个会偏好长度的代理奖励，整个 DPO 路径碰都不碰。这就是「扔掉奖励模型」的字面意思：代码里它真的不在调用链上。

## β 和 RLHF 的 KL 系数：同一根缰绳

很多人把 DPO 的 $\beta$ 当成一个新超参去试。错。它和第 6 章 RLHF 目标里那个 KL 系数 $\beta$ 控制的是**同一个量：策略离参考多远**。`stage07-dpo` 的 β 扫描把两条 KL 曲线叠在一起证这件事：

| β | RLHF KL | DPO KL | RLHF真奖 | DPO真奖 |
|---|---|---|---|---|
| 0.1 | 2.268 | 0.341 | -37.488 | 1.572 |
| 0.2 | 2.191 | 0.337 | -37.001 | 1.560 |
| 0.5 | 1.179 | 0.264 | -26.884 | 1.138 |
| 1.0 | 0.328 | 0.154 | -13.725 | 0.119 |
| 2.0 | 0.077 | 0.065 | -7.431 | -1.064 |
| 3.0 | 0.033 | 0.035 | -5.753 | -1.619 |

两列 KL 都随 $\beta$ 增大**单调收紧**（RLHF 从 2.268 压到 0.033，DPO 从 0.341 压到 0.035）。这不是巧合：RLHF 的 KL 是目标里显式写的惩罚项，β 越大惩罚越重，策略越不敢动；DPO 的 KL 是隐式的——β 在 margin 里当乘子，β 越大梯度越早饱和（下一节细说），策略也越不敢离开参考。两种形态，一根缰绳。

所以第一条可迁移直觉：**调 DPO 的 β，就是在调你愿意让策略离 SFT 多远**。和 RLHF 调 KL 系数是一回事，不是新东西。

## 失败模式一：β 太大 → 梯度饱和 → 欠训练

但有一个地方会把你坑得很惨，因为方向是反的。先看 `stage07-dpo` 的 C1 表（参考策略真奖励 -3.144，含 9.6% 的灾难性 `#3`；纯 `#1` 真奖励 1.572 是最优）：

| β | 干净真奖励 | #1 概率 | 诊断 |
|---|---|---|---|
| 0.1 | 1.572 | 1.000 | 充分收敛到纯 #1 |
| 0.5 | 1.138 | 0.979 | |
| 1.0 | 0.119 | 0.928 | |
| 3.0 | -1.619 | 0.823 | 梯度早死，几乎没离开参考 |

在 DPO 里，**β 越大，真奖励越差**。为什么？回到那行代码 `g = 1/(1+exp(margin))`，`margin = β·(...)`。β 大 → margin 一上来就大 → `g = 1-σ(margin) → 0` → 梯度更新趋零 → 策略还没把质量集中到 `#1` 就停了，停在平庸的参考附近。β=3.0 时 `#1` 概率才爬到 0.823，没收敛干净，真奖励直接掉成负的。

这是个**反直觉陷阱**：在 RLHF 里「KL 系数大 = 保守 = 安全」，你大概率会下意识把 DPO 的 β 也往大了设求稳。结果在 DPO 里「β 大 = 欠训练 = 卡在平庸参考附近」。同一根缰绳，但 RLHF 那头大 β 是「拉得紧、漂移小但还在训」，DPO 这头大 β 是「梯度提前死了、根本没训起来」。别照搬 RLHF 的调参直觉——这是把两个 β 当同一个量之后唯一需要记的例外。

## 失败模式二：DPO 也会 reward hacking，污染源换成了标签

有种说法是「DPO 没有 RM，所以不会 reward hacking（钻奖励空子）」。不对。hacking 没消失，只是可被污染的信号从 RM 换成了偏好标签本身。`stage07-dpo` 的 C2 用一个**弱参考**（均匀分布，没有好答案可回退）+ 脏标签把它逼出来：

| β | 翻转0% | 翻转30% | 噪声损失 |
|---|---|---|---|
| 0.3 | 0.978 | -4.872 | 5.850 |
| 0.5 | -1.264 | -7.832 | 6.567 |
| 1.0 | -5.787 | -9.463 | 3.676 |

弱参考下，30% 标签翻转把真奖励从 0.978 拽到 -4.872。机制是：DPO 的隐式 KL 缰绳锚在 $\pi_{\text{ref}}$ 上，可均匀参考里没有「好答案」当锚点，被翻转标错的偏好对就直接把策略拽到坏菜单项上，没有回退余地。

这里要注意一个诚实的细节，C2 用的是弱参考、不是小 β——在前面 C1 那种**好 SFT 参考**下，小 β 反而是最抗噪的一端（小 β 让 g 在多步里保持 ~0.5，策略充分收敛到好模式）。代码注释把这个区分写死了，没有为了讲故事假装「小 β 一定过拟合」。

推论很硬：**DPO 的隐式 KL 缰绳只在锚点本身够好时才保命**。好的 SFT 参考不是 DPO 的可选项，是它的前提条件。如果你的 SFT 模型很烂、又拿脏偏好数据跑 DPO，你会得到一个比 SFT 还差的模型，而且没有 RM 那一层把噪声平均掉的缓冲。

## 失败模式三：标注质量是 DPO 的硬上限

C3 把噪声鲁棒性量化了（同 β=0.5）：

```text
RLHF (写死的代理RM，与标签翻转解耦) 真奖励 -26.884
DPO  翻转0%  真奖励 1.138
DPO  翻转30% 真奖励 -0.776
```

DPO 的真奖励因 30% 噪声下降 **168.2%**（从 1.138 到 -0.776）。因为 DPO 直连标签，标注质量就是它的硬天花板——garbage in, garbage out，没有中间层替它消化噪声。

这张表必须配一句诚实说明，否则会误导：表里 RLHF 用的是「写死的代理 RM」，它的偏置（偏好长度）和标签翻转是解耦的，所以数字不随翻转率变。**这不代表 RLHF 真的抗噪**。真实世界里 RM 要在脏对上训练而退化（见第 5 章：翻转率从 0% 升到 30%，RM 干净留出准确率从 0.777 单调掉到 0.755，且需要 early-stop 才不至于更差）。两种方法都吃标注质量；DPO 少了一层 RM 失真，但也少了一层把噪声平均掉的缓冲。哪个更糟取决于你的脏法——别把这张 toy 表当成「DPO 比 RLHF 更怕噪声」的证据，它只证明「DPO 直连标签」这一条结构事实。

## ⚡ 前沿：DPO 之后的变体洪流，都在动损失里的某一项

把 DPO 拆到损失项级别，你就能看懂它后面那一串变体到底改了什么，而不是把它们当成一堆并列的新算法名词。回到那个核心损失：

```text
L = -log σ( β·[ (logπ(y_w)-logπ_ref(y_w)) - (logπ(y_l)-logπ_ref(y_l)) ] )
```

每个变体动的都是这里面一个具体的零件：

- **IPO**：认为 sigmoid 的对数似然在干净数据上会过度推大 margin（过拟合偏好），把损失换成一个**有界的平方目标**，让 margin 收敛到一个目标值而不是无穷大——动的是「外层那个 $-\log\sigma$ 用什么形状」。
- **KTO**：扔掉成对结构，改成**单条样本 + 好/坏二元标签**（借前景理论的效用函数），这样不需要严格配对的偏好对——动的是「输入要不要是 pair」。
- **SimPO**：直接**去掉 $\pi_{\text{ref}}$**，用长度归一化的平均对数概率当隐式奖励，省掉那份参考模型的显存和前向——动的是「要不要保留参考项 $\log\pi_{\text{ref}}$」。注意这恰好放弃了我们 C2 里强调的那根锚点缰绳，是一笔明确的 trade-off。
- **ORPO**：把偏好对齐**合进 SFT 阶段**，在监督损失上加一个赔率比（odds ratio）惩罚项，一步到位不分两段——动的是「要不要单独一个对齐阶段」。

看出来没有？它们不是四个互相竞争的「更好的 DPO」，而是在「损失形状 / 是否成对 / 是否要参考项 / 是否要独立阶段」这四个旋钮上各拧一个。哪个组合最优，**目前没有通用解**——不同模型规模、不同数据质量、不同 β 区间下,公开 benchmark 给出的排名互相打架，正在研究中。你能从损失项级别看懂它们改了哪一项、为什么改，比记住哪个「在某榜单上 SOTA」有用得多,因为那个榜单的结论半年就翻篇。

## 小结与去向

DPO 在这个合成世界里，用约 **1/2.0** 的成本匹配甚至超过 RLHF；它的 β 是隐式 KL 缰绳，但调参方向和 RLHF 相反（大 β = 欠训练）；它没了 RM 但 hacking 没消失,只是污染源变成了偏好标签,弱参考 + 脏标签照样崩。两种方法都不能凭空补偿脏标注。

最后一句诚实话，和第 6 章一致：合成环境的绝对数字偏乐观，能迁移到真实训练的是**相对趋势**——DPO 成本更低、KL 随 β 单调收紧、弱参考叠噪声会把 DPO 拖崩。绝对值别拿去汇报。

下一章（第 8 章）讲 GRPO，它走的是另一条路：保留在线采样、但扔掉 value/critic 网络，用一组采样的相对优势当基线。如果说 DPO 是「扔掉 RM 和 rollout」，GRPO 是「保留 rollout、扔掉 critic」——两章合起来看，你会发现后训练这一代算法的主线就是「哪个移动部件能省掉、省掉之后这根 KL 缰绳由谁来拉」。
