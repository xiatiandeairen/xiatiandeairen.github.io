先把这一章要打碎的错觉摆出来:**网络越深越强**——这个直觉在 2015 年之前是常识,2015 年之后被一组反常数据按在地上。一个 56 层的普通卷积网络,训练误差比 20 层的还高。注意是训练误差,不是测试误差,所以这不是过拟合(模型记住了训练集、泛化不动);是优化本身出了问题——更深的网络明明有能力表达更浅网络能表达的一切(后面那些层学成恒等映射就行),却学不到。这个现象有个名字叫退化问题(degradation,加深反而变差)。

第 05 章我们用 BatchNorm(批归一化)把"较深"的网络稳住了——它让每层输入分布不乱跑,梯度不至于一上来就炸或者灭。但 BN 解决的是"训练能不能启动",不是"能不能堆到几百层"。继续加深,即便有 BN,朴素卷积栈照样退化。本章配套代码 `examples/vision-from-scratch/src/stage06-residual.ts` 做的事很克制:**它把变量砍到只剩一个——有没有 skip(跳跃连接)**,然后用真实数字告诉你这一个变量值多少。

为了让 skip 的贡献成为唯一变量,这份代码故意**不在主干放 BatchNorm**。原因写在文件头注释里(`stage06-residual.ts` 第 22-25 行):BN 本身就是一种缓解梯度消失的手段,主干每块都塞 BN,朴素版的浅层梯度也不会消失了——那就把本章想隔离的效应给盖住了。所以这里把 BN 剥掉,让 skip 的作用单独显形。这是个实验设计决策,不是说真实 ResNet 不用 BN——恰恰相反,真实 ResNet 每个残差块里 BN 是标配。

## 残差块:把"学 H(x)"改成"学 H(x)−x"

普通卷积块算的是 `y = F(x)`:输入进去,一通卷积+ReLU,出来一个新表示。残差块只多加一步——把输入 `x` 原样加回输出:

```
y = relu(F(x) + x)
```

这个 `+ x` 就是 shortcut(也叫 skip connection,跳跃连接)。它看着像个无关痛痒的小动作,但它把整块要学的目标换了。原来这块要学的是完整的映射 `H(x)`;现在 `F(x)` 学的是 `H(x) − x`,也就是**残差**(residual,差值)。

为什么这么换是关键?

## ✦ 当最优解接近恒等,学一个接近 0 的扰动比学完整恒等映射容易得多

退化问题的核心矛盾是:更深的网络里,很多块的最优行为其实就是"别捣乱,把输入原样传下去",也就是学一个恒等映射 `H(x) = x`。

让一堆卷积+ReLU 去精确拟合恒等映射,意外地难——你得让一组非线性变换的输出恰好等于输入,差一点都会沿深度累积放大。但换成残差形式后,要恒等只需要 `F(x) = 0`,也就是把那块的权重全压到接近 0。**学一个"接近 0 的扰动"远比学一个"精确的恒等变换"容易**——前者是默认起点(权重初始化本来就在 0 附近),后者要费力拟合。

这一句就是 ResNet(2015)那篇论文的全部赌注。把目标从 `H(x)` 改成 `H(x) − x`,凭这一改,可训练深度从几十层推到了上千层(原文做到 1202 层仍能优化,虽然那个深度上泛化开始反噬)。改的不是网络的表达能力——朴素栈理论上能表达的它都能表达——改的是**优化的难易度**。这是个关于 loss landscape(损失曲面)和梯度的故事,不是关于容量的故事。记住这个区分,后面 ⚡ 那节会回到它。

现在看代码怎么落地。本章用的是一个带缩放的残差块 `ScaledResidualBlock`(`stage06-residual.ts` 第 72-81 行),`forward` 就一行:

```ts
override forward(x: Tensor): Tensor {
  // identity shortcut: F and x must be shape-identical (guaranteed by shape-preserving F).
  return this.branch.forward(x).mulScalar(this.alpha).add(x).relu();
}
```

`branch` 是 `F`(一个 3×3 卷积 + ReLU 的块),`alpha` 是缩放系数——为什么需要它,失败模式 A 会讲。`add(x)` 就是那条 shortcut,`x` 原样加回来。autograd(自动微分)会自动处理反向,我们不用手写梯度。注意前提:**`F(x)` 和 `x` 形状必须完全一样才能相加**——这里靠 pad=1 的 3×3 卷积保证输出尺寸不变。形状一旦不一样,失败模式 B 会炸。

core 里还有个不带缩放的版本 `ResidualBlock`(`core/nn.ts` 第 299-315 行),它做的是 `relu(F(x) + shortcut(x))`,branch 不缩放。本章在失败模式演示里会用到它——因为它正好暴露裸 skip 的两个坑。

## 发现 1:朴素版梯度随深度消失,残差版保持健康

先讲机制,因为这是整章的物理基础。反向传播穿过一个朴素块时,上游梯度要乘以这块的 Jacobian(雅可比矩阵,即输出对输入的偏导)`J_F`。穿过 `d` 个堆叠的朴素块,梯度就是一连串 Jacobian 的**连乘**:`J_1 · J_2 · ... · J_d`。ReLU 之后每个 `‖J‖` 通常小于 1,连乘就几何级数地缩小——堆得越深,第一层(stem,最浅那个可学层)收到的梯度越接近 0。梯度到不了第一层,第一层就学不动。这就是梯度消失。

残差块算的是 `y = F(x) + x`,对 `x` 求导是 `J_F + I`(`I` 是单位矩阵)。这个 `+ I` 给每条反向路径加了一条**绕过 `F` 的恒等通路**,连乘里永远有一项是 1,乘积塌不到 0。

这不是嘴上说说。`stage06-residual.ts` 的发现 1 直接测了这件事:用同一个 batch、同一套逐块相同的初始化,只改"有没有 skip",做**一次前向+反向,不训练**(`initShallowGrad`,第 128-139 行),量 stem 卷积权重的梯度 L2 范数。不训练是故意的——排除掉优化器、NaN 这些干扰,纯看架构对梯度流的影响。实测(训练前,单次反向):

| 深度 | 朴素堆叠 stem 梯度 | 残差版 stem 梯度 | 残差/朴素 |
|---|---|---|---|
| 4 | 3.180e-1 | 3.421e-1 | 1.1x |
| 6 | 2.876e-1 | 5.440e-1 | 1.9x |
| 8 | 1.703e-1 | 1.175e+0 | 6.9x |
| 10 | 1.855e-1 | 1.288e+0 | 6.9x |
| 14 | 1.036e-1 | 2.334e+0 | 22.5x |

读法:朴素版随深度单调往下走(3.18e-1 → 1.04e-1,信号越来越到不了第一层);残差版反而稳住甚至增大,深度 14 时残差版的 stem 梯度是朴素版的 **22.5 倍**。`+ I` 那条高速路把梯度原样送到了底。

## 发现 2:朴素版卡在随机基线,残差版训到 100%

梯度消失只是中间变量,真正要看的是"能不能学会"。发现 2 在深度=10 下训练两个网络各 200 步,逐块相同初始化,喂相同的 batch 顺序(`trainNet`,第 150-179 行),唯一变量还是 skip。

学习率取在一个**故意挑出来的档位** `lr=0.1`(代码第 55-58 行有解释):低到残差版稳定,高到朴素版卡住。这个 gap 就是这一节的全部信息——`lr=0.05` 时两个都能学,`lr=0.1` 朴素卡死残差照学,`lr≥0.2` 连残差都发散。我们取那个能把差距显出来的值。实测末步:

- 朴素堆叠:loss=1.3280,acc=43.8%
- 残差版:loss=0.0687,acc=100.0%
- 基线参考:4 类随机猜测的 loss = ln(4) = 1.3863

朴素版的 loss 1.3280 紧贴随机基线 1.3863——**它根本没学会**,准确率 43.8% 也就比瞎猜(25%)好一点点,基本是噪声。残差版 loss 掉到 0.0687,准确率 100%。ASCII loss 曲线里残差版从 9.0774(注意它初始 loss 反而更高,这是失败模式 A 的伏笔——稍后讲)一路砸到 0.0668;朴素版的曲线在 1.5848 和 0.8059 之间来回横跳,从没真正收敛。

这就是把发现 1 和发现 2 串起来的因果链:**朴素版第一层梯度被深度吃掉 → 第一层学不动 → 整个网络卡在随机基线;残差版梯度健康 → 同样的学习率下能一路训到位**。这两个数字(22.5x 梯度比、43.8% vs 100% 准确率)是本章最硬的两块证据。

> ⚠️ 诚实标注:12×12 toy 图、6 通道、单线程 f64。绝对的 loss 值、梯度量级、步数都是 toy-optimistic(玩具规模偏乐观),别把"200 步到 100%"当成真实数据集的预期。能迁移的是**相对趋势**——朴素梯度随深度消失、残差梯度存活,以及残差能在朴素卡死的学习率下训练。墙钟数字是真的:全部探针+训练实测 8301 ms(`performance.now()` 实测)。

## 失败模式 A:裸 skip 不归一化,前向激活会爆炸

到这里 skip 听起来是免费午餐。不是。看上面残差版的初始 loss——9.0774,比朴素版高得多。这不是 bug,这是裸 skip 的代价正在显形。

机制:残差块的恒等项 `+ x` 让激活**沿深度累加**。第一块输出 `F_1(x) + x`,第二块在它上面再加自己的输入……每过一块,激活的方差就叠一次。`d` 个块叠下来,激活标准差大致按 `√d` 增长。不缩放、不归一化,深度一上去激活就发散,loss 跟着飙。

`stage06-residual.ts` 用 core 那个不缩放的 `ResidualBlock`(alpha=1)对比了一下(失败模式 A,第 228-240 行)。同样深度,未缩放 vs 缩放后(branch 乘 1/√d)的初始 loss:

| 深度 | 未缩放残差 初始 loss | 缩放后(branch×1/√d) 初始 loss |
|---|---|---|
| 6 | 8.16 | 1.64 |
| 10 | 18.16 | 2.82 |
| 14 | 25.90 | 4.69 |
| 20 | 24.18 | 10.66 |

未缩放版 loss 随深度一路飙到 25.90;缩放后压在个位数,深度 20 也才 10.66——可训练。本章用的解药是**便宜版**:把残差分支乘 `1/√DEPTH`(代码第 59-63 行,`RESIDUAL_ALPHA = 1 / Math.sqrt(DEPTH)`)。`d` 条 skip 叠加方差累加,激活 std 涨 `√d`,那就每条分支除以 `√d` 把它压回 O(1)。

但请注意:**这是 toy 里的廉价替代品,真实 ResNet 不这么做**。真实做法是每个残差块里塞 BatchNorm(第 05 章那一套),让归一化把激活方差摁住。本章主干为了隔离 skip 效应特意拿掉了 BN(开头讲过),所以才需要这个 `1/√d` 缩放来顶替 BN 的活。换句话说,**skip 和归一化是配套的**:光有 skip,梯度通了但激活会炸;skip + 归一化(或缩放),梯度通了激活也稳——两者一起才是能堆深的完整配方。这是初学者最容易漏的一环。

## 失败模式 B:通道数一变,identity skip 立刻 shape mismatch

第二个坑在堆深时几乎必踩。真实网络不可能每块都保持通道数不变——你总要在某些层加宽通道(比如 6 通道变 12 通道)来增加容量。这时 `F(x)` 出来是 12 通道,但 identity shortcut 的 `x` 还是 6 通道,两个张量形状对不上,`F(x) + x` 直接非法。

core 的 `ResidualBlock` 对这个情况做了显式检查(`core/nn.ts` 第 309-312 行):

```ts
if (f.size !== s.size)
  throw new Error(
    `ResidualBlock: branch output ${f.shape} != shortcut ${s.shape}; pass a projection shortcut.`,
  );
```

`stage06-residual.ts`(第 243-257 行)故意构造一个 6→12 通道的加宽块、配 identity skip,触发这个错误。实测抛出:

```
ResidualBlock: branch output 2,12,12,12 != shortcut 2,6,12,12; pass a projection shortcut.
```

这个错误信息的设计值得说一句:它**同时给出两边的 shape**(branch 是 `2,12,12,12`,shortcut 是 `2,6,12,12`),一眼就能看出是通道维 12 vs 6 不匹配,而不是只甩一句"shape mismatch"让你自己去 print。排查崩溃时,带两边形状的错误信息能省掉一轮 debug。

修复是标准做法:**1×1 卷积 projection shortcut**(投影捷径)。不走 identity,而是让 shortcut 也过一个 1×1 卷积,把 6 通道映射到 12 通道,空间尺寸不变,这样 `F(x)` 和 `shortcut(x)` 都是 12 通道,相加合法(`stage06-residual.ts` 第 259-267 行)。实测:

```
输入 shape: [2,6,12,12]  ->  输出 shape: [2,12,12,12]
F(x) 与 shortcut(x) 都是 12 通道, 相加合法; 空间尺寸仍为 12x12。
```

1×1 卷积在这里干的就是"换通道数、不动空间"的活——这正是第 07 章搭真正的分类网络时,在每个下采样+加宽的节点上要反复用到的对齐手段。两种 shortcut 分工清楚:**形状不变用 identity(零参数、最省);形状要变用 1×1 projection(带一点参数,负责对齐维度)**。ResNet 原文也是这么分的。

## ⚡ shortcut 到底解决了什么,至今没有定论

把这一章的机制讲完了,但有个诚实的话必须放在最后:**shortcut 为什么有效,学界到现在没有共识。** 这是个正在研究、没有通用结论的问题。

本章从头到尾讲的是**梯度视角**——`+ I` 给梯度开了一条不衰减的高速路,这是 ResNet 原文(2015)的解释,也是发现 1 直接量到的现象。但 2018 年有一篇 loss landscape(损失曲面)可视化的工作给了另一套说法:它把高维损失曲面投影到二维画出来,发现没有 skip 的深网损失曲面极其崎岖、布满局部坑洼,加上 skip 之后曲面被显著**抹平**,优化器从此好走多了。按这一派的说法,skip 的核心贡献不是修梯度,而是把优化地形从悬崖峭壁变成缓坡。

两套解释——"修梯度流"和"平滑损失曲面"——不互斥,但侧重点不同,谁是主因仍在争。本章的 toy 只能干净地量到梯度那一面(发现 1),量不到曲面形状,所以我没法替你下结论。这是个开放问题。

不过有一件事是确定的:**不管真正的机制是哪个,'加 x 再归一化'这个模块已经赢了。** 它不止活在卷积网络里——Transformer(注意力网络,现代大模型的骨架)的每一层都是 `x + Attention(x)` 和 `x + FFN(x)`,残差连接同样不可或缺,拿掉它深层 Transformer 根本训不动。从 2015 的 ResNet 到今天的大语言模型,"残差 + 归一化"已经是所有现代深网雷打不动的标配模块。机制还在争,但这个模块的地位早已无可争议。
