先把账算清楚，因为这一笔账就是整本书存在的理由。一张 CIFAR 大小的图，32×32 RGB，是 32·32·3 = 3072 个数。如果你像在 tiny-dl 里训 nanoGPT 那样，第一层直接上一个全连接（fully connected，每个输出都连到每个输入）把它接到一个 1024 维的隐层，这一层的权重矩阵就是 3072×1024 ≈ **314 万个参数**——一层。换成一个 3×3、作用在 3 通道上的卷积核，参数是 3·3·3 = **27 个**。

这不是「卷积比较省内存」这种工程口径的优化。27 和 314 万的差距背后是一个结构性的先验被编码进了网络：**同一个边缘检测器，滑过整张图的每个位置**。全连接给每个像素位置都配了一套独立的权重，等于假设「图像左上角的猫和右下角的猫是两件无关的事」；卷积则假设「检测一条边的算子，在哪儿都一样好用」，于是让所有位置共享这 27 个数。参数量的量级从 O(像素²) 塌缩到 O(核²)。本书要从零搭的，就是这个塌缩背后的全部机制。

这一章不写代码，只回答一个先决问题：**既然 tiny-dl 已经能训模型，图像为什么还要单开一支？** 答案有两半——维度灾难，和平移不变性的缺失。把这两半讲透，全书的章节安排就是它们的自然推论。

## 全连接到底坏在哪：两个独立的失败

很多人把「图像要用 CNN」当成一句口诀背下来，但口诀挡不住你在实践里踩坑。全连接处理图像有两个**互相独立**的失败模式，分清楚它们，你才知道卷积的每个设计是在解决哪一个。

### 失败一：维度灾难——参数随分辨率平方爆炸

回到开头那笔账。3072→1024 的全连接是 314 万参数，这还只是 32×32 的玩具分辨率。换成一张普通的 224×224 RGB 输入（这是 ResNet、ViT 这类真实骨干的标准输入尺寸），第一层全连接接到 1024 维就是 224·224·3·1024 ≈ **1.5 亿参数**——一层就顶得上整个 nanoGPT。参数随输入边长**平方**增长，这就是维度灾难（curse of dimensionality，高维空间里样本变得极度稀疏、参数需求爆炸）在图像上的具体形态。

为什么这么多参数是坏事，而不只是「慢一点」？两个连锁后果：

- **数据不够喂**。1.5 亿参数要从有限的图片里估出来，监督信号被稀释，过拟合（overfitting，模型背下训练集而非学到规律）几乎注定。本书第 08 章会在合成图上演示 train/test gap 怎么随增强缩小——那是过拟合的直接可视化。
- **梯度没结构可循**。每个权重各自为政，网络没有任何理由认为「相邻像素相关」「同一模式可能出现在不同位置」。它得从零用数据把这些常识学回来，而这些常识本可以**免费**写进结构里。

卷积对维度灾难的回应就是开头那个权重共享：一个核扫全图，参数量与输入尺寸**解耦**——32×32 和 224×224 用同一个 3×3 核，核还是 27 个数。本书 `examples/vision-from-scratch/src/core/nn.ts` 里的 `Conv2d` 构造函数把这件事写死了，参数只跟核的形状有关，跟输入多大无关：

```ts
// examples/vision-from-scratch/src/core/nn.ts —— Conv2d 构造
const fanIn = inC * kernel * kernel;
this.weight = this.param(Tensor.from([outC, inC, kernel, kernel], () => kaiming(fanIn, rng)));
this.bias = bias ? this.param(Tensor.zeros([outC])) : null;
```

注意 `weight` 的形状是 `[outC, inC, kernel, kernel]`——**没有 H、W**。输入图多大，这层都是这么多参数。这就是维度灾难被结构性绕开的样子。

### 失败二：平移不变性缺失——全连接学不到「猫挪了一下还是猫」

第二个失败和参数量无关，哪怕你有无限数据也治不好。全连接层的每个输入维度对应一个**固定的像素位置**。把一张图整体右移 5 个像素，对全连接来说，几乎所有输入维度的值都变了——它眼里这是一张**完全不同的输入**。它要学会「右移的猫还是猫」，唯一办法是把猫出现在每个位置的样本都见过一遍，分别记下来。这叫缺乏**平移不变性**（translation invariance，物体平移后识别结果不变）。

这是个 deal-breaker，不是优化问题。现实里你永远凑不齐「猫在每个像素位置」的训练数据，全连接也就永远学不全。

卷积自带平移**等变性**（translation equivariance，输入平移、输出跟着平移）：因为是同一个核在滑窗，输入里的边挪到哪，输出的响应就跟到哪。再叠上池化（pooling，把局部响应汇总成一个值，见第 02 章）做下采样，等变性就近似升级成不变性——这正是 `core/nn.ts` 里 `MaxPool2d` / `AvgPool2d` 存在的理由，它们薄薄包一层 `core/autograd.ts` 的 `maxpool2d` / `avgpool2d` 算子。

> **把两个失败分开记**：维度灾难靠**权重共享**解（少参数）；平移不变性靠**滑窗 + 池化**解（同一算子扫全图）。它们恰好都由「卷积」这一个操作同时满足，所以容易被混为一谈。但调试时分清楚很值钱——参数爆炸时你该看通道数和核大小，识别对位置敏感时你该看池化和数据增强。

## 卷积的代价：天下没有免费的先验

讲到这里像是卷积全是好处，那就成了 survey 稿。把代价讲清楚才诚实。**卷积的强归纳偏置（inductive bias，模型对解的内置假设）是把双刃剑**：

- **局部性假设可能错**。3×3 核一次只看 3×3 的邻域，要看更大范围只能靠叠层把感受野（receptive field，一个输出像素「能看到」的输入区域）撑大。本书第 04 章专门拆这个——感受野怎么随深度和步长增长，以及为什么浅层网络「看不全」一个大物体。如果你的任务真需要长程依赖（比如判断图像两个远端角落的关系），堆卷积是低效的绕路。
- **权重共享假设也可能错**。「同一个算子在哪都适用」对自然图像大体成立，但对「位置本身有语义」的输入就不一定——比如人脸总在证件照中央、文档版面有固定结构。这时全局位置信息反而是被卷积主动丢掉的。

这就接到了本章的前沿钩子。

⚡ **卷积已死还是没死？这场争论目前没有定论。** 2020 年的 ViT（Vision Transformer）用纯注意力（attention，让每个位置直接和所有位置交互，正是 tiny-dl 第 06 章手写的那套）在大规模数据上打败了 CNN，一度让人宣告卷积过时——因为注意力没有局部性这个硬假设，理论上能学到任意范围的关系。但 2022 年的 ConvNeXt 反手证明：把 ViT 那套现代训练配方（更激进的数据增强、更长训练、LayerNorm、更大核）原样搬给一个纯卷积网络，它**依然顶尖**。结论被推翻成：当年 CNN 不是输在结构，是输在训练配方。再到 2023 年以后，主流视觉骨干普遍走**卷积 + 注意力混合**（卷积管局部和早期下采样、注意力管全局），两种归纳偏置各取所长。

这件事**目前没有通用解**：给定一个新任务、一个新的数据规模，到底该偏卷积的强先验（数据少时省样本）还是偏注意力的弱先验（数据多时上限高），没有可以套的公式，仍是活跃的研究。**而你能不能读懂这场争论，取决于你是否真懂卷积的归纳偏置是什么、它假设了什么、什么时候这假设会坏**——这本书就是带你把这个「懂」从口诀变成手写过的代码。

## 全书地图：从 matmul 反向，到一个会训练的 CNN

这本书的定位是 tiny-dl 的**续作而非重启**。tiny-dl 已经从零搭好了一个会反向传播的张量引擎、优化器、训练循环，最后训出 nanoGPT（见 tiny-dl 第 08 章 `examples/tiny-dl-from-scratch/src/stage08-nanogpt.ts`）。那套 `core` 里的 autograd、SGD/Adam、cross-entropy、训练循环，本书**全部沿用，一行不重讲**。我们只补图像缺的那一根支柱：卷积。

配套代码在 `examples/vision-from-scratch/`，结构和 tiny-dl 一脉相承——`src/core/` 是一次写好、各 stage 只调用不改的冻结底座，`src/stage01..08-*.ts` 是逐章可跑脚本。按 README，每个 stage 都是离线、确定性的（固定种子 `core/rng.ts`，同机 bit-for-bit 可复现），不下数据集、不联网、不调任何 API。

各章和对应代码的对照：

- **第 01 章 卷积**（`src/stage01-convolution.ts`）：手写 `conv2d`。这里有个值得你记住的设计决定——它不写 7 重循环的直接卷积，而是走 **im2col**（把每个滑窗位置摊平成一行，整个卷积变成一次 matmul）。`core/autograd.ts` 里的注释把动机说得很直白：这样做是为了让卷积的反向**复用 tiny-dl 早就测过的 matmul 反向**（`dA = dC@B^T, dB = A^T@dC`），唯一新增的反向只剩 col2im（im2col 的转置）。这就是「续作」的含金量——新算子的正确性建立在旧底座已验证的梯度上。
- **第 02 章 池化**（`src/stage02-pooling.ts`）：maxpool / avgpool 的前向 + 梯度路由。maxpool 的反向是「梯度只回流到当时取最大的那个位置」，这个路由逻辑是新手最容易写错的地方。
- **第 03 章 卷积反向**（`src/stage03-conv-backward.ts`）：用 `core/metrics.ts` 的 `gradCheck`（数值梯度对比解析梯度）验证 conv/pool 的反向是对的。这一章是全书「诚实数字」的地基——后面所有训练曲线，都站在这里被 gradCheck 验证过的梯度之上。
- **第 04 章 感受野**（`src/stage04-receptive-field.ts`）：感受野随深度/步长增长，ASCII 可视化。回答上一节「卷积为什么看不全大物体」。
- **第 05 章 BatchNorm**（`src/stage05-batchnorm.ts`）：训练用 batch 统计、推理用 running 统计这条经典分叉。`core/nn.ts` 的 `BatchNorm2d` 手写了一个 fused 反向，注释点明原因：BN 的伴随梯度会把同一 channel 里所有元素耦合在一起，不能逐元素简单求。
- **第 06 章 残差**（`src/stage06-residual.ts`）：同一个网络加不加 skip connection 的收敛差距。`core/nn.ts` 的 `ResidualBlock` 注释一句话点破为什么残差能训得更深——`out = relu(F(x) + shortcut(x))`，那个加法让梯度有一条**导数恒为 1 的直通路**绕过 F，对抗梯度消失。
- **第 07-08 章 真训练**（`src/stage07-tiny-cnn.ts` / `src/stage08-augmentation.ts`）：把前面所有零件（conv / BN / maxpool / linear + Adam + cross-entropy）组装成一个分类器，在 `core/data.ts` 生成的合成几何图上端到端训练，输出混淆矩阵；第 08 章再用随机平移/翻转演示数据增强怎么缩小 train/test gap。

一句**关于数字的诚实声明**（沿用 README 的口径，本书必须守住）：这些 stage 跑在**合成 toy 图像**上，绝对准确率偏乐观，别拿去对标真实数据集。可迁移的是**机制和趋势**——loss 会降、残差比 plain 收敛快、BatchNorm 让训练更稳、增强缩小 gap——不是某个具体的准确率数字。本章是全景导览、不绑定独立 stage 输出，所以这里不报任何运行数字；从第 01 章起，每个数字都来自对应 stage 的真实运行。开头那个 **27 vs 约 314 万** 的对比是纯算术（3·3·3 与 3072·1024），不是跑出来的，可自行验算。

## 读这本书之前，先确认你站在哪

如果你还没读过 tiny-dl，本书的第 01 章会假设你已经懂：什么是计算图、反向传播为什么是图的拓扑逆序遍历、matmul 的反向长什么样（`dA = dC@B^T`）。这些是 tiny-dl 第 01-02 章的内容，本书直接拿来用、不重证。

如果你已经能在脑子里跑通 nanoGPT 的一次反向，那你离手写 CNN 只差一个概念：**卷积不是新的反向传播，是同一个 matmul 反向被 im2col 包了一层壳**。这正是第 01 章要带你亲手验证的第一件事。翻过去，我们从把一张图摊成一个矩阵开始。
