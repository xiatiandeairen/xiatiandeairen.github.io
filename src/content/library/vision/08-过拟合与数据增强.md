第 07 章你把 conv → ReLU → pool → linear 接成一个 tiny CNN，在合成形状上跑出漂亮的准确率，本该收工。这一章我先把它的「漂亮」拆穿：换一批更难的数据，同一个网络立刻露馅。先看 `examples/vision-from-scratch/src/stage08-augmentation.ts` 跑出来的基线：

```
dataset: 3 classes (vertical / horizontal / diagonal strokes), 36 train imgs, 120 test imgs, 16x16
chance accuracy = 33.3% (3 balanced classes)
--- 基线: 无增强 (baseline, no augmentation) ---
  train acc = 100.0%   test acc = 88.3%
  过拟合鸿沟 train - test = +11.7pp
```

训练集 36 张图，网络把它背到了 100%；换到从没见过的 120 张测试图上，掉到 88.3%。这 **11.7pp 的差**（pp = percentage points，百分点）就是「过拟合」(overfitting，背训练样本而非学会规律) 的体温计。本章只做一件事：用从零写的数据增强 (data augmentation，对训练图做保标签变换造更多样本) 把这道鸿沟按下去，并对每种手段诚实记账——哪个有用、哪个白费、哪个反而把网络搞坏。

## 先把过拟合「制造」出来：为什么训练集要小到 36 张

很多人讲过拟合是讲一堆正则化公式。但你得先有一道看得见的鸿沟，否则所有手段的增量都淹没在噪声里。stage08 的数据配置是刻意的：

```typescript
const trainPerClass = 12;   // 训练集每类只给 12 张 → 逼出过拟合
const testPerClass = 40;    // 测试集每类 40 张，是更稳的泛化估计
```

为什么 train 给 12、test 给 40？**过拟合是小数据现象**。数据一多，连没增强的网络也学得会规律，没有鸿沟可演示——你想看的对比就消失了。所以这里反过来「饿」着网络：让它样本少到只能靠死记。第 00 章讲过卷积用权重共享 (weight sharing，同一组卷积核扫过全图) 换来参数少、泛化好的承诺；可参数再少，36 张图配上一个有几百维分类头的网络，容量仍远超数据，于是它选了最省力的路——把这 36 张连噪声一起背下来。

这里的笔画数据集 `makeStrokeDataset` 比第 07 章的实心形状更刁钻，data.ts 的注释把原因说死了：

```typescript
// MNIST-like single-stroke digit-ish glyphs. NOT real MNIST — a tiny set of 3 stroke
// patterns (vertical "1", horizontal-top "7"-ish, diagonal "/"). Purpose: a second toy
// task with thinner, more translation-sensitive structure to contrast with solid shapes.
```

笔画又细又对位置敏感（一根 vertical bar 平移两格，落在不同卷积核位置上的响应就变了），正好放大泛化的难度。每张图里笔画的横坐标 `cx`、长度 `len`、起点 `top` 都是随机的（见 `makeStrokeDataset` 内 `randint(rng, ...)`），所以测试集那 120 张是训练集里没出现过的具体位置/长度组合——这正是泛化要跨过去的 gap。

> 诚实边界：这套合成笔画干净、可分性远高于真手写字。**绝对准确率偏乐观**，不要把 88.3%、100% 这些数字当作能迁到真 MNIST 或照片。可迁移的只有相对结构：无增强有鸿沟 → 安全增强收窄 → 破坏语义的增强有害。本章后面每个数字都按这个原则读。

## 三种增强：每一种都是一句「位置/强度无关」的归纳偏置

数据增强的本质不是「凭空造数据」，而是**把你对任务的先验编码进训练分布**。你知道一根竖笔平移两格还是竖笔，于是你造出平移版本喂给网络，相当于明说「位置无关」。stage08 把每种手段写成一个纯函数 `(sample, rng) => sample`，这样消融 (ablation，单独开关一个变量测它的贡献) 时模型代码、训练循环、初始权重全相同，唯一变量就是这个变换——增量才归得了因。

### 随机平移：和卷积的归纳偏置同向叠加

```typescript
const augShift: Augment = (s, rng) => {
  const dx = Math.round(uniform(rng, -2, 2));
  const dy = Math.round(uniform(rng, -2, 2));
  return shiftImage(s, dx, dy);
};
```

`shiftImage`（在 `core/data.ts`）把图整体挪 (dx, dy)，移出画框的内容直接丢弃而不是绕回来——它的注释解释了为什么：

```typescript
// Out-of-frame content is dropped (not wrapped) — wrapping would teach a false torus
// topology the conv never sees at test time.
```

绕回 (wrap) 会教网络一个测试时根本不存在的环面拓扑，是隐蔽的标签污染，所以宁可丢。看消融结果：

```
随机平移 random shift ±2               train 100.0%  test 100.0%  Δtest +11.7pp  gap +0.0pp
```

单这一项就把测试准确率从 88.3% 拉到 100%，鸿沟从 11.7pp 直接归零。**这就是本章的专家钩子** ✦：数据增强对 CNN 特别有效，恰恰因为卷积已经内建了平移不变性 (translation invariance，物体挪位置识别结果不变)。你做随机平移增强，等于在教网络「位置无关」——这跟卷积权重共享的归纳偏置 (inductive bias，模型结构自带的先验假设) **同向叠加**：结构上卷积核扫全图本就倾向位置无关，数据上你再把不同位置的同类样本喂进去，两股力气往一个方向使，于是收益巨大。

反过来想：对一个没有这种偏置的模型——比如把 16×16 图拉平成 256 维直接喂全连接网络——同样的平移增强收益会小得多。全连接网络里像素 (3,5) 和像素 (3,7) 是两个毫不相干的权重，你平移一张图，对它而言几乎是一张全新的、要重新记的图；而对 CNN，平移后的图和原图共享同一套卷积核，增强提供的「同类不同位」信息能被结构高效吸收。**增强的收益高低，取决于它和模型归纳偏置是否对齐。** 这是「数据侧优化」和「模型侧优化」不是两件独立事情的根本原因。

### 水平翻转：在这个类集里安全，但是「碰巧」安全

```typescript
水平翻转 horizontal flip               train 100.0%  test  89.2%  Δtest +0.8pp  gap +10.8pp
```

水平翻转只带来 +0.8pp，几乎没用。为什么对平移那么有效、对翻转却近乎无效？因为这个任务的难点是**位置敏感**（笔画细、对位置响应敏感），平移增强正打在痛点上；而左右镜像并没有制造网络欠缺的那类不变性——竖笔翻转还是竖笔，横笔翻转还是横笔，信息增量小。

更要紧的是 hFlip 在这里**是「碰巧」安全的**，代码注释把这个边界讲得很细：

```typescript
// hFlip maps '/' to a backslash shape but the dataset has a SINGLE diagonal class, so the
// label is preserved (no other class it could be confused with).
```

水平翻转会把 `/` 变成 `\`——这俩在人眼里是不同笔画，但这个数据集**只有一个对角类**，没有第二个对角类去和它竞争，所以标签侥幸没破。换句话说，hFlip 安全是因为类别集合「不够细」，而不是因为翻转本身保标签。下面那个失败案例，正是同一个陷阱在类别集合「够细」时爆掉。

### 高斯噪声与 weight decay：和增强正交的另一条腿

```typescript
高斯噪声 gaussian noise σ=0.15         train 100.0%  test  88.3%  Δtest +0.0pp  gap +11.7pp
weight decay 1e-3 (无增强)            train 100.0%  test  88.3%  Δtest +0.0pp  gap +11.7pp
```

两者单独开都是 +0.0pp。先别下「没用」的结论——这是诚实记账要小心的地方。这两项是**正则化**而非「制造新视角」：噪声让网络别死磕某几个精确像素值，weight decay (权重衰减，惩罚大权重让模型更平滑) 让权重别长太大。它们攻击的是「过拟合」这个病的另一个侧面。weight decay 在这个 toy 里没单独起效，看实现就明白它确实接进了优化器：

```typescript
// core/optim.ts，SGD.step():
let g = p.grad[j];
if (this.weightDecay !== 0) g += this.weightDecay * p.data[j];  // 耦合 L2 形式，折进梯度
```

它确实在每步把 `wd * param` 折进梯度了，只是在 36 张图、平移已能完全解决的场景里，它的边际贡献被吃掉了。这恰好引出一条要诚实承认的方法论限制：**这些 Δ 是 N=1 的固定种子结果**。代码注释自己点破了：

```typescript
// Run-to-run nondeterminism is zero (fixed seeds), so these deltas are exact for this
// config — but on real data they'd be noisy; we'd need multiple seeds to trust the sign.
```

固定种子让数字可复现，但这意味着每个 Δ 只是**一次**抽样。在真实数据上这些增量会抖，要多个种子跑才敢相信正负号。所以「噪声/weight decay = 0pp」只能说「在这个配置下没看到收益」，不能说「无效」。这正是 know 里那条教训——N=1 的命中可能是运气，feasibility 要 ≥3 个独立 sample——在这章的现身。

### 组合：三者叠加 + weight decay

```
--- 组合: shift + hFlip + noise + weight decay ---
  train acc = 100.0%   test acc = 100.0%
  鸿沟 train - test = +0.0pp   (baseline 鸿沟 +11.7pp)
  test 增量 vs baseline = +11.7pp
```

四样一起上，测试 100%、鸿沟归零。注意 loss 曲线（见 stage08 输出）从 1.1426 降到 0.0038，比基线（0.9910 → 0.0004）起点更高、终点更高——这是增强的副作用也是它的工作方式：**训练变难了**。每个 epoch 网络看到的都是抖动过的不同视图，没法再靠死记冲到接近零的 train loss，被迫去学更鲁棒的特征。「train loss 降不到那么低」在这里不是坏消息，是正则在起作用的指纹。

> Dropout 这里没出现在消融表，但它和 weight decay 是同一类「模型侧正则」：训练时随机丢一部分神经元 (dropout，按概率置零激活防共适应)，逼网络别依赖某几个特征的固定组合。在这个 36 张图的 toy 上，平移增强已经把鸿沟打到零，再叠 dropout 边际为零、徒增训练噪声，所以代码没演示它——这是一个该承认「不必要」而不是「遗漏」的取舍。真实规模上 dropout 才会和增强形成互补。

## 失败模式：垂直翻转把对角线变成另一条对角线

这是本章最该记住的一段。前面 hFlip 侥幸安全，是因为类别集合不够细；现在换垂直翻转，类别集合的细度足以让它爆掉：

```
--- 失败模式: 垂直翻转 (label-breaking vertical flip) ---
  train acc = 38.9%   test acc = 46.7%
  test 增量 vs baseline = -41.7pp  <- 注意符号
  ✓ 如预期: 破坏标签语义的增强使 test 准确率不升反降。
    垂直翻转把对角 '/' 变成 '\', 等于给最位置敏感的类别注入了错误标签。
```

注意符号：**-41.7pp**。一个增强不仅没帮忙，还把测试准确率从 88.3% 砸到 46.7%，离 33.3% 的瞎猜基线只剩一步。代码注释把机制讲透了：

```typescript
// Vertical flip — the FAILURE-MODE augmentation. UNSAFE: mirroring top<->bottom turns the
// diagonal '/' (drawn top-left to bottom-right) into '\' (top-right to bottom-left).
// (Same trap as flipping a handwritten '6' into a '9'.)
```

垂直翻转把 `/`（左上到右下）变成 `\`（右上到左下），但你还硬给它打上「这是 `/`」的标签。这等于往最位置敏感的类别里灌**错误标签**——和把手写 `6` 翻成 `9` 还说它是 `6` 是同一个陷阱。网络被两组矛盾的监督信号撕扯，学不出任何东西。

教训不是「翻转危险」，而是更普适的一条：**增强必须保标签 (label-invariant)，「更多变换」不等于「更好泛化」。** 增强是把先验编码进数据，一旦你编码进去的「先验」是错的（`/` 翻转后还是 `/`），它就成了系统性的标签噪声，比不增强糟得多。这也回扣前面 hFlip 的分析：判断一个变换安不安全，不能只看变换本身，要看它在**当前类别集合**里会不会把一类映到另一类的语义上。同一个翻转，在「只有一个对角类」时安全，在「区分两个对角方向」时致命。

汇总表把三种结局并在一起：

```
=== 汇总 (按 test acc 排序) ===
  配置                   train     test      Δtest
  combo+wd             100.0%   100.0%   +11.7pp
  baseline 无增强         100.0%    88.3%    +0.0pp
  失败:垂直翻转               38.9%    46.7%   -41.7pp
```

注意最后一行连 train acc 都只有 38.9%——破坏语义的增强不光毁泛化，连训练都拟合不上去，因为监督信号自相矛盾。这是个有用的诊断信号：**如果加了某个增强后连 train loss 都降不下去，先怀疑这个增强在破坏标签，而不是去调学习率。**

## 收官：从权重共享的承诺到增强兑现的泛化

把整本书的线收一下。第 00 章给的承诺是：卷积用权重共享（同一套核扫全图）换来参数少、且自带平移不变性的归纳偏置——这是「卷积为什么适合视觉」的理论一半。第 01 章手写了那次扫全图的卷积，第 04 章算清了感受野 (receptive field，输出一个值看进了输入多大一片) 怎么随层叠加，让你直观看到「位置无关」是怎么从结构里长出来的。但理论上的不变性是**潜力**，不是**保证**：本章基线证明了，一个内建平移不变性的网络，喂它太少且位置单一的数据，照样过拟合到 11.7pp 鸿沟。

数据增强是把那份潜力兑现成实际泛化的那一步。随机平移增强之所以一击见效（+11.7pp，鸿沟归零），正因为它和卷积的归纳偏置同向——结构倾向位置无关，数据补上各种位置的同类样本，两边对齐，泛化就出来了。这就是「卷积为何适合视觉」的闭环：**结构提供偏置，增强提供与偏置对齐的数据，二者相乘**。理解了这层对齐关系，你才知道为什么图像任务上几乎默认要做增强，而表格数据上随手平移/翻转往往毫无意义——那里的特征没有「平移后还是同类」的物理。

## ⚡ 前沿：增强本身正在变成被优化的对象

本章手做的平移/翻转/噪声只是入门级、靠人工先验挑的固定变换。从 2019 年起，研究方向把「增强」本身变成了一个**可搜索、可学习的对象**，和「调模型」并列成为吃准确率的两大杠杆：

- **AutoAugment** 用强化学习去搜索「该用哪些变换、各自概率和幅度多大」的策略组合——把本章你手工拍板的「平移 ±2、翻转概率 0.5」交给搜索器去定。
- **RandAugment** 发现昂贵的搜索可以塌缩成两个超参（选几个变换、统一幅度多大），大幅降低搜索成本还几乎不掉点。
- **MixUp / CutMix** 更激进：把两张图（及其标签）按比例线性混合、或把一张图的一块补丁贴到另一张上并按面积混标签。注意这直接挑战了本章那条「增强必须保标签」的铁律——它不是保标签，而是**显式地构造软标签** (soft label，标签不再是 0/1 而是按混合比例的概率)，让决策边界更平滑。这说明本章的铁律是「入门安全区」，前沿在更精确的标签建模下故意越界。

这里要诚实标注两点。其一，**「数据侧优化」整体是开放问题**：没有一套通用增强策略对所有任务/数据都最优，什么变换有效高度依赖数据的对称性——平移对自然图像有效，对医学切片可能因为破坏空间标记反而有害，目前仍要逐任务实验，无通用解，正在研究。其二，**toy 规模能观察到的只有方向，不是数值**：本章在 36 张合成图上看到的「平移增强收窄鸿沟」这个**正负号**是可迁移的，但 +11.7pp 这个**绝对值偏乐观**，真实数据上增强的收益通常是个位数百分点且需多种子验证才敢信。把方向当结论、把数值当参考，是读这一整本 toy 实现唯一安全的姿势。
