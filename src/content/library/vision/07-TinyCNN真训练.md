前六章每一章都在单独验证一个零件：卷积的前向（第 01 章）、池化（第 02 章）、卷积的反向梯度（第 03 章）、感受野（第 04 章）、BatchNorm（第 05 章，"批归一化"，把每层激活拉回稳定分布）、残差连接（第 06 章）。每个零件都用 `core/_smoke` 单独跑过数值梯度对拍——但那只证明了"这个算子的导数算对了"，没证明它们拼起来能学。本章就是那个集成测试：把卷积、池化、Linear、Adam、交叉熵全串成一个分类器，端到端训一遍，看三件事——loss 会不会降、测试准确率能不能超过随机基线、以及**哪两类被搞混**。

代码在 `examples/vision-from-scratch/src/stage07-tiny-cnn.ts`，全程单线程 CPU，几秒跑完。下面所有数字都是这个文件运行时实打实算出来的（墙钟时间走 `performance.now()`），不是我编的；toy 估算的地方我会标。

## 网络长什么样，以及那个最容易踩的形状坑

`buildTinyCnn` 把两个卷积块叠起来，每块是 `Conv-BN-ReLU-MaxPool`，最后展平接一个 Linear 分类头：

```typescript
const block = (inC: number, outC: number): Module[] => {
  const mods: Module[] = [new Conv2d(inC, outC, 3, rng, { padding: 1 })];
  if (useBatchNorm) mods.push(new BatchNorm2d(outC)); // BN 稳住激活分布
  mods.push(new ReLU(), new MaxPool2d(2)); // ReLU 后 2x2 池化把 H,W 各砍半
  return mods;
};
const net = new Sequential([
  ...block(1, C1),    // 16x16 -> 8x8
  ...block(C1, C2),   // 8x8 -> 4x4
  new Flatten(),      // -> (N, C2*4*4)
  new Linear(flatFeatures, classes, rng),
]);
```

这里有个新手必踩的坑：Linear 头的输入维度 `flatFeatures` 必须**精确等于**最后一个卷积块输出展平后的长度，差一个数 matmul（矩阵乘法）就直接抛形状错误。很多人靠脑补这个数，脑补错了就看到经典的 `expected (N, X) got (N, Y)`。代码不靠脑补，它用一行算出来：

```typescript
const afterPools = IMG >> 2; // 两次 2x2 池化 => 除以 4 => 4
const flatFeatures = C2 * afterPools * afterPools;
```

输入 16×16，两次 `padding=1` 的 3×3 卷积是尺寸保持的（输出还是 16×16、8×8），真正缩小空间的是两次 2×2 池化，各砍一半：`16 → 8 → 4`。所以喂给分类头的特征向量是 `C2 × 4 × 4`。`C2=16` 时就是 256。运行输出印证：

```
网络: 2x[Conv-BN-ReLU-MaxPool] + Flatten(256) + Linear -> 4; 参数量 = 2324
```

整个网络才 **2324 个参数**——比你想象的小一个数量级。这是故意的：4 类干净几何图根本不需要大网络，参数少反而让"它到底学到了什么"看得更清楚。

### 为什么 eval 时必须冻结 BN，否则一张图的预测会取决于同批的别人

`evalDataset` 里有个看着多余、实际是 BN 的头号 bug 防线的细节：

```typescript
function evalDataset(net: Module, ds: ImageDataset) {
  net.setTraining(false);  // 关键：BN 切到 frozen running stats
  // ...
  const result = noGrad(() => { /* 纯推理，不建反向图 */ });
}
```

`setTraining(false)` 让 BatchNorm 改用训练期攒下来的**全局滑动统计量**（running mean/var），而不是当前 batch 的统计量。为什么非这样不可？BN 在训练时是拿"这一批 32 张图"的均值方差去归一化的。如果 eval 还用 batch 统计，那么单张图的预测就会**取决于它碰巧和谁分在同一批**——同一张猫，跟一批狗在一起和跟一批猫在一起，归一化结果不同，预测可能翻车。这是 BN 最经典的坑：测试结果不可复现，换个 batch 大小准确率就变。`noGrad` 是顺带的优化——推理不需要反向图，关掉省内存。

## 训练前先看随机基线：25% 不是巧合

训练任何分类器之前，先问一句"瞎猜能得多少分"。4 类均衡，瞎猜期望 25%。运行输出：

```
训练前 test: loss=1.5676, acc=25.8% (随机基线 ~25%)
```

25.8% 就是没学之前的水位线，`loss=1.5676` 约等于 `ln(4)=1.386` 附近（均匀分布的交叉熵理论值）。**任何低于这条线的训练都是负学习，任何声称"有效"的模型必须显著高于它。** 把基线打印出来是个好习惯——我见过太多人兴奋地报告"模型 60% 准确率"，结果那任务随机基线就是 50%。

## loss 曲线：形状比数字可迁移

健康学习率（Adam，lr=0.003）训 12 个 epoch，144 步，真实墙钟 **1316ms**（约 9.14ms/步）。ASCII loss 曲线：

```
*                                                         1.5779 (max)
 *
  **
    *
     ****
         *** *
            * ***************
                             ***************************  0.0248 (min)
```

注意这条曲线的形状：**先陡降，后拉平**。这是几乎所有正常训练的共同签名——开头梯度大、参数离最优远，loss 哗哗掉；后面进入收敛区，曲线压平。⚡ 这里要诚实地划清界限：这条曲线的**形状**是可迁移的（你在 ImageNet 上训 ResNet 也是这个形状，只是更慢更抖），但**绝对数字**（最终 loss 掉到 0.0248）是 toy 任务的乐观值，别拿去对标真实数据集。

最终结果：

```
最终: train acc=100.0% (loss=0.0334), test acc=99.2% (loss=0.0615)
train-test acc gap = 0.8 个百分点 (gap 越大越过拟合; 增广见 stage08)
```

`train-test gap` 这个数我专门打出来了：训练集 100%、测试集 99.2%，差 0.8 个百分点。这个 gap 是过拟合的诚实度量——gap 越大，说明模型越是在"背"训练集而非"学"规律。0.8 个百分点很小，因为这任务太干净。真实图像上这个 gap 动辄十几二十个点，怎么把它压下去就是下一章数据增强（data augmentation，对训练图做随机变换扩充多样性）要打的仗。

## 混淆矩阵：模型把哪两类搞混，暴露了它"看"的是什么

光看准确率太粗。混淆矩阵 `M[真][预测]` 才告诉你**错在哪**：

```
     t\p   circle   square    cross triangle
  circle       34        0        0        0
  square        0       28        1        0
   cross        0        0       30        0
triangle        0        0        0       27
最易混: 真实 "square" 被预测成 "cross" 共 1 次 (几何上这两类局部边缘最相似)
```

对角线是答对的，非对角线是混淆。这里唯一的错误是 1 张 square 被认成 cross。代码自动把最严重的混淆挑出来，不用你扫矩阵：

```typescript
let worst = { i: -1, j: -1, n: 0 };
for (let i = 0; i < full.classes; i++)
  for (let j = 0; j < full.classes; j++)
    if (i !== j && after.confusion[i][j] > worst.n) worst = { i, j, n: after.confusion[i][j] };
```

为什么是 square↔cross 混而不是 square↔circle？因为方块和十字都由**直角与直线边缘**构成，局部边缘特征最像；圆是纯曲线，和谁都不像。⚡ 这就是本章最可迁移的洞察：**混淆模式跟着几何/语义相似度走，几何相近的类先混**。这个规律在真实数据上照样成立——猫 vs 狗比猫 vs 汽车更容易混。绝对错误次数（1 次）是 toy 的，但"谁和谁混"这个结构会迁移。

## ✦ 第一层卷积核自发长成边缘检测器——这是诊断 CNN 是否正常学习的最快手段

现在是本章的高光。我们把训练后第一层的 4 个 3×3 卷积核用 ASCII 热力图画出来（`@`/`%` 表示大值、空格表示接近零）：

```
  kernel #0:
    @@::..
    @@  **
    ++@@--
      range [-0.509, 0.423]
  kernel #1:
    ==##**
    ++  @@
    ##++--
      range [-0.796, 0.733]
  kernel #2:
    ==  @@
    ++::++
      ##
      range [-0.513, 0.720]
  kernel #3:
    ++++##
    ::..@@
      ##**
      range [-0.713, 0.989]
```

看出来了吗？这些核不是噪声——它们有**结构**：一侧偏正、一侧偏负，形成方向性的明暗对比。这正是**边缘/方向检测器**（Gabor-like，类似生物视觉皮层 V1 区简单细胞的感受野模式）的形态。

✦ 关键判断在这里：**没人设计这些核去检测边缘，是数据逼出来的。** 我们只给了随机初始化和"分对四种形状"这一个目标，梯度下降自己发现"检测边缘"是分辨几何形状最高效的第一步特征。这跟 1962 年 Hubel 和 Wiesel 在猫视觉皮层测到的方向选择性细胞、跟 AlexNet 第一层学出的 Gabor 滤波器，是同一个现象在不同尺度上的复现。

更实用的是：**可视化第一层卷积核是判断 CNN 是否"正常学习"的最快诊断手段。** 训练后核呈现结构化的边缘/角点模式 → 网络在正常学习；核如果是一片噪声、没有任何空间结构 → 几乎可以断定 learning rate 或初始化出了问题，赶紧查。这比盯 loss 曲线快得多——下一节就证明为什么盯 loss 会被骗。

```typescript
function snapshotFirstKernels(net: Sequential): Float64Array {
  const conv1 = net.parameters()[0]; // 第一个注册参数 = block-1 卷积权重
  return Float64Array.from(conv1.data); // 拷贝，与后续 in-place 更新解耦
}
```

注意这里用 `Float64Array.from` 做了一次**拷贝**——因为优化器是原地（in-place）更新权重的，不拷贝的话你拿到的"训练后快照"会被后续步骤继续改写。这是后面"发散前/后对比"能成立的前提。

## 失败模式：把 lr 调到 5，看 loss 怎么骗你

happy path 太顺了。一个只见过收敛曲线的人，根本认不出崩溃的训练长什么样。所以本章后半段故意制造一次发散，并且——**结果跟教科书说的不一样**，这恰恰是要讲的点。

民间智慧（folklore）说："学习率太高 → loss 变 NaN（非数，溢出的标志）。" 我们把 lr 调到 5（健康值 0.003 的约 1667 倍），用**无 BN** 的网络 + SGD 训练，期待看到 NaN。结果：

```
失败模式演示: lr 调到 5 (健康值 0.003 的 ~1667x)
无 BN 网络 + SGD(lr=5): loss 峰值 21.59, 出现 NaN? 否
  -> loss 没有 NaN 反而是个陷阱: crossEntropy 数值稳定 (减 max + log 加 1e-12),
     所以 loss 封顶在 ~21.6 而非发散; 单看 loss 你会以为只是『没学好』。
```

**没有 NaN。** 为什么？因为我们在第 03/05 章把 `crossEntropy` 写得太稳了——softmax 减去行最大值防上溢、log 加 `1e-12` 防 log(0)。这些数值稳定技巧的副作用是：哪怕 logits 大得离谱，loss 也只会**封顶**在 21.6 左右，而不是溢出成 NaN。于是发散后的 loss 曲线长这样：

```
*                                                         11.3593 (max)
   *
 ** **           *          ** *  ***    **       * ****
      *********** **********  * **   ****  ******* *      1.4908 (min)
```

高位震荡、平台、不收敛——但**没崩成 NaN**。这是个陷阱：一个只盯 loss 的人会以为"模型只是没学好，再调调超参就行"，完全意识不到模型已经废了。

### ✦ 真正的崩溃信号在权重量级，不在 loss

那怎么看出它真的发散了？看**第一层卷积核的量级**：

```
真正的崩溃信号在权重, 不在 loss —— 看第一层卷积核的量级:
  初始化第一层核:  L2=3.615, peak|w|=9.45e-1
  发散后第一层核:  L2=2.02e+4, peak|w|=7.09e+3
  训练全程峰值 peak|w| = 7.09e+3 (从 O(1) 暴涨, 数量级即崩溃强度)
  核 non-finite 比例: 初始 0% -> 发散后 0%
```

初始化时核的最大绝对值是 `0.945`（O(1) 量级），发散后飙到 `7090`——**暴涨约 7000 倍，跨了近 4 个数量级**。而 `non-finite 比例始终是 0%`：没有一个权重变成 NaN/Inf，它们全是有限数，只是大得荒谬。loss 被数值稳定技巧封了顶，但权重没有任何东西拦着它疯长。

把发散前后的核画出来，肉眼就能读出异常——看 heatmap 的 `range`：

```
初始化时的第一层核 (随机但 O(1), 结构平滑):
  kernel #0:
    @@::
    @@..++
    **##--
      range [-0.518, 0.478]

发散后的第一层核 (range 暴露了异常量级):
  kernel #0:
      ++**
    --..**
    ##@@%%
      range [-4.545, -1.262]
```

初始核 range 是 `[-0.518, 0.478]`，对称、平滑、O(1)。发散后 range 变成 `[-4.545, -1.262]`——全负、偏移、量级失控。**这就是肉眼可读的崩溃。** 监控代码很朴素，就是全程追踪峰值绝对值：

```typescript
function peakFiniteAbs(t: Tensor): number {
  let mx = 0;
  for (let i = 0; i < t.size; i++) {
    const a = Math.abs(t.data[i]);
    if (Number.isFinite(a) && a > mx) mx = a;
  }
  return mx;
}
```

实战结论：**排查 lr/初始化问题时，先看第一层核是否还是 O(1) 且有结构，比反复盯 loss 快得多。** loss 会因为数值稳定而撒谎（封顶而非发散），权重量级不会。

### 一个故意不当作教训的反直觉点：BN 不能拦住权重爆炸

代码里这个失败演示**特意关掉了 BN**，因为 BN 会重新归一化激活、压住这个 lr 驱动的爆炸，演示就不明显了。但这里藏着一个反直觉、容易被误传成"BN 能稳住权重"的点，源码注释里明确否认了它：

> 有了 BN，卷积权重反而能漂得**更**自由——BN 让前置卷积的尺度对前向输出无关紧要，于是 SGD 会把那些权重推到天文数字，而 loss 还保持有界。

换句话说，"看权重量级"是鲁棒的崩溃信号，"BN 能约束权重大小"则是**假的**，本章不做这个声称。这是个微妙但重要的区分：BN 稳的是激活分布，不是权重范数。

## ⚡ 那个 99% 是个善意的谎言：toy 和真实之间的鸿沟

回到开头那个让人愉悦的 99.2%。我必须诚实：**这个数字严重偏乐观，别信。** 这四类合成几何图是干净、低噪声、彼此高度可分的——`makeShapeDataset` 只加了 0.08 的噪声。真实图像有什么？光照变化、遮挡、视角、类内差异（一万种长得不一样的"狗"都叫狗）、背景干扰。这些 toy 数据一概没有。

⚡ 这道从 toy 到真实的鸿沟，**目前没有通用的弥合解，仍是开放问题**——域偏移（domain shift，训练分布和真实分布不一致）下的鲁棒泛化是整个领域还在啃的硬骨头。但有一件事现在就能说清楚：**可迁移的是 loss 曲线的形状（先陡降后平）和混淆的结构（几何/语义相近的类先混），不是那个绝对准确率数字。** 你把本章的网络原样搬到真实照片上，准确率会暴跌，但"先陡降后拉平"的曲线形状和"相似类先混"的混淆模式依然成立。

弥合这道鸿沟的第一块战场，就是下一章（第 08 章）的**数据增强与正则化**：既然真实世界有光照/视角/遮挡的多样性而我们的训练集没有，那就人工制造这种多样性——随机翻转、平移、加噪、cutout，逼模型学不变性而不是死记硬背。本章那个 0.8 个百分点的 train-test gap，到了真实数据上会膨胀成主要矛盾，而增强正是压它的主力工具。

## 小结：这一章到底证明了什么

- 第 01-06 章的零件**真能合体学习**：2324 个参数的小网络，从 25.8% 随机基线训到测试集 99.2%，loss 曲线先陡降后拉平。
- **第一层卷积核自发收敛成边缘检测器**（Gabor-like），不是设计的、是数据逼的；可视化它是诊断"是否正常学习"最快的手段（✦）。
- **混淆矩阵暴露模型在看什么**：square↔cross 相混，因为它们局部边缘最像——几何相近的类先混（可迁移）。
- **失败模式给了反直觉教训**：数值稳定的 loss 会封顶而非 NaN，掩盖发散；真正肉眼可读的崩溃信号是第一层核 peak|w| 从 O(1) 暴涨到 7e3（✦）。
- **那个 99% 是 toy 乐观值**：toy↔真实的鸿沟无通用解（⚡），可迁移的是曲线形状与混淆结构，弥合它是第 08 章数据增强的战场。

下一章我们就动手做增强，看那个看似无害的 0.8 个百分点 gap，在更难的数据上会变成什么样。
