先把一句话钉死:深层神经元不"看整张图",它看一个有限的窗口,这个窗口随深度变大、随 stride (步长,卷积核每次滑动的格数) **乘性**变大。很多人写检测器时把感受野 (receptive field,一个输出神经元能影响它的输入像素集合) 当成"反正网络够深就能看全图"的玄学,结果造出一个物理上看不见比自己窗口大的目标的检测器,或者一个顶层神经元其实只盯着输入角落一个 7×7 小块的分类器。这一章我们把感受野变成一个**测出来的整数**,不是背出来的公式。

本章对应的可跑代码是 `examples/vision-from-scratch/src/stage04-receptive-field.ts`,它编译后纯离线运行,固定种子 (seed 404),同一台机器每次跑出逐字节相同的数。下面引用的所有数字都来自它的真实输出。它的 RF 整数和梯度回流都是用第 03 章那个 grad-check 过 (`<1e-8`) 的真 conv2d / autograd 算出来的——没有一个是写死的常量,`[MATCH]` 标记是拿公式和一次新鲜测量去比对。

## 递推公式:两个量一起往下滚

感受野的公式不止一行,因为有两个量耦合着往下累加。一个是 jump (跳距,相邻两个输出单元在输入上隔几个像素),一个是 field (感受野边长本身)。从输入起步 `j₀=1, r₀=1`(一个输入像素只看自己),每过一层:

```
jump:  j_l = j_{l-1} × stride_l
field: r_l = r_{l-1} + (kernel_l − 1) × j_{l-1}
```

整个公式的关键全在 `(kernel−1)` 被前一层的 jump 乘了。一旦某层 stride 把 jump 撑大了,后面每一层的核都"够"到那么多倍远。代码里这两行就是 `computeRfTrace`:

```typescript
field = field + (kernel - 1) * (useJump ? jumpBefore : 1);
jump = jump * stride; // jump only ever GROWS through stride; =1 forever if all strides 1
```

`useJump` 那个开关是后面 PART 3 要演示的 bug——把它关掉就是经典的低估写法。先看干净情形:三层 3×3、stride 全 1、无 padding。stage04 PART 1 打出的递推表:

```
  layer |   spec    | jump | receptive field
  ------+-----------+------+-----------------
     in |    --     |    1 |  1x1 (one pixel)
      1 |    3x3 s1 |    1 |  3x3
      2 |    3x3 s1 |    1 |  5x5
      3 |    3x3 s1 |    1 |  7x7
```

stride 全 1 时 jump 永远是 1,公式退化成 `r += kernel−1`,三层 3×3 就是 `1 + 2 + 2 + 2 = 7`。第 3 层一个单元看 7×7。这是 textbook 结论。但 textbook 结论会撒谎(PART 3 见),所以我们要测。

### 失败模式:输入太小,footprint 被边界裁掉

为什么实测前要先操心输入尺寸?因为脉冲法 (impulse method,往输入打一个亮点看它扩散到多大) 测出来的 footprint 会被特征图边界**裁掉**。如果最深那层特征图本身就比 7×7 小,亮点扩散到边上就被切了,你会测出一个偏小的假值。stage04 注释里把这条钉死:

```typescript
// SIZE INVARIANT: input must be big enough that the deepest feature map is itself >= the
//   7x7 field, else the footprint gets CLIPPED by the map boundary and under-reports. With
//   H=15 and three stride-1 3x3 layers the output is 9x9 — room for the full 7x7 spread.
```

所以代码用 15×15 输入:经三层 stride-1 3×3 无 padding 后输出 9×9,装得下完整的 7×7 扩散。这是测量本身的前置条件,不是公式问题——记错它你会以为是网络结构错。

## 两种独立测法,互相交叉验证

### 测法一:前向脉冲(亮点扩散)

往输入正中打一个值为 1 的像素(其余全 0),用**全 1 核**前向。为什么全 1 核?因为正权重不会抵消,"这个输出像素有没有收到亮点的任何贡献"就等价于"它是不是非零"——一个干净的 footprint,不会因为正负号相消出现假零。stage04 PART 1 逐层量 footprint:

```
  input  feature map 15x15, non-zero footprint = 1x1
  conv1  feature map 13x13, non-zero footprint = 3x3
  conv2  feature map 11x11, non-zero footprint = 5x5
  conv3  feature map 9x9, non-zero footprint = 7x7
```

1 → 3 → 5 → 7,逐层和递推表对上。最深那层特征图画成 ASCII 热力图,就是一个亮点 fan-out 后的样子(中间最亮 49,因为全 1 核三层下来 = 7×7 个 1 求和):

```
    ..........
  ..::--==--::..
  ..--##%%##--..
  ..==%%@@%%==..
  ..--##%%##--..
  ..::--==--::..
    ..........
  range [0.000, 49.000]
```

### 测法二:梯度脉冲(回流到哪些输入像素)

前向脉冲有个软肋——一旦有 stride 下采样,输出在缩小的坐标系里,你还得换算回输入坐标才能读 RF。**梯度脉冲**没这个问题:往某一个深层单元的 grad 塞一个 1,反向传播,凡是拿到非零梯度的输入像素,按导数定义就是"能改变这个单元的像素",正是它的感受野——而且直接落在输入坐标里,免疫 stride 下采样。这是 RF 的严格定义。代码里 `measureRfByGradient` 用 one-hot 把单个单元选成标量 loss 再 `backward()`:

```typescript
const oneHot = Tensor.zeros(out.shape);
oneHot.data[oy * outW + ox] = 1; // (1,1,outH,outW) -> flat index oy*outW+ox
const loss = out.mul(oneHot).sum();
loss.backward();
return { box: nonzeroBox(input.grad, input.shape[2], input.shape[3]), outH, outW };
```

对 9×9 输出的正中单元测,结果:

```
Gradient impulse (center unit of 9x9 output): RF = 7x7 input pixels
  theory 7x7 vs measured 7x7  [MATCH]
```

理论 7×7、前向脉冲 7×7、梯度脉冲 7×7,三个独立来源对齐。画在 15×15 输入网格上就是正中一个 7×7 实心块。这种"公式 + 两种测量"的三角验证,比单背公式可靠得多——任何一个对不上,要么公式错、要么实现错,当场暴露。

## ✦ 为什么是"小核深堆"而不是一个大核

现在能回答这本书反复要绑的工程判断了:既然两层 3×3 的感受野是 5×5(`1+2+2`)、三层是 7×7,那为什么不直接用一个 5×5 核、一个 7×7 核?

算笔账。两层 3×3 覆盖 5×5 视野,参数是 `2 × (3×3) = 18`;一层 5×5 同样覆盖 5×5,参数 `25`。三层 3×3 覆盖 7×7 是 `3 × 9 = 27`;一层 7×7 是 `49`。**同样的感受野,小核深堆参数更少**——核边长是平方进感受野是线性,深堆永远占便宜。更关键的是中间多塞了非线性:两层 3×3 之间夹一次 ReLU (第 03 章那个"死神经元"主角),一层 5×5 是纯线性一锤子。同样的视野,深堆能拟合更复杂的函数。

这就是 VGG (2014, Simonyan & Zisserman) 把网络**全用 3×3 堆叠**的根据,也是 2014 年后 CNN 设计的默认哲学:能用多层小核就别用一个大核,参数更省、非线性更多、感受野还一样大。第 06 章的残差网络把这条推到极致——3×3 堆到几十上百层,靠的是残差连接让这么深的堆叠还能训得动。"小核深堆"和"残差让深堆可训"是一对孪生前提。

## stride 的失败模式:忘了它会乘,公式就撒谎

回到那个 `useJump` 开关。这是本章的头条 bug,因为它静默——不报错,只是让你以为深层单元看一个小窗口,其实它看一个大窗口。

把第 2 层换成 stride 2(下采样),其余不变:3×3 s1 → 3×3 s2 → 3×3 s1。stage04 PART 3 把正确递推、低估写法、实测三个值并排打出来:

```
  layer |   spec    | jump | receptive field
  ------+-----------+------+-----------------
     in |    --     |    1 |  1x1 (one pixel)
      1 |    3x3 s1 |    1 |  3x3
      2 |    3x3 s2 |    2 |  5x5
      3 |    3x3 s1 |    2 |  9x9
  correct final RF = 9x9
  naive  final RF = 7x7   <- forgot to multiply jump by stride
  the naive value UNDER-reports by 2 pixels per side (22% smaller field).
```

正确算法里第 2 层 stride 2 把 jump 从 1 撑到 2,于是第 3 层那个 `(3−1)` 被乘了 2,贡献 4 而不是 2,最终 9×9。低估写法(`useJump=false`,把 jump 钉死在 1)算出 7×7,**每边少 2 像素,整个视野小 22%**。

谁说了算?梯度脉冲。它落在真实输入坐标里,不管你公式怎么记:

```
Gradient impulse (center unit of 5x5 output): MEASURED RF = 9x9 input pixels
    naive formula   : 7x7     [WRONG — underestimate]
    correct formula : 9x9     [MATCH]
```

实测 9×9,坐实正确公式、判死低估写法。**stride 对感受野是乘性的**——只数核大小、不数 stride,你会以为深层单元看一个小窗口,它其实看一个大得多的窗口。这种错只看代码看不出来(网络照样训照样有 loss 下降),梯度脉冲一测立刻现形。这也是为什么这本书坚持"测,别背"。

## ✦⚡ 有效感受野:理论窗口里大部分像素几乎不出力

到这里还有个更深的坑,而且它是个**活跃研究方向**。前面全 1 核测出来的 RF 是一个均匀实心块——块里每个像素权重一样。但真实网络的核不是全 1,是随机初始化、训练出来的权重。换成 N(0,1) 高斯随机核再测梯度脉冲,RF 的**轮廓**还是 7×7(支撑集没变),但梯度**幅度**完全不是平的:

```
Random-kernel gradient impulse: RF outline = 7x7 (same support as all-ones), but look at the magnitudes:
  center |grad| = 4.3018,  corner |grad| = 0.0850,  center/corner = 50.6x
```

中心像素的 |grad| 是角落的 **50.6 倍**。为什么?因为中心像素喂给输出的路径数远多于边缘像素——从中心出发能经过更多种核位置组合到达那个输出单元,边缘只有寥寥几条路。多层卷积叠起来,这种路径数差异让梯度幅度从中心向边缘衰减,近似一个高斯包。这就是**有效感受野** (effective receptive field, ERF):理论 RF 说"这 7×7 都能影响输出",但有效 RF 说"其中只有中心一小撮真正出力,边缘几乎是零"。

这个鸿沟有真实后果。Luo et al. (NeurIPS 2016, *Understanding the Effective Receptive Field*) 给出关键结论:ERF 随网络深度只按 √深度 增长,而且呈高斯衰减——意味着你堆很多层把理论 RF 撑到覆盖全图,有效看到的可能还是中间一小块。⚡**理论感受野和有效感受野之间的鸿沟,至今没有通用的弥合解,是个活跃方向。**两条主线都在试图把边缘像素"激活"起来:空洞卷积 (dilated/atrous conv,核元素之间插空,不加参数就扩大 jump) 用稀疏采样强行拉大有效覆盖;大核卷积的回潮——RepLKNet (Ding et al., CVPR 2022) 直接上 **31×31** 的核,论文的论点正是"想要大有效感受野,与其堆很多层不如直接放大核",这是对 VGG 时代"小核深堆"哲学的一次正面挑战。哪种路线在什么场景更优,目前没有定论,要看任务、算力和你测出来的 ERF 长什么样——所以还是那句话:测它,别信默认。

## 小结与接口

- 感受野是测出来的整数,不是背出来的公式。三角验证:递推表 + 前向脉冲(亮点扩散)+ 梯度脉冲(梯度回流),三者对齐才可信(本章三处都 7×7 [MATCH])。
- 测前先查 SIZE INVARIANT:最深特征图必须 ≥ 目标 RF,否则 footprint 被边界裁掉,低估。
- 小核深堆 > 单个大核:同样 5×5 视野,两层 3×3 用 18 参数 vs 一层 5×5 用 25,中间还多一次非线性。VGG 全 3×3 吃的就是这个红利,第 06 章残差网络把它推到极深。
- stride 对感受野是乘性的:它撑大 jump,后面每层的核都够得更远。只数核、不数 stride = 静默低估(本章 22% 的差),梯度脉冲一测现形。
- ⚡ 理论 RF 远大于有效 RF(本章中心/角落 50.6×),弥合二者无通用解;空洞卷积与大核回潮(RepLKNet 31×31)是当前两条活跃路线。

下一章把这些卷积块组装进真正能跑分类的网络;第 06 章会回到"深度"这个主题——当你想用感受野覆盖全图而不得不堆很多层时,残差连接是让这种深堆还能训得动的那块关键拼图。
