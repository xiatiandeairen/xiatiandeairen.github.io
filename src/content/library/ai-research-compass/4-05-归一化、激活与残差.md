> 这一章把你从"知道 Transformer 里有 LayerNorm、残差、GELU 这些零件、大致知道它们各干啥"带到"能从优化动力学推出为什么不加它们深层网络就训不动、能手推 LayerNorm 的反向、能讲清 RMSNorm 凭什么敢去掉减均值、为什么门控 FFN(SwiGLU)比裸 ReLU 强而代价是多一个矩阵、Pre-LN 和 Post-LN 的梯度行为差在哪"。学完你能从零写出数值正确的 LayerNorm / RMSNorm / SwiGLU-FFN,并在面对"loss 不降 / 深层不收敛 / 训练发散"时知道先去看哪个稳定器。

## 为什么这一章绕不过去

前面几章我们搭出了 Transformer 的两个主干算子:自注意力(token 之间混信息)和前馈网络 FFN(逐 token 做非线性变换)。但如果你真的只把这两个算子堆叠 24 层、48 层,然后开 Adam 去训,**绝大概率会发现:loss 卡住不动,或者几百步就发散成 NaN。** 不是注意力写错了,是你少了三个让深层网络"训得动"的稳定器。

这三个稳定器是:**归一化(normalization)、激活函数(activation)、残差连接(residual connection)**。它们不直接提供"表达能力"——一个没有归一化的网络理论上能拟合的函数集合并不更小。它们提供的是**可优化性(trainability)**:让梯度能稳定地流过几十层、让每一层的输入分布不至于在训练中剧烈漂移、让恒等映射成为优化器唾手可得的起点。

为什么这三个东西要放在一章讲?因为**它们解决的是同一个问题的三个侧面**:深层复合函数 `f₍L₎ ∘ … ∘ f₍₂₎ ∘ f₍₁₎(x)` 在前向时激活尺度会逐层放大或衰减、在反向时梯度会逐层连乘而指数级爆炸或消失。

- 残差把"连乘"变成"连加",给梯度修一条高速公路;
- 归一化把每层输入的尺度钉在一个固定范围,阻止前向激活和反向梯度漂移;
- 激活函数提供非线性,同时它的形状(是否平滑、是否有死区、是否带门控)直接影响梯度能不能传、能传多准。

**贯穿全章的判断标准**:评价一个稳定器,不看它让模型"更强"了多少,而看它让深层网络的**优化曲面(loss landscape)**变平滑了多少、让梯度的尺度变稳定了多少。表达能力是注意力和 FFN 的事;这一章的三个零件,卖的是"训得动"。

我们按"先解决梯度怎么传(残差)→ 再解决尺度怎么稳(归一化)→ 最后挑非线性的形状(激活)"的逻辑推进。但因为残差和归一化在结构上耦合(Pre-LN / Post-LN),我会先建立残差的核心直觉,讲归一化时再回头把两者的配合讲透。

## 一、残差连接:把梯度的"连乘"改成"连加"

### 问题:深了之后,为什么 plain 网络反而更差

2015 年之前一个反直觉的实验事实是:把一个能训练的网络单纯加深(比如从 20 层加到 56 层),训练误差不降反升。注意是**训练误差**升高,不是测试误差——这排除了过拟合,说明问题出在**优化**:更深的网络明明表达能力是更强的超集(深网络至少能用恒等层模拟浅网络),却连训练集都拟合不好。这个现象叫**退化(degradation)**。He 等人 2015 年的 ResNet 用残差连接给出了解法。

退化的根子是梯度。考虑一个朴素(plain)的 L 层网络,第 l 层是 `hₗ = fₗ(hₗ₋₁)`,损失 L 对第 l 层输入的梯度由链式法则给出:

```
∂L/∂hₗ = ∂L/∂h_L · ∏(k=l+1..L) ∂hₖ/∂hₖ₋₁
                    └────────────┬────────────┘
                     L-l 个雅可比矩阵连乘
```

这个**雅可比矩阵的连乘**是灾难的来源。如果每个 `∂hₖ/∂hₖ₋₁` 的谱范数(最大奇异值)平均是 ρ,那么这条梯度的尺度大致按 ρ^(L-l) 走:

- ρ < 1 → 梯度对浅层指数级衰减(**梯度消失,vanishing**),浅层几乎不更新;
- ρ > 1 → 梯度指数级放大(**梯度爆炸,exploding**),数值溢出成 NaN。

ρ 恰好恒等于 1 是测度零事件——随机初始化几乎不可能让所有层的雅可比谱范数都精确卡在 1。所以朴素深层网络的默认命运就是消失或爆炸。

### 推导:残差为什么让梯度"加一条不衰减的路"

残差连接的改动小到一行:把 `hₗ = fₗ(hₗ₋₁)` 改成

```
hₗ = hₗ₋₁ + fₗ(hₗ₋₁)
```

`fₗ` 叫**残差分支(residual branch)**,那条直接把 `hₗ₋₁` 加过来的连接叫**跳跃连接(skip / shortcut)**。现在重新算雅可比:

```
∂hₗ/∂hₗ₋₁ = I + ∂fₗ/∂hₗ₋₁
            └┬┘   └────┬────┘
           恒等项     残差项
```

关键就在这个**恒等项 I**。把 L-l 层的雅可比连乘展开:

```
∏(k=l+1..L) (I + Jₖ)        其中 Jₖ = ∂fₖ/∂hₖ₋₁
  = I + Σ Jₖ + Σ JₖJⱼ + …    (展开后是所有子集乘积之和)
    └┬┘
   恒等路径:无论多少层,这一项恒为 I,梯度原样直达
```

展开式里**永远有一项是 I**,它对应"梯度沿跳跃连接一路直达、不经过任何 `fₖ`"的路径。这意味着:

```
∂L/∂hₗ = ∂L/∂h_L · (I + 高阶项)
       = ∂L/∂h_L + ∂L/∂h_L · (高阶项)
         └────┬────┘
       至少这一份梯度不衰减地传到第 l 层
```

**即便所有残差分支的雅可比 Jₖ 都很小(残差项趋于 0),浅层依然能收到 `∂L/∂h_L` 这一份完整梯度。** 梯度高速公路就是这个意思:连乘里被强行塞进一个恒等项,把"指数衰减"降级成"最坏也有一份原样到达"。这就是残差解决退化的数学本质。

### 另一个等价直觉:恒等映射变成"免费的起点"

换个角度。深层网络要拟合的真实映射记为 H(x)。朴素网络让每层直接学 H,而残差网络让每层学的是**残差** F(x) = H(x) − x,因为输出是 x + F(x)。

为什么这更好学?因为**最优解里有大量层其实应该接近恒等映射**(尤其网络容量过剩时)。朴素网络要让一堆非线性层精确拟合出恒等函数 H(x)=x,这对优化器很难;而残差网络只需要把 F(x) 压到 0——把权重推向 0 是优化器最擅长的事(L2 正则、小初始化都天然往这个方向走)。**残差把"学恒等"这个难题,变成了"把残差分支关小"这个易题。** 这也解释了一个常见做法:把残差分支的最后一层(或 LayerNorm 的 γ)初始化成接近 0,让网络从"纯恒等堆叠"出发,训练初期梯度行为极其稳定,再慢慢长出非线性。

### 代码:残差块的骨架

```python
import torch, torch.nn as nn

class ResidualBlock(nn.Module):
    """Transformer 子层的统一形态:x + sublayer(norm(x))  ← Pre-LN
       sublayer 可以是注意力,也可以是 FFN。"""
    def __init__(self, dim, sublayer):
        super().__init__()
        self.norm = nn.LayerNorm(dim)
        self.sublayer = sublayer

    def forward(self, x):
        return x + self.sublayer(self.norm(x))   # 跳跃连接 x 原样加回,梯度高速路
```

注意这里 norm 放在残差分支**内部、sublayer 之前**——这是 Pre-LN,下一节会讲为什么现代大模型几乎都这么放。

### 分析:残差几乎零成本

- **参数量**:0。跳跃连接没有任何可学参数。
- **计算量**:一次逐元素加法,相对 GEMM 可忽略。
- **显存**:前向多存一份 `hₗ₋₁`(但它本来也要存,用于别处),额外开销近 0;反向时加法的梯度是直接复制,无额外缓存。

**残差是深度学习里性价比最高的设计之一:近乎零成本,换来可以堆几十上百层的能力。**

## 二、LayerNorm:把每层输入的尺度钉死

### 问题:残差解决了梯度路径,但尺度还在漂

光有残差不够。`hₗ = hₗ₋₁ + fₗ(hₗ₋₁)` 这个加法有个隐患:**激活尺度会沿深度累积。** 每加一个残差分支,`hₗ` 的方差就可能比 `hₗ₋₁` 大一点;堆几十层下来,深层激活的数值范围可能比浅层大一两个数量级。激活尺度乱了,会连带两个麻烦:

1. 进入注意力的 `QKᵀ/√d` 尺度失控,softmax 容易饱和(进入梯度接近 0 的平台区);
2. 不同层、不同样本的激活尺度差异巨大,优化器很难用同一个学习率伺候所有层。

更广地说,前面层的参数一更新,后面层看到的输入分布就变了——这个现象当年被命名为**内部协变量偏移(internal covariate shift)**。归一化要做的就是**在每一层入口,把激活的尺度强行拉回一个固定的、与 batch 和深度无关的范围**。

### 推导:LayerNorm 的前向

LayerNorm(Ba 等,2016)沿**特征维**(feature / hidden 维,记作 D)对**每一个样本、每一个位置**单独归一化。设某个 token 的隐藏向量是 x ∈ ℝᴰ:

```
μ   = (1/D) Σ(i=1..D) xᵢ                 ← 这个 token 自己 D 个特征的均值(标量)
σ²  = (1/D) Σ(i=1..D) (xᵢ − μ)²          ← 方差(标量,有偏估计,除 D 不是 D-1)
x̂ᵢ = (xᵢ − μ) / √(σ² + ε)               ← 标准化:零均值、单位方差
yᵢ = γᵢ · x̂ᵢ + βᵢ                        ← 仿射重标定,γ、β ∈ ℝᴰ 可学
```

两点必须讲清:

**(1) 为什么要除以 `√(σ²+ε)` 而不是 `√(σ²)`。** ε(典型 1e-5 ~ 1e-6)是数值护栏。若某个 token 的所有特征恰好相等,σ²=0,除以 √0 直接 NaN。加上 ε 保证分母恒为正。**ε 不是可有可无的工程脏补丁,它界定了归一化在 σ→0 时的行为**:当真实方差远大于 ε,它几乎不影响结果;当方差趋于 0,它防止梯度爆炸到无穷。

**(2) 为什么还要 γ、β。** 标准化把分布钉死成零均值单位方差,这其实**限制了表达能力**——也许某一层就是需要一个尺度为 5、偏置为 −2 的激活分布。γ(scale)和 β(shift)是逐特征的可学参数,允许网络在归一化之后**把分布重新拉到它想要的均值和方差**。极端情况下,若 γ=√(σ²+ε)、β=μ,LayerNorm 退化成恒等映射。**所以 LayerNorm 不是"强制零均值单位方差",而是"先标准化掉不可控的漂移,再用可学参数把可控的尺度还回来"。** γ 初始化为 1、β 初始化为 0,即从恒等附近出发。

### 推导:为什么不用 BatchNorm

BatchNorm(BN,2015)和 LN 形状像、本质不同:**BN 沿 batch 维统计**,即对一个特征通道,跨这个 batch 里所有样本算 μ、σ²;LN 沿特征维统计,只看单个样本自己。一张图说清差异(设张量形状 [B, T, D] = [batch, 序列长, 特征]):

```
BatchNorm:  对每个特征 d,统计跨 (B,T) 所有元素 → μ_d, σ_d   (统计量依赖 batch 和序列长)
LayerNorm:  对每个 (b,t),统计跨 D 个特征      → μ_{b,t}, σ_{b,t}  (每个 token 自己算,与 batch 无关)
```

BN 在 CNN 视觉任务里非常成功,但在**变长序列的语言任务**里有三个致命问题,这正是 Transformer 选 LN 的原因:

1. **batch 统计量不稳定,且依赖 batch 大小。** BN 的 μ、σ 是对当前 mini-batch 估计的,batch 小时(大模型训练里单卡 batch 经常很小)估计噪声大;LLM 推理时常常是 batch=1 逐 token 生成,根本没有"batch 内其他样本"可统计。
2. **序列变长 → 统计被 padding 污染。** 不同样本序列长度不同,要 pad 到等长。BN 跨 (B,T) 求和时,这些 padding 位置要么被错误地算进统计量,要么需要复杂的 mask 逻辑。LN 只看单个 token 的 D 个特征,**与序列里有多少 token、哪些是 padding 完全无关**。
3. **训练/推理不一致。** BN 训练时用 batch 统计、推理时用训练期累积的 running mean/var,这套切换在序列分布随位置变化时容易失配。LN **训练和推理用完全相同的公式**,无切换、无 running stats。

**一句话总结:LN 逐样本逐位置归一化,把统计量从"跨样本"收回到"样本内部",于是天然免疫 batch 大小、序列长度、训练/推理切换这三件序列任务的麻烦事。** 这不是经验偏好,是序列建模的结构性需求决定的。

### 推导:LayerNorm 的反向(手推 + 数值验证)

这是必须会推的基本功,面试和读源码都绕不开。给定上游梯度 g = ∂L/∂y ∈ ℝᴰ,要求 ∂L/∂x、∂L/∂γ、∂L/∂β。先记 std = √(σ²+ε)。

γ、β 的梯度最简单(注意要在 batch/序列维上求和,因为 γ、β 被所有 token 共享):

```
∂L/∂γᵢ = Σ over (b,t)  gᵢ · x̂ᵢ
∂L/∂βᵢ = Σ over (b,t)  gᵢ
```

x 的梯度是难点,关键在于 **μ 和 σ 都是 x 的函数**,所以 x 的每个分量都通过三条路径影响 y:直接路径、经 μ 的路径、经 σ 的路径。先把对 x̂ 的梯度记下来:

```
dx̂ᵢ = ∂L/∂x̂ᵢ = gᵢ · γᵢ
```

然后逐项链式(把 x̂ⱼ = (xⱼ−μ)/std 对 xᵢ 求偏导,μ 和 std 都依赖所有 xₖ):

```
推导骨架(沿特征维 D,下标 i,j 都是特征维):
  ∂x̂ⱼ/∂xᵢ = (1/std)·[ δᵢⱼ − 1/D − x̂ᵢ·x̂ⱼ/D ]
            └─┬─┘  └─┬─┘  └────┬────┘  └────┬────┘
           缩放   直接项  经 μ 的项    经 σ² 的项

  ∂L/∂xᵢ = Σⱼ dx̂ⱼ · ∂x̂ⱼ/∂xᵢ
         = (1/std)·[ dx̂ᵢ − (1/D)Σⱼ dx̂ⱼ − x̂ᵢ·(1/D)Σⱼ dx̂ⱼ·x̂ⱼ ]
```

写成更好记的"减两个均值"形式(mean 沿特征维):

```
∂L/∂x = (1/std) · ( dx̂ − mean(dx̂) − x̂ · mean(dx̂ ⊙ x̂) )
                    └─┬─┘  └───┬───┘  └────────┬────────┘
                  原始梯度  减掉均值分量   减掉沿 x̂ 方向的分量
```

这个公式有一个**漂亮的几何意义**:LayerNorm 的反向把上游梯度投影掉了两个方向——常向量 **1**(对应 μ 这个自由度)和 x̂ 方向(对应缩放这个自由度)。因为前向时 LN 移除了均值和尺度这两个自由度,反向时这两个方向上的梯度就被相应地"扣掉"。可以验证 `∂L/∂x` 在特征维上的和恒为 0(被减去 mean 保证),且与 x̂ 正交。

我用 numpy 做了有限差分验证(γ、β 随机,上游梯度随机,中心差分 h=1e-6):

```
解析 ∂L/∂x  vs  有限差分:  最大绝对误差 1.4e-9     ✓ 公式正确
∂L/∂x 在特征维求和:        ~1e-16(机器精度)        ✓ 与 1 正交
∂L/∂x 与 x̂ 的点积:        ~1e-5(受 h² 截断限制)    ✓ 与 x̂ 正交
```

**这两个正交性不是巧合,而是"归一化移除了两个自由度 → 反向自动消除这两个方向的梯度"的直接后果。** 理解了这点,你就明白为什么 LN 能"平滑损失面":它把损失对 x 的梯度约束在一个 D−2 维子空间里,削掉了让激活整体平移、整体放缩这两类对损失无意义却数值上很大的方向,从而让优化更稳。

### 代码:从零实现 LayerNorm(含手写反向)

```python
import torch, torch.nn as nn

class LayerNorm(nn.Module):
    def __init__(self, dim, eps=1e-5):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(dim))   # scale,初始化 1 → 从恒等出发
        self.beta  = nn.Parameter(torch.zeros(dim))   # shift,初始化 0
        self.eps   = eps

    def forward(self, x):                              # x: [..., D]
        mu  = x.mean(-1, keepdim=True)                 # 沿特征维
        var = x.var(-1, unbiased=False, keepdim=True)  # 有偏方差(除 D)
        xhat = (x - mu) / torch.sqrt(var + self.eps)   # 数值护栏 eps
        return self.gamma * xhat + self.beta

# 手写反向(给定 dy = dL/dy),用来对照理解上面的推导
def layernorm_backward(dy, x, gamma, eps=1e-5):
    D    = x.shape[-1]
    mu   = x.mean(-1, keepdim=True)
    var  = x.var(-1, unbiased=False, keepdim=True)
    std  = torch.sqrt(var + eps)
    xhat = (x - mu) / std
    dxhat = dy * gamma
    dx = (1.0 / std) * (dxhat
                        - dxhat.mean(-1, keepdim=True)            # 减 mean(dx̂)
                        - xhat * (dxhat * xhat).mean(-1, keepdim=True))  # 减沿 x̂ 分量
    dgamma = (dy * xhat).sum(dim=tuple(range(dy.dim()-1)))        # 在非特征维求和
    dbeta  = dy.sum(dim=tuple(range(dy.dim()-1)))
    return dx, dgamma, dbeta
```

### 分析:LayerNorm 的开销

- **参数量**:2D(γ 和 β 各 D 个)。对 d_model=4096 的层,每个 LN 就 8192 个参数——相对一个 4096×4096≈1678 万参数的投影矩阵,**LN 参数占比可忽略**。
- **计算量**:几次沿 D 的归约(求 μ、σ²)和逐元素运算,FLOPs 是 O(D),相对 FFN 的 O(D²) 矩阵乘可忽略。
- **真正的成本是访存与同步,不是算力。** LN 要先读完整行才能算出 μ、σ²,再回头逐元素改写,是典型的 **memory-bound** 算子;在大模型里 LN 的 kernel 启动和访存常成为非 GEMM 时间的大头。这也是为什么后面的 RMSNorm 值得做——它能省掉一遍归约。

## 三、RMSNorm:敢去掉减均值的简化

### 问题:LN 的减均值那一步,真的有必要吗

RMSNorm(Zhang & Sennrich,2019)提出一个大胆的简化:**只做缩放,不做平移**——既不减均值,也(默认)不要 β。它把归一化简化为除以**均方根(Root Mean Square)**:

```
RMS(x) = √( (1/D) Σ(i=1..D) xᵢ²  +  ε )

yᵢ = ( xᵢ / RMS(x) ) · γᵢ
```

对照 LN,差异一目了然:**LN 减了 μ 再除以以 μ 为中心的标准差;RMSNorm 不减 μ,直接除以未中心化的均方根,也不加偏置 β。**

### 推导:为什么去掉减均值还能 work

LN 的两个作用是"re-centering(把均值拉到 0)"和"re-scaling(把尺度归一)"。RMSNorm 的核心假设是:**真正提供训练稳定性的是 re-scaling,re-centering 的贡献小到可以省。** 给一个直观论证:

LN 反向的几何分析告诉我们,归一化的稳定作用主要来自"约束梯度尺度、削掉无意义的整体放缩方向"。re-centering 削掉的是"整体平移"这个自由度;但在有残差和后续仿射的网络里,均值偏移这个自由度本来就比较容易被其它层吸收。RMSNorm 赌的就是:**省掉 re-centering,训练稳定性几乎不掉,但能省一遍沿特征维的归约和一次减法。**

定量上看省了什么:

```
LayerNorm:   要算 μ(一遍归约)、再算 σ²(依赖 μ,又一遍归约)、减 μ、除 std、乘 γ、加 β
RMSNorm:     只算 Σxᵢ²(一遍归约)、除 RMS、乘 γ
             省掉:一遍归约(均值)、一次逐元素减法、加 β 这一步
```

在 memory-bound 的归一化算子里,**少一遍对整行的归约 + 少一次逐元素遍历,是实打实的访存节省**(粗略 ~10–30% 的归一化开销,具体随实现和硬件而异,**待核**)。LLaMA、PaLM、T5(用其变体)等现代大模型大量采用 RMSNorm,正是因为它在效果几乎不变的前提下更省、更易实现。

**这里有个值得记的方法论**:RMSNorm 是"假设某个组件不重要 → 删掉它 → 实测确认效果不掉"这种**消融驱动(ablation-driven)简化**的典范。它不是发明新机制,而是大胆删减并验证。读论文时遇到"某经典模块的简化版",第一反应应该是"它删了什么、凭什么敢删、删了之后省在哪"。

### 代码:RMSNorm

```python
class RMSNorm(nn.Module):
    def __init__(self, dim, eps=1e-6):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(dim))   # 只有 scale,没有 shift
        self.eps   = eps

    def forward(self, x):                             # x: [..., D]
        # rms = sqrt(mean(x^2) + eps);用 rsqrt 省一次除法
        rms = torch.rsqrt(x.pow(2).mean(-1, keepdim=True) + self.eps)
        return x * rms * self.gamma                   # 不减均值、不加 beta
```

工程注意:RMS 在低精度下要小心。`x.pow(2)` 在 fp16/bf16 下对大激活可能溢出或丢精度,实现里通常**先把 x 升到 fp32 算 RMS、再降回**(LLaMA 官方实现就是这么做的)。这一点和上一门 MLSys 课"混合精度"那章的结论一致:归约类操作要在高精度下做。

### 分析:RMSNorm vs LayerNorm

| 维度 | LayerNorm | RMSNorm |
|------|-----------|---------|
| 减均值(re-center) | 有 | **无** |
| 偏置 β | 有 | **无(默认)** |
| 参数量 | 2D | **D** |
| 归约次数(概念上) | 2(μ、σ²) | **1(均方)** |
| 数值护栏 | √(σ²+ε) | √(mean(x²)+ε) |
| 典型采用 | 原始 Transformer、BERT、GPT-2 | **LLaMA、PaLM、多数现代 LLM** |

参数量从 2D 减到 D,看似省得不多(LN 参数本来就占比极小),**RMSNorm 真正的收益在计算/访存而非参数**——别被参数表误导。

## 四、激活函数:从死神经元到门控

### 问题:ReLU 简单好用,但有个"死亡"的坑

FFN 的非线性来自激活函数。最经典的是 **ReLU(x) = max(0, x)**:正半轴恒等、负半轴归零。它解决了 sigmoid/tanh 的饱和问题(sigmoid 在两端梯度趋 0,深层必梯度消失),计算又极快。但 ReLU 有个著名的坑——**死神经元(dead/dying ReLU)**:

```
ReLU'(x) = 1 (x>0);  0 (x<0)
```

ReLU 在负半轴**梯度恒为 0**。如果某个神经元因为一次大的梯度更新,使得它对所有输入的预激活都落到负区(比如 bias 被推得很负),那么它的梯度永远是 0,**再也收不到任何更新信号,等于永久死亡**。在大学习率或初始化不当时,可能有相当比例的神经元就这么死掉,白白浪费容量。

### 推导:GELU——给负区一条"平滑的活路"

修法的核心思路:**别在 0 处硬切断,给负区一点平滑的、非零的梯度。** GELU(Gaussian Error Linear Unit,Hendrycks & Gimpel,2016)是 BERT、GPT 系列的选择。它的定义是:**用输入自身的标准正态累积分布(CDF)作为门**——

```
GELU(x) = x · Φ(x)        Φ 是标准正态 N(0,1) 的累积分布函数
        = x · (1/2)·[ 1 + erf( x / √2 ) ]
```

直觉:Φ(x) ∈ (0,1) 是"这个输入有多大概率为正"的软判断。x 越大,Φ(x) 越接近 1,GELU≈x(像 ReLU 的正区);x 越负,Φ(x) 越接近 0,GELU≈0(像 ReLU 的负区);但**过渡是平滑的**,负区附近 GELU 取一个小的负值再回到 0,处处可导、梯度处处非零(除极限),没有死区。

erf 有点慢,常用 tanh 近似(很多实现的默认):

```
GELU(x) ≈ 0.5·x·( 1 + tanh[ √(2/π) · (x + 0.044715·x³) ] )
                              └──┬──┘
                          √(2/π) = 0.7978845608…   (Python 验算一致)
```

我做了数值对比,精确版与 tanh 近似在 x∈[−3,3] 上**最大偏差约 4e-4**,实践中无差异:

```
x= -3:  exact=-0.004050   tanh=-0.003637
x= -1:  exact=-0.158655   tanh=-0.158808
x=  0:  exact= 0.000000   tanh= 0.000000
x=  1:  exact= 0.841345   tanh= 0.841192
x=  3:  exact= 2.995950   tanh= 2.996363
```

顺带认识一个近亲 **SiLU / Swish**:`Swish(x) = x·σ(x)`(σ 是 sigmoid)。它和 GELU 形状几乎一样(都是平滑、负区有小回弯的"自门控"激活)。Swish 在 x<0 有个极小值,我数值求得在 **x ≈ −1.278 处取最小值 ≈ −0.278**——这个"负区小回弯"正是它和 ReLU 的关键区别:负区不是死的。下一节的 SwiGLU 里 Swish 会扮演主角。

### 推导:GLU 门控家族与 SwiGLU——为什么"乘起来"更强

到这里激活还都是逐元素的标量函数。**门控线性单元(GLU,Gated Linear Unit)**引入一个结构性的新东西:**用一路信号去"门控"(逐元素乘)另一路信号**。原始 GLU(Dauphin 等,2016):

```
GLU(x) = (xW + b) ⊙ σ(xV + c)
          └──┬──┘   └────┬────┘
          内容分支     门控分支(σ 把它压到 (0,1) 当"开关")
```

⊙ 是逐元素乘(Hadamard)。直觉:门控分支 σ(xV) 是一组 (0,1) 之间的"阀门",**逐维决定内容分支的每个特征放过多少**。这是**数据依赖(input-dependent)的信息筛选**——阀门开多大取决于当前输入 x,而不是固定的。

Shazeer 2020 年(《GLU Variants Improve Transformer》)把门换成各种激活做消融,发现用 Swish 当门的 **SwiGLU** 在 Transformer FFN 里效果最好。把它代入 FFN,得到现代大模型(LLaMA、PaLM 等)的标准 FFN:

```
传统 FFN:    FFN(x) = max(0, xW₁) W₂              (一个升维矩阵 + 一个降维矩阵)

SwiGLU-FFN:  FFN(x) = ( Swish(xW₁) ⊙ (xV) ) W₂     (两个升维矩阵 W₁、V,一个降维 W₂)
                       └────┬────┘  └┬┘
                       门控分支    内容分支(无激活,线性)
```

**为什么门控提升质量?** 给一个机制层面的论证,而非"实验上更好"这种空话:

1. **数据依赖的乘法 = 比加法更强的特征交互。** 传统 FFN 是 `σ(xW₁)W₂`,中间隐层各维相互独立地过激活再线性组合;SwiGLU 引入了 `gate ⊙ value` 这个**乘法交互**,让一个特征能动态地放大或抑制另一个特征。乘法门控能表达"仅当特征 A 激活时才让特征 B 通过"这类条件逻辑,这是纯加性结构难以高效表达的。
2. **更平滑、更精细的门。** Swish 当门比硬 ReLU 提供逐维连续的 (0,1) 开度,梯度信息更丰富,门可以"半开"。
3. **缓解死神经元。** Swish 负区非零,门控分支也不会整片死掉。

### 推导:SwiGLU 的"代价"与参数补偿——为什么隐层缩到 ⅔

天下没有免费午餐。传统 FFN 用 **2 个矩阵**(W₁ 升维、W₂ 降维),SwiGLU 用 **3 个矩阵**(W₁、V 两个升维 + W₂ 降维)。如果隐藏维 d_ff 不变,SwiGLU-FFN 的参数和计算就比传统 FFN 多 50%。为公平比较,实践中**把 SwiGLU 的隐层维缩小,补偿掉多出来的矩阵**。算一笔账(d 是 d_model,h 是 FFN 隐层维):

```
传统 FFN 参数:  W₁(d×h) + W₂(h×d)            = 2dh
SwiGLU 参数:    W₁(d×h') + V(d×h') + W₂(h'×d) = 3dh'

令两者相等:  2dh = 3dh'  ⟹  h' = (2/3)·h
```

**所以把 SwiGLU 的隐层维设成传统 FFN 的 ⅔,参数量和计算量就对齐了,可以公平对比。** 这就是为什么你在 LLaMA 配置里看到 FFN 隐层不是标准的 4×d_model,而是约 4×(2/3)×d_model ≈ (8/3)·d_model(再往上取整到硬件友好的倍数)。Shazeer 的论文正是在**参数对齐**的前提下报告 SwiGLU 更优,这个"⅔ 缩放"是让对比公平的关键,新手很容易漏掉而误以为 SwiGLU 是靠堆参数赢的。

### 代码:SwiGLU-FFN

```python
import torch.nn.functional as F

class SwiGLU_FFN(nn.Module):
    """LLaMA 风格 FFN。hidden 应取传统 4*dim 的约 2/3 以对齐参数量。"""
    def __init__(self, dim, hidden=None):
        super().__init__()
        if hidden is None:
            hidden = int(2/3 * 4 * dim)        # 4*dim 的 2/3 做参数补偿
        self.w1 = nn.Linear(dim, hidden, bias=False)   # 门控分支的升维
        self.v  = nn.Linear(dim, hidden, bias=False)   # 内容分支的升维
        self.w2 = nn.Linear(hidden, dim, bias=False)   # 降维

    def forward(self, x):
        gate = F.silu(self.w1(x))              # Swish(xW1) = SiLU,作门
        return self.w2(gate * self.v(x))       # 门 ⊙ 内容,再降维
```

### 分析:激活的成本对比

| 激活 | 形式 | 负区梯度 | 平滑 | FFN 矩阵数 | 备注 |
|------|------|----------|------|-----------|------|
| ReLU | max(0,x) | 0(**死区**) | 否(0 处不可导) | 2 | 最快,有死神经元风险 |
| GELU | x·Φ(x) | 非零小负 | 是 | 2 | BERT/GPT;tanh 近似够用 |
| Swish/SiLU | x·σ(x) | 非零小负 | 是 | 2 | 与 GELU 形状近似 |
| SwiGLU | Swish(xW)⊙(xV) | 非零 | 是 | **3(隐层缩 ⅔ 补偿)** | LLaMA;门控,数据依赖筛选 |

逐元素激活(ReLU/GELU/SiLU)本身计算量都是 O(元素数),可忽略;**SwiGLU 的成本是结构性的多一个矩阵乘**,通过 ⅔ 缩放后总量持平,只是把"一个大矩阵 + 一个激活"换成"两个略小矩阵 + 一个门控乘",换来更强的特征交互。

## 五、把三者装回 Transformer:Pre-LN vs Post-LN

### 问题:LayerNorm 到底放残差的哪一侧

残差和归一化在结构上必须配合。有两种放法,差别决定了深层 Transformer 能不能直接训。

```
Post-LN(原始 Transformer 2017):  x ← LayerNorm( x + Sublayer(x) )
                                            └──残差在 LN 内部──┘

Pre-LN(GPT-2 之后,现代主流):    x ← x + Sublayer( LayerNorm(x) )
                                       └─跳跃连接在 LN 外部─┘
```

### 推导:为什么现代大模型几乎都用 Pre-LN

回到第一节的梯度高速公路。残差能让梯度直达浅层,**前提是那条跳跃连接上没有东西衰减梯度**。看两种放法的跳跃路径:

```
Post-LN:  梯度回传到浅层,必须穿过每一层的 LayerNorm(LN 在残差相加之后)。
          每过一个 LN,梯度被它的雅可比缩放一次 → "纯恒等"路径被破坏,深层梯度仍可能失控。

Pre-LN:   跳跃连接 x 是"裸"的,LN 只作用在残差分支内部。
          ∂hₗ/∂hₗ₋₁ = I + ∂/∂hₗ₋₁[ Sublayer(LN(hₗ₋₁)) ]
          那个 I 是干净的恒等项,没有任何 LN 挡在主干路上 → 梯度高速路完整保留。
```

**结论:Pre-LN 保住了那条"恒等项 I"不被 LN 污染,所以深层 Pre-LN Transformer 不需要复杂的 warmup 也能稳定训练;Post-LN 则对学习率、warmup 极其敏感,层一深就容易发散。** 这是现代大模型(GPT-2 起,几乎所有 LLM)清一色用 Pre-LN 的根本原因。代价是 Pre-LN 的主干 `x` 会随深度累积(没有 LN 把它拉回),所以**整个网络最后通常再补一个 final LayerNorm** 把输出尺度收住。

权衡也要诚实说:Post-LN 在层数不太多、调好 warmup 时,最终质量有时略优(因为它对每层输出都归一化,正则更强),早期一些工作偏好它。但**在"能不能稳定训深"这个第一优先级上,Pre-LN 完胜**,所以成了默认。

### 代码:一个完整的 Pre-LN Transformer 块

```python
class TransformerBlock(nn.Module):
    """Pre-LN + 残差 + RMSNorm + SwiGLU,现代 LLM 的标准块形态。"""
    def __init__(self, dim, attn):
        super().__init__()
        self.norm1 = RMSNorm(dim)
        self.attn  = attn                       # 自注意力(前面章节实现)
        self.norm2 = RMSNorm(dim)
        self.ffn   = SwiGLU_FFN(dim)

    def forward(self, x):
        x = x + self.attn(self.norm1(x))        # Pre-LN:norm 在残差分支内
        x = x + self.ffn(self.norm2(x))         # 跳跃连接 x 全程裸露,梯度高速路完整
        return x
```

## 六、设计权衡与常见坑

- **ε 放在根号里面还是外面、放多大都有讲究。** LN 是 √(σ²+ε)、RMSNorm 是 √(mean(x²)+ε),ε 在根号内。放小了(如 1e-8)在 bf16 下可能被舍入吃掉失去护栏作用;放大了会在激活方差本来就小的层引入偏差。RMSNorm 因不减均值,RMS 通常比 LN 的 std 略大,ε 取值常更小(1e-6)。
- **RMSNorm/LN 的归约务必在 fp32 做。** 低精度下平方和 / 方差容易溢出或丢精度,先升精度算统计量再降回。这是 NaN 的常见来源。
- **Post-LN 直接训深 = 自找麻烦。** 若你复现某论文用了 Post-LN 还不稳,先确认 warmup、学习率,或直接换 Pre-LN。别在 Post-LN 上死磕大学习率。
- **SwiGLU 忘了缩 ⅔ 隐层 = 偷偷多加 50% 参数。** 这会让你的"SwiGLU vs 传统 FFN"对比不公平,误判收益来源。要么对齐参数量,要么对齐计算量,明确说清比的是哪个。
- **去掉 β 不等于去掉所有 bias 都安全。** RMSNorm 去掉 β 是经过验证的;但盲目把网络里所有 Linear 的 bias 都删掉,有时影响小有时影响表达,要 ablation,别想当然。
- **死神经元未必报错,只是悄悄浪费容量。** 用 ReLU 时,监控"预激活恒为负的神经元比例";比例高就调小学习率、改初始化,或换 GELU/SiLU。
- **归一化是 memory-bound,别只盯 FLOPs。** 评估归一化开销要看访存和 kernel 启动,不是浮点运算数;这也是 RMSNorm 省一遍归约值钱的原因(呼应 MLSys 课的 kernel fusion)。
- **γ 初始化为 0 的技巧。** 把残差分支末端(或 LN/RMSNorm 的 γ)初始化成 0,让网络从纯恒等堆叠出发,极深网络训练初期更稳——这是很多大模型的隐藏配方。

## 七、动手练习

1. **(推导题)手推 LayerNorm 反向。** 从 yᵢ = γᵢ·x̂ᵢ + βᵢ、x̂ᵢ=(xᵢ−μ)/√(σ²+ε) 出发,完整推出 ∂L/∂xᵢ,务必把"经 μ 的路径"和"经 σ² 的路径"分别写出来再合并,得到 `∂L/∂x = (1/std)(dx̂ − mean(dx̂) − x̂·mean(dx̂⊙x̂))`。*提示*:先算 ∂L/∂σ²、∂L/∂μ 两个标量梯度,再把 xᵢ 通过直接项、μ、σ² 三条路径的贡献加起来;推完用 numpy 写有限差分(中心差分 h=1e-6)验证,目标误差 < 1e-7。

2. **(分析题)BN 为什么不适合变长序列。** 用 [B, T, D] 张量写清 BN 和 LN 各自沿哪些维统计,然后具体论证:(a) 推理 batch=1 时 BN 的统计量从哪来、为什么不稳;(b) 序列 pad 到不等长时 BN 跨 (B,T) 求和会被 padding 污染、为什么 LN 免疫;(c) BN 训练/推理用不同统计量,在序列分布随位置变化时为什么容易失配。*提示*:核心是 BN 的统计量"跨样本",LN"样本内"。

3. **(编码题)实测死神经元 + 对比三种激活。** 搭一个 4 层 MLP,分别用 ReLU / GELU / SiLU 训同一个回归任务。每个 epoch 统计"预激活对整个 batch 恒为负的神经元比例"(ReLU 死神经元指标),画出三条激活的收敛曲线和死神经元曲线。*提示*:故意用偏大的学习率放大 ReLU 死神经元现象;GELU/SiLU 因负区梯度非零,死神经元比例应显著更低。

4. **(编码 + 分析题)SwiGLU 参数对齐对比。** 实现 SwiGLU-FFN 和传统 ReLU-FFN,先验证"SwiGLU 隐层取 ⅔×(4d) 时两者参数量相等",再在同一任务上对比收敛。然后做消融:把 SwiGLU 隐层也设成 4d(不缩 ⅔),观察参数多 50% 后效果变化,讨论"SwiGLU 的提升有多少来自门控机制、多少来自参数量"。*提示*:打印两种配置的 `sum(p.numel())` 确认参数账;公平对比必须先对齐参数或计算量。

## 八、源码 / 论文导读

- **残差**:He 等《Deep Residual Learning for Image Recognition》(ResNet,2015)——读引言里对 degradation 的实验描述,以及残差块那一页;这是"训练误差随深度反升"现象的原始出处。
- **LayerNorm**:Ba、Kiros、Hinton《Layer Normalization》(2016)——读它与 BatchNorm 的对比一节,理解"沿特征维 vs 沿 batch 维"。
- **RMSNorm**:Zhang & Sennrich《Root Mean Square Layer Normalization》(2019)——读它对"re-centering 是否必要"的消融,这是消融驱动简化的范本。
- **GELU**:Hendrycks & Gimpel《Gaussian Error Linear Units (GELUs)》(2016)——看 x·Φ(x) 的动机和 tanh 近似式。
- **门控 FFN**:Dauphin 等《Language Modeling with Gated Convolutional Networks》(GLU,2016)给出门控线性单元;Shazeer《GLU Variants Improve Transformer》(2020)给出 SwiGLU 并在**参数对齐**下做消融——重点看它怎么把隐层缩到 ⅔ 做公平对比。
- **Pre-LN 分析**:Xiong 等《On Layer Normalization in the Transformer Architecture》(2020)——理论分析 Pre-LN 为何能免 warmup 稳定训深,印证本章梯度高速路论证。
- **开源实现**:
  - **nanoGPT**(Karpathy):`model.py` 里 `LayerNorm`、`Block`(Pre-LN 残差结构)、`MLP`——最干净的对照,先读它建立全局感。
  - **HF transformers**:`models/llama/modeling_llama.py` 的 `LlamaRMSNorm`(注意它把输入升 fp32 算 RMS)、`LlamaMLP`(SwiGLU,`down_proj(act(gate_proj(x)) * up_proj(x))`)——对照本章 RMSNorm / SwiGLU 代码逐行印证。
  - **x-transformers**(lucidrains):把各种 norm 位置、激活变体做成可配置开关,适合做练习里的消融对比。

## 九、小结与承上启下

这一章我们没有给 Transformer 增加任何表达能力,却让它从"堆深就崩"变成"能稳定训几十上百层"。三个稳定器各司其职:

- **残差** `y = x + F(x)` 把梯度的连乘改成连加,塞进一个恒等项 I 当梯度高速路,顺带把"学恒等"的难题降级成"关小残差分支"的易题;
- **归一化** 把每层输入尺度钉在固定范围:LN 沿特征维逐样本归一化,天然免疫 batch 大小 / 变长序列 / 训练推理切换(这三点正是 BN 在序列任务里的死穴);RMSNorm 进一步赌掉 re-centering,省一遍归约,成为现代 LLM 默认;
- **激活** 从 ReLU 的死区,进化到 GELU/SiLU 的平滑自门控,再到 SwiGLU 的数据依赖乘法门控(代价是多一个矩阵,用 ⅔ 隐层缩放做参数补偿);
- **Pre-LN** 把 LN 放进残差分支内部,保住裸露的跳跃连接和干净的恒等项,这是深层 Transformer 不需要繁琐 warmup 就能训的关键。

到这里,你手上已经有了一个**能稳定训练的 Transformer 块**:注意力 + SwiGLU-FFN,外面包 Pre-RMSNorm 和残差。但有一个我们一直回避的问题:注意力本身对 token 顺序是不敏感的(打乱输入顺序,注意力的输出只是跟着换位,不会"知道"谁在前谁在后)。语言显然是有序的——**模型怎么知道哪个词在第几个位置?** 这就是下一章**位置编码(positional encoding)**要解决的:从绝对位置编码到 RoPE 旋转位置编码,如何把"顺序"这个信息注入一个本质上无序的注意力机制里。我们已经把网络"训得动"了,下一步是让它"看得懂顺序"。
