> 这一章把你从"会写 PPO、能在 dense reward 环境里收敛"，带到"理解为什么长 horizon + 稀疏奖励是 RL 最硬的结构性难题，并能用势函数塑形、HER、内在奖励、课程学习这套工具系统性地把功劳分对、把样本效率提上来"。这是连接经典 RL 与后面 RLHF / agent 训练的关键一章——因为 agent 任务（多步工具调用、长链推理）的奖励几乎总是稀疏且延迟的。

## 1. 为什么单独讲这一章：信用分配是 RL 区别于监督学习的根

监督学习里没有"信用分配"问题。你给模型一张猫的图片，label 就是"猫"，梯度直接告诉每个参数往哪个方向走。**反馈是即时的、密集的、且精确对齐到每个输入。**

RL 不是。考虑一盘围棋：你下了 250 手，最后输了。**这 250 手里到底哪几手是败着？**reward 只在终局给一个 −1。你拿到的训练信号是"整条 250 步的轨迹，标量 −1"。把这 −1 的功过分摊回每一步动作——这就是**信用分配（credit assignment）**。它是 RL 的核心难题，没有之一。

形式化一下。一条长度为 T 的轨迹 τ = (s₀,a₀,r₀,s₁,a₁,r₁,…,s_{T-1},a_{T-1},r_{T-1},s_T)，策略梯度（REINFORCE）告诉我们：

```
∇θ J(θ) = E_τ[ Σₜ ∇θ log πθ(aₜ|sₜ) · Gₜ ]
其中 Gₜ = Σ_{k≥t} γ^{k-t} r_k   （从 t 时刻往后的回报）
```

注意 Gₜ：动作 aₜ 的梯度权重是**它之后所有奖励的折扣和**。这里藏着两个致命问题，本章要逐一拆解：

**问题 A（信用分配本身）**：如果 reward 只在终局非零（稀疏），那么对所有 t，Gₜ ≈ γ^{T-t} · r_T。同一条轨迹上所有动作只共享终局这一个标量信号，梯度无法区分某一步是好棋还是臭棋（且早期步还被 γ^{T-t} 进一步压低）——它们拿到的回报来自同一个终局奖励。**稀疏奖励让信用分配在数学上退化成"全体连坐"。**

**问题 B（长 horizon 的方差爆炸 + 误差累积）**：T 越大，Gₜ 这个随机变量的方差越大（后面会算）；同时如果你用 bootstrapping（如 TD），每步的估计误差会沿着 T 步链式累积放大。

这一章的全部内容，都是在回答：**奖励稀疏、延迟、horizon 很长时，怎么把功劳分对、怎么压住方差与误差累积。** 这正是 agent 训练（一个任务几十步工具调用，只有最后成功/失败一个信号）和推理模型训练（一道题几千 token，只有答案对错一个信号）的核心痛点。

## 2. 把"信用分配难"算清楚：稀疏奖励下梯度信号为什么退化

先把"难"量化，否则后面所有方法都是空谈。

### 2.1 稀疏奖励让 advantage 趋同

策略梯度的真正驱动量不是 Gₜ，而是 **advantage** Aₜ = Qπ(sₜ,aₜ) − Vπ(sₜ)，它衡量"在 sₜ 选 aₜ 比平均水平好多少"。理想的信用分配就是：每一步分到的功劳 = 这一步动作带来的 advantage。

现在假设奖励极端稀疏：只有终局 s_T 给 reward R，中间全为 0。那么对中间任意状态 sₜ：

```
Qπ(sₜ,aₜ) = E[ γ^{T-t} R | sₜ, aₜ ]
Vπ(sₜ)    = E[ γ^{T-t} R | sₜ ]
Aₜ = Qπ(sₜ,aₜ) − Vπ(sₜ)
   = γ^{T-t} ( E[R | sₜ,aₜ] − E[R | sₜ] )
```

两个观察：

1. **advantage 被 γ^{T-t} 指数压扁。** 离终局越远（t 越小），同样的"动作好坏差异" E[R|sₜ,aₜ]−E[R|sₜ] 被 γ^{T-t} 这个 < 1 的因子压得越小。γ=0.99、T−t=200 时，γ²⁰⁰ ≈ 0.134；T−t=500 时 γ⁵⁰⁰ ≈ 0.0067。**前期关键决策的信用信号几乎被折现到噪声水平之下。**

2. **estimation 噪声不衰减。** 我们不知道真实的 E[R|sₜ,aₜ]，只能用蒙特卡洛采样估。采样估计的标准差不随 γ 衰减，但真实信号衰减了——**信噪比随 horizon 指数恶化。**

这就是稀疏奖励让信用分配困难的**数学本质**：不是"信息没有"，而是"信息被折现 + 被噪声淹没"，且随 horizon 指数级恶化。

### 2.2 长 horizon 的方差爆炸（必须会算）

REINFORCE 估计量 ĝ = (Σₜ ∇log π(aₜ|sₜ)) · G₀。我们算它的方差量级。假设各步得分函数 ∇log π(aₜ|sₜ) 近似独立、每步方差 σ²_s，回报 G₀ = Σₜ γᵗ rₜ。

先看回报本身的方差。设每步 rₜ 独立、方差 σ²_r：

```
Var[G₀] = Var[ Σₜ γᵗ rₜ ] = Σₜ γ^{2t} σ²_r = σ²_r · (1 − γ^{2T}) / (1 − γ²)
当 T→∞ ：Var[G₀] → σ²_r / (1 − γ²)
```

关键看 **effective horizon** H_eff = 1/(1−γ)。γ=0.99 → H_eff=100；γ=0.999 → H_eff=1000。回报方差大致随 1/(1−γ²) ≈ H_eff/2 线性增长。**你为了能传播长程信用而调大 γ，代价就是方差线性放大。** 这是 RL 里最基本的一组矛盾：γ 大 → 能看到远处的 reward（信用能传过去），但 → 方差爆炸（学不稳）。

再看完整梯度估计量。粗略地，ĝ 把 T 项得分函数乘以一个高方差的 G₀，方差量级 ~ O(T · σ²_s · Var[G₀]) ~ O(T · H_eff)。**方差随 horizon 超线性增长**——这就是为什么纯 MC 策略梯度在长 horizon 上几乎不可用，必须靠 baseline、GAE、价值函数 bootstrapping 来压（这部分是上一章 actor-critic / GAE 的内容，本章默认你已掌握，专注信用分配维度）。

> 小结一组对偶矛盾，记牢：**γ↑ → bias↓（看得远，信用传得到）但 variance↑；bootstrapping（TD）→ variance↓ 但 bias↑（且误差沿 horizon 累积）。** 长程稀疏问题就卡在这个三角里，下面所有方法本质都是在重塑这个权衡。

## 3. 奖励塑形与势函数定理：在不改最优策略的前提下加密信号

### 3.1 朴素 reward shaping 的陷阱

最直接的想法：既然终局才有 reward，那我手工加一些中间奖励——比如机器人离目标近了就给正分。形式上把奖励改成 R'(s,a,s') = R(s,a,s') + F(s,a,s')，F 是塑形项。

**坑：随便加 F 会改变最优策略。** 经典反例：你给"离目标近"加奖励，但如果 F 设计不当，agent 会发现一个**刷分回路**——在靠近目标的某个圈里反复横跳，每次"靠近"都拿塑形奖励，永远不真正到达目标。**这就是 reward hacking 的雏形：agent 优化了你写的 proxy reward，而不是你真正想要的 true reward。** 新手在 RL 里最常踩、也最难 debug 的坑，几乎都是 reward 设计出的 hacking。

我们需要一个**充分条件**：什么样的 F 能保证"加密了学习信号、但绝不改变最优策略"？答案是 Ng, Harada & Russell (1999) 的 **potential-based shaping（基于势函数的塑形）**。

### 3.2 定理与完整证明（本章核心推导）

**定理（策略不变性）**：设势函数 Φ : S → ℝ（任意实值状态函数）。若塑形项取

```
F(s,a,s') = γ·Φ(s') − Φ(s)
```

则原 MDP M=(S,A,P,R,γ) 与塑形后 MDP M'=(S,A,P,R+F,γ) 拥有**完全相同的最优策略集**。更精确地，二者的最优 Q 函数差一个只依赖状态的常数项：

```
Q*_{M'}(s,a) = Q*_M(s,a) − Φ(s)
```

**证明。** 我们直接验证：如果 Q*_M 满足 M 的 Bellman 最优方程，那么 Q'(s,a) := Q*_M(s,a) − Φ(s) 满足 M' 的 Bellman 最优方程。由 Bellman 最优解的唯一性（γ<1 时 Bellman 最优算子是压缩映射，不动点唯一），这就证明了 Q'=Q*_{M'}。

M 的 Bellman 最优方程：

```
Q*_M(s,a) = E_{s'~P(·|s,a)}[ R(s,a,s') + γ · max_{a'} Q*_M(s',a') ]      (★)
```

M' 的奖励是 R'=R+F，写出 M' 的 Bellman 最优算子作用在候选解 Q'(s,a)=Q*_M(s,a)−Φ(s) 上，看是否自洽。

先算 M' 方程的右边 RHS：

```
RHS = E_{s'}[ R'(s,a,s') + γ · max_{a'} Q'(s',a') ]
    = E_{s'}[ R(s,a,s') + F(s,a,s') + γ · max_{a'} ( Q*_M(s',a') − Φ(s') ) ]
```

代入 F(s,a,s') = γΦ(s') − Φ(s)。注意 Φ(s') 与 max 里的 a' 无关，可提到 max 外：

```
γ · max_{a'} ( Q*_M(s',a') − Φ(s') ) = γ · max_{a'} Q*_M(s',a') − γ·Φ(s')
```

代回：

```
RHS = E_{s'}[ R(s,a,s') + (γΦ(s') − Φ(s)) + γ·max_{a'}Q*_M(s',a') − γΦ(s') ]
```

把 +γΦ(s') 和 −γΦ(s') 消掉：

```
RHS = E_{s'}[ R(s,a,s') + γ·max_{a'}Q*_M(s',a') ] − Φ(s)
```

E_{s'} 里那一坨正是 (★) 的右边，等于 Q*_M(s,a)。所以：

```
RHS = Q*_M(s,a) − Φ(s) = Q'(s,a) = LHS    ✓
```

候选解 Q' 精确满足 M' 的 Bellman 最优方程。由压缩映射不动点唯一，Q*_{M'} = Q*_M − Φ。

**推出策略不变性。** 在任意状态 s 下：

```
π*_{M'}(s) = argmax_a Q*_{M'}(s,a) = argmax_a ( Q*_M(s,a) − Φ(s) )
```

Φ(s) 是常数（对 a 不变），减一个与 a 无关的常数不改变 argmax：

```
           = argmax_a Q*_M(s,a) = π*_M(s)    ∎
```

证毕。**最优策略逐状态相同。**

### 3.3 直觉：为什么"差分形式"是关键，以及 telescoping 视角

为什么偏偏是 γΦ(s')−Φ(s) 这个差分形式？换个角度看——沿一条轨迹累加塑形奖励，它会**望远镜式（telescoping）地全部抵消**，只剩端点：

```
Σ_{t=0}^{T-1} γᵗ F(sₜ,sₜ₊₁)
 = Σₜ γᵗ ( γΦ(sₜ₊₁) − Φ(sₜ) )
 = Σₜ ( γ^{t+1}Φ(sₜ₊₁) − γᵗΦ(sₜ) )
 = γᵀΦ(s_T) − Φ(s₀)        （中间项两两抵消）
```

所以塑形给整条轨迹增加的总回报 = γᵀΦ(s_T) − Φ(s₀)。如果起点 s₀ 固定，且 episodic 任务里终止态 Φ(s_T)=0（或 T→∞ 时 γᵀ→0），那么**塑形对任何策略的轨迹回报增加的是同一个常数 −Φ(s₀)**。所有策略的回报被平移同一个量，排序不变 → 最优策略不变。这是定理的"轨迹视角"，和上面的"Bellman 视角"互为印证。

**这就是势函数塑形的精髓：它本质是给状态价值估计塞进先验（Φ 就是你猜的 V 的形状），让中间步骤立刻有了密集的、可区分的学习信号，但因为是差分/望远镜结构，长期回报排序丝毫不变——你白嫖了密集信号，没付改变最优解的代价。**

### 3.4 怎么选 Φ、代价是什么

- **理想 Φ ≈ V\*。** 如果 Φ 恰好等于最优状态价值 V*，那么塑形后 advantage 在每一步都被对齐到最优——信用分配瞬间完美。当然你不知道 V*，但任何对它的合理近似（如 "到目标的负距离"）都能把信号大幅加密。
- **代价 1：你不知道 V\*，错的 Φ 只是不改最优策略，但会改变学习动力学。** Φ 形状差，会引导探索走弯路、甚至先变慢再变快。策略不变 ≠ 收敛速度不变。
- **代价 2：动作依赖的塑形 F(s,a,s') 一般会破坏不变性。** 定理只对"状态势差"成立。Wiewiora 等把它推广到 potential-based advice（含动作的势），但要求 Φ(s,a) 与策略的 Q 一致地构造，超出本章范围（标记为进阶阅读）。**新手常犯：把塑形项写成依赖动作的形式，结果偷偷改了最优策略还不自知。**
- **实践锚点：** 在 RLHF / 推理训练里，"过程奖励模型（PRM）给每步打分"本质就是一种 reward shaping。它**不保证**是势函数形式，因此**可能改变最优策略**、诱发 reward hacking——这是当前推理模型训练里一个真实且活跃的风险点（后续 RLHF 章节展开）。

## 4. HER：从失败中也学到东西（事后诸葛亮式重标）

势函数塑形要求你**会设计 Φ**。但很多 goal-conditioned 任务（"把方块推到坐标 g"）你压根不知道好的 Φ，而且 reward 是二值稀疏的：到达目标 g 给 +0，否则 −1（或到达给 1 否则 0）。随机策略几乎永远到不了 g，于是**几乎每条轨迹的 reward 都一样（全是失败）**，梯度里没有任何可区分信号——纯随机探索撞不到目标，就永远学不动。

**Hindsight Experience Replay（HER，Andrychowicz et al. 2017）** 的洞见极其朴素却深刻：

> 就算你没到达目标 g，你总归到达了**某个**状态 s'。那就假装 s' **本来就是**你的目标——相对于"到达 s'"这个目标，你这条轨迹是**成功**的！

也就是说，一条"失败抵达 g"的轨迹，对"目标 = 你实际抵达的某个状态"而言是完美示范。**失败轨迹被事后重标（relabel）为成功轨迹**，从而产生大量非平凡的正信号。这要求两个前提：(1) 任务是 **goal-conditioned** 的，策略和奖励都吃目标 g：π(a|s,g)、r(s,a,g)；(2) 奖励可以对任意重标目标重新计算——通常 r(s,a,g)=−[‖achieved(s') − g‖ > ε]，给定 s' 和任意 g 都能算。

### 4.1 HER 的目标重标伪代码

```python
# HER —— 在 off-policy 算法（如 DDPG/SAC/DQN）的 replay buffer 写入阶段做目标重标
# 关键：每条 transition 既按"原目标"存一份，也按若干"事后目标"重标后各存一份

def store_episode_with_her(episode, buffer, reward_fn, k=4, strategy="future"):
    """
    episode: [(s_t, a_t, s_{t+1}, g)  for t in 0..T-1]，g 是该回合的原始目标
    reward_fn(s_next, g): 给定到达状态与目标，重新计算奖励（HER 的硬前提：奖励可重算）
    k: 每个 transition 额外采样多少个事后目标
    strategy: 事后目标的采样策略，"future" 是论文里最有效的
    """
    T = len(episode)
    for t in range(T):
        s, a, s_next, g = episode[t]

        # (1) 原始 transition：用真实目标 g，奖励通常是稀疏的失败信号
        buffer.add(s, a, s_next, g, reward_fn(s_next, g))

        # (2) 事后重标：采样 k 个"假装的目标"
        for _ in range(k):
            g_new = sample_hindsight_goal(episode, t, strategy)
            # 用同一段真实经验 (s,a,s_next)，但换上重标目标 g_new
            # 注意：s,a,s_next 不变（物理转移与目标无关），只有 g 和 reward 变
            r_new = reward_fn(s_next, g_new)   # 相对 g_new，这步很可能是"成功"，给出非平凡信号
            buffer.add(s, a, s_next, g_new, r_new)


def sample_hindsight_goal(episode, t, strategy):
    T = len(episode)
    if strategy == "future":
        # 从 t 之后（含 t）实际到达的状态里随机挑一个，把它的 achieved-goal 当作目标
        # 直觉：这个目标"在未来真的被达成了"，所以这条 (s,a) 对它是成功示范
        j = random.randint(t, T - 1)
        return achieved_goal(episode[j][2])            # episode[j][2] 即该 transition 的 s_next
    elif strategy == "final":
        return achieved_goal(episode[T - 1][2])        # 用整段最后到达的状态当目标
    # 还有 "episode" / "random" 等策略，future 通常最优
```

落地要点：HER **必须**配 off-policy 算法（DDPG/SAC/DQN 这类用 replay buffer 的），因为重标后的 (s,a,s_next,g_new,r_new) 是"假目标下的经验"，相对当前策略是 off-policy 的——on-policy 的 PPO 不能直接这么用（这是新手常见误配）。

### 4.2 为什么 HER 提升样本效率：信号密度的视角

定量直觉。设目标空间命中半径 ε，随机策略一条长度 T 的轨迹**恰好到达原目标 g** 的概率为 p（稀疏任务里 p 极小，可能 1e-4 量级）。无 HER 时，正信号出现的期望要等 ~1/p 条轨迹——样本几乎全浪费。

HER 把"成功"的定义从"到达唯一的 g"放宽成"到达你实际经过的任意状态"。**每一条轨迹都至少'成功到达'了它自己经过的 T 个状态。** 于是非平凡（非全等）信号的产出率从 ~p 跃升到接近 1——**每条轨迹都贡献可学习的信号**。样本效率的提升量级大致是 1/p：原来 1/p 条轨迹才出一个正信号，现在条条都出。

更深一层：HER 实际上是**自动构造了一门课程（curriculum）**。训练早期策略很烂，只能到达离起点近的状态，于是 future 策略采样到的目标也都是"近且容易达成"的——agent 先学会近目标；随着策略变强能走更远，重标目标自然变难。**难度随能力自动爬升，无需人工设计课程**——这是 HER 被反复称道的隐藏红利。

> 一句话记牢：**势函数塑形是"我替你把信号加密（注入先验 Φ）"；HER 是"我把你的失败重新定义成成功（重标目标）"。前者要你懂任务结构，后者只要任务是 goal-conditioned + 奖励可重算。**

## 5. 内在奖励与好奇心：当连失败轨迹都没信息时

HER 要求任务 goal-conditioned。但很多探索难题（Montezuma's Revenge 这类，几百步才碰到第一个外部奖励）连目标都没有显式坐标。这时需要**内在奖励（intrinsic reward）**：让 agent 对"新颖/意外"本身产生奖励，从而在外部奖励出现前就主动探索。

总奖励 r = r_ext + β · r_int，β 是内在权重。两条主流构造：

**(a) 基于计数 / 伪计数的新颖性奖励。** 给访问次数少的状态更高奖励：r_int(s) = 1/√N(s)，N(s) 是访问计数。状态空间连续时用密度模型估计**伪计数（pseudo-count）**（Bellemare et al. 2016）。直觉：鼓励去没去过的地方。

**(b) 基于预测误差的好奇心（ICM，Pathak et al. 2017）。** 训一个前向模型预测下一状态的特征 φ(sₜ₊₁)，把**预测误差**当内在奖励：

```
r_int(sₜ,aₜ) = η · ‖ f_forward(φ(sₜ), aₜ) − φ(sₜ₊₁) ‖²
```

预测越不准（越意外）→ 内在奖励越高 → agent 越想去。ICM 的关键设计：φ 不是原始像素，而是用一个**逆动力学模型**（从 sₜ,sₜ₊₁ 反推 aₜ）学出的特征——这样 φ 只编码"agent 动作能影响的部分"，**过滤掉 agent 控制不了的随机噪声**。

**这里有个 RL 经典大坑——"噪声电视（noisy-TV）问题"。** 如果直接用原始观测的预测误差当好奇心，agent 会被一个永远随机闪烁的电视屏幕**永久吸引**：屏幕内容不可预测 → 预测误差永远很高 → 内在奖励永远很高 → agent 站着看电视再也不探索。**预测误差混淆了"我没学会"（epistemic，应该奖励）和"环境本质随机"（aleatoric，不该奖励）。** ICM 用逆动力学特征部分缓解；RND（Random Network Distillation, Burda et al. 2018）用"预测一个固定随机网络的输出"绕开——因为目标网络是确定性的，纯随机环境噪声不会带来持续误差。新手上手内在奖励，几乎一定先撞上 noisy-TV，务必心里有数。

内在奖励的另一个代价：**它是非平稳的 reward**（随着你越来越熟悉某状态，r_int 会衰减），这让价值函数的学习目标在动，训练更难稳定。通常要给 r_int 单独的 value head 或随训练 anneal β。

## 6. 课程学习与多智能体非平稳性（信用分配的两个延伸战场）

### 6.1 课程学习：把长 horizon 拆成可学的阶梯

**课程学习（curriculum learning）** 的思想：不要一上来就让 agent 解最难的稀疏长程任务，而是**从易到难排个序**，让它先在短 horizon / 密集信号的简化版本上学会基本技能，再逐步加难。这直接缓解信用分配——短任务里功劳好分，学到的子技能迁移到长任务后，相当于把长链拆成几段已经"分好功"的子链。

几种典型做法：

- **手工课程**：人工设计难度递增的任务序列（起点离目标由近到远、障碍由少到多）。
- **自动课程 / 目标生成**：用一个 generator 自动产出"当前刚好在能力边界上"的任务（如 Goal GAN）。最有效的难度是**成功率约 50% 的任务**——太易学不到新东西，太难没有信号。
- **逆向课程（reverse curriculum）**：从**紧挨目标的状态**开始训练（这时成功容易、信用好分），再逐步把起点往后挪。本质上是在用"离目标的距离"做难度轴。
- **HER 是隐式自动课程**（第 4.2 已述）：无需显式设计，难度随能力自然爬升。

课程学习与势函数塑形是互补的两种"注入先验"方式：塑形改奖励（信号维度），课程改任务分布（数据维度）。

### 6.2 多智能体的非平稳性（简述，点到为止）

当环境里有多个同时学习的 agent，信用分配难度再上一个数量级，叠加一个新难题：**非平稳性（non-stationarity）**。

从任一个 agent i 的视角看，其他 agent j 的策略 π_j 在不断变化（它们也在学）。于是 agent i 面对的"环境转移 + 奖励"实际是 P(s'|s,a_i; π_{-i}) ——**依赖于别人不断变化的策略**。这意味着：

- **环境对 i 而言不再是平稳 MDP**，违反了 Q-learning / 策略梯度收敛性证明的基础假设（那些证明都要求转移核固定）。i 昨天学到的 Q 值，今天因为别人变了策略就失效了。
- **多智能体信用分配（multi-agent credit assignment）**：团队拿到一个共享 reward，到底是哪个 agent 的功劳？（如 QMIX 用可分解的联合 value、COMA 用反事实 baseline 来分摊。）

主流缓解思路是**集中训练、分散执行（CTDE, centralized training with decentralized execution）**：训练时用一个能看到全局信息（含他人动作）的 critic 来稳定学习目标，执行时每个 agent 只用自己的局部观测。这把非平稳性收进训练阶段。本章只点出这是一个独立战场，细节留给多智能体专题。

## 7. 长程 agent 的误差累积：每步小错如何级联放大

最后回到本章一开始的问题 B，把"误差累积"算清楚——这对理解后面 **agent 训练**（多步工具调用）至关重要。

### 7.1 行为克隆 / 自回归 rollout 的二次型误差（compounding error）

考虑一个学到的策略每步以概率 ε 犯一个"会让你偏离训练分布"的错。关键在于：**一旦偏离，你进入了训练时没见过的状态分布，后续动作更可能错**——错误不是独立叠加，而是级联放大。

经典结果（Ross & Bagnell 的 behavior cloning 分析，DAgger 论文 2011 的核心引理）：若每步偏离训练分布的概率为 ε，horizon 为 T，则相对专家的总代价上界是 **O(ε · T²)**，而不是朴素期待的 O(ε·T)。

直觉推导（白板版）：

```
设第 t 步首次"脱轨"的概率 ~ ε（在还没脱轨的前提下）。
一旦在第 t 步脱轨，从 t 到 T 这段（约 T−t 步）都处于分布外，
   每步都可能继续累积代价，这段的代价 ~ O(T − t)。
对首次脱轨时刻 t 求期望（脱轨大致均匀地可能发生在任意一步）：
   总代价 ~ Σ_{t=1}^{T} P(首次脱轨在 t) · O(T − t)
         ~ ε · Σ_{t=1}^{T} (T − t)
         ~ ε · T²/2  =  O(ε·T²)
```

**关键就是那个 T²。** horizon 翻倍，误差不是翻倍而是翻四倍。这解释了为什么：

- **长链推理 / 长程 agent 极其脆弱**：一道 50 步的推理，即使每步正确率 99%，全对的概率约 0.99⁵⁰ ≈ 0.605——单步看着很高，全程一塌糊涂。50 步降到 0.6，100 步就掉到 0.99¹⁰⁰ ≈ 0.366。**这是 horizon 对乘性正确率的指数惩罚，和上面的二次型分布漂移共同构成长程 agent 的两道天花板。**
- **必须 on-policy 纠偏**：DAgger 的解法是让专家在 agent **自己访问到的（可能脱轨的）状态**上标注，把 O(εT²) 压回 O(εT)。这正是为什么 agent 训练强调用 RL（on-policy，在自己 rollout 的状态上学）而非纯 behavior cloning（off-policy，只在专家分布上学）。

### 7.2 把这一章的工具映射到长程 agent 训练

```
长程 agent 的痛点              本章对应的武器
────────────────────────────────────────────────────────────
奖励只在任务末尾（成功/失败） → 势函数塑形 / 过程奖励(PRM, 注意 hacking)
                                HER（若任务 goal-conditioned）
中间步骤无信号、探索撞不到    → 内在奖励 / 好奇心驱动探索
任务太难、一步到位学不动      → 课程学习（由易到难 / 逆向课程）
每步小错级联（O(εT²)）        → on-policy 纠偏(DAgger 思想)、价值 bootstrapping 压方差
长 horizon 方差爆炸          → baseline / GAE / 较大但受控的 γ（上一章）
```

## 8. 设计权衡与常见坑（务必内化）

- **γ 的双刃剑**：γ↑ 让远处 reward 的信用能传播回来（解决稀疏），但方差随 ~1/(1−γ²) 放大，且 bootstrapping 误差沿更长链累积。长程任务往往要 γ≈0.99~0.999，必须同时上强 baseline / GAE。
- **reward shaping 的 hacking 风险**：只有 **γΦ(s')−Φ(s)** 形式的势函数塑形被证明不改最优策略。任何 ad-hoc 中间奖励（尤其依赖动作、依赖时间、可重复触发的）都可能被 agent 找到刷分回路。**上线前务必问自己：这个塑形项是不是势函数形式？如果不是，最优策略可能被我悄悄改了。**
- **PRM / 过程奖励不是势函数**：RLHF / 推理训练里给每步打分的过程奖励模型，一般**不**满足势函数形式，因此**有改变最优策略、诱发 hacking 的风险**——这是当前活跃的研究与工程风险点（具体缓解手段待核，留给 RLHF 章节）。
- **内在奖励的 noisy-TV**：直接用观测预测误差当好奇心，会被纯随机源（闪烁屏、骰子）永久套牢。用逆动力学特征（ICM）或固定随机网络蒸馏（RND）缓解。且内在奖励是非平稳的，建议单独 value head + anneal β。
- **HER 必须配 off-policy**：重标经验对当前策略是 off-policy 的，只能进 DDPG/SAC/DQN 的 replay buffer，**不能直接喂 on-policy 的 PPO**。
- **课程的难度甜点**：最有效的训练任务是成功率 ~50% 的任务；太易无增量，太难无信号。自动课程的核心就是把任务难度持续锚在能力边界上。
- **别把"策略不变"当成"学习速度不变"**：势函数塑形保证最优解不变，但 Φ 选得差会显著拖慢甚至误导收敛——它改变的是学习动力学，不是终点。

## 9. 动手练习

**练习 1（推导题，核心）：势函数塑形的策略不变性。**
设 Φ:S→ℝ，塑形项 F(s,a,s')=γΦ(s')−Φ(s)。
(a) 不照抄正文，自己写出原 MDP 与塑形 MDP 各自的 Bellman 最优方程，验证 Q*_{M'}=Q*_M−Φ 满足塑形 MDP 的方程，并说明为什么"满足方程"就等于"它就是最优解"（提示：γ<1 时 Bellman 最优算子是 ‖·‖_∞ 下模为 γ 的压缩映射，不动点唯一）。
(b) 用 telescoping 给出第二种证明：计算 ΣₜγᵗF(sₜ,sₜ₊₁)，说明它对所有策略只差一个依赖 s₀ 的常数，从而轨迹回报排序不变。
(c) 举一个**依赖动作**的塑形 F(s,a,s') 的反例，说明它会改变最优策略（提示：构造一个对某个 a 额外加正奖励的 F，让一个原本次优的动作变成最优）。

**练习 2（分析题）：为什么稀疏奖励让信用分配指数级困难。**
在只有终局 reward R、中间为 0、γ<1 的设定下：
(a) 证明对中间状态 sₜ，advantage Aₜ = γ^{T-t}(E[R|sₜ,aₜ]−E[R|sₜ])，并解释 γ^{T-t} 这个因子对"前期关键决策可学习性"的指数级压制。
(b) 取 γ=0.99，分别算 t 离终局 100、300、500 步时信用信号被压缩的倍数。
(c) 结合 2.2 节，说明为什么"调大 γ 让信用传得更远"和"方差爆炸"是一对不可兼得的矛盾，给出 Var[G₀] 随 γ 的表达式。

**练习 3（分析题）：HER 为何提升样本效率。**
(a) 设随机策略一条轨迹到达原目标 g 的概率为 p，用 HER（k 个 future 目标）后，定量说明非平凡正信号的产出率如何从 ~p 提升到接近 1，并估计样本效率提升的量级。
(b) 解释为什么 HER 等价于一个**自动课程**：early 训练时 future 采样到的目标有什么特征？随训练如何变化？
(c) 为什么 HER 不能直接用在 on-policy 的 PPO 上？（提示：重标经验相对当前策略的 on/off-policy 性质。）

**练习 4（编码题）：在简单 grid-world 上对比稀疏 vs 势函数塑形。**
实现一个 N×N grid-world：起点左上、目标右下，到达目标 reward +1，其余 step reward 0（稀疏），γ=0.99。
(a) 跑 tabular Q-learning，记录"首次到达目标"所需的 episode 数。
(b) 加入势函数 Φ(s) = −曼哈顿距离(s, goal)，塑形项 F=γΦ(s')−Φ(s)，重跑，对比收敛速度。
(c) 故意把塑形项改成**非势函数**形式（如"每次靠近目标一格 +0.1，且不减去 Φ(s)"），观察 agent 是否学出刷分回路（在某些格子反复横跳）。把三条学习曲线画在一起。
提示：tabular Q-learning 几十行即可；关键是观察 (b) 收敛显著加快、(a) 极慢、(c) 出现 reward hacking。

## 10. 源码 / 论文导读

- **Ng, Harada, Russell (1999), "Policy Invariance under Reward Transformations"**：势函数塑形的原始论文。重点读定理 1 及其证明（即本章 §3.2），以及他们对"为什么必须是势差形式"的必要性讨论（不仅充分，在一定意义上也是必要的）。
- **Wiewiora et al. (2003), potential-based advice**：把势函数推广到含动作的 Φ(s,a)，是练习 1(c) 的进阶答案——动作依赖塑形要怎么改造才仍保持不变性。（进阶）
- **Andrychowicz et al. (2017), "Hindsight Experience Replay" (NeurIPS)**：HER 原文。重点读 Section 3（goal relabeling 的形式化）和 4（future/final/episode/random 四种策略的消融）。配合 OpenAI Baselines / Stable-Baselines3 里 `HerReplayBuffer` 的实现读，看工业级重标是怎么和 replay buffer 解耦的。
- **Pathak et al. (2017), ICM "Curiosity-driven Exploration"**：内在奖励 + 逆动力学特征。重点读它如何用逆模型学特征以过滤不可控噪声（noisy-TV 缓解）。**Burda et al. (2018), RND**：更简单稳健的内在奖励基线，工程上常用。
- **Bellemare et al. (2016), 伪计数（pseudo-counts）**：把"访问计数"推广到高维连续状态的密度模型视角。
- **Ross, Gordon, Bagnell (2011), DAgger "A Reduction of Imitation Learning..."**：§7.1 的 O(εT²) 复合误差与 on-policy 纠偏的来源。读它的 no-regret 分析，理解为什么必须在 agent 自己访问的状态上学习。
- **开源代码**：
  - **CleanRL**：单文件实现，读它的 PPO + GAE 看长 horizon 方差怎么被 baseline/GAE 压（与本章 §2.2 互补）。
  - **Stable-Baselines3 `HerReplayBuffer`**：HER 的生产级实现，对照本章 §4.1 伪代码。
  - **TRL / verl**：RLHF / 推理模型训练框架。看它们怎么处理"只有序列末尾一个 reward"的稀疏信用分配，以及（若有）过程奖励/KL 正则的接入点——这是本章思想在大模型对齐里的直接落地（待核：各框架对过程奖励的具体支持程度随版本变化）。

## 11. 小结与承上启下

这一章把 RL 最硬的结构性难题之一拆透了：

1. **信用分配是 RL 的根本难题**——稀疏/延迟奖励让"哪一步动作负责"在数学上退化（advantage 被 γ^{T-t} 指数压扁、被噪声淹没），长 horizon 又放大方差（~1/(1−γ²)）和误差累积（O(εT²)）。
2. **势函数塑形**用一个被严格证明的差分结构 F=γΦ(s')−Φ(s)，在**不改变最优策略**的前提下注入密集先验信号——我们给了 Bellman 与 telescoping 两种完整证明，并点出它和过程奖励/reward hacking 的关系。
3. **HER** 用"事后重标目标"把失败轨迹变成成功示范，以 ~1/p 量级提升样本效率，且自带隐式课程。
4. **内在奖励**在连失败都无信息时驱动探索，但要警惕 noisy-TV 与非平稳性。
5. **课程学习**从数据分布维度拆解长 horizon；**多智能体非平稳性**是信用分配的延伸战场。

**承上**：本章默认你已掌握上一章的 actor-critic / GAE / baseline——那是压方差的"通用引擎"，本章是在它之上解决"信号本身稀疏/延迟/长程"的专门武器。

**启下**：所有这些思想会在后面的 **RLHF 与推理模型训练**章节里以新形态重现——RLHF 的奖励就是"整段回答一个标量"（极端稀疏 + 长 horizon），过程奖励模型（PRM）是 reward shaping 的现代变体（且**不保证**势函数性质，hacking 风险真实存在），KL 正则是另一种"塑形/约束"。理解了本章的信用分配第一性原理，你才能看懂为什么大模型对齐里那些训练技巧是必然而非偶然——它们都是在同一组 bias-variance-credit 矛盾里做的工程选择。
