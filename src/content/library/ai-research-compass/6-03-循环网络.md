> 这一章把你从"知道 RNN 是处理序列的网络"带到"能在白板上推出 BPTT 的梯度连乘项、说清梯度为什么会消失或爆炸、徒手实现一个 LSTM cell、并解释 cₜ 的梯度为什么是加性的从而把远程依赖学进来"。这是 Transformer 出现前序列建模的主力路线,把它的机制和它的死穴都讲透,你才会真正理解后面注意力为什么是一次范式跃迁。

## 一句话引言

循环网络的核心思想只有一句话:**用一个随时间更新的隐状态 hₜ,把"到目前为止读到的全部历史"压缩进一个固定维度的向量**,再用同一组权重在每个时刻复用。这个"参数共享 + 状态递推"的设计让它能处理任意长度的序列,但也埋下了致命问题——梯度沿时间连乘,要么消失要么爆炸。本章从递推方程推起,用链式法则把这个连乘项算出来,看清它为什么注定不稳定;然后看 LSTM 如何用一条加性的"细胞状态高速路"绕过连乘,把长程依赖救回来。

## 1. 动机与预备:为什么需要"循环"

### 1.1 序列建模的根本约束

上一章(02 语言模型基础)讲到,神经语言模型用一个固定窗口的前文(比如前 n−1 个词)经 MLP 估计 `P(wₜ | wₜ₋ₙ₊₁ ... wₜ₋₁)`。这有两个硬伤:

- **窗口固定**。你只能看到前 n−1 个词。要看更远,就得把网络第一层的参数量按 n 线性堆大,而且永远有个上限。
- **没有位置无关的复用**。"猫"这个词出现在窗口第 1 位还是第 3 位,在 MLP 里走的是完全不同的权重。语言里"同一个模式可以出现在任何位置"这个先验没被利用。

**循环网络(Recurrent Neural Network, RNN;Elman, 1990)的破局点**:不要固定窗口,而是维护一个**隐状态(hidden state)hₜ**,它在每读入一个新 token 时被更新一次:

```
hₜ = f(hₜ₋₁, xₜ)
```

`hₜ₋₁` 概括了"从开头到 t−1 的全部历史",`xₜ` 是当前输入。理论上,只要 hₜ 维度够大、f 够强,hₜ 就能记住任意远的历史——这是 RNN 区别于固定窗口模型的关键承诺。而且**同一个 f(同一组权重)在每个时刻复用**:这就是"参数共享在时间维度上的版本",对应 CNN 在空间维度上的卷积核共享。

### 1.2 预备:你需要熟悉的三件事

- **链式法则的矩阵形式**。若 z = g(y),y = h(x),则 ∂z/∂x = (∂z/∂y)(∂y/∂x),其中每个 ∂·/∂· 是雅可比矩阵(Jacobian)。多步复合就是雅可比连乘。
- **逐元素激活的雅可比是对角阵**。若 y = φ(a)(φ 逐元素作用),则 ∂y/∂a = diag(φ′(a)),一个对角矩阵,对角线是每个分量的导数。
- **矩阵范数与谱半径**。矩阵 W 的谱范数 ‖W‖₂ 等于它的最大奇异值;谱半径 ρ(W) 是最大特征值的模。连乘 Wᵏ 的增长率由这两个量主导:大致 ‖Wᵏ‖ 随 ρ(W)ᵏ 量级变化。**这是后面判断梯度消失/爆炸的核心工具。**

## 2. 普通 RNN:递推、展开与前向

### 2.1 递推方程

最常见的 Elman RNN,在时刻 t:

```
hₜ = tanh(W_h hₜ₋₁ + W_x xₜ + b_h)        # 隐状态更新
ŷₜ = softmax(W_y hₜ + b_y)                # 输出(以语言模型为例,预测下一个词)
```

维度:设隐状态维度为 d_h,输入维度为 d_x,词表大小为 V。则 W_h ∈ ℝ^{d_h×d_h},W_x ∈ ℝ^{d_h×d_x},W_y ∈ ℝ^{V×d_h}。**注意 W_h、W_x、W_y 不带时间下标——所有时刻共享同一组权重**。初始隐状态 h₀ 通常取零向量或可学习参数。

为什么激活用 tanh 而不是 ReLU?两个原因:(1)tanh 输出有界于 (−1, 1),让隐状态不会在递推中无限膨胀;(2)tanh 在 0 附近近似线性、关于原点对称,作为状态更新比较稳。代价是它两侧饱和(导数趋 0),这正是后面梯度消失的帮凶之一。

### 2.2 时间展开(unrolling)

RNN 是一个带环的图(hₜ 依赖 hₜ₋₁)。要算梯度,先把这个环按时间"展开"成一个很深的前馈网络:

```
x₁ → [cell] → h₁ → [cell] → h₂ → ... → [cell] → h_T
       ↑                ↑                       ↑
      h₀               h₁                     h_{T-1}
每个 [cell] 是同一组权重 (W_h, W_x, W_y) 的同一份拷贝
```

**关键观察**:展开后这是一个深度为 T 的网络,而且每一层的权重是绑定(tied)的。序列有多长,这个网络就有多深。一个 100 词的句子,就是一个 100 层的网络——而且这 100 层共享参数。深度网络的所有梯度病症(消失/爆炸),RNN 会因为"深 = 序列长"而无可避免地撞上。

### 2.3 前向代码

```python
import torch
import torch.nn as nn

class VanillaRNNCell(nn.Module):
    def __init__(self, d_x, d_h):
        super().__init__()
        self.W_h = nn.Linear(d_h, d_h, bias=False)
        self.W_x = nn.Linear(d_x, d_h, bias=True)   # bias 合并到这里

    def forward(self, x_t, h_prev):
        # x_t: (B, d_x), h_prev: (B, d_h)
        return torch.tanh(self.W_h(h_prev) + self.W_x(x_t))

def run_rnn(cell, xs, h0):
    # xs: (T, B, d_x) —— 时间在第 0 维
    h = h0
    hs = []
    for t in range(xs.size(0)):
        h = cell(xs[t], h)      # 同一个 cell 在每步复用 → 参数共享
        hs.append(h)
    return torch.stack(hs)      # (T, B, d_h)
```

这段代码的 `for` 循环就是"循环"二字的字面含义,也是 RNN 不可并行的根源(2.4 节会回到这一点)。

## 3. BPTT:沿时间反向传播,以及梯度连乘从哪来

这一节是全章的数学核心。我们要把"损失对早期权重的梯度"写成显式的连乘,看清它为什么不稳定。

### 3.1 损失与总梯度

以语言模型为例,总损失是各时刻交叉熵之和:`L = Σₜ Lₜ`,其中 `Lₜ = −log P(目标词 | h_t)`。

我们关心 ∂L/∂W_h(对共享权重的梯度)。因为 W_h 在每个时刻都被用到,根据多元函数对共享变量求导的法则,要**把每个时刻经过 W_h 的贡献全部累加**:

```
∂L/∂W_h = Σ_{t=1}^{T} ∂Lₜ/∂W_h
```

而每个 Lₜ 依赖 hₜ,hₜ 依赖 hₜ₋₁、hₜ₋₂、……、h₁(链条一直回溯到序列开头)。所以:

```
∂Lₜ/∂W_h = Σ_{k=1}^{t}  (∂Lₜ/∂hₜ) · (∂hₜ/∂h_k) · (∂h_k/∂W_h)
                          └─────┬────┘   └────┬───┘   └────┬───┘
                          顶层误差信号    跨时间传播项     局部直接项
```

这就是 **BPTT(Backpropagation Through Time;Werbos, 1990;Rumelhart et al., 1986)**:把误差从时刻 t 一路反传到时刻 k。注意 `∂h_k/∂W_h` 是把 h_k 看作只在第 k 步显式依赖 W_h(其余依赖通过 h_{k−1} 走 ∂hₜ/∂h_k 那一项),这样不重不漏。整个推导的危险全部集中在中间那个 **跨时间传播项 ∂hₜ/∂h_k**。

### 3.2 跨时间传播项展开成连乘

`∂hₜ/∂h_k` 表示"第 k 步的隐状态扰动一点,会让第 t 步的隐状态变多少"。由链式法则,它是相邻步雅可比的连乘:

```
∂hₜ/∂h_k = (∂hₜ/∂hₜ₋₁)·(∂hₜ₋₁/∂hₜ₋₂)· ... ·(∂h_{k+1}/∂h_k)
         = ∏_{j=k+1}^{t}  ∂h_j/∂h_{j-1}
```

现在算单步雅可比 ∂h_j/∂h_{j−1}。回忆 h_j = tanh(W_h h_{j−1} + W_x x_j + b)。记预激活 a_j = W_h h_{j−1} + W_x x_j + b。对 h_{j−1} 求导,逐元素 tanh 贡献一个对角阵,线性项贡献 W_h:

```
∂h_j/∂h_{j-1} = diag(tanh′(a_j)) · W_h
```

其中 tanh′(a) = 1 − tanh²(a) ∈ (0, 1],在 a=0 处取最大值 1,两侧饱和趋 0。代回连乘:

```
∂hₜ/∂h_k = ∏_{j=k+1}^{t}  diag(tanh′(a_j)) · W_h
```

**这就是 RNN 的命门**:从第 k 步传到第 t 步,要把同一个矩阵 W_h 连乘 (t−k) 次,中间还各夹一个对角阵 diag(tanh′)。t−k 就是"依赖跨越的时间步数",也就是远程依赖的距离。距离越远,连乘项越多。

### 3.3 消失与爆炸:定量条件

对连乘做范数上界(用 ‖AB‖ ≤ ‖A‖·‖B‖):

```
‖∂hₜ/∂h_k‖ ≤ ∏_{j=k+1}^{t} ‖diag(tanh′(a_j))‖ · ‖W_h‖
           ≤ (γ · ‖W_h‖)^{t-k}
```

这里 γ = max tanh′ = 1(对角阵谱范数 = 对角元最大绝对值,而 tanh′ ≤ 1)。令 β = γ·‖W_h‖。于是:

```
‖∂hₜ/∂h_k‖  ≲  β^{t-k},   β = γ·‖W_h‖,  γ ≤ 1
```

**这是一个几何(指数)级数**,跨越距离 (t−k) 是指数。三种结局:

- **梯度消失(vanishing)**:若 β < 1(典型情况,因为 ‖W_h‖ 常被初始化/正则到 1 附近,而 tanh′ 在非零预激活处严格 < 1,把有效因子压到 1 以下)。则 β^{t-k} → 0 当 t−k 变大。**含义**:相距 50 步的两个 token 之间的梯度信号衰减到接近 0,模型学不到"50 步前那个词"对当前预测的影响——**远程依赖学不进来**。这不是优化没收敛,是梯度从源头就被几何衰减抹掉了。
- **梯度爆炸(exploding)**:若 β > 1(当 W_h 的谱半径明显大于 1)。则 β^{t-k} → ∞。**含义**:梯度数值溢出成 NaN/Inf,一步更新把参数打飞,loss 突然跳成天文数字。
- **临界**:β = 1 是不稳定的刀刃,实践中几乎不会刚好停在这。

**更精确的判据用谱半径**:乘积的增长率本质由 W_h 的谱半径 ρ(W_h) 决定(配合 tanh′ 的收缩)。Pascanu, Mikolov & Bengio(2013)给出的充分条件是:**最大奇异值 σ_max(W_h)=‖W_h‖₂ < 1/γ 时梯度必然消失**(因为每一步都是收缩映射;这里用的是谱范数而非谱半径,与上文 (γ·‖W_h‖)^{t−k} 的 ‖W_h‖ 一致——谱半径只出现在爆炸的必要条件 ρ(W_h)>1/γ 里)。要长期不消失,需要 ρ(W_h) ≥ 1,但那又把你推向爆炸边缘——**这是一个鱼与熊掌的两难,普通 RNN 没有出路**。

> 直觉版:把 W_h 当成一个"每步乘一次的数 β"。β=0.9,乘 50 次 ≈ 0.005;β=1.1,乘 50 次 ≈ 117。没有一个固定的 β 能让信号既不衰减又不爆炸地走过几十上百步。

### 3.4 截断 BPTT 与梯度裁剪:工程上的两个对策

**截断 BPTT(truncated BPTT, TBPTT)**:序列很长时,完整展开 T 步内存吃不消(要存所有中间 h_t 用于反传),而且远端梯度本来也消失了。所以实践中把序列切成长度 k₂ 的块,前向可以连续(隐状态跨块传递),但反向只回传 k₁ 步就截断。代价:**模型再也学不到超过截断窗口的依赖**——这是用截断主动放弃远程信号,换可训练性。

**梯度裁剪(gradient clipping;Pascanu et al., 2013)**:专治爆炸,不治消失。对全局梯度向量 g,若其范数超过阈值 θ 就等比例缩回:

```
if ‖g‖ > θ:
    g ← (θ / ‖g‖) · g
```

```python
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
# 注意:这是按【整个模型梯度拼成的大向量的 L2 范数】裁剪,
# 不是逐参数裁剪——保留梯度方向,只缩模长。clip 要在 loss.backward() 之后、
# optimizer.step() 之前调用。
```

裁剪保留方向只缩模长,所以不改变"往哪走",只防止"一步走太远"。**关键认知:裁剪能救爆炸,但对消失无能为力——消失是信号本身没了,缩放救不回零。消失必须靠改结构,这就引出 LSTM。**

## 4. LSTM:用一条加性的细胞状态绕开连乘

### 4.1 设计动机:把"记忆"和"输出"分离,并让记忆走加法

普通 RNN 的病根在 3.2:状态更新是 hₜ = tanh(W_h hₜ₋₁ + …),每步都被 W_h 乘一次、被 tanh 压一次,于是反传时连乘 (diag(tanh′)·W_h)。**要根治,就得让"携带长期信息的那条状态"在相邻步之间的关系不再是'乘一个矩阵',而是'加'**——加法的导数是 1,不衰减。

LSTM(Long Short-Term Memory;Hochreiter & Schmidhuber, 1997)的核心创新就是:引入一条独立的**细胞状态(cell state)cₜ**专门承载长期记忆,它的更新以**加法**为主干;再用三个**门(gate)**——可学习的、取值 (0,1) 的开关——来控制信息的擦除、写入、读出。门用 sigmoid σ(取值 0~1,0=全关,1=全开),候选值用 tanh(取值 −1~1)。

### 4.2 完整方程组

设 [hₜ₋₁, xₜ] 表示把上一步隐状态和当前输入拼接(concatenate)。LSTM 在时刻 t:

```
fₜ = σ(W_f [hₜ₋₁, xₜ] + b_f)        # 遗忘门 forget gate:决定从 cₜ₋₁ 擦掉多少旧记忆
iₜ = σ(W_i [hₜ₋₁, xₜ] + b_i)        # 输入门 input gate:决定写入多少新信息
c̃ₜ = tanh(W_c [hₜ₋₁, xₜ] + b_c)     # 候选细胞 candidate:这一步"想写入"的新内容
cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ           # ★细胞状态更新:加性主干★
oₜ = σ(W_o [hₜ₋₁, xₜ] + b_o)        # 输出门 output gate:决定从 cₜ 读出多少给 hₜ
hₜ = oₜ ⊙ tanh(cₜ)                  # 隐状态(对外输出 + 传给下一步)
```

⊙ 是逐元素相乘(Hadamard product)。逐行读懂这个机制:

- **fₜ ⊙ cₜ₋₁**:遗忘门对旧记忆做逐元素门控。fₜ 的某个分量接近 1 → 这一维记忆完整保留;接近 0 → 这一维记忆被清空。
- **iₜ ⊙ c̃ₜ**:输入门决定新候选 c̃ₜ 的每一维写入多少。
- **cₜ = 两者相加**:这是与普通 RNN 最本质的区别——**新细胞状态 = 保留的旧记忆 + 选择性写入的新记忆,是加法**。没有一个无条件的 W_c 矩阵去乘 cₜ₋₁。
- **hₜ = oₜ ⊙ tanh(cₜ)**:细胞状态 cₜ 是"内部记忆",不直接暴露;输出门 oₜ 决定这一步对外读出多少。**记忆(cₜ)和输出(hₜ)被解耦**——你可以把某个信息长期存在 cₜ 里却暂时不读出(oₜ 关),需要时再读。这是普通 RNN 做不到的(它的 hₜ 既是记忆又是输出,二者强绑死)。

参数量:有 4 个权重矩阵(f, i, c̃, o 各一个),每个 ∈ ℝ^{d_h×(d_h+d_x)},外加 4 个偏置。所以 LSTM 的参数量约是同维度普通 RNN 的 **4 倍**(普通 RNN 只有 W_h、W_x 两块,合起来一块 ℝ^{d_h×(d_h+d_x)})。这 4 倍的代价,买来的是可训练的长程依赖。

### 4.3 ★核心推导:cₜ 对 cₜ₋₁ 的梯度为什么是加性的★

这是 LSTM 全部魔力的数学来源,务必算清。我们要算 ∂cₜ/∂cₜ₋₁,看它和普通 RNN 的 diag(tanh′)·W_h 有何本质不同。

从 cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ 出发。注意 fₜ、iₜ、c̃ₜ 都通过 hₜ₋₁ 间接依赖更早的状态,但**cₜ₋₁ 在本式里直接出现在 fₜ ⊙ cₜ₋₁ 这一项**。先看这条直接路径(也是主导路径):

```
∂cₜ/∂cₜ₋₁  ⊇  diag(fₜ)        # 来自 fₜ ⊙ cₜ₋₁ 对 cₜ₋₁ 求导(fₜ 视为这一步的门控系数)
```

完整地,fₜ、iₜ、c̃ₜ 还各自通过 hₜ₋₁ = oₜ₋₁⊙tanh(cₜ₋₁) 依赖 cₜ₋₁,所以严格的雅可比是:

```
∂cₜ/∂cₜ₋₁ = diag(fₜ)
          + [来自 fₜ 经 hₜ₋₁ 的项] + [来自 iₜ 经 hₜ₋₁ 的项] + [来自 c̃ₜ 经 hₜ₋₁ 的项]
```

**但关键在于:第一项 diag(fₜ) 是一条不经过任何门内部 sigmoid 饱和、不连乘权重矩阵的直接通道**。沿着细胞状态这条线一路反传 (t−k) 步,主导项是:

```
∂cₜ/∂c_k  ≈  ∏_{j=k+1}^{t} diag(f_j)     # 加性高速路上的主导梯度
```

对比普通 RNN 的 ∏ diag(tanh′)·W_h:

| | 普通 RNN | LSTM 细胞状态 |
|---|---|---|
| 单步雅可比主导项 | diag(tanh′(a_j)) · **W_h** | diag(**f_j**) |
| 是否连乘权重矩阵 | 是(W_h 反复乘) | **否** |
| 单步因子取值 | 受 ‖W_h‖ 摆布,可 >1 或 <1 | f_j ∈ (0,1),由门学出来 |
| 当因子≈1 | 需要 ρ(W_h)≈1 但那会爆炸 | **遗忘门学到≈1 即可,无副作用** |

**这就是机制的全部**:如果某条记忆需要长期保留,模型只要把对应维度的遗忘门 f_j **学到接近 1**,那么 ∏ f_j ≈ 1,梯度沿细胞状态几乎**无损**地传过几十上百步——这条路被形象地称为 **constant error carousel(恒定误差传送带)** 或"梯度高速路"。而 f_j 接近 1 完全不会引起爆炸(它最多是 1),也不依赖 W_h 的谱半径。普通 RNN 想让因子接近 1 必须 ρ(W_h)≈1,代价是其他方向上爆炸;LSTM 用逐维、逐步可学的门把这个"接近 1"做成了**无副作用、可选择**的。

> 一句话总结机制:普通 RNN 的记忆每步被**乘**一个矩阵(几何衰减无法避免);LSTM 的记忆每步被**加**新内容、被一个学出来的、可以接近 1 的门**保留**,于是梯度不再被迫几何衰减。**消失问题从"乘法的宿命"变成了"门可以学着不衰减"。**

注意:LSTM 是**缓解(mitigate)**而非**根除**梯度消失。如果遗忘门学成了小于 1,梯度照样衰减;但区别在于,现在"是否衰减"是数据驱动可学的,而不是被矩阵谱半径锁死。实践中给遗忘门偏置 b_f 初始化一个正值(常取 +1,Jozefowicz et al., 2015)能让训练初期 fₜ≈σ(正数)>0.5,默认倾向"记住",显著加速收敛——这是个值得记住的工程技巧。

### 4.4 从零实现 LSTM cell

```python
import torch
import torch.nn as nn

class LSTMCellScratch(nn.Module):
    """从零实现 LSTM cell,显式写出三门 + 细胞更新,体现 4.2 的方程组。"""
    def __init__(self, d_x, d_h):
        super().__init__()
        self.d_h = d_h
        # 把 4 个门的权重合并成一个大矩阵,一次 matmul 算完 4 个预激活(工程常用优化)
        # 输入是 [h_prev, x] 拼接,维度 d_h + d_x;输出 4*d_h(对应 f, i, c̃, o)
        self.W = nn.Linear(d_h + d_x, 4 * d_h, bias=True)
        # 遗忘门偏置初始化为正(4.3 末尾的技巧):默认倾向"记住"
        with torch.no_grad():
            self.W.bias[:d_h].fill_(1.0)   # 前 d_h 个偏置对应遗忘门 f

    def forward(self, x_t, state):
        h_prev, c_prev = state              # 都是 (B, d_h)
        z = self.W(torch.cat([h_prev, x_t], dim=-1))   # (B, 4*d_h),一次算完
        f, i, g, o = z.chunk(4, dim=-1)     # 切成 4 块:遗忘/输入/候选/输出 的预激活
        f = torch.sigmoid(f)                # 遗忘门 ∈ (0,1)
        i = torch.sigmoid(i)                # 输入门 ∈ (0,1)
        g = torch.tanh(g)                   # 候选细胞 c̃ ∈ (-1,1)
        o = torch.sigmoid(o)                # 输出门 ∈ (0,1)

        c = f * c_prev + i * g              # ★加性细胞更新★:这一行就是不消失的关键
        h = o * torch.tanh(c)               # 输出门控读出
        return h, (h, c)

def run_lstm(cell, xs, h0, c0):
    """xs: (T, B, d_x)。返回每步 h。"""
    h, c = h0, c0
    outs = []
    for t in range(xs.size(0)):
        h, (h, c) = cell(xs[t], (h, c))     # 同一 cell 每步复用 → 时间维参数共享
        outs.append(h)
    return torch.stack(outs)                # (T, B, d_h)
```

可对照 PyTorch 官方 `torch.nn.LSTMCell` 与 `torch.nn.LSTM`(后者是 cuDNN 加速的融合实现,把 for 循环下沉到 CUDA kernel)。**学习时务必先用上面的 scratch 版跑通、肉眼比对每步 c 和 h,再用官方版求性能**。一个验证小实验:构造一个"记住序列第一个 token、在最后一步输出"的玩具任务(中间塞 50 个噪声 token),普通 RNN 几乎学不会,LSTM 能学会——这就是 4.3 推导在实验上的体现。

## 5. GRU:更省的门控,以及双向与堆叠

### 5.1 GRU:把三门压成两门

**GRU(Gated Recurrent Unit;Cho et al., 2014)**是 LSTM 的简化:**去掉独立的细胞状态 cₜ,只保留 hₜ;把遗忘门和输入门合并成一个更新门**。方程:

```
zₜ = σ(W_z [hₜ₋₁, xₜ])              # 更新门 update gate:旧状态 vs 新状态的混合比例
rₜ = σ(W_r [hₜ₋₁, xₜ])              # 重置门 reset gate:计算候选时遗忘多少旧状态
h̃ₜ = tanh(W_h [rₜ ⊙ hₜ₋₁, xₜ])      # 候选状态(注意 rₜ 门控了 hₜ₋₁)
hₜ = (1 − zₜ) ⊙ hₜ₋₁ + zₜ ⊙ h̃ₜ    # ★凸组合更新:加性,且系数互补★
```

最后一行是关键:`hₜ = (1−zₜ)⊙hₜ₋₁ + zₜ⊙h̃ₜ` 是一个**凸组合(convex combination)**——保留旧状态的比例 (1−zₜ) 和写入新状态的比例 zₜ **强制互补、和为 1**。和 LSTM 一样,这给出加性更新:当 zₜ≈0 时 hₜ≈hₜ₋₁,状态被原样搬运,梯度 ∂hₜ/∂hₜ₋₁ 主导项 ≈ diag(1−zₜ) ≈ I,**梯度不衰减**。机制和 LSTM 同源,只是用一个门同时管"忘旧"和"纳新"(LSTM 用 fₜ、iₜ 两个独立门,可以"既不忘也不纳"或"既忘又纳",GRU 被迫此消彼长)。

**参数对比**:GRU 有 3 个权重矩阵(z, r, h̃),LSTM 有 4 个。所以 GRU 参数约是 LSTM 的 **3/4**,计算也更省。

**LSTM vs GRU 怎么选**(Chung et al., 2014;Greff et al., 2017 的大规模对比结论,具体相对优劣随任务而异,**精确名次待核**):

- **效果上常常打平**,没有一个在所有任务上压倒另一个。
- **GRU 参数少、训得快**,小数据/算力紧时优先。
- **LSTM 的细胞-隐状态解耦**(记忆可存而不读)在某些需要长期精确记忆的任务上略占优;遗忘门的独立性给了更细的控制。
- 实践建议:**默认 GRU 起步,效果不够再换 LSTM**;但在 Transformer 时代,这俩主要出现在轻量级、流式、或资源极受限的场景。

### 5.2 双向 RNN:让每个位置同时看到左右文

很多任务(命名实体识别、词性标注——下一章 04 的主题)在预测位置 t 的标签时,**右边的上下文和左边一样重要**。比如判断"bank"是"银行"还是"河岸",得看后面的词。单向 RNN 只能从左到右,看不到右文。

**双向 RNN(Bidirectional RNN;Schuster & Paliwal, 1997)**:跑两个独立的 RNN,一个正向(→)一个反向(←),把两个隐状态拼接:

```
h→ₜ = RNN_fwd(x₁ ... xₜ)            # 从左读到 t
h←ₜ = RNN_bwd(x_T ... xₜ)            # 从右读到 t
hₜ = [h→ₜ ; h←ₜ]                    # 拼接,维度变 2·d_h
```

```python
# PyTorch 一行开启:
bilstm = nn.LSTM(d_x, d_h, bidirectional=True, batch_first=False)
# 输出维度自动变成 2*d_h
```

代价与约束:(1)输出维度翻倍;(2)**必须看到整个序列才能算反向**——所以**双向 RNN 不能做自回归生成/流式解码**(你不知道未来 token),只适合"输入序列已知、要给每个位置打标签或做整句表示"的任务。这一点直接通向第 6 章 ELMo(用双向 LSTM 做上下文相关词表示)和 BERT 的双向理解。

### 5.3 堆叠(深层)RNN

把多层 RNN 上下摞起来:第 1 层的输出序列 (h⁽¹⁾₁ ... h⁽¹⁾_T) 当第 2 层的输入序列。更深 → 更强的表示能力,但也更难训(梯度要穿过"时间深度 × 层深度")。实践常用 2~4 层,配合层间 dropout(注意:dropout 只加在**层与层之间**或非循环连接上,不要直接 drop 循环连接的隐状态,否则破坏记忆——variational dropout, Gal & Ghahramani 2016 给了正确做法)。

## 6. RNN 的根本局限:为什么注意力必然到来

LSTM/GRU 缓解了梯度消失,RNN 一度是序列建模的绝对主力(2014–2017 的机器翻译、语音、序列标注几乎被它统治)。但它有两个**结构性、无法靠调参解决**的局限,直接催生了注意力与 Transformer。

**局限一:顺序计算,无法并行(致命)**。hₜ 必须等 hₜ₋₁ 算完才能算(看 2.3 的 for 循环)。一个长度为 T 的序列,前向/反向都需要 **T 步串行**,无论你有多少 GPU 核心都没法把这 T 步并行掉。时间复杂度的"墙"是 O(T) 的串行深度。**在 GPU/TPU 这种大规模并行硬件上,这是毁灭性的**——训练吞吐被串行长度卡死。这正是 Transformer 最大的卖点:**自注意力让所有位置的表示一次性并行算出**(代价是 O(T²) 的两两交互,但那是可并行的矩阵乘,硬件吃得下)。详见算法课 02(注意力机制的数学)与 09(高效注意力)对这一权衡的完整讨论。

**局限二:长程依赖仍然有限**。LSTM 把梯度从"几何必然衰减"改善成"门可学着不衰减",但(1)信息仍要**逐步串行搬运**穿过中间所有位置——相距 200 步的两个 token,信息要在固定维度的 cₜ 里"挤过"中间 200 步的反复读写,容量瓶颈和干扰仍在;(2)它没有"直接寻址任意位置"的能力。而注意力给出的是 **O(1) 路径长度**:任意两个位置之间一步直连(query 直接和任意 key 算相似度),不需要逐步搬运。**路径长度从 RNN 的 O(T) 降到注意力的 O(1)**——这是 Bahdanau 注意力(算法课 02 的起点)和 Transformer 解决长程依赖的根本机制。

**一句话承上启下**:RNN 教会我们"用状态递推 + 参数共享处理变长序列",LSTM 教会我们"用加性门控开梯度高速路";但"串行不可并行"和"路径长度 O(T)"这两座山,循环结构本身翻不过去——**注意力机制就是为推平这两座山而生的**。这正是下一段技术史的开端(算法课 02/03 把这段讲透)。

## 7. 设计权衡与常见坑

- **tanh vs ReLU 作 RNN 激活**:普通 RNN 别随便换 ReLU。ReLU 无上界,正向连乘极易让隐状态爆炸;tanh 的有界性是稳定的一部分。要用 ReLU 类需配特殊初始化(IRNN:W_h 初始化为单位阵,Le et al., 2015),否则崩。
- **梯度裁剪只治爆炸不治消失**(第 3.4 节)。新手常见误区:loss 不降就猛调裁剪阈值——裁剪救不了"信号本身衰减为零"。消失要靠 LSTM/GRU 结构或更短的截断窗口/任务重构。
- **遗忘门偏置初始化为正**(b_f≈+1):最高性价比的 LSTM 调参技巧。不设的话训练初期 fₜ≈0.5,记忆默认半衰,长程任务收敛慢甚至学不动。
- **clip 的位置**:必须在 `backward()` 之后、`optimizer.step()` 之前。放错位置等于没裁。
- **双向 RNN 不能自回归生成**:它需要完整序列算反向。把 BiLSTM 用在语言模型/解码器上是典型错误。它的位置是"输入已知的打标签/整句编码"。
- **变长 batch 的 padding 与 mask**:一个 batch 内句子长度不一,要 pad 对齐。但 pad 位置不能参与 loss、也不该污染隐状态——PyTorch 用 `pack_padded_sequence`/`pad_packed_sequence` 让 cuDNN 跳过 pad。忘记 pack 会让模型把大量 pad 当真实 token 学,指标虚高/训练异常。
- **dropout 别加在循环连接上**:直接 drop 隐状态会随机抹掉记忆、破坏时间一致性。用 variational/recurrent dropout(同一序列内各时刻用同一个 dropout mask)或只在层间/输入输出加。
- **截断 BPTT 的隐藏陷阱**:跨块传隐状态时,记得 `h.detach()`(切断跨块的梯度),否则计算图无限增长导致显存爆 + 实际上没真截断。
- **梯度消失的"假象"调试**:打印各时间步对早期权重的梯度范数,若呈指数衰减(画出来近似直线 in log scale),就是 vanishing 实锤,而非学习率问题——这是 3.3 推导的直接诊断手段。

## 10. 小结与承上启下

这一章我们把循环网络的机制和死穴都拆到了底:

- **RNN 用隐状态递推 hₜ=tanh(W_h hₜ₋₁+W_x xₜ) + 参数共享**处理变长序列,但时间展开后是一个深度=序列长的网络。
- **BPTT 的跨时间梯度是 ∏ diag(tanh′)·W_h 的连乘**,范数受 (γ·‖W_h‖)^{距离} 这个几何项支配:β<1 消失、β>1 爆炸,且**不存在两全的固定 W_h**。梯度裁剪治爆炸不治消失。
- **LSTM 用一条加性细胞状态 cₜ=fₜ⊙cₜ₋₁+iₜ⊙c̃ₜ 开梯度高速路**:∂cₜ/∂cₜ₋₁ 的主导项是 diag(fₜ) 而非连乘 W,遗忘门学到≈1 时梯度近乎无损穿过长距离,且不引起爆炸——这就是缓解消失的全部机制。**GRU 用互补的更新门做同源的加性更新,更省参**。
- **双向 RNN 拼前后向**给每位置左右文(不能自回归);堆叠加深表示。
- **RNN 的两座翻不过的山**:串行不可并行(O(T) 串行深度)、长程路径长度 O(T)。

**承上**:第 02 章的神经语言模型用固定窗口,本章的 RNN 把它升级成"无限历史"的递推。**启下**:第 04 章会把本章的 BiLSTM 接上 CRF 层做序列标注(NER/POS),你会看到"神经特征 + 结构化解码"的经典组合;第 05 章讲 seq2seq 与翻译时,你会看到 RNN 编码器的"信息瓶颈"如何逼出注意力。而注意力如何一举推平本章末尾那两座山(并行 + O(1) 路径),其完整数学机制在**算法课 02(注意力机制的数学)、03(Transformer Block)** 里讲透——读到那里时,请带着本章的两个局限回头看,你会理解 Transformer 不是凭空出现,而是对 RNN 结构性缺陷的精确回应。

### 练习 1:BPTT 连乘项推导与消失/爆炸条件

**第一部分:推出 ∂hₜ/∂h_k = ∏ⱼ diag(tanh′(a_j))·W_h**

从递推式出发,记预激活 a_j = W_h h_{j−1} + W_x x_j + b,则 h_j = tanh(a_j)。

单步雅可比 ∂h_j/∂h_{j−1}:tanh 逐元素作用,其雅可比是对角阵 diag(tanh′(a_j));a_j 对 h_{j−1} 的导数是 W_h(x_j 与 h_{j−1} 无关,被消掉)。由链式法则:

```
∂h_j/∂h_{j-1} = diag(tanh′(a_j)) · W_h
```

多步靠链式法则把相邻单步雅可比连乘(注意顺序:外层在左):

```
∂hₜ/∂h_k = (∂hₜ/∂hₜ₋₁)(∂hₜ₋₁/∂hₜ₋₂)···(∂h_{k+1}/∂h_k)
         = ∏_{j=k+1}^{t} diag(tanh′(a_j)) · W_h
```

得证。这是一个长度为 (t−k) 的矩阵连乘,每个因子都含同一个 W_h。

**第二部分:范数上界 ‖∂hₜ/∂h_k‖ ≤ (γ·‖W_h‖)^{t−k}**

用子乘性 ‖AB‖ ≤ ‖A‖·‖B‖ 把连乘的范数拆开:

```
‖∂hₜ/∂h_k‖ ≤ ∏_{j=k+1}^{t} ‖diag(tanh′(a_j))‖ · ‖W_h‖
```

对角阵的谱范数 = 对角元绝对值的最大值。令 γ = max_a tanh′(a),则每个 ‖diag(tanh′(a_j))‖ ≤ γ。连乘共 (t−k) 个相同上界:

```
‖∂hₜ/∂h_k‖ ≤ (γ · ‖W_h‖)^{t-k}
```

得证。记 β = γ·‖W_h‖,这是一个**几何(指数)级数**,指数就是依赖跨越的时间步数。

**(a) γ 的精确值及原因**

```
tanh′(a) = 1 − tanh²(a)
```

因 tanh(a) ∈ (−1, 1),故 tanh²(a) ∈ [0, 1),所以 tanh′(a) ∈ (0, 1]。最大值在 tanh(a)=0 即 **a=0**(预激活为零)处取得:

```
γ = max tanh′(a) = tanh′(0) = 1 − 0 = 1
```

**结论:γ = 1**,在预激活=0 处取得;偏离 0 越远 tanh 越饱和,tanh′ 越趋于 0。(验算:1 − tanh(0)² = 1.0,已核对。)

**(b) 梯度必然消失的充分条件(谱半径表述)**

因 γ=1,有 β = ‖W_h‖₂ = σ_max(W_h)(谱范数 = 最大奇异值)。当 β < 1 时,(t−k)→∞ 使 β^{t−k}→0,梯度上界趋零,即必然消失。用 W_h 的量表述其充分条件:

```
σ_max(W_h) = ‖W_h‖₂ < 1/γ = 1   （谱范数判据,每步都是收缩映射）
```

若题面要求用谱半径 ρ(W_h) 表述:由于 ρ(W_h) ≤ σ_max(W_h),严格的"必然消失"充分条件用的是**谱范数** σ_max(W_h) < 1/γ(对应 Pascanu et al. 2013);**谱半径只出现在爆炸的必要条件 ρ(W_h) > 1/γ 里**。换言之:

- ρ(W_h) < 1/γ = 1 ⇒ 长期梯度衰减(消失,因为乘积的长期增长率由 ρ 决定);
- 要长期不消失,必须 ρ(W_h) ≥ 1。

直觉验算:β=0.9 乘 50 次 ≈ 0.0052,信号几乎归零;远程依赖学不进来。

**(c) 为什么不存在两全的固定 W_h**

把每步的有效因子近似成一个标量 β。损失对 k 步前状态的梯度量级 ≈ β^{t−k}:

- 要**不爆炸**,需对所有距离都有界,即 β ≤ 1;
- 要**不消失**(远程梯度不衰减到零),需 β ≥ 1。

两者**同时**只能在 β=1 这条刀刃上满足,而:(1)实际中 tanh′ 在非零预激活处严格 < 1,把有效因子继续往下压,即便 ‖W_h‖=1 整体仍 < 1 倾向消失;(2)β=1 是不稳定平衡,任意扰动都会偏向消失或爆炸,无法稳定维持。更本质地,W_h 是矩阵:其不同特征方向有不同的 |λᵢ|,只要谱半径 ρ ≥ 1 保住某些方向不消失,谱范数 σ_max ≥ ρ ≥ 1 就让另一些方向(或同方向的瞬态)爆炸——**单个固定矩阵无法在所有方向上同时把因子稳定在 1**。

验算佐证:β=0.9 → 0.0052(消失);β=1.1 → 117(爆炸);中间没有一个固定 β 能让信号既不衰减又不爆炸地走过 50 步。**这正是普通 RNN 的死结,只能靠改结构(LSTM/GRU)绕开。**

### 练习 2:cₜ 梯度为何是加性的

**第一部分:从直接路径得到主导项 diag(fₜ)**

从细胞状态更新出发:

```
cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ
```

把 fₜ、iₜ、c̃ₜ 暂时视为这一步的常系数(只看 cₜ₋₁ 的直接出现),逐元素乘 fₜ ⊙ cₜ₋₁ 对 cₜ₋₁ 的雅可比就是把 fₜ 放到对角线上的对角阵;iₜ ⊙ c̃ₜ 这一项不含 cₜ₋₁,导数为 0。于是直接路径贡献:

```
∂cₜ/∂cₜ₋₁ ⊇ diag(fₜ)
```

这是**主导项**:它不经过任何门内部的 sigmoid(无饱和压缩),也不连乘任何权重矩阵。

**第二部分:完整雅可比还含哪些经 hₜ₋₁ 的项**

fₜ、iₜ、c̃ₜ 实际都依赖 hₜ₋₁,而 hₜ₋₁ = oₜ₋₁ ⊙ tanh(cₜ₋₁) 又依赖 cₜ₋₁。把这条间接路径加上,完整雅可比为:

```
∂cₜ/∂cₜ₋₁ = diag(fₜ)                                  ← 直接路径(主导)
          + diag(cₜ₋₁) · (∂fₜ/∂hₜ₋₁) · (∂hₜ₋₁/∂cₜ₋₁)   ← fₜ 经 hₜ₋₁
          + diag(c̃ₜ)  · (∂iₜ/∂hₜ₋₁) · (∂hₜ₋₁/∂cₜ₋₁)   ← iₜ 经 hₜ₋₁
          + diag(iₜ)  · (∂c̃ₜ/∂hₜ₋₁) · (∂hₜ₋₁/∂cₜ₋₁)   ← c̃ₜ 经 hₜ₋₁
```

其中 ∂hₜ₋₁/∂cₜ₋₁ = diag(oₜ₋₁ ⊙ tanh′(cₜ₋₁));∂fₜ/∂hₜ₋₁ = diag(fₜ⊙(1−fₜ))·W_f[:, h 部分](sigmoid 导数 σ′=σ(1−σ)),iₜ、c̃ₜ 类似(c̃ 用 tanh′)。这三项都**穿过 sigmoid/tanh 的导数(<1,饱和时趋 0)并连乘权重矩阵**,因此通常被压得很小,不构成长程主导通道。

**第三部分:f_j≈1 时梯度近乎无损,且为何不爆炸**

沿细胞状态从 k 反传到 t,只保留主导项,得到对角阵连乘(对角阵连乘仍是对角阵,逐元素相乘):

```
∂cₜ/∂c_k ≈ ∏_{j=k+1}^{t} diag(f_j) = diag(∏_{j=k+1}^{t} f_j)
```

当某维记忆需要长期保留、模型把对应维度遗忘门各步学到 f_j ≈ 1:

```
∏_{j=k+1}^{t} f_j ≈ 1  ⇒  ∂cₜ/∂c_k ≈ I
```

梯度沿这条"恒定误差传送带 / constant error carousel"几乎**无损**地穿过 (t−k) 步——远程依赖的梯度信号不被几何抹掉。

**为何"≈1"不引起爆炸**:遗忘门由 sigmoid 产生,f_j ∈ (0, 1),严格地 **f_j ≤ 1**。对角阵 diag(f_j) 的谱范数 = max_i f_{j,i} ≤ 1,所以每一步**永不放大**,连乘只会 ≤ 1,**永远不会爆炸**。对比普通 RNN:它想让因子≈1 必须 ρ(W_h)≈1,而那会在其他方向把 σ_max 推到 >1 导致爆炸(练习 1c);LSTM 用**逐维、逐步可学**的门把这个"接近 1"做成了无副作用、可选择的。

验算佐证:f_j 取 [0.95,1] 之间随机,60 步后乘积每维 ≈ 0.20(远胜 vanilla β=0.9 的 0.0018),且最大值 ≤ 1 不放大;若 f_j 真取 1,则乘积恒为 1,梯度完全无损。

**结论:LSTM 把"消失=乘法的宿命"改成了"门可以学着不衰减"——这是缓解(mitigate)而非根除:f_j 若学成 <1 梯度照样衰减,但'是否衰减'变成数据驱动可学,而非被矩阵谱半径锁死。** 工程上把 b_f 初始化为正值(常取 +1)让训练初期 fₜ>0.5 默认倾向"记住",显著加速收敛。

### 练习 3:长程记忆玩具任务(实现 + 实验)

任务:序列第 0 步给一个 0/1 标志位(one-hot 编码),中间塞 T 个随机噪声 token,要求模型在**最后一步**输出该标志位。这是经典 copy/adding task 的变体。预期:vanilla RNN 在 T≥20 后崩,LSTM 到 T=100 仍能学会。

> 依赖:`pip install torch`(CPU 即可,本任务维度小、几百步训练,无需 GPU)。下面是可直接运行的完整脚本。

```python
import torch
import torch.nn as nn

torch.manual_seed(0)

# ---------- 两个 cell:vanilla RNN 与 scratch LSTM(来自第 2.3 / 4.4 节) ----------
class VanillaRNNCell(nn.Module):
    def __init__(self, d_x, d_h):
        super().__init__()
        self.W_h = nn.Linear(d_h, d_h, bias=False)
        self.W_x = nn.Linear(d_x, d_h, bias=True)
    def forward(self, x_t, h_prev):
        return torch.tanh(self.W_h(h_prev) + self.W_x(x_t))

class LSTMCellScratch(nn.Module):
    def __init__(self, d_x, d_h):
        super().__init__()
        self.d_h = d_h
        self.W = nn.Linear(d_h + d_x, 4 * d_h, bias=True)
        with torch.no_grad():
            self.W.bias[:d_h].fill_(1.0)        # 遗忘门偏置初始化为 +1(4.3 技巧)
    def forward(self, x_t, state):
        h_prev, c_prev = state
        z = self.W(torch.cat([h_prev, x_t], dim=-1))
        f, i, g, o = z.chunk(4, dim=-1)
        f, i, o = torch.sigmoid(f), torch.sigmoid(i), torch.sigmoid(o)
        g = torch.tanh(g)
        c = f * c_prev + i * g                  # ★加性细胞更新★
        h = o * torch.tanh(c)
        return h, (h, c)

# ---------- 合成数据:flag 在第 0 步,中间 T 个噪声,最后一步要预测 flag ----------
# 词表:0/1 是 flag 的 one-hot 占两维;噪声 token 用额外 N_NOISE 维随机 one-hot
N_NOISE = 8
D_X = 2 + N_NOISE     # 前 2 维给 flag,后 N_NOISE 维给噪声字母表
def make_batch(B, T):
    flag = torch.randint(0, 2, (B,))                      # (B,) 目标 0/1
    seq_len = T + 1
    x = torch.zeros(seq_len, B, D_X)
    # 第 0 步:flag 的 one-hot(放在前两维)
    x[0, torch.arange(B), flag] = 1.0
    # 第 1..T 步:噪声 one-hot(放在后 N_NOISE 维),与 flag 无关
    noise = torch.randint(0, N_NOISE, (T, B))
    for t in range(1, seq_len):
        x[t, torch.arange(B), 2 + noise[t-1]] = 1.0
    return x, flag                                        # x:(T+1,B,D_X), flag:(B,)

# ---------- 通用训练/评测 ----------
class Model(nn.Module):
    def __init__(self, kind, d_h=64):
        super().__init__()
        self.kind = kind
        self.d_h = d_h
        self.cell = VanillaRNNCell(D_X, d_h) if kind == "rnn" else LSTMCellScratch(D_X, d_h)
        self.readout = nn.Linear(d_h, 2)                  # 二分类:flag 是 0 还是 1
    def forward(self, x):
        B = x.size(1)
        h = torch.zeros(B, self.d_h)
        c = torch.zeros(B, self.d_h)
        for t in range(x.size(0)):
            if self.kind == "rnn":
                h = self.cell(x[t], h)
            else:
                h, (h, c) = self.cell(x[t], (h, c))
        return self.readout(h)                            # 只用最后一步 h 预测

def train_eval(kind, T, steps=600, B=64, lr=1e-2):
    model = Model(kind)
    opt = torch.optim.Adam(model.parameters(), lr=lr)
    lossf = nn.CrossEntropyLoss()
    grad_norms = []                                       # 记录早期权重梯度范数(实证 3.3)
    for step in range(steps):
        x, y = make_batch(B, T)
        logits = model(x)
        loss = lossf(logits, y)
        opt.zero_grad()
        loss.backward()
        # 记录 vanilla RNN 输入层(最早被用到的权重)梯度范数
        if kind == "rnn":
            grad_norms.append(model.cell.W_x.weight.grad.norm().item())
        torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)  # 治爆炸,不治消失
        opt.step()
    # 评测准确率
    with torch.no_grad():
        x, y = make_batch(512, T)
        acc = (model(x).argmax(-1) == y).float().mean().item()
    return acc, grad_norms

if __name__ == "__main__":
    print(f"{'T':>5} | {'vanilla RNN':>12} | {'LSTM':>8}")
    for T in (5, 20, 50, 100):
        acc_rnn, gn = train_eval("rnn", T)
        acc_lstm, _ = train_eval("lstm", T)
        print(f"{T:>5} | {acc_rnn:>12.3f} | {acc_lstm:>8.3f}")
    # 画 vanilla RNN 早期权重梯度范数随训练步(log 轴近似直线 = 指数衰减实锤)
    # import matplotlib.pyplot as plt
    # plt.semilogy(gn); plt.xlabel("step"); plt.ylabel("||grad W_x||"); plt.show()
```

**预期结果(随机性导致具体数值会有波动,定性结论稳健;精确百分比待核):**

```
    T |  vanilla RNN |     LSTM
    5 |        ~1.00 |    ~1.00      # 短依赖,两者都会
   20 |   ~0.5–0.7   |    ~1.00      # vanilla 开始退化(0.5 = 瞎猜)
   50 |       ~0.50  |   ~0.95+      # vanilla 崩到随机水平
  100 |       ~0.50  |   ~0.90+      # vanilla 完全学不会,LSTM 仍能学
```

**实验解读:**

- vanilla RNN 在 T≥20 后准确率塌向 0.5(二分类的随机基线),因为 flag 的梯度信号要穿过 T 步连乘,被几何衰减抹掉——这正是第 3.3 节"远程依赖学不进来"的实证。
- LSTM 到 T=100 仍能学会:遗忘门把承载 flag 的那一维学到 f_j≈1,梯度沿细胞状态高速路无损回传到第 0 步——第 4.3 节推导的实验体现。
- **诊断手段(对应 3.3 末尾)**:打印 vanilla RNN 输入层 `W_x` 梯度范数随训练步,在 log 纵轴上画出来近似直线/快速塌向 0,即 vanishing 实锤,而非学习率问题。
- 工程注意:`h.detach()` 在本任务序列短、整段反传,故不需要;若做长序列截断 BPTT 才需跨块 detach。LSTM 的 b_f=+1 初始化对 T=100 收敛很关键,去掉后会明显变慢。

### 练习 4:LSTM vs GRU vs 双向(对比与辨析)

**(a) 参数量公式,验证 GRU ≈ LSTM 的 3/4**

设隐状态维 d_h、输入维 d_x。两者门的输入都是拼接向量 [hₜ₋₁, xₜ],维度 d_h + d_x。每个"门/候选"是一个权重矩阵 ∈ ℝ^{d_h×(d_h+d_x)} 外加偏置 ∈ ℝ^{d_h}。

- **LSTM**:4 块(遗忘 f、输入 i、候选 c̃、输出 o):

```
P_LSTM = 4 · [ d_h·(d_h + d_x) + d_h ] = 4·d_h·(d_h + d_x + 1)
```

- **GRU**:3 块(更新 z、重置 r、候选 h̃):

```
P_GRU  = 3 · [ d_h·(d_h + d_x) + d_h ] = 3·d_h·(d_h + d_x + 1)
```

比值:

```
P_GRU / P_LSTM = 3/4 = 0.75   （精确等于 3/4,不只是约等于)
```

验算(d_h=256, d_x=100):P_LSTM = 365568,P_GRU = 274176,比值 = 0.75。**已核对,恰好 3/4。** 严格说"GRU 是 LSTM 的 3/4"在含偏置时也精确成立,因为偏置项也是按块数 4:3 同比例缩放。

**(b) 为何 GRU 的 (1−zₜ) 与 zₜ 必须互补,而 LSTM 的 fₜ、iₜ 独立;LSTM 多了什么表达能力**

GRU 的状态更新是**凸组合**:

```
hₜ = (1 − zₜ) ⊙ hₜ₋₁ + zₜ ⊙ h̃ₜ
```

"保留旧"的系数 (1−zₜ) 与"写入新"的系数 zₜ **由同一个门 zₜ 导出,被结构强制和为 1**——这是设计上把两个动作压成一个门以省参的结果。后果:**只要多写入新内容(zₜ 大),就必然少保留旧记忆(1−zₜ 小),反之亦然——此消彼长,无法解耦。**

LSTM 用两个**独立**的门:

```
cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ      （fₜ、iₜ 各自独立产生,不受"和为1"约束)
```

因此 LSTM 能表达 GRU 凸组合**做不到**的状态组合:

| 想要的动作 | LSTM (fₜ, iₜ) | GRU 能否做到 |
|---|---|---|
| 既保留旧、又不写新(纯保鲜) | f≈1, i≈0 | ✗(zₜ→0 保旧但也不写;但 GRU 候选受 rₜ 影响,语义不完全等价) |
| 既清空旧、又大量写新 | f≈0, i≈1 | ✓(zₜ≈1) |
| **既不保留旧、也不写新(清零)** | f≈0, i≈0 | ✗(zₜ 任何取值都至少保留旧或写入新之一) |
| 同时大量保留旧 + 大量写新(累加) | f≈1, i≈1 | ✗(1−zₜ 与 zₜ 不能同时大) |

**LSTM 多出的表达能力:它能独立地决定"忘多少"和"纳多少",尤其能表达"f≈1, i≈0 的纯长期保鲜"和"f≈1, i≈1 的边记边累加"这类 GRU 因系数互补而无法表达的状态。** 加上 cₜ/hₜ 解耦(输出门 oₜ 让记忆可存而不读),LSTM 在需要长期精确记忆的任务上有更细的控制。代价是多 1/4 参数、训练略慢——所以实践常"默认 GRU 起步,效果不够再换 LSTM"。

**(c) 为何双向不能用于自回归语言模型,但适合 NER/POS;如何通向 ELMo/BERT**

双向 RNN 拼接正反两向隐状态:

```
h→ₜ = RNN_fwd(x₁ … xₜ)      # 用到 t 及左侧
h←ₜ = RNN_bwd(x_T … xₜ)      # 用到 t 及【右侧到序列末尾】
hₜ = [h→ₜ ; h←ₜ]
```

- **不能用于自回归语言模型**:自回归 LM 在预测位置 t 的词时,**只能依赖 x_<t**(t 左侧),目标恰恰是 xₜ 及其右侧还未生成的内容。而反向分支 h←ₜ 计算时**必须看到 x_t…x_T 整个右侧序列**——这等于"用答案预测答案",造成标签泄漏(label leakage),且推理时未来 token 根本不存在,无法流式/自回归解码。把 BiLSTM 当语言模型/解码器是典型错误。
- **适合 NER / POS**:序列标注是"输入整句已知,给每个位置打标签"。判断 "bank" 是银行还是河岸、判断一个词的词性,**右文和左文同等重要**(如 "river bank" vs "bank account")。输入序列完整可见,双向恰好让每个位置同时看到左右上下文,显著提升标注质量。代价:输出维度翻倍(2·d_h)、必须拿到完整序列才能算。
- **如何通向第 6 章 ELMo / BERT**:ELMo 正是用**双向 LSTM**在大规模语料上预训练,产出"上下文相关词表示"(同一个词在不同句子里向量不同)——这是双向编码思想的直接延伸。BERT 进一步用 Transformer 的双向自注意力做"深度双向"理解,并用 **Masked Language Model** 巧妙绕开"双向会看到答案"的泄漏问题(遮住目标词再预测,从而既双向又不自我泄漏)。**这条线索说明:双向编码在"理解类任务"上的价值,从 BiLSTM(本章)一路延续到 ELMo、BERT(第 6 章);而它与自回归生成的天然冲突,也正是 BERT(双向理解)与 GPT(单向生成)分道扬镳的根源。**
