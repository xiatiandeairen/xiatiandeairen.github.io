> 这一章把你从"会写代码、数学也不差,但对 NLP 只有'就是处理文字嘛'这种模糊印象",带到"能精确说清语言这种数据相对图像/语音到底特殊在哪、这些特殊性如何决定了 NLP 全部任务与方法的形态、为什么 NLP 的评测本身是一个比建模更难的元问题"。读完你应该能:用第一性原理列出语言的六个根本难点并各举可计算的例子;画出整个 NLP 任务地图并解释每条任务边沿(理解/生成、token 级/序列级)如何对应难点;手算句法歧义的解析树数量(Catalan 数)和 Zipf 长尾的覆盖率;讲清"为什么 NLP 评测比图像分类难"这个贯穿全课的暗线。这是整门课的问题框架——后面每一章(表示、序列模型、CRF、翻译、预训练、RAG、评测)本质上都是在攻其中一两个难点。

## 一、动机:先承认"语言难",再问"难在哪"

如果你做过图像或数值类的机器学习,会带着一个隐含假设进 NLP:**输入是某种张量,定义损失,梯度下降,搞定。** 这个范式没错,但它会让你低估一件事——**为什么同样是"感知输入→产生输出",图像分类在 2015 年就被刷到超人类,而"让机器真正理解一段话"到今天(LLM 时代)仍然是开放问题。**

差距不在算力,也不主要在模型容量。差距在**数据本身的性质**。语言这种数据有一组图像/语音都不具备(或不那么强烈具备)的内禀属性,正是这组属性决定了:NLP 需要的不是"更大的卷积核",而是一整套专门的表示方法、序列建模工具、结构预测算法和评测哲学。

**本章的全部任务,就是把"语言难"这句空话,拆成六个可以精确陈述、甚至可以计算的难点,并说明每个难点如何派生出本课后面要讲的具体方法。** 我先把这条主干画出来,后面逐个补完:

```
语言的六大内禀难点                     → 派生出的方法 / 本课对应章节
────────────────────────────────────────────────────────────────
1. 离散符号(非连续,无天然度量)        → 词向量/嵌入(把离散映到连续) ……… 第 1 章
2. 组合性(有限词→无限句,结构敏感)     → 句法/语义结构、CRF、解析 …… 第 4 章
3. 歧义(词义/句法/指代/语用 多层)      → 上下文模型、消歧、语言模型 … 第 2-3 章
4. 长程依赖(相关的词隔很远)            → RNN/LSTM、注意力、长上下文 … 第 3、5 章
5. 世界知识与常识(意义在文本之外)      → 预训练、检索增强(RAG) …… 第 6-7、10 章
6. 稀疏 / Zipf 长尾(尾部永远见不全)    → 子词切分、平滑、迁移 …… 第 1、8 章
────────────────────────────────────────────────────────────────
横切一切的元问题:评测难(无唯一正解)   → 生成评测与忠实性 …………… 第 13 章
```

**这一章不教你建模,只教你"问题长什么样"。** 一个常见误区是新手直接跳进 Transformer,却说不清自己在解决什么——结果是会调包、不会判断方法是否对路。本章就是给你那把判断的尺子。

**给做过工程、ML 从零的读者的迁移提示:** 你熟悉的连续优化范式在 NLP 里依然成立,但有三处"水土不服"会反复咬你:(1) 输入是离散 token,没有"把图片旋转 0.3 度"这种平滑扰动,数据增强、梯度、距离度量全都要重新定义;(2) 输出常常不是一个标量或一个类别,而是一个**有内部结构的序列**(一棵树、一串标签、一段文本),损失要定义在结构上;(3) 监督信号稀缺且主观——同一个翻译有无数种正确写法。把这三点记牢,你就能预判后面每个方法"在补哪个洞"。

## 二、难点一:离散符号——语言没有天然的"距离"

### 问题

图像的像素是 0–255 的连续值,语音是连续波形。它们天然活在欧氏空间里:两张图"接近"就是像素差小,你可以做插值、加高斯噪声、算梯度,一切微积分工具直接可用。

**语言的原子是离散符号(token)。** "猫"和"狗"之间没有"0.7 只猫 + 0.3 只狗"这种中间态;"猫"的下一个字符不是"猫 + ε"。最朴素的表示是 **one-hot**:词表大小 V,第 i 个词表示成一个长度 V、只有第 i 位是 1 的向量。

### 机制 / 推导:one-hot 为什么是"反语义"的

设词表 V,词 wᵢ 的 one-hot 向量是 eᵢ ∈ {0,1}ⱽ。考察任意两个不同词的关系:

```
内积:        eᵢ · eⱼ = 0           （i≠j，任意两词正交）
欧氏距离:    ‖eᵢ − eⱼ‖₂ = √2       （任意两个不同词，距离全相等）
余弦相似度:  cos(eᵢ, eⱼ) = 0        （任意两词，相似度全为 0）
```

这三行是 one-hot 的死刑判决:**在 one-hot 空间里,"猫"到"狗"的距离,和"猫"到"微积分"的距离,严格相等(都是 √2)。** 表示里完全没有语义——所有词等距、互相正交。模型拿到这种输入,等于拿到一堆没有任何先验关系的 ID。

更糟的是**维度灾难**:V 在真实语料里是几万到几十万,one-hot 是极度高维且稀疏(全是 0,只有一个 1)。一个把 one-hot 接全连接层的模型,第一层权重就是 V × d 的矩阵——参数巨大,而且对每个词都得从零学起,词与词之间不共享任何统计强度。

### 代码:量化 one-hot 的"无语义"

```python
import numpy as np

V = 50000
def onehot(i): 
    v = np.zeros(V); v[i] = 1.0; return v

cat, dog, calculus = onehot(7), onehot(42), onehot(31337)
def cos(a, b):  # 余弦相似度
    return a @ b / (np.linalg.norm(a)*np.linalg.norm(b) + 1e-12)

print(cos(cat, dog), cos(cat, calculus))   # 0.0  0.0  —— 全相等，毫无语义
print(np.linalg.norm(cat - dog), np.linalg.norm(cat - calculus))  # √2  √2
```

### 分析:这个难点派生出什么

整个**第 1 章(文本表示:one-hot → word2vec → GloVe)** 就是为了治这个病。核心思路是**分布式表示(distributed representation)**:把每个词映射到一个低维稠密向量(嵌入,embedding),让语义近的词向量也近。其理论依据是 **分布假设(distributional hypothesis):一个词的意义由它出现的上下文决定**(Firth, 1957,"You shall know a word by the company it keeps")。word2vec / GloVe 就是把这个假设变成可优化目标——这是第 1 章的主线,这里只点出"为什么非做不可"。

> **关键结论:离散性意味着语义不在符号本身,而必须从"符号如何共同出现"中统计地恢复出来。** 这一句话是整个词向量乃至预训练时代的种子。

注意一个常被忽略的代价:把离散变连续之后,你获得了微积分,但**失去了精确的可枚举性**。one-hot 至少能精确说"这是第 7 个词";嵌入空间里"猫"是一个稠密向量,你无法保证它不和别的概念纠缠。离散↔连续的来回转换(嵌入进、softmax 出)是 NLP 的基本节律,第 1 章和算法课的 softmax 输出层都在处理这个边界。

## 三、难点二:组合性——有限的词,无限的句子

### 问题

英语词表撑死几十万词,但**合法句子有无穷多个**:任意一句话,你都能再嵌套一个从句、再加一个修饰让它更长。这就是语言的**组合性(compositionality)**:整体的意义由部分的意义以及它们的**组合方式**决定(Frege 原则)。

组合性带来两个直接后果,都很要命:

1. **训练时见过的句子,推理时几乎永远见不到第二次。** 句子级别的"查表"彻底失效。模型必须从词和小片段泛化到从未见过的整句——这叫**组合泛化(compositional generalization)**。
2. **同样的词,排列/嵌套方式不同,意义天差地别。** 表示必须对**结构**敏感,而不只是对"出现了哪些词"敏感。

### 机制 / 推导:词袋为什么不够,以及结构的组合爆炸

先看一个反例,说明"只看出现了哪些词"(词袋,bag-of-words)必然丢失意义:

```
句 A: "狗 咬 人"
句 B: "人 咬 狗"
```

两句的词集合完全相同 {狗, 咬, 人},词频也相同。任何只基于词袋的表示,A 和 B 必然得到**完全一样的向量**——但它们意义相反(后者是新闻)。**语序携带意义,词袋系统性地丢弃语序。** 这就是为什么需要序列模型(第 3 章 RNN/LSTM)和位置编码(算法课),它们的存在理由就是"让模型看得见顺序与结构"。

再看结构本身有多复杂。给定 n 个词,把它们组合成一棵**二叉句法树**(每次把相邻两个成分合并)有多少种方式?这是经典的 Catalan 数:n 个叶子的满二叉树数目是 `C(n-1)`,其中

```
Catalan(k) = comb(2k, k) / (k+1)
n 个词 → 解析树数 = Catalan(n-1)
```

我把小数值算出来(已用代码验证):

```
词数 n     可能的二叉解析树数 Catalan(n-1)
  3                2
  5               14
  7              132
 10            4 862
 20    1 767 263 190   （~1.77 × 10⁹）
渐近:  Catalan(n-1) ~ 4^(n-1) / ((n-1)^1.5 · √π)   —— 随词数指数爆炸
```

**一句 20 个词的话,纯句法上就有近 18 亿种可能的成分结构。** 这还没算每个词本身的词性歧义、词义歧义。"理解一句话"在最坏情况下是在一个指数大的结构空间里搜索——这正是为什么句法分析(parsing)是 NLP 的硬核子领域,以及为什么我们需要概率来给这些结构排序(谁更可能),而不是枚举。

### 代码:词袋丢序的可计算证据

```python
from collections import Counter

def bow(sent):                       # 词袋:词→频次，丢弃顺序
    return Counter(sent.split())

a = bow("狗 咬 人")
b = bow("人 咬 狗")
print(a == b)                        # True  —— 两句在词袋下完全无法区分

# Catalan 数:n 个词的二叉解析树数量
from math import comb
def parse_trees(n):
    k = n - 1
    return comb(2*k, k) // (k + 1)
for n in (3, 5, 7, 10, 20):
    print(n, parse_trees(n))         # 2 14 132 4862 1767263190
```

### 分析:这个难点派生出什么

组合性是**第 4 章(序列标注与结构预测:HMM、CRF、维特比)** 和句法/语义分析的全部动机:输出不是一个标签,而是一个**结构**(一串标签、一棵树),所以损失要定义在结构上,推理要在结构空间里搜索(维特比就是在指数空间里做多项式时间的最优搜索)。组合性也是衡量"模型是不是真懂"的试金石——**组合泛化是当前 LLM 仍未完全解决的能力**:模型见过"红色的车"和"蓝色的杯子",能否正确理解从没见过的"蓝色的车"?这类系统性泛化测试(如 SCAN、COGS 数据集)反复显示神经模型会在组合新颖性上栽跟头,这是第 15 章(前沿)的开放问题。

> **关键结论:语言的意义是"可组合"的,这同时是它表达力无穷的来源,也是泛化与搜索困难的根源。词袋丢结构,序列/树模型才补得回来。**

## 四、难点三:歧义——同一串符号,多种意思

歧义是 NLP 最具标志性的难点,也是新手最容易低估的。**歧义不是边角情况,而是自然语言的常态**——人类靠上下文和常识瞬间消歧,几乎察觉不到歧义存在,于是误以为语言是清晰的。机器没有这层默认上下文,歧义就赤裸裸地暴露出来。歧义至少有四层,逐层递进:

### 4.1 词义歧义(lexical / word-sense)

同一个词形有多个义项。英语 "bank" = 河岸 / 银行;中文"打" = 击打 / 打电话 / 打毛衣 / 一打(12 个)。消歧(Word Sense Disambiguation, WSD)必须靠上下文:

```
"他去 bank 取钱"        → 银行
"他坐在 bank 上钓鱼"    → 河岸
```

**词义歧义是"上下文化表示"存在的根本理由。** 静态词向量(word2vec,第 1 章)给每个词形一个固定向量,无法区分 bank 的两个义项——这是它的硬伤;而 BERT(第 6 章)给出的是**随上下文变化的向量**,同一个 bank 在两句里向量不同。所以这一难点直接解释了"为什么从静态词向量进化到上下文嵌入是范式跃迁"。

### 4.2 句法歧义(syntactic / structural):本课的标志性例子

同一串词,可以挂成不同的句法结构,导致不同意思。最经典的例子是 **介词短语附着歧义(PP-attachment)**:

```
"I saw the man with the telescope."
读法 1:  我用望远镜看见了那个人   （with the telescope 修饰动词 saw → 我有望远镜）
读法 2:  我看见了那个带着望远镜的人 （with the telescope 修饰名词 man  → 他有望远镜）
```

两种读法对应两棵不同的解析树,词完全一样、顺序完全一样,**纯靠括号怎么打就变了意思**:

```
读法 1（介词短语挂到 VP）：
    saw [ the man ] [ with the telescope ]      ← 工具状语，修饰 saw

读法 2（介词短语挂到 NP）：
    saw [ the man [ with the telescope ] ]      ← 定语，修饰 man
```

句子里 PP-attachment 点越多,歧义组合越多——再次回到第三节那个 Catalan 爆炸:**句法歧义就是"组合性"在解析层面的具体表现。** 真实句子里这种歧义层出不穷("我看见在山上的人用望远镜"——挂法更多)。

### 4.3 指代歧义(referential / anaphora)

代词、省略指向谁?这需要跨句子追踪。Winograd Schema 是刻意设计来考这个的:

```
"市政府拒绝给抗议者颁发许可，因为他们担心暴力。"  → "他们" = 市政府
"市政府拒绝给抗议者颁发许可，因为他们宣扬暴力。"  → "他们" = 抗议者
```

只改了动词("担心" vs "宣扬"),"他们"的指代就翻转了。**消解它需要常识**(谁会担心暴力?谁会宣扬暴力?),纯语法和纯统计共现都不够。这条线连到第 11 章(信息抽取与知识)的指代消解(coreference),也连到难点五(世界知识)。

### 4.4 语用歧义(pragmatic):字面之外的意思

意义不止于字面。"你能把盐递给我吗?"字面是问你**有没有能力**,语用上是**请求**;反讽("天气真好啊"在暴雨天)字面与意图相反。语用层的意义依赖说话场景、共同知识、意图——这是 NLP 最难、也最少被经典任务覆盖的一层,直到对话系统(第 12 章)和指令对齐(强化学习课的 RLHF)才被正面处理。

### 机制:消歧的统一形式——给读法排概率

四层歧义有一个统一的应对范式:**不要追求唯一解,而要给每种解释一个概率,选最可能的。** 形式化:给定观测的词串 x,候选解释 y(义项 / 解析树 / 指代),求

```
ŷ = argmax_y  P(y | x)  =  argmax_y  P(x | y) · P(y)
                                         ↑           ↑
                                   结构生成 x 的似然   该解释的先验合理性
```

这个 `argmax P(y|x)` 是 NLP 几乎所有"消歧 / 结构预测 / 解码"任务的共同骨架:WSD、句法分析、机器翻译、序列标注全是它的特例。**整门课很大一部分就是在教"如何参数化并高效计算这个 argmax"**——n-gram 语言模型(第 2 章)估 `P(y)`,CRF(第 4 章)把它做成结构化条件概率,Transformer 解码(算法课)把它做成自回归。

### 代码:用上下文给词义打分(消歧的最小骨架)

```python
# 玩具版 WSD：用上下文词袋与各义项的"典型语境"算相似度，选最高的义项
contexts = {
    "river":   {"water", "fish", "boat", "sit"},   # 河岸义的典型语境
    "finance": {"money", "loan", "account", "withdraw"},
}
def disambiguate(sentence_words):
    s = set(sentence_words)
    scores = {sense: len(s & ctx) for sense, ctx in contexts.items()}
    return max(scores, key=scores.get), scores

print(disambiguate(["he", "withdraw", "money", "bank"]))   # ('finance', ...)
print(disambiguate(["he", "sit", "bank", "fish"]))         # ('river',   ...)
```

这只是把"靠上下文消歧"做成了可计算的最小例子;真实系统用上下文嵌入(第 6 章)取代这种硬编码集合,但**内核都是"上下文决定读法"**。

### 分析

> **关键结论:歧义不是 bug,是自然语言用有限符号表达无限意义的必然代价。NLP 的核心动作不是"找到唯一正确答案",而是"在多种解释上估计概率并择优"。** 这也是为什么概率论、信息论是 NLP 的母语——一切都建立在"语言是带噪声、带歧义的随机过程"这个世界观上。

歧义还埋下了**评测难**的第一颗雷:既然一句话可能有多个合理读法/译法,那"标准答案"就不唯一,后面第七节会展开。

## 五、难点四:长程依赖——相关的词隔得很远

### 问题

句子里有语义/语法关系的两个词,可能隔着很长的距离。主谓一致就是典型:

```
"The keys to the cabinet that I bought last year ... are on the table."
                                                      ↑are 由 keys（复数）决定
```

`are` 要跟最前面的 `keys`(复数)一致,而它们之间隔了一长串修饰("to the cabinet that I bought last year")。模型必须把"主语是复数"这个信息**保持很多步**,直到用得上。其他例子:篇章里前文设定的事实约束后文("张三是医生……他的诊断")、长文档里跨段落的指代。

### 机制 / 推导:为什么"距离"是建模的天敌

把句子看成时间序列,逐词处理(这是 RNN 的视角,第 3 章)。要让第 t 步用到第 k 步(k ≪ t)的信息,信息得在隐状态里**连续传递 t − k 步**。问题在于,反向传播时梯度要沿着这条链连乘,信息/梯度在长链上会**指数衰减或爆炸**。

最小数学模型:一个线性循环 hₜ = w · hₜ₋₁(标量版)。那么

```
∂hₜ/∂hₖ = w^(t−k)
|w| < 1  →  w^(t−k) → 0    （梯度消失:远处的信息传不过来）
|w| > 1  →  w^(t−k) → ∞    （梯度爆炸:数值发散）
```

只有 |w| 恰好 = 1 才不衰减不爆炸,而训练中这是测度零的刀刃。向量版结论一致:梯度范数被雅可比矩阵最大奇异值的 (t−k) 次方主导。**这就是"梯度消失/爆炸"问题的本质——它不是工程 bug,而是长链连乘的数学必然。** 这是第 3 章 RNN 的核心痛点,也是 **LSTM 用门控和一条"加法式"的细胞状态高速路来缓解它**的根本动机(LSTM 让信息默认"直传"而非每步乘一个矩阵,从而把连乘变成连加,梯度不再指数衰减——细节第 3 章推导)。

更进一步,**注意力机制(Seq2Seq,第 5 章;Transformer,算法课)直接放弃"逐步传递"**:它让第 t 个位置一步到位地、用 O(1) 的路径长度直接看任意位置 k,从根本上消除了"距离导致的衰减"。**这就是注意力为什么在长程依赖上完胜 RNN 的第一性原因——它把"信息传播路径长度"从 O(距离) 砍到 O(1)。** 代价是每步要看所有位置,计算量从 O(n) 变 O(n²)(算法课"高效注意力"专章处理这个代价)。

### 代码:数值演示梯度沿长链消失

```python
import torch
T = 200
w = torch.tensor(0.9, requires_grad=True)   # |w|<1
h = torch.tensor(1.0)
for _ in range(T):                            # 模拟 200 步线性循环
    h = w * h
h.backward()
print(w.grad.item())   # ≈ T·w^(T-1) ≈ 1.6e-7 ，数量级极小 —— 远处梯度几乎消失

# 对比 |w|>1 会爆炸（把 0.9 改 1.1 重跑，梯度会变成巨大值）
```

### 分析

长程依赖把 NLP 的序列建模分成清晰的几代,正是本课第 3、5 章和算法课的主线:

```
RNN（逐步传递，路径 O(n)）    → 长程靠不住，梯度消失
LSTM/GRU（门控+加法高速路）  → 缓解但仍有限，长文档吃力
注意力 / Transformer（路径 O(1)）→ 直连任意距离，但 O(n²) 计算
长上下文优化（稀疏/线性注意力）→ 在 O(1) 路径与可承受计算间找平衡
```

> **关键结论:NLP 序列模型进化史,很大程度上是一部"如何让相距很远的两个词高效互相影响"的历史。** 记住这条线,你读后面每个架构都能立刻定位它在解长程依赖的哪个子问题。

## 六、难点五 & 六:世界知识/常识,与 Zipf 长尾稀疏

这两个难点放一起讲,因为它们共同回答一个问题:**为什么"只看文本本身"永远不够。**

### 6.1 难点五:意义不全在文本里(世界知识与常识)

很多句子的正确理解依赖**文本之外**的知识:

```
"杯子倒了，咖啡洒了一地。"
```

要理解这句,你得知道:杯子里装着液体、液体受重力会流出、地是水平面会承接……**这些常识没有任何一个词显式写出来。** 经典难题 Winograd Schema(见 4.3)整个就是构造"语法上无法消解、必须靠世界知识"的指代题。

形式化地看,语言是一个**有损编码**:说话人把脑中丰富的情境压缩成一句话,默认听者用共享的世界模型解压。机器没有那个世界模型,就解不开。这条难点定义了两条本课主线:

- **预训练(第 6-7 章):** 让模型在海量文本上学习,把语料里隐含的世界规律(共现、因果、事实)压进参数。这正是"为什么大规模预训练能涌现常识"的根。算法课从语言建模=压缩的角度证明了"压缩逼着模型发现规律";本课从 NLP 任务角度强调"这些规律就是任务所需的世界知识"。
- **检索增强(RAG,第 10 章):** 参数里记不全、会过期的知识(谁是现任 CEO、某 API 的最新签名),就在推理时从外部库**检索**回来拼进上下文。RAG 的存在理由就是承认"世界知识太多太动态,塞不进固定参数"。

> **关键结论:语言是对世界的有损引用,理解语言 = 理解语言 + 拥有世界模型。NLP 要么把世界知识学进参数(预训练),要么实时检索进来(RAG)——这是第 6-7、10 章的分工。**

### 6.2 难点六:Zipf 长尾——尾巴永远见不全

#### 问题

词频极度不均。**Zipf 定律(Zipf's law):** 把词按频次从高到低排名,第 r 名的频次近似满足

```
f(r) ∝ 1 / rˢ ，  s ≈ 1   （英文等多数语言经验值接近 1）
```

即频次乘排名近似常数:最常见的词出现得极频繁(the、的、是),而绝大多数词出现得极少。后果是**重尾(heavy tail)**:海量低频词构成长尾,你的训练语料无论多大,**测试时总会遇到训练里没见过或只见过几次的词(未登录词 OOV、稀有词)。**

#### 推导:长尾到底有多重(可计算)

设 s = 1,词表大小 V。归一化常数是调和数 `H_V = Σ_{r=1}^{V} 1/r`。第 r 个词的概率

```
P(r) = (1/r) / H_V ，   H_V = Σ_{r=1}^V 1/r ≈ ln V + γ   （γ ≈ 0.5772，欧拉常数）
```

`H_V` 随 V 只**对数增长**——这是重尾的数学指纹:概率衰减得慢,头部吃不掉多少质量,尾巴占了一大块。前 k 个词覆盖的 token 比例 ≈ H_k / H_V。我把真实量级算出来(代码已验证):

```
词表 V = 10000，s = 1：
  前 10   个词覆盖   ~30% 的 token
  前 100  个词覆盖   ~53%
  前 1000 个词覆盖   ~77%
  前 5000 个词覆盖   ~93%   ← 剩下一半的词表只占 ~7% 的出现量
```

**头部一百个词就占了一多半的文本,而后面成千上万的词共同分摊那条又长又薄的尾巴。** 这意味着:稀有词的统计样本永远不够,模型对它们的估计天然不可靠。

与 Zipf 互补的是 **Heaps 定律(Heaps' law):** 语料里不同词类型数 V 随 token 总数 n 增长为 `V ≈ K · nᵝ`(β ≈ 0.4–0.6)。它是 Zipf 的推论:**新词的出现速度是次线性的但永不停止——你读得越多,见到的新词越多,但永远见不全。** 这就是"OOV 是宿命"的数学根据。

#### 代码:从语料拟合 Zipf 并量化覆盖

```python
import re, math
from collections import Counter

text = open("corpus.txt", encoding="utf-8").read().lower()
freq = Counter(re.findall(r"[a-z]+", text))          # 英文按词；中文需先分词
ranked = freq.most_common()                          # 按频次降序

# Zipf 检验：log(rank) 与 log(freq) 近似一条斜率 ≈ -s 的直线
for r in (1, 10, 100, 1000):
    if r <= len(ranked):
        word, f = ranked[r-1]
        print(f"rank {r:5d}  freq {f:7d}  rank*freq {r*f}")   # 近似常数 → Zipf

# 覆盖率：前 k 个词占多少 token
total = sum(freq.values())
cum = 0
for k, (_, f) in enumerate(ranked, 1):
    cum += f
    if k in (10, 100, 1000, 5000):
        print(f"top-{k}: {cum/total:.1%} of tokens")
```

#### 分析

Zipf 长尾派生出本课多个工程支柱:

- **子词切分(subword,第 8 章,BPE/WordPiece/SentencePiece):** 不要以"词"为单位(尾巴无穷长、OOV 必然),而是切成更小、可复用的子词片段。这样任何生僻词、拼写错误、新词都能由已知子词拼出来——**OOV 问题被结构性地消解**。这是现代 LLM 分词器的核心,它的存在理由就是 Zipf。
- **平滑(smoothing,第 2 章 n-gram):** 没见过的 n-gram 概率不能给 0(否则整句概率归零),要从见过的事件"匀"一点概率给没见过的。这是统计 NLP 时代对抗稀疏的经典手艺。
- **迁移学习 / 预训练:** 在大语料上学,让稀有现象至少见过几次;再迁移到小数据任务上。低资源语言、专业领域的 NLP 困难,本质都是 Zipf 长尾 + 数据稀缺。

> **关键结论:Zipf 长尾保证了"测试集里永远有训练集没充分覆盖的部分"。NLP 的鲁棒性问题,一大半是在和这条长尾搏斗。** 这也是为什么"在 benchmark 上刷高分"和"在真实长尾输入上稳健"是两回事(第 14 章 鲁棒性会展开)。

## 七、任务地图:把 NLP 整片领域装进一张二维表

理解了难点,再看 NLP 的任务全景就有了坐标。**用两个正交的轴来组织所有任务**,你会发现整个领域井然有序。

**轴一:理解(understanding)vs 生成(generation)。** 输入是文本、输出是某种判断/结构(理解),还是输出也是文本(生成)?
**轴二:token 级 vs 序列级。** 输出是对每个 token 各产生一个结果(token 级,如给每个词标词性),还是对整段输入产生一个整体结果(序列级,如判断整句情感)?

```
                token 级输出                    序列级 / 整体输出
            ┌──────────────────────────┬──────────────────────────────┐
 理 解      │ 序列标注:                │ 文本分类:                    │
(判别)      │  · 分词 (word seg)       │  · 情感分析                  │
            │  · 词性标注 (POS)        │  · 主题分类 / 意图识别       │
            │  · 命名实体识别 (NER)    │  · 自然语言推理 (NLI:蕴含)  │
            │  · 句法/语义角色标注     │  · 语义相似度 / 检索匹配     │
            │   → 第 4 章 CRF/维特比   │   → 第 6 章 BERT 微调        │
            ├──────────────────────────┼──────────────────────────────┤
 生 成      │ (逐 token 生成，但通常   │ 序列到序列 (seq2seq):        │
(产出文本)  │  整体当作序列级任务):    │  · 机器翻译 (MT)             │
            │  自回归语言模型本身      │  · 摘要 (summarization)      │
            │   → 算法课语言建模       │  · 问答 (QA) / 对话          │
            │                          │  · 数据→文本、改写、纠错     │
            │                          │   → 第 5、7、12 章           │
            └──────────────────────────┴──────────────────────────────┘
 横切:信息抽取(IE)= 序列标注+关系判断的组合(第 11 章);
       RAG = 检索(理解/匹配)+ 生成(序列级)的拼接(第 10 章)。
```

几个关键观察,直接连回难点:

- **序列标注(分词/POS/NER)的难点是结构 + 歧义。** 标签之间不独立(动词后面更可能是名词),所以不能逐 token 独立分类,要用 **CRF** 把整条标签序列联合建模(第 4 章)。这是难点二(组合性)在最朴素任务上的体现。
- **文本分类看着简单,坑在长程与世界知识。** 一句话的情感可能被结尾一个转折词("……但是很失望")整个翻转——长程依赖;反讽则需要语用与常识。
- **机器翻译是 NLP 的"果蝇"(drosophila)——所有难点的集大成者:** 词义歧义(选对译词)、句法歧义(重排语序)、长程依赖(一致性)、世界知识(习语、文化专名)、且输出是序列级生成。**注意力机制最早就是在机器翻译里被发明出来的(第 5 章,Bahdanau 2014),用来解决"长句翻译时该对齐到源句哪个词"——这是注意力的诞生地**,算法课讲它的成熟形态,第 5 章讲它为什么诞生。
- **理解任务多用判别式模型(BERT 系,第 6 章);生成任务用自回归/seq2seq(GPT/T5,第 7 章)。** 这条分界对应了预训练的两大范式,第 6-7 章会讲透"为什么理解偏好双向编码、生成必须单向解码"。
- **LLM 时代的大变化(第 9 章):上面几乎所有格子,都可以被同一个大模型用 prompt 统一来做。** 分类→让它输出标签词,NER→让它输出 JSON,翻译→直接 prompt。这是本课的主线高潮:**经典 NLP 的任务边界没有消失,而是被"统一成文本到文本"的范式重新组织了。** 但被统一不等于被解决——歧义、长尾、忠实性这些难点依然在,只是换了战场(第 13、14、15 章)。

> **这张表是你读任何 NLP 论文/系统的定位仪:先问它在哪个格子(理解还是生成、token 级还是序列级),再问它在攻六大难点里的哪几个。** 两个问题答完,你就知道它的方法形态大概长什么样、该和谁比、用什么指标评。

## 八、元问题:为什么 NLP 的评测这么难(贯穿全课的暗线)

这是本章最重要、却最容易被新手忽视的一节。**在图像分类里,评测是"已解决"的:有标准答案(这张图是猫还是狗),算准确率即可,客观、可复现、无争议。NLP 不是这样,而且越往生成走越不是。** 评测难是 NLP 区别于很多领域的元特征,我把它拆成四个层层加深的原因——它们直接来自前面的难点,并为**第 13 章(文本生成的评测:为什么"打分"这么难)** 完整埋线。

**原因一:很多任务没有唯一正确答案(直接来自歧义 + 组合性)。**
一句中文"我没说他偷了钱",英文有至少五种合理译法(取决于重音落在哪个词);一篇文章的合格摘要有无数种写法,用词、详略各异都可能对。**分类任务的"对/错"二元判断,在生成任务里坍塌了**——你拿模型输出去和某一个参考答案做字符串比对,模型给出一个完全正确但用词不同的答案,会被判错。这是离散符号 + 组合性 + 歧义三个难点叠加的直接后果。

**原因二:参考答案有限,但合理输出无限(来自组合性的无穷性)。**
为缓解原因一,评测会给多个参考(multi-reference)。但第三节算过,合法句子有无穷多;给三五个参考,覆盖率仍接近零。**这导致自动指标(如机器翻译的 BLEU、摘要的 ROUGE,本质是和参考的 n-gram 重叠率)系统性偏差:** 它奖励"和参考用词像",而非"意思对、表达好"。一个换了同义词、调了语序的好译文,n-gram 重叠低,BLEU 就低——指标和人类判断脱节。第 13 章会推导 BLEU/ROUGE 的公式并精确分析它在哪儿失灵。

**原因三:人评(human eval)是金标准,但贵、慢、且主观。**
既然自动指标不可靠,最终得靠人打分。但人评有自己的病:**成本高**(雇人逐条读、打分,慢且贵)、**主观**(同一篇摘要,不同人评分差异大,要算标注者间一致性 inter-annotator agreement 来确认靠不靠谱)、**不可复现**(换一批标注者、换一套打分指引,结论可能变)。**于是 NLP 评测陷入两难:自动指标便宜但不准,人评准但贵且不稳。** 整个生成评测领域(第 13 章)就是在这个两难里找折中——包括近年用强模型当裁判(LLM-as-a-judge),但裁判本身又有偏见(偏好长答案、偏好自己风格),套娃式地把评测难度又推高一层。

**原因四:忠实性(faithfulness)无法靠表面匹配检验(来自世界知识 + 生成的开放性)。**
生成模型会**流畅地胡说**——语法完美、读着可信,但事实错误(幻觉,hallucination)。一个摘要可能通顺漂亮却**捏造了原文没有的信息**。这种错误用 n-gram 重叠根本测不出来(措辞可能还很像参考),必须专门检验"输出是否被源/事实支撑"。**忠实性评测是生成式 NLP 最前沿、最棘手的评测问题**,第 13 章会专门处理。

把它和图像分类对照,差异就一目了然:

```
                 图像分类                NLP(尤其生成)
正确答案          唯一、客观              常常多个、主观、无穷
评测方式          准确率(比对标签)       自动指标偏差大 + 人评贵且主观
可复现性          高                      低(依赖标注者与指引)
新型失败模式      —                       流畅地胡说(幻觉)，表面匹配测不出
```

> **关键结论:NLP 的评测难度根植于语言本身的歧义性、组合无穷性与意义的世界依赖。这不是"还没找到好指标"的工程滞后,而是"正确答案本就不唯一"的本质困难。** 因此**贯穿本课的一条暗线是:每讲一个任务,都要同时问"它怎么评测、指标可信吗"**;到第 13 章我们把这条线收口,系统讲生成评测与忠实性。**评测难,是你判断一个 NLP 进展是真进步还是刷分的唯一防线。**

## 九、范式简史:一句话串起从规则到大模型

最后用一条时间线把整个领域的方法演化串起来——**这条线就是本课的叙事主线**,后面每章都是它的一个站点。我用"它如何应对前面六大难点"作为评判,而不是单纯罗列技术。

```
范式            年代(约)    核心思想                    如何应对难点 / 局限
──────────────────────────────────────────────────────────────────────────
1. 规则         1950s-80s   人工写语法规则、词典         精确但脆;歧义/长尾/常识写不尽 → 规则爆炸，覆盖率低
2. 统计         1990s-2000s 从语料估概率 P(y|x)         用概率应对歧义(argmax)、用平滑应对稀疏;
                            (n-gram、HMM、CRF)          但表示靠人工特征，长程与语义弱
3. 神经         2013-2017   学分布式表示 + 序列网络      词向量治离散(难点1)，RNN/LSTM 治长程(难点4)，
                            (word2vec、LSTM、seq2seq)   注意力诞生;仍需大量任务标注
4. 预训练       2018-2019   自监督预训练 + 下游微调      预训练吸收世界知识(难点5)、迁移缓解稀疏(难点6);
                            (BERT、GPT、T5)             "预训练+微调"成为统一配方
5. 大模型统一   2020-now    一个大模型 + prompt 通吃     上节那张任务表几乎全格子被 prompt 统一(第9章);
                            (GPT-3/ICL、指令对齐)       难点未消失，转入忠实性/对齐/长尾鲁棒(第13-15章)
──────────────────────────────────────────────────────────────────────────
```

**读这条线的正确姿势:不是"新范式淘汰旧范式",而是"每代用更少的人工先验、换取更强的从数据中自动获取结构与知识的能力"。** 规则把知识全写死在代码里;统计把知识挪到"从语料估的概率"里但表示还靠人工;神经连表示都让数据学;预训练连任务知识都先无监督学好;大模型连"任务定义"都用自然语言 prompt 给。**先验逐级从代码移向数据,这就是 NLP 七十年的主旋律。** 你会在本课每一章看到这条主旋律的一个具体音符。

本课的章节排布正是沿这条线展开:表示(第 1 章)→ 语言模型(第 2 章)→ 序列网络(第 3 章)→ 结构预测(第 4 章)→ 翻译与注意力诞生(第 5 章)→ 判别式/生成式预训练(第 6-7 章)→ 子词与多语言(第 8 章)→ LLM 统一(第 9 章)→ RAG(第 10 章)→ 信息抽取(第 11 章)→ 对话(第 12 章)→ 评测与忠实性(第 13 章)→ 鲁棒性偏见安全(第 14 章)→ 前沿(第 15 章)→ Capstone(第 16 章)。

## 十、设计权衡与常见坑

把本章的认知压缩成一组"做 NLP 时反复有用"的判断准则与避坑清单:

- **坑 1:把 NLP 当成"图像分类换了输入"。** 离散性让你不能直接套连续世界的工具——没有平滑扰动做数据增强(改一个词可能完全变义)、距离要重新定义、输出常是结构而非标量。**反例提醒:** 别用图像里那套"加噪声/几何变换"思维做文本增强,文本增强(同义替换、回译)极易改变标签。

- **坑 2:迷信单一自动指标。** 看到论文报 BLEU/ROUGE/Accuracy 涨了几个点就信,是新手最大的坑。**先问:这个指标测的是不是我真正关心的东西?** 生成任务尤其要警惕 n-gram 重叠类指标(第八节原因二)。**养成"指标 + 人评 + 案例分析"三件套**的习惯。

- **坑 3:忽视长尾,只看平均。** 在均匀采样的测试集上看平均准确率,会被高频样本主导,掩盖模型在稀有现象(罕见词、长难句、少数群体语言)上的崩溃。**Zipf 保证长尾永远存在**,要专门切出长尾子集评测(第 14 章)。

- **坑 4:用词袋/无序模型处理语序敏感任务。** "狗咬人/人咬狗"的例子要刻进肌肉记忆。**任何丢弃语序的表示(纯词袋、纯平均池化),在语序携带意义的任务上都有上限。**

- **权衡 1:表示的可解释性 vs 表达力。** one-hot 完全可解释(就是 ID)但无语义;稠密嵌入表达力强但难解释。整个领域选了后者并用可解释性研究(算法课/第 15 章)去补这个洞。

- **权衡 2:长程能力 vs 计算成本。** RNN 便宜(O(n))但长程弱;注意力长程强但 O(n²)。没有免费午餐,长上下文是一个持续在做的工程权衡(算法课高效注意力专章)。

- **权衡 3:参数化知识 vs 检索知识。** 知识塞进参数(预训练)→ 快但会过期、会幻觉、改不动;知识放外部库(RAG)→ 可更新、可溯源但依赖检索质量、引入延迟。第 10 章会量化这个权衡。

- **坑 5:以为 LLM 统一了任务 = 解决了任务。** prompt 能做不等于做对、做稳、做得忠实。**被统一的是接口,不是难点。** 歧义、长尾、忠实性原封不动地留在那里——这是本课后半程(第 13-15 章)的全部理由。

## 十一、动手练习

> 这些练习不需要训练任何模型,目的是把本章的概念变成你自己能算、能举例、能论证的东西。

**练习 1(歧义四连,概念)。** 自己各造一个例子,覆盖词义歧义、句法歧义、指代歧义、语用歧义四类,并对每个例子写清:(a) 哪两种(或更多)读法;(b) 人类靠什么消歧(上下文?常识?语调?);(c) 本课哪一章的方法主要负责处理这类歧义。
*提示:句法歧义优先想 PP-attachment 或"修饰范围"型(如"老人和小孩的玩具");语用歧义想反讽或间接言语行为(请求/承诺)。*

**练习 2(组合泛化为什么难,论证)。** 用第三节的 Catalan 数论证:为什么"句子级别查表/记忆"不可行,以及为什么模型必须从词/短语泛化到整句。然后构造一个最小例子说明组合泛化的考法:列出训练集见过的几个"形容词+名词"组合,给出一个未见过的新组合,说明一个只会记忆共现的模型会怎么错。
*提示:训练见过 {红的车、蓝的杯},测试给"蓝的车";想想纯共现统计模型为什么可能给出低概率甚至拒识。可参考 SCAN / COGS 这类组合泛化数据集的设计思路(可只描述,不必下载)。*

**练习 3(Zipf 实测,编程)。** 找一份纯文本语料(英文用 NLTK 自带的 Gutenberg / Brown 语料,或任意一本电子书 .txt;中文需先用 jieba 之类分词)。用第六节的代码:(a) 画 log(rank)–log(freq) 图,目测斜率是否 ≈ −1;(b) 计算前 10 / 100 / 1000 个词覆盖多少 token;(c) 把语料切成两半,统计后半部分里有多少词从未在前半出现(OOV 率),并用 Heaps 定律解释为什么语料越大 OOV 率下降但永不归零。
*提示:画图前对 freq 和 rank 取对数;Zipf 直线在头部更准、极尾因采样噪声会偏离,这本身就值得讨论。覆盖率用累积频次/总频次。*

**练习 4(为什么 NLP 评测比图像分类难,综合论述)。** 写一段 300–500 字的论证,从本章六大难点出发,解释 NLP(尤其生成任务)评测为何本质上难于图像分类。要求至少把"正确答案不唯一""参考有限而合理输出无穷""人评贵且主观""流畅地胡说(幻觉)用表面匹配测不出"这四点各自挂到它来源的难点上(歧义/组合性/世界知识),并各举一个具体任务说明(如机器翻译、摘要)。
*提示:这道题是第 13 章的预习。好答案会明确指出"这不是工程滞后,而是'正确答案本就不唯一'的本质困难",并据此给出"指标+人评+案例"三件套的实践建议。*

## 十二、源码 / 论文导读

按"先读综述建立地图,再读原典理解机制"的顺序:

- **教科书地图(强烈建议作为全程参考):** Jurafsky & Martin, *Speech and Language Processing*(SLP,第 3 版网络草稿免费)。本章对应它的前言/导论、句法歧义与解析章、n-gram 与稀疏/平滑章、词义消歧章。**先通读它的"导论"和"N-gram Language Models"两章,你对本章六大难点会有更系统的落点。** 另一本 Manning & Schütze, *Foundations of Statistical NLP* 是 Zipf/Heaps、统计 NLP 的经典出处(具体页码以你手头版本为准,标注为待核)。

- **离散性 → 分布式表示(难点一,接第 1 章):** Firth (1957) 的分布假设原话;Mikolov et al. (2013) word2vec 两篇(*Efficient Estimation* 与 *Distributed Representations*)。**先看 word2vec 论文里"为什么用上下文预测词/词预测上下文"的动机段。**

- **组合性与解析(难点二,接第 4 章):** 任意一本 NLP 教材的 CFG/PCFG 章节;了解 CKY 解析与 Catalan 数的关系。**重点理解"为什么解析要用概率给树排序",而非枚举所有 Catalan 棵树。**

- **歧义与世界知识(难点三、五):** Winograd Schema Challenge(Levesque et al., 2012)论文,理解它如何刻意构造"语法无法消解、必须靠常识"的题——这是检验"是否真懂"的经典探针。

- **长程依赖(难点四,接第 3、5 章):** Hochreiter & Schmidhuber (1997) LSTM 原论文的"constant error carousel"一节,理解加法式细胞状态如何对抗梯度消失;Bengio et al. (1994) 关于长程依赖与梯度消失的早期分析;Bahdanau et al. (2014) 注意力机制在机器翻译里的诞生(第 5 章详读)。

- **评测难(第八节,接第 13 章):** Papineni et al. (2002) BLEU 原论文(读它的动机与定义,理解 n-gram 精确率 + 简短惩罚 brevity penalty);Lin (2004) ROUGE。**带着批判读:它们各自假设了什么、在什么情况下与人类判断脱节。**

- **开源实现(用来把概念落到代码):**
  - `nltk`:自带 Gutenberg/Brown 语料、`FreqDist`(做练习 3 的 Zipf 统计)、WordNet(看词义义项)。
  - HuggingFace `tokenizers` / `transformers`:看 BPE/WordPiece 分词器实现,理解子词如何应对 Zipf(第 8 章);看 `AutoModel` 的嵌入层如何把 token id 映成向量(难点一的落地)。
  - `sentence-transformers`、`faiss`:第 10 章 RAG 会用,本章可暂存。

> 本章是"问题导览",上面多数论文的**机制推导**在本课后续各章或算法课/强化学习课里展开;这里你只需读它们的**动机/问题定义部分**,把"它在解六大难点的哪一个"对上号即可。

## 十三、小结与承上启下

这一章没教你任何一个模型,但给了你**读完整门课所需的问题坐标系**。把它压成必须记住的几条:

- **语言有六个内禀难点:离散(无天然度量)、组合(有限词→无限句、结构敏感)、歧义(词义/句法/指代/语用四层)、长程依赖、需要世界知识与常识、Zipf 长尾稀疏。** 你能各举一个可计算/可举证的例子。
- **这六个难点不是并列的清单,而是 NLP 全部方法的发源地:** 离散→嵌入(第 1 章)、组合→结构预测与 CRF(第 4 章)、歧义→上下文模型与 `argmax P(y|x)`(贯穿)、长程→RNN/LSTM/注意力(第 3、5 章)、世界知识→预训练与 RAG(第 6-7、10 章)、长尾→子词与平滑(第 1、8 章)。
- **任务地图用"理解 vs 生成"和"token 级 vs 序列级"两个轴组织,** 是你定位任何 NLP 工作的坐标。机器翻译是集所有难点之大成的"果蝇",也是注意力的诞生地。
- **评测难是 NLP 的元问题,根植于"正确答案本就不唯一",不是工程滞后。** 这条暗线贯穿全课,第 13 章收口;它是你区分"真进步"与"刷分"的唯一防线。
- **范式从规则→统计→神经→预训练→大模型,主旋律是"先验逐级从代码移向数据"。** 大模型统一了任务接口,但没有消灭难点——难点转入忠实性、对齐、长尾鲁棒,正是本课后半程的主题。

**承上启下:** 既然语义不在离散符号本身、而在"符号如何共同出现",那么 NLP 的第一块地基必然是——**如何把离散的词映射成一个携带语义的连续向量。** 这正是下一章(第 1 章《文本表示:从 one-hot 到 word2vec 与 GloVe》)的全部内容:我们将把本章点到的"分布假设"变成一个可优化的目标函数,亲手推导 word2vec 的 skip-gram 损失与负采样,看清"猫"和"狗"的向量是怎么自己靠近的。**带着本章这张难点地图进第 1 章,你会清楚地看到:词向量,就是我们对"离散性"这第一个难点的第一记重拳。**
