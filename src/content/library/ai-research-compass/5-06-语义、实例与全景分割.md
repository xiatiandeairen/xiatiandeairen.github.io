> 这一章把你从"会做图像分类、知道有分割这回事"带到"能在白板上推清 FCN 为什么需要 skip、转置卷积棋盘伪影的代数根源、RoIAlign 相对 RoI Pooling 把量化误差降了多少,能徒手写出双线性上采样和 Dice loss,能读懂 Mask R-CNN 与 DeepLab 源码的每个关键模块"。分类只回答"图里有什么",分割回答"每个像素是什么、属于哪个个体"——这是从 1 个标签到 H×W 个标签的跃迁,所有难点都从这个维度爆炸里长出来。

## 一句话引言

分割任务的本质是**稠密预测(dense prediction)**:输出的空间分辨率必须和输入对齐,网络要同时具备"看得广(大感受野、强语义)"和"定位准(高分辨率、保边界)"这对矛盾的能力。本章从这对矛盾出发,推透三件事:**怎么把分类网络改造成逐像素预测器(FCN/U-Net/DeepLab)**、**怎么在检测框架里抠出像素级掩码而不引入致命的对齐误差(Mask R-CNN + RoIAlign)**、**怎么评测和对付类别不均衡(mIoU / AP^mask / Dice)**。

## 1. 动机:从一个标签到 H×W 个标签

### 1.1 三种分割任务,先把边界划清

混淆这三个任务是新手最常见的概念错误。用一张"路上有两个人、一辆车、一片天空"的图来区分:

| 任务 | 输出 | 区分同类个体? | 处理背景(stuff)? | 典型评测 |
|---|---|---|---|---|
| **语义分割** (semantic) | 每像素一个类别 | 否(两个人都是 `person`) | 是(天空/路面也分类) | mIoU |
| **实例分割** (instance) | 每个物体一个掩码+类别 | 是(person#1, person#2) | 否(只管可数的 thing) | AP^mask |
| **全景分割** (panoptic) | 每像素 = (类别, 实例id) | 是(对 thing) | 是(对 stuff) | PQ |

两个关键术语,记牢:

- **thing**:可数、有清晰个体边界的物体——人、车、狗。实例分割只关心 thing。
- **stuff**:不可数、无个体概念的区域——天空、草地、马路。语义分割平等对待 thing 和 stuff,但不区分 thing 的个体。

**全景分割(Kirillov et al., 2019)是前两者的并集**:对 stuff 像素只给类别(等价语义分割),对 thing 像素给"类别 + 实例 id"(等价实例分割),且要求**每个像素恰好被分配一次**(无重叠、无空洞)。这个"恰好一次"的约束,正是它比"语义+实例两个模型拼起来"难的地方——拼接会产生掩码重叠和冲突,全景要求一个统一的、无歧义的像素划分。

### 1.2 核心矛盾:分辨率 vs 语义

为什么不能直接拿 ResNet 输出每像素类别?因为分类网络为了拿大感受野和强语义,**一路下采样**。看一个典型 backbone 的空间分辨率轨迹(输入 224×224):

```
stage     输出步长(stride)   特征图大小      感受野(量级)
conv1     /2                 112×112        ~7
maxpool   /4                 56×56          ~11
res2      /4                 56×56          ~35
res3      /8                 28×28          ~99
res4      /16                14×14          ~291
res5      /32                7×7            ~483   ← 语义最强,但只剩 7×7
```

**矛盾摆出来了**:res5 语义最强(感受野近 500,几乎覆盖全图,知道"这是车"),但分辨率只有 7×7——把它放大回 224×224,每个特征像素要负责 32×32 的图像块,边界全糊。反过来,conv1 分辨率高(112×112,边界清晰),但感受野只有 7,只看得到局部纹理,根本不知道这是车还是路。

```
深层特征:  语义强 ✓   分辨率低 ✗   →  知道"是什么",不知道"边界在哪"
浅层特征:  语义弱 ✗   分辨率高 ✓   →  知道"边界在哪",不知道"是什么"
```

**所有语义分割架构,本质都是在回答同一个问题:如何把深层的语义和浅层的分辨率重新组合起来。** FCN 用 skip 加法、U-Net 用对称 concat、DeepLab 用空洞卷积"干脆不下采样到那么低"。下面逐个推。

## 2. FCN:把分类网络改造成逐像素预测器

### 2.1 问题:全连接层杀死了空间结构

经典分类网络末端是 `flatten → FC → softmax`。FC 层做了两件致命的事:**(1) 把 H×W×C 的特征图压成一个向量,空间位置信息被抹平;(2) 权重维度和输入尺寸绑死**(FC 的权重矩阵是 `(H·W·C) × num_classes`,换个输入尺寸就不能用了)。

Long, Shelhamer & Darrell(2015,CVPR,FCN)的洞察:**全连接层可以等价改写成卷积层**。

**FC→Conv 的等价推导。** 设最后一层特征图是 `7×7×512`,FC 把它映射到 4096 维。FC 看到的是 flatten 后的 `7·7·512 = 25088` 维向量,权重 `W ∈ ℝ^{4096×25088}`。现在用一个卷积层:**核大小 = 输入特征图大小 = 7×7,输入通道 512,输出通道 4096**。这个卷积核的参数量是 `7·7·512·4096`,和 FC 的 `25088·4096` **完全相等**(`7·7·512 = 25088`)。当输入恰好是 7×7 时,这个卷积只能放下一个核位置,输出 `1×1×4096`——和 FC 输出在数值上**逐元素相同**。

```
FC:    输入 flatten 成 25088 向量,  W: 4096×25088,  输出 4096
等价 Conv: kernel 7×7×512 → 4096,   参数 7·7·512·4096 = 25088·4096,  输入 7×7 时输出 1×1×4096
```

**改成卷积后发生了什么神奇的事**:卷积对输入尺寸**不再敏感**。喂一张 384×384 的图,backbone 出来是 `12×12×512`,这个 7×7 卷积可以在上面滑动,输出 `6×6×4096`——一张**密集的预测热力图**,而不是单个向量。换句话说,FC→Conv 让网络从"输出一个类别"变成"输出一张逐空间位置的类别图"。这就是 "fully **convolutional** network" 名字的来历:**全卷积 = 处处可滑动 = 天然输出稠密预测**。

最后接一个 `1×1` 卷积把通道压到 `num_classes`,得到 `H'×W'×num_classes` 的粗粒度分割 logits。剩下的问题:H'×W' 太小(/32),怎么放大回原图?

### 2.2 转置卷积:可学习的上采样,以及它怎么"反着算"

**转置卷积(transposed convolution,又叫 deconvolution / fractionally-strided convolution)** 是 FCN 用来上采样的可学习算子。名字里的 "deconvolution" 是误导——它**不是卷积的逆运算**,而是"卷积的梯度算子(伴随)被拿来当前向用"。

先把普通卷积写成矩阵乘法,转置卷积的"转置"就一目了然。考虑 1D 情形:输入 `x ∈ ℝ⁴`,卷积核 `k = [k₀, k₁, k₂]`,stride=1,no padding,输出 `y ∈ ℝ²`。卷积 `y = C x` 的矩阵 C 是:

```
        x₀  x₁  x₂  x₃
y₀ =    k₀  k₁  k₂  0
y₁ =    0   k₀  k₁  k₂

C ∈ ℝ^{2×4}  (输出短,输入长 → 下采样/降维)
```

**转置卷积就是用 Cᵀ 做前向**:`x' = Cᵀ y'`,把 `y' ∈ ℝ²` 映回 `ℝ⁴`:

```
Cᵀ ∈ ℝ^{4×2}:
       y₀  y₁
x'₀ =  k₀  0
x'₁ =  k₁  k₀
x'₂ =  k₂  k₁
x'₃ =  0   k₂
```

**关键直觉**:普通卷积是"多个输入加权求和成一个输出"(many→one);转置卷积反过来,"一个输入散播(scatter)到多个输出,再把重叠处累加"(one→many,overlap-add)。这正是上采样需要的——把小特征图的每个值"摊开"到大特征图的一片区域。

**输出尺寸公式。** 普通卷积:`o = ⌊(i − k + 2p)/s⌋ + 1`。转置卷积是它的逆向解(给定 o 反解 i),输出尺寸:

```
o_transposed = s·(i − 1) + k − 2p + output_padding
```

例:`i=2, k=4, s=2, p=1` → `o = 2·1 + 4 − 2 = 4`,把 2 放大成 4(2 倍上采样)。stride 在这里起的是"放大倍数"的作用——stride=2 意味着输入相邻两个值在输出上间隔 2 个像素,中间的空隙由核填充。

### 2.3 棋盘伪影(checkerboard artifacts):机制与代数根源

转置卷积有一个臭名昭著的毛病:输出常出现**棋盘状的明暗格子**。这不是 bug,是数学结构的必然产物。

**根源:不均匀重叠(uneven overlap)。** 当 `kernel_size` 不能被 `stride` 整除时,overlap-add 过程里不同输出位置被覆盖的次数不同。用 1D 最小例子算清楚:`k=3, s=2`,输入每个值散播 3 格,步长 2,看每个输出位置被几个输入贡献:

```
输入位置:    x₀          x₁          x₂
散播(k=3):  [a b c]      [a b c]      [a b c]
对齐到输出(s=2):
  out:  0   1   2   3   4   5   6
  x₀:   a   b   c
  x₁:           a   b   c
  x₂:                   a   b   c
覆盖次数: 1   1   2   1   2   1   1
              ↑       ↑
        out2,out4 被 2 个核覆盖,out1,out3,out5 只被 1 个

→ 偶数位置幅度系统性偏高,奇数位置偏低 ⇒ 周期性明暗 = 棋盘
```

2D 情形下两个方向的不均匀叠乘,棋盘格更明显。常见触发组合:`k=3,s=2`、`k=4,s=3`,以及最经典的 `k=4,s=2,p=1`(虽然 4 能被 2 整除,但 padding 让边界覆盖次数不均,中心区域反而没事,边界有伪影)。

**两条修法,理解为什么有效:**

1. **核大小被 stride 整除**(如 `k=4,s=2`,中心区每点恰好被 4 个核均匀覆盖)。能消除中心区伪影,但边界仍可能不均。
2. **resize-conv(Odena et al., 2016 推荐)**:先用**双线性/最近邻插值**做无参数上采样到目标尺寸,再接一个普通卷积学特征。**为什么这条最干净**:双线性插值的权重是**解析、均匀、与位置无关**的,不存在 overlap-add 的不均匀;卷积只负责学语义,不负责放大。代价是插值不可学(但实践证明够用),且多一次卷积。现代分割/生成模型(U-Net 变体、许多 diffusion 的 decoder)默认走这条路。

```python
# 棋盘伪影的两种上采样对比
import torch.nn as nn

# 方式A:转置卷积(可能棋盘) —— k 能被 s 整除以缓解
up_deconv = nn.ConvTranspose2d(C, C, kernel_size=4, stride=2, padding=1)

# 方式B:resize-conv(推荐,无棋盘) —— 上采样与学特征解耦
up_resize = nn.Sequential(
    nn.Upsample(scale_factor=2, mode='bilinear', align_corners=False),
    nn.Conv2d(C, C, kernel_size=3, padding=1),   # 普通卷积只负责学特征
)
```

> 坑:`align_corners` 参数。`align_corners=True` 把角点像素中心对齐到 [-1,1] 的端点,`False` 则按像素"面积中心"对齐。两者在不同分辨率间不一致会导致掩码偏移半个像素——训练和推理必须用**同一个设置**,否则评测掉点且难查。PyTorch 默认 `False`,torchvision 分割模型也多用 `False`。

### 2.4 为什么必须 skip:浅层细节的代数注入

FCN 直接把 /32 的 logits 一步双线性放大 32 倍(称 **FCN-32s**),结果**边界极糊**——因为 /32 特征里一个像素管 32×32 图像块,放大只是把这个粗块插值平滑,边界信息根本没被编码进去,插值变不出来。

FCN 的解法是 **skip connection(跳跃连接)**:把浅层(高分辨率)特征的预测加到深层上采样后的预测上。**FCN-16s** 的做法:

```
深层 res5 → 1×1 conv → score32 (/32)
score32 → 转置卷积 ×2 → up_score (/16)          # 先放大到 /16
浅层 res4(pool4) → 1×1 conv → score16 (/16)     # 浅层直接出预测
fused = up_score + score16                       # 逐元素相加(同分辨率)
fused → 转置卷积 ×16 → 最终预测 (原分辨率)
```

**为什么是加法、为什么有效。** 两路都是 `num_classes` 通道的 logits,空间分辨率对齐到 /16 后逐元素相加,等价于**把两个独立分类器的 logits 做集成(ensemble)**:深层提供"这块大概是车"的强语义先验,浅层提供"车的边界精确在这条线上"的高频细节。加法让最终 logits 在边界处由浅层主导、在区域内部由深层主导。**FCN-8s** 再融一层 pool3(/8),边界进一步变锐。论文里 32s→16s→8s 的 mIoU 单调上升,直接证明了浅层细节的价值(具体数值见 §6,此处不背)。

**注意 skip 的本质**:它不是在"融合特征",而是在"融合不同尺度的预测",绕过了"深层特征丢掉的高频无法重建"这个信息论硬限制——丢了的信息只能从还保留它的浅层那里**取**,不能从深层**变**出来。

## 3. U-Net:对称编码器-解码器,小数据之王

### 3.1 问题与设计:把 skip 推到极致

FCN 的 skip 只融了 2~3 层,且是**预测层面**(先各自 1×1 到 num_classes 再加)的融合。Ronneberger, Fischer & Brox(2015,MICCAI,U-Net)面向**医学图像**(标注极少、边界要求极高)把 skip 推到极致:**每个下采样层级都有一条对称的 skip,且融合发生在特征层面(concat),而非预测层面**。

```
编码器(收缩)                         解码器(扩张)
x →[conv×2]→ f1 ──────skip(concat)──────→ [concat]→[conv×2]→ out
        ↓ pool                                    ↑ up
       [conv×2]→ f2 ──skip──→ [concat]→[conv×2]→ ↑ up
              ↓ pool                        ↑ up
             [conv×2]→ f3 ─skip─→ [concat] ↑ up
                    ↓ pool              ↑ up
                   [conv×2]→ bottleneck ↑ up
```

形状像字母 U,故名。两点和 FCN 的关键差异:

1. **concat 而非 add**:解码器把上采样特征和对应编码器特征在**通道维拼接**,后续卷积自己学怎么混合。比 FCN 的固定加法表达力更强——网络可以学到"边界处多信浅层、内部多信深层"的非线性组合,而不是被迫等权相加。代价是通道翻倍、显存和计算更大。
2. **解码器是对称的、逐级上采样**:每级放大 2 倍并融一次 skip,细节逐步恢复,而非 FCN 那样一步放大 16/8 倍。

### 3.2 为什么 U-Net 在小数据上特别强

这是一个常被问到、值得讲透的点,有三个原因:

- **skip 提供了梯度高速路**:解码器浅层能直接从编码器对应层拿到梯度和特征,缓解深网络在小数据下的优化困难(类似残差的作用)。
- **没有全连接、纯卷积**:参数量相对可控,且卷积的**平移等变性(translation equivariance)** 是强归纳偏置,小数据下不容易过拟合。医学图像里器官可能出现在任意位置,平移等变正好匹配。
- **重度数据增强友好**:原论文用**弹性形变(elastic deformation)** 作增强——这恰好模拟生物组织的真实形变,极大扩充了有效样本。U-Net 的成功一半归功于这个增强策略,不只是架构。

> 一句话:U-Net = 强归纳偏置(全卷积)+ 极致 skip(逐级 concat)+ 任务匹配的增强。这套组合到今天仍是 diffusion 模型 denoiser 的主干结构(diffusers 库的 `UNet2DModel`),足见其生命力。

## 4. DeepLab:空洞卷积——不下采样也能拿大感受野

### 4.1 问题:能不能根本就别把分辨率降那么低?

FCN/U-Net 是"先降到 /32,再想办法升回来"。DeepLab(Chen et al., 系列工作 v1~v3+)换了个思路:**既然下采样是分辨率损失的根源,那就在保持分辨率的同时把感受野做大**。工具是**空洞卷积(atrous / dilated convolution)**。

### 4.2 空洞卷积:扩感受野不降分辨率的代数

普通 3×3 卷积采样 3 个连续位置。空洞卷积引入**膨胀率(dilation rate)r**,在采样点之间插 `r−1` 个"洞"(跳过的位置),核**不变胖**(参数量不变)但**视野变大**:

```
普通卷积 (r=1):    采样位置  • • •           覆盖 3 个像素
空洞卷积 (r=2):    采样位置  •   •   •       覆盖 5 个像素(中间跳1)
空洞卷积 (r=4):    采样位置  •       •       •   覆盖 9 个像素(中间跳3)

1D 输出:  y[i] = Σₖ w[k] · x[i + r·k]      (r·k 就是"带洞"的采样偏移)
```

**有效核大小(感受野贡献)推导。** 对 k×k 的核、膨胀率 r,等效核大小:

```
k_eff = k + (k − 1)·(r − 1)
```

3×3、r=2 → `k_eff = 3 + 2·1 = 5`(等效 5×5 的视野,但只有 9 个参数,不是 25 个)。r=4 → `k_eff = 3 + 2·3 = 9`。**关键收益:感受野随 r 线性增长,参数量和计算量不变**(仍是 9 次乘加)。

**怎么用它替代下采样。** 标准 ResNet 在 res4/res5 用 stride=2 下采样。DeepLab 把这些 stride 改成 1(**不再下采样**),同时把后续卷积的 dilation 调大来补偿丢失的感受野。这样输出步长(output stride)可以保持在 /8 甚至 /16,而**感受野和原来 /32 时一样大**。结果:特征图分辨率高 4 倍(/8 vs /32),语义却不打折——直接解决了 §1.2 的矛盾。

```python
# 把 ResNet 的 res5 从"下采样+r=1"改成"不下采样+r=2",感受野不变、分辨率翻倍
# 原始: stride=2, dilation=1   →  分辨率减半
# 改后: stride=1, dilation=2   →  分辨率不变,k_eff=5 补偿感受野
import torch.nn as nn
conv_res5 = nn.Conv2d(512, 512, kernel_size=3, stride=1,
                      padding=2, dilation=2)   # padding=dilation 保持尺寸
```

> 坑:**gridding / 棋盘采样问题**。若连续多层用同一个 r(比如都 r=2),采样点会落在固定的稀疏网格上,相邻像素用到的输入完全不重叠,信息出现"格子状空洞"。解法是用**锯齿状膨胀率**(如 r=1,2,3 循环,称 hybrid dilated convolution),让连续层的采样覆盖连续。

### 4.3 ASPP:多尺度物体怎么办

单一膨胀率只对应单一尺度。图里既有近处的大车又有远处的小车,需要**多尺度并联**。**ASPP(Atrous Spatial Pyramid Pooling)** 把若干不同膨胀率的空洞卷积**并联**,各自捕获一个尺度,再拼接融合:

```
input feature
  ├── 1×1 conv                    (尺度: 点)
  ├── 3×3 conv, r=6               (中尺度)
  ├── 3×3 conv, r=12              (大尺度)
  ├── 3×3 conv, r=18              (更大尺度)
  └── image pooling → 1×1 → upsample   (全局上下文:全图平均池化)
       ↓ 全部 concat → 1×1 conv 融合 → 输出
```

**为什么要那条 image pooling 分支**:当膨胀率大到接近特征图尺寸时,空洞卷积会退化(有效采样点大量落在 padding 上,变成只看中心一个点)。全局平均池化提供一个**真正的全图上下文**作为兜底,告诉网络"整张图的大概语境"。这是 DeepLabv3 相对 v2 的重要改进。

### 4.4 CRF 后处理:早期手段,为什么后来被弃

DeepLab v1/v2 在网络输出后接一个 **全连接条件随机场(fully-connected CRF,Krähenbühl & Koltun, 2011)** 做后处理,锐化边界。它的能量函数:

```
E(label assignment) = Σᵢ ψ_unary(xᵢ)  +  Σᵢ<ⱼ ψ_pair(xᵢ, xⱼ)

ψ_unary(xᵢ) = −log P(xᵢ)                            # 网络给的逐像素概率(数据项)
ψ_pair(xᵢ,xⱼ) = μ(xᵢ,xⱼ)·[ w₁·exp(−|pᵢ−pⱼ|²/2θα² − |Iᵢ−Iⱼ|²/2θβ²)  # 外观核:位置近且颜色近
                          + w₂·exp(−|pᵢ−pⱼ|²/2θγ²) ]                  # 平滑核:位置近
```

**直觉**:成对项鼓励"位置相邻 + 颜色相近的像素取相同标签"——这正好把网络糊掉的边界,用原图的颜色/位置不连续性重新对齐。用平均场近似可高效求解。

**为什么 v3 之后弃用 CRF**:(1) 不可微、是独立后处理,无法端到端训练;(2) 引入一堆需手调的超参(θα,θβ,θγ,w);(3) 当骨干网络(空洞卷积 + ASPP)本身分辨率和边界已经够好时,CRF 带来的提升微乎其微,不值这份复杂度。**这是一个典型的"算法进步消解了后处理"的案例**——早期补丁的价值随主模型变强而归零。

## 5. Mask R-CNN:在检测框架里抠像素,以及 RoIAlign 的对齐革命

### 5.1 问题:实例分割为什么不能照搬语义分割

语义分割不区分个体,而实例分割必须把每个物体单独抠出来。**Mask R-CNN(He et al., 2017,ICCV best paper)** 的思路:**先检测(找出每个实例的框),再在框内做二值语义分割(前景/背景)**。它在 Faster R-CNN(两阶段检测器)上**并行加一个掩码分支**:

```
图 → backbone+FPN → RPN(出候选框)
  → 对每个候选框 RoIAlign 抠出固定大小特征(如 14×14)
      ├── 分类分支 → 类别
      ├── 框回归分支 → 精修框
      └── 掩码分支(小FCN) → 每类一张 m×m 二值掩码(如 28×28)
```

三个分支**并行、解耦**。掩码分支为**每个类别**预测一张独立的二值掩码(K 个类就 K 张),最后**只取分类分支选中的那个类**对应的掩码——这叫 **decouple mask & class**。为什么解耦?如果让一张掩码同时承担"分割 + 分类",类间会竞争(softmax 跨类归一化);解耦后每类掩码独立 sigmoid,**掩码只管"是不是前景",分类的事交给分类分支**,二者不抢。论文消融证明这个解耦显著优于耦合设计。

### 5.2 RoIAlign:为什么"量化取整"对像素级任务是灾难

这是本章技术含量最高、也最值得推透的一点。

Faster R-CNN 用 **RoI Pooling** 把任意大小的候选框特征池化成固定大小(如 7×7)。它做了**两次量化取整(quantization)**:

```
量化1:把浮点框坐标 round 到整数特征格   x_feat = round(x_image / stride)
量化2:把框划成 k×k 个 bin 时,每个 bin 的边界再 round 到整数
```

**对检测,这点误差无所谓**(框本来就允许 IoU 误差);**对像素级掩码,这是灾难**。算一笔量化误差账:

**量化误差推导。** 设 backbone 步长 `stride = 16`。RPN 给出一个候选框,左上角图像坐标 `x = 200`。

```
映射到特征图:  x_feat = 200 / 16 = 12.5
RoI Pooling 量化:  round(12.5) → 12 或 13(取决于舍入约定/实现)
误差(特征格):     |12.5 − 13| = 0.5 个特征像素
折算回原图:        0.5 × 16 = 8 个像素 !!!
```

**一次量化就在原图上引入 8 像素的偏移**;框划 bin 时第二次量化再叠加。对一个 28×28 的掩码,8 像素相当于掩码边界整体平移了将近 1/4——**预测的掩码和物体真实边界系统性错位**,小物体直接报废。

**RoIAlign 的解法:全程不取整,用双线性插值。**

```
1. 框坐标保留浮点:  x_feat = 12.5,不 round
2. 每个 bin 内取若干采样点(如 2×2=4 个),采样点坐标也是浮点
3. 每个浮点采样点的值,用双线性插值从相邻 4 个整数特征格算出
4. bin 内对采样点做平均(或 max)池化
```

**双线性插值公式**(采样点 (x,y),落在整数格 `(x0,y0),(x1,y1)` 之间,`dx=x−x0, dy=y−y0`):

```
f(x,y) = (1−dx)(1−dy)·f(x0,y0) + dx(1−dy)·f(x1,y0)
       + (1−dx)dy·f(x0,y1)     + dx·dy·f(x1,y1)
```

四个角权重之和恒为 1,且**关于采样点坐标处处可微**——梯度能传回特征图,整个 RoIAlign 端到端可训练,且**零量化误差**。

**误差对比小结:**

| | 框坐标 | bin 边界 | 原图误差量级 | 可微 |
|---|---|---|---|---|
| RoI Pooling | round(取整) | round(取整) | ~stride/2 ≈ 8 px(stride=16) | 对坐标不可微 |
| **RoIAlign** | 浮点保留 | 浮点保留 | **0**(双线性精确插值) | **处处可微** |

论文消融:仅把 RoI Pooling 换成 RoIAlign,mask AP 大幅提升,且**stride 越大(深层特征)提升越显著**——完全符合上面"误差 = stride/2"的推导,stride 大则量化误差大,对齐的收益就大。**这是"理论误差量级精确预测实验收益"的漂亮案例,务必理解。**

```python
# RoIAlign 核心思想:双线性采样(简化版,展示机制,非高效实现)
import torch

def bilinear_sample(feat, x, y):
    """feat: [C,H,W];(x,y)浮点坐标 → 双线性插值出 [C] 向量。"""
    x0, y0 = int(x), int(y)
    x1, y1 = x0 + 1, y0 + 1
    dx, dy = x - x0, y - y0
    # 四角加权,权重和恒为 1
    v = ((1-dx)*(1-dy)*feat[:, y0, x0] + dx*(1-dy)*feat[:, y0, x1]
       + (1-dx)*dy   *feat[:, y1, x0] + dx*dy   *feat[:, y1, x1])
    return v   # 对 x,y 可微 → 梯度可回传

def roi_align_single(feat, box, out=7, samples=2):
    """box=(x1,y1,x2,y2)浮点(特征图坐标),不做任何取整。"""
    C = feat.shape[0]
    x1, y1, x2, y2 = box
    bin_w, bin_h = (x2 - x1) / out, (y2 - y1) / out
    res = torch.zeros(C, out, out)
    for i in range(out):
        for j in range(out):
            acc = 0.0
            for si in range(samples):           # bin 内规则采样点
                for sj in range(samples):
                    sx = x1 + (j + (sj + 0.5)/samples) * bin_w  # 浮点采样坐标
                    sy = y1 + (i + (si + 0.5)/samples) * bin_h
                    acc = acc + bilinear_sample(feat, sx, sy)
            res[:, i, j] = acc / (samples * samples)            # bin 内平均
    return res
# 生产环境用 torchvision.ops.roi_align,这里只为讲清"浮点采样+双线性"的机制
```

### 5.3 全景分割怎么统一

朴素做法:跑一个语义模型(管 stuff)+ 一个实例模型(管 thing),后处理融合。问题是两路掩码会重叠/冲突,违反"每像素恰好一次"。两条主流统一路线:

- **Panoptic FPN(Kirillov et al., 2019)**:在 Mask R-CNN 的 FPN 上再加一个轻量语义分割分支,thing 用实例分支、stuff 用语义分支,用一套启发式(按置信度/面积)解决冲突。工程简单但仍有手工融合规则。
- **Mask2Former / MaskFormer(Cheng et al., 2021-2022)**:把三种分割**统一成"预测一组(掩码 + 类别)对"** 的集合预测问题(借鉴 DETR 的 query + 二分匹配),一个模型一套 loss 同时做语义/实例/全景,**彻底取消了 thing/stuff 的架构区分**。这是当前 SOTA 范式,下一章 Transformer 检测会展开 query-based 的机制(此处只点到)。

## 6. 评测与类别不均衡

### 6.1 mIoU:语义分割的标准度量

**IoU(Intersection over Union,交并比)** 对单个类:

```
IoU_c = |预测为c ∩ 真值为c| / |预测为c ∪ 真值为c|  =  TP / (TP + FP + FN)
```

**mIoU = 各类 IoU 的算术平均**(对 num_classes 个类平均)。为什么用 IoU 而不用逐像素准确率(pixel accuracy)?**因为类别极不均衡**:一张街景图 90% 是天空和马路,即使把所有像素都判成"马路",pixel accuracy 也能到 90%,但这模型毫无用处。IoU 对每个类**独立**算交并比再平均,小类(行人、交通灯)和大类(天空)**权重相同**,逼着模型把小类也分对。**这是 IoU 相对 accuracy 的本质优势。**

### 6.2 AP^mask:实例分割的度量

实例分割借用检测的 **AP(Average Precision)**,但**把"框的 IoU"换成"掩码的 IoU"** 来判定一个预测是否命中真值(TP)。`AP@0.5` 用 0.5 的掩码 IoU 阈值,COCO 的主指标 `AP` 是 0.5:0.05:0.95 共 10 个阈值下 AP 的平均。阈值越高,对掩码边界精度要求越苛刻——这也是为什么 RoIAlign 的对齐改进在高 IoU 阈值下收益尤其大。

### 6.3 类别不均衡:Dice loss 与加权交叉熵

逐像素**交叉熵(cross-entropy, CE)** 在极端不均衡下会被多数类主导:大量背景像素的梯度淹没了少数前景像素,模型倾向于"全判背景"。两个对策:

**(1) 加权交叉熵**:给每类一个权重 `w_c`(常用类频率倒数 `w_c ∝ 1/freq_c` 或其平方根),放大小类的损失:

```
L_wce = − Σ_pixels  w_{c*} · log P(c*)        c* 为该像素真值类
```

简单直接,但权重需手调,且对极端不均衡(前景 <1%)仍不够。

**(2) Dice loss**:直接优化"重叠率"这个评测目标本身。Dice 系数(= F1 的集合形式):

```
Dice = 2|P ∩ G| / (|P| + |G|)     P=预测前景概率图, G=真值掩码(0/1)

软化为可微损失(用概率 p∈[0,1] 代替硬集合):
L_dice = 1 − (2·Σᵢ pᵢ gᵢ + ε) / (Σᵢ pᵢ + Σᵢ gᵢ + ε)
```

**为什么 Dice 对不均衡鲁棒——关键推导。** 注意分子分母**只涉及前景**(gᵢ=1 的项),**完全不含背景项**(gᵢ=0 时 `pᵢ·gᵢ=0`,只在分母 `Σpᵢ` 里出现一点)。所以**背景像素再多,也不会线性地放大损失**——Dice 衡量的是前景的相对重叠比例,天然对前景/背景比例不敏感。这正是 CE 缺的东西:CE 是逐像素独立求和,背景越多总损失里背景占比越大;Dice 是一个**全局比值**,与背景数量近似无关。`ε`(如 1)是平滑项,防止某图无前景时分母为 0,并稳定梯度。

**实践常用 CE + Dice 的加权和**:CE 提供稳定的逐像素梯度(优化前期收敛快),Dice 直接对齐评测目标(对不均衡和小目标友好),互补。

```python
import torch
import torch.nn.functional as F

def dice_loss(logits, target, eps=1.0):
    """logits:[N,1,H,W] 前景 logit;target:[N,1,H,W] 0/1 掩码。"""
    p = torch.sigmoid(logits)                      # 概率化
    p = p.flatten(1)                               # [N, H*W]
    g = target.flatten(1).float()
    inter = (p * g).sum(dim=1)                     # 交(只前景贡献)
    union = p.sum(dim=1) + g.sum(dim=1)            # 模长和(背景仅微弱进入)
    dice = (2 * inter + eps) / (union + eps)
    return (1 - dice).mean()

def combo_loss(logits, target, w_dice=1.0, w_bce=1.0):
    bce = F.binary_cross_entropy_with_logits(logits, target.float())
    return w_bce * bce + w_dice * dice_loss(logits, target)   # 互补组合
```

**Dice loss 梯度的坑**:Dice 是非逐像素的比值,梯度对预测概率**高度非线性**,前期(预测全是 0.5)时梯度可能不稳定/偏小;纯 Dice 训练有时收敛困难。这就是为什么很少单用 Dice,几乎总配 BCE/CE 打底。Focal loss(对易分样本降权)是另一条对付不均衡的正交路线,常与上面组合。

## 7. 设计权衡与常见坑

把全章的权衡和易错点收成一张清单——这些是新手反复栽跟头的地方:

- **上采样三选一**:转置卷积(可学、可能棋盘)、双线性 resize-conv(无棋盘、推荐)、PixelShuffle(亚像素,常用于超分)。优先 resize-conv,只在确需可学放大且控好 `k%s==0` 时用转置卷积。
- **skip 用 add 还是 concat**:add(FCN,省显存,要求通道一致)vs concat(U-Net,表达力强,通道翻倍费显存)。小数据/要边界精度选 concat;算力紧选 add。
- **空洞卷积的 gridding**:连续层勿用相同 r,用锯齿状 r(1,2,3)避免采样网格化空洞。
- **output stride 的取舍**:DeepLab 推理时常把 output stride 从 16 降到 8 提精度,但**显存和 FLOPs 约 4 倍增长**(特征图面积翻 4 倍)。这是精度/成本的硬权衡。
- **RoIAlign 的 `align_corners` / 采样点数 / 坐标系**:训练推理必须一致;掩码分支输出 28×28 后双线性放大回框尺寸时,半像素对齐错误会系统性掉点,且极难定位。
- **掩码分辨率 vs 框**:Mask R-CNN 掩码只在 28×28 上预测再放大,**对极大物体的细节不足**(28×28 撑不起 800px 的物体边界)——这是该框架的固有上限,PointRend / 高分辨率掩码分支是补救方向。
- **类别不均衡别只换 loss**:loss(Dice/加权 CE/Focal)、采样(过采样小类)、增强(针对小类)是三条正交手段,严重不均衡时应组合。
- **评测对齐**:mIoU 要确认是否含 ignore label(很多数据集有 255=ignore,算 IoU 时必须屏蔽,否则数字虚高或虚低)。

- **FCN**:Long et al. 2015《Fully Convolutional Networks for Semantic Segmentation》——重点读 §3.3"upsampling is backwards convolution"(转置卷积本质)和 §4.2 skip 融合(32s/16s/8s 消融)。代码看 `torchvision.models.segmentation.fcn_resnet50`,关注 `FCNHead`(就是 1×1 conv + dropout + 1×1 到 num_classes)。
- **U-Net**:Ronneberger et al. 2015。读 Fig.1(U 形结构)和 §3 的弹性形变增强。实现可看 `MONAI` 库的 `UNet`(医学专用,接口清晰)或 diffusers 的 `UNet2DModel`(看 skip concat 怎么在 `up_blocks` 里取 `down_blocks` 的输出)。
- **DeepLab**:Chen et al.《DeepLabv3》(2017,arXiv,空洞卷积 + ASPP 集大成)。重点读 §3.3 ASPP 和那条 image pooling 分支的动机。代码:`torchvision.models.segmentation.deeplabv3_resnet50`,核心看 `ASPP` 模块和 `ASPPConv`(带 dilation 的 3×3)。空洞卷积如何改 ResNet 的 stride,看 torchvision `resnet.py` 里 `replace_stride_with_dilation` 参数。
- **Mask R-CNN**:He et al. 2017。**必读 §3 RoIAlign**(对齐误差的核心论证)和掩码分支解耦的消融(Table 2)。实现首选 **detectron2** 的 `roi_heads/mask_head.py` 和 `poolers.py`(`ROIAlign` 的封装);PyTorch 算子在 `torchvision.ops.roi_align` / `RoIAlign`。
- **全景**:Kirillov et al. 2019《Panoptic Segmentation》(定义 + PQ 指标)和《Panoptic FPN》;进阶看 Cheng et al.《Mask2Former》(统一范式,detectron2 的 `MaskFormer` 项目)。
- **数值玩具**:动手在 `torchvision.ops.roi_align` 上喂一个浮点框,对比手算双线性结果;再用 `nn.ConvTranspose2d(1,1,4,2,1)` 配全 1 权重看输出的棋盘格,验证 §2.3 的覆盖次数推导。

> 具体 benchmark 数值(各模型在 PASCAL VOC / Cityscapes / COCO 上的 mIoU、AP 分数,以及 ASPP 默认膨胀率组合等超参)随论文版本和实现差异较大,本章刻意不背具体数字,以原论文表格为准【待核】。

## 10. 小结与承上启下

这一章把"像素级理解"的三条技术主线讲透了:

1. **稠密预测的核心矛盾是分辨率 vs 语义**,FCN(skip 加法)、U-Net(对称 concat)、DeepLab(空洞卷积保分辨率 + ASPP 多尺度)是三种代表性解法,本质都在"重新组合深层语义与浅层细节"。
2. **实例分割靠"检测 + 框内二值分割"**,而 **RoIAlign 用双线性插值消除 RoI Pooling 的量化误差**——我们精确算出了 stride=16 时一次量化≈8 像素的偏移,这正是它对 mask AP 大幅提升的代数解释。
3. **评测和不均衡**:mIoU/AP^mask 为什么优于 pixel accuracy,Dice loss 为什么对前景/背景比例不敏感,都给了推导。

**在整门课里的位置**:前几章你打通了卷积、感受野、backbone(分类)和检测的基础;这一章把输出从"框"推进到"像素"。**承上**——RoIAlign、FPN、两阶段范式都直接复用自检测章;**启下**——本章末尾提到的 MaskFormer/Mask2Former 把分割统一成"集合预测",依赖的 query + 二分匹配机制,正是下一章 **Vision Transformer 与 DETR 系列**要从第一性原理推开的内容。当稠密预测遇上 Transformer,分类/检测/分割的架构边界会被进一步抹平——那是视觉走向通用骨干的关键一跳。
