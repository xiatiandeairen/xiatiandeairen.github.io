> 这一章把你从"知道模型有个固定上下文窗口"带到"能从第一性原理推出为什么直接外推会崩、能亲手推导 Position Interpolation / NTK-aware / YaRN 三种方法都在如何改 RoPE 的频率与尺度、能定量解释 lost-in-the-middle 的成因、并能讲清评测长上下文为什么'大海捞针刷满分'≠'真能用'"——直至你能照着伪代码自己改一份 RoPE 的频率表,把一个 4K 训练的模型外推到 32K 而不崩。

## 一、动机:窗口不是越大越好,而是"撑大了还能用"才难

先把问题摆正。**长上下文不是一个工程参数,而是一连串相互纠缠的难题。** 一个朴素的想法是:既然 Transformer 的注意力理论上能看任意长的序列(没有 RNN 那种隐状态瓶颈),那把窗口设大不就行了?把 `max_position` 从 2048 改成 1048576,重新 forward 一遍?

不行,而且会以三种不同的方式失败,这三种失败正好对应本章的三条主线:

- **外推失败(能不能跑出合理输出):** 模型在长度 L_train(比如 4K)上训练,推理时喂 32K。无论是绝对位置编码还是 RoPE,模型从没见过那些位置对应的编码值/旋转角,送进去得到的是**训练分布之外的输入**,输出直接退化成乱码。这是位置编码的外推问题,第三、四、五节解决。
- **利用失败(跑出来了但用不好):** 就算让模型"读完"了 100K token,它对**中段**信息的利用显著差于头尾——这就是 lost-in-the-middle。长 ≠ 用得好。第六节剖析成因。
- **成本失败(算得起吗):** 注意力是 `O(N²)`,KV cache 显存是 `O(N)`。N 从 2K 到 1M(约 512 倍),平方项涨了约 26 万倍。这一块在第九章(高效注意力)和 MLSys 课(FlashAttention、KV cache / PagedAttention)里专门处理,本章第七节做量化收口,把账算清楚。

**这一章的核心抓手是位置编码的外推。** 因为它是长上下文里最"数学"、最能从第一性原理讲透、也最能用几十行代码改出来的部分——而且现代几乎所有长上下文模型(Llama、Qwen、Yi、Mistral 的长版本)的扩窗,本质上都是在**改 RoPE 的频率**。把这件事讲透,你就握住了长上下文的钥匙。

预备知识:本章重度依赖第四章(位置编码,从正弦到 RoPE)。下面会快速重建 RoPE 的关键事实,但如果你对"RoPE 为什么编码的是相对位置"还没把握,建议先回去把第四章的旋转推导过一遍。

## 二、重建地基:RoPE 到底在编码什么

把 RoPE(Rotary Position Embedding,旋转位置编码,Su et al. 2021,RoFormer)的关键结论重新推一遍。我们只需要三件事:它怎么作用、它编码的是什么、它的"频率"是什么。这三件事是后面所有外推方法的施力点。

### 2.1 机制:把 query/key 按位置旋转

设 query 向量 q、key 向量 k 都是 d 维(d 为 head 维度,偶数)。RoPE 把它们**两两配对**成 d/2 对,第 m 对(m = 0, 1, …, d/2−1)是分量 `(q[2m], q[2m+1])`,把它当成复平面上一个二维向量。在位置 p 处,RoPE 把第 m 对旋转一个角度 `p · θ_m`:

```
旋转角度       φ = p · θ_m
旋转后的第 m 对:
  [ q'[2m]   ]   [ cos(pθ_m)  -sin(pθ_m) ] [ q[2m]   ]
  [ q'[2m+1] ] = [ sin(pθ_m)   cos(pθ_m) ] [ q[2m+1] ]
```

不同的对用**不同的旋转频率** θ_m,这是 RoPE 的灵魂。频率定义为:

```
θ_m = base^(-2m/d)        base = 10000(沿用正弦编码的默认底数)
```

m=0 时 θ₀ = 1(转得最快,高频);m 越大 θ_m 越小,最大那对 θ ≈ base^(-1) = 1/10000(转得最慢,低频)。

### 2.2 关键性质:内积只依赖相对位置 (i − j)

这是 RoPE 唯一真正重要的性质,务必能自己证。用复数记法最干净:把第 m 对 `(q[2m], q[2m+1])` 写成复数 `qₘ = q[2m] + i·q[2m+1]`。位置 p 处的旋转就是乘以 `e^(i p θ_m)`:

```
位置 i 的 query 第 m 对:  qₘ · e^(i·i·θ_m)        (前一个 i 是虚数单位,后一个 i 是位置)
位置 j 的 key   第 m 对:  kₘ · e^(i·j·θ_m)

它们的内积(取实部)= Re[ (qₘ e^(i·i·θ_m)) · conj(kₘ e^(i·j·θ_m)) ]
                   = Re[ qₘ · conj(kₘ) · e^(i·(i−j)·θ_m) ]
```

最后一行里,**位置 i、j 只以差值 (i − j) 出现**。也就是说,RoPE 下两个 token 的注意力打分,只取决于它们的**相对距离**,与它们的绝对位置无关。这是 RoPE 比绝对位置编码优越的核心理由,也是后面理解外推的支点。

### 2.3 频率即"波长":高频管局部,低频管全局

把 θ_m 翻译成更有物理直觉的量——**波长 λ_m**,即旋转一整圈(2π)需要走过多少个位置:

```
λ_m = 2π / θ_m = 2π · base^(2m/d)
```

- m=0(最高频):λ₀ = 2π ≈ 6.28 个 token 转一圈。**它分辨的是相邻几个 token 的精细顺序**(局部分辨率)。
- m = d/2−1(最低频):λ = 2π · base^((d−2)/d) ≈ 54410 个 token 才转一圈(若把指数粗略当成 1,则 ≈ 2π·base ≈ 62832,量级一致)。**它编码的是大尺度的位置信息**,在几千 token 的范围内角度变化很小、近乎单调。

**这张"频率谱"是理解一切外推方法的地图:高频维度负责区分近处 token 谁先谁后,低频维度负责区分远处 token 大致在哪。** 记住这句话,下面三节本质上都是在回答一个问题:**外推时,这条频率谱该怎么改,才能既不让模型懵、又保住该保的分辨率?**

## 三、为什么直接外推会崩:训练分布之外的旋转角

### 3.1 问题:模型只见过 [0, L_train) 范围的角度

模型在长度 L_train 上训练时,对第 m 对,它见过的旋转角范围是:

```
φ_m ∈ [0, L_train · θ_m)
```

对**低频维度**(θ_m 很小),L_train · θ_m 这个上界往往**还不到 2π,甚至远不到**。举例:最低频 θ ≈ 1/10000,L_train = 4096,那么训练中见过的最大角度只有 `4096/10000 ≈ 0.41 弧度`——连四分之一圈都没转到。模型对这一维度的认知,完全局限在 `[0, 0.41]` 这一小段弧上。

现在推理时把序列拉到 32K。位置 30000 在这一维上的角度变成 `30000/10000 = 3.0 弧度`。**这是模型在训练里从未见过的角度区间** `(0.41, 3.0]`。注意力打分函数(那些 Wq、Wk 权重)是针对小角度区间拟合出来的,送进去一个超出区间 7 倍的角度,等于让一个只在 0–10℃ 标定过的温度计去测 70℃——读数完全不可信。**输出 perplexity(困惑度,衡量预测下一个词不确定性的指标)会爆炸式上升,生成退化成乱码。** 这就是直接外推崩溃的根因。

### 3.2 为什么不是所有维度都一样糟:高频反而"见过整数圈"

值得注意的细节:**高频维度其实见过完整的周期。** 最高频 θ₀=1,在 L_train=4096 里转了 `4096/(2π) ≈ 651` 圈,各种角度都见过。所以高频维度本身外推时角度并不"陌生"——陌生的是低频维度。这个非均匀性,正是 NTK-aware(第五节)能比 PI(第四节)做得更好的物理基础:**不该对所有频率一视同仁地缩放。** 先记下这个伏笔。

### 3.3 一个常被忽略的二阶问题:注意力熵随长度漂移

即便位置角度的问题解决了,还有一个量级问题。softmax 注意力对 N 个 token 打分,N 越大,概率被摊到越多 token 上,**注意力分布的熵自然变大、每个 token 分到的权重变小**(极端情况下趋近均匀分布,信息坍塌)。训练在 N=4K 时 softmax 的"温度感"是针对那个长度调好的;推到 64K,同样的 logit 尺度下分布会过度平坦。这是 YaRN(第六节)要额外引入"温度修正"的原因。先埋下,后面接。

## 四、Position Interpolation:把外推变成内插

### 4.1 核心思路:不外推位置,而是压缩位置

Position Interpolation(PI,位置插值,Chen et al. 2023,Meta)的想法极其简洁,**一句话:既然模型只认识 [0, L_train) 的位置,那就把要处理的长序列的位置坐标线性压回这个范围。**

设目标长度 L_target = 32768,训练长度 L_train = 4096,缩放因子:

```
s = L_target / L_train = 32768 / 4096 = 8
```

PI 把位置 p 替换成 p/s 再送进 RoPE:

```
原本:   RoPE 在位置 p 旋转 p · θ_m
PI 后:  RoPE 在位置 p 旋转 (p/s) · θ_m
```

现在看位置 30000:旋转角变成 `(30000/8) · θ_m = 3750 · θ_m`。而 3750 < 4096 = L_train,**所有位置的旋转角都落回了训练见过的 [0, L_train·θ_m) 区间内**。外推问题被转化成了内插问题——模型不需要面对陌生角度,只需要在见过的角度之间做插值,而神经网络对"见过的区间内插值"是擅长的。

### 4.2 等价视角:PI 就是把所有频率同比缩小

把 `(p/s)·θ_m` 重新组合一下:

```
(p/s) · θ_m = p · (θ_m / s)
```

**这等价于保持位置不变、把每一维的频率 θ_m 同比例除以 s。** 这就是开篇那句话"所有外推方法都是在改 RoPE 频率"的第一个实例。PI 的改法是:**全谱乘以 1/s,一视同仁。**

代码上 PI 只改一行——构造频率表时把 base 之外乘个尺度,或直接缩位置:

```python
def rope_freqs_pi(dim, base=10000.0, scale=1.0):
    # dim = head_dim;返回 dim/2 个旋转频率 θ_m
    m = torch.arange(0, dim, 2).float() / dim          # [0/d, 2/d, ..., (d-2)/d]
    theta = base ** (-m)                                # θ_m = base^(-2m/d),标准 RoPE
    return theta / scale                                # PI:整谱同比缩小 s 倍 -> 频率 / s

def apply_rope(x, theta, positions):
    # x: [..., seq, dim];positions: [seq];theta: [dim/2]
    ang = positions[:, None] * theta[None, :]           # [seq, dim/2],每个位置×每个频率
    cos, sin = ang.cos(), ang.sin()
    x1, x2 = x[..., 0::2], x[..., 1::2]                 # 偶/奇分量配成对
    # 二维旋转:每对 (x1, x2) 转 ang
    return torch.stack([x1*cos - x2*sin,
                        x1*sin + x2*cos], dim=-1).flatten(-2)
```

注意 `rope_freqs_pi(dim, scale=8.0)` 就实现了 8 倍 PI。**整个 PI 的工程改动就是这一个 `/ scale`。**

### 4.3 代价:高频局部分辨率被牺牲

PI 不是免费的。把全谱频率都除以 s,意味着**高频维度的波长也被拉长了 s 倍**:λ₀ 从 6.28 个 token 变成 `6.28 × 8 ≈ 50` 个 token 才转一圈。后果是**相邻 token 的旋转角差从 θ₀ 缩小到 θ₀/s**——原本相邻两个 token 在最高频维上差 1 弧度,现在只差 0.125 弧度。**模型区分"谁在前谁在后"的局部分辨率被压扁了 8 倍。**

这正是 PI 论文里强调"PI 需要少量微调(论文用约 1000 步)才能恢复性能"的原因:模型要重新适应被压缩的高频分辨率。**PI 的本质权衡:用牺牲局部精细分辨率,换取低频维度不外推到陌生区间。** 它简单、稳、但"一刀切"地伤害了不该伤的高频。下一节就是来修这个问题的。

### 4.4 量化:外推前后的角度范围

把账算清。设 head 维 d=128,base=10000,L_train=4096:

```
最高频 m=0:   θ₀ = 1
  原始最大角:4096 × 1 = 4096 rad(转了约 651 圈,角度饱和、见过所有相位)
  PI(s=8)后:位置 32768 的最大角 = 32768/8 × 1 = 4096 rad —— 与训练一致 ✓

最低频 m=63:  θ = 10000^(-126/128) ≈ 1.16e-4
  原始最大角:4096 × 1.16e-4 ≈ 0.475 rad(不到 1/12 圈)
  PI(s=8)后:32768/8 × 1.16e-4 ≈ 0.475 rad —— 与训练一致 ✓
```

**两端都精确落回训练区间,这就是 PI 能 work 的数学保证。** 但代价是高频维度的"有效信息密度"被稀释了 s 倍——它转的圈数没变(还是 651 圈),却要覆盖 8 倍长的序列。

## 五、NTK-aware 缩放:高频少动、低频多动

### 5.1 问题:PI 错在哪

回看第 3.2 节的伏笔:**高频维度本来不需要缩放**(它在训练里见过完整周期,外推到更大位置只是又转了几圈,角度并不陌生);**真正需要缩放的是低频维度**(它在训练里连一圈都没转完,外推会冲进陌生区间)。

PI 对所有频率同比除以 s,等于**对本来好好的高频维度也下了重手**,白白牺牲了局部分辨率。NTK-aware 的洞察就是:**按频率分段处理——高频几乎不动,低频充分缩放。**

(名字里的 NTK 来自 Neural Tangent Kernel 理论的一个直觉:神经网络难以学习高频信号。bloc97 在 Reddit/社区里提出该方法时借用了这个直觉,故称 NTK-aware。这是社区起源、后被广泛采用的方法,严格的理论归属【待核】,但下面的数学是确定且可验算的。)

### 5.2 机制:不改频率公式,改底数 base

NTK-aware 不直接缩放每个 θ_m,而是**改 RoPE 频率公式里的底数 base**:把 base 放大成 base'。回看频率公式 `θ_m = base^(-2m/d)`:

```
m=0:      θ₀ = base^0 = 1            —— 与 base 无关!改 base 完全不影响最高频
m=d/2−1:  θ ≈ base^(-1) = 1/base    —— 与 base 成反比,改 base 直接缩放最低频
```

**这是关键的数学巧合:改 base 对 θ_m 的影响随 m 单调变化——m 越小(越高频)影响越小,m 越大(越低频)影响越大。** 正好契合"高频少动、低频多动"的需求。NTK-aware 选一个新底数 base',使得**最低频维度的缩放恰好达到 s 倍**(等价于 PI 对最低频的处理),而高频几乎不动。

求 base':要让最低频维度 `m = d/2 − 1`(此处 `2m/d ≈ (d−2)/d`)的有效缩放为 s,需要:

```
原最低频  θ_low      = base^(-(d-2)/d)
要求新最低频 θ'_low  = θ_low / s

θ'_low = base'^(-(d-2)/d) = base^(-(d-2)/d) / s
=> base'^(-(d-2)/d) = base^(-(d-2)/d) · s^(-1)
两边取 (-d/(d-2)) 次方:
=> base' = base · s^(d/(d-2))
```

**结论(可背):NTK-aware 把底数从 base 改成 `base · s^(d/(d-2))`。** 对 d=128、s=8:`base' = 10000 × 8^(128/126) ≈ 10000 × 8^1.016 ≈ 10000 × 8.27 ≈ 82700`。注意指数 d/(d-2) ≈ 1.016 略大于 1,是为了精确补偿最低频那一维不是恰好 `2m/d=1`(而是 (d-2)/d)的边界效应。很多实现直接近似成 `base · s`,差别不大。

### 5.3 代码:一行改 base

```python
def rope_freqs_ntk(dim, base=10000.0, scale=1.0):
    # NTK-aware:不缩频率,改底数。高频(小 m)几乎不变,低频(大 m)被压到 ~1/scale
    base_prime = base * (scale ** (dim / (dim - 2)))    # base · s^(d/(d-2))
    m = torch.arange(0, dim, 2).float() / dim
    return base_prime ** (-m)                           # 用新底数算频率
```

对比 PI:PI 是 `theta / scale`(全谱平移),NTK 是 `换底数`(谱形被重新拉伸)。**画出 log θ_m vs m 的曲线就一目了然:PI 把整条直线向下平移,NTK 把直线以"高频端为锚点"顺时针压低斜率——高频端几乎不动,低频端被压下去。**

### 5.4 分析:为什么 NTK 常能"免微调"外推

PI 伤害了高频分辨率,所以需要微调恢复。NTK 几乎不动高频,**局部分辨率基本保住**,因此在实践中 NTK-aware 常能在**完全不微调**的情况下,把模型外推到 2–4 倍而 perplexity 不明显恶化(这是它在社区一夜走红的原因)。

代价在另一头:NTK 对最低频的缩放不像 PI 那样"刚好压回 L_train",在极长外推(8× 以上)时,**中间频段的处理不够干净**——有些中频维度既没被充分缩放(可能小幅超出训练区间),又损失了一点分辨率,出现"两头不讨好"的中频。这正是 YaRN 要进一步精修的地方。

## 六、YaRN:逐维分段 + 温度修正,把外推做到极致

### 6.1 YaRN 的两个独立改进

YaRN(Yet another RoPE extensioN,Peng et al. 2023)= **NTK-by-parts(逐维分段插值)+ attention 温度修正**。它把前面所有方法的优点合并,并补上第 3.3 节埋的熵漂移问题。两个改进是正交的,分开讲。

### 6.2 改进一:NTK-by-parts——按波长分三段处理

YaRN 不用单一缩放,而是**逐维度判断该不该插值**,判据是该维度的**波长 λ_m 相对训练长度 L_train 的比例** `r_m = L_train / λ_m`(这个维度在训练序列里转了几圈):

```
r_m = L_train / λ_m = L_train · θ_m / (2π)
```

- **高频维度(r_m 大,转了很多圈,如 r > 32):波长远短于训练长度,模型见过完整周期,不该动它**——保持原频率 θ_m(尺度因子 = 1,纯外推但安全,因为它见过所有相位)。
- **低频维度(r_m 小,连一圈都没转完,如 r < 1):必须插值**——按 PI 方式缩放(频率 ÷ s)。
- **中频维度(1 ≤ r ≤ 32):线性渐变**——用一个随 r 平滑从"插值"过渡到"不插值"的 ramp 系数 γ(r) ∈ [0,1] 混合两种处理。

```
λ_m 短(高频)  ───►  不插值,θ_m 不变       (它见过完整周期)
λ_m 中         ───►  γ(r) 平滑混合
λ_m 长(低频)  ───►  插值,θ_m / s          (它从没转完一圈,外推会崩)
```

**这就是把第 3.2 节"高频见过整数圈、低频没见过"的观察,精确量化成了逐维度的处理策略。** PI 一刀切、NTK 用 base 隐式分段,YaRN 则**显式按波长分三段**,是三者里对频率谱最精细的控制。

### 6.3 改进二:温度修正——补偿注意力熵漂移

回到第 3.3 节:序列变长,softmax 注意力分布变平。YaRN 的修正是**给注意力 logits 乘一个常数温度因子**:

```
attention = softmax( (QKᵀ/√d) · 1/t )      等价于把 logits 放大 1/t 倍
```

其中 1/t > 1(放大 logits → 让 softmax 更"尖" → 抵消因长度增加而过度平坦的分布)。YaRN 论文给出一个经验拟合:`√(1/t) ≈ 0.1·ln(s) + 1`(s 为缩放因子;此为论文给出的经验公式形式,具体常数【待核,以论文为准】)。

**这个修正的妙处在于它"免费":** 1/t 是个常数,可以**直接吸收进 RoPE 的 cos/sin 表里**(把 cos、sin 都乘以 √(1/t)),推理时零额外开销、不改任何 kernel。这是 YaRN 在工程上干净利落的关键。

### 6.4 代码:YaRN 频率表(分段 + 温度)

```python
def rope_freqs_yarn(dim, base, scale, L_train,
                    lo_rounds=1.0, hi_rounds=32.0):
    # 1) 标准频率与每维"转过的圈数" r_m
    m     = torch.arange(0, dim, 2).float() / dim
    theta = base ** (-m)                               # 原始 θ_m
    wavelen = 2 * math.pi / theta                      # λ_m
    r = L_train / wavelen                              # r_m:训练序列里转过几圈

    # 2) ramp 系数 γ ∈ [0,1]:r<lo -> 1(纯插值);r>hi -> 0(不插值);中间线性
    gamma = ((r - lo_rounds) / (hi_rounds - lo_rounds)).clamp(0, 1)
    gamma = 1 - gamma                                  # 翻转:低频(r 小)γ→1

    # 3) 逐维混合:插值频率 θ/scale 与原频率 θ
    theta_interp = theta / scale
    theta_yarn   = gamma * theta_interp + (1 - gamma) * theta

    # 4) 温度修正:吸收进 cos/sin 幅度(此处返回因子,apply 时乘上)
    mscale = 0.1 * math.log(scale) + 1.0               # √(1/t),经验式【待核】
    return theta_yarn, mscale

# apply 时:cos, sin = (ang.cos()*mscale, ang.sin()*mscale)
```

### 6.5 三方法对比小结

```
方法          频率谱改法              高频分辨率  低频外推安全  熵修正  需微调
─────────────────────────────────────────────────────────────────────
直接外推      不改                   保留        崩(陌生角)  无      —(崩)
PI            全谱 θ/s               全被牺牲 ✗  安全 ✓       无      需要
NTK-aware     改底数 base·s^(d/d-2)  几乎保留 ✓  较安全       无      常免
YaRN          逐维分段 + θ/s 混合    保留 ✓      安全 ✓       有 ✓    少量更佳
```

**一句话收束第三到六节:外推的全部艺术,就是在 RoPE 的频率谱上,决定每一维度该被缩放多少——高频维度因为见过完整周期所以可以不动甚至外推,低频维度因为没转完一圈所以必须插值压回训练区间。PI / NTK / YaRN 是这个谱系上从粗到精的三档。**

## 七、利用难点:Lost in the Middle 与长上下文的成本墙

外推解决了"能不能跑"。这一节面对两个"跑出来了但仍然棘手"的问题。

### 7.1 Lost in the Middle:长 ≠ 用得好

Liu et al. 2023("Lost in the Middle")的核心实测发现:**把关键信息放在长上下文的不同位置,模型的检索/利用准确率呈 U 形曲线——开头和结尾高,中间显著塌陷。** 一段几十个文档的上下文,答案藏在第 1 个或最后 1 个文档时模型答得好,藏在正中间时准确率可能掉十几到几十个百分点(具体数值随模型/任务,【以原论文 figure 为准】)。

为什么会这样?有几个相互叠加的成因,逐个剖析:

**成因一:注意力被稀释(softmax 的容量有限)。** 第 3.3 节算过:N 个 token 抢一个 softmax 的概率质量,N 越大,单个中段 token 能分到的注意力权重越小。而开头 token 往往因为是"上下文的锚"被反复关注,结尾 token 因为离当前生成位置近(相对距离小、RoPE 打分天然偏高)也被强关注。**夹在中间、既不近也无特殊地位的 token,在注意力的零和竞争里被边缘化。**

**成因二:位置偏置(训练数据的结构性偏好)。** 自然语料里,**重要信息常出现在段落/文档的开头和结尾**(主题句、总结句)。模型在预训练中学到了这个先验:开头结尾更值得关注。这个偏置在长上下文里被放大成对中段的系统性忽视。

**成因三:因果注意力的累积衰减 + 外推副作用。** 因果掩码下,靠后的 token 能注意到更多历史(注意力对象多),靠前的 token 被注意到的"次数"也多(后面每个 token 都可能回看它)。中段 token 两头不沾。再叠加外推方法(尤其 PI)对中远距离分辨率的损伤,中段的相对位置信号本就更模糊。

**这三条共同决定:lost-in-the-middle 不是某个 bug,而是 softmax 注意力 + 数据先验 + 因果结构的结构性产物。** 它没有像外推那样干净的数学解——缓解手段(重排上下文把关键信息放两头、用检索缩短有效上下文、训练时显式做中段监督)都是部分缓解,不是根治。**这是你向别人解释"为什么不能无脑塞 100K 上下文"时最该讲清的一点。**

### 7.2 成本墙:把账算清(呼应 MLSys 课)

长上下文的第三种失败是成本。两笔账,都得会算:

**账一:注意力计算 `O(N²)`。** 注意力矩阵是 N×N,每个元素一次点积 `O(d)`,总计 `O(N²·d)`。N 从 2K 到 1M(约 512 倍),平方项涨 **约 26 万倍**。这就是为什么长上下文必须配合稀疏/线性注意力(第九章)或 FlashAttention 这类 IO 优化(MLSys 课第九章把 `O(N²)` 访存降到分块流式)——否则 1M 上下文一次 forward 的注意力计算量在工程上不可承受。

**账二:KV cache 显存 `O(N)`。** 沿用 MLSys 课第十章推的公式,每 token 的 KV 字节:

```
b_tok = 2 · L · d_model · P
```

对一个 L=40、d_model=5120、BF16(P=2)的模型,`b_tok ≈ 800 KB/token`(数字示意,真实模型以其 config 为准【待核】)。那么:

```
2K  上下文:800KB × 2048    ≈ 1.6 GB
32K 上下文:800KB × 32768   ≈ 25 GB    (单卡已吃紧)
1M  上下文:800KB × 1048576 ≈ 800 GB   (远超单卡,必须跨卡/分页/量化)
```

**显存随 N 线性涨,但绝对量级在百万 token 时已是 TB 级。** 这就是为什么百万级上下文不只是"改个 RoPE 频率"就行——它强依赖 GQA/MQA(砍 KV 头)、KV 量化、PagedAttention(MLSys 课第十章)、乃至跨卡 KV 切分。**外推解决"位置对不对",MLSys 解决"算得起、放得下"。两者缺一不可,这也是本章把成本账显式接到 MLSys 课的原因。**

## 八、设计权衡与常见坑

- **坑一:把 PI 的 scale 当成"越大越好"。** scale 越大,高频分辨率被压得越狠,短文本性能掉得越多。**外推不是免费午餐:扩窗 8 倍,几乎一定以牺牲一部分原长度能力为代价**(YaRN 的分段就是为了把这个代价压到最小)。生产中常用"动态 NTK":序列短时 scale=1(不损伤),只在序列长度超过 L_train 时才按需调大 scale。
- **坑二:改了频率却忘了 cos/sin 表是预计算缓存的。** 多数实现(nanoGPT、HF)把 RoPE 的 cos/sin 按 `max_position` **预计算并缓存**。你改了频率公式或 scale,**必须重建这张缓存表**,否则改了个寂寞。HF 里对应 `rotary_emb` 模块的 buffer。
- **坑三:训练-推理长度不匹配未被正视。** 如果你打算让模型在 32K 上工作,**最好至少在长样本上做少量继续训练(continual pretrain)或微调**。纯靠 zero-shot 外推(尤其 PI)通常达不到原生长上下文模型的质量。但长文档数据本身稀缺、且 `O(N²)` 让长样本训练极贵——这是长上下文训练的真实困境:**你想要的长样本最少,训练它们又最贵。**
- **坑四:用"大海捞针"满分就宣布长上下文能用。** 见下节练习。检索一个孤立事实(needle)和在长上下文里做多跳推理是两种能力,前者满分不代表后者可用。
- **坑五:外推后不重新校准 RoPE 之外的东西。** 有些模型有 attention sinks(前几个 token 当"注意力垃圾桶")、ALiBi 等其他位置机制;长上下文改造时要确认你改的是不是模型实际用的位置机制——别对一个用 ALiBi 的模型套 RoPE 插值。
- **权衡总览:** PI = 最简单、最稳、伤高频、需微调;NTK-aware = 免微调、保高频、极长时中频不干净;YaRN = 最优但实现最复杂、需调 lo/hi 阈值和温度。**默认推荐:中等外推(2–4×)用动态 NTK,大幅外推(8×+)且能微调用 YaRN。**

## 九、动手练习

**练习 1(推导题):证明 PI 把外推变内插。** 设 L_train、目标长度 L_target、缩放 s = L_target/L_train。写出 PI 处理后,位置 p ∈ [0, L_target) 经过 RoPE 旋转的角度表达式,证明对每个维度 m,该角度都落在训练见过的区间 [0, L_train·θ_m) 内。
> 提示:PI 把旋转角从 `p·θ_m` 改成 `(p/s)·θ_m`。代入 p 的最大值 L_target−1,证明 `(L_target/s)·θ_m = L_train·θ_m`,即上界恰好等于训练上界。再说明这就是"内插"(在见过的角度区间内取值)而非"外推"。

**练习 2(推导题):推导 NTK-aware 的 base'。** 要求改底数后最低频维度(取 `2m/d = (d−2)/d`)的有效缩放恰为 s,推出 base' = base·s^(d/(d−2))。再令 d→∞,说明它退化成什么。
> 提示:令 `base'^(-(d-2)/d) = base^(-(d-2)/d)/s`,两边取 `-d/(d-2)` 次方解出 base'。d→∞ 时指数 d/(d−2)→1,故 base' → base·s——即"近似把 base 直接乘 s"那个常见简化的来历。

**练习 3(编码题):亲手把 4K 模型外推到 16K 并验证不崩。** 取一个 RoPE 小模型(nanoGPT 风格,或 HF 上任意 RoPE 模型),分别用直接外推、PI(s=4)、NTK(s=4)三种方式重建频率表,在一段 12K token 文本上测 perplexity。
> 提示:复用本章 `rope_freqs_pi` / `rope_freqs_ntk`,**记得重建 cos/sin 缓存**(坑二)。你应观察到:直接外推 ppl 爆炸(几十上百),PI/NTK 的 ppl 接近原长度水平,且 NTK 通常略优、对短文本几乎无损。把三条 ppl-vs-位置曲线画出来。

**练习 4(分析题):设计一个能区分"检索"与"推理"的长上下文评测。** 说明为什么"大海捞针"(Needle in a Haystack:在长无关文本里插一句事实,问模型这句话)即使满分也不能证明模型"会用"长上下文;设计一个需要**跨多个分散位置聚合信息才能答对**的任务。
> 提示:大海捞针只考"单点检索",且 needle 通常和背景文本风格迥异(易被注意力高亮)。真正的难题是 multi-hop:答案依赖第 3K、第 40K、第 90K token 处三条线索的组合(如"A 在第 3K 段提到,B 在第 40K 段提到,问 A 和 B 的关系")。再结合 lost-in-the-middle 设计:把关键线索故意放在中段,看准确率是否塌陷。思考:你的指标该报"平均准确率"还是"按位置分桶的准确率"?(后者才能暴露 U 形曲线。)

## 十、源码 / 论文导读

- **RoPE 原始:** Su et al. 2021《RoFormer: Enhanced Transformer with Rotary Position Embedding》。重点读第 3 节旋转矩阵构造和"相对位置"性质的证明(对应本章第二节)。
- **Position Interpolation:** Chen et al. 2023《Extending Context Window of Large Language Models via Positional Interpolation》(Meta)。重点读它对"为什么直接外推会产生灾难性 attention score"的分析,以及只需约 1000 步微调的实验。
- **NTK-aware:** 起源于社区(bloc97 在 LocalLLaMA 社区的帖子),没有正式 paper 形态的原始出处【待核确切引用】;其数学被 YaRN 论文规范化收录,可在 YaRN 论文里读到 NTK-aware 与 "NTK-by-parts" 的统一表述。
- **YaRN:** Peng et al. 2023《YaRN: Efficient Context Window Extension of Large Language Models》。重点读第 3 节:NTK-by-parts 的 ramp 函数定义、attention 温度因子的推导与"吸收进 cos/sin"的工程技巧(对应本章第六节)。
- **Lost in the Middle:** Liu et al. 2023《Lost in the Middle: How Language Models Use Long Contexts》。看那张著名的 U 形准确率曲线和位置消融实验(对应第七节)。
- **开源实现:**
  - HuggingFace transformers:`modeling_rope_utils.py`(在它里面集中实现了 `_compute_default_rope_parameters` / `_compute_linear_scaling_rope_parameters`(PI)/ `_compute_dynamic_ntk_parameters` / `_compute_yarn_parameters`——一份文件读完四种方法,强烈推荐对照本章源码)。模型 config 里 `rope_scaling = {"type": "yarn"/"dynamic"/"linear", "factor": s}` 就是切换它们。
  - nanoGPT:RoPE 实现最精简,适合先看清"频率表 + 旋转"的骨架,再去看 HF 的扩展。
  - x-transformers:`RotaryEmbedding` 支持多种插值开关,适合做消融实验脚手架。
- **大海捞针评测:** Needle in a Haystack(gkamradt 的开源评测脚本)是事实上的社区基准;RULER(NVIDIA 2024)是更严肃的、含多跳与聚合任务的长上下文基准,呼应本章练习 4 的诉求【RULER 具体任务集以其 repo 为准】。

## 十一、小结与承上启下

这一章把"长上下文"拆成了三个本质不同的问题,并把最数学的那个讲透了:

1. **能不能跑(外推):** 直接外推崩在"低频维度遇到训练分布外的旋转角"。PI / NTK-aware / YaRN 是同一思想——**改 RoPE 频率谱**——的三档精度。核心口诀:**高频见过整数圈可不动,低频没转完一圈必须插值压回。**
2. **用不用得好(lost-in-the-middle):** softmax 容量稀释 + 数据位置先验 + 因果结构,共同造成中段利用塌陷。没有干净解,只有缓解。
3. **算不算得起(成本):** 注意力 `O(N²)`、KV cache `O(N)`,百万 token 时是 TB 级显存,强依赖第九章的高效注意力和 MLSys 课的 KV 管理。

在整门课里的位置:本章站在**第四章(位置编码)的肩膀上**——没有 RoPE 的相对位置性质和频率直觉,本章无从谈起;它**向第九章(高效注意力)交棒**——外推让序列"可以"很长,但要让它"跑得动"得靠打破 `O(n²)`;它**与 MLSys 课(FlashAttention、KV cache / PagedAttention)咬合**——算法侧解决"位置对不对",系统侧解决"放不放得下"。读完这三块,你才算真正握住了"百万 token"这件事的全貌。

下一章(第十一章,PEFT)我们转向另一条主线:**在不动全部权重的前提下适配大模型**——而长上下文外推后的少量微调,正是 PEFT 最自然的用武之地之一。
