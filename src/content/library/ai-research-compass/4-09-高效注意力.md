> 这一章把你从"知道注意力是 `softmax(QKᵀ/√d)V`、隐约觉得它在长序列上很贵"带到"能精确算出那个二次墙到底卡在哪、能从第一性原理推导稀疏 / 线性 / SSM 三条路线各自把复杂度降到多少、为什么降、代价是什么,直至能照伪代码实现一个 O(n) 的线性注意力并解释它为何牺牲了精确性"。读完你应当能在面试或设计评审里,把"这四条路线分别在'精确性 vs 复杂度'谱系的哪个位置、何时该选哪个"讲到不含糊。

## 一、动机:二次墙是结构性的,不是工程没优化好

上一门 MLSys 课的 FlashAttention 那章,我们证明了一件重要的事:**标准注意力慢,主要慢在访存(memory-bound),不在算力。** FlashAttention 把那个 N×N 中间矩阵留在片上、不写回 HBM,于是不改变 FLOPs 就把墙钟时间砍掉近一个量级。

但请注意 FlashAttention 解决的是哪一层问题——**它优化的是"同样多的计算怎么更省地搬数据",FLOPs 本身一分没少,仍是 Θ(N²d)。** 这意味着:当序列长到一定程度,即使数据流动已被压榨到极致,纯计算量 N² 这一项迟早会重新成为瓶颈;而显存里那个 Θ(N²) 的注意力矩阵(以及它带来的反向激活)也总有装不下的一天。

这一章换一个角度问问题:**能不能从算法上,让注意力的计算量和显存本身就不再是 N²?** 这是一个比 FlashAttention 更激进的目标——FlashAttention 是"精确地、更省地算 O(N²)";本章的三条路线大多是"用某种近似或结构假设,把 O(N²) 直接降到次二次甚至线性"。**这两件事正交、可叠加:你完全可以用 FlashAttention 去实现一个稀疏注意力的稠密块。** 把这层关系想清楚,是本章的第一个认知锚点。

**先把"二次墙"算成一组能验算的数,这样后面每条路线省了多少才有参照。**

### 1.1 标准注意力的复杂度墙:精确账

记号(单头,self-attention,前向):

- N:序列长度(token 数)
- d:head 维度(head dimension),常见 64 / 128
- Q, K, V ∈ ℝ^{N×d}

三步与各自的时间 / 空间成本:

```
S = Q Kᵀ        # S ∈ ℝ^{N×N}     时间 2N²d FLOP，空间 N² 个数
P = softmax(S)  # 逐行归一化       时间 Θ(N²)（exp/加/除，常数小），空间 N²
O = P V         # O ∈ ℝ^{N×d}     时间 2N²d FLOP，空间 Nd
─────────────────────────────────────────────────────────
时间 ≈ 4N²d FLOP（两次矩阵乘主导）
空间 ≈ Θ(N²)   （那张 N×N 的注意力矩阵；FlashAttention 能把它从 HBM 抹掉，但"逻辑上存在"的 N² 计算量还在）
```

**结论 1(复杂度墙):标准注意力时间 Θ(N²d)、显存 Θ(N²)。两者都对 N 是平方。** 把数字代进去感受一下量级:N 从 2K 涨到 32K(16 倍),N² 涨 256 倍。这就是为什么 32K、128K、1M 上下文不是"多等一会儿"的问题,而是"根本算不动、装不下"的问题——这堵墙是注意力定义里 `每个 token 看所有 token` 这条全连接假设直接带来的,**不是哪个 kernel 没写好**。

下面三条路线,本质都是在动这条全连接假设:

- **路线一(稀疏):** 不让每个 token 看所有 token,只看一个稀疏子集 → 把 N² 的"对数量"砍成近线性。
- **路线二(线性):** 保持每个 token 形式上看所有 token,但用核函数把 softmax 拆开、改变运算顺序,利用矩阵乘结合律绕开 N×N → O(N)。
- **路线四(抛弃注意力):** 干脆不用"两两打分"这种交互方式,改用线性递归(SSM)建模序列 → 训练近线性、推理 O(1) 状态。

(路线三"架构侧省 KV"动的不是计算复杂度而是推理显存,单列一节。)

我们逐条推导。

## 二、路线一:稀疏注意力——砍掉大部分连接

### 2.1 问题:全连接是不是必要的?

自注意力的代价全来自"N 个 query 各自要和 N 个 key 打分"这张完全二部图,共 N² 条边。**核心假设性观察:语言/序列里,真正重要的依赖大多是局部的(相邻词、同一句话),少数是远程的(指代、文档级主题)。如果绝大多数 token 对之间的注意力权重本来就接近 0,那么算它们就是浪费。** 稀疏注意力的赌注就是:**用一个预先设计好的、稀疏的注意力模式(attention pattern)去逼近那张稠密图,只算"可能重要"的那些边。**

把注意力矩阵看成一个 N×N 的 0/1 掩码 mask:mask[i,j]=1 表示允许 i 看 j。标准注意力 mask 全 1(或下三角全 1)。稀疏注意力让 mask 里每行只有 k 个 1(k ≪ N),于是:

```
时间 ≈ Θ(N · k · d)    （每个 query 只和 k 个 key 打分）
空间 ≈ Θ(N · k)        （只存非零位置的分数）
```

**只要 k 是常数或 O(log N) 或 O(√N),复杂度就从 Θ(N²) 降到 Θ(N·k·d) 的近线性。** 问题只剩一个:**mask 该怎么设计,既稀疏又不漏掉重要依赖?** 这是稀疏注意力流派的全部战场。下面看两个有代表性、且被工程验证过的设计。

### 2.2 Longformer:滑动窗口 + 全局 token

Longformer(Beltagy et al., 2020)用三种模式的叠加:

```
1) 滑动窗口（sliding window）：token i 只看 [i-w/2, i+w/2] 这 w 个邻居。
   → 每行 w 个 1，时间 Θ(N·w·d)。w 是固定窗口宽度（如 512）。
2) 空洞窗口（dilated window）：窗口内每隔 dilation 个取一个，
   → 不增加边数却扩大感受野（借鉴空洞卷积），让高层能看更远。
3) 全局 token（global attention）：少数特殊位置（如 [CLS]、问答里的问题 token）
   → 这些位置"看所有、被所有看"，是稀疏图里的 hub。设有 g 个全局 token，
     它们贡献 Θ(g·N) 的边（g 远小于 N，如 g 是常数）。
```

为什么这套组合能 work,需要把两件事讲清:

**(a) 感受野如何随深度增长(为什么局部窗口能传递全局信息)。** 单层滑窗里,token i 只直接看到宽 w 的邻居。但叠 L 层后,信息能传多远?这和 CNN 的感受野(receptive field)推导完全一样:

```
每过一层，i 的信息能向外多扩散 w/2（窗口半径）。
L 层后的感受野半径 ≈ L · (w/2)
→ 感受野宽度 ≈ L·w
```

所以只要 **L·w ≥ N**,理论上最深层的 token 就能(经过多跳中转)接收到全序列信息。代价是:**远程信息要经过多层中转才能到达,不是一跳直达;中转越多,信号越被稀释、越容易在层间被覆盖掉。** 这就是稀疏注意力丢失远程交互的数学本质——不是看不到,而是看得"间接而模糊"。空洞窗口正是为了用同样的边数换更大的单层扩散半径,缩短中转跳数。

**(b) 全局 token 为什么关键。** 局部窗口的多跳中转对"需要精确远程对齐"的任务(如抽取式问答里答案与问题的对应)不够。全局 token 充当**信息枢纽(hub)**:任何位置到任何位置的最短路径长度,因为有 hub,从"O(N/w) 跳"降到"≤2 跳"(i → hub → j)。**这是稀疏图论里经典的"小世界"构造——加少数高连接度节点,直径骤降。** 代价是这 g 个 token 要做稠密注意力,贡献 Θ(g·N),所以 g 必须远小于 N。

**Longformer 总复杂度:Θ(N·w·d + g·N·d)。当 w、g 视为常数,这是 Θ(N·d) 的线性复杂度。** 这是它能处理几千到上万 token 文档的根据。

### 2.3 BigBird:随机 + 窗口 + 全局,以及"为什么随机有用"

BigBird(Zaheer et al., 2020)在 Longformer 的窗口+全局之外,加了**随机注意力(random attention)**:每个 token 再随机看 r 个其它位置。三件套:

```
窗口（locality） + 全局（hub） + 随机（r 个随机边）
每行 ≈ w + g + r 个 1，仍是常数 → Θ(N) 复杂度
```

随机这一招看着随意,背后其实有图论根据,值得讲透:

**核心论证:把注意力 mask 看成一张图,我们希望它在"稀疏"的同时仍是一个好的信息传播图——即任意两点间最短路径短(图直径小)、且没有瓶颈割边。** 随机正则图(random regular graph)有一个深刻性质:**适当度数的随机图是 expander(扩张图),其直径是 O(log N),且谱隙(spectral gap)大、信息混合快。** 直觉:纯局部窗口图直径是 Θ(N/w)(信息要沿链条一格格爬),而加入随机长边后,如同"在链条上随机搭天桥",任意两点期望几跳可达——这正是稀疏图能逼近稠密图信息流的理论支撑。BigBird 论文据此论证:**窗口+全局+随机的稀疏注意力,在理论上仍是图灵完备 / 通用序列函数逼近器**(具体定理表述与条件见原文,此处只取直觉)。

**一句话对比:Longformer 用"窗口+全局"靠多层中转 + hub 传播信息;BigBird 再加"随机长边"把图直径压到 O(log N),用 expander 性质论证稀疏图不损失表达力。两者复杂度都是 Θ(N)(常数度数下)。**

### 2.4 稀疏注意力的代价与坑(算法 + 工程)

- **代价是"硬编码的归纳偏置"。** 稀疏 mask 是人工设计的"哪些边重要"的先验。一旦真实任务的重要依赖落在 mask 之外(比如某个远程指代恰好不在窗口、没被随机选中、又不是全局 token),这条交互就**被结构性地抹掉了**,模型再训也补不回来。这与标准注意力"让数据自己决定看哪里"形成根本对立——**稀疏注意力是用"省算力"换"少一点自由度"。**
- **不规则稀疏对 GPU 极不友好。** GPU 喜欢稠密、规整的矩阵乘。随机/空洞这种散点访问会让显存访问不连续(gather/scatter),实际加速远不及理论值。**真正落地的稀疏注意力几乎都用"块稀疏(block-sparse)"**:把序列分成块,以块为单位决定看不看,块内仍是稠密小矩阵乘(可以用 FlashAttention 算)。这样既稀疏又规整。这是"理论复杂度"和"实测加速比"之间最大的鸿沟——**新手常被论文里的 Θ(N) 误导,以为一定快,实际不做块化会比稠密还慢。**
- **短序列上得不偿失。** 当 N 不大(比如 ≤2K),N·w 和 N² 没拉开差距,稀疏的开销(mask 构造、不规则访问)反而让它比稠密注意力慢。**稀疏注意力只在长序列才划算。**

## 三、路线二:线性注意力——用核函数绕开 softmax

这条路线最优雅,也最容易让人误解,需要把推导一步步铺开。

### 3.1 问题:N² 到底卡在哪个运算上?

回看 `O = softmax(QKᵀ)V`。N² 来自 `S = QKᵀ`(N×d 乘 d×N,得到 N×N)。**如果没有中间这个 softmax,我们能不能换个乘法顺序躲开 N×N?** 看一个纯线性的版本(暂时把 softmax 拿掉):

```
若 O = (Q Kᵀ) V                  # 先算 QKᵀ：N×d · d×N = N×N，再 ·V，时间 Θ(N²d)
则由矩阵乘结合律 O = Q (Kᵀ V)     # 先算 KᵀV：d×N · N×d = d×d，再 Q·：N×d · d×d
```

**关键观察:同一个结果,两种括号方式,复杂度天差地别。** 算右边:

```
KᵀV : d×N 乘 N×d → d×d 矩阵，时间 Θ(N·d²)
Q(KᵀV) : N×d 乘 d×d → N×d，时间 Θ(N·d²)
总时间 Θ(N·d²)，而且中间最大矩阵是 d×d，与 N 无关 → 空间 Θ(d²)
```

**结论(线性注意力的全部数学根据):若注意力可以写成纯矩阵乘 `Q(KᵀV)` 而无中间 softmax,则利用结合律先算 KᵀV(d×d),总复杂度从 Θ(N²d) 降到 Θ(Nd²),对 N 线性。** 当 N ≫ d(长序列),Nd² ≪ N²d,这是数量级的胜利。

**但这里有一道坎:softmax 是非线性的,`softmax(QKᵀ)` 不能简单写成 `Q' K'ᵀ` 的形式,结合律用不上。** 线性注意力流派的全部技巧,就是用一个**核函数(kernel)/ 特征映射 φ** 把 softmax 近似掉,让它重新变成可结合的矩阵乘。

### 3.2 核函数视角:softmax 是一种相似度,换一种相似度就能拆开

标准注意力第 i 个 query 的输出(写成求和形式,先不减 max):

```
Oᵢ = Σⱼ [ exp(qᵢ·kⱼ) / Σⱼ' exp(qᵢ·kⱼ') ] · vⱼ
   = Σⱼ exp(qᵢ·kⱼ) vⱼ  /  Σⱼ exp(qᵢ·kⱼ')
```

把 `exp(qᵢ·kⱼ)` 看成一个**相似度核** sim(qᵢ, kⱼ) = exp(qᵢ·kⱼ)。线性注意力的核心想法:**换一个能"分解"的相似度核,即找一个特征映射 φ: ℝ^d → ℝ^m,使 sim(q,k) ≈ φ(q)·φ(k)。** 一旦能这样分解,代入上式:

```
Oᵢ = Σⱼ φ(qᵢ)·φ(kⱼ) vⱼ  /  Σⱼ φ(qᵢ)·φ(kⱼ')
```

注意 φ(qᵢ) 与求和变量 j 无关,可以**提到求和号外**(这是关键一步):

```
分子 = φ(qᵢ)ᵀ [ Σⱼ φ(kⱼ) vⱼᵀ ]        # 中括号是一个 m×d 矩阵，记 S = Σⱼ φ(kⱼ)vⱼᵀ
分母 = φ(qᵢ)ᵀ [ Σⱼ φ(kⱼ) ]            # 中括号是一个 m 维向量，记 z = Σⱼ φ(kⱼ)

Oᵢ = φ(qᵢ)ᵀ S  /  (φ(qᵢ)ᵀ z)
```

**这里发生了魔法:S = Σⱼ φ(kⱼ)vⱼᵀ 和 z = Σⱼ φ(kⱼ) 都和 i 无关——它们是对所有 key/value 的一次性聚合,算一遍 Θ(N·m·d) 就够,与具体哪个 query 无关。** 然后每个 query 只需做 φ(qᵢ)ᵀS(m×d 的乘)和 φ(qᵢ)ᵀz(m 维点积),各 Θ(m·d) 和 Θ(m)。

复杂度核算:

```
预聚合 S, z :  遍历 N 个 (kⱼ,vⱼ)，每个算 φ(kⱼ)vⱼᵀ 是 Θ(m·d)  → 总 Θ(N·m·d)
所有 query :   N 个，每个 Θ(m·d)                              → 总 Θ(N·m·d)
─────────────────────────────────────────────────────────────
总时间 Θ(N·m·d)，空间 Θ(m·d)（存 S）+ Θ(m)（存 z），与 N 无关
```

**结论 2(线性注意力 O(N)):若 softmax 核被特征映射 φ 替换为 φ(q)·φ(k) 形式,则注意力可写成 `Oᵢ = φ(qᵢ)ᵀS / (φ(qᵢ)ᵀz)`,其中 S、z 是对全体 key/value 的一次性求和。总复杂度 Θ(N·m·d),对 N 线性,显存 Θ(m·d) 与 N 无关。** 当特征维 m 取成 Θ(d) 量级,就是 Θ(Nd²),正好对上 3.1 的结合律分析——φ(K)ᵀφ(Q) 的结合律版本。

这正是动手练习要你独立复现的推导:**O(N) 的根源不是任何近似带来的,而是"φ(q) 能提到对 j 的求和号外,从而把对所有 key 的聚合 S/z 复用给每个 query"——本质就是矩阵乘结合律的求和形式。**

### 3.3 自回归(因果)情形:从"一次性聚合"到"前缀和递推"

上面的 S、z 是对**全部** j 求和,这适用于双向(encoder)场景。但自回归生成里,第 i 个 token 只能看 j ≤ i(因果掩码)。这时 S、z 不能一次性算完——它们对每个 i 是不同的前缀和:

```
Sᵢ = Σ_{j≤i} φ(kⱼ) vⱼᵀ      # i 位置的累计状态（m×d 矩阵）
zᵢ = Σ_{j≤i} φ(kⱼ)          # i 位置的累计归一化（m 维）
Oᵢ = φ(qᵢ)ᵀ Sᵢ / (φ(qᵢ)ᵀ zᵢ)
```

而 Sᵢ、zᵢ 满足一个**线性递推**:

```
Sᵢ = Sᵢ₋₁ + φ(kᵢ) vᵢᵀ
zᵢ = zᵢ₋₁ + φ(kᵢ)
```

**这是线性注意力一个极其深刻的结论:在因果情形下,它等价于一个 RNN——状态是固定大小的 (S, z)(m×d 矩阵 + m 维向量),每步 O(1) 更新(与已生成长度 i 无关)。** 对比标准注意力的自回归生成:第 i 步要存 i 个 K/V 并和它们全部打分,状态随 i 线性增长、每步 Θ(i·d)。

**这一点直接解释了线性注意力在推理上的杀手锏:推理时是 O(1) 状态、O(1) 每步——KV cache 不再随序列增长!** 这与下面要讲的 SSM 殊途同归(它们在数学上是近亲),也是长序列推理最看重的性质。代价我们马上讲。

训练时,这个前缀和可以并行算(parallel scan,见路线四),所以训练仍是近线性、可并行,不像传统 RNN 必须串行。

### 3.4 φ 怎么选:从"随便挑"到 Performer 的随机特征

φ 的选择直接决定近似质量。两类代表:

**(a) 简单正特征(linear attention,Katharopoulos et al., 2020)。** 直接取 `φ(x) = elu(x) + 1`(或 `softmax`、`relu` 等),保证 φ 的每个分量为正(注意力权重应非负)。**这根本不试图逼近 exp 核,只是换了一个自己定义的、可分解的相似度。** 简单、快,但表达力和精确 softmax 差距明显——它本质是个不同的模型,不是 softmax 的近似。

**(b) 随机特征逼近 exp 核(Performer,Choromanski et al., 2021)。** Performer 想真正**无偏地逼近 softmax 的 exp 核**。它用的数学工具是随机傅里叶特征(random features)的一个变体——FAVOR+(Fast Attention Via positive Orthogonal Random features)。核心恒等式(高斯核的随机特征展开,这里给出 Performer 用的正特征版本):

```
对 exp(qᵀk)（softmax 的核），存在一个随机映射 φ 使
    E[ φ(q)ᵀ φ(k) ] = exp(qᵀk)        # 期望意义下无偏

Performer 的正随机特征（简化形式）：
    φ(x) = exp(-‖x‖²/2) / √m · [ exp(ω₁ᵀx), ..., exp(ω_mᵀx) ]
其中 ωₛ ~ N(0, I_d) 是 m 个独立高斯随机向量（实践中做正交化）
```

为什么这个 φ 的内积期望等于 exp(qᵀk)?用高斯积分(这是 random features 的标准推导,值得过一遍):

```
E_ω[ exp(ωᵀq) exp(ωᵀk) ]                       # ω ~ N(0, I)
  = E_ω[ exp(ωᵀ(q+k)) ]
  = exp( ‖q+k‖² / 2 )                          # 高斯矩母函数 E[exp(ωᵀa)] = exp(‖a‖²/2)
  = exp( (‖q‖² + 2qᵀk + ‖k‖²) / 2 )
所以
  exp(-‖q‖²/2) · E_ω[exp(ωᵀq)exp(ωᵀk)] · exp(-‖k‖²/2)
  = exp( (‖q‖²+2qᵀk+‖k‖²)/2 - ‖q‖²/2 - ‖k‖²/2 )
  = exp( qᵀk )                                  ✓ 正是 softmax 的核
```

把那两个 `exp(-‖·‖²/2)` 分别并进 φ(q)、φ(k),就得到上面的 φ。用 m 个独立 ω 取样本均值来近似这个期望,样本越多越准。**用正特征(全 exp,恒正)而非原始的 cos/sin 三角随机特征,是为了保证 φ ≥ 0,从而 φ(q)ᵀφ(k) ≥ 0——避免注意力权重出现负数和分母接近 0 的数值灾难。这是 Performer 的关键工程贡献,论文里证明三角特征会让方差在 softmax 的小值区爆炸。**

**正交随机特征(让 m 个 ω 两两正交)能进一步降低估计方差**,这是 FAVOR+ 里的"O"(orthogonal)。

### 3.5 量化分析:方差、近似误差与"为什么质量打折"

线性注意力(尤其 Performer)是**有偏 / 有方差的近似**,必须把代价算清,否则就是盲目乐观:

- **方差随特征维 m 衰减。** 用 m 个随机特征做蒙特卡洛估计,方差 ∝ 1/m。**要把估计误差压到可接受,m 往往需要不小**(和 d 同量级甚至更大),这会侵蚀线性注意力的速度优势——若 m 取到 N 量级就完全失去意义。实践中 m 取 Θ(d log d) 之类。
- **误差在"尖锐注意力"处最严重。** softmax 在某个 query 高度集中在少数 key(分布尖锐、低熵)时,exp 核的值跨度极大,随机特征逼近的相对方差最大。**直觉:线性注意力擅长"平滑、弥散"的注意力,不擅长"精确检索某一个 token"。** 这恰好是它在需要精确长程检索的任务(如 needle-in-a-haystack、精确复制)上明显弱于 softmax 的原因——也是后来一系列"线性注意力召不回长程信息"实证的根。
- **归一化的数值坑。** 分母 `φ(qᵢ)ᵀz` 在近似下可能非常小甚至理论上为负(若 φ 不保正),导致输出爆炸。**这就是为什么必须用正特征(φ≥0)** ,以及实践中要给分母加一个小 ε 稳定。
- **没有 max 归一化的稳定性。** softmax 减 max 的数值技巧在这里用不了(我们绕开了 softmax),exp(ωᵀx) 在 ‖x‖ 大时仍可能溢出,需要对 q/k 做缩放或归一化(如先除 √d、或 layernorm)。

**一句话总结路线二的取舍:线性注意力把复杂度从 Θ(N²d) 干到 Θ(Nmd) 的线性、推理变成 O(1) 状态的 RNN,代价是把"精确的、可任意尖锐的 softmax"换成"有方差的、偏平滑的近似核",在需要精确长程检索的任务上质量明显下降。它是"线性复杂度"和"召回精度"之间的硬权衡——没有免费午餐。**

### 3.6 代码:O(N) 线性注意力的核心实现

下面给两个版本,体现 3.2 的"先聚合 KV"和 3.3 的"因果递推"两套机制。聚焦机制,省略 batch/head 维和边界。

```python
import torch

# --- 版本 A：非因果（双向），先聚合 KᵀV，对 N 线性 ---
def linear_attention_bidirectional(Q, K, V, feature_map):
    # Q,K,V: [N, d]；feature_map: φ，把 [*, d] 映射到 [*, m]，要求输出 ≥ 0
    phi_q = feature_map(Q)          # [N, m]
    phi_k = feature_map(K)          # [N, m]

    # 关键：先把所有 key/value 聚合成与 N 无关的 S 和 z（这一步是 O(N) 的源头）
    # S = Σⱼ φ(kⱼ) vⱼᵀ  ∈ [m, d]    （爱因斯坦求和：对 j 求和）
    S = torch.einsum('nm,nd->md', phi_k, V)   # [m, d]，与 N 无关！
    z = phi_k.sum(dim=0)                       # [m]，分母聚合 Σⱼ φ(kⱼ)

    # 每个 query 复用同一个 S、z：分子 φ(q)ᵀS，分母 φ(q)ᵀz
    num = torch.einsum('nm,md->nd', phi_q, S)  # [N, d]
    den = torch.einsum('nm,m->n', phi_q, z)    # [N]
    return num / (den[:, None] + 1e-6)         # ε 防止分母为 0

# --- 版本 B：因果（自回归），等价于固定大小状态的 RNN，每步 O(1) ---
def linear_attention_causal(Q, K, V, feature_map):
    N, d = Q.shape
    m = feature_map(Q[:1]).shape[-1]
    phi_q = feature_map(Q)          # [N, m]
    phi_k = feature_map(K)          # [N, m]

    S = torch.zeros(m, d)           # 累计状态 Sᵢ = Σ_{j≤i} φ(kⱼ)vⱼᵀ
    z = torch.zeros(m)              # 累计归一化 zᵢ = Σ_{j≤i} φ(kⱼ)
    O = torch.zeros(N, d)
    for i in range(N):              # 真实训练用 parallel scan 并行；这里展示递推语义
        S = S + torch.outer(phi_k[i], V[i])   # Sᵢ = Sᵢ₋₁ + φ(kᵢ)vᵢᵀ   —— O(1) 更新
        z = z + phi_k[i]                       # zᵢ = zᵢ₋₁ + φ(kᵢ)
        O[i] = (phi_q[i] @ S) / (phi_q[i] @ z + 1e-6)
    return O
# 注意：版本 B 的状态 (S, z) 大小恒为 m×d + m，与 i 无关
#       —— 这就是"推理 O(1) 状态、不需要随序列增长的 KV cache"的代码证据。
```

`feature_map` 可以是简单正特征 `lambda x: torch.nn.functional.elu(x) + 1`(linear attention),也可以是 Performer 的 FAVOR+ 随机特征(需先采样并正交化一组 ω,然后 `φ(x)=exp(-‖x‖²/2)·exp(xWω)/√m`)。**对拍提示:当 feature_map 取 elu+1 时,版本 A 和"对 elu+1 核做精确稠密注意力"应数值相等,可用 `allclose` 验证结合律没写错;但它和真正的 softmax 注意力不会相等——因为换了核。**

## 四、路线三:架构侧省 KV——MQA / GQA / 滑窗

前两条路线动的是**计算复杂度**(FLOPs)。但长序列推理还有另一堵墙是**显存**:KV cache 随序列和 batch 线性膨胀(MLSys 课 KV cache 那章已逐字节推导过显存公式)。这一节从**模型架构**角度看怎么省 KV——它不改变注意力的 O(N²) 计算本质,只减小那个"每 token 要存的 K/V 字节数"的系数。**这条路线是当前生产大模型几乎都在用的,务必和前两条区分开:前两条改复杂度的"阶",这条改的是显存的"系数"。**

### 4.1 KV cache 的字节账(快速回顾)

标准多头注意力(MHA),H 个头,每头 d_head 维,L 层,序列长 S,每数 b 字节,batch 大小 B:

```
KV cache 字节 = B · L · S · 2 · H · d_head · b
                              ↑
                            K 和 V 各一份
```

注意里面有个 `2·H·d_head`——**K、V 的头数 H 直接是系数。如果能让多个 query 头共享同一份 K/V,这个 H 就能压下来。** 这正是 MQA/GQA 的全部想法。

### 4.2 MQA 与 GQA:KV 头共享

```
MHA（标准）:   H 个 Q 头，H 个 K 头，H 个 V 头        —— KV 系数 = H
MQA（Multi-Query Attention，Shazeer 2019）:
               H 个 Q 头，但只有 1 个 K 头、1 个 V 头   —— KV 系数 = 1，省 H 倍
               所有 query 头共享同一份 K、V
GQA（Grouped-Query Attention，Ainslie et al. 2023）:
               H 个 Q 头分成 G 组，每组共享 1 份 K/V    —— KV 系数 = G（1<G<H）
               是 MHA（G=H）和 MQA（G=1）之间的插值
```

**机制:Q 仍是 H 个头各看各的子空间(保留多头的表达多样性),但 K/V 投影只产出 1 份(MQA)或 G 份(GQA),在打分时被多个 Q 头复用。** 量化收益直接:

```
GQA 的 KV cache 字节 = B · L · S · 2 · G · d_head · b
相比 MHA 省 H/G 倍。MQA（G=1）省 H 倍——如 H=32 时 KV cache 直接小 32 倍。
```

**为什么不全用 MQA(G=1)省到极致?因为质量。** MQA 让所有 query 头共享一份 K/V,等于强行抹掉了"不同头关注不同 key 子空间"的能力,**实测会有质量损失、训练也更不稳**。GQA 是工程上找到的甜点:**用 G(如 8)份 K/V,绝大部分 KV 显存照样省下来(省 H/G 倍),质量损失却远小于 MQA。** 这是典型的"复杂度系数 vs 质量"的连续权衡——把 G 当旋钮在两端之间调。

**这条路线还有一个隐藏价值,呼应 MLSys:** decode 是 memory-bound 的,瓶颈是从 HBM 搬 K/V。**省 KV 不只省显存容量,还直接减少 decode 时要搬运的 K/V 字节数,从而提升 memory-bound 的 decode 吞吐。** 所以 GQA 几乎是当前推理友好型模型的标配(LLaMA-2/3 较大版本、多数开源大模型都用 GQA[待核:各模型具体 G 值])。

### 4.3 滑动窗口注意力(Mistral):把 KV cache 上界也压成常数

GQA 压的是 KV 的"头数"系数,但 KV cache 仍随序列长 S 线性增长(那个 S 还在公式里)。**滑动窗口注意力(Sliding Window Attention, SWA)**(Mistral 7B 采用)直接动 S:**每个 token 只看最近 W 个 token(因果滑窗),于是任意时刻只需缓存最近 W 个位置的 K/V,KV cache 大小封顶在 Θ(W),不再随总长度 S 增长。**

```
普通因果注意力 decode：KV cache ∝ S（已生成长度），无上界
滑窗（窗口 W）decode：    只需保留最近 W 个 K/V → KV cache ∝ min(S, W)，封顶常数
```

它本质就是路线一的滑动窗口,但**用途侧重在推理显存**(把 KV cache 钉死成常数),而非训练计算复杂度。远程信息怎么办?和 Longformer 同理——**靠多层堆叠的感受野传递:L 层、窗口 W,理论感受野 ≈ L·W,只要够大就能(经多跳)覆盖更远。** Mistral 的设计点是用相对小的 W(如 4096)配足够的层数,让有效上下文远超单层窗口。代价同样是远程交互是间接多跳、可能被稀释。

**一句话:路线三不碰注意力的算法复杂度,而是从架构上减小"每 token 的 KV 字节"——MQA/GQA 压头数系数(省 H/G 倍 KV、提 decode 吞吐),滑窗把 KV cache 上界钉成常数。它们和前两条路线正交,可叠加,且是当前生产模型的主流选择。**

## 五、路线四:抛弃注意力——状态空间模型 SSM 与 Mamba

最激进的路线:**根本不用"两两打分"这种交互。** 既然线性注意力在因果情形下已经退化成一个"固定大小状态的线性 RNN"(3.3 节),那不如直接从"线性递归 + 状态"这个第一性出发,设计一个比注意力更原生地线性、且推理 O(1) 状态的序列模型。这就是状态空间模型(State Space Model, SSM),以及它的当红代表 Mamba。

### 5.1 从连续状态空间到离散线性递归

SSM 的根来自经典控制论里的连续线性系统。一个输入 u(t)、输出 y(t)、隐状态 h(t) ∈ ℝ^Nstate 的连续状态空间方程:

```
h'(t) = A h(t) + B u(t)        # 状态怎么随时间演化（A 是 Nstate×Nstate 状态转移）
y(t)  = C h(t) + D u(t)        # 从状态读出输出（D 通常并成残差，常略去）
```

要用在离散 token 序列上,需要**离散化(discretization)**——把连续方程在步长 Δ 下变成离散递推。常用零阶保持(Zero-Order Hold, ZOH)离散化,得到:

```
hₜ = Ā hₜ₋₁ + B̄ uₜ              # 离散状态转移
yₜ = C hₜ

其中（ZOH）：
  Ā = exp(Δ A)
  B̄ = (ΔA)⁻¹ (exp(ΔA) − I) · ΔB    ≈ Δ B   （当 Δ 小时的一阶近似）
```

**这就是一个线性 RNN:状态 hₜ 是固定大小 Nstate 维向量,每步用 Ā、B̄、C 做线性更新,与已处理长度无关。** 推理时:**O(1) 状态(只存 hₜ,Nstate 维)、O(1) 每步计算——和线性注意力一样,没有随序列增长的 KV cache。** 这是 SSM 对注意力最根本的优势。

### 5.2 关键难题一:线性 RNN 怎么并行训练?——卷积视角 + 并行扫描

线性 RNN 推理 O(1) 很美,但训练若必须串行(像传统 RNN 一步等一步),就无法吃满 GPU,失去对 Transformer 的竞争力。SSM 的突破在于:**线性递归可以并行计算,有两条等价路径。**

**(a) 卷积视角(LTI 情形,A/B/C 不随时间变)。** 把递推展开(设 h₋₁=0):

```
h₀ = B̄u₀
h₁ = ĀB̄u₀ + B̄u₁
h₂ = Ā²B̄u₀ + ĀB̄u₁ + B̄u₂
...
yₜ = C hₜ = Σ_{k=0}^{t} C Āᵏ B̄ · u_{t−k}
```

**这正好是一个卷积!** 定义卷积核(SSM kernel):

```
K̄ = ( CB̄, CĀB̄, CĀ²B̄, ..., CĀ^{N-1}B̄, ... )     # 长度 = 序列长
则  y = K̄ * u                                       # u 与 K̄ 的因果卷积
```

于是训练时可以**用 FFT 把这个长卷积 O(N log N) 算出来**(而非串行 O(N) 步),完美并行。**这是 S4(Structured State Space, Gu et al. 2022)的核心:把线性 RNN 当成一个超长卷积核,FFT 并行训练;推理时再切回 O(1) 的递归形式。同一个模型,两副面孔——训练用卷积,推理用递归。**

**(b) 并行扫描视角(prefix scan)。** 递推 `hₜ = Āhₜ₋₁ + B̄uₜ` 是一种"带状态的前缀运算"。前缀和(prefix sum)有经典的并行算法(Blelloch scan),复杂度 O(N) 工作量、O(log N) 深度(并行步数)。线性递推因为转移是线性的、可结合(两步合成一步:`(A₂,b₂)∘(A₁,b₁)=(A₂A₁, A₂b₁+b₂)`),所以同样能用 **associative parallel scan** 在 O(log N) 深度内算完。**Mamba 用的就是这条路径(硬件感知的并行扫描),而非 FFT——因为 Mamba 的转移矩阵随输入变化(见下节),不再是固定卷积核,FFT 用不了,只能 scan。**

**这一步是把 SSM 从"理论上线性"变成"实际上能在 GPU 上训得快"的关键。** 注意 3.3 节的线性注意力因果递推 `Sᵢ=Sᵢ₋₁+φ(kᵢ)vᵢᵀ` 也是这种可结合递推——所以线性注意力训练同样可以 parallel scan。**线性注意力和 SSM 在数学结构上是近亲:都是"固定大小状态的线性递归 + 可并行扫描"。**

### 5.3 关键难题二:固定状态怎么记得住长程?——HiPPO 与结构化 A

线性 RNN 的老问题:**状态 h 大小固定(Nstate 维),要把任意长历史压进这 Nstate 维,凭什么不丢信息?** 朴素初始化的 A 会让远处信息指数衰减(记忆很短)。S4 的关键理论武器是 **HiPPO(High-order Polynomial Projection Operators)**:

**HiPPO 给出一类特殊的 A 矩阵,使得隐状态 h(t) 最优地保存了"输入历史在一组正交多项式基(如勒让德多项式)上的投影系数"。** 直觉:**与其让状态随便记,不如让它持续维护"用 Nstate 个系数对整段历史做最优多项式逼近"——这样固定 Nstate 维状态就能以可控精度压缩任意长历史,且有逼近论保证(逼近误差随 Nstate 多项式衰减,而非指数遗忘)。** 这是 SSM 能在长程依赖 benchmark(如 Long Range Arena)上一度超过 Transformer 的理论根据。

S4 还把 A 做**结构化**(对角 + 低秩,或后续工作直接用对角 A,即 S4D / DSS),这样:

- `Āᵏ = exp(ΔA)ᵏ` 在对角结构下能高效算(对角阵幂就是逐元素幂);
- 卷积核 K̄ 的计算从 O(N²) 降到近线性。

**否则一般 Nstate×Nstate 的 A 求幂是 O(Nstate³),SSM 根本算不起。结构化 A(尤其对角化)是 SSM 工程可行的前提,和 HiPPO 的记忆理论是 SSM 两条腿。**

### 5.4 Mamba:让 SSM "看内容"——选择性(selectivity)

S4 虽好,但有个根本短板:**它是 LTI(线性时不变)的——A、B、C、Δ 对所有 token 都一样,不随输入内容变化。** 这意味着 SSM 像一个固定的卷积滤波器,**无法做"内容相关的选择"**:它不能根据当前 token 决定"这个信息要不要记进状态、要不要忽略"。而这恰恰是注意力最强的地方(注意力天然内容相关——打分就是看 q、k 的内容)。这导致 S4 在需要内容选择的任务(如选择性复制、语言建模里的精确召回)上弱于 Transformer。

**Mamba(Gu & Dao, 2023)的核心创新:让 SSM 的参数 B、C、Δ 成为输入的函数(selective SSM / S6)。**

```
S4（LTI）:        Δ, B, C 是固定参数，对所有 token 相同
Mamba（selective）: Δₜ = softplus(Linear_Δ(uₜ))   # 步长随输入变
                    Bₜ = Linear_B(uₜ)              # 输入矩阵随输入变
                    Cₜ = Linear_C(uₜ)              # 输出矩阵随输入变
                    （A 仍是结构化的、可学的固定对角阵）
递推变成时变：  hₜ = Āₜ hₜ₋₁ + B̄ₜ uₜ,   Āₜ = exp(Δₜ A)
```

**这一改的意义:Δₜ 随输入变,等价于让模型对每个 token 动态决定"步长 / 遗忘强度"——Δ 大则 Āₜ=exp(ΔA) 让旧状态衰减得多(更关注当前、遗忘历史),Δ 小则几乎保留旧状态(跳过当前、维持记忆)。这就给了 SSM 内容相关的"选择性门控":重要的 token 写入状态,不重要的(如填充词)被跳过。** 这正是 Mamba 在语言建模上追平甚至超过同规模 Transformer 的关键。

**代价:时变的 Āₜ 让卷积视角失效(不再是固定卷积核,FFT 用不了)。** 所以 Mamba 必须用 5.2(b) 的**并行扫描**,并配一个**硬件感知(hardware-aware)的 scan kernel**(把状态保持在 SRAM、用 kernel fusion 避免把巨大的中间状态写回 HBM——这思路和 FlashAttention 一脉相承)。**这是 Mamba 论文标题"linear-time sequence modeling with selective state spaces"里那个 hardware-aware 的由来:没有这个 kernel,时变 SSM 的并行扫描会因访存而很慢。**

### 5.5 SSM vs 注意力:本质区别与各自长短

把这章最该带走的对比讲透。**两者的根本差异是"如何存储和访问历史":**

```
                  标准注意力                    SSM / Mamba
─────────────────────────────────────────────────────────────────
历史存储        全量 KV cache（无损保留每个      固定大小状态 h（Nstate 维，
                token 的 K/V），大小 ∝ S         有损压缩整段历史），大小 = 常数
访问历史方式    每步对全部历史 token 显式打分     只能通过当前状态 h 间接访问，
                （内容寻址，可精确检索任意位置）   无法"回头精确读"某个具体旧 token
推理每步成本    Θ(S·d)（与已生成长度成正比）      Θ(Nstate·d)（常数，与长度无关）
推理显存        Θ(S)（KV cache 随长度涨）         Θ(Nstate)（状态固定，不涨）
训练复杂度      Θ(N²d)                           近线性（卷积 O(NlogN) 或 scan O(N)）
内容选择        天生（打分即内容相关）            S4 没有；Mamba 用 selective 补上
精确长程检索    强（直接寻址历史任一 token）       弱（信息已被压进固定状态，难无损召回）
```

**最核心的一句:注意力是"无损存全部历史 + 每步重新检索"(空间换精度),SSM 是"有损压缩成固定状态 + 递推更新"(精度换空间和速度)。** 这决定了它们的长短:

- **注意力强在精确检索**(needle-in-haystack、精确复制、in-context learning 里精确抓取示例),因为它真的把每个历史 token 都留着、能逐个精确打分。
- **SSM/Mamba 强在长序列效率**(推理 O(1) 状态、训练近线性),弱在"固定状态装不下要精确召回的全部细节"——所以它在某些精确检索任务上仍落后于注意力。
- **这正催生了混合架构**:很多近期模型把 Mamba 层和注意力层交替堆叠(如 Jamba 等[待核:具体配比]),让 SSM 层承担大部分廉价的序列建模、少量注意力层负责精确检索——**取两者之长。这是当前长上下文架构最活跃的方向之一。**

### 5.6 Mamba 选择性扫描:核心伪代码

```python
# Selective SSM (Mamba S6) 单层前向的机制骨架（省略卷积分支、归一化、batch/通道维）
# u: [N, D] 输入序列；A: [D, Nstate] 可学的（每通道一个对角 SSM，结构化）
def selective_ssm(u, A, W_delta, W_B, W_C):
    N, D = u.shape
    Nstate = A.shape[1]

    # --- 关键：B, C, Δ 都随输入 u 变（这是 Mamba 区别于 S4 的"选择性"）---
    delta = softplus(u @ W_delta)        # [N, D]   每个 token、每通道一个步长 Δₜ
    B = u @ W_B                          # [N, Nstate]  输入矩阵随 token 变
    C = u @ W_C                          # [N, Nstate]  输出矩阵随 token 变

    # 离散化（ZOH 的对角近似）：Āₜ = exp(Δₜ ⊙ A)，B̄ₜ ≈ Δₜ ⊙ Bₜ
    dA = torch.exp(delta[..., None] * A)         # [N, D, Nstate]  时变状态转移
    dB = delta[..., None] * B[:, None, :]        # [N, D, Nstate]

    # --- 递推（真实实现用 hardware-aware parallel scan，O(logN) 深度；此处展示语义）---
    h = torch.zeros(D, Nstate)
    y = torch.zeros(N, D)
    for t in range(N):
        h = dA[t] * h + dB[t] * u[t][:, None]    # hₜ = Āₜ⊙hₜ₋₁ + B̄ₜ uₜ —— 固定大小状态
        y[t] = (h * C[t][None, :]).sum(-1)       # yₜ = Cₜ hₜ
    return y
# 对比注意力：这里没有任何 N×N 打分；状态 h 大小恒为 D×Nstate，与 t 无关 → 推理 O(1)。
# Δₜ 随输入变 → 模型动态决定每个 token 写入/遗忘多少状态 → 内容相关的"选择"。
```

## 六、设计权衡与常见坑(横向)

把四条路线放到一张"精确性 vs 复杂度"的谱系上,这是本章最该内化的全局图:

```
精确  ←──────────────────────────────────────────────→  线性/高效
精确性高，复杂度高                                    复杂度低，精确性打折
│                                                                    │
标准注意力 ── (FlashAttention：精确，只省访存，仍 O(N²) FLOP) ──┐    │
│                                                              │     │
稀疏注意力（块稀疏）：近线性，丢"mask 外"的远程交互 ──────────┤     │
│                                                              │     │
GQA/滑窗：不改复杂度阶，省 KV 显存系数（与上面正交，可叠加）──┘     │
│                                                                    │
线性注意力 / SSM(S4) / Mamba：O(N) 训练、O(1) 推理状态，弱精确检索 ──┘
```

逐条坑点(都是评审/实战里真摔过的):

- **"次二次"≠"一定更快"。** 稀疏/线性的渐进优势只在 N 足够大时显现;N 小时常数项和不规则访问让它们慢于经过 FlashAttention 优化的稠密注意力。**先估算你的 N 落在交叉点哪一侧。** 块稀疏、kernel 友好的实现才能把理论优势变成实测加速。
- **FlashAttention 和这章不是竞品,是正交工具。** 反复强调:FA 是精确 O(N²) 的访存优化;本章是降复杂度阶的算法。**两者叠加**——稀疏注意力的稠密块、线性注意力的内层、SSM 的扫描,都可以也应该用 IO 感知 kernel 实现。把它们当对立选项是常见误区。
- **线性/SSM 的"精确长程检索"短板是结构性的,不是调参能补的。** 把无穷历史压进固定状态,信息论上就会有损。**需要精确召回(RAG、长文档精确问答、in-context 精确抓取)的场景,纯线性方案要谨慎;混合架构(少量注意力层兜底检索)往往是更稳的选择。**
- **稀疏 mask 的设计是硬归纳偏置,会"结构性致盲"。** 重要依赖一旦落在 mask 外就永久丢失。设计 mask 等于赌"重要的边都在我画的图里"——赌错了模型再训也救不回。
- **MQA 太激进会掉点。** 把 KV 头压到 1(MQA)省显存最多,但质量损失和训练不稳常不可接受;**GQA 的 G(如 8)是更稳的甜点**。别为省那点显存把质量赌进去。
- **Performer 等随机特征:m 太小则方差大、近似差;m 太大则吃掉线性优势。** m 的选择是精度-速度的直接旋钮,且对"尖锐注意力"任务再大也救不回相对方差。
- **离散化和数值。** SSM 的 Ā=exp(ΔA) 要稳定就得控制 Δ 和 A 的谱(A 的实部应 ≤ 0,否则状态发散);Δ 用 softplus 保正。这些数值细节做错了训练直接 NaN。

## 七、动手练习

1.(推导题,核心)**独立证明线性注意力为何 O(N)。** 从 `Oᵢ = Σⱼ [sim(qᵢ,kⱼ)/Σⱼ' sim(qᵢ,kⱼ')] vⱼ`、设 sim(q,k)=φ(q)·φ(k) 出发,推出 `Oᵢ = φ(qᵢ)ᵀS/(φ(qᵢ)ᵀz)`,明确写出 S、z 的定义,并核算预聚合 + 所有 query 的总复杂度,论证它是 Θ(N·m·d)。*提示:全部关键在"φ(qᵢ) 与求和变量 j 无关,可提到 Σⱼ 外面",从而 S=Σⱼφ(kⱼ)vⱼᵀ 只算一次被所有 query 复用——这就是结合律的求和形式。再做因果版:写出 Sᵢ=Sᵢ₋₁+φ(kᵢ)vᵢᵀ 的递推,论证它是固定大小状态的 RNN、推理 O(1)。*

2.(分析题,对比谱系)**把稀疏 / 线性 / SSM 放到"精确性 vs 复杂度"坐标上对比。** 对每条路线回答:(a) 训练时间复杂度、(b) 推理每步成本与显存随序列长的增长、(c) 它牺牲了什么精确性、(d) 最适合 / 最不适合的任务各一例。*提示:抓住"是否无损保留全部历史 + 能否精确寻址任一旧 token"这条主线——它同时决定复杂度和精确检索能力,是两者权衡的同一个根。重点写清楚:为什么"省复杂度"和"丢精确长程检索"在这三条路线里是同一枚硬币的两面。*

3.(编码题)**实现并对拍线性注意力的结合律。** 用 PyTorch 写 `linear_attention_bidirectional`(先聚合 S、z 的 O(N) 版),feature_map 取 `elu(x)+1`;再写一个"对同一个 elu+1 核做精确稠密注意力"的朴素 O(N²) 版,用 `torch.allclose` 验证两者数值相等(验证结合律没写错)。然后把 feature_map 换成 softmax 核对应的精确注意力,展示线性版与真 softmax **不相等**(因为换了核)。*提示:第一组 allclose 应通过(同一个核的两种算法),第二组应明显不等(不同核)——这恰好把"O(N) 来自结合律(精确无损)"和"质量损失来自换核(近似)"两件事在代码里分开,这是本章最容易混淆的点。*

4.(推导/思考题)**推导 SSM 的卷积形式,并说明 Mamba 为何不能用它。** 从 `hₜ=Āhₜ₋₁+B̄uₜ, yₜ=Chₜ`(LTI)展开,推出 `yₜ=Σₖ CĀᵏB̄·u_{t−k}`,写出卷积核 K̄,说明训练为何能用 FFT 做到 O(N log N)。再论证:当 Ā 随 token 变成 Āₜ(Mamba 的 selective),为什么这个卷积核展开不再成立、只能退回并行扫描。*提示:卷积核 K̄=（CB̄,CĀB̄,CĀ²B̄,…）要求 Ā 对所有位置相同(时不变),才能写成"位置差 t−k 的固定函数"。Āₜ 时变后,uₜ₋ₖ 的系数变成 `C Āₜ Āₜ₋₁…Āₜ₋ₖ₊₁ B̄ₜ₋ₖ`,依赖绝对位置而非仅位置差,不再是卷积——但仍是可结合的线性递推,故 parallel scan 仍可用。*

## 九、小结与承上启下

这一章我们把"打破注意力二次墙"的四条路线讲到能各自验算复杂度、各自说清代价:

```
复杂度墙：标准注意力 时间 Θ(N²d)、显存 Θ(N²)，源自"每 token 看所有 token"的全连接假设
  ├─ 路线一 稀疏：砍连接，每行 k 个边 → Θ(N·k·d) 近线性
  │         （窗口靠多层感受野 L·W；全局 token 当 hub；随机边让图成 expander、直径 O(logN)）
  │         代价：硬编码 mask 致盲、不规则访问需块化
  ├─ 路线二 线性：换核 φ 使 sim=φ(q)·φ(k)，结合律先聚合 S=Σφ(k)vᵀ → Θ(N·m·d)
  │         因果下退化成"固定状态线性 RNN"，推理 O(1) 状态
  │         代价：有方差近似（∝1/m），偏平滑、弱精确检索（Performer 用正随机特征保稳定）
  ├─ 路线三 省 KV：不改复杂度阶，只压显存系数
  │         MQA/GQA 共享 KV 头省 H/G 倍、提 decode 吞吐；滑窗把 KV cache 钉成常数
  │         与一、二正交可叠加，是当前生产模型主流
  └─ 路线四 抛弃注意力(SSM/Mamba)：连续系统离散化成线性递归 hₜ=Āhₜ₋₁+B̄uₜ
            训练用卷积(FFT, O(NlogN))或并行扫描(O(logN)深度)；推理 O(1) 状态
            HiPPO 保证固定状态的长程记忆；Mamba 让 B/C/Δ 随输入变补上内容选择
            代价：固定状态有损，弱精确检索 → 催生与注意力的混合架构
```

**最该带走的一句话:这四条路线本质都在权衡同一枚硬币——"是否无损保留全部历史并能精确寻址任一旧 token"。注意力把它做到极致(空间换精度,O(N²));稀疏/线性/SSM 各以不同方式放弃一部分"无损全检索"来换线性复杂度。没有哪条是普适最优,选型取决于你的序列长度、是否需要精确长程检索、推理显存预算。**

这一章在整门课里的位置:它紧接 MLSys 的 FlashAttention(那是"精确 O(N²) 怎么更省地算"),把视角推进到"能不能根本不算 O(N²)"。**理解了"固定状态 vs 全量 KV"这条主线,你就握住了长上下文架构的核心张力**——后续讲超长上下文、检索增强、混合架构(注意力+SSM)时,都会回到这枚硬币。从这里再往前,我们会进入**位置编码与长度外推**(RoPE / ALiBi:让注意力本身能泛化到比训练更长的序列)——那是另一条让 Transformer 吃长序列的、与本章正交的关键拼图。
