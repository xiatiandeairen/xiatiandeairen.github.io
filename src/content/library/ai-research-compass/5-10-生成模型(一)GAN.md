> 这一章把你从"知道 GAN 是俩网络互相对抗生成图片"带到"能在白板上推出最优判别器 D\*、把 GAN 目标化简成 JS 散度、从第一性原理讲清模式崩溃和梯度消失为什么必然发生,再用 Wasserstein 距离精确地说明 WGAN 为什么修好了梯度、并能照着实现 WGAN-GP 的梯度惩罚"。读完你对 GAN 的训练动力学不应有任何黑箱:它不是"调参玄学",而是一个**目标函数的几何性质决定的、可被精确分析的优化问题**。

## 一句话引言

GAN 的本质是一个**双人零和博弈**:生成器 G 想让自己造的假样本分布 p_g 骗过判别器,判别器 D 想精确区分真假。这一章从博弈的 minimax 目标讲起,先证明"固定 G 时最优判别器有闭式解",再把这个闭式解代回去证明**GAN 在数学上等价于最小化 p_data 与 p_g 之间的 Jensen-Shannon 散度**——这一步既解释了 GAN"在学分布匹配",又精确暴露了它的死穴:**JS 散度在两个分布不重叠时是常数,梯度恒为 0**。WGAN 的全部贡献就是把这个"几何上失效的距离"换成一个"几何上良性的距离"(Wasserstein 距离),代价是要求判别器满足 1-Lipschitz 约束,而这个约束如何工程化实现(weight clipping → 梯度惩罚)正是 WGAN 系列的演进主线。

## 1. 动机:为什么需要"对抗"这种训练范式

### 1.1 生成模型的根本任务与"似然"这条老路

生成建模的目标朴素得不能再朴素:**给定一堆样本(比如几万张人脸),学一个能产生"看起来像是同分布"的新样本的模型**。形式化:数据来自某个我们不知道的真实分布 p_data(x),x ∈ ℝᴰ(D 是像素维度,一张 256×256×3 的图就是 D ≈ 20 万维)。我们想得到一个可采样的模型分布 p_g(x),使 p_g ≈ p_data。

传统做法是**显式建模似然(likelihood)**:参数化一个 p_θ(x),然后最大化训练集上的对数似然 Σᵢ log p_θ(xᵢ)。这条路有两个分支:

- **自回归(autoregressive)**:把联合分布按链式法则拆成 `p(x) = ∏ₜ p(xₜ | x_{<t})`,逐像素/逐 token 建模(PixelRNN、PixelCNN)。能算精确似然,但生成要逐元素串行,256×256 的图要采 20 万步,慢得离谱。
- **变分(VAE)**:引入隐变量 z,建模 `p(x) = ∫ p(x|z)p(z) dz`,但这个积分算不动,只能优化它的下界(ELBO)。VAE 能训稳、能算似然下界,但生成的图像**普遍偏糊**——后面 11、12 章会讲它的根因(像素级重建损失 + 后验坍缩)。

这里有一个贯穿生成模型的**核心张力**:

> 要算似然 p_θ(x),你就得对 x 的密度有显式表达式;但一个"能直接采样、表达力强"的生成器(比如把高斯噪声经过一个深 CNN 变成图)的输出密度 p_g(x) **根本没有闭式表达式**,因为它是一个高度非线性变换下的 pushforward 分布,算它需要对网络求逆 + 雅可比行列式,实际不可行。

### 1.2 GAN 的破局点:不算似然,改成"分布比较"

Goodfellow et al.(2014,NeurIPS)的洞见是:**我们其实不需要 p_g(x) 的解析式,只要能从 p_g 采样,再有一个办法判断"p_g 采的样本和 p_data 采的样本像不像",就能优化 p_g。**

这个"判断像不像"的办法,他们用一个**可学习的判别器(discriminator)** D 来充当。于是生成模型问题被改写成一个博弈:

- **生成器 G**:一个确定性映射 `G: ℝ^{d_z} → ℝᴰ`,把简单噪声 z ~ p_z(通常是标准高斯或均匀分布)映射成"假"样本 G(z)。G(z) 的分布记为 p_g。G 不需要密度表达式,只需要能采样——喂个 z 进去出来一张图,天经地义。
- **判别器 D**:一个映射 `D: ℝᴰ → [0,1]`,输出"输入 x 是真样本的概率"。D(x) → 1 表示判为真,D(x) → 0 表示判为假。

**这就是 GAN 范式的全部立足点:把"无法计算的似然最大化"替换成"可计算的、由神经网络近似的分布距离最小化"。** 后面整章的数学,都是在精确回答两个问题:(1) 这个博弈的最优解到底对应什么分布距离?(2) 这个距离的几何性质如何决定训练好不好训?

> 直觉类比:VAE/自回归是"画完之后用尺子量自己画得准不准"(需要显式的度量);GAN 是"画完之后请一个会进步的鉴赏家来挑刺,你照着鉴赏家的反馈改"。鉴赏家(D)越强,逼着画家(G)越接近真实。

## 2. 博弈目标:minimax 与价值函数

### 2.1 价值函数的来历(从交叉熵推起,不要直接背)

D 的任务是一个**二分类**:真样本(来自 p_data)标签为 1,假样本(来自 p_g)标签为 0。对一个二分类器,标准损失是**二元交叉熵(BCE)**。设真样本占一半、假样本占一半(GAN 训练时每个 batch 一半真一半假),D 要最大化的就是"正确分类的对数似然":

```
对真样本 x ~ p_data,标签 1: 贡献 log D(x)         (D(x) 越接近 1 越好)
对假样本 G(z), z ~ p_z,标签 0: 贡献 log(1 − D(G(z)))  (D(G(z)) 越接近 0 越好)
```

把两部分按各自分布取期望并相加,得到 D 要**最大化**的目标:

```
V(D, G) = E_{x~p_data}[log D(x)]  +  E_{z~p_z}[log(1 − D(G(z)))]
```

这正是 GAN 论文里的**价值函数(value function)** V(D, G)。注意它就是负的 BCE 损失(在等量真假样本的设定下),没有任何神秘之处——**D 最大化 V 等价于最小化它作为分类器的交叉熵**。

而 G 的目标恰好相反:G 希望它造的假样本让 D 认不出来,即让 `log(1 − D(G(z)))` 这一项变小(D(G(z)) → 1)。注意第一项 `E[log D(x)]` 完全不含 G,对 G 是常数。所以 **G 最小化 V**。

两者目标完全对立,组合成一个 **minimax(极小极大)博弈**:

```
min_G  max_D  V(D, G)  =  E_{x~p_data}[log D(x)] + E_{z~p_z}[log(1 − D(G(z)))]
```

**读这个式子的正确姿势**:它是一个二人零和博弈的鞍点(saddle point)问题。`max_D` 在里层:对当前固定的 G,先把 D 训到最优(最会挑刺的鉴赏家);`min_G` 在外层:再调 G 去对抗这个最优的 D。理论分析按这个顺序(先内层后外层)走,但工程实现里我们不可能每步把 D 训到收敛——这个理论与实践的裂缝,正是训练不稳定的来源之一(§4)。

### 2.2 把 G(z) 的期望换成对 p_g 的期望

为后面推导方便,做一个关键的记号转换。`G(z)` 当 z ~ p_z 时服从分布 p_g,所以对噪声 z 的期望可以等价写成对生成分布 p_g 的期望(这是 **LOTUS,无意识统计学家定律**:E_{z}[f(G(z))] = E_{x~p_g}[f(x)]):

```
V(D, G) = E_{x~p_data}[log D(x)] + E_{x~p_g}[log(1 − D(x))]
        = ∫ p_data(x) log D(x) dx  +  ∫ p_g(x) log(1 − D(x)) dx
        = ∫ [ p_data(x) log D(x) + p_g(x) log(1 − D(x)) ] dx
```

最后一步把两个积分合并到同一个 dx 下——**这是整章最关键的一步代数操作**,它让我们可以"逐点(对每个 x)" 地优化被积函数,从而求出最优 D。记住这个合并技巧。

## 3. 核心推导(一):固定 G,最优判别器 D\* 的闭式解

### 3.1 逐点最优化

固定 G(因而 p_g 固定),我们要对 D 最大化 V。注意被积函数

```
f(D) = p_data(x) · log D  +  p_g(x) · log(1 − D)
```

对每一个 x 是**互相独立的**(D(x) 在不同 x 处取值彼此无约束,假设 D 是任意可测函数,不受网络容量限制)。因此最大化整个积分,等价于**对每个 x 独立地最大化 f(D)**。这是个一元微积分问题。

记 a = p_data(x),b = p_g(x)(对固定 x 是两个非负常数),最大化 `g(y) = a log y + b log(1 − y)`,y ∈ (0,1):

```
g'(y) = a/y − b/(1 − y)
令 g'(y) = 0:
        a/y = b/(1 − y)
        a(1 − y) = b·y
        a − a·y = b·y
        a = (a + b)·y
        y* = a / (a + b)
```

验证是极大:`g''(y) = −a/y² − b/(1−y)² < 0`(a, b ≥ 0,不同时为 0),故 g 严格凹,y\* 是唯一全局最大。代回 a、b:

```
┌─────────────────────────────────────────────────┐
│  D*(x) = p_data(x) / ( p_data(x) + p_g(x) )       │   (最优判别器)
└─────────────────────────────────────────────────┘
```

**这个结果极其漂亮,值得逐条品**:

- **它有清晰的概率含义**:D\*(x) 正是"在等先验下,样本 x 来自真实分布的后验概率" P(real | x)。这正是任何二分类器在数据无限、容量无限时应当逼近的贝叶斯最优解。
- **三个特殊点**:若某处只有真数据(p_g = 0),D\* = 1(笃定为真);只有假数据(p_data = 0),D\* = 0;**两分布完全重合(p_data = p_g)处,D\* = 1/2**——鉴赏家彻底分不清真假,只能瞎猜。这给了我们一个判断收敛的"温度计":**理想收敛时 D 在所有样本上输出 1/2**。
- **它是"最理想"的 D**,实际网络只能逼近。但理论分析就用这个理想 D\* 代回去,看 G 在面对一个完美鉴赏家时被逼向哪里。

## 4. 核心推导(二):代回 D\*,GAN 在最小化 JS 散度

### 4.1 推出 −log4 + 2·JSD

把 D\* 代回 V(D, G),得到一个只关于 G(只关于 p_g)的"虚拟损失",记为 C(G) = V(D\*, G):

```
C(G) = E_{x~p_data}[ log  p_data/(p_data + p_g) ]
     + E_{x~p_g}  [ log  p_g    /(p_data + p_g) ]
```

直接看不出是什么。**配凑技巧**:在每个 log 里凑一个 1/2,把分母从 (p_data + p_g) 变成 (p_data + p_g)/2,这样比值就变成"分布除以两者的平均分布"——这正是 KL 散度需要的形式。逐步推:

```
第一项 = E_{x~p_data}[ log  p_data / (p_data + p_g) ]
       = E_{x~p_data}[ log  ( p_data / ((p_data+p_g)/2) ) · (1/2) ]
       = E_{x~p_data}[ log  p_data / ((p_data+p_g)/2) ]  +  E_{x~p_data}[ log (1/2) ]
       = KL( p_data ‖ (p_data+p_g)/2 )  −  log 2
```

这里用了 KL 散度的定义 `KL(P‖Q) = E_{x~P}[log P(x)/Q(x)] = ∫ P log(P/Q)`。同理第二项:

```
第二项 = E_{x~p_g}[ log  p_g / (p_data + p_g) ]
       = KL( p_g ‖ (p_data+p_g)/2 )  −  log 2
```

两项相加:

```
C(G) = KL(p_data ‖ M) + KL(p_g ‖ M) − 2 log 2,   其中 M = (p_data + p_g)/2
```

而 **Jensen-Shannon 散度(JSD)** 的定义恰好是:

```
JSD(p_data ‖ p_g) = ½ KL(p_data ‖ M) + ½ KL(p_g ‖ M),   M = (p_data+p_g)/2
```

所以 `KL(p_data‖M) + KL(p_g‖M) = 2·JSD(p_data‖p_g)`。代入,且 2 log 2 = log 4:

```
┌──────────────────────────────────────────────────────┐
│   C(G) = −log 4 + 2 · JSD( p_data ‖ p_g )              │
└──────────────────────────────────────────────────────┘
```

### 4.2 全局最优:p_g = p_data

JS 散度有两条已知性质(都可从 KL ≥ 0 直接得到):

- **非负**:JSD ≥ 0(因为它是两个 KL 的非负加权和,而 KL ≥ 0 由 Jensen 不等式给出)。
- **当且仅当两分布相等时为 0**:JSD(P‖Q) = 0 ⟺ P = Q(几乎处处)。

因此 C(G) ≥ −log 4,**且等号当且仅当 p_g = p_data 时取到**。这就是 GAN 的**全局最优定理**:

> 当判别器达到最优、且生成器使 C(G) 最小化时,生成分布精确等于真实分布 p_g = p_data,此时 C(G) = −log4,且 D\*(x) ≡ 1/2。

**这一节的产出是整个 GAN 理论的灵魂**:它把一个"两个网络互相打架"的工程直觉,变成了一句精确的数学陈述——**GAN(在最优 D 假设下)是在以 JS 散度为度量,最小化生成分布与真实分布的距离**。GAN 不是"骗过分类器"那么玄,它在做**分布匹配(distribution matching)**。这也是为什么 GAN 能生成多样、清晰的样本:它直接逼近整个分布,而不像 VAE 那样优化逐像素重建误差。

但是——**这个漂亮结论里藏着一颗定时炸弹,炸弹的引信就是"JS 散度"本身的几何缺陷**。下一节引爆它。

## 5. 训练病理:为什么 GAN 这么难训(从 JS 的几何缺陷推根因)

GAN 出了名地难训,常见症状有三个:**训练不稳定/不收敛、模式崩溃(mode collapse)、梯度消失(vanishing gradient)**。新手往往以为是"调参不到位",但 §4 的推导让我们能从根上解释其中两个。

### 5.1 梯度消失:JS 在不重叠时是常数

**关键事实:真实图像分布 p_data 集中在高维像素空间里一个极低维的"流形(manifold)"上。** 直觉:自然图像只占 256×256×3 ≈ 20 万维空间里微不足道的一小块(随机采像素几乎永远是雪花噪点,不是图)。同理,G(z) 把一个 d_z 维(比如 128 维)的噪声映射进 20 万维空间,p_g 也支撑在一个**至多 128 维的流形**上。

**两个低维流形在高维空间里"几乎必然不相交"**(或交集是零测集)。这件事的后果是致命的——考虑一个一维卡通例子:

```
设 p_data 是 x=0 处的点质量(支撑在 {0}),
   p_g    是 x=θ 处的点质量(支撑在 {θ}),θ ≠ 0。
两者支撑不重叠。计算 JSD:
   在 x=0:  M(0)=½,p_data(0)=1, → KL(p_data‖M) 贡献 1·log(1/½)=log2
   在 x=θ:  M(θ)=½,p_g(θ)=1,   → KL(p_g‖M)    贡献 1·log(1/½)=log2
   JSD = ½(log2) + ½(log2) = log2   (常数!与 θ 多大无关)
```

**只要 θ ≠ 0,无论 θ = 0.001 还是 θ = 1000,JSD 恒等于 log2。** 这意味着:

```
∇_θ JSD = 0      (当支撑不重叠时,JS 散度关于 G 参数的梯度恒为 0)
```

而 G 是靠 ∇_G C(G) = 2∇_G JSD 来更新的。**梯度恒为 0 ⟹ G 收不到任何"往哪个方向挪能更接近 p_data"的信号。** 翻译成训练现象:

> **当 D 训得太好(逼近 D\*)、而此刻 p_g 与 p_data 还不重叠时,D 能完美区分真假(在真样本上输出 ≈1,假样本上 ≈0),G 的损失饱和、梯度趋近 0,G 学不动了。** 这就是"判别器太强反而训不动生成器"的精确机理。

Arjovsky & Bottou(2017)在《Towards Principled Methods...》里把这件事严格化:在合理假设下,当 D 接近最优,传给 G 的梯度范数趋于 0(原始损失)或方差爆炸/方向乱跳(下面的 −log D 替代损失)。**这不是 bug,是 JS 散度这个度量的几何性质决定的必然。**

### 5.2 非饱和损失:工程上的第一个补丁(及其代价)

Goodfellow 原论文已经预见到梯度消失,给了一个工程补丁:**别让 G 最小化 `log(1 − D(G(z)))`,改成最大化 `log D(G(z))`**(等价于最小化 `−log D(G(z))`)。这叫**非饱和损失(non-saturating loss)**。

为什么?看两个损失对 D(G(z)) 的梯度行为(记 d = D(G(z)) ∈ (0,1)):

```
原始(minimax)损失对 G:  ℓ = log(1 − d)
   d 很小(G 很烂、刚开始训)时,dℓ/dd = −1/(1−d) ≈ −1,梯度极小 → 学不动
非饱和损失对 G:           ℓ' = −log d
   d 很小时,dℓ'/dd = −1/d → −∞,梯度极大 → 训得动
```

**直觉**:G 最烂的时候(被 D 一眼识破,d≈0)恰恰最需要大梯度去快速改进,原始损失却在这时给最小梯度(饱和),非饱和损失反过来给最大梯度。所以**实际中几乎所有原始 GAN 用的都是非饱和损失**。

**代价/坑**:非饱和损失虽然缓解了梯度消失,但它对应的不再是干净的 JSD 最小化。Arjovsky 证明它隐式优化的目标含有一项 `KL(p_g‖p_data) − 2·JSD`(符号反常),其中 `KL(p_g‖p_data)` 这一项的不对称性会**鼓励模式崩溃**:KL(p_g‖p_data) 对"p_g 在 p_data ≈ 0 处还有质量"惩罚极重(分母趋 0,log 趋 +∞),但对"p_data 有质量而 p_g 漏掉了某些众数(mode)"几乎不惩罚。结果 G 学会**宁可只覆盖少数几个最有把握的众数、绝不冒险**——这正是模式崩溃的诱因之一。

### 5.3 模式崩溃:博弈的非合作本质

**模式崩溃**:G 只生成少数几种(甚至一种)样本——比如训练人脸数据,G 反复生成同一张脸,或只生成几种发型。它们都能骗过当前的 D,所以 G 的损失看着不高,但**多样性彻底丧失**。

除了 §5.2 的 KL 不对称诱因,更根本的机理来自**博弈的动力学**:

- 理论分析假设"先把 D 训到最优(内层 max),再更新 G"。但工程上我们交替更新(D 走一步,G 走一步),**G 看到的永远是一个滞后、不完美的 D**。
- G 找到一个能骗过**当前** D 的众数后,会一头扎进去(把所有质量塌到那个众数)。D 随后学会识破这个众数,G 就整体平移到下一个能骗过 D 的众数。**G 和 D 陷入"打地鼠"式的循环追逐,而不是收敛到覆盖全部众数的均衡。** 这是 minimax 鞍点用一阶交替梯度法去找时的典型病态——梯度场存在旋转分量(rotational dynamics),容易绕圈不收敛。

**一句话总结三大病的根**:模式崩溃 + 梯度消失这两大病,深层都指向同一件事——**JS 散度(及其变体)在"两个分布几乎不重叠"这个高维生成的常态下,要么给不出梯度(消失),要么给出鼓励塌缩的扭曲梯度(崩溃)。修复必须从"换一个几何性质良好的分布距离"下手。** 这就把舞台交给了 Wasserstein 距离。

## 6. WGAN:用 Wasserstein 距离换掉 JS

### 6.1 推土机距离的定义与直觉

**Wasserstein-1 距离(又叫 Earth-Mover / 推土机距离,记 W₁)** 衡量"把一堆土(分布 P)搬成另一堆土(分布 Q)所需的最小搬运代价",代价 = Σ(每份土的质量 × 搬运距离):

```
W₁(P, Q) = inf_{γ ∈ Π(P,Q)}  E_{(x,y)~γ}[ ‖x − y‖ ]
```

- `Π(P,Q)` 是所有**联合分布(运输方案)** γ(x,y) 的集合,其两个边缘分布分别是 P 和 Q(即 ∫γ(x,y)dy = P(x),∫γ(x,y)dx = Q(y))。一个 γ 就是一套"从 x 处搬多少土到 y 处"的搬运计划。
- `inf`(下确界)表示取**最优搬运方案**的代价。

**为什么它比 JS 强?回到 §5.1 的卡通例子**(p_data 在 0,p_g 在 θ):

```
唯一的搬运方案就是把 θ 处的全部质量搬到 0,距离 |θ|。
W₁(p_data, p_g) = |θ|        ← 随 θ 连续变化,处处可导,∂W₁/∂θ = sign(θ)
对比 JSD = log2(常数),∇=0。
```

**这就是 WGAN 的全部魔力来源**:即使两个分布支撑完全不重叠,W₁ 仍然**随分布参数连续、可导地变化**,并指出"该往哪个方向挪、挪多远"。它给的梯度始终有用,不会饱和。Arjovsky, Chintala & Bottou(2017,WGAN)正是用这个例子论证 W₁ 在弱拓扑下比 JS/KL 更"连续"。

> 一句话:JS 散度只看"两堆土重不重叠"(重叠=0,不重叠=常数,像个台阶函数),Wasserstein 看"两堆土差多远"(像个斜坡)。斜坡处处有坡度可以下,台阶在平台上无坡可下。

### 6.2 Kantorovich-Rubinstein 对偶:把 inf 变成可优化的 sup

上面的 W₁ 定义含一个对**所有运输方案求下确界**的优化,在高维上完全没法直接算。**Kantorovich-Rubinstein 对偶定理**把它等价转化为一个对函数求上确界的形式:

```
┌──────────────────────────────────────────────────────────┐
│  W₁(P, Q) = sup_{ ‖f‖_L ≤ 1 }  E_{x~P}[f(x)] − E_{x~Q}[f(x)] │
└──────────────────────────────────────────────────────────┘
```

其中 sup 取遍所有满足 **1-Lipschitz 约束** 的函数 f。**1-Lipschitz** 指 `‖f‖_L ≤ 1`,即对任意 x₁, x₂ 有 `|f(x₁) − f(x₂)| ≤ ‖x₁ − x₂‖`(函数变化速度不超过 1,梯度范数处处 ≤ 1)。

(此对偶的完整证明属于最优传输理论,涉及线性规划对偶,这里**不展开证明,直接使用**;读者只需接受这个等价。它的直觉:那个最会"放大 P 与 Q 之间差异"、但又不能变化太快(Lipschitz 约束防止它在重叠处也胡乱拉开)的函数 f,其拉开的最大间隙就等于搬运代价。)

这个对偶形式简直是为 GAN 量身定做的:把 f 用一个神经网络参数化(记 f_w),它扮演的角色和判别器极像,但 WGAN 论文管它叫 **critic(评论家)** 而非 discriminator——因为它**不再输出 [0,1] 的概率,而是输出一个实数打分**,且没有 sigmoid、没有 log。于是 WGAN 的目标变成:

```
min_G  max_{w: f_w 是1-Lipschitz}  E_{x~p_data}[ f_w(x) ]  −  E_{z~p_z}[ f_w(G(z)) ]
```

- **critic 的训练**(内层 max):最大化 `E[f_w(真)] − E[f_w(假)]`,即让 critic 给真样本打高分、假样本打低分,两者均值之差就近似 W₁。
- **G 的训练**(外层 min):固定 critic,最小化 `−E_z[f_w(G(z))]`,即让 G 造的样本被 critic 打高分(往真样本那边挪)。G 的损失梯度是 `−E_z[∇_G f_w(G(z))]`,由于 f_w 处处梯度有界且非零(Lipschitz 但非常数),**这个梯度始终有用**——梯度消失问题从根上消失。

**WGAN 相对原始 GAN 的几个实操改动(都源于上面的数学)**:

1. critic 最后一层去掉 sigmoid(输出实数打分,不是概率)。
2. 损失里没有 log(直接是打分的差)。
3. critic 要训得比 G 多(每次更新 G 前,critic 更新 n_critic 步,典型 5 步),因为内层 max 要尽量逼近真 W₁,W₁ 估计越准,给 G 的梯度方向越对——这点和原始 GAN "别把 D 训太强"恰好相反(原始 GAN 里 D 太强会梯度消失,WGAN 里 critic 越准越好)。

### 6.3 如何强加 1-Lipschitz 约束:weight clipping → 梯度惩罚

对偶式只有在 f_w **确实满足 1-Lipschitz** 时才等于 W₁。怎么逼一个神经网络满足这个约束?这是 WGAN 工程化的核心难点,有两代方案。

**方案一:权重裁剪(weight clipping,原始 WGAN)**。每次更新后,把 critic 的所有权重 clamp 到 `[−c, c]`(c 是小常数,如 0.01)。直觉:网络是若干线性层 + 1-Lipschitz 激活(ReLU/LeakyReLU 是 1-Lipschitz)的复合,限制每层权重的范数,就限制了整体的 Lipschitz 常数(到某个 K,对偶式算出的就是 K·W₁,差个常数因子不影响优化方向)。

**weight clipping 的坑(WGAN 原作者自己承认的)**:

- **容量浪费 / 病态权重分布**:clip 把权重逼到 ±c 两个极端值附近(像被两堵墙夹住),critic 退化成一个"几乎二值权重"的简单函数,表达力严重受限,难以拟合复杂的最优 f。
- **梯度爆炸/消失对 c 极度敏感**:c 大了梯度容易爆,c 小了梯度容易消,几乎没有好用的 c,调参苦不堪言。

**方案二:梯度惩罚(gradient penalty,WGAN-GP)**。Gulrajani et al.(2017)的关键观察:**1-Lipschitz 等价于"f 的梯度范数处处 ≤ 1";而对偶的最优 f\*,在最优运输路径上梯度范数恰好等于 1(几乎处处)。** 与其硬裁权重,不如**软性地惩罚 critic 的输入梯度范数偏离 1**:

```
GP = λ · E_{x̂ ~ p_x̂} [ ( ‖∇_x̂ f_w(x̂)‖₂ − 1 )² ]
```

- **惩罚目标是 (‖∇‖ − 1)² 而非 max(0, ‖∇‖−1)²**:为什么是双边惩罚到恰好 = 1,而不是只罚 >1?因为最优 critic 的梯度范数本就**等于** 1(不是 ≤1),双边惩罚把 critic 往最优解推。这是一个**重要而反直觉的设计点**,新手常误写成单边。
- **x̂ 在哪里采?** 在真假样本对之间**随机插值**:`x̂ = ε·x_real + (1−ε)·x_fake`,ε ~ Uniform[0,1]。理由:理论上要在全空间罚梯度做不到,而最优运输路径恰好落在真假样本的连线上,所以在插值点上罚是对最优 f\* 所在区域的有效采样。
- **λ 典型取 10**(WGAN-GP 论文默认值)。
- **WGAN-GP 不再用 BatchNorm**(改用 LayerNorm 或不用),因为 BN 把整个 batch 的样本耦合,使"逐样本的梯度范数"这个量定义不清(梯度惩罚是 per-sample 的)。这是一个实现时极易踩的坑。

完整的 WGAN-GP critic 损失:

```
L_critic = E_{x~p_g}[ f_w(x) ] − E_{x~p_data}[ f_w(x) ] + λ·E_{x̂}[ (‖∇_x̂ f_w(x̂)‖₂ − 1)² ]
                └──── 注意符号:这里写成"假−真",critic 做最小化等价于前面"真−假"做最大化 ────┘
```

## 7. 代码:从原始 GAN 训练循环到 WGAN-GP 梯度惩罚

下面给可照着实现的 PyTorch 核心片段。**只写体现机制的部分**,网络结构(DCGAN 风格的转置卷积/卷积)用占位省略。

### 7.1 原始 GAN(非饱和损失)训练循环

```python
import torch, torch.nn as nn

bce = nn.BCEWithLogitsLoss()  # 注意:内部含 sigmoid+log,数值更稳;故 D 输出 logits 不加 sigmoid

def train_gan_step(G, D, real, opt_g, opt_d, z_dim, device):
    B = real.size(0)
    z = torch.randn(B, z_dim, device=device)
    real_label = torch.ones (B, 1, device=device)
    fake_label = torch.zeros(B, 1, device=device)

    # ---- (1) 训练 D:最大化 log D(x) + log(1 − D(G(z))) ----
    fake = G(z).detach()                       # detach:更新 D 时不让梯度流回 G
    loss_d_real = bce(D(real), real_label)     # 真样本判为 1
    loss_d_fake = bce(D(fake), fake_label)     # 假样本判为 0
    loss_d = loss_d_real + loss_d_fake
    opt_d.zero_grad(); loss_d.backward(); opt_d.step()

    # ---- (2) 训练 G:非饱和损失,最大化 log D(G(z)) ⟺ 最小化 BCE(D(G(z)), 1) ----
    z = torch.randn(B, z_dim, device=device)
    fake = G(z)                                # 这里不 detach,梯度要流进 G
    loss_g = bce(D(fake), real_label)          # 关键:用 real_label(1),即 −log D(G(z))
    opt_g.zero_grad(); loss_g.backward(); opt_g.step()
    return loss_d.item(), loss_g.item()
```

**易踩坑标注**:(1) 更新 D 时对 `fake` 一定要 `.detach()`,否则 D 的梯度会误更新 G;(2) G 的损失用 `real_label` 而不是 `1 − fake_label`,这才是非饱和损失 −log D(G(z));(3) 用 `BCEWithLogitsLoss` + D 输出 logits,比手写 `log(sigmoid(.))` 数值稳定得多。

### 7.2 WGAN-GP:梯度惩罚的实现(核心)

```python
def gradient_penalty(critic, real, fake, device):
    B = real.size(0)
    # 1) 在真假样本之间逐样本随机插值:x̂ = ε·real + (1−ε)·fake
    eps = torch.rand(B, 1, 1, 1, device=device)        # 每个样本一个 ε(图像是 4D 张量)
    x_hat = (eps * real + (1 - eps) * fake).requires_grad_(True)

    # 2) critic 对插值点打分
    score = critic(x_hat)                              # shape [B, 1]

    # 3) 求 score 对输入 x̂ 的梯度(create_graph=True 以便对该梯度二次求导)
    grad = torch.autograd.grad(
        outputs=score, inputs=x_hat,
        grad_outputs=torch.ones_like(score),
        create_graph=True, retain_graph=True,
    )[0]                                               # ∇_x̂ f(x̂),shape 同 x̂

    # 4) 逐样本的梯度 L2 范数,再惩罚其偏离 1(双边!)
    grad = grad.view(B, -1)
    grad_norm = grad.norm(2, dim=1)                    # ‖∇‖₂ per sample
    gp = ((grad_norm - 1.0) ** 2).mean()               # (‖∇‖−1)²,不是 max(0,·)
    return gp


def train_wgan_gp_step(G, C, real, opt_g, opt_c, z_dim, device,
                       n_critic=5, lam=10.0):
    B = real.size(0)
    # ---- (1) 训练 critic n_critic 步:max  E[C(real)] − E[C(fake)] − λ·GP ----
    for _ in range(n_critic):
        z = torch.randn(B, z_dim, device=device)
        fake = G(z).detach()
        # Wasserstein 估计:critic 给真打高分、假打低分;最小化"假−真"= 最大化"真−假"
        loss_c = C(fake).mean() - C(real).mean() + lam * gradient_penalty(C, real, fake, device)
        opt_c.zero_grad(); loss_c.backward(); opt_c.step()
        # 注意:WGAN-GP 不需要 weight clipping(GP 已经管住了 Lipschitz)

    # ---- (2) 训练 G 一步:min −E[C(G(z))],即让假样本被打高分 ----
    z = torch.randn(B, z_dim, device=device)
    loss_g = -C(G(z)).mean()
    opt_g.zero_grad(); loss_g.backward(); opt_g.step()
    return loss_c.item(), loss_g.item()
```

**实现要点回顾**:`create_graph=True` 是关键——梯度惩罚里 `gp` 本身是 critic 参数的函数,要对它再反向传播一次(二阶导),所以构建梯度的计算图必须保留;critic 损失里**没有 log、没有 sigmoid**;`n_critic=5`、`lam=10` 是 WGAN-GP 论文默认值;**优化器用 Adam 时 WGAN-GP 论文建议 β₁=0、β₂=0.9**(原始 WGAN 用 RMSProp,因为权重裁剪和动量配合不好;GP 版本可以用 Adam,但建议低 β₁)。

## 8. 量化分析:参数量、显存、复杂度

### 8.1 训练成本的结构

GAN 训练每个 step 的计算量,记 C_G、C_D 分别为 G、D 的单次前向 FLOPs:

```
原始 GAN 每 step:  ~ (前向+反向) × (G 一次 + D 两次≈D(real)、D(fake))
                   ≈ 3·(C_G + 2·C_D)    (前向 C_G+2C_D,反向≈2×前向)
WGAN-GP 每 step:   critic 走 n_critic 步,每步含一次额外的"对输入求梯度+二阶反传"
                   ≈ n_critic·(C_D 的前向反向 + GP 的二阶开销) + (C_G + C_D)
```

**两个量化结论**:

- **WGAN-GP 显著更贵**:n_critic=5 意味着 critic 的计算约为 G 的 5 倍;梯度惩罚的 `create_graph=True` 二阶反传,**显存和计算各再增加约一倍 critic 的开销**(要存高阶计算图)。粗算 WGAN-GP 比原始 GAN 单 step 慢约 **3–6×**(取决于 n_critic 和 GP 占比)。这是用计算换稳定性。
- **梯度惩罚的显存峰值**:二阶导要保留 critic 前向的中间激活两次(一次算 score,一次算 ∇score 的图),**插值 batch 的激活显存约翻倍**。OOM 时优先降 batch 或 critic 宽度。

### 8.2 生成器参数量与感受野(以 DCGAN 风格为例)

DCGAN 的 G 是一串**转置卷积(transposed conv,又叫 fractionally-strided conv)** 上采样:从 1×1 的噪声逐步放大到目标分辨率。每层把空间尺寸翻倍。转置卷积输出尺寸:

```
H_out = (H_in − 1)·S − 2P + K            (转置卷积,与普通卷积的 (H−K+2P)/S+1 互为逆)
例:H_in=4, S=2, P=1, K=4 → H_out=(4−1)·2−2+4=8   (空间翻倍)
```

单个卷积/转置卷积层参数量(忽略 bias):`K·K·C_in·C_out`。一个把 100 维噪声生成 64×64×3 图的典型 DCGAN-G(通道数 1024→512→256→128→3,5 个转置卷积),量级在**几百万参数**(待核具体数值,取决于通道配置)。

**一个对生成模型很重要的量化直觉——转置卷积的"棋盘伪影(checkerboard artifact)"**:当 K 不能被 S 整除时(如 K=3, S=2),转置卷积的输出会出现**周期性的明暗格子**,因为不同输出位置被卷积核覆盖的次数不均匀(有的位置被加了 2 次核,有的 1 次)。Odena et al.(2016,Distill)指出修法:用 K 能被 S 整除(如 K=4,S=2),或改用"最近邻/双线性上采样 + 普通卷积"替代转置卷积。**这是 GAN 生成图里最常见的可视瑕疵之一,看到规则格子先怀疑这里。**

## 9. StyleGAN:风格注入与渐进生成(机制简述)

WGAN-GP 解决了"训得稳不稳",但要生成**高分辨率、高保真、可控**的图,还需要架构创新。StyleGAN 系列(Karras et al., 2018 起)是 GAN 图像质量的巅峰,这里只讲它的两个核心思想(细节留给后续阅读)。

**思想一:渐进式生成(progressive growing,源自 ProGAN, 2017)**。不是一上来就训 1024×1024,而是**从 4×4 开始训,稳定后逐步加层把分辨率翻倍**(4→8→16→…→1024),新层用淡入(fade-in)平滑接入。直觉:低分辨率先学好整体结构(大轮廓),高分辨率层只需补细节,把一个极难的高维匹配问题**分解成一串由易到难的子问题**,极大稳定了训练。

**思想二:风格注入(style-based generator)**。StyleGAN 不把噪声 z 直接喂进卷积栈,而是:

```
z ──(8 层 MLP,映射网络)──▶ w  (中间隐空间 W,更"解耦")
w ──(每层 affine)──▶ (yₛ, y_b)  每层一组缩放/偏置
然后用 AdaIN(自适应实例归一化)把 w 的"风格"注入每个卷积层的特征:
   AdaIN(featᵢ, y) = yₛ,ᵢ · (featᵢ − μ(featᵢ))/σ(featᵢ) + y_b,ᵢ
```

`AdaIN` 先把特征图逐通道标准化(去掉原有风格),再用从 w 算出的 (yₛ, y_b) 重新注入新风格。**关键收益**:不同层注入的 w 控制不同尺度的属性(粗层管姿态/脸型,细层管发色/雀斑),实现了**属性解耦与可控编辑**(改某层 w 只动对应尺度的属性);中间隐空间 W 比原始高斯 Z 更线性、更解耦,这是 StyleGAN 能做语义插值/编辑的基础。(StyleGAN2 进一步用 weight demodulation 替代 AdaIN 以消除水滴伪影——待核细节,留给阅读。)

## 10. 设计权衡与常见坑(汇总)

- **原始 GAN vs WGAN-GP**:原始 GAN 更轻更快,但训练不稳、需小心平衡 D/G 强度("D 别太强");WGAN-GP 稳得多、损失值有意义(loss 约等于 W₁ 估计,可作收敛指标),但**慢 3–6×、显存高**。资源紧、分辨率低优先原始 GAN(用非饱和损失);要稳要高质量上 WGAN-GP 或更现代的损失(hinge loss,SAGAN/BigGAN 用)。
- **"D 该多强"的方向相反,别记混**:原始 GAN 里 D 太强 → JS 饱和 → G 梯度消失,所以要平衡;WGAN 里 critic 越准 → W₁ 估计越准 → G 梯度方向越对,所以 n_critic > 1 主动多训 critic。**这是新手最容易记反的一点。**
- **梯度惩罚的双边性**:必须罚 (‖∇‖−1)² 让范数趋向 **恰好 1**,而非单边 max(0,‖∇‖−1)²。错写成单边会让 critic 偏离对偶最优。
- **WGAN-GP 禁用 BatchNorm**:GP 是 per-sample 的,BN 把 batch 内样本耦合会破坏定义,改用 LayerNorm/InstanceNorm 或不用归一化。
- **棋盘伪影**:转置卷积 K 不被 S 整除会出格子,改 K=4/S=2 或上采样+卷积。
- **GAN 没有干净的"loss 下降=变好"指标**(尤其原始 GAN):D loss、G loss 此消彼长,看 loss 几乎判断不了质量。**实际靠 FID(Fréchet Inception Distance)等样本质量/多样性指标评估**(下一章或评测章详述),以及肉眼看样本。WGAN 的 critic loss 是个例外的、相对有意义的指标。
- **训练崩溃的急救顺序**:先看是不是模式崩溃(样本变单一)→ 试 WGAN-GP / 加噪 / minibatch discrimination;看是不是梯度消失(D 准确率瞬间 100%、G 不动)→ 用非饱和损失 / 降 D 学习率 / 改 WGAN;看是不是数值爆炸 → 降 lr、查 BN/GP 配置。

## 13. 小结与承上启下

这一章的主线是**一个目标函数的几何性质如何决定一类生成模型的命运**:

- GAN 的对抗博弈,在最优判别器假设下**精确等于以 JS 散度做分布匹配**(D\* 闭式解 + JSD 推导,§3–§4)——这把"两个网络打架"的玄学还原成了清晰的优化目标。
- 但 JS 散度在高维生成的常态(两个低维流形几乎不重叠)下**几何性质恶劣**:要么给不出梯度(消失),要么给出鼓励塌缩的扭曲梯度(崩溃)。这是 GAN 难训的根,不是调参不力(§5)。
- **WGAN 用 Wasserstein 距离换掉 JS**,因为后者即使分布不重叠也连续可导地提供梯度;代价是要求 critic 满足 1-Lipschitz,实现上从粗糙的 weight clipping 演进到 WGAN-GP 的梯度惩罚(§6–§7)。StyleGAN 则在架构层面用渐进生成 + 风格注入把质量与可控性推到巅峰(§9)。

**它在整门课里的位置**:GAN 代表生成建模的**隐式密度(implicit density)** 路线——不写出 p_g(x),只学采样器,靠对抗信号逼近分布。它生成快(单次前向出图)、质量高,但训练脆、覆盖度难保证、没有显式似然。下一章我们转向**显式/可处理密度的另一条主线——VAE 与归一化流(Normalizing Flow)**,看它们如何用变分下界或可逆变换把似然算出来,以及为什么它们更稳但更糊。再往后的**扩散模型(Diffusion)** 章会发现:扩散用"逐步去噪 + 分数匹配"绕开了 GAN 的对抗不稳定,又拿回了堪比 GAN 的样本质量,几乎统一了当代图像生成——而理解它的"为什么稳",恰恰要靠你在本章建立的"分布距离的梯度好不好用"这把尺子。**本章建立的'用分布距离的几何性质来诊断训练动力学'这个分析框架,会一直陪你读到扩散模型与流匹配(flow matching)。**

## 练习参考答案

### 练习 1:默写最优判别器 D* 与 JS 散度的完整推导链

#### (a) 推出 D*(x) = p_data/(p_data + p_g),二阶导验证极大

固定 G,把价值函数写成对 x 的单一积分(用 LOTUS 把 E_z 换成 E_{p_g}):

```
V(D,G) = ∫ [ p_data(x)·log D(x) + p_g(x)·log(1 − D(x)) ] dx
```

被积函数对每个 x 互相独立(把 D 当作任意可测函数,不受网络容量限制),所以"最大化整个积分"等价于"对每个 x 逐点最大化被积函数"。记 a = p_data(x)、b = p_g(x)(对固定 x 是两个非负常数),令 y = D(x) ∈ (0,1),求

```
g(y) = a·log y + b·log(1 − y)
```

的极大。求一阶导并令其为 0:

```
g'(y) = a/y − b/(1 − y) = 0
  ⟹ a/y = b/(1 − y)
  ⟹ a(1 − y) = b·y
  ⟹ a − a·y = b·y
  ⟹ a = (a + b)·y
  ⟹ y* = a / (a + b)
```

**二阶导验证是极大**:

```
g''(y) = −a/y² − b/(1 − y)²
```

因 a, b ≥ 0 且不同时为 0,对任意 y ∈ (0,1) 有 g''(y) < 0,故 g 在 (0,1) 上**严格凹**,驻点 y* 是**唯一全局最大**。代回 a = p_data(x)、b = p_g(x):

```
┌──────────────────────────────────────────────┐
│  D*(x) = p_data(x) / ( p_data(x) + p_g(x) )    │
└──────────────────────────────────────────────┘
```

> 数值验算(a=0.7, b=0.3):网格搜索 g(y) 的最大点 = 0.70000,与公式 a/(a+b)=0.7 一致;g''(0.7) = −4.76 < 0,确认极大。

它的概率含义:**D*(x) 就是等先验下"x 来自真实分布"的后验概率 P(real | x)**,即贝叶斯最优分类器。三个特殊点:只有真数据处 D*=1;只有假数据处 D*=0;**两分布重合处 p_data=p_g ⟹ D*=1/2**。

#### (b) 代回 D*,用"凑 1/2 配 KL"推出 C(G) = −log4 + 2·JSD(p_data‖p_g)

把 D* 代回 V,得到只关于 G 的虚拟损失 C(G) = V(D*, G):

```
C(G) = E_{x~p_data}[ log  p_data/(p_data + p_g) ]
     + E_{x~p_g}  [ log  p_g    /(p_data + p_g) ]
```

**关键技巧**:在每个 log 里乘进一个 1/2,把分母凑成"平均分布" M = (p_data + p_g)/2,使比值变成"分布 / 两者平均",这正是 KL 散度需要的形式。逐项推第一项:

```
E_{p_data}[ log  p_data/(p_data+p_g) ]
  = E_{p_data}[ log ( p_data / ((p_data+p_g)/2) ) · (1/2) ]      ← 分子分母同乘 1/2
  = E_{p_data}[ log  p_data / M ]  +  E_{p_data}[ log (1/2) ]
  = KL(p_data ‖ M)  −  log 2                                     ← 用 KL(P‖Q)=E_P[log P/Q]
```

同理第二项:

```
E_{p_g}[ log  p_g/(p_data+p_g) ] = KL(p_g ‖ M) − log 2
```

两项相加:

```
C(G) = KL(p_data ‖ M) + KL(p_g ‖ M) − 2·log 2 ,   M = (p_data + p_g)/2
```

而 JS 散度定义为 JSD(P‖Q) = ½·KL(P‖M) + ½·KL(Q‖M),所以

```
KL(p_data‖M) + KL(p_g‖M) = 2·JSD(p_data‖p_g)
```

又 2·log 2 = log 4,代入:

```
┌──────────────────────────────────────────────────┐
│   C(G) = −log 4 + 2 · JSD( p_data ‖ p_g )          │
└──────────────────────────────────────────────────┘
```

> 数值验算(随机离散分布 P、Q,各 5 个 bin):直接按定义算 C(G) = −1.194456;按公式 −log4 + 2·JSD = −1.194456,完全一致。

#### (c) 全局最优在 p_g = p_data,且此时 D* ≡ 1/2

JS 散度的两条性质(均由 KL ≥ 0 得到,KL ≥ 0 本身由 Jensen 不等式给出):

- **非负**:JSD = ½·KL(p_data‖M) + ½·KL(p_g‖M) ≥ 0(两个非负 KL 的非负加权和)。
- **等号条件**:JSD(P‖Q) = 0 ⟺ KL(P‖M)=KL(Q‖M)=0 ⟺ P = M 且 Q = M ⟺ P = Q(几乎处处)。

因此由 C(G) = −log4 + 2·JSD ≥ −log4,**且等号当且仅当 JSD = 0,即 p_g = p_data(几乎处处)时取到**。这就是全局最优:

```
min_G C(G) = −log4 ,   在 p_g = p_data 处取得
```

此时把 p_g = p_data 代入 (a) 的闭式解:

```
D*(x) = p_data / (p_data + p_data) = 1/2   (对所有 x)
```

即收敛时判别器对所有样本输出 1/2 —— 鉴赏家彻底分不清真假。

> 数值验算:令 p_g = p_data 时直接算 C(G) = −1.386294 = −log4,确认下界取到。

**结论:在最优判别器假设下,GAN 等价于以 JS 散度为度量做分布匹配;其全局最优是生成分布精确等于真实分布(p_g = p_data),此时 C(G) = −log4 且 D* ≡ 1/2。**

### 练习 2:用一维两点分布定量解释梯度消失与 Wasserstein 的优越

设 p_data = δ(x)(质量全在 x=0),p_g = δ(x−θ)(质量全在 x=θ),θ ≠ 0。两者支撑 {0} 与 {θ} 不重叠。

#### (a) JSD(p_data‖p_g) 关于 θ,证明 θ≠0 时为常数 log2

平均分布 M = (p_data + p_g)/2 在 x=0 与 x=θ 两点各放 ½ 的质量。逐点算两个 KL:

```
在 x=0:  p_data 放全部质量 1,M(0)=½  → KL(p_data‖M) = 1·log(1 / ½) = log 2
在 x=θ:  p_g    放全部质量 1,M(θ)=½  → KL(p_g‖M)    = 1·log(1 / ½) = log 2
(其余位置 p_data、p_g 均为 0,KL 被积项为 0)
```

代入 JSD 定义:

```
JSD(p_data‖p_g) = ½·log2 + ½·log2 = log 2   (与 θ 的具体取值无关)
```

只要 θ ≠ 0,无论 θ = 0.001 还是 1000,JSD 恒为 log2 ≈ 0.693。因此

```
∂ JSD / ∂θ = 0      (θ ≠ 0)
```

> 数值验算:θ ∈ {0.001, 1, 1000} 时 JSD 均 = 0.693147 = log2。

#### (b) W₁(p_data, p_g) 关于 θ

两点分布之间唯一的运输方案就是把 x=θ 处的全部质量(质量 1)搬到 x=0,搬运距离 |θ|:

```
W₁(p_data, p_g) = inf_γ E_γ[‖x−y‖] = 1·|θ| = |θ|
```

(一维下 W₁ 等于两个 CDF 之差的积分;两点分布退化成 |θ|。)其导数:

```
∂ W₁ / ∂θ = sign(θ)    (θ ≠ 0,大小恒为 1)
```

> 数值验算:θ ∈ {0.001, 1, 1000} 时 W₁ = {0.001, 1, 1000} = |θ|,随 θ 连续变化。

#### (c) 为什么 JS 训不动 G、而 Wasserstein 能

- G 靠 ∇_θ(损失)更新。用 JS 时损失 = −log4 + 2·JSD,梯度 ∝ ∂JSD/∂θ = **0**(只要 p_g 与 p_data 还不重叠)。**梯度恒为 0 ⟹ G 收不到"该往哪挪、挪多远"的任何信号,完全学不动**——θ=1000 和 θ=0.001 在 JS 看来"一样远(都=log2)",像一个台阶函数,平台上无坡可下。这正是"判别器训得太好(逼近 D*)、而 p_g 与 p_data 不重叠时 G 梯度消失"的精确机理。
- 用 W₁ 时损失 = |θ|,∂W₁/∂θ = sign(θ) **始终非零且指向 0**。无论分布离得多远,Wasserstein 都给出"往 −sign(θ) 方向挪"的有效梯度,像一个斜坡处处有坡度可下。

**结论:JS 散度对"不重叠"只输出常数(台阶),梯度为 0,在高维生成"两个低维流形几乎必然不相交"的常态下必然梯度消失;Wasserstein 距离随分布参数连续可导地变化(斜坡),即使支撑完全不重叠也持续提供有用梯度。这就是 WGAN 把 JS 换成 W₁ 的根本动机。**

### 练习 3:解释模式崩溃为何与 KL 的不对称有关

#### (a) KL(p_g‖p_data) 的积分式与两种情形下被积项的趋向

```
KL(p_g ‖ p_data) = ∫ p_g(x) · log ( p_g(x) / p_data(x) ) dx
```

被积项是 p_g(x)·log(p_g/p_data)。分析两种失配:

- **情形一:p_g 在某处有质量,但 p_data 在该处 ≈ 0(G 造出真实没有的样本)。** 此处分子 p_g > 0、分母 p_data → 0,故 log(p_g/p_data) → +∞,且前面乘的 p_g > 0 不为零 ⟹ **被积项 → +∞,惩罚极重(发散)**。
- **情形二:p_data 在某处有质量,但 p_g 在该处 ≈ 0(G 漏掉了某个众数 / mode dropping)。** 此处分子 p_g → 0,而 x·log x → 0(p_g·log p_g → 0),分母 p_data 有限 ⟹ **被积项 → 0,几乎不惩罚**。

> 数值验算:
> - p_g 覆盖 p_data 的"空洞"(p_g=0.5 处 p_data≈1e-6):KL(p_g‖p_data) ≈ 6.561(大)。
> - p_g 漏掉 mode(只覆盖 3 个 mode 中的 1 个):KL(p_g‖p_data) ≈ 1.079(有限、小)。
> 两者对比鲜明:同样是分布失配,"造假货"被重罚,"漏众数"几乎不罚。

#### (b) 据此说明 KL(p_g‖p_data) 鼓励保守、塌向少数众数

这个 KL 是**不对称的、零容忍 (zero-forcing / mode-seeking) 的**:它对"p_g 在 p_data≈0 处还有质量"(造出不真实的样本)施加发散级惩罚,却对"p_g 漏掉 p_data 的众数"几乎不罚。最小化它的理性策略因此是:

> **宁可只覆盖少数几个最有把握(p_data 高)的众数,把质量都压在那里,也绝不冒险在 p_data 低的区域放质量。**

结果 G 主动放弃覆盖度、把概率质量塌缩到一两个安全众数上——这正是**模式崩溃**。这与梯度消失的根同源:章节 §5.2 指出,GAN 的非饱和损失隐式优化的目标里含一项 KL(p_g‖p_data),它的这种不对称性是模式崩溃的诱因之一。

**结论:KL(p_g‖p_data) 是 mode-seeking 的非对称散度——重罚"造假"、轻纵"漏众数",因此最小化它会把 G 推向"少覆盖、求稳"的保守解,塌向少数众数,这正是模式崩溃在度量层面的成因。**(对照:KL(p_data‖p_g) 反过来是 mass-covering / mean-seeking 的,会逼 G 覆盖所有众数,代价是糊成一片——这也解释了 VAE 偏糊的倾向。)

### 练习 4:给 §7.2 的 WGAN-GP 加梯度范数 sanity check,并对照单边惩罚

#### 思路

梯度惩罚 (‖∇‖−1)² 把 critic 在插值点的输入梯度范数**双向**推向恰好 1(对偶最优 critic 的梯度范数本就 ≡ 1)。所以训练若干步后,在一个 batch 的插值点上统计 ‖∇_x̂ f(x̂)‖₂ 的均值,应当收敛到 1 附近——这是检查"Lipschitz 约束是否被有效施加"的探针。对照实验把双边 (‖∇‖−1)² 误改成单边 max(0,‖∇‖−1)²:单边只在范数 >1 时罚、范数 <1 时完全不管,于是当对抗压力把某些区域的梯度范数压到 <1 时,单边版不会把它拉回 1,统计均值会**偏低**,critic 偏离对偶最优。

#### 探针 + 对照(在 §7.2 的 `gradient_penalty` 上加一行返回,改动最小)

```python
import torch, torch.nn as nn

def gradient_penalty(critic, real, fake, device, one_sided=False):
    """在 §7.2 基础上:(1) 加 one_sided 开关做对照;(2) 额外返回梯度范数均值做 sanity check。"""
    B = real.size(0)
    eps = torch.rand(B, 1, 1, 1, device=device)               # 图像是 4D;1维数据用 rand(B,1)
    x_hat = (eps * real + (1 - eps) * fake).requires_grad_(True)
    score = critic(x_hat)
    grad = torch.autograd.grad(
        outputs=score, inputs=x_hat,
        grad_outputs=torch.ones_like(score),
        create_graph=True, retain_graph=True,                  # 二阶导:gp 要对 critic 参数再反传
    )[0]
    grad_norm = grad.view(B, -1).norm(2, dim=1)                # ‖∇‖₂ per sample

    if one_sided:                                              # ❌ 错误版:只罚 >1
        gp = (torch.clamp(grad_norm - 1.0, min=0.0) ** 2).mean()
    else:                                                      # ✅ 正确版:双边,推向恰好 =1
        gp = ((grad_norm - 1.0) ** 2).mean()

    # sanity check:detach 出来纯统计,不参与反传
    return gp, grad_norm.mean().detach()


@torch.no_grad()
def report_grad_norm(critic, real, fake, device, one_sided=False):
    """训练循环里每隔 N 步调用:打印插值点梯度范数均值(应 ≈ 1)。"""
    # 注意:这里要临时开梯度,@no_grad 仅防止统计本身建图;实际用下面 with 包住
    pass

def critic_step(G, C, real, opt_c, z_dim, device, n_critic=5, lam=10.0, one_sided=False):
    B = real.size(0)
    last_gnorm = None
    for _ in range(n_critic):
        z = torch.randn(B, z_dim, device=device)
        fake = G(z).detach()
        gp, gnorm = gradient_penalty(C, real, fake, device, one_sided)
        # critic 做最小化:写成"假−真"等价于最大化"真−假"
        loss_c = C(fake).mean() - C(real).mean() + lam * gp
        opt_c.zero_grad(); loss_c.backward(); opt_c.step()
        last_gnorm = gnorm
    return loss_c.item(), last_gnorm.item()                    # 把 ‖∇‖均值带出来打印
```

#### 预期现象(及成因)

- **正确(双边)实现**:训练充分后 ‖∇_x̂ f‖₂ 的 batch 均值应**收敛到 1 附近**——双边惩罚从两侧把范数夹向 1,这是"对偶最优 critic 梯度范数 ≡ 1"在数值上的体现。
- **错误(单边)实现**:均值会**偏低于 1**。因为单边 max(0,‖∇‖−1)² 在范数 <1 的区域梯度为 0、完全不施加约束;当对抗项把某些区域的梯度范数压到 <1 时,没有任何力把它拉回 1,critic 偏离 1-Lipschitz 对偶最优,W₁ 估计变差,样本质量与稳定性随之退化。

> 机制验证(纯 numpy 解析玩具,CPU 秒级,不依赖 torch):
> 用标量 critic f(x)=w·x(梯度范数 = |w|),施加一个想把范数推离 1 的"对抗压力",对比两种惩罚下的稳定点:
> - 对抗想**增大**范数(初始 |w|=2)时:双边收敛到 ≈1.015、单边也收敛到 ≈1.015(范数 >1,两者都罚,行为一致)。
> - 对抗想**减小**范数时:双边把范数拉回 ≈0.985(双边在 <1 侧仍施力);**单边掉到 0、统计被对抗项主导而偏离**(单边 <1 不罚)。
> 这干净地复现了"单边版在范数 <1 时失效、导致均值偏低"的核心结论。
>
> (附:用 torch 在 1 维玩具数据上实跑 §7.2 完整循环也能跑通,但 1 维 WGAN-GP 玩具对学习率/步数/网络宽度极敏感,实跑出的 ‖∇‖具体数值在 run 间剧烈波动、不稳定——这本身印证了章节 §10 所说"GAN 没有干净的 loss 指标、训练脆"。因此此处用解析玩具锁定机制,**不写死某个具体收敛数值(具体数值随任务/超参波动,待核)**。)

**结论:正确的双边梯度惩罚把插值点的输入梯度范数双向夹向恰好 1,这正是 sanity check 应观测到的现象;误用单边惩罚会在范数 <1 区域失去约束、使均值偏低并偏离对偶最优——这印证了 §6.3 强调的"梯度惩罚必须是双边、不能写成单边"这一反直觉而关键的设计点。**
