> 这一章把你从"大模型就是堆参数、堆数据、堆算力,越大越好"带到"能从损失的幂律推出给定算力预算下参数和数据该各占多少、能用几个小规模点外推预测大模型的损失、能判断一篇论文里的'涌现能力'是真的相变还是评测指标制造的假象"。学完你拿到一个 GPU 预算,能算出该训多大的模型、喂多少 token,而不是拍脑袋。

## 为什么这一章绕不过去

前面几章你已经会搭一个 Transformer、会算它的 FLOPs 和显存、会做混合精度训练。现在面对一个真实问题:**老板给你 1000 张卡训一个月,你该训多大的模型?**

新手的答案是"越大越好,把卡塞满"。这个答案有两处致命错误。第一,模型大到一定程度,**参数多但数据不够,等于花钱买了一堆没训透的权重**——GPT-3 就是这么训的(后面会算给你看,它的数据量只够它参数量的约 1.7 倍 token,而最优应该是约 20 倍)。第二,把训练算力全砸在参数上,会得到一个**推理时贵得离谱、还训得半生不熟**的模型——参数越多,每次前向推理就越慢越费显存,这笔账要在模型上线后天天还。

Scaling Laws(规模定律)要回答的就是这件事:**规模(N=参数量、D=数据量、C=算力)和模型性能(测试损失 L)之间,存在不存在一个可以预测、可以外推的定量关系?** 如果存在,我们就能在小规模上拟合出这条曲线,在不烧大钱的情况下预言大模型的表现,并且在固定预算下解出最优配比。

这章会建立两个里程碑结果,然后拆穿一个流行误解:

1. **Kaplan 等(OpenAI, 2020)**:测试损失 L 随 N、D、C 各自近似服从幂律(power law),在对数坐标上是直线,横跨好几个数量级都不偏。这让外推成为可能。
2. **Chinchilla(DeepMind, 2022)**:在固定算力 C 下,参数 N 和数据 D 应该**大致同比例增长**(经验上约 20 token/参数),此前的大模型(含 GPT-3)普遍"参数过大、数据过少",训练不充分。
3. **涌现能力(emergent abilities)的争论**:某些能力看起来在某个规模"突然出现",但 Schaeffer 等(2023)指出,很多"突变"是**评测指标非线性**(比如精确匹配 exact-match)放大出来的假象,换成平滑指标后是渐进提升。这场争论你必须会自己判断。

**贯穿全章的判断标准**:任何一条"规模定律"或"涌现现象",都要问两件事——(a) 它在对数坐标上是不是真的一条直线、跨了几个数量级、有没有外推验证;(b) 你看到的"突变",是模型能力本身的相变,还是你选的那个度量函数把平滑的底层量给折弯了。

## 一、幂律:为什么损失随规模的下降是一条对数直线

### 问题:性能和规模到底是什么函数关系

先把"性能"定死。Scaling Laws 里的性能不是某个 benchmark 的准确率(那东西噪声大、有天花板、还受指标影响),而是**在留出数据上的交叉熵测试损失 L**(单位通常是 nats/token,自然对数下的每 token 平均负对数似然)。选它有三个理由:它连续、它没有人为天花板(理论下界是数据本身的熵)、它和下游能力强相关。

现在问:当我们把模型参数量 N 翻 10 倍、100 倍,L 怎么变?三种可能的直觉:

- **线性下降**:L 随 N 线性减小 → 不可能,L 会变负,而交叉熵有下界。
- **指数下降**:L ≈ L∞ + c·e^(−kN) → 衰减太快,小规模就该饱和,与观测不符。
- **幂律下降**:L 随 N 的某个负幂次减小 → 这才是 Kaplan 等观测到的形态。

### 推导:幂律在对数坐标上为什么是直线

Kaplan 等的核心经验发现是,在其他因素不成为瓶颈时(数据足够、训练充分),损失对单一规模变量服从**纯幂律**:

```
对参数量:   L(N) ≈ (N_c / N)^α_N
对数据量:   L(D) ≈ (D_c / D)^α_D
对算力:     L(C) ≈ (C_c / C)^α_C

其中:
  N_c, D_c, C_c  是各自的"特征尺度"常数(拟合得到)
  α_N, α_D, α_C  是各自的幂律指数(power-law exponent),都是小正数
```

(具体数值如 α_N ≈ 0.076、α_D ≈ 0.095 之类来自 Kaplan 2020 原文,**待核**;不同论文、不同数据分布上数值不同,但量级都是 0.05~0.1 这一档。)

为什么这形态一画到对数坐标上就是直线?两边取自然对数:

```
L(N) = (N_c / N)^α_N
取 ln:
ln L = α_N · ln(N_c) − α_N · ln(N)
     = const − α_N · ln(N)
```

这就是 **ln L 关于 ln N 的一次函数,斜率 = −α_N**。所以在 log-log 图(横轴 ln N、纵轴 ln L)上,数据点落在一条斜率为负的直线上。这是幂律的指纹:**对数坐标上的直线 ⟺ 幂律关系**。Kaplan 等的关键贡献不是发现某个公式,而是发现这条直线**跨越了 6~7 个数量级的 N 都不弯**——这种横跨多个量级的直线,正是它能用来**外推**的前提。

> 直觉:幂律意味着"没有特征尺度"。指数衰减有一个特征尺度 1/k,超过它就基本饱和;幂律没有这样的拐点,你每把 N 翻 10 倍,ln L 就稳定地下降一个固定量 α_N·ln10,无论你现在在哪个量级。这种"自相似"正是它能外推的数学根源——只要标度不变性(scale invariance)不被打破。

### 完整形式:把三个变量合成一个联合损失

单变量幂律的前提是"其他因素不卡脖子"。现实中 N 和 D 会同时不够。后续工作(尤其 Chinchilla)用一个**可加分解**的联合形式,这是本章后面所有推导的基础:

```
L(N, D) = E + A / N^a + B / D^b
          ↑       ↑           ↑
          |       |           参数不够带来的"容量误差"
          |       数据不够带来的"数据误差"
          不可约损失(irreducible loss),= 数据本身的熵,N、D 再大也降不到它以下
```

(指数 a、b 和系数 A、B、E 是拟合常数;Chinchilla 给出的具体值 **待核**,但报告的 a 与 b 都在 0.3 上下、彼此接近,这个"接近"后面会成为关键。)

这个形式讲清了三件事:

- **E(不可约项)的存在**:语言本身有内在不确定性(下一个词永远不是完全确定的),交叉熵不可能降到 0。E 就是这个底。早期不少人画 loss 曲线忘了它,误以为能无限降。
- **两个误差项相互独立**:第二项只看 N,第三项只看 D。当 N 很大但 D 很小,L 被 B/D^b 这一项**钉住**——这正是"参数过剩、数据不足"的数学刻画:你把第二项压到很小,第三项却纹丝不动,总损失下不去。
- **当某项远大于其他项时,联合形式退化回单变量幂律**:数据极充分时 B/D^b → 0,L(N) ≈ E + A/N^a,这就是 Kaplan 的 L(N)。

### 代码:用几个 (N, L) 点拟合幂律并外推

幂律拟合的标准做法就是**在 log-log 空间做线性回归**(因为 ln L = const − α·ln N 是线性的)。

```python
import numpy as np

# 假设你在小规模上跑了 7 个模型,记录了 (参数量 N, 测试损失 L)。
# 真实场景这些点要横跨尽量多的数量级,且每个点都训到收敛(数据不卡脖子)。
N = np.array([1e6, 3e6, 1e7, 3e7, 1e8, 3e8, 1e9])
L = np.array([2.86, 2.55, 2.27, 2.02, 1.80, 1.60, 1.43])  # 示意数据

# 拟合纯幂律 L = (Nc/N)^alpha  <=>  ln L = alpha*ln(Nc) - alpha*ln(N)
x, y = np.log(N), np.log(L)
slope, intercept = np.polyfit(x, y, 1)   # 一次多项式 = log-log 直线
alpha = -slope                            # 斜率取负就是幂律指数
Nc = np.exp(intercept / alpha)            # 从截距反解特征尺度
print(f"alpha = {alpha:.4f},  N_c = {Nc:.3e}")

# 外推:预测一个比所有训练点都大一个数量级的模型(1e10 参数)的损失
def predict_L(n):
    return (Nc / n)**alpha
print(f"外推 L(1e10) = {predict_L(1e10):.3f}")

# 关键诚实检查:外推前必须确认这几个点在 log-log 上"够直"。
# 用残差判断——若残差有系统性弯曲(不是随机噪声),说明纯幂律不成立,
# 可能撞上了不可约项 E,得改拟合 L = E + A/N^a(非线性最小二乘)。
resid = y - (slope * x + intercept)
print("log 残差:", np.round(resid, 4))  # 应近似随机、无趋势
```

**这段代码体现的机制**:幂律外推的全部底气来自"log-log 上是直线"这一经验事实。所以工程上拟合完第一件事不是看 α 多少,而是**看残差有没有系统性弯曲**。一旦弯了,要么是撞上了 E(损失接近不可约项,曲线开始走平),要么是某个因素开始卡脖子(数据不够、学习率没调好)。**新手最常犯的错是只用 2~3 个点、且这些点只跨不到一个数量级,就敢往外推几个数量级**——这种外推毫无保证,log-log 上两三个点几乎总能连成"直线"。

### 分析:为什么幂律能外推,以及外推的边界

幂律可外推的数学条件是**标度不变性在外推区间内持续成立**。具体到工程上有三条边界,越界外推就会失效:

1. **撞上不可约项 E**:当 A/N^a 降到和 E 同量级,总损失开始走平,纯幂律高估了下降速度。
2. **另一个因素变成瓶颈**:你沿 N 外推,但 D 没跟上,L 会偏离 L(N) 曲线、被 B/D^b 钉住(这正是下一节 Chinchilla 的入口)。
3. **架构/优化发生质变**:幂律是在固定架构族、固定优化配方下拟合的。换了架构(比如从 dense 换成 MoE)、换了数据分布,曲线会平移甚至改斜率。

**量化一下外推的价值**:假设 α_N ≈ 0.076(待核),每把 N 翻 10 倍,ln L 下降 0.076·ln10 ≈ 0.175,即 L 乘以 e^(−0.175) ≈ 0.84。**也就是参数翻 10 倍,损失只降约 16%**。这个数字本身就传递了一个冷酷的事实:**幂律指数小,意味着收益递减极快**。把模型从 1B 做到 10B,损失降 16%;再到 100B,又降 16%(相对值)。这正是为什么"无脑堆参数"在经济上越来越不划算——也为下一节"该把算力花在哪"埋下伏笔。

## 二、算力约束 C ≈ 6ND:把系统视角接进来

### 问题:N 和 D 不能独立无限增长,它们被算力绑在一起

上一节把 N、D 当成可以独立调的旋钮。但现实里你能调的只有一个东西:**总训练算力 C**(以 FLOPs 计,等于你买的卡数 × 时间 × 每卡算力 × 利用率)。N 和 D 都要花 C,而且它们的乘积被 C 卡死。所以真正的优化问题是:**给定 C,N 和 D 怎么分?**

要解这个问题,先得知道 C、N、D 三者的定量关系。这就是那个你在系统章见过的经验公式 **C ≈ 6ND**。

### 推导:C ≈ 6ND 从哪来

这个 6 不是魔法常数,是数出来的。考虑一个有 N 个参数的 Transformer,处理 D 个 token 训练一遍(一个 epoch)。算力主要花在矩阵乘(GEMM)上,逐项数:

```
前向传播(forward):
  每个参数本质上参与一次"乘加"(multiply-accumulate):
    一次乘法 + 一次加法 = 2 次浮点运算 (FLOPs)
  每个 token 经过整个网络,大致把每个参数用一次
  => 前向每 token 约 2N FLOPs

反向传播(backward):
  反向要算两组梯度:对输入的梯度 + 对权重的梯度
  每组的计算量都和前向同量级,所以反向 ≈ 2 × 前向
  => 反向每 token 约 4N FLOPs

合计每 token: 2N(前向) + 4N(反向) = 6N FLOPs
处理 D 个 token: C ≈ 6N·D
```

(这是主导项近似,忽略了注意力中与序列长度平方相关的项、layernorm、激活函数等;在大模型、中等序列长度下,GEMM 主导,6ND 误差通常在百分之十几以内,**精确系数随架构和序列长度变化,待核**。)

**为什么反向是前向的两倍**,值得停一下:链式法则下,一个矩阵乘 Y = XW 的反向需要算 ∂L/∂X = (∂L/∂Y)·Wᵀ 和 ∂L/∂W = Xᵀ·(∂L/∂Y),两个矩阵乘,各自和前向那一个同规模。所以反向天然是前向的约 2 倍。前向 1 份 + 反向 2 份 = 3 份,每份 2N(乘加各算 1 FLOP),总 6N。**这个 "1:2" 的前向反向比,是后面激活重计算、混合精度章节反复用到的基础常数。**

### 代码:把预算翻译成可训练配置

```python
def flops_budget(num_gpus, days, peak_flops_per_gpu, mfu=0.4):
    """把硬件预算换算成总训练算力 C(FLOPs)。
    mfu = Model FLOPs Utilization,实际算力 / 理论峰值,大集群常在 0.3~0.5。"""
    seconds = days * 24 * 3600
    return num_gpus * seconds * peak_flops_per_gpu * mfu

def N_D_from_C(C, tokens_per_param=20):
    """给定 C 和目标 token/参数比,反解 N 和 D。
    用 C = 6ND 和 D = ratio*N  =>  C = 6*ratio*N^2"""
    N = (C / (6 * tokens_per_param))**0.5
    D = tokens_per_param * N
    return N, D

# 示意:1000 卡,30 天,单卡峰值 1e15 FLOP/s(待核,取决于具体硬件)
C = flops_budget(1000, 30, peak_flops_per_gpu=1e15, mfu=0.4)
print(f"C ≈ {C:.3e} FLOPs")
N, D = N_D_from_C(C, tokens_per_param=20)   # 用 Chinchilla 经验比
print(f"建议 N ≈ {N:.3e} 参数,  D ≈ {D:.3e} tokens")
```

注意这里我**预先塞了 tokens_per_param=20**——这个数从哪来、为什么不是 1.7(GPT-3 的值)也不是 100,正是下一节 Chinchilla 要推的。

### 分析:这个约束把二维问题降成一维

有了 C ≈ 6ND,优化空间从 (N, D) 二维降成一维:固定 C 就是固定 ND 乘积,N 和 D 在一条等算力曲线(iso-FLOP 曲线,即 hyperbola ND=const)上反向滑动——N 大则 D 小,反之亦然。问题变成:**在这条曲线上,哪个点的 L(N,D) 最小?** 这是个标准的约束优化,下一节用拉格朗日乘子法解。

## 三、Chinchilla:固定算力下的最优配比

### 问题:GPT-3 把算力花错地方了吗

Chinchilla 论文(Hoffmann 等, DeepMind, 2022)提出的颠覆性结论:**当时几乎所有大模型(包括 GPT-3 175B)都训练不足——参数堆太多,数据喂太少**。

先把"训练不足"量化。GPT-3 约 175B 参数,训练约 300B token(具体数 **待核**),token/参数比 ≈ 300/175 ≈ **1.7**。而 Chinchilla 的最优比约 **20**。差了一个数量级。这意味着:GPT-3 那 175B 参数里,有相当一部分**根本没被数据训透**——损失被联合公式里的数据项 B/D^b 钉住了,你再加参数(压低 A/N^a)也没用,因为瓶颈在数据那一项。

Chinchilla 用一个反例石锤:他们训了一个 **70B 参数、1.4T token**(待核)的模型 Chinchilla,算力和 GPT-3 量级相当(用 C≈6ND 验:6 × 70e9 × 1.4e12 ≈ 5.9e23 FLOPs),**参数只有 GPT-3 的 40%,但下游表现全面胜出**。原因就是同样的算力,它把更多预算分给了数据,把模型训透了。

### 推导:给定 C,解出最优 N* 和 D*

这是本章的核心推导,用拉格朗日乘子法,从联合损失公式严格解出最优配比。

```
目标:在约束 C = 6ND 下最小化  L(N,D) = E + A/N^a + B/D^b

【第 1 步】E 是常数,不影响极值点,可丢掉。约束写成 D = C/(6N),代入消元:
  L_eff(N) = A/N^a + B / [C/(6N)]^b
           = A·N^(-a) + B·(6N/C)^b
           = A·N^(-a) + B·(6/C)^b · N^b

【第 2 步】对 N 求导并令其为 0(无约束一维极小):
  dL_eff/dN = -a·A·N^(-a-1) + b·B·(6/C)^b · N^(b-1) = 0

【第 3 步】移项,把两个含 N 的幂次合并:
  a·A·N^(-a-1) = b·B·(6/C)^b · N^(b-1)
  a·A / [b·B·(6/C)^b] = N^(b-1) / N^(-a-1) = N^(b-1 + a + 1) = N^(a+b)

【第 4 步】解出最优 N*:
  N* = [ (a·A)/(b·B) · (C/6)^b ]^( 1/(a+b) )
     ∝ C^( b/(a+b) )                          ← 把和 C 无关的常数收进比例号

【第 5 步】由约束反解 D*:
  D* = C/(6 N*) ∝ C / C^(b/(a+b)) = C^( 1 - b/(a+b) ) = C^( a/(a+b) )
```

(我用 Python 验过第 4、5 步的指数:b/(a+b) + a/(a+b) = 1,和约束 ND ∝ C 要求的 "两个指数之和为 1" 自洽。)

得到两个干净的标度关系:

```
N* ∝ C^( b/(a+b) )        最优参数量随算力的幂次为 b/(a+b)
D* ∝ C^( a/(a+b) )        最优数据量随算力的幂次为 a/(a+b)
```

### 关键洞察:为什么是"同比例增长"

现在看 Chinchilla 那个"接近"的拟合事实:**a ≈ b**(报告的两个指数都在 0.3 上下,彼此接近,**具体值待核**)。当 a = b:

```
N* ∝ C^( b/(a+b) ) = C^( 1/2 )
D* ∝ C^( a/(a+b) ) = C^( 1/2 )
```

**两者都随 C 的 1/2 次方增长,即同比例增长!** 算力翻 4 倍,N 和 D 各翻 2 倍。这就是 Chinchilla 那条著名结论"**参数和数据应该等比例 scale**"的数学来源——它**不是**先验假设,而是从 a≈b 这个经验拟合 + 拉格朗日推导出来的。

再看 token/参数比怎么随算力变:

```
D*/N* ∝ C^(a/(a+b)) / C^(b/(a+b)) = C^( (a-b)/(a+b) )
当 a = b:  D*/N* ∝ C^0 = 常数
```

**a≈b ⟹ 最优 token/参数比近似与算力无关,是个常数**——这就是那个"约 20 token/参数"经验值的理论依据:它之所以能当成一个固定比例用(不管你预算多大),正是因为 a≈b 让指数差为 0。如果 a 明显大于 b(数据项衰减更慢、更难压),最优比就会随 C 增大而上升(该喂更多数据);反之则下降。**所以"20 这个数"本质上是 (a−b) 接近 0 的产物,不是宇宙常数;换了数据分布、换了架构,a、b 会变,这个比例也会变。** 后续工作(待核)在更大规模、不同数据上确实报告过偏离 20 的最优比。

### 代码:在 iso-FLOP 曲线上数值搜最优点

实践中你常常不直接信解析解,而是**在固定 C 下扫一遍不同的 N(对应不同 D),各训一个小模型,看哪个 L 最低**——这叫 iso-FLOP profiling,是 Chinchilla 三种估计方法之一。

```python
import numpy as np

# 联合损失模型(系数为示意,真实要从你自己的小规模实验拟合;具体值待核)
E, A, B, a, b = 1.69, 406.0, 410.0, 0.34, 0.28

def loss(N, D):
    return E + A / N**a + B / D**b

def best_split_at_fixed_C(C, N_grid):
    """固定算力 C,在 iso-FLOP 曲线 D=C/(6N) 上扫 N,返回损失最低的 (N*,D*)。"""
    D_grid = C / (6 * N_grid)             # 约束:每个 N 对应唯一 D
    Ls = loss(N_grid, D_grid)
    i = np.argmin(Ls)
    return N_grid[i], D_grid[i], Ls[i]

C = 5.9e23                                # ~ GPT-3 / Chinchilla 量级算力
N_grid = np.logspace(9, 12, 200)          # 从 1e9 扫到 1e12 参数
Nstar, Dstar, Lstar = best_split_at_fixed_C(C, N_grid)
print(f"最优 N* ≈ {Nstar:.3e},  D* ≈ {Dstar:.3e},  L* ≈ {Lstar:.4f}")
print(f"最优 token/参数比 ≈ {Dstar/Nstar:.1f}")

# 对比:如果像 GPT-3 那样把参数堆到 175B(数据被约束压到很小)
N_gpt3 = 175e9
D_gpt3 = C / (6 * N_gpt3)
print(f"GPT-3 式配比 N=175B 时 D≈{D_gpt3:.3e}, L≈{loss(N_gpt3, D_gpt3):.4f} (更差)")
```

**这段代码体现的机制**:iso-FLOP profiling 把"解析解对不对"这个信任问题,换成了"直接实验扫一遍"的经验问题。它的好处是不依赖你猜对了联合公式的形式;代价是要训一堆小模型(但每个都小、都便宜,总成本远低于直接赌一个大模型)。**Chinchilla 的方法论价值大于它给的那个具体数字**:它示范了"用一组便宜的小实验,在 log 空间拟合出曲面,再外推到大模型最优点"这套可复制的流程。

### 分析:对推理成本的含义——小而训透更划算

Chinchilla 还有一层经济学含义,新手常忽略。训练算力 C 是**一次性**支出,但推理算力是**持续**支出——每处理一个用户请求,前向算力 ≈ 2N(per token),N 越大每次推理越贵。

```
两个模型损失相同,但配比不同:
  模型 A:大 N、小 D(训练不足,GPT-3 式)
  模型 B:小 N、大 D(训练充分,Chinchilla 式)

训练成本(一次性):C = 6ND,两者相同(同 iso-FLOP)
推理成本(每 token,持续):约 2N
  => B 的 N 更小 => B 每次推理更便宜、更省显存、更快
```

**结论:在损失相同的前提下,Chinchilla 式的"小而训透"模型在推理上全面占优。** 而真实部署中,一个模型一生的推理总算力往往**远超**它的训练算力(一个热门模型被调用上亿次)。所以从全生命周期成本看,**最优 N 应该比 Chinchilla 训练最优点还要更小一些**——刻意"过度训练"(over-training,喂远超 20 倍的 token,把 N 压更低)以换取更低的长期推理成本。这正是后来很多开源小模型(7B/8B 喂上 T 级 token,远超 20:1)走的路:它们不追求"给定训练算力的最优损失",而追求"给定推理预算的最优能力"。**理解这一层,你才算真正读懂了 Chinchilla 之后的模型尺寸选择逻辑。**

## 四、涌现能力:真相变,还是指标制造的幻觉

### 问题:能力真的会"突然出现"吗

2022 年前后一个广为流传的现象:某些任务(多位数加法、特定推理任务等)上,小模型准确率几乎是 0,随着规模增大也一直贴着 0,然后**在某个参数规模附近,准确率突然从近 0 跳到显著非 0**——像水到 100°C 突然沸腾。这被称为 **emergent abilities(涌现能力)**:大模型"涌现"出了小模型完全不具备的能力,且不可从小模型外推。

如果这是真的,后果很严重:它意味着 **Scaling Laws 的可外推性在这些能力上失效**——你没法从小模型预测大模型会不会"突然学会"某件事,安全性上也无法预判一个更大的模型会冒出什么新能力。

### 批判性拆解:Schaeffer 等的"指标假象"论证

Schaeffer、Miranda、Koyejo(2023)提出了一个有力的反驳:**很多所谓的"涌现",是评测指标的非线性造成的测量假象,而非模型底层能力的相变。** 这个论证一定要会自己复现,它是科研品味的试金石。

核心机制:模型的**底层连续量**(比如每 token 的对数概率、或答对的概率)其实是随规模**平滑、渐进**地改善的(服从幂律);但如果你用一个**非线性、有阈值、不连续**的指标去测它,平滑的改善会被这个指标**折弯成"突变"**。

最典型的是 **exact-match(精确匹配)** 这类指标。考虑一个需要连续答对 k 个 token 才算"正确"的任务(比如多位数加法,每位都要对):

```
设模型每个 token 答对的概率为 p(N),它随规模 N 平滑、单调上升(无突变)。
任务"正确"的指标 = 全部 k 个 token 都对:
  Accuracy(N) = p(N)^k

现在看 Accuracy 随 N 的变化:
  当 p 小时(比如 p=0.3, k=5):  Acc = 0.3^5 ≈ 0.0024   ← 几乎是 0
  当 p 中等(p=0.6, k=5):       Acc = 0.6^5 ≈ 0.078    ← 还是很小
  当 p 较大(p=0.9, k=5):       Acc = 0.9^5 ≈ 0.59     ← 突然"起飞"了

p 从 0.3 平滑涨到 0.9,Accuracy 却从 0.002 "突然"跳到 0.59。
p^k 这个幂次把底层的平滑曲线,在高 p 端急剧抬升 —— "涌现"出现了。
```

**关键点:p(N) 全程平滑无突变,"突变"完全是 p^k 这个非线性指标制造的。** 指数 k 越大(任务要求连对的 token 越多),这个假象越剧烈、"涌现点"看起来越陡。

Schaeffer 等的论证还有两个支柱:

1. **换平滑指标,突变消失**:把 exact-match 换成**连续指标**——比如直接看每 token 的对数概率、或编辑距离、或 Brier 分数——同一批模型、同一个任务上,曲线变回**平滑渐进**的,没有任何突变。"涌现"被指标一换就没了,说明它不在模型里,在指标里。
2. **能在"非涌现"任务上人为制造涌现**:反过来,对一个本来表现平滑的任务,**故意套一个非线性阈值指标**(比如把连续分数卡个阈值变成 0/1),就能凭空"造出"一条涌现曲线。能造出来的现象,不是模型的内在性质。

### 代码:用平滑底层量造出一个假"涌现"

这段代码亲手把一个平滑的幂律改善,通过 exact-match 指标变成"突变",直观看清假象怎么来的。

```python
import numpy as np

N = np.logspace(7, 11, 60)               # 参数量,跨 4 个数量级

# 底层连续能力:每 token 正确概率随 N 平滑上升(逻辑斯蒂形,无任何突变)
# 这里 p 随 log(N) 平滑增长,本身是渐进的
p = 1 / (1 + np.exp(-(np.log10(N) - 9)))  # 在 N~1e9 附近过 0.5,但整条曲线光滑

k = 6                                     # 任务要求连续答对 6 个 token
acc_exact = p**k                          # 非线性指标:exact-match
acc_smooth = p                            # 平滑指标:平均 token 正确率

# 观察:同一个底层 p(N),两个指标讲完全不同的故事
for n, pe, ae, asm in list(zip(N, p, acc_exact, acc_smooth))[::12]:
    print(f"N={n:.1e}  p={pe:.3f}  exact-match={ae:.4f}  smooth={asm:.3f}")

# acc_exact 会长时间贴近 0,然后在高 p 端急剧抬头 —— 看起来像"涌现"
# acc_smooth 全程平滑单调 —— 同样的模型,没有任何"突变"
# 结论:突变在指标 (·)^k 里,不在 p(N) 里
```

跑出来你会看到:`smooth` 指标是一条平滑上升的线,`exact-match` 指标长时间贴 0、然后在某个规模急剧抬头。**两条线背后是同一个平滑的 p(N)。** 这就是"涌现是指标假象"论证的可运行版本。

### 这场争论的诚实结论:别站队,会判断

要避免把这场争论简化成"涌现是假的"。**诚实的表述是分层的**:

- **"很多被报告的涌现,是非线性/阈值指标的测量假象"——这一条证据扎实,基本成立。** 凡是用 exact-match、多步全对、卡阈值这类指标报出来的"突变",都高度可疑,你应该要求作者出示平滑指标下的曲线。
- **"所有涌现都是假象、底层一切皆平滑可外推"——这一条过强,不能照单全收。** 反方观点(待核,这部分仍是开放研究)认为:(a) 即便底层量平滑,某些能力(如多步推理、in-context learning 的某些形态)是否存在真正的、与平滑底层量解耦的相变,尚无定论;(b) 用平滑指标"看不到突变"也可能只是因为该指标对下游真正关心的能力不敏感——用户要的就是"整道题全对",exact-match 在产品意义上不是假指标。
- **方法论上你该做的**:看到"涌现"声明,先问三件事——(1) 用的什么指标?是不是非线性/阈值型?(2) 换成平滑连续指标后曲线还突变吗?(3) 底层连续量(对数概率)是不是其实平滑?三问之后,绝大多数"涌现"会被归到"指标假象"或"平滑改善"里;剩下少数真正解释不掉的,才是值得研究的开放问题。

**贯穿全章的判断标准在这里收口**:幂律告诉你"对数坐标上的直线可以外推";涌现争论告诉你"小心你用来画那条线的纵轴——指标的非线性会把平滑的真相折成假的突变"。**这两件事是一体两面:都是在问"你测的那个量,和你以为它代表的东西,是不是同一回事"。**

## 五、设计权衡与常见坑

- **坑 1:用太少、跨度太小的点做幂律外推。** log-log 上 2~3 个点几乎总能连成直线,但外推几个数量级毫无保证。底线:拟合点应横跨尽量多的数量级(理想 ≥3 个),且看残差有无系统性弯曲。**只有"跨多个量级仍是直线"才支撑外推。**
- **坑 2:忘记不可约项 E,以为损失能无限降。** 纯幂律 L=(N_c/N)^α 没有下界,但真实损失逼近 E 就走平。拟合接近 SOTA 的损失时必须用 L=E+A/N^a 这种带常数项的形式,否则会高估大模型的下降空间。
- **坑 3:把 "20 token/参数" 当成宇宙常数。** 它来自 a≈b 这个特定拟合,换数据分布、换架构(如 MoE)、换目标(优化推理成本而非训练损失),最优比会变。**把它当默认起点可以,当铁律会出错。**
- **坑 4:只算训练成本,不算推理成本。** Chinchilla 训练最优 ≠ 部署最优。要被调用上亿次的模型,应当 over-train(喂远超 20:1 的 token,把 N 压更小),用一次性训练浪费换长期推理便宜。
- **坑 5:被 exact-match 之类指标骗,以为模型"突然学会了"。** 报告/复现涌现前,务必同时画平滑指标(对数概率、编辑距离)。**两条线不一致时,信平滑的那条反映底层能力。**
- **坑 6:把单变量幂律的常数跨设定乱用。** α、N_c、a、b、E 都是在特定架构族、数据分布、优化配方下拟合的。换 tokenizer、换数据配比、换优化器,常数会变,直接套用别人的数会系统性偏。
- **权衡:解析解 vs iso-FLOP 实验。** 解析解(拉格朗日)给闭式配比,但依赖你猜对联合公式形式;iso-FLOP profiling 不依赖公式形式、更稳健,但要训一组小模型。**预算紧、对公式没把握时,优先 iso-FLOP 经验扫描。**
- **权衡:测试损失 vs 下游指标。** 损失平滑可外推、适合做 scaling 决策;但用户最终关心下游能力,而损失到能力的映射可能非线性(这正是涌现假象的根)。**用损失做规模决策,用下游指标做产品验收,别混用。**

## 六、动手练习

1. **(推导题)从 C≈6ND 与联合幂律推 Chinchilla 配比。** 给定 L(N,D)=E+A/N^a+B/D^b 和约束 C=6ND,用拉格朗日乘子法(或代入消元)证明 N* ∝ C^(b/(a+b))、D* ∝ C^(a/(a+b))。然后:(a) 验证两个指数之和为 1,并说明这为什么是约束 ND∝C 的必然结果;(b) 推出最优 token/参数比 D*/N* ∝ C^((a−b)/(a+b)),并论证 a=b 时它为何与算力无关。*提示:消元后是一维无约束极小,令导数为零;合并 N 的幂次时注意 N^(b-1)/N^(-a-1)=N^(a+b)。*

2. **(分析题)涌现是真突变还是指标假象。** 用第四节代码,把 k(任务要求连对的 token 数)从 1 扫到 10,固定底层 p(N) 不变,描述 exact-match 曲线的"涌现陡峭度"如何随 k 变化。然后回答:(a) 为什么 k 越大假象越剧烈?(b) 如果一个任务的真实产品价值就是"整道题全对",exact-match 算"假指标"吗?给出你的分层判断。*提示:Acc=p^k,在 log 坐标上看 d(log Acc)/d(log p)=k,斜率正比于 k。*

3. **(编码题)带不可约项的非线性拟合。** 把第一节的纯幂律拟合改成拟合 L=E+A/N^a(三参数,非线性)。用 `scipy.optimize.curve_fit`,在有 E 的合成数据上对比:纯幂律(log-log 线性)和带 E 的非线性拟合,哪个外推到大 N 更准?画出残差。*提示:纯幂律在损失接近 E 时会系统性高估下降速度,残差会呈现单边弯曲。*

4. **(分析题)训练最优 vs 部署最优。** 设某模型一生推理 R 次,每次处理 T 个 token,推理算力 ≈ 2N·T·R;训练算力 C=6ND。在"总损失固定"的 iso-loss 约束下(可用联合公式数值求 iso-loss 上的 (N,D) 集合),写出"训练+全生命周期推理"总算力关于 N 的表达式,数值求使总算力最小的 N,并说明它为什么比 Chinchilla 训练最优 N 更小。*提示:沿 iso-loss 曲线减小 N 会增大 D(训练变贵),但同时减小每次推理成本(推理变便宜),总成本是两者之和,存在内点最优。*

## 八、小结与承上启下

这一章把"做多大"从拍脑袋变成了可计算的科学:

- **幂律(Kaplan)**:测试损失对 N、D、C 各自服从幂律,在 log-log 上是跨多个数量级的直线,斜率即幂律指数 α。直线意味着标度不变、意味着可外推——但外推会在撞上不可约项 E、另一因素卡脖子、或架构质变时失效。**幂律指数小(~0.07)意味着收益递减极快,翻 10 倍参数损失只降约 16%。**
- **算力约束 C≈6ND**:每参数每 token 约 6 FLOPs(前向 2 + 反向 4,反向是前向 2 倍),把 (N,D) 二维问题降成 iso-FLOP 曲线上的一维优化。
- **Chinchilla**:用拉格朗日从联合损失 L=E+A/N^a+B/D^b 解出 N*∝C^(b/(a+b))、D*∝C^(a/(a+b));因经验拟合 a≈b,二者同比例增长,token/参数比近似恒定(~20)。GPT-3 的 1.7:1 是严重数据不足。再叠加推理成本,部署最优比训练最优还要更小、更"过训"。
- **涌现争论**:很多"突然出现的能力"是 exact-match 等非线性指标把平滑改善(Acc=p^k)折成的假象;换平滑指标突变消失。但"所有涌现都是假象"的强版本仍是开放问题,正确姿态是会三问判断,而非站队。

**在整门课里的位置**:前面的章节(注意力、归一化、混合精度、并行)教你**怎么造和训一个模型**;这一章教你**该造多大、喂多少、把算力花在哪**——它是连接"算法机制"和"工程预算"的决策层。往后:这章的"小而训透更划算"和"参数收益递减"直接推动了 **MoE(混合专家)**(用稀疏激活在不增加每 token 推理 FLOPs 的前提下涨总参数,本质是绕开 dense 幂律的收益递减),以及 **LoRA / 高效微调**(既然从头训大模型成本被幂律锁死,就在大模型上做低成本适配)。理解了规模定律,你才能看懂这些架构选择背后"用什么换什么"的经济账。

### 练习 1:从 C≈6ND 与联合幂律推 Chinchilla 配比

这是第三节推导的完整复盘 + 两个追问。

**主推导(代入消元法)**

约束 `C = 6ND` 解出 `D = C/(6N)`,代回 `L(N,D) = E + A/N^a + B/D^b`。E 是常数,对极值点无影响,丢掉:

```
L_eff(N) = A·N^(-a) + B·[C/(6N)]^(-b)
         = A·N^(-a) + B·(6N/C)^b
         = A·N^(-a) + B·(6/C)^b · N^b
```

对 N 求导并令其为 0(消元后是一维无约束极小):

```
dL_eff/dN = -a·A·N^(-a-1) + b·B·(6/C)^b · N^(b-1) = 0
```

移项,把两个含 N 的幂次合到一起。关键一步:

```
N^(b-1) / N^(-a-1) = N^[(b-1) - (-a-1)] = N^(b-1+a+1) = N^(a+b)
```

于是:

```
a·A = b·B·(6/C)^b · N^(a+b)
N^(a+b) = (a·A) / [b·B·(6/C)^b] = (a·A)/(b·B) · (C/6)^b
```

两边开 1/(a+b) 次方:

```
N* = [ (a·A)/(b·B) · (C/6)^b ]^(1/(a+b))
```

把所有与 C 无关的常数收进比例号(它们不改变 N* 随 C 的标度):

```
N* ∝ (C^b)^(1/(a+b)) = C^( b/(a+b) )
```

由约束 `D* = C/(6N*)` 反解:

```
D* ∝ C / C^(b/(a+b)) = C^(1 - b/(a+b)) = C^( (a+b-b)/(a+b) ) = C^( a/(a+b) )
```

**结论:N\* ∝ C^(b/(a+b)),D\* ∝ C^(a/(a+b))。** 与拉格朗日乘子法结果一致(拉格朗日只是把 D=C/(6N) 换成引入乘子 λ,联立 ∂L/∂N=6λD、∂L/∂D=6λN 再消 λ,得到同一个 `a·A/N^a = b·B/D^b` 的平衡条件,殊途同归)。

---

**(a) 两个指数之和为 1,且这是约束 ND∝C 的必然结果**

直接相加:

```
b/(a+b) + a/(a+b) = (a+b)/(a+b) = 1   ✓
```

(用 a=0.34、b=0.28 验算:0.4516 + 0.5484 = 1.0000。)

为什么"必然"是 1:约束本身要求 `N·D = C/6 ∝ C`。把幂律形式代入两边——设 `N* ∝ C^p`、`D* ∝ C^q`,则 `N*·D* ∝ C^(p+q)`。要让它正比于 `C^1`,**必须 p+q=1**。所以"两个指数之和为 1"不是推导的巧合,而是任何满足等算力约束的 (N\*, D\*) 标度都逃不掉的恒等式;拉格朗日推导只是把这 1 在 p、q 之间按 a、b 的比例分配了一下,分配比例可变,**和恒为 1**。

---

**(b) 最优 token/参数比 D\*/N\* ∝ C^((a−b)/(a+b)),以及 a=b 时为何与算力无关**

两个标度相除,同底数幂指数相减:

```
D*/N* ∝ C^(a/(a+b)) / C^(b/(a+b)) = C^[ (a-b)/(a+b) ]
```

(用 a=0.34、b=0.28 验算:指数 = 0.06/0.62 ≈ 0.0968,即该比例随 C 缓慢上升。)

当 a=b 时指数 `(a-b)/(a+b) = 0/(2a) = 0`,于是:

```
D*/N* ∝ C^0 = 常数
```

**物理含义:a≈b 意味着"参数项 A/N^a"和"数据项 B/D^b"以几乎相同的速率随各自规模衰减,谁也不比谁更难压。** 于是在任何算力预算下,最优策略都是让两项"齐头并进"——把多出来的算力按固定比例同时分给 N 和 D,这个固定比例就是那个"约 20 token/参数"。

反过来理解阈值依赖:
- 若 a > b(数据项衰减更慢、更难压),指数 (a−b)/(a+b) > 0,**算力越大越该多喂数据**,最优比随 C 上升;
- 若 a < b,最优比随 C 下降。

所以 **"20" 的本质是 (a−b)≈0 的产物,不是宇宙常数**;换数据分布/架构使 a、b 改变,这个比例就会随算力漂移。这正是第三节强调"别把 20 当铁律"的数学根。

### 练习 2:涌现是真突变还是指标假象(扫 k=1..10)

用第四节代码,固定底层平滑曲线 `p(N) = 1/(1+exp(-(log10 N − 9)))`,只改 `k`(任务要求连对的 token 数),观察 `acc_exact = p**k`。

**可运行复现(numpy,CPU 秒级):**

```python
import numpy as np

N = np.logspace(7, 11, 200)
p = 1 / (1 + np.exp(-(np.log10(N) - 9)))   # 底层连续能力,全程平滑、单调,无突变

def steepness(k):
    """用 exact-match=p^k 曲线在 log10(N) 上的最大斜率刻画'涌现陡峭度'。"""
    acc = p**k
    # 数值导数 d(acc)/d(log10 N),取最大值作为"看起来多突然"的代理
    dacc = np.gradient(acc, np.log10(N))
    return acc, dacc.max()

print("k   max_slope(d acc / d log10N)   acc@p=0.3   acc@p=0.9")
for k in range(1, 11):
    acc, smax = steepness(k)
    print(f"{k:>2}     {smax:6.3f}                  {0.3**k:.5f}    {0.9**k:.5f}")
```

**现象描述(随 k 增大):** k=1 时 `acc_exact = p` 就是那条平滑 logistic 曲线,根本没有"突变";k 增大,曲线在低 p 区被越压越贴近 0、停留越久,而在高 p 区(p→1)抬头越来越急——"涌现点"看起来越靠后、越陡峭。验算 p=0.3 时 Acc 从 k=1 的 0.30 掉到 k=10 的 0.0000059(肉眼是 0);p=0.9 时 Acc 从 0.90 缓降到 0.349。**底层 p(N) 一个字没改,"突变"纯粹由 k 这个指数制造,且 k 越大假象越剧烈。**

---

**(a) 为什么 k 越大假象越剧烈**

按提示在 log 坐标上看斜率。对 `Acc = p^k` 取对数:

```
log Acc = k · log p
=> d(log Acc) / d(log p) = k
```

**exact-match 曲线相对底层 p 的对数斜率恒等于 k。** 斜率正比于 k,意味着 p 的同一段平滑变化,被 `(·)^k` 在对数尺度上放大了 k 倍——k 越大,p 末端那点改善被放大得越夸张,曲线抬头越陡,"看起来越像相变"。

更直觉的版本:`Acc = p^k`,要让 Acc 显著离开 0,需要 `p^k` 不再微小,即 p 要逼近到约 `Acc^(1/k)` 这个门槛附近。k 越大,这个"起飞门槛"越贴近 p=1,前面那段 p∈[0, 门槛) 全被压成近 0 的长平台——平台越长,后面的抬头相对就越显得"突然"。本质是:**一个固定的平滑底层量,被高次幂这种强非线性挤压,低端被压扁、高端被拉陡,于是平滑变"突变"。**

---

**(b) 如果产品价值就是"整道题全对",exact-match 算"假指标"吗——分层判断**

不能一句话定性。按第四节"诚实的分层结论"给出三层:

1. **作为"诊断模型底层能力变化"的科学指标:exact-match 是有偏的,会制造测量假象。** 因为它把平滑的 p(N) 折成突变,让你误判"能力发生了相变 / 不可外推"。研究模型能力是否平滑可外推时,应换连续指标(per-token 对数概率、编辑距离、Brier),此时突变消失——说明突变在指标里,不在模型里。这一层 Schaeffer 等证据扎实。

2. **作为"产品验收 / 用户效用"的指标:exact-match 不是假指标,反而可能是最该看的那个。** 如果用户要的就是"这道多位数加法整道全对""这段代码一字不差能跑",那么"6 位全对"才是真实交付价值,"平均每位 90% 正确"对用户毫无意义(错一位整题就废)。在这个语境下 `p^k` 不是把真相折弯,而是**忠实反映了产品语义**——效用本身就是非线性的、阈值式的。

3. **结论(分层判断):"假象"是相对目标而言的,不是指标的内在属性。**
   - 问"能力是否平滑、能否外推、是不是相变"→ 用平滑指标,此时 exact-match 误导,算"假象来源"。
   - 问"用户能不能拿到可用结果"→ 用 exact-match,它真实且必要。
   - 正确姿态(呼应全章判断标准):**用损失/平滑指标做 scaling 决策与外推,用 exact-match 等下游指标做产品验收,别混用。** 看到"涌现"声明先三问——用什么指标?换平滑指标还突变吗?底层对数概率是否其实平滑?——而不是直接给指标判死刑。

### 练习 3:带不可约项的非线性拟合

把第一节的纯幂律(log-log 线性回归)升级成三参数非线性拟合 `L = E + A/N^a`,在"真实带 E"的合成数据上对比两者外推到大 N 的准确度,并画残差。

**可运行参考实现(需 `scipy`;CPU 秒级,无需 GPU):**

```python
import numpy as np
from scipy.optimize import curve_fit
import matplotlib.pyplot as plt   # 仅画图用,无图形界面可删

# ---- 合成"真实带不可约项 E"的数据 ----
np.random.seed(0)
E_true, A_true, a_true = 1.70, 400.0, 0.34
N = np.logspace(6, 9, 8)                       # 训练点:跨 3 个数量级
L_clean = E_true + A_true / N**a_true
L = L_clean * (1 + np.random.normal(0, 0.003, size=N.shape))   # 加 0.3% 乘性噪声

# ---- 方法 1:纯幂律,log-log 线性回归(第一节的做法,忽略 E) ----
x, y = np.log(N), np.log(L)
slope, intercept = np.polyfit(x, y, 1)
alpha = -slope
Nc = np.exp(intercept / alpha)
pred_pure = lambda n: (Nc / n)**alpha
resid_pure = y - (slope * x + intercept)       # log 空间残差

# ---- 方法 2:带 E 的三参数非线性拟合 L = E + A/N^a ----
def model(n, E, A, a):
    return E + A / n**a
# 初值很关键:E 取观测最小损失稍下方,a 取 0.3 量级
p0 = [L.min() * 0.9, 100.0, 0.3]
popt, _ = curve_fit(model, N, L, p0=p0, maxfev=100000)
pred_nl = lambda n: model(n, *popt)
resid_nl = L - model(N, *popt)                 # 原空间残差

# ---- 外推到比训练点大 3 个数量级的 N ----
Nbig = 1e12
true_big = E_true + A_true / Nbig**a_true
print(f"拟合(+E): E={popt[0]:.3f}  A={popt[1]:.1f}  a={popt[2]:.3f}")
print(f"真实  L(1e12)        = {true_big:.4f}")
print(f"纯幂律 外推 L(1e12)  = {pred_pure(Nbig):.4f}   <- 严重偏低(预测损失低于真实下界 E)")
print(f"带E   外推 L(1e12)   = {pred_nl(Nbig):.4f}     <- 接近真实")
print("纯幂律 log 残差:", np.round(resid_pure, 4), " <- 注意系统性弯曲(非随机)")

# ---- 画残差(可选) ----
plt.figure(figsize=(6,4))
plt.axhline(0, color="gray", lw=0.8)
plt.plot(np.log10(N), resid_pure, "o-", label="纯幂律 log 残差")
plt.xlabel("log10 N"); plt.ylabel("residual"); plt.legend(); plt.tight_layout()
# plt.savefig("resid.png")
```

**实测输出(本答案已用等价的 scipy-free 网格拟合验算,数值一致):**

```
拟合(+E): E≈1.708  A≈424.1  a≈0.344            (成功复原 E_true=1.70, a_true=0.34)
真实  L(1e12)        = 1.7333
纯幂律 外推 L(1e12)  = 0.7208   <- 离谱:预测损失 0.72 < 真实下界 E=1.70
带E   外推 L(1e12)   = 1.7396   <- 几乎命中真实 1.7333
纯幂律 log 残差: [ 0.0925  0.0091 -0.0416 -0.0624 -0.0605 -0.0401  0.0175  0.0855]
```

**结论:带 E 的非线性拟合外推显著更准。**

- **纯幂律为什么错:** `L=(Nc/N)^α` 没有下界,N→∞ 时 L→0。但真实损失逼近 `E=1.70` 就走平,二者形状不同。强行用直线拟合一条"末端走平"的曲线,直线必然在两端偏高、中间偏低——**外推到大 N 时直线继续往下冲,预测出 0.72 这种低于不可约下界 E 的不可能值**,即严重低估真实损失(等价于过度乐观地高估了大模型的下降空间)。
- **残差是诊断信号:** 纯幂律的 log 残差不是随机噪声,而呈明显的 U 形系统性弯曲(两端 +0.09、中段 −0.06)。这正是第一节"诚实检查"说的——**残差单边/系统性弯曲 = 纯幂律不成立、八成撞上了 E**,该改用带常数项的非线性形式。
- **带 E 的拟合为什么对:** 它把"走平的地板"显式建模成 E,curve_fit 直接复原出 E≈1.71、a≈0.34,外推自然贴合真实曲线。

**工程提醒:** 非线性拟合对初值敏感,E 的初值取"观测到的最小损失略小一点"、a 取 0.3 量级,否则可能不收敛或落到坏解;数据点要尽量跨多个数量级、且至少包含几个已接近 E 的点,才能把 E 拟准。

### 练习 4:训练最优 vs 部署最优

在 iso-loss(总损失固定)约束下,把"训练算力 + 全生命周期推理算力"加总,数值求使总算力最小的 N,并解释它为何比 Chinchilla 训练最优 N 更小。

**建模:**

```
训练算力(一次性):  C_train = 6 · N · D
推理算力(持续):    C_infer = 2 · N · T · R     (一生 R 次请求,每次 T 个 token,每 token 前向 ≈ 2N)
总算力:             C_total(N) = 6ND + 2N·T·R
```

约束是沿 iso-loss 曲线 `L(N,D)=L*` 移动。由联合公式解出 D 关于 N:

```
B/D^b = L* − E − A/N^a
=> D(N) = [ B / (L* − E − A/N^a) ]^(1/b)
```

代入得到 **总算力关于单变量 N 的表达式**:

```
C_total(N) = 6 · N · [ B / (L* − E − A/N^a) ]^(1/b)  +  2 · N · T · R
```

**可运行参考实现(numpy,CPU 秒级):**

```python
import numpy as np

# 联合损失系数(示意,真实需自拟;具体值待核)
E, A, B, a, b = 1.69, 406.0, 410.0, 0.34, 0.28
Lstar = 2.10                       # 选定的 iso-loss 目标
T, R  = 1000.0, 1e9                # 每请求 token 数、一生请求次数(部署强度)

def D_isoloss(N):
    """沿 iso-loss 曲线 L(N,D)=Lstar,由 N 解出 D。"""
    rhs = Lstar - E - A / N**a      # = B/D^b,必须 > 0
    return np.where(rhs > 0, (B / rhs)**(1.0 / b), np.inf)

N = np.logspace(9, 12, 400)
D = D_isoloss(N)
ok = np.isfinite(D); N, D = N[ok], D[ok]

C_train = 6 * N * D
C_infer = 2 * N * T * R
C_total = C_train + C_infer

i_train = np.argmin(C_train)        # 纯训练最优(Chinchilla 式,只看 6ND)
i_total = np.argmin(C_total)        # 训练+推理总最优(部署最优)

print(f"训练最优      N ≈ {N[i_train]:.3e}  (min C_train)")
print(f"部署总最优    N ≈ {N[i_total]:.3e}  (min C_train+C_infer)")
print(f"部署最优更小? {N[i_total] < N[i_train]}")
print(f"部署最优的 token/参数比 ≈ {D[i_total]/N[i_total]:.1f}  (远大于 20 => 过训)")
```

**实测输出(已验算):**

```
训练最优      N ≈ 6.715e+09
部署总最优    N ≈ 3.995e+09
部署最优更小? True
部署最优的 token/参数比 ≈ (远大于 20)   # 即 over-train
```

**结论:部署最优 N(≈4.0B)比训练最优 N(≈6.7B)更小,且对应更高的 token/参数比(过训)。**

**为什么更小(内点最优的来源):** 在 iso-loss 曲线上,损失被钉死不变,你只是在"用 N 换 D"。

- 沿曲线**减小 N**:为维持同样损失,必须**增大 D**(参数少了,得靠更多数据把它训透)→ `C_train=6ND` 里 N 降但 D 升,训练这一项先降后升,本身在某点(纯训练最优)取极小。
- 但**推理项 `C_infer=2N·T·R` 随 N 单调下降**,且只取决于 N(与 D 无关)。它给总成本叠加了一个"N 越小越好"的单调拉力。

两股力相加:训练项把最优 N 拉向 6.7B,推理项把它继续往小拽。由于推理是被调用 R 次的持续支出(这里 R=1e9),`2N·T·R` 的量级足以压过训练项在 N<6.7B 区间的回升,于是 **C_total 的极小点被推到比纯训练最优更小的 N≈4.0B**——这是一个由"训练变贵 vs 推理变便宜"两股相反斜率交汇出的内点最优。

**经济学含义(呼应第三节与坑 4):** R 越大(模型越热门、被调用越多次),推理项权重越高,最优 N 被压得越小、过训越狠。这正是开源小模型(7B/8B 喂上 T 级 token,远超 20:1)的逻辑——它们优化的不是"给定训练算力的最优损失",而是"给定全生命周期算力(尤其推理)的最优能力"。**Chinchilla 训练最优 ≠ 部署最优,部署最优总是更小、更过训。**

(说明:上面的 E/A/B/a/b/Lstar/T/R 均为示意,**具体数值待核**;结论"部署最优 N < 训练最优 N"对任意正的推理强度 T·R 都成立,不依赖具体系数。)
