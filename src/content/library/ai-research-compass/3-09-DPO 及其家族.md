> 这一章把你从"知道 RLHF = 训奖励模型 + 跑 PPO"带到"能在白板上从 KL 正则化的 RLHF 目标一步步推出 DPO 损失,并解释清楚奖励模型为什么能被消掉、β 和 π_ref 在数学上各扮演什么角色"。读完你应当能向别人证明:DPO 不是一个工程 trick,而是 RLHF 目标在 Bradley-Terry 假设下的一个**精确的等价重写**——它把"先学奖励再做 RL"两步,合并成了一次封闭形式可解的分类损失。

## 为什么需要这一章:RLHF 的三步流水线太重了

先把上一章(PPO)和 RLHF 的标准流程钉死,这是本章的出发点。一个工业级的 RLHF 流水线长这样:

1. **SFT(监督微调)**:在高质量人类示范上做监督微调,得到一个基础策略,记作 π_ref(reference policy,参考策略)。它既是后面 RL 的初始化,也是 KL 约束的锚点。
2. **RM(奖励建模)**:收集人类**成对偏好**数据——给同一个 prompt x 两个回答 y_w(winner,人类更喜欢)和 y_l(loser),标注哪个更好。用这些数据拟合一个神经网络奖励模型 r_φ(x, y),让它在偏好数据上的预测概率最大。
3. **RL(PPO 优化)**:固定 r_φ,用 PPO 最大化"奖励减 KL 惩罚",让策略 π_θ 在涨奖励的同时别离 π_ref 太远。

这套流程能 work(InstructGPT、ChatGPT 都靠它),但它**重得离谱**,而且每一步都埋着坑:

- **要同时持有四个大模型**:训练时显存里通常同时活着 π_θ(actor,要更新)、π_ref(算 KL,冻结)、r_φ(算奖励,冻结)、以及 PPO 的 critic(价值网络,要更新)。四个百亿级模型同框,工程门槛和显存压力都极大。
- **RM 是一层间接,会被钻空子**:策略优化的是"奖励模型给的分",不是"真实人类偏好"。一旦策略找到了 RM 的某个盲区(比如发现"回答里多堆几个 markdown 标题就被打高分"),它会疯狂地往那个方向跑——这就是 **reward hacking(奖励黑客)**。RM 越偏离真实分布的区域,它的预测越不可信,而 RL 偏偏最擅长把策略推到那些区域。
- **PPO 超参极其敏感**:学习率、KL 系数、clip 范围、GAE 的 λ、rollout 长度……任何一个没调好都可能让训练发散或退化(模型只会说"我无法回答")。这是 on-policy RL 的通病:训练数据由当前策略实时采样,分布一直在变,稳定性天生脆弱。

**一个自然的问题是:我们真的需要那个显式的奖励模型,和那一整套 on-policy RL 吗?** RLHF 的最终产物是一个策略 π_θ,奖励模型 r_φ 只是个中间产物——训练完就扔掉了。能不能跳过它,直接从偏好数据训出策略?

DPO(Direct Preference Optimization,直接偏好优化,Rafailov et al., 2023)给出的回答是:**能,而且不损失任何东西**。它的核心洞察用一句话说就是——**RLHF 那个带 KL 约束的优化问题,最优策略有闭式解;反过来,这个闭式解允许我们用策略本身来表达奖励;把这个表达代回偏好概率公式,奖励模型就被消掉了,只剩下一个直接作用在策略上的、长得像逻辑回归的分类损失。**

这章的全部分量都压在那个推导上。所以推进路线是:

1. 写清 RLHF 的 KL 正则化目标,推出它的**闭式最优策略** π*(用两种方法:逐点求导 + 配分函数法,以及"凑成 KL"的优雅证法);
2. 从 π* **反解出奖励** r,看到奖励 = β·对数概率比 + 一个只依赖 x 的项;
3. 把 r 代入 **Bradley-Terry 偏好模型**,见证配分函数 Z(x) 在差值里**精确抵消**,得到 **DPO 损失**;
4. 解读这个损失的梯度,理解它"拉高 winner、压低 loser"的几何;搞清 β 和 π_ref 各管什么;
5. 拆 DPO 的家族:IPO、KTO、ORPO、SimPO,各自在补 DPO 的哪个洞;
6. 量化分析:DPO 相对 RLHF 在样本效率、稳定性、天花板上的得失,以及它的几个致命坑。

### 预备:把记号和两个前置事实摆好

后面推导密集,先统一记号,并明确两个我们直接拿来用的前置结论(它们在前面的章节里应已建立)。

**记号。**
- x:prompt(输入提示);y:completion(模型生成的完整回答,是一个 token 序列)。
- π_θ(y|x):当前策略对回答 y 的概率,等于各 token 条件概率的连乘 ∏ₜ π_θ(yₜ | x, y_{<t})。注意它是**整段回答的序列概率**,不是单个 token。
- π_ref(y|x):参考策略(通常就是 SFT 后的模型),冻结不动。
- r(x, y):奖励函数,给(prompt, 回答)打一个标量分。
- β > 0:KL 惩罚的强度系数(温度),控制"涨奖励"和"贴紧 π_ref"之间的权衡。
- (x, y_w, y_l):一条偏好样本,y_w 是人类偏好的(chosen),y_l 是被拒的(rejected)。
- σ(z) = 1 / (1 + e^{−z}):logistic sigmoid。

**前置事实一:KL 散度的定义。** 两个分布 p、q 在同一支撑集上的 KL 散度(Kullback-Leibler divergence,相对熵)是

```
KL(p ‖ q) = Σ_y p(y) · log( p(y) / q(y) ) = E_{y~p}[ log p(y) − log q(y) ]
```

它非负,当且仅当 p = q 时为 0。直觉:用 q 去编码服从 p 的数据,平均要多花多少 nat。本章它扮演"皮筋":惩罚策略偏离 π_ref。

**前置事实二:Bradley-Terry 偏好模型。** 这是奖励建模的统计学地基。它假设:人类偏好 y_w 胜过 y_l 的概率,由两者**奖励之差**经过一个 sigmoid 决定:

```
P(y_w ≻ y_l | x) = σ( r(x, y_w) − r(x, y_l) )
               = exp(r(x,y_w)) / ( exp(r(x,y_w)) + exp(r(x,y_l)) )
```

它的含义:奖励差越大,人类越确定地偏好 y_w;奖励相等时概率退化为 0.5(纯随机)。**标准的奖励建模,就是用最大似然在偏好数据上拟合 r_φ,使上式概率最大**——这本质是一个二分类(逻辑回归)。记住这个式子,它是后面"消掉 Z(x)"的舞台。

## 第一步:RLHF 目标的闭式最优策略

### 问题:KL 正则化的奖励最大化

RLHF 第三步要解的优化问题是:在给定一个奖励 r(x, y) 和参考策略 π_ref 的前提下,找一个策略 π 最大化奖励,同时被 KL 拴住别跑太远。形式化(对固定的 x,期望对 y ~ π(·|x) 取):

```
max_π  E_{y~π(·|x)}[ r(x, y) ]  −  β · KL( π(·|x) ‖ π_ref(·|x) )         …… (目标 J)
```

第一项让策略追高奖励,第二项(乘了 β)惩罚它偏离 π_ref。β → 0 时只顾刷奖励(容易 reward hacking 和模式崩溃),β → ∞ 时纹丝不动等于 π_ref。这是个**带约束(π 必须是合法概率分布,即非负且归一)的泛函优化问题**。下面解它,这是全章的数学核心。

### 解法一:逐点拉格朗日 + 配分函数(机械但稳)

把目标 J 展开,用 KL 的定义代入(为简洁,下面固定 x,把 π(·|x) 简记为 π(y),π_ref 同理):

```
J(π) = Σ_y π(y) · r(x,y)  −  β · Σ_y π(y) · log( π(y) / π_ref(y) )
```

约束是 Σ_y π(y) = 1(概率归一)。用一个拉格朗日乘子 λ 处理归一约束(非负约束我们事后验证最优解自动满足,无需显式引入),构造拉格朗日量:

```
L(π, λ) = Σ_y π(y)·r(x,y) − β·Σ_y π(y)·log(π(y)/π_ref(y)) + λ·( Σ_y π(y) − 1 )
```

对某个具体的 π(y) 求偏导(把 y 当成一个独立变量,逐点优化),令其为 0。注意 d/dπ(y) [ π(y)·log(π(y)/π_ref(y)) ] = log(π(y)/π_ref(y)) + π(y)·(1/π(y)) = log(π(y)/π_ref(y)) + 1:

```
∂L/∂π(y) = r(x,y) − β·[ log(π(y)/π_ref(y)) + 1 ] + λ = 0
```

解出 log(π(y)/π_ref(y)):

```
β·log( π(y)/π_ref(y) ) = r(x,y) + λ − β
log( π(y)/π_ref(y) )  = (1/β)·r(x,y) + (λ−β)/β
π(y) = π_ref(y) · exp( (1/β)·r(x,y) ) · exp( (λ−β)/β )
```

最后那个 exp((λ−β)/β) 是个**不依赖 y、只依赖 x 的常数**(因为 λ 是对固定 x 的归一乘子)。它的作用就是把整个分布归一化。我们不去解 λ,而是直接令归一化常数为

```
Z(x) = Σ_y π_ref(y) · exp( (1/β)·r(x,y) )         …… 配分函数(partition function)
```

于是把常数 exp((λ−β)/β) 定为 1/Z(x),得到**闭式最优策略**:

```
┌─────────────────────────────────────────────────────────┐
│   π*(y|x) = (1/Z(x)) · π_ref(y|x) · exp( r(x,y) / β )    │   …… (★ 闭式最优策略)
└─────────────────────────────────────────────────────────┘
```

验证三件事就放心了:(1) 它非负(π_ref ≥ 0,exp > 0,Z > 0),所以前面省略的非负约束自动满足;(2) 它归一(Σ_y π* = (1/Z)·Σ_y π_ref·exp(r/β) = Z/Z = 1);(3) 它是极大值而非极小值——因为 KL 项关于 π 是严格凸的,奖励项是线性的,整个 J 是严格凹的,逐点导数为零的驻点就是全局唯一最大值。

**(★) 是整章的引擎。** 它说:RLHF 的最优策略,就是把参考策略 π_ref 按"奖励的指数"做一次**重加权(reweighting)**——奖励越高的回答,概率被放大得越多,放大倍数是 exp(r/β)。β 控制这个放大的锐度:β 小则指数陡峭,高奖励回答被极端放大(贴近贪心);β 大则曲线平缓,接近原样保留 π_ref。这个解也叫 **Gibbs 分布 / Boltzmann 分布**,在统计物理里 r/β 对应"能量/温度",β 就是温度。

### 解法二:把目标凑成一个 KL(优雅,看透本质)

解法一机械可靠,但解法二能让你"一眼看穿"为什么最优解长那样,值得掌握。技巧是:**把整个目标 J 重写成一个负的 KL 散度加常数**,而 KL 在两分布相等时取最小(此处取负后即最大),最优策略就直接读出来了。

从 J(对固定 x,最大化等价于最小化 −J/β):

```
−J/β = E_{y~π}[ log(π(y)/π_ref(y)) − (1/β)·r(x,y) ]
     = E_{y~π}[ log( π(y) / ( π_ref(y)·exp(r(x,y)/β) ) ) ]
```

分母 π_ref(y)·exp(r/β) 还不是一个概率分布(不归一)。我们**人为补上配分函数 Z(x) 把它归一化**:定义 π*(y) = (1/Z(x))·π_ref(y)·exp(r(x,y)/β)——这正是(★)。于是 π_ref(y)·exp(r/β) = Z(x)·π*(y),代回:

```
−J/β = E_{y~π}[ log( π(y) / (Z(x)·π*(y)) ) ]
     = E_{y~π}[ log(π(y)/π*(y)) ] − log Z(x)
     = KL( π(·|x) ‖ π*(·|x) ) − log Z(x)
```

看清楚了:**最小化 −J/β,等价于最小化 KL(π ‖ π*)**(因为 log Z(x) 不含 π,是常数)。而 KL 散度在两分布相等时唯一取得最小值 0。所以最优策略就是

```
π_opt(·|x) = π*(·|x) = (1/Z(x))·π_ref(·|x)·exp(r(·|x)/β)
```

和解法一**完全一致**。这个证法的妙处在于它把"最优"的来源讲透了:RLHF 目标本质上就是在**让你的策略去匹配那个由奖励重加权出来的目标分布 π***,β 决定了重加权的强度。两种解法殊途同归,你应该都能独立写出来。

## 第二步:从最优策略反解奖励

(★) 给的是"已知 r 求 π*"。DPO 的关键一步是**反过来**:把(★)两边取对数,解出 r。从

```
π*(y|x) = (1/Z(x)) · π_ref(y|x) · exp( r(x,y)/β )
```

两边除以 π_ref、取对数:

```
log( π*(y|x) / π_ref(y|x) ) = r(x,y)/β − log Z(x)
```

移项,**用最优策略表达奖励**:

```
┌────────────────────────────────────────────────────────────┐
│   r(x,y) = β · log( π*(y|x) / π_ref(y|x) )  +  β · log Z(x)  │   …… (◆ 奖励的策略表达)
└────────────────────────────────────────────────────────────┘
```

这是 DPO 论文里最关键的一个等式(对应原文 Eq. 5)。它说:**任何一个奖励函数 r,只要它对应的最优策略是某个 π*,那么 r 就可以写成"β 乘以 π* 相对 π_ref 的对数概率比,再加上一个只依赖 x 的项 β·logZ(x)"。** 注意 Z(x) 仍然是个对 y 求和的天文数字(要遍历所有可能的回答序列),**根本算不出来**——但马上你会看到,我们不需要算它。

这里有个需要严谨对待的逻辑点(DPO 论文的 Lemma 也强调了):**对每一个奖励函数 r,都存在唯一对应的最优策略 π*;反过来,(◆) 把奖励参数化成了策略 π 的形式**。也就是说,我们可以**不再单独参数化一个奖励网络 r_φ**,而是直接令

```
r_θ(x,y) := β · log( π_θ(y|x) / π_ref(y|x) ) + β · log Z(x)
```

把策略 π_θ 自己当成隐式的奖励模型。**"语言模型本身就是奖励模型"——这就是 DPO 标题里 "Your Language Model Is Secretly a Reward Model" 的全部含义。** 但 Z(x) 这个拦路虎还在,下一步它会消失。

## 第三步:代入 Bradley-Terry,见证 Z(x) 抵消

现在把(◆)的奖励表达代入第一步预备里的 Bradley-Terry 偏好模型。回忆 BT 模型只关心**奖励之差** r(x, y_w) − r(x, y_l)。我们来算这个差(把 π* 换成我们要训练的 π_θ):

```
r_θ(x,y_w) − r_θ(x,y_l)
 = [ β·log(π_θ(y_w|x)/π_ref(y_w|x)) + β·logZ(x) ]
   − [ β·log(π_θ(y_l|x)/π_ref(y_l|x)) + β·logZ(x) ]
```

**关键时刻**:y_w 和 y_l 共享同一个 prompt x,所以两项里的 β·logZ(x) **完全相同**,相减直接抵消:

```
r_θ(x,y_w) − r_θ(x,y_l)
 = β·log(π_θ(y_w|x)/π_ref(y_w|x)) − β·log(π_θ(y_l|x)/π_ref(y_l|x))
```

**这就是 DPO 的全部魔法。** 那个永远算不出来的配分函数 Z(x),因为它只依赖 x 而不依赖具体回答 y,在"成对偏好的奖励差"里被精确消掉了——这正是 DPO 必须用**成对数据**、必须用 **BT(差值)模型**的根本原因:它需要一个减法来杀掉 Z(x)。

把这个差代回 BT 模型 P(y_w ≻ y_l | x) = σ(r(x,y_w) − r(x,y_l)),偏好概率变成纯粹由策略表达的式子:

```
P_θ(y_w ≻ y_l | x) = σ( β·log(π_θ(y_w|x)/π_ref(y_w|x)) − β·log(π_θ(y_l|x)/π_ref(y_l|x)) )
```

最后一步:对偏好数据集 D = {(x, y_w, y_l)} 做**最大似然估计**——让模型预测的偏好概率最大,等价于最小化负对数似然。于是得到:

```
┌──────────────────────────────────────────────────────────────────────────────┐
│  L_DPO(π_θ; π_ref) =                                                           │
│   − E_{(x,y_w,y_l)~D} [ log σ( β·log(π_θ(y_w|x)/π_ref(y_w|x))                   │
│                                  − β·log(π_θ(y_l|x)/π_ref(y_l|x)) ) ]          │
└──────────────────────────────────────────────────────────────────────────────┘
                                                                    …… (DPO 损失)
```

**到此推导闭环。** 我们从 RLHF 那个需要奖励模型 + on-policy RL 的复杂目标(目标 J)出发,一路推到一个**只需要成对偏好数据、可以离线监督训练、长得像逻辑回归**的损失。没有显式奖励模型,没有 PPO,没有 critic,没有采样 rollout。整条推导链:目标 J → 闭式解 π*(★) → 反解奖励 r(◆) → 代入 BT 差值消掉 Z → 最大似然 = L_DPO。**你应该能合上书把这五步在白板上独立复现——这是本章的硬指标。**

### 一个常被忽略的前提:它们真的等价吗?

诚实地说清边界:DPO 损失和"RM + PPO"在数学上等价,**严格成立有两个前提**:(1) 人类偏好真的服从 Bradley-Terry 模型;(2) 你能在整个策略空间里精确优化到最优。现实中两条都只是近似——偏好数据有噪声、BT 假设未必精确、神经网络优化也到不了全局最优。所以 DPO ≈ RLHF,而不是字面上的恒等。这个"≈"里藏着 DPO 的全部坑(第五节细讲),也是 IPO 等变体要修补的地方。**不要把它当成无条件的免费午餐。**

## 第四步:解读 DPO 损失——梯度的几何与超参的角色

推完公式,要理解它在训练时**实际在干什么**。先定义一个简写,叫**隐式奖励(implicit reward)**:

```
r̂_θ(x,y) = β · log( π_θ(y|x) / π_ref(y|x) )       (略去对 y 无关的 β·logZ)
```

DPO 损失就是 −E[ log σ( r̂_θ(x,y_w) − r̂_θ(x,y_l) ) ]。对它求梯度(令 Δ = r̂_θ(x,y_w) − r̂_θ(x,y_l),用 d/du logσ(u) = σ(−u) = 1−σ(u)):

```
∇_θ L_DPO = − E_{(x,y_w,y_l)} [ σ(−Δ) · ∇_θ Δ ]
          = − E [ σ( r̂_θ(x,y_l) − r̂_θ(x,y_w) )
                  · β·( ∇_θ log π_θ(y_w|x) − ∇_θ log π_θ(y_l|x) ) ]
```

逐项读这个梯度,每一块都有清晰含义:

- **∇_θ log π_θ(y_w|x) − ∇_θ log π_θ(y_l|x)**:更新方向是"**拉高 winner 的对数概率、压低 loser 的对数概率**"。这就是偏好优化最朴素的诉求,符合直觉。
- **σ( r̂_θ(x,y_l) − r̂_θ(x,y_w) ) 这个权重**:它是**自适应的难度门控**。当模型已经"答对了"(给 y_w 的隐式奖励远高于 y_l,Δ 很大正),这个权重 σ(负大数) ≈ 0,梯度几乎为零——**已经学会的样本不再浪费力气**。当模型"答反了"(给 y_l 的奖励反而更高,Δ 为负),权重 σ(正数) → 1,梯度满格——**模型把全部注意力放在还没学对、甚至学反了的样本上**。DPO 论文特别强调:正是这个权重项防止了模型退化,**朴素地只最大化 winner 概率(不带这个动态权重)会导致模型崩溃**。

**β 的角色(务必讲透)。** β 在两个地方出现,身份是统一的:它是 **KL 约束的强度**,等价地是**隐式奖励的尺度**。
- β 大 ⇒ KL 惩罚强 ⇒ 策略被牢牢拴在 π_ref 附近,更新保守,不容易跑偏但天花板低、学得慢。
- β 小 ⇒ KL 惩罚弱 ⇒ 策略可以大幅偏离 π_ref,更新激进,可能学得快但容易过拟合偏好、退化、或对 π_ref 失去锚定。
- 从梯度看:β 是梯度的整体缩放因子(∇Δ 里那个 β),同时藏在 r̂ 里影响权重 σ(·) 的锐度。**典型取值常在 0.1 量级(具体随任务与数据,待核;不同实现/论文报告的甜点区间不同)。**

**π_ref 的角色(同样务必讲透)。** π_ref 在 DPO 里有两个不可替代的作用:
1. **锚点 / 正则化器**:损失里出现的全是 π_θ 相对 π_ref 的**比值**,π_ref 把"绝对概率"变成"相对参考的偏移"。这等价于 RLHF 里的 KL 皮筋——防止模型为了拉偏好而把整个分布扭曲到不像话。**如果把 π_ref 去掉(令它为均匀分布),DPO 就退化成直接最大化 log π_θ(y_w) − log π_θ(y_l),实践中极易过拟合和退化。**
2. **数学上的必需品**:它来自(★)闭式解,是 KL 约束的对象,不是可有可无的工程选项。

一个微妙但重要的工程事实:**π_ref 在训练全程冻结,每步都要前向算 π_ref(y_w|x) 和 π_ref(y_l|x)。** 这意味着显存里要同时放 π_θ 和 π_ref 两个模型。比 RLHF 的四个少一半,但不是零——这正是 ORPO 等变体想进一步省掉的东西。

### 代码:DPO 损失的 PyTorch 实现

下面是 DPO 损失的核心实现,只保留体现机制的部分(省略数据加载、分布式等噪声)。关键在于:**对一段回答,序列对数概率 = 各 token 对数概率之和**;然后算两个 logratio、相减、过 logσ。

```python
import torch
import torch.nn.functional as F

def sequence_logprob(logits, labels, prompt_mask):
    """
    把模型对一段回答的 token-level logits,汇总成序列级对数概率 log π(y|x)。
    logits:      [B, T, V]  模型在每个位置对下一个 token 的预测分布(未归一)
    labels:      [B, T]     真实 token id(把回答区段填真值,prompt 区段任意)
    prompt_mask: [B, T]     1 表示该位置属于 prompt(不计入回答概率),0 表示属于回答
    返回:        [B]        每个样本的 log π(y|x) = Σ_t log π(y_t | x, y_<t)
    """
    # 标准的 teacher-forcing 错位:用第 t 步 logits 预测第 t+1 个 token
    logits = logits[:, :-1, :]              # [B, T-1, V]
    labels = labels[:, 1:]                  # [B, T-1]
    resp_mask = (1 - prompt_mask)[:, 1:]    # [B, T-1] 只在回答区段为 1

    logp = F.log_softmax(logits, dim=-1)                       # 归一成对数概率
    token_logp = logp.gather(-1, labels.unsqueeze(-1)).squeeze(-1)  # [B, T-1] 取真值 token 的 logp
    return (token_logp * resp_mask).sum(dim=-1)                # 回答区段求和 -> [B]

def dpo_loss(policy_logits_w, policy_logits_l,    # 策略 π_θ 对 winner / loser 的 logits
             ref_logits_w,    ref_logits_l,       # 参考 π_ref(冻结)对应的 logits
             labels_w, labels_l, pmask_w, pmask_l,
             beta=0.1):
    # 1) 四个序列对数概率
    logp_pi_w  = sequence_logprob(policy_logits_w, labels_w, pmask_w)  # log π_θ(y_w|x)
    logp_pi_l  = sequence_logprob(policy_logits_l, labels_l, pmask_l)  # log π_θ(y_l|x)
    with torch.no_grad():                                              # π_ref 冻结,不回传
        logp_ref_w = sequence_logprob(ref_logits_w, labels_w, pmask_w) # log π_ref(y_w|x)
        logp_ref_l = sequence_logprob(ref_logits_l, labels_l, pmask_l) # log π_ref(y_l|x)

    # 2) 两个 logratio = log(π_θ/π_ref),即隐式奖励 r̂/β
    logratio_w = logp_pi_w - logp_ref_w     # winner 的 log 比值
    logratio_l = logp_pi_l - logp_ref_l     # loser  的 log 比值

    # 3) 相减得到 Δ/β,再乘 β 得到隐式奖励差 Δ(配分函数 Z 已在此差值中抵消)
    delta = beta * (logratio_w - logratio_l)

    # 4) 过 logσ 取最大似然 = 最小化 -logσ(Δ)
    #    用 F.logsigmoid 保证数值稳定(等价 -softplus(-delta),不会溢出)
    loss = -F.logsigmoid(delta).mean()

    # 监控量:隐式奖励 / 偏好准确率(delta>0 即模型把 winner 排在前面)
    chosen_reward   = (beta * logratio_w).detach()
    rejected_reward = (beta * logratio_l).detach()
    reward_acc      = (delta > 0).float().mean()
    return loss, {"reward_acc": reward_acc,
                  "reward_margin": (chosen_reward - rejected_reward).mean()}
```

四个易错点,新手几乎都会踩:
1. **序列概率是 token 对数概率之**和**,不是均值**——长回答天然 logp 更负(后面 SimPO 会盯上这件事)。
2. **prompt 区段不能计入回答概率**——只对 completion 的 token 求和,否则把 prompt 也优化了。
3. **π_ref 必须 no_grad 冻结**——它是锚点不是被训对象,且省一半反传显存。
4. **用 F.logsigmoid 而非手写 log(sigmoid(·))**——后者在 |Δ| 大时数值溢出;logsigmoid 内部用 softplus 稳定实现。

## 第五步:DPO 的家族——每个变体在补哪个洞

DPO 优雅,但第三节末尾那个"≈"暴露了它的弱点。家族成员都是冲着某个具体缺陷来的。逐个讲清"它修什么、怎么修"。

### IPO:修 BT 假设下的过拟合

**问题。** DPO 的推导依赖 Bradley-Terry 模型。当某对样本的偏好是**确定性的**(人类 100% 选 y_w,数据里这对从不翻车),BT 模型为了让 σ(Δ) → 1,会驱使 Δ = β·(logratio_w − logratio_l) → +∞。也就是说,**DPO 会无限制地拉大隐式奖励差,把 logratio_l 压向 −∞**——这意味着把 loser 的概率压到接近 0,即使 loser 本身可能并不糟。KL 正则在这里**失效了**:理论上 β 应该拦住这种过度偏离,但当偏好确定时,损失的最优解就在无穷远处,模型会一路狂奔过拟合,π_ref 的锚定形同虚设。这是 DPO 在**干净/确定偏好数据**上的典型失败模式。

**IPO(Identity Preference Optimization,Azar et al., 2023)怎么修。** 它不套 sigmoid 这个无界的链接函数,而是把目标改成**让隐式奖励差去回归一个有限的目标值**(具体是让 logratio 之差等于 1/(2β) 这样的常数),用一个平方损失(均方误差形式)代替 logσ:

```
L_IPO = E_{(x,y_w,y_l)} [ ( (logπ_θ(y_w)−logπ_ref(y_w)) − (logπ_θ(y_l)−logπ_ref(y_l)) − 1/(2β) )² ]
```

平方损失有**有限的最优点**:它要的是"奖励差等于某个常数",而不是"奖励差越大越好"。所以即使偏好确定,模型也只把差拉到目标值就停,不再无限发散。**代价**是它放弃了 BT 的概率解释,且对目标常数的选择敏感。一句话:IPO = 把 DPO 的"越分越开"换成"分到刚好就停"。(上式形式按 IPO 论文意旨给出,常数项的精确写法以原文为准,待核。)

### KTO:免成对数据,只要"好/坏"二元标签

**问题。** DPO/IPO 都要**成对**数据(同一 prompt 的 y_w 和 y_l),标注昂贵——你得让人对着两个回答比较。但现实中大量反馈是**单边**的:用户点了个赞或踩、一条回答被标"可接受"或"不可接受",没有配对。

**KTO(Kahneman-Tversky Optimization,Ethayarajh et al., 2024)怎么做。** 它借鉴**前景理论(prospect theory,Kahneman & Tversky)**:人对收益和损失的感受是不对称的(损失带来的痛 > 等量收益带来的爽,即损失厌恶),且都相对一个**参考点**衡量。KTO 把这套人类价值函数搬进损失:对每条样本(x, y),先算它的隐式奖励 r̂_θ = β·log(π_θ(y|x)/π_ref(y|x)) 和一个参考点(通常用 batch 内 KL 项的估计当基准),然后:
- 若 y 是**好**样本:用一个**凹**的价值函数奖励"奖励超过参考点"的部分(收益区,边际递减);
- 若 y 是**坏**样本:用损失厌恶的方式惩罚"奖励高于参考点"(损失区,惩罚更陡)。

**好处**:只需单条样本的二元 desirable/undesirable 标签,标注成本骤降,且天然能处理好坏样本数量不平衡。**代价**:引入了价值函数的形状超参,理论解释从"概率似然"换成了"行为经济学",离 BT 的干净推导更远。一句话:KTO = 把"哪个更好"的成对比较,换成"这个好不好"的单边判断 + 损失厌恶。

### ORPO:连参考模型都省掉,把对齐塞进 SFT

**问题。** DPO 训练时还要持有冻结的 π_ref,占显存;而且流程上仍是"先 SFT 得到 π_ref,再单独跑一轮 DPO",两阶段。

**ORPO(Odds Ratio Preference Optimization,Hong et al., 2024)怎么做。** 它把偏好对齐**直接融进 SFT**,一个阶段搞定,**不需要参考模型**。损失 = 标准 SFT 的负对数似然(在 y_w 上)+ 一个 **odds ratio(几率比)**惩罚项。几率(odds)定义为 odds(y|x) = π(y|x) / (1 − π(y|x)),惩罚项大致是 −log σ( log( odds_θ(y_w|x) / odds_θ(y_l|x) ) ),用几率比把 winner 相对 loser 拉开。关键是:**它直接用当前模型 π_θ 的 odds,不引用任何 π_ref**,因为 SFT 项本身就提供了"别跑太远"的锚定(模型在 y_w 上做最大似然,自然贴住数据分布)。

**好处**:省掉 π_ref(显存减半)、单阶段(无需先 SFT 再对齐)。**代价**:失去了 KL 约束那种显式的偏离控制,锚定全靠 SFT 项,理论保证更弱。一句话:ORPO = 把对齐惩罚塞进 SFT 损失,用 odds ratio 代替 KL,免参考模型。

### SimPO:连参考模型也不要,且对齐"长度归一化的概率"

**问题(补充,非用户必答但重要)。** DPO 用序列对数概率之和当隐式奖励,这与**生成时**的解码目标(常按长度归一化)不一致,会引入**长度偏置**——模型可能学会"回答越长隐式奖励越高",输出注水变长。

**SimPO(Simple Preference Optimization,Meng et al., 2024)怎么做。** 两个改动:(1) 用**长度归一化的平均对数概率**(序列 logp 除以 token 数)当隐式奖励,直接对齐生成目标、消长度偏置;(2) **去掉 π_ref**,并在奖励差里加一个**目标 margin γ**(要求 winner 的归一化奖励比 loser 高出至少 γ)。损失形如 −logσ( (β/|y_w|)·logπ_θ(y_w) − (β/|y_l|)·logπ_θ(y_l) − γ )。**好处**:无参考模型、显式去长度偏置、常报告有竞争力的效果。**代价**:去掉 π_ref 后锚定更弱,对 γ 和数据质量更敏感。

一句话总览这个家族的演化轴:**DPO(成对+BT+参考)→ IPO(防确定性过拟合)→ KTO(去成对,要单边标签)→ ORPO/SimPO(去参考模型,塞进/简化训练)**。它们都在同一张光谱上权衡"理论严谨 vs 工程轻量 vs 数据成本"。

## 量化分析:DPO 相对 RLHF 的得与失

把账算清楚,别只停在"DPO 更简单"的感性层面。

**计算与显存复杂度。**
- RLHF(PPO):训练时同框 4 个模型(actor + ref + reward + critic),其中 2 个要更新;每步要**在线采样 rollout**(自回归生成,慢且非并行)、算优势、多个 PPO epoch。
- DPO:同框 2 个模型(π_θ + π_ref),仅 1 个更新;**数据完全离线**(偏好对预先采好),每步只是 4 次前向 + 1 次反传,**没有自回归采样**。
- **结论:DPO 的单步成本和显存约为 PPO 的一半甚至更低,且因无在线采样,吞吐高得多、复现容易得多。** 这是 DPO 最硬的工程优势。

**稳定性(偏差-方差视角)。**
- PPO 是 on-policy RL,梯度本质是策略梯度估计,**方差大**(靠 GAE、baseline、clip 压方差),训练曲线抖、对超参极敏感。
- DPO 是离线监督式损失(本质逻辑回归),**梯度是低方差的全监督梯度**,训练像普通微调一样平稳。**这是 DPO"稳定、易调"口碑的数学根源**:它把一个高方差的 RL 优化,换成了低方差的分类优化。

**样本效率与数据分布。** 这是 DPO 的**软肋**,务必讲清:
- DPO 是**离线**的,只能从固定的偏好数据集学习,数据由**某个旧策略**采出。一旦 π_θ 被更新,它就**偏离了生成偏好数据的那个分布**——这是经典的 **distribution shift(分布偏移)**问题。模型在数据没覆盖到的区域,隐式奖励的外推是不可信的,可能学出奇怪行为。
- RLHF 是 **on-policy**:rollout 由当前策略实时生成,奖励模型对**当前策略的输出**打分,数据分布始终跟着策略走,能持续探索新区域。**这给了 RLHF 在困难任务上更高的天花板**——它能发现并强化离线数据里压根不存在的好回答。
- **结论:DPO 用"放弃 on-policy 探索"换来了"稳定 + 轻量"。在偏好数据覆盖良好的对齐任务上,二者天花板相近;在需要探索、数据覆盖差的硬任务上,RLHF/on-policy 方法往往更高(待核:具体差距随任务与数据规模,文献结论不完全一致,有工作报告 DPO 持平甚至更好,也有报告 PPO 在某些设定上更优)。**

**一个理论上的 subtlety:DPO 的可行域比 RLHF 大。** 有分析指出(DPO 后续理论工作),DPO 损失最小化不要求隐式奖励对应一个合法的、能由 π_ref 重加权得到的策略,**它可能找到 RLHF 永远到不了的解**——其中有些是好的(更灵活),有些是病态的(把 OOD 区域的概率乱推)。这把双刃剑既是 DPO 灵活性的来源,也是它某些失败模式的根源。(此为活跃研究议题,具体结论待核。)

### DPO 的常见坑(RL 对齐高发区)

- **坑 1:winner 和 loser 的概率一起降。** 很多人盯着"winner 概率有没有涨",却发现训练中 **log π_θ(y_w) 和 log π_θ(y_l) 双双下降**,只是 loser 降得更快(margin 在涨,损失在降,但绝对概率都在掉)。这是 DPO 被反复报告的现象:模型可能在**整体压低偏好数据区域的概率**,把概率质量挪到了 OOD 的地方,埋下退化隐患。**监控时不能只看 reward margin,要同时看 chosen reward 的绝对走向。** 缓解手段之一是加一个 SFT 正则项(在 y_w 上的 NLL),把 winner 概率拽住。
- **坑 2:π_ref 选错。** π_ref 必须是**在偏好数据分布上合理**的模型(通常就是同一批数据 SFT 出来的)。如果 π_ref 和偏好数据来自不同分布,logratio 会失真,训练跑偏。**经验法则:π_ref 用与偏好数据同源的 SFT 模型。**
- **坑 3:β 调反方向。** β 太小,模型疯狂偏离 π_ref、过拟合偏好、输出退化(复读、注水、谄媚);β 太大,几乎学不动。新手常因为"loss 降得慢"就调小 β,结果触发退化。**先从中等 β(如 0.1)起,观察 chosen reward 是否塌陷再调。**
- **坑 4:长度偏置 / 谄媚(sycophancy)。** 因为序列 logp 是求和,DPO 隐含偏好更长的回答;偏好数据若本身带"长答案更受喜欢"的标注偏置,会被放大成注水。SimPO 的长度归一化、或显式长度去偏是应对手段。
- **坑 5:把 DPO 当成无脑替代 RLHF。** DPO 在数据覆盖好的对齐上性价比极高,但在需要在线探索的硬任务(复杂推理、agent)上,纯离线 DPO 天花板可能不够,需要 on-policy 方法(PPO/GRPO)或迭代式 DPO(用当前模型重新采样偏好对再训,把 DPO 半 on-policy 化)。

## 动手练习

**练习 1(推导题,核心)。** 不看本章,从 RLHF 目标
```
max_π  E_{y~π}[ r(x,y) ] − β·KL(π ‖ π_ref)
```
出发,完整推出 DPO 损失。要求写清:(a) 用拉格朗日法或"凑 KL"法解出闭式最优策略 π*(y|x) = (1/Z(x))·π_ref(y|x)·exp(r(x,y)/β);(b) 反解出 r(x,y) = β·log(π*/π_ref) + β·logZ(x);(c) 代入 Bradley-Terry,**明确指出 Z(x) 为什么在差值里抵消**;(d) 写出最终 L_DPO。*提示:卡在哪一步就回到对应小节,但第二遍必须独立完成——这是检验你是否真的"会推"的唯一标准。*

**练习 2(分析题)。** 解释 β → 0 和 β → ∞ 两个极限下,(★)的 π* 分别趋向什么分布?对应 DPO 训练会出现什么行为?再回答:为什么去掉 π_ref(令其为均匀分布)后 DPO 容易退化?*提示:从(★)看 β 对指数项锐度的影响;从 DPO 梯度看 π_ref 如何把"绝对概率"变成"相对比值",失去它意味着什么约束消失了。*

## 小结与承上启下

这一章把对齐这条线上最优雅的一个结果讲透了:**RLHF 那个需要奖励模型 + on-policy RL 的复杂目标,在 Bradley-Terry 假设下有一个精确的等价重写——DPO 损失。** 推导链条值得刻进肌肉记忆:KL 正则目标 → 闭式最优策略 π* = (1/Z)·π_ref·exp(r/β) → 反解出奖励 r = β·log(π/π_ref) + β·logZ → 代入 BT 偏好差让 Z 抵消 → 最大似然得到 L_DPO。整个魔术的支点,是配分函数 Z(x) 只依赖 prompt、在成对差值里被精确消掉。

把 DPO 放进整门课的坐标里:上一章(PPO)是 on-policy RL 对齐的主力,门槛高但天花板高、能探索;这一章 DPO 是离线偏好优化的代表,用"放弃探索"换"稳定 + 轻量",在数据覆盖好的对齐任务上性价比极高。两者不是替代而是**互补的两极**——你要根据"数据是否离线、任务是否需要探索、工程预算多少"来选,甚至把它们缝起来(迭代式 DPO、先 DPO 再 PPO)。

接下来的章节会把镜头转向**推理模型**:当奖励不再来自学出来的 RM、也不来自成对偏好,而是来自**可验证的信号**(数学答案对不对、代码能不能跑过测试),对齐和推理训练会长出全新的形态——RLVR 与 GRPO。你在这一章建立的"KL 约束下的策略优化"直觉,会在那里继续发力:GRPO 仍然是 KL 约束 + 策略梯度,只是把 PPO 的 critic 用"组内相对奖励"替换掉了。DPO 教会你"奖励可以被消掉",下一站会教你"奖励可以被验证器替代"。

### 练习 1:从 RLHF 目标完整推出 DPO 损失

这是本章硬指标,下面给出完整五步,每一步都自洽闭合。

**(a) 解闭式最优策略 π\*。**

固定一个 prompt x,简记 π(y)=π(·|x)、π_ref(y)=π_ref(·|x)。目标(对固定 x)是

```
J(π) = Σ_y π(y)·r(x,y) − β·Σ_y π(y)·log( π(y)/π_ref(y) )      约束:Σ_y π(y) = 1
```

引入归一约束的拉格朗日乘子 λ(非负约束事后验证自动满足):

```
L(π,λ) = Σ_y π(y)·r(x,y) − β·Σ_y π(y)·log(π(y)/π_ref(y)) + λ·(Σ_y π(y) − 1)
```

对单个 π(y) 逐点求偏导。用到 d/dπ(y) [ π(y)·log(π(y)/π_ref(y)) ] = log(π(y)/π_ref(y)) + 1,令导数为 0:

```
∂L/∂π(y) = r(x,y) − β·[ log(π(y)/π_ref(y)) + 1 ] + λ = 0
```

解出对数比值,再取指数:

```
log( π(y)/π_ref(y) ) = (1/β)·r(x,y) + (λ−β)/β
π(y) = π_ref(y)·exp( r(x,y)/β )·exp( (λ−β)/β )
```

最后那个因子只依赖 x、不依赖 y,作用就是归一化。直接令归一化常数为配分函数

```
Z(x) = Σ_y π_ref(y)·exp( r(x,y)/β )
```

得到**闭式最优策略**:

```
π*(y|x) = (1/Z(x))·π_ref(y|x)·exp( r(x,y)/β )            …… (★)
```

三项验证:非负(π_ref≥0、exp>0、Z>0,故省略的非负约束自动满足);归一(Σ π* = (1/Z)·Σ π_ref·exp(r/β) = Z/Z = 1);是最大值(KL 项关于 π 严格凸、奖励项线性,故 J 严格凹,唯一驻点即全局最大)。

**(等价的「凑 KL」证法,验证同一结果。)** 最大化 J 等价于最小化 −J/β:

```
−J/β = E_{y~π}[ log(π(y)/π_ref(y)) − r(x,y)/β ]
     = E_{y~π}[ log( π(y) / (π_ref(y)·exp(r/β)) ) ]
```

补上 Z(x) 把分母归一化,即 π_ref(y)·exp(r/β) = Z(x)·π\*(y):

```
−J/β = E_{y~π}[ log( π(y) / (Z(x)·π*(y)) ) ] = KL( π ‖ π* ) − log Z(x)
```

log Z(x) 与 π 无关,故最小化 −J/β ⇔ 最小化 KL(π‖π\*),在 π=π\* 处唯一取得最小 0。两法结论一致。

**(b) 反解奖励 r。** 把(★)两边除以 π_ref 再取对数:

```
log( π*(y|x)/π_ref(y|x) ) = r(x,y)/β − log Z(x)
```

移项得**奖励的策略表达**:

```
r(x,y) = β·log( π*(y|x)/π_ref(y|x) ) + β·log Z(x)          …… (◆)
```

把待训练策略 π_θ 替换 π\*,就用策略本身隐式地参数化了奖励:r_θ(x,y) = β·log(π_θ/π_ref) + β·logZ(x)。Z(x) 是对所有回答序列求和的天文数字,算不出来——但下一步它会消失。

**(c) 代入 Bradley-Terry,Z(x) 抵消。** BT 模型只关心奖励之差。算 r_θ(x,y_w) − r_θ(x,y_l):

```
r_θ(x,y_w) − r_θ(x,y_l)
 = [ β·log(π_θ(y_w|x)/π_ref(y_w|x)) + β·logZ(x) ]
   − [ β·log(π_θ(y_l|x)/π_ref(y_l|x)) + β·logZ(x) ]
```

**关键:y_w、y_l 共享同一个 prompt x**,所以两项的 β·logZ(x) 是**同一个数**,相减精确抵消:

```
r_θ(x,y_w) − r_θ(x,y_l) = β·log(π_θ(y_w|x)/π_ref(y_w|x)) − β·log(π_θ(y_l|x)/π_ref(y_l|x))
```

Z(x) 只依赖 x 不依赖具体回答,因此在「同 prompt 成对回答的奖励差」里被一个减法杀掉——这就是 DPO 必须用**成对数据 + BT(差值)模型**的根本原因:它需要这个减法。(数值核验:在离散 y-空间上随机取 π_ref 与 r,反解(◆)后两两作差,得到的差与真实 r 差完全相等,误差约 1e-16。)

**(d) 最大似然得到 L_DPO。** 把差代回 P(y_w≻y_l|x)=σ(r(x,y_w)−r(x,y_l)):

```
P_θ(y_w ≻ y_l | x) = σ( β·log(π_θ(y_w|x)/π_ref(y_w|x)) − β·log(π_θ(y_l|x)/π_ref(y_l|x)) )
```

对偏好集 D 做最大似然(= 最小化负对数似然):

```
L_DPO(π_θ; π_ref) = − E_{(x,y_w,y_l)~D} [ log σ( β·log(π_θ(y_w|x)/π_ref(y_w|x))
                                              − β·log(π_θ(y_l|x)/π_ref(y_l|x)) ) ]
```

**推导闭环:目标 J → 闭式解 π\*(★) → 反解奖励 r(◆) → 代入 BT 差值消掉 Z → 最大似然 = L_DPO。** 全程无显式奖励模型、无 PPO、无 critic、无 rollout。

### 练习 2:β 两极限下 π* 的行为,以及去掉 π_ref 为何退化

**β → 0(KL 惩罚→0,只顾刷奖励)。** 看(★)的指数项 exp(r/β):β→0 时指数被无限放大,r 的微小差异被指数级拉开。π\* 把几乎全部概率质量压到**奖励最高的那个(那些)回答**上,退化为对 argmax_y r(x,y) 的(近似)**确定性 / one-hot 分布**(贪心)。

- 对应 DPO 训练行为:KL 约束几乎消失,策略可以无限偏离 π_ref。模型疯狂追逐隐式奖励差,极易 **reward hacking / 模式崩溃**——输出退化(复读、注水、谄媚、只说固定套话),多样性塌掉。loss 可能降得很快,但 chosen reward 的绝对概率往往一起塌陷。

**β → ∞(KL 惩罚→∞,纹丝不动)。** exp(r/β)→exp(0)=1,(★)退化为 π\* = π_ref。奖励完全失去影响力,最优策略就是参考策略本身。

- 对应 DPO 训练行为:KL 皮筋绷到极紧,梯度的整体缩放因子 β 虽大,但隐式奖励差 r̂=β·log(π_θ/π_ref) 要求 π_θ 几乎贴死 π_ref,**模型学不动**,偏好信号推不动策略,训练近乎停滞、对齐无效果。

**结论:** β 是指数项锐度 / KL 强度的统一旋钮。β 小 → 分布尖锐、激进、易退化;β 大 → 贴住 π_ref、保守、学不动。实践取中等值(如 0.1 量级,具体甜点区随任务待核),先观察 chosen reward 是否塌陷再调。

**为什么去掉 π_ref(令其均匀)会退化。** 均匀分布对每个 y 是常数 1/|Y|,代入隐式奖励:

```
r̂_θ(x,y) = β·log( π_θ(y|x) / (1/|Y|) ) = β·log π_θ(y|x) + β·log|Y|
```

常数项 β·log|Y| 在 winner−loser 差里抵消,于是 DPO 退化为**直接最大化 log π_θ(y_w) − log π_θ(y_l)**(绝对对数概率之差)。问题出在两处:

1. **失去锚点 / 正则化**:原损失里全是 π_θ 相对 π_ref 的**比值**,π_ref 把「绝对概率」翻译成「相对参考的偏移」,等价于 RLHF 的 KL 皮筋。去掉它,就没有任何东西拦着模型为了拉大偏好差而把整个分布扭曲到不像话——可以无节制地推高 log π_θ(y_w)、压低 log π_θ(y_l),哪怕 loser 并不糟,实践中**极易过拟合偏好并退化**(与练习 4 的「崩溃」同源)。
2. **数学上的必需品**:π_ref 来自(★)闭式解,是 KL 约束的对象,不是可选工程项;均匀分布并非任何真实 SFT 模型,logratio 失去意义。

(后续变体 ORPO/SimPO 确实去掉了显式 π_ref,但它们另寻锚定:ORPO 靠 SFT 项把模型拽在数据分布上,SimPO 靠长度归一化 + margin γ,而非简单地令 π_ref 均匀。)

### 练习 3:把 dpo_loss 改写成 IPO,并对比 logratio_l 曲线

IPO 的唯一改动:把无界的 `-logσ(Δ)` 换成**有限最优点的平方损失**,让 `(logratio_w − logratio_l)` 回归常数 `1/(2β)`。

下面给两份代码。第一份是按本章 `dpo_loss` 接口的 **IPO 损失 PyTorch 实现**(直接替换那一行);第二份是一个**纯 numpy、CPU 秒跑、无需任何大依赖**的最小对比实验,把两个 logratio 当作可学标量参数,复现「DPO 把 logratio_l 推向 −∞、IPO 收敛到有限值」。

**(1) IPO 损失(PyTorch,替换 dpo_loss 的第 3–4 步)。**

```python
import torch.nn.functional as F

def ipo_loss(policy_logits_w, policy_logits_l, ref_logits_w, ref_logits_l,
             labels_w, labels_l, pmask_w, pmask_l, beta=0.1):
    # 1) 四个序列对数概率(sequence_logprob 沿用本章实现)
    logp_pi_w  = sequence_logprob(policy_logits_w, labels_w, pmask_w)
    logp_pi_l  = sequence_logprob(policy_logits_l, labels_l, pmask_l)
    with torch.no_grad():
        logp_ref_w = sequence_logprob(ref_logits_w, labels_w, pmask_w)
        logp_ref_l = sequence_logprob(ref_logits_l, labels_l, pmask_l)

    # 2) 两个 logratio = log(π_θ/π_ref)
    logratio_w = logp_pi_w - logp_ref_w
    logratio_l = logp_pi_l - logp_ref_l

    # 3) IPO:让 (logratio_w − logratio_l) 回归有限目标 1/(2β),用平方损失
    #    对比 DPO 的 -F.logsigmoid(beta*(logratio_w - logratio_l))
    h = logratio_w - logratio_l                  # 隐式奖励差 / β
    loss = ((h - 1.0 / (2.0 * beta)) ** 2).mean()  # 有限最优点:差到目标就停,不发散
    return loss, {"h_mean": h.detach().mean()}
```

要点:平方损失的最优解是「差 = 常数」,而非「差越大越好」,所以即使偏好 100% 确定,模型把差拉到 1/(2β) 就停手,不会把 loser 压向 −∞。代价是放弃了 BT 的概率解释,且对目标常数敏感。(常数项 1/(2β) 按 IPO 论文意旨给出,精确写法以原文为准,待核。)

**(2) 最小可跑对比实验(纯 numpy,CPU 直接 `python3 xxx.py`;若想画图把 matplotlib 部分解注释)。**

```python
import numpy as np
# import matplotlib.pyplot as plt   # 可选:画图时解注释

def sigmoid(z): return 1.0 / (1.0 + np.exp(-z))

# 把 winner / loser 的 logratio 当作两个可学标量参数 lw, ll。
# 「偏好极度确定」= winner 永远被偏好,DPO 会把 Δ=β(lw-ll) 推向 +∞、ll 推向 -∞。
beta, lr, steps = 0.1, 0.5, 4000

# ---- DPO: L = -log σ(β(lw-ll)),  dL/dΔ = -σ(-Δ) ----
lw, ll = 0.0, 0.0
dpo_ll = []
for _ in range(steps):
    delta = beta * (lw - ll)
    g = -sigmoid(-delta)               # dL/dΔ
    lw -= lr * (g * beta)              # dΔ/dlw = +β
    ll -= lr * (g * (-beta))           # dΔ/dll = -β
    dpo_ll.append(ll)

# ---- IPO: L = ((lw-ll) - 1/(2β))^2,  dL/d(·) = 2·resid ----
lw2, ll2, tgt = 0.0, 0.0, 1.0 / (2 * beta)
ipo_ll = []
for _ in range(steps):
    resid = (lw2 - ll2) - tgt
    g = 2 * resid
    lw2 -= lr * (g * 1.0)
    ll2 -= lr * (g * (-1.0))
    ipo_ll.append(ll2)

for s in (100, 500, 1000, 2000, 4000):
    print(f"step {s:5d} | DPO ll={dpo_ll[s-1]:9.3f} | IPO ll={ipo_ll[s-1]:9.4f}")
print(f"末步 DPO Δ={beta*(lw-ll):.3f}(持续增大), IPO (lw-ll)={lw2-ll2:.5f} 目标={tgt:.5f}")

# plt.plot(dpo_ll, label="DPO loser logratio"); plt.plot(ipo_ll, label="IPO loser logratio")
# plt.xlabel("step"); plt.ylabel("logratio_l"); plt.legend(); plt.savefig("ipo_vs_dpo.png")
```

```
step   500 | DPO ll=   -7.521 | IPO ll=   0.0000
step  1000 | DPO ll=  -10.891 | IPO ll=   0.0000
step  2000 | DPO ll=  -14.484 | IPO ll=   0.0000
step  4000 | DPO ll=  -18.107 | IPO ll=   0.0000
末步 DPO Δ=3.621(持续增大), IPO (lw-ll)=0.00000 目标=5.00000
```

**结论:** DPO 的 logratio_l 单调下滑、无收敛趋势(−2.2→−7.5→−10.9→−14.5→−18.1,Δ 持续增大),正是「偏好确定时把 loser 概率压向 −∞」的过拟合发散;IPO 在有限步内把 `(lw−ll)` 钉到目标 1/(2β)=5 后就**停在有限值**(本对称设定里 lw=−ll,故 ll 稳定在 0),不再发散。要复现发散,关键是构造「偏好极度确定」的样本(winner 恒优),否则 DPO 的 σ(−Δ) 权重会自然衰减、看不到失控。

> 说明:本实验把序列 logratio 抽象成自由标量,聚焦「有界 vs 无界损失」这一机制差异;接到真实模型时,只需把 `lw/ll` 换成 `dpo_loss`/`ipo_loss` 里算出的 `logratio_w/logratio_l`,结论形态不变。画图时解注释 matplotlib 即可得到 logratio_l 随步数的两条曲线(DPO 持续下探、IPO 平台收敛)。

### 练习 4:推导 DPO 梯度,解释自适应权重,以及去掉权重为何崩

**推导 ∇_θ L_DPO。** 记隐式奖励 r̂_θ(x,y) = β·log(π_θ(y|x)/π_ref(y|x)),并令

```
Δ = r̂_θ(x,y_w) − r̂_θ(x,y_l) = β·[ log(π_θ(y_w|x)/π_ref(y_w|x)) − log(π_θ(y_l|x)/π_ref(y_l|x)) ]
```

则 L_DPO = − E[ log σ(Δ) ]。用关键导数 **d/du log σ(u) = 1 − σ(u) = σ(−u)**:

```
∇_θ L_DPO = − E[ d/dΔ logσ(Δ) · ∇_θ Δ ] = − E[ σ(−Δ) · ∇_θ Δ ]
```

其中 π_ref 冻结、不含 θ,所以

```
∇_θ Δ = β·( ∇_θ log π_θ(y_w|x) − ∇_θ log π_θ(y_l|x) )
```

又 σ(−Δ) = σ( r̂_θ(x,y_l) − r̂_θ(x,y_w) ),代入即得:

```
∇_θ L_DPO = − E_{(x,y_w,y_l)} [ σ( r̂_θ(x,y_l) − r̂_θ(x,y_w) )
                                · β·( ∇_θ log π_θ(y_w|x) − ∇_θ log π_θ(y_l|x) ) ]
```

这就证明了**梯度前的权重正是 σ(r̂_θ(x,y_l) − r̂_θ(x,y_w))**。

**这个权重为什么是「自适应难度门控」(一句话)。** 它衡量「模型把 loser 排在 winner 前面的程度」:模型已学会时(r̂_w ≫ r̂_l,Δ 大正)权重 σ(负大数)≈0、梯度几乎为零(已学会就停手),模型学反时(r̂_l > r̂_w,Δ 负)权重 σ(正数)→1、梯度满格(把全部力气放到学错/最难的样本上)。

**去掉权重为何会崩。** 若改成朴素地最大化 log π_θ(y_w|x) − log π_θ(y_l|x),等价于把上式权重恒置为常数(去掉 σ(·) 这个动态门控),梯度永远满格、且**没有有限的停止点**:

- 失去「已学会就停手」机制:对每条样本,梯度都一味地推高 log π_θ(y_w)→0(概率→1)、压低 log π_θ(y_l)→−∞(概率→0),不管该样本是否已经分对、也不管 loser 是否其实不糟。
- 后果是**模型崩溃 / 退化**:概率质量被无节制地从 loser 区域抽走、过度堆到 winner 上(甚至挪到 OOD 区域),分布被扭曲到不合理(复读、注水、谄媚、多样性塌陷)。这正是 DPO 论文强调的——**正是 σ(·) 权重项防止了退化,朴素最大化偏好差会让模型崩**。它与练习 2「去掉 π_ref」的失败同源:都丢掉了「适可而止」的约束(一个是动态难度门控,一个是 KL 锚点),让损失的最优解跑到无穷远。
