> 这一章把你从"知道 CNN 是一堆卷积加池化"带到"能逐层算出任意 CNN 架构的感受野、参数量、FLOPs,能从第一性原理解释每一次架构突破到底解决了上一代的什么瓶颈,能亲手写出 VGG block 与 Inception block 并说清它们的设计权衡"。这不是一份"架构图册"——每一个架构(LeNet / AlexNet / VGG / GoogLeNet / ResNeXt / DenseNet / SENet / EfficientNet / MobileNet)我们都问同一个问题:**它前面那一代卡在哪了,它用什么数学/工程手段把这个瓶颈撬开,代价是什么**。读完你应该能把"为什么是 3×3 不是 5×5""为什么 1×1 卷积能降算力""为什么深度/宽度/分辨率要联合缩放"这类问题从头推一遍,而不是背结论。这一章是后面所有视觉骨干网络(检测、分割、自监督、ViT 对比)的地基。

## 一、动机:把"特征工程"这件事交给梯度

在深度学习之前,计算机视觉的主流范式是**手工特征 + 浅层分类器**:你设计 SIFT、HOG 这类描述子去提取边缘、纹理、梯度方向直方图,再喂给 SVM。这套范式的天花板很低,原因是**人脑设计的特征是固定的、低层的,无法随任务和数据自适应**。

卷积神经网络(CNN)的根本主张是:**不要手工设计特征,而是把"提取什么特征"也变成可优化的参数,用梯度下降端到端学出来**。CNN 之所以是视觉的天然选择,是因为它把图像的两个先验硬编码进了网络结构:

- **局部性(locality)**:一个像素的语义主要由它邻域决定,所以用小的卷积核只看局部窗口,而不是像全连接那样让每个输出连接所有像素。
- **平移等变性(translation equivariance)**:猫在图像左上角还是右下角都是猫,所以同一个卷积核**在整张图上共享权重**滑动,而不是为每个位置学一套独立权重。

这两个先验直接带来**参数量的巨幅压缩与样本效率的提升**。我们先把这个量化出来,这是理解后面一切的基础。

**给有工程/数学背景、视觉从零的读者的迁移提示**:你熟悉"定义损失、梯度下降优化参数"这个范式,CNN 没有跳出它。CNN 唯一陌生的地方在于**它的参数被结构性地约束成了"在空间上共享的小滤波器"**。本章要让你形成的核心肌肉是:看到任意一个卷积层配置,能立刻心算出它的输出尺寸、参数量、FLOPs、以及它对最终感受野的贡献。这三个量(尺寸 / 参数 / 算力)外加感受野,就是评判一切 CNN 架构设计的四把尺子。

### 1.1 先把四把尺子立起来:卷积层的尺寸、参数、FLOPs、感受野

考虑一个标准 2D 卷积层:输入特征图通道数 Cᵢₙ、空间尺寸 H×W,卷积核大小 K×K,输出通道数 Cₒᵤₜ,步长 S,padding P。

**输出空间尺寸**(单边)的公式必须烂熟于心:

```
H_out = ⌊(H − K + 2P) / S⌋ + 1
```

直觉:能放下卷积核的起始位置有多少个。`H−K` 是核左边缘能滑动的最大距离,加 `2P` 是两侧补零放宽了边界,除以步长 S 是每次跳 S 格,`+1` 算上起始位置那一格。

**参数量**(权重,先不含 bias):

```
Params = Cₒᵤₜ × Cᵢₙ × K × K
```

直觉:输出的每个通道都需要一组"看遍所有输入通道的 K×K 立方体"的权重,共 Cₒᵤₜ 个这样的立方体。注意**参数量与特征图空间尺寸 H、W 无关**——这正是权重共享的结果,是 CNN 相对全连接最关键的省参来源。

**FLOPs**(浮点运算次数,这里按乘加各算一次,即乘加对 = 2 FLOPs;很多论文只算乘加对的个数 MACs,差一个 2 倍因子,读数时务必看清口径):

```
MACs = H_out × W_out × Cₒᵤₜ × Cᵢₙ × K × K
FLOPs ≈ 2 × MACs
```

直觉:输出特征图上每个位置(共 H_out·W_out·Cₒᵤₜ 个)都要做一次 `Cᵢₙ·K·K` 规模的点积。**和参数量不同,FLOPs 与空间尺寸成正比**——这就是为什么浅层(分辨率大、通道少)往往是算力大头,深层(分辨率小、通道多)往往是参数大头。这个不对称是后面 EfficientNet 复合缩放和 MobileNet 设计的物理根源。

**感受野(receptive field)**:输出特征图上一个神经元,能"看到"原始输入图像上多大的区域。这是衡量一层/一个网络"视野"的核心量。单层 K×K 卷积感受野就是 K。多层堆叠时按下面的递推累加,我们在 VGG 一节会完整推导。

我把这四把尺子单独拎出来,是因为**后面每一次架构革命,本质都是在这四个量之间重新分配预算**:AlexNet 用 GPU 撑起了 FLOPs 预算;VGG 在固定感受野下省参数、加非线性;Inception 用 1×1 卷积砍 FLOPs;ResNeXt/MobileNet 用分组卷积同时砍参数和 FLOPs;EfficientNet 则研究如何在总预算给定时把它们三者按最优比例分配。

## 二、LeNet-5(1998):卷积 + 池化范式的雏形

LeNet-5 由 Yann LeCun 等人提出,用于手写数字识别(MNIST 那一类任务)。它第一次把**卷积层(提局部特征)→ 下采样/池化层(降分辨率、增大感受野、引入一点平移不变性)→ 全连接层(分类)**这条流水线跑通,这套宏观结构几乎被之后二十年所有 CNN 沿用。

它的典型结构(经典版本,细节按 LeCun 原文,激活为 tanh/sigmoid):

```
输入 32×32×1
→ Conv 5×5, 6 个核     → 28×28×6     （C1）
→ 平均池化 2×2, stride 2 → 14×14×6     （S2）
→ Conv 5×5, 16 个核    → 10×10×16    （C3）
→ 平均池化 2×2, stride 2 → 5×5×16      （S4）
→ 全连接 120 → 全连接 84 → 输出 10
```

LeNet 的**历史意义**在于确立了两个直觉:

1. **逐级抽象**:浅层卷积学边缘/笔画,深层卷积在前者基础上组合出更复杂的部件。下采样让深层神经元的感受野覆盖越来越大的输入区域,从而能"看到"更全局的结构。
2. **池化的作用**:平均/最大池化在缩小特征图的同时,提供了对小幅平移的**不变性**(invariance,注意区别于卷积的等变性)——目标在窗口内挪动几个像素,池化输出基本不变。

LeNet 当时没有大规模铺开,原因不是架构错,而是**算力(没有 GPU)、数据(没有大规模标注集)、以及深层网络的优化手段(tanh/sigmoid 饱和导致梯度消失)都还不到位**。这三个缺口,正是 14 年后 AlexNet 一次性补上的。

## 三、AlexNet(2012):三个关键改动引爆深度学习

AlexNet(Krizhevsky、Sutskever、Hinton)在 ImageNet 大规模图像分类竞赛(ILSVRC-2012,1000 类、百万级图片)上把 top-5 错误率从此前最好的约 26% 大幅降到约 16%(具体数值「待核」,量级无误),用压倒性优势宣告了深度 CNN 的时代。它的网络比 LeNet 深、宽得多(5 个卷积层 + 3 个全连接层,约 6000 万参数「待核」),但真正让它成立的是**三个具体技术,每一个都针对一个明确的优化/正则瓶颈**。

### 3.1 ReLU:把梯度消失的源头之一直接拆掉

LeNet 用的 sigmoid σ(x)=1/(1+e^(−x)) 和 tanh,导数有个致命问题:**在 |x| 较大时导数趋近于 0(饱和区)**。我们算一下 sigmoid 的导数:

```
σ(x) = 1/(1+e^(−x))
σ'(x) = σ(x)·(1−σ(x))
```

σ'(x) 在 x=0 时取最大值 0.25,只要 x 偏离 0,导数就迅速衰减到接近 0。在反向传播里,梯度是逐层导数连乘的(链式法则):

```
∂L/∂(浅层) = ∂L/∂(深层) · ∏(每层激活导数 · 权重)
```

每经过一个 sigmoid,梯度就被乘上一个 ≤0.25 的因子。**L 层网络梯度量级可能被压到 0.25^L 量级**——10 层就衰减到约 10⁻⁶,浅层几乎收不到学习信号。这就是**梯度消失**(vanishing gradient)。

ReLU(Rectified Linear Unit)的定义朴素到极致:

```
ReLU(x) = max(0, x)
ReLU'(x) = 1  (x>0);  0  (x<0)
```

它的关键性质:**在正区间导数恒为 1,梯度连乘时不衰减**(也不爆炸)。这让梯度能畅通地传回浅层,深网络第一次变得可训练。代价是 x<0 时导数为 0,可能导致**死亡 ReLU**(神经元永久不激活),这催生了 LeakyReLU、PReLU、GELU 等变体(后续章节细讲)。此外 ReLU 计算只是一个比较+取最大,比 exp 快得多,这在当年算力紧张时也是实打实的收益。

**这里要建立的核心因果**:AlexNet 能比 LeNet 深这么多,首要功臣不是参数多,而是 ReLU 把"深"这件事在优化上变得可行。

### 3.2 Dropout:用随机失活做集成正则

参数一多(几千万),在百万级数据上仍会过拟合。Dropout 的做法是**训练时以概率 p 随机把一部分神经元的输出置零**:

```
训练时: y = mask ⊙ x / (1−p),  mask 中每个元素以概率 (1−p) 取 1、p 取 0
推理时: y = x   （不丢弃,因训练时已除以 (1−p) 做了期望校正）
```

这里除以 `(1−p)` 是 **inverted dropout** 的标准做法,目的是保持训练/推理时该层输出的**期望一致**:训练时每个元素被保留的概率是 (1−p),保留后又除以 (1−p),所以 E[y]=x,推理时直接用 x 即可,无需再缩放。

它为什么能正则?两个等价直觉:(1)**强迫特征冗余**:任何神经元都可能突然消失,网络不能让某个输出过度依赖单个特征,逼出更鲁棒、更分散的表征;(2)**近似模型集成**:每个 mask 相当于一个子网络,Dropout 近似在指数多个共享权重的子网络上做集成平均,而集成是经典的降方差手段。

### 3.3 数据增强 + GPU:把数据预算和算力预算同时拉满

- **数据增强(data augmentation)**:随机裁剪、水平翻转、颜色抖动(AlexNet 用了 PCA 颜色扰动)。本质是**利用图像任务已知的不变性先验,人工扩充训练分布**——翻转后还是同一个物体,所以可以白嫖一倍数据,直接降过拟合。
- **GPU**:AlexNet 把模型拆到两块 GTX 580(各 3GB 显存)上并行训练,这是显存约束下的工程妥协,也第一次证明 **GPU 是深度学习的算力基础设施**。

**AlexNet 的总结论**:它本身的架构并不优雅(大核 11×11、人为的双 GPU 分组),真正的贡献是**用 ReLU 解决可训练性 + Dropout/增强解决过拟合 + GPU 解决算力**,把"深度学习能 work"这件事在大规模数据上一次性验证。之后所有工作都是在这个既定事实上做架构精化。

## 四、VGG(2014):用堆叠 3×3 小卷积把"深度即非线性"做到极致

VGG(Simonyan & Zisserman, 牛津 VGG 组)提出了一个极简却影响深远的主张:**抛弃大卷积核,整个网络只用 3×3 卷积堆叠 + 2×2 池化,靠加深层数(VGG-16 / VGG-19)提升性能**。这个选择背后是一个可以严格推导的洞察:**多个小卷积堆叠,能用更少参数、更多非线性,达到和单个大卷积相同的感受野**。这是本章第一个必须自己会推的核心结论。

### 4.1 感受野递推公式(先建立工具)

设第 l 层的卷积核大小 kₗ、步长 sₗ。记 rₗ 为第 l 层输出神经元在**输入图像**上的感受野大小,jₗ 为该层相邻神经元在输入上的间隔(jump,即累计步长)。递推公式:

```
j₀ = 1,  r₀ = 1                （输入像素自己:间隔 1,感受野 1）
jₗ = j_{l−1} × sₗ
rₗ = r_{l−1} + (kₗ − 1) × j_{l−1}
```

第二式的直觉:在第 l 层,一个核横跨 kₗ 个上一层神经元,相邻上一层神经元在输入上间隔 j_{l−1},所以这个核相比单个上一层神经元,额外多覆盖了 `(kₗ−1)×j_{l−1}` 的输入范围,叠加到上一层已有的感受野 r_{l−1} 上。

### 4.2 两个 3×3 = 一个 5×5(感受野),三个 3×3 = 一个 7×7

现在用上面的公式,在 **stride=1** 的纯卷积堆叠下推导。stride=1 时 jₗ 恒为 1,递推简化为 rₗ = r_{l−1} + (kₗ−1)。

```
单个 5×5:         r = 1 + (5−1) = 5
两个 3×3 串联:
  第1层 3×3:      r₁ = 1 + (3−1) = 3
  第2层 3×3:      r₂ = 3 + (3−1) = 5    ✓ 等于一个 5×5

单个 7×7:         r = 1 + (7−1) = 7
三个 3×3 串联:
  r₁=3, r₂=5, r₃ = 5 + 2 = 7            ✓ 等于一个 7×7
```

**结论一(感受野等价)**:n 个 3×3 卷积串联(stride=1)的感受野是 `2n+1`。两个 3×3 ≡ 一个 5×5;三个 3×3 ≡ 一个 7×7。

### 4.3 参数量对比:小核堆叠省参数

设输入输出通道都是 C(同维,便于公平比较),忽略 bias。

```
一个 5×5 卷积:        参数 = C × C × 5 × 5 = 25 C²
两个 3×3 卷积:        参数 = 2 × (C × C × 3 × 3) = 18 C²
                      省了 (25−18)/25 = 28%

一个 7×7 卷积:        参数 = C × C × 7 × 7 = 49 C²
三个 3×3 卷积:        参数 = 3 × (C × C × 3 × 3) = 27 C²
                      省了 (49−27)/49 ≈ 45%
```

**结论二(省参数)**:相同感受野下,小核堆叠的参数量显著更少,且感受野越大、堆叠越深,省得越多。

### 4.4 非线性更多:这才是更关键的收益

参数省下是其一,**更本质的收益是非线性增多**。一个 5×5 卷积后只跟一个 ReLU,整体是"一次线性变换 + 一次非线性";而两个 3×3 之间各插一个 ReLU,就有了"线性 → ReLU → 线性 → ReLU"。**两层非线性比一层能表达更复杂的决策函数**——直觉上,单个 ReLU 只能切一刀(分段线性的一个折点),两个 ReLU 堆叠能构造更精细的分段线性函数,函数类的表达能力更强。

这就是 VGG 的核心设计哲学:**把一个"宽而浅"的大核,拆成"窄而深"的小核串联,在不损感受野的前提下,同时拿到更少参数和更强非线性。** 这个"用深度换表达力"的思想,直接铺垫了后面 ResNet 把网络推到上百层的动机。

### 4.5 VGG block 的 PyTorch 实现

```python
import torch.nn as nn

def vgg_block(in_ch, out_ch, num_convs):
    """一个 VGG block:num_convs 个 3×3 卷积(全程保持空间尺寸),末尾一次 2×2 池化下采样。
    关键点:padding=1 配合 K=3、stride=1,保证卷积不改变 H、W(same 卷积)。
    """
    layers = []
    for i in range(num_convs):
        layers += [
            nn.Conv2d(in_ch if i == 0 else out_ch, out_ch,
                      kernel_size=3, padding=1),   # H、W 不变:(H−3+2·1)/1+1 = H
            nn.ReLU(inplace=True),                 # 每个 3×3 后都接非线性 —— VGG 的灵魂
        ]
    layers.append(nn.MaxPool2d(kernel_size=2, stride=2))  # 空间尺寸减半
    return nn.Sequential(*layers)

# VGG-16 的卷积部分:每个 block 内卷积数与输出通道
# (num_convs, out_ch): (2,64) (2,128) (3,256) (3,512) (3,512)
```

### 4.6 VGG 的代价(坑)

VGG 优雅,但有个大坑:**它的参数 90% 以上压在最后的全连接层**。VGG-16 约 1.38 亿参数「待核」,其中第一个全连接层 `7×7×512 → 4096` 一层就吃掉约 1.0 亿参数。这说明 **VGG 时代还没意识到全连接层是参数黑洞**——这个浪费被下一节的 GoogLeNet 用全局平均池化(global average pooling)彻底解决。记住这个对比:它是"卷积省参 vs 全连接耗参"这一矛盾的最佳教材。

## 五、GoogLeNet / Inception(2014):多尺度并行 + 1×1 瓶颈

GoogLeNet(Szegedy 等,Google)和 VGG 同年夺冠,走的却是完全不同的路:**不靠单纯加深,而是设计一个更精巧的基本模块 Inception,在控制算力的前提下做"多尺度特征提取"**。它要解决两个问题:(1)物体在图像里尺度不一,该用多大的核?(2)怎么在不爆炸算力的前提下加宽网络?

### 5.1 Naive Inception:多尺度并行的直觉与它的灾难

朴素想法:**既然不知道该用 1×1、3×3 还是 5×5,那就全都用,并行算完再把通道拼起来(concat)**,让网络自己学着用哪个尺度。

```
        ┌─ 1×1 conv ─┐
输入 ───┼─ 3×3 conv ─┼─→ 通道维 concat → 输出
        ├─ 5×5 conv ─┤
        └─ 3×3 maxpool ┘
```

这个想法对(多尺度感知),但**直接做算力会炸**。算一下:输入 28×28×192,若 5×5 分支直接输出 32 通道:

```
5×5 分支 MACs = 28×28 × 32 × 192 × 5×5 ≈ 1.2 × 10⁸
```

仅一个分支就上亿 MACs,几个模块、几十个通道分支叠起来完全不可接受。更糟的是 concat 让通道数只增不减,层层放大。

### 5.2 1×1 卷积:降算力的真正机理(本章必考)

Inception 的破局点是 **1×1 卷积**(由 "Network in Network" 一文引入)。先想清楚 1×1 卷积到底在干什么:**它在每个空间位置上,对该位置的 Cᵢₙ 维通道向量做一次线性变换,映射到 Cₒᵤₜ 维**,完全不混合空间邻域(因为核就 1×1)。所以它等价于"在通道维度上的一个全连接层,在所有空间位置共享"。

它的两个用途:

1. **跨通道信息融合**:在通道维做线性组合,允许网络重组特征。
2. **降维瓶颈(bottleneck)——降算力的关键**:用 1×1 把通道数先压低,再做昂贵的大核卷积,算完(可选)再升回去。

**为什么能降算力?定量推导**。沿用 5×5 分支,输入 28×28×192,想最终输出 32 通道。

```
方案 A(直接 5×5):
  MACs = 28×28 × 32 × 192 × 25 ≈ 1.20 × 10⁸

方案 B(先 1×1 降到 16 通道,再 5×5 到 32 通道):
  1×1 降维:  28×28 × 16 × 192 × 1   ≈ 2.41 × 10⁶
  5×5 卷积:  28×28 × 32 × 16  × 25  ≈ 1.00 × 10⁷
  合计 ≈ 1.24 × 10⁷

A / B ≈ 1.20×10⁸ / 1.24×10⁷ ≈ 9.7 倍
```

**结论:加一层 1×1 降维瓶颈,5×5 分支算力降到约 1/10。** 机理一句话:**昂贵的大核卷积 FLOPs 正比于 Cᵢₙ×Cₒᵤₜ,而 1×1 卷积很便宜(核只有 1×1),先用它把 Cᵢₙ 砍小,大核就只在低维上运算了。** 1×1 自身的开销远小于它为大核省下的开销,净赚。这就是"瓶颈结构"——先掐细再放开——的算力本质,后面 ResNet bottleneck、ResNeXt、MobileNet 全在用它。

### 5.3 带降维的 Inception 模块 + 全局平均池化

实际 Inception 模块在 3×3、5×5 前各加 1×1 降维,在 maxpool 后加 1×1 调整通道:

```
        ┌─ 1×1 ─────────────────────┐
        ├─ 1×1(降维) → 3×3 ────────┤
输入 ───┤                           ├─→ concat → 输出
        ├─ 1×1(降维) → 5×5 ────────┤
        └─ 3×3 maxpool → 1×1 ───────┘
```

此外 GoogLeNet 还做了一件影响深远的事:**用全局平均池化(GAP)替代 VGG 那个巨大的全连接层**。GAP 把最后的 H×W×C 特征图在空间上直接求平均,得到一个 C 维向量,**零参数**,再接一个小分类头。回顾 VGG 那 1 亿参数的全连接,GAP 几乎把它清零,这是 GoogLeNet 参数量(约 600 万「待核」)只有 VGG-16(1.38 亿「待核」)零头的主因。GAP 还有正则效果(强制每个通道对应一类语义的全局响应)。

### 5.4 Inception block 的 PyTorch 实现

```python
import torch
import torch.nn as nn

class Inception(nn.Module):
    def __init__(self, in_ch, c1, c3_reduce, c3, c5_reduce, c5, pool_proj):
        super().__init__()
        # 分支1:单 1×1
        self.b1 = nn.Conv2d(in_ch, c1, kernel_size=1)
        # 分支2:1×1 降维 → 3×3(padding=1 保持空间尺寸)
        self.b2 = nn.Sequential(
            nn.Conv2d(in_ch, c3_reduce, 1), nn.ReLU(True),
            nn.Conv2d(c3_reduce, c3, kernel_size=3, padding=1), nn.ReLU(True))
        # 分支3:1×1 降维 → 5×5(padding=2 保持空间尺寸)
        self.b3 = nn.Sequential(
            nn.Conv2d(in_ch, c5_reduce, 1), nn.ReLU(True),
            nn.Conv2d(c5_reduce, c5, kernel_size=5, padding=2), nn.ReLU(True))
        # 分支4:3×3 池化(保持尺寸) → 1×1 调通道
        self.b4 = nn.Sequential(
            nn.MaxPool2d(3, stride=1, padding=1),
            nn.Conv2d(in_ch, pool_proj, 1), nn.ReLU(True))

    def forward(self, x):
        # 四个分支输出空间尺寸全相同,沿通道维(dim=1)拼接
        return torch.cat([self.b1(x), self.b2(x), self.b3(x), self.b4(x)], dim=1)
```

注意所有分支的 padding 都精心设置成**保持 H、W 不变**,否则无法 concat。这是实现 Inception 时最容易踩的坑:**四个分支的输出空间尺寸必须严格一致**。

## 六、为什么"更深更好"?以及随之而来的退化问题

VGG 和 GoogLeNet 都用经验证明了"更深更好",但这背后有理论支撑也有陷阱,这一节把它讲清楚,作为引出 ResNet 的桥梁。

### 6.1 深度为什么带来表达力:指数级的分段线性区域

一个直观且有理论依据(Montufar 等关于深度网络分段线性区域数目的工作)的论证:**带 ReLU 的网络表示的是分段线性函数,而它能划分出的线性区域数目,随深度指数增长、随宽度多项式增长。** 也就是说,**加深比加宽在"提升函数复杂度"上效率更高**。直觉:每加一层非线性,相当于在前一层已经折叠的空间上再折一次,折叠次数随层数指数累积。这解释了为什么 VGG 把核拆细加深、为什么深度是性能的主要杠杆。

### 6.2 退化问题:深到一定程度,训练误差反而上升

但实践中出现了一个反直觉现象(He 等在 ResNet 论文中明确指出):**把一个浅网络加深(比如从 20 层堆到 56 层),不仅测试误差变差,连训练误差也变高了。**

这关键在于**它不是过拟合**——过拟合是训练误差低、测试误差高。这里训练误差本身就上升,说明是**优化问题**:更深的网络更难被优化到一个好解。

逻辑上这很荒谬:**理论上深网络的表达能力 ⊇ 浅网络**。因为如果新增的层都学成恒等映射(identity,输出=输入),深网络就退化成那个浅网络,训练误差至少不该更差。**但实验表明,SGD 很难让一堆非线性层去逼近恒等映射**——让 `ReLU(Wx+b) ≈ x` 这件事对优化器并不容易。这就是**退化问题(degradation)**:不是网络没能力表达,而是当前的优化方法找不到那个好解。

### 6.3 这直接引出残差学习

ResNet 的破题思路异常简洁(下章细讲,这里点出动机闭环):既然让层去学恒等映射很难,那就**改变学习目标**——不让一组层去学习目标映射 H(x),而是让它学习**残差** F(x) = H(x) − x,然后输出 `F(x) + x`(通过一条 shortcut 把输入直接加到输出)。

```
普通层:   输出 = H(x)              想退化成恒等 ⇒ 需让 H(x)=x,难
残差块:   输出 = F(x) + x          想退化成恒等 ⇒ 只需 F(x)=0,即把权重压到 0,易
```

**关键洞察:把权重学成接近 0,远比把一堆非线性层学成精确的恒等变换容易。** 这一个改动让上百层、上千层网络变得可训练,是深度学习架构史上最重要的突破之一。本章只到此为止建立动机,机制留给下一章。

## 七、现代演进:在"基本模块"层面继续榨取效率

ResNet 之后,架构创新的主战场从"宏观怎么连"转向"**微观的 block 怎么设计,才能在固定算力/参数预算下榨出更多性能**"。下面四个方向各代表一种正交的思路。

### 7.1 ResNeXt(2017):分组卷积引入"基数"这第三个维度

ResNeXt(Xie 等)在 ResNet bottleneck 基础上,把中间那个 3×3 卷积换成**分组卷积(grouped convolution)**,并提出**基数(cardinality)**这个新维度——即分组的数量。

**分组卷积是什么**:把输入的 Cᵢₙ 个通道平均切成 g 组,每组 Cᵢₙ/g 通道,各自独立做卷积,输出也分 g 组拼起来。**组间不通信**。它的参数量和 FLOPs 相对普通卷积降低为 1/g:

```
普通卷积参数:   Cₒᵤₜ × Cᵢₙ × K²
分组卷积参数:   g × (Cₒᵤₜ/g) × (Cᵢₙ/g) × K² = (Cₒᵤₜ × Cᵢₙ × K²) / g
```

ResNeXt 的论点:**在相同 FLOPs/参数预算下,增加基数(分更多组、每组更窄)比单纯加深或加宽更能提升精度**。直觉是分组卷积让网络学习多条相对独立的变换路径再聚合(split-transform-aggregate),类似 Inception 的多分支但结构高度规整(所有分支拓扑相同)。基数因此成为深度、宽度之外**调节模型的第三个旋钮**。

### 7.2 DenseNet(2017):密集连接做极致特征复用

DenseNet(Huang 等)把残差的"相加"换成"**拼接**",并且让一个 block(dense block)内**每一层都接收前面所有层的特征图作为输入**:

```
ResNet:    xₗ = F(x_{l−1}) + x_{l−1}              （特征相加)
DenseNet:  xₗ = F([x₀, x₁, ..., x_{l−1}])         （前面所有层特征 concat 后再卷积)
```

每层只新产出很少的通道(增长率 growth rate k,如 k=12 或 32),但因为能直接访问前面所有层的特征,**特征复用极度充分,参数效率很高**(用很少参数达到 ResNet 相当的精度)。它的副作用是:concat 让通道线性累积,中间特征图的显存占用大(很多层都要保留),**DenseNet 是典型的"省参数但费显存"**——训练时显存往往是瓶颈。这是一个重要的工程权衡:参数量小 ≠ 显存小、≠ 训练快。

### 7.3 SENet(2017):通道注意力,显式重标定特征

SENet(Hu 等)提出 **Squeeze-and-Excitation(SE)模块**,核心思想:**普通卷积默认所有输出通道同等重要,但其实不同通道对当前输入的重要性不同,应该让网络自适应地给每个通道乘上一个重要性权重。** 这是"通道注意力"的开山之作。

SE 模块三步(squeeze → excitation → scale):

```
输入特征 U ∈ ℝ^(H×W×C)
1) Squeeze:全局平均池化,把每个通道压成一个标量
   z_c = (1/(H·W)) · Σ_{i,j} U_{i,j,c}         得到 z ∈ ℝ^C  （每通道的全局描述)
2) Excitation:两层全连接学通道间关系(中间降维比 r,如 r=16)
   s = σ( W₂ · ReLU(W₁ · z) )                   s ∈ (0,1)^C  （每通道的门控权重)
       W₁ ∈ ℝ^(C/r × C),  W₂ ∈ ℝ^(C × C/r)
3) Scale:把权重逐通道乘回原特征
   X̃_{:,:,c} = s_c · U_{:,:,c}
```

**为什么 squeeze 用全局池化**:要判断一个通道整体上重不重要,需要全局信息(感受野扩到整张图),而单个空间位置看不到全局,所以先 GAP 把空间信息压缩成每通道一个标量。**为什么 excitation 要降维再升维(中间 C/r)**:既减少参数(`2C²/r` 而非 `C²`),又给通道间关系建模引入瓶颈与非线性。SE 模块**极轻**(参数和算力增加都很小),却能即插即用地稳定涨点,所以被海量后续网络(包括 EfficientNet)直接吸收为标准组件。

### 7.4 MobileNet / ShuffleNet:移动端的深度可分离卷积

移动端要求模型在手机 CPU 上实时跑,核心矛盾是 **FLOPs 和参数都要极小,但精度别掉太多**。MobileNet(Howard 等)的武器是**深度可分离卷积(depthwise separable convolution)**,把一个标准卷积**分解成两步**:

```
标准卷积:同时做"空间滤波"+"通道融合",一步到位
深度可分离 = Depthwise(逐通道做空间滤波,不混通道) + Pointwise(1×1,只混通道)
```

- **Depthwise 卷积**:对每个输入通道用一个独立的 K×K 核做空间卷积(即分组卷积里 g=Cᵢₙ 的极端情形),**不跨通道**。
- **Pointwise 卷积**:就是 1×1 卷积,负责跨通道融合。

**算力压缩比的推导**(输入 Cᵢₙ、输出 Cₒᵤₜ、核 K×K、输出空间 H_out·W_out,记 N=H_out·W_out):

```
标准卷积 MACs:        N × Cₒᵤₜ × Cᵢₙ × K²
深度可分离 MACs:
  depthwise:          N × Cᵢₙ × K²                    （每通道一个核,不乘 Cₒᵤₜ)
  pointwise(1×1):    N × Cₒᵤₜ × Cᵢₙ × 1²
  合计:               N × Cᵢₙ × (K² + Cₒᵤₜ)

压缩比 = (K² + Cₒᵤₜ) / (Cₒᵤₜ × K²) = 1/Cₒᵤₜ + 1/K²
```

当 K=3、Cₒᵤₜ 较大(如 256)时,压缩比 ≈ 1/256 + 1/9 ≈ 0.115,**算力降到约 1/8 ~ 1/9**。机理:**标准卷积把"空间滤波"和"通道融合"耦合在一个昂贵操作里(FLOPs ∝ Cᵢₙ·Cₒᵤₜ·K²),深度可分离把它分解成两个总乘加次数远更少的操作(FLOPs ∝ Cᵢₙ·(K²+Cₒᵤₜ))**。代价是表达能力略有损失(空间与通道不再联合建模),靠加宽/加深和后续设计补回。ShuffleNet 进一步用**分组 1×1 + channel shuffle**(打乱通道顺序让组间信息流动)压低 1×1 的开销,解决分组卷积组间不通信的问题。

## 八、EfficientNet(2019):复合缩放的推导

到这里,所有"怎么设计 block"的招数都摆出来了。EfficientNet(Tan & Le)回答的是一个更高层的问题:**给定一个好的基线网络,当你有更多算力预算想把它放大时,应该把预算花在加深(depth)、加宽(width)、还是提分辨率(resolution)上?** 这是"模型缩放(model scaling)"问题。

### 8.1 三个缩放维度,以及单独放大各自的饱和

```
深度 d:层数 ×d     —— 更多层,更强的特征抽象与更大感受野
宽度 w:通道 ×w     —— 更多通道,捕获更细粒度特征,易并行
分辨率 r:输入 H×W ×r —— 更大输入,保留更多细节
```

EfficientNet 的关键经验观察:**单独放大任何一个维度,精度都会很快饱和**。

- 只加深:梯度/优化变难(即便有残差,收益递减),且感受野早已覆盖全图后继续加深收益小。
- 只加宽:浅而宽的网络抓不到高层语义,宽到一定程度精度平台化。
- 只提分辨率:输入细节变多,但**网络深度/宽度不够,感受野和容量跟不上**——大图里的大目标需要更大感受野(更深)和更多通道去表征。

**核心耦合直觉**:三者互相依赖。**分辨率提高了,就需要更大感受野(更多层=更深)去覆盖变大的目标,也需要更多通道(更宽)去表征更丰富的细节**。所以理性的做法是**三个维度按比例协同放大**,而不是死磕一个。

### 8.2 为什么 FLOPs 对三者的敏感度不同(必须算清)

放大对 FLOPs 的影响不对称,这是复合缩放配比的物理依据:

```
深度 ×d:   层数翻 d 倍,FLOPs ∝ d          （线性)
宽度 ×w:   每层 Cᵢₙ、Cₒᵤₜ 都翻 w 倍,而卷积 FLOPs ∝ Cᵢₙ·Cₒᵤₜ ⇒ FLOPs ∝ w²  （平方)
分辨率 ×r: H、W 都翻 r 倍,FLOPs ∝ H·W ⇒ FLOPs ∝ r²                       （平方)
```

所以总 FLOPs 大致正比于 `d · w² · r²`。**宽度和分辨率对算力是平方敏感,深度只是线性**——这是后面约束方程里 w、r 带平方而 d 不带的根本原因。

### 8.3 复合缩放公式的推导

EfficientNet 用一个**复合系数 φ(compound coefficient)**统一控制三者,固定它们的相对比例:

```
深度:    d = α^φ
宽度:    w = β^φ
分辨率:  r = γ^φ
约束:    α · β² · γ² ≈ 2,   且 α≥1, β≥1, γ≥1
```

**这个约束怎么来的(关键推导)**:由 8.2,总 FLOPs ∝ `d·w²·r² = α^φ · (β^φ)² · (γ^φ)² = (α·β²·γ²)^φ`。如果强制 `α·β²·γ² ≈ 2`,那么:

```
FLOPs 放大倍数 = (α·β²·γ²)^φ ≈ 2^φ
```

也就是说,**φ 每增加 1,总 FLOPs 大约翻一倍**,而 α、β、γ 决定这翻倍的预算如何在深/宽/分三者间分配。论文先在小模型(EfficientNet-B0)上用网格搜索固定 φ=1 找到一组最优 (α, β, γ)(原文约 α≈1.2, β≈1.1, γ≈1.15「待核」),之后**保持这个比例不变**,只调 φ 就能得到 B1~B7 一整族模型。

**为什么这套优于单维放大**:它把"加多少层、加多少通道、用多大图"这三个本该联合搜索(搜索空间巨大)的决策,降维成**搜一次比例 + 沿一条射线缩放**。在相同 FLOPs 下,EfficientNet 系列显著优于同算力的 ResNet/ResNeXt(具体精度数值「待核」)。直觉上它让三个维度的边际收益保持平衡,而不是把预算全砸在一个早已饱和的维度上。

EfficientNet 的基线 B0 本身是用神经架构搜索(NAS)得到的,主体由 **MBConv**(MobileNet 的倒残差 + 深度可分离卷积 + SE 模块)构成——你看,**它是前面所有技术的集大成:深度可分离(MobileNet)+ SE 通道注意力(SENet)+ 残差(ResNet)+ 复合缩放(本节)**。这就是为什么把它放在本章压轴。

## 九、设计权衡与常见坑

把全章的权衡收拢成一张"决策表",这是从"会算"到"会判断"的关键:

- **参数量 ≠ 算力 ≠ 显存 ≠ 速度,四个量经常背离,别用一个代另一个判断**。VGG:参数巨大(全连接),但卷积部分算力还好。DenseNet:参数小但显存大(concat 要保留大量中间特征)。MobileNet:FLOPs 极小,但 depthwise 卷积**访存密集、计算密度低**,在 GPU 上未必比 FLOPs 更高的标准卷积快(GPU 喜欢大而规整的矩阵乘)。**移动端真正该优化的常常是"延迟"而非 FLOPs,二者不等价**。
- **1×1 卷积是现代 CNN 的瑞士军刀**:降维/升维、跨通道融合、瓶颈降算力,处处是它。看到一个不熟的 block,先找 1×1 在哪、在降维还是升维,基本就抓住了它的算力结构。
- **感受野要"够用且匹配目标尺度"**:不是越大越好。检测小目标需要保留高分辨率浅层特征(感受野别太大),检测大目标需要深层大感受野。这是后续 FPN 等多尺度结构的动机。
- **新手坑一:Inception 各分支输出尺寸不一致导致 concat 报错**——padding 必须精确配平。
- **新手坑二:把 BatchNorm 的训练/推理模式搞错**(BN 在推理时用running 统计量;本章为聚焦未展开 BN,但实际所有现代 CNN 都依赖它稳定训练,务必单独吃透)。
- **新手坑三:Dropout 推理时忘了关 / 或自己实现时忘了 inverted dropout 的 `/(1−p)` 缩放,导致训练推理期望不一致**。
- **新手坑四:堆很深的普通卷积(无残差)却收敛不动**——大概率是退化问题,加残差连接,而不是一味调学习率。
- **缩放坑**:盲目只加宽或只加深去追精度,性价比远低于按 EfficientNet 比例联合缩放;但 EfficientNet 的最优比例是在其特定基线上搜出来的,**换了基线/任务不一定照搬**。

## 十、动手练习

**练习 1(推导题,感受野与参数)**:用第 4.1 节的递推公式,证明 stride=1 下 n 个 3×3 卷积串联的感受野为 `2n+1`。再设输入输出通道均为 C,推导"四个 3×3 串联"与"单个 9×9"的参数量,算出节省百分比。*提示:四个 3×3 感受野 2·4+1=9,与 9×9 相同;参数 4·9C²=36C² vs 81C²。*

**练习 2(分析题,1×1 降算力)**:输入特征图 14×14×512,你要用 3×3 卷积输出 512 通道。(a)算直接做的 MACs;(b)设计一个"1×1 降到 128 → 3×3 → 1×1 升回 512"的瓶颈,算总 MACs 与加速比;(c)用一句话说清 1×1 降算力的机理。*提示:大核 FLOPs ∝ Cᵢₙ·Cₒᵤₜ,先用便宜的 1×1 把 Cᵢₙ 压小。*

**练习 3(推导题,复合缩放)**:已知卷积 FLOPs 对深度线性、对宽度和分辨率平方敏感。(a)推导总 FLOPs ∝ `d·w²·r²`;(b)解释 EfficientNet 约束 `α·β²·γ²≈2` 为何能让"φ 每加 1,FLOPs 翻倍";(c)论证为什么联合缩放优于把同样的 FLOPs 预算全部投到单一维度。*提示:从边际收益递减 + 三维度互相依赖(分辨率↑需要更深以扩感受野、更宽以增容量)两个角度答。*

**练习 4(编码题,深度可分离卷积)**:用 PyTorch 的 `nn.Conv2d(..., groups=...)` 实现一个深度可分离卷积模块(depthwise 用 `groups=in_ch` + pointwise 用 1×1),并写一个脚本对比它与等价标准卷积在固定输入下的参数量和理论 MACs,验证压缩比公式 `1/Cₒᵤₜ + 1/K²`。*提示:depthwise 时令 `out_channels=in_channels, groups=in_channels`,其参数量应为 `in_ch·K²`;pointwise 即 `kernel_size=1` 的普通卷积。*

## 十二、小结与承上启下

这一章你应该带走两样东西。其一是**四把尺子的心算能力**:任给一个卷积配置,能立刻算出输出尺寸 `(H−K+2P)/S+1`、参数 `Cₒᵤₜ·Cᵢₙ·K²`、MACs `N·Cₒᵤₜ·Cᵢₙ·K²`、以及它对感受野的贡献 `r_{l−1}+(kₗ−1)·j_{l−1}`。其二是**一条架构演进的因果链**:

```
LeNet 立范式(卷积+池化+全连接)
  → AlexNet 用 ReLU/Dropout/GPU 让"深且大"在大数据上 work
  → VGG 用 3×3 堆叠把"深度=非线性=表达力"做到极致(省参 + 多非线性)
  → GoogLeNet 用多尺度并行 + 1×1 瓶颈 + GAP,在控算力下加宽
  → 深度的红利撞上退化问题 → ResNet 用残差破局(下章)
  → block 微观精化:ResNeXt(基数)、DenseNet(特征复用)、SENet(通道注意力)、MobileNet(深度可分离)
  → EfficientNet 用复合缩放统一回答"预算多了往哪加"
```

注意 know 提示里那条诚实纪律:本章凡是 benchmark 具体分数、精确参数量、超参默认值,我都标了「待核」,因为它们随版本/复现而漂移,但**所有公式、推导、复杂度结论都可自行验算且自洽**,这才是你要长在身上的部分。

**承上启下**:第六章我们只点到 ResNet 的动机(把学恒等映射的难题转成学残差为零的易题),下一章会把残差学习讲透——为什么 `F(x)+x` 在反向传播上构造了一条"梯度高速公路"让上千层可训练,以及 BatchNorm 如何与残差协同稳定训练。再往后,当 CNN 的归纳偏置(局部性、平移等变)从"优势"逐渐变成"限制",我们会进入 Vision Transformer:它**丢掉卷积的硬先验,用注意力让模型自己学全局关系**——而你会发现,本章学到的"感受野、算力配比、瓶颈降维"这些尺子,在分析 ViT 时一样好用。
