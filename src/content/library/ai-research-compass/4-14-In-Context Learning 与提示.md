> 这一章把你从"模型要学新任务就得拿数据做梯度更新"带到"能从第一性原理说清:为什么 GPT-3 仅靠 prompt 里几个示例、不动一个权重就能做新任务;这个现象在数学上可能是哪一种机制(隐式贝叶斯、隐式梯度下降、任务向量),每种假说的证据强在哪、漏洞在哪;Chain-of-Thought 为什么把准确率拉高不是玄学而是给自回归生成多腾了计算预算;以及 prompt 工程那些'换个示例顺序就掉 10 个点'的脆弱性背后的机理"。学完你能自己设计一个 few-shot/CoT prompt 并预测它会怎么坏,也能读懂这一摊还在打架的机制论文。

## 为什么这一章绕不过去

前两章你刚学完微调:要让一个预训练模型会做新任务,标准答案是**拿带标签的数据,对权重做梯度下降**(全量或 PEFT)。这套范式从 BERT 时代统治到今天,它的世界观是清晰的——**学习 = 改权重**。

然后 GPT-3(Brown et al., 2020)做了一件违反这个世界观的事。你不给它任何梯度更新,只在输入里写:

```
将英文翻译成法语:
sea otter => loutre de mer
cheese => fromage
plush giraffe =>
```

模型直接续写出 `girafe en peluche`。它**没有更新任何参数**——权重和你问它"今天天气怎样"时一模一样——却完成了一个它当前 prompt 才被"告知"的任务。Brown 等把这叫 **in-context learning(上下文学习,ICL)**:模型在前向传播这一瞬间,**仅凭上下文里的几个示例,就调整了自己的行为**,像是"现学现用",但没有任何持久的权重改变。

这件事为什么是颠覆性的,要从三个层面体会:

1. **它打破了"学习必须改权重"的等式。** 传统机器学习里,"模型学到了 X"和"模型的参数被 X 的梯度改过"是同义词。ICL 里,行为变了,参数没动。那"学习"发生在哪?——发生在**激活(activation)**里,在那次前向传播的中间张量里。这逼着我们重新定义什么叫"学习"。

2. **它是 scaling 的涌现产物,小模型几乎没有。** 同样的 prompt 喂给 GPT-2(1.5B)和 GPT-3(175B),小模型基本不吃 few-shot 这一套,大模型才显著起效。ICL 不是被显式训练出来的目标(预训练目标只是"预测下一个词"),而是规模够大后**自己冒出来的能力**(前置见《08-Scaling Laws 与涌现》)。这意味着它的机制藏在"为什么一个只学过预测下一个词的网络,会顺带学会现学现用"里。

3. **它是 prompt 工程整个学科的地基。** 你今天用的所有"提示技巧"——few-shot 举例、Chain-of-Thought 让它先想再答、角色扮演——本质都是在**操纵 ICL**:用上下文把模型推到你想要的行为分布上。不理解 ICL 的机制,prompt 工程就只能靠玄学和运气。

**这一章的诚实前提:ICL 的机制至今是一个开放问题。** 我不会假装有定论。我会做三件事——(a) 把现象本身的边界讲清(zero/one/few-shot 怎么分、它对什么敏感);(b) 呈现三大主流机制假说,每一个都给出它的数学构造或经验证据,以及它解释不了的地方;(c) 把 Chain-of-Thought 从"魔法咒语"还原成"给自回归生成腾计算预算"的可分析机制。**贯穿全章的判断标准**:任何关于 ICL"为什么 work"的说法,都要问——它能不能解释 ICL 的脆弱性(对示例顺序/格式极度敏感),能不能解释它的涌现性(为什么小模型没有),它的证据是构造性的(我能搭一个网络复现)还是相关性的(我观察到激活长得像)。

## 一、把现象钉死:zero / one / few-shot 与"学习"发生在哪

### 问题:同一个模型,凭什么 prompt 里多塞几个例子就变强

先把术语和现象的边界划清,否则后面机制讨论会悬空。Brown 等定义了一个**示例数量 K**(放进 prompt 的"输入→输出"演示对的个数),据此分三档:

```
zero-shot (K=0):  只给任务描述,不给任何示例
  "把英文翻译成法语:  cheese =>"

one-shot  (K=1):  给一个示例
  "把英文翻译成法语:  sea otter => loutre de mer ;  cheese =>"

few-shot  (K=10~100): 给若干示例(受上下文窗口限制)
  "把英文翻译成法语:
   sea otter => loutre de mer
   peppermint => menthe poivrée
   ... (K 个)
   cheese =>"
```

GPT-3 论文的核心经验曲线是:**在大多数任务上,准确率随 K 增大而上升(few > one > zero),且模型越大,这条曲线越陡、绝对水平越高。** 小模型(如 GPT-2 量级)的曲线几乎是平的——给它示例它也不太会用。**"会用示例"这件事本身,是随规模涌现的。**

这里要立刻消除一个新手的误解。**ICL 里没有任何参数更新。** 看到"few-shot learning"里有"learning"就以为偷偷训了几步——没有。整个过程是**一次普通的前向传播**:把 K 个示例和最后的查询拼成一长串 token,喂进去,读出最后位置的 logits,采样得答案。权重在前后字节级相同。这就是为什么它有时被更准确地叫 **in-context learning** 而非 fine-tuning——"学习"完全发生在那一次前向的激活动态里,前向结束、激活释放,"学到的东西"就没了(下一个请求要重新在 prompt 里给示例)。

### 机制:学习为什么"能"发生在前向里——一个信息论视角

要理解 ICL 怎么可能,先回到模型到底被训练成了什么。自回归语言模型优化的是**给定前文、预测下一个词的条件分布** `p(x_t | x_{<t})`。预训练语料是整个互联网,里面**天然充满"任务"的结构**:一篇维基百科是"主题→展开",一段代码是"注释→实现",一个问答帖是"问→答",一份词典是"词→释义"。

关键洞察:**为了把"预测下一个词"做到极致,模型被迫去建模这些隐藏结构。** 当语料里反复出现"英文词 => 法语词"的模式,模型要降低这一段的损失,就必须学会:看到这个格式 ⇒ 这是翻译任务 ⇒ 下一个 token 应该是法语翻译。它不是被人教"这叫翻译任务",而是因为**正确识别出这个隐含任务能让下一词预测更准**,于是顺带学会了。

把这个直觉量化。设语料里一段文本背后有一个潜在的"任务/概念"θ(可能是"翻译""情感分类""续写诗"),文本是按 `p(x | θ)` 生成的,而 θ 本身有个先验 `p(θ)`。一个把 `p(x)` 学到极好的模型,等价于学到了边缘分布

```
p(x) = ∫ p(x | θ) · p(θ) dθ        ……(把所有可能任务积掉)
```

而要让条件预测 `p(x_t | x_{<t})` 准,模型实际上需要隐式地维护**后验** `p(θ | x_{<t})`——"看了前文这么多,现在最可能是哪个任务"。**ICL 的可能性就藏在这个后验里:prompt 里那 K 个示例,就是在更新这个 θ 的后验,把概率质量集中到"翻译"这个任务上,从而让续写落在翻译的分布里。** 这正是下一节"隐式贝叶斯"假说的雏形。注意此刻还没有任何机制细节,只是论证了"为什么纯粹学下一词预测,会留出 ICL 的可能空间"。

### 代码:用真实接口观察 ICL 现象(可跑的最小框架)

下面是构造 few-shot prompt 并读出预测的核心骨架。重点不在调包,在于**让你看清"输入是一长串拼接、输出只取最后位置"这个机制本身**。

```python
# 用 HuggingFace 因果语言模型观察 ICL(机制示意,非性能优化)
import torch
from transformers import AutoTokenizer, AutoModelForCausalLM

tok = AutoTokenizer.from_pretrained("gpt2-large")      # 模型越大 ICL 越明显
model = AutoModelForCausalLM.from_pretrained("gpt2-large").eval()

def build_fewshot_prompt(demos, query, template="{x} => {y}"):
    """demos: [(x, y), ...] 示例对;query: 待预测的 x。
    机制核心:把 K 个 '示例' 和 query 用同一个模板拼成一条文本。"""
    lines = [template.format(x=x, y=y) for (x, y) in demos]   # K 行示例
    lines.append(template.format(x=query, y="").rstrip())      # 末行只给 x,留空 y
    return "\n".join(lines)

demos = [("happy", "positive"), ("terrible", "negative"),
         ("delightful", "positive"), ("awful", "negative")]   # 情感分类的 4 个 shot
prompt = build_fewshot_prompt(demos, query="wonderful")

ids = tok(prompt, return_tensors="pt").input_ids
with torch.no_grad():
    logits = model(ids).logits          # 一次前向,无任何梯度/更新
next_tok_logits = logits[0, -1]         # ★ 只取最后一个位置:它编码了"在此上下文下,下一词是什么"

# 在候选标签 token 上比较概率(分类型 ICL 的标准读法)
for label in [" positive", " negative"]:
    tid = tok(label).input_ids[0]
    print(label, next_tok_logits[tid].item())
# 期望:" positive" 的 logit 更高 —— 模型"在上下文里学会了"这个情感分类任务
```

两个机制要点用注释标了:**(1) K 个示例和 query 走的是同一个模板**(格式一致是 ICL 起效的前提,后面讲脆弱性会回到这);**(2) 全部信息汇聚到最后一个位置的 logits**——因果注意力让最后位置能"看"到前面所有示例,这就是"学习"在前向里发生的物理位置。

### 分析:ICL 的算力账与它和微调的本质差异

ICL 没有训练成本,但有**推理成本**,而且这笔账常被低估。设示例平均长度 ℓ token、共 K 个示例、查询长 q token,则每次推理的 prompt 长度

```
L = K·ℓ + q
```

而自注意力对序列长度是 **O(L²)** 的(前置见《02-注意力机制的数学》《10-长上下文》)。这意味着:

- **few-shot 的代价是平方级膨胀的 prompt。** K 从 0 加到 32,L 翻好几倍,单次推理的注意力计算按 O(L²) 增长,KV cache 显存按 O(L) 线性增长。**ICL 把"训练成本"换成了"每一次推理都要重付的上下文成本"**——你每问一次,那 K 个示例都得重新进一次前向。微调正相反:一次性付训练成本,推理时 prompt 干净。

```
ICL 与微调的成本结构对比:
  微调:  训练成本 = 一次性(梯度更新);推理成本 = 干净 prompt
  ICL:   训练成本 = 0;推理成本 = 每次都背 O((K·ℓ)²) 的示例上下文
  → 任务稳定、高频调用 → 微调更划算
  → 任务多变、长尾、无标注数据 → ICL 几乎是唯一选择(零成本切任务)
```

- **这也解释了 ICL 受上下文窗口硬约束**:K 不能无限大,塞不下更多示例。这是它相对微调的根本天花板,也催生了"把示例蒸馏进权重"的研究(如 (IA)³ 论文标题就直接说 PEFT 比 ICL 更便宜)。

到这里现象层面讲清了:ICL 是一次普通前向、学习发生在激活、随规模涌现、用平方级上下文成本换零训练成本。**下面进入这一章真正难的部分——这件事在机制上到底是怎么发生的。我会给三种主流假说,这是当前研究的最前沿,也是最容易被讲成玄学的地方。**

## 二、机制假说之一:隐式贝叶斯推断——模型在"认任务"

### 问题:示例到底是"被学习"还是"被用来定位任务"

第一种主流解释,把 ICL 看成**任务识别(task recognition / location)而非任务学习**。核心主张:**预训练让模型内部隐式地学到了一大堆"任务/概念"以及它们的先验;prompt 里的示例不是教模型一个新任务,而是帮模型从已有任务库里"认出"现在该用哪个,然后按那个任务的分布续写。** 代表性形式化来自 Xie et al. (2022, "An Explanation of In-context Learning as Implicit Bayesian Inference")。

### 推导:为什么后验会随示例增多而集中到正确任务

接着上一节的潜变量设定,把 ICL 写成一个贝叶斯推断过程。设有一族任务 θ,预训练让模型逼近了

```
p(下一段输出 | prompt) = ∫ p(输出 | θ) · p(θ | prompt) dθ
                                         ↑
                         给定 prompt(含 K 个示例)对任务的后验
```

prompt 由 K 个示例对 (x₁,y₁),…,(x_K,y_K) 加查询 x_query 组成。用贝叶斯公式展开后验:

```
p(θ | 示例) ∝ p(θ) · ∏ᵢ p(xᵢ, yᵢ | θ)        ……(假设示例在给定 θ 下近似条件独立)

取对数:
log p(θ | 示例) = log p(θ) + Σᵢ log p(xᵢ, yᵢ | θ) + const
```

关键看 **Σᵢ log p(xᵢ, yᵢ | θ)** 这一项。设真实任务是 θ*。对每个示例,在 θ* 下的对数似然 `log p(xᵢ,yᵢ | θ*)` 平均而言**高于**在错误任务 θ′ 下的 `log p(xᵢ,yᵢ | θ′)`(因为示例确实是按 θ* 的格式生成的)。于是随 K 增大:

```
log p(θ* | 示例) − log p(θ′ | 示例)
  ≈ [log p(θ*) − log p(θ′)] + Σᵢ [log p(xᵢ,yᵢ|θ*) − log p(xᵢ,yᵢ|θ′)]
                                  ↑
              每一项期望 > 0,记其期望为 KL 散度相关的正量 → 随 K 线性累加
```

第二项是 K 个正期望项之和,**随 K 线性增长并压过先验差**。所以后验质量 `p(θ* | 示例)` 随示例增多**指数式集中**到真实任务上。一旦后验塌缩到 θ*,续写就按 `p(输出 | θ*)` 走——模型"做对了任务"。

> **直觉一句话:示例不是训练数据,是给模型"指认任务"的证据。** 你给它看几对"词→法语",不是在教它翻译(它早会了),是在用证据把它内部的"现在该翻译"这个后验顶上去。K 越多,证据越足,认得越准。

这个推导还顺手解释了 GPT-3 的一个反直觉发现的"为什么":论文里它**对 K 的边际收益递减**——从 0 到 1 个示例提升巨大,从 32 到 64 提升甚微。贝叶斯视角下这是必然的:后验集中是指数式的,**前几个示例就把概率质量基本压到 θ* 上了,再加示例只是边际地继续压一个已经接近 1 的后验。**

### 这个假说的强证据:Min et al. 的"标签可以是错的"

隐式贝叶斯最有名的支撑实验来自 Min et al. (2022, "Rethinking the Role of Demonstrations")。他们做了一件挑衅的事:**把 few-shot 示例里的标签随机打乱(故意标错),ICL 准确率竟然几乎不掉。**

```
正常 few-shot:        "happy => positive ; awful => negative ; ..."  → 准确率 A
标签随机化的 few-shot: "happy => negative ; awful => positive ; ..."  → 准确率 ≈ A (只略降)
对照(完全无示例):    zero-shot                                       → 准确率 ≪ A
```

这个结果对"任务学习"说是毁灭性的:**如果示例是在教模型输入→输出的映射,把映射全标错就该崩盘。** 但它对"任务识别"说是自然的——**示例真正传递的信息不在 (x,y) 的正确配对里,而在"格式长这样、标签空间是 {positive, negative}、输入分布是这类句子"这些元信息里。** 这些元信息足以让模型认出"哦这是情感二分类",然后它**调用自己预训练里早学好的情感判断**去做查询,而不是从示例里现学映射。Min 等据此提炼:示例真正起作用的成分是**标签空间、输入分布、格式**,而非具体的 input-label 对应。

### 局限:贝叶斯说解释不了什么

诚实地讲它的漏洞:

1. **它解释不了"真能学新映射"的情形。** 后续工作(如 Wei et al. 2023 在大模型上)发现:**模型足够大、示例足够多时,翻转标签确实会让性能往翻转后的方向走**——说明大模型不只是"认任务",也能从示例里**真的学进去一个与预训练相悖的映射**。纯任务识别说覆盖不了这个,这把我们推向第二种假说(隐式梯度下降)。
2. **它把 θ 当成离散的"任务库"是个理想化。** 真实模型里没有一个干净的任务先验 `p(θ)`,这套贝叶斯只是**对模型行为的一个高层描述**,不是机制级证据——它没告诉你"后验更新"在 Transformer 的哪个权重、哪层注意力里物理地发生。这是它和下一个假说的根本档次差异:**贝叶斯是行为层的相关性解释,梯度下降说试图给构造性机制。**

## 三、机制假说之二:隐式梯度下降——前向传播在激活里"训练"了一个小模型

### 问题:Transformer 的前向,能不能算出一步梯度下降

这是三种假说里**数学上最硬核、也最令人震惊**的一个:**Transformer 的前向传播,可以在激活里模拟对示例做梯度下降。** 即模型不是"认任务",而是真的在前向过程中,**用 prompt 里的示例当训练集,在内部跑了几步(线性回归式的)优化,得到一个临时的"小模型",再用它预测查询。** 代表性工作:von Oswald et al. (2023, "Transformers Learn In-Context by Gradient Descent")、Akyürek et al. (2022)、Dai et al. (2022, 把注意力解读为"元梯度")。

### 推导:一层线性注意力 = 一步梯度下降(构造性证明)

这是本章最关键的推导,**要给到能复现的程度**。考虑最简单的 ICL 任务:线性回归。示例是 (x₁,y₁),…,(x_K,y_K),xᵢ ∈ ℝᵈ,yᵢ ∈ ℝ,假设 yᵢ = w*ᵀxᵢ。我们要预测查询 x_query 的 y。

**先看显式梯度下降会做什么。** 用最小二乘损失,权重从 w₀=0 出发,学习率 η,对损失 `L(w) = ½ Σᵢ (wᵀxᵢ − yᵢ)²` 做一步梯度下降:

```
∇_w L |_{w=0} = Σᵢ (w₀ᵀxᵢ − yᵢ) xᵢ = Σᵢ (0 − yᵢ) xᵢ = −Σᵢ yᵢ xᵢ

w₁ = w₀ − η·∇L = 0 + η·Σᵢ yᵢ xᵢ = η·Σᵢ yᵢ xᵢ

对查询的预测:
ŷ_query = w₁ᵀ x_query = η · Σᵢ yᵢ · (xᵢᵀ x_query)        ……(★ 记住这个式子)
```

注意 (★) 的结构:**预测 = 对每个示例的标签 yᵢ 加权求和,权重是 xᵢ 与 x_query 的内积。**

**再看一层线性自注意力会算什么。** 把每个示例打包成一个 token,值向量(value)里放标签 yᵢ,键向量(key)放 xᵢ,查询位置的 query 向量放 x_query(略去 softmax,用线性注意力——这是 von Oswald 等的设定,使等价精确):

```
线性注意力对查询位置的输出 = Σᵢ (query · keyᵢ) · valueᵢ
                          = Σᵢ (x_queryᵀ xᵢ) · yᵢ          ……(把 query=x_query, key=xᵢ, value=yᵢ 代入)
```

**把这一项和 (★) 逐项对比:**

```
一步梯度下降的预测:  ŷ = η · Σᵢ yᵢ (xᵢᵀ x_query)
一层线性注意力输出:       Σᵢ (x_queryᵀ xᵢ) yᵢ

两者完全同构!  (内积对称 xᵢᵀx_query = x_queryᵀxᵢ;η 由注意力的投影权重吸收)
```

**结论(可验算):一层线性自注意力,在恰当的权重配置下,精确地实现了"对示例做一步梯度下降并预测查询"。** 学习率 η 被编码进 Q/K/V 的投影矩阵里。把这个推广开:

- **多层注意力 ⇒ 多步梯度下降。** 第 ℓ 层在第 ℓ−1 层的"权重估计"基础上再走一步,激活里逐层精炼出越来越好的 w。von Oswald 等构造性地证明了一个 L 层线性 Transformer 可以模拟 L 步 GD,并在实验里发现**真实训练出来的 Transformer,其逐层行为高度吻合 GD 的逐步轨迹**(不只是"能",而是"训练后真的趋向这么做")。
- **softmax 注意力 ⇒ 近似一种预条件/核化的梯度下降**(更复杂,但精神相同)。

> **直觉:Transformer 前向 = 一个"会在激活里临时搭一个学习器"的元学习器(meta-learner)。** 预训练没有教它任何具体任务,但教会了它**一套"看到示例就地拟合"的算法**,这套算法被编码在权重里;每次前向,它就用这套算法、拿当前 prompt 的示例当数据,临时跑出一个小模型。**权重不变,变的是每次前向用这套固定算法对不同示例算出的不同临时解。** 这就是"不更新权重也能学"在机制上最干净的回答。

### 代码:复现"线性注意力 ≈ 一步 GD"

下面用 numpy 把上面的等价性**数值验证**出来——这是你该亲手跑一遍的核心实验。

```python
import numpy as np
rng = np.random.default_rng(0)

d, K = 5, 20
w_star = rng.normal(size=d)                  # 真实权重
X = rng.normal(size=(K, d))                  # K 个示例输入 (key)
y = X @ w_star                               # 对应标签 (value)
x_q = rng.normal(size=d)                     # 查询

# --- 路线 A:显式一步梯度下降,w0=0 ---
eta = 0.1
grad = -(y[:, None] * X).sum(axis=0)         # ∇L|_{w=0} = -Σ yᵢxᵢ
w1 = -eta * grad                             # w1 = η Σ yᵢ xᵢ
pred_gd = w1 @ x_q                           # ŷ = w1ᵀ x_q

# --- 路线 B:一层线性注意力(query=x_q, key=X, value=y) ---
attn_scores = X @ x_q                        # ⟨xᵢ, x_q⟩,K 维
pred_attn = eta * (attn_scores * y).sum()    # Σ (xᵢᵀx_q)·yᵢ,乘同一个 η

print(pred_gd, pred_attn, np.isclose(pred_gd, pred_attn))
# 输出两个相等的数,True —— 线性注意力一层 = 一步 GD,数值验证通过
```

跑出来两个值相等。这不是巧合,是上面推导的逐项对应。**你照着推导能复现,这就是"构造性证据"的含金量——它不是观察到激活长得像 GD,而是证明了等价并能复刻。**

### 局限:构造能做 ≠ 训练出来的模型真这么做

这个假说最强,但也最容易被过度推销,务必看清边界:

1. **"存在一组权重能模拟 GD"不等于"预训练自然学到的就是这组"。** 构造性证明给的是**可表达性(expressivity)**:Transformer 有能力实现 GD。但真实预训练出来的模型是不是真在用这套算法,是经验问题。von Oswald 等在**小型合成线性回归**任务上给了较强的对齐证据(逐层轨迹吻合 GD);但**大型语言模型在自然语言任务上是不是也在跑梯度下降,远没有定论**——自然语言任务不是干净的线性回归,这个外推是这一假说最大的问号。「这一步从合成任务到真实 LLM 的推广强度,学界仍在争论,待核」
2. **它和 softmax、归一化、MLP 的真实结构有距离。** 干净的等价建立在**线性注意力**上;真实 Transformer 有 softmax、LayerNorm、FFN,等价关系变成"近似"且推导复杂得多,机制是否仍是 GD 不再一目了然。
3. **它解释不了 Min 等"标签错了也不掉点"的现象。** 如果模型真在拿示例做监督回归,标签全错应该把它带偏——但实验显示(在那个规模下)没怎么偏。**所以梯度下降说和任务识别说各自只覆盖一部分现象**:小模型/少示例更像"认任务"(标签不重要),大模型/多示例更像"真在学映射"(标签开始重要)。**两个假说很可能不是对立,而是在不同 regime 下各自主导。** 这是目前比较被接受的折中认识。

## 四、机制假说之三:任务向量 / 函数向量——示例被压成激活里的一个向量

### 问题:示例携带的"任务信息",能不能被定位成一个具体向量并搬走

第三种假说更偏**机制可解释性(mechanistic interpretability)**,主张:**ICL 过程中,prompt 里那些示例被模型压缩成激活空间里的一个(或少数几个)"任务向量(task vector)/ 函数向量(function vector)";这个向量编码了"现在要做的任务",可以被提取出来、注入到一个完全没有示例的前向里,从而让模型在零示例下也执行该任务。** 代表性工作:Hendel et al. (2023, "In-Context Learning Creates Task Vectors")、Todd et al. (2023, "Function Vectors in Large Language Models")。

### 机制:提取与移植——一个可证伪的强主张

这个假说之所以有分量,是因为它给出一个**可证伪、可干预**的实验协议,不只是观察:

```
任务向量假说的核心实验(intervention 范式):
1. 跑一遍带示例的 few-shot prompt(任务 = 翻译),
   在某一层 ℓ、某个位置(通常是分隔符 "=>" 或最后示例处)记下激活向量 v。
2. 另起一个全新前向,prompt 里【只有查询、没有任何示例】:"cheese"。
3. 在同一层 ℓ 的对应位置,把记下的 v 直接【加进/替换】激活,继续前向。
4. 观察:模型在零示例下,是否输出了翻译(fromage)?

  → 若成立:说明任务信息确实被压缩进了 v,且 v 足以驱动任务执行。
  → 这是【可证伪】的:如果换任务的 v 注入后行为不变,假说就垮了。
```

Hendel 等和 Todd 等报告:**在多种任务上,这个移植确实部分奏效**——注入提取出的任务向量,能让无示例的前向以显著高于随机的概率执行对应任务。Todd 等还进一步定位到**少数几个注意力头**对构造这个函数向量贡献最大(用因果干预/activation patching 确认)。

> **直觉:few-shot 的 K 个示例,在某一层被模型"读完并总结成一句话",这句话就是激活里的一个向量,内容是"接下来请做翻译"。** 后续的层不再回看示例,只看这个总结向量行事。**移植实验证明这个'总结'是可分离的实体**——你能把它抠出来贴到别处。

### 与前两个假说的关系:不矛盾,是不同抽象层

务必讲清三个假说**不是三选一**,而是在**不同抽象层**描述同一件事,且互相咬合:

```
            行为层(它在做什么)                         机制层(怎么实现的)
隐式贝叶斯:  "在更新任务后验 p(θ|示例)"     ←———相当于———  任务向量就是这个后验的一个【点估计/编码】
任务向量:    "把示例压成一个 θ 的表示 v"     ←———可能由———  某些注意力头在前向里【聚合示例算出 v】
隐式GD:      "在激活里对示例做梯度下降"       ←———其解就是———  逐层精炼出的那个 v / 临时权重

一种自洽的统一叙事:
  注意力头读完示例 → (隐式GD式地)聚合出一个任务向量 v(≈ 后验 p(θ|示例) 的编码)
  → 后续层条件在 v 上续写 → 行为上看就是"认出并执行了任务"
```

**三者各自的证据类型也正好互补**:贝叶斯给行为层的数学描述(后验集中),GD 给可表达性的构造证明(线性注意力=GD),任务向量给机制层的因果干预证据(抠出来能移植)。一个负责任的工程师的认识应该是:**ICL 大概率是"前向里聚合示例→形成任务表示→条件续写"这一过程,三个假说是从不同角度照它。没有任何一个被证明是全貌,把任何一个当定论都是不诚实的。**

### 局限

- **移植只是"部分"奏效**,准确率往往低于原始 few-shot——说明任务信息不全在那一个向量里,还有分布在别处的成分。任务向量是**一个重要载体,但不是唯一载体**。
- 提取依赖**选对层和位置**,换层就失效,鲁棒性有限;"是不是每个任务都有干净的单一向量"也没有定论(复杂任务可能需要多向量/子空间)。「待核:跨模型、跨任务的普适性证据仍不充分」

## 五、Chain-of-Thought:把"先想再答"从玄学还原成算力账

### 问题:让模型先写推理步骤,凭什么能把复杂推理准确率拉高几十个点

Chain-of-Thought(CoT,思维链,Wei et al., 2022)是 prompt 工程里效果最戏剧化的发现。做法极简:**在 few-shot 示例里,不直接给"问题→答案",而是给"问题→推理过程→答案";模型在回答查询时,也会先生成推理步骤,再给最终答案。**

```
标准 few-shot:
  "Q: 食堂有 23 个苹果,用掉 20 个,又买 6 个,现在有几个?  A: 9"            ← 直接给答案
  对难题:模型经常答错。

CoT few-shot:
  "Q: 食堂有 23 个苹果,用掉 20 个,又买 6 个,现在有几个?
   A: 原来有 23 个。用掉 20 个,剩 23−20=3 个。又买 6 个,3+6=9 个。答案是 9。"  ← 给出推理过程
  对同样的难题:准确率大幅提升(GSM8K 类算术/多步推理任务上提升尤其明显)。
```

Wei 等报告 CoT 在多步算术、常识、符号推理上带来**显著且随规模放大**的提升,且和 ICL 一样**是涌现的**(小模型用 CoT 反而可能更差)。问题是:**模型权重没变、信息没变,凭什么"先把步骤写出来"就更准?** 这绝不是玄学,有两个可以量化的机制。

### 机制一:把多步计算"摊进"自回归生成,绕开单步前向的计算上限

这是最本质的解释,**要从 Transformer 单次前向的计算能力上限讲起**。

一个固定深度 L、固定宽度的 Transformer,**对单个 token 位置做一次前向,所执行的串行计算步数是有限且固定的**(大致正比于层数 L)。这是一个硬约束:**让它在"一个 token 的前向"里完成一个需要 T 步串行依赖的计算(比如 T 位数连续进位加法、T 步逻辑推导),当 T 超过它的有效深度时,它在原理上就算不完。** 这不是训练不够,是**计算深度不够**——一个 L 层网络无法在一次前向里模拟比 O(L) 更深的串行计算链。

CoT 怎么破这个局?**它把"一步算完"改成"分多个 token、每个 token 算一点、把中间结果写进序列、下一步再读回来接着算"。**

```
无 CoT:要在【单个 token 的一次前向】里算完整个多步推理
        → 串行步数 T 受限于网络深度 L,T > L 时算不动 → 出错

有 CoT:把推理拆成 T 个中间 token,逐个生成
        token 1: 算出中间结果 r₁,写进序列
        token 2: 注意力【读回】r₁,在其基础上算 r₂,写进序列
        ...
        token T: 读回 r_{T-1},算出最终答案
        → 每个 token 仍只用 O(L) 深度,但【整条推理链的有效深度 ≈ L × T】
        → 自回归 + 注意力把"中间结果"当成可读写的外部草稿纸(scratchpad)
```

**关键洞察:自回归生成 + 注意力 = 一条可读写的"工作记忆磁带"。** 每生成一个推理 token,就把一个中间结果固化进上下文;后续 token 通过注意力把它读回来,在其上继续算。这等价于**用序列长度(token 数)换取计算深度**——CoT 让总有效计算量从 O(L) 提升到约 O(L × 生成步数)。这就是为什么它对**串行依赖深的多步任务**(算术、逻辑链)提升最大,而对一步就能判的任务(简单分类)几乎没用。**这个解释有理论支撑**:近年有工作证明,带 CoT(中间 token)的 Transformer 表达能力**严格强于**无 CoT 的同规模模型,本质就是把计算从"深度"摊到了"长度"上。「具体复杂度类的刻画(如 CoT 让 Transformer 可模拟更大的电路类)是活跃理论方向,精确表述待核」

### 机制二:给"思考"腾出 token 预算 + 条件化到正确推理分布

第二个机制和自回归的概率结构有关,补充第一个:

```
无 CoT 直接出答案:模型必须在【最终答案那个 token】上,
   一次性把全部概率质量押对。p(答案 | 问题) 这个分布,要跨越"问题→答案"的巨大语义鸿沟,
   高熵、难压中。

有 CoT:把生成分解为
   p(答案, 推理 | 问题) = p(推理 | 问题) · p(答案 | 推理, 问题)
   ↑ 每一步 p(stepᵢ | step_{<i}, 问题) 都是【局部、低难度】的下一步预测,
     模型在预训练里见过海量"推理文本"(数学题解、代码、论证),这些条件分布它压得很准。
   → 一旦正确的推理链被生成出来,p(答案 | 推理, 问题) 几乎是确定性的(顺着推理读出答案)。
```

换句话说:**CoT 把"一个巨大的、高熵的跳跃"分解成"一串小的、低熵的、模型擅长的局部预测"**,每一步都把模型**条件化(condition)**到更接近答案的上下文上。到最后一步,问题已被推理链化简到答案几乎呼之欲出。这也解释了 CoT 的一个脆弱性:**如果中间某步推错,错误会沿链传播**(后续步骤条件在错误中间结果上),所以 CoT 不是万能,它把"一步错"换成了"链式累积误差"的风险。

### 代码:CoT prompt 模板与"zero-shot CoT"

```python
def build_cot_prompt(demos, query):
    """demos: [(question, reasoning, answer), ...] —— 每个示例含【推理过程】。
    机制核心:示例里展示'怎么一步步想',诱导模型对 query 也生成推理链。"""
    blocks = []
    for q, reasoning, a in demos:
        blocks.append(f"Q: {q}\nA: {reasoning} 所以答案是 {a}。")   # 把推理写进示例
    blocks.append(f"Q: {query}\nA:")                              # query 留空,等模型自己生成推理
    return "\n\n".join(blocks)

# Zero-shot CoT(Kojima et al., 2022):不给推理示例,只加一句【咒语】
def zero_shot_cot(question):
    # "Let's think step by step." 这句话本身就把模型推到"先生成推理"的分布上
    return f"Q: {question}\nA: 让我们一步一步思考。"
# 神奇之处:仅这一句,在大模型上就能触发多步推理、显著提升——
# 因为预训练语料里"一步步思考"后面【几乎总是跟着推理过程】,这句话起到任务/格式定位作用(呼应贝叶斯说)
```

注意 **zero-shot CoT** 这个彩蛋:Kojima 等(2022)发现连示例都不用给,只在问题后加一句"Let's think step by step",大模型就会自己展开推理、准确率大涨。**用前面的机制看毫不神秘**:这句话在预训练语料里后面几乎总跟着推理文本,它把模型的续写分布**定位**到"推理模式",从而触发机制一(摊开计算)和机制二(分解为局部预测)。这是隐式贝叶斯(任务定位)和 CoT(摊开计算)两个机制的一次漂亮合流。

### 分析:CoT 的成本与适用边界

```
CoT 的代价:生成 token 数从 ~1(直接答案)涨到 T(推理链长度)
  → 推理延迟与成本 ≈ ×T(逐 token 自回归,每个都要一次前向)
  → 用算力(更多生成步)换准确率,本质是【test-time compute】的早期形态

适用:串行依赖深的任务(多步算术、逻辑、代码推理)收益大
不适用:单步可判的任务(简单分类/检索)—— 加 CoT 徒增成本甚至引入跑偏风险
涌现性:小模型用 CoT 常【更差】(推理链生成质量不够,错误累积反而拖累)——与 ICL 同样是规模涌现
```

**一句话定位 CoT:它是"用生成长度换计算深度"的算力交易。** 你多花 T 倍生成成本,换取把超出单次前向深度的串行计算变得可行。理解了这一点,后续课程里的 self-consistency(采样多条推理链投票)、tree-of-thought、乃至 o1 系列"长思考"(把 test-time compute 推到极致)都顺理成章——**它们都是同一条主线:在推理时花更多算力让模型多想,而不是只靠预训练那一锤子。**

## 六、Prompt 工程的科学:为什么"换个顺序就掉 10 个点"

ICL 强大,但**脆弱得反直觉**。同一批示例,换个顺序、换个标点、换个标签词,准确率可能剧烈波动。这不是模型"不稳定"的玄学,每条脆弱性都能用前面的机制解释。把它讲成科学,你才能稳定地用好 prompt。

### 现象与机理:三个主要敏感维度

**1. 示例顺序敏感(order sensitivity)。** Lu et al. (2022) 发现,**仅打乱 few-shot 示例的顺序,同一组示例的准确率可以从接近 SOTA 跌到接近随机。** 机理有二:(a) 自回归 + 因果注意力下,**靠后的示例对最后查询位置的影响通常更大**(位置更近、注意力衰减更少),所以"哪个示例排最后"会偏置预测——这是 recency 效应;(b) 某些顺序会让模型产生**majority-label 偏置**(比如末尾连续几个都是 positive,模型倾向多输出 positive)。

**2. 标签词与格式敏感。** 把标签从 `positive/negative` 换成 `A/B` 或 `好/坏`,性能会变;分隔符从 `=>` 换成 `:` 也会变。机理:**ICL 高度依赖"格式/标签空间"这些元信息来定位任务**(正是 Min 等的发现)。换标签词等于换了任务定位的锚点,模型对"这个标签词在预训练里关联什么"很敏感。

**3. 标签分布偏置(label bias)。** 模型对某些标签词有**先验偏好**(比如 " positive" 这个 token 本身在语料里更常见、先验概率更高),导致预测向高频标签倾斜。Zhao et al. (2021, "Calibrate Before Use") 给了一个干净的修正:

```
Contextual Calibration(上下文校准)思路:
1. 喂一个【内容为空】的查询(如 "N/A" 或空字符串)给同一个 few-shot prompt,
   读出各标签的概率 p_cf(content-free 概率)—— 这反映模型对标签的【纯先验偏置】。
2. 用 p_cf 去【除/校准】真实查询的标签概率,抵消先验偏置:
   p_calibrated(y | x) ∝ p(y | x) / p_cf(y)
3. 等价于学一个仿射变换 W·p + b 把 content-free 输入的预测拉回均匀分布。
   → 实测能显著降低对顺序/格式的敏感,并提升平均准确率。
```

### 工程准则:从机制推出的实操建议

把上面的机理翻译成可执行的 prompt 工程准则(每条都标了它对应哪个机制):

```
1. 示例选择:选与查询【语义相近】的示例(检索式 few-shot / kNN-prompting)。
   依据:贝叶斯说——相近示例对正确任务 θ* 的似然贡献更大,后验集中更快。
2. 示例数量:不是越多越好。前几个收益最大(后验指数集中),
   注意上下文成本 O(L²) 和 recency 干扰。依据:贝叶斯 + 成本分析。
3. 顺序:对顺序极敏感的任务,用【多个顺序投票/采样】或做校准,别赌单一顺序。
   依据:order sensitivity + recency 效应。
4. 格式一致:示例与查询用【完全相同】的模板和分隔符。
   依据:格式是任务定位的元信息,不一致 = 任务定位漂移。
5. 标签词:用预训练里【语义清晰、频率均衡】的标签词;必要时做 contextual calibration。
   依据:label bias + 标签空间敏感。
6. 复杂推理:加 CoT(few-shot 给推理过程,或 zero-shot 加"一步步思考"),
   并可叠加 self-consistency(采样多条链投票)。依据:CoT 的计算深度论。
7. 评估纪律:报告【多个示例集 × 多个顺序】的均值±方差,而非挑一个最好的 prompt 报。
   依据:ICL 高方差,单点结果不可信(呼应 know:N=1 不能声明 feasibility)。
```

**最重要的一条 meta 准则:永远报告方差,不要 cherry-pick prompt。** ICL 的高方差意味着"我调出一个 prompt 跑了一次得 90 分"几乎没有信息量——它可能是这组示例这个顺序的运气。**学术诚实和工程可靠都要求你跨多个示例集、多个顺序评估,报均值和波动。**

## 设计权衡与常见坑

**1. 把 ICL 当成"免费的微调"——成本被隐藏在每次推理里。** ICL 训练成本为零,但每次调用都要重付 O((K·ℓ)²) 的上下文成本。高频、稳定的任务,几十万次调用累加的推理成本会远超一次微调。**判据:任务稳定且高频 → 微调;任务多变、长尾、无标注 → ICL。**

**2. 以为示例标签必须正确——在某些 regime 下不是。** Min 等显示(在那个规模/任务下)标签错了也不太掉点,因为起作用的是格式/标签空间/输入分布。**但别过度推论**:大模型 + 多示例时标签开始真的重要(Wei 2023)。**坑:在"标签不重要"的认知下随便标,结果换到大模型/复杂任务上被反咬。**

**3. 把某一个机制假说当定论去做产品决策。** 隐式贝叶斯、隐式 GD、任务向量都只覆盖部分现象,且大模型 + 自然语言上的机制远未定论。**坑:基于"ICL 就是梯度下降"去设计某个依赖它精确成立的系统,在真实 LLM 上可能不成立。** 当工具用其直觉,别当定理用。

**4. CoT 到处加。** CoT 只在串行依赖深的任务上有正收益;简单任务上它增加成本、还可能因为生成跑偏引入错误。小模型用 CoT 常更差(推理链质量不够)。**坑:无脑给所有任务套"一步步思考",拖慢且可能掉点。**

**5. CoT 的误差沿链传播。** 中间一步错,后续都条件在错误上。CoT 把"单点错"换成"链式累积误差"。**缓解:self-consistency(多采样投票)、让步骤可验证(如代码可执行)。**

**6. Prompt 评估只跑一次、只报最好。** ICL/CoT 高方差,单次或 cherry-pick 的结果不可复现、会误导决策。**铁律:多示例集 × 多顺序,报均值±方差。**(直接呼应 know:N=1 不能声明 feasibility。)

**7. 忽视位置/recency 效应做长 few-shot。** 示例越多越长,靠后的示例影响越大、靠前的可能被"淹没",且超长 prompt 触发长上下文的"中间遗忘"(lost in the middle,前置见《10-长上下文》)。**把最相关的示例放在离查询近的位置,通常更稳。**

**8. 标签词随手选。** 选了一个低频或多义的标签 token,label bias 会把预测带偏。**用语义清晰、频率均衡的词,必要时 contextual calibration。**

## 动手练习

**练习 1(编码 + 分析题:示例顺序与数量的影响)。** 用 `gpt2-large`(或更大模型)和本章 `build_fewshot_prompt`,在一个二分类任务(如情感)上:(a) 固定一组 K=4 示例,枚举它全部排列(4!=24 种顺序),统计在一个小测试集上的准确率分布,报均值、标准差、最好/最差。(b) 把 K 从 0 扫到 8(每个 K 随机采若干组示例),画准确率随 K 的曲线。(c) 用你跑出的数据,验证或反驳两个论断:"顺序影响很大""收益随 K 递减"。
- *提示*:(a) 用 `itertools.permutations`。准确率读法:比较候选标签 token 的 logit 取 argmax。你大概率会看到最好与最差顺序差距明显(order sensitivity),且方差不小——这正是"必须报方差"的实证理由。(b) 曲线大概率前段陡、后段平(呼应后验指数集中)。

**练习 2(推导题:把"一步 GD = 一层线性注意力"自己推一遍)。** 不看正文第三节,自己完成:(a) 对最小二乘损失 `L(w)=½Σᵢ(wᵀxᵢ−yᵢ)²`,从 w₀=0 出发写出一步梯度下降后的 w₁,并写出对查询的预测 ŷ=w₁ᵀx_q。(b) 写出一层线性注意力(query=x_q, key=xᵢ, value=yᵢ)对查询位置的输出。(c) 逐项对比,指出学习率 η 被吸收进了注意力的哪个部分,并说明为什么用 softmax 注意力时这个等价只剩"近似"。
- *提示*:(a) ∇L|₀ = −Σ yᵢxᵢ,w₁ = ηΣyᵢxᵢ,ŷ = ηΣyᵢ(xᵢᵀx_q)。(b) Σ(x_qᵀxᵢ)yᵢ。(c) η 进 Q/K/V 投影的缩放;softmax 把"线性加权 Σ scoreᵢ·valueᵢ"变成"归一化指数加权",不再是纯线性,故等价退化为近似(对应某种核化/预条件 GD)。

**练习 3(分析题:三种机制假说的对照与判别)。** 做一张表,对隐式贝叶斯、隐式梯度下降、任务向量三个假说,填:(a) 核心主张一句话;(b) 最强证据(及证据类型:行为相关 / 构造可表达 / 因果干预);(c) 它解释不了的一个现象;(d) 它对"翻转示例标签会怎样"分别预测什么。最后回答:Min 等"标签错了不掉点"和 Wei 2023"大模型翻标签会跟着翻"两个看似矛盾的结果,如何用"不同 regime 下不同机制主导"来调和?
- *提示*:(d) 贝叶斯/任务识别预测"标签不重要(只要格式/标签空间对)";梯度下降预测"标签重要(它是监督信号)";调和点——小模型/少示例 regime 偏任务识别(标签不重要),大模型/多示例 regime 偏真学映射(标签重要),两假说各管一段。

**练习 4(编码 + 推导题:验证 CoT 是"用长度换深度",并实现 contextual calibration)。** (a) 设计一个**多步算术**任务(如连续 5 次加减),分别用"直接答案"和"CoT"prompt 测同一模型,比较准确率;再设计一个**单步分类**任务做对照,验证"CoT 只对深串行任务有用"。(b) 实现本章的 contextual calibration:用 content-free 输入("N/A")估计各标签先验 p_cf,对真实预测做 `p/p_cf` 校准,比较校准前后的准确率与对顺序的敏感度(配合练习 1 的顺序枚举)。
- *提示*:(a) 直接答案在步数变多时应明显掉点(单次前向深度不够),CoT 应保持;单步任务上两者接近甚至 CoT 略差(印证适用边界)。(b) 校准后准确率通常上升、方差下降;直觉:你把"模型对空输入就有的偏好"减掉了,剩下的才是输入真正带来的信号。

## 小结与承上启下

这一章的全部分量压在一个反范式的事实上:**学习不一定要改权重——一个只被训练去"预测下一个词"的大模型,在规模够大后,会在前向传播的激活里临时"学会"一个 prompt 当场告知的新任务,且这种能力是涌现的。** 由此:

- **现象**:zero/one/few-shot 随 K 和规模提升,学习发生在激活(不是参数),代价是每次推理 O(L²) 的上下文成本。
- **三大机制假说,互补而非互斥**:(1) **隐式贝叶斯**——示例更新任务后验、模型在"认任务"(行为层证据,Min 等标签随机化);(2) **隐式梯度下降**——前向可构造性地模拟对示例做 GD,一层线性注意力 = 一步 GD(可表达性证据,本章给了可验算的推导);(3) **任务向量**——示例被压成激活里一个可提取、可移植的向量(因果干预证据)。**一个自洽叙事**:注意力头读完示例聚合出任务向量(≈后验编码,可能以 GD 式精炼),后续层条件其上续写。**没有一个是定论,大模型 + 自然语言上的机制仍是开放问题。**
- **Chain-of-Thought**:不是玄学,是**用生成长度换计算深度**——把超出单次前向深度(O(L))的多步串行计算,摊进 T 个中间 token,用自回归 + 注意力当可读写草稿纸,有效深度升到 O(L×T);同时把"一个高熵跳跃"分解成"一串模型擅长的低熵局部预测"。代价是 ×T 的推理成本,只对深串行任务有正收益。
- **Prompt 工程是科学**:顺序/格式/标签的敏感性都能从机制推出,工程准则(相近示例、格式一致、校准、报方差)有据可依;**铁律是报均值±方差、不 cherry-pick**。

**在整门课里的位置。** 前面你学的是"模型怎么被造出来、怎么被训练、怎么被低成本微调";这一章是**第一次把视角从"训练时"切到"推理时"**——发现推理时本身就是一个可以注入信息、可以"现学"、可以用算力换智能(CoT)的舞台。**承上**:ICL 是 scaling(第 8 章)的涌现产物,prompt 直接作用在你学过的注意力(第 2/3 章)和上下文(第 10 章)上,与 PEFT(第 11 章)构成"改权重 vs 改上下文"的一对范式。**启下**:理解了"推理时可以用更多计算让模型更聪明"(CoT 是它的萌芽),你就握住了通往后续两大方向的钥匙——一是**对齐**(RLHF/DPO 把"按指令/偏好行事"训进权重,让 zero-shot 就听话,减少对 few-shot 的依赖),二是**推理时计算扩展(test-time scaling)**(self-consistency、tree-of-thought、长思考的 o1 系列),它们都是 CoT"花更多算力多想一会儿"这条主线的极致延伸。**至此你已经能读懂 ICL 这摊还在打架的机制论文,也能科学地而非玄学地设计和评估一个 prompt。**
