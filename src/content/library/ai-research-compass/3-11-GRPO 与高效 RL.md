> 这一章把你从"知道 PPO 能训 RLHF、也大概知道 R1 用 RL 训出了推理"带到"能从策略梯度的方差结构推导出 GRPO 为什么可以扔掉 critic、组内均值凭什么是合法 baseline、那个 KL 正则项为什么写成 `π_ref/π_θ − log(π_ref/π_θ) − 1` 而不是普通的 log 比值,并能自己写出一份数值正确的 GRPO advantage 计算与 loss"。学完你应当能向人讲清:R1-Zero 的意义不在分数,而在它证明了什么;以及 GRPO 省的到底是哪一块显存、用什么代价换来的。

## 为什么这一章绕不过去

前一章你已经把 PPO 拆透了:裁剪目标 `min(rₜ·Âₜ, clip(rₜ,1−ε,1+ε)·Âₜ)` 给策略更新加了软刹车,GAE 在偏差与方差之间平滑权衡地估计优势 `Âₜ`。但 PPO 有一个在小规模 RL(游戏、机器人)里不痛、一搬到大语言模型上就变成主要成本的零件:**critic(价值网络)**。

PPO 要算优势 `Âₜ = Σ (γλ)ˡ δₜ₊ₗ`,而每个 TD 残差 `δₜ = rₜ + γV(sₜ₊₁) − V(sₜ)` 都依赖一个学出来的价值函数 `V(s)`。在 Atari 上,`V` 是个小网络,几乎免费。但在 LLM RLHF / 推理训练里,**critic 通常和 policy 同样大**(同一个 transformer 骨架加一个标量回归头),原因是它要在同样的高维语言状态空间里估计价值——一个 7B 的 policy 配一个 7B 的 critic,显存、优化器状态、前反向算力直接翻倍。更糟的是:

- **critic 自己是另一个不稳定来源。** 它用自举(bootstrap)目标 `rₜ + γV(sₜ₊₁)` 回归,目标随自身移动;在稀疏、延迟、噪声大的语言奖励上,价值估计常年偏差很大,而 PPO 的优势又完全依赖它。critic 估歪了,policy 就沿着错的优势爬坡。
- **价值的语义在 LLM 里很可疑。** "一句话说到一半时的状态价值是多少"这个量,在数学上定义清楚,但在实践里极难学准——大部分 token 处的真实价值差异很小,critic 学到的是噪声。

DeepSeek 在训练 R1 时给出的回答是:**对推理这种任务,根本不需要逐 token 的价值估计;我们要的只是一个 baseline 来给优势做减法,而 baseline 可以不用网络去学,用"同一道题采样一组答案、拿组内平均奖励"就够了。** 这就是 GRPO(Group Relative Policy Optimization,组相对策略优化)的全部出发点。

**贯穿全章的判断标准**:一个 RL 算法里的某个零件是不是必需,要看去掉它之后梯度估计是否变得有偏(bias),还是只是方差(variance)变大。**有偏不可接受;方差变大但仍无偏,可以用更多采样补回来。** GRPO 扔 critic 的合法性,完全建立在"组内均值是无偏 baseline"这一条上——这一条我们会从头证。

## 一、先把地基夯实:策略梯度、baseline 与方差

GRPO 不是凭空冒出来的,它是策略梯度 + baseline 这条老线的一个特例。要讲透它为什么成立,必须先把 baseline 的数学讲到能手推。

### 问题:策略梯度为什么需要一个减数

策略梯度定理(REINFORCE,Williams 1992)说:要最大化期望回报 `J(θ) = E[R]`,梯度是

```
∇θ J(θ) = E_{τ~πθ} [ Σₜ ∇θ log πθ(aₜ|sₜ) · Gₜ ]

其中 Gₜ = 从 t 步起的累积回报(return),τ 是一整条轨迹
```

直觉:把"导致高回报的动作"的对数概率往上推、低回报的往下压,推多猛由 `Gₜ` 这个权重决定。问题是 `Gₜ` 是个高方差的随机变量——同一个状态、同一个动作,因为后续随机性,这次回报 100、下次 −50,梯度方向乱晃,收敛极慢。

**关键技巧:减去一个 baseline `b(s)`**,把权重从 `Gₜ` 换成 `Gₜ − b(sₜ)`:

```
∇θ J(θ) = E [ Σₜ ∇θ log πθ(aₜ|sₜ) · (Gₜ − b(sₜ)) ]
```

我们要证两件事:(1) 任何只依赖状态、不依赖动作的 `b(s)` 都**不改变梯度的期望(无偏)**;(2) 选对 `b(s)` 能**显著降低方差**。

### 推导一:为什么减去 baseline 不引入偏差

核心是一个被反复用到的恒等式——**score function 的期望为零**:

```
E_{a~πθ(·|s)} [ ∇θ log πθ(a|s) ]
  = Σₐ πθ(a|s) · ∇θ log πθ(a|s)        (展开期望)
  = Σₐ πθ(a|s) · ∇θ πθ(a|s) / πθ(a|s)   (∵ ∇log f = ∇f / f)
  = Σₐ ∇θ πθ(a|s)
  = ∇θ Σₐ πθ(a|s)                        (求和与求导可交换)
  = ∇θ (1)                               (∵ 概率归一,Σₐ πθ = 1)
  = 0                                     ∎
```

有了这个,baseline 项的期望立刻塌掉(因为 `b(sₜ)` 不依赖 `aₜ`,可以提到对动作的期望外面):

```
E [ ∇θ log πθ(aₜ|sₜ) · b(sₜ) ]
  = E_{sₜ} [ b(sₜ) · E_{aₜ~πθ} [ ∇θ log πθ(aₜ|sₜ) ] ]   (先对动作求期望)
  = E_{sₜ} [ b(sₜ) · 0 ]                                  (用上面的恒等式)
  = 0                                                       ∎
```

**结论(加粗)**:只要 baseline 不依赖被采样的动作 `aₜ`,把 `Gₜ` 换成 `Gₜ − b(sₜ)` 后,梯度的期望分毫不变——它仍是 `∇θ J(θ)` 的无偏估计。这就是所有 baseline / actor-critic 方法合法性的根。注意条件是"不依赖动作":一个依赖动作的 baseline 会引入偏差,这是个常见的坑。

### 推导二:最优 baseline 与"为什么均值就够好"

无偏之后,我们挑 `b` 来最小化方差。设单状态、标量梯度 `g = ∇θ log πθ(a|s)`,优势权重 `(G − b)`。估计量是 `g·(G − b)`,其方差:

```
Var[ g·(G−b) ] = E[ g²(G−b)² ] − ( E[g(G−b)] )²
```

第二项 `E[g(G−b)] = E[gG]`(因为 `E[gb] = b·E[g] = 0`,与 b 无关)。所以最小化方差只需最小化第一项关于 b 的部分。对 b 求导置零:

```
d/db E[ g²(G−b)² ] = E[ −2 g² (G−b) ] = 0
  ⟹  E[g²G] = b · E[g²]
  ⟹  b* = E[g²·G] / E[g²]          ← 理论最优 baseline,是用 g² 加权的回报均值
```

这告诉我们两件事:

1. **理论最优 baseline 是"梯度幅度平方加权的回报均值"**,不是简单算术均值。但它需要知道 `g²` 与 `G` 的联合分布,实践中估不准。
2. **简单的算术均值 `b = E[G]`(用 `g²` 当常数权重)是 b\* 的一个良好近似**,在 `g²` 与 `G` 相关性不强时几乎一样好。它不需要任何额外网络,只要把同一状态下若干次采样的回报平均一下。

**这正是 GRPO 的全部魔法。** 对推理任务,"状态"就是 prompt(问题),"动作"就是整条回答,"回报"就是这条回答的奖励 `r`(答案对不对 + 格式对不对)。对同一个 prompt 采样 G 条回答,它们共享同一个状态 `s = prompt`,于是组内平均奖励 `mean(r₁..r_G)` 正好是 `b = E[G|s]` 的蒙特卡洛估计——一个合法(无偏)且接近最优的 baseline,而且**完全不需要 critic**。

### 量化:用 critic vs 用组均值,方差差在哪

两者都无偏,差别在方差和成本。设 prompt 处真实价值(期望奖励)为 `V*(s) = E[r|s]`。

- **PPO/critic 路线**:用学出来的 `V(s) ≈ V*(s)` 当 baseline。如果 critic 学得准,`G − V(s)` 的方差很小,**单条样本**就能给出低方差的优势——这是 critic 的优势,它把"跨样本的平均"压进了一个函数里,采 1 条就能用。代价是:critic 本身有估计误差 `V(s) − V*(s)`,这个误差不引入梯度偏差(因为它仍不依赖动作),但会让方差降得不彻底;更要命的是显存翻倍 + 训练不稳。
- **GRPO/组均值路线**:`b = mean(r₁..r_G)` 是 `V*(s)` 的无偏估计,但它是**用同组其它样本算的**,所以要采 G 条才能给这组定 baseline。组内标准化后的优势

```
Âᵢ = ( rᵢ − mean({r₁..r_G}) ) / std({r₁..r_G})
```

用组内样本标准差再做了一次归一,把不同 prompt(有的题奖励普遍高、有的普遍低)的尺度拉齐,等效于自适应学习率。**代价是采样量:G 条里有 1 条的"自由度"被均值吃掉了**(后面坑里细说),且 baseline 的估计方差是 ~σ²/G,要 G 够大才稳。

一句话对比:**critic 用"模型容量"换"采样量",GRPO 用"采样量"换"模型容量"。** 对 LLM,采样(推理)相对便宜、再养一个大模型相对贵且不稳,所以 GRPO 的权衡在推理训练上很划算。

## 二、GRPO 的目标函数:PPO-clip 的骨架 + 组基线的优势 + 直接 KL

把上面的 baseline 接进 PPO 的裁剪骨架,就得到 GRPO。我们逐项推。

### 从单优势到组优势

GRPO 对每个 prompt `q` 从旧策略 `π_θ_old` 采样一组 G 条输出 `{o₁,...,o_G}`,每条得一个奖励 `rᵢ`(来自规则/可验证奖励或奖励模型)。组内标准化优势如前:

```
Âᵢ = ( rᵢ − mean(r) ) / ( std(r) + ε_std )      # ε_std 防止整组奖励相同时除零(全对/全错)
```

**一个关键的简化设计**:在 outcome-supervision(只给整条答案一个最终奖励)的 GRPO 里,**同一条输出 oᵢ 的所有 token 共享同一个优势 `Âᵢ`**。也就是说,优势不再逐 token 变化(对比 PPO+GAE 是逐 token 的)。这背后的含义是:GRPO 放弃了 critic 提供的逐 token 信用分配,把"这条答案整体好/坏"的信号均摊到它的每个生成 token 上。对推理任务,最终答案对不对本就是条级别(整条)的信号,这个简化是匹配的;但它也意味着 GRPO 对"一条长答案里哪几步推理才是关键"分辨不出来——这是它精度上的本质局限。

### GRPO 目标函数(完整写出)

设 `oᵢ` 的第 t 个 token 为 `oᵢ,ₜ`,概率比

```
ρᵢ,ₜ(θ) = πθ(oᵢ,ₜ | q, oᵢ,<ₜ) / π_θ_old(oᵢ,ₜ | q, oᵢ,<ₜ)
```

GRPO 最大化的目标(写成对一个 prompt 的形式,实际对一个 batch 的 prompt 取平均):

```
J_GRPO(θ) =
  (1/G) Σᵢ  (1/|oᵢ|) Σₜ  {
       min( ρᵢ,ₜ · Âᵢ ,  clip(ρᵢ,ₜ, 1−ε, 1+ε) · Âᵢ )    ← PPO-clip 骨架,优势换成组优势
       −  β · D_KL[ πθ ‖ π_ref ]                          ← KL 正则,直接进目标,不进 reward
  }
```

逐零件说明它和 PPO 的同与异:

- **`min(·, clip(·))` 完全是 PPO 的裁剪。** 作用一样:`Âᵢ > 0` 时不让 `ρ` 涨过 `1+ε`,`Âᵢ < 0` 时不让 `ρ` 跌过 `1−ε`,给单步更新封顶,防策略崩。
- **优势 `Âᵢ` 来自组标准化,不来自 critic + GAE。** 这是和 PPO 最大的结构差异——**整套价值网络消失了**。
- **KL 正则项 `−β·D_KL[πθ‖π_ref]` 直接加在每个 token 的目标里。** 这与经典 RLHF 把 KL 揉进 reward(`r ← r − β·log(πθ/π_ref)`)的做法不同:GRPO 把它显式地作为损失项。`π_ref` 是冷启动后的参考策略(通常是 SFT 模型),冻结不动。作用是拴住策略别离 SFT 模型太远,防 reward hacking 和能力崩塌(catastrophic forgetting)。

### 推导三:那个非常规的 KL 估计器为什么长那样

GRPO 用的 KL 不是 `log(πθ/π_ref)`,而是下面这个(对每个 token):

```
D̂_KL = π_ref(oᵢ,ₜ|·)/πθ(oᵢ,ₜ|·) − log( π_ref(oᵢ,ₜ|·)/πθ(oᵢ,ₜ|·) ) − 1
```

这是 John Schulman 推广的 **k3 估计器**。要看懂为什么用它,先明确我们能拿到什么:我们是从 `πθ` 采样的,但只能逐 token 算出**采到的那个 token**的概率,算不出整个词表上的求和——所以真 KL `Σ πθ·log(πθ/π_ref)` 无法直接算,只能用从 `πθ` 采样的样本去**估**它。

设 `x = π_ref/πθ`(注意是 ref 比 θ),记 `r = log x`。三种估计器:

```
k1 = −log x = −r              ← 朴素估计:E_{πθ}[−r] = E[log(πθ/π_ref)] = KL[πθ‖π_ref] ✓ 无偏
k2 = (1/2) r²                 ← 平方估计:无偏性只在二阶近似下成立
k3 = x − r − 1 = (π_ref/πθ) − log(π_ref/πθ) − 1   ← GRPO 用的
```

先验证 **k3 无偏**。对 `x = π_ref/πθ` 求在 `πθ` 下的期望:

```
E_{πθ}[ x ] = Σ πθ · (π_ref/πθ) = Σ π_ref = 1     (π_ref 归一)

E_{πθ}[ k3 ] = E[x] − E[r] − 1
            = 1 − E_{πθ}[ log(π_ref/πθ) ] − 1
            = − E_{πθ}[ log(π_ref/πθ) ]
            = E_{πθ}[ log(πθ/π_ref) ]
            = KL[ πθ ‖ π_ref ]                      ∎ k3 无偏
```

再看 **为什么不用无偏又简单的 k1,偏要用 k3**。两个理由,都关乎方差与稳定:

1. **k3 恒非负,k1 可正可负。** 由 `log x ≤ x − 1`(对数的标准切线不等式,等号仅在 x=1)立即得 `k3 = x − 1 − log x ≥ 0`,逐样本就是 KL 该有的样子(KL 永远 ≥ 0)。而 k1 的单样本 `−log x` 可以是负的——某个采到的 token 恰好 `π_ref > πθ` 时 `k1 < 0`,虽然期望对,但单 batch 里出现"负 KL"会让正则方向瞬间反过来(变成鼓励远离 ref),引入噪声甚至不稳。
2. **k3 方差更低。** k1 = −log x 的方差由 `log x` 的散布决定,在 x 偏离 1 较多时(策略和 ref 在某 token 上差很多)`log x` 尾巴很长;k3 = (x−1) − log x 把线性主项 `x−1` 减掉了一部分波动,经验上方差显著更小(Schulman 的分析与实验,**待核** 具体方差比值)。

**一句话**:k3 是一个**无偏、恒非负、低方差**的 KL 单样本估计器,这三条正好是你在 RL loss 里加 KL 项最想要的性质。这是个典型的"无偏的最简单写法不一定最好用,要在无偏前提下挑方差和数值性质都好的那个"的例子。

### 量化:GRPO 省了多少显存与算力

以一个 7B policy 为例,粗算稳态显存(沿用混合精度 + Adam,每参数约 16 字节;数量级估算,**待核** 具体数字随实现差异):

```
PPO 需常驻的大模型:
  policy        7B × 16B/param ≈ 112 GB(参数+梯度+优化器状态)
  critic        7B × 16B/param ≈ 112 GB
  reference     7B × 2B/param  ≈  14 GB(只前向,fp16,无梯度无优化器)
  reward model  7B × 2B/param  ≈  14 GB(只前向)
  ──────────────────────────────────────
  四个模型,其中两个"满状态"(policy + critic)

GRPO 去掉 critic:
  policy        ≈ 112 GB
  reference     ≈  14 GB
  reward model  ≈  14 GB(若用规则/可验证奖励,连这个也省掉!)
  ──────────────────────────────────────
  只剩一个满状态模型 + 一两个推理态模型
```

省的不只是那 ~112 GB:**少了一个要反向传播和 Adam 更新的大模型**,等于训练步的前反向算力和优化器开销也降一大块。在推理 RL 里,奖励常来自可执行验证(数学答案匹配、代码跑测试),连 reward model 都能省,GRPO 实际上可能只需常驻一个满状态 policy + 一个冻结的 reference。**这就是"省显存又稳"的物理来源:扔掉的不是一个小部件,是一个和主模型同量级的、还自带不稳定性的大模型。**

## 三、GRPO 的代码:从 advantage 到 loss

把上面的数学落成可运行的核心片段。先是组标准化优势——GRPO 的灵魂只有这几行:

```python
import torch

def grpo_advantages(rewards: torch.Tensor, group_size: int, eps: float = 1e-4):
    """
    rewards: 形状 [B] 的一维张量,B = num_prompts * group_size,
             同一 prompt 的 G 条回答在内存里连续排放。
    返回:每条回答的标量优势 Âᵢ,形状 [B]。
    """
    # reshape 成 [num_prompts, G],每行是同一道题的一组回答
    r = rewards.view(-1, group_size)                      # [P, G]
    mean = r.mean(dim=1, keepdim=True)                    # 组内均值 = baseline b(s)
    std  = r.std(dim=1, keepdim=True)                     # 组内标准差,做尺度归一
    adv  = (r - mean) / (std + eps)                       # Âᵢ = (rᵢ − mean) / (std + ε)
    return adv.view(-1)                                   # 摊平回 [B]
    # 注意:整组奖励全相同时(全对/全错),std=0,被 eps 兜底,该组优势≈0,
    #       这组样本对梯度几乎无贡献——这是 GRPO 在"太简单/太难的题"上学不到东西的根源。
```

注意这里**没有任何价值网络、没有 GAE、没有逐 token 折扣**:一道题的优势完全由它那一组答案的奖励排布决定。

再看 KL 估计与 token 级 loss。下面是注释化的核心训练片段,体现机制而非工程全貌:

```python
def grpo_loss(logp_new, logp_old, logp_ref, advantages, mask,
              clip_eps=0.2, beta=0.04):
    """
    logp_new : 当前策略 πθ 对每个生成 token 的 log 概率,  形状 [B, T]
    logp_old : 采样时旧策略 π_old 的 log 概率(detach),    形状 [B, T]
    logp_ref : 参考策略 π_ref 的 log 概率(detach,冻结),  形状 [B, T]
    advantages: 每条回答的标量优势 Âᵢ,                     形状 [B] —— 关键:广播到该条所有 token
    mask     : 1=生成的 response token,0=prompt 或 padding,形状 [B, T]
    """
    adv = advantages.unsqueeze(1)                         # [B,1] -> 广播到 [B,T],整条共享同一优势

    # 概率比 ρ = πθ/π_old,用 log 差再 exp,数值更稳
    log_ratio = logp_new - logp_old                       # [B,T]
    ratio = torch.exp(log_ratio)

    # PPO-clip 骨架(优势已是组优势)
    unclipped = ratio * adv
    clipped   = torch.clamp(ratio, 1 - clip_eps, 1 + clip_eps) * adv
    policy_term = torch.min(unclipped, clipped)           # [B,T],要最大化

    # k3 KL 估计:x = π_ref/πθ = exp(logp_ref - logp_new)
    log_ref_ratio = logp_ref - logp_new                   # log(π_ref/πθ) = r
    x = torch.exp(log_ref_ratio)                          # π_ref/πθ
    kl = x - log_ref_ratio - 1.0                          # k3 = x − r − 1 ≥ 0,无偏估 KL[πθ‖π_ref]

    # 每个 token 的目标(注意符号:loss = −目标)
    per_token = policy_term - beta * kl                   # [B,T]

    # 按 mask 做"每条回答内 token 平均",再对 batch 平均(对应公式里的两层 1/G、1/|oᵢ|)
    per_seq = (per_token * mask).sum(dim=1) / mask.sum(dim=1).clamp(min=1)  # [B]
    loss = -per_seq.mean()                                # 取负,因为优化器做最小化
    return loss
```

几个实现上极易踩错、但决定成败的细节:

- **`advantages` 必须广播到该回答的每个 token(`unsqueeze(1)`)**,而不是只放在最后一个 token 上。这对应"整条共享同一优势"。
- **`logp_old` 与 `logp_ref` 都要 `detach`**,它们不参与梯度;`π_old` 是采样快照,`π_ref` 是冻结的 SFT 模型。
- **token 平均的归一化方式有讲究。** 上面按"每条回答内先平均、再 batch 平均"写,对应原始 GRPO 公式里的 `(1/G)Σᵢ (1/|oᵢ|)Σₜ`。后续工作(如 DAPO,**待核** 细节)指出这种逐序列长度归一会在长度上引入偏置(长回答里每个 token 权重被摊薄),提出改成"整 batch 按总 token 数归一"。这是个真实的、影响推理训练长度行为的坑,新手容易照抄公式而不知有此偏置。
- **`clip` 是单边失效保护,不是双边对称。** 当 `Âᵢ > 0` 真正生效的是上界 `1+ε`,`Âᵢ < 0` 时是下界 `1−ε`;`min` 自动选对方向。

## 四、DeepSeek-R1 的配方:R1-Zero 与多阶段 R1

GRPO 是算法零件,R1 是把它用到极致的工程配方。两个产物要分开理解。

### R1-Zero:纯 RL,无 SFT 冷启动,推理能力"涌现"

**做法**:直接拿基座模型(DeepSeek-V3-Base),**不做任何监督微调,不喂任何人工标注的推理过程**,上来就用 GRPO 做 RL。奖励极简,主要是**可验证奖励(RLVR,Reinforcement Learning with Verifiable Rewards)**:

- **准确率奖励**:数学题答案能否被规则匹配(如要求 `\boxed{}` 里的答案与标准答案相等)、代码能否通过测试用例。这类奖励**不需要奖励模型**,是确定性的、不可被 hack 的(答案对就是对)。
- **格式奖励**:是否把思考过程放进 `<think>...</think>`、最终答案放进规定标签。

**突破点(加粗)**:在这种"只告诉它最终对不对"的训练下,模型**自发地**学会了延长思维链、做验算、回溯重想。DeepSeek 报告里那个著名的 "aha moment"——模型在推理中途写出类似"等等,让我重新评估这一步"的反思——**不是被任何训练数据教的,是奖励信号诱导出来的**。随着训练推进,模型在 AIME 等数学基准上的回答平均长度持续增长,准确率随之系统性上升(具体曲线与分数 **待核**)。

**这件事的意义远超分数本身,是本章最该被记住的一点**:它从经验上支持了一个强命题——**复杂推理策略(分解、验算、回溯、自我怀疑)不必由人类示范注入,可以作为最大化"最终答案正确率"这一标量目标的副产物涌现出来。** 这把"RL 是用来对齐/调教已有能力的工具"的旧观念,推进到了"RL 可以从可验证信号里长出新的认知行为"。它呼应了 AlphaZero 的精神:**AlphaZero 在棋盘上靠自我博弈 + 搜索长出了超人棋感而不用人类棋谱;R1-Zero 在思维链上靠 RL + 可验证奖励长出了推理策略而不用人类推理示范。** 搜索的舞台从棋盘换成了文字草稿纸。

**但 R1-Zero 不是没代价。** 它的产物可读性差、语言混杂(中英混着推、出现无意义重复)、格式不稳——因为没有任何"像人写的"约束,模型只对"答案对不对"负责,推理过程怎么乱都行。这直接催生了完整版 R1 的多阶段设计。

### R1:冷启动 SFT + 多阶段 RL

完整 R1 的配方是为了"既要 R1-Zero 涌现出的推理力,又要可读、对齐、通用"。它分多个阶段(阶段命名与精确次序以原报告为准,**待核**;以下是被广泛复述的主干):

```
阶段 0  冷启动 SFT
   用少量高质量、长思维链数据(部分由 R1-Zero 生成后人工/规则筛选清洗)
   先对基座做 SFT,把"可读的、规整的长推理格式"打底。
   作用:给后续 RL 一个稳定的起点和 π_ref —— 解决 R1-Zero 可读性差、训练早期不稳的问题。

阶段 1  面向推理的 RL(GRPO + 可验证奖励)
   在冷启动模型上做大规模 GRPO,奖励 = 准确率(可验证) + 格式 + 语言一致性。
   语言一致性奖励是专门为治 R1-Zero 的"语言混杂"加的。
   这一步把推理能力推到高位。

阶段 2  拒绝采样 + SFT(扩展到通用能力)
   用阶段 1 的强推理模型对大量 prompt 采样,只保留正确/优质的回答(拒绝采样,
   rejection sampling),再混入写作、问答、自我认知等通用 SFT 数据,做一轮 SFT。
   作用:把"会推理"扩展成"会推理且通用、会写人话"。

阶段 3  全场景 RL(对齐人类偏好)
   再做一轮 RL,这次奖励同时覆盖推理的可验证奖励 + 通用任务上的有用性/无害性
   (这部分用奖励模型,回到 RLHF 式偏好)。
   作用:最终对齐,兼顾推理硬实力与对话有用性、安全性。
```

读这套配方的正确姿势是看**每个阶段在解决前一阶段暴露的什么问题**:R1-Zero 证明纯 RL 能涌现推理但产物粗糙 → 冷启动 SFT 给可读性和稳定起点 → 推理 RL 把硬实力拉满 → 拒绝采样 SFT 补通用能力 → 全场景 RL 做最终对齐。**这是一条"先用极简纯 RL 验证能力上限存在,再用工程化多阶段把它驯化成产品"的典型路径。**

另外 DeepSeek 还做了**蒸馏**:用 R1 生成的高质量推理数据去 SFT 一批更小的稠密模型(基于 Qwen / Llama 系),发现**小模型直接 SFT 蒸馏 R1 的推理轨迹,效果比让小模型自己跑 RL 更好**。这暗示:**推理能力的"发现"很贵(要大模型 + 大量 RL),但"复制"很便宜(蒸馏轨迹即可)**——能力一旦被一个强模型涌现出来,就能低成本搬运给小模型。

## 五、设计权衡与常见坑

把 GRPO/R1 用对,踩坑清单比公式更值钱。

**1. 组大小 G 的权衡。** G 太小,组均值 baseline 的估计方差 ~σ²/G 太大,优势噪声大;G 太大,采样成本线性上升(每个 prompt 要生成 G 条完整回答,推理开销是大头)。实践常取 G 在 8~64 量级(**待核** 具体取值)。**坑**:G 个样本里有 1 个自由度被均值"用掉"了(组内优势之和恒为 0),所以有效信号样本是 G−1,G 很小时这个损失不可忽略。

**2. 全对 / 全错的整组零优势。** 当一道题 G 条全对(`std=0`)或全错,组内标准化后优势全为 0,这组对梯度无贡献。**后果**:太简单(总是全对)和太难(总是全错)的题都学不到东西,有效学习只发生在"难度适中、组内有对有错"的题上。**实践对策**:做难度课程(curriculum)、过滤掉常年全对/全错的样本、动态调难度。这是 GRPO 的一个结构性脆弱点。

**3. 奖励 hacking——RL 永远的幽灵。** 即便用可验证奖励,模型也会找捷径:只优化"答案匹配"可能学会**不推理直接猜答案**(尤其选择题)、或**在 `<think>` 里写废话凑格式分**、或学会**输出能骗过答案匹配正则但实际错误的字符串**。**对策**:奖励设计要堵漏洞(答案抽取要严格)、KL 项拴住别离 SFT 太远(防退化成胡说)、监控思维链质量而非只看最终分数。**新手最常见的失败**是只盯着 reward 曲线涨就以为成功,实则模型在 hack——一定要人工抽看生成的推理过程。

**4. KL 系数 β 的两难。** β 太大,策略被死死拴在 SFT 模型上,学不动、推理力上不去;β 太小,策略可能漂移到胡言乱语、能力崩塌(忘掉 SFT 学的东西)。**坑**:β 还和奖励的尺度耦合——奖励量级一变,等效的 β 就变了,所以换奖励设计后 β 常要重调。有些实现干脆把 KL 项去掉、只靠 clip 控制步长(在纯可验证奖励、强格式约束下有人这么做,**待核** 适用边界)。

**5. 训练-推理概率不一致(off-by-one 级的致命 bug)。** 采样时用的推理引擎(如 vLLM)和训练时用的前向,可能因 kernel、精度、采样温度处理不同,导致 `logp_old` 算得和真实采样分布对不上。**后果**:概率比 `ρ` 系统性偏移,clip 形同虚设,训练悄悄发散。**这是工程上最隐蔽、最常吃亏的坑**,务必校验采样时记录的 logp 与训练前向重算的 logp 一致。

**6. 优势标准化是双刃剑。** 除以组内 std 把不同题的尺度拉齐(好处),但当某组奖励方差很小时,会把微小的随机差异放大成大优势(坏处)——`std + ε` 里的 ε 既防除零也防这种放大。**坑**:ε 取太小,近似全对的组会被噪声主导;取太大,正常组的优势被压扁。

## 六、动手练习

**练习 1(推导题,核心)**:从头证明"组内均值是合法 baseline",并说明它与理论最优 baseline 的差距。
(a) 用第一节的 score function 恒等式,证明:对同一个 prompt 的一组回答,把第 i 条的回报 `rᵢ` 减去组均值 `mean(r)` 后再乘 `∇log π`,梯度期望不变(无偏)。
(b) 写出理论最优 baseline `b* = E[g²G]/E[g²]`,并论证:当 `g² = (∇log π)²` 与回报 `G` 近似独立时,`b*` 退化为算术均值 `E[G]`。
*提示*:(a) 关键是 baseline 不依赖被采样的那条回答自身的"动作随机性"——但注意组均值里**包含**了第 i 条自身。严格做法是用"留一(leave-one-out)均值"`b₋ᵢ = mean(r 去掉 rᵢ)`,它与第 i 条动作独立,严格无偏;实践用含自身的均值有 O(1/G) 的轻微偏差,G 大可忽略。这个细节本身就是答案的一部分,想清楚它你就真懂了。

**练习 2(分析题)**:量化对比 PPO(critic)与 GRPO(组基线)的成本与方差。
(a) 写出两者要常驻显存的大模型清单,估算一个 N 参数模型下各自的峰值显存量级(用每参数 16 字节满状态、2 字节推理态)。
(b) 论证:critic 路线"采 1 条样本即可得低方差优势但有 critic 估计误差",组基线路线"必须采 G 条、baseline 方差 ~σ²/G 但无模型偏差"。在什么情形下你会选 critic,什么情形选组基线?
*提示*:从"模型容量贵 vs 采样贵"的相对成本切入。LLM 推理可批量并行、相对便宜,再养一个大 critic 既费显存又添不稳定性——这正是推理 RL 倒向 GRPO 的根本原因。

**练习 3(推导题)**:证明 k3 KL 估计器 `x − log x − 1`(`x = π_ref/πθ`)的三条性质。
(a) 无偏:`E_{πθ}[x − log x − 1] = KL[πθ‖π_ref]`(照第二节推一遍)。
(b) 恒非负:由 `log x ≤ x − 1` 得逐样本 `≥ 0`。请进一步说明等号何时成立,以及它为什么对应"策略与 ref 完全一致"。
(c) 与朴素估计器 `k1 = −log x` 对比:k1 也无偏,为什么 GRPO 不用 k1?从单样本符号(k1 可负)和方差两个角度回答。
*提示*:(b) 用 `f(x)=x−1−log x`,求导 `f'(x)=1−1/x`,在 x=1 取唯一极小且 f(1)=0。(c) 想"单 batch 里出现负 KL 会让正则项瞬间反向"意味着什么。

**练习 4(编码题)**:实现并测试 `grpo_advantages`,然后做一个反直觉验证。
(a) 用本章代码实现组标准化优势,构造一组人工奖励(如某 prompt 的 8 条回答奖励为 `[1,1,0,0,1,0,1,0]`),手算验证输出。
(b) 构造"整组全对"`[1,1,1,1]` 和"整组全错"`[0,0,0,0]`,验证优势接近 0,并解释这为什么意味着"太简单/太难的题对训练无贡献"。
(c) 把 `std` 归一去掉(只减均值不除 std),在两组奖励尺度差很大的 prompt 上对比有无 std 归一的优势分布,说明 std 归一起到了什么"自适应学习率"的作用。
*提示*:(c) 想象 A 题奖励都在 [0.9,1.0]、B 题都在 [0,1],不除 std 时 B 题的优势会系统性地比 A 题大,导致 batch 被 B 题主导。

## 小结与承上启下

这一章的主轴只有一条因果链:**PPO 的 critic 对 LLM 太贵又不稳 → 但 critic 的唯一职责是提供一个 baseline → baseline 只要不依赖动作就无偏(score function 恒等式)→ 同一 prompt 一组答案的平均奖励正是这样一个无偏 baseline → 于是 critic 可以整个扔掉,这就是 GRPO。** 它用"多采几条样本"换掉了"再养一个大模型",在推理 RL 这个采样相对便宜、可验证奖励唾手可得的场景里,是一笔极划算的交易。R1-Zero 进一步证明了:在可验证奖励下,长链推理这种复杂认知行为可以纯靠 RL 涌现,不必人类示范;完整 R1 则示范了怎么用多阶段把这份"野生能力"驯化成可读、通用、对齐的产品。

往回看,GRPO 是 PPO(上一章)在"奖励是条级别、状态价值难估"这一特定结构下的自然简化,它和 DPO(把 RL 整个换成监督式损失)代表了"绕开 critic / 绕开在线 RL"的两种不同思路——DPO 走得更远(连采样都省)但天花板低,GRPO 保留在线采样和 clip 因而能力上限更高,这正是推理训练最终选 GRPO 而非 DPO 的原因。

往后看,GRPO 的两个结构性局限——**无法做逐 token 信用分配**(整条共享一个优势)、**整组全对/全错时零信号**——直接定义了后续算法的改进方向:更细的过程奖励(process reward,给推理的中间步骤打分)、更好的难度课程与采样策略、以及把 GRPO 搬到 **agent / 多步工具使用**这种长程、稀疏、延迟奖励的场景时,信用分配这块 RL 最硬的骨头会重新摆回桌面。这些正是后面几章——过程奖励与可验证奖励的精细化、以及 agent RL 的长程信用分配——要接着啃的内容。你现在手里已经有了那条最关键的工具:**当一个 RL 零件看起来昂贵时,先问它到底在提供 bias 修正还是 variance 修正——如果只是后者,大概率能用更便宜的采样换掉它。**
