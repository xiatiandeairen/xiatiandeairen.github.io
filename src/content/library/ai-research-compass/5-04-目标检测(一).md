> 这一章把你从"会做图像分类"带到"能在白板上推出 bbox 回归的偏移参数化、徒手实现 IoU 与 NMS、讲清 RPN 为什么能取代选择性搜索、并算清 RoI Pooling 的量化误差从哪来"。目标检测是从"图里有什么"跨到"什么东西在哪、有多大"的第一步,两阶段路线是这一步最干净、最值得吃透的范式。

## 一句话引言

目标检测要同时做两件事:**定位(在哪,bbox)+ 分类(是什么)**。两阶段路线(R-CNN 家族)把它拆成"先提出一批可能含物体的候选框,再对每个候选框分类+精修",一步步把候选框生成、特征共享、固定尺寸池化、端到端训练这四个瓶颈逐个解决,最终在 Faster R-CNN 里收敛成一个完全可微、用网络自己生成候选框(RPN)的统一架构。本章把这条演进里的每一个数学机制——IoU、anchor、偏移回归、smooth L1、NMS、RoI Pooling——全部推透。

## 1. 动机:检测比分类难在哪

### 1.1 任务定义与输出空间

图像分类的输出是**一个**类别(或一个类别分布):输入一张图 → 输出 "猫"。目标检测的输出是**一个变长集合**:输入一张图 → 输出 {(框₁, 类别₁, 置信度₁), (框₂, 类别₂, 置信度₂), ...}。一张图里可能有 0 个、1 个、几十个物体,数目事先不知道。

一个**边界框(bounding box, bbox)**通常用 4 个数表示。两种等价的常见编码:

```
角点式(corner):  (x₁, y₁, x₂, y₂)   左上角 + 右下角
中心式(center):  (xc, yc, w, h)     中心坐标 + 宽高
```

互相转换:`xc=(x₁+x₂)/2, yc=(y₁+y₂)/2, w=x₂−x₁, h=y₂−y₁`。两种编码后面都要用:IoU/NMS 用角点式算面积最方便,bbox 回归用中心式做参数化最自然(§4)。

检测难在三处,后面整章都在拆这三个难点:

- **输出是变长集合**。分类网络输出固定维度向量,检测要输出"个数不定的结构化对象"。怎么让固定结构的网络吐出变长结果?——靠"在大量候选位置上各做一次判断,再筛掉冗余"。
- **定位需要回归连续坐标**。分类只需把概率质量压到正确类别;定位要预测 4 个**连续**的几何量,且要对物体尺度、长宽比鲁棒。
- **正负样本极度不均衡**。一张图里"含物体"的区域是少数,"背景"是绝大多数。任何"在很多位置上判断有无物体"的方案,天然面对 1:1000 量级的正负失衡(§6)。

### 1.2 评价指标:IoU 与 mAP(先建立直觉)

要谈"框预测得准不准",得先有**框与框的相似度**。这就是 **IoU(Intersection over Union,交并比)**:两个框交集面积除以并集面积。

```
IoU(A, B) = area(A ∩ B) / area(A ∪ B) = area(A ∩ B) / (area(A) + area(B) − area(A ∩ B))
```

IoU ∈ [0, 1]:完全不重叠为 0,完全重合为 1。它对**平移和缩放都敏感**(框偏一点、大小不对,IoU 都掉),又对**绝对尺度无关**(同样形状的大框小框,只要相对错位一致,IoU 一样)。这个"尺度无关"性质正是它适合做检测度量的原因——大物体小物体用同一个阈值衡量。

检测的总指标是 **mAP(mean Average Precision)**:对每个类别,在某个 IoU 阈值下(经典 PASCAL VOC 用 0.5,COCO 用 0.5:0.95 多阈值平均),把预测按置信度排序,逐个判定 TP/FP(预测框与某个未匹配的真值框 IoU ≥ 阈值且类别对 → TP,否则 FP),画出 precision-recall 曲线并求曲线下面积(AP),再对所有类别取平均。本章不展开 mAP 计算细节(放数据/评测章),但你要记住:**IoU 是检测里一切"准不准"判断的原子操作**——训练时用它分配正负样本,推理时用它做 NMS,评测时用它判 TP/FP。下面先把 IoU 实现出来。

### 1.3 代码:IoU 的向量化实现

```python
import torch

def box_iou(boxes1, boxes2):
    """
    boxes1: [N, 4]  boxes2: [M, 4],  均为角点式 (x1, y1, x2, y2)
    返回 iou: [N, M],  iou[i, j] = IoU(boxes1[i], boxes2[j])
    """
    # 各自面积:[N], [M]。clamp(min=0) 防止退化框(x2<x1)给出负面积
    area1 = (boxes1[:, 2] - boxes1[:, 0]).clamp(min=0) * (boxes1[:, 3] - boxes1[:, 1]).clamp(min=0)
    area2 = (boxes2[:, 2] - boxes2[:, 0]).clamp(min=0) * (boxes2[:, 3] - boxes2[:, 1]).clamp(min=0)

    # 广播求交集左上角(取两框 x1/y1 的较大者)、右下角(取 x2/y2 的较小者)
    # boxes1[:, None, :2] -> [N,1,2];boxes2[:, :2] -> [M,2];广播成 [N,M,2]
    lt = torch.max(boxes1[:, None, :2], boxes2[None, :, :2])   # [N, M, 2]
    rb = torch.min(boxes1[:, None, 2:], boxes2[None, :, 2:])   # [N, M, 2]

    wh = (rb - lt).clamp(min=0)                # 交集宽高,无重叠时 clamp 到 0 -> [N, M, 2]
    inter = wh[:, :, 0] * wh[:, :, 1]          # 交集面积 [N, M]

    union = area1[:, None] + area2[None, :] - inter
    return inter / union.clamp(min=1e-6)       # 防 0 除
```

要点:**交集 = max(左上) 到 min(右下) 围出的矩形**,无重叠时这个矩形宽或高为负,clamp 到 0 让交集面积归零。`clamp(min=1e-6)` 防止两个零面积退化框相除出 NaN。复杂度 O(N·M),全程张量广播,无 Python 循环——这是检测代码里最高频的算子,务必写成向量化。

## 2. R-CNN:把检测硬拆成"提框 + 分类"

### 2.1 核心思路

R-CNN(Girshick et al., 2014)的逻辑极其朴素,却第一次把 CNN 用进检测并大幅刷新精度:

```
1. region proposal:用"选择性搜索"(Selective Search)在图上生成 ~2000 个候选区域(region proposals)
2. warp + CNN:把每个候选区域裁剪出来、缩放(warp)到固定尺寸(227×227),各自送进一个 CNN(AlexNet)提特征
3. 分类:对每个区域的 CNN 特征,用一组每类一个的线性 SVM 判类别
4. 框精修:再用一组类相关的线性回归器,对框做一次坐标精修(bbox regression)
5. NMS:对每个类别做非极大值抑制,去掉重复检测
```

**选择性搜索(Selective Search, Uijlings et al., 2013)**:一种不依赖学习的候选框生成法。先做过分割(把图切成很多小超像素),再按颜色、纹理、尺寸、填充度等相似度**自底向上贪心合并**相邻区域,合并过程中每一级的区域外接框都作为候选。它的设计目标是**高召回**:宁可多吐框(2000 个),也要尽量保证真实物体被某个框覆盖(后续阶段可以筛掉多余的,但漏掉的找不回来)。这是"提框阶段优先召回、判别阶段优先精度"这一分工的源头。

### 2.2 为什么慢:重复计算

R-CNN 精度好,但慢到几乎不能用(单图推理几十秒)。瓶颈在第 2 步:**2000 个候选区域,每个都独立缩放后跑一遍完整 CNN**。这些候选区域在原图上**大量重叠**,重叠部分的卷积特征被反复计算了无数遍。CNN 前向是检测里最贵的部分,把它重复 2000 次,代价线性爆炸。

```
R-CNN 单图 CNN 前向次数 ≈ 2000(每个 proposal 一次)
其中绝大部分卷积是在重叠区域上重复算的浪费
```

还有两个结构性问题:(1) 训练是**多阶段流水线**——CNN 微调、SVM 训练、回归器训练分三步独立做,无法端到端,特征和分类器各自为政;(2) 特征要**写到磁盘**再喂 SVM,存储和 IO 开销巨大。这三个问题(重复卷积、多阶段、磁盘 IO)直接定义了 Fast R-CNN 要解决什么。

> 一句话记住 R-CNN 的历史地位:它证明了"CNN 特征 + 区域提议"这条路能把检测精度拉上一个台阶,同时把"为什么慢"的答案明明白白摆出来——**特征没共享**。

## 3. Fast R-CNN:共享卷积 + RoI Pooling

### 3.1 关键洞察:卷积只跑一次

Fast R-CNN(Girshick, 2015)的破局点:**整张图只过一次卷积主干,得到一张特征图;所有候选框共享这张特征图**。候选框不再各自裁原图跑 CNN,而是把自己在原图上的坐标**映射到特征图上的对应区域**,直接从特征图上"取"自己那块特征。

```
原图 H×W  ──卷积主干(下采样 stride=S)──▶  特征图 (H/S)×(W/S)×C
proposal 在原图坐标 (x1,y1,x2,y2)  ──除以 S──▶  特征图上的 RoI 区域
```

这一下把"跑 2000 次 CNN"变成"跑 1 次 CNN + 2000 次廉价的特征切片"。**重复卷积的浪费被彻底消除**,这是从几十秒到亚秒级的根本原因。

### 3.2 RoI Pooling:把任意大小区域变成固定尺寸

新问题来了:不同 proposal 在特征图上对应的区域**大小不一**(大物体的 RoI 大、小物体的 RoI 小),但后面接的全连接分类/回归头要求**固定维度输入**。怎么把任意 h×w 的 RoI 变成固定的 H'×W'(比如 7×7)?

**RoI Pooling** 的做法:把 RoI 在特征图上的区域**均匀切成 H'×W' 个格子(bin)**,对每个格子内的特征做 max pooling,得到 H'×W' 的固定输出。

```
RoI 在特征图上大小 h × w,目标输出 H' × W'(如 7×7)
每个 bin 的大小:  bin_h = h / H',  bin_w = w / W'
第 (i, j) 个 bin 覆盖的特征图区域:
    行 ∈ [ floor(i·bin_h),  ceil((i+1)·bin_h) )
    列 ∈ [ floor(j·bin_w),  ceil((j+1)·bin_w) )
对该区域做 max pooling -> 输出第 (i, j) 个值
```

不管 RoI 多大,输出永远是 H'×W'×C。这样所有 proposal 经过 RoI Pooling 后都有相同维度,可以拼成 batch 一起过全连接头。

**RoI Pooling 的两次取整(量化)**——这是本章重点之一,也是下一章 RoIAlign 的引子。注意上面有两处把连续坐标"对齐到整数网格"的操作:

```
量化 ①(RoI 边界):proposal 坐标除以 stride S 后一般不是整数,
                   RoI Pooling 把它 round/floor 到整数特征图坐标。
量化 ②(bin 边界):h/H' 一般不整除,bin 边界 floor/ceil 到整数行列。
```

**量化误差的量级估计**。设特征图下采样步长 S(如 ResnNet C4 特征 S=16)。量化①把 RoI 在原图上的边界对齐到了"S 的整数倍"网格,单边最大偏差约 S/2 个像素;量化②再在 bin 内引入约 (bin 尺寸) 量级的对齐偏差。两者叠加:

```
RoI 边界在原图上的对齐误差 ≈ ±S/2 像素(S=16 时约 ±8 像素)
单个 bin 在原图上的尺寸 ≈ (bin_w · S) 像素,bin 内特征"错位半个 bin"是常态
```

对一个 16×16 像素的小物体,±8 像素的边界误差几乎等于物体本身一半的尺寸——**小目标定位被量化严重伤害**。这就是 Fast/Faster R-CNN 在小目标上精度受限的一个直接原因,也是 Mask R-CNN(He et al., 2017)提出 **RoIAlign**(用双线性插值,不做任何取整)来解决的问题,下一章详讲。

### 3.3 代码:RoI Pooling 的机制示意

```python
import torch
import torch.nn.functional as F

def roi_pool_single(feat, roi, output_size, spatial_scale):
    """
    机制示意版(逐 bin max pool),理解量化用;生产请用 torchvision.ops.roi_pool。
    feat:         [C, Hf, Wf]   主干输出的单图特征
    roi:          (x1, y1, x2, y2)  proposal 在【原图】上的角点坐标
    output_size:  (H', W'),如 (7, 7)
    spatial_scale: 1/stride,如 1/16,把原图坐标缩到特征图坐标
    """
    C, Hf, Wf = feat.shape
    Hp, Wp = output_size

    # ---- 量化①:RoI 边界对齐到整数特征图坐标(这一步引入边界量化误差)----
    x1 = int(round(roi[0] * spatial_scale)); y1 = int(round(roi[1] * spatial_scale))
    x2 = int(round(roi[2] * spatial_scale)); y2 = int(round(roi[3] * spatial_scale))
    roi_w = max(x2 - x1, 1); roi_h = max(y2 - y1, 1)   # 保底 1,防退化

    bin_h = roi_h / Hp; bin_w = roi_w / Wp
    out = feat.new_full((C, Hp, Wp), float("-inf"))

    for i in range(Hp):
        for j in range(Wp):
            # ---- 量化②:bin 边界 floor/ceil 到整数行列 ----
            hs = y1 + int(torch.floor(torch.tensor(i * bin_h)))
            he = y1 + int(torch.ceil(torch.tensor((i + 1) * bin_h)))
            ws = x1 + int(torch.floor(torch.tensor(j * bin_w)))
            we = x1 + int(torch.ceil(torch.tensor((j + 1) * bin_w)))
            hs, he = max(0, hs), min(Hf, max(he, hs + 1))
            ws, we = max(0, ws), min(Wf, max(we, ws + 1))
            out[:, i, j] = feat[:, hs:he, ws:we].amax(dim=(1, 2))  # bin 内 max pool
    return out  # [C, H', W']
```

两个 `round`/`floor`/`ceil` 就是量化误差的全部来源。RoIAlign 把它们换成"采样点 + 双线性插值",完全不取整——记住这个对照,下一章会直接拿它做实验。

### 3.4 多任务损失:分类与回归一起学

Fast R-CNN 把分类和框回归放进**同一个网络的两个并行头**,用一个联合损失同时训练(端到端,除了候选框生成还靠外部选择性搜索之外)。对每个 RoI,损失为:

```
L = L_cls(p, u) + λ · [u ≥ 1] · L_loc(tᵘ, v)
```

- `p`:分类头输出的 (K+1) 类概率(K 个前景类 + 1 个背景类),`L_cls(p, u) = −log p_u` 是标准交叉熵,`u` 是真值类别。
- `v = (vx, vy, vw, vh)`:真值框相对该 RoI 的回归目标(偏移参数化,§4)。
- `tᵘ`:回归头为类别 u 预测的偏移。注意回归头对**每个前景类各输出一组 4 个偏移**(class-specific regression),损失只取真值类 u 对应的那 4 个。
- `[u ≥ 1]`:**艾弗森括号(指示函数)**,前景(u≥1)时为 1、背景(u=0)时为 0——**背景框不计回归损失**(背景没有"正确的框",回归没有意义)。
- `λ`:平衡两项的权重,原文取 1。

这个"分类交叉熵 + 仅前景计的定位损失 + 指示函数门控"的结构,会原样传给 Faster R-CNN 的两个阶段。下一节专门把 `L_loc` 里的偏移参数化和 smooth L1 推透——这是检测损失里最值得讲清的部分。

## 4. bbox 回归:偏移参数化与 smooth L1

### 4.1 为什么回归偏移,而不是绝对坐标

一个最自然的想法是:让网络直接输出框的绝对坐标 (x1,y1,x2,y2)。这在检测里几乎不可行,原因有三:

1. **尺度问题**。绝对坐标的取值范围随图像分辨率从几到几千。网络要在如此大的动态范围里回归,数值上极难训练,且换个分辨率就失效。
2. **缺乏平移/尺度不变性**。同一个物体出现在图左上和右下,绝对坐标完全不同,但"该框相对它所在候选框该怎么调整"应该是同一种模式。直接回归绝对坐标,网络得为每个位置单独学一套,样本效率极低。
3. **有现成的参考系**。每个预测都附着在一个候选框(R-CNN 里是 proposal,Faster R-CNN 里是 anchor)上。**预测"相对这个参考框要怎么微调"远比预测绝对位置容易**——把绝对定位问题降成了一个局部的、归一化的小修正问题。

所以检测统一用**回归相对参考框的偏移**。设参考框(anchor 或 proposal)中心宽高为 (xa, ya, wa, ha),真值框为 (x, y, w, h),定义回归目标:

```
tx = (x  − xa) / wa          ty = (y  − ya) / ha
tw = log(w / wa)             th = log(h / ha)
```

网络预测 (t̂x, t̂y, t̂w, t̂h),推理时反解回真实框:

```
x̂  = t̂x · wa + xa           ŷ  = t̂y · ha + ya
ŵ  = wa · exp(t̂w)           ĥ  = ha · exp(t̂h)
```

### 4.2 为什么这样参数化(逐项推导其合理性)

**中心偏移除以宽高(tx, ty)**:中心位移 `(x−xa)` 的"合理量级"应随框大小缩放——大框允许大位移,小框只允许小位移。除以 `wa, ha` 后,`tx, ty` 变成"以参考框自身尺寸为单位的相对位移",**对框尺度归一化**。这样不同大小的框,回归目标落在同一量级(通常 ~O(1)),网络好学。

**宽高取对数(tw, th)**:为什么不直接回归 `w/wa`?三个理由,推一遍:

```
理由A(非负约束自动满足):
    宽高必须 > 0。若回归 w/wa 这个比值,网络输出可能为负 -> 框宽为负,非法。
    改回归 log(w/wa),则 ŵ = wa·exp(t̂w),exp 恒正 -> 宽高天然 > 0,无需额外约束。

理由B(乘性误差对称):
    框尺度是乘性的:"预测大了 2 倍"和"预测小了 2 倍"应同等惩罚。
    log 把乘性变加性:log(2·wa/wa)=+log2,log(0.5·wa/wa)=−log2,绝对值相等 -> 对称。
    若回归比值 w/wa,预测 2 倍的误差是 |2−1|=1,预测 0.5 倍是 |0.5−1|=0.5,不对称、偏向高估。

理由C(梯度/量级稳定):
    物体尺度跨几个数量级(小到 16px,大到几百 px),w/wa 比值动态范围大;
    log 压缩动态范围,使 tw 落在小区间,数值稳定。
```

一句话:**(tx,ty) 用"除宽高"实现尺度归一化,(tw,th) 用 log 实现非负性 + 乘性对称 + 动态范围压缩**。这套参数化从 R-CNN 沿用到 Faster R-CNN、SSD、YOLO 系列,是检测的事实标准。

### 4.3 smooth L1:为什么不用 L2

定位损失逐项作用在 4 个偏移上:`L_loc(t, v) = Σ_{k∈{x,y,w,h}} smooth_L1(t_k − v_k)`。其中

```
              ⎧ 0.5 · x²         if |x| < 1
smooth_L1(x) = ⎨
              ⎩ |x| − 0.5       otherwise
```

**为什么不用 L2(平方损失)**。L2 = x²,其梯度 dL/dx = 2x:误差越大,梯度越大。检测训练里,回归目标常含**离群点(outlier)**——比如某个 anchor 与真值框匹配得很勉强,偏移目标 v 很大。L2 下这种大误差样本贡献的梯度 ∝ 误差,会**主导整个 batch 的梯度方向,把训练带偏甚至梯度爆炸**。

**为什么不用 L1(绝对值损失)**。L1 = |x|,梯度恒为 ±1,对离群点鲁棒。但 L1 在 x=0 处**不可导**(梯度从 −1 突跳到 +1),在最优解附近来回振荡、不收敛到精确值。

**smooth L1 取两者之长**:小误差区(|x|<1)用二次,梯度 ∝ x,在 0 附近平滑、能精细收敛;大误差区(|x|≥1)切换成线性,梯度被**截断在 ±1**,离群点不再主导梯度。验证它确实平滑拼接(函数值与一阶导都连续):

```
连续性 @ x=1:
    左段值 0.5·1² = 0.5;  右段值 |1| − 0.5 = 0.5      ✓ 函数值连续
导数:
    左段 d/dx(0.5x²) = x,  在 x=1 处 = 1
    右段 d/dx(|x|−0.5) = sign(x), 在 x=1⁺ 处 = 1        ✓ 一阶导连续
梯度幅度上界:
    |d smooth_L1/dx| = min(|x|, 1) ≤ 1   <-- 关键:大误差梯度被钳在 1
```

最后这行 `|grad| ≤ 1` 是 smooth L1 的灵魂:**任何样本对回归梯度的贡献绝对值不超过 1**,从根上掐死离群点对优化的破坏。这就是检测损失普遍用 smooth L1(及其后继 Huber loss / L1 loss + 各种 IoU loss)而非 L2 的原因。

> 顺带:smooth L1 就是 β=1 的 **Huber loss**。后续工作(如 Faster R-CNN 的某些实现)会引入超参 β 控制拐点位置:`|x|<β` 用 `0.5x²/β`,否则 `|x|−0.5β`,β 越小越接近纯 L1。这是「待核」具体框架默认值的地方,但机制不变。

## 5. anchor 与 RPN:Faster R-CNN 的核心

### 5.1 anchor:在特征图上铺满"预设框"

Fast R-CNN 还有一条命脉攥在外人手里:候选框仍由选择性搜索生成,它**跑在 CPU 上、约 2 秒/图**,成了新的瓶颈(卷积加速后,提框反而成了最慢环节)。Faster R-CNN(Ren et al., 2015,NeurIPS)的核心贡献:**用一个小网络 RPN(Region Proposal Network)在特征图上自己生成候选框,与检测网络共享主干卷积,几乎零额外成本**。

RPN 的基础是 **anchor(锚框)**:在特征图的**每个空间位置**,预设一组固定大小和长宽比的参考框。原文用 3 种尺度 × 3 种长宽比 = **9 个 anchor / 位置**:

```
尺度(scale):    {128², 256², 512²}  像素面积(在原图尺度上)
长宽比(ratio):   {1:2, 1:1, 2:1}
每位置 anchor 数 k = 3 × 3 = 9
```

特征图大小为 (H/S)×(W/S)(S 为主干下采样步长,典型 16),则全图 anchor 总数:

```
anchor 总数 = (H/S) × (W/S) × k
例:H=W=800, S=16 -> 50×50×9 = 22500 个 anchor
```

anchor 给了网络一个"密集铺满全图、覆盖各种尺度长宽比"的参考框集合。**网络不从零猜框,而是判断每个 anchor 含不含物体、并预测它该怎么微调**——这正是 §4 偏移回归的用武之地。

**为什么 anchor 能取代选择性搜索**:选择性搜索靠手工规则在图上找候选;anchor 是"先验地、均匀密集地撒框,再让网络打分筛选"。只要 anchor 的尺度/长宽比覆盖了数据集里物体的分布,密集铺设就能保证高召回——而"打分筛选"是网络擅长的、可学习的、能跑在 GPU 上的。**手工提框 → 密集先验 + 学习筛选**,这是从 Fast 到 Faster 的范式跃迁。

### 5.2 RPN 结构:滑窗预测 objectness + 偏移

RPN 是接在主干特征图上的一个**轻量全卷积小网络**:

```
主干特征图 [C, Hf, Wf]
   │
   ├─ 3×3 卷积(512 通道) + ReLU    <-- 在特征图上"滑窗",每个位置看 3×3 邻域
   │        输出 [512, Hf, Wf]
   │
   ├─ 1×1 卷积 -> 分类分支:  [2k, Hf, Wf]   每位置 k 个 anchor,各 2 个 logit(物体/背景)
   └─ 1×1 卷积 -> 回归分支:  [4k, Hf, Wf]   每位置 k 个 anchor,各 4 个偏移 (tx,ty,tw,th)
```

几个关键点:

- **objectness 是二分类**,不分具体类别——RPN 只回答"这个 anchor 里有没有**某个**物体"(前景 vs 背景),具体是猫是狗留给第二阶段。这让 RPN 极轻量、且与类别数解耦。
- **全卷积、滑窗即卷积**。"在每个位置滑窗预测"在实现上就是 1×1 卷积——卷积本身就是滑窗。所以 RPN 对全图所有 anchor 的预测**一次卷积前向同时算完**,极快。
- **回归是 anchor-specific**,每个 anchor 各预测自己的 4 个偏移,反解时叠加到该 anchor 的 (xa,ya,wa,ha) 上(§4.1)。

### 5.3 RPN 的训练:正负样本分配与采样

RPN 怎么知道哪个 anchor 该判前景?**用 IoU 把 anchor 和真值框匹配**,定义正负标签(原文规则):

```
正样本(label = 1)条件(满足其一):
    (a) 与某个真值框 IoU 最高的 anchor(保证每个真值框至少有一个正 anchor)
    (b) 与任意真值框 IoU > 0.7 的 anchor
负样本(label = 0):
    与所有真值框 IoU < 0.3 的 anchor
忽略(不计损失):
    IoU 介于 0.3 ~ 0.7 之间的 anchor(标签模糊,不参与训练)
```

规则 (a) 是兜底:若某真值框恰好没有任何 anchor 与它 IoU>0.7(小物体或奇怪长宽比常见),它仍能通过"IoU 最高"拿到一个正样本,**否则这个物体在训练里完全没有正样本去学**。

**正负不均衡与采样**。一张图 ~2 万个 anchor,正样本(前景)往往只有几十个,负样本上万——若全用,负样本梯度淹没一切,网络直接学成"全判背景"。Faster R-CNN 的对策:**每图随机采样 256 个 anchor 组成一个 mini-batch,正负比例尽量 1:1(正样本不足 128 个时用负样本补足)**。

```
RPN 每图采样:  256 个 anchor,正:负 ≈ 128:128
正样本不足则用负样本填满 256
```

RPN 的损失沿用 §3.4 的多任务结构:`L = (1/N_cls)Σ L_cls(pᵢ, pᵢ*) + λ(1/N_reg)Σ pᵢ* · L_loc(tᵢ, tᵢ*)`,分类用二分类交叉熵,定位用 smooth L1,**且 `pᵢ*` 因子保证只有正 anchor 计回归损失**(背景 anchor 没有目标框)。这种"采样平衡 + 仅正样本计回归"是两阶段法压制不均衡的标准手段;后续单阶段法因正负更悬殊,催生了 **Focal Loss**(下一章/检测二),那是另一条解法线。

### 5.4 Faster R-CNN 全流程与四步训练

把两个阶段拼起来,Faster R-CNN = **共享主干 + RPN(阶段一)+ RoI 头(阶段二,即 Fast R-CNN 头)**:

```
图 ─主干卷积─▶ 特征图
                 ├─▶ RPN ─▶ ~2000 个 proposal(objectness 排序取 top-N + NMS)
                 │            │
                 └────────────┴─▶ RoI Pooling(从同一特征图取每个 proposal 的特征)
                                     │
                                     └─▶ FC ─▶ 分类头(K+1 类)+ 回归头(类相关偏移)
                                                  │
                                                  └─▶ 每类 NMS ─▶ 最终检测结果
```

注意 RPN 输出的 proposal 在送进 RoI 头前,**先按 objectness 排序取 top-N、再做一次 NMS** 去冗余(训练时 N≈2000,测试时常取更少如 300)。这是 NMS 在 Faster R-CNN 里出现的第一处(第二处在最终输出),下一节专讲 NMS。

**端到端 vs 四步交替训练**。RPN 和 RoI 头共享主干,但二者训练目标不同,怎么联合优化主干?原文给了 **4-step alternating training**:

```
step1: 用 ImageNet 预训练主干初始化,单独训 RPN
step2: 用 step1 的 RPN 产生的 proposal,单独训 Fast R-CNN 头(此时主干又被独立微调,两者还不共享)
step3: 用 step2 的主干,固定主干、只微调 RPN 特有层  <-- 从这步起主干被共享
step4: 固定主干,只微调 Fast R-CNN 特有层
```

后续工作证明可以**近似联合训练(approximate joint training)**一步到位(把两个损失加起来,忽略 proposal 坐标对 RoI Pooling 的梯度),精度相当且简单得多,现代实现(detectron2 等)默认走联合训练。4-step 是历史方案,理解其"为什么要交替"(让两个任务都能影响共享主干)即可,实操用联合训练。

## 6. NMS:非极大值抑制

### 6.1 问题:同一个物体被框很多次

无论 RPN 还是最终输出,同一个物体附近会有**大量高度重叠、都判为高分**的框(密集 anchor + 相邻位置预测相似)。检测的最终输出要求"一个物体一个框",必须去冗余。**NMS(Non-Maximum Suppression,非极大值抑制)** 是标准做法:**在一堆重叠框里,只保留得分最高的,抑制掉与它 IoU 过高的其余框**。

### 6.2 算法(贪心版,完整步骤)

```
输入:框集合 B = {b₁,...,bₙ},对应分数 S = {s₁,...,sₙ},IoU 阈值 τ(如 0.5)
输出:保留框集合 K

1. K ← ∅
2. while B 非空:
3.     m ← argmax(S)                  # 取当前最高分的框
4.     K ← K ∪ {b_m};  从 B、S 中移除 b_m
5.     for 剩下每个 b_i ∈ B:
6.         if IoU(b_m, b_i) > τ:       # 与刚保留的框重叠过高
7.             从 B、S 中移除 b_i       # 抑制掉(认为是同一物体的重复检测)
8. return K
```

直觉:**得分最高的框最可信,把它定为"代表";凡是和它重叠到一定程度(IoU>τ)的,都判定为"在框同一个物体",删掉**。剩下的框要么是别的物体、要么是重叠不那么高的,进入下一轮继续选代表。NMS 按类别独立做(不同类的框不互相抑制)。

### 6.3 代码:NMS 实现

```python
import torch

def nms(boxes, scores, iou_threshold):
    """
    boxes:  [N, 4] 角点式 (x1,y1,x2,y2)
    scores: [N]
    返回:保留框的索引(按分数降序),[K]
    生产用 torchvision.ops.nms(C++/CUDA 实现,快得多)。
    """
    if boxes.numel() == 0:
        return torch.empty((0,), dtype=torch.long, device=boxes.device)

    x1, y1, x2, y2 = boxes[:, 0], boxes[:, 1], boxes[:, 2], boxes[:, 3]
    areas = (x2 - x1).clamp(min=0) * (y2 - y1).clamp(min=0)
    order = scores.argsort(descending=True)        # 分数从高到低的索引

    keep = []
    while order.numel() > 0:
        i = order[0].item()                        # 当前最高分
        keep.append(i)
        if order.numel() == 1:
            break
        rest = order[1:]                           # 其余候选

        # 当前框 i 与其余所有框的交集(向量化)
        xx1 = torch.max(x1[i], x1[rest]); yy1 = torch.max(y1[i], y1[rest])
        xx2 = torch.min(x2[i], x2[rest]); yy2 = torch.min(y2[i], y2[rest])
        w = (xx2 - xx1).clamp(min=0);     h = (yy2 - yy1).clamp(min=0)
        inter = w * h
        iou = inter / (areas[i] + areas[rest] - inter).clamp(min=1e-6)

        # 只保留与 i 重叠不超过阈值的框,进入下一轮
        order = rest[iou <= iou_threshold]
    return torch.tensor(keep, dtype=torch.long, device=boxes.device)
```

实现要点:外层 while 每轮固定保留 1 个框、剔除与它高重叠的若干框,**循环次数 = 最终保留框数 K**;内层 IoU 计算向量化(对当前框 vs 剩余所有框)。

### 6.4 复杂度与坑

```
朴素 NMS 复杂度:O(N²) 最坏(每轮与 O(N) 个框算 IoU,共 O(N) 轮)
                 实际 ~ O(N·K),K 为保留数,通常远小于 N
排序成本:O(N log N)
```

**常见坑与变体**:

- **阈值 τ 的两难**。τ 太大(如 0.7):靠得近的两个**真**物体(两个并排的人)会被误抑成一个,**漏检**;τ 太小(如 0.3):同一物体的重复框删不干净,**冗余多**。没有万能值,需按数据调。
- **硬抑制丢真目标**。贪心 NMS 是"硬删":一旦 IoU>τ 直接归零。**Soft-NMS**(Bodla et al., 2017)改成按 IoU **衰减**重叠框的分数(`s_i ← s_i · f(IoU)`,如线性或高斯衰减)而非直接删,对密集场景的召回更友好。
- **跨类问题**。标准 NMS 按类独立做,故同一位置可同时输出"猫"和"狗"两个框;若要求每位置最多一个框,需用 class-agnostic NMS 或 batched 技巧。
- **可微性**。贪心 NMS 不可微,无法端到端训练——它是**推理时的后处理**。让检测"无 NMS"是后来 DETR(Carion et al., 2020)用集合预测 + 匈牙利匹配解决的方向(检测后续章节)。

## 7. 量化分析:参数量、FLOPs、速度对比

把三代方法放一起,看每个改进省了什么(数值为量级估计,精确值「待核」原论文,会随主干/分辨率变化):

```
                  候选框来源          CNN 前向次数/图      端到端     单图推理(量级)
R-CNN             选择性搜索(CPU)    ~2000(每框一次)    否(3阶段)  ~数十秒
Fast R-CNN        选择性搜索(CPU)    1(共享卷积)        近似(除提框)  ~2 秒(瓶颈在提框)
Faster R-CNN      RPN(GPU,共享)    1(共享卷积)        是          ~0.1-0.2 秒
```

**改进的因果链一目了然**:

```
R-CNN  →  Fast R-CNN:  把"2000 次 CNN"变"1 次 CNN + RoI Pooling"
                        瓶颈从【重复卷积】转移到【CPU 选择性搜索】
Fast   →  Faster:       把"CPU 选择性搜索 ~2s"变"GPU RPN ~10ms"
                        提框成本几乎归零,整条链全在 GPU、可端到端
```

**RPN 的额外参数量**很小:一个 3×3×C×512 卷积 + 两个 1×1 卷积(到 2k 和 4k 通道)。以 C=512、k=9 估:

```
3×3 卷积:   3·3·512·512 ≈ 2.36 M 参数
1×1 cls:    1·1·512·(2·9) = 512·18 ≈ 9.2 K
1×1 reg:    1·1·512·(4·9) = 512·36 ≈ 18.4 K
RPN 总计 ≈ 2.4 M 参数  <-- 相对主干(ResNet-50 ~25M)和 RoI 头(两个 4096 FC ~几十M)很轻
```

**RoI 头的 FC 才是参数大头**:Fast R-CNN 头的两个 4096 维全连接,仅第一个就 7·7·512·4096 ≈ 100 M 量级参数(随 RoI Pooling 输出尺寸和通道变化)。这也解释了后续 Light-Head R-CNN、用全卷积头替代大 FC 等优化方向——**两阶段的计算/参数大头在第二阶段的 per-RoI 计算上**,RoI 数越多越贵,这是它比单阶段慢的根本(单阶段没有 per-RoI 的重型头)。

## 8. 设计权衡与常见坑

- **两阶段 vs 单阶段的本质权衡**:两阶段先粗筛(RPN 把 2 万 anchor 砍到几百 proposal)再精判,**精度高但慢**(per-RoI 重型头);单阶段(YOLO/SSD/RetinaNet)在密集 anchor 上直接出类别+框,**快但正负更不均衡**(无粗筛环节),靠 Focal Loss 等补。理解两阶段的"粗筛"价值,才懂单阶段为何更难训。
- **anchor 设计是手工先验**,尺度/长宽比要匹配数据集物体分布。检测细长物体(如行人、文字行)却只配 1:1、2:1 的 anchor,召回会差。这也是 anchor-free 方法(FCOS、CenterNet)想去掉的"超参负担"(检测后续)。
- **正负样本分配是检测的隐形核心**。IoU 阈值(0.3/0.7)、采样比例(1:1)、采样数(256)这些看似工程的超参,直接决定模型学到什么。**不少检测涨点工作本质是在改样本分配策略**(ATSS、OTA 等)。
- **坐标系混乱**是头号 bug 来源:原图坐标 / 特征图坐标 / 归一化坐标 / 角点式 / 中心式来回转,一处 stride 或 0.5 偏移写错,框就系统性偏移。**写检测代码先把每个张量的坐标系标在注释里**。
- **RoI Pooling 的量化伤小目标**(§3.2),小物体检测优先 RoIAlign。
- **回归目标要做标准化**。实践中常对 (tx,ty,tw,th) 减均值除标准差(原文用固定的 (0,0,0,0)/(0.1,0.1,0.2,0.2) 量级常数),否则回归损失与分类损失量级失衡。这是「待核」具体数值、但务必知道存在的工程细节。
- **NMS 阈值与置信度阈值是两个独立旋钮**,别混。前者控重叠去重,后者控低分过滤;调 recall/precision 要分清在动哪个。
- **smooth L1 的 β**(拐点)在不同框架默认不同,改主干/分辨率时若回归不稳可调它。

### 练习 1:bbox 偏移参数化的独立推导、反解与「不取 log」的不对称性

**第一步:从三条要求出发,独立推出参数化。**

设参考框(anchor/proposal)中心宽高为 (xa, ya, wa, ha),真值框为 (x, y, w, h)。我们要为每个几何量构造一个回归目标 t,使其满足给定要求。

(1)中心坐标 (tx, ty) —— 要求「对框尺度归一化」。

原始位移是 `x − xa`(横向)和 `y − ya`(纵向)。问题在于:一个 100px 宽的框偏移 5px 是「几乎没动」,一个 10px 宽的框偏移 5px 却是「移了半个身位」。同样的绝对位移,对大框小框意义完全不同 —— 直接回归绝对位移,网络要为不同尺度各学一套,无法归一化。

把位移除以参考框自身的宽/高,得到「以框尺寸为单位」的相对位移:

```
tx = (x − xa) / wa
ty = (y − ya) / ha
```

这样无论框多大,「移动了自身宽度的多少倍」都落在同一量级(典型 ~O(1)),满足尺度归一化。

(2)宽高 (tw, th) —— 要求「预测恒正」+「乘性误差对称」。

宽高必须 > 0。若直接回归比值 `r = w/wa`,网络输出 r̂ 是无约束实数,可能为负,反解出负宽,非法。需要一个把无约束实数映成正数的可逆函数 —— 最自然的是 exp,其反函数是 log:

```
tw = log(w / wa)
th = log(h / ha)
```

反解 ŵ = wa·exp(t̂w),exp 恒正,**恒正要求自动满足**,无需任何额外约束。

再看乘性对称:框尺度是乘性量,「预测大了 2 倍」和「预测小了 2 倍」应受同等惩罚。log 把乘性变加性:

```
log(2·wa / wa) = +log2 ≈ +0.693
log(0.5·wa / wa) = −log2 ≈ −0.693
```

二者绝对值相等 —— 在以 |t̂w − tw| 为度量的损失下,过大 2 倍与过小 2 倍惩罚相同,**乘性误差对称要求满足**。(附带好处:log 把跨数量级的尺度比值压进小区间,数值稳定,即理由 C。)

**第二步:反解公式(推理时由预测偏移恢复真实框)。** 把上面四式反代:

```
x̂ = t̂x · wa + xa          ŷ = t̂y · ha + ya
ŵ = wa · exp(t̂w)          ĥ = ha · exp(t̂h)
```

**第三步:若把 tw 改成回归 w/wa(不取 log),破坏哪条要求?**

破坏两条:

- **恒正要求**:比值 w/wa 的回归输出无界,可能为负,反解出负宽,非法。
- **乘性对称要求**:见下方数值例子。

(尺度归一化那条与 tw 无关,不受影响;真正被打破的是「恒正」和「乘性对称」。)

**第四步:数值例子 —— 高估 2 倍 vs 低估 2 倍的 L1 惩罚不对称。**

设真值 w = wa(即真值比值 = 1,对应回归目标 v = w/wa = 1)。逐项算 L1 = |预测比值 − 真值比值|:

```
高估 2 倍:预测 ŵ = 2·wa,预测比值 = 2,L1 = |2 − 1| = 1.0
低估 2 倍:预测 ŵ = 0.5·wa,预测比值 = 0.5,L1 = |0.5 − 1| = 0.5
```

**高估 2 倍受到的惩罚(1.0)是低估 2 倍(0.5)的 2 倍 —— 不对称,且系统性偏向「不敢往大预测」**。

对照取 log 后(回归目标 v = log(w/wa) = 0):

```
高估 2 倍:t = log2 ≈ +0.693,L1 = |0.693 − 0| = 0.693
低估 2 倍:t = log(0.5) ≈ −0.693,L1 = |−0.693 − 0| = 0.693
```

**两者惩罚完全相等,对称性恢复。** 这就是宽高必须取 log 的核心理由。

### 练习 2:实现 NMS、手算验证、与 torchvision 对拍、Soft-NMS 救回并排目标

> 依赖:`torch`、`torchvision`(CPU 即可)。NMS 本身仅需 torch;对拍需 torchvision。本题维度极小,CPU 秒级跑完。

**(1)NMS 实现(照 §6.3)+ 人工样本 + 手算验证。**

```python
import torch

def nms(boxes, scores, iou_threshold):
    """boxes:[N,4] 角点式(x1,y1,x2,y2); scores:[N]; 返回保留索引(按分数降序)"""
    if boxes.numel() == 0:
        return torch.empty((0,), dtype=torch.long, device=boxes.device)
    x1, y1, x2, y2 = boxes[:, 0], boxes[:, 1], boxes[:, 2], boxes[:, 3]
    areas = (x2 - x1).clamp(min=0) * (y2 - y1).clamp(min=0)
    order = scores.argsort(descending=True)        # 分数从高到低
    keep = []
    while order.numel() > 0:
        i = order[0].item()                        # 当前最高分作为代表
        keep.append(i)
        if order.numel() == 1:
            break
        rest = order[1:]
        # 代表框 i 与其余所有框的交集(向量化)
        xx1 = torch.max(x1[i], x1[rest]); yy1 = torch.max(y1[i], y1[rest])
        xx2 = torch.min(x2[i], x2[rest]); yy2 = torch.min(y2[i], y2[rest])
        w = (xx2 - xx1).clamp(min=0);     h = (yy2 - yy1).clamp(min=0)
        inter = w * h
        iou = inter / (areas[i] + areas[rest] - inter).clamp(min=1e-6)
        order = rest[iou <= iou_threshold]         # 只保留重叠 <= 阈值的进下一轮
    return torch.tensor(keep, dtype=torch.long, device=boxes.device)


# ---- 人工样本:两簇各自高度重叠 + 一簇独立 ----
boxes = torch.tensor([
    [ 50.,  50., 150., 150.],   # 0 簇A,score 0.95(簇A最高 -> 应保留)
    [ 55.,  55., 158., 158.],   # 1 簇A,score 0.90,与0 IoU≈0.81 -> 被抑制
    [ 60.,  48., 155., 150.],   # 2 簇A,score 0.88,与0高度重叠 -> 被抑制
    [300., 300., 400., 400.],   # 3 独立簇,score 0.80(应保留)
    [305., 302., 402., 398.],   # 4 与3 IoU≈0.90 -> 被抑制
])
scores = torch.tensor([0.95, 0.90, 0.88, 0.80, 0.70])

# 手算预期(IoU 阈值 0.5):
#   选 0(最高)-> 抑制 1、2(都与0 IoU>0.5);剩 3、4
#   选 3(剩余最高)-> 抑制 4(IoU>0.5);剩空
#   => 保留 {0, 3}
print("nms keep:", nms(boxes, scores, 0.5).tolist())   # 期望 [0, 3]
```

**手算预期 = [0, 3]**,与实现输出一致(已用等价 numpy 镜像验证:`thr=0.5 -> [0,3]`)。

**(2)与 torchvision 对拍(比较索引集合)。**

```python
import torchvision

mine = set(nms(boxes, scores, 0.5).tolist())
ref  = set(torchvision.ops.nms(boxes, scores, 0.5).tolist())
assert mine == ref, f"mismatch: {mine} vs {ref}"
print("对拍通过:", mine, "==", ref)   # {0,3} == {0,3}
```

注意:对拍比较的是**保留索引的集合**(两实现选框逻辑相同,但当出现分数并列时内部顺序可能不同),所以用 `set(...)` 比较而非逐元素相等。题面也明确「比较的是索引集合」。

**(3)进阶:Soft-NMS(高斯衰减)在并排目标上救回硬 NMS 的漏检。**

Soft-NMS 不删框,而是按 IoU 衰减重叠框分数 `s_i ← s_i · exp(−IoU² / σ)`,最后用一个分数阈值截断。

```python
def soft_nms_gaussian(boxes, scores, sigma=0.5, score_thr=0.3):
    """高斯 Soft-NMS:衰减而非删除;返回最终保留索引"""
    boxes = boxes.clone()
    s = scores.clone().float()
    x1, y1, x2, y2 = boxes[:, 0], boxes[:, 1], boxes[:, 2], boxes[:, 3]
    areas = (x2 - x1).clamp(min=0) * (y2 - y1).clamp(min=0)
    alive = list(range(len(boxes)))
    kept_order = []
    while alive:
        m = max(alive, key=lambda j: s[j].item())  # 当前最高分
        alive.remove(m); kept_order.append(m)
        if not alive:
            break
        rest = torch.tensor(alive)
        xx1 = torch.max(x1[m], x1[rest]); yy1 = torch.max(y1[m], y1[rest])
        xx2 = torch.min(x2[m], x2[rest]); yy2 = torch.min(y2[m], y2[rest])
        w = (xx2 - xx1).clamp(min=0); h = (yy2 - yy1).clamp(min=0)
        inter = w * h
        iou = inter / (areas[m] + areas[rest] - inter).clamp(min=1e-6)
        s[rest] = s[rest] * torch.exp(-(iou ** 2) / sigma)   # 高斯衰减
    # 衰减后按分数阈值截断
    return [j for j in kept_order if s[j].item() >= score_thr]


# 构造两个【真】目标(两个紧挨的人),IoU≈0.33:
#   人A x∈[100,200], 人B x∈[150,250],纵向同高
crowd = torch.tensor([
    [100., 100., 200., 300.],   # 0 人A(真),0.90
    [150., 100., 250., 300.],   # 1 人B(真),0.82,与A IoU≈0.33
])
crowd_scores = torch.tensor([0.90, 0.82])

print("硬 NMS τ=0.3:", nms(crowd, crowd_scores, 0.3).tolist())          # [0] -> 人B 被误删(漏检)
print("Soft-NMS:    ", soft_nms_gaussian(crowd, crowd_scores, 0.5, 0.3)) # [0,1] -> 人B 救回
```

**对比结论**(已验算):两个真目标 IoU = 0.333。在 τ = 0.3 的硬 NMS 下,人B 被当作人A 的重复检测**直接删掉(漏检 → 保留 [0])**;Soft-NMS 把人B 分数从 0.82 高斯衰减到约 **0.657**(仍 ≥ 0.3 阈值),**人B 被救回(保留 [0, 1])**。这正是 Soft-NMS 在拥挤/密集场景比硬 NMS 召回更友好的根因 —— 把「一刀切删除」换成「软性降权」,避免误杀靠得近的真目标。

### 练习 3:RoI Pooling 量化误差(S=16,框 (50,50,66,66),输出 2×2)

物体真值框 (x1,y1,x2,y2) = (50, 50, 66, 66),原图上宽高 = 16×16(小物体);stride S = 16,即 `spatial_scale = 1/16`;目标 RoI 输出 2×2。

**(a)量化①:RoI 边界对齐到特征图整数坐标,再反映射回原图,误差几像素?**

先把原图坐标除以 S 映到特征图坐标:

```
x1: 50 / 16 = 3.125      x2: 66 / 16 = 4.125
y1: 50 / 16 = 3.125      y2: 66 / 16 = 4.125
```

RoI Pooling 把它 round 到整数特征图坐标(对齐到网格):

```
3.125 → round → 3        4.125 → round → 4
```

反映射回原图(乘回 S = 16):

```
x1': 3 × 16 = 48         x2': 4 × 16 = 64
y1': 3 × 16 = 48         y2': 4 × 16 = 64
```

对齐后的框 = **(48, 48, 64, 64)**,而原框是 (50, 50, 66, 66)。

```
左/上边界误差:48 − 50 = −2 像素
右/下边界误差:64 − 66 = −2 像素
对齐后框宽高 = 16×16(与原框宽高碰巧相同,但整体向左上平移了 2px)
```

**本例单边边界误差 = 2 像素(整框系统性偏移 2px)。**

> 说明:正文 §3.2 给的 ±S/2 ≈ ±8px 是量化①的**最坏情况上界**(当原图坐标恰好落在网格中点 ±8 处时,round 把它推到最远的整数倍)。本例 3.125/4.125 离整数都很近,round 后单边只偏 2px —— 是该上界范围内的一个具体取值,二者不矛盾:**最坏 ±8px,本例 2px**。

**(b)每个 bin 在原图上覆盖多大区域?**

RoI 在特征图上的宽/高 = 4 − 3 = 1 个特征单元。输出 2×2,故每个 bin 在特征图上 = 1 / 2 = **0.5 个特征单元**。换算回原图:

```
每个 bin 在原图上 ≈ 0.5 × S = 0.5 × 16 = 8 像素(×8 像素)
```

即量化② 在把 0.5 这种非整数 bin 边界 floor/ceil 到整数行列时,会引入约「半个特征单元 = 8px」量级的错位。

**(c)为什么对 16px 小物体量化误差严重?RoIAlign 如何消除①②两处取整?**

论证:

- 物体本身只有 16×16 px。量化① 带来的 **2px 单边偏移占物体尺寸的 1/8**;而最坏情况(±8px)下偏移可达物体半个身位 —— 对小物体,几像素的绝对误差就是**相对上的巨大定位漂移**,而同样的几像素误差对一个 200px 大物体几乎可忽略。量化误差是**绝对量**,小物体的**相对**承受力最差,这就是 Fast/Faster R-CNN 小目标精度受限的直接来源。
- 更糟的是 RoI 在特征图上只占 1×1 单元,却要切成 2×2 个 bin。bin 边界 0.5 经 floor/ceil 取整后,多个 bin 会塌缩/重叠到同一个或相邻的整数特征格 —— 2×2 输出里**几乎没有独立的特征信息**,空间分辨率被取整彻底抹平。

RoIAlign 的消除机制:

- **不做任何取整**。量化①:RoI 边界用浮点特征图坐标(3.125、4.125)直接保留,不 round。量化②:bin 边界也用浮点(每 bin = 0.5 特征单元)直接保留。
- 在每个 bin 内**按固定规则取若干采样点**(典型每 bin 4 个),每个采样点落在连续坐标上(非整数格点);用**双线性插值**从其周围 4 个整数特征点算出该采样点的值,再对 bin 内采样点做(平均/最大)池化。
- 因为采样点坐标和插值全程是连续的、可微的,**①② 两处「连续坐标 → 整数网格」的取整被彻底取消**,RoI 与 bin 的边界精确对应到原图位置。这把小目标的定位漂移从「±几像素绝对误差」降到亚像素级,也使梯度能精确回传到特征坐标 —— 这正是下一章 RoIAlign 撑起 Mask R-CNN 实例分割(掩码对像素级对齐极敏感)的原因。

### 练习 4:anchor 数量与召回(800×800,stride=16,k=9)

**第一步:全图 anchor 总数。**

特征图尺寸 = (H/S) × (W/S) = 800/16 × 800/16 = 50 × 50。每位置 k = 9 个 anchor:

```
anchor 总数 = (800/16) × (800/16) × 9 = 50 × 50 × 9 = 22500 个
```

**第二步:为何 4:1 细长物体在该 anchor 配置下召回低(定性 + 定量佐证)。**

anchor 长宽比只有 {1:2, 1:1, 2:1}(即 r = w/h ∈ {0.5, 1, 2}),最「细长」的也只到 2:1。一个长宽比 4:1 的真值物体,与这三种 anchor 的形状差距都很大。

定量佐证(同面积、轴对齐居中,取最理想情形下的 IoU 上界):

```
4:1 物体 vs anchor 2:1 :IoU ≈ 0.55   ← 最接近的一种,也只到 0.55
4:1 物体 vs anchor 1:1 :IoU ≈ 0.33
4:1 物体 vs anchor 1:2 :IoU ≈ 0.22
```

(实际中 anchor 与物体还有位置/尺度错配,IoU 只会更低,上面是「形状差异」单因素下的乐观上界。)

由 §5.3 的正负分配规则:

- 正样本主条件是「与真值框 IoU > 0.7」。该物体与**所有** anchor 的最大 IoU 也只有约 0.55 < 0.7 —— **过不了 0.7 的主阈值,几乎拿不到「IoU>0.7」的正样本**。
- 只能靠兜底规则 (a)「与该真值框 IoU 最高的那个 anchor」强行分到**一个**正样本。于是这类物体在训练中正样本极度稀少(每个真值框仅 1 个,且那个 anchor 形状还很不贴),回归头要从「2:1 拉成 4:1」这种大形变去学,目标偏移大、smooth L1 也压不出好结果。
- 训练信号既少又差 → 模型对 4:1 物体学不到可靠的 objectness 与回归 → 推理时这类框 objectness 打分低、定位偏 → **召回低**。

**第三步:两种改法。**

改法 A —— **改 anchor 配置(在 anchor-based 框架内补先验)**:

- 把数据集物体的长宽比分布统计出来,给 anchor 增加贴近的长宽比,如把 ratio 扩成 {1:4, 1:2, 1:1, 2:1, 4:1}(甚至按 k-means 在训练集 GT 框上聚类自动选 anchor 形状,如 YOLOv2 的 dimension cluster)。
- 这样 4:1 物体能找到 IoU 与之 > 0.7 的 anchor,顺利进入正样本集,回归目标也变小。代价是 anchor 数变多(k 增大 → 总数与计算线性上升)、调参负担加重,且仍是「手工先验」,换个数据集还要重调。

改法 B —— **换 anchor-free 方法(根除「靠预设长宽比匹配」这一假设)**:

- 用 FCOS / CenterNet 这类无 anchor 检测器:不预设任何固定长宽比的框,而是在特征图每个落入物体内部的位置上,**直接回归该点到物体四条边的距离**(FCOS 的 l, t, r, b)或先定位中心点再回归宽高(CenterNet)。
- 物体多细长,网络就回归出多细长的 (l, t, r, b),**完全不受预设 ratio 集合的约束**,4:1、10:1 同等对待,从根上消除「anchor 长宽比覆盖不全 → 召回低」的问题,也卸掉了 anchor 尺度/比例/数量这一堆超参负担。代价是正负样本分配规则要重新设计(用「点是否在 GT 框内 / 是否靠近中心」替代 IoU 阈值),且早期 anchor-free 在重叠物体的中心点冲突上需额外处理(如多级 FPN 分摊)。
