> 这一章把你从"会用 policy gradient / Q-learning 这类 model-free 算法在固定环境里学策略",带到"理解为什么'学一个模型 + 在模型里搜索'是通往超人表现的另一条主路线,能从第一性原理推导 MCTS 的 UCT 选择规则,能讲清 AlphaGo→AlphaZero→MuZero 这条飞轮为什么能在零人类知识下 bootstrap 到超人,并能照着伪代码实现一个 MCTS 主循环"。读完你应当能向别人证明:**自博弈 + 搜索之所以能持续变强,不是工程玄学,而是一次"策略改进算子"在自身上的不动点迭代**。

## 为什么需要这一章:model-free 的样本墙

前几章的算法——REINFORCE、actor-critic、PPO、DQN——有一个共同的隐含假设:**环境是一个黑箱,你只能"试一下、看反馈"**。你给环境一个动作 a,它吐回下一个状态 s' 和奖励 r,但你不知道这个 (s,a) → (s', r) 的映射本身长什么样。你能做的只有反复试错,从经验里直接估计价值或策略。这类方法叫 **model-free**(无模型)。

它有一个无法回避的代价:**样本效率低**。每一条经验只用一次(on-policy 的 PPO)或被反复回放有限次(off-policy 的 DQN),而且每条经验只在它实际访问到的那个 (s,a) 上提供信息。要覆盖一个大状态空间,你得真的去访问无数次。在 Atari 上 DQN 需要几千万帧,在很多真实任务里(机器人、推荐、对弈)这个代价根本付不起——真实交互慢、贵、甚至危险。

**model-based RL(有模型强化学习)的核心赌注是:如果我能学到环境的动力学 `p(s', r | s, a)`,我就可以在脑子里"想象"地走很多步,而不必每一步都真的去环境里试。** 这带来两个本质能力,是 model-free 永远拿不到的:

1. **样本高效(sample efficiency)**:一条真实经验可以用来训练模型,而训练好的模型能生成任意多条"想象经验"去训练策略。真实交互的次数被摊薄。
2. **规划(planning)**:有了模型,你可以在做决策前**向前搜索**——模拟"如果我走这步,对手/环境会怎么回应,再往后会怎样"——用前瞻把当前策略放大成一个更强的决策。model-free 的 agent 只能"条件反射",model-based 的 agent 能"深思熟虑"。

这条路线的巅峰就是 DeepMind 的 AlphaGo / AlphaZero / MuZero:它们的超人表现**不是靠更大的网络硬背出来的,而是靠"搜索"和"学习"互相放大**。这一章要讲透的,就是这个放大机制为什么在数学上成立。

我们的推进路线:

1. 先把 model-based 与 model-free 的取舍量化,讲清"模型从哪来、模型误差怎么伤人";
2. Dyna 架构——把真实经验和想象经验统一进一个学习循环的最简范式;
3. 蒙特卡洛树搜索(MCTS)四阶段的完整机制 + UCT 规则从 UCB 的推导;
4. AlphaGo→AlphaZero→MuZero 的演进,每一步砍掉了什么先验、为什么还能更强;
5. 推导"自博弈飞轮"的不动点视角:MCTS 是一个策略改进算子,网络是它的摊销器(amortizer)。

### 预备:把环境形式化为 MDP,并区分"模型"指的是什么

后面所有讨论都建立在马尔可夫决策过程(MDP)上。一个 MDP 由五元组 `(S, A, P, R, γ)` 给出:状态集 S、动作集 A、转移核 `P(s'|s,a)`、奖励 `R(s,a)` 或 `R(s,a,s')`、折扣 γ ∈ [0,1)。

**"模型"在这里特指 P 和 R**,即环境的动力学。注意区分两种东西,新手极易混:

- **价值函数 / 策略**:`V(s)`、`Q(s,a)`、`π(a|s)`,是"在这个环境里怎么做好"。
- **环境模型**:`P(s'|s,a)`、`R(s,a)`,是"这个环境本身怎么运转"。

model-free 只学前者;model-based 额外学(或已知)后者。AlphaZero 属于"模型已知"(围棋规则给定,转移完全确定);MuZero 属于"模型未知,需要学",而且它只学一个**够用来规划的隐式模型**,这是后话。

还要建立一个贯穿全章的直觉——**Bellman 期望与最优方程**,它是"规划"二字的数学根:

```
V^π(s)   = E_{a~π}[ R(s,a) + γ·E_{s'~P}[ V^π(s') ] ]      (策略评估)
V*(s)    = max_a [ R(s,a) + γ·E_{s'~P}[ V*(s') ] ]         (最优方程)
Q*(s,a)  = R(s,a) + γ·E_{s'~P}[ max_{a'} Q*(s',a') ]
```

**有了模型 P、R,你就能在不碰真实环境的情况下,反复用这个递推式做 backup(回传)——这就是 planning 的本质:在模型里做 Bellman 迭代。** model-free 之所以慢,是因为它必须用采样的 (s,a,r,s') 去近似上面那个对 P 的期望;model-based 直接掌握 P,省掉了采样这一步的方差。

## 第一节:model-based 的好处与它的阿喀琉斯之踵

### 好处一:样本复杂度的量级差异(直觉化推导)

为什么"学模型"能省样本?给一个干净的直觉论证。

考虑表格型(tabular)情形,|S| 个状态、|A| 个动作。要把策略学好,你至少得知道每个 (s,a) 的转移和奖励。

- **model-free(如 Q-learning)**:你要直接估 `Q(s,a)`,它有 |S|·|A| 个数。但每个 Q 值是一个对未来的长期期望,估准它需要在该 (s,a) 上采样很多次,且误差会沿轨迹传播。
- **model-based**:你要估的是 `P(s'|s,a)`(|S|·|A|·|S| 个数)和 `R(s,a)`。表面上参数更多,但**每访问一次 (s,a) 都直接给 P 和 R 提供一个无偏样本**,而且一旦 P、R 估准,所有的 Q、V 都能通过 planning(在模型里跑 Bellman 迭代)精确算出来,不需要再为每个长期价值单独采样。

关键差别在于**信息的复用**:model-free 里"这个动作长期好不好"这件事,必须靠在真实环境里反复走完整轨迹来回答;model-based 把它拆成"一步动力学(局部、好估)"+"在模型里组合出长期后果(免费的计算)"。一步动力学是局部的、密集可观测的;长期后果是计算,不是采样。**这就是 model-based 样本高效的根:把'昂贵的真实采样'换成了'便宜的模型内计算'。**

理论上,在表格型 MDP 里,达到 ε-最优所需的样本数,model-based 方法(如 R-max、MBIE)的样本复杂度上界与 model-free 的最优界在主导项上是同阶的,但 model-based 在常数和实践收敛速度上通常显著更优;具体的 minimax 样本复杂度界(关于 |S|、|A|、1/(1-γ)、1/ε 的幂次)请参原始论文【待核:此处精确幂次依赖于设定与算法,不凭记忆给死】。我们这一章不纠缠这个界的精确形式,只取它的定性结论:**模型把"长期信用分配"从采样问题变成了计算问题**。

### 阿喀琉斯之踵:模型误差与复合误差(compounding error)

model-based 不是免费午餐。它有一个致命弱点,新手必须刻进脑子:**模型是学出来的,带误差;而规划要在模型里走很多步,误差会逐步累积、放大**。

把它量化。设真实一步转移算子作用在价值上的误差,在学到的模型 `P̂` 下每步引入的偏差上界为 ε_model(某种范数下)。如果你在模型里向前 rollout(展开)H 步去估计回报,误差不是 H·ε_model 这么温柔——因为后面每一步的输入状态本身就是前面错误状态的函数,误差会**沿轨迹复合**。一个常被引用的定性结论:

```
H 步想象 rollout 的价值估计误差 ~ O( ε_model · (1 + γ + γ² + ... ) )
                                  在 H 较大时受 1/(1-γ) 量级支配,
                                  且状态分布偏移会使有效误差进一步放大。
```

直觉版:**你在一个略微失真的世界模型里规划得越深,你越可能在规划一个根本不存在的世界。** 模型在真实数据覆盖到的状态上还算准,但 rollout 一旦走到训练数据没覆盖的状态(分布外,out-of-distribution),模型会胡说,而策略会**专门利用模型的这些错误区域**——它发现"在模型里往这个方向走能拿高分",于是兴高采烈地优化,结果一到真实环境就崩。这是 model-based 版本的 **reward hacking / model exploitation**,是这条路线上最经典的坑。

应对这个坑的几条主线(后面会落到具体算法上):

- **短 rollout**:别在模型里走太深(如 Dyna 里常用 1 步;MBPO 用短分支 rollout),把复合误差按住。
- **集成模型 + 不确定性**:用模型集成(ensemble)估计认知不确定性,在不确定的地方惩罚或不信任(PETS、MBPO 的思路)。
- **不学完整模型,只学"够搜索用"的模型**:MuZero 的关键一招——它根本不要求隐式模型能重建真实观测,只要求它在"预测价值/策略/奖励"这件事上有用。这绕开了"精确重建一个复杂世界"这个几乎不可能的任务。

记住这条主线张力:**model-based 用模型误差换样本效率**。整条 MuZero 路线的精妙之处,很大程度上就是在管理这个张力。

## 第二节:Dyna —— 真实经验与想象经验的统一循环

Dyna(Sutton, 1990)是 model-based RL 最简洁、最该先理解的范式。它的洞察:**为什么要把"从真实经验学"和"从模型学"分成两件事?把它们塞进同一个学习循环,用同一套更新规则。**

Dyna-Q 的结构(以 tabular Q-learning 为底座):

```
初始化 Q(s,a) 任意, 模型 Model(s,a) 为空
循环(每一步真实交互):
  (a) 在真实环境:由当前 Q 选动作(如 ε-greedy),执行,得到 (s, a, r, s')
  (b) 直接 RL 更新(用真实经验):
        Q(s,a) ← Q(s,a) + α [ r + γ·max_{a'} Q(s',a') − Q(s,a) ]
  (c) 学模型(记住这次转移):
        Model(s,a) ← (r, s')          # tabular 下就是确定性地记住
  (d) 规划(用想象经验做 n 次 planning backup):
        重复 n 次:
          随机取一个曾访问过的 (s̃, ã)
          (r̃, s̃') ← Model(s̃, ã)       # 从模型"回忆/想象"一条经验
          Q(s̃,ã) ← Q(s̃,ã) + α [ r̃ + γ·max_{a'} Q(s̃',a') − Q(s̃,ã) ]
```

**关键观察:步骤 (b) 和 (d) 用的是同一个 Q-learning 更新式**,区别仅在于经验的来源——(b) 来自真实环境,(d) 来自模型。这就是 Dyna 的全部精髓:模型的作用是当一个"经验生成器",把稀缺的真实交互**摊销**成大量的内部更新。

为什么这能加速?在 tabular 确定性环境里,一次真实经验只更新一个 Q 值;但通过模型反复回放历史转移,价值信息能在状态空间里**更快地反向传播**。经典的"Dyna 迷宫"实验显示:同样的真实步数下,n 越大(每步规划次数越多),找到最优路径所需的真实交互回合数越少——因为奖励信息通过模型 backup 一晚上就传遍了整张图,而纯 Q-learning 要靠真的走很多趟才能把信息一格一格往回带。

**Dyna 的现代回声**:你可以把经验回放(experience replay)看成 Dyna 的一个特例——回放缓冲区是一个"非参数化的、只会重放真实历史的模型"。而真正的 model-based 方法(如 MBPO,Model-Based Policy Optimization)把 Model 换成一个神经网络动力学模型,用它生成**短分支想象 rollout**,喂给一个 model-free 的策略优化器(如 SAC)。MBPO 的设计正是直面第一节那个复合误差:**只从真实状态出发走很短的几步想象,绝不长程 rollout**,以此把模型误差按在可控范围内,同时享受样本效率。

**坑提醒**:Dyna 在确定性 tabular 下很干净;一旦环境随机、状态连续、模型用神经网络近似,(c) 步的"记住转移"就变成"拟合一个带误差的分布",(d) 步的想象就可能放大误差。**n(规划步数)不是越大越好**——模型越差,想象越多反而把策略往错误方向推得越狠。这就是为什么纯 Dyna 在复杂环境里要配合短 rollout 和不确定性估计。

## 第三节:蒙特卡洛树搜索(MCTS)—— 在模型里做局部最优规划

前面的 planning(Dyna 的 backup、value iteration)是"全局、均匀"地更新价值。但当状态空间巨大(围棋约 10¹⁷⁰ 个局面)时,你不可能更新所有状态。你需要的是**只在当前真正要决策的那个状态附近,集中算力做深度前瞻**。这就是 MCTS 解决的问题。

**MCTS 的定位:它是一个"在线规划"算法。给定当前状态 s₀ 和一个模型(能模拟 (s,a)→s' 和判断终局/给奖励),它通过反复模拟,在 s₀ 处构建一棵搜索树,最终输出"在 s₀ 该走哪个动作"。** 它不需要遍历整个状态空间,只把算力花在"看起来更有希望"的分支上——这是它能对付天文数字状态空间的根本原因。

### 四个阶段

每一次 MCTS 模拟(simulation,也叫一次"playout")由四个阶段组成。树中每个节点是一个状态,每条边 (s,a) 维护两个统计量:**访问次数 `N(s,a)`** 和 **价值估计 `Q(s,a)`**(该边下所有模拟回报的平均)。

```
一次模拟:
1. 选择(Selection):从根 s₀ 出发,在已经在树中的节点上,
   反复用"树策略"(下面的 UCT)选动作往下走,直到到达一个"叶节点"
   (它的某个子节点还没被扩展过)。
2. 扩展(Expansion):为这个叶节点新增一个(或若干)子节点,
   把它加入树,初始化其 N=0, Q=0。
3. 模拟/评估(Simulation / Rollout / Evaluation):
   从新节点估计它的价值。经典 MCTS 用"随机 rollout 到终局"得到一个回报;
   AlphaZero 用价值网络 V(s) 直接给出估值(不再 rollout 到底)。
4. 回传(Backpropagation / Backup):把这次模拟得到的回报 G,
   沿着刚才走过的路径,逐边更新统计:
       N(s,a) ← N(s,a) + 1
       Q(s,a) ← Q(s,a) + (G − Q(s,a)) / N(s,a)   # 增量平均
   (零和对弈里 G 的符号要按当前行动方逐层取反,见下面代码注释)
```

跑足够多次模拟后,**根节点各动作的访问次数分布 `N(s₀, a)` 就是 MCTS 的输出**——访问最多的动作通常就是最优选择。注意一个微妙但重要的点:**最终选动作通常看访问次数 N 而不是价值 Q**,因为 N 综合了"价值高"和"被反复验证过",对噪声更鲁棒;高 Q 但只访问一两次的动作可能只是运气好。

### UCT:把选择阶段变成一个多臂老虎机问题

四阶段里,**第 1 步"选择"是 MCTS 的灵魂**,因为它决定了算力往哪儿花。这里有一个根本的**探索-利用(exploration-exploitation)**两难:

- **利用**:多走当前看起来 Q 高的动作(它可能真是好棋)。
- **探索**:也得给访问次数少的动作机会(它可能其实更好,只是还没被试够,Q 估计不可信)。

UCT(Upper Confidence bounds applied to Trees,Kocsis & Szepesvári, 2006)的洞察:**把"在一个节点选哪个动作"看成一个多臂老虎机(multi-armed bandit)问题,直接套用 bandit 的最优探索策略 UCB1。** 这是 MCTS 真正的理论支柱,值得完整推导。

#### 从 Hoeffding 不等式推导 UCB1

先看 bandit 问题本身。K 个臂,拉第 i 个臂得到一个 [0,1] 范围内的随机奖励,真实均值 μᵢ 未知。你拉了总共 n 次,其中第 i 个臂拉了 Nᵢ 次,样本均值是 Q̂ᵢ。你想最小化"后悔"(regret)——即没有每次都拉最优臂带来的累计损失。

核心工具是 **Hoeffding 不等式**:对 [0,1] 区间内独立同分布、均值 μ 的随机变量,样本均值 Q̂(基于 m 个样本)满足

```
P( Q̂ − μ ≥ u ) ≤ exp( −2 m u² )      （单边）
P( μ − Q̂ ≥ u ) ≤ exp( −2 m u² )
```

直觉:样本均值偏离真值超过 u 的概率,随样本数 m 指数下降。

现在我们想构造一个**置信上界(upper confidence bound)**:一个量 `Q̂ᵢ + uᵢ`,使得真实均值 μᵢ 以高概率不超过它。给定置信水平,我们要 u 满足 `exp(−2 Nᵢ u²) = δ`(δ 是允许的失败概率)。解出:

```
exp(−2 Nᵢ uᵢ²) = δ
⟹  2 Nᵢ uᵢ² = ln(1/δ)
⟹  uᵢ = √( ln(1/δ) / (2 Nᵢ) )
```

UCB1 的精妙在于**让 δ 随时间收紧**:取 δ = n^{−4}(随总拉取次数 n 越来越小,要求置信越来越高),代入:

```
uᵢ = √( ln(n⁴) / (2 Nᵢ) ) = √( 4 ln n / (2 Nᵢ) ) = √( 2 ln n / Nᵢ )
```

于是得到 **UCB1 选择规则**:每一步选

```
a = argmaxᵢ [ Q̂ᵢ + √( 2 ln n / Nᵢ ) ]
                └─利用─┘  └────探索奖励────┘
```

**为什么这个形式是对的(两个极限验证)**:

- 当某个臂 Nᵢ 很小(很少被试),探索项 √(2 ln n / Nᵢ) 很大 → 它的 UCB 被抬高 → 倾向于去试它(探索)。
- 当 Nᵢ 很大(试了很多次),探索项 → 0,只剩 Q̂ᵢ → 倾向于按真实价值选(利用)。
- 分子 ln n 随总次数缓慢增长,保证**没有任何臂会被永远饿死**:即使一个臂暂时看着差,只要别人一直被拉、n 在涨,迟早它的探索项会重新把它抬上来再确认一次。

UCB1 的著名结论:它的累计后悔是 **O(ln n)**(对数后悔),并且这是不可能再改进的渐近最优阶——任何"合理"的 bandit 算法后悔下界也是 Ω(ln n)。**这就是为什么 MCTS 敢用它:它在数学上保证了"既不会因贪心而错过好动作,也不会因瞎探索而浪费太多算力"。**

#### O(ln n) 后悔的完整证明骨架(讲透必须自己能复现)

只把结论抛出来不算讲透。下面给出 Auer et al.(2002)定理 1 的证明骨架,理解它你就理解了"为什么 √(2 ln n/N) 这个探索项的尺度刚刚好"。回忆后悔的分解(见下一节也会用):次优臂 i 的差距 `Δᵢ = μ* − μᵢ > 0`,则

```
R_n = Σ_{i: Δᵢ>0} Δᵢ · E[Nᵢ(n)]      （每拉一次次优臂 i 损失 Δᵢ;最优臂不产生后悔)
```

所以**只需界住每个次优臂的期望拉取次数 E[Nᵢ(n)]**。核心引理:

**只要次优臂 i 已被拉了 ℓ := ⌈8 ln n / Δᵢ²⌉ 次,它的探索项就收缩到 ≤ Δᵢ/2。** 验算:

```
√(2 ln n / Nᵢ) ≤ √(2 ln n / ℓ) ≤ √( 2 ln n / (8 ln n / Δᵢ²) ) = √(Δᵢ²/4) = Δᵢ/2
```

此时,UCB 还会错选次优臂 i(它的乐观上界压过最优臂 *)只可能因为下面三种小概率事件之一发生(否则不可能,可由不等式联立推出):

```
(E1) Q̂* ≤ μ* − √(2 ln t / N*)     最优臂被严重低估(样本均值跌破其置信下界)
(E2) Q̂ᵢ ≥ μᵢ + √(2 ln t / Nᵢ)     次优臂被严重高估(样本均值冲破其置信上界)
(E3) μ* < μᵢ + 2·√(2 ln t / Nᵢ)    探索项还没收缩到位(即 Nᵢ < ℓ)
```

逻辑链:若 E1、E2 都不发生,则 `Q̂* > μ* − 探索*` 且 `Q̂ᵢ < μᵢ + 探索ᵢ`;代入"i 被选中"的条件 `Q̂ᵢ + 探索ᵢ ≥ Q̂* + 探索*`,消去得 `μᵢ + 2·探索ᵢ > μ*`,即 E3。而当 Nᵢ ≥ ℓ 时,由上面引理 `2·探索ᵢ ≤ Δᵢ = μ* − μᵢ`,E3 不可能。**所以一旦 Nᵢ 超过 ℓ,错选只能由 E1 或 E2 引起。**

用 Hoeffding 把 E1、E2 的概率界住:对固定的 N*、Nᵢ,各自 ≤ exp(−2·(2 ln t)) = t⁻⁴。对所有 (t, N*, Nᵢ) 三重求和——t 从 1 到 n,N* 与 Nᵢ 各从 1 到 t,所以是 `Σ_{t=1}^{n} t·t·t⁻⁴ = Σ t⁻²`,这是收敛级数,上界 ≤ Σ_{t=1}^∞ 2t⁻² = π²/3。把"必拉的 ℓ 次"与"运气事件的常数项"加起来:

```
E[Nᵢ(n)] ≤ ⌈8 ln n / Δᵢ²⌉ + 1 + π²/3
            └─ 探索项收缩前不得不付的次数 ─┘  └─ 小概率运气项之和(常数)─┘
```

代回后悔分解:

```
R_n = Σ_{i:Δᵢ>0} Δᵢ·E[Nᵢ(n)]
    ≤ Σ_{i:Δᵢ>0} ( 8 ln n / Δᵢ + Δᵢ·(1 + π²/3) )
    = O( ln n · Σ_i 1/Δᵢ )                      ▮ 对数后悔成立
```

**这条证明把"探索项尺度"和"目标"精确绑定了**:正因为想让次优臂只被拉 O(ln n) 次,探索半径就必须取 √(2 ln n/N) 这个量级——大了浪费(过度探索),小了置信带不够、错选概率求和不收敛(欠探索导致线性后悔)。UCT 把这套尺度原封不动搬到树上,正是它合理性的根。

#### UCT = 在树的每个节点跑 UCB1

把 UCB1 搬到 MCTS 的选择阶段,把"总拉取次数 n"换成"父节点访问次数 N(s)",把"臂 i 的次数 Nᵢ"换成"边 (s,a) 的访问次数 N(s,a)",把样本均值换成 `Q(s,a)`,就得到 **UCT 选择规则**:

```
a* = argmax_a [ Q(s,a) + c · √( ln N(s) / N(s,a) ) ]
```

其中 `N(s) = Σ_a N(s,a)` 是该节点总访问数,**c 是探索常数**(把理论里的 √2 换成一个可调超参,因为实际回报范围、价值尺度都和理想 [0,1] bandit 不同,c 要按经验调)。

**这里有一个深刻的细节,讲透 MCTS 必须点破**:树里的每个节点其实**不是一个标准 bandit**——它的"臂的奖励分布"不是固定的,而是随着子树被搜索得越来越深、估计越来越准而**在漂移**(non-stationary)。所以 UCT 把 UCB1 直接套上去,严格来说是一个近似。Kocsis & Szepesvári 的贡献正是证明了:即便如此,**只要每个节点都用 UCB 类策略,MCTS 的根节点动作选择会收敛到 minimax 最优**(在合适条件下,选到次优动作的概率随模拟次数多项式衰减)。也就是说:**算力给够,MCTS 会逼近"完美前瞻"的结果**。这条收敛性是整套方法在博弈上敢用的底气。

### 代码:MCTS + UCT 主循环(伪代码)

下面是体现机制的核心实现,带零和博弈的符号处理。重点看 `select` 里的 UCT、`backup` 里的增量平均与符号取反。

```python
import math

class Node:
    def __init__(self, state, parent=None, prior=0.0):
        self.state = state
        self.parent = parent
        self.children = {}      # action -> Node
        self.N = 0              # 访问次数
        self.W = 0.0            # 累计价值(回报之和)
        self.prior = prior      # P(a|s),来自策略网络;经典 MCTS 可设为均匀
    @property
    def Q(self):                # 平均价值
        return self.W / self.N if self.N > 0 else 0.0

def uct_score(child, parent_N, c):
    # Q + c * sqrt(ln N_parent / N_child);N_child=0 时探索项视为 +inf 必被选
    if child.N == 0:
        return float('inf')
    exploit = -child.Q          # negamax:子节点 Q 是对手视角,父节点要最小化它
    explore = c * math.sqrt(math.log(parent_N) / child.N)
    return exploit + explore

def mcts(root_state, model, num_simulations, c=1.4):
    root = Node(root_state)
    for _ in range(num_simulations):
        node = root
        path = [node]
        # 1. 选择:沿 UCT 最大的子节点下行,直到遇到尚未展开的节点(无子节点,本实现一次展开全部合法动作)或终局
        while node.children and not model.is_terminal(node.state):
            action = max(node.children,
                         key=lambda a: uct_score(node.children[a], node.N, c))
            node = node.children[action]
            path.append(node)
        # 2. 扩展:若非终局,展开所有合法动作为子节点(prior 可由策略网络给)
        if not model.is_terminal(node.state):
            for a in model.legal_actions(node.state):
                s_next = model.transition(node.state, a)     # 用模型走一步
                node.children[a] = Node(s_next, parent=node, prior=1.0)
            # 选一个新子节点作为本次评估的叶子
            node = next(iter(node.children.values()))
            path.append(node)
        # 3. 评估:经典 MCTS 随机 rollout 到终局;AlphaZero 用价值网络替代这一步
        value = rollout(node.state, model)   # 返回相对"叶节点行动方"的回报
        # 4. 回传:沿路径更新;零和博弈中每上一层,价值对当前方取反
        for n in reversed(path):
            n.N += 1
            n.W += value
            value = -value          # 关键:对手视角下,我的好就是你的坏
    # 输出:按访问次数选动作(比按 Q 更鲁棒)
    return max(root.children, key=lambda a: root.children[a].N)

def rollout(state, model):
    # 经典 MCTS:用随机策略快速模拟到终局,返回 ±1/0 这类终局回报
    cur = state
    while not model.is_terminal(cur):
        a = model.random_action(cur)
        cur = model.transition(cur, a)
    return model.terminal_value(cur)   # 相对当前行动方
```

**复杂度与方差分析**:

- 每次模拟的代价 ≈ 树深度 D(选择下行)+ rollout 长度 L(评估)。总代价 = num_simulations × (D + L)。这正是 MCTS 的可控之处:**用模拟次数直接换决策质量,是一个 anytime 算法**(随时可停,停得越晚越准)。
- **方差来源**:经典 rollout 用随机策略走到底,回报方差很大,需要很多次模拟才能把 Q 估准。**AlphaZero 用价值网络 V(s) 替代 rollout** 正是为了砍掉这个方差——一次确定性的网络前向给出低方差估值,等价于把"无数次随机 rollout 的期望"摊销进了网络。这是 AlphaGo 系列相对经典 MCTS 的关键工程跃迁之一。
- **探索项的尺度敏感**:c 和 Q 必须在可比尺度上。AlphaZero 用的不是上面这个朴素 UCT,而是 **PUCT** 变体(下一节),它把策略网络给的先验 prior 也乘进探索项,让搜索一开始就偏向"网络认为好的"动作,大幅提高搜索效率。

## 第四节:AlphaGo → AlphaZero → MuZero —— 砍先验,长本事

这条演进线是"学习 + 搜索"路线的教科书。看清每一代**砍掉了什么人类先验,却反而更强**,你就理解了这套方法的力量来自哪里。

### AlphaGo(2016):搜索 + 学习的首次大成,但还拄着人类拐杖

AlphaGo 用了三个网络/组件:

1. **策略网络(policy network)**:先用**人类高手棋谱**做监督学习(behavior cloning),学 `p(a|s)` 模仿人类落子;再用自博弈 policy gradient 强化。
2. **价值网络(value network)**:估 `V(s)`,预测从该局面出发的胜率。
3. **MCTS**:用策略网络给搜索提供先验(往哪些动作扩展)、用价值网络 + 快速 rollout 评估叶节点,把搜索结果汇总成最终落子。

它的关键依赖:**人类棋谱**——策略网络的起点是模仿人类。这既是助力(冷启动快)也是天花板(被人类经验束缚)。

### AlphaZero(2017):扔掉人类数据,一个网络,一条飞轮

AlphaZero 做了三个激进简化,结果更强、更通用(同一套算法通杀围棋、国际象棋、将棋):

1. **零人类数据**:完全从随机初始化的网络开始**自博弈(self-play)**,不看任何人类棋谱。
2. **单一双头网络** `f_θ(s) = (p, v)`:一个网络同时输出策略分布 `p(·|s)` 和价值 `v(s)`,共享表示。
3. **MCTS 用 PUCT,且 MCTS 的输出反过来当训练目标**(下面详述)——这是飞轮的心脏。

AlphaZero 的搜索选择规则是 **PUCT**(Predictor + UCT):

```
a* = argmax_a [ Q(s,a) + c_puct · P(s,a) · √(Σ_b N(s,b)) / (1 + N(s,a)) ]
```

对比上一节的 UCT,差别在探索项:这里乘了网络给的先验 **P(s,a) = p(a|s)**,且形式是 `√(ΣN) / (1+N)` 而非 `√(ln N / N)`。直觉:**让搜索一开始就重点试"网络认为有希望"的动作**(P 大的探索项大),而不是均匀地试所有动作——在围棋这种巨大动作空间里,这是把搜索算力花在刀刃上的关键。随着某动作被访问得多(N 大),探索项衰减,逐渐让位给实测 Q。【待核:PUCT 探索项的精确常数与是否含额外缩放,以原论文/伪代码为准;这里给的是被广泛复述的形式】

#### 训练目标:MCTS 是"策略改进算子",网络是"摊销器"

这是整章最该被讲透的一点。AlphaZero 的训练循环:

```
自博弈生成数据:
  对局中每个状态 s,跑 N 次 MCTS,得到根节点访问分布
      π_MCTS(a|s) = N(s,a)^{1/τ} / Σ_b N(s,b)^{1/τ}   （τ 是温度）
  按 π_MCTS 落子;整局下完得到最终胜负 z ∈ {+1, −1}(对每个状态按其行动方记号)

训练网络 f_θ(s) = (p, v),最小化:
  L(θ) = Σ_s [ (z − v(s))²            # 价值头:回归真实对局结果
              − π_MCTS(s)·log p(s)     # 策略头:交叉熵,模仿 MCTS 的访问分布
              + λ‖θ‖² ]                # 正则
```

**为什么这个循环会持续变强?把它看成两个算子的交替不动点迭代。**

定义两个映射:

- **改进算子(improvement)`I`**:给定当前网络策略 `p_θ`,MCTS 用它当先验、用 `v_θ` 当叶子评估,做深度搜索,输出 `π_MCTS`。**关键断言:在算力足够时,`π_MCTS` 是比 `p_θ` 更好的策略。** 凭什么?因为 MCTS 是一次前瞻——它不是直接信 `p_θ`,而是用 `p_θ` 当起点、向前模拟很多步、用价值反传修正,本质上是在做一步(带函数近似的)**策略改进 / 前瞻 backup**。这正对应策略迭代里的 policy improvement step:用一步 lookahead 得到的贪心策略不差于原策略(policy improvement theorem 的精神)。搜索越深、模拟越多,`π_MCTS` 相对 `p_θ` 的改进越大。
- **摊销算子(projection / amortization)`P`**:训练让网络 `p_θ` 去拟合 `π_MCTS`(策略头的交叉熵),让 `v_θ` 去拟合对局真实结果 z(价值头的回归)。这是把"昂贵的、每步要跑 N 次模拟才能得到的搜索结果",**蒸馏 / 摊销进一根快速前向的网络**里。下次搜索的起点 `p_θ` 因此被抬高了。

交替执行 `θ ← P(I(θ))`:**搜索把策略往上抬一截(改进),训练把这一截固化进网络(摊销),抬高后的网络又让下一轮搜索能搜得更好,如此螺旋上升。** 这就是自博弈飞轮:

```
        ┌──────────────────────────────────────────┐
        │                                            │
   网络 p_θ, v_θ ──当MCTS先验/评估──▶ MCTS 搜索        │
        ▲                              │             │
        │                              ▼             │
   训练(摊销)◀──π_MCTS, z 当目标── 更强的 π_MCTS      │
        │                                            │
        └────────────────────────────────────────────┘
   每转一圈:策略被"搜索"放大,再被"学习"固化,起点更高
```

**为什么零人类知识也能持续变强(自博弈的自举)**:对手就是自己。当你变强,你的对手(自己)也同步变强,于是你永远面对一个"和你水平相当、略弱于你下一步要达到的水平"的对手——这是一个**自动生成的、难度始终匹配的课程(curriculum)**。你不需要人类提供难度合适的对局,系统自己造。配合"搜索改进 + 学习摊销"这个不动点迭代,只要改进算子 I 真的能把策略往上抬(算力够、模型/评估够准),整个系统就会从随机水平 bootstrap 到超人——因为**它的每一步训练目标(π_MCTS)永远略好于它当前的策略**,它在追一个被自己不断推高的目标。

**这件事为什么能超越人类**:人类棋谱是天花板对模仿学习成立,但对"自我改进的搜索"不成立。AlphaZero 不模仿任何人,它只服从一个客观信号——**胜负**(z)。胜负不撒谎、不带人类偏见。一旦把"变强"接到一个不带人类天花板的客观信号上,再用搜索+摊销的飞轮去逼近它的不动点,超越人类只是算力和时间问题。AlphaGo Zero 在围棋上从零自学几天就超越了所有人类与早期版本,正是这个机制的实证。

#### 一个易被忽略的探索细节:根节点的 Dirichlet 噪声

自博弈如果完全按网络先验走,容易陷入重复套路(探索不足,飞轮转死)。AlphaZero 在**根节点的先验上注入 Dirichlet 噪声**:`P(s₀,a) ← (1−ζ)·p_θ(a|s₀) + ζ·η_a`,η ~ Dir(α)。这保证每局开局都有随机扰动,逼迫搜索去试一些网络不看好的动作——**这是飞轮不退化、能持续发现新策略的关键工程保险**。新手做自博弈最常见的失败就是忘了根噪声,导致多样性崩塌、棋力停滞。

### MuZero(2019 预印本 / Nature 2020):连规则都不要——在隐空间学一个"够规划用"的模型

AlphaZero 仍有一个强先验:**它需要环境的真实模型**(围棋/象棋规则给定,MCTS 扩展节点时用真实规则算 `transition(s,a)`)。可现实世界(Atari、真实控制)规则未知。**MuZero 的飞跃:不要真实规则,自己学一个隐式模型;而且这个模型不需要能重建真实观测,只需要能支撑 MCTS 做出正确的价值/策略预测。**

MuZero 学三个函数(都是神经网络):

```
表示函数  h:  s₀ = h(o₁,...,o_t)          # 把观测历史编码成根隐状态 s₀
动力学    g:  s_{k},  r_k = g(s_{k−1}, a_k)  # 在"隐状态空间"里走一步,顺带预测奖励
预测      f:  p_k, v_k = f(s_k)             # 从隐状态预测策略和价值
```

**核心理解:这里的隐状态 `s_k` 不对应任何真实游戏状态,也不要求能解码回画面。** 它纯粹是一个**为了让 f、g 的预测准确而存在的内部表示**。MuZero 把 MCTS 整个搬进这个隐空间:根节点是 h(观测),每次扩展用 g 在隐空间走一步并拿到预测奖励 r_k,叶子评估用 f 给的 v_k,选择用 PUCT(先验来自 f 给的 p_k)。整棵搜索树活在网络自己造的隐空间里。

训练时,把 h、g、f 沿"实际走过的动作序列"展开 K 步,**端到端联合训练**三个目标:

```
对从真实轨迹采样的一段 (o_t, a_{t+1}, ..., a_{t+K}):
  s₀ = h(o_t)
  for k = 1..K:  s_k, r_k = g(s_{k−1}, a_{t+k}),  p_k, v_k = f(s_k)
  L = Σ_{k=0..K} [
        ℓ_reward(  r_k,        u_{t+k} )      # 预测奖励 vs 真实奖励
      + ℓ_value(   v_k,        z_{t+k} )      # 预测价值 vs n-step 自举回报(用 MCTS 价值/真实回报)
      + ℓ_policy(  p_k,    π_MCTS_{t+k} )      # 预测策略 vs 该步 MCTS 访问分布
      ]
```

**这里是 MuZero 最反直觉、也最该讲透的设计**:训练信号里**完全没有"重建观测/状态"这一项**。模型从不被要求"预测下一帧画面长什么样"。它只被要求:**沿真实动作序列展开后,每一步预测的奖励、价值、策略要对**。换句话说,MuZero 学的是一个**价值等价(value-equivalent)模型**——只要它在"能算出正确的规划量(奖励、价值、策略)"这件事上等价于真实模型,它内部到底怎么表示状态、是否对应物理现实,**根本不重要**。

为什么这是神来之笔?回到第一节的阿喀琉斯之踵:**学一个能精确重建复杂世界的模型几乎不可能(想想预测 Atari 下一帧的每个像素),而且重建误差会毒害规划。MuZero 直接放弃重建,只学"规划所需的最小充分信息"。** 它把建模目标从"复现世界"收窄到"复现规划相关的量",一举绕开了"完美世界模型"这个不可能任务。这也是为什么 MuZero 能用同一套算法在围棋/象棋(达到 AlphaZero 水平)和 Atari(达到当时 SOTA)上都work——它对环境的要求降到了最低:**只要能交互、有奖励信号即可,连规则都不用告诉它**。

**砍先验的总轨迹**,一句话串起来:

```
AlphaGo:  需要 人类棋谱 + 真实规则 + 多个网络
AlphaZero:需要        真实规则 + 单一网络   （砍掉人类数据,自博弈自举)
MuZero:   只需要               单一网络框架 （连规则也砍掉,隐空间学价值等价模型)
```

每砍掉一层人类/环境先验,方法就更通用一步——而表现不降反升。**这有力地说明:这套'搜索放大 + 学习摊销'的飞轮才是表现的真正来源,人类先验只是可有可无的脚手架。** 这正是"the bitter lesson"(苦涩的教训,Sutton)的最佳注脚:**长期看,能随算力扩展的通用方法(搜索 + 学习)总会碾压依赖人类知识的专用方法。**

## 第五节:把飞轮的"会变强"讲到能验算的程度

前面说"搜索是策略改进算子",这里给一个更扎实的论证骨架,让你能自己验算"为什么 π_MCTS 不差于 p_θ"。

考虑理想化情形:MCTS 做了无穷多次模拟,且叶子评估 v_θ 准确。则在每个节点,UCT/PUCT 的访问分布会集中到使 `Q(s,a)` 最大的动作上——也就是 MCTS 趋向于对当前价值估计做**贪心 lookahead**。回忆策略迭代里的 **policy improvement theorem**:

```
设 π' 是对 Q^π 的贪心策略:π'(s) = argmax_a Q^π(s,a)。
则对所有 s:  Q^π(s, π'(s)) = max_a Q^π(s,a) ≥ Q^π(s, π(s)) = V^π(s)。
由此(沿 Bellman 算子单调性逐步展开):  V^{π'}(s) ≥ V^π(s),对所有 s。
```

推导要点(可自己补全):令 π' 贪心于 Q^π,则 `V^π(s) ≤ Q^π(s,π'(s)) = E[r + γ V^π(s')]`;把右边的 V^π 再用同样不等式替换一次、反复代入,得到一个单调不减的序列收敛到 `V^{π'}`,故 `V^{π'} ≥ V^π`。**结论:贪心于当前价值的新策略,价值不会变差。**

MCTS 不是精确的一步贪心(它是带前瞻、带函数近似、有限模拟的近似版),但**精神一致**:`π_MCTS` 是一个"用 v_θ 做多步前瞻后得到的、比直接信 p_θ 更接近贪心"的策略。因此在足够算力下,`π_MCTS` 对应的价值 ≥ p_θ 对应的价值。训练把 p_θ 拉向 π_MCTS,等于把网络策略往"价值更高"的方向推。

**这就闭环了**:

- 改进算子 I 把策略价值单调抬高(policy improvement 的精神);
- 摊销算子 P 把抬高后的策略固化进网络,几乎不丢失(只要网络容量够、拟合够好);
- 价值头同时被真实对局结果 z 校准,保证 v_θ 不漂(防止改进算子用错误评估做出错误前瞻——这正是防 model exploitation 的关键:**价值最终锚定在客观胜负上**)。

**新手坑(讲透必须点的几条)**:

- **价值头漂移导致飞轮反向**:若 v_θ 系统性高估某类局面,MCTS 会被骗着往那儿搜,π_MCTS 学坏,网络更信那类局面——正反馈崩坏。锚定客观 z、足够的探索(根噪声)是刹车。
- **搜索深度不够时 I 不再是改进**:模拟次数太少,π_MCTS 可能还不如 p_θ(没搜出名堂,反而被噪声带偏)。改进算子有效的前提是算力门槛,达不到飞轮不转甚至倒转。
- **温度 τ 的作用**:训练早期用高温(τ=1,按访问数比例采样,鼓励探索),对局后段/评估用低温(τ→0,取访问最多,纯利用)。温度调度不当会让自博弈数据多样性不足或决策不够果断。

## 设计权衡与常见坑(汇总)

**model-based vs model-free 怎么选**:

- 真实交互**昂贵/危险/慢**(机器人、医疗、对弈)→ 倾向 model-based,样本效率是刚需。
- 环境**模拟器免费且快**(很多游戏)→ model-free(PPO 等)往往更简单稳健,不必背模型误差的包袱。**这点反直觉但重要:有大量廉价样本时,model-free 的简单性常常赢**。
- 环境**动力学复杂难建模**(高维像素)→ 若要 model-based,走 MuZero 式"价值等价模型",别试图重建世界。

**MCTS 工程坑**:

- 探索常数 c / c_puct 对棋力极敏感,且与价值尺度耦合,必须按经验调,不能照搬 √2。
- 零和博弈回传时**符号取反**别忘(代码里 `value = -value`),漏了会训练出"帮对手赢"的滑稽策略。
- 输出动作用**访问次数 N** 而非 Q,鲁棒性差很多。
- 自博弈务必加**根 Dirichlet 噪声**,否则多样性崩塌。
- 搜索深度/模拟次数是 anytime 旋钮:**算力换质量**,但低于阈值时改进算子失效。

**model-based 通病——模型利用(model exploitation)**:策略会专门钻模型的错误区域刷高想象回报。对策:短 rollout、模型集成估不确定性、价值锚定客观信号(MuZero 的 z)。**这是这条路线最致命、最隐蔽的坑,一旦出现表现为"模型内分数飞涨、真实表现崩盘"。**

## 动手练习

1. **(推导题)从 UCB 到 UCT。** 用 Hoeffding 不等式,完整推导 UCB1 的探索项为什么是 √(2 ln n / Nᵢ)。提示:写出"真实均值超过置信上界的概率 ≤ exp(−2Nᵢuᵢ²)",令该概率等于 n⁻⁴,解出 uᵢ;再说明为何选 n⁻⁴ 这个衰减速度能同时保证(a)对数后悔 O(ln n) 和(b)任何臂不被永久饿死。进一步:把 n→N(s)、Nᵢ→N(s,a),写出 UCT,并解释 c 为何不能直接取 √2。

2. **(分析题)自博弈为何零人类知识仍能持续变强。** 用"策略改进算子 I + 摊销算子 P"的框架论证:为什么交替执行 `θ ← P(I(θ))` 会让策略价值单调上升?明确指出(a)I 对应策略迭代里的哪一步、依赖什么定理;(b)P 在什么条件下"几乎不丢失改进";(c)为什么对手是自己能提供"难度自动匹配的课程";(d)哪一项设计防止飞轮被错误价值带崩(提示:价值头锚定什么)。再答:若把价值头训练目标 z(客观胜负)换成"人类专家对局面的打分",这个超越人类的论证会在哪里断掉?

3. **(编码题)实现并验证 MCTS。** 用本章伪代码,在井字棋(Tic-Tac-Toe)上实现 MCTS+UCT(rollout 用随机策略)。验证:(a)模拟次数→∞ 时,MCTS 永不输(井字棋最优策略至少平局);(b)画"模拟次数 vs 选到最优首手的频率"曲线,观察收敛;(c)把 rollout 换成一个粗糙的手写评估函数,比较收敛速度差异——体会"低方差评估替代随机 rollout"的价值(这正是 AlphaZero 用价值网络的动机)。提示:终局回报用 +1/0/−1,回传记得逐层取反。

4. **(分析+推导题)复合误差与 rollout 深度。** 设学到的模型每步价值偏差上界为 ε,折扣为 γ。论证:从真实状态出发在模型里 rollout H 步估计回报,其偏差大致受 ε·(1−γ^H)/(1−γ) 量级支配(给出几何级数求和),并说明当 H→∞ 时上界趋于 ε/(1−γ)。据此解释:(a)为什么 MBPO 用短分支 rollout 而非长程;(b)为什么 MuZero 用"价值等价模型"而非重建模型能侧面缓解这个问题(提示:重建误差 vs 价值预测误差,哪个直接进入规划)。

## 小结与承上启下

这一章给了你强化学习的**第二条主路线**:不只是从经验里直接学策略(model-free),而是**学一个模型 + 在模型里搜索**,用搜索做深度前瞻、用学习把前瞻结果摊销回网络。核心要带走的三件事:

1. **model-based 用模型误差换样本效率**,而管理模型误差(短 rollout、不确定性、价值等价)是这条路线的工程主线;Dyna 是把真实经验与想象经验统一进一个学习循环的最简范式。
2. **MCTS 的灵魂是把"选哪个动作搜"当 bandit 问题**,UCT 直接源自 UCB1 的对数后悔最优性,你已能从 Hoeffding 不等式把它推出来;它是一个 anytime 的、用算力换决策质量的局部规划器。
3. **AlphaGo→AlphaZero→MuZero 是一条"砍人类/环境先验、表现反升"的飞轮**:搜索是策略改进算子、网络是摊销器,交替迭代逼近一个被客观胜负信号锚定的不动点——这就是零人类知识 bootstrap 到超人的数学根据,也是"苦涩的教训"的最佳实证。

**承上**:前几章的 policy gradient / actor-critic / 价值方法,在这里以两种身份回归——它们既是 model-free 的对照组(凸显 model-based 的样本效率优势),又是 model-based 内部的"被摊销对象"(AlphaZero 的策略头本质是一个被 MCTS 监督的策略网络)。policy improvement theorem 在第五节直接撑起了飞轮的正确性。

**启下**:这条"搜索 + 学习"的飞轮,正是当前**大模型推理(reasoning)**路线的精神先祖。把"棋盘 + 落子"换成"问题 + 推理步骤(token)",把"胜负 z"换成"答案对错/可验证奖励",把"MCTS 前瞻"换成"采样多条推理链 + 重排/自洽",AlphaZero 的飞轮就变成了 **o1 / R1 式推理模型的训练范式雏形**——用搜索/采样放大当前策略、用 RL 把放大后的好轨迹摊销回模型。后续讲推理模型与 RLHF 时,你会反复看到这一章的影子:**reasoning 本质上就是在'思维空间'里做 AlphaZero。** 下一章我们就从这个类比出发,进入推理模型与可验证奖励的世界。
