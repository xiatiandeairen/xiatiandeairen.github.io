> 这一章把你从"知道 GPT 是预测下一个词、BERT 是完形填空"带到"能从概率分解、梯度信号密度、任务统一性三个层面推清楚为什么生成式自回归(autoregressive, AR)在大模型时代赢了,能精确说出掩码语言模型(masked LM, MLM)只预测 15% token 背后的信息论与一致性权衡,能从头推导 teacher forcing 为什么能让一次前向并行算出所有位置的 loss、以及它埋下的 exposure bias 是怎么累积误差的"。学完你能自己写出一个数值正确的自回归训练目标(shift + 因果掩码 + 交叉熵),能为任意一个任务设计出三种范式各自的输入输出构造,并能讲清楚"单向不如双向"这个代价为什么被生成能力和通用性盖过。这是理解后续指令微调、in-context learning、RLHF 全部建立在"续写"这一行为上的根。

## 一、动机:预训练目标是模型"学什么"的总开关

前面几章我们把 Transformer 的**结构**(注意力、位置编码、归一化、前馈)拆透了。但结构只是"模型能算什么"的容器。**真正决定一个几百亿参数的模型从海量无标注文本里学到了什么的,是预训练目标(pre-training objective)——也就是我们让它在每一个训练样本上去最大化或最小化的那个标量。** 同一个 Transformer 主干,配上不同的预训练目标,会长成完全不同的东西:GPT 和 BERT 的骨架几乎一样(都是堆叠的 self-attention 块),但因为目标不同,一个天生会生成、另一个天生擅理解却不能直接写句子。

这章要回答的核心问题有三个,层层递进:

1. **三大预训练范式各自在数学上"喂给模型什么信号"?** 自回归 LM、掩码 LM、去噪/seq2seq,它们的输入输出构造、损失定义、梯度信号密度各不相同。把这三者并排写清楚,是后面一切比较的基础。
2. **为什么是生成式自回归赢了,而不是理解能力更强的双向掩码?** 2018-2019 年 BERT 在几乎所有 NLU(自然语言理解)榜单上碾压同期 GPT;但 2020 年之后,大模型清一色走 AR 路线。这个反转不是偶然,背后是任务统一性、可生成性、与 in-context learning 的契合、scaling 友好这几条结构性优势,共同盖过了"单向看不到未来 token"这个明显的表征劣势。
3. **AR 的训练机制里有什么必须吃透的细节和坑?** teacher forcing 如何借因果掩码把"逐词生成"的串行任务变成"一次前向并行算全部位置 loss"的并行任务;它带来的 exposure bias(暴露偏差)为什么会在推理时累积误差。

**给有扎实工程数学基础的读者一句迁移提示:** 你可以把预训练目标理解为"自监督学习里那个自动构造监督信号的规则"。监督学习里标签是人给的;自监督里标签是从数据自身按某个规则抠出来的(把后一个词当标签、把被遮的词当标签、把被破坏的 span 当标签)。**不同范式 = 不同的"从无标注数据里自动造标签"的规则**,而这个规则决定了梯度往哪个方向推、信号有多密、学到的表征能直接干什么。

下面先把语言建模的概率地基立起来,这是三大范式共同的出发点。

## 二、预备:语言建模的概率分解

一段文本是一个 token 序列 `x = (x₁, x₂, …, x_T)`,每个 `xₜ` 取自词表 V(大小 |V| 通常 3 万到 20 万,**待核**,取决于 tokenizer)。**语言建模(language modeling)的终极目标是建模这段文本的联合概率 `p(x₁, …, x_T)`。** 但联合分布是一个 |V|^T 维的对象,天文数字,无法直接建模。

概率论给了一个不损失任何信息的分解——**链式法则(chain rule of probability)**:

```
p(x₁, x₂, …, x_T) = p(x₁) · p(x₂|x₁) · p(x₃|x₁,x₂) · … · p(x_T|x₁,…,x_{T-1})
                  = ∏_{t=1}^{T} p(xₜ | x_{<t})
```

其中 `x_{<t}` 记 `(x₁, …, x_{t-1})`,叫**前缀(prefix)** 或上下文。这一步是恒等的,没有任何近似:任何联合分布都能这样拆成一串条件分布的连乘。**自回归语言模型的全部数学,就是用一个神经网络去拟合右边那个条件分布 `p_θ(xₜ | x_{<t})`。** 给定前缀,网络输出词表上的一个分布(softmax 头),我们让真实的下一个词在这个分布下的概率尽可能大。

这里有一个值得停下来体会的点:**链式法则的分解方向(从左到右)是人为选的,不是唯一的。** 我们也可以从右到左分解 `p(x) = ∏ p(xₜ | x_{>t})`,或者按任意排列顺序分解(后面 XLNet 就是这么干的)。三大范式的本质分歧,很大程度上就是**"如何分解这个联合概率、用哪些上下文来预测哪些 token"** 的不同选择:

- **自回归 LM**:严格从左到右,`p(xₜ | x_{<t})`,只用左侧上下文。
- **掩码 LM**:不做严格分解,而是建模 `p(x_masked | x_visible)`——给定可见部分,独立地预测每个被遮位置,用了**双向**上下文。
- **去噪/seq2seq**:先用一个破坏函数把 `x` 变成 `x̃`,再建模 `p(x | x̃)`,即从被破坏的输入重建原文。

把这三种"分解/重建方式"逐一推清楚,就能理解它们能力差异的根源。

## 三、范式一:自回归语言建模(GPT 路线)

### 3.1 目标函数与最大似然

自回归 LM 直接对链式法则的每一项取对数、求和,得到序列的对数似然。在整个语料 D 上做**最大似然估计(maximum likelihood estimation, MLE)**,等价于最小化负对数似然(negative log-likelihood, NLL):

```
L_AR(θ) = - E_{x ~ D} [ Σ_{t=1}^{T} log p_θ(xₜ | x_{<t}) ]
```

对单个位置 t,`p_θ(xₜ | x_{<t})` 由网络在该位置输出的 logits 经 softmax 得到。设网络在位置 t(即"看完 x_{<t} 后")输出 logits 向量 `zₜ ∈ ℝ^|V|`,则:

```
p_θ(· | x_{<t}) = softmax(zₜ)
log p_θ(xₜ | x_{<t}) = zₜ[xₜ] - log Σ_{v∈V} exp(zₜ[v])
```

第二行就是大家熟悉的**交叉熵(cross-entropy)**:目标 token 的 logit 减去 log-sum-exp 归一化项。**整条序列的 loss = 每个位置交叉熵的和(或平均)。** 这个目标的美妙之处在于:它对序列里**每一个 token 都产生一个监督信号**——一段长度 T 的文本贡献 T 个预测任务。这一点在第六节比较信号密度时是关键。

### 3.2 因果掩码:如何用一次前向并行算出全部位置的 loss

朴素地想,"预测每个位置的下一个词"似乎得串行做 T 次:先用 x₁ 预测 x₂,再用 x₁x₂ 预测 x₃……每次喂的前缀长度不同。**如果真这么训练,长度 T 的序列要跑 T 次前向,慢到不可接受。**

Transformer 的关键工程胜利,是用**因果掩码(causal mask, 也叫下三角掩码)** 让一次前向就并行算出所有位置的预测。机制如下。在 self-attention 里,位置 i 的输出是对所有位置 value 的加权和,权重来自 `QKᵀ/√d` 的 softmax。我们在 softmax 之前,把所有"未来位置"(j > i)对应的注意力分数设成 -∞:

```
注意力分数矩阵 S = QKᵀ / √d        (形状 T×T,S[i,j] = 位置 i 对位置 j 的关注度)

加因果掩码:  S[i,j] ← -∞   当 j > i        (位置 i 不许看位置 j>i)

A = softmax(S, dim=-1)              (每行归一化;-∞ 经 exp 变 0,未来位置权重恰好为 0)
```

因为 `exp(-∞) = 0`,被掩位置在 softmax 后权重严格为零。**于是位置 i 的表征只依赖 x₁…xᵢ,绝不会"偷看"到 i 之后的 token。** 这就保证了:网络在位置 i 的输出 `zᵢ`(进而 `p_θ(·|x_{<i+1})`)只基于左侧上下文——和我们逐个前缀串行喂进去得到的结果**在数学上完全一致**,但一次前向就全算出来了。

**关键洞察:因果掩码把"自回归"这个串行的概率结构,翻译成了一个可以高度并行的矩阵运算。** 这是 AR 能高效大规模训练的工程命门,和上一章 Transformer 抛弃 RNN 的动机一脉相承——RNN 的串行依赖在时间维上无法并行,而带因果掩码的 attention 把时间依赖编码进了一个能并行计算的掩码矩阵里。

### 3.3 shift:输入和标签错一位

训练时怎么构造监督?设原始 token 序列为 `x = (x₁, …, x_T)`。我们要让"位置 t 的输出"去预测 `x_{t+1}`。所以:

```
输入 (input)  : (x₁, x₂, …, x_{T-1})      → 喂给网络
标签 (target) : (x₂, x₃, …, x_T)          → 即输入整体右移一位

位置 t 的网络输出 zₜ 负责预测 标签[t] = x_{t+1}
```

这个"标签 = 输入右移一位"的操作就是 **shift(移位)**。它配合因果掩码,使得位置 t 的预测只用到了 x_{<t+1},而要预测的恰是 x_{t+1}——没有任何信息泄漏。下面的代码把这套机制完整实现一遍。

### 3.4 代码:自回归 LM 的训练目标

```python
import torch
import torch.nn.functional as F

def autoregressive_loss(logits, input_ids, ignore_index=-100):
    """
    自回归 LM 的核心训练目标:shift + 因果掩码下的交叉熵。
    logits:    (B, T, V)  模型在每个位置对词表的打分。前提是模型内部已用因果掩码,
                          所以 logits[:, t] 只依赖 input_ids[:, :t+1]。
    input_ids: (B, T)     一个 batch 的 token 序列。
    返回:      标量平均交叉熵(每个被预测 token 的平均 NLL)。
    """
    # ---- shift: 用前 T-1 个位置的输出,去预测第 2..T 个 token ----
    shift_logits = logits[:, :-1, :].contiguous()   # (B, T-1, V) 位置 0..T-2 的预测
    shift_labels = input_ids[:, 1:].contiguous()    # (B, T-1)    目标是 token 1..T-1
    # 位置 t 的 logits 预测 input_ids[t+1] —— 这就是"预测下一个词"

    # ---- 交叉熵: 每个位置一个 |V| 类分类 ----
    loss = F.cross_entropy(
        shift_logits.view(-1, shift_logits.size(-1)),  # (B*(T-1), V)
        shift_labels.view(-1),                          # (B*(T-1),)
        ignore_index=ignore_index,    # padding 位置标 -100, 不计入 loss
    )
    return loss


def build_causal_mask(T, device):
    """构造 T×T 因果掩码:位置 i 只能看 j<=i。返回加到注意力分数上的偏置矩阵。"""
    # 上三角(不含对角线)为 True 的位置是"未来",要被屏蔽
    future = torch.triu(torch.ones(T, T, dtype=torch.bool, device=device), diagonal=1)
    mask = torch.zeros(T, T, device=device)
    mask.masked_fill_(future, float('-inf'))   # 未来位置填 -inf, 经 softmax 后权重为 0
    return mask    # 用法: attn_scores = QK^T/sqrt(d) + mask; A = softmax(attn_scores, -1)
```

几个**新手必须看清的实现要点**:(1) `shift_logits` 取 `[:, :-1]`、`shift_labels` 取 `[:, 1:]`,错一位——漏掉这步会让模型"预测当前词"即看到答案,loss 会异常地低到接近 0,这是最常见的 bug。(2) `ignore_index=-100` 让 padding 不污染 loss。(3) 因果掩码用加 -inf 而非乘 0,因为它作用在 softmax **之前**的 logits 上;乘 0 会在 softmax 后破坏归一化。

### 3.5 复杂度与信号密度分析

**计算复杂度**:一次前向算出全部 T 个位置的 loss,主导项是 attention 的 O(T²·d) 和 FFN 的 O(T·d²)。与"只做一次预测"相比,**算 T 个位置的 loss 几乎是免费的**(就是同一次前向的输出层多取了 T 个位置)。这是 AR 训练效率的核心。

**信号密度**:长度 T 的序列产生 **T 个**监督信号(每个 token 都被预测一次)。设语料共 N 个 token,则 AR 在一个 epoch 内提供约 N 个预测任务——**几乎把数据的每一个 token 都榨成了一次学习机会**。记住这个数字,第六节比较时它是 MLM 的约 6-7 倍。

## 四、范式二:掩码语言建模(BERT 路线)

### 4.1 目标:用双向上下文做完形填空

掩码 LM 换了一种构造监督的规则:**随机选一部分 token 遮住(替换成特殊符号 [MASK]),让模型用剩下的双向上下文去还原被遮的 token。** 形式化:设掩码位置集合为 M ⊂ {1,…,T},`x_M` 为被遮的 token,`x_{\M}` 为可见部分(其中 M 处填 [MASK]),目标是:

```
L_MLM(θ) = - E_{x ~ D, M} [ Σ_{t ∈ M} log p_θ(xₜ | x_{\M}) ]
```

**关键差异在条件部分:`x_{\M}` 是整段可见上下文,包含 t 左侧和右侧的词。** 所以掩码 LM 是**双向(bidirectional)** 的——预测一个位置时左右都看。实现上,它**不加因果掩码**,attention 是全连接的(每个位置看所有位置),被遮位置的真实身份不在输入里(已被 [MASK] 替换),所以"看右侧"不算作弊。

这就是掩码 LM 表征更强的根源:**人类理解一个词的含义本来就要看上下文两边**(英语里 "bank" 是河岸还是银行,得看后文)。双向上下文让每个 token 的表征融合了完整语境,这对分类、抽取、句子关系判断这类**理解(NLU)** 任务极其有利。BERT 在 2018 年几乎刷爆所有 NLU 榜单,正是吃了双向的红利。

### 4.2 为什么只预测 15%?——信息论与一致性的双重权衡

BERT 原文(Devlin et al., 2019)选了一个看起来很随意的数字:**随机选 15% 的 token 作为预测目标**。这个 15% 不是拍脑袋,背后有两层逻辑,必须讲透。

**第一层:掩码率是"任务难度"和"信号密度"的权衡。** 设掩码率为 r。

- r 太高(比如遮 50%):可见上下文太少,完形填空变成几乎无信息的瞎猜,每个空缺周围的语境被破坏得太厉害,模型学不到稳定的语义。极端 r=1 就是没有上下文的无条件生成,退化成纯先验。
- r 太低(比如遮 1%):每个 batch 里被预测的 token 太少,**梯度信号极其稀疏**——大部分算力花在编码那些不产生 loss 的可见 token 上,训练效率低下。

15% 是经验上**让上下文仍足够丰富、同时信号密度还不至于太低**的折中点。**注意这里和 AR 的本质区别:AR 每个 token 都贡献 loss(信号密度 ≈100%),MLM 只有约 15% 的 token 贡献 loss——这是 MLM 训练效率天然低于 AR 的根本原因之一(见第六节定量)。**

**第二层:被选中的 15% 还要再做 80/10/10 拆分,解决 train-test 不一致。** 这是更精妙的一点。如果被选中的 token 一律换成 [MASK],会有一个**分布不匹配(distribution mismatch)** 问题:[MASK] 这个符号只在预训练出现,下游微调和推理时输入里根本没有 [MASK]。模型可能学成"只在看到 [MASK] 时才认真编码语义,看到真实 token 就偷懒",导致预训练学到的能力在下游用不上。BERT 的解法是把被选中的 15% 再拆:

```
被选中的 token (占全体 15%) 中:
  80%  → 替换成 [MASK]          (主任务:从上下文还原)
  10%  → 替换成一个随机 token    (制造噪声:迫使模型不能盲信输入 token,要核对上下文)
  10%  → 保持原 token 不变       (但仍要预测它:让模型对"未被遮的真实 token"也输出正确分布)
```

**这套拆分的目的是:让模型对"输入里的每一个真实 token"都保持谨慎的语义编码,而不是只对 [MASK] 位置卖力。** 后两种情况(随机替换、保持不变)制造了"输入 token 不一定可信/不一定是答案"的压力,逼模型始终依赖上下文做判断,从而缩小了预训练(有 [MASK])和下游(无 [MASK])的行为差距。**这是一个用训练时的人为扰动去对齐 train/test 分布的经典技巧**,理解它对后面理解很多正则化和数据增强手段都有迁移价值。

### 4.3 掩码 LM 致命的结构缺陷:不能直接生成,且预测独立

掩码 LM 的目标决定了它**无法直接做生成**。原因有二:

**其一,它建模的是 `p(x_M | x_{\M})`,需要一个"已经存在、只是被挖了空"的句子作为条件。** 而生成任务是从零开始、从左到右地造出整句话——没有可见上下文可填。掩码 LM 没有"给定前缀续写下一个词"的能力对象。

**其二,被遮的多个位置是条件独立预测的。** 看损失:`Σ_{t∈M} log p_θ(xₜ | x_{\M})`,每个被遮位置都基于同一个 `x_{\M}` 独立预测,彼此之间**没有建模依赖**。它假设:

```
p(x_M | x_{\M}) ≈ ∏_{t∈M} p(xₜ | x_{\M})      ← 条件独立假设, 这是个近似!
```

这个条件独立假设在被遮 token 相互关联时是错的(比如遮掉 "New [MASK]" 里的 York,和遮掉 "[MASK] York" 里的 New,如果同时遮,模型不知道两者要配套)。**AR 的链式法则是精确分解(无近似),MLM 的多位置预测是有近似的——这是 AR 在"建模真实联合分布"上更干净的一个理论优势。**

正因这两点,BERT 类模型擅长"理解/打分/抽取",但你没法让它像 GPT 那样写一篇文章。**这个"不能生成"的缺陷,是它在大模型时代被 AR 盖过的直接技术原因。**

## 五、范式三:去噪 / seq2seq(T5、BART 路线)与 prefix-LM 折中

### 5.1 去噪自编码:把破坏-重建统一成 text-to-text

第三条路线想兼得"双向编码的理解力"和"能生成"。代表是 **T5(Raffel et al., 2020)** 和 **BART(Lewis et al., 2020)**,它们用**编码器-解码器(encoder-decoder)** 结构 + **去噪(denoising)** 目标:

```
原文:        Thank you for inviting me to your party last week.
破坏(corrupt): Thank you <X> me to your party <Y> week.       ← 随机 mask 掉若干 span
目标(target):  <X> for inviting <Y> last <Z>                  ← 让解码器生成被删掉的内容
```

机制是:**编码器双向地读被破坏的输入 `x̃`(理解力来源),解码器自回归地生成被破坏掉的内容(生成力来源)。** 目标函数是一个条件 AR:

```
L_denoise(θ) = - E_{x ~ D} [ Σ_{t} log p_θ(yₜ | y_{<t}, x̃) ]
```

其中 `x̃` 是破坏后的输入(编码器看,双向),`y` 是要重建的目标序列(解码器自回归生成)。**注意目标端仍然是严格自回归的 `p(yₜ|y_{<t}, x̃)`——所以它天然能生成。** BART 用的破坏方式更杂(token 删除、文本填充、句子打乱、文档旋转等),T5 主打 **span corruption(片段破坏)**:连续遮掉若干小段、每段用一个哨兵 token(sentinel,如 `<X> <Y>`)占位。T5 论文里这个被破坏 token 的比例约 **15%、平均 span 长度约 3**(**待核**,以原文 sweep 的最优配置为准)。

**T5 最大的范式贡献是 "text-to-text":把一切 NLP 任务都统一成"输入一段文本、输出一段文本"。** 翻译是文本到文本,分类是输出类别名这个"文本",摘要、问答、相似度打分(输出一个数字字符串)全都套进这一个框架。任务的区别只体现在输入前缀的指令(如 `"translate English to German: ..."`)。**这种统一性是后来"用一个生成模型干所有事"思路的直接先声**——只不过 T5 用 encoder-decoder 实现,GPT 用 decoder-only 实现。

### 5.2 prefix-LM:在一个 decoder 里折中单向与双向

还有一种优雅的折中:**prefix-LM(前缀语言模型)**。它在**单个 decoder** 里把序列分成两段——前缀(prompt/输入)和续写(target):

```
注意力规则:
  前缀部分(x₁..x_k):  内部全连接(双向),前缀里每个 token 互相都能看
  续写部分(x_{k+1}..): 因果(单向),只能看前缀全部 + 续写中自己左侧
```

这样**前缀获得了双向编码的理解力,续写保持自回归的生成力**,而且不需要独立的 encoder——用一个特制的注意力掩码就能实现(掩码矩阵在前缀块是全 1,在续写块是下三角)。UniLM、GLM 等模型走的就是这条路,Google 的 UL2 把它作为多种目标之一统一训练。**prefix-LM 是"理解 vs 生成"光谱上 AR 与双向之间的连续插值,理解它有助于看清:单向 vs 双向不是非此即彼,而是注意力掩码形状的设计选择。**

### 5.3 三范式输入输出构造对照表

把三大范式(加 prefix-LM)的核心区别压成一张表,这是本章最该背下来的一页:

| 维度 | 自回归 LM (GPT) | 掩码 LM (BERT) | 去噪 seq2seq (T5/BART) | prefix-LM |
|------|----------------|----------------|------------------------|-----------|
| 结构 | decoder-only | encoder-only | encoder-decoder | decoder-only(特制掩码) |
| 上下文方向 | 单向(左→右) | 双向 | 编码器双向 / 解码器单向 | 前缀双向 / 续写单向 |
| 建模对象 | `p(xₜ\|x_{<t})` 精确分解 | `p(x_M\|x_{\M})` 条件独立近似 | `p(y\|x̃)` 重建 | 同 AR,但前缀双向编码 |
| 预测哪些 token | 全部(信号密度≈100%) | 约 15% | 被破坏的 span | 续写部分 |
| 能否直接生成 | ✅ 天生能 | ❌ 不能 | ✅ 能 | ✅ 能 |
| 擅长 | 生成、续写、通用 | 理解/分类/抽取 | 有明确输入的生成(翻译/摘要) | 理解+生成折中 |
| 代表训练技巧 | shift + 因果掩码 | 15% + 80/10/10 | span corruption + sentinel | 分块注意力掩码 |

## 六、核心论证:为什么生成式自回归赢了

现在把全章最重要的问题正面回答透。**2018-2019 年双向的 BERT 在 NLU 榜单上明显强于同期 GPT;但 2020 年之后大模型几乎全是 decoder-only AR。** 这个反转由四条结构性优势驱动,它们共同盖过了"单向看不到未来"的表征劣势。逐条论证。

### 6.1 任务统一性:一切皆续写

AR 把**所有任务都化约成"给定一段文本,续写下一段文本"** 这一个原语。分类?让模型续写类别名。翻译?续写译文。问答?续写答案。摘要、改写、代码、对话,统统是"在某个前缀后面接着写"。**这种统一性比 T5 的 text-to-text 更彻底**:T5 还要区分 encoder 输入和 decoder 输出两个序列,AR 连这个区分都没有——prompt 和 completion 是同一个序列的前后两段,用同一套参数、同一个目标处理。

**这条优势的深层后果是:训练目标和使用方式完全一致。** 预训练时模型学的就是"续写",推理时干的也是"续写",中间没有任何 train/use 的形式 gap。对比 BERT:预训练学的是"填 [MASK]",下游却要接一个新的分类头去做别的事,预训练目标和下游用法是错位的。**AR 的"预训练即最终形态"是它能 scaling 到通用智能的关键架构选择。**

### 6.2 与 in-context learning / 指令对齐天然契合

**in-context learning(上下文学习, ICL)** 指模型在不更新参数的情况下,仅凭 prompt 里给的几个示例就完成新任务。这个能力**只在 AR 范式里自然涌现**,原因是:AR 训练时见过海量"前面的内容约束后面的内容"的模式(文档里前文定义的概念后文会沿用、列表前几项的格式后几项会跟随)。**ICL 本质上就是"用 prompt 里的示例当作前缀,约束续写"——这正是 AR 训练目标的直接外推。** 模型学会的"根据前缀的统计规律续写"能力,在推理时被当成"根据示例做类比"来用。

掩码 LM 没有这个能力:它的训练目标是"填空",不存在"用前文示例引导后文生成"这种结构,所以 BERT 不能 few-shot prompting。**指令微调(instruction tuning)和 RLHF 也都建立在续写之上**——"指令→回答"就是一个前缀-续写对,RLHF 优化的是续写策略。**整个"prompt → 模型续写 → 对齐人类偏好"的现代 LLM 范式,地基就是 AR 的续写行为。** 这是 AR 赢的最关键一条:它通向了一个掩码 LM 根本到不了的能力空间。

### 6.3 可生成性:刚需,而双向无法直接提供

如前(4.3)所证,掩码 LM 不能直接生成完整序列。而**"生成"是绝大多数有商业价值的任务的核心**:写作、对话、代码、翻译、摘要,本质都是生成。一个不能生成的模型,无论理解力多强,都无法独立完成这些任务,只能当别的系统的特征提取器。**当目标从"理解一句话"转向"生成有用的内容",不能生成就是出局条件。** 去噪 seq2seq 虽然能生成,但需要"被破坏的输入"或明确的输入序列作为条件,在"从一个开放式 prompt 自由生成"这件事上不如 decoder-only AR 自然。

### 6.4 scaling 友好:目标简单、信号密集、无结构特异性

最后一条是工程与经验的:**AR 目标在 scaling(扩大模型和数据)时表现最干净。**

- **信号密度高(定量)**:第三节算过,AR 每个 token 都贡献 loss,长度 T 的序列产 T 个监督信号。MLM 只有约 15% 的 token 贡献 loss。**在相同的"过 token 数"下,AR 提供的预测任务数约是 MLM 的 1/0.15 ≈ 6.7 倍。** 换句话说,处理同样多的原始 token,AR 从中榨取的学习信号密集得多,这让它在给定算力下数据效率更高(**注:这是一阶近似论证;精确的样本效率还受任务难度、表征质量影响,但信号密度差异是主因之一**)。
- **目标无结构特异性**:AR 只需 decoder-only + 因果掩码,没有 encoder-decoder 的跨注意力、没有 [MASK] 比例和 80/10/10 这类需要调的超参。结构越简单,越容易 scale、越容易在不同规模上保持稳定。
- **scaling law 经验**:decoder-only AR 的 loss 随参数/数据/算力的幂律下降关系被反复验证(GPT-3、Chinchilla 等),且外推性好——这给了"无脑加大就更强"的信心,而双向/去噪目标的 scaling 行为研究得没这么充分(**待核**:不同范式 scaling law 的系统对比仍是开放问题)。

### 6.5 那"单向不如双向"的代价去哪了?

诚实地说,**单向确实有代价**:预测位置 t 时看不到 t 之后的 token,纯粹做表征(给定整句话编码每个词)时,AR 的表征理论上不如双向充分,这也是 BERT 当年在 NLU 榜单领先的原因。**但这个代价在大模型尺度下被三件事抵消:**

1. **规模补偿**:当模型足够大、数据足够多,单向模型也能学到极强的上下文表征——少看一侧的信息损失,被海量参数和数据带来的统计规律弥补掉大半。GPT-3 之后,AR 模型在很多 NLU 任务上已能用 few-shot 追平甚至超过专门微调的 BERT。
2. **用途变了**:大模型时代的核心需求是**生成和通用交互**,不是给定完整句子做编码打分。在"生成"这个主战场上,双向的表征优势根本用不上(生成时本来就没有"未来 token"可看,因果约束是生成任务的内在要求,不是缺陷)。
3. **通用性碾压专用性**:一个能 zero/few-shot 干上千种任务的 AR 模型,价值远高于一个 NLU 榜单高几分但每个任务都要单独微调、且不能生成的 BERT。**"一个模型干所有事"的杠杆,压倒了"单个理解任务上高几分"的局部优势。**

**一句话总结这场胜负:不是 AR 的表征比双向好,而是 AR 选对了"生成+统一+可 scaling"这条能通向通用智能的路,而双向选了一条天花板更低(困在理解、不能生成、不能 ICL)的路。范式之争的胜负手是"能力上限"和"通用性",不是"单个理解任务的精度"。**

## 七、训练机制深挖:teacher forcing 与 exposure bias

AR 训练里有一对必须吃透的概念,它们是后续推理、采样、甚至 RLHF 动机的根。

### 7.1 teacher forcing:训练时喂真实前缀

回看 AR 的损失 `Σ_t log p_θ(xₜ|x_{<t})`:计算位置 t 的预测时,条件 `x_{<t}` 用的是**真实(ground-truth)前缀**,而不是模型自己上一步预测出来的 token。**这种"训练时始终喂真实历史"的做法叫 teacher forcing(教师强制)**——好比写作文时,每写一个词老师都先把前面正确的内容告诉你,你只需在正确前缀上接下一个词。

teacher forcing 之所以是默认选择,是因为它**让 3.2 节的并行成为可能**:既然每个位置的条件都是固定的真实前缀,各位置的预测就互不依赖,可以靠因果掩码一次前向全部并行算出。如果训练时也用模型自己生成的 token 当前缀(下一步依赖上一步的实际输出),就退化成串行,且每个 batch 的计算图随机,无法高效训练。**teacher forcing = 训练并行性的前提。**

### 7.2 exposure bias:训练和推理的分布错位

teacher forcing 埋了一个隐患。**训练时**,模型在每一步看到的前缀都是**真实的、高质量的**;但**推理(自由生成)时**,没有真实答案可喂,模型只能把**自己上一步生成的 token** 接到前缀里,作为下一步的条件:

```
训练(teacher forcing):  预测 xₜ 的条件 = (真实 x₁, 真实 x₂, …, 真实 x_{t-1})
推理(自由生成):         预测 x̂ₜ 的条件 = (生成 x̂₁, 生成 x̂₂, …, 生成 x̂_{t-1})
```

问题来了:模型**只在"真实前缀"这个分布上被训练过,从没见过"自己生成的、可能含错误的前缀"这个分布。** 这种"训练时暴露的输入分布"和"推理时实际遭遇的输入分布"之间的不一致,就是 **exposure bias(暴露偏差)**。

它的后果是**误差累积(error accumulation)**:推理时一旦某一步采到一个不那么好的 token,这个 token 就进入了后续所有步骤的条件——而这个"含错误的前缀"是训练分布里没有的,模型在它上面的预测更不可靠,更容易再出错,错误像滚雪球一样放大。直觉模型:

```
设每步"进入一个未见过/异常状态"的概率为 ε(很小),且一旦进入就难自我纠正。
则生成 T 步后仍"全程留在训练见过的良性分布内"的概率 ≈ (1-ε)^T,随 T 增大指数衰减。
→ 序列越长,越可能漂移到训练分布之外,这是长文本生成质量下降、跑题、重复的一个理论来源。
```

(这是一个**直觉性的上界论证**,真实情况下各步并非独立、模型也有一定自我纠正能力,所以不会真的指数崩溃;但"长序列更易漂移出训练分布"这个定性结论是稳健的。)

**理解 exposure bias 极其重要,因为它是好几个后续主题的共同动机:**

- **为什么需要采样策略(temperature、top-p、top-k)**:贪心解码容易陷入退化(重复、空洞),合理的随机采样在一定程度上缓解分布漂移;但采样本身也可能采到坏 token,是把双刃剑。
- **为什么 RLHF / RL 微调有用**:RL 训练时模型用的是**自己生成的序列**(on-policy),并对整条生成序列的好坏给反馈——这等于让模型在"自己生成的分布"上学习,**直接对症 exposure bias**(让训练分布和推理分布对齐)。这是从纯 MLE 预训练走向 RL 后训练的一条根本理由,你在 RLHF 章节会反复见到。
- **历史上的缓解尝试**:scheduled sampling(训练时按概率混入模型自己的预测当前缀,在 teacher forcing 和 free-running 之间退火)等方法,本质都是想缩小这个 train/test 分布差。

**记住这个因果链:teacher forcing 换来了训练并行性,代价是 exposure bias;而 RL 后训练之所以能锦上添花,很大程度正是因为它在"模型自己的生成分布"上优化,补上了 MLE 预训练留下的这个缺口。**

## 八、设计权衡与常见坑

**坑 1:shift 错位/不 shift,导致信息泄漏。** 忘记把 label 右移一位、或 shift 方向反了,会让位置 t 的输出能"看到"它要预测的 token,loss 异常低(接近 0)却毫无泛化。**自检方法:一个没训练的随机初始化模型,正确实现下 loss 应约等于 log|V|**(均匀分布的交叉熵);如果初始 loss 远低于这个值,几乎肯定是泄漏。

**坑 2:把因果掩码当成"乘 0"而非"加 -inf"。** 掩码作用在 softmax **之前**的 logits 上,要加 -inf(经 exp 变 0);若在 softmax **之后**乘 0,会破坏概率归一化(剩余权重和不为 1),数值与梯度都错。

**坑 3:误以为掩码 LM 能稍加改造就生成。** MLM 没有"逐步续写"的训练信号,硬要它生成需要 Gibbs 采样式的迭代填空,质量和效率都远不如 AR。**别在 BERT 上做生成,选错工具。**

**坑 4:把 MLM 的 15% 当成不可改的魔法常数。** 后续研究(如 RoBERTa 的动态掩码、以及一些更大掩码率的工作)表明最优掩码率与模型规模、数据量有关,**15% 是 BERT 当时的经验最优,不是普适定律**(**待核**:更大模型下更高掩码率有时更好,具体随设置而变)。

**坑 5:忽视 exposure bias,纯靠加大 MLE 预训练想解决生成跑题/重复。** MLE + teacher forcing 有结构性的 train/test 分布错位,加数据能缓解但难根除;长序列生成质量问题往往要靠**解码策略 + RL 后训练**这套组合拳,而非单纯堆预训练。

**坑 6:在 decoder-only 上做分类还硬接双向。** decoder-only 模型做判别任务时,常见做法是取最后一个 token 的隐状态或用 prompt 让它"续写"出标签;**强行去掉因果掩码改成双向会破坏与预训练的一致性**,通常得不偿失。要双向理解,该选/微调一个 encoder 模型。

**权衡总览:三大范式不是"谁绝对强",而是占据"理解力 ↔ 生成力 / 专用 ↔ 通用"光谱的不同位置。** 若任务是纯判别(分类/检索/抽取)且数据可微调,encoder 双向仍是性价比之选;若有明确输入序列的条件生成(翻译/摘要),encoder-decoder 仍有结构优势;**但若目标是通用、可交互、可 few-shot 的生成式智能,decoder-only AR 是唯一通向那里的范式。** 工具要按任务选,而大模型时代的主线任务恰好让 AR 成为默认。

## 九、动手练习

**练习 1(构造题,必做):为同一任务写出三种范式的输入输出。** 取"情感分类:判断 'this movie is great' 是正面还是负面"这个任务,分别写出:(a) 自回归 LM 怎么把它构造成续写(给出 prompt 和期望续写);(b) 掩码 LM 怎么把它构造成填空(给出带 [MASK] 的输入和被预测目标,以及怎么读出分类结果);(c) T5 text-to-text 怎么构造(给出带任务前缀的输入序列和目标序列)。*提示:AR 用 "...is great. Sentiment:" 让模型续写 "positive";MLM 把标签词位置设成 [MASK] 比较 "positive"/"negative" 哪个概率高;T5 输入 "sst2 sentence: this movie is great" 输出 "positive"。想清楚每种范式"标签从哪来、怎么读出"。*

## 十、源码 / 论文导读

**论文(按推荐阅读顺序):**
- **Radford et al., 2018, "Improving Language Understanding by Generative Pre-Training"(GPT-1)** + **Radford et al., 2019, "Language Models are Unsupervised Multitask Learners"(GPT-2)** —— AR 路线的起点。重点读 GPT-2 里"语言模型是无监督多任务学习器"的论证——**它第一次明确提出"一切任务都能表达成续写 p(output|input,task)"**,这是本章第六节"任务统一性"的原始出处。
- **Devlin et al., 2019, "BERT: Pre-training of Deep Bidirectional Transformers for Language Understanding"** —— 掩码 LM 的奠基作。**精读第 3.1 节 "Masked LM"**:15% 掩码率、80/10/10 拆分的原始动机就写在这里(以及那段关于 [MASK] 在微调时不出现导致 mismatch 的讨论)。这是理解本章第四节最权威的一手材料。
- **Raffel et al., 2020, "Exploring the Limits of Transfer Learning with a Unified Text-to-Text Transformer"(T5)** —— **读它的核心贡献"text-to-text 统一框架",以及它对各种预训练目标(BERT 式、prefix-LM、去噪、不同掩码率/span 长度)做的大规模系统对比实验那几节**——这是少有的把"不同范式正面 PK"做成 controlled experiment 的工作,直接支撑本章第五、六节的论证。
- **Lewis et al., 2020, "BART"** —— 去噪自编码 + seq2seq,看它列举的多种 noising 方式(token masking / deletion / text infilling / sentence permutation / document rotation)如何对应不同的重建难度。
- **Brown et al., 2020, "Language Models are Few-Shot Learners"(GPT-3)** —— **重点读 in-context learning(few-shot prompting)那部分**:它实证了 AR 模型仅凭 prompt 示例就能做新任务,是本章第六节"AR 与 ICL 天然契合"最有力的经验证据。
- **(进阶)Tay et al., 2022/2023, "UL2: Unifying Language Learning Paradigms"** —— 把多种预训练目标(不同去噪模式 + prefix-LM)统一在一个训练方案里,适合读完本章后回看"范式能不能融合"这个开放问题。

**开源实现:**
- **nanoGPT(Karpathy)** —— **最该读的入门实现**。看 `model.py` 里因果掩码怎么构造(`torch.tril` 或 `is_causal=True`)、`train.py` 里 loss 怎么算(取 `logits[:, :-1]` 和 `targets[:, 1:]` 做 cross-entropy,和本章 3.4 代码逐行对应)。**整个 AR 训练目标在 nanoGPT 里就是几十行可读代码,强烈建议照着跑一遍并做练习 4。**
- **HuggingFace transformers** —— 对照看三类模型的 head:`GPT2LMHeadModel`(AR,内部就是 shift + CrossEntropyLoss,`labels` 传进去会自动右移)、`BertForMaskedLM`(MLM,只对 [MASK] 位置算 loss)、`T5ForConditionalGeneration`(seq2seq)。**读 `GPT2LMHeadModel.forward` 里 `shift_logits / shift_labels` 那几行,会发现和本章代码一模一样**;再看 `DataCollatorForLanguageModeling`,15% 掩码和 80/10/10 拆分就在这个 collator 里(**待核**:具体类名/参数随版本变化,以你用的版本为准)。
- **x-transformers(lucidrains)** —— 想看 prefix-LM、各种注意力掩码变体的简洁实现,看它的 attention mask 相关参数,能直观体会"单向 vs 双向 vs 前缀双向只是掩码形状不同"。

**阅读策略:先跑 nanoGPT 把 AR 目标落到能改的代码,再读 BERT 3.1 节和 T5 的目标对比实验建立"为什么是 AR"的判断,最后用 GPT-3 的 ICL 那节把这条逻辑链闭环。** 三者读完,你对"预训练目标决定模型能力上限"会有刻进直觉的理解。

## 十一、小结与承上启下

这一章我们把"模型到底学什么"这个总开关拆透了:

1. **三大范式的数学本质是"如何分解/重建联合概率 p(x)"的不同选择**:AR 严格从左到右精确分解 `∏ p(xₜ|x_{<t})`;MLM 用双向上下文做有条件独立近似的填空 `p(x_M|x_{\M})`;去噪 seq2seq 是破坏-重建 `p(y|x̃)`。prefix-LM 则用注意力掩码形状在单向与双向之间连续插值。
2. **AR 的训练机制**:shift(标签右移一位)+ 因果掩码(-inf 屏蔽未来),让一次前向并行算出全部 T 个位置的 loss——把串行的概率结构翻译成可并行的矩阵运算,这是 AR 高效 scaling 的工程命门。
3. **生成式自回归赢的四条结构性理由**:任务统一性(一切皆续写,训练即最终形态)、与 ICL/指令对齐天然契合(通向掩码 LM 到不了的能力空间)、可生成性(刚需,双向无法直接提供)、scaling 友好(信号密度高约 6.7×、目标无结构特异性)。**胜负手是能力上限和通用性,而非单个理解任务的精度——"单向不如双向"的表征代价被规模补偿、用途转向生成、通用性碾压专用性这三件事盖过。**
4. **teacher forcing ↔ exposure bias 这对因果**:前者换来训练并行性,代价是 train/test 分布错位与长序列误差累积——而这正是解码策略和 RL 后训练存在的根本理由之一。

**这一章在整门课里的位置**:它承接前面的 Transformer 结构章(结构是容器,目标是灵魂),向后则是整条主线的地基。**因为预训练目标选定了 decoder-only AR,后面所有内容才成立**:tokenizer 与词表(决定 |V| 和 softmax 那一项)、scaling law(AR loss 的幂律外推)、指令微调(prompt→completion 就是前缀-续写对)、in-context learning(prompt 示例约束续写)、以及 RLHF/RL 后训练(在模型自己生成的分布上优化,对症 exposure bias)。**你在本章建立的"一切皆续写、teacher forcing 留下 exposure bias 缺口"这两条主轴,会一路贯穿到课程后半的对齐与推理训练**——到那时你会清楚地看到:RLHF 不是凭空冒出来的新东西,而是 MLE 预训练这套范式留下的缺口必然要被补上的下一块拼图。
