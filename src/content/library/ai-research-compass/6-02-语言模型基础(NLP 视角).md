> 这一章把你从"知道语言模型是给序列算概率"带到"能亲手用计数估出一个 n-gram 模型、能从第一性原理说清它为什么必然在真实语料上崩(数据稀疏 → 零概率),能推导并实现加 k 平滑/回退/插值,能讲透 Kneser-Ney 的接续多样性直觉为什么对罕见词更对,能手算困惑度,并能解释 Bengio 2003 用词嵌入 + MLP 是如何一举绕开稀疏诅咒、从而接上整个预训练时代"。这是 NLP 视角的语言模型地基:它和算法课 00 章互补——**那一章从信息论讲透"交叉熵=最大似然=KL=无损压缩"这条等价链,本章不重复推导,只引用结论;本章的重心是统计 NLP 特有的东西:计数估计、稀疏、平滑、回退、PPL 作为评测工具,以及神经 LM 为什么是稀疏问题的解药。**

## 一、动机:语言模型到底要解决什么,以及为什么"数数"是第一个答案

先把对象钉死。一个**语言模型(language model, LM)** 就是一个对 token 序列赋予概率的函数:给一段序列 w = (w₁, w₂, …, w_T),它要回答 `P(w)` 是多少。一个好的 LM 给"今天天气不错"高概率,给"今天天气矩阵积分"低概率。

> 给有工程数学基础、NLP 从零的读者:你可以把 LM 完全类比成"对离散序列的密度估计"。连续数据你会拟合高斯/混合模型;这里数据是离散符号序列,你要拟合的是一个定义在 `V^T`(V 是词表,T 是长度)这个天文级离散空间上的分布。整章的张力都来自一件事——**这个空间太大,你永远见不全,所以纯靠数数(最大似然)必然在没见过的地方给出零概率,而零概率对一个概率模型是灾难性的。** 平滑(smoothing)就是统计 NLP 几十年里专门对付这件事的技术,Kneser-Ney 是其巅峰;神经 LM 则换了一条路彻底绕开它。

为什么是这章的对象?因为**语言模型是 NLP 的中心任务**,这一点本章末尾会展开论证(自监督、可迁移)。但历史上,在没有神经网络、没有大算力的年代,人们解决"给序列算概率"的第一个、也是统治了 NLP 二三十年的答案,就是最朴素的一招:**数数**。统计每种序列在大语料里出现了多少次,用频率当概率。本章就从这一招的成功与失败讲起。

### 1.1 第一性原理:自回归分解——把"给整段算概率"拆成"逐词预测"

和算法课 00 章一样,起点是概率论的链式法则(乘法公式)。它是**恒等式,不是近似**:

```
P(w₁, w₂, …, w_T)
  = P(w₁) · P(w₂|w₁) · P(w₃|w₁,w₂) · … · P(w_T | w₁,…,w_{T-1})
  = ∏_{t=1}^{T} P(wₜ | w_<t)        其中 w_<t = (w₁,…,w_{t-1})
```

这叫**自回归分解**。它把"建模 `V^T` 上的联合分布"这个不可能任务,精确等价转化成"对每个位置 t,给定全部历史 w_<t,预测当前词的条件分布 `P(wₜ|w_<t)`"。注意此刻还没有任何近似——这只是把一个大问题改写成一串小问题。

但小问题里仍藏着原来的困难:条件 `w_<t` 的长度随 t 增长,历史的可能取值数也是指数级。`P(wₜ|w_<t)` 需要为**每一种可能的历史**都存一个 V 上的分布,你根本数不过来。于是出现了第一次、也是决定性的一次**近似**——马尔可夫假设。

## 二、n-gram:用马尔可夫假设把历史截断成定长

### 2.1 马尔可夫假设:只看前 n−1 个词

**n-gram 模型的全部假设浓缩成一句话:当前词只依赖前面有限的 n−1 个词,更早的历史一概忽略。** 形式化:

```
P(wₜ | w_<t)  ≈  P(wₜ | w_{t-n+1}, …, w_{t-1})
```

这就是 (n−1) 阶**马尔可夫假设**。代入自回归分解:

```
P(w) ≈ ∏_{t=1}^{T} P(wₜ | w_{t-n+1:t-1})
```

- n=1,**unigram**:`P(wₜ)`,词与词独立,纯词频。
- n=2,**bigram**:`P(wₜ|w_{t-1})`,只看前一个词。
- n=3,**trigram**:`P(wₜ|w_{t-2}, w_{t-1})`,看前两个词。工业界经典语音/输入法长期主力。

**为什么必须截断?** 因为要存的参数量是 `|V|^n` 量级(n-gram 表的大小)。bigram 是 `|V|²`,trigram 是 `|V|³`。词表 V=5 万时,trigram 理论上 1.25×10¹⁴ 个条目——已经远超任何语料能见到的不同 n-gram 数(实际只有极小一部分非零)。再往上 n=4、5,参数空间爆炸,见过的比例急剧趋零。**n 是一个偏差-方差权衡的旋钮:n 大,条件更长、模型更有"上下文",但见过的 n-gram 比例骤降、估计方差爆炸;n 小,见得多、估计稳,但丢上下文、偏差大。** 这个权衡是 n-gram 时代一切技术(尤其平滑与回退)的根源。

### 2.2 最大似然估计:概率就是计数比

定下了 n,怎么估 `P(wₜ|w_{t-n+1:t-1})`?用**最大似然估计(MLE)**。结论极简单:**条件概率 = n-gram 计数 ÷ 前缀 (n−1)-gram 计数**。以 bigram 为例:

```
P_MLE(wₜ | w_{t-1}) = count(w_{t-1}, wₜ) / count(w_{t-1})
```

count(a,b) 是 bigram "a b" 在语料里出现的次数,count(a) 是 unigram "a" 的次数(等价于所有以 a 开头的 bigram 计数之和)。

**这个"计数比"不是拍脑袋,而是 MLE 的解析解,可以推。** 设语料是一串 token,把它看成多项分布的采样。固定上文 a,所有"a 后面跟什么"构成一个 V 类多项分布,参数 θ_v = P(v|a)。在语料里,上文是 a 的位置共出现 N_a = count(a) 次,其中下文为 v 的有 c_v = count(a,v) 次(Σ_v c_v = N_a)。在 i.i.d. 多项假设下,这部分数据的对数似然:

```
ℓ(θ) = Σ_{v∈V} c_v · log θ_v ,   约束 Σ_v θ_v = 1, θ_v ≥ 0
```

用拉格朗日乘子求条件极值:

```
L = Σ_v c_v log θ_v + λ (1 − Σ_v θ_v)
∂L/∂θ_v = c_v/θ_v − λ = 0      ⟹   θ_v = c_v / λ
代入约束 Σ_v θ_v = 1:  Σ_v c_v/λ = 1  ⟹  λ = Σ_v c_v = N_a
所以  θ_v = c_v / N_a = count(a,v) / count(a)         ∎
```

**所以"用频率当概率"严格就是该多项模型的最大似然解。** 这个推导对任意 n 同理(把上文 a 换成 (n−1)-gram)。记住这条:**n-gram MLE = 计数比,是 MLE 严格解,不是近似。** 它唯一的问题不在"对不对",而在"见没见过"。

### 2.3 致命问题:数据稀疏 → 零概率 → 整段似然崩塌

MLE 的灾难在分子上:**只要某个 n-gram 在训练语料里没出现过(count=0),它的 MLE 概率就是 0。** 而由自回归连乘:

```
P(w) = ∏ₜ P(wₜ|·)
```

**任何一个因子为 0,整段序列概率就是 0,对数似然是 −∞,困惑度是 +∞。** 这就是**数据稀疏(data sparsity)** 问题:语言的组合是长尾的,绝大多数语法完全合法、语义完全正常的 n-gram,你的训练语料根本没见过。一句全新的、完全通顺的句子,只要含一个没见过的 trigram,模型就判它"绝不可能出现"——这显然荒谬。

**量化一下稀疏有多严重。** 这是齐普夫定律(Zipf's law)的直接后果:词频按排名近似服从 `频率 ∝ 1/排名`,长尾极长。对 bigram、trigram,组合后长尾更夸张。一个经验事实(Brown 等早期实验,具体数值随语料而定):即使在上千万词的语料上训练 trigram,测试集里仍有相当比例的 trigram 是训练时**从未见过**的。**模型见过的 n-gram 永远是冰山一角,且 n 越大冰山越小。** 这不是数据不够的偶然问题,而是语言组合空间无限、语料有限的必然问题——再加十倍数据,新见到的句子里照样有没见过的 n-gram。

所以,**平滑不是可选的优化,是 n-gram LM 能不能用的生死线。** 它的核心思想统一成一句话:**从"见过的"事件那里挪一点概率质量,匀给"没见过的"事件,让任何 n-gram 都不至于是绝对的 0。** 这在统计上叫"折扣(discounting)"。下面把主流平滑法从最笨到最强讲一遍。

## 三、平滑(一):加 k 与折扣——最直觉但最粗的一类

### 3.1 加一平滑(Laplace):假装每个都多见了一次

最朴素:估计前,假装词表里每个可能的下文都已经被见过 1 次。bigram 形式:

```
P_Laplace(wₜ | w_{t-1}) = (count(w_{t-1}, wₜ) + 1) / (count(w_{t-1}) + |V|)
```

分子 +1,分母 +|V|(因为每个 v∈V 都加了 1,共加 |V|)。**为什么分母加 |V| 而不是别的?为了让分布仍归一化**——验证:`Σ_v (count(a,v)+1) = count(a) + |V|`,正好是分母,所以 `Σ_v P_Laplace(v|a) = 1`。✓ 这是检验任何平滑公式是否合法的硬标准:**归一化必须成立,否则它不是概率。**

加一的贝叶斯解释也很干净:它等价于给多项分布加一个**对称 Dirichlet(α=1) 先验**做 MAP 估计——"先验认为每个下文都微弱可能"。

**但加一在 NLP 里几乎总是太猛了。** 直觉:词表 V 动辄几万,某个上文 a 只出现了 10 次,你却"凭空"给全词表每个词都注水 1 次、共注 |V|≈5 万次质量进去——真实的 10 次观测被这 5 万次虚构观测彻底淹没。结果是**把大量概率质量从见过的 n-gram 抽走,过度分配给海量没见过的 n-gram**,严重低估了见过的、高估了没见过的。所以实践里退一步:

### 3.2 加 k 平滑(Lidstone):把注水量调小

把 1 换成一个小常数 k(0<k<1,如 0.01):

```
P_{add-k}(wₜ | w_{t-1}) = (count(w_{t-1}, wₜ) + k) / (count(w_{t-1}) + k·|V|)
```

归一化同样成立(分母里 `k·|V|` 与分子每项 +k 对应)。k 是超参,用**留出集(held-out set)** 调,使留出集 PPL 最低。k→0 退回 MLE(稀疏崩),k=1 是 Laplace(注水过度),好的 k 通常很小。

**加 k 的根本缺陷(理解它才能理解为什么需要更聪明的方法):它对所有没见过的 n-gram 一视同仁,都给同样的概率。** 但"美国 总统"没见过和"美国 薛定谔"没见过,合理性天差地别——前者只是语料没覆盖,后者本就罕见。加 k 完全无视这种差异。这正是后面回退/插值/Kneser-Ney 要解决的。

### 3.3 Good-Turing:用"只出现一次的事件"估计"没出现的事件"(思想)

加 k 注水量是拍的;**Good-Turing 折扣**给出一个有理论依据的折扣量。核心思想极漂亮:**用语料里出现 r+1 次的 n-gram 的数量,去重估出现 r 次的 n-gram 的"真实"期望频次。** 记 N_r 为"恰好出现 r 次的不同 n-gram 的个数",则把出现 r 次的事件的折扣后计数定为:

```
r* = (r+1) · N_{r+1} / N_r
```

特别地,**没出现过的事件(r=0)** 总质量被估为 `N_1 / N`(N 是总 token 数)——也就是说,**"只出现一次的事件有多少",直接告诉你"没出现的事件大概占多少质量"**(直觉:见过一次的东西越多,说明长尾越厚,没见过的也越多)。这就把"留多少质量给 unseen"从超参变成了由数据自身估出来的量。Good-Turing 是后续 Katz 回退的折扣基础;本章不展开它的全部推导(涉及缺失质量的偏差修正),你记住这个**"用 N₁ 估 unseen 质量"** 的核心思想即可。

## 四、平滑(二):回退与插值——按需借用低阶信息

加 k/Good-Turing 都在"同一阶"内分配质量。更聪明的思路是:**高阶 n-gram 没见过时,退而求其次,借用低阶(更短、见得多)的 n-gram 的信息。** 这就是回退与插值——它们用**不同阶模型的层级关系**来对抗稀疏,是从"孤立计数"走向"层级估计"的关键一步。

### 4.1 回退(Katz back-off):见过就用高阶,没见过才退

**回退(back-off)** 的逻辑像 if-else:trigram 见过就用 trigram;没见过,退回 bigram;bigram 还没见过,退回 unigram。形式(以三阶为例,记上文 c = (w_{t-2}, w_{t-1})):

```
P_katz(wₜ | c) =
   ┌ d · P_MLE(wₜ | c)              若 count(c, wₜ) > 0      （见过:用折扣后的高阶）
   └ α(c) · P_katz(wₜ | w_{t-1})    若 count(c, wₜ) = 0      （没见过:退到低阶，乘回退权重）
```

- d 是折扣系数(<1，常用 Good-Turing 给),把见过的 trigram 概率压低一点,**腾出**总质量 `1 − Σ_{见过} d·P_MLE`。
- α(c) 是**回退权重**,把腾出的质量按低阶 bigram 分布,分给那些 trigram 没见过的下文。α(c) 由"归一化必须成立"唯一确定:腾出多少质量,就乘多大的 α 让退下去的部分恰好补满,使 `Σ_v P_katz(v|c)=1`。

**关键性质:回退是"非黑即白"——高阶见过就完全不看低阶,只在高阶为 0 时才退。** 这是它和插值的根本区别。

### 4.2 插值(interpolation):各阶按权重恒混合

**插值(interpolation)** 不做 if-else,而是**永远把各阶混在一起**,无论高阶见没见过:

```
P_interp(wₜ | w_{t-2},w_{t-1}) =
      λ₃ · P_MLE(wₜ | w_{t-2}, w_{t-1})
    + λ₂ · P_MLE(wₜ | w_{t-1})
    + λ₁ · P_MLE(wₜ)
   其中 λ₁+λ₂+λ₃ = 1,  λᵢ ≥ 0
```

权重 λ 用留出集学(可用 EM,或简单网格搜),也可让 λ 依赖上文计数(上文见得多就更信高阶)。

**回退 vs 插值的取舍:**
- **回退**信"只要高阶见过,高阶就是更可信的",低阶只在高阶完全失效时兜底。理论更干净(配 Good-Turing),但实现要小心算 α(c) 的归一化。
- **插值**永远让低阶参与,实现简单、鲁棒,即使高阶见过,低阶也提供平滑。**现代 Kneser-Ney 用的是插值形式**(见下),实践中插值往往更好调、更稳。
- 两者都体现同一信念:**低阶 n-gram 是高阶稀疏时的"后备知识源"。** 差别只在"什么时候、以多大权重"用它。

## 五、平滑(三):Kneser-Ney——为什么"接续多样性"比频次更对

Kneser-Ney(KN)是 n-gram 平滑的**集大成者与事实上的最优**(Chen & Goodman 1998 的大规模实证里,interpolated/modified KN 几乎在所有设置下 PPL 最低)。它的精髓只有一句话,但极其深刻——**低阶分布不该用"词出现了多少次"来估,而该用"这个词跟在多少种不同上文后面出现过"来估。** 这就是**接续多样性(continuation diversity)**。下面把这个反直觉的设计讲透,因为它是整个 n-gram 平滑历史的智力高点。

### 5.1 问题:用 unigram 频次做回退,会被高频搭配词坑

经典反例(教科书级):考虑词 **"Francisco"**。它在语料里频次可能很高——但**几乎只出现在 "San Francisco" 里**,前面永远是 "San"。现在 bigram "reading Francisco" 没见过,要回退到 unigram `P(Francisco)`。如果 unigram 用普通频次估,`P(Francisco)` 会很高(因为 "Francisco" 总词频高),于是模型会觉得"reading 后面接 Francisco"挺合理——**这是错的**,Francisco 几乎不能自由地跟在任意词后面,它只配跟 "San"。

**症结:回退/低阶分布回答的问题,不是"这个词常不常见",而是"这个词作为一个全新上文的接续,有多自由、多百搭"。** "Francisco" 词频高但**接续多样性极低**(只跟一种上文),不该在新上文里被给高回退概率。普通 unigram 频次混淆了这两件事。

### 5.2 解法:把低阶概率改成"接续计数"

KN 把低阶(回退)分布的估计,从"频次"改成"**这个词出现在多少种不同上文之后**"的计数,叫**接续计数(continuation count)**:

```
N₁₊(• w) = |{ w' : count(w', w) > 0 }|
         = 词 w 前面出现过的不同词 w' 的种类数（"• "表示对前驱求种类）
```

然后**接续概率(continuation probability)** 正比于它:

```
P_continuation(w) = N₁₊(• w) / Σ_{w''} N₁₊(• w'')
                  = (w 跟过的不同上文数) / (所有词跟过的不同上文数之和)
```

回到反例:`N₁₊(• Francisco)` ≈ 1(只跟过 "San"),哪怕它总频次很高,接续概率也极低;而像 "time"、"day" 这种能跟在成百上千种词后面的词,接续概率高。**这恰好就是"作为一个新上下文的接续,这个词有多百搭"——正是回退时该问的问题。** KN 用接续计数替代频次,就修好了 Francisco 这类病。

> 为什么这对罕见词尤其好(直接回答练习):罕见词的频次本就小、噪声大,用频次估低阶极不稳;而**接续多样性是个更稳健、更贴合"可组合性"的信号**。一个罕见但用法灵活的词(能接在多种上文后)会得到合理的回退概率;一个看似不罕见但只在固定搭配里出现的词(如 Francisco)则被正确压低。KN 用"种类数"而非"次数",把估计建立在**结构多样性**而非**绝对频次**上,所以对长尾/罕见情形更稳、更对。

### 5.3 完整公式:absolute discounting + 接续概率(插值形式)

KN 的高阶部分用**绝对折扣(absolute discounting)**:从每个见过的 n-gram 计数里**减去一个固定常数 D**(0<D<1,经验上 D 与 N₁/(N₁+2N₂) 相关),腾出质量按**接续概率**插值低阶。bigram 形式:

```
P_KN(wₜ | w_{t-1}) = max(count(w_{t-1}, wₜ) − D, 0) / count(w_{t-1})
                   + λ(w_{t-1}) · P_continuation(wₜ)

其中  λ(w_{t-1}) = (D / count(w_{t-1})) · N₁₊(w_{t-1} •)
      N₁₊(w_{t-1} •) = 词 w_{t-1} 后面跟过的不同词的种类数（这次对后继求种类）
```

逐项拆解,确保你能复现:
- `max(count−D, 0)/count(w_{t-1})`:高阶 MLE 减去绝对折扣 D 后的概率(减一个常数,而不是按比例缩,这就是"绝对"折扣;对高频低频一视同仁地各减 D)。
- `λ(w_{t-1})`:腾出的总质量。`N₁₊(w_{t-1} •)` 是 w_{t-1} 后面有多少种不同的词见过(每种贡献了一份被减掉的 D),所以腾出量 = `D × 种类数 / count`。
- `P_continuation(wₜ)`:把腾出的质量按**接续概率**(而非普通频次)分配——这是 KN 的灵魂。
- **归一化成立**:腾走的总量 `Σ_v D·[count(w_{t-1},v)>0] / count(w_{t-1})` 恰等于 `λ(w_{t-1})`,而 `Σ_v P_continuation(v)=1`,所以补回来恰好是 λ,总和为 1。✓

**modified KN**(Chen & Goodman)进一步对计数为 1、2、≥3 的 n-gram 用三个不同折扣 D₁、D₂、D₃₊,而非单一 D,贡献了又一点 PPL。这是神经 LM 之前 n-gram 平滑的天花板。

### 5.4 一句话收束这条平滑线

从加一到 KN,是一条清晰的认知升级:**加 k**——盲目均匀注水;**Good-Turing**——用 N₁ 估 unseen 质量;**回退/插值**——稀疏时借低阶;**Kneser-Ney**——借低阶时,低阶概率要用"接续多样性"而非"频次"。**每一步都在更精细地回答同一个问题:"没见过的、或见得少的 n-gram,概率到底该给多少。"** 这就是统计 NLP 与稀疏搏斗的全部智慧结晶。

## 六、困惑度:n-gram 时代的评测标尺(NLP 视角)

困惑度(perplexity, PPL)的**信息论身份**(=exp(交叉熵)=几何平均概率的倒数=与 KL/压缩的等价)在算法课 00 章已推透,**这里不重复**。本节只补 NLP 视角下用它评测 n-gram 模型时的实操与坑。

定义重述(就是平均负对数似然取指数):

```
PPL(w) = exp( − (1/T) Σ_{t=1}^{T} ln P(wₜ | 历史) )
       = ( ∏ₜ P(wₜ | 历史) )^(−1/T)        ← 每步条件概率几何平均的倒数
```

**直觉(沿用 00 章结论):PPL = 模型每步平均相当于在多少个等概率选项里犹豫。** PPL 越低越好。在 n-gram 语境下还要特别注意:

- **PPL 必须在独立的测试集上算,且测试集要用平滑后的概率。** 没平滑时,测试集一旦含训练未见的 n-gram,某个因子为 0,PPL=∞——这恰恰是用 PPL 暴露"没做平滑"的方式。
- **比较平滑方法,就是比同一测试集上谁的 PPL 低。** 这是当年所有平滑论文(包括 Chen & Goodman)的实验范式:固定语料、固定 n、固定测试集,横扫各平滑法看 PPL。
- **跨模型比 PPL 必须 tokenizer 与语料完全一致**(00 章的坑,这里强调:n-gram 时代还要管 `<UNK>`(未登录词)的处理是否一致——把 OOV 都映射成 `<UNK>` 会人为压低 PPL,不同 OOV 策略的 PPL 不可比)。
- **句首/句尾标记 `<s>`/`</s>` 要算进去且约定一致**:bigram 第一个词要用 `P(w₁|<s>)`,句尾要乘 `P(</s>|w_T)` 才能让概率在所有长度的句子上正确归一化,否则模型无法表达"句子该结束了"。

## 七、代码:bigram 计数 + 加 k 平滑 + 困惑度,从零实现

下面这段是**可照着复现**的最小完整实现,体现机制:统计 bigram/unigram 计数 → 加 k 平滑给出条件概率 → 在测试集上算 PPL。刻意不用任何库,让每一步透明。

```python
import math
from collections import defaultdict, Counter

BOS, EOS, UNK = "<s>", "</s>", "<unk>"

def tokenize(line):
    # 玩具分词：空格切；真实场景用 BPE/分词器（见第 08 章子词切分）
    return line.strip().split()

def build_vocab(corpus, min_count=1):
    """统计词频，低频词将被映射为 <unk>，从而让模型有能力给未登录词概率"""
    freq = Counter(tok for line in corpus for tok in tokenize(line))
    vocab = {w for w, c in freq.items() if c >= min_count}
    vocab |= {BOS, EOS, UNK}
    return vocab

def normalize(line, vocab):
    """加句首句尾标记；不在词表内的词替换为 <unk>"""
    toks = [t if t in vocab else UNK for t in tokenize(line)]
    return [BOS] + toks + [EOS]

class BigramAddK:
    def __init__(self, vocab, k=0.01):
        self.vocab = vocab
        self.V = len(vocab)          # 词表大小，加 k 平滑分母要用
        self.k = k
        self.uni = Counter()         # count(w_{t-1})  —— 前缀计数
        self.bi  = defaultdict(Counter)  # count(w_{t-1}, w_t)

    def fit(self, corpus):
        for line in corpus:
            toks = normalize(line, self.vocab)
            for prev, cur in zip(toks[:-1], toks[1:]):
                self.uni[prev] += 1
                self.bi[prev][cur] += 1

    def cond_prob(self, prev, cur):
        # 加 k 平滑：(count(prev,cur)+k) / (count(prev)+k*V)
        # 即使 prev 从没作为上文出现过(count=0)，分母 k*V>0，也不会除零
        num = self.bi[prev][cur] + self.k
        den = self.uni[prev] + self.k * self.V
        return num / den

    def perplexity(self, corpus):
        log_sum, n_tokens = 0.0, 0
        for line in corpus:
            toks = normalize(line, self.vocab)
            for prev, cur in zip(toks[:-1], toks[1:]):
                p = self.cond_prob(prev, cur)
                log_sum += math.log(p)   # 自然对数 → 配 exp 得 PPL
                n_tokens += 1            # 预测的 token 数（不含 BOS 作为被预测对象）
        return math.exp(-log_sum / n_tokens)

# --- 用法示例 ---
train = ["the cat sat on the mat",
         "the dog sat on the log",
         "the cat ate the fish"]
test  = ["the cat sat on the log"]

vocab = build_vocab(train, min_count=1)
lm = BigramAddK(vocab, k=0.01)
lm.fit(train)
print("PPL =", lm.perplexity(test))
```

**机制要点(也是新手最常踩的坑):**
1. **`cond_prob` 的分母用 `count(prev) + k*V`,不是 `+k`。** 这是归一化的命门:对固定 prev,`Σ_cur (count+k) = count(prev) + k*V`,只有分母这样写,各 cur 概率才加和为 1。写成 `+k` 是最常见的 bug,会让"概率"不归一、PPL 失去意义。
2. **必须加 `<s>`/`</s>`。** 没有 `</s>`,模型不会"主动结束句子",且不同长度句子的概率无法正确比较。
3. **PPL 用自然对数 ln 配 `exp`**(若想报 bits-per-token 则全程用 log₂ 配 2^·,两套别混——00 章坑 4)。
4. **`<unk>` 处理决定 OOV 概率**:训练时把低频词折成 `<unk>` 学到它的概率,测试遇到没见过的词也映射成 `<unk>`,才不会概率为 0。报告 PPL 必须说明 OOV 策略。

要做 add-k 调参,把上面 `k` 在留出集上网格搜,选 PPL 最低的 k:

```python
def tune_k(lm_class, vocab, train, dev, ks=(1.0, 0.5, 0.1, 0.05, 0.01, 0.005)):
    best_k, best_ppl = None, float("inf")
    for k in ks:
        lm = lm_class(vocab, k=k); lm.fit(train)
        ppl = lm.perplexity(dev)
        if ppl < best_ppl:
            best_k, best_ppl = k, ppl
    return best_k, best_ppl   # 通常落在很小的 k，印证 §3.1 “加一太猛”
```

## 八、神经语言模型(Bengio 2003):用嵌入一举绕开稀疏诅咒

n-gram 与稀疏搏斗了二十年,平滑越做越精(直到 KN),但**稀疏的根在表示上没解决**:n-gram 把每个词当成**离散原子(one-hot)**,"cat" 和 "dog" 在模型眼里是两个毫不相干的符号,正交、零相似度。于是"the cat ate"见过、"the dog ate"没见过,n-gram **无法把前者的统计知识迁移给后者**——哪怕 cat 和 dog 在分布上极像。平滑只能在"频次/接续"层面挪质量,**挪不动"语义相似的词应该共享统计"这件事**。这是 n-gram 范式的天花板。

### 8.1 核心思想:把词嵌入连续空间,让相似词共享统计

**Bengio 等(2003)的《A Neural Probabilistic Language Model》给出了换道方案:不再用 one-hot,而是给每个词学一个低维稠密向量(词嵌入,word embedding),用一个前馈神经网络(MLP)从前 n−1 个词的嵌入,直接预测下一个词的分布 `P(wₜ|w_{t-n+1:t-1})`。**

为什么这一招就缓解稀疏?**因为在连续嵌入空间里,模型可以学到 cat 和 dog 的向量很接近;于是即便 "the dog ate" 这个具体组合没在训练里出现,网络对它的预测也会和见过的 "the cat ate" 相近——统计强度通过"相似的输入产生相似的输出"这一神经网络的连续性,自动在相似词之间共享了。** 这正是 n-gram 做不到的:离散 one-hot 之间没有"接近"可言,而稠密向量有几何结构,泛化沿着相似度流动。**稀疏诅咒的根(表示离散),被"用连续向量表示词"从根上拔掉了。**

### 8.2 结构与机制(可复现)

记上下文为前 n−1 个词,词表 V,嵌入维度 m,隐藏层维度 h。

```
1. 查表得嵌入：对每个上文词 w_{t-i}，取其嵌入行向量 e_i = C[w_{t-i}] ∈ ℝ^m
                （C ∈ ℝ^{|V|×m} 是嵌入矩阵，可训练；这是“词向量”首次作为可学参数登场）
2. 拼接：x = [e_1; e_2; …; e_{n-1}] ∈ ℝ^{(n-1)m}
3. 隐藏层（非线性）：a = tanh(W x + b)，W ∈ ℝ^{h×(n-1)m}
4. 输出 logits：z = U a + (可选直连项 V_d x) + d，  U ∈ ℝ^{|V|×h}，z ∈ ℝ^{|V|}
5. softmax：P(wₜ = v | 上文) = exp(z_v) / Σ_j exp(z_j)
```

(第 4 步原文有一个从输入直连到输出的 skip 项 V_d x,可选;softmax 数值稳定要先减 max,见算法课 00 章。)

训练就是最小化负对数似然(=交叉熵,等价链见 00 章),对 C、W、b、U、d 一起做反向传播梯度下降。**注意嵌入矩阵 C 是被一起学出来的**——这就是后来 word2vec/GloVe 乃至所有现代模型"词向量"概念的源头。

```python
import torch, torch.nn as nn

class BengioNPLM(nn.Module):
    """A Neural Probabilistic Language Model (Bengio et al., 2003) 的最小实现。
       输入：前 n-1 个词的 id；输出：下一个词在词表上的 logits。"""
    def __init__(self, vocab_size, n, emb_dim=60, hidden=100):
        super().__init__()
        self.context = n - 1
        self.C = nn.Embedding(vocab_size, emb_dim)          # 嵌入矩阵 C：相似词将学到相近向量
        self.hidden = nn.Linear((n - 1) * emb_dim, hidden)  # W, b
        self.out = nn.Linear(hidden, vocab_size)            # U, d
        # 论文还有 input->output 直连项，这里省略以突出主干

    def forward(self, ctx_ids):           # ctx_ids: (batch, n-1)
        e = self.C(ctx_ids)               # (batch, n-1, emb_dim)
        x = e.flatten(start_dim=1)        # 拼接 -> (batch, (n-1)*emb_dim)
        a = torch.tanh(self.hidden(x))    # 非线性隐藏层
        return self.out(a)                # logits (batch, vocab_size)；交给 CrossEntropyLoss
```

### 8.3 它解决了什么、又留下了什么——通向 RNN/Transformer

**解决了:** (1) 稀疏——相似词共享统计,泛化到没见过的词组合;(2) 参数从 `O(|V|^n)`(n-gram 表)降到 `O(|V|·m + (n-1)m·h + h·|V|)`,即**参数量对 n 几乎线性而非指数**(嵌入 + 两层全连接),不再随上下文长度爆炸。

**没解决(正是后续章节的动机):**
- **上下文仍是定长 n−1。** Bengio 的 MLP 把前 n−1 个词拼接成固定长度输入,**和 n-gram 一样有马尔可夫截断**,看不到更早历史。**要建模任意长依赖,需要循环结构(RNN/LSTM,本课第 03 章)用一个隐状态压缩全部历史,或自注意力(Transformer,见算法课 02-03 章)让每个位置直接访问全部历史。**
- **输出 softmax 在 `|V|` 上,词表大时计算昂贵。** 原文为此提了重要性采样等近似;后来有层级 softmax、负采样、自适应 softmax 等(部分见算法课分词与嵌入章)。

**这就是从 n-gram 到神经 LM 的范式跃迁的全部要义:把"离散计数 + 平滑"换成"连续表示 + 可微优化",稀疏问题被表示层的泛化能力消解。** 后面所有发展——RNN/LSTM 拿掉定长限制、Transformer 拿掉串行限制并放大上下文、再到大规模预训练——都是沿着 Bengio 这一步"用神经网络学 `P(wₜ|历史)`"继续往前。

## 九、为什么 LM 是 NLP 的中心任务:自监督与可迁移

这一节回答一个战略问题:为什么整门 NLP 课要把语言模型放在如此核心的位置?因为 LM 有两个别的任务没有的、决定性的属性。

**第一,语言模型是天然的自监督(self-supervised)任务,标注成本为零。** 训练 LM 不需要人工标签——**文本自己就是标签**:每个位置的"下一个词"就是监督信号,免费、无限。NLP 的核心痛点历来是**标注数据稀缺**(情感、句法树、翻译对都贵)。而 LM 可以在整个互联网的纯文本上训练,数据量比任何标注集大几个数量级。这是 LM 能"吃下"海量数据、从而学到广泛语言知识的前提。

**第二,语言建模目标"逼"模型学到可迁移的通用语言知识。** 沿用 00 章的论证:要把下一个词预测得准(=把语言压得短),模型不得不在参数里内化语法、语义、世界知识、甚至浅层推理——因为这些都是"下一个词是什么"的决定因素。于是**一个为预测下一个词训练出来的模型,其内部表示对几乎所有下游 NLP 任务都有用**:分类、抽取、问答、翻译……只需在预训练 LM 上做少量适配。

**这两点合起来,就是"预训练 + 微调/提示"范式的根基,也是 NLP 在 2018 年后被彻底重写的原因。** 路线图清晰:Bengio(2003)证明神经 LM 可学词的连续表示 → word2vec/ELMo 把"学表示"规模化 → BERT(判别式预训练,本课第 05 章)/ GPT(生成式预训练,第 06 章)把整个语言模型作为可迁移基座 → LLM 时代用一个 LM + 提示统一几乎所有 NLP 任务(第 09 章)。**本章讲的"给序列建 `P(w)`"这件最朴素的事,正是这一切的种子。** 你现在站在 n-gram 与神经 LM 的交界点上,后面整门课都是这棵种子长出来的树。

## 十、设计权衡与常见坑

**1. n 的选择是偏差-方差权衡,不是越大越好。** n 大上下文多但稀疏重、估计方差爆炸;n 小稳但短视。trigram 是经典平衡点。**坑:** 以为"上下文越长越好"就上 5-gram、6-gram,结果绝大多数 n-gram 没见过,全靠回退,实际等价于低阶模型还更慢。

**2. 不做平滑的 n-gram 不能用,加一平滑几乎总是太猛。** 默认就该用插值/回退,追求最优用 modified Kneser-Ney。**坑:** 课堂示例图省事用加一,在真实大词表上 PPL 会很差,误以为 n-gram 本身不行——其实是平滑选错了。

**3. 归一化是平滑公式的硬约束,务必验算 `Σ_v P(v|·)=1`。** 加 k 分母必须 `+k·|V|`;回退的 α(c)、KN 的 λ 都由归一化唯一确定。**坑:** 自己改平滑公式时漏掉归一化,得到的"概率"加和不为 1,PPL 失去意义甚至 <1。

**4. PPL 不可跨 tokenizer / 跨 OOV 策略 / 跨语料比较。** (承 00 章)报告 PPL 必须连同分词、`<unk>` 处理、测试集一起说明。**坑:** 把不同设置下的 PPL 数字直接比大小下结论。

**5. 句首句尾标记必须一致处理。** bigram 起始 `P(w₁|<s>)`、终止 `P(</s>|w_T)` 都要算进 PPL,否则不同长度句子概率不可比、模型不会结束句子。**坑:** 算 PPL 时忘了 `</s>`,或把 `<s>` 也当成被预测的 token 计入分母。

**6. Kneser-Ney 的低阶用接续计数,不是普通频次。** 这是 KN 比其他平滑强的唯一原因。**坑:** 实现 KN 时低阶仍用频次估,退化成普通 absolute discounting,丢掉 KN 的精髓(Francisco 问题复发)。

**7. 神经 LM 不是没有截断——Bengio 2003 仍是定长上下文。** **坑:** 以为"用了神经网络就建模了无限历史"。真正去掉定长限制要等 RNN/LSTM(第 03 章)和自注意力(算法课)。

## 十一、动手练习

**练习 1(推导,验证理解):n-gram MLE 与零概率。**
(a) 仿照 §2.2,对 trigram 写出 `P(wₜ|w_{t-2},w_{t-1})` 的最大似然解,并用拉格朗日乘子法证明它等于计数比 `count(w_{t-2},w_{t-1},wₜ)/count(w_{t-2},w_{t-1})`。(b) 给出一个具体例子说明:为什么测试集里只要含一个训练未见的 trigram,**整段**句子的 MLE 概率就是 0、PPL 就是 +∞。(c) 用加 0.5 平滑改写该 trigram 概率,写出分母,并验证对固定上文 `Σ_v P=1`。
*提示:trigram 分母是 `count(w_{t-2},w_{t-1}) + k·|V|`;零概率来自连乘里任一因子为 0。*

**练习 2(机制,解释 KN):接续计数为什么对罕见词更好。**
设玩具语料里 "Francisco" 出现 100 次但**只**跟在 "San" 后面,"glass" 出现 10 次但跟在 8 种不同的词后面。(a) 分别算两者的接续计数 `N₁₊(• w)`。(b) 说明普通 unigram 频次会给谁更高的回退概率、KN 接续概率会给谁更高,哪个更符合"作为新上文的接续该给多少概率"的直觉。(c) 用一句话概括:为什么 KN 用"跟过多少种上文"而非"出现多少次"来估低阶,对长尾/罕见词更稳健。
*提示:`N₁₊(• Francisco)=1`,`N₁₊(• glass)=8`;接续概率正比于接续计数。*

**练习 3(手算 + 编程,算 PPL):小语料的困惑度。**
训练语料只有一句:`<s> the cat sat </s>`。(a) 用 bigram MLE(不平滑)写出 `P(the|<s>)`、`P(cat|the)`、`P(sat|cat)`、`P(</s>|sat)`。(b) 在测试句 `<s> the cat sat </s>` 上手算 PPL(此时与训练相同,应得多少?)。(c) 改测试句为 `<s> the dog sat </s>`,不平滑时 PPL 是多少、为什么;再用 §7 代码加 k=0.1 平滑重算 PPL,对比。
*提示:训练只有一句时,各 MLE 概率都是 1(每个上文只见过一种下文),不平滑 PPL=1;含 "dog"(未见)时不平滑出现 0 → PPL=∞;平滑后有限。*

**练习 4(实现,进阶):把 §7 的 add-k bigram 升级成插值或 Kneser-Ney。**
(a) 实现 §4.2 的线性插值 `λ₃·trigram + λ₂·bigram + λ₁·unigram`,在留出集上用网格搜或简化 EM 学 λ。(b) (挑战)实现 §5.3 的插值型 bigram Kneser-Ney,正确计算接续计数 `N₁₊(• w)` 与 `N₁₊(w •)`、绝对折扣 D、回退权重 λ(w_{t-1}),并验证归一化。(c) 在同一测试集上比较 add-k、插值、KN 三者的 PPL,确认 KN 最低。
*提示:接续计数要扫一遍语料的全部 bigram,用集合统计"每个词前面/后面出现过的不同词种类";D 可先固定 0.75 或用 `D=N₁/(N₁+2N₂)`。*

- **教科书:** Jurafsky & Martin, *Speech and Language Processing*(第 3 版在线草稿)的 **"N-gram Language Models"** 章——n-gram、各类平滑、PPL、`<UNK>`/句界处理讲得最系统,本章的工程细节(句界、OOV)与之对齐。**待核**:第 3 版具体章节号随在线版本更新而变,引用时以你手头版本为准。

## 十三、小结与承上启下

这一章你应当带走的硬核结论:

- **语言模型=给序列建 `P(w)=∏ₜ P(wₜ|w_<t)`(自回归分解,恒等式)。** 直接建联合分布不可行,必须分解。
- **n-gram=马尔可夫截断 `P(wₜ|w_<t)≈P(wₜ|w_{t-n+1:t-1})` + MLE 计数比。** MLE=计数比是拉格朗日乘子可推的严格解,n 是偏差-方差旋钮。
- **n-gram 的死穴是数据稀疏:任何未见 n-gram → 0 概率 → 整段似然崩 → PPL=∞。** 这是语言组合空间无限、语料有限的必然,不是数据量问题。
- **平滑是生死线,且有清晰的认知阶梯:** 加 k(盲目均匀注水,几乎总过猛)→ Good-Turing(用 N₁ 估 unseen)→ 回退/插值(稀疏时借低阶)→ **Kneser-Ney(借低阶时用"接续多样性"而非频次,对罕见词更稳,n-gram 平滑的天花板)**。每步都在更精细地回答"没见过的 n-gram 概率给多少"。
- **困惑度是评测标尺**(信息论身份见 00 章),NLP 实操要管 OOV、句界、tokenizer 一致性,跨设置不可比。
- **Bengio 2003 换道:用词嵌入 + MLP 学 `P(wₜ|历史)`,让相似词共享统计,从表示层根除稀疏诅咒,参数对 n 线性而非指数。** 但它仍是定长上下文。

**承上启下:** 我们已经看清,"用连续表示 + 可微优化"取代"离散计数 + 平滑",是 NLP 摆脱稀疏的关键一跃;但 Bengio 的 MLP 还卡在**定长上下文**这道坎上,和 n-gram 一样看不到任意长的历史。**下一章(第 03 章,循环网络)** 就来解决这个问题:RNN 用一个随时间更新的隐状态去**压缩任意长的历史**,理论上让 `P(wₜ|w_<t)` 真正条件于全部前文——但随之而来的是梯度消失/爆炸这个新难题,以及 LSTM 用门控机制对它的精巧修复。再往后,序列标注(第 04 章 HMM/CRF/维特比)会把"给序列打标签"这个结构预测问题讲透,Seq2Seq(第 05 章)则在机器翻译里催生了注意力——而注意力,正是通向 Transformer、通向你已在算法课里见过的那个统治性架构的门。**本章建立的"`P(wₜ|历史)`、稀疏-平滑、PPL"这套语言,会一路用到底。**
