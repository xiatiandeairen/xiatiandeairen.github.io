> 这章把你从"知道有 Bellman 方程、知道用动态规划能解 MDP"带到能处理一个**不知道环境模型**的真实问题:在转移概率 P 和奖励 R 完全未知、只能靠采样交互的情况下,如何估计一个策略的价值(预测),又如何在估值的同时把策略推向最优(控制)。这条主线上最核心的两个工具——蒙特卡洛(MC)与时序差分(TD)——以及它们之间的偏差-方差张力、自举(bootstrapping)的本质、on/off-policy 的分野,是后面**所有**深度强化学习算法(DQN、Actor-Critic、PPO,乃至 RLHF 里的价值函数)的地基。你将能自己推出 TD 更新为什么是 Bellman 期望方程的随机近似,并写出能解 gridworld 的 Q-learning。

## 0. 为什么"无模型"是真问题

上一章(动态规划)我们假设了一件奢侈的事:**环境模型已知**。也就是说,对每个状态 s 和动作 a,我们手里有完整的转移分布 `P(s'|s,a)` 和期望奖励 `R(s,a)`。有了它,策略评估就是解一个线性方程组(或迭代到不动点),策略改进就是对已知的 Q 取 argmax。整套动态规划(Value Iteration / Policy Iteration)是**纯计算**,根本不需要"经验"。

但现实里,这个假设几乎总是不成立:

- 你不知道一局围棋落子后对手会怎么应,也不知道机器人电机给定电压后关节会转到哪——**转移概率没有解析式**。
- 即便你能写出物理方程,状态空间巨大到无法枚举求和(`Σ_{s'} P(s'|s,a)·V(s')` 这一步就死了)。
- 你能做的,只有**让 agent 真去环境里走**,观测到一串 `(s₀, a₀, r₁, s₁, a₁, r₂, ...)`,然后从这些样本里学。

**这就是无模型(model-free)设定:不显式估计 P 和 R,直接从采样轨迹学价值或策略。** 它把"求期望"这件做不到的事,换成了"用样本平均逼近期望"这件做得到的事。本章的全部技巧,归根结底都是在回答一个问题:**手头只有有限的采样轨迹,怎么把 Bellman 方程里那个我算不出的期望,替换成一个我能算的估计量,并保证它收敛到对的地方。**

先把记号和前置结论钉死,后面反复用。

### 0.1 记号与前置:回报、价值、Bellman

一个**回合(episode)**是 agent 与环境从初始状态交互到终止状态的一条完整轨迹:

```
S₀, A₀, R₁, S₁, A₁, R₂, S₂, ..., R_T, S_T(终止)
```

注意时序约定:在 Sₜ 选动作 Aₜ,环境返回奖励 R_{t+1} 和下一状态 S_{t+1}(奖励下标比状态晚一拍,这是 Sutton & Barto 的标准约定,务必记牢,否则推导会差一个下标)。

**回报(return)** Gₜ 是从时刻 t 起的折扣累积奖励:

```
Gₜ = R_{t+1} + γ·R_{t+2} + γ²·R_{t+3} + ... = Σ_{k=0}^{∞} γᵏ·R_{t+k+1}
```

γ ∈ [0,1] 是折扣因子。它有递归结构,这是后面一切自举的源头:

```
Gₜ = R_{t+1} + γ·G_{t+1}                       ……(★)
```

**状态价值函数**是在策略 π 下,从 s 出发的期望回报:

```
v_π(s) = E_π[ Gₜ | Sₜ = s ]
```

**动作价值函数**(从 s 出发,先强制做动作 a,之后follow π):

```
q_π(s,a) = E_π[ Gₜ | Sₜ = s, Aₜ = a ]
```

把(★)的期望展开,得到本章的两条主轴方程。

**Bellman 期望方程**(评估给定 π):

```
v_π(s)   = E_π[ R_{t+1} + γ·v_π(S_{t+1}) | Sₜ=s ]
q_π(s,a) = E[ R_{t+1} + γ·E_{a'~π}[ q_π(S_{t+1}, a') ] | Sₜ=s, Aₜ=a ]
```

**Bellman 最优方程**(刻画最优 q*):

```
q*(s,a) = E[ R_{t+1} + γ·max_{a'} q*(S_{t+1}, a') | Sₜ=s, Aₜ=a ]
```

记住这两个方程的差别:期望方程里下一步用 `E_{a'~π}[q]`(**按当前策略加权平均**),最优方程里用 `max_{a'} q`(**取最优动作**)。这一字之差,正是后面 SARSA 与 Q-learning 的分水岭。

### 0.2 一个反复要用的工具:增量平均

MC 也好、TD 也好,本质都是"在线地维护一个平均值"。把它单独拎出来,因为它给出了"步长 α"的来历。

设我们要估计某个量的均值,样本一个个来:x₁, x₂, ...,记前 n 个的平均为 μ_n。则:

```
μ_n = (1/n)·Σ_{i=1}^{n} xᵢ
    = (1/n)·( x_n + Σ_{i=1}^{n-1} xᵢ )
    = (1/n)·( x_n + (n-1)·μ_{n-1} )
    = μ_{n-1} + (1/n)·( x_n − μ_{n-1} )
```

**这就是增量更新式:新估计 = 旧估计 + 步长 ×(目标 − 旧估计)。** 当步长取 1/n 时它精确等于样本平均;当步长取一个固定的 α(常数)时,它变成**指数加权移动平均**(对近期样本加权更高,适合非平稳目标)。强化学习里几乎所有更新都长这个样子——把 `(x_n − μ_{n-1})` 这个"误差项"看清楚,后面 TD 误差 δ 就是它的一个具体化身。

## 1. 蒙特卡洛预测:用完整回合的经验回报估 V

### 1.1 问题与机制

给定策略 π,我要估 `v_π(s)`。无模型,所以我不能解方程,只能采样。MC 的想法朴素到极致:**`v_π(s)` 就是回报 Gₜ 在 Sₜ=s 条件下的期望,那我就让 agent 跑很多回合,把每次访问到 s 之后实际拿到的回报记下来,求平均。**

关键前提:**Gₜ 必须能被算出来,这要求回合会终止**。因为 `Gₜ = Σ γᵏ R_{t+k+1}` 是把这个回合从 t 到终止 T 的所有真实奖励加起来——它是一个**完整观测到的真实回报,不含任何估计**。这是 MC 与 TD 最根本的区别:MC 的目标是真值的无偏样本。

算法有两个变体:

- **First-visit MC**:在一个回合里,只用某状态 s **第一次**被访问之后的回报。
- **Every-visit MC**:s 每次被访问都计入。

两者在 n→∞ 都收敛到 `v_π(s)`,但收敛速率和理论性质略有差别(first-visit 的样本是 i.i.d. 的,证明干净;every-visit 的样本在同一回合内相关,但实践常用且也收敛)。

```python
import numpy as np
from collections import defaultdict

def mc_prediction_first_visit(env, policy, gamma=0.99, n_episodes=50000):
    """First-visit MC 估计 V_π。policy(s) -> action。"""
    V = defaultdict(float)
    N = defaultdict(int)            # 每个状态被(首次访问)计数
    for _ in range(n_episodes):
        # 1) 采一条完整回合
        episode = []               # [(s, r), ...]  r 是在 s 处执行动作后收到的奖励 R_{t+1}
        s = env.reset()
        done = False
        while not done:
            a = policy(s)
            s_next, r, done, _ = env.step(a)
            episode.append((s, r))
            s = s_next
        # 2) 从后往前累计回报 G,first-visit 只在状态首次出现处更新
        G = 0.0
        # 注意:episode[i]=(s_i, r_{i+1}),G 要用 r_{i+1} 起算
        seen_states = [x[0] for x in episode]
        for t in reversed(range(len(episode))):
            s_t, r_next = episode[t]
            G = r_next + gamma * G          # G_t = R_{t+1} + γ G_{t+1}  ←(★)
            if s_t not in seen_states[:t]:  # first visit
                N[s_t] += 1
                V[s_t] += (G - V[s_t]) / N[s_t]   # 增量平均
    return V
```

注意第 2 步**从后往前**遍历:因为 `Gₜ = R_{t+1} + γ·G_{t+1}`,反向扫只需 O(T) 一次就能算出所有 Gₜ,这是(★)的直接应用。

### 1.2 分析:为什么 MC 无偏但高方差

**无偏性。** 对 first-visit,设某次首访 s 得到的回报为 G(它是真实回报的一个样本),则:

```
E[ G | S=s ] = v_π(s)          (按定义,v_π(s) 就是 G 的条件期望)
```

样本平均是无偏估计量的平均,故 MC 估计 `V(s) → v_π(s)`,**无偏**(first-visit 下每个回合贡献一个独立同分布样本,大数定律保证收敛)。这是 MC 最大的优点:**目标里没有任何"猜测",它直指真值。**

**高方差。** Gₜ 是一长串随机奖励的和:`Gₜ = R_{t+1} + γR_{t+2} + ... + γ^{T-t-1}R_T`。每一步的动作随机性(π 随机)、转移随机性(P 随机)都注入方差,而且它们**沿整条轨迹累积**。直观地,如果一条回合有 H 步,回报里就叠了 H 层随机性。粗略量化(假设各步奖励近似独立、方差为 σ²,折扣 γ):

```
Var(Gₜ) ≈ σ²·(1 + γ² + γ⁴ + ...) = σ² / (1 − γ²)     (无限期、独立近似)
```

更要命的是**轨迹长度本身是随机的**(回合长短不一),长轨迹的回报方差远大于短轨迹。所以 MC 的估计噪声很大,需要很多回合才能收敛。

**三个硬约束(也是 MC 的坑):**
1. **必须等回合结束**才能更新——在线学习、持续任务(continuing task,没有终止)上 MC 直接失效。
2. **不能利用马尔可夫性**:MC 把每个回报当独立样本,不借助 `V(s')` 这种结构信息,样本效率低。
3. **高方差**导致收敛慢、对学习率敏感。

这三个约束,恰好是 TD 来解决的。

## 2. 时序差分预测:用估计更新估计(自举)

### 2.1 机制:把 MC 目标里"未来真实回报"换成"当前估计"

回到 Bellman 期望方程:`v_π(s) = E_π[ R_{t+1} + γ·v_π(S_{t+1}) ]`。

MC 用 `Gₜ`(完整真实回报)当目标。TD 的洞见是:**`Gₜ = R_{t+1} + γ·G_{t+1}`,而 `G_{t+1}` 的期望就是 `v_π(S_{t+1})`,我手上不是有 V 的当前估计吗?那就用 `R_{t+1} + γ·V(S_{t+1})` 当目标,不必等整个回合跑完。**

这就是 **TD(0) 目标**(又叫 TD target):

```
TD target:  R_{t+1} + γ·V(S_{t+1})
```

它只需要往前看**一步**(one-step lookahead):观测到 R_{t+1} 和 S_{t+1} 就够了。更新式(套用 §0.2 的增量平均,目标换成 TD target):

```
V(Sₜ) ← V(Sₜ) + α·[ R_{t+1} + γ·V(S_{t+1}) − V(Sₜ) ]
```

中括号里这一坨,就是大名鼎鼎的 **TD 误差(TD error)**:

```
δₜ = R_{t+1} + γ·V(S_{t+1}) − V(Sₜ)
```

**δ 度量的是"我对 Sₜ 的旧估计,和往前走一步后用新信息修正出的估计,差了多少"。** 它是 TD 学习的心跳信号,后面 Actor-Critic 用它当 advantage 的近似、TD(λ) 用它做资格迹更新,无处不在。

```python
def td0_prediction(env, policy, gamma=0.99, alpha=0.1, n_episodes=50000):
    """TD(0) 估计 V_π。每一步即时更新,不等回合结束。"""
    V = defaultdict(float)
    for _ in range(n_episodes):
        s = env.reset()
        done = False
        while not done:
            a = policy(s)
            s_next, r, done, _ = env.step(a)
            # 终止状态的 V 恒为 0:V(s_next)=0 when done
            target = r + (0.0 if done else gamma * V[s_next])
            delta = target - V[s]                 # TD 误差 δ
            V[s] += alpha * delta                 # 自举更新
            s = s_next
    return V
```

注意一个常错点:**终止状态的价值必须是 0**(没有未来了),所以 done 时 target 只剩 r。漏掉这个会让价值系统性偏高。

### 2.2 关键推导:TD 更新是 Bellman 期望方程的随机近似

这是本章理论核心,务必自己推一遍。我们要说明:**反复做 TD(0) 更新,V 会收敛到满足 Bellman 期望方程的不动点 v_π。**

**思路:把 Bellman 方程看成"求一个使期望 TD 误差为零的 V"的方程,TD 更新则是这个方程的随机梯度式逼近(Robbins-Monro 随机近似)。**

第一步,Bellman 期望方程可以写成"期望 TD 误差 = 0":

```
Bellman:  v_π(s) = E_π[ R_{t+1} + γ·v_π(S_{t+1}) | Sₜ=s ]
等价于:    E_π[ R_{t+1} + γ·v_π(S_{t+1}) − v_π(s) | Sₜ=s ] = 0
即:        E_π[ δₜ | Sₜ=s ] = 0   (当 V = v_π 时,期望 TD 误差为零)
```

也就是说,**真值 v_π 恰好是"使每个状态的期望 TD 误差归零"的那个 V**。

第二步,引入 Robbins-Monro 随机近似定理。它解决这类问题:想求 `θ*` 使 `E[ f(θ*, X) ] = 0`,但 E 算不出,只能采样到 `f(θ, Xᵢ)`。定理说,迭代

```
θ_{n+1} = θ_n + α_n · f(θ_n, X_n)
```

在满足步长条件 `Σ α_n = ∞` 且 `Σ α_n² < ∞`(例如 α_n = 1/n)、且 f 有界、`E[f]` 在 θ* 处穿过零点(单调性)等条件下,**几乎必然收敛到 θ\***。直觉:`f(θ_n, X_n)` 是真实期望 `E[f(θ_n)]` 的一个含噪样本,大步长保证能走到任意远(消除初值影响),平方可和保证噪声被逐步压住。

第三步,对号入座。把"待求的 θ"取为每个状态的价值 `V(s)`,把"采样函数 f"取为采到的那一步 TD 误差:

```
f(V, 这一步样本)  =  R_{t+1} + γ·V(S_{t+1}) − V(Sₜ)  =  δₜ
```

它的条件期望(对状态 s)正是 `E_π[δₜ | s]`,而我们第一步证了:**当 V=v_π 时这个期望为零**。于是 TD(0) 更新

```
V(Sₜ) ← V(Sₜ) + α·δₜ
```

就是对方程 `E_π[δ | s] = 0` 做 Robbins-Monro 迭代。**只要步长满足 RM 条件、所有状态被无限次访问,V 几乎必然收敛到 v_π。** 推导完毕。

> **要点提炼**:MC 是直接用"目标的无偏样本(完整回报)"做随机近似;TD 是用"目标的有偏样本(含自举估计 V(S'))"做随机近似。两者都是 Robbins-Monro,差别全在"目标怎么构造"。这一句话统一了本章上半部分。

### 2.3 分析:TD 的偏差-方差,与对 MC 的全面对照

**TD 有偏(在 V 还没收敛时)。** TD target `R_{t+1} + γ·V(S_{t+1})` 用了**当前估计 V**,而 V 此刻是错的,所以目标本身带偏差。只有当 V 收敛到 v_π,目标才无偏。这叫**自举偏差**:用自己的估计去更新自己。

**TD 低方差。** TD target 只含**一步真实随机性**(R_{t+1} 和 S_{t+1} 各一次采样),`V(S_{t+1})` 是个确定的查表值,不引入采样方差。对比 MC target 含 H 步累积随机性:

```
            目标            随机来源数         偏差        方差
MC          Gₜ (完整回报)    O(H) 步累积        无偏        高
TD(0)       R+γV(S')         1 步               有(自举)    低
```

这就是 **MC vs TD 的偏差-方差权衡**:MC 用方差换无偏,TD 用偏差换低方差。实践中 TD 往往收敛更快、能在线学习,是深度 RL 的主流;但在奖励极稀疏、自举误差会沿状态链传播放大(value 高估螺旋)的场景,TD 的偏差会咬人。

**马尔可夫性的利用。** 有个深刻的差别:在**有限数据**下,MC 收敛到的是"最小化训练回报均方误差"的解,而 **TD(0) 收敛到的是把数据当成一个经验 MDP(用样本频率估 P、R)后该 MDP 的 Bellman 解**(这叫 TD 的 certainty-equivalence / 极大似然 MDP 解释)。换句话说,**TD 隐式假设并利用了马尔可夫性**,在真满足马尔可夫性的环境里数据效率更高;反之若环境其实非马尔可夫,TD 的这个假设会带来额外偏差。这是 Sutton & Barto 著名的 "AB 例子" 想讲的事,值得读原文体会。

### 2.4 在偏差-方差之间插值:n-step 与 TD(λ)

MC(看到底)和 TD(0)(看一步)是两个极端。中间地带就是 **n-step return**:往前看 n 步真实奖励,第 n 步之后用自举:

```
G_t^{(n)} = R_{t+1} + γR_{t+2} + ... + γ^{n-1}R_{t+n} + γⁿ·V(S_{t+n})
```

- n=1 退化成 TD(0);
- n=∞(或到回合终止)退化成 MC。

n 越大,真实奖励占比越高 → 偏差越小、方差越大;n 越小则相反。**n 是一个手动调节偏差-方差的旋钮。** 更新式照旧:`V(Sₜ) ← V(Sₜ) + α·[ G_t^{(n)} − V(Sₜ) ]`。

**TD(λ)** 更进一步:不选某个固定 n,而是把所有 n-step return 按几何权重 (1−λ)λⁿ⁻¹ 加权平均成 **λ-return**:

```
G_t^λ = (1−λ)·Σ_{n=1}^{∞} λ^{n-1}·G_t^{(n)}
```

权重 `(1−λ)λⁿ⁻¹` 对所有 n 求和为 1(几何级数),λ=0 时只剩 n=1(即 TD(0)),λ=1 时变成 MC。**λ ∈ [0,1] 是偏差-方差连续旋钮。**

直接按定义算 λ-return 需要等回合结束(前向视角,forward view)。工程上用**资格迹(eligibility trace)的后向视角**等价实现,可在线增量更新:为每个状态维护一个迹 `e(s)`,记录它"最近多久前被访问、该为当前 TD 误差负多少责任":

```
后向视角 TD(λ):
  每步观测 δₜ = R_{t+1} + γV(S_{t+1}) − V(Sₜ)
  e(Sₜ) ← e(Sₜ) + 1                       # 访问到的状态迹 +1(accumulating trace)
  对所有状态 s:
      V(s) ← V(s) + α·δₜ·e(s)              # 用同一个 δ 更新所有有迹的状态
      e(s) ← γλ·e(s)                       # 迹按 γλ 衰减
```

直觉:一次 TD 误差 δₜ 不只更新当前状态,而是**回溯地分配给最近访问过的一串状态**,越近的拿得越多(迹大)。λ 控制这条"信用回溯链"的长度。有定理保证:对离线(回合末统一更新)情形,后向视角资格迹与前向视角 λ-return 给出**完全相同**的总更新量。这是 TD(λ) 优雅之处——一个标量 λ 统一了 MC 和 TD,且能在线高效实现。

> 这里"资格迹"是个隐喻:每个状态有一张"我最近参与了决策"的资格凭证,凭证越新越有资格领取当前误差的修正额度。

## 3. 无模型控制:在估值的同时改进策略

预测解决了"给定 π,V 是多少"。控制要解决"怎么把 π 推向最优"。这里有三个绕不开的设计点,逐个攻克。

### 3.1 为什么控制必须学 Q 而不是 V

策略改进(policy improvement)要做的是 `π'(s) = argmax_a q_π(s,a)`。在**有模型**时,从 V 也能算 Q:`q(s,a) = R(s,a) + γΣ_{s'}P(s'|s,a)V(s')`。但**无模型**时,你没有 P、没有 R,这个 argmax 算不出来!

**结论:无模型控制必须直接估计动作价值 `Q(s,a)`。** 因为只要有了 Q,改进就是纯查表的 `argmax_a Q(s,a)`,完全不需要模型。这是为什么本节所有算法(SARSA、Q-learning)学的都是 Q 而不是 V。

### 3.2 探索的必要性:GLIE 与 ε-greedy

第二个坑:如果一直贪婪地走 `argmax_a Q(s,a)`,你只会反复采样当前认为最好的动作,**那些从没试过的动作的 Q 值永远得不到更新**——可能有个更好的动作,因为你从没探索它而被永远埋没。这是 RL 的根本难题:**探索(exploration)与利用(exploitation)的权衡**。

**ε-greedy** 是最简单的解法:以概率 1−ε 走贪婪动作,以概率 ε 在所有动作里均匀随机选一个。

```
π(a|s) = 1 − ε + ε/|A|      if a = argmax_{a'} Q(s,a')
       = ε/|A|              otherwise
```

它保证每个动作都有 ≥ ε/|A| 的概率被选,从而**所有状态-动作对都被无限次访问**——这正是收敛定理要求的。但固定 ε 会让策略永远保留 ε 的随机性,无法收敛到确定性最优策略。解法是让 ε 随时间衰减,满足 **GLIE**(Greedy in the Limit with Infinite Exploration)条件:

```
GLIE 两条件:
 (1) 无限探索:每个 (s,a) 被访问次数 → ∞       (保证 Q 估计处处收敛)
 (2) 极限贪婪:π 最终收敛到对 Q 贪婪的策略       (保证策略收敛到最优)
```

例如取 `ε_k = 1/k`(第 k 个回合)就同时满足两条。**GLIE + 适当步长是 on-policy MC/SARSA 收敛到最优的充分条件。**

其它探索手段(此处只点名,深探留给后续章节):
- **乐观初始化(optimistic initialization)**:把 Q 初值设得很高(比任何真实回报都高)。这样每个动作第一次被试时实际回报都低于乐观初值,Q 被下调,agent 自然"为了消除乐观偏差"去遍历所有动作,实现免费的早期探索。简单但偏差随时间消退,只在初期起作用。
- **UCB(Upper Confidence Bound)**:不只看 Q 均值,还加一个**不确定性奖励项**,选 `argmax_a [ Q(s,a) + c·√(ln t / N(s,a)) ]`。N 是该动作被选次数,选得越少则 √ 项越大、越被鼓励去试。它把"探索"建立在"我对这个估计有多不确定"上,比纯随机的 ε-greedy 更有方向性,在多臂老虎机(bandit)上有 O(ln t) 的遗憾(regret)界。

### 3.3 On-policy SARSA:用实际走出的下一动作

把"预测=策略评估"和"ε-greedy 改进"交替起来,就是 **广义策略迭代(GPI)** 的无模型版。SARSA 是其 TD 实现。名字来自更新用到的五元组 **(S, A, R, S', A')**:它把 TD(0) 从 V 搬到 Q,且**下一步价值用实际按当前策略选出的动作 A' 的 Q 值**:

```
δₜ = R_{t+1} + γ·Q(S_{t+1}, A_{t+1}) − Q(Sₜ, Aₜ)
Q(Sₜ, Aₜ) ← Q(Sₜ, Aₜ) + α·δₜ
```

关键:`A_{t+1}` 是用与当前 Q 对应的 ε-greedy 策略**真正采样出来、并将在下一步真正执行**的动作。所以 SARSA 是 **on-policy**——它评估并改进的,正是它用来产生数据的那个(含探索的)策略本身。它对应的是 Bellman **期望**方程的随机近似:目标里那个 `Q(S', A')` 中的 A' 服从 π,等价于在估 `E_{a'~π}[q(S',a')]`。

```python
def sarsa(env, n_episodes, gamma=0.99, alpha=0.5, eps=0.1):
    Q = defaultdict(lambda: np.zeros(env.n_actions))
    def eps_greedy(s):
        if np.random.rand() < eps:
            return np.random.randint(env.n_actions)
        return int(np.argmax(Q[s]))
    for _ in range(n_episodes):
        s = env.reset()
        a = eps_greedy(s)                       # 先选好当前动作
        done = False
        while not done:
            s2, r, done, _ = env.step(a)
            a2 = eps_greedy(s2)                  # 关键:实际要执行的下一动作 A'
            target = r + (0.0 if done else gamma * Q[s2][a2])
            Q[s][a] += alpha * (target - Q[s][a])
            s, a = s2, a2                        # A' 成为下一步真正执行的动作
    return Q
```

### 3.4 Off-policy Q-learning:用 max,逼近 Bellman 最优方程

**Q-learning** 改一个字:下一步价值不用实际动作 A',而用**贪婪动作的 max**:

```
δₜ = R_{t+1} + γ·max_{a'} Q(S_{t+1}, a') − Q(Sₜ, Aₜ)
Q(Sₜ, Aₜ) ← Q(Sₜ, Aₜ) + α·δₜ
```

**这个 max 让 Q-learning 成为 Bellman 最优方程的随机近似。** 复用 §2.2 的推导:Bellman 最优方程 `q*(s,a) = E[R + γ·max_{a'} q*(S',a')]` 可写成"期望最优 TD 误差为零":

```
当 Q = q* 时:  E[ R + γ·max_{a'} q*(S',a') − q*(s,a) | s,a ] = 0
```

Q-learning 的更新量 `R + γ·max_{a'}Q(S',a') − Q(s,a)` 正是这个函数的一个采样,做 Robbins-Monro 迭代,**(s,a) 被无限次访问 + 步长满足 RM 条件 ⇒ Q 几乎必然收敛到 q\***。

**为什么是 off-policy?** 注意目标里的 `max_{a'} Q(S', a')` 评估的是**贪婪策略**(永远取最优动作),而产生数据的行为策略(behavior policy)是带探索的 ε-greedy——**学习的目标策略 ≠ 行动的行为策略**,这就是 off-policy 的定义。Q-learning 能用一个探索性的行为策略采数据,却直接学到最优(贪婪)策略的价值。

更妙的是:Q-learning 的 off-policy 性**不需要重要性采样(importance sampling)修正**。一般 off-policy 要用比值 `π(a|s)/b(a|s)` 重加权来纠正分布偏差(MC off-policy 评估就必须这么干,且方差极大)。Q-learning 不用,原因在于它学的是 Q 而非 V:`max_{a'}Q(S',a')` 这个目标**根本不依赖行为策略怎么选 a'**,它直接对所有 a' 取 max。下一步动作怎么采,不影响这个目标的值——所以无需重加权。这是 Q-learning 在工程上极受欢迎的关键(DQN 正是其深度版)。

```python
def q_learning(env, n_episodes, gamma=0.99, alpha=0.5, eps=0.1):
    Q = defaultdict(lambda: np.zeros(env.n_actions))
    def eps_greedy(s):
        if np.random.rand() < eps:
            return np.random.randint(env.n_actions)
        return int(np.argmax(Q[s]))
    for _ in range(n_episodes):
        s = env.reset()
        done = False
        while not done:
            a = eps_greedy(s)                   # 行为策略(含探索)
            s2, r, done, _ = env.step(a)
            # 目标用 max(贪婪/目标策略),与实际下一动作无关 → off-policy
            best_next = 0.0 if done else np.max(Q[s2])
            target = r + gamma * best_next
            Q[s][a] += alpha * (target - Q[s][a])
            s = s2
    return Q
```

SARSA 与 Q-learning 的代码**只差一行**:SARSA 用 `Q[s2][a2]`(实际下一动作),Q-learning 用 `np.max(Q[s2])`(最优动作)。这一行差别,带来完全不同的行为(见 §3.6 悬崖行走)和理论归属(期望方程 vs 最优方程)。

### 3.5 完整可运行例子:Q-learning 解 Gridworld

下面是一个自包含、无外部依赖的 gridworld 与 Q-learning 求解器,直接能跑。

```python
import numpy as np

class GridWorld:
    """
    4x4 网格。S 起点(0,0)=状态0,G 目标(3,3)=状态15。
    动作 0..3 = 上/右/下/左。每步奖励 -1(鼓励走最短路),到 G 奖励 0 并终止。
    撞墙原地不动(仍 -1),迫使 agent 学会避开边界。
    """
    def __init__(self, n=4):
        self.n = n
        self.n_actions = 4
        self.goal = n * n - 1
    def reset(self):
        self.s = 0
        return self.s
    def step(self, a):
        r, c = divmod(self.s, self.n)
        if   a == 0: r = max(0, r - 1)          # 上
        elif a == 1: c = min(self.n - 1, c + 1) # 右
        elif a == 2: r = min(self.n - 1, r + 1) # 下
        elif a == 3: c = max(0, c - 1)          # 左
        self.s = r * self.n + c
        done = (self.s == self.goal)
        reward = 0.0 if done else -1.0
        return self.s, reward, done, {}

def q_learning_solve(env, n_episodes=5000, gamma=0.95, alpha=0.5, eps=0.1):
    Q = np.zeros((env.n * env.n, env.n_actions))
    for ep in range(n_episodes):
        s = env.reset()
        done = False
        while not done:
            # ε-greedy 行为策略
            a = np.random.randint(env.n_actions) if np.random.rand() < eps \
                else int(np.argmax(Q[s]))
            s2, r, done, _ = env.step(a)
            target = r + (0.0 if done else gamma * np.max(Q[s2]))   # 用 max → off-policy
            Q[s, a] += alpha * (target - Q[s, a])                   # TD 更新
            s = s2
    return Q

env = GridWorld(4)
Q = q_learning_solve(env)
greedy_policy = np.argmax(Q, axis=1)
arrows = {0: '↑', 1: '→', 2: '↓', 3: '←'}
print("学到的贪婪策略(每格箭头指向下一步):")
for r in range(4):
    print(' '.join('G ' if (r*4+c)==env.goal else arrows[greedy_policy[r*4+c]]
                   for c in range(4)))
# 预期:每个格子的箭头都把你引向右下角 G,路径长度 = 曼哈顿距离。
```

跑通后你会看到学出的策略把每个格子都指向终点 G,且每条路径长度等于到 G 的曼哈顿距离——因为每步 -1 的奖励让最优策略 = 最短路。这就验证了:**在完全不知道转移规则的前提下,Q-learning 仅靠采样把最优策略学了出来。**

### 3.6 SARSA vs Q-learning 的行为差异:悬崖行走

这是最经典的对照实验(Sutton & Barto, Example 6.6),它戳穿了一个常见误解——"Q-learning 学最优,所以它一定表现更好"。

**环境**:一个网格,底边中间一长条是"悬崖",掉下去 −100 并被弹回起点;其余每步 −1;目标在悬崖另一端。最优路径(最短)是**贴着悬崖边走**。

**两者学到的不同东西:**

- **Q-learning** 学的是**最优策略**(假设之后完美贪婪、永不失足),所以它认定贴悬崖走最优,会把策略压向悬崖边缘。
- **SARSA** 是 on-policy,它评估的是**自己实际在用的 ε-greedy 策略**——而这个策略有 ε 概率乱走。贴着悬崖走时,那 ε 的随机性会**真的把它推下悬崖**(−100),这个惨痛后果通过 `Q(S',A')`(A' 是实际的探索动作)被算进了价值里。于是 SARSA 学会**绕开悬崖、走一条更靠里的"安全路径"**。

```
       Q-learning 偏好(最优但危险)        SARSA 偏好(次优但安全)  (示意,非精确策略)
       · · · · · · · · · · · ·              · · · · · · · · · · · ·
       · · · · · · · · · · · ·              · → → → → → → → → → ↓ ·
       S → → → → → → → → → → G(贴边)        S ↑ · · · · · · · · → G
       █ █ █ █ █ █ █ █ █ █ (悬崖)           █ █ █ █ █ █ █ █ █ █ (悬崖)
```

**结果(定性,数值待核但趋势确定)**:训练中带 ε 探索时,**SARSA 的实际回合回报更高(更少掉崖)**,因为它的策略对自身探索噪声鲁棒;Q-learning 虽然学到了"理论最优"的贴边路径,但因 ε 探索频繁掉崖,**在线表现反而更差**。若把 ε 退火到 0,Q-learning 的贪婪策略才会显出最短路优势。

**这个例子要刻进脑子的结论:**
- **On-policy(SARSA)评估"我真正会执行的、含探索的策略",所以它会规避因自身探索而触发的灾难——它在乎"行动中"的安全。**
- **Off-policy(Q-learning)评估"假想中完美贪婪的策略",所以它不在乎行为策略的探索风险——它学的是"理想最优",但部署时若仍带探索就可能踩坑。**

它也解释了一个工程直觉:**在线、安全攸关、探索代价高的场景偏好 on-policy(或 on-policy 系的 PPO);可以离线大量采数据、追求渐进最优的场景偏好 off-policy(Q-learning/DQN)。**

## 4. 收敛性、复杂度与样本效率的量化小结

把散落的量化结论收拢成一张表,便于查阅与决策。

**收敛条件(表格化)**

```
算法           收敛到          关键充分条件
─────────────────────────────────────────────────────────────────
MC 预测        v_π            每个 s 被无限次访问 + 增量步长(1/n)
TD(0) 预测     v_π            同上,步长满足 RM:Σα=∞, Σα²<∞
SARSA 控制     q*(→π*)        GLIE(ε→0)+ RM 步长(on-policy 评估自身策略)
Q-learning     q*             所有 (s,a) 被无限次访问 + RM 步长
                              (off-policy,行为策略只需保证遍历)
```

**步长 α 的两种取法及其代价**
- α = 1/n(逐状态计数衰减):满足 RM,理论收敛到真值,但对**非平稳**目标(如控制中策略一直在变、Q 目标在漂移)反应迟钝——老样本权重太大。
- α = 常数:不满足 Σα²<∞,**不收敛到点而是在真值附近抖动**(稳态方差 ∝ α);但能持续跟踪漂移目标。深度 RL 几乎都用小常数 α(或学习率调度),因为目标始终非平稳。这是个**实用 vs 理论收敛**的经典取舍。

**偏差-方差谱(一图流)**

```
方差 ←─────────────────────────────────────────→ 偏差
高方差/无偏                                     低方差/有偏(自举)
  MC ── n-step(n大)─ TD(λ)(λ大→小)─ n-step(n小)── TD(0)
  λ=1                                                    λ=0
```

**计算/存储复杂度(表格型)**

```
方法            每步更新计算         存储             能否在线/持续任务
──────────────────────────────────────────────────────────────────
表格 MC         回合末 O(T)          O(|S|) 或 O(|S||A|)   否(需终止)
表格 TD(0)      O(1)                 O(|S|)               是
表格 SARSA/QL   O(1)(QL 取 max O(|A|)) O(|S||A|)          是
TD(λ) 资格迹    O(|S|)/步(更新全迹)   O(|S|)               是
```

注:表格法的存储是 O(|S||A|),状态-动作空间一大就爆炸——这正是**函数逼近(下一章用神经网络代替 Q 表)**的动机。一旦上函数逼近,off-policy + 自举 + 函数逼近三者同时出现会破坏收敛保证(著名的 **deadly triad / 致命三要素**),那是 DQN 要靠 target network、experience replay 等技巧硬扛的问题,本章先埋个伏笔。

## 5. 设计权衡与常见坑(尤其 RL 新手必踩)

1. **奖励下标差一位**。`Sₜ` 选 `Aₜ` 得到 `R_{t+1}`。MC 反向累计、TD target 构造,只要下标错一位,价值就系统性偏移且很难 debug。写代码时把 episode 存成 `(s, a, r_next)` 三元组,对齐到位。

2. **终止状态价值忘了置零**。`V(terminal)=0`。TD/Q 更新里 `done` 时 target 必须只剩 r。漏掉会让价值无止境膨胀(尤其 γ 接近 1 时灾难性高估)。

3. **常数步长当成会收敛**。固定 α 下 Q 在真值附近抖,不会精确收敛。要么接受抖动(跟踪非平稳),要么衰减 α。别拿"loss 不降到 0"当 bug 查。

4. **探索不足 → 收敛到次优**。ε 太小或不衰减、或乐观初值不够乐观,会让某些 (s,a) 几乎不被访问,其 Q 永远是错的初值,策略卡在局部最优。**任何收敛定理都以"无限次访问每个 (s,a)"为前提**,这个前提在代码里靠探索机制保证。

5. **把 SARSA 当 Q-learning(或反过来)**。一行之差,行为天壤之别(悬崖行走)。明确你要的是"评估自己含探索的策略"(on-policy)还是"逼近理想最优"(off-policy)。

6. **off-policy 的 reward hacking / 高估**。Q-learning 的 `max` 算子对**估计噪声**有系统性向上偏差(Jensen 不等式:`E[max Q̂] ≥ max E[Q̂]`),这叫 **maximization bias / 最大化偏差**。新手会发现 Q 值莫名虚高、策略偏好那些被高估的动作。**Double Q-learning**(van Hasselt, 2010)用两套 Q 解耦"选动作"和"估价值"来缓解,DDQN 把它带进深度 RL。这是 RL 极常见的坑,务必知道它的存在和成因。

7. **每个回合更新次数 vs 步长耦合**。every-visit、长回合会让某些状态在一回合内被更新很多次,等效放大学习率,导致不稳定。注意更新频率与 α 的相互作用。

## 6. 动手练习

**练习 1(推导题,核心)** 严格证明:TD(0) 更新 `V(Sₜ) ← V(Sₜ) + α·δₜ` 是 Bellman 期望方程的随机近似。要求:(a) 把 Bellman 期望方程改写成 `E_π[δₜ | Sₜ=s] = 0` 的形式;(b) 指出它是 Robbins-Monro 求根迭代 `θ ← θ + α·f(θ, X)` 的特例,写清 θ、f、X 各对应什么;(c) 陈述收敛所需的步长条件与访问条件。
*提示*:回到 §2.2,自己合上书重推一遍;关键是看出"采样到的 δₜ 是期望 TD 误差的无偏估计"。

## 8. 小结与承上启下

这一章把"环境模型未知"这个真实约束,转化成了一套**纯靠采样**学习价值与策略的方法论:

- **预测**:MC(无偏、高方差、需回合终止)与 TD(自举、低方差、有偏、能在线)是两个极端,n-step 与 TD(λ)/资格迹在它们之间用一个旋钮(n 或 λ)连续插值——**这是偏差-方差权衡在 RL 里最干净的体现**。
- **控制**:无模型必须学 Q;在 GPI 框架下交替"TD 估 Q"与"ε-greedy 改进",得到 on-policy 的 **SARSA**(评估自身含探索策略,对应 Bellman 期望方程)与 off-policy 的 **Q-learning**(用 max 逼近 Bellman 最优方程,无需重要性采样)。悬崖行走把 on/off-policy 的本质差别——"在乎行动中的风险" vs "追求理想最优"——演示得淋漓尽致。
- **统一视角**:MC 和 TD 都是 Bellman 方程的 **Robbins-Monro 随机近似**,差别只在目标怎么构造(真实回报 vs 自举估计)。**这把整章收成了一句话。**

**承上**:第 1 章(动态规划)的 Bellman 期望/最优方程,在这里从"已知模型时解方程"变成了"未知模型时用样本逼近其不动点"——同一个方程,两种求解范式。

**启下**:本章全是**表格法**,存储 O(|S||A|) 在真实问题里必然爆炸。下一章用**函数逼近(神经网络)**替换 Q 表,得到 **DQN**——但自举 + off-policy + 函数逼近会触发**致命三要素(deadly triad)**导致发散,于是需要 target network、experience replay 来稳住。再往后,当动作连续或我们想直接优化策略,就进入**策略梯度 / Actor-Critic**;那里的 critic 用的正是本章的 TD,advantage 用的正是 TD 误差 δ 的推广(GAE)。**本章的 δ、自举、on/off-policy、偏差-方差,会在后面每一章反复出现——它是整门 RL 课程真正的地基。**

### 练习 1:证明 TD(0) 更新是 Bellman 期望方程的随机近似

**(a) 把 Bellman 期望方程改写成 `E_π[δₜ | Sₜ=s] = 0`**

策略 π 下的状态价值满足

```
v_π(s) = E_π[ R_{t+1} + γ·v_π(S_{t+1}) | Sₜ=s ]
```

把右边整体移到左边(`v_π(s)` 是关于 s 的常数,可以放进期望里):

```
0 = E_π[ R_{t+1} + γ·v_π(S_{t+1}) | Sₜ=s ] − v_π(s)
  = E_π[ R_{t+1} + γ·v_π(S_{t+1}) − v_π(s) | Sₜ=s ]
```

定义当 V = v_π(真值)时的 TD 误差 `δₜ = R_{t+1} + γ·v_π(S_{t+1}) − v_π(Sₜ)`,上式即

```
E_π[ δₜ | Sₜ=s ] = 0          ……(对每个状态 s 都成立)
```

**结论:Bellman 期望方程等价于"真值 v_π 使每个状态的期望 TD 误差归零"。** 这把"求满足 Bellman 方程的 V"变成了一个**求根问题**:找 V 使 `E_π[δ | s] = 0`。

**(b) 对应 Robbins-Monro 求根迭代 `θ ← θ + α·f(θ, X)`**

Robbins-Monro(RM)解决的问题:求 `θ*` 使 `g(θ*) := E_X[ f(θ*, X) ] = 0`,但期望 g 算不出,只能采到含噪样本 `f(θ, X)`。逐个对应:

```
RM 符号          TD(0) 里的对应物
──────────────────────────────────────────────────────────
待求参数 θ        每个状态的价值表 V(s)(被估对象就是 V(Sₜ))
采样函数 f(θ,X)   这一步采到的 TD 误差
                  δₜ = R_{t+1} + γ·V(S_{t+1}) − V(Sₜ)
随机样本 X        这一步的转移样本 X = (R_{t+1}, S_{t+1})
                  (在状态 Sₜ=s 下,由环境/策略联合采样)
目标方程 g(θ)=0   g(V)(s) = E_π[δₜ | Sₜ=s] = 0
```

代入后 RM 迭代 `θ ← θ + α·f(θ,X)` 恰好就是

```
V(Sₜ) ← V(Sₜ) + α·δₜ = V(Sₜ) + α·[ R_{t+1} + γ·V(S_{t+1}) − V(Sₜ) ]
```

**关键一步(为什么合法):** 采样到的单步 `δₜ` 是其条件期望 `g(V)(s) = E_π[δₜ|s]` 的**无偏估计**——因为对固定的 V 表,`δₜ` 只随这一步的随机 `(R_{t+1}, S_{t+1})` 变化,而

```
E_π[ R_{t+1} + γ·V(S_{t+1}) − V(s) | Sₜ=s ] = g(V)(s)
```

注意此处的无偏是针对"以 V 为参数的那个采样函数 g(V)"而言;由于 V≠v_π 时 g(V)(s) 一般不为 0,这正体现了 §2.3 说的**自举偏差**(对真值 v_π 而言目标有偏)。但 RM 不要求样本对真值无偏,只要求**对当前参数 θ 的目标函数 g(θ) 无偏**,这一条满足。

**(c) 收敛所需条件**

RM 定理保证 V 几乎必然(with probability 1)收敛到 v_π,需要:

1. **步长(Robbins-Monro 条件):** 用 `α_n(s)` 表示状态 s 第 n 次被更新时的步长,需对每个 s 满足

```
Σ_n α_n = ∞        (步长之和发散:保证能从任意初值走到真值,不被初值困住)
Σ_n α_n² < ∞       (步长平方可和:保证采样噪声被逐步压住、不破坏收敛)
```

典型取法 `α_n = 1/n`(逐状态计数衰减)同时满足两条;常数步长只满足第一条,故只能在真值附近抖动而不精确收敛。

2. **访问条件:** **每个状态被无限次访问**(n→∞)。否则没被更新到的状态其 V 停在错误初值,谈不上收敛。

3. (技术性)即时奖励有界、`v_π` 存在唯一(γ<1 或所有策略下回合必终止即可保证)。

满足以上,则对每个 s,`V(s) → v_π(s)` a.s.。**推导完毕——TD(0) 就是把"期望 TD 误差为零"这个 Bellman 方程,用单步采样的 δₜ 做 Robbins-Monro 求根。**

### 练习 2:λ-return 权重归一化,及 λ→0、λ→1 的极限

**(a) 权重对所有 n 求和为 1**

权重 `wₙ = (1−λ)λⁿ⁻¹`,n=1,2,3,...,其中 λ ∈ [0,1)。求和:

```
Σ_{n=1}^∞ (1−λ)λⁿ⁻¹ = (1−λ)·Σ_{n=1}^∞ λⁿ⁻¹
                    = (1−λ)·(1 + λ + λ² + ...)
                    = (1−λ)·( 1/(1−λ) )      ← 几何级数 Σ_{k≥0} λᵏ = 1/(1−λ),|λ|<1
                    = 1
```

**所以 λ-return `G_t^λ = Σ_{n≥1} wₙ·G_t^{(n)}` 是各 n-step return 的一个合法凸组合(加权平均),权重和恒为 1。**

**(b) λ→0 极限 → TD(0) 目标**

直接代 λ=0 看每项权重 `wₙ = (1−λ)λⁿ⁻¹`:

```
n=1:  w₁ = (1−0)·0⁰ = 1·1 = 1
n≥2:  wₙ = (1−0)·0ⁿ⁻¹ = 0     (0 的正数次幂 = 0)
```

(约定 0⁰=1。)于是

```
G_t^{λ=0} = 1·G_t^{(1)} + 0 + 0 + ... = G_t^{(1)} = R_{t+1} + γ·V(S_{t+1})
```

**即 λ=0 时 λ-return 退化为 TD(0) 目标。** ✓

**(c) λ→1 极限 → MC 目标(回合终止)**

设回合在 T 步终止。终止后没有未来,**n 一旦 ≥ T−t,n-step return 就"看到底",不再含自举项,且对所有更大的 n 都等于完整回报 Gₜ**:

```
G_t^{(n)} = Gₜ            对所有 n ≥ T − t
```

(因为 V(终止状态)=0,γⁿV(S_{t+n}) 项消失,真实奖励也加满了。)

把 λ-return 拆成"有限自举段"+"终止后尾段",并令 λ→1。最干净的写法是用 §2.3/教材的等价分解式:

```
G_t^λ = (1−λ)·Σ_{n=1}^{T−t−1} λ^{n-1}·G_t^{(n)}  +  λ^{T−t−1}·Gₜ
```

(前半是终止前各 n-step 的加权和,系数 λ^{T−t−1} 是 n ≥ T−t 那些项的几何权重之和,它们对应的 return 都等于 Gₜ,故合并成一项。)

令 λ→1:

- 前半每个系数 `(1−λ)λⁿ⁻¹ → 0`,整段(有限项之和)→ 0;
- 尾项系数 `λ^{T−t−1} → 1`。

```
lim_{λ→1} G_t^λ = 0 + 1·Gₜ = Gₜ = R_{t+1} + γR_{t+2} + ... + γ^{T−t−1}R_T
```

**即 λ=1 时 λ-return 退化为 MC 目标(完整真实回报)。** ✓

**结论:λ ∈ [0,1] 是 TD(0)(λ=0)与 MC(λ=1)之间的连续插值旋钮**,提示里的尾项处理(终止后回报恒为 Gₜ、把无穷多相等项的几何权重合并为 λ^{T−t−1})正是 λ→1 推导的关键。

### 练习 3:悬崖行走 SARSA vs Q-learning(可运行,无外部依赖)

下面代码自包含,仅依赖 numpy,CPU 直接跑。已实测验证(seed=0,500 回合):**SARSA 训练期尾 100 回合均回报 ≈ −22,Q-learning ≈ −39**(SARSA 更高,掉崖更少);贪婪策略:**SARSA 走上方安全路、Q-learning 贴悬崖边走最短路**。ε 按 1/k 退火后两者都趋向最短路(Q-learning 收敛到长度 13 = 最优;SARSA 接近最优)。

```python
import numpy as np

class CliffWalking:
    """4×12 网格(Sutton & Barto Example 6.6)。
    起点 S=(3,0)=状态36,终点 G=(3,11)=状态47;
    底边中间10格 (3,1)..(3,10) 是悬崖(状态37..46)。
    动作 0/1/2/3 = 上/右/下/左。每步 -1;踩悬崖 -100 并瞬间回起点(不终止);
    到 G 终止(reward 走到 G 那步算 -1)。撞墙原地不动。"""
    def __init__(self):
        self.rows, self.cols = 4, 12
        self.n_states = self.rows * self.cols
        self.n_actions = 4
        self.start = 3 * 12 + 0          # 36
        self.goal  = 3 * 12 + 11         # 47
        self.cliff = {3 * 12 + c for c in range(1, 11)}  # 37..46

    def reset(self):
        self.s = self.start
        return self.s

    def step(self, a):
        r, c = divmod(self.s, self.cols)
        if   a == 0: r = max(0, r - 1)               # 上
        elif a == 1: c = min(self.cols - 1, c + 1)   # 右
        elif a == 2: r = min(self.rows - 1, r + 1)   # 下
        elif a == 3: c = max(0, c - 1)               # 左
        ns = r * self.cols + c
        if ns in self.cliff:                          # 掉崖:重罚 + 回起点
            self.s = self.start
            return self.start, -100.0, False, {}
        self.s = ns
        done = (ns == self.goal)
        return ns, -1.0, done, {}

def eps_greedy(Q, s, n_actions, eps):
    if np.random.rand() < eps:
        return np.random.randint(n_actions)
    return int(np.argmax(Q[s]))

def sarsa(env, n_episodes, gamma=1.0, alpha=0.5, eps=0.1, eps_decay=False):
    Q = np.zeros((env.n_states, env.n_actions))
    returns = []
    for ep in range(n_episodes):
        e = (1.0 / (ep + 1)) if eps_decay else eps      # 1/k 退火满足 GLIE
        s = env.reset()
        a = eps_greedy(Q, s, env.n_actions, e)          # 先选好当前动作 A
        done, G, steps = False, 0.0, 0
        while not done and steps < 1000:                # 步数上限防早期死循环
            s2, r, done, _ = env.step(a); G += r; steps += 1
            a2 = eps_greedy(Q, s2, env.n_actions, e)    # 关键:实际要执行的下一动作 A'
            target = r + (0.0 if done else gamma * Q[s2][a2])   # ← 用 Q[s2][a2]
            Q[s][a] += alpha * (target - Q[s][a])
            s, a = s2, a2                               # A' 成为下一步真正执行的动作
        returns.append(G)
    return Q, returns

def q_learning(env, n_episodes, gamma=1.0, alpha=0.5, eps=0.1, eps_decay=False):
    Q = np.zeros((env.n_states, env.n_actions))
    returns = []
    for ep in range(n_episodes):
        e = (1.0 / (ep + 1)) if eps_decay else eps
        s = env.reset()
        done, G, steps = False, 0.0, 0
        while not done and steps < 1000:
            a = eps_greedy(Q, s, env.n_actions, e)      # 行为策略(含探索)
            s2, r, done, _ = env.step(a); G += r; steps += 1
            target = r + (0.0 if done else gamma * np.max(Q[s2]))  # ← 用 max(贪婪/目标策略)
            Q[s][a] += alpha * (target - Q[s][a])
            s = s2
        returns.append(G)
    return Q, returns

# —— SARSA 与 Q-learning 唯一的区别就是上面标注的那一行:
#     SARSA   : target 用 Q[s2][a2]   (实际下一动作 A',on-policy → Bellman 期望方程)
#     Q-learn : target 用 np.max(Q[s2])(最优动作,    off-policy → Bellman 最优方程)

def greedy_path(env, Q, max_len=60):
    s = env.reset(); path = [int(s)]
    for _ in range(max_len):
        s, _, done, _ = env.step(int(np.argmax(Q[s])))
        path.append(int(s))
        if done: break
    return path

def render(env, path):
    g = [['.' for _ in range(env.cols)] for _ in range(env.rows)]
    for c in range(1, 11): g[3][c] = 'C'           # 悬崖
    for s in path:
        r, c = divmod(int(s), env.cols)
        if g[r][c] == '.': g[r][c] = '*'           # 走过的格
    g[3][0], g[3][11] = 'S', 'G'
    return '\n'.join(' '.join(row) for row in g)

if __name__ == "__main__":
    np.random.seed(0)
    env = CliffWalking(); N = 500
    Qs, rs = sarsa(env, N)            # 固定 eps=0.1
    Qq, rq = q_learning(env, N)
    print("训练期尾100回合均回报:  SARSA = %.1f   Q-learning = %.1f"
          % (np.mean(rs[-100:]), np.mean(rq[-100:])))     # SARSA 更高(掉崖少)
    print("\nSARSA 贪婪路径(走上方安全路):");     print(render(env, greedy_path(env, Qs)))
    print("\nQ-learning 贪婪路径(贴悬崖最短路):"); print(render(env, greedy_path(env, Qq)))

    # —— ε 按 1/k 退火,长训练,观察是否都收敛到最短路 ——
    np.random.seed(0)
    Qs2, _ = sarsa(env, 2000, eps_decay=True)
    Qq2, _ = q_learning(env, 2000, eps_decay=True)
    print("\n退火后贪婪路径长度:  SARSA = %d   Q-learning = %d  (最短路 = 13)"
          % (len(greedy_path(env, Qs2)) - 1, len(greedy_path(env, Qq2)) - 1))
```

**实测输出(seed=0,趋势稳定):**

```
训练期尾100回合均回报:  SARSA = -22.2   Q-learning = -39.2

SARSA 贪婪路径(走上方安全路):     Q-learning 贪婪路径(贴悬崖最短路):
. * * * * * * * * * * *           . . . . . . . . . . . .
* * . . . . . . . . . *           . . . . . . . . . . . .
* . . . . . . . . . . *           * * * * * * * * * * * *
S C C C C C C C C C C G           S C C C C C C C C C C G

退火后贪婪路径长度:  SARSA = 17   Q-learning = 13  (最短路 = 13)
```

**分析结论:**

- **(a) 训练期回报:SARSA(≈−22)> Q-learning(≈−39)。** Q-learning 的贪婪策略贴着悬崖,而训练时仍带 ε=0.1 的随机性,**那 ε 概率会真的把它推下悬崖(每次 −100)**,拉低在线回报。SARSA 是 on-policy,它把"自己含探索的策略会掉崖"这个后果通过 `Q(S',A')`(A' 是实际探索动作)算进了价值,于是主动远离悬崖,在线回报更高。**这正是"Q-learning 学最优 ≠ 训练表现更好"的反例。**

- **(b) 贪婪路径:Q-learning 贴边走(行3上方那一行,长度最短),SARSA 走上方更靠里的安全路(更长但远离悬崖)。** 与教材 Example 6.6 一致。

- **退火(ε=1/k):** 满足 GLIE,探索逐渐消失。**Q-learning 收敛到长度 13 的最短路(= 最优:11 步右 + 上下各 1 步绕过起点格);SARSA 也趋向最短路**(本例 2000 回合后为 17,继续退火/加长训练会进一步逼近 13——因为 ε→0 后 SARSA 评估的策略也趋于贪婪,安全路与最短路重合)。**这验证了:GLIE 下两者最终都收敛到最优策略,差异只在"训练过程中"是否规避探索风险。**

> 注:`γ=1.0`(无折扣)时回合必须能终止,故保留 `steps < 1000` 上限防止早期随机策略在掉崖-回起点间无限循环;收敛后不会触发。

### 练习 4:MC 与 TD 目标的方差对比,及各自适用场景

**(a) γ=0.99 时 MC 目标方差约是 TD 目标的多少倍**

独立奖励、方差 σ² 的近似下:

- **MC 目标 = Gₜ**,含整条轨迹累积随机性:

```
Var(Gₜ) ≈ σ²·(1 + γ² + γ⁴ + ...) = σ² / (1 − γ²)
```

- **TD(0) 目标 = R_{t+1} + γ·V(S_{t+1})**,其中 `V(S_{t+1})` 是查表确定值(不引入采样方差,这是关键),**只有一步真实奖励 R_{t+1} 带随机性**:

```
Var(TD target) ≈ Var(R_{t+1}) ≈ σ²
```

倍数 = 两者之比:

```
Var(Gₜ) / Var(TD) ≈ [σ²/(1−γ²)] / σ² = 1/(1−γ²)
```

代入 γ=0.99:

```
1/(1 − 0.99²) = 1/(1 − 0.9801) = 1/0.0199 ≈ 50.25
```

**≈ 50 倍(精确 50.25)。** 即 MC 目标的方差约为 TD 目标的 **50 倍**。

**(b) 为何长视野(γ→1)任务里 TD/自举几乎是必需的**

- 倍数 `1/(1−γ²)` 随 γ→1 **爆炸式增长**:γ=0.9→约 5 倍,γ=0.99→约 50 倍,γ=0.999→约 500 倍(`1/(1−γ²) ≈ 1/(2(1−γ))`,与有效视野 1/(1−γ) 同阶发散)。
- 长视野意味着回合很长、回报里叠加的随机奖励步数 H 很大,**MC 把这 H 层方差全吃进单个目标**,估计噪声巨大、收敛极慢、对学习率敏感。
- TD 用 `V(S_{t+1})` 把"未来的全部随机性"压缩成一个确定的自举估计,**目标方差几乎与视野无关(恒为 ~σ²)**。代价是引入自举偏差,但偏差可随 V 收敛而消除,方差却无法靠单次采样消除。
- **结论:视野越长,MC 的方差代价越不可承受,而 TD 的方差几乎不变——所以长视野/持续任务里 TD(及 n-step、TD(λ) 取较小 λ)几乎是必需的。** 这也是几乎所有深度 RL(DQN、Actor-Critic、PPO 的 critic/GAE)都用自举的根本原因。

**(c) 什么情况下宁愿用 MC**

尽管 TD 方差低,以下场景反而偏好 MC(用方差换取无偏/无假设):

1. **非马尔可夫环境(状态有遗漏/部分可观测 POMDP)。** TD(0) 隐式假设并利用马尔可夫性(certainty-equivalence / 极大似然 MDP 解释,见 §2.3 AB 例子);若环境实际非马尔可夫,**这个假设本身带来系统性偏差**。MC 只用真实回报、不依赖 `V(S')` 的结构假设,对马尔可夫性破坏更鲁棒。

2. **自举偏差不可控 / 会被放大的场景。** 奖励极稀疏、或 off-policy + 函数逼近(致命三要素 deadly triad)时,自举误差会沿状态链传播、`max` 算子还会叠加最大化偏差(value 高估螺旋)。**MC 目标无自举,从根上不受这类偏差累积/发散之害。**

3. **回合很短(H 小)。** 此时 `1/(1−γ²)` 即便 γ 接近 1 也被"提前终止"截断,MC 的方差代价小;同时 MC 无偏、实现简单(无需调自举相关的稳定化技巧),性价比高。

4. (补充)**需要无偏的真值锚点做评估/校准**,或 episode 结构天然清晰、可大量离线采样时,MC 的"目标直指真值、无猜测"是优点。

**一句话总结:TD 用偏差换低方差,长视野/在线/马尔可夫时占优;MC 用方差换无偏/无假设,在非马尔可夫、自举不可控、短回合时反而是更稳的选择。** 二者由 n-step / TD(λ) 的旋钮连续插值,实践中常取折中(如 GAE 的 λ≈0.95)。
