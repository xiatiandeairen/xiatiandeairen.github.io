> 这一章把你从"会用贝尔曼方程求价值、再从价值贪心地导出策略"带到"直接把策略本身参数化成 π_θ(a|s),对一个明确的标量目标 J(θ) 做梯度上升"。核心是把**策略梯度定理**从第一性原理推到尾——它的全部魔法在一行恒等式(log-derivative trick)上:它让我们绕过了对环境转移 p(s'|s,a) 求导这件根本做不到的事。读完你能自己默写出这条定理的完整证明、能证明 baseline 不引入偏差、能讲清为什么用 advantage 能把方差压下去,并能照着伪代码实现一个带 reward-to-go 与 baseline 的 REINFORCE。这是后面 Actor-Critic(05 章)、PPO(06 章)、RLHF(08 章)、GRPO(11 章)全部策略优化方法的共同地基。

## 一、动机:为什么要从"价值"转到"策略"

前几章我们走的是**价值类(value-based)** 路线:学一个 `Q(s,a)` 或 `V(s)`,然后通过 `a* = argmax_a Q(s,a)` 这样的贪心(或 ε-贪心)间接得到策略。这条路在离散动作、表格或中等规模问题上非常成功(Q-learning、DQN)。但它有几处**结构性的别扭**,而这些别扭恰好是大模型对齐场景的核心痛点:

**第一,argmax 要求动作空间可枚举。** 连续动作(机器人关节力矩、连续控制)里 `argmax_a Q(s,a)` 本身就是一个内层优化问题,不好解。而语言模型每步要从几万到十几万 token 的词表里选——虽然离散但极大,且我们要的不是"选概率最高的一个 token",而是要一个**分布**(采样温度、top-p 都建立在策略是分布这件事上)。价值类方法没有显式的策略分布对象,这层很不自然。

**第二,价值类天然倾向确定性策略,处理随机策略很别扭。** 但很多场景的最优策略本身就是随机的:石头剪刀布的纳什均衡是均匀混合;部分可观(POMDP)下,随机策略可以隐式地做"信息收集"。语言生成更是如此——我们要的就是一个能采样出多样输出的随机策略。

**第三,价值类的"目标"和"我们真正想要的"隔了一层。** 我们真正想最大化的是期望回报,但价值类去拟合贝尔曼方程(让 `Q` 自洽),再寄希望于"`Q` 准了 → 贪心策略就好"。这中间有 deadly triad(函数逼近 + bootstrap + off-policy 三者凑齐会发散,详见 03 章)等一系列已知的不稳定。

**策略类(policy-based)方法换一个思路:既然我们真正想要的是一个好策略,那就直接把策略参数化成 π_θ(a|s),定义一个明确的标量目标 J(θ),然后老老实实对 θ 做梯度上升。** 这就回到了我们最熟悉的范式:定义一个可微目标,求梯度,更新参数。监督学习的全部工程直觉(SGD、Adam、minibatch)都能搬过来。

这里的关键转变是:**π_θ 是一个我们自己设计、完全可控、对 θ 处处可微的对象。** 离散动作就用 softmax 头:`π_θ(a|s) = softmax(f_θ(s))[a]`;连续动作就让网络输出一个高斯的均值和方差 `π_θ(·|s) = N(μ_θ(s), σ_θ(s)²)`。这两种参数化都是我们说了算的,求 ∇_θ log π_θ(a|s) 是常规的反向传播。

但马上有一个让所有 RL 新手卡住的问题:**目标 J(θ) 里含有环境的转移概率 p(s'|s,a) 和初始分布,而这些是环境给的黑盒,我们既不知道它的解析式、更没法对它求导。** 整条轨迹的分布依赖 θ(因为动作是策略采的)又依赖 p(我们不可微的部分)。

**这一章 90% 的智慧,就是用一个三行的恒等式优雅地把 p 对梯度的贡献彻底消掉。** 这就是策略梯度定理。

**给做过监督学习的工程老兵的迁移提示:** 你已经会"对损失求梯度、SGD 更新"。这里唯一陌生的是:**目标是一个对随机轨迹的期望,而采样这个随机性的分布本身依赖你要优化的参数 θ。** 监督学习里数据分布是固定的(与模型参数无关),所以你可以直接对 loss 求梯度;RL 里"数据"(轨迹)是策略生成的,改 θ 就改了数据分布,这是策略梯度推导里所有技巧的来源,也是后面"为什么要 on-policy、为什么要重要性采样"的根。

## 二、问题设定:目标函数与它为什么不好直接求导

先把对象写清楚。沿用 00–01 章的 MDP 设定:状态 s ∈ S,动作 a ∈ A,转移 `p(s'|s,a)`,奖励 `r(s,a)`,折扣 γ ∈ [0,1),初始状态分布 `ρ₀(s₀)`。

一条**轨迹(trajectory)** 是 `τ = (s₀, a₀, s₁, a₁, …, s_{T-1}, a_{T-1}, s_T)`。在策略 π_θ 和环境 p 共同作用下,这条轨迹出现的概率是:

```
p_θ(τ) = ρ₀(s₀) · ∏_{t=0}^{T-1} π_θ(a_t | s_t) · p(s_{t+1} | s_t, a_t)
```

读这一行要看清三种因子:`ρ₀` 是环境给的初始分布(与 θ 无关),`π_θ(a_t|s_t)` 是**我们的策略(含 θ)**,`p(s_{t+1}|s_t,a_t)` 是**环境转移(不含 θ,黑盒,不可导)**。这三者相乘,是因为给定 s_t,动作 a_t 只由策略决定;给定 (s_t, a_t),下一个状态 s_{t+1} 只由环境决定(马尔可夫性)。

轨迹的**回报(return)** 记为 `G(τ)`。最常见的两种写法:无折扣的有限步 `G(τ) = Σ_{t=0}^{T-1} r(s_t,a_t)`,或折扣的 `G(τ) = Σ_{t=0}^{T-1} γ^t r(s_t,a_t)`。下面的推导对两种都成立(γ 只是奖励上的一个常数权重,不影响对 θ 求导的结构),为了式子干净,正文先用通用的 `G(τ)`,需要逐步回报时再展开。

**优化目标** 就是期望回报:

```
J(θ) = E_{τ ~ p_θ(τ)} [ G(τ) ] = ∫ p_θ(τ) · G(τ) dτ
```

（离散情形把 ∫ 换成 Σ,推导完全一样。）我们要做的是 `θ ← θ + α · ∇_θ J(θ)`,梯度**上升**(因为是最大化回报，不是最小化损失)。

现在直面那个困难。机械地把 ∇_θ 推进积分:

```
∇_θ J(θ) = ∇_θ ∫ p_θ(τ) · G(τ) dτ
         = ∫ ∇_θ [ p_θ(τ) ] · G(τ) dτ        ← G(τ) 不显含 θ(回报由 reward 决定，reward 是环境的)
```

注意这里 `G(τ)` 对 θ 没有显式依赖：奖励 `r(s,a)` 是环境定义的，给定一条具体轨迹 τ，它的回报就是个确定的数。θ 的影响**全在 p_θ(τ) 里**（θ 改变了哪些轨迹更可能出现）。

那 `∇_θ p_θ(τ)` 呢？把上面 p_θ(τ) 的连乘取对数会更好处理，但**直接对连乘求导会牵扯到 p(s'|s,a) 的导数——而我们对环境根本无法求导**。这就是死结。log-derivative trick 就是来解这个死结的。

## 三、核心推导:log-derivative trick 与策略梯度定理

### 3.1 一行恒等式:log-derivative trick

整个策略梯度的命门是这条**初等微积分恒等式**（也叫 likelihood-ratio / score-function trick）：

```
∇_θ p_θ(τ) = p_θ(τ) · ∇_θ log p_θ(τ)
```

证明只用链式法则。因为 `∇_θ log p_θ(τ) = (1 / p_θ(τ)) · ∇_θ p_θ(τ)`（对数的导数是 1/自变量 乘以内层导数），两边同乘 `p_θ(τ)` 即得。**它的价值在于：把"对概率本身求导"换成"概率 × 对 log 概率求导"，而 log 把连乘变成连加，求导后各项独立。**

代回 ∇_θ J(θ)：

```
∇_θ J(θ) = ∫ ∇_θ p_θ(τ) · G(τ) dτ
         = ∫ p_θ(τ) · ∇_θ log p_θ(τ) · G(τ) dτ          ← 用 log-derivative trick
         = E_{τ ~ p_θ} [ ∇_θ log p_θ(τ) · G(τ) ]          ← ∫ p_θ(τ)·(...) dτ 就是期望的定义
```

**这一步是质变**：右边重新变回了一个"对 p_θ 的期望"。期望意味着——**我们可以用蒙特卡洛采样去无偏估计它**。只要按当前策略 π_θ 跑几条轨迹，对每条算 `∇_θ log p_θ(τ) · G(τ)` 再平均，就是 ∇_θ J(θ) 的无偏估计。这就是把"不可解的积分"变成"可采样估计"的关键。

### 3.2 关键一击：环境转移项被求导消掉

现在算 `∇_θ log p_θ(τ)`。把 p_θ(τ) 的连乘取对数，连乘变连加：

```
log p_θ(τ) = log ρ₀(s₀) + Σ_{t=0}^{T-1} [ log π_θ(a_t|s_t) + log p(s_{t+1}|s_t,a_t) ]
```

对 θ 求梯度。**逐项看哪些含 θ：**

```
∇_θ log p_θ(τ) = ∇_θ log ρ₀(s₀)                    ← 不含 θ，梯度 = 0
               + Σ_t ∇_θ log π_θ(a_t|s_t)           ← 含 θ，保留！
               + Σ_t ∇_θ log p(s_{t+1}|s_t,a_t)     ← 不含 θ，梯度 = 0
```

**这就是整章最关键的一击**：初始分布 `ρ₀` 和环境转移 `p(s'|s,a)` 都不含 θ，对 θ 求梯度直接归零。剩下的只有我们自己的策略项。于是：

```
∇_θ log p_θ(τ) = Σ_{t=0}^{T-1} ∇_θ log π_θ(a_t|s_t)
```

请停下来体会这件事的分量：**我们完全不需要知道环境的动力学，也不需要它可微，就能算出整条轨迹概率的对数梯度。** log 把连乘拆成连加，求导后那些不含 θ 的因子（哪怕它们是不可知的黑盒）全部变成加性的零项。这正是 model-free（无模型）强化学习能成立的数学根基——**我们对世界一无所知，照样能做梯度上升。**

代回 3.1 的结果，得到**策略梯度定理(Policy Gradient Theorem)的轨迹形式**：

```
┌─────────────────────────────────────────────────────────────┐
│  ∇_θ J(θ) = E_{τ ~ π_θ} [ ( Σ_{t=0}^{T-1} ∇_θ log π_θ(a_t|s_t) ) · G(τ) ]  │
└─────────────────────────────────────────────────────────────┘
```

直觉解读：**`∇_θ log π_θ(a_t|s_t)` 是"让动作 a_t 在状态 s_t 下更可能发生"的方向**（它在统计学里叫 score function，得分函数）。乘上回报 `G(τ)` 当权重——**回报高的轨迹，把它里面用过的所有 (状态, 动作) 决策都朝"更可能"的方向推；回报低的（甚至为负）就往"更不可能"的方向推。** 一句话：**好的行为加权重、坏的行为减权重，按回报大小调整每个决策的概率。** 这就是策略梯度的全部直觉，朴素到近乎"试错学习"的数学化身。

这个估计是**无偏(unbiased)** 的：它没有用任何 bootstrap、没有用任何近似模型，纯粹是定义式的恒等变形 + 蒙特卡洛采样。无偏是策略梯度相对价值类(TD 是有偏的)的一个本质优点。代价是——它**方差极大**，这是后面三节(因果性、baseline、advantage)要逐层解决的核心矛盾。

### 3.3 因果性：reward-to-go(把无关的过去回报删掉)

轨迹形式有个明显的浪费。看 t 时刻那一项 `∇_θ log π_θ(a_t|s_t)`，它乘的是**整条轨迹的总回报 G(τ)**，其中包含了 `r₀, r₁, …, r_{t-1}` 这些**发生在 a_t 之前**的奖励。但物理上 t 时刻的动作 a_t **不可能影响过去的奖励**(因果律)。把这些与 a_t 无关的项乘进去，期望上虽然不变(下面会证它们贡献为零)，但白白增大了方差。

**因果性(causality）说的是：第 t 步的决策，只该为它之后（含当步）的回报负责。** 我们想把权重从 `G(τ)` 换成只算 t 之后的回报，记为 **reward-to-go(待领回报)**：

```
G_t = Σ_{k=t}^{T-1} γ^{k-t} · r(s_k, a_k)      ← 从 t 开始往后累计的（折扣）回报
```

下面证明：把 G(τ) 换成 G_t 不改变梯度的期望（无偏），只是降方差。核心是一个引理。

**引理（过去的奖励对当前 score 的期望贡献为零）：** 对任意 t' < t，

```
E_{τ ~ π_θ} [ ∇_θ log π_θ(a_t|s_t) · r(s_{t'}, a_{t'}) ] = 0
```

直觉：`r(s_{t'},a_{t'})` 在时刻 t' 已经"尘埃落定"，给定它之后，未来动作 a_t 的 score function 平均下来是零（score 的期望恒为零，下面 4.1 会专门证）。

证明（对未来变量取条件期望，用 score 期望为零）：

```
E[ ∇_θ log π_θ(a_t|s_t) · r(s_{t'},a_{t'}) ]
  = E_{s_{t'},a_{t'},…,s_t} [ r(s_{t'},a_{t'}) · E_{a_t ~ π_θ(·|s_t)} [ ∇_θ log π_θ(a_t|s_t) ] ]
                                                   └─────────── 这一项 = 0（见 4.1 score 期望为零）
  = E[ r(s_{t'},a_{t'}) · 0 ]
  = 0
```

第一步是全期望公式（先对 a_t 之前的所有变量取期望，再对 a_t 取条件期望）；关键在于：给定 s_t，`r(s_{t'},a_{t'})`（t' < t，是过去量）相对 a_t 是常数，可以提到内层期望外面；而内层 `E_{a_t}[∇_θ log π_θ(a_t|s_t)] = 0`（4.1 会证）。所以整项为零。∎

因此，G(τ) 里属于"t 之前"的奖励项，乘上 t 时刻的 score 后期望都是零——删掉它们不改变期望。得到**策略梯度定理的 reward-to-go 形式**：

```
∇_θ J(θ) = E_{τ ~ π_θ} [ Σ_{t=0}^{T-1} ∇_θ log π_θ(a_t|s_t) · G_t ]
```

这就是本章标题里那条式子。**它和 3.2 的轨迹形式期望相等，但方差更小**——因为每个 score 项乘的是更"干净"、数值更小的 G_t，而不是把无关的过去回报一并塞进去放大噪声。这是免费的降方差（不引入任何偏差），实现里**永远应该用 reward-to-go 而不是总回报**，是新手最容易漏掉的一个正确性/效率细节。

### 3.4 期望形式（连接 Q 函数，为 05 章铺路）

reward-to-go `G_t` 是 `Q^{π_θ}(s_t, a_t)` 的一个**无偏单样本估计**（因为 `Q^π(s,a) = E_π[G_t | s_t=s, a_t=a]`，这是 03 章的定义）。把对 G_t 的采样换成它的期望，策略梯度定理还有一个更"理论"的等价写法：

```
∇_θ J(θ) = E_{s ~ d^π, a ~ π_θ(·|s)} [ ∇_θ log π_θ(a|s) · Q^{π}(s,a) ]
```

其中 `d^π(s)` 是策略 π 下的（折扣）状态访问分布。这是 Sutton 等人 1999 年论文里那条经典形式。**两点要记牢：(1) 期望里的状态是从 π_θ 自己诱导的分布 d^π 采的——这就是 on-policy（同策略）的由来：数据必须来自当前策略，换了策略分布就变了；(2) 把 G_t 换成它的期望 Q^π，再换成"Q − baseline"的优势，就直接通向 05 章 Actor-Critic：用一个学出来的 critic 去估 Q 或 advantage，替代高方差的蒙特卡洛 G_t。** 这一章我们先用最朴素的蒙特卡洛 G_t，把方差问题暴露出来，下一章再请 critic 来救场。

## 四、降方差：baseline 与 advantage

### 4.1 score function 的期望恒为零（baseline 的全部依据）

策略梯度估计方差大的一个直接原因：`G_t` 总是正的（如果奖励非负），导致**所有动作的概率都被往上推**，只是推的力度不同。我们更希望的是"比平均好的动作往上推、比平均差的往下压"——这需要一个**参照基线**。但加基线会不会改变梯度？答案是不会，依据是下面这条**比 3.1 还基础的恒等式**：

**任何合法概率分布的 score function，期望恒为零。** 即对任意只依赖 s（不依赖 a）的函数 `b(s)`：

```
E_{a ~ π_θ(·|s)} [ ∇_θ log π_θ(a|s) ] = 0
```

证明（只用"概率归一"这一个事实）：

```
E_{a ~ π_θ(·|s)} [ ∇_θ log π_θ(a|s) ]
  = Σ_a π_θ(a|s) · ∇_θ log π_θ(a|s)
  = Σ_a π_θ(a|s) · ( 1/π_θ(a|s) ) · ∇_θ π_θ(a|s)     ← log 导数：∇log f = (1/f)∇f
  = Σ_a ∇_θ π_θ(a|s)
  = ∇_θ Σ_a π_θ(a|s)                                  ← 梯度与有限求和可交换
  = ∇_θ 1                                             ← 概率归一：Σ_a π_θ(a|s) = 1，恒等于常数 1
  = 0
```

最后一步是命门：**不管 θ 怎么变，π_θ(·|s) 永远是个合法分布，概率和恒为 1，常数的梯度是零。** 这条性质在统计学里叫"score 的期望为零"，是极大似然、Fisher 信息、自然梯度的共同基石。（连续动作把 Σ 换成 ∫，并需要"梯度与积分可交换"的正则性条件，常见参数化如高斯都满足，此处不展开。）

### 4.2 baseline 不引入偏差：完整证明

现在从 reward-to-go 形式里，给每个时刻减去一个**只依赖状态、不依赖动作**的基线 `b(s_t)`：

```
∇_θ J(θ) ≈ E [ Σ_t ∇_θ log π_θ(a_t|s_t) · ( G_t − b(s_t) ) ]
```

要证明这等于原梯度，只需证明**减去的那部分期望为零**，即对每个 t：

```
E_{τ ~ π_θ} [ ∇_θ log π_θ(a_t|s_t) · b(s_t) ] = 0
```

证明（对动作 a_t 取条件期望，把 b(s_t) 提出来，再用 4.1）：

```
E_τ [ ∇_θ log π_θ(a_t|s_t) · b(s_t) ]
  = E_{s_t} [ E_{a_t ~ π_θ(·|s_t)} [ ∇_θ log π_θ(a_t|s_t) · b(s_t) ] ]      ← 全期望，先固定 s_t 对 a_t 求内层
  = E_{s_t} [ b(s_t) · E_{a_t ~ π_θ(·|s_t)} [ ∇_θ log π_θ(a_t|s_t) ] ]      ← b(s_t) 不依赖 a_t，提到内层期望外
  = E_{s_t} [ b(s_t) · 0 ]                                                 ← 内层正是 4.1 的 score 期望 = 0
  = 0                                                                      ∎
```

**关键前提一再强调：b 只能依赖状态 s_t，绝不能依赖动作 a_t。** 一旦 b 依赖了 a_t，它就不能提到对 a_t 的期望之外，4.1 也不再适用，就会引入偏差——这是新手设计 baseline 时最危险的坑（比如有人想当然地用 `b(s,a)=Q(s,a)`，那其实是合法的，得到的是优势，见 4.4；但如果用一个依赖 a 的、与 score 相关的乱七八糟的量，就破坏无偏）。

**所以：减去任意状态相关 baseline 都不改变梯度的期望（无偏），但能改变方差。** 这给了我们一个免费的旋钮——选一个好的 b，把方差压下去，且不付任何偏差代价。

### 4.3 为什么 baseline 能降方差：把方差算出来

无偏不等于有用，我们要的是**降方差**。直觉先行：如果不减 baseline，回报恒正时所有 score 都被同号放大；减一个接近平均回报的 b，让"高于平均"的项为正、"低于平均"的为负，相互抵消，估计量的幅度变小、方差变小。下面把它量化。

考虑单个时刻 t，记 score 向量 `g = ∇_θ log π_θ(a_t|s_t)`，要估计的是 `g·(G_t − b)`。它无偏（4.2 已证），所以减小方差就是减小它的二阶矩。看它的方差（对单个标量分量或用 trace 看整体，思路相同）：

```
Var[ g·(G_t − b) ] = E[ g²·(G_t − b)² ] − ( E[g·(G_t − b)] )²
```

第二项是梯度的平方，**与 b 无关**（4.2：期望不随 b 变）。所以**最小化方差 ⟺ 最小化 E[ g²·(G_t − b)² ]**。把它对标量 b 求导并令其为零（这是一个关于 b 的二次函数，有唯一最小）：

```
d/db  E[ g²·(G_t − b)² ] = E[ g²·2·(G_t − b)·(−1) ] = 0
  ⟹  E[ g²·(G_t − b) ] = 0
  ⟹  b* = E[ g²·G_t ] / E[ g² ]      ← 理论最优 baseline：以 g² 为权重的 G_t 加权平均
```

**结论：理论最优的 baseline 是"用 score 模长平方 g² 加权的 reward-to-go 期望"**，是一个被 g² 加权的均值。它和 `E[G_t] = V(s_t)`（无权重的均值）不完全相等，但**非常接近**——这就是为什么实践中直接取 **b(s_t) = V^π(s_t)（状态价值函数，即"在 s_t 处按当前策略往后能拿到的平均回报"）** 作为 baseline，效果已经很好，且有清晰的语义。它把"这个状态本身好不好"的部分从回报里扣掉，剩下的才是"这个动作相对该状态平均水平的增量"。

直觉再说一遍：**回报 G_t 里混了两部分信息——"我处在一个好状态(s_t 本身回报就高)"和"我在这状态里选了个好动作"。我们只想用后者来调整策略，因为前者不是这个动作的功劳。减去 V(s_t) 正好把"状态本身好不好"这层共因扣掉，留下纯粹归功于动作的信号，噪声(方差)自然小。**

### 4.4 优势函数 advantage：策略梯度的标准形式

取 `b(s) = V^π(s)`，定义**优势函数(advantage function)**：

```
A^π(s,a) = Q^π(s,a) − V^π(s)
```

它度量"在状态 s 选动作 a，比起在 s 按当前策略平均地随便选，好多少"。`A > 0` 表示这个动作优于平均，`A < 0` 表示劣于平均。注意 `V^π(s) = E_{a~π}[Q^π(s,a)]`，所以**优势的期望恒为零**：`E_{a~π}[A^π(s,a)] = 0`——平均而言没有动作"超过平均"，这正是 baseline 把恒正信号"居中"的体现。

把 G_t（它是 Q 的无偏估计）减去 V(s_t)，得到**策略梯度定理的优势形式**（最常用的形式，PPO/A2C/GRPO 全用它）：

```
∇_θ J(θ) = E_{s ~ d^π, a ~ π_θ} [ ∇_θ log π_θ(a|s) · A^π(s,a) ]
```

**这是工业界策略优化的标准表达式，值得背下来。** 它把"按回报加权"升级成"按优势加权"——只朝着"比平均好的动作"使劲，方差小、信号纯。后面 05 章的 GAE 就是在精细地、低方差地估这个 `A^π(s,a)`（在偏差和方差之间用 λ 做插值）；06 章 PPO 把 `∇log π · A` 这个目标包上一层 clip 防止步子太大；11 章 GRPO 干脆用"同一 prompt 采一组答案、组内标准化"来估 A，省掉训 critic 的开销。一条主线贯穿全部后续章节。

## 五、REINFORCE 算法：把定理变成可跑的代码

把上面的 reward-to-go + baseline 形式落地，就是经典的 **REINFORCE 算法**（Williams, 1992；REINFORCE 是 "REward Increment = Nonnegative Factor × Offset Reinforcement × Characteristic Eligibility" 的缩写，但记住它是"蒙特卡洛策略梯度"就够了）。它是最朴素、最纯粹的策略梯度算法——**用整条采样轨迹的蒙特卡洛回报当梯度权重，不用任何 bootstrap/critic（带 baseline 时 baseline 可以是一个简单的拟合 V，也可以是常数均值）。**

完整流程（带 reward-to-go 与 value baseline）：

```
REINFORCE with baseline（蒙特卡洛策略梯度）
─────────────────────────────────────────────
初始化策略参数 θ，价值(baseline)参数 φ
循环：
  1. 用当前策略 π_θ 与环境交互，采 N 条完整轨迹 {τⁱ}   ← on-policy：数据必须来自当前 θ
  2. 对每条轨迹的每个时刻 t，算 reward-to-go：
        Gₜⁱ = Σ_{k≥t} γ^{k-t} · rₖⁱ
  3. 拟合 baseline（回归 V_φ 去逼近 Gₜ，最小化 (V_φ(sₜ) − Gₜ)² ）
  4. 算优势估计：     Âₜⁱ = Gₜⁱ − V_φ(sₜⁱ)
  5.（可选但强烈推荐）对一个 batch 内所有 Âₜⁱ 做标准化：Â ← (Â − mean) / (std + ε)
  6. 策略梯度（注意是上升，所以损失取负）：
        L(θ) = − (1/N) Σᵢ Σₜ log π_θ(aₜⁱ|sₜⁱ) · Âₜⁱ
        θ ← θ + α · ∇_θ (−L)   （即对 L 做下降）
  7. 更新 baseline：φ ← φ − α_v · ∇_φ Σ (V_φ(sₜ) − Gₜ)²
```

PyTorch 风格的核心实现片段（只写体现机制的部分，省略 env/网络定义噪声）：

```python
import torch

def reward_to_go(rewards, gamma):
    # rewards: 一条轨迹的逐步奖励 list[float]，返回每步的 reward-to-go
    G = torch.zeros(len(rewards))
    running = 0.0
    for t in reversed(range(len(rewards))):   # 从后往前累加，O(T) 而非 O(T²)
        running = rewards[t] + gamma * running
        G[t] = running
    return G

def reinforce_update(policy, value_fn, opt_pi, opt_v, trajectories, gamma):
    # trajectories: list of dict，每条含 states / actions / rewards
    all_logp, all_adv, all_v_target, all_states = [], [], [], []

    for traj in trajectories:
        states  = traj["states"]          # [T, obs_dim]
        actions = traj["actions"]         # [T]（离散）或 [T, act_dim]（连续）
        G = reward_to_go(traj["rewards"], gamma)   # [T]，蒙特卡洛回报，Q 的无偏估计

        dist  = policy(states)            # 一个分布对象（Categorical 或 Normal）
        logp  = dist.log_prob(actions)    # [T] = log π_θ(aₜ|sₜ)，这是 score 的来源
        v     = value_fn(states).squeeze(-1)        # [T] = baseline V_φ(sₜ)

        adv = G - v.detach()              # 优势 Â = G − V；detach！baseline 的梯度不能流进策略损失
                                          # （baseline 只通过 value loss 学，混进来会破坏无偏的推导前提）
        all_logp.append(logp); all_adv.append(adv)
        all_v_target.append(G); all_states.append(states)

    logp = torch.cat(all_logp)            # [ΣT]
    adv  = torch.cat(all_adv)
    adv  = (adv - adv.mean()) / (adv.std() + 1e-8)   # batch 内标准化，进一步压方差（引入轻微偏差，工程常用）

    # 策略损失：梯度上升 → 损失取负号；这一项的梯度恰好是 −∇log π·Â，下降它即上升 J
    policy_loss = -(logp * adv).mean()

    opt_pi.zero_grad(); policy_loss.backward(); opt_pi.step()

    # baseline 用 MSE 回归到蒙特卡洛回报（独立的优化目标）
    v_pred = value_fn(torch.cat(all_states)).squeeze(-1)
    value_loss = torch.nn.functional.mse_loss(v_pred, torch.cat(all_v_target))
    opt_v.zero_grad(); value_loss.backward(); opt_v.step()
```

实现里三个**必须看懂的机制点**：

1. **`dist.log_prob(actions)` 就是 `log π_θ(a|s)`，反向传播它得到的就是 score `∇_θ log π_θ`。** PyTorch 的 `Categorical`（离散）/`Normal`（连续）分布对象帮你把这一步自动微分了——这就是"我们能对策略求导、却不用对环境求导"在代码里的样子。

2. **`G - v.detach()`：baseline 必须 detach。** 推导（4.2）成立的前提是 baseline 只依赖状态、且其取值在对 a 求期望时是常数。如果让 V_φ 的梯度通过 advantage 流进策略损失，就违背了这个前提、引入偏差，还会造成两个目标互相干扰。baseline 只通过它自己的 `value_loss` 去学。

3. **梯度上升 = 对 `-(logp*adv)` 做梯度下降。** 这里没有"真正的损失函数"——`-(logp*adv).mean()` 是一个**代理目标(surrogate)**，它本身的数值毫无意义（不是误差），唯一的作用是**它的梯度等于策略梯度的负值**。新手常犯的错是盯着这个 loss 数值看收敛——它会上下乱跳甚至越训越大，这正常；要看的是 episode return。

## 六、量化分析:方差、偏差、复杂度、收敛性

**偏差(bias)。** 不带 baseline 的纯 REINFORCE 用蒙特卡洛 G_t（或 G(τ)）做权重，是 ∇_θ J(θ) 的**无偏估计**（三、四节的恒等变形 + 蒙特卡洛，没有任何近似）。这是它相对 TD / Actor-Critic（用 bootstrap 的 V 估计，有偏）的本质优点。但 batch 内 advantage 标准化（第 5 步）会引入轻微偏差——除以 std 改变了梯度的尺度，严格说不再无偏；工程上几乎总是用，因为方差收益远大于偏差代价。

**方差(variance)。** 这是 REINFORCE 的阿喀琉斯之踵。蒙特卡洛回报 G_t 是 T−t 个随机奖励的累加，其方差**随轨迹长度近似线性增长**（粗略地，若各步奖励方差为 σ²、弱相关，则 `Var[G_t] ≈ O((T−t)·σ²)`，折扣会压缩远期项的贡献）。再加上 score 向量 `∇log π` 自身的随机性，整个梯度估计的方差很大。后果是：**需要很多条轨迹才能让梯度估计稳定，样本效率低。** 三道降方差手段的层次要记清：
- **reward-to-go**（3.3）：删掉无关的过去回报，无偏降方差，免费，必用。
- **baseline V(s)**（4.2–4.3）：扣掉"状态本身好不好"的共因，无偏降方差，强烈推荐。
- **advantage / GAE**（4.4 + 05 章）：用 critic 估优势，方差更小但引入偏差，是 bias-variance 的可调权衡（GAE 的 λ 就是那个旋钮）。

**复杂度。** 单条长度 T 的轨迹：reward-to-go 用从后往前的累加是 `O(T)`（新手常误写成对每个 t 重新累加 → `O(T²)`，长序列下灾难，LLM 的 T 可达数千）。一次梯度计算是 N 条轨迹的前向反向，`O(N·T·|θ前向成本|)`，与监督学习一个 minibatch 同量级。**主要开销不在算梯度，而在采样**——每次更新都要用当前策略重新与环境交互（on-policy），这是策略梯度方法样本昂贵的根源。

**为什么必须 on-policy(同策略)。** 策略梯度定理里的期望 `E_{τ~π_θ}` 和 `E_{s~d^π}` 都是**对当前策略 π_θ 诱导的分布**取的。一旦 θ 更新，这个分布就变了，旧策略采的轨迹不再服从新分布，直接拿来用就是有偏的。**所以朴素策略梯度每更新一步就得扔掉旧数据重新采样——这是它样本效率低的根本原因，也是 06 章 PPO 引入"重要性采样比 r_t(θ) = π_θ/π_old + clip"想缓解的问题：在小范围内复用旧数据多更新几步。**

**收敛性。** 策略梯度是对 J(θ) 的**随机梯度上升**。在标准随机近似条件下（学习率满足 `Σα_t = ∞, Σα_t² < ∞`，梯度估计无偏且方差有界），它收敛到 J(θ) 的一个**驻点(stationary point)**——注意是**局部最优 / 驻点，不是全局最优**（J(θ) 对神经网络参数一般非凸）。这跟监督学习里 SGD 的收敛保证是同一类结果，没有更强。**实践含义：策略梯度对初始化、随机种子敏感，不同 run 结果可能差很多；报告结果要跑多个种子(待核:学术界常见 3–10 个种子,具体看会议惯例)。**

## 七、设计权衡与常见坑

**坑 1:把代理目标的 loss 数值当训练信号看。** `-(logp*adv).mean()` 不是误差，它的值乱跳是正常的。**唯一可信的训练信号是 episode return（和它的滑动平均）。** 想监控梯度健康度，看梯度范数、看策略的熵。

**坑 2:advantage 不标准化 → 梯度尺度漂移、训练爆炸或停滞。** 奖励量级随训练变化时，未归一化的 advantage 会让有效学习率隐式漂移。**batch 内做 `(Â−mean)/(std+ε)` 几乎是标配**（代价是轻微偏差，见第六节）。

**坑 3:baseline 不 detach,梯度串台。** 让 V_φ 的梯度流进策略损失，破坏 4.2 无偏证明的前提，两个目标互相污染。代码里 `G - v.detach()` 的 detach 不能漏。

**坑 4:reward-to-go 写成 O(T²)。** 长序列(尤其 LLM)下是性能灾难,且容易把折扣 γ^{k-t} 的指数算错。**从后往前一次累加,O(T)。**

**坑 5:步长太大 → 策略崩溃且无法恢复。** 策略梯度有个价值类没有的恶性正反馈:一步把 θ 更新过头 → 策略变差 → 接下来用这个差策略采的数据更差 → 梯度估计更不可靠 → 越训越崩,且**坏策略采不到好轨迹,没法自我纠正**。这就是 06 章要引入信赖域 / clip 的根本动机。**朴素 REINFORCE 对学习率极其敏感,这是它难以直接用于大模型的首要原因。**

**坑 6:熵塌缩(entropy collapse)/ 过早确定性。** 策略可能过早地对某些动作变得几乎确定(熵趋于 0),从此停止探索、卡在次优。**常见对策:在损失里加一个熵正则项 `+ β·H(π_θ(·|s))` 鼓励策略保持随机性**(详见 14 章稳定性)。

**坑 7:reward hacking(奖励钻空子)——所有 RL 的头号陷阱。** 策略梯度忠实地最大化你给的奖励;如果奖励函数有漏洞,它会找到漏洞而不是你想要的行为(经典例子:赛艇游戏里 agent 原地转圈刷分而不去终点)。**在 RLHF/RLVR 里这尤其致命**:奖励模型是学出来的、不完美,策略会专门攻击 RM 的弱点,输出高分但实际很差的内容。这是后续 08/10/11/14 章反复出现的主题——**奖励设计比算法本身更决定成败**。

## 十、小结与承上启下

这一章我们完成了 RL 课程里一次根本的范式切换:**从"学价值再贪心导出策略"(value-based),转到"直接参数化策略、对期望回报做梯度上升"(policy-based)。** 核心收获:

1. **策略梯度定理**:`∇_θ J(θ) = E_{τ~π_θ}[ Σ_t ∇_θ log π_θ(a_t|s_t) · G_t ]`。它的全部魔法在 **log-derivative trick** `∇p = p·∇log p`——log 把轨迹概率的连乘拆成连加,求导后**环境转移 p(s'|s,a) 和初始分布 ρ₀ 因不含 θ 而归零**,于是我们无需对环境求导、无需知道环境模型,就能做梯度上升。这是 model-free RL 的数学根基。
2. **三层降方差**:reward-to-go(因果性,无偏)→ baseline V(s)(无偏,扣掉状态共因)→ advantage `A = Q − V`(策略梯度的标准形式)。其中"任何状态相关 baseline 都不改变梯度期望"由"score 期望恒为零"`E_{a~π}[∇log π] = 0` 保证,而这条又只依赖"概率归一"这一个事实。
3. **REINFORCE**:把定理落地成可跑代码——无偏但高方差、必须 on-policy、对步长极其敏感。

但我们也清楚地看到了**没解决的问题**:(a) 蒙特卡洛 G_t 方差太大、样本效率低;(b) 必须用一个学出来的 baseline,但用什么、怎么估优势才能既低方差又不太偏?(c) on-policy 导致每步都要重新采样,贵;(d) 步长稍大策略就崩。

这四个问题正是接下来两章的主线:**05 章 Actor-Critic 与 GAE** 请来一个 critic 网络去估优势(用 bootstrap 换更低方差,GAE 的 λ 在偏差和方差间插值),系统性地把方差压下来;**06 章 TRPO / PPO** 解决步长与样本复用——用信赖域 / clip 约束每步策略不偏离旧策略太远,在小范围内安全地复用旧数据多更新几步。再往后,**08 章 RLHF** 把"动作=生成 token、奖励=人类偏好模型"代入这套框架,**11 章 GRPO** 则用"组内标准化"巧妙替代 critic baseline。**你在本章建立的 `∇log π · A` 这条主轴,会一路贯穿到课程结尾的对齐与推理训练。** 把这一章的两个证明(策略梯度定理、baseline 无偏)练到能默写,后面所有策略优化方法你都只是在改"A 怎么估、步子怎么控"这两件事而已。

### 练习 1：从零默写策略梯度定理

完整推导链，分三步。

**(a) 把 ∇_θ 推进期望，说明 G(τ) 不显含 θ。**

目标是对随机轨迹的期望：

```
J(θ) = E_{τ ~ p_θ(τ)} [ G(τ) ] = ∫ p_θ(τ) · G(τ) dτ
```

对 θ 求梯度，梯度算子与积分(对 τ 的积分，与 θ 无关)可交换，且 ∇_θ 只作用在显含 θ 的因子上：

```
∇_θ J(θ) = ∇_θ ∫ p_θ(τ) · G(τ) dτ
         = ∫ ∇_θ [ p_θ(τ) · G(τ) ] dτ
         = ∫ [ ∇_θ p_θ(τ) ] · G(τ) dτ          ← G(τ) 不显含 θ，被当常数提出
```

为什么 **G(τ) 不显含 θ**：一条轨迹 τ = (s₀,a₀,…,s_T) 一旦给定，它的回报 G(τ) = Σ_t γ^t r(s_t,a_t) 就是由环境奖励函数 r 在这条确定轨迹上算出的一个确定的数，跟 θ 没有任何函数关系。θ 的全部影响在于"哪些轨迹更容易被采到"，也就是**只藏在分布 p_θ(τ) 里**，不在 G 里。所以 ∇_θ 只落到 p_θ(τ) 上。

(若用离散/求和写法，把 ∫ 换成 Σ_τ，每一步同理。)

**(b) log-derivative trick，把它变回期望形式。**

直接面对 ∇_θ p_θ(τ) 不好处理(p_θ 是一长串连乘，里面有不可导的环境项)。用恒等式：

```
∇_θ p_θ(τ) = p_θ(τ) · ∇_θ log p_θ(τ)
```

证明只用链式法则：∇_θ log p_θ(τ) = (1/p_θ(τ)) · ∇_θ p_θ(τ)，两边乘 p_θ(τ) 即得(要求 p_θ(τ) > 0，对 softmax / 高斯参数化恒成立)。代回：

```
∇_θ J(θ) = ∫ [ ∇_θ p_θ(τ) ] · G(τ) dτ
         = ∫ p_θ(τ) · ∇_θ log p_θ(τ) · G(τ) dτ        ← 代入 trick
         = E_{τ ~ p_θ} [ ∇_θ log p_θ(τ) · G(τ) ]        ← ∫ p_θ(τ)·(·) dτ 就是期望定义
```

质变之处：右边重新是"对 p_θ 的期望"，因此**可以用蒙特卡洛采样无偏估计**——按当前 π_θ 跑 N 条轨迹，对每条算 ∇_θ log p_θ(τ)·G(τ) 再平均即可。trick 的作用是把"对概率本身求导"换成"概率 × 对 log 概率求导"，而 log 会把连乘拆成连加(下一步用)。

**(c) 展开 ∇_θ log p_θ(τ)，逐项归零——全章命门。**

轨迹概率是连乘：

```
p_θ(τ) = ρ₀(s₀) · ∏_{t=0}^{T-1} π_θ(a_t|s_t) · p(s_{t+1}|s_t,a_t)
```

取 log，连乘变连加：

```
log p_θ(τ) = log ρ₀(s₀) + Σ_{t=0}^{T-1} [ log π_θ(a_t|s_t) + log p(s_{t+1}|s_t,a_t) ]
```

对 θ 求梯度，逐项看含不含 θ：

```
∇_θ log p_θ(τ) = ∇_θ log ρ₀(s₀)                  ← ρ₀ 是环境给的初始分布，不含 θ → 梯度 = 0
               + Σ_t ∇_θ log π_θ(a_t|s_t)         ← 这是我们的策略，含 θ → 保留
               + Σ_t ∇_θ log p(s_{t+1}|s_t,a_t)   ← p 是环境转移，不含 θ → 梯度 = 0
```

**为什么 ρ₀ 和 p(s'|s,a) 的梯度是零**：θ 是策略 π_θ 的参数。初始状态分布 ρ₀(s₀) 由环境(任务设定)决定，环境转移 p(s_{t+1}|s_t,a_t) 由环境动力学决定——这两者都是与 θ 无关的常量函数(把它们看成 θ 的函数，就是不含 θ 的常数函数，对 θ 求导即 0)。**正是因为 log 先把连乘拆成了连加，这些不含 θ 的因子才以"加性项"的形式出现，求导后干净地变成零，彼此不耦合。** 若不取 log，∇_θ 作用在连乘上会用乘法法则牵扯出 ∇_θ p(s'|s,a) 这种我们根本算不出的项。

于是：

```
∇_θ log p_θ(τ) = Σ_{t=0}^{T-1} ∇_θ log π_θ(a_t|s_t)
```

代回 (b)，得到**策略梯度定理的轨迹形式**：

```
∇_θ J(θ) = E_{τ ~ π_θ} [ ( Σ_{t=0}^{T-1} ∇_θ log π_θ(a_t|s_t) ) · G(τ) ]
```

**"为什么不需要环境可微"的一句话总结：环境的动力学(ρ₀、p)虽然是不可知、不可导的黑盒，但它们不含策略参数 θ；log-derivative trick 把轨迹概率的连乘拆成连加后，对 θ 求导时这些黑盒项作为常数全部归零，只剩下我们完全可控、处处可微的策略项 ∇_θ log π_θ(a_t|s_t)。** 这就是 model-free 强化学习能成立的数学根基——对世界一无所知，照样能做梯度上升。∎

### 练习 2：证明 baseline 不改变梯度的期望

**目标**：对任意只依赖状态的 b(s)，证明对每个 t

```
E_{τ ~ π_θ} [ ∇_θ log π_θ(a_t|s_t) · b(s_t) ] = 0
```

**第一步：先证基础引理(score 期望为零)。** 对固定状态 s：

```
E_{a ~ π_θ(·|s)} [ ∇_θ log π_θ(a|s) ] = 0
```

证明只用"概率归一"这一个事实：

```
E_{a ~ π_θ(·|s)} [ ∇_θ log π_θ(a|s) ]
  = Σ_a π_θ(a|s) · ∇_θ log π_θ(a|s)
  = Σ_a π_θ(a|s) · ( 1/π_θ(a|s) ) · ∇_θ π_θ(a|s)     ← ∇log f = (1/f)∇f
  = Σ_a ∇_θ π_θ(a|s)                                  ← π_θ(a|s) 约掉
  = ∇_θ Σ_a π_θ(a|s)                                  ← 梯度与有限求和可交换
  = ∇_θ 1                                             ← 概率归一：Σ_a π_θ(a|s) ≡ 1
  = 0                                                 ← 常数的梯度为零
```

命门在最后两步：**无论 θ 怎么变，π_θ(·|s) 始终是合法分布，概率和恒等于常数 1，常数对 θ 的梯度是零。** (连续动作把 Σ 换成 ∫，需"梯度与积分可交换"的正则性条件，高斯等常见参数化满足。)∎(引理)

**第二步：用全期望公式 + 引理证主结论。**

```
E_τ [ ∇_θ log π_θ(a_t|s_t) · b(s_t) ]
  = E_{s_t} [ E_{a_t ~ π_θ(·|s_t)} [ ∇_θ log π_θ(a_t|s_t) · b(s_t) | s_t ] ]   ← 全期望：先固定 s_t 对 a_t 求内层
  = E_{s_t} [ b(s_t) · E_{a_t ~ π_θ(·|s_t)} [ ∇_θ log π_θ(a_t|s_t) | s_t ] ]   ← b(s_t) 与 a_t 无关，提到内层期望外
  = E_{s_t} [ b(s_t) · 0 ]                                                     ← 内层正是引理 = 0
  = 0
```

(全期望公式的合法性：t 时刻的 (s_t, a_t) 边际分布存在，可先对 s_t 之前的所有变量及 s_t 取外层期望，再对 a_t 取条件内层期望。)∎

把它叠加到 reward-to-go 形式上：

```
E [ Σ_t ∇_θ log π_θ(a_t|s_t) · ( G_t − b(s_t) ) ]
  = E [ Σ_t ∇_θ log π_θ(a_t|s_t) · G_t ]  −  Σ_t E[ ∇_θ log π_θ(a_t|s_t) · b(s_t) ]
  = ∇_θ J(θ)  −  0  =  ∇_θ J(θ)
```

所以**减去任意状态相关 baseline 都不改变梯度的期望(无偏)**，只改变方差(练习 3)。

**为什么 b 绝对不能依赖 a？哪一步崩掉？**

崩在第二步的"提出"那一行。如果 b 依赖动作、写成 b(s_t, a_t)，那么它对 a_t **不是常数**，无法提到 `E_{a_t}[·]` 之外：

```
E_{a_t} [ ∇_θ log π_θ(a_t|s_t) · b(s_t, a_t) ]  ≠  b · E_{a_t}[ ∇_θ log π_θ ]
```

此时右边不再化成"b × 0"，引理(score 期望为零)用不上，剩下的是 ∇log π 与 b 的协方差项，一般非零，于是**引入偏差**。

一个细节澄清：用 b(s,a) = Q^π(s,a) 看似"依赖 a"，但它恰好等于 V(s) + A(s,a)，其中 A 那部分对 a 的期望为零，所以减 Q 等价于减 V 再额外用 A 当权重——结果是合法的优势形式(见 4.4)，并不破坏无偏。真正危险的是用一个**与 score function 本身相关、又依赖 a 的随意量**当 baseline，那会让上式的协方差项不为零，破坏无偏。这就是"b 只能依赖状态"这条铁律的来由。

### 练习 3：定量解释为什么 advantage 降方差

**(a) 减常数 baseline 后估计量的方差，求最优 b*。**

固定时刻 t，记 score 向量 g = ∇_θ log π_θ(a_t|s_t)，估计量为 g·(G_t − b)。先看它的方差(对标量分量、或对各分量取 trace 看整体，思路一致)：

```
Var[ g·(G_t − b) ] = E[ g²·(G_t − b)² ] − ( E[ g·(G_t − b) ] )²
```

关键观察：**第二项(一阶矩的平方)与 b 无关**——由练习 2，E[g·(G_t−b)] = E[g·G_t] − b·E[g] = E[g·G_t] − 0，因为 E[g] = E[∇log π] = 0(score 期望为零)。所以减 b 不动一阶矩，**最小化方差 ⟺ 最小化二阶矩 E[g²·(G_t−b)²]**。

把二阶矩看成关于标量 b 的函数 f(b) = E[g²(G_t−b)²]，展开是 b 的二次式，且二次项系数 E[g²] ≥ 0，所以是开口向上的抛物线，有唯一最小。对 b 求导并令零：

```
f'(b) = E[ g²·2·(G_t − b)·(−1) ] = −2·E[ g²·(G_t − b) ] = 0
  ⟹  E[ g²·G_t ] − b·E[ g² ] = 0
  ⟹  b* = E[ g²·G_t ] / E[ g² ]
```

验算二阶导：f''(b) = 2·E[g²] > 0，确为极小。

**结论：最优常数 baseline 是"以 score 模长平方 g² 为权重的 reward-to-go 加权平均" b* = E[g²·G_t] / E[g²]。**

**为什么它接近 V(s) 而非恰好等于**：状态价值 V(s_t) = E[G_t | s_t] 是 **g² 权重全为 1 时**的普通(无权)均值。b* 则是用 g² 加权的均值。两者只在"g² 与 G_t 不相关"时才严格相等：

```
若 Cov(g², G_t) = 0:  E[g²·G_t] = E[g²]·E[G_t]  ⟹  b* = E[G_t] = V(s_t)
```

实际中 g² 与 G_t 弱相关，b* 与 V(s) 有偏差但很接近；而 V(s) 有清晰语义("状态本身的平均好坏")、容易用一个 critic 回归估计，所以工程上直接取 **b(s_t) = V^π(s_t)** 作 baseline，已经几乎拿到全部降方差收益。

**(b) 直觉论证：两层降方差。**

**为什么 reward-to-go(G_t) 比总回报 G(τ) 方差小：** t 时刻的 score 项 ∇log π_θ(a_t|s_t) 在总回报形式里乘的是 G(τ) = (r₀+…+r_{t-1}) + G_t，前半段是 **a_t 之前**就已经发生、与 a_t 无任何因果关系的奖励。由练习 2 那类引理，这些过去项乘 score 后**期望为零**(对它们不负责)——也就是说它们对梯度均值毫无贡献，却把一堆与决策无关的随机奖励塞进权重，纯粹放大噪声。换成 G_t 后，权重只保留"t 之后、由 a_t 实际影响的"那段回报，是更干净、数值更小的随机量，方差更低。这是**免费的降方差(完全不引入偏差)**。

定量旁证：若各步奖励方差约 σ²、弱相关，则 Var[G(τ)] ≈ O(T·σ²)，而 Var[G_t] ≈ O((T−t)·σ²) ≤ Var[G(τ)]，靠后的 t 削减尤其明显。

**为什么 advantage(G_t − V(s_t)) 比 reward-to-go 还小——两层信息分解：**

回报 G_t 里混了两层信息：

```
G_t  =  V(s_t)              +  ( G_t − V(s_t) )
        └─ 状态本身好不好       └─ 这个动作相对该状态平均水平的增量(优势)
         (跟 a_t 选得好不好无关)
```

第一层 V(s_t) 是"我恰好身处一个回报本就高的状态"带来的，**不是这个动作的功劳**，对所有动作都一样(它是 a_t 的共因/公共偏移)，却随状态在轨迹间剧烈波动，是 G_t 方差的主要来源之一。我们调整策略时只想用第二层——纯粹归功于"在这个状态选了这个动作"的增量信号。减去 V(s_t) 正好把第一层共因扣掉(其期望由练习 2 保证不动梯度均值)，把恒正的回报"居中"成有正有负的优势：比平均好的动作 A>0 往上推、比平均差的 A<0 往下压，相互抵消使估计量幅度变小、方差进一步下降。

层级总结(方差逐级下降，前两级免费/无偏)：

```
G(τ) 总回报   →  G_t reward-to-go   →   A = G_t − V(s_t) 优势
（含无关过去）   （删过去，无偏降方差）    （再扣状态共因，无偏降方差）
```

### 练习 4：实现并对比 REINFORCE 三种变体

下面是一份 **CPU 即可运行** 的完整脚本，在 CartPole-v1 上对比三个版本：① 纯 G(τ) 总回报、② reward-to-go、③ reward-to-go + value baseline + advantage 标准化。三者**共用同一份训练循环**，只在"权重那一项"分叉。脚本对 5 个种子取均值±std，画出学习曲线(return vs 环境步数)与梯度范数曲线。

依赖与运行说明：`pip install torch gymnasium matplotlib numpy`(CartPole 用纯 CPU 几分钟内跑完，无需 GPU)。`gymnasium` 若用旧版 `gym`，把 `import gymnasium as gym` 改回 `import gym` 并把 `env.step` 返回值的 5 元组改 4 元组即可。

```python
import numpy as np
import torch
import torch.nn as nn
from torch.distributions import Categorical
import gymnasium as gym
import matplotlib.pyplot as plt

# ---------------- 网络：策略 + 价值(baseline) ----------------
class Policy(nn.Module):
    def __init__(self, obs_dim, n_act, hidden=64):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(obs_dim, hidden), nn.Tanh(),
            nn.Linear(hidden, n_act))           # logits
    def forward(self, x):
        return Categorical(logits=self.net(x))  # 返回分布对象，自动微分 log_prob

class Value(nn.Module):
    def __init__(self, obs_dim, hidden=64):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(obs_dim, hidden), nn.Tanh(),
            nn.Linear(hidden, 1))
    def forward(self, x):
        return self.net(x).squeeze(-1)

# ---------------- reward-to-go：从后往前 O(T) 累加 ----------------
def reward_to_go(rewards, gamma):
    G = np.zeros(len(rewards), dtype=np.float32)
    running = 0.0
    for t in reversed(range(len(rewards))):     # 关键：O(T) 而非 O(T^2)
        running = rewards[t] + gamma * running
        G[t] = running
    return G

# ---------------- 采一个 batch 的若干完整轨迹 ----------------
def collect(env, policy, gamma, batch_steps=2000):
    S, A, R_full, R_togo, ep_returns, lengths = [], [], [], [], [], []
    steps = 0
    while steps < batch_steps:
        s, _ = env.reset()
        ep_r = []
        done = False
        while not done:
            st = torch.as_tensor(s, dtype=torch.float32)
            with torch.no_grad():
                a = policy(st).sample().item()
            s2, r, term, trunc, _ = env.step(a)
            done = term or trunc
            S.append(s); A.append(a); ep_r.append(r)
            s = s2; steps += 1
        ep_ret = float(sum(ep_r))               # G(τ)：整条轨迹总回报
        R_full.extend([ep_ret] * len(ep_r))     # 变体①：每步权重 = 总回报
        R_togo.extend(reward_to_go(ep_r, gamma)) # 变体②③：reward-to-go
        ep_returns.append(ep_ret); lengths.append(len(ep_r))
    return (torch.as_tensor(np.array(S), dtype=torch.float32),
            torch.as_tensor(np.array(A), dtype=torch.long),
            torch.as_tensor(np.array(R_full), dtype=torch.float32),
            torch.as_tensor(np.array(R_togo), dtype=torch.float32),
            float(np.mean(ep_returns)))

# ---------------- 一次更新：三变体只差 weight 那一项 ----------------
def update(variant, policy, value, opt_pi, opt_v, batch):
    S, A, R_full, R_togo, _ = batch
    dist = policy(S)
    logp = dist.log_prob(A)                      # log π_θ(a|s)，score 来源

    if variant == "full_return":                 # ① 纯总回报 G(τ)
        weight = R_full
    elif variant == "reward_to_go":              # ② reward-to-go
        weight = R_togo
    elif variant == "baseline_adv":              # ③ rtg + value baseline + 标准化
        v = value(S)
        adv = R_togo - v.detach()                # baseline 必须 detach
        weight = (adv - adv.mean()) / (adv.std() + 1e-8)  # batch 内标准化
    else:
        raise ValueError(variant)

    # 代理目标：梯度上升 → 取负号做下降
    policy_loss = -(logp * weight).mean()
    opt_pi.zero_grad(); policy_loss.backward()
    grad_norm = torch.nn.utils.clip_grad_norm_(policy.parameters(), 1e9)  # 只测范数，不裁剪
    opt_pi.step()

    if variant == "baseline_adv":                # baseline 用 MSE 回归到 rtg，独立目标
        v_pred = value(S)
        value_loss = nn.functional.mse_loss(v_pred, R_togo)
        opt_v.zero_grad(); value_loss.backward(); opt_v.step()

    return float(grad_norm)

# ---------------- 单次完整训练(一个种子、一个变体) ----------------
def train_one(variant, seed, n_updates=120, batch_steps=2000, gamma=0.99):
    torch.manual_seed(seed); np.random.seed(seed)
    env = gym.make("CartPole-v1")
    env.reset(seed=seed)
    obs_dim = env.observation_space.shape[0]
    n_act = env.action_space.n
    policy, value = Policy(obs_dim, n_act), Value(obs_dim)
    opt_pi = torch.optim.Adam(policy.parameters(), lr=3e-3)
    opt_v  = torch.optim.Adam(value.parameters(),  lr=1e-2)

    rets, gnorms, xsteps = [], [], []
    total = 0
    for _ in range(n_updates):
        batch = collect(env, policy, gamma, batch_steps)
        total += len(batch[0])
        gn = update(variant, policy, value, opt_pi, opt_v, batch)
        rets.append(batch[4]); gnorms.append(gn); xsteps.append(total)
    env.close()
    return np.array(xsteps), np.array(rets), np.array(gnorms)

# ---------------- 多种子聚合 + 画图 ----------------
def run_all(seeds=(0, 1, 2, 3, 4)):
    variants = ["full_return", "reward_to_go", "baseline_adv"]
    labels = {"full_return": "1) full return G(tau)",
              "reward_to_go": "2) reward-to-go",
              "baseline_adv": "3) rtg + baseline + norm"}
    results = {}
    for v in variants:
        runs_r, runs_g, xs = [], [], None
        for sd in seeds:
            xs, r, g = train_one(v, sd)
            runs_r.append(r); runs_g.append(g)
        results[v] = (xs, np.array(runs_r), np.array(runs_g))

    fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(13, 5))
    for v in variants:
        xs, R, G = results[v]
        mR, sR = R.mean(0), R.std(0)
        ax1.plot(xs, mR, label=labels[v])
        ax1.fill_between(xs, mR - sR, mR + sR, alpha=0.2)
        ax2.plot(xs, G.mean(0), label=labels[v])
    ax1.set(xlabel="env steps", ylabel="episode return (mean +/- std over seeds)",
            title="Learning curves"); ax1.legend(); ax1.grid(alpha=0.3)
    ax2.set(xlabel="env steps", ylabel="policy grad norm",
            title="Gradient norm"); ax2.legend(); ax2.grid(alpha=0.3)
    plt.tight_layout(); plt.savefig("reinforce_variants.png", dpi=120)
    print("saved reinforce_variants.png")

if __name__ == "__main__":
    run_all()
```

**机制要点(对应正文)：**
- 三个版本**唯一的差别**就是 `update()` 里 `weight` 那一行：① 用整条轨迹总回报、② 用 reward-to-go、③ 用减去 detached baseline 并标准化后的优势。训练循环、采样、网络完全复用——这正是正文"三者只差权重那一项"的体现。
- `reward_to_go()` 从后往前一次累加是 **O(T)**；初学者写成对每个 t 重新循环求和会变 O(T²)，长序列灾难。
- 变体③里 `R_togo - v.detach()`：**baseline 必须 detach**，否则 V 的梯度会窜进策略损失，破坏练习 2 的无偏前提。baseline 只通过它自己的 `value_loss`(MSE 回归到 rtg)学习。
- `policy_loss = -(logp * weight).mean()` 是**代理目标**，数值本身无意义、会乱跳；唯一可信的训练信号是 `episode return`(图一的纵轴)。
- `clip_grad_norm_(..., 1e9)` 这里阈值取极大，只为**测量**梯度范数、不真正裁剪——用于画图二对比三变体的梯度幅度。

**预期定量结论(典型现象，具体曲线随机器/版本略变，标「待核」)：**
- **学习曲线抖动 / 方差逐级下降**：① 总回报版抖动最剧烈、上升最慢、种子间 std 最大；② reward-to-go 明显更平滑、更快(免费降方差，无偏)；③ baseline+标准化版最平滑、最快达到接近 500 的满分回报、种子间 std 最小。
- **样本效率逐级提升**：达到同一回报阈值(如均值 ≥195)所需的环境步数，③ < ② < ①。
- **梯度范数**：① 的 grad norm 最大且波动最剧烈(权重恒正、量级大)；② 因权重更小更干净而下降；③ 因 advantage 标准化把尺度固定在 ~1，grad norm 最稳定、不随回报量级漂移。
- 这与正文三层降方差(reward-to-go → baseline → advantage)的预测完全一致。**公平对比的前提是固定同一组种子取均值±std**——单种子曲线说明不了问题，因为策略梯度对种子敏感。
