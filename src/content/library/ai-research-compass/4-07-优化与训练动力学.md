> 这一章把你从"知道大模型用 AdamW、要 warmup、会偶尔训崩"带到"能从梯度尺度差异推出为什么必须用自适应优化器、能手推 Adam 的偏差校正每一步、能说清 AdamW 与 L2 正则在数学上不是一回事、能从残差累积的方差爆炸推出初始化的 1/√(2L) 缩放、并能在看到一根 loss spike 时按机制定位是数值溢出、坏 batch、学习率过大还是注意力 logit 爆炸"。学完你能自己写出一个数值正确、能稳定收敛的 AdamW + warmup-cosine 训练循环,并在面对发散时知道先看哪个量。

## 为什么要单独讲"怎么把损失降下去"

前面几章你已经把 Transformer 的**前向**讲透了:注意力怎么算、残差和 LayerNorm 怎么搭、参数量和显存怎么估。但一个能正确前向的模型,离一个**能训出来**的模型还隔着一条又深又黑的沟。这条沟里全是尸体:同样的架构、同样的数据,换个优化器收敛速度差三倍;学习率调大一点点,loss 在第 2000 步突然从 2.1 飙到 NaN;去掉 warmup,前 500 步直接发散;初始化的残差分支不缩放,深层激活的方差随层数线性膨胀,把激活直接顶到 fp16 的溢出边界。

**这些都不是玄学,每一条都有可推的数学机制。** 这一章的任务就是把这些机制一根一根拆开。我们要回答四个问题,而且每个都要推到你能自己复现的程度:

1. **为什么 Transformer 几乎只用 Adam 系优化器,不用裸 SGD?** —— 因为不同参数的梯度尺度可以差几个数量级,单一全局学习率对它们没法同时合适。
2. **Adam 到底在算什么,偏差校正那个 1/(1−βᵗ) 是怎么来的?** —— 它是指数滑动平均在冷启动时系统性偏小的精确修正,可以严格推出来。
3. **为什么 AdamW 把权重衰减从梯度里拎出来是对的,而把它写进梯度(L2 正则)是错的?** —— 因为自适应缩放会把不同参数的权重衰减强度搅成一团乱麻,破坏"衰减"的本意。
4. **训练为什么会崩,怎么稳住?** —— warmup、梯度裁剪、残差缩放初始化,每一个都对应一种确定的崩法。

贯穿全章有一个判断标准,和混合精度那章一脉相承:**一个优化技巧是不是必需,要看去掉它之后是否有一类参数/一段训练阶段会系统性地学不好或学崩。** 能让某类参数走错方向、或让某个阶段必然失稳的,是必需品;只是让收敛快一点点的,是调参。

预备知识:你需要会标量对向量求导(梯度是什么)、知道链式法则、理解期望和方差的基本运算、记得指数滑动平均(EMA)的形式。前面架构章里的残差连接、LayerNorm、注意力 logit `QKᵀ/√d` 会在本章被反复引用。

## 一、从 SGD 到动量:优化的最朴素形态先讲清楚

### 问题:我们到底在解一个什么数学问题

训练就是在求解一个无约束优化问题:

```
min_θ  L(θ) = E_{(x,y)~D}[ ℓ(f_θ(x), y) ]
```

θ 是几亿到几千亿个参数拼成的一个超高维向量,L 是这些参数上的标量损失。我们没法算遍整个数据分布 D 的期望,只能用一个 mini-batch 估计:

```
g_t = ∇_θ L_batch(θ_t)   ← 用一个 batch 估计的随机梯度,是真梯度的无偏估计
                            E[g_t] = ∇L(θ_t),但有方差
```

**这里有两个事实贯穿整章。** 第一,g_t 是**随机**的——它带噪声,噪声方差随 batch size 增大而减小(方差 ∝ 1/B)。第二,L 是**非凸**的——有无数局部极小、鞍点、平坦区和陡峭峡谷。所有优化器设计上的纠结,都来自这两个事实:噪声让你不能完全相信单步梯度,非凸让你不能用任何依赖凸性的保证。

### 推导:SGD 与动量

最朴素的随机梯度下降(SGD):

```
θ_{t+1} = θ_t − lr · g_t
```

它的问题在病态的损失曲面上暴露得最清楚。设想一个二维二次曲面,在两个方向上的曲率(Hessian 特征值)差很多——一个方向陡(λ_max 大)、一个方向平(λ_min 小)。这种曲面的**条件数** κ = λ_max/λ_min 很大,称为病态(ill-conditioned)。SGD 在这种曲面上的行为是:沿陡方向来回震荡,沿平方向龟速爬行。要不震荡,lr 必须 ≤ 2/λ_max(否则陡方向发散);但这样平方向的步长 lr·λ_min 就小到 ≈ 2λ_min/λ_max = 2/κ,收敛慢得和 κ 成正比。

**动量(momentum)就是为了治这个震荡。** 它不直接用当前梯度,而是用梯度的指数滑动平均:

```
v_t = β · v_{t−1} + g_t          ← 动量缓冲,β 通常 0.9
θ_{t+1} = θ_t − lr · v_t
```

直觉:把优化想成一个小球滚下山。SGD 是每一步看一眼当前坡度就走;动量是小球有惯性,过去的速度会累积。在来回震荡的陡方向,相邻几步的梯度方向相反,累加后**相互抵消**;在持续向下的平方向,相邻梯度方向一致,累加后**相互增强**。结果就是震荡被抑制、有效步长被放大。

定量看:在持续同向的方向上,稳态速度是几何级数求和 `v_∞ = g·(1 + β + β² + …) = g/(1−β)`。β=0.9 时这是 **10 倍** 放大。这就是动量在病态方向上"加速"的来源——它把有效学习率沿一致方向放大了 1/(1−β) 倍。

```python
# SGD with momentum 的核心,就这几行
def sgd_momentum_step(params, grads, buffers, lr=0.1, beta=0.9):
    for p, g, v in zip(params, grads, buffers):
        v.mul_(beta).add_(g)        # v = beta*v + g
        p.add_(v, alpha=-lr)        # p = p - lr*v
    # 注意:有的实现写成 v = beta*v + (1-beta)*g(EMA 形式),
    # 那种下面 lr 要相应放大 1/(1-beta) 才等价,别混用。
```

### 分析:动量没解决的那个问题

动量解决了"沿同一方向累积加速、沿震荡方向抵消"。但它**没有**解决一个更根本的问题:**不同参数的合适步长本来就不一样**,而 SGD/动量对所有参数用**同一个全局 lr**。在一个百亿参数的 Transformer 里,embedding 的某一行(只被几个罕见 token 激活)的梯度,和 LayerNorm 的某个 scale 参数的梯度,尺度可以差几个数量级。一个全局 lr 不可能同时对它们都合适——对前者太小学不动,对后者太大就震荡。

这就是自适应优化器登场的地方。

## 二、Adam:为每个参数算一个自适应步长

### 问题:能不能让每个参数有自己的有效学习率

我们想要的是:**梯度一直很大的参数,步子迈小一点(别冲过头);梯度一直很小的参数,步子迈大一点(别原地踏步)。** 也就是说,有效步长应该和该参数梯度的"典型大小"成反比。

怎么估计"梯度的典型大小"?用梯度平方的滑动平均——它衡量的是该参数梯度的二阶矩(未中心化的方差 + 均值平方)。这正是 Adam(Adaptive Moment Estimation,Kingma & Ba, 2015)的核心思想:**同时维护梯度的一阶矩(均值,用于动量方向)和二阶矩(平方均值,用于自适应缩放)。**

### 推导:Adam 的完整更新规则

Adam 维护两个 EMA 缓冲,逐元素(element-wise,即每个参数分量独立)地算:

```
一阶矩(动量):   m_t = β₁ · m_{t−1} + (1−β₁) · g_t        β₁ 默认 0.9
二阶矩(梯度平方):v_t = β₂ · v_{t−1} + (1−β₂) · g_t⊙g_t    β₂ 默认 0.999
```

(⊙ 是逐元素乘,g_t⊙g_t 就是每个分量各自平方。)初始化 m₀ = 0,v₀ = 0。

**关键问题:m₀=v₀=0 这个冷启动会带来系统性偏差。** 我们把 m_t 的递推展开看清楚。设各步梯度近似平稳、期望为 E[g],则:

```
把递推一路展开:
  m_t = (1−β₁) · Σ_{i=1}^{t} β₁^{t−i} · g_i

两边取期望(假设 E[g_i] ≈ E[g] 在窗口内近似不变,可提出来):
  E[m_t] = (1−β₁) · E[g] · Σ_{i=1}^{t} β₁^{t−i}

那个几何级数:Σ_{i=1}^{t} β₁^{t−i} = β₁^{t−1} + β₁^{t−2} + … + β₁⁰
            = (1 − β₁^t) / (1 − β₁)

所以:
  E[m_t] = E[g] · (1 − β₁^t)
```

**看出问题了:E[m_t] = (1−β₁ᵗ)·E[g],而不是 E[g]。** 在 t 很小的早期,β₁ᵗ 接近 1,(1−β₁ᵗ) 接近 0,m_t 被严重低估。t=1 时 E[m₁]=(1−β₁)E[g]=0.1·E[g],只有真值的十分之一!这就是 v₀=m₀=0 把估计往零拽的后果。

**偏差校正(bias correction)就是把这个 (1−β₁ᵗ) 除回去:**

```
m̂_t = m_t / (1 − β₁^t)      ← 除以刚才推出的系统性缩小因子,还原成无偏估计
v̂_t = v_t / (1 − β₂^t)      ← 二阶矩同理,因子是 (1−β₂^t)
```

校正后 E[m̂_t] = E[g],E[v̂_t] ≈ E[g²],偏差消除。注意 t→∞ 时 β₁ᵗ→0,校正因子→1,校正自动失效——它只在早期起作用,这正是我们要的。

**最后是参数更新:**

```
θ_{t+1} = θ_t − lr · m̂_t / (√v̂_t + ε)        ε 默认 1e-8,防除零
```

逐元素看这个更新:分子 m̂_t 是该参数梯度的(去偏)均值方向,分母 √v̂_t 是该参数梯度大小的(去偏)均方根(RMS)。**两者一除,有效步长就和该参数自己的梯度尺度无关了**——梯度大的分母大、步长被压回去,梯度小的分母小、步长被放大。这就是"每个参数自己的自适应学习率"。

更精确地:如果某参数梯度近似常数 g(无噪声),则 m̂≈g,v̂≈g²,更新量 ≈ lr·g/(|g|+ε) ≈ lr·sign(g)。**有效步长被归一化到约 ±lr,与梯度绝对大小脱钩。** 这是 Adam 的灵魂:它把更新步长**规范化**了,所以一个全局 lr 能同时适配尺度差几个数量级的所有参数。

### 为什么 Adam 特别适合 Transformer

把上面那条性质和第一节末尾的痛点接上,就是答案:

- **Transformer 各参数梯度尺度差异极大。** Embedding(尤其稀疏激活的词向量)、attention 投影、FFN、LayerNorm 的 scale/bias,它们的梯度量级天然不同。**待核**:不同来源对"差几个数量级"的具体测量不一,但定性结论一致——SGD 单一 lr 无法同时适配,实践中裸 SGD 训练大 Transformer 极难收敛。
- **Adam 把每个参数的步长归一化到 ~lr 量级**,所以一组超参(lr、β、ε)能跨这些异质参数通用。
- **对稀疏/罕见特征友好。** 一个很久才被激活一次的 embedding 行,二阶矩 v 会衰减得很小,下次它有梯度时分母小、步长大,不会因为长期不更新就学不动。

代价也要讲清楚:Adam 要额外存 **m 和 v 两份和参数同形的状态**(fp32 下每参数 8 字节,叠加 fp32 参数主副本和混合精度的 fp16 参数/梯度,正是混合精度那章"每参数约 16 字节"账本里的两项)。对千亿参数模型,这是几百 GB 的优化器状态,直接逼出 ZeRO/优化器状态分片(那是 MLSys 那门课的内容)。

```python
# Adam 单步,逐行对应上面的公式
def adam_step(p, g, m, v, t, lr=1e-3, b1=0.9, b2=0.999, eps=1e-8):
    m.mul_(b1).add_(g, alpha=1 - b1)            # m = b1*m + (1-b1)*g
    v.mul_(b2).addcmul_(g, g, value=1 - b2)     # v = b2*v + (1-b2)*g*g
    m_hat = m / (1 - b1**t)                      # 偏差校正(t 从 1 开始计)
    v_hat = v / (1 - b2**t)
    p.addcdiv_(m_hat, v_hat.sqrt().add_(eps), value=-lr)  # p -= lr * m_hat/(sqrt(v_hat)+eps)
    # 工程实现常把校正吸收进 lr:lr_t = lr * sqrt(1-b2^t)/(1-b1^t),
    # 然后直接用未校正的 m,v,更省一次张量分配。数学等价。
```

> **一个常见误解先澄清:** Adam 不是"二阶优化器"。它没有用 Hessian。√v̂ 只是梯度平方的滑动平均,是对梯度尺度的一阶统计,不是曲率。它有时被叫"对角预条件(diagonal preconditioning)",因为效果上像用一个对角矩阵 diag(1/√v̂) 预条件了梯度,但这个对角阵来自梯度二阶矩,不是 Hessian 对角线。把 Adam 说成二阶方法是错的。

## 三、AdamW:权重衰减为什么必须从梯度里拎出来

### 问题:L2 正则和权重衰减,在 SGD 里一样,在 Adam 里不一样

先看一个几乎所有人都以为等价、但在自适应优化器下**不等价**的东西。

**权重衰减(weight decay)** 的本意:每一步把参数往零拉一点,θ ← (1−λ)·θ,防止权重无限增长、起到正则作用。

**L2 正则** 的做法:在损失里加一项 (λ/2)·‖θ‖²,它的梯度是 λ·θ,加进总梯度里。

在**纯 SGD**下,这俩完全等价:

```
SGD + L2:   θ ← θ − lr·(g + λθ) = θ − lr·g − lr·λ·θ = (1 − lr·λ)·θ − lr·g
                                                        ↑ 这一项正是权重衰减
```

所以历史上大家混着叫,代码里 `weight_decay` 参数其实是把 λθ 加进梯度——在 SGD 时代这没毛病。

**但在 Adam 里,灾难发生了。** 如果你把 λθ 加进梯度 g,那么这个 λθ 会**一起过自适应缩放**:

```
Adam + L2(把 λθ 混进 g):
  g'_t = g_t + λ·θ_t
  然后 m,v 都基于 g' 算,更新 ≈ lr · (g + λθ 的去偏动量) / (√(g+λθ 的二阶矩) + ε)

问题:权重衰减项 λθ 被除以了 √v̂(每个参数各自的梯度 RMS)!
```

### 推导:为什么这破坏了权重衰减的本意

权重衰减想要的是:**每个参数都按同一个相对比例 λ 往零收缩**,θ ← (1−lr·λ)θ。这个收缩强度应该和"这个参数的梯度有多大"**无关**——大权重该收、小权重该收,一视同仁。

但 Adam+L2 把 λθ 塞进分母 √v̂ 里除了一下。结果是:

```
有效衰减强度 ∝ λ / (√v̂ + ε)

→ 梯度一直很大的参数(√v̂ 大):分母大,实际被衰减得很轻;
→ 梯度一直很小的参数(√v̂ 小):分母小,实际被衰减得很重。
```

**这完全拧了。** 梯度大往往意味着这个参数正在被强烈地学习、很重要,你却几乎不正则它;梯度小的参数你反而往死里收缩。权重衰减"按统一比例收缩、与梯度尺度解耦"的本意被自适应缩放彻底搅乱。更糟的是,实际衰减强度变成了 v̂ 的函数,而 v̂ 在训练中一直变,于是有效正则强度变得不可控、不可解释。

**AdamW(Loshchilov & Hutter, 2017/2019)的修正干净利落:把权重衰减从梯度里彻底拎出来,直接作用在参数上,绕过自适应缩放。**

```
AdamW 更新:
  m_t = β₁·m_{t−1} + (1−β₁)·g_t          ← g_t 里【不】含 λθ
  v_t = β₂·v_{t−1} + (1−β₂)·g_t⊙g_t       ← 同样不含
  m̂_t = m_t/(1−β₁^t),  v̂_t = v_t/(1−β₂^t)
  θ_{t+1} = θ_t − lr·( m̂_t/(√v̂_t + ε) + λ·θ_t )
                       └── 自适应那部分 ──┘   └─ 衰减独立加 ─┘
```

注意最后一行:自适应更新 m̂/(√v̂+ε) 和衰减项 λθ **是相加的两个独立项**,衰减项**没有**被 √v̂ 除。这样每个参数的衰减强度都是统一的 lr·λ,和它的梯度尺度无关——权重衰减的本意被还原了。

> **这是一个 spec 的 NL 含义和 code 实现脱节的经典案例**:文档说"weight decay",你按 L2 实现塞进梯度,在 SGD 下对、在 Adam 下语义就变了。用户读到 `weight_decay=0.1` 以为是"统一收缩",代码却算成了"按梯度尺度反比收缩"。**实现带语义负载的字段前,先问"用户读到这个数会理解成什么、我算的是不是那个含义"。**

### 分析:为什么这个修正在大模型上影响很大

实证上,AdamW 比 Adam+L2 在泛化和可调性上明显更好,这也是为什么**现代大模型几乎全用 AdamW**。具体改善幅度依模型和任务而定**待核**,但机制是确定的:解耦后权重衰减的强度可独立调节、可解释,且不再随训练阶段漂移。

```python
# AdamW 的 step,和上面 Adam 只差最后那一行衰减
def adamw_step(p, g, m, v, t, lr=1e-3, b1=0.9, b2=0.999, eps=1e-8, wd=0.1):
    m.mul_(b1).add_(g, alpha=1 - b1)
    v.mul_(b2).addcmul_(g, g, value=1 - b2)
    m_hat = m / (1 - b1**t)
    v_hat = v / (1 - b2**t)
    p.addcdiv_(m_hat, v_hat.sqrt().add_(eps), value=-lr)  # 自适应部分
    p.add_(p, alpha=-lr * wd)                              # 解耦的权重衰减:θ -= lr*wd*θ
    # 关键:wd 项【不】经过 m,v,直接作用在 p 上。这一行的位置就是 AdamW 的全部秘密。
```

**一个被广泛采用的工程细节:权重衰减通常只施加在 2D 权重矩阵上,不施加在 1D 参数(LayerNorm 的 scale/bias、各种 bias)上。** 原因:对 LayerNorm 的 scale 做衰减会把它往零压,削弱归一化的表达力;bias 衰减意义也不大。所以实践中会按参数维度分两组,一组 wd>0、一组 wd=0(nanoGPT 的 `configure_optimizers` 正是这么分的,见导读)。

## 四、学习率调度:warmup 与衰减各治一种病

学习率不是一个常数,而是一条随训练步数变化的曲线。一条典型的大模型 lr 曲线分两段:**先线性升(warmup),再缓慢降(cosine/线性衰减)。** 这两段各自治一种确定的病。

### 4.1 为什么需要 warmup:早期 Adam 二阶矩不可信 + Pre-LN 早期梯度大

**warmup** 指训练最初的若干步(典型几百到几千步,占总步数百分之几),把 lr 从 0(或很小值)**线性升到** 峰值 lr。为什么不一上来就用峰值?有两个独立的机制,叠加起来让早期格外脆弱:

**机制一:早期 v̂ 估计不可信,等价学习率被放大。** 回到 Adam 的更新 lr·m̂/(√v̂+ε)。训练刚开始时,v 只累积了几步,样本极少,√v̂ 的估计方差很大。更要命的是冷启动:v₀=0,头几步 v 还没"充满",即便有偏差校正,v̂ 的取值也极不稳定。当某一步 v̂ 偶然偏小,分母就偏小,**那一步的有效步长被放大**,可能把参数一脚踹到很差的区域,而早期参数还在随机初始化附近、损失曲面陡峭,一脚踹偏就可能再也回不来或引发连锁发散。warmup 用很小的 lr 度过这段 v̂ 不可信的窗口,等 v 累积够多个 batch、估计稳定了,再升到峰值。

**机制二:Pre-LN 架构早期深层梯度偏大。** 现代 Transformer 几乎都用 Pre-LN(LayerNorm 放在残差分支输入处,x + Sublayer(LN(x)))。Pre-LN 的好处是训练稳(下一节细讲),但它有个副作用:训练初期,各残差分支的输出还没被学得很小,深层的输出方差随层数累积膨胀(下一节会精确推这个膨胀),反传回来早期梯度偏大、尺度不均。**待核**:不同分析对"Pre-LN 早期梯度行为"的刻画有差异,但 warmup 缓解早期大梯度是社区一致的经验。一个广为引用的解释(Liu et al., 2020, "On the Variance of the Adaptive Learning Rate and Beyond",RAdam 那篇)就把 warmup 的必要性归因于早期自适应学习率的高方差,并提出 RAdam 用解析的方差修正来部分替代 warmup。

两个机制都指向同一个药方:**早期用小 lr 保命,等统计量稳了再放开。**

### 4.2 为什么需要衰减:噪声地板与精细收敛

训练后期 lr 要往下降,典型是 **cosine 衰减**(从峰值按半个余弦曲线降到一个小的末值,常见降到峰值的 10% 或 0)。为什么?

随机梯度有噪声(g_t 是 batch 估计)。在接近极小点时,梯度的真实信号变小,但噪声不会跟着变小——于是参数在极小点附近做一个**布朗运动式的抖动**,抖动幅度大致正比于 lr(更准确地,稳态抖动方差 ∝ lr·噪声方差/B,这是 SGD 作为离散化随机微分方程的经典结果,**待核** 具体常数)。lr 不降,你就永远卡在这个"噪声地板"上下不去。**衰减 lr 等于逐步缩小抖动幅度,让参数能沉降到更精细的极小点。** 这也解释了为什么衰减末段往往能看到 loss 再降一截。

cosine 比线性常被偏好的经验理由:它在中段保持较高 lr(多探索),末段快速压低(精细收敛),形状上更"先充分再收尾"。但这更多是经验,理论上的最优调度仍是开放问题。

```python
import math

def lr_at(step, base_lr, warmup_steps, total_steps, min_lr_ratio=0.1):
    """warmup-cosine:先线性升到 base_lr,再 cosine 降到 base_lr*min_lr_ratio。"""
    if step < warmup_steps:
        return base_lr * step / max(1, warmup_steps)        # 线性 warmup
    # cosine 衰减阶段
    progress = (step - warmup_steps) / max(1, total_steps - warmup_steps)
    progress = min(1.0, progress)                            # 超出总步数后钳在末值
    cosine = 0.5 * (1.0 + math.cos(math.pi * progress))      # 1 → 0
    min_lr = base_lr * min_lr_ratio
    return min_lr + (base_lr - min_lr) * cosine              # base_lr → min_lr
```

### 4.3 梯度裁剪:给单步更新设一个防爆上限

即使有 warmup,偶尔一个坏 batch(异常样本、长尾分布)会产生一个量级远超平常的梯度,一步就把参数顶飞。**梯度裁剪(gradient clipping)按全局范数裁**:把所有参数的梯度拼成一个大向量,算它的 L2 范数,如果超过阈值就整体等比缩放回阈值。

```
g_all = concat(所有参数的梯度)              ← 看成一个超长向量
total_norm = ‖g_all‖₂ = √(Σ_p Σ_i g[p][i]²)  ← 全局 L2 范数
if total_norm > max_norm:
    scale = max_norm / (total_norm + 1e-6)
    g ← g · scale     (对【每一个】参数的梯度都乘同一个 scale)
```

**关键是"全局"而非逐参数。** 全局范数裁剪等比缩放所有梯度,**保持梯度向量的方向不变**,只缩短长度——相当于只对那些异常大的更新临时踩一脚刹车,正常步几乎不受影响(因为正常步的 total_norm 低于阈值,scale=1)。如果改成逐参数各裁各的,会改变不同参数梯度的相对比例,扭曲更新方向。max_norm 常取 1.0(**待核**:这是常见默认,具体随模型而定)。

```python
def clip_grad_global_norm(params, max_norm=1.0):
    total_norm = torch.sqrt(sum(p.grad.detach().pow(2).sum() for p in params if p.grad is not None))
    scale = max_norm / (total_norm + 1e-6)
    scale = min(scale, 1.0)                  # 只缩不放:范数没超阈值就不动
    for p in params:
        if p.grad is not None:
            p.grad.mul_(scale)
    return total_norm                        # 返回裁前范数,这是最重要的训练监控量之一
```

**把裁前的 total_norm 打到日志里**——它是诊断训练健康度的头号指标。正常训练它平稳小幅波动;一旦你看到它突然跳到平时的几十上百倍,那就是一根 loss spike 的前兆(下一节展开)。

## 五、初始化与残差:为什么深层会方差爆炸,以及 1/√(2L) 是怎么来的

这一节是本章数学最硬的一块,也是"为什么深 Transformer 难训"的根。**坏的初始化会让深层激活的方差随层数线性甚至指数膨胀,直接顶到数值上限或让梯度爆炸。** 我们要把这个膨胀精确推出来,再推出修正它的缩放因子。

### 5.1 推导:残差累积导致激活方差线性膨胀

考虑一个 Pre-LN 残差网络的前向。第 ℓ 层:

```
x_{ℓ+1} = x_ℓ + F_ℓ(x_ℓ)        F_ℓ 是第 ℓ 个子层(attention 或 FFN,内含 LN)
```

做一个简化但抓住本质的假设:**每个子层的输出 F_ℓ(x_ℓ) 与输入 x_ℓ 近似不相关,且方差大致为一个常数 σ_F²**(LayerNorm 把输入归一化到单位尺度,再过线性层,初始化下输出方差大致可控为常数,这是合理的一阶近似)。在不相关假设下,和的方差等于方差的和:

```
Var(x_{ℓ+1}) = Var(x_ℓ) + Var(F_ℓ(x_ℓ))
             = Var(x_ℓ) + σ_F²

设 Var(x₀) = σ₀²,一路累加 L 层:
  Var(x_L) = σ₀² + L · σ_F²
```

**结论:激活方差随层数 L 线性增长。** 一个 L=100 层的网络,深层激活方差是浅层的约 100 倍,标准差约 10 倍。这有两个直接危害:

1. **数值危害。** 激活幅度随深度线性放大,深层很容易顶到 fp16 的表示上限(~65504,见混合精度那章),溢出成 inf,训练 NaN。
2. **优化危害。** 输入到每个子层的尺度随深度漂移,LayerNorm 虽然每层重新归一化,但残差主干 x 本身在变大,使得"子层输出相对主干的比例"随深度变小——深层子层的相对贡献被稀释,梯度信号不均,早期训练不稳(这也回连了上一节 warmup 的机制二)。

### 5.2 推导:残差分支缩放 1/√(2L) 怎么压住方差

修正思路:既然每层加 σ_F² 导致线性累积,那就把每个子层的输出**乘一个小因子 α**,让累积可控:

```
x_{ℓ+1} = x_ℓ + α · F_ℓ(x_ℓ)
Var(x_L) = σ₀² + L · α² · σ_F²
```

要让 L 层累积后总方差不爆(比如想让残差分支总贡献和初始方差同量级,Var(x_L) ≈ σ₀² + σ_F² 这一类目标),需要 L·α² = O(1),即:

```
α = O(1/√L)
```

**这就是 1/√L 缩放的来源——它精确抵消了 L 层线性累积里的那个 L。** 具体常数因模型设计而异:

- GPT-2(Radford et al., 2019)的做法:对**残差路径上的投影层**(attention 输出投影、FFN 第二层)的**初始化权重**乘 1/√N,其中 **N = 残差连接的层数 = 2·n_layer**(因为每个 Transformer block 有两个残差相加,attention 一个、FFN 一个)。这就是题目里 **1/√(2·层数)** 的出处。nanoGPT 里这一行明确写着按 `1/sqrt(2 * n_layer)` 缩放(见导读)。
- 缩放作用在初始化的权重标准差上,等价于在训练初期把 α 压到 1/√(2L) 量级,让 5.1 的累积方差从 L·σ_F² 降到约 σ_F²/2 量级,与初始激活同尺度。

```python
# nanoGPT 风格:对残差投影层做 1/sqrt(2*n_layer) 初始化缩放
import math
def init_weights(module, n_layer):
    if isinstance(module, nn.Linear):
        std = 0.02                                    # 基础初始化 std(GPT-2 默认)
        if getattr(module, "_is_residual_proj", False):
            std *= 1.0 / math.sqrt(2 * n_layer)       # ← 残差分支投影:抵消 L 层累积
        nn.init.normal_(module.weight, mean=0.0, std=std)
        if module.bias is not None:
            nn.init.zeros_(module.bias)
    elif isinstance(module, nn.Embedding):
        nn.init.normal_(module.weight, mean=0.0, std=0.02)
# 标记哪些是残差投影:在 block 里构造 c_proj 时设 c_proj._is_residual_proj = True
```

**为什么是初始化阶段做、而不是永久乘 α?** 初始化只是给训练一个数值健康的起点;训练开始后,如果模型需要某个子层贡献大,它可以通过学习把该层权重放大。我们要防的是**初始的随机权重就让深层爆掉**,让训练根本起不来。这也是为什么 Pre-LN + 残差缩放初始化 + warmup 三件套常一起出现——它们都在保护**早期**这段最脆弱的训练阶段。

### 5.3 Pre-LN vs Post-LN:为什么现代模型选 Pre-LN

顺带把架构章可能埋的伏笔接上。原始 Transformer 是 **Post-LN**:`x_{ℓ+1} = LN(x_ℓ + F_ℓ(x_ℓ))`,LayerNorm 在残差相加之后。问题:LN 在主干路径上,反传时梯度每过一层都被 LN 的雅可比缩放一次,深层梯度容易指数级衰减/放大,所以原始 Transformer **离不开 warmup**,去掉就发散。

**Pre-LN**:`x_{ℓ+1} = x_ℓ + F_ℓ(LN(x_ℓ))`,LN 只在残差分支里。好处:**残差主干是一条干净的恒等路径**,梯度可以无衰减地直接从顶层流到底层(这正是残差网络 He et al. 的核心洞见),训练稳得多,对 warmup 的依赖大大降低。代价:就是 5.1 推的那个——主干方差随层数累积膨胀,所以 Pre-LN 模型最后一层之前通常**再加一个 final LayerNorm** 把累积的大方差归一化回来。**现代大模型几乎全用 Pre-LN(常配 RMSNorm)**,代价用残差缩放初始化和 final norm 补掉。

## 六、训练为什么会崩:loss spike 的成因与对策

把前面所有机制攒起来,现在能系统地讲**loss spike**(loss 突然飙升,有时回落、有时直接 NaN 发散)。这是大模型训练最常见也最让人头疼的现象。**每一种成因都对应前面某个机制的失效**,诊断的关键是按机制逐个排查,而不是瞎调参。

### 6.1 四类主要成因(按机制归类)

**成因一:数值溢出(对应第五节 + 混合精度章)。** fp16 范围窄(上限 ~65504),深层激活方差膨胀(5.1)或某个中间量异常大,溢出成 inf,inf 参与运算产生 NaN,NaN 一旦出现就通过梯度污染全部参数,loss 永久 NaN。**对策**:bf16(动态范围和 fp32 一样宽,基本免溢出,这是大模型转向 bf16 的核心理由)、loss scaling(fp16 时)、残差缩放初始化、final norm。

**成因二:坏数据 batch。** 某个 batch 含异常样本(超长重复、乱码、标注错误),产生量级远超平常的梯度,一步把参数顶飞。**对策**:梯度裁剪(第四节,这是这类 spike 的头号防线)、数据清洗、有些工作会跳过梯度范数异常的 batch(skip-step:检测到 total_norm 超阈值就丢弃这一步的更新)。

**成因三:学习率过大(对应第二、四节)。** lr 超过损失曲面局部曲率能承受的上限,更新冲过头,loss 上跳。早期尤其危险(v̂ 不可信 + 曲面陡)。**对策**:warmup、整体调小峰值 lr、cosine 衰减。**判据**:如果 spike 总在 lr 较高的阶段出现、调小 lr 后消失,基本可确认是这类。

**成因四:注意力 logit 爆炸(Transformer 特有)。** 这一类值得单独推一下,因为它是 Transformer 独有的、且很隐蔽。

### 6.2 注意力 logit 爆炸的机制

回忆注意力打分 `softmax(QKᵀ/√d)`。logit 是 qᵢ·kⱼ/√d。如果训练中 Q、K 的某些维度的幅度被学得很大,点积 qᵢ·kⱼ 就可能变得极大。softmax 对**大的输入差**极其敏感:

```
设两个 logit 是 z 和 z+Δ,softmax 后大的那个的概率 ≈ 1/(1 + e^{−Δ})
Δ=10 时已 ≈ 0.99995,Δ=30 时 e^{30}≈1e13,概率饱和到 1,且
反传时 softmax 的雅可比 ∂softmax/∂z = diag(p) − p·pᵀ,在 p 接近 one-hot 时趋近 0
```

后果有两面:**前向上**,logit 过大使 e^z 溢出(fp16 下 z>11 就 e^z>65504 溢出,bf16 也会在更大处溢出),attention 输出 NaN;**反传上**,softmax 饱和成 one-hot,雅可比趋零,梯度消失,该处注意力学不动,而别处可能因为这个不稳定产生大梯度。实践中观察到的现象是:某些注意力头的 logit 在训练中持续增长,最终触发 spike。

**对策(几种,都在压 logit 的尺度):**
- **QK-Norm**:在算点积前对 Q、K 各做一次归一化(如 RMSNorm/L2 归一化),把它们的幅度钉住,点积就不会爆。这是近年大模型(如一些开源大模型)采用的有效手段,**待核** 具体哪些模型默认开启。
- **logit soft-capping**:对 attention logit 过一个 `c·tanh(logit/c)` 把它软钳在 ±c(Gemma 等用过,**待核** 细节),代价是引入额外非线性、可能轻微伤性能。
- **z-loss**:对 softmax 的归一化分母(log-sum-exp,即 logZ)加一个小的正则项 `α·(logZ)²`,把 logit 整体往小拉,防止 log-sum-exp 漂得太大。PaLM 等用过(**待核** 系数)。

### 6.3 诊断流程(把机制变成可执行的排查顺序)

看到 loss spike,按这个顺序查(从最易确认到最难):

```
1. 看 grad total_norm 日志:
   - spike 前 total_norm 是否暴涨? → 是 → 坏 batch 或 logit 爆炸(成因二/四)
                                      → 否 → 可能纯数值/lr 问题
2. 是否出现 NaN/inf?用 fp16? → 是 → 高度怀疑数值溢出(成因一),
                                     先试 bf16 或检查 loss scale 是否过大
3. spike 是否集中在 lr 高的阶段?调小 lr / 加长 warmup 后是否消失?
   → 是 → 学习率过大(成因三)
4. 定位到具体 batch:把触发 spike 的 batch 捞出来人工看,是否异常数据?
   → 是 → 坏 batch(成因二),清洗 + 梯度裁剪 + skip-step
5. 监控各层 attention logit 的 max abs:某些头是否在持续增长?
   → 是 → logit 爆炸(成因四),上 QK-Norm / soft-cap / z-loss
```

**最重要的工程纪律:留 checkpoint + 留监控量。** spike 往往在某一步突然发生,如果你有定期 checkpoint,可以回滚到 spike 前、跳过坏 batch、或临时降 lr 度过;如果你日志里有 grad norm、loss scale、各层激活/logit 的统计,定位会快得多。**裸训练不带这些监控,等于蒙眼开车。**

## 七、设计权衡与常见坑

把全章的权衡和坑集中起来,这是最该反复看的一节。

**优化器选择的权衡。**
- **AdamW 是默认**,因为它对 Transformer 异质梯度尺度鲁棒、超参好调。代价是每参数额外 8 字节状态(m+v,fp32),逼出优化器状态分片。
- **裸 SGD/动量**省状态(动量只多 4 字节),但对大 Transformer 极难收敛,基本不用于预训练。
- **新兴优化器**(如 Lion、Adafactor、Shampoo 等)各有取舍:Adafactor 用因子化二阶矩省显存(把 v 从矩阵压成两个向量),Lion 只存一阶动量更省、用 sign 更新,但调参更敏感。**待核**:这些在超大规模上能否稳定打赢 AdamW 仍有争议,选型前务必小规模验证。

**超参的典型取值与坑(都标待核,务必以你的具体设置和最新实践为准):**
- β₁=0.9, β₂=0.999 是 Adam 默认,但**大模型常把 β₂ 调到 0.95**——更短的二阶矩窗口对预训练后期的梯度尺度变化响应更快,也更稳。**待核** 具体值随模型。
- ε=1e-8 是默认,但 fp16/bf16 下 ε 太小可能在分母被舍入掉,有时调到 1e-6 或更大更稳(混合精度章的数值视角)。
- weight decay 常见 0.1 量级,**只加在 2D 权重上**,别加在 LayerNorm/bias 上。
- warmup 常见占总步数的几个百分点(几百到几千步);grad clip max_norm 常见 1.0。

**最常踩的坑清单:**
1. **把 weight decay 实现成 L2(塞进梯度)还以为和 AdamW 一样。** 在 Adam 下语义全变(第三节)。用框架的 `AdamW` 而非 `Adam(..., weight_decay=)`(后者历史上是 L2 实现,**待核** 各框架行为)。
2. **偏差校正的 t 从 0 开始计数。** t 必须从 1 开始,t=0 时 1−β⁰=0 直接除零。
3. **去掉 warmup 训 Pre-LN/原始 Transformer。** 早期 v̂ 不可信 + 大梯度,大概率前几百步发散。
4. **残差投影不做 1/√(2L) 缩放训深网络。** 深层激活方差爆,顶到 fp16 上限或梯度爆。
5. **不监控 grad norm。** spike 来了不知道为什么、不知道是哪步、回滚都没 checkpoint。
6. **逐参数裁剪梯度而非全局范数裁剪。** 扭曲更新方向。
7. **fp16 不开 loss scaling,或 scale 设死不动态调整。** 小梯度下溢归零(混合精度章)。
8. **batch size 变了 lr 不跟着调。** batch 变大梯度噪声变小,常需相应调大 lr(有 √B 或线性缩放经验法则,**待核** 具体规则)。

## 八、动手练习

**练习 1(推导题,必做)。** 完整推导 Adam 的二阶矩偏差校正因子。已知 `v_t = β₂·v_{t−1} + (1−β₂)·g_t²`,v₀=0。(a) 把递推展开成关于 {g_1², …, g_t²} 的显式求和。(b) 假设各步 E[g_i²] ≈ E[g²] 在窗口内近似不变,求 E[v_t],并证明 E[v_t] = E[g²]·(1−β₂ᵗ)。(c) 由此说明为什么校正因子是 1/(1−β₂ᵗ),并解释 t→∞ 时校正为何自动消失。(d) 进阶:如果梯度实际上不平稳(E[gᵢ²] 随 i 变),这个校正还无偏吗?定性说明偏差方向。
*提示*:(a)(b) 完全照搬正文第二节对 m_t 的推法,把 (1−β₁) 换成 (1−β₂)、g 换成 g²、几何级数 Σβ₂^{t−i} 照算。(d) 想:校正基于"窗口内 E[g²] 不变"的假设,若 g² 正在快速下降(接近收敛),v_t 里还混着早期的大 g²,会偏大还是偏小?

**练习 2(对比分析题,必做)。** 用一个**单参数**的例子,把 AdamW 和 Adam+L2 的有效权重衰减强度算出来并对比。设标量参数 θ,两种情形:(i) 该参数梯度一直很大,√v̂ ≈ 10;(ii) 梯度一直很小,√v̂ ≈ 0.01(忽略 ε)。weight decay 系数 λ=0.1,lr=1e-3。(a) 分别写出 AdamW 和 Adam+L2 在两种情形下,**单步里权重衰减项**对 θ 的实际收缩量(衰减项贡献的 Δθ)。(b) 指出在 Adam+L2 下,情形 (i) 和 (ii) 的有效衰减强度相差多少倍,而 AdamW 下相差多少倍。(c) 用一句话说明这个差异为什么破坏了"权重衰减"的本意。
*提示*:AdamW 衰减项 Δθ = −lr·λ·θ,和 √v̂ 无关。Adam+L2 里 λθ 被并进梯度再除以 (√v̂+ε),衰减项的有效贡献 ≈ −lr·λθ/√v̂。代入两个 √v̂ 比一下,Adam+L2 下两情形差 10/0.01=1000 倍,AdamW 下完全相同。

**练习 3(分析题)。** 用第五节 5.1 的方差累积模型分析一个问题:某人训练一个 48 层 Pre-LN Transformer,fp16,**没做**残差投影缩放(α=1),初始化让每个子层输出方差 σ_F²≈1,初始激活方差 σ₀²≈1。(a) 估算第 48 层(共 96 个残差相加)的激活方差和标准差。(b) 如果激活分量近似高斯,3σ 处的激活幅度大约是多少?离 fp16 上限 65504 还有多远?(c) 现在加上 1/√(2L) 缩放(L=48),重算 (a)(b),说明缩放把方差从多少压到了多少。(d) 这能完全杜绝溢出吗?为什么实践中还要配 bf16 / final norm / grad clip?
*提示*:(a) 96 个残差,Var ≈ 1 + 96·1 = 97,σ≈9.8。(b) 3σ≈30,离 65504 还远,但这只是单分量 3σ,极值随维度和训练推进会更大,且这只是初始化时,训练中权重被学大后 σ_F² 会涨。(c) α²=1/96,Var ≈ 1 + 96·(1/96)·1 = 2,σ≈1.4。(d) 缩放只保初始化健康,训练动态、坏 batch、logit 爆炸仍可能溢出,故需多层防护。

**练习 4(编码题)。** 用 numpy(不用 PyTorch)从零实现 AdamW,在一个**二维病态二次函数** `L(θ) = 0.5·(a·θ₁² + θ₂²)`(取 a=100,条件数 100)上分别跑 SGD、SGD+momentum、AdamW,从同一初始点 θ=(1,1) 出发,画出三者的损失下降曲线和参数轨迹。(a) 观察 SGD 在陡方向(θ₁)的震荡和平方向(θ₂)的龟速。(b) 观察 AdamW 如何通过逐坐标自适应步长把两个方向的有效步长拉平。(c) 试着把 AdamW 的 lr 调大到震荡,体会自适应也有上限。(d) 在 AdamW 里故意把偏差校正去掉(直接用 m,v 不除 (1−βᵗ)),对比前几十步的轨迹差异,亲眼看到冷启动偏差让早期步长偏小。
*提示*:梯度解析可写:∂L/∂θ₁=a·θ₁,∂L/∂θ₂=θ₂。这是确定性梯度(无噪声),正好单独看自适应机制本身。AdamW 在这种各向异性曲面上的优势会非常直观——它把 θ₁ 方向(梯度大)的步长压住、θ₂ 方向(梯度小)的步长放大,两个方向几乎同速收敛。去掉偏差校正后,头几步 m,v 偏小,m/√v 的比值受影响,轨迹起步会和正确版本不同。

## 九、源码 / 论文导读

**论文(读哪部分):**
- **Adam**:Kingma & Ba, 2015, "Adam: A Method for Stochastic Optimization"。重点读 **Algorithm 1**(完整更新规则,逐行对照本章第二节代码)和 **Section 3 "Initialization Bias Correction"**(偏差校正的推导,练习 1 的来源,原文推法和本章一致)。
- **AdamW**:Loshchilov & Hutter, 2017/2019, "Decoupled Weight Decay Regularization"。重点读把 **L2 正则与权重衰减解耦**的那张算法对比图和实验——它直观展示了 Adam 下两者不等价(本章第三节)。
- **残差缩放 / 深网络初始化**:GPT-2 论文(Radford et al., 2019)正文里关于残差投影按 1/√N 缩放初始化的那一句;以及 "Improving Transformer Optimization Through Better Initialization"、Fixup、DeepNet(Wang et al., 2022,"DeepNet: Scaling Transformers to 1,000 Layers")这条线——DeepNet 给出了让千层 Transformer 稳定的精确残差缩放公式,是 5.2 的深化。
- **warmup 与自适应学习率方差**:Liu et al., 2020, "On the Variance of the Adaptive Learning Rate and Beyond"(RAdam)。重点读它把 warmup 必要性归因于**早期自适应学习率高方差**的分析(本章 4.1 机制一)。
- **Pre-LN vs Post-LN**:Xiong et al., 2020, "On Layer Normalization in the Transformer Architecture"。重点看它对两种 LN 位置的**梯度尺度分析**,论证 Pre-LN 为何对 warmup 依赖更低(本章 5.3)。
- **训练不稳定 / spike**:PaLM 论文(Chowdhery et al., 2022)里关于 **loss spike 与 z-loss、skip batch** 的工程描述;OPT-175B 的训练日志(Zhang et al., 2022,附 logbook)是难得的真实 spike 与对策记录。**待核** 具体超参。

**开源实现(读哪个模块):**
- **nanoGPT**(karpathy):读 `model.py` 的 **`_init_weights`** 和构造里对 `c_proj` 按 `1/sqrt(2*n_layer)` 缩放那一段(本章 5.2 代码的原型);读 **`configure_optimizers`** —— 它把参数按维度分成 weight-decay 组(2D)和 no-decay 组(1D),正是第三节末尾讲的纪律,是把"wd 只加 2D"落地的最干净参照。
- **PyTorch**:读 `torch.optim.AdamW` 的 `step`(对照第三节代码看解耦衰减落在哪一行),以及 `torch.nn.utils.clip_grad_norm_`(对照第四节看全局范数裁剪)。也对比 `torch.optim.Adam` 的 `weight_decay` 参数实现,确认它是 L2 式(塞梯度)还是解耦式——**这是坑 1 的现场,亲自去源码里确认你用的框架版本到底怎么实现**。
- **HF Transformers**:读 `optimization.py` 里的 **`get_cosine_schedule_with_warmup`** 和 `get_linear_schedule_with_warmup`(对照第四节调度代码),以及 `Trainer` 里 grad clip 和 lr scheduler 的接线方式,看一个生产训练循环怎么把本章三件套(AdamW + warmup-cosine + grad clip)串起来。
- **x-transformers**(lucidrains):读它对 **QK-Norm**、**residual scaling** 等稳定 trick 的开关实现,是把第六节那些 logit 爆炸对策落地的现成参照。

## 十、小结与承上启下

这一章的核心是四条,每条背后都有可推的数学:

1. **自适应优化是 Transformer 的刚需,不是调参偏好。** 各参数梯度尺度差几个数量级,单一全局 lr 不可能同时合适;Adam 用梯度二阶矩把每个参数的有效步长归一化到 ~lr,一组超参才能通吃异质参数。偏差校正 1/(1−βᵗ) 是 EMA 冷启动系统性偏小的精确修正,能严格推出来。
2. **AdamW 把权重衰减从梯度里拎出来,是数学上的必需而非工程美化。** 衰减项一旦被自适应分母 √v̂ 除过,有效衰减强度就和梯度尺度反相关,彻底拧了"统一比例收缩"的本意。解耦后衰减强度可控、可解释。
3. **学习率调度的两段各治一种确定的病。** warmup 度过早期 v̂ 不可信 + Pre-LN 大梯度的脆弱窗口;衰减缩小末期的噪声地板抖动,让参数沉降到精细极小。梯度裁剪按全局范数给单步设防爆上限,保方向只缩长度。
4. **训崩有确定机制,诊断靠监控不靠玄学。** 残差累积让激活方差随层数线性膨胀(Var(x_L)=σ₀²+L·σ_F²),1/√(2L) 缩放精确抵消那个 L;loss spike 的四类成因——数值溢出、坏 batch、lr 过大、注意力 logit 爆炸——各对应一个失效机制,按 grad norm / NaN / lr 阶段 / 坏 batch / logit max 的顺序逐个排查。

**在整门课里的位置:** 前面的架构章给了你**模型长什么样**(注意力、残差、归一化、初始化的前向结构);这一章给了你**怎么让它真的训出来**——优化器、调度、稳定性。两者是同一枚硬币的两面:5.1 的方差累积、4.1 的 Pre-LN 早期梯度、6.2 的 attention logit 爆炸,全都是架构选择在**优化动力学**上的回响,脱离架构谈优化、或脱离优化谈架构,都只懂一半。

**往后看两条线。** 第一条接 **MLSys / 分布式训练**:本章反复出现的"Adam 每参数 8 字节状态"会在那里被 ZeRO/优化器状态分片拆开,本章的"fp16 溢出 / bf16 免溢出"正是混合精度那章的延续,梯度裁剪的全局范数在数据并行下还要跨设备 all-reduce 才能算对。第二条接 **scaling law 与超参迁移**:本章的 lr、warmup、β₂、batch size 这些超参在不同模型规模间怎么迁移(如 μP / maximal update parametrization 让超参跨规模可转移),是把"小模型调好的超参直接用到大模型"变成可能的关键,也是真正的大规模训练敢一次性投入算力的底气所在。没有这一章,你能在白板上画出一个完美的 Transformer,但亲手训它会在第一周被 NaN、spike 和不收敛反复教育——和上一门课里 RL 训练被熵塌缩教育,是同一种"原理对、动力学崩"的痛,而第一性原理的诊断思路完全相通。
