## 一、动机:为什么要"不用标签"学特征

前面几章我们把视觉的判别式主干(CNN、ResNet、ViT)和它们在检测、分割上的用法拆透了。但有一个我们一直默认、却从没质疑的前提:**这些模型几乎都靠 ImageNet 的 1000 类人工标签做预训练**,然后把学到的主干迁移到下游任务。这个前提有两个根本问题。

**第一,标注昂贵且不可扩展。** ImageNet 的 ~128 万张图、1000 个类别,是几万人时的众包标注成果。但互联网上有数十亿张无标注图像,而真正稀缺、决定模型上限的恰恰是"未标注数据的规模"。如果学好特征必须先有人工标签,那数据规模就被标注预算死死卡住——这与上一门课里"语言模型靠海量无标注文本 scaling"的成功路径完全相反。

**第二,分类标签是一个信息瓶颈。** 一张图被压成一个 1000 类里的某一个 id,丢掉了图像里绝大部分结构信息(物体的姿态、纹理、空间关系、多个物体的共现)。**用一个 10 bit 的标签(log₂1000 ≈ 9.97 bit)去监督一张百万像素的图,监督信号的"信息带宽"低得可怜。** 模型只学到"区分这 1000 类需要的那点特征",对类别定义之外的视觉结构一无所知。

**自监督学习(self-supervised learning, SSL)的核心思想:不向人要标签,而是从数据本身按某个规则自动造出监督信号(这种自造的任务叫 pretext task,代理任务),用它预训练一个通用主干,再迁移到真正关心的下游任务。** "self-supervised"里的 self 就是指"监督信号来自数据自身,不来自外部标注"。

这和上门课讲的语言模型预训练是同一个哲学:语言模型把"预测下一个词"当作自造监督;视觉自监督则要回答一个更难的问题——**图像是连续的高维信号,没有天然的离散 token 序列,该怎么从一张图里凭空造出有用的监督信号?** 这一章就是讲清这个问题的三大主流答案:

1. **对比学习(contrastive learning)**:不去重建像素,而是学一个"哪些图是同一个东西"的相似性结构。代表是 SimCLR、MoCo。
2. **非对比的自蒸馏(non-contrastive / self-distillation)**:连负样本都不要,只拉近正样本——但这立刻引出一个尖锐的理论问题:**为什么模型不直接把所有输入都映射到同一个常数向量(那样正样本距离恒为 0,loss 完美最小)?** BYOL、DINO、SimSiam 是怎么躲开这个"塌缩(collapse)"陷阱的,是本章的智力高峰。
3. **掩码图像建模(masked image modeling, MIM)**:借鉴 BERT 的完形填空,遮住一部分图像让模型重建。代表是 MAE、BEiT。

**给有工程数学基础读者的迁移提示:** 自监督的本质是"用一个不需要标签的目标函数,逼模型学到对下游任务有用的不变性和结构"。难点全在于**设计一个既不平凡(不能被常数解或恒等映射作弊)、又能逼出语义的目标**。后面你会看到,对比学习靠负样本防平凡解,非对比靠精巧的不对称结构防塌缩,掩码建模靠"重建任务本身足够难"防平凡解——三条路防的是同一个敌人:**平凡解(trivial solution)**。

我们先建立一个所有方法共享的几何直觉:特征空间和不变性。

## 二、预备:表示空间、不变性与"什么是好特征"

把一张图 x 经过编码器 f_θ 映射成一个 d 维向量 z = f_θ(x) ∈ ℝ^d。我们想学的"好特征"到底是什么?自监督给出的操作性定义是:

**好特征 = 对"语义无关的变换"不变(invariant),对"语义相关的差异"敏感(discriminative)。**

什么叫语义无关的变换?对同一张猫的照片做随机裁剪、改变颜色、左右翻转、加点模糊——**变换后还是同一只猫,语义没变。这些变换叫数据增强(data augmentation)。** 一个好的特征应该让"同一张图的不同增强视图"映射到相近的位置(不变性),同时让"不同图"映射到分开的位置(可区分性)。

这就给出了自监督最朴素的监督信号构造规则,叫**实例判别(instance discrimination)**:

> 把每一张图自身当成一个"类别"。同一张图的两个随机增强视图互为**正样本对(positive pair)**,任意两张不同图互为**负样本(negative)**。学习目标:拉近正样本对的特征,推开负样本。

注意这里没有用任何人工标签——"类别"就是"图像实例自身的 id",由数据自动给出。N 张图就是 N 个"类别",每个类别只有一个样本(它的各种增强视图)。

这里有一个必须先点破的张力,它贯穿全章:

```
若只"拉近正样本"而不做任何约束:
  最优解是 f_θ(x) ≡ c (常数), 此时所有正样本距离 = 0, loss 达到最小。
  但这个特征对任何下游任务都没用 —— 这就是"塌缩(collapse)"。
```

**所以自监督的全部技术难点,就是在"拉近正样本"的同时,用某种机制阻止模型走捷径塌缩到常数。** 三大范式就是三种不同的"防塌缩"答案。带着这个问题往下看。

数据增强的选择本身极其关键。SimCLR 的消融发现:**随机裁剪(random crop)+ 颜色抖动(color jitter)的组合是最重要的两个增强**;尤其颜色抖动不可或缺,否则模型会走捷径——直接靠"同一张图的两个裁剪共享颜色直方图"来认出正样本,而不去学语义(**这是一个深刻的坑:增强太弱,模型会找到比'学语义'更省力的作弊路径**)。记住这个直觉:**增强定义了"我们要求模型对什么不变",增强设计错了,学到的不变性就是错的。**

## 三、对比学习:把"认出正样本"做成一个分类问题

### 3.1 从相似度到 InfoNCE:完整推导

对比学习的目标函数叫 **InfoNCE**(Noise-Contrastive Estimation 的信息论变体,由 Oord et al. 2018 在 CPC 工作里提出)。我们从第一性原理把它推出来,而不是直接抛公式。

设一个 query 向量 q(某个增强视图的特征),它对应一个正样本 key k⁺(同一张图另一个视图的特征),以及若干负样本 key k₁, k₂, …(其他图的特征)。把正样本和所有负样本放在一起,记 key 集合为 {k₀, k₁, …, k_K},其中恰好有一个是正样本 k⁺。

**核心想法:把"在这一堆 key 里认出哪个是正样本"变成一个 (K+1) 类的分类问题。** 这正好是 softmax 分类的形式。我们需要一个"相似度打分"作为 logits。用**点积相似度**(向量先做 L2 归一化,即除以自己的模长,使其落在单位球面上,这时点积等于余弦相似度):

```
sim(q, k) = q · k / (‖q‖ · ‖k‖)        (余弦相似度, 范围 [-1, 1])
归一化后(‖q‖=‖k‖=1): sim(q, k) = q · k
```

把相似度除以一个**温度(temperature)τ > 0** 当作 logits,做 softmax,得到"q 与第 i 个 key 配对"的概率:

```
p(i | q) = exp(sim(q, kᵢ) / τ) / Σⱼ exp(sim(q, kⱼ) / τ)
```

我们希望模型把概率质量都压到正样本 k⁺ 上,即最大化 p(正样本 | q),等价于最小化它的负对数似然。设正样本下标为 +,得到**单个 query 的 InfoNCE 损失**:

```
L(q) = - log [ exp(sim(q, k⁺) / τ) / Σ_{i=0}^{K} exp(sim(q, kᵢ) / τ) ]
```

写开:

```
L(q) = - sim(q, k⁺)/τ + log Σ_{i=0}^{K} exp(sim(q, kᵢ)/τ)
       └── 拉近正样本 ──┘   └──── 推开所有 key(含正样本)的 log-sum-exp ────┘
```

**这就是完整的 InfoNCE。它本质上就是一个温度为 τ 的 (K+1) 类 softmax 交叉熵,其中"正确类别"就是正样本。** 前一项把 q 往 k⁺ 拉,后一项(log-sum-exp,是 max 的光滑近似)把 q 从所有 key(主要是 K 个负样本)推开。两项一拉一推,正好实现"拉近正、推开负"。

**为什么它叫 Info(信息)NCE?** 一个关键理论结论:**最小化 InfoNCE 等价于最大化 q 与 k⁺ 之间互信息(mutual information)的一个下界**。可以证明(Oord et al. 2018):

```
I(q; k⁺) ≥ log(K+1) − L_InfoNCE
```

即互信息 I 被 `log(K+1) − loss` 下界。这给出两个一阶推论:**(1) 负样本越多(K 越大),这个下界 log(K+1) 越松、能逼近的互信息上限越高——这是"对比学习需要大量负样本"的理论根据;(2) 最小化 loss 就是在抬高互信息下界,即逼正负视图共享尽可能多的语义信息。** (注:这个下界相当松,"InfoNCE 好用是因为它最大化互信息"这个解释近年受到质疑——后续工作发现互信息估计准不准和表示好不好并不强相关,真正起作用的可能是"对齐 + 均匀性"那套几何,见 3.4。这里把下界作为历史动机和直觉给出,**严格的因果解释仍是开放问题,标注为需谨慎对待**。)

### 3.2 温度 τ 到底在调什么:梯度视角

τ 是对比学习里最神秘也最重要的超参,新手常以为它只是个无关紧要的缩放。**讲透 τ 必须看它对梯度的作用。** 把 sim 简记 s,softmax 概率记 pᵢ。对负样本 kᵢ 的相似度 sᵢ 求导:

```
∂L/∂sᵢ = (1/τ) · pᵢ          (i 为负样本)
∂L/∂s₊ = (1/τ) · (p₊ − 1)    (正样本)
```

其中 `pᵢ = exp(sᵢ/τ) / Σ exp(sⱼ/τ)`。**关键洞察:τ 越小,softmax 越"尖锐(sharp)",概率分布越集中在相似度最高的那个 key 上。** 这意味着:

- **τ 小(尖锐)**:梯度几乎只关注"和 q 最像的那个负样本"(最难的负样本,hard negative)。模型被强烈推开那些最难区分的负样本,学到的特征**局部更可分、对 hard negative 更敏感**,但也更容易过度关注个别难样本、对噪声敏感。
- **τ 大(平滑)**:梯度被均摊到所有负样本上,对所有负样本一视同仁地温和推开。特征分布更均匀,但区分 hard negative 的能力下降。

**用一句话概括:τ 控制对比损失对"困难负样本"的关注程度——小 τ 让模型死磕最难的负样本,大 τ 让模型雨露均沾。** SimCLR 实验中 τ 的常用值在 **0.1~0.5** 量级(**待核**,具体最优值随增强、batch、架构变化;SimCLR 论文报告 τ=0.1 附近较优,MoCo v2 也在 0.07~0.2 量级)。这就是为什么 τ 调不好会显著掉点:它直接改写了梯度往哪个方向、用多大力气推。

**新手坑:温度必须放在归一化之后的相似度上。** 如果忘了对特征做 L2 归一化,点积的尺度会随特征模长漂移,τ 的"尖锐度"含义就失控了(模长大的样本相当于自带小温度)。**归一化 + 温度是一对,缺一不可。**

### 3.3 代码:InfoNCE loss 的 PyTorch 实现

下面用 SimCLR 的形式实现 InfoNCE。一个 batch 有 N 张图,每张图生成 2 个增强视图,共 2N 个视图;对每个视图,它的正样本是"同一张图的另一个视图",其余 2N−2 个视图都是负样本。

```python
import torch
import torch.nn.functional as F

def info_nce_simclr(z, temperature=0.1):
    """
    SimCLR 形式的 InfoNCE。
    z: (2N, d)  —— 一个 batch 的 2N 个视图特征。约定排列为
       [view1_of_img1, ..., view1_of_imgN, view2_of_img1, ..., view2_of_imgN]
       即第 i 个和第 i+N 个互为正样本对。
    返回: 标量 loss(2N 个 query 的平均交叉熵)。
    """
    N2 = z.shape[0]
    N = N2 // 2

    # ---- 1. L2 归一化:把特征投到单位球面,使点积 = 余弦相似度 ----
    z = F.normalize(z, dim=1)                       # (2N, d), 每行模长=1

    # ---- 2. 相似度矩阵:sim[i,j] = z_i · z_j ----
    sim = z @ z.t()                                 # (2N, 2N)
    sim = sim / temperature                         # 除以温度 τ

    # ---- 3. 屏蔽自己和自己(对角线),不能把自己当正/负样本 ----
    self_mask = torch.eye(N2, dtype=torch.bool, device=z.device)
    sim.masked_fill_(self_mask, float('-inf'))      # 对角填 -inf, softmax 后权重为 0

    # ---- 4. 每个 query 的正样本下标:i -> (i+N) mod 2N ----
    # 第 i 行(view i)的正样本是第 (i+N)%2N 列
    pos_idx = (torch.arange(N2, device=z.device) + N) % N2   # (2N,)

    # ---- 5. InfoNCE = 以 pos_idx 为正确类别的交叉熵 ----
    # cross_entropy(logits, target): logits 每行做 log-softmax, 取 target 类的负对数
    loss = F.cross_entropy(sim, pos_idx)
    return loss
```

**逐点讲清机制:**(1) `z @ z.t()` 一次性算出所有 2N×2N 对相似度,正样本和负样本都在这张矩阵里;(2) 对角线必须屏蔽——`sim(z_i, z_i)=1/τ` 是最大值,不屏蔽的话模型会把"自己"当正样本,任务退化;(3) 把问题交给 `F.cross_entropy`,它内部就是 `−log[exp(logit[target]) / Σexp(logit)]`,与 3.1 推导的 InfoNCE 逐字对应——**正样本是"正确类别",其余列(除自己外)自动充当负样本的 log-sum-exp**。这就是为什么对比学习能直接复用分类的交叉熵机器。

**复杂度**:相似度矩阵是 (2N)² 个点积,每个 d 维,故 **O(N²·d)** 的计算和 **O(N²)** 的显存。这立刻暴露 SimCLR 的命门:**负样本来自同 batch,要更多负样本就得更大 batch,而显存随 N² 增长**。SimCLR 论文用到 **batch size 4096~8192**(**待核**,需 TPU pod 级算力),正是为了凑足负样本——这也直接催生了 MoCo。

### 3.4 一个更干净的几何解释:对齐 + 均匀性

Wang & Isola (2020) 给了对比学习一个比互信息更可验证的解释,值得记住,因为它直接说明"为什么有负样本就不塌缩"。他们证明 InfoNCE 在 K→∞ 时等价于优化两个量:

```
对齐(alignment): 正样本对在球面上应尽量靠近     —— minimize E‖f(x)−f(x⁺)‖²
均匀性(uniformity): 所有特征应尽量均匀铺满单位球面 —— 来自负样本项的排斥, 类似球面上的静电势能最小化
```

**这两项是对抗的:对齐想把点聚拢,均匀性(负样本排斥)想把点摊开。塌缩(所有点重合)会让均匀性项爆炸(球面上点全堆一处,排斥能无穷大),所以负样本项天然惩罚塌缩。** 这就是对比学习不塌缩的根本原因——**负样本提供了一个"把特征摊开"的排斥力,常数解在这个力下能量极高,被排除。** 记住这个图景:它是理解下一节"非对比方法没有负样本却也不塌缩"为什么反直觉、需要别的机制的对照基准。

## 四、MoCo:用动量队列把负样本数量从 batch 里解耦

### 4.1 问题:负样本越多越好,但 batch 不能无限大

3.3 已论证:InfoNCE 的互信息下界 log(K+1) 随负样本数 K 变松,**负样本越多通常越好**;但 SimCLR 把负样本绑在 batch 里,显存 O(N²) 限制了 K。**MoCo(He et al. 2020)的目标:把"负样本数量 K"和"batch size N"解耦,用小 batch 也能有海量负样本。**

朴素想法:维护一个**队列(queue)**,把过去若干 batch 编码出的 key 缓存起来当负样本,用完入队、最老的出队。这样队列长度 K(=负样本数)可以远大于 batch size,且不占梯度显存(队列里的 key 是常量,不反传)。

但这立刻有个**致命问题:表示不一致(representation inconsistency)**。编码器 f_θ 每个 step 都在更新,队列里旧 batch 的 key 是用"几十上百步之前的旧编码器"算的,而当前 query 用的是"最新编码器"。**新 query 和陈旧 key 算相似度,相当于拿两把不同的尺子量东西,对比信号被噪声污染。** 队列越长,最老的 key 越陈旧,越不一致。

### 4.2 动量编码器:让 key 编码器缓慢平滑地跟随

MoCo 的核心创新是**用两个编码器**:

- **query 编码器 f_q(参数 θ_q)**:正常用梯度更新,编码当前 batch 的 query。
- **key 编码器 f_k(参数 θ_k,momentum encoder, 动量编码器)**:**不直接用梯度更新**,而是做 θ_q 的**指数移动平均(exponential moving average, EMA)**:

```
θ_k ← m · θ_k + (1 − m) · θ_q          (每个 step 更新, m 是动量系数, 接近 1, 如 0.999)
```

**为什么这样能解决不一致?** 关键在 m 接近 1(0.999~0.9999)。θ_k 是 θ_q 的极缓慢平滑版本:每步只挪一点点,所以**θ_k 在很多步内变化极小**。这带来两个好处:

1. **队列里不同时刻入队的 key,都是用"几乎相同的、缓慢漂移的 key 编码器"算的**,彼此一致性高——解决了陈旧 key 与当前的尺子不一致问题。
2. θ_k 是 θ_q 的时间平均,**更平滑稳定**,不像 θ_q 那样每步剧烈抖动。

**用 EMA 反推 θ_k 的"记忆长度":** 展开 EMA,θ_k 在 t 步是过去 θ_q 的加权平均,权重以 m^k 衰减。**有效平均窗口约为 1/(1−m) 步**(几何级数 Σ m^k = 1/(1−m))。m=0.999 → 窗口约 1000 步;m=0.9999 → 约 10000 步。**这个窗口要和队列长度匹配:窗口太短(m 太小)key 编码器变化快、队列里 key 不一致;窗口太长(m 太大)key 编码器更新太慢、跟不上 query 编码器的进步。** MoCo 论文消融显示 **m=0.999 远优于 m=0.9 或 m=0(后者直接训不起来)**——这定量证明了"缓慢一致更新"是 MoCo 能 work 的命门。

### 4.3 代码:MoCo 的动量更新 + 队列字典

```python
import torch
import torch.nn.functional as F

class MoCo(torch.nn.Module):
    def __init__(self, encoder_q, encoder_k, dim=128, K=65536, m=0.999, T=0.07):
        super().__init__()
        self.m, self.T, self.K = m, T, K
        self.encoder_q = encoder_q                       # 梯度更新
        self.encoder_k = encoder_k                       # 动量更新, 不接梯度
        # 用 q 的参数初始化 k, 并冻结 k 的梯度
        for p_q, p_k in zip(self.encoder_q.parameters(), self.encoder_k.parameters()):
            p_k.data.copy_(p_q.data)
            p_k.requires_grad = False
        # 队列字典: K 个负样本 key, 注册为 buffer(不是参数, 不反传)
        self.register_buffer("queue", F.normalize(torch.randn(dim, K), dim=0))
        self.register_buffer("queue_ptr", torch.zeros(1, dtype=torch.long))

    @torch.no_grad()
    def _momentum_update(self):
        # θ_k ← m·θ_k + (1−m)·θ_q  : EMA 缓慢跟随
        for p_q, p_k in zip(self.encoder_q.parameters(), self.encoder_k.parameters()):
            p_k.data = self.m * p_k.data + (1.0 - self.m) * p_q.data

    @torch.no_grad()
    def _dequeue_and_enqueue(self, keys):
        # 新 batch 的 key 入队, 最老的出队(环形缓冲)
        bs = keys.shape[0]
        assert self.K % bs == 0  # 教学简化: 与官方 moco 一致, 假设队列长能整除 batch, 切片不跨队尾回绕
        ptr = int(self.queue_ptr)
        self.queue[:, ptr:ptr + bs] = keys.t()           # 写入(dim, bs)
        self.queue_ptr[0] = (ptr + bs) % self.K

    def forward(self, im_q, im_k):
        # ---- query 分支: 接梯度 ----
        q = F.normalize(self.encoder_q(im_q), dim=1)     # (N, dim)

        # ---- key 分支: 不接梯度, 用动量编码器 ----
        with torch.no_grad():
            self._momentum_update()                      # 先更新 key 编码器
            k = F.normalize(self.encoder_k(im_k), dim=1) # (N, dim)

        # ---- InfoNCE logits ----
        # 正样本: 当前 batch 内 q 与对应 k 的点积 (N, 1)
        l_pos = torch.einsum('nc,nc->n', q, k).unsqueeze(-1)
        # 负样本: q 与队列里 K 个旧 key 的点积 (N, K)
        l_neg = torch.einsum('nc,ck->nk', q, self.queue.clone().detach())
        logits = torch.cat([l_pos, l_neg], dim=1) / self.T   # (N, 1+K)

        # 正样本永远在第 0 列, 故 target 全 0 —— 又是一个分类交叉熵
        labels = torch.zeros(logits.shape[0], dtype=torch.long, device=q.device)
        loss = F.cross_entropy(logits, labels)

        self._dequeue_and_enqueue(k)                     # 当前 batch 的 key 入队
        return loss
```

**机制要点串讲:**(1) 负样本数 K(这里 65536)由队列长度决定,**和 batch size N 完全无关**——这就是"解耦"的字面意思,N 可以小到 256,K 仍是 6 万级;(2) 队列里的 key 用 `.detach()`,不反传,所以 K 个负样本几乎不增加显存(只多存 dim×K 个数,O(K·d) 而非 SimCLR 的 O(N²));(3) key 编码器全程 `no_grad`,只靠 `_momentum_update` 的 EMA 更新;(4) 又把对比变成 target=0 的交叉熵,和 InfoNCE 一字不差。

**一句话总结 MoCo 的设计:** **把负样本搬进一个不反传的队列(解耦数量与 batch),再用动量 EMA 保证队列里 key 编码一致(解耦数量与一致性的矛盾)。** 这两步合起来,让小 batch 也能享受海量、一致的负样本。MoCo v2 后来吸收了 SimCLR 的 projection head 和更强增强,进一步缩小了与 SimCLR 的差距。

### 4.4 SimCLR vs MoCo 对照

| 维度 | SimCLR | MoCo |
|------|--------|------|
| 负样本来源 | 同 batch 其他视图 | 动量队列(跨 batch 缓存) |
| 负样本数 K | = 2N−2(绑死 batch) | = 队列长度(独立于 batch,可达 6 万+) |
| 显存随负样本 | O(N²) | O(K·d),队列不反传 |
| key 编码器 | 与 query 共享(同一个) | 独立的动量 EMA 编码器 |
| 大 batch 依赖 | 强(需 4096+) | 弱(256 即可) |
| 防塌缩机制 | 负样本排斥(同对比) | 负样本排斥(同对比) |

**两者本质都是对比学习(都靠负样本防塌缩),区别只在"负样本怎么来、够不够多、一不一致"这个工程问题上。** 这张表回答了练习里"MoCo 动量队列为何解耦负样本数"的问题。

## 五、非对比方法:没有负样本,为什么不塌缩?

### 5.1 尖锐的问题

对比学习靠负样本的排斥力防塌缩(4.4 的"均匀性"项)。**但负样本很烦:要大 batch 或大队列,还可能把"语义相同但属于不同图"的样本误当负样本推开(false negative,假负样本)。** 自然的问题:**能不能只拉近正样本、完全不要负样本?**

但 2.0 节已经论证:只拉近正样本,最优解是 `f_θ(x) ≡ c`(常数塌缩),loss=0 却毫无用处。**所以"无负样本"方法的全部本事,就是设计一种结构,让常数解在优化上不可达或不被偏好。** BYOL、SimSiam、DINO 给了三套互相关联的答案,这是本章最反直觉、面试最爱问、也最容易讲错的部分。我们逐个拆。

### 5.2 BYOL:在线-目标双网络 + 预测头 + stop-gradient

**BYOL(Grill et al. 2020,Bootstrap Your Own Latent)** 的结构:两个网络,一个 query 视图走**在线网络(online network)**,另一个视图走**目标网络(target network)**。

```
视图1 ──[在线编码器 f_θ]──[投影头 g_θ]──[预测头 q_θ]──> 预测 p₁
视图2 ──[目标编码器 f_ξ]──[投影头 g_ξ]──────────────> 目标 z₂  (stop-gradient!)

损失: L = ‖ normalize(p₁) − normalize(z₂) ‖²   (再对称地交换视图1/2 算一遍)
目标网络参数: ξ ← τ·ξ + (1−τ)·θ   (又是 EMA, 不接梯度)
```

三个关键设计,**缺一个就塌缩**:

1. **stop-gradient(梯度停止)**:目标分支 z₂ 不反传梯度(`.detach()`)。损失只更新在线网络,目标网络靠 EMA 跟随。
2. **预测头 q_θ(predictor)**:在线分支比目标分支**多一个预测头**,造成两个分支**结构不对称(asymmetric)**。
3. **EMA 目标网络**:目标网络是在线网络的动量平均(同 MoCo 的 EMA)。

**为什么这套不塌缩?——讲透这个问题。** 直觉上,如果在线和目标都输出常数 c,loss=0,塌缩似乎是个解。但实证上 BYOL 不塌缩。机制性的解释有几层:

**第一层(经验事实):消融证明预测头 + stop-gradient 这对组合是防塌缩的关键。** 去掉预测头 q_θ(让两分支对称)→ 立刻塌缩;不做 stop-gradient(让目标也反传)→ 也塌缩。**所以防塌缩的不是某个单一组件,而是"不对称预测 + stop-gradient + EMA"这套耦合结构。**

**第二层(动力学论证,关键):** 一个广为接受的解释来自 BYOL 后续分析和 SimSiam 论文。考虑梯度怎么流:loss 推动在线分支的预测 p₁ 去匹配一个**被 detach 的、由 EMA 缓慢变化的目标 z₂**。**因为目标被 stop-gradient 固定为一个"当前的、不可被梯度直接拽向常数的靶子",优化过程不是'同时把两边拉向同一点',而是'让在线预测去追一个缓慢移动的固定靶'。** 在这种"追逐固定靶"的动力学下,**常数解虽然是损失的一个驻点,但不是优化轨迹会收敛到的稳定吸引子**——预测头 q_θ 的存在使得在线网络可以学到一个非平凡的、能预测目标的映射,而不必让编码器输出退化。

**第三层(更强的理论,谨慎陈述):** Tian et al. (2021) "Understanding self-supervised learning dynamics without contrastive pairs" 用线性化分析证明:**在预测头 + stop-gradient 的设置下,即使没有负样本,优化动力学中存在一个机制(与预测头的特征值结构、EMA 共同作用)主动避开塌缩解**——预测头本质上扮演了"对消塌缩方向"的角色。直觉是:预测头让在线分支能在特征空间里做一个旋转/变换去匹配目标,这个自由度使得"编码器输出有结构"比"输出常数"在优化上更容易达到低 loss。(**严格的充要条件和它对超参的依赖仍是活跃研究,这里给出主流理解,标注为非完全 settled。**)

**给读者一个可操作的记忆:BYOL 不塌缩 = stop-gradient 把目标变成"不会被拽向常数的固定靶" + 预测头制造不对称让在线网络有"非平凡地预测靶子"的自由度 + EMA 让靶子缓慢一致地移动。三者去掉任一个,塌缩立刻发生。**

### 5.3 SimSiam:把 BYOL 砍到只剩 stop-gradient

**SimSiam(Chen & He 2021)** 做了一个极简化实验,直接证明"防塌缩的最小必要件"是什么。它在 BYOL 基础上**去掉了 EMA 目标网络**(两分支共享同一个编码器),也去掉了大 batch、负样本、动量——只保留**预测头 + stop-gradient**:

```
视图1 ─[编码器 f]─[投影 g]─[预测 q]─> p₁ ─┐
视图2 ─[编码器 f]─[投影 g]──────────> z₂  │  (z₂ stop-gradient)
                                          └─> L = − cos(p₁, stopgrad(z₂))
                          (对称地再算 视图2→p₂ vs stopgrad(z₁))
```

**SimSiam 的核心消融(本章最该记的一个实验):去掉 stop-gradient,模型立刻塌缩(loss 瞬间掉到最小值 −1,所有输出变成同一个常数向量,在 ImageNet 上分类准确率掉到随机水平);加回 stop-gradient,正常学习。** 这个对照极其干净地证明:**stop-gradient 是无负样本方法不塌缩的关键,而 EMA、动量、大 batch、负样本都不是必需的。**

```python
import torch
import torch.nn.functional as F

def simsiam_loss(p1, z1, p2, z2):
    """
    SimSiam 损失。p 是预测头输出, z 是投影头输出。
    关键: 对 z 做 stop-gradient(.detach()), 这是不塌缩的命门。
    """
    def D(p, z):
        p = F.normalize(p, dim=1)
        z = F.normalize(z.detach(), dim=1)     # ★ stop-gradient: 目标侧不反传
        return -(p * z).sum(dim=1).mean()      # 负余弦相似度
    # 对称: 两个视图互为对方的预测目标
    return 0.5 * D(p1, z2) + 0.5 * D(p2, z1)
```

**SimSiam 论文给的解释:这套结构隐式地在做一个类 EM(期望最大化)的交替优化**——stop-gradient 把一侧暂时"冻结"成目标,另一侧去拟合它,角色对称交替,相当于交替更新"聚类中心(目标)"和"分配(预测)"。这个交替过程有一个非平凡的不动点,而常数塌缩不是它收敛到的解。**记住结论:stop-gradient 把"同时拉两边塌成一点"的退化优化,变成了"交替地一侧固定、另一侧拟合"的良性优化。**

### 5.4 DINO:自蒸馏 + 中心化/锐化,涌现出分割能力

**DINO(Caron et al. 2021,self-DIstillation with NO labels)** 是非对比里最重要的一个,因为它(a) 把范式推到 ViT 上,(b) 揭示了一个惊人的涌现现象,(c) 用一套优雅的"中心化 + 锐化"防塌缩。

**结构是知识蒸馏(knowledge distillation)的自监督版本:** 学生网络 g_θs 和教师网络 g_θt(EMA 跟随学生,同 MoCo/BYOL),都输出一个 **K 维的概率分布**(softmax,K 是某个原型数,如 65536,不是类别),损失是让学生分布去匹配教师分布的**交叉熵**:

```
学生: ps = softmax(g_θs(view_s) / τs)
教师: pt = softmax( (g_θt(view_t) − C) / τt )    ← 注意减去 center C, 且温度 τt < τs
损失: L = − Σ pt · log ps      (跨多个增强视图做; 教师只看全局大裁剪, 学生看全局+局部小裁剪)
教师参数: θt ← λ·θt + (1−λ)·θs  (EMA)
center:   C  ← m·C + (1−m)·mean_batch(g_θt)   (EMA 跟踪教师输出的 batch 均值)
```

**防塌缩的两个对抗操作(这是 DINO 的精髓):**

塌缩在 DINO 里有两种形式:**(a) 所有输出都塌成同一个分布;(b) 某一维主导(分布退化成 one-hot 常数)。** DINO 用两个互相对抗的操作分别压制:

1. **中心化(centering)**:教师 logits 减去一个 EMA 跟踪的均值 C。**它防止"某一维总是主导"——把各维拉回中心,鼓励分布更均匀。** 但中心化单独用会推向"均匀分布塌缩"(所有维都一样)。
2. **锐化(sharpening)**:教师用**更低的温度 τt**(比学生 τs 小),使教师分布更尖锐。**它防止"塌成均匀分布"——逼教师给出有信息的、尖锐的目标。** 但锐化单独用会推向"one-hot 主导塌缩"。

**关键:中心化(推向均匀)和锐化(推向尖锐)方向相反,互相平衡,共同把教师分布锁在"既不均匀塌缩、也不 one-hot 塌缩"的健康区间。** 这是一个非常漂亮的"两个相反力达成平衡"的设计——类比对比学习里"对齐 vs 均匀性"那对张力,DINO 用"锐化 vs 中心化"在无负样本下复现了同样的平衡。**没有负样本,但用两个相反的归一化操作造出了等效的张力。**

**涌现的分割能力(emergent segmentation):** DINO 最震撼的发现是——**用 ViT 做 DINO 自监督训练后,ViT 最后一层的自注意力图(attention map)直接清晰地勾勒出图像里物体的轮廓,无需任何分割标签或监督。** 把 [CLS] token 对各 patch 的注意力可视化,前景物体被干净地分割出来。这是一个强信号:**纯自监督 + ViT 自发学到了物体的语义边界这种高层结构**,而这是 CNN 自监督里没有这么明显观察到的。这个现象后来直接启发了无监督分割、开放词表分割等方向(也和后面第 15 章的视觉基础模型相关)。**它定性地证明了"不用标签也能学到逼近人工分割的语义结构",是自监督价值的最有力实证之一。**

### 5.5 三类非对比方法小结

| 方法 | 防塌缩机制 | 是否要 EMA | 是否要预测头 | 关键消融 |
|------|-----------|-----------|------------|---------|
| BYOL | stop-grad + 预测头 + EMA 目标 | 是 | 是 | 去预测头或去 stop-grad → 塌缩 |
| SimSiam | stop-grad + 预测头(无 EMA) | 否 | 是 | 去 stop-grad → 立刻塌缩 |
| DINO | 中心化(防某维主导) + 锐化(防均匀) + stop-grad + EMA | 是 | 否(用原型分布) | 去中心化或去锐化 → 对应两种塌缩 |

**贯穿三者的统一直觉:无负样本方法必须靠"不对称 + stop-gradient"把'两边同时塌成一点'的退化优化打断,再用某种机制(预测头的自由度 / 中心化锐化的张力)制造一个非平凡的稳定解。** 这是本章要求你能复述的核心论证。

## 六、掩码图像建模:把 BERT 搬到图像上

### 6.1 从语言完形填空到图像完形填空

上门课讲过 BERT 的掩码语言建模:遮住 15% 的 token 让模型还原。一个自然的想法:**图像也切成 patch(ViT 已经这么做了),遮住一部分 patch,让模型重建——这就是掩码图像建模(MIM)。** 但图像和语言有一个本质差异,决定了掩码率天差地别:

```
语言: token 离散、信息密集、冗余低 —— 遮 15% 已经够难(每个词都重要)
图像: 像素连续、空间冗余极高     —— 遮 15% 太简单(从邻居插值就能填出来)
```

**这是理解 MAE 高掩码率的钥匙。** 图像有强烈的**空间冗余(spatial redundancy)**:相邻像素高度相关,一小块缺失可以靠周围像素**低层插值(interpolation)**直接补出来,根本不需要理解语义。如果掩码率低,模型会走捷径——学成一个"局部纹理插值器",而不是"语义理解器"。**要逼模型学语义,就必须把掩码率提到"无法靠局部插值补全、必须依赖对整体内容的理解才能重建"的程度。**

### 6.2 MAE:掩 75%、非对称编码器-解码器、重建像素

**MAE(He et al. 2022,Masked Autoencoder)** 是这个思路的极简且强力的实现。三个核心设计:

**设计 1:极高掩码率(~75%)。** 随机遮掉 75% 的 patch,只留 25% 可见。如 6.1 所论证,**高掩码率消除了"局部插值"这条捷径,逼模型从稀疏的可见 patch 推断整体内容,从而学语义。** MAE 消融显示掩码率从 10% 一路提到 75%(甚至 80%)线性任务性能都在涨,75% 附近最优——**这个高得反直觉的最优掩码率,本身就是"图像冗余大"这个信息论事实的实验证据。**

**设计 2:非对称编码器-解码器(asymmetric)。** 这是 MAE 高效的关键。

```
编码器(重, 如 ViT-Large): 只处理 25% 的可见 patch(被掩的根本不输入!)
解码器(轻, 浅层小 Transformer): 输入 = 编码后的可见 token + 共享的 [MASK] token(放在被掩位置)
                                 输出: 重建所有被掩 patch 的像素
```

**编码器完全不看被掩的 patch**——被掩 patch 连进都不进编码器。这带来巨大的计算节省:重型编码器只处理 1/4 的 token。**因为 ViT 的 self-attention 是 O(n²),token 数减到 1/4,编码器 attention 计算量降到约 (1/4)² = 1/16**(FFN 部分降到 1/4)。这让 MAE 能用同样算力训练大得多的编码器或训更多 epoch。轻量解码器只在预训练用,下游微调时丢弃,只留编码器。

**设计 3:重建目标是原始像素(可选归一化)。** 损失是被掩 patch 的像素 MSE(均方误差):

```
L = (1/|M|) Σ_{p ∈ M} ‖ decoder(p) − normalize(pixels_p) ‖²    (只在被掩 patch M 上算)
```

MAE 发现对每个 patch 的像素先做**逐 patch 归一化(减均值除标准差)**再算 MSE,效果更好(让模型关注相对结构而非绝对亮度)。**注意 loss 只在被掩 patch 上计算,可见 patch 不算重建 loss(它们是"已知条件",重建它们没意义)——这点和 BERT 只对 [MASK] 位置算 loss 同构。**

### 6.3 代码:MAE 掩码-重建的核心前向

```python
import torch

def random_masking(x, mask_ratio=0.75):
    """
    对 patch 序列随机掩码。返回保留的可见 patch、以及恢复原顺序需要的索引。
    x: (B, N, D)  —— B 张图, 每张 N 个 patch token, 维度 D。
    """
    B, N, D = x.shape
    len_keep = int(N * (1 - mask_ratio))          # 保留的 patch 数, 75% 掩 -> 保留 25%

    # 每个样本生成随机打分, 排序 -> 随机选取要保留的子集
    noise = torch.rand(B, N, device=x.device)
    ids_shuffle = torch.argsort(noise, dim=1)     # 升序: 前 len_keep 个被保留
    ids_restore = torch.argsort(ids_shuffle, dim=1)  # 逆排列, 用于把 token 放回原位

    ids_keep = ids_shuffle[:, :len_keep]                       # 保留的 patch 下标
    x_visible = torch.gather(                                  # 取出可见 patch
        x, dim=1, index=ids_keep.unsqueeze(-1).expand(-1, -1, D))

    # 生成二值 mask(0=可见, 1=被掩), 训练时算 loss 用
    mask = torch.ones(B, N, device=x.device)
    mask[:, :len_keep] = 0
    mask = torch.gather(mask, dim=1, index=ids_restore)        # 恢复到原 patch 顺序
    return x_visible, mask, ids_restore


def mae_forward(patches, encoder, decoder, mask_token, pos_embed, mask_ratio=0.75):
    """
    MAE 前向骨架(省略 patch embedding/位置编码的细节, 突出掩码-重建机制)。
    patches: (B, N, D) 已做 patch embedding 的 token(未加 mask token)。
    """
    # ---- 1. 随机掩码, 只保留 25% 可见 patch ----
    x_vis, mask, ids_restore = random_masking(patches, mask_ratio)

    # ---- 2. 重型编码器: 只处理可见 patch(被掩的根本不进来 -> 省 16x attention) ----
    latent = encoder(x_vis)                        # (B, len_keep, D_enc)

    # ---- 3. 把 [MASK] token 填回被掩位置, 恢复成完整长度 N ----
    B, N = mask.shape
    n_mask = N - latent.shape[1]
    mask_tokens = mask_token.expand(B, n_mask, -1)             # 共享的可学习 [MASK]
    x_full = torch.cat([latent, mask_tokens], dim=1)          # (B, N, D)
    # 用 ids_restore 把 token 放回它们原本的空间位置
    x_full = torch.gather(
        x_full, dim=1, index=ids_restore.unsqueeze(-1).expand(-1, -1, x_full.shape[-1]))
    x_full = x_full + pos_embed                               # 加位置编码(让 mask token 知道自己在哪)

    # ---- 4. 轻量解码器: 重建所有 patch 的像素 ----
    pred = decoder(x_full)                         # (B, N, patch_pixels)

    # ---- 5. 损失: 只在被掩 patch 上算像素 MSE ----
    return pred, mask     # 调用方用 mask 选出被掩 patch 算 MSE
```

```python
def mae_loss(pred, target_pixels, mask):
    """pred/target: (B, N, patch_pixels); mask: (B, N), 1=被掩。只在被掩 patch 上算 MSE。"""
    loss = (pred - target_pixels) ** 2           # 逐像素平方误差
    loss = loss.mean(dim=-1)                      # (B, N) 每个 patch 的均方误差
    loss = (loss * mask).sum() / mask.sum()       # 只对被掩 patch(mask=1)求平均
    return loss
```

**机制串讲:**(1) `random_masking` 的 argsort 技巧是无替换随机抽样的标准写法,`ids_restore` 保证后面能把 token 放回原空间位置;(2) **编码器只吃 `x_vis`(25%)是 MAE 高效的全部秘密**;(3) `mask_token` 是一个**共享的可学习向量**——所有被掩位置用同一个 [MASK] 占位,靠位置编码区分它们在哪;(4) loss 只在 `mask=1` 处算,可见 patch 不参与。

### 6.4 BEiT:预测离散 token 而非像素

**BEiT(Bao et al. 2021,BERT pre-training of image transformers)** 走了另一条路:**不重建连续像素,而是预测被掩 patch 的离散视觉 token。** 它先用一个预训练的 **dVAE / tokenizer** 把每个 patch 编码成一个离散 codebook id(类似把图像 patch "分词"),然后像 BERT 一样**预测被掩 patch 的离散 token id**(分类交叉熵,不是 MSE)。

**像素重建 vs 离散 token 预测的权衡:**

```
MAE(像素 MSE):  目标简单直接、无需额外 tokenizer; 但像素含大量低层细节(高频噪声),
                 模型可能浪费容量去拟合无语义的细节。
BEiT(离散token): 离散 token 已是"语义压缩"过的目标, 逼模型预测高层语义而非像素细节;
                 但依赖一个额外预训练的 tokenizer(dVAE), 流程更复杂、目标质量受 tokenizer 限制。
```

**核心区别是"重建目标在抽象层次上的高低"**:像素是最低层(MAE 靠高掩码率 + 编码器不看被掩 patch 来补偿"目标太低层"),离散 token 是更高层(BEiT 把语义抽象前置到 tokenizer 里)。两条路都成功了,说明 **MIM 的关键不在"重建什么",而在"任务足够难、逼模型理解全局"** ——这与对比/非对比方法防平凡解是同一个底层逻辑。

### 6.5 对比学习 vs 掩码建模:两条路的性质差异

| 维度 | 对比/非对比(SimCLR/MoCo/DINO) | 掩码建模(MAE/BEiT) |
|------|------------------------------|---------------------|
| 监督信号 | 视图间的相似性结构(实例判别) | 重建被掩内容(完形填空) |
| 依赖数据增强 | 强依赖(增强定义不变性) | 弱依赖(随机掩码本身就是增强) |
| 防平凡解机制 | 负样本排斥 / stop-grad+预测头 | 重建任务够难(高掩码率) |
| 学到的特征 | 线性可分性强(linear probing 高) | 微调潜力强(fine-tuning 高),线性探测相对弱 |
| batch 敏感性 | 较敏感(尤其 SimCLR) | 不敏感 |
| 典型主干 | CNN / ViT 都行 | 主要 ViT(天然 patch) |

**一个常被问的现象:为什么 MAE 的 linear probing(冻结主干只训一个线性分类头)往往不如对比方法,但 fine-tuning(微调整个主干)却很强?** 一种解释:**对比学习直接优化"特征的线性可分性"(InfoNCE 就是个线性 softmax),所以冻结主干线性探测就好;而 MAE 学的是"重建所需的丰富局部+全局表示",这些特征不是为线性可分而生,需要微调去重组才能发挥**(**待核**:这是主流解释,精确机制仍在研究)。**给读者的实践判断:下游能微调全部参数 → MIM 类很有竞争力;只能冻结主干做线性/少量参数适配 → 对比类通常更省事。**

## 七、设计权衡与常见坑

**坑 1:忘记 L2 归一化特征。** 对比学习里特征必须归一化到单位球面,温度 τ 才有"控制 softmax 尖锐度"的明确含义。不归一化 → 点积尺度随模长漂移 → τ 失控、训练不稳。**归一化和温度是一对。**

**坑 2:数据增强太弱,模型走捷径。** 增强决定"要求模型对什么不变"。SimCLR 已证明去掉颜色抖动会让模型靠"两个裁剪共享颜色直方图"作弊认正样本,根本不学语义。**自监督的性能对增强极其敏感,增强设计错了,学到的不变性就是错的——这是和监督学习最不同的一点。**

**坑 3:在无负样本方法里删掉 stop-gradient。** SimSiam 已干净证明:去掉 stop-gradient,模型秒塌缩(所有输出变常数,loss 掉到最小,下游随机猜)。**stop-gradient 不是可选优化技巧,是无负样本方法不塌缩的命门。** 同理 BYOL/DINO 里删预测头、删中心化或锐化,都会触发对应形式的塌缩。

**坑 4:把 MAE 的高掩码率当成超参随便调小。** 75% 是被"图像冗余大、低掩码率会让模型退化成插值器"这个机制决定的最优区间,**调到 15%(BERT 那套)会让 MAE 学不到语义**(模型靠局部插值就能重建,任务太简单)。语言和图像的最优掩码率差 5 倍,根源是信息冗余度不同,不能跨域照搬。

**坑 5:用 MAE 的特征做 linear probing 然后断言"MAE 不如对比学习"。** 评测协议本身有偏向:linear probing 偏爱对比方法,fine-tuning 偏爱 MIM。**评测自监督方法一定要说清用的是 linear probing 还是 fine-tuning,两个协议结论可能相反。** 拿错协议下结论是常见的误判。

**坑 6:误以为"自监督预训练 ≥ 监督预训练"是无条件的。** 自监督的优势在"无标注数据规模可以无限大"和"特征更通用";但在数据量受限、且下游任务恰好和 ImageNet 类别高度对齐时,监督预训练仍可能更强。**自监督的价值要在"数据规模可扩展 + 下游任务多样/与分类标签不对齐"的场景下才充分体现。**

**权衡总览:三类范式占据不同生态位。** 要 linear probing 友好、特征即插即用 → 对比/非对比(DINO 尤其强,且涌现分割);要省去负样本和大 batch 的工程麻烦、且下游能全参微调 → MAE 类掩码建模;要把视觉特征对齐到文本语义、拿零样本能力 → 下一章的 CLIP(对比学习推广到图文对)。**没有银弹,选哪个取决于"下游怎么用特征"和"有多少算力/数据"。**

## 十、小结与承上启下

这一章我们把"不用标签怎么学好特征"从第一性原理拆透了:

1. **自监督的本质是"从数据自身造监督信号(pretext task)",而全部技术难点是防止模型走捷径塌缩到平凡解(常数 / 恒等映射)。** 三大范式是三种防平凡解的答案。
2. **对比学习**把"认出正样本"做成温度 τ 的 (K+1) 类分类,损失是 InfoNCE `L=−log[exp(sim(q,k⁺)/τ)/Σexp(sim(q,kᵢ)/τ)]`;它靠**负样本的排斥力(均匀性)** 防塌缩;τ 控制对 hard negative 的关注度。**SimCLR** 靠大 batch 提供负样本,**MoCo** 用**动量 EMA 编码器 + 队列字典**把负样本数从 batch size 里解耦出来(队列解耦数量、EMA 解耦一致性)。
3. **非对比方法**(BYOL/SimSiam/DINO)证明了**无负样本也能不塌缩**:靠**stop-gradient(把目标变成不被拽向常数的固定靶)+ 不对称(预测头 / 原型分布)+ EMA**。SimSiam 的"删 stop-gradient 立刻塌缩"是最干净的实证;DINO 用**中心化(防某维主导)+ 锐化(防均匀)** 这对相反的力达成平衡,并**涌现出无标签分割能力**。
4. **掩码图像建模**(MAE/BEiT)把 BERT 完形填空搬到图像:**因图像空间冗余大,必须用 ~75% 高掩码率消除局部插值捷径、逼模型学语义**;MAE 的**非对称设计(编码器只看 25% 可见 patch)** 把 attention 算量降到约 1/16,是高效关键。**对比方法 linear probing 强、MIM fine-tuning 强**,评测协议决定结论。

**这一章在整门课里的位置**:它紧接 ViT(第 7 章)——**DINO 和 MAE 都建立在 ViT 的 patch 化之上,是"ViT + 自监督"的结晶**。向后,它是**第 9 章 CLIP 的直接地基**:CLIP 就是把本章的对比学习从"同一张图的两个增强视图"推广到"配对的(图,文)对"——一个 batch 里 N 个图文对构成 N×N 相似度矩阵,对角线是正样本,本质就是本章的 InfoNCE 用在跨模态上,由此涌现出零样本分类能力。**你在本章建立的"InfoNCE = 温度 softmax 分类、负样本提供排斥力、防塌缩是核心矛盾"这三条主轴,会直接迁移到 CLIP**;而 MAE 的"掩码-重建"思路也会在后面的生成模型(第 11 章扩散的去噪重建)里以另一种形式再次出现。**到那时你会清楚看到:对比学习不是只能学单模态特征,它一旦接上文本,就直接打开了"用语言描述就能分类任意图像"的零样本大门。**

### 练习 1:InfoNCE 与温度 τ 的作用

**(a) 从 (K+1) 类分类视角写出 InfoNCE，并展开成两部分**

设 query 为 q，key 集合为 {k₀, …, k_K} 共 K+1 个，其中恰好一个是正样本 k⁺，其余 K 个是负样本。所有向量先做 L2 归一化，相似度取点积（= 余弦相似度）：sᵢ = sim(q, kᵢ) = q · kᵢ。

把"在 K+1 个 key 里认出正样本"建模成一个 (K+1) 类 softmax 分类，正确类别就是正样本所在的那一类。以 sᵢ/τ 作为 logits，"q 与第 i 个 key 配对"的概率为：

```
p(i | q) = exp(sᵢ/τ) / Σ_{j=0}^{K} exp(sⱼ/τ)
```

最大化 p(正样本 | q) 等价于最小化其负对数似然，得到单个 query 的 InfoNCE 损失：

```
L(q) = − log [ exp(s₊/τ) / Σ_{i=0}^{K} exp(sᵢ/τ) ]
```

展开成两部分（log 商 = log 分子 − log 分母）：

```
L(q) = − s₊/τ              +  log Σ_{i=0}^{K} exp(sᵢ/τ)
       └─ 拉近正样本项 ─┘     └──── log-sum-exp 推开项 ────┘
```

- 第一项 −s₊/τ：减小它要求 s₊ 增大，即把 q 拉向正样本 k⁺。
- 第二项 log Σ exp(sᵢ/τ)：它是 max 的光滑近似，减小它要求所有 sᵢ（主要是 K 个负样本）整体变小，即把 q 从所有 key 推开。

**结论：InfoNCE 就是一个温度为 τ 的 (K+1) 类 softmax 交叉熵，"拉近正 + 推开负"由"线性项 − log-sum-exp 项"这一拉一推实现。**

**(b) 求 ∂L/∂sᵢ 并论证小 τ 让梯度集中在最难负样本上**

令 zᵢ = sᵢ/τ，pᵢ = softmax(z)ᵢ。先求 L 对 logits zᵢ 的导数。softmax 的雅可比是 ∂pⱼ/∂zᵢ = pⱼ(δᵢⱼ − pᵢ)。而 L = −log p₊，所以

```
∂L/∂zᵢ = Σⱼ (∂L/∂pⱼ)(∂pⱼ/∂zᵢ)
        = (−1/p₊) · ∂p₊/∂zᵢ
        = (−1/p₊) · p₊(δ_{i,+} − pᵢ)
        = pᵢ − δ_{i,+}
```

即对负样本（i ≠ +）：∂L/∂zᵢ = pᵢ；对正样本：∂L/∂z₊ = p₊ − 1。

再用链式法则 zᵢ = sᵢ/τ，故 ∂L/∂sᵢ = (∂L/∂zᵢ)·(∂zᵢ/∂sᵢ) = (∂L/∂zᵢ)·(1/τ)：

```
∂L/∂sᵢ = pᵢ / τ            (i 为负样本)        ← 即题目要证的式子
∂L/∂s₊ = (p₊ − 1) / τ      (正样本)
```

（数值验证：随机取 5 个 key、τ=0.2，解析梯度与中心差分数值梯度逐元素吻合，最大误差 ≈ 5×10⁻¹⁰；softmax 链式结果 ∂L/∂zᵢ = pᵢ − [i=+] 也逐元素吻合。）

论证"小 τ 梯度集中在最难负样本上"：负样本 kᵢ 收到的梯度大小正比于 pᵢ = exp(sᵢ/τ)/Σexp(sⱼ/τ)。τ 越小，sⱼ/τ 之间的差异被放大，softmax 越尖锐，概率质量 pᵢ 越集中在相似度 sᵢ 最大的那个 key（也就是最像 q、最难区分的负样本——hard negative）上。于是该负样本拿到几乎全部梯度，模型被强烈推开它；其余容易负样本的 pᵢ≈0，几乎不产生梯度。**所以 τ 越小，梯度越集中在最难的负样本上，模型相当于在做近似的 hard-negative mining。**

**(c) τ→0 与 τ→∞ 两个极端**

**τ→0：softmax 退化成 argmax，梯度几乎只作用于相似度最高的那个负样本，对比学习退化成纯 hard-negative mining——只死磕最难区分的单个负样本，对噪声/离群点极其敏感；τ→∞：所有 pᵢ→均匀(1/(K+1))，梯度被均摊到全部 key 上，退化成"对所有负样本一视同仁地平均推开"，完全丧失区分 hard negative 的能力（特征趋于均匀但不够判别）。**

### 练习 2:MoCo 动量队列为何解耦负样本数

**(a) SimCLR 负样本数为何被 batch size 死死绑定，及显存复杂度**

SimCLR 的负样本只来自同一个 batch：一个 batch 有 N 张图、生成 2N 个增强视图，对任一 query 视图，正样本是同图的另一视图，其余 2N−2 个视图全部充当负样本。因此负样本数 K = 2N − 2 是 batch size 的线性函数——**要更多负样本，唯一办法就是把 batch 调更大，二者被死死绑定**。

显存复杂度：核心开销是 2N×2N 的相似度矩阵（每个元素是一对 d 维向量点积），计算量 O(N²·d)、矩阵显存 O(N²)，且这些都在计算图里需要反传。**显存随 batch 呈平方 O(N²) 增长**，这就是 SimCLR 必须上到 batch 4096~8192（TPU pod 级，**待核**）才能凑足负样本、普通单机难以复现的根因。

**(b) MoCo 队列如何让 K 独立于 N，且几乎不增显存**

MoCo 维护一个长度为 K 的队列（queue），缓存过去若干 batch 编码出的 key，当负样本用；每来一个新 batch，新 key 入队、最老的 key 出队（环形缓冲）。于是**负样本数 K = 队列长度，由队列容量决定，与当前 batch size N 完全无关**——N 可以小到 256，K 仍可设到 65536。

为什么几乎不增显存：队列里的 key 是过去算好的常量，前向时用 `.detach()` 切断梯度、不参与反传。它们只占静态存储 dim×K 个数，即 **O(K·d) 的纯存储，没有计算图、不占激活/反传显存**。对比 SimCLR 是 O(N²) 的带梯度矩阵：MoCo 用"廉价的不反传存储"换来了海量负样本，把"负样本数量"和"梯度显存"解耦开。

**(c) 重点：只加队列不加动量编码器会怎样，EMA 解决了什么，m 该选大还是小**

只加队列、让 key 编码器和 query 编码器同步用梯度更新（每步都剧烈变化），会出现**表示不一致（representation inconsistency）**：编码器每个 step 都在更新，队列里旧 batch 的 key 是用"几十上百步之前的旧编码器"算的，而当前 query 用的是"最新编码器"。**新 query 和陈旧 key 算相似度，相当于拿两把不同的尺子量东西**——对比 logits 被这种"尺子漂移"噪声严重污染，队列越长、最老的 key 越陈旧、越不一致，训练退化甚至训不起来。

动量 EMA 解决的正是这个一致性问题。key 编码器参数 θ_k 不直接吃梯度，而是做 query 编码器 θ_q 的指数移动平均：

```
θ_k ← m · θ_k + (1 − m) · θ_q        (每步更新, m 接近 1)
```

m 接近 1 时 θ_k 每步只挪一点点，在很多步内几乎不变。于是**不同时刻入队的 key 都是用"几乎相同、缓慢漂移的同一把尺子"算出来的，彼此一致性高**；同时 θ_k 作为时间平均也更平滑稳定。

定量看 m 选大还是小，用有效平均窗口。展开 EMA，θ_k 是历史 θ_q 的几何加权平均，权重以 mᵏ 衰减，几何级数 Σ_{k≥0} mᵏ = 1/(1−m)，故**有效一致性窗口 ≈ 1/(1−m) 步**（数值验证：m=0.999 的几何级数部分和 = 1000.0，正是 1/(1−m)）：

```
m=0.9    → 窗口 ≈ 10 步
m=0.99   → 窗口 ≈ 100 步
m=0.999  → 窗口 ≈ 1000 步
m=0.9999 → 窗口 ≈ 10000 步
```

**这个窗口要和队列长度匹配，存在双向权衡：m 太小（窗口太短）→ key 编码器变化太快、队列里 key 仍不一致，等于没解决问题（MoCo 消融里 m=0 直接训不起来）；m 太大（窗口太长）→ key 编码器更新太慢、跟不上 query 编码器的进步，目标过于陈旧。** MoCo 论文消融显示 m=0.999 远优于 m=0.9 或 m=0，定量证明"应选接近 1 的大 m，给约 1000 步的一致窗口"是 MoCo 能 work 的命门。

**一句话总结：队列把"负样本数量"从 batch 里解耦出来，动量 EMA 把"队列内 key 的一致性"从"编码器每步剧变"里解耦出来——两步合起来才让小 batch 也能享受海量且一致的负样本。**

### 练习 3:BYOL/DINO/SimSiam 为何不塌缩

**(a) 证明"只拉近正样本、无任何约束"的最优解是常数塌缩**

设只拉近正样本对的损失（以平方距离为例，归一化与否不改变结论）：

```
L(θ) = E_{x, (x⁺ 是 x 的另一增强视图)} ‖ f_θ(x) − f_θ(x⁺) ‖²
```

L 是非负量（每项 ‖·‖² ≥ 0），故 L ≥ 0，下确界是 0。取常数映射 f_θ(x) ≡ c（任意常数向量），则对任意正样本对都有 f_θ(x) − f_θ(x⁺) = c − c = 0，于是每一项都为 0，L = 0 达到全局最小。

**结论：在"只拉近正样本、无任何其他约束"下，常数解 f_θ(x) ≡ c 是损失的全局最优解（loss=0），这就是塌缩（collapse）。** 它对所有输入输出同一个向量，对任何下游任务都不携带信息——所以无负样本方法必须额外设计机制，让这个常数解在优化上不可达或不被偏好。（对照：对比学习靠负样本的"均匀性"排斥力——常数解会让球面上所有点堆在一处、排斥能无穷大，从而被天然排除；无负样本方法没有这个排斥力，得换别的机制。）

**(b) SimSiam：去掉 stop-gradient 会怎样，stop-gradient 把优化变成什么**

实证结果：**去掉 stop-gradient，SimSiam 立刻塌缩**——损失瞬间掉到最小值（负余弦的最小 −1），所有输出退化成同一个常数向量，ImageNet 分类准确率掉到随机水平；加回 stop-gradient 则正常学习。这是 SimSiam 论文最干净的一个对照实验，证明 stop-gradient 是无负样本方法不塌缩的命门（而 EMA、动量、大 batch、负样本都不是必需的）。

机制解释：没有 stop-gradient 时，损失同时对两个分支反传，优化器可以"同时把两边一起拽向同一点"——这条路径直接通向常数解，最省力，于是塌缩。**stop-gradient 把目标分支 z 固定（detach）成一个当前不可被梯度直接拽动的靶子，于是优化从"同时拉两边塌成一点"变成了"一侧暂时冻结成固定目标、另一侧（带预测头）去拟合这个目标"的良性过程。** SimSiam 论文把它解释为一个类 EM（期望最大化）的交替优化：stop-gradient 那侧像被固定的"聚类中心/目标"，预测那侧像"分配/拟合"，两个角色对称交替更新。这个交替过程有一个非平凡的不动点，而常数塌缩不是它会收敛到的稳定解——预测头提供的自由度让"编码器输出有结构"比"输出常数"更容易达到低 loss。

**(c) DINO：中心化与锐化各防哪种塌缩，为何必须同时存在**

DINO 里塌缩有两种形式：(a) 输出塌成均匀分布（所有维都一样，没有信息）；(b) 某一维总是主导（分布退化成 one-hot 常数）。DINO 用两个方向相反的操作分别压制：

- **中心化（centering）**：教师 logits 减去一个 EMA 跟踪的 batch 均值 C（C ← m·C + (1−m)·mean_batch(教师输出)）。它把各维拉回中心、抵消"某一维系统性偏高"，**防止"某一维主导（one-hot 塌缩）"，方向是推向均匀分布**。但若只用中心化，会把分布推得过平，导向"均匀分布塌缩"。
- **锐化（sharpening）**：教师用比学生更低的温度 τt（τt < τs），使教师分布更尖锐。它逼教师给出有信息、有区分度的目标，**防止"塌成均匀分布"，方向是推向尖锐（one-hot）**。但若只用锐化，又会把分布推成 one-hot，导向"某维主导塌缩"。

**为什么必须同时存在：中心化（推向均匀）和锐化（推向尖锐）方向正好相反、互相对抗。两者平衡时，教师分布被锁在"既不均匀塌缩、也不 one-hot 塌缩"的健康区间；缺任何一个，剩下那个力没有制衡，就会把分布单向推到自己对应的那种塌缩。** 这正是用一对相反的归一化操作、在无负样本条件下复现了对比学习里"对齐 vs 均匀性"那种张力。

**贯穿三者的统一直觉：无负样本方法都靠"不对称（预测头 / 原型分布）+ stop-gradient"把'两边同时塌成一点'的退化优化打断，再用某种机制（预测头的自由度 / 中心化-锐化的张力）制造一个非平凡的稳定解。**

### 练习 4:75% 掩码率与 MAE 计算量

**(a) 为何图像最优掩码率(~75%)远高于语言(~15%)，低掩码率会让 MAE 学成什么**

根源是两种模态的信息冗余度不同：

```
语言: token 离散、信息密集、冗余低 → 遮 15% 已经够难(每个词都很重要, 难以从上下文直接补出)
图像: 像素连续、空间冗余极高     → 遮 15% 太简单(相邻像素高度相关, 缺的一小块靠周围低层插值就能补)
```

图像有强烈的**空间冗余（spatial redundancy）**：相邻 patch/像素高度相关，少量缺失可以靠周围像素做**低层插值（interpolation）**直接补出来，根本不需要理解语义。所以低掩码率下重建任务太简单，模型会走捷径——**学成一个"局部纹理插值器"，只会从邻居外推像素，而学不到对图像整体内容的语义理解**（这与对比/非对比方法防"平凡解/捷径"是同一个底层逻辑：任务太容易就会被一个无语义的平凡解攻克）。

要逼模型学语义，就必须把掩码率提到"无法靠局部插值补全、必须依赖对整体内容的理解才能重建"的程度，对图像而言这个临界点远高于语言，约在 75%（MAE 消融显示掩码率从 10% 一路提到 75%~80%，线性探测性能持续上涨，75% 附近最优）。**这个高得反直觉的最优掩码率本身就是"图像冗余大"这一信息论事实的实验证据。结论：低掩码率会让 MAE 退化成局部插值器、学不到语义。**

**(b) 编码器只处理 25% 可见 patch，attention 与 FFN 各降到几分之几**

设全部 patch 数为 n，掩码率 75% → 编码器只处理可见 patch 数 n' = 0.25 n（被掩 patch 根本不输入编码器）。

- self-attention 计算量是 O(token数²)。token 数从 n 降到 0.25 n，平方关系：

```
(n')² / n² = (0.25)² = 0.0625 = 1/16
```

**编码器 attention 计算量降到原来的 (1/4)² = 1/16。**

- FFN（逐 token 的前馈）计算量是 O(token数)，线性关系：

```
n' / n = 0.25 = 1/4
```

**FFN 计算量降到原来的 1/4。**

（数值验证：keep=0.25 → attention 比例 0.0625=1/16，FFN 比例 0.25=1/4。）

**(c) 非对称设计为何能在相同算力下训更大编码器或更多 epoch**

MAE 的非对称在于：**重型编码器（如 ViT-Large）只处理 25% 可见 patch，把最贵的 attention 算量压到约 1/16；被掩 patch 不进编码器，只在后面由一个轻量浅层解码器（输入 = 编码后的可见 token + 共享 [MASK] token）负责重建。** 于是单次前向里，承担绝大部分参数和计算的编码器只跑 1/4 的 token，整体训练算量大幅下降。

把省下来的算力重新分配，就能在相同算力预算下选择：(i) 训练大得多的编码器（每步更便宜，可堆更多参数），或 (ii) 跑更多 epoch / 更多数据（每步快，可跑更多步）。而那个轻量解码器**只在预训练阶段使用、下游微调时整个丢弃，只保留编码器**——它的额外开销很小，且不进入下游推理成本。**所以非对称设计（重编码器只看可见 patch + 轻解码器只在预训练用）是 MAE 既高效又可扩展（Scalable）的关键。**
