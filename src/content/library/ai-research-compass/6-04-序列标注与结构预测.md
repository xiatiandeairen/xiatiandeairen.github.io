> 这一章把你从"会给整段文本打一个分类标签"带到"给序列的每一个位置打一个互相关联的标签,并在指数级大的标签序列空间里高效地训练与解码"。我们要讲透的核心矛盾是:**序列标注的输出不是 n 个独立的分类决策,而是一个有内部结构(标签之间彼此约束)的整体**。逐位置独立 softmax 会因为忽略这层结构而系统性犯错。我们会从生成式的 HMM 起步(交代前向-后向与维特比这两套动态规划),再把它"判别式化"得到**线性链 CRF**——把配分函数 Z(x) 的前向算法、对数似然的梯度、维特比解码三件事从第一性原理推到能照着实现的程度,并讲清 CRF 为什么严格优于逐位置 softmax。最后把 CRF 接到神经网络上(BiLSTM-CRF、BERT+CRF),交代它在预训练时代的位置。读完你能自己推导 CRF 的全部公式、手写维特比、并判断一个抽取任务到底要不要 CRF 层。

## 一、动机:序列标注是什么,以及为什么它不是"n 个独立分类"

很多 NLP 任务的本质是:**给定一个长度为 n 的 token 序列 `x = (x₁, …, xₙ)`,给每个位置 t 赋一个标签 `yₜ`,得到等长的标签序列 `y = (y₁, …, yₙ)`。** 这类任务统称**序列标注(sequence labeling / tagging)**,典型代表:

- **分词(word segmentation)**:中文没有空格,要判断每个字是不是词边界。常用 4 标签集 `{B, M, E, S}`(词首/词中/词尾/单字成词)。
- **词性标注(POS tagging)**:给每个词标 `名词/动词/形容词…`。
- **命名实体识别(NER, Named Entity Recognition)**:从文本里框出人名、地名、机构名等"实体"。这是本章贯穿的主例。

NER 的输出不是天然的"每位置一个标签",因为一个实体可能跨多个 token(`"中华人民共和国"` 是一个机构/地名)。我们需要一种编码,把"分块(chunking)"问题转成"逐位置标注"问题。最经典的是 **BIO 标注体系**(也叫 IOB):

```
标签集 = { O, B-T, I-T  for each entity type T }
  O    : Outside,不属于任何实体
  B-T  : Begin,类型为 T 的实体的第一个 token
  I-T  : Inside,类型为 T 的实体的非首 token
```

例句(PER=人名, ORG=机构):

```
token:  马   云   创   立   了   阿   里   巴   巴
label:  B-PER I-PER O   O   O   B-ORG I-ORG I-ORG I-ORG
```

解码时,**一段连续的 `B-T I-T I-T …` 就还原成一个类型为 T 的实体**;遇到 `O` 或新的 `B` 就切断。这样,任意"框出若干不重叠片段并给每段分类"的 chunking 任务,都被 BIO 无损地编码成了逐位置分类。(还有 BIOES/BILOU 变体,额外用 `E`/`S` 显式标实体结尾和单 token 实体,有时能略微提精度,机制完全一样,后面会提。)

**现在抛出本章的核心问题。** 既然每个位置要打一个标签,最朴素的做法是:对每个位置 t 单独跑一个分类器,`P(yₜ | x)`,各位置独立地取 argmax。这正是"逐位置独立 softmax(per-position / token-level softmax)"。它**错在哪里**?

错在它假设了**各位置标签条件独立**:`P(y | x) = ∏ₜ P(yₜ | x)`。但 BIO 标签之间有硬性的结构约束:

- **`I-PER` 不能直接跟在 `O` 后面。** "Inside-人名"必须前接一个人名 token(`B-PER` 或 `I-PER`),否则这个"Inside"无所依附,是非法序列。
- **`I-ORG` 不能跟在 `B-PER` 后面。** 类型必须一致:`B-PER` 开头的实体只能用 `I-PER` 延续。
- 反过来,`B-PER` 之后**很可能**是 `I-PER`(人名常多于一字),这是软偏好。

逐位置 softmax 看不到这些。它可能给出 `O I-PER O` 这种**逻辑上不可能**的序列——因为它对每个位置的决策是孤立的,位置 2 的分类器不知道位置 1 输出了什么。实践中这会产生大量"碎片化"和"非法转移"的错误。**我们需要的是对整个标签序列联合建模 `P(y | x)`,显式地把"标签 → 标签"的转移偏好编码进去。** 这就引出了 HMM 和 CRF。

**一句话定调本章的方法演进:HMM(生成式,建模 `P(x,y)`)→ 线性链 CRF(判别式,直接建模 `P(y|x)`,是 HMM 的判别式升级)→ 神经特征 + CRF(BiLSTM-CRF / BERT-CRF)。** 这三步共享同一套动态规划(前向、维特比),理解了 HMM 的 DP,CRF 的 DP 就是同构的。

## 二、HMM:生成式序列模型与它的两套动态规划

### 2.1 模型定义与三个独立性假设

**隐马尔可夫模型(HMM, Hidden Markov Model)** 把标签 `y` 看成"隐藏状态(hidden state)",把 token `x` 看成隐藏状态"发射(emit)"出来的"观测(observation)"。它是一个**生成式**模型——直接对联合分布 `P(x, y)` 建模,假设数据是这样"生成"的:先按马尔可夫链走出一串隐藏标签,每走到一个标签就吐出一个观测词。

完整的联合概率(为了边界整齐,引入一个虚拟起始状态 `y₀ = START`):

```
P(x, y) = ∏_{t=1}^{n}  P(yₜ | yₜ₋₁) · P(xₜ | yₜ)
                       └─转移概率─┘   └─发射概率─┘
```

这一行依赖三个**独立性假设**:

1. **一阶马尔可夫假设(转移)**:当前标签只依赖前一个标签,`P(yₜ | y₁:ₜ₋₁) = P(yₜ | yₜ₋₁)`。
2. **观测独立假设(发射)**:当前观测只依赖当前标签,与其它位置的标签和观测都无关,`P(xₜ | x, y) = P(xₜ | yₜ)`。
3. **时齐性(homogeneous)**:转移概率 `P(y'|y)` 和发射概率 `P(x|y)` 不随位置 t 变化(同一套参数全程复用)。

参数因此就两张表:**转移矩阵 A**,`A[i][j] = P(yₜ=j | yₜ₋₁=i)`,大小 `|Y|×|Y|`;**发射矩阵 B**,`B[j][w] = P(xₜ=w | yₜ=j)`,大小 `|Y|×|V|`(V 是词表)。**有标注数据时,HMM 的训练就是数频率(最大似然),根本不用梯度下降**:

```
A[i][j] = count(i→j 转移) / count(从 i 出发的转移)
B[j][w] = count(标签 j 上出现词 w) / count(标签 j 出现总次数)
```

(实际要做平滑,如加一平滑/Laplace,避免未见过的词或转移概率为 0 把整条路径乘成 0。)这个"数频率即训练"的简洁性,是 HMM 在前深度学习时代流行的重要原因。

### 2.2 三个基本问题与前向算法

HMM 有三个经典问题(Rabiner 1989 的综述把这套讲得最清楚,见导读):

1. **评估(likelihood)**:给定参数和观测 x,算 `P(x) = Σ_y P(x, y)`(对所有可能标签序列求和)。
2. **解码(decoding)**:给定 x,求最可能的标签序列 `y* = argmax_y P(y | x)`。这是序列标注真正要的。
3. **学习(learning)**:无标注时用 Baum-Welch(EM)估参数。有标注时直接数频率(上一节)。

问题 1 的 `Σ_y` 看似要枚举 `|Y|ⁿ` 条序列,**指数级**。但马尔可夫结构允许用**动态规划**把它压成线性。定义**前向变量(forward variable)**:

```
α_t(j) = P(x₁, x₂, …, xₜ,  yₜ = j)
       = "观测到前 t 个词、且第 t 步落在标签 j" 的联合概率
```

递推关系(从 t-1 到 t):要落在 (t, j),必须先在 t-1 落在某个 i,再从 i 转移到 j,再由 j 发射出 xₜ:

```
初始化:  α_1(j) = P(START → j) · P(x₁ | j)            ← A[START][j] · B[j][x₁]
递推:    α_t(j) = [ Σ_i  α_{t-1}(i) · A[i][j] ] · B[j][xₜ]
终止:    P(x)  = Σ_j  α_n(j)
```

**关键在 `Σ_i` 这一步:它把"所有以 i 结尾的前缀路径概率"先汇总了,后面所有要经过 (t,j) 的路径都复用这个汇总值,不再重复展开前缀。** 复杂度从 `O(|Y|ⁿ)` 降到 `O(n·|Y|²)`(n 个位置,每个位置 `|Y|` 个目标状态,每个目标状态对 `|Y|` 个来源求和)。这是本章所有 DP 的共同骨架,务必看透这个"汇总-复用"。

对称地有**后向变量** `β_t(i) = P(x_{t+1}, …, xₙ | yₜ = i)`(给定 t 步在 i,之后观测的条件概率),递推方向相反:

```
β_n(i) = 1
β_t(i) = Σ_j  A[i][j] · B[j][x_{t+1}] · β_{t+1}(j)
```

前向-后向合起来,可以算任意位置的**后验** `P(yₜ = j | x) = α_t(j)·β_t(j) / P(x)`,这是 Baum-Welch(EM 估参)的 E 步所需,也是后面理解 CRF 梯度的钥匙(CRF 的梯度恰好是边缘期望,而边缘期望就靠前向-后向算)。

### 2.3 维特比解码:动态规划求最优标签序列

问题 2(解码)要的是 `argmax_y P(y|x)`。注意 `P(y|x) = P(x,y)/P(x)`,而 `P(x)` 与 y 无关,所以 `argmax_y P(y|x) = argmax_y P(x,y)`——**解码只需最大化联合概率**。

把前向算法里的 `Σ`(求和)换成 `max`(取最大),就得到**维特比算法(Viterbi, 1967,源自卷积码译码)**。直觉:`Σ` 是"把所有路径加起来",`max` 是"只留最好的那条路径"。定义**维特比变量**:

```
δ_t(j) = max over (y₁…yₜ₋₁)  P(x₁:ₜ, y₁:ₜ₋₁, yₜ = j)
       = "前 t 步、第 t 步落在标签 j" 的所有路径里,概率最大的那条的概率
```

递推:

```
初始化:  δ_1(j) = A[START][j] · B[j][x₁]
递推:    δ_t(j) = [ max_i  δ_{t-1}(i) · A[i][j] ] · B[j][xₜ]
         ψ_t(j) = argmax_i  δ_{t-1}(i) · A[i][j]      ← 回溯指针:(t,j) 的最优前驱
终止:    最优末状态 = argmax_j δ_n(j)
回溯:    从末状态沿 ψ 逆推,还原整条 y*
```

`ψ_t(j)`(backpointer)记下"到达 (t,j) 的最优路径上一格是谁",这样正向算完只拿到最优概率,再**从尾向头沿指针回溯**就拿回整条最优序列。复杂度同样 `O(n·|Y|²)`。

**为什么 max 能这么贪?——最优子结构(Bellman 最优性原理)。** 到达 (t,j) 的全局最优路径,它在 (t-1) 处经过的那个状态 i,这条路径的前缀(从头到 (t-1,i))一定也是"到达 (t-1,i) 的最优路径"。若不然,把前缀换成更优的,整条会更优,矛盾。所以我们只需为每个 (t-1, i) 保留一条最优前缀,无需展开指数多条。**这与强化学习里 DP 解 MDP、贝尔曼最优方程是同一个原理**(详见强化学习课《贝尔曼方程与动态规划》一章),序列标注只是把它用在了"标签链"上。

### 2.4 工程必修:在对数空间做,别在概率空间做

n 个 `(0,1)` 概率连乘会迅速下溢到浮点 0。**所有实现都在对数空间做**:乘变加,`max` 不变(log 单调),前向里的 `Σ` 变成 **log-sum-exp**:

```
log(Σ_i exp(s_i))  ——数值稳定写法——  m + log(Σ_i exp(s_i − m)),  m = max_i s_i
```

减去最大值 m 再 exp,防止 `exp` 上溢;算完加回 m。维特比因为只有 `max`,直接在对数空间 `δ̃_t(j) = max_i [δ̃_{t-1}(i) + log A[i][j]] + log B[j][xₜ]` 即可。**记住:CRF 里完全一样——配分函数用 log-sum-exp 前向,解码用 max 维特比。HMM 的 DP 就是 CRF 的 DP,只是分数从"log 概率"换成了"任意实值打分"。**

### 2.5 HMM 的根本局限:为什么要换成 CRF

HMM 简洁优雅,但两个假设在真实 NLP 里很硬:

**(1) 观测独立假设太强,无法用丰富特征。** HMM 规定 `xₜ` 只由 `yₜ` 生成,于是每个观测只能是"一个词"。但判断一个 token 是不是人名,我们想用的特征很多:它**是否大写开头**、**是否在词典里**、**前一个词是不是"先生/总统"**、**后缀是不是 -ing**、词形、字符 n-gram……这些是**重叠的、相互依赖的**特征。生成式模型要对观测建模 `P(x|y)`,一旦特征重叠,就得对特征间的相关性也建模(否则重复计数),这在生成式框架里极难——你得写出 `P(大写, 在词典中, 后缀… | y)` 的联合分布。**判别式模型不建模 `P(x)`,只建模 `P(y|x)`,可以随便往里塞任意重叠特征而无需担心它们的联合分布**,这是 CRF 相对 HMM 最实质的解放。

**(2) 生成式建模了我们不关心的 `P(x)`。** 我们要的是 `P(y|x)`,HMM 却花力气去拟合 `P(x,y)`,把建模能力浪费在生成观测上。当模型容量有限时,这种"为不关心的目标分配容量"会损害我们真正关心的判别边界。

**(3) 标签偏置(label bias),针对 HMM 的有向判别式表亲 MEMM。** 历史上从 HMM 到 CRF 中间有个 MEMM(最大熵马尔可夫模型),它是判别式但**逐位置局部归一化** `P(yₜ|yₜ₋₁, x)`。局部归一化导致"标签偏置问题":转移出度小的状态对观测不敏感(它的概率几乎被迫全分给少数后继,无论观测说什么)。**CRF 用全局归一化(整条序列共享一个配分函数 Z(x))根治了标签偏置**,这是 Lafferty 等人 2001 年提出 CRF 的核心动机。理解这点能帮你回答面试高频题"CRF 为什么比 MEMM 好"。

## 三、线性链 CRF:判别式结构预测的标准答案

### 3.1 从能量模型到 CRF 的定义

**条件随机场(CRF, Conditional Random Field, Lafferty/McCallum/Pereira 2001)** 直接对条件分布 `P(y|x)` 建模,且**全局归一化**。我们只关心序列标注用的**线性链 CRF(linear-chain CRF)**:标签之间是一条链(yₜ 只与 yₜ₋₁ 相邻),这与 HMM 的图结构相同,但建模方向反过来。

定义一条 `(x, y)` 配对的**非归一化打分(score / 负能量)** 为"所有位置的发射分 + 转移分"之和:

```
score(x, y) = Σ_{t=1}^{n}  [ E(t, yₜ)  +  T(yₜ₋₁, yₜ) ]
                            └发射分数┘   └转移分数┘
```

- **发射分数(emission)** `E(t, yₜ)`:位置 t 取标签 yₜ 的"局部适配度",可以是任意关于 x 和位置 t 的函数。在 BiLSTM-CRF 里它就是神经网络在位置 t 输出的、对应标签 yₜ 的那个 logit。
- **转移分数(transition)** `T(i, j)`:从标签 i 转到标签 j 的偏好,是一张可学习的 `|Y|×|Y|` 矩阵,**与位置无关**(线性链的时齐性)。这就是把"`I-PER` 不能跟 `O`"这类约束学进来的地方——非法转移会学到很负的分数。

(边界:`y₀ = START`、`y_{n+1} = STOP` 引入 `T(START, y₁)` 与 `T(yₙ, STOP)`,让模型学"什么标签适合开头/结尾",实现时给转移矩阵加两行两列即可。)

把打分通过 **softmax 跨整个序列空间** 归一化,得到条件概率:

```
P(y | x) = exp(score(x, y)) / Z(x)
Z(x)     = Σ_{y′ ∈ Yⁿ}  exp(score(x, y′))      ← 配分函数,对所有 |Y|ⁿ 条标签序列求和
```

**`Z(x)`(配分函数 / partition function)是全部难点所在。** 它对**指数级多**条候选序列 y′ 求和,保证 `Σ_y P(y|x) = 1`。注意这个归一化是**全局的**:分母只有一个 Z(x),整条序列共享——这正是它区别于逐位置 softmax(每个位置一个分母)和 MEMM(每步一个局部分母)的根本,也是它无标签偏置的原因。

**与传统特征 CRF 的关系(交代术语,便于读论文)。** 经典 CRF 写成特征函数加权和:`score(x,y) = Σₜ Σₖ wₖ · fₖ(yₜ₋₁, yₜ, x, t)`,特征 fₖ 由人工设计(如"yₜ₋₁=B-PER 且 yₜ=I-PER 且 xₜ 大写"),wₖ 是学的权重。我们上面的 `E + T` 是它的特例与现代化:发射分把所有"只与 (yₜ, x, t) 有关"的特征聚合成一个神经网络输出,转移分把所有"只与 (yₜ₋₁, yₜ) 有关"的特征聚合成一张矩阵。**机制完全一致,只是特征从手工变成了神经网络自动学。**

### 3.2 配分函数 Z(x) 的前向算法(本章最重要的推导)

直接按定义算 `Z(x) = Σ_{y′} exp(score(x,y′))` 要枚举 `|Y|ⁿ` 条序列,不可行。**但 score 在标签链上是可分解的(每项只涉及相邻两个标签),于是和 HMM 前向一模一样的动态规划适用。** 这是 CRF 能落地的命门,务必自己推一遍。

定义 CRF 的**前向变量**(在对数空间,记 `α_t(j)` 为到位置 t、第 t 步标签为 j 的所有前缀路径打分的 **log-sum-exp**):

```
α_t(j) = log  Σ over (y₁…yₜ₋₁)  exp( score 的前 t 项之和, 且 yₜ = j )
```

直觉:`exp(α_t(j))` = "所有以标签 j 结束的、长度为 t 的前缀路径"的 `exp(部分打分)` 之和。我们要证它能递推。**推导(把第 t 项从求和里拆出来):**

```
设前缀打分 s(y₁:ₜ) = Σ_{k=1}^{t} [E(k, yₖ) + T(yₖ₋₁, yₖ)]
则 s(y₁:ₜ) = s(y₁:ₜ₋₁) + E(t, yₜ) + T(yₜ₋₁, yₜ)        ← 打分可加,第 t 项只连 (yₜ₋₁, yₜ)

exp(α_t(j))
  = Σ_{y₁:ₜ₋₁}  exp( s(y₁:ₜ₋₁) + E(t,j) + T(yₜ₋₁,j) )      ← 固定 yₜ=j
  = exp(E(t,j)) · Σ_i [ exp(T(i,j)) · Σ_{y₁:ₜ₋₂: yₜ₋₁=i} exp(s(y₁:ₜ₋₁)) ]
                                       └────── 这内层正是 exp(α_{t-1}(i)) ──────┘
  = exp(E(t,j)) · Σ_i  exp(α_{t-1}(i)) · exp(T(i,j))
```

两边取 log,得到**对数空间的递推**:

```
初始化:  α_1(j) = E(1, j) + T(START, j)
递推:    α_t(j) = E(t, j) + logsumexp_i [ α_{t-1}(i) + T(i, j) ]
终止:    log Z(x) = logsumexp_j [ α_n(j) + T(j, STOP) ]
```

**对照 HMM 前向 `α_t(j) = [Σ_i α_{t-1}(i)·A[i][j]]·B[j][xₜ]`:结构完全同构——`Σ` 对应 `logsumexp`,概率乘 `A·B` 对应分数加 `E+T`。** 这就是 2.4 节说的"分数从 log 概率换成任意实值"的兑现:HMM 里 `log A[i][j] = log P(j|i)`、`log B[j][xₜ] = log P(xₜ|j)` 是受概率公理约束(每行非负且和为 1)的特殊分数;CRF 把这两个约束彻底松开,`T` 和 `E` 是任意实数,由判别式目标学出来。

复杂度 `O(n·|Y|²)`,**线性级**。指数级的 `Σ_{y′}` 被这个"汇总-复用"的 DP 化解了。**这是 CRF 全部工程可行性的根基,也是练习一要你独立复现的核心。**

### 3.3 训练:负对数似然及其梯度

CRF 是判别式模型,训练目标是**最大化训练数据的条件对数似然**,等价于最小化**负对数似然(NLL)**损失。对单个样本 `(x, y*)`(y* 是金标签):

```
L(θ) = − log P(y* | x) = − [ score(x, y*) − log Z(x) ]
     = log Z(x) − score(x, y*)
```

**这个损失有极漂亮的直觉:`score(x, y*)` 推高金标序列的分数,`log Z(x)` 压低所有序列的总分。** 训练就是"把对的那条路顶上去,把整个空间的配分压下来"。两项都已可高效计算:`score(x, y*)` 是 O(n) 直接累加金标路径的 `E+T`;`log Z(x)` 是 3.2 的前向算法 O(n·|Y|²)。

**梯度——理解 CRF 的最后一块拼图。** 以转移参数 `T(i,j)` 为例,对损失求偏导:

```
∂L/∂T(i,j) = ∂ log Z(x)/∂T(i,j)  −  ∂ score(x,y*)/∂T(i,j)
```

第二项是金标里"i→j 这个转移是否出现"的计数(0/1)。第一项是经典的**指数族恒等式:配分函数的对数关于某参数的导数 = 该参数对应特征的期望**。展开:

```
∂ log Z/∂T(i,j) = (1/Z) · ∂Z/∂T(i,j)
                = Σ_{y′}  P(y′|x) · [∂ score(x,y′)/∂T(i,j)]
                = Σ_{y′}  P(y′|x) · count_{i→j}(y′)
                = E_{y′ ~ P(·|x)} [ #(i→j 转移出现在 y′ 中) ]    ← 模型期望计数
```

所以:

```
∂L/∂T(i,j) = E_{y′~P(·|x)}[count_{i→j}(y′)]  −  count_{i→j}(y*)
            └────── 模型期望(model expectation)──────┘   └─经验计数(empirical)─┘
```

**这就是判别式概率模型梯度的通用形态:模型期望 − 经验观测。** 梯度为零 ⟺ 模型对每个特征的期望计数等于训练集里的真实计数(矩匹配 / moment matching)。其中"模型期望计数"靠**前向-后向**算(2.2 节的 β 在这里登场):边缘后验 `P(yₜ₋₁=i, yₜ=j | x) ∝ α_{t-1}(i)·exp(T(i,j)+E(t,j))·β_t(j)`,对 t 求和就是 `i→j` 的期望出现次数。发射参数同理(经验=金标在位置 t 的标签,期望=后验 `P(yₜ=j|x)`)。

**但在神经 CRF 实现里你几乎不必手写前向-后向。** 因为 `log Z(x)` 已用前向算法(3.2)以可微算子(logsumexp、加法)搭出计算图,**直接 `loss.backward()` 自动微分就会算出与前向-后向数学上完全一致的梯度**(autograd 对 logsumexp 求导,自动等价于做了后向递推)。这是现代实现相对经典 CRF 的巨大简化:**你只需正确实现前向算 log Z,梯度白送。** 理解上面手推是为了知道"梯度在算什么、为什么收敛到矩匹配",实现上交给 autograd。

### 3.4 解码:维特比(与 HMM 同构)

预测时要 `y* = argmax_y P(y|x) = argmax_y score(x,y)`(Z(x) 与 y 无关,可丢)。把 3.2 前向里的 `logsumexp` 换成 `max`,就是 CRF 的维特比,**与 HMM 维特比同构**:

```
初始化:  δ_1(j) = E(1,j) + T(START,j)
递推:    δ_t(j) = E(t,j) + max_i [ δ_{t-1}(i) + T(i,j) ]
         ψ_t(j) = argmax_i [ δ_{t-1}(i) + T(i,j) ]
终止:    best = argmax_j [ δ_n(j) + T(j, STOP) ]
回溯:    沿 ψ 从 best 逆推得 y*
```

**关键认知:CRF 训练用 `logsumexp` 前向(要对所有路径求和算 Z),解码用 `max` 维特比(只要最优一条路径)。一个 sum 一个 max,是同一套 DP 骨架的两种半环(semiring)实例**——sum-product 半环 vs max-plus 半环。看懂这点,HMM/CRF 的前向与维特比在你眼里就是一份代码换个算子。复杂度 `O(n·|Y|²)`。

### 3.5 为什么 CRF 严格优于逐位置 softmax(把动机收口)

现在能精确回答开篇的问题了。逐位置 softmax 做 `P(y|x)=∏ₜ P(yₜ|x)`,**每个位置独立局部归一化**;CRF 做全局归一化的 `P(y|x)=exp(score)/Z(x)`。三点本质差异:

1. **CRF 显式建模标签转移 `T(i,j)`,逐位置 softmax 完全没有这一项。** 于是"`I` 不能接 `O`"、"类型必须一致"这类结构约束,CRF 能学进转移矩阵(非法转移学到极负分,维特比解码自动绕开),逐位置 softmax 学不到,会输出非法序列。这是最直接的差距。
2. **全局归一化 vs 局部归一化。** CRF 在整条序列空间上比较,优化的就是"整条序列对不对";逐位置 softmax 各管各的位置。即便你想用启发式后处理修非法序列,那也是事后补丁,不是端到端优化的目标。
3. **解码是全局最优。** CRF 维特比给出 `argmax_y score(x,y)`,是序列级最优解;逐位置 argmax 是 n 个局部最优拼起来,整体未必最优,且不保证合法。

**代价与边界(诚实地说):CRF 不是免费的。** 转移矩阵带来 `O(|Y|²)` 的解码代价,标签集很大时(几百类)会显著变慢;它带来的提升,在**特征/表示足够强**时会缩小。后面第五节会讲到:当用 BERT 这种强上下文编码器后,模型已能从上下文隐式学到大量标签依赖,**BERT+CRF 相对纯 BERT 的增益在很多基准上变得很小甚至可忽略**——这是预训练时代对 CRF 价值的重要修正,不能停留在"CRF 永远更好"的旧认知。

## 四、PyTorch 实现:线性链 CRF 的打分、log Z 与维特比

下面给一份**体现机制、可照着实现**的核心代码。聚焦三件事:金标路径打分 `score(x,y*)`、配分函数 `log Z(x)`(前向)、维特比解码。发射分数 `E`(形状 `[seq_len, num_tags]`)假定由上游编码器(BiLSTM/BERT)给出,这里只实现 CRF 层。为聚焦机制,先写单序列版(batch、mask 在注释里点明)。

```python
import torch
import torch.nn as nn

class LinearChainCRF(nn.Module):
    def __init__(self, num_tags: int):
        super().__init__()
        self.num_tags = num_tags
        # 转移矩阵:trans[i, j] = 从标签 i 转移到标签 j 的分数(与位置无关)
        self.trans = nn.Parameter(torch.randn(num_tags, num_tags) * 0.01)
        # START -> 各标签 / 各标签 -> STOP 的转移(边界打分,可学)
        self.start_trans = nn.Parameter(torch.randn(num_tags) * 0.01)
        self.stop_trans  = nn.Parameter(torch.randn(num_tags) * 0.01)

    def _score_gold(self, emissions: torch.Tensor, tags: torch.Tensor) -> torch.Tensor:
        # emissions: [L, K] 发射分数;tags: [L] 金标签索引。返回标量 score(x, y*)
        L = tags.size(0)
        score = self.start_trans[tags[0]] + emissions[0, tags[0]]      # 起始转移 + 首位发射
        for t in range(1, L):
            score = score + self.trans[tags[t-1], tags[t]]            # 转移分 T(y_{t-1}, y_t)
            score = score + emissions[t, tags[t]]                     # 发射分 E(t, y_t)
        score = score + self.stop_trans[tags[-1]]                     # 终止转移
        return score

    def _log_Z(self, emissions: torch.Tensor) -> torch.Tensor:
        # 前向算法,对数空间。返回标量 log Z(x)
        L, K = emissions.shape
        # alpha[j] = log-sum-exp 的前向变量;初始化 = 起始转移 + 首位发射
        alpha = self.start_trans + emissions[0]                       # [K]
        for t in range(1, L):
            # 广播构造 [K_from, K_to]:alpha(i) + T(i, j) + E(t, j)
            broadcast_alpha = alpha.unsqueeze(1)                      # [K, 1] over i(来源)
            emit_t          = emissions[t].unsqueeze(0)              # [1, K] over j(目标)
            scores = broadcast_alpha + self.trans + emit_t           # [K, K]
            alpha = torch.logsumexp(scores, dim=0)                   # 对来源 i 求和 -> [K]
        alpha = alpha + self.stop_trans                              # 终止转移
        return torch.logsumexp(alpha, dim=0)                         # 对末状态 logsumexp -> 标量

    def nll(self, emissions: torch.Tensor, tags: torch.Tensor) -> torch.Tensor:
        # 负对数似然损失 = log Z(x) - score(x, y*);autograd 自动给出"模型期望-经验计数"梯度
        return self._log_Z(emissions) - self._score_gold(emissions, tags)

    @torch.no_grad()
    def viterbi_decode(self, emissions: torch.Tensor):
        # 维特比解码:把前向的 logsumexp 换成 max,并记回溯指针。返回最优标签序列(list)
        L, K = emissions.shape
        delta = self.start_trans + emissions[0]                      # [K]
        backptr = []                                                 # 每步记 [K] 的最优前驱
        for t in range(1, L):
            scores = delta.unsqueeze(1) + self.trans                 # [K_from, K_to](解码忽略发射先比转移)
            best_prev = scores.argmax(dim=0)                         # [K] 每个目标 j 的最优来源 i
            delta = scores.max(dim=0).values + emissions[t]          # [K] 取 max 再加本步发射
            backptr.append(best_prev)
        delta = delta + self.stop_trans
        best_last = int(delta.argmax().item())                       # 最优末状态
        # 回溯:从末状态沿 backptr 逆推
        best_path = [best_last]
        for bp in reversed(backptr):
            best_last = int(bp[best_last].item())
            best_path.append(best_last)
        best_path.reverse()
        return best_path
```

**这段代码兑现了前三节的全部机制,逐处对应:**
- `_log_Z` 的循环就是 3.2 的递推 `α_t(j) = E(t,j) + logsumexp_i[α_{t-1}(i)+T(i,j)]`,用广播把它向量化成一次 `[K,K]` 运算。
- `viterbi_decode` 与 `_log_Z` **结构完全相同**,只把 `logsumexp(dim=0)` 换成 `max(dim=0)` 并多记一个 `argmax` 当回溯指针——这就是 3.4 说的"sum 半环 ↔ max 半环换算子"。
- `nll` 不需要手写梯度:`_log_Z` 由可微算子搭成,`backward()` 自动得到 3.3 推的"模型期望 − 经验计数"。

**工程要点(批量与掩码,实现时务必处理):** 真实训练是 batch + 变长序列,需要 `mask`(`[B, L]`,padding 位为 0)。前向递推里 padding 步要"冻结" alpha(`alpha = mask_t * new_alpha + (1-mask_t) * alpha`),否则 padding 的发射分会污染 log Z;`score_gold` 累加时也要乘 mask 跳过 padding;终止转移要加在每条序列**真实最后一个非 padding 位**上(用 `mask.sum(1)-1` 取末位索引)。**别在 padding 上图省事——这是 BiLSTM-CRF 最常见的 silent bug,loss 看着在降但解码全乱。**

**别重复造轮子:** 生产中直接用成熟实现 `pytorch-crf`(`from torchcrf import CRF`,提供 `forward` 算 NLL、`decode` 做维特比,已处理 batch/mask),或 AllenNLP 的 `ConditionalRandomField`(还支持 `allowed_transitions`,可硬性禁止非法转移)。上面手写版是给你"看透机制"用的。

## 五、把 CRF 接到神经网络:BiLSTM-CRF 与预训练时代

### 5.1 BiLSTM-CRF:经典神经序列标注架构

经典 CRF 的发射分靠人工设计特征。**BiLSTM-CRF(Huang/Xu/Yu 2015;Lample 等 2016 用于 NER)** 把这步换成神经网络:

```
词/字 embedding  →  BiLSTM(双向 LSTM,拿到每个位置的上下文表示 hₜ)
                 →  线性层 W·hₜ + b  得到 [L, num_tags] 的发射分数 E
                 →  CRF 层(转移矩阵 T + 维特比解码)
```

- **BiLSTM**(双向 LSTM,详见本课《循环神经网络:RNN/LSTM/GRU》一章)给每个位置一个**融合了左右文**的表示 `hₜ`,线性层把它投影成"该位置各标签的发射 logit"。**它负责"这个 token 局部看像什么标签"。**
- **CRF 层**只加一张 `|Y|×|Y|` 转移矩阵,**负责"标签之间怎么衔接合法/合理"**,并用维特比做全局最优解码。
- 整个网络端到端训练:loss 是 CRF 的 NLL,梯度从 CRF 经发射分数一路回传到 LSTM 和 embedding。

**为什么这个组合好:发射分(神经网络,捕捉无穷丰富的上下文特征)与转移分(CRF 矩阵,捕捉标签结构约束)各司其职,互补。** BiLSTM 解决了 HMM"观测独立、特征贫乏"的问题;CRF 解决了逐位置 softmax"无标签依赖、局部归一化"的问题。2015–2018 年间它是几乎所有序列标注任务的 SOTA 基线。

### 5.2 BERT+CRF 与纯 BERT:预训练时代 CRF 还要不要

2018 年 BERT(详见本课《预训练语言模型:BERT 与掩码语言建模》一章)出现后,序列标注范式再变:**把 BiLSTM 编码器换成预训练 Transformer**,在每个 token 表示上接分类头。两种主流配法:

- **纯 BERT(token classification)**:BERT 输出 → 线性层 → 逐位置 softmax,**没有 CRF**。
- **BERT+CRF**:BERT 输出 → 线性层(发射分)→ CRF 层(转移 + 维特比)。

**关键判断(易踩的认知坑):预训练强编码器在很大程度上削弱了 CRF 的必要性。** 因为 BERT 的自注意力已让每个 token 表示充分看到全句上下文,**逐位置分类头其实已经隐式地"知道"邻居标签倾向**(它看得到邻居 token),很多本来要靠转移矩阵显式建模的依赖,被强表示隐式吸收了。经验上(具体数值随数据集/标签集变化,此处定性):

- 在**标签集大、转移约束强、训练数据较少**时,加 CRF 仍常有可见提升(转移矩阵直接禁掉非法序列,省得模型从数据里学"`I` 不接 `O`")。
- 在**标签集小、数据充足、用强预训练模型**时,BERT+CRF 相对纯 BERT 的提升常常很小,甚至持平,而 CRF 引入 `O(|Y|²)` 解码开销和实现复杂度。这时很多工程选择直接纯 BERT。

**「待核」:BERT+CRF 相对纯 BERT 的具体增益量级(如 CoNLL-2003 NER 上是否 < 0.5 F1)随论文与实现差异较大,需查具体 benchmark,不要记一个固定数字。** 但定性结论可靠:**CRF 的边际价值随编码器变强而下降**——这是判断"要不要 CRF 层"的第一性原理,别无脑套用"序列标注就得加 CRF"的旧经验。

### 5.3 还有一类做法:把序列标注当生成

最前沿的抽取范式(LLM 时代)甚至不走"逐位置打标签":直接用 seq2seq / 自回归 LLM,把"抽出的实体"作为**文本生成**出来,或用受约束解码强制产出结构化标签序列。这条线与本章的判别式结构预测是不同范式,**详见本课信息抽取/LLM 抽取章**。本章要你掌握的是判别式结构预测这条经典且至今在低延迟、可控、小模型场景仍是主力的路线。

## 六、设计权衡与常见坑

- **要不要 CRF 层?——按"编码器强度 × 标签结构强度 × 数据量"判断,别默认加。** 弱编码器(BiLSTM)、强标签约束、小数据 → 加 CRF 收益大;强编码器(BERT/更大)、弱约束、大数据 → 纯分类头常够用。先跑纯分类头基线,再用 CRF 做消融,用数据说话(对应用户知识库:N=1 不算验证,要消融对比)。
- **BIO vs BIOES。** BIOES 显式标实体尾(E)和单 token 实体(S),给转移矩阵更多结构信号,有时小幅提升,但标签集从 `2k+1` 涨到 `4k+1`(k 为实体类型数),`O(|Y|²)` 转移和解码更贵。**默认 BIO,精度敏感再试 BIOES,务必同口径消融。**
- **概率空间下溢——必在对数空间。** 这是最经典的坑。前向用 logsumexp(且减最大值防溢出),维特比用 max+加法。手写 CRF 若在概率空间乘,长序列必下溢成 0,loss 变 NaN/inf。
- **padding 掩码处理错。** 第四节(批量与掩码工程要点)的 silent bug:padding 位的发射分混进 log Z,或终止转移加错位置,会让 loss 正常下降但解码全错。务必单测"含 padding 的 batch 与逐条无 padding 解码结果一致"。
- **维特比一定要回溯指针。** 只记最优分数 δ 拿不到序列,必须存 backpointer `ψ` 并从尾回溯。新手常忘 ψ,只能拿到末状态。
- **转移矩阵的硬约束 vs 软学习。** CRF 默认从数据**软学**转移偏好(非法转移学成负分)。若数据少、想保证绝对不输出非法序列,可在解码时把已知非法转移的分数设为 `-∞`(AllenNLP `allowed_transitions` 即此),把 BIO 的硬规则注入维特比。**软学不保证 100% 合法,硬约束更稳但需手写规则。**
- **标签偏置认知。** 别把 MEMM 当 CRF。MEMM 局部归一化有标签偏置,CRF 全局归一化没有——这是 CRF 设计的核心动机,也是高频面试题。
- **评测要用实体级 F1,不是 token 级准确率。** 序列标注真正关心的是"实体框对了没",评测用 chunk/span 级的 precision/recall/F1(`seqeval` 库;实体边界和类型都对才算对),token 级准确率会被大量 `O` 标签虚高(详见本课评测章)。

## 九、小结与承上启下

这一章我们把**序列标注作为结构预测**讲透了:

- **核心矛盾**:输出是有内部结构的标签序列,不是 n 个独立分类;逐位置 softmax 因忽略标签依赖会输出非法序列,必须联合建模 `P(y|x)`。
- **HMM**:生成式 `P(x,y)=∏P(yₜ|yₜ₋₁)P(xₜ|yₜ)`,前向-后向算似然/边缘,维特比解码,`O(n|Y|²)`;局限是观测独立(特征贫乏)、生成式(浪费容量建模 `P(x)`)。
- **线性链 CRF**:判别式 + 全局归一化 `P(y|x)=exp(score)/Z(x)`,`score=Σₜ[E+T]`。**配分函数 Z(x) 用前向算法 `O(n|Y|²)` 算(本章最重要的推导),训练最小化 NLL=`log Z − score(y*)`,梯度是"模型期望 − 经验计数"(autograd 白送),解码用维特比。** CRF 优于逐位置 softmax 的三点:建模转移、全局归一化、全局最优解码。
- **神经化**:BiLSTM-CRF(神经发射 + CRF 转移)是经典 SOTA;预训练时代 BERT+CRF 与纯 BERT 并存,**CRF 的边际价值随编码器变强而下降**,要按编码器强度/约束强度/数据量判断,而非默认加。
- **贯穿的统一视角**:HMM 与 CRF 的前向、维特比是**同一套 DP 骨架**,sum 半环(算 Z)与 max 半环(解码)只是换个算子。

**承上**:本章的发射分数依赖一个能给每个 token 好表示的编码器——上一章的 RNN/LSTM/GRU 正是它,BiLSTM-CRF 把两者拼起来。**启下**:第五节反复点到的"BERT 让 CRF 边际价值下降",指向下一章**预训练语言模型(BERT 与掩码语言建模)**——为什么一个预训练的强编码器能隐式吸收这么多结构信息,以致显式的转移矩阵变得可有可无?那是序列标注乃至整个 NLP 范式从"任务专用结构"转向"通用预训练表示"的转折点,也是本课从"经典序列模型"迈入"预训练时代"的门槛。

### 练习 1:推导并实现 CRF 配分函数前向算法,并用玩具例对拍暴力枚举

**(a) 从定义推出递推。**

打分按位置可分解,每一项只涉及相邻两个标签:

```
score(y₁:ₙ) = Σ_{t=1}^{n} [ E(t, yₜ) + T(yₜ₋₁, yₜ) ]      (含 y₀=START)
```

定义前缀打分 `s(y₁:ₜ) = Σ_{k=1}^{t} [E(k, yₖ) + T(yₖ₋₁, yₖ)]`。关键恒等式(把第 t 项拆出来):

```
s(y₁:ₜ) = s(y₁:ₜ₋₁) + E(t, yₜ) + T(yₜ₋₁, yₜ)
```

第 t 项 `E(t,yₜ)+T(yₜ₋₁,yₜ)` 只依赖 yₜ 和 yₜ₋₁,与更早的标签无关,这是能做 DP 的全部理由。

定义前向变量(对数空间,固定第 t 步标签为 j、对所有前缀求 log-sum-exp):

```
α_t(j) = log  Σ over (y₁…yₜ₋₁)  exp( s(y₁:ₜ) ),   其中 yₜ = j
```

对 `exp(α_t(j))` 做代数(固定 yₜ=j,把第 t 项从指数里乘出来):

```
exp(α_t(j))
  = Σ_{y₁:ₜ₋₁} exp( s(y₁:ₜ₋₁) + E(t,j) + T(yₜ₋₁, j) )
  = exp(E(t,j)) · Σ_{y₁:ₜ₋₁} exp( s(y₁:ₜ₋₁) ) · exp( T(yₜ₋₁, j) )      ← E(t,j) 与求和变量无关,提出
```

把对 `y₁:ₜ₋₁` 的求和按 yₜ₋₁=i 分组(先选 t-1 步落在哪个 i,再对更早的前缀求和):

```
  = exp(E(t,j)) · Σ_i  exp(T(i,j)) · [ Σ_{y₁:ₜ₋₂, yₜ₋₁=i} exp(s(y₁:ₜ₋₁)) ]
                                       └──────── 这内层 = exp(α_{t-1}(i)) ────────┘
  = exp(E(t,j)) · Σ_i  exp(α_{t-1}(i)) · exp(T(i,j))
```

内层求和正是 `exp(α_{t-1}(i))` 的定义(t-1 步固定落在 i 的前缀 log-sum-exp 取 exp)。两边取 log,把乘积里的求和写成 logsumexp:

```
α_t(j) = E(t,j) + log Σ_i exp( α_{t-1}(i) + T(i,j) )
       = E(t,j) + logsumexp_i [ α_{t-1}(i) + T(i,j) ]        ✅ 即所求递推
```

完整三步(含边界 START/STOP):

```
初始化:  α_1(j)   = E(1, j) + T(START, j)
递推:    α_t(j)   = E(t, j) + logsumexp_i [ α_{t-1}(i) + T(i, j) ]
终止:    log Z(x) = logsumexp_j [ α_n(j) + T(j, STOP) ]
```

复杂度 `O(n·|Y|²)`——指数级的 `Σ_{y′}` 被"先汇总 `Σ_i`、后续路径复用该汇总值"的 DP 化解。**这与 HMM 前向 `α_t(j)=[Σ_i α_{t-1}(i)·A[i][j]]·B[j][xₜ]` 完全同构:`Σ`↔`logsumexp`,概率乘 `A·B`↔分数加 `T+E`。**

**(b) 玩具例对拍(|Y|=2, n=3,前向 vs 暴力枚举 8 条序列)。**

下面用 **numpy(CPU,无需 GPU/torch)** 实现前向 log Z,并暴力枚举全部 `2³=8` 条序列直接算 `log Σ exp(score)`,验证两者相等。已实跑验证通过。

```python
import numpy as np
from itertools import product

def logsumexp(a, axis=None):
    a = np.asarray(a, dtype=float)
    m = np.max(a, axis=axis, keepdims=True)
    out = m + np.log(np.sum(np.exp(a - m), axis=axis, keepdims=True))  # 减最大值防溢出
    return np.squeeze(out, axis=axis) if axis is not None else float(out.reshape(-1)[0])

np.random.seed(0)
K, n = 2, 3
E     = np.random.randn(n, K)   # 发射分 E[t, j]
T     = np.random.randn(K, K)   # 转移分 T[i, j]
start = np.random.randn(K)      # START -> j
stop  = np.random.randn(K)      # j -> STOP

def score(path):                # 单条序列打分,O(n) 直接累加
    s = start[path[0]] + E[0, path[0]]
    for t in range(1, n):
        s += T[path[t-1], path[t]] + E[t, path[t]]
    return s + stop[path[-1]]

# 暴力:枚举全部 2^3=8 条序列,直接算 log Σ exp(score)  ← 金标准
logZ_brute = logsumexp([score(p) for p in product(range(K), repeat=n)])

# 前向算法(对数空间)
alpha = start + E[0]                                  # α_1 = E(1,·)+T(START,·)
for t in range(1, n):
    scores = alpha[:, None] + T + E[t][None, :]       # [K_i, K_j] = α_{t-1}(i)+T(i,j)+E(t,j)
    alpha  = logsumexp(scores, axis=0)                # 对来源 i 做 logsumexp -> α_t(·)
logZ_fwd = logsumexp(alpha + stop)                    # +T(·,STOP) 后对末状态 logsumexp

print(logZ_brute, logZ_fwd, abs(logZ_brute - logZ_fwd))
assert abs(logZ_brute - logZ_fwd) < 1e-9
```

实跑输出:

```
8.328115789006043  8.328115789006045  1.78e-15      # 差值 ~机器精度,PASS
```

**两者数值相等(差异仅 ~1e-15 浮点误差),证明前向递推实现正确。** 把上面 `logsumexp(..., axis=0)` 换成 `.max(0)` 并记 argmax 即得维特比,可同样对拍 `argmax_y score` —— 我也跑了一遍,维特比解码结果 `(0,0,0)` 与暴力枚举最优序列一致(score=7.4156),PASS。这个"前向 vs 暴力""维特比 vs 暴力枚举 argmax"的小例子对拍,是检验任何 CRF 实现对错的金标准。

### 练习 2:HMM / CRF / 逐位置 softmax 三者五维对比

| 维度 | HMM | 线性链 CRF | 逐位置 softmax |
|---|---|---|---|
| (a) 生成式 vs 判别式 | **生成式**,建模 `P(x,y)` | **判别式**,建模 `P(y|x)` | **判别式**,建模 `P(yₜ|x)`(各位置独立) |
| (b) 是否建模标签转移 | **是**,转移矩阵 `A[i][j]=P(yₜ=j|yₜ₋₁=i)` | **是**,可学转移分 `T(i,j)`(任意实值) | **否**,完全没有"标签→标签"项 |
| (c) 归一化范围 | 全局(联合分布天然归一,Σ_{x,y}=1) | **全局**,整条序列共享一个配分函数 `Z(x)` | **局部**,每个位置一个独立 softmax 分母 |
| (d) 能否用任意重叠特征 | **不能**,观测独立假设要求 `P(xₜ|yₜ)`,重叠特征需建模联合分布,极难 | **能**,不建模 `P(x)`,发射分可塞任意重叠特征(大写/词典/后缀/上下文…) | **能**(发射侧同 CRF),但缺转移建模 |
| (e) 训练方式 | 有标注:**数频率**(MLE,A、B 计数比值 + 平滑),无需梯度 | **梯度下降**:最小化 NLL `=logZ−score(y*)`,需前向算 logZ(梯度=模型期望−经验计数,autograd 白送) | **普通交叉熵**,各位置独立反传 |

补充两点:HMM 是**局部归一化的有向判别表亲 MEMM** 的对照——MEMM 也是判别式但逐步局部归一化 `P(yₜ|yₜ₋₁,x)`,因此有**标签偏置**;CRF 用全局 `Z(x)` 根治标签偏置(Lafferty 2001 的核心动机)。逐位置 softmax 可看作连转移项也丢掉的退化判别式。

**为什么逐位置 softmax 会输出 `O I-PER O` 而 CRF 不会(一句话):** 逐位置 softmax 没有转移项 T、且对每个位置**独立取 argmax**,位置 t 的决策看不到位置 t-1 输出了什么,因此无法表达"`I-PER` 必须前接 `B-PER`/`I-PER`"这条硬约束,可能在前一格是 `O` 的情况下仍给该格选 `I-PER`;而 CRF 把这种非法转移在矩阵 `T(O, I-PER)` 上学成极负分,维特比在**整条序列**上最大化 `score=ΣE+ΣT`,会自动绕开经过该极负转移的路径,从而结构上避免非法序列。

### 练习 3:NER 为什么用 BIO + 转移约束

**(a) BIO 如何无损编码 chunking,以及解码如何还原实体。**

chunking 任务的本质是"在序列上框出若干**互不重叠**的连续片段(span),并给每段一个类型 T"。BIO 用逐位置标签编码:

```
标签集 = { O } ∪ { B-T, I-T : 每种实体类型 T }
  片段 [起, 止] 类型 T  →  起位标 B-T,(起+1 … 止)各位标 I-T
  不属于任何片段的位置  →  O
```

因为每个 token 恰好落在"某片段内首位 / 某片段内非首位 / 片段外"三种互斥情形之一,这个映射是**单射且可逆**的——给定任意一组不重叠片段,有唯一 BIO 序列;反之亦然,故"无损"。

**解码还原(从 BIO 序列切回实体):** 从左到右扫描,遇到 `B-T` 开启一个类型为 T 的新实体并记起点;**只要后续是同类型 `I-T` 就把它并入当前实体**(延长终点);遇到 `O`、遇到新的 `B-*`、或遇到类型不一致的标签,就**封口当前实体并按需开启新实体**。即"一段连续的 `B-T I-T I-T …` 合并成一个类型 T 的实体"。例:`B-PER I-PER O O O B-ORG I-ORG I-ORG I-ORG` → 实体 {PER: 位置 1–2, ORG: 位置 6–9}。

**(b) ≥3 条 BIO 硬约束 + CRF 如何学成负分。**

合法 BIO 序列(无 START 边界讨论时)必须满足:

1. **`I-T` 不能跟在 `O` 后面。** "Inside-T"必须前接一个属于类型 T 的 token,`O` 之后无所依附,非法。
2. **`I-T` 不能跟在不同类型 `B-T'`/`I-T'`(T′≠T)后面。** 类型必须一致:`B-PER` 开头的实体只能用 `I-PER` 延续,出现 `B-PER → I-ORG` 非法。
3. **句首(START 之后)不能直接是 `I-T`。** 第一个 token 不可能是"非首位 Inside",合法开头只能是 `O` 或 `B-T`。
   (可再补:`I-T` 的合法前驱**只有** `B-T` 或同类型 `I-T` 两种。)

**CRF 怎么处理:** 转移分 `T(i,j)` 是一张可学矩阵,训练数据里这些非法转移**从不出现**(经验计数为 0),而 NLL 梯度 `∂L/∂T(i,j)=模型期望计数 − 经验计数` 会持续把"模型仍给非法转移分配的概率质量"往下压,于是 `T(O, I-PER)`、`T(B-PER, I-ORG)` 等被学成**很负的分**。维特比解码最大化整条 `score` 时,任何经过这些极负转移的路径打分都极低,自动被绕开,从而(软性地)避免输出非法序列。若要**绝对**禁止,可在解码时直接把这些转移分置为 `-∞`(AllenNLP 的 `allowed_transitions` 即此硬约束做法)。

**(c) 思考题:只用 `{实体类型, O}`(不区分 B/I)会丢什么。**

会丢失**实体边界(切分)信息**,导致**相邻同类型实体粘连**:若两个相邻的同类型实体首尾相接(中间无 `O` 隔开),只标类型时这两段会得到一串完全相同的类型标签,解码无法判断这是"一个长实体"还是"两个相邻实体",边界丢失、无法切分。例:`马云` 和 `李彦宏` 若相邻且都标 `PER PER PER PER`,无 B/I 就分不出是 1 个还是 2 个人名。`B-` 标签的唯一作用正是"显式标记每个实体的起点",从而对同类型相邻实体也能切出边界——这就是为什么必须用 BIO 而非只标类型。(BIOES/BILOU 进一步用 `E`/`S` 显式标尾和单 token 实体,给转移更多结构信号,代价是标签集从 `2k+1` 涨到 `4k+1`、`O(|Y|²)` 更贵。)

### 练习 4(进阶):手写 CRF 加 batch + mask,并写对拍测试

核心机制:**padding 步要"冻结" alpha/delta**(`新值⊙mask + 旧值⊙(1−mask)`),否则 padding 的发射分会污染 log Z;**终止转移 STOP 必须加在每条序列真实最后一个非 padding 位**(用 `lengths=mask.sum(1)` 定位);维特比回溯**只走真实步数**。下面给 **numpy CPU 可跑**的批量版 + 对拍测试,已实跑验证通过;末尾附等价的 torch 骨架(逻辑一一对应,torch 版需要 `pip install torch`,CPU 即可)。

```python
import numpy as np

def logsumexp(a, axis):
    m = np.max(a, axis=axis, keepdims=True)
    return np.squeeze(m + np.log(np.sum(np.exp(a - m), axis=axis, keepdims=True)), axis=axis)

# ---------- 单序列参考实现(无 padding),作对拍金标准 ----------
def single_logZ(E, T, start, stop):                  # E:[L,K]
    L = E.shape[0]; alpha = start + E[0]
    for t in range(1, L):
        alpha = logsumexp(alpha[:, None] + T + E[t][None, :], axis=0)
    return logsumexp(alpha + stop, axis=0)

def single_viterbi(E, T, start, stop):
    L, K = E.shape; delta = start + E[0]; bptr = []
    for t in range(1, L):
        sc = delta[:, None] + T
        bptr.append(sc.argmax(0)); delta = sc.max(0) + E[t]
    last = int((delta + stop).argmax()); path = [last]
    for bp in reversed(bptr):
        last = int(bp[last]); path.append(last)
    path.reverse(); return path

# ---------- 批量 + mask 版 ----------
def batch_logZ(E, T, start, stop, mask):             # E:[B,L,K]  mask:[B,L] (pad=0)
    B, L, K = E.shape
    alpha = start[None] + E[:, 0]                                  # [B,K]
    for t in range(1, L):
        sc = alpha[:, :, None] + T[None] + E[:, t][:, None, :]     # [B,K_i,K_j]
        new_alpha = logsumexp(sc, axis=1)                          # 对来源 i 求和 -> [B,K]
        m = mask[:, t][:, None]                                    # [B,1]
        alpha = m * new_alpha + (1 - m) * alpha                    # ★ padding 步冻结 alpha
    alpha = alpha + stop[None]                                     # STOP(对 pad 序列,后续步已冻结=末位值)
    return logsumexp(alpha, axis=1)                                # [B]

def batch_viterbi(E, T, start, stop, mask):
    B, L, K = E.shape
    delta = start[None] + E[:, 0]; bptrs = []
    for t in range(1, L):
        sc = delta[:, :, None] + T[None]                           # [B,K_i,K_j]
        bptrs.append(sc.argmax(axis=1))                            # [B,K]
        new_delta = sc.max(axis=1) + E[:, t]
        m = mask[:, t][:, None]
        delta = m * new_delta + (1 - m) * delta                    # ★ padding 步冻结 delta
    delta = delta + stop[None]
    lengths = mask.sum(1).astype(int)                              # ★ 每条真实长度
    paths = []
    for b in range(B):
        last = int(delta[b].argmax()); path = [last]
        for t in range(lengths[b] - 1, 0, -1):                     # ★ 只回溯真实步数
            last = int(bptrs[t - 1][b, last]); path.append(last)
        path.reverse(); paths.append(path)
    return paths

# ---------- 对拍测试:批量结果 == 逐条去 padding 单独解码 ----------
np.random.seed(1)
K = 3
T = np.random.randn(K, K); start = np.random.randn(K); stop = np.random.randn(K)
lengths = [4, 2, 3]; Lmax = max(lengths); B = len(lengths)
E = np.random.randn(B, Lmax, K)
mask = np.zeros((B, Lmax))
for b, l in enumerate(lengths):
    mask[b, :l] = 1.0
    E[b, l:] = 123456.0          # padding 发射分故意设成离谱大值:掩码正确则不影响结果

bz, bpaths = batch_logZ(E, T, start, stop, mask), batch_viterbi(E, T, start, stop, mask)
for b, l in enumerate(lengths):
    sz, sp = single_logZ(E[b, :l], T, start, stop), single_viterbi(E[b, :l], T, start, stop)
    assert abs(bz[b] - sz) < 1e-6, (bz[b], sz)        # logZ 一致
    assert bpaths[b] == sp, (bpaths[b], sp)           # 解码序列一致
print("PASS: 批量 logZ / viterbi 与逐条无 padding 完全一致")
```

实跑输出(三条不同长度序列,padding 发射分被设成 123456 故意挖坑):

```
seq 0 len=4: logZ batch=6.894869 single=6.894869 diff=0.0 | path [0,0,0,0] == [0,0,0,0]
seq 1 len=2: logZ batch=1.856403 single=1.856403 diff=0.0 | path [1,1]     == [1,1]
seq 2 len=3: logZ batch=5.917884 single=5.917884 diff=0.0 | path [1,1,2]   == [1,1,2]
PASS
```

**测试逼出的三类 silent bug(故意把 padding 发射分设成 123456 来暴露):**
1. **不冻结 alpha/delta** → padding 步的离谱发射分混进递推,batch logZ 与单条不符。
2. **STOP 转移加错位置**(加到 padding 末位而非真实末位)→ pad 序列打分错。本实现因 padding 步已冻结,`alpha+stop` 等价于把 STOP 加在真实末位;若不冻结则必错。
3. **回溯走满 Lmax 而非真实长度** → 短序列尾部混入 padding 步的 backpointer,解码序列错。

只有三处全对,断言才会通过。这正是第四节强调的"必须单测含 padding 的 batch 与逐条无 padding 解码一致"。

**等价的 torch 骨架(依赖:`pip install torch`,CPU 即可,无需 GPU):**

```python
import torch
def batch_logZ_torch(E, T, start, stop, mask):       # E:[B,L,K] mask:[B,L]
    B, L, K = E.shape
    alpha = start.unsqueeze(0) + E[:, 0]
    for t in range(1, L):
        sc = alpha.unsqueeze(2) + T.unsqueeze(0) + E[:, t].unsqueeze(1)   # [B,K,K]
        new_alpha = torch.logsumexp(sc, dim=1)                            # [B,K]
        m = mask[:, t].unsqueeze(1)
        alpha = m * new_alpha + (1 - m) * alpha                           # 冻结 padding
    alpha = alpha + stop.unsqueeze(0)
    return torch.logsumexp(alpha, dim=1)             # [B];nll = batch_logZ_torch - gold_score
```
torch 版把 `logsumexp` 换成可微算子后,`(logZ − score(y*)).sum().backward()` 即可端到端训练,梯度自动等于"模型期望 − 经验计数",无需手写前向-后向。生产中可直接用 `torchcrf.CRF`(已内置 batch/mask 处理)。
