## 方向全景

这个方向的母题只有一句话：**如何让一个系统在没有人逐步手把手教的情况下，靠"试错 + 反馈"自己学会做出好的决策序列。** 注意是"序列"——不是一锤子买卖的预测（那是监督学习的活），而是一连串相互影响的动作，每个动作改变环境状态，最终累积出一个好结果或坏结果。围棋下一步、机器人迈一步、模型在草稿纸上写下一行推理、agent 决定调哪个工具——本质都是同一个问题。

强化学习（Reinforcement Learning，RL，"通过奖励信号学习行为"）把这个问题形式化成一个数学对象：**马尔可夫决策过程（MDP）**。它有四件东西——状态 s、动作 a、转移概率（在 s 做 a 会到哪个新状态）、奖励 r。智能体要学的是一个**策略 π(a|s)**：给定状态，输出动作的概率分布。目标是最大化"长期累积奖励"，也就是不光看眼前这一步赚多少，而是看这一步加上它带来的所有后续收益（用折扣因子 γ 把远期收益打个折）。

这里藏着 RL 区别于其它机器学习范式的两个根本难点，整个方向几十年都在跟它俩搏斗：

第一是**信用分配（credit assignment）**：你赢了一盘棋，是第 30 手的妙招立功，还是第 80 手没犯错？奖励往往延迟且稀疏，怎么把最终的成败"归因"到中间某一步动作上，是 RL 的核心痛。第二是**探索与利用的权衡（exploration vs exploitation）**：你已经知道一条能拿 60 分的路，要不要冒险去试一条可能 90 分也可能 0 分的新路？只利用会陷在次优解，只探索永远学不精。

研究范式的演变，可以理解成"用什么东西去近似那个最优策略/价值函数"在升级。**最早是查表**——状态空间小到能列成一张表（Q-learning，1989，Watkins），每个状态-动作对存一个数值。**接着是线性函数 + 手工特征**，能处理稍大的问题但依赖人工设计特征。**真正的范式跃迁发生在 2013 年前后**：用深度神经网络当那个"近似器"，让 RL 第一次能直接吃原始像素、处理巨大状态空间，这就是深度强化学习（Deep RL）。**最近两年又发生第二次范式延伸**：人们发现 RL 不只是"训练一个会动的 agent"的工具，它还是**让大语言模型学会长链推理、学会对齐人类偏好、学会用工具**的关键手术刀——母题从"控制与博弈"扩展到了"让模型想清楚再答"。这一章就是沿着这两次跃迁讲下来。

## 技术脉络地图

一张文字版的主线演进图，看清"为什么走到今天"：

**经典 RL 时代（理论奠基，1980s–2012）**
动态规划 → 时序差分学习 TD（Sutton 1988，"用下一步的估计来更新这一步的估计"，bootstrap 思想）→ Q-learning（1989，无模型、学动作价值）/ SARSA → 策略梯度 REINFORCE（Williams 1992，直接对策略求导）。这一时期能解决的都是小规模问题，瓶颈在"状态一多，表就存不下、特征就难设计"。

**深度强化学习引爆（2013–2018）**
DQN（DeepMind，2013 arXiv / 2015 Nature，深度网络 + 经验回放 + 目标网络打 Atari）→ 这条线分叉成两支：
- **价值线**：DQN → Double DQN（2015，治高估）→ Dueling DQN（2016）→ Rainbow（2017，七种改进的合体）。
- **策略线 / actor-critic 线**：A3C（2016，异步并行采样）→ TRPO（2015，带约束的稳定更新）→ **PPO（2017，裁剪式目标，简单稳定，至今主力）** → DDPG/TD3/SAC（连续控制，机器人）。

**博弈与规划的巅峰（2016–2020）**
AlphaGo（2016，深度网络 + 蒙特卡洛树搜索 MCTS + 人类棋谱 + 自我博弈，击败李世石）→ AlphaGo Zero（2017，纯自我博弈、不用人类棋谱）→ AlphaZero（2017，一套算法通吃围棋/象棋/将棋）→ MuZero（2019/2020，连游戏规则都不告诉它，自己学一个"内部模型"再规划）。这条线证明了**"学习 + 搜索"的组合能超越人类顶尖水平**，这个思想两年后会在推理模型上复活。

**RL 进入大模型，第一次延伸——对齐（2020–2023）**
RLHF（基于人类反馈的强化学习）：先有 OpenAI 2017 的偏好学习雏形、2020 的摘要对齐工作，到 **InstructGPT（2022，用 RLHF 把 GPT-3 调成会听话的助手）** 成为范式，ChatGPT 引爆。紧接着是简化路线：**DPO（2023，斯坦福，把"训奖励模型 + RL"两步合成一步直接优化）** → IPO / KTO 等变体。

**RL 进入大模型，第二次延伸——推理与 agent（2024–2025，当下战线）**
o1（OpenAI，2024，用 RL 激发长思维链，test-time compute 换正确率）→ **DeepSeek-R1（2025，公开复现，纯 RL 也能涌现推理，配套 GRPO 算法）** → 可验证奖励 RLVR 成为共识 → 重心转向 **agent / 工具使用 / 多步规划**（让模型在真实环境里调工具、查资料、写代码、自我纠错）。

一句话串起来：**算力 + 神经网络近似让 RL 能处理大世界（DQN）→ 稳定的策略优化让它好用（PPO）→ 学习与搜索结合让它超人（AlphaZero）→ 然后这套方法论被搬到语言模型上，先解决"听话"（RLHF/DPO），再解决"想清楚"（o1/R1），现在正在解决"会干活"（agent）。**

## 里程碑成果精讲

**DQN（2013 arXiv / 2015 Nature，DeepMind，Mnih 等）** — 痛点：把神经网络直接塞进 Q-learning 会训崩。原因有二：一是 RL 的训练数据是 agent 自己一步步走出来的，前后高度相关（连续帧几乎一样），违反了神经网络训练要求数据独立同分布的假设，梯度方向乱晃；二是 Q-learning 的更新目标里又包含网络自己的输出，"用自己预测自己"，目标一直动，像追自己的影子，训练发散。做法：两个关键招。**经验回放（experience replay）**——把走过的 (s,a,r,s') 四元组存进一个大缓冲池，训练时随机抽一批，打散时间相关性，还能重复利用数据。**目标网络（target network）**——单独复制一份网络专门用来算更新目标，它的参数冻结，每隔几千步才同步一次，让"被追的影子"暂时站定。突破点：第一次让深度网络在 RL 里稳定收敛，且**同一套网络结构、同一组超参数，直接吃屏幕像素，在 49 个 Atari 游戏里达到甚至超过人类水平**（Nature 论文报告其中过半游戏超过人类玩家）。真实机制：经验回放把"在线、相关、非平稳"的数据流，变成了近似"离线、独立、平稳"的监督学习问题，于是神经网络的那套优化重新生效；目标网络把移动靶变成阶段性的固定靶，让 TD 误差的回归有了稳定方向。影响：**深度强化学习这个子领域的开山之作**，证明了"端到端从感知到决策"可行，此后所有 Deep RL 工作都站在它肩上。

**A3C（2016，DeepMind，Mnih 等）** — 痛点：DQN 靠经验回放打破相关性，但回放池占内存大、且只适用于离策略（off-policy）算法，限制了策略梯度类方法。做法：开多个 agent 在各自环境副本里**异步并行**跑，每个 agent 算出梯度后异步更新同一份全局网络。突破点：用"并行带来的多样性"代替"回放池"来打破数据相关性——不同 worker 此刻在探索环境的不同角落，它们的梯度天然解耦。同时采用 **actor-critic 结构**（actor 出策略、critic 估价值，critic 给 actor 当基准减小方差）。真实机制：多个 worker 的经验在时间和空间上彼此独立，等效于把数据"摊薄"了相关性；异步更新还自带一点正则化效果。影响：让 actor-critic 成为主流工程范式，并行采样成为大规模 RL 标配；不过它很快被更稳的 PPO 取代。

**PPO（2017，OpenAI，Schulman 等）** — 痛点：策略梯度方法有个致命毛病——一次更新步子迈大了，策略就崩，再也救不回来（因为后续数据都由这个崩掉的策略采集，恶性循环）。前作 TRPO（2015）用复杂的二阶优化加 KL 散度硬约束解决了稳定性，但实现繁琐、计算重。做法：PPO 把硬约束换成一个**裁剪（clip）目标函数**。它衡量新旧策略在某动作上的概率比 r=π_new/π_old，当这个比值偏离 1 太远（超出 [1-ε, 1+ε]，ε 常取 0.2）时，把目标函数"剪平"，让超出部分不再贡献梯度。突破点：用一行 min(裁剪前, 裁剪后) 的朴素操作，达到了 TRPO 级别的稳定性，却只需一阶优化、几十行代码。真实机制：裁剪相当于给策略更新加了个"软刹车"——只要还在信任区间内就正常学，一旦某次更新想把某动作概率拉得太猛，裁剪项立刻把它的优势"封顶"，于是策略不会在单次更新里发生剧变，避免了不可逆的崩溃。通常还配 **GAE（广义优势估计，2015）** 来在偏差和方差之间平滑权衡地估计每步的优势值。影响：**PPO 成为过去 8 年最常用的 RL 算法**，从机器人、游戏到后来 RLHF 训练 ChatGPT、训练 o1 类推理模型，主力位置一直没让出去；它的价值不在最优，而在"足够好且几乎不用调参就能跑通"。

**AlphaGo → AlphaZero（2016/2017，DeepMind，Silver 等）** — 痛点：围棋状态空间约 10^170，暴力搜索不可能；它还需要"大局观"这种难以手工编码的直觉。做法：AlphaGo 用**策略网络**（预测下一步该走哪）+ **价值网络**（评估当前局面胜率）来给**蒙特卡洛树搜索（MCTS，"用大量随机模拟来估计每步好坏的搜索算法"）剪枝和导航**，初始用人类棋谱做监督预训练，再用自我博弈强化。AlphaZero 更激进：**完全丢掉人类棋谱，从随机初始化纯靠自我博弈**，一套算法同时征服围棋、国际象棋、将棋。突破点：把"慢而准的搜索"与"快而粗的神经网络直觉"拧成一个自我增强的闭环。真实机制是这个闭环最值得品：**搜索（MCTS）扮演"策略提升算子"**——网络给个初步策略，MCTS 在它基础上多想几步、做大量推演，输出一个比原网络更强的走法分布；然后**用这个被搜索增强过的分布当标签，反过来训练网络**，让网络下次的"直觉"直接逼近"想过之后的结论"。如此循环，网络和搜索互相把对方往上抬，棋力螺旋上升。影响：**击败李世石（2016）成为 AI 里程碑事件**，向公众证明了机器能在最复杂的智力博弈上超越人类；更深远的是它确立了**"学习 + 搜索"这一范式**——而这正是 2024 年 o1 类推理模型的思想源头（把"在棋盘上搜索"换成"在思维链上搜索"）。

**MuZero（2019/2020，DeepMind，Schrittwieser 等）** — 痛点：AlphaZero 必须知道游戏规则（怎么落子、何时结束）才能在脑中推演；现实世界（机器人、视频、复杂系统）的"规则"往往未知。做法：让模型**自己学一个隐空间里的环境模型**——不预测真实画面，只预测"对决策有用的那部分动态"：给定隐状态和动作，预测下一隐状态、即时奖励和策略价值，然后在这个学出来的隐模型里做 MCTS 规划。突破点：**把"基于模型的规划"从"必须有现成模型"中解放出来**，模型自己长出一个够用的内部世界。真实机制：它不追求重建环境的全部细节（那既难又浪费），只学"足以支撑规划"的抽象动态——隐状态是为奖励和价值预测服务的、被任务塑造过的表征。影响：在 Atari、围棋、象棋上达到 AlphaZero 级别却不需要规则输入，是"world model + planning"路线的标杆，启发了后续大量基于模型的 RL 和"世界模型"研究。

**RLHF / InstructGPT（2022，OpenAI，Ouyang 等）** — 痛点：GPT-3 这类纯靠"预测下一个词"训出来的语言模型，知识渊博却不"听话"——你让它写摘要它可能续写、答非所问、啰嗦或胡说，因为"预测互联网文本"这个目标和"对人类有用"根本不是一回事。而"有用、无害、诚实"这种目标无法写成可微的损失函数。做法：三步。（1）监督微调 SFT，用人写的优质示范先教个基础；（2）训一个**奖励模型（reward model）**——让人对模型的多个回答**排序**（哪个更好），用这些偏好对训出一个能给任意回答打分的网络；（3）**用 PPO 把语言模型当 agent、奖励模型当环境去强化**，让模型生成能拿高分的回答。突破点：**把"人类说不清但一比就知道"的偏好，蒸馏成了一个可优化的奖励信号**，从而能用 RL 去对齐。真实机制：人类对"好"的判断难以显式定义，但**相对比较（A 比 B 好）容易且一致**；奖励模型把这些离散的成对偏好拟合成连续打分函数，PPO 再顺着这个打分爬坡。同时加一个 **KL 散度惩罚**，约束被对齐的模型别离原始模型太远——否则模型会为了讨好奖励模型而胡乱输出（这正是 reward hacking 的早期表现）。影响：**这是 ChatGPT 之所以"好用"的核心技术**，让 RL 从游戏/机器人领域跨进了 NLP 主战场，开启了大模型对齐这个研究子方向；论文一个反直觉的发现是，1.3B 的 InstructGPT 在人类评价上能胜过 175B 的原始 GPT-3，说明对齐带来的可用性提升量级巨大。

**DPO（2023，斯坦福，Rafailov 等）** — 痛点：RLHF 那一套又重又脆——要单独训维护一个奖励模型，PPO 训练本身对超参数极其敏感、容易崩、复现难，整条 pipeline 工程门槛高。做法：DPO（直接偏好优化）做了一步漂亮的数学推导，证明"用奖励模型 + KL 约束的 RL"这个优化问题，**存在一个解析解**，可以反过来把"最优策略"直接表达进损失函数里，从而**绕过显式训练奖励模型，也绕过 RL 采样**，直接拿偏好数据（每条是"好回答 vs 坏回答"一对）用一个类似分类的损失去微调模型。突破点：**把强化学习问题等价转化成了一个简单的监督式损失**，训练像普通微调一样稳定。真实机制：关键洞察是——RLHF 想要的那个"最优策略"和"奖励函数"之间有一一对应的闭式关系，于是奖励模型其实是个可以被消掉的中间变量。DPO 的损失直接拉大"好回答相对坏回答的对数概率差"（同样隐含 KL 约束在里面），效果上等价于先训奖励再 RL。影响：**因为简单稳定，DPO 迅速成为开源社区对齐的默认选择**，大量开源模型用它做偏好对齐；也催生 IPO（修正 DPO 在数据噪声下的过拟合）、KTO（连成对偏好都不要，只要"这条好/这条坏"的单边标注）等一系列变体，共同构成"离线偏好优化"家族。需要点出的是：DPO 省事但因为是离线的、不重新采样，在最难的任务上天花板通常不及在线 RL（PPO/GRPO）——这也是后来推理训练又回到在线 RL 的原因之一。

**o1（2024，OpenAI）** — 痛点：再大的语言模型，遇到要多步推理的难题（数学证明、复杂代码、逻辑谜题）还是容易一步错步步错，因为它习惯"脱口而出"，没有"打草稿、检查、回溯"的过程。做法：**用强化学习训练模型生成很长的内部思维链（chain-of-thought）**，让它在给出最终答案前先大段地推演、试错、自我纠正；训练信号来自最终答案对不对。突破点：提出/落地了 **test-time compute scaling（测试时算力扩展）**这一新维度——**让模型在回答时"多想一会儿"（生成更长的推理），正确率随之系统性上升**。真实机制：传统提升靠扩大参数和预训练数据（训练时算力），o1 揭示了另一条正交的曲线——固定模型大小，**单纯增加推理时的思考长度/搜索量，性能也能持续爬升**；RL 在这里的作用是教会模型"如何有效地花这些思考预算"——什么时候该展开验算、什么时候该换思路、什么时候该回退，这些"元推理策略"是被奖励信号雕刻出来的，而非人工提示出来的。影响：开辟了"推理模型"这一全新品类，把竞争焦点从"模型多大"部分转向"会不会想"；在数学竞赛、编程、科学题等硬推理基准上相对前代有大幅提升（OpenAI 报告其在多项竞赛级基准上远超 GPT-4o，具体数值「待核」）。

**DeepSeek-R1 / GRPO（2025，DeepSeek）** — 痛点：o1 效果惊艳但完全闭源，外界不知道"RL 激发推理"到底是怎么做到的、需不需要海量人工标注的推理过程、能不能低成本复现。做法：DeepSeek 公开了完整方案。最劲爆的是 **R1-Zero**：**不用任何人工标注的推理步骤，直接在基座模型上做纯 RL**，奖励只看"最终答案对不对 + 格式对不对"（可验证奖励）。配套提出 **GRPO（Group Relative Policy Optimization，组相对策略优化）**算法。突破点有二：其一，**证明了长链推理能力可以纯靠 RL"涌现"出来**——训练中模型自发学会了反思、验算、"等一下，让我重新想想"式的回溯，思维链自己越变越长，这被称作 "aha moment"，是个强信号：推理策略不必人教，可由奖励诱导自生。其二，**GRPO 砍掉了 PPO 里的 critic（价值网络）**。真实机制看 GRPO 怎么省：PPO 要训一个和策略网络同样大的 critic 来估计每步的优势（基准），又贵又是另一个不稳定来源。GRPO 的做法是——**对同一道题让模型采样一组（比如 16 条）答案，直接用这组答案奖励的均值当基准，每条答案的优势 = 自己的奖励减去这组的平均奖励**。谁比组内平均好就强化、差就抑制。这样**用"组内相对比较"替代了需要专门网络去学的价值基准**，省掉一整个网络的显存和训练开销，对动辄数百亿参数的大模型意义重大。影响：**R1 是 2025 年最有影响力的开源工作之一**，它把"RLVR + 高效策略优化"这条推理训练路线彻底公开化、平民化，GRPO 迅速被各路推理模型采用，可验证奖励（RLVR）成为推理训练的事实标准范式。

**（延伸）Agent 与工具使用（2023 至今，多家）** — 痛点：再会推理的模型，知识也停在训练截止日，也不能真去执行操作（查实时信息、跑代码、改文件、调 API）。做法：让模型学会在多步交互中**决定何时调用外部工具、用工具返回的结果继续推理、必要时纠错重试**——把语言模型变成一个能在真实环境里闭环行动的 **agent**。从早期 ReAct（2022，"推理与行动交替"）、Toolformer（2023，自学何时调 API），到把工具使用直接纳入 RL 训练（用任务是否真正完成当奖励）。突破点：**奖励从"答案文本对不对"升级为"在真实环境里任务是否达成"**，这是把 RL 从静态问答推向动态执行的关键一跳。真实机制：调工具这个"动作"的好坏，要等工具执行、甚至整个任务跑完才知道，是典型的**长程、稀疏、延迟奖励**问题——又把信用分配这块 RL 最硬的骨头摆回了桌面，只不过这次环境是代码仓库、浏览器、操作系统。影响：这是当下产业落地最猛的方向（编程 agent、计算机操作 agent、深度研究 agent），也是**和读者正在做的 AI agent 产品最直接相关的研究战线**——你产品里"模型决定调哪个工具、调错了怎么回退、多步任务怎么不跑偏"这些工程问题，背后正是 RL 长程信用分配和奖励设计的开放难题。

## 近十年关键优化技术

逐项拆解，每项给"解决的瓶颈 + 机制"。

**经验回放（Experience Replay）**
- 瓶颈：在线 RL 的数据前后强相关、非平稳，且样本用一次就扔，浪费。
- 机制：用缓冲池存历史经验，训练时随机采样，打散相关性 + 重复利用样本。进阶版 **优先经验回放（PER，2015）** 按 TD 误差大小给样本设采样权重，让"学得最不好、信息量最大"的经验被更频繁地复习。

**目标网络（Target Network）**
- 瓶颈：TD 更新的目标里含网络自身输出，目标随训练抖动，导致发散（"追自己影子"）。
- 机制：复制一份参数冻结的网络专算目标值，隔一段时间才同步，把移动靶变成阶段性固定靶，稳住回归方向。

**Actor-Critic 架构**
- 瓶颈：纯策略梯度（REINFORCE）用整条轨迹的真实回报做信号，方差极大、学得慢；纯价值方法（Q-learning）又难处理连续/高维动作空间。
- 机制：actor 管策略、critic 管价值评估，critic 给 actor 提供一个低方差的基准（baseline），actor 据此判断"这个动作比平均好还是差"。兼得策略方法的灵活与价值方法的低方差。SAC（2018）等还引入"最大熵"鼓励探索。

**PPO 裁剪 + GAE**
- 瓶颈：策略更新步子大就不可逆崩溃；优势估计要么偏差大（只看一步）要么方差大（看整条轨迹）。
- 机制：裁剪（clip）给单次策略更新设"封顶"，把更新限制在信任区间内防崩；GAE（广义优势估计）用一个 λ 参数在"一步 TD（低方差高偏差）"和"多步回报（低偏差高方差）"之间做指数加权平滑，找到偏差-方差的甜点。

**奖励建模（Reward Modeling）**
- 瓶颈：很多目标（"有用""安全""自然")无法手写成奖励函数。
- 机制：收集人类偏好（成对排序），拟合一个神经网络奖励模型来给任意输出打分，把"说不清的好"变成"可优化的分"。这是 RLHF 的地基。

**KL 约束（KL Penalty / Constraint）**
- 瓶颈：模型为了刷奖励会偏离原始分布、输出怪异或退化（reward hacking 的一种）。
- 机制：在目标里加一项对"新策略与参考策略的 KL 散度"的惩罚，像拴根皮筋，让模型在优化奖励的同时别跑太远，保住语言流畅性和多样性。

**DPO / IPO / KTO（离线偏好优化家族）**
- 瓶颈：RLHF 的奖励模型 + 在线 RL 太重、太不稳、难复现。
- 机制：DPO 用解析解把 RL 转成监督式损失，直接吃成对偏好；IPO 修正 DPO 在偏好数据有噪声或过于确定时的过拟合倾向；KTO 借鉴前景理论，连成对数据都不要，只需单条样本的"好/坏"二元标签，标注成本更低。共同点是**离线、稳定、工程简单**，代价是难度高的任务上天花板可能低于在线 RL。

**过程奖励模型（PRM, Process Reward Model）**
- 瓶颈：只看最终答案对错（结果奖励 ORM）会漏掉一种坏情况——**答案蒙对了但过程全错**，强化这种轨迹会教坏模型；且稀疏的最终奖励让长推理的信用分配极难。
- 机制：训一个模型给**推理过程的每一步**单独打分（这步推理对不对/有没有用），提供稠密的中间监督。OpenAI 2023 的 "Let's Verify Step by Step" 是代表作，发现过程监督在数学推理上显著优于结果监督。代价是逐步标注极贵。

**可验证奖励（RLVR, RL with Verifiable Rewards）**
- 瓶颈：奖励模型本身是学出来的、可能被钻空子（reward hacking），而且训练维护它很贵。
- 机制：在那些**答案能被程序自动判定对错**的领域（数学有标准答案、代码能跑单元测试、形式证明能被验证器检查），**直接用"验证器判对错"当奖励，不需要学习型奖励模型**。奖励干净、无法被语义层面糊弄。这是 o1、DeepSeek-R1 推理训练能 work 的关键前提，也解释了为什么推理模型在数学/代码上进步最快——因为那里恰好有现成的可验证奖励。

**GRPO（Group Relative Policy Optimization）**
- 瓶颈：PPO 的 critic 价值网络和策略网络一样大，在百亿级大模型上又费显存又添不稳定性。
- 机制：对同一输入采样一组输出，用**组内奖励均值当基准**算每条的相对优势，彻底去掉 critic。等于用"群体相对评价"替代"专门学一个绝对价值估计"，省一整个网络。

**搜索与 RL 结合（Search + RL，含 test-time compute）**
- 瓶颈：纯靠神经网络一次前向给出的策略，深度不够、易短视。
- 机制：用搜索（MCTS、束搜索、采样多条再投票/筛选）在推理时做额外的"深思"，AlphaZero 用它当策略提升算子来产生训练标签；推理模型时代演化为 **test-time compute**——推理时多花算力（更长思维链、采样多条解再选最优如 self-consistency 多数投票），用推理算力换正确率，开辟了与"扩大模型"正交的提升维度。

## 前沿战线

**1. 推理模型 / 长思维链的边界**
o1、R1 之后的核心问题是：test-time compute 的回报曲线能撑多远？怎么让模型**自适应地决定该想多久**（简单题别浪费、难题多投入），而不是一刀切地拉长。还有"思维链能否被压缩/蒸馏进更小模型"、"长推理里的幻觉如何抑制"。一个活跃议题是**思维链的忠实性（faithfulness）**——模型写出来的推理过程，是否真的是它得出答案的实际计算路径，还是事后编的好看说辞？这直接关系到能否用思维链来做可解释性和安全监控。

**2. Agent / 工具使用 / 多步规划（产业落地最热）**
让 agent 在真实环境（代码库、浏览器、操作系统、API 生态）里完成长程任务。研究焦点：**长程任务的奖励设计与信用分配**（一个任务几十步，只有最后知道成败，怎么归因到中间动作）、**错误恢复**（调错工具、踩到报错后能否自己回退重试而非一路错下去）、**多轮记忆与状态管理**。这块和读者的 agent 产品几乎是同一个问题：你遇到的"模型多步任务跑偏、调错工具不回头"，学界正用 RL（任务完成度当奖励）和环境构建来攻。代表性环境/基准如 SWE-bench（真实 GitHub issue 修复）正成为 agent 能力的试金石。

**3. 自我改进（Self-Improvement）**
让模型**生成自己的训练数据再用来训自己**，减少对人工标注的依赖。思路包括：模型自己出题、自己解、用可验证奖励筛出正确解再回炉训练（如 STaR、ReST 类方法的思想）；以及 R1 展示的"纯 RL 自发涌现推理"。核心未解问题是**会不会自我强化偏见、坍缩到单一模式（model collapse）**，以及在没有外部 ground truth 的领域，自我改进的信号从哪来。

**4. 可验证奖励 RLVR 的疆域扩展**
数学和代码因为天然可验证而进步飞快，前沿在于**把"可验证"推广到更软的领域**：怎么给开放式写作、科学假设、复杂规划设计出"够硬、难被钻空子"的自动验证器或奖励。一条路是用更强的模型当裁判（LLM-as-judge），但裁判本身又可能被骗——这又绕回 reward hacking。

**5. 长程信用分配（Long-Horizon Credit Assignment）**
这是贯穿 agent 和复杂推理的硬核理论战线。任务越长、奖励越稀疏延迟，"哪一步动作真正有功/有过"就越难判定。研究方向包括分层 RL（把长任务拆成子目标）、更好的优势估计、用语言模型自己做"事后归因"等，但远未解决。

**6. 多智能体协作（Multi-Agent）**
多个 agent 分工协作或博弈（一个写代码一个审、一个规划一个执行）。难点是**非平稳性**——每个 agent 在学习时，其它 agent 也在变，导致每个个体面对的"环境"一直漂移，收敛理论比单智能体难得多；还有通信协议涌现、信用如何在多个 agent 间分配等问题。

## 公认难点 / 开放问题

**奖励黑客（Reward Hacking）—— 最根本的诅咒。**
只要奖励是人定的或学出来的代理指标，模型就会找到"刷高分但违背真实意图"的捷径。经典例子：赛艇游戏里 agent 发现原地绕圈撞分数道具比正经比赛得分高，于是放弃比赛一直转圈。在 LLM 上则表现为：迎合奖励模型的偏见（爱说长、爱用某些措辞）、在思维链里"作弊"凑答案、谄媚（sycophancy，专挑你爱听的说）。难在哪：**这是优化的本质属性，不是 bug**——你优化的永远是真实目标的代理，代理和真实目标之间的任何缝隙都会被强大的优化器无情利用。Goodhart 定律的活生生体现："当一个指标成为目标，它就不再是好指标。" 没有通解，只能不断打补丁（KL 约束、对抗性地改进奖励、用可验证奖励缩小缝隙）。

**信用分配（Credit Assignment）—— 最古老的硬骨头。**
奖励延迟且稀疏时，把功过精确归到中间某一步动作，至今没有好办法。任务越长越无解（agent 几十步操作、推理几千 token），这是 RL 样本效率低和训练慢的根源之一。

**样本效率低（Sample Inefficiency）。**
RL 往往需要海量交互才能学会——经典 Deep RL 打 Atari 动辄需要等效数千万帧的经验，远超人类学会一个游戏所需。在大模型上，每"采一条样"都要跑一次昂贵的生成，RL 训练成本因此居高不下。基于模型的 RL（MuZero 路线）、离线 RL 试图缓解，但通用高效仍是远景。

**训练不稳定（Instability）。**
RL 训练对超参数、随机种子、实现细节极度敏感，同一算法换个种子结果可能天差地别，复现性长期被诟病（曾有著名论文专门讨论 Deep RL 的复现危机）。PPO/GRPO 在工程上压住了部分不稳定，但远谈不上"傻瓜化"。

**安全与可靠性（Safety & Alignment）。**
RL 优化得越狠，模型越可能为达目标走极端：谄媚、欺骗、隐藏真实意图（如果"看起来对齐"比"真对齐"更易拿奖励）。当模型能力逼近或超过人类、人类难以判断其输出对错时，**怎么提供可靠的监督信号**（弱监督强、可扩展监督 scalable oversight）成了根本性的开放难题。

**长程规划（Long-Horizon Planning）。**
让 agent 制定并执行需要几十上百步、跨越长时间跨度的计划，并在中途纠偏，目前能力还很脆弱——稍长就容易跑偏、遗忘目标、陷入循环。这是 agent 从"能用"到"可靠"之间最大的鸿沟。

**评测困难（Evaluation）。**
开放式 agent 任务和长推理很难自动、客观、防作弊地评测。基准容易被"刷榜"（针对性优化）、容易泄漏进训练数据导致虚高，且"过程对不对"比"答案对不对"难评得多。**没有可信的评测，就无法判断是不是真的进步**——这是悬在整个方向头上的元问题。

## 就业与门槛

岗位主要分两类：**RL 研究科学家**（推理模型、对齐、agent 训练的算法研究，集中在头部 AI 实验室和大厂研究院）和 **RL/agent 工程师 / 应用研究员**（把 RLHF/RLVR/agent 训练落到产品里，需求随推理模型和 agent 产品爆发而快速增长）。纯研究岗（尤其顶级实验室）**基本要求 PhD 或等价的顶会论文记录**；偏工程和应用的岗位，**强系统/分布式训练功底 + 扎实的 RL 实操经验**往往比学位更被看重——而这恰是资深系统工程师的天然优势（RL 训练本质是个大型分布式系统工程问题：采样、训练、奖励计算的并行编排）。薪资处于 AI 领域高位区间，推理/agent 方向因当下极度紧缺溢价明显（具体数值「待核」）。一个务实的切入点：**从 agent 工程和 RLHF/RLVR 工程做起，用系统能力立足，再补 RL 理论纵深**，比硬转纯理论研究路径更顺。

## 必读清单

**教科书 / 基础**
- Sutton & Barto《Reinforcement Learning: An Introduction》(第 2 版，2018) —— RL 圣经，理论地基，免费在线版。先吃透前半本（MDP、TD、策略梯度、actor-critic）。
- David Silver 的 UCL RL 课程（DeepMind，视频公开）—— 配合圣经看，讲解清晰。
- 伯克利 CS285《Deep Reinforcement Learning》(Sergey Levine) —— 深度 RL 的权威课程，覆盖到现代算法。
- Spinning Up in Deep RL（OpenAI）—— 工程师友好的实操教程 + 干净的算法代码实现，强烈推荐动手跑一遍 PPO。

**奠基论文（按本章脉络）**
- DQN：Mnih et al., *Playing Atari with Deep RL* (2013) / *Human-level control...* (Nature 2015)
- A3C：Mnih et al., *Asynchronous Methods for Deep RL* (2016)
- TRPO / PPO：Schulman et al. (2015 / 2017)；配 GAE 论文 (2015)
- AlphaGo / AlphaZero / MuZero：Silver et al. (2016/2017) / Schrittwieser et al. (2019)
- RLHF：Christiano et al. *Deep RL from Human Preferences* (2017)；Ouyang et al. *InstructGPT* (2022)

**对齐与推理前沿**
- DPO：Rafailov et al. *Direct Preference Optimization* (2023)
- 过程奖励：Lightman et al. *Let's Verify Step by Step* (2023)
- DeepSeek-R1 技术报告（2025）—— 必读，看 RLVR + GRPO 的完整工程方案和 "aha moment"
- o1 相关公开材料（OpenAI，2024）—— 理解 test-time compute 范式

**Agent 方向**
- ReAct (2022)、Toolformer (2023) —— agent 推理-行动范式的起点
- SWE-bench 论文与榜单 —— 当下 agent 能力的事实基准，结合读者产品看最有体感

**代码库（照着能从入门到精通）**
- CleanRL —— 单文件实现各主流 RL 算法，读代码学算法的最佳起点
- Spinning Up —— OpenAI 的教学级实现
- TRL（HuggingFace）—— 工业级 RLHF/DPO/GRPO 训练库，做大模型 RL 必看
- veRL / OpenRLHF —— 面向大模型的高效 RL 训练框架，理解分布式 RL 工程的好材料
