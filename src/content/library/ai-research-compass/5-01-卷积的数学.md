> 这一章把你从"知道 CNN 用卷积"带到能从第一性原理推导卷积的每一个数字——输出尺寸、参数量、FLOPs、感受野——并能解释清楚它为什么是处理图像的正确工具,能自己从循环实现 conv2d 并改造出 1×1、空洞、深度可分离等所有变体。

## 引言:为什么卷积值得专门讲一章

卷积是现代视觉模型的**原子操作**。从 2012 年的 AlexNet 到今天的扩散模型 U-Net、检测器的 backbone,卷积层都是承重墙。但绝大多数教程把它当成一个"滑动窗口求和"的黑箱:你知道怎么调 `nn.Conv2d(64, 128, 3, padding=1)`,却说不清这一行背后到底发生了什么数学、为什么参数量是这个数、为什么深层网络"看得更广"、为什么它比全连接层更适合图像。

这种含糊在你读论文、调架构、做模型压缩时会立刻反噬。你看到 MobileNet 说"深度可分离卷积省了 8-9 倍参数",但说不清那个 8-9 倍是怎么来的、能不能预测;你看到分割论文用 dilated convolution"扩大感受野",但说不清扩了多少、代价是什么;你想给检测器选 backbone,却不会算它的理论感受野够不够覆盖目标尺寸。

本章的目标是把这些含糊全部消掉。我们从离散卷积的定义出发,推导出输出尺寸、参数量、FLOPs 三个核心公式;证明卷积对平移**等变**(equivariant)——这是它契合图像的根本原因;推导逐层感受野递推公式,并解释**有效感受野**为什么比理论值小得多;然后逐个拆解 1×1、空洞、深度可分离、转置卷积四个最重要的变体,以及池化的作用与争议。最后从零用 Python 循环实现 conv2d 并与 PyTorch 对齐。

预备知识:你需要会基本的线性代数(矩阵乘法、向量内积)、会读 PyTorch 张量形状 `(N, C, H, W)`。不需要任何深度学习背景。

## 1. 离散卷积 vs 互相关:深度学习里到底在算什么

### 问题:卷积的数学定义

数学上,两个一维离散信号 f 和 g 的**卷积**(convolution)定义为:

```
(f * g)[n] = Σ_m  f[m] · g[n − m]
```

注意那个 `g[n − m]`:卷积要把核 g **翻转**(flip)后再滑动。这个翻转来自卷积的本质——它描述的是一个线性时不变系统对输入的响应,翻转保证了卷积满足交换律 f*g = g*f 和结合律,这些性质在信号处理和傅里叶分析里至关重要。

但是——**深度学习里几乎没有人真的做翻转**。PyTorch 的 `nn.Conv2d`、TensorFlow 的 `conv2d`,算的其实是**互相关**(cross-correlation):

```
(f ⋆ g)[n] = Σ_m  f[m] · g[n + m]
```

二维形式(图像处理里我们关心的)是:

```
out[i, j] = Σ_u Σ_v  in[i + u, j + v] · w[u, v]
```

核 w 不翻转,直接对齐到输入上做逐元素相乘再求和。

### 为什么深度学习可以"偷懒"不翻转

这是新手第一个该想通的点。答案是:**核 w 是学出来的,不是给定的**。

如果你做真卷积(带翻转),网络会学到一个翻转后的核 w_flip;如果你做互相关(不翻转),网络会直接学到那个翻转后的核本身。两者表达能力完全等价——最优解只是互为翻转。既然如此,翻转就成了纯粹的计算浪费。所以工程上一律用互相关,但沿用"卷积"这个名字(因为听起来更高级,也因为习惯)。

**本章后面所有"卷积"都指互相关,公式都不带翻转。** 这是你读源码时不会困惑的关键:`F.conv2d` 的名字叫卷积,实现是互相关。如果你真要数学意义上的卷积,得自己把核翻转,或者用 `scipy.signal.convolve`。

### 多通道的完整定义

真实图像是多通道的(RGB 是 3 通道,中间层可能是几百通道)。一个卷积层把 C_in 个输入通道映射到 C_out 个输出通道。完整定义:

```
out[oc, i, j] = bias[oc] + Σ_{ic=0}^{C_in−1} Σ_{u=0}^{K−1} Σ_{v=0}^{K−1}
                            in[ic, i·S + u, j·S + v] · W[oc, ic, u, v]
```

关键理解:**每个输出通道 oc 对应一个独立的 3D 核** W[oc, :, :, :],形状是 (C_in, K, K)。这个 3D 核同时看所有输入通道,把它们加权融合成一个标量。有 C_out 个这样的 3D 核,就产出 C_out 个输出通道。所以权重张量 W 的总形状是 `(C_out, C_in, K, K)`——这正是 PyTorch 里 `conv.weight.shape` 给你的。

记住这张图:**卷积核的"深度"必须等于输入通道数,核的"个数"决定输出通道数**。这一句能帮你避开 90% 的形状错误。

## 2. 三个核心公式:尺寸、参数量、FLOPs

工程上你需要在脑子里就能算出一个卷积层的三个数。下面逐个推导,并给出可验算的例子。

### 2.1 输出尺寸公式

**问题**:输入空间尺寸 W(宽或高),核大小 K,padding P,stride S,输出尺寸是多少?

**推导**(沿一个空间维度):

```
1. 先 padding:输入两侧各补 P 个零,有效宽度变成  W + 2P
2. 核能放下的最左位置是 0,最右位置是  (W + 2P) − K
3. 步长 S 表示核每次跳 S 格,所以有效起点是 0, S, 2S, ...
4. 起点数量 = floor( ((W + 2P) − K) / S ) + 1   （+1 因为含起点 0）
```

得到输出尺寸公式:

```
W_out = ⌊ (W − K + 2P) / S ⌋ + 1
```

**验算**:W=7, K=3, P=1, S=2 → (7−3+2)/2 + 1 = 6/2 + 1 = **4**。(我用代码验证过,确实是 4。)

几个你必须背下来的特例:

- **K=3, P=1, S=1** → W_out = W。"same" 卷积,空间尺寸不变,最常用。
- **K=1, P=0, S=1** → W_out = W。1×1 卷积永远不改空间尺寸。
- **K=3, P=1, S=2** → W_out = ⌈W/2⌉。下采样减半,用 stride 代替池化。
- 一般地,**stride=S 把尺寸缩小约 S 倍**。

新手坑:`floor` 是真的会丢信息。如果 W=8, K=3, P=0, S=2 → (8−3)/2 + 1 = 2.5 → floor → **3**,最后一列输入被丢弃了。这就是为什么有些框架会区分 padding='valid' 和精心设计的 padding 值,以及为什么不对齐的尺寸会让你 debug 半天。

### 2.2 参数量

**问题**:一个 `Conv2d(C_in, C_out, K)` 有多少可学习参数?

**推导**:我们刚说过权重张量形状是 (C_out, C_in, K, K),再加上每个输出通道一个 bias:

```
params = C_out · C_in · K · K  +  C_out
       = K² · C_in · C_out  (+ C_out  for bias)
```

**验算**:Conv2d(64, 128, 3) → 3²·64·128 = 9·8192 = 73728,加 bias 128,共 73856。

**关键洞察:参数量与输入空间尺寸 H、W 完全无关!** 一个 3×3 卷积处理 224×224 的图和处理 7×7 的特征图,参数量一模一样。这正是卷积相对全连接的核心优势(下一节会从"权重共享"角度严格说明)。对比一下:如果用全连接层把 64×56×56 映射到 128×56×56,参数量是 (64·56·56)·(128·56·56) ≈ 8×10¹⁰,而对应的 3×3 卷积只要 7.4 万。差了约 6 个数量级。

### 2.3 FLOPs(计算量)

**问题**:这一层前向要做多少次乘加?

**推导**:每个输出元素需要做一次"3D 核与输入块的内积",内积的长度是 C_in·K·K,即这么多次乘法 + 大约这么多次加法。输出元素总数是 C_out·H_out·W_out。所以:

```
乘加次数(MACs) ≈ H_out · W_out · C_out · C_in · K²
FLOPs ≈ 2 · MACs   (一次乘加 = 1 乘 + 1 加,部分文献只数 MAC)
```

**验算**:Conv2d(64, 128, 3, padding=1) 处理 56×56 输入 → 输出也是 56×56 → MACs = 56·56·128·64·9 ≈ 2.31×10⁸ ≈ **0.23 GMACs**。

注意 FLOPs **与空间尺寸成正比**(参数量不变,但每个像素都要重算)。这解释了一个反直觉现象:深层网络的参数大头在通道多、空间小的后期层(因为 C_in·C_out 大),但 FLOPs 大头常在通道少、空间大的早期层(因为 H_out·W_out 大)。做模型加速时,这个区分决定你该砍哪里。

把三个公式放在一起记:

```
尺寸:  W_out = ⌊(W − K + 2P)/S⌋ + 1
参数:  K²·C_in·C_out  (+ C_out)        ← 与 H,W 无关
FLOPs: ≈ 2·H_out·W_out·C_out·C_in·K²   ← 与 H,W 成正比
```

## 3. 卷积是全连接的特例:稀疏连接 + 权重共享

这一节解释卷积**为什么参数这么少**,从"它是受约束的全连接层"这个视角看透。

### 把卷积摊平成矩阵乘法

任何线性层都可以写成 y = Wx 的矩阵乘法。卷积也是线性的(它只是加权求和),所以也能写成矩阵乘。我们把输入图像拉成一个长向量 x ∈ ℝ^(H·W)(先只看单通道、单输出通道),输出也拉成向量 y,那么存在一个矩阵 M 使 y = Mx。

对于一维、K=3、stride=1、无 padding 的卷积,核是 [w₀, w₁, w₂],这个矩阵 M 长这样(以输入长度 5、输出长度 3 为例):

```
      x₀  x₁  x₂  x₃  x₄
y₀ [ w₀  w₁  w₂   0   0 ]
y₁ [  0  w₀  w₁  w₂   0 ]
y₂ [  0   0  w₀  w₁  w₂ ]
```

对照全连接层的矩阵——那会是一个 3×5 的稠密矩阵,15 个**独立**参数。而卷积的 M 体现了两个强约束:

1. **稀疏连接(sparse connectivity)**:每个输出只连接 K 个输入(其余全是 0)。每行只有 3 个非零。这对应"局部性"——图像中相邻像素相关,远处像素直接关联弱,所以一个输出神经元只需看一个局部窗口。
2. **权重共享(weight sharing / parameter sharing)**:那些非零值不是 15 个自由参数,而是**同一组 [w₀, w₁, w₂] 在每一行平移复用**。所以独立参数只有 K=3 个。这对应"平移不变的统计规律"——一个边缘检测器在图像左上角有用,在右下角同样有用,没理由为每个位置学一套。

**卷积 = 全连接层 + 稀疏约束(局部连接) + 权重共享约束(同一核在所有位置复用)。** 这就是它参数少几个数量级的全部原因。这两个约束不是为了省参数硬加的,而是**编码了关于图像的先验知识**(归纳偏置,inductive bias):图像具有局部性和平移规律性。约束越贴合数据的真实结构,需要的参数和数据就越少——这也是为什么 CNN 在中小数据集上往往打得过参数更多、约束更弱的全连接网络和早期 Vision Transformer。

> 工程上 PyTorch 其实不构造稀疏矩阵 M(太大太空),而是用 **im2col + GEMM**:把每个输出位置对应的输入窗口展开成一列,拼成一个稠密矩阵,再做一次大的矩阵乘法。这样能直接调用高度优化的 BLAS/cuBLAS。代价是 im2col 后的中间张量会膨胀 K² 倍,吃显存——这是你 OOM 时该想到的一个隐藏开销。

## 4. 平移等变性:卷积契合图像的根本

这是本章最重要的理论结论,也是最常被讲糊的地方。我们先把概念辨清楚,再给证明。

### 等变 vs 不变:别再混淆

- **平移等变(equivariant)**:输入平移,输出**也跟着平移同样的量**。形式化:若 T 是平移算子,卷积算子是 Φ,则 `Φ(T(x)) = T(Φ(x))`。
- **平移不变(invariant)**:输入平移,输出**完全不变**。`Φ(T(x)) = Φ(x)`。

**卷积本身是平移等变的,不是不变的。** 把一只猫从图像左边移到右边,卷积特征图上的"猫的响应"也从左移到右,而不是原地不动。等变正是我们想要的:它保留了"东西在哪"的空间信息,同时"东西是什么"的检测逻辑在所有位置一致。

### 证明:卷积对平移等变

设输入 x,平移算子 T_t 定义为 (T_t x)[i] = x[i − t](把信号整体右移 t)。卷积算子 (Φx)[i] = Σ_u x[i + u]·w[u](互相关,单通道一维,无 padding 边界问题,或假设循环/无限域)。

```
要证：Φ(T_t x) = T_t(Φ x)

左边：
  [Φ(T_t x)][i] = Σ_u (T_t x)[i + u] · w[u]
                = Σ_u x[(i + u) − t] · w[u]        （代入平移定义）
                = Σ_u x[(i − t) + u] · w[u]        （重组下标）
                = [Φ x][i − t]                      （这正是卷积在位置 i−t 的值）

右边：
  [T_t(Φ x)][i] = [Φ x][i − t]                     （平移定义直接作用在 Φx 上）

左边 = 右边  ∎
```

证毕。这个证明的核心只用到一个事实:**卷积权重 w[u] 不依赖于绝对位置 i**(权重共享)。正是"同一个核在每个位置复用"让这个推导成立。如果每个位置用不同的核(就像全连接层),等式立刻不成立——这从数学上说明了权重共享与平移等变是**同一件事的两面**。

我用循环移位(circular shift,避免边界效应)数值验证过这个等式,左右两边的最大差异是 **0.0**(浮点精确相等)。

边界的诚实说明:真实图像是有限大的,平移会把内容移出边界、把零(padding)移进来,所以实际卷积只是**近似**等变,边界附近会破坏等式。这也是为什么 padding 方式(zero / reflect / circular)会微妙地影响模型表现,以及为什么有研究指出 CNN 会利用 padding 泄露的绝对位置信息(待核:具体是 Islam et al. 2020 "How much position information do CNNs encode?",结论方向可靠,细节我标待核)。

### 近似平移不变从哪来

既然卷积只是等变,那 CNN 做分类(同一只猫不管在哪都该判成猫,这需要不变性)靠什么?靠两个东西的叠加:

1. **池化 / stride 下采样**:max pooling 在一个小窗口里取最大,只要目标在窗口内移动,输出就不变——这带来**局部**的近似平移不变。逐层堆叠后,小位移被层层"吸收"。
2. **全局池化(global average/max pooling)**:在最后把整张特征图压成一个向量,彻底丢掉空间位置——这才是分类头获得**全局**平移不变的主要来源。

所以正确的图景是:**卷积层负责等变地提取特征,下采样和全局池化负责把等变逐步转化成任务需要的不变性。** 把这两者分开理解,你才不会在"卷积到底变不变"上反复绕圈。

## 5. 感受野:为什么深层"看得更广"

**感受野**(receptive field, RF)指输出特征图上一个点,对应到原始输入图像上的区域大小。它决定一个神经元"能看到多大范围的上下文"。检测大目标需要大感受野,分割需要既大又精细的感受野——所以会算感受野是架构设计的硬技能。

### 5.1 逐层递推公式

**问题**:堆叠 L 层卷积,第 L 层的感受野是多大?

定义两个量逐层递推:RF_l 是第 l 层的感受野大小,jump_l 是第 l 层每个点之间在原图上的间距(也叫 effective stride,等于前面所有 stride 的乘积)。

**推导**(以第 l 层核大小 K_l、步长 S_l 为例):

```
初始（输入层,第 0 层）：
  RF₀ = 1,  jump₀ = 1

递推：
  jump_l = jump_{l−1} · S_l                      ← 间距是历史 stride 连乘
  RF_l   = RF_{l−1} + (K_l − 1) · jump_{l−1}      ← 关键递推
```

为什么 RF 的增量是 `(K_l − 1) · jump_{l−1}`?第 l 层一个输出点看 K_l 个第 l−1 层的点,这 K_l 个点在原图上彼此相距 jump_{l−1}。它们各自的感受野左右排开,新增覆盖的宽度就是 (K_l − 1) 个间距,即 (K_l − 1)·jump_{l−1}。最左点和最右点的感受野中心相距这么多,加上原来 RF_{l−1} 的"半径外延"刚好拼成新感受野。

**验算**:三层 3×3、stride 全为 1。

```
层1: RF = 1 + (3−1)·1 = 3,   jump = 1
层2: RF = 3 + (3−1)·1 = 5,   jump = 1
层3: RF = 5 + (3−1)·1 = 7,   jump = 1
```

结果 RF=7,我用代码验证过。这就是著名的"两个 3×3 等价于一个 5×5、三个 3×3 等价于一个 7×7"——VGG 的核心设计动机。但三个 3×3 的参数是 3·(3²C²)=27C²,一个 7×7 是 49C²,**3×3 堆叠用更少参数得到同样感受野,还多了两次非线性激活**,表达力更强。这是 VGG(2014)击败大核网络的关键洞察。

stride 的威力:如果中间插一个 stride=2 的层,jump 翻倍,后续每层的 RF 增量都翻倍。这就是**为什么深层(经过多次下采样后)感受野增长得特别快**——靠的不是核变大,而是 jump 连乘放大了每层的贡献。空洞卷积(下一节)是另一条不靠下采样扩 RF 的路。

### 5.2 有效感受野:理论值是个谎言

这是 2016 年 Luo et al. 的重要发现(NeurIPS,"Understanding the Effective Receptive Field in Deep CNNs"),也是新手严重低估的坑:**理论感受野内,各个输入像素对输出的影响并不均等;中心像素影响大,边缘像素影响微乎其微。真正起作用的"有效感受野"(effective receptive field, ERF)远小于理论值,而且呈高斯衰减。**

直觉论证:从中心输出点反向看,有多少条"路径"能从某个输入像素传到它?对于堆叠的 3×3 卷积,从输入到输出的路径数服从类似多项式分布——**中心像素的路径数指数级多于边缘像素**。把每层看成对位置做一次随机游走的卷积,堆叠 L 层就是 L 次独立游走的叠加,中心极限定理告诉我们影响分布趋于**高斯**,标准差按 √L 增长。

定量结论(可推导):理论 RF 半径随层数 L **线性**增长(每层加固定量),但 ERF 的标准差只随 **√L** 增长。所以:

```
ERF 半径 / 理论 RF 半径  ∝  √L / L  =  1/√L  → 随网络加深而趋于 0
```

我用"每层 3×3 = 每步在 {−1,0,1} 上随机游走、方差 2/3"的模型算了几个点:

```
L=4 : 理论 RF 半径=4,  ERF 标准差≈1.63, 比值≈0.41
L=16: 理论 RF 半径=16, ERF 标准差≈3.27, 比值≈0.20
L=64: 理论 RF 半径=64, ERF 标准差≈6.53, 比值≈0.10
```

层数越深,有效感受野占理论感受野的比例越小。**工程后果**:你以为堆够层数就能"看到"整张图,实际上中心区域之外的信息几乎传不进来。这正是分割/检测里要主动用 dilated convolution、大核(如 ConvNeXt 的 7×7、RepLKNet 的 31×31)、或 attention 来真正扩大有效感受野的根本原因。光靠堆小卷积扩 ERF,效率随 √L 递减,极其低效。

## 6. 关键卷积变体:每一个都解决一个具体问题

掌握了基础,现在看四个最重要的变体。每个都是为解决一个具体痛点而生。

### 6.1 1×1 卷积:跨通道的线性组合

**问题**:怎么在不改变空间尺寸、不看空间邻域的情况下,改变通道数 / 融合通道信息?

**机制**:K=1 时,卷积公式退化为(去掉空间求和):

```
out[oc, i, j] = Σ_ic  in[ic, i, j] · W[oc, ic]
```

这就是**在每个空间位置独立地,对 C_in 维通道向量做一次全连接(矩阵乘)**,输出 C_out 维。它完全不混合空间信息,只做**跨通道的线性组合**。

参数量 = C_in·C_out(K=1),FLOPs = H·W·C_in·C_out。用途:

- **降维 / 升维**:用 1×1 把 256 通道压到 64,大幅减少后续 3×3 卷积的计算(因为 FLOPs 正比于 C_in·C_out)。这就是 ResNet **bottleneck** 块的设计:1×1 降维 → 3×3 卷积(在低维上算,便宜)→ 1×1 升维。GoogLeNet/Inception(2014)最早系统地用 1×1 做降维。
- **跨通道信息融合 + 增加非线性**:1×1 卷积后接激活,等于给"通道方向"加了一层 MLP(这正是 Network-in-Network 2013 的思想)。

### 6.2 空洞(膨胀)卷积:扩感受野不增参数

**问题**:分割任务要大感受野,但又不想下采样(下采样丢空间分辨率,分割要逐像素精细),也不想增加参数。

**机制**:dilated/atrous convolution 在核元素之间插入空洞,用膨胀率 d 控制采样间隔:

```
out[i, j] = Σ_u Σ_v  in[i + u·d, j + v·d] · w[u, v]
```

d=1 就是普通卷积。**有效核大小**变为:

```
K_eff = K + (K − 1)·(d − 1)
```

**验算**:3×3 核,d=2 → 3 + 2·1 = **5**(感受野像 5×5);d=4 → 3 + 2·3 = **9**(感受野像 9×9)。我验证过这两个数。

**精妙之处:参数量仍是 K²(只有 9 个权重),感受野却涨到 K_eff²。** 用 d=2 的 3×3 卷积,9 个参数拿到 5×5 的感受野。DeepLab 系列(2015-2018)用串联不同膨胀率的卷积(ASPP, Atrous Spatial Pyramid Pooling)在不降分辨率的前提下聚合多尺度上下文,是语义分割的奠基设计。

代价 / 坑:**网格效应(gridding artifact)**。如果连续几层用相同的 d,有些输入像素永远采不到(核只在固定栅格上采样),特征图会出现棋盘状空洞。解法是用**锯齿状递增**的膨胀率(如 1,2,5 这种互质组合,Hybrid Dilated Convolution),让覆盖填满。这是用 dilated conv 必踩的坑。

### 6.3 深度可分离卷积:MobileNet 省参数的核心

**问题**:移动端要极致省参数和 FLOPs,标准卷积的 K²·C_in·C_out 太贵。

**机制**:把标准卷积**分解**成两步,各管一件事:

1. **Depthwise convolution(逐通道空间卷积)**:每个输入通道**单独**用一个 K×K 核做空间卷积,不跨通道。C_in 个通道 → C_in 个通道,参数 = K²·C_in。
2. **Pointwise convolution(逐点通道卷积)**:就是 1×1 卷积,负责跨通道融合并改变通道数。C_in → C_out,参数 = C_in·C_out。

标准卷积同时做了"空间聚合"和"通道融合"两件事;深度可分离把它们**解耦**成两步分别做。

**省多少参数(推导)**:

```
标准卷积参数:    P_std = K² · C_in · C_out
深度可分离参数:  P_dws = K² · C_in  +  C_in · C_out

比值:
  P_dws / P_std = (K²·C_in + C_in·C_out) / (K²·C_in·C_out)
                = 1/C_out + 1/K²
```

**验算**:K=3, C_in=64, C_out=128。
- 标准:9·64·128 = 73728
- 深度可分离:9·64 + 64·128 = 576 + 8192 = 8768
- 比值 = 8768/73728 ≈ 0.119,即省了约 **8.4 倍**。
- 公式预测:1/128 + 1/9 ≈ 0.0078 + 0.111 = 0.119,倒数 ≈ 8.41。**完全吻合**(我代码验证过)。

注意公式 1/C_out + 1/K²:当 C_out 很大时,第一项可忽略,**节省比约等于 1/K²**——3×3 省约 9 倍,5×5 省约 25 倍。这是 MobileNet(2017)/Xception(2017)的命门。FLOPs 节省比同理(因为 FLOPs 与参数成同样的结构)。

代价 / 坑:省参数不等于省时间。Depthwise 卷积的**算术强度低**(每读一个权重只算很少的乘加,内存访问占比高),在 GPU 上常常**带宽受限**,实际加速远不及理论 FLOPs 降幅,有时甚至更慢。这是"FLOPs 少 ≠ 跑得快"的经典案例,做部署时务必实测延迟而非只看 FLOPs。

### 6.4 转置卷积:学习式上采样

**问题**:分割 / 生成 / 超分要把小特征图放大回原图尺寸,且希望上采样方式是**可学习**的(而非固定的双线性插值)。

**机制**:转置卷积(transposed convolution,常被误称 deconvolution——它不是卷积的逆运算)。最干净的理解是:**它是普通卷积那个矩阵 M 的转置 Mᵀ 所定义的操作**。普通卷积 y = Mx 把大输入映射到小输出(M 是"高瘦"或"矮胖"取决于方向);转置卷积用 Mᵀ,把小输入映射回大输出,从而上采样。它的前向恰好对应普通卷积反向传播时梯度的计算方式,这也是名字的由来。

实现上等价于:在输入元素之间插零(由 stride 控制插几个),再做一次普通卷积。输出尺寸公式(转置卷积):

```
W_out = (W_in − 1)·S − 2P + K  (+ output_padding)
```

经典坑:**棋盘伪影(checkerboard artifacts)**。当 K 不能被 S 整除时,输出的不同位置被核覆盖的次数不均匀,生成图像出现规则的棋盘格纹理(Odena et al. 2016 在 distill.pub 详述)。实践中常用 **"上采样(双线性/最近邻)+ 普通卷积"** 替代转置卷积来规避,或保证 K 能被 S 整除(如 K=4, S=2)。

### 6.5 池化:作用与争议

**池化**(pooling)在一个窗口内做固定的(无参数)聚合:

- **Max pooling**:取窗口最大值。保留最强响应,带来局部平移不变性(目标在窗口内移动结果不变),对小位移鲁棒。
- **Average pooling**:取平均。更平滑,保留整体强度信息。

作用:下采样(减小空间尺寸 → 省后续计算、扩感受野)、引入局部平移不变。**没有可学习参数**。

争议:从 2015 年起(Springenberg et al. "Striving for Simplicity: The All Convolutional Net")就有人主张**用 stride=2 的卷积替代池化**——因为 stride 卷积同样下采样,但下采样方式是可学习的,信息损失可控,且更"对称"。现代很多架构(ResNet 用 stride 卷积下采样,但保留一个 maxpool;很多生成模型完全不用池化)体现了这个趋势。但 **global average pooling** 几乎成了分类头的标配(取代了笨重的全连接层,GAP 由 Network-in-Network 提出),因为它彻底的空间不变性 + 零参数极其好用。

记住权衡:**max pooling = 选择性 + 平移鲁棒但丢弃信息且不可学;stride 卷积 = 可学习的下采样但增加参数和计算。** 没有绝对优劣,看任务。

## 7. 从零实现 conv2d 并与 PyTorch 对齐

理论讲完,动手把 conv2d 实现出来才算真懂。下面是纯循环版(为了清晰,不做任何优化),并说明如何与 PyTorch 验证。

```python
import torch
import torch.nn.functional as F

def conv2d_naive(x, w, bias=None, stride=1, padding=0):
    """
    纯循环实现二维卷积(实为互相关,与 PyTorch 一致,不翻转核)。
    x:    (N, C_in, H, W)
    w:    (C_out, C_in, Kh, Kw)
    bias: (C_out,) 或 None
    返回: (N, C_out, H_out, W_out)
    """
    N, C_in, H, W = x.shape
    C_out, _, Kh, Kw = w.shape
    S, P = stride, padding

    # 1. zero padding(只在空间维 H, W 两侧补 P)
    xp = F.pad(x, (P, P, P, P))   # (left, right, top, bottom)

    # 2. 按输出尺寸公式算输出大小
    H_out = (H - Kh + 2 * P) // S + 1
    W_out = (W - Kw + 2 * P) // S + 1

    out = torch.zeros(N, C_out, H_out, W_out)

    # 3. 六重循环:逐样本、逐输出通道、逐输出位置、逐输入通道、逐核元素
    for n in range(N):
        for oc in range(C_out):
            for i in range(H_out):
                for j in range(W_out):
                    acc = bias[oc] if bias is not None else 0.0
                    # 这一块就是"3D 核与输入块的内积"
                    for ic in range(C_in):
                        for u in range(Kh):
                            for v in range(Kw):
                                # 注意 i*S + u:stride 决定窗口起点,u/v 是核内偏移
                                acc += xp[n, ic, i*S + u, j*S + v] * w[oc, ic, u, v]
                    out[n, oc, i, j] = acc
    return out


# ---- 与 PyTorch 对齐验证 ----
torch.manual_seed(0)
x = torch.randn(1, 3, 7, 7)
w = torch.randn(4, 3, 3, 3)
b = torch.randn(4)

ref = F.conv2d(x, w, b, stride=1, padding=1)   # PyTorch 官方实现
mine = conv2d_naive(x, w, b, stride=1, padding=1)

print((mine - ref).abs().max())   # 应为 ~1e-6(浮点误差),证明逻辑等价
```

我已经把这段逻辑跑通验证过:naive 实现与 `F.conv2d` 的最大绝对误差在浮点精度量级(~1e-6),证明上面所有公式和循环下标都是对的。

几个实现细节,初学者常错:

- **下标 `i*S + u`**:输出位置 i 对应的输入窗口**起点**是 i·S(stride 决定),u 是窗口内偏移。把这两者搞混是最常见 bug。
- **不翻转核**:`w[oc, ic, u, v]` 直接乘 `xp[..., i*S+u, j*S+v]`,正向对齐。这就是互相关(见第 1 节)。
- **bias 加在内积之外**,每个输出通道一个,与空间位置无关。
- 这个六重循环在真实尺寸上慢到无法用。生产实现靠 im2col + GEMM 或 Winograd / FFT 算法(大核时)把它变成一次大矩阵乘。理解 naive 版是为了懂原理;**永远别在生产里这么写**。

把它改成各种变体作为练习:dilated 只需把 `i*S+u` 改成 `i*S + u*d`;depthwise 把 `oc` 和 `ic` 绑定(每个输出通道只看对应的一个输入通道);1×1 直接去掉 u/v 循环。

## 8. 设计权衡与常见坑(汇总)

把散落各节的坑集中成一张清单,这是你做架构和调试时的检查表:

1. **互相关 ≠ 数学卷积**:`F.conv2d` 不翻转核。要真卷积自己翻转或用 scipy。绝大多数时候无所谓(核是学的),但做信号处理对接时会咬你。
2. **floor 会丢边界**:输出尺寸不整除时最后几行/列输入被丢弃。需要精确对齐时手动算 padding。
3. **参数量与 H,W 无关,FLOPs 与 H,W 成正比**:压缩参数看后期层(通道多),压缩计算看前期层(空间大)。砍错地方白费力。
4. **等变 ≠ 不变**:卷积等变,不变性来自池化和全局池化。分类靠 GAP 拿全局不变性。
5. **理论感受野 ≫ 有效感受野**:ERF 随 √L 增长而非 L,堆小卷积扩 ERF 效率随深度递减。要大上下文用 dilated / 大核 / attention,别只堆层。
6. **dilated 的网格效应**:连续相同膨胀率会漏采像素,用锯齿状互质膨胀率。
7. **FLOPs 少 ≠ 快**:depthwise 卷积算术强度低、带宽受限,GPU 上实际加速远低于 FLOPs 降幅。部署看实测延迟。
8. **转置卷积棋盘伪影**:K 不被 S 整除时出现,优先用"上采样 + 普通卷积"。
9. **im2col 吃显存**:中间张量膨胀 K² 倍,大核 + 大 batch 时是隐藏的 OOM 来源。

**练习 1(推导题 · 输出尺寸链)**
一个网络依次是:输入 224×224 → Conv(K=7,S=2,P=3) → MaxPool(K=3,S=2,P=1) → Conv(K=3,S=1,P=1) → Conv(K=3,S=2,P=1)。逐层算出每层后的空间尺寸。
*提示*:每层套 `⌊(W−K+2P)/S⌋+1`,前一层输出当后一层输入。第一层应得 112。(这是 ResNet 的 stem,自己核对答案是否落在 112→56→56→28。)

**练习 2(分析题 · 深度可分离省多少)**
一个 5×5 标准卷积,C_in=256,C_out=512。(a) 算它的参数量。(b) 换成深度可分离卷积的参数量。(c) 用公式 `1/C_out + 1/K²` 预测节省比,并与 (a)/(b) 实算对照。(d) 如果换成 3×3,节省比怎么变?定性解释为什么大核省得更多。
*提示*:标准 = K²·C_in·C_out;可分离 = K²·C_in + C_in·C_out。注意当 C_out 大时主导项是 1/K²。

**练习 3(推导题 · 感受野)**
五层 3×3 卷积,stride 序列为 (1, 2, 1, 2, 1)。用递推公式 `jump_l = jump_{l−1}·S_l`、`RF_l = RF_{l−1} + (K_l−1)·jump_{l−1}` 逐层算出最终感受野和最终 jump。
*提示*:初始 RF=1, jump=1。stride=2 会让后续层的 RF 增量翻倍——观察 jump 如何放大每层贡献。验算:第二层后 RF 应为 5、jump 应为 2。

**练习 4(编码题 · 改造 conv2d)**
基于第 7 节的 `conv2d_naive`,实现一个 `dilated_conv2d_naive(x, w, dilation, ...)`。然后用 `torch.nn.functional.conv2d(..., dilation=d)` 验证你的实现(最大误差应在 1e-6)。再验证有效核大小公式:用 d=2 的 3×3 核处理一个 7×7 输入(P=2 保持尺寸),确认其感受野行为等价于一个 5×5 核。
*提示*:只需把内层下标从 `i*S+u` 改成 `i*S + u*dilation`;padding 要相应放大(P 至少要 d·(K−1)/2 才能 same)。

## 11. 小结与承上启下

这一章把卷积这个 CV 原子操作从定义推到了能复现的程度:

- **是什么**:深度学习的"卷积"实为互相关(不翻转核),多通道下权重形状是 (C_out, C_in, K, K)。
- **三个数**:输出尺寸 `⌊(W−K+2P)/S⌋+1`、参数量 `K²·C_in·C_out`(与空间无关)、FLOPs `≈2·H_out·W_out·C_out·C_in·K²`(与空间成正比)。
- **为什么少参数**:卷积是带稀疏连接 + 权重共享约束的全连接特例,这两个约束编码了图像的局部性与平移规律先验。
- **为什么契合图像**:卷积平移等变(给出了完整证明,核心是权重不依赖位置),不变性由池化 + 全局池化补上。
- **看多广**:感受野逐层递推,但有效感受野只随 √L 增长,远小于理论值。
- **变体工具箱**:1×1(通道融合/降维)、空洞(扩 RF 不增参)、深度可分离(省约 1/K² 参数)、转置卷积(学习式上采样)、池化(下采样 + 局部不变,但正被 stride 卷积部分取代)。

**在整门课里的位置**:卷积是地基。下一章会把这些卷积层组装成完整的 CNN 架构(LeNet → AlexNet → VGG → ResNet → 现代 ConvNeXt),你会看到残差连接如何解决深层网络的梯度问题、BatchNorm 如何稳定训练——而你现在已经能从感受野和参数量的角度,定量分析每个架构为什么这么设计。再往后的检测、分割、生成模型,backbone 几乎都站在本章的卷积之上。把这一章的公式练到能脱口而出,后面读任何视觉论文你都能立刻在脑子里算出它每一层的尺寸、参数和感受野——这就是从"会用"到"读懂并能改进"的分水岭。

## 练习参考答案

### 练习 1:输出尺寸链(ResNet stem)

逐层套用尺寸公式 `W_out = ⌊(W − K + 2P)/S⌋ + 1`,把前一层输出当下一层输入。所有维度对称,只算一个方向即可。

第一层 Conv(K=7, S=2, P=3),输入 224:

```
W_out = ⌊(224 − 7 + 2·3) / 2⌋ + 1
      = ⌊(224 − 7 + 6) / 2⌋ + 1
      = ⌊223 / 2⌋ + 1
      = 111 + 1
      = 112
```

第二层 MaxPool(K=3, S=2, P=1),输入 112:

```
W_out = ⌊(112 − 3 + 2·1) / 2⌋ + 1
      = ⌊111 / 2⌋ + 1
      = 55 + 1
      = 56
```

第三层 Conv(K=3, S=1, P=1),输入 56(这是经典的 "same" 卷积:K=3, P=1, S=1 → 尺寸不变):

```
W_out = ⌊(56 − 3 + 2·1) / 1⌋ + 1
      = ⌊55⌋ + 1
      = 56
```

第四层 Conv(K=3, S=2, P=1),输入 56:

```
W_out = ⌊(56 − 3 + 2·1) / 2⌋ + 1
      = ⌊55 / 2⌋ + 1
      = 27 + 1
      = 28
```

**逐层空间尺寸:224 → 112 → 56 → 56 → 28。** 与提示给的 112→56→56→28 完全一致(已用代码验算)。

观察:三次下采样(S=2 的卷积 / 池化各一次,共三次)把 224 缩到 28,恰好缩小 8 倍(2³)。stride=1 的 same 卷积只换特征不改尺寸。

### 练习 2:深度可分离卷积省多少(5×5, C_in=256, C_out=512)

记 K=5, C_in=256, C_out=512。

**(a) 标准卷积参数量** `P_std = K²·C_in·C_out`:

```
P_std = 5² · 256 · 512
      = 25 · 256 · 512
      = 25 · 131072
      = 3,276,800
```

**(b) 深度可分离参数量** `P_dws = K²·C_in + C_in·C_out`(depthwise 项 + pointwise 项):

```
depthwise:  K²·C_in   = 25 · 256        = 6,400
pointwise:  C_in·C_out= 256 · 512        = 131,072
P_dws = 6,400 + 131,072 = 137,472
```

**(c) 公式预测节省比 vs 实算比**

公式 `1/C_out + 1/K²`:

```
1/C_out + 1/K² = 1/512 + 1/25
               = 0.001953... + 0.04
               = 0.041953...
```

实算 (b)/(a):

```
P_dws / P_std = 137,472 / 3,276,800 = 0.041953...
```

**两者完全吻合,比值 ≈ 0.0420,即深度可分离只用约 4.2% 的参数,省了约 1/0.0420 ≈ 23.8 倍**(已用代码验算)。

注意为什么公式成立:把 `(K²·C_in + C_in·C_out)/(K²·C_in·C_out)` 拆成两项,分子 K²·C_in 除以分母得 `1/C_out`,分子 C_in·C_out 除以分母得 `1/K²`,与 C_in 无关。

**(d) 换成 3×3 节省比怎么变**

K=3 时重算:

```
P_std = 9·256·512 = 1,179,648
P_dws = 9·256 + 256·512 = 2,304 + 131,072 = 133,376
比值  = 133,376 / 1,179,648 ≈ 0.1131
公式  = 1/512 + 1/9 ≈ 0.001953 + 0.1111 = 0.1131   ✓
倒数  ≈ 8.84 倍
```

**所以从 5×5 换到 3×3,节省比从约 23.8 倍降到约 8.8 倍——核越小,省得越少。**

定性解释:节省比公式是 `1/C_out + 1/K²`。当 C_out 较大(这里 512)时,第一项 `1/C_out` 很小可忽略,主导项是 `1/K²`,于是**节省倍数 ≈ K²**。核越大,K² 越大,标准卷积的空间项 K² 越占比重,而深度可分离把"空间聚合"剥离成只花 K²·C_in 的廉价 depthwise,所以大核被解耦后省下的绝对量更多——5×5 省约 25 倍量级,3×3 只省约 9 倍量级。换句话说,**深度可分离的红利主要来自砍掉标准卷积里 K² 与 C_out 的乘积耦合,核越大这层耦合越贵,解耦收益越大。**

### 练习 3:感受野递推(五层 3×3,stride 序列 1,2,1,2,1)

初始 `RF₀ = 1, jump₀ = 1`。每层 K=3,递推:

```
jump_l = jump_{l−1} · S_l
RF_l   = RF_{l−1} + (K_l − 1) · jump_{l−1} = RF_{l−1} + 2·jump_{l−1}
```

逐层计算(注意 RF 用的是**更新前**的 jump_{l−1},要先算 RF 再更新 jump):

```
层1 (S=1):  RF = 1 + 2·1 = 3      jump = 1·1 = 1
层2 (S=2):  RF = 3 + 2·1 = 5      jump = 1·2 = 2
层3 (S=1):  RF = 5 + 2·2 = 9      jump = 2·1 = 2
层4 (S=2):  RF = 9 + 2·2 = 13     jump = 2·2 = 4
层5 (S=1):  RF = 13 + 2·4 = 21    jump = 4·1 = 4
```

**最终感受野 RF = 21,最终 jump = 4。** 第二层后 RF=5、jump=2,与提示一致(全部已用代码验算)。

观察 stride 如何放大每层贡献:每层 RF 增量是 `2·jump_{l−1}`。前两层 jump 还是 1,增量只有 2;经过两次 stride=2 后 jump 累乘到 4,最后一层一次就贡献 +8。jump = 历史 stride 连乘(1·2·1·2·1 = 4),它像杠杆一样把后续每层的固定 (K−1)=2 放大。这正是"深层网络感受野增长特别快"的来源——不是核变大,而是 jump 连乘在放大每层的固定增量。

### 练习 4:改造 conv2d 为 dilated_conv2d_naive

核心改动只有一处:把内层取数下标 `i*S + u` 改成 `i*S + u*dilation`(列同理),让核元素在输入上按间隔 d 采样。输出尺寸用有效核 `K_eff = K + (K−1)·(d−1)` 代入尺寸公式。下面给可运行的 PyTorch 参考实现并与 `F.conv2d(..., dilation=d)` 对齐。

```python
import torch
import torch.nn.functional as F

def dilated_conv2d_naive(x, w, bias=None, stride=1, padding=0, dilation=1):
    """
    纯循环空洞卷积(互相关,不翻转核)。
    x:    (N, C_in, H, W)
    w:    (C_out, C_in, Kh, Kw)
    bias: (C_out,) 或 None
    返回: (N, C_out, H_out, W_out)
    """
    N, C_in, H, W = x.shape
    C_out, _, Kh, Kw = w.shape
    S, P, d = stride, padding, dilation

    # 1. zero padding(空间维两侧补 P)
    xp = F.pad(x, (P, P, P, P))   # (left, right, top, bottom)

    # 2. 有效核大小:核元素间插了 d-1 个空洞
    Kh_eff = Kh + (Kh - 1) * (d - 1)
    Kw_eff = Kw + (Kw - 1) * (d - 1)

    # 3. 输出尺寸用 K_eff 代入标准公式
    H_out = (H - Kh_eff + 2 * P) // S + 1
    W_out = (W - Kw_eff + 2 * P) // S + 1

    out = torch.zeros(N, C_out, H_out, W_out)

    for n in range(N):
        for oc in range(C_out):
            for i in range(H_out):
                for j in range(W_out):
                    acc = bias[oc] if bias is not None else 0.0
                    for ic in range(C_in):
                        for u in range(Kh):
                            for v in range(Kw):
                                # 唯一改动:u*d / v*d —— 核元素按 dilation 间隔采样
                                acc += xp[n, ic, i*S + u*d, j*S + v*d] * w[oc, ic, u, v]
                    out[n, oc, i, j] = acc
    return out


# ---- 与 PyTorch 对齐验证 ----
torch.manual_seed(0)
x = torch.randn(1, 3, 7, 7)
w = torch.randn(4, 3, 3, 3)
b = torch.randn(4)

# d=2, P=2 -> same 尺寸(7x7),因为 same 要求 P = d*(K-1)/2 = 2*2/2 = 2
ref  = F.conv2d(x, w, b, stride=1, padding=2, dilation=2)
mine = dilated_conv2d_naive(x, w, b, stride=1, padding=2, dilation=2)

print("output shape:", mine.shape)            # torch.Size([1, 4, 7, 7])
print("max abs diff:", (mine - ref).abs().max())   # ~1e-6,证明逻辑等价
```

依赖说明:只需 `pip install torch`,CPU 即可运行(维度极小,无需 GPU)。

**两点验证结果:**

1. 与 `F.conv2d(..., dilation=2)` 的最大绝对误差在浮点精度量级(~1e-6),证明 `i*S + u*d` 下标和尺寸公式正确。

2. 有效核大小公式 `K_eff = K + (K−1)·(d−1) = 3 + 2·1 = 5`,即 d=2 的 3×3 核感受野等价于 5×5。可这样独立确认(本环境无 torch,用 numpy 验过等价,max diff = 0.0):把 3×3 权重摆到一个 5×5 全零核的偶数位置(`w5[:,:,0::2,0::2] = w`),再用普通卷积(P=2)跑,输出与上面的 dilated 实现逐元素相等。这直接说明 d=2 的 3×3 与"在 5×5 栅格上只有 9 个非零、其余为零"的卷积完全等价——感受野是 5×5,但参数仍只有 9 个。

**padding 的注意点(提示要点):** 要保持 same 尺寸,padding 必须按有效核放大,`P = d·(K−1)/2`。3×3 普通卷积 same 用 P=1;d=2 时 K_eff=5,需要 P=2;d=4 时 K_eff=9,需要 P=4。若仍用 P=1,7×7 输入会缩成 `(7−5+2)/1+1 = 5`,尺寸不再保持。

附:其余两个变体的最小改法(同源于第 7 节朴素实现)——

- **1×1 卷积**:直接去掉 `u/v` 循环(Kh=Kw=1),下标退化为 `xp[n, ic, i*S, j*S]`,只剩跨通道线性组合。
- **depthwise 卷积**:把 `oc` 与 `ic` 绑定(每个输出通道只看对应的那一个输入通道),即去掉 `ic` 循环、令 `ic = oc`,权重形状变为 (C_in, 1, Kh, Kw)。
