## 一句话引言

前面所有章节都在做"2D 图像 → 标签 / 另一张 2D 图"的事。这一章第一次把**三维**摆上桌:给定一组**带相机位姿的 2D 照片**,我们要重建出一个**能从任意新视角渲染出照片级图像的三维场景表示**。核心洞察只有一个——**把"渲染"做成一个可微分的函数,然后用真实照片当监督,把场景表示当参数去优化它**。NeRF 把场景表示成一个连续函数(一个 MLP),3DGS 把它表示成一堆显式的高斯椭球;两条路的全部分歧,都源于"用什么数据结构装这个三维场景"这一个选择。本章从体渲染的物理方程推起,把这两条路讲到你能复现的程度。

## 1. 预备:从多视图到三维,任务到底是什么

### 1.1 相机模型:三维点怎么落到像素上

要从 2D 反推 3D,先得说清 3D 怎么变成 2D。**针孔相机模型(pinhole camera)**把这件事压缩成两步线性映射。

第一步,世界坐标系下的一个点 `X_world = (X, Y, Z)` 经过**外参(extrinsics)**——一个旋转 R(3×3 正交阵)加一个平移 t(3 维向量)——变到相机坐标系:

```
X_cam = R · X_world + t
```

外参编码的就是**相机在世界里的位姿(pose)**:它在哪、朝哪看。第二步,相机坐标系下的点经过**内参(intrinsics)**矩阵 K 投影到像素平面:

```
       ⎡ fₓ  0  cₓ ⎤
   K = ⎢ 0  f_y c_y ⎥        ⎡u⎤            ⎡X_cam⎤
       ⎣ 0   0   1 ⎦      s· ⎢v⎥ = K ·      ⎢Y_cam⎥
                             ⎣1⎦            ⎣Z_cam⎦
```

其中 fₓ, f_y 是焦距(以像素为单位),(cₓ, c_y) 是主点(通常接近图像中心),s 是齐次坐标的尺度因子(等于 Z_cam,做透视除法用)。**关键事实:这个投影是多对一的——一条从相机光心出发、穿过像素 (u,v) 的射线上的所有 3D 点,都投影到同一个像素。** 深度信息在投影里丢掉了。

### 1.2 反问题:从多张图恢复三维

单张图无法恢复深度(刚说的多对一)。但**多张不同视角的图**可以:同一个 3D 点在不同视角下投影到不同像素,这些投影约束联立起来就能三角化(triangulate)出它的 3D 位置——这是经典多视图几何(multi-view geometry)的核心,**SfM(Structure-from-Motion)**正是干这个的:输入一堆无序照片,输出每张照片的相机位姿 (R, t, K) 加一团稀疏 3D 点。**COLMAP** 是事实标准的 SfM 工具。

本章把任务设定钉死:

- **输入**:N 张图像 + 每张图的相机内外参 (Kᵢ, Rᵢ, tᵢ)。位姿通常由 COLMAP 预先算好,**本章不重建位姿,假定位姿已知**。
- **输出**:一个三维场景表示,使得对任意新位姿都能渲染出该视角下的图像。
- **监督**:就是这 N 张输入图本身——渲染出的图要和真图一致。

这是一个**无显式三维标注**的问题:我们从没拿到过场景的真实几何(没有点云真值、没有 mesh),全靠"渲染要像照片"这个 2D 约束反推 3D。**能做到这一点的前提,是渲染过程对场景参数可微**——这就是为什么下一节要先把渲染写成一个可微方程。

> 直觉:传统图形学是"已知三维场景 → 渲染出图"(正问题,可微性无所谓);神经渲染是"已知图 → 反求三维场景"(反问题),它要求渲染管线从头到尾可微,这样才能用梯度下降把"渲染误差"灌回场景表示里。**整个领域的技术核心,就是设计可微的渲染管线。**

## 2. 体渲染:把"沿光线积分颜色"写成可微方程

### 2.1 为什么是体渲染,不是表面渲染

经典图形学渲染**表面**(surface):光线打到第一个不透明表面就停,取那点的颜色。但"光线撞到的第一个表面在哪"这个操作是**不可微的**——它是一个 argmax / 硬阈值,对几何参数求导处处为零或不存在。

体渲染(volume rendering)换了个物理模型:**不假设有硬表面,而是把空间看成一团有密度的"雾"**,光线穿过时被连续地吸收和散射。空间每一点有一个**体密度(volume density)σ ≥ 0**(可理解为"光线在该点被一个微粒拦截/终止的概率密度")和一个**颜色 c**。这个模型天然是连续、可微的——没有任何硬判断,只有积分。这正是 NeRF 选它的根本原因。

### 2.2 连续体渲染方程的推导

考虑一条从相机光心 o 沿方向 d 射出的光线 `r(t) = o + t·d`,t 从近界 t_n 到远界 t_f。我们要算这条光线最终汇聚到像素上的颜色 C(r)。

先定义**透射率(transmittance)T(t)**:光线从 t_n 走到 t 这一段,**没有被任何东西拦截、顺利存活下来的概率**。在 [t, t+dt] 这一小段里,光线被拦截的概率是 σ(t)·dt(密度乘步长)。存活概率每走一步乘一个 (1 − σ·dt),取连续极限得到一个微分方程:

```
[T 满足的微分方程:存活率的衰减速度正比于当前密度]
  dT/dt = −σ(t)·T(t)

[解这个一阶线性 ODE(分离变量)]
  dT/T = −σ(t) dt
  ln T(t) − ln T(t_n) = −∫ₜₙᵗ σ(s) ds      (T(t_n)=1,起点必存活)
  ⟹  T(t) = exp( −∫ₜₙᵗ σ(s) ds )            ★ 透射率
```

T(t) 是单调递减的:走得越深、穿过的密度越多,存活率越低。

现在算颜色。光线在 [t, t+dt] 处**首次被拦截**(即恰好走到这里才终止)的概率是:"走到 t 还活着" × "在这一步被拦截" = `T(t)·σ(t)·dt`。一旦在 t 处终止,光线就取走该点的颜色 c(r(t), d)(注意颜色依赖于位置**和视角方向 d**——同一点从不同角度看可以不同颜色,这是镜面高光的来源)。把所有可能的终止位置按概率加权积分:

```
[连续体渲染方程]
  C(r) = ∫ₜₙ^{t_f}  T(t) · σ(t) · c(r(t), d)  dt ,      T(t)=exp(−∫ₜₙᵗ σ ds)     ★★
```

这就是 NeRF 论文里那个核心方程。**它把"一条光线的颜色"写成了关于 σ 和 c 的可微泛函**——只要 σ、c 对参数可微,C(r) 就对参数可微。这是整个 NeRF 能用梯度下降训练的地基。

读这个方程抓三点:权重 `w(t) = T(t)·σ(t)` 是一个**沿光线的概率密度**(它在哪里大,光线就"看到"哪里);σ 决定几何(物体在哪),c 决定外观(看起来什么颜色);T 保证了**遮挡(occlusion)**——被前面密度挡住的后方区域 T 已经很小,贡献自动被压低。

### 2.3 离散化:从积分到 alpha 合成(本章关键推导)

计算机算不了连续积分,得沿光线采 N 个点 t₁ < t₂ < … < t_N 做数值积分。设相邻采样点间隔 δᵢ = tᵢ₊₁ − tᵢ。**核心近似:在每个小区间 [tᵢ, tᵢ₊₁] 内假设 σ 和 c 是常数(分别为 σᵢ, cᵢ)**(分段常数近似)。下面把 ★★ 严格离散出来,这一步很多人只会背结论,我们推。

先看一个小区间 [tᵢ, tᵢ₊₁] 内,透射率怎么衰减。区间内 σ≡σᵢ 是常数:

```
[区间内透射率从 T(tᵢ) 衰减到 T(tᵢ₊₁)]
  T(tᵢ₊₁) = T(tᵢ)·exp(−∫_{tᵢ}^{tᵢ₊₁} σᵢ ds) = T(tᵢ)·exp(−σᵢ·δᵢ)

[记区间起点累计透射率 Tᵢ ≜ T(tᵢ),它等于前面所有区间存活率连乘]
  Tᵢ = exp(−Σⱼ₌₁^{i−1} σⱼ·δⱼ) = ∏ⱼ₌₁^{i−1} exp(−σⱼδⱼ)
```

再算第 i 个区间对总颜色的贡献。该区间内 σ≡σᵢ, c≡cᵢ,代入 ★★ 的被积式并在 [tᵢ, tᵢ₊₁] 上积分:

```
[第 i 段贡献 = ∫ T(t)σᵢ cᵢ dt ,区间内 T(t)=Tᵢ·exp(−σᵢ(t−tᵢ))]
  贡献ᵢ = σᵢ cᵢ ∫_{tᵢ}^{tᵢ₊₁} Tᵢ·exp(−σᵢ(t−tᵢ)) dt
        = Tᵢ σᵢ cᵢ · [ −(1/σᵢ)·exp(−σᵢ(t−tᵢ)) ]_{tᵢ}^{tᵢ₊₁}
        = Tᵢ σᵢ cᵢ · (1/σᵢ)·( 1 − exp(−σᵢδᵢ) )
        = Tᵢ · ( 1 − exp(−σᵢ·δᵢ) ) · cᵢ
```

注意 σᵢ 在积分里被精确消掉了——这不是粗暴近似,是分段常数假设下的**精确积分**。定义第 i 个采样点的**不透明度(alpha)**:

```
  αᵢ ≜ 1 − exp(−σᵢ·δᵢ)        ★ alpha:这一小段把光"吃掉"的比例,∈[0,1]
```

把所有区间的贡献加起来,得到 NeRF 真正用来跑的**离散 alpha 合成(alpha compositing)公式**:

```
[离散体渲染 = 沿光线的 alpha 合成]
  Ĉ(r) = Σᵢ₌₁^N  Tᵢ · αᵢ · cᵢ ,     其中  Tᵢ = ∏ⱼ₌₁^{i−1} (1 − αⱼ)        ★★★
```

最后这个 Tᵢ 的连乘形式也值得验算:`Tᵢ = exp(−Σⱼ<ᵢ σⱼδⱼ) = ∏ⱼ<ᵢ exp(−σⱼδⱼ) = ∏ⱼ<ᵢ (1−αⱼ)`,和 ★ 的定义一致。**这个公式和图形学里 from-back-to-front 的 over 合成、和半透明图层叠加,是同一个东西**——NeRF 的渲染本质就是"把一串带不透明度的彩色采样点从前往后叠起来"。

三个性质:αᵢ∈[0,1] 是天然的、σᵢ→∞ 时 αᵢ→1(完全不透明,像硬表面),σᵢ→0 时 αᵢ→0(透明,空气);权重 wᵢ = Tᵢαᵢ 满足 Σwᵢ ≤ 1(等号当光线被完全吸收时成立),它就是连续权重 T(t)σ(t) 的离散版;**整个 ★★★ 对 σᵢ 和 cᵢ 处处可微**——这才是能反传梯度的关键。

### 2.4 代码:体渲染 alpha 合成

```python
import torch

def volume_render(sigma, color, t_vals, dirs):
    """
    把网络在采样点上的输出合成成像素颜色。这是 NeRF 渲染的心脏。
    sigma : (R, N)      每条光线 N 个采样点的体密度 σ ≥ 0
    color : (R, N, 3)   每个采样点的 RGB ∈ [0,1]
    t_vals: (R, N)      沿光线的采样深度 t(已排序)
    dirs  : (R, 3)      每条光线的方向(未归一化,模长用于把 δ 换算成真实距离)
    返回   : rgb (R,3), weights (R,N), depth (R,)
    """
    # 1) 相邻采样点的间隔 δ。最后一段补一个大值(到无穷远)
    deltas = t_vals[:, 1:] - t_vals[:, :-1]                 # (R, N-1)
    inf = torch.full_like(deltas[:, :1], 1e10)
    deltas = torch.cat([deltas, inf], dim=-1)               # (R, N)
    # t 是沿单位方向的参数,乘 ||d|| 还原成世界空间真实距离(★ 易错点)
    deltas = deltas * torch.norm(dirs, dim=-1, keepdim=True)

    # 2) alpha = 1 - exp(-σ·δ)   ← 公式 ★
    alpha = 1.0 - torch.exp(-torch.relu(sigma) * deltas)    # σ 过 relu 保证非负

    # 3) 透射率 Tᵢ = ∏_{j<i}(1-αⱼ)  ← 用 cumprod 实现,且要 exclusive(不含当前点)
    ones = torch.ones_like(alpha[:, :1])
    trans = torch.cumprod(torch.cat([ones, 1.0 - alpha + 1e-10], dim=-1), dim=-1)
    trans = trans[:, :-1]                                   # 去掉最后多出的一项 → exclusive

    # 4) 权重 wᵢ = Tᵢ·αᵢ,再加权求和  ← 公式 ★★★
    weights = trans * alpha                                 # (R, N)
    rgb = torch.sum(weights[..., None] * color, dim=-2)     # (R, 3)
    depth = torch.sum(weights * t_vals, dim=-1)             # 期望深度,可视化几何用
    return rgb, weights, depth
```

`cumprod` 那行的 `exclusive`(把 1 拼到最前、再砍掉最后一项)是头号 off-by-one 坑:Tᵢ 必须是**前 i−1 个点**的存活率连乘,**不含当前点**;写成 inclusive 会让每个点被自己的 alpha 多衰减一次,渲出来整体偏暗。

## 3. NeRF:用一个 MLP 当作连续场景函数

### 3.1 把场景表示成一个函数

NeRF(Neural Radiance Fields,神经辐射场,Mildenhall et al., 2020)的全部赌注押在一句话上:**整个三维场景就是一个连续函数,用一个 MLP 拟合它**:

```
  F_θ : (x, d) ↦ (c, σ)
       输入  位置 x=(x,y,z) ∈ ℝ³  +  视角方向 d ∈ S²⊂ℝ³(3 维笛卡尔单位向量,常由 (θ,φ) 转来)
       输出  颜色 c=(r,g,b)        +  体密度 σ ∈ ℝ≥0
```

这就是"5D 输入(3 位置 + 2 方向)→ 4D 输出(3 颜色 + 1 密度)"的含义。注意两个刻意的设计:

- **σ 只依赖位置 x,不依赖方向 d**。物理上"某点有多实"是几何属性,不该随你从哪看而变。这个约束强制网络学到**多视角一致(multi-view consistent)的几何**——从任何角度看,物体都在同一个地方。这是 NeRF 几何质量的关键归纳偏置。
- **c 依赖位置 x 和方向 d**。同一点从不同角度颜色可以不同,这样才能表达镜面反射、高光。

为落实"σ 不看方向",NeRF 的 MLP 结构是分段的:**先用一个深 MLP 从 x 算出密度 σ 和一个中间特征,再把该特征和方向 d 拼起来过一个浅头算 c**。

### 3.2 训练:渲染图与真图的像素 L2 损失

有了 F_θ 和 §2 的可微渲染,训练就是一个端到端回归:

1. 从训练图里随机抽一批像素,每个像素按相机参数反算出一条光线 r。
2. 沿每条光线采 N 个点,送 F_θ 得 {σᵢ, cᵢ},用 ★★★ 渲染出 Ĉ(r)。
3. 与该像素的真实颜色 C_gt(r) 算 L2:

```
  L = Σ_{r ∈ batch}  ‖ Ĉ(r) − C_gt(r) ‖²₂              ★ NeRF 的全部损失
```

就这么朴素——**没有任何三维标注,只有"渲染像不像照片"**。梯度从这个 L2 经 ★★★ 的可微合成、再经 F_θ,一路灌回 MLP 的权重 θ。一个场景训练到收敛,θ 里就"长"出了这个场景的完整几何与外观。**每个场景一套独立的 θ,NeRF 不跨场景泛化**(原版如此)。

### 3.3 关键推导:位置编码——为什么裸 MLP 学不了高频

直接拿原始坐标 (x,y,z,θ,φ) 喂 MLP,会得到一张**糊成一团**的重建——高频细节(边缘、纹理、清晰轮廓)全丢了。这不是调参问题,是**深层机制**问题,必须讲透。

**现象**:用 ReLU MLP 直接拟合 `f(x)`(x 是低维坐标),网络强烈偏向输出**低频、过度平滑**的函数。

**原因(谱偏差,spectral bias)**:神经网络的训练存在 **spectral bias / frequency principle**(Rahaman et al., 2019;Xu et al., 2019)——**梯度下降优先拟合低频成分,高频成分收敛极慢**。可以从神经正切核(NTK)视角理解:用 NTK 近似,网络在函数空间的学习动态由核 k(x, x') 主导,而对于标准 MLP 配以低维输入,这个核近似是一个**平稳的、随距离单调衰减的核**(类似一个宽的高斯/拉普拉斯核)。这种核的特征谱里**高频特征对应的特征值极小**,而梯度下降下每个特征模式的收敛速度正比于其特征值——于是高频模式收敛慢到训练时长内基本学不出来。直观说:**标准 MLP 自带一个"低通滤波器",越细的细节被压得越狠。**

**解法:位置编码(positional encoding)γ(·)**。把每个标量坐标 p 用一组不同频率的正余弦映射到高维:

```
  γ(p) = ( sin(2⁰πp), cos(2⁰πp), sin(2¹πp), cos(2¹πp), …, sin(2^{L−1}πp), cos(2^{L−1}πp) )
```

对位置 x 的三个分量各做(NeRF 原版 L=10),对方向 d 的分量各做(L=4)。纯编码维度从 3 膨胀到 3·2·10 = 60(位置)、从 3 到 3·2·4 = 24(方向);若再拼上原始坐标(下面代码 include_input=True 默认即如此,NeRF 原版亦然),则位置为 60+3=63、方向为 24+3=27。

**为什么这能解决问题——两个层次的论证**:

(1) **改变了核的频谱**。把输入经过 Fourier 特征映射 γ 后再喂 MLP,等价于把上面那个"随距离单调衰减、高频被压死"的平稳核,**换成一个由 γ 的频率集合决定的核**(Tancik et al., 2020《Fourier Features》给出:复合核 ≈ Σ 各频率分量,频谱被 γ 显式拓宽到包含高频)。频谱里有了高频分量,梯度下降就能在合理时间内学出高频细节。**位置编码本质是给 NTK 的频谱"注入高频带宽"**。

(2) **拉开了相邻输入的距离**。两个空间上很近的点 p, p+ε,在原始坐标里几乎一样(MLP 难以给它们不同输出,这正是"过度平滑"的来源)。但经过高频项 sin(2^{L−1}πp) 后,即便 ε 很小,只要 2^{L−1}π·ε 不可忽略,这两点的编码就**显著分开**——MLP 于是有能力给空间上极近的两点赋予截然不同的颜色/密度,这就是"能表达锐利边缘"的微观机制。

**频率上限是个权衡旋钮**:L 太小 → 仍然糊;L 太大 → 编码里出现比像素采样还密的高频,网络会去拟合噪声、出现"褶皱"伪影(aliasing)。这也是后续 Mip-NeRF 改进的切入点(用积分位置编码按采样体积自适应地衰减过高频率)。

```python
import torch

class PositionalEncoding(torch.nn.Module):
    """γ(p):把低维坐标映射到高频傅里叶特征,解开 MLP 的谱偏差。"""
    def __init__(self, num_freqs=10, include_input=True):
        super().__init__()
        self.include_input = include_input
        # 频率 2^0π … 2^{L-1}π
        self.freqs = 2.0 ** torch.arange(num_freqs) * torch.pi  # (L,)

    def forward(self, x):                     # x: (..., D), D=3
        out = [x] if self.include_input else []
        for f in self.freqs:
            out.append(torch.sin(f * x))      # (..., D)
            out.append(torch.cos(f * x))      # (..., D)
        return torch.cat(out, dim=-1)         # (..., D*2L (+D));include_input=True 时多出 +D 的原始坐标
```

### 3.4 分层采样:别在空气里浪费算力

★★★ 的积分质量取决于采样点是否落在"有东西"的地方。但一条光线绝大部分是空气(σ≈0),均匀采样会把算力浪费在虚空里,真正的表面附近反而采样不足。NeRF 用**分层采样(hierarchical sampling)**——同时训练两个 MLP:

1. **粗网络(coarse)**:沿光线**均匀分层**采 N_c 个点(把 [t_n, t_f] 分成 N_c 段,每段内随机抽一个,stratified sampling,保证既覆盖全程又有随机性防过拟合栅格),渲染得到一组权重 wᵢ = Tᵢαᵢ。
2. **重要性重采样**:把归一化后的权重 ŵᵢ = wᵢ/Σwⱼ 当成一个沿光线的**分段常数概率密度(PDF)**,用逆变换采样(inverse-transform sampling)从这个 PDF 里再抽 N_f 个点——**权重大的地方(表面附近)抽得密**。
3. **细网络(fine)**:用 N_c + N_f 个点合并渲染,得到最终颜色。

两个网络各自和真值算 L2,损失相加。**机制上,粗网络的权重在告诉细网络"表面大概在哪",细网络把采样预算集中投到那里**——这是把固定采样预算的效率提升数倍的标准做法。原版 N_c=64, N_f=128(「待核:具体数值随实现而异」)。

```python
def sample_pdf(bins, weights, n_samples):
    """逆变换采样:从粗网络权重定义的 PDF 里按重要性抽样。bins:(R,N_c) 采样深度边界。"""
    weights = weights + 1e-5                                  # 防全零光线除零
    pdf = weights / weights.sum(-1, keepdim=True)             # 归一化成 PDF
    cdf = torch.cumsum(pdf, dim=-1)                           # 累积分布 CDF
    cdf = torch.cat([torch.zeros_like(cdf[:, :1]), cdf], -1)  # 前补 0
    u = torch.rand(*cdf.shape[:-1], n_samples, device=cdf.device)  # 均匀采样
    # 对每个 u 在 CDF 上二分查找它落在哪个 bin,再线性插值出对应深度 → 权重大处抽得密
    idx = torch.searchsorted(cdf, u, right=True)
    # 注意 idx 最大可达 N_c+1(cdf 前补 0 后长度 N_c+1),照抄当 bin 索引会越界,需 clamp(idx-1, 0, N_c-1)
    # (插值还原 t 的细节略,核心是 CDF^{-1}(u))
    return idx
```

### 3.5 数值分析:NeRF 的算力账

把 NeRF 慢的根因算清楚。一张 H×W 图像有 H·W 条光线,每条采 N=N_c+N_f 个点,每个点过一次 MLP。

- **单次前向的 MLP 调用数** ≈ H·W·N。取 800×800 图、N=192:`800·800·192 ≈ 1.23×10⁸` 次 MLP 前向,**渲染一张图就要上亿次网络调用**。这是 NeRF 渲染慢到分钟级的根本原因——表示是隐式的,任何查询都得跑一遍网络。
- **MLP 本身**:原版 8 层、每层 256 维全连接,单次前向约 `8 × (256×256) ≈ 5×10⁵` 次乘加(MACs)的量级。乘上上面的调用数,单图前向 FLOPs 在 10¹³~10¹⁴ 量级。
- **参数量**:整个场景就是这个 MLP——`~8层×256×256 ≈ 5×10⁵` 个参数,**约 1~5 MB(「待核」)**。这是 NeRF 的最大优点:**存储极省**,几 MB 装下整个三维场景。
- **训练时间**:原版单场景在单卡上**训练 1~2 天(「待核,随实现/分辨率而异」)**。慢在每次迭代都要对一批光线跑上面那套上亿次前向 + 反传。

一句话总结 NeRF 的画像:**表示连续、质量高、存储极省;但训练慢、渲染慢(都因为"隐式 = 任何查询都得跑网络")。** 这个"慢"正是 3DGS 要解决的痛点。

## 4. 3D Gaussian Splatting:把场景换成一堆显式高斯

### 4.1 换数据结构:从连续函数到离散基元

3D Gaussian Splatting(3DGS,Kerbl et al., 2023)不改变"用可微渲染 + 照片监督反求三维"这个大框架,**只换了一件事:用什么装场景**。NeRF 用一个 MLP(隐式、连续);3DGS 用**一大堆显式的各向异性三维高斯(anisotropic 3D Gaussians)**——可以想成空间里飘着几十万到上百万个**半透明的彩色椭球**,每个椭球第 k 个由四组参数描述:

```
  位置     μ_k ∈ ℝ³                  椭球中心在哪
  协方差   Σ_k ∈ ℝ^{3×3}(对称正定)   椭球的形状/朝向/大小(各向异性)
  不透明度 o_k ∈ [0,1]                这个椭球多实
  颜色     由球谐系数(SH)表达          随视角变化的 RGB(对应 NeRF 里 c 依赖 d)
```

单个三维高斯的空间分布:`G_k(x) = exp( −½ (x−μ_k)ᵀ Σ_k⁻¹ (x−μ_k) )`。**全部参数都是显式可优化的张量**,没有网络——这是和 NeRF 最根本的分野。

**协方差怎么参数化**(关键工程细节):Σ 必须**对称正定**,直接优化 6 个自由分量会在梯度更新中破坏正定性。3DGS 把 Σ 分解成"缩放 + 旋转":`Σ = R·S·Sᵀ·Rᵀ`,其中 S=diag(sₓ,s_y,s_z) 是缩放(优化 log 尺度保正),R 由单位四元数 q 给出的旋转矩阵。**优化四元数 q 和缩放 s,而非直接优化 Σ**,从而无论梯度怎么走,重构出的 Σ 都恒正定。这是一个"用合适的参数化保证约束"的范本。

### 4.2 可微光栅化:splatting 是怎么渲染的

NeRF 是 ray-marching(沿光线步进采样查询);3DGS 是 **rasterization / splatting**(把每个三维基元"拍扁"投到屏幕上叠起来),走的是 GPU 光栅化的老路,所以快。流程:

1. **投影(EWA splatting)**:把每个三维高斯投影到二维图像平面,得到一个**二维高斯("splat",一个椭圆斑点)**。三维协方差 Σ 经投影变换 + 透视近似(用投影的雅可比 J 做局部线性化)得到二维协方差 `Σ' = J·W·Σ·Wᵀ·Jᵀ`(W 是视角变换),这是 EWA(Elliptical Weighted Average)splatting 的标准做法。
2. **按深度排序**:对当前视角,把所有高斯按到相机的深度排序(这样才能正确做前后遮挡合成)。这是 3DGS 实时性的关键之一——一次排序后整图共享。
3. **逐像素 alpha 合成**:每个像素位置 p,把覆盖它的高斯按深度从前往后做 alpha 合成,**和 §2.3 完全同一个 ★★★ 公式**:

```
  C(p) = Σ_{k 按深度排序}  c_k · α_k · ∏_{j<k}(1 − α_j)
         其中  α_k = o_k · G'_k(p)
              G'_k(p) = exp(−½ (p−μ'_k)ᵀ Σ'_k⁻¹ (p−μ'_k))   ← 二维高斯在该像素的值
```

**注意这里 ★★★ 又出现了**——不透明度 α_k 现在是"高斯的不透明度 o_k 乘上二维高斯在该像素处的衰减值"。这个合成对所有高斯参数(μ, Σ, o, SH)**解析可微**,梯度能直接回传到每个高斯的张量上,**全程没有神经网络的前向**。

### 4.3 训练:自适应密度控制

3DGS 的训练循环:从 SfM 给的**稀疏点云初始化**一批高斯(每个点放一个小高斯)→ 渲染 → 和真图算损失(L1 + D-SSIM 的组合,不是纯 L2)→ 反传更新所有高斯参数。

但固定数量的高斯不够——初始点稀,有的区域欠拟合(高斯太大太糊)、有的过拟合(一个大高斯横跨细节区)。3DGS 的关键创新是**自适应密度控制(adaptive density control)**:周期性地

- **致密化(densify)**:对"视图空间位置梯度大"(说明此处误差大、欲动)的高斯:若它**小**(under-reconstruction)→ **克隆**一个;若它**大**(over-reconstruction,一个大高斯盖了细节)→ **分裂**成两个更小的。
- **剪枝(prune)**:删掉不透明度 o 低于阈值(几乎透明、无贡献)的高斯,以及尺寸异常大的。
- **不透明度周期重置**:定期把所有 o 压低再让优化重新"竞争",清除冗余高斯,控制总数。

**机制上,这是一个让基元数量随场景复杂度自适应增长/收缩的过程**——简单区域少几个大高斯,细节区域自动分裂出许多小高斯。最终高斯数量在几十万到几百万级。

### 4.4 数值分析:3DGS 为什么快、代价是什么

把 3DGS 的账和 NeRF 对账,这是本章要你建立的核心定量直觉。

- **渲染速度**:光栅化是 GPU 原生操作,**没有逐采样点的网络前向**;一次深度排序 + 高度并行的逐 tile alpha 合成,**渲染从 NeRF 的分钟级降到实时(≥30~100+ FPS,「待核,随场景/硬件」)**——快了**几个量级**。这是 3DGS 出圈的直接原因。
- **训练速度**:同样没有网络,前向/反传都是对显式参数的解析运算,**训练从 NeRF 的天级降到分钟~小时级(「待核」)**。
- **存储/显存——代价在这里**:设有 M 个高斯,每个约存:位置 3 + 缩放 3 + 旋转(四元数)4 + 不透明度 1 + SH 系数(3 阶约 48 个浮点)≈ **~59 个 float ≈ 236 字节/高斯(「待核,随 SH 阶数」)**。取 M=10⁶:`10⁶ × 236 B ≈ 236 MB` 一个场景。**相比 NeRF 的几 MB MLP,显式表示的存储大了约两个量级**——这是 3DGS 的核心代价,也是后续一大堆"压缩高斯"工作的动机(向量量化、剪枝、SH 蒸馏等)。
- **复杂度对比**:NeRF 渲染 ∝ (像素数 × 每光线采样数 × MLP前向);3DGS 渲染 ∝ (高斯数排序 M·logM + 像素覆盖的高斯数 alpha 合成)。后者常数项小得多且全程 GPU 并行。

### 4.5 隐式 MLP vs 显式高斯:根本取舍

这是本章最该带走的一张思维图。两条路的全部分歧,源于**用什么数据结构表示三维场景**:

| 维度 | NeRF(隐式连续场) | 3DGS(显式离散基元) |
|---|---|---|
| 场景表示 | 一个 MLP F_θ(连续函数) | 一堆三维高斯(离散参数集) |
| 渲染方式 | ray-marching,沿光线查询网络 | rasterization / splatting,投影叠加 |
| 查询代价 | 任意点都要跑一次网络(贵) | 直接读参数(便宜) |
| 训练速度 | 天级 | 分钟~小时级 |
| 渲染速度 | 分钟级(离线) | 实时(几个量级快) |
| 存储 | 几 MB(极省) | 几百 MB(代价) |
| 编辑性 | 难(几何隐藏在网络权重里) | 易(可直接移动/删除某个高斯) |
| 连续/分辨率 | 连续场,理论任意分辨率、外推平滑 | 离散基元,极端缩放/空洞处易露馅 |

一句话讲透这个取舍:**隐式表示把场景压进网络权重——连续、省存储、但"读"很贵(任何查询都要算一遍网络),所以慢;显式表示把场景摊成一堆基元——"读"极便宜(直接取参数,GPU 光栅化),所以快、可编辑,代价是占存储大、且离散基元在连续性/外推上不如连续场优雅。** 这不是谁淘汰谁,而是"算力 vs 存储 vs 连续性"的工程定位不同:要实时交互/可编辑选 3DGS,要极致省存储/连续表示选 NeRF 系。

## 5. 设计权衡与常见坑

- **位姿质量决定一切。** NeRF/3DGS 都假定相机位姿准。COLMAP 位姿带误差时,重建直接糊成双影。**这是新手最大、最隐蔽的坑——不是模型不行,是位姿不准**。调不出好结果先回去查 SfM 重建质量(重投影误差、配准图数)。
- **NeRF:位置编码频率 L 是糊/伪影的旋钮。** L 太小整体糊,太大出现高频褶皱伪影(aliasing)。**别盲目调大**;有抗锯齿需求看 Mip-NeRF 的积分位置编码。
- **NeRF:near/far 界没设好,采样全打偏。** t_n, t_f 框错(把场景框在外面或框太松)→ 采样点落不到表面 → 学不出几何。前景为主的场景还要处理无界背景(NeRF++ / Mip-NeRF 360 的反球面参数化)。
- **NeRF:体渲染 δ 没乘方向模长。** §2.4 代码里 `deltas *= ‖d‖` 这步漏了,等价于深度尺度错,几何被压扁。off-by-one(Tᵢ 用 inclusive cumprod)会整体偏暗——用"喂常数密度看能否渲出预期透明度"自测。
- **3DGS:协方差直接优化会丢正定性。** 必须走 `Σ=RSSᵀRᵀ`(优化四元数+log缩放),否则数值上 Σ⁻¹ 会爆。这是工程上的硬约束,不是可选项。
- **3DGS:无密度控制就废一半。** 不做 densify/prune,高斯要么过少(糊)要么爆炸增长(显存 OOM)。densify 的梯度阈值、prune 的不透明度阈值、不透明度重置周期是三个核心超参,调不好要么糊要么炸显存。
- **3DGS:深度排序是近似。** 全局按高斯中心深度排序,在高斯互相穿插时不是严格正确的可见性顺序,极端视角下会有 popping(闪烁/翻面)伪影。
- **3DGS 存储是真痛点。** 百万高斯几百 MB,移动端/大场景需要压缩。别以为它全面优于 NeRF——**用存储和一定的伪影换来了速度**。
- **两者都默认静态、稠密视角、固定光照。** 动态场景、稀疏视角(few-shot)、变光照都需专门变体(各有大量后续工作),原版方法直接上会失败。

## 6. 动手练习

1. **(推导题)从连续体渲染推出离散 alpha 合成。** 从 `C(r)=∫ T(t)σ(t)c(t)dt`、`T(t)=exp(−∫σds)` 出发,在区间 [tᵢ,tᵢ₊₁] 内假设 σ≡σᵢ、c≡cᵢ,严格积分出第 i 段贡献,推出 `αᵢ=1−exp(−σᵢδᵢ)` 和 `Ĉ=ΣTᵢαᵢcᵢ, Tᵢ=∏_{j<i}(1−αⱼ)`。**关键检查点**:第 i 段积分里 σᵢ 必须被精确消掉(不是近似);验证 `Tᵢ=exp(−Σ_{j<i}σⱼδⱼ)` 与 `∏(1−αⱼ)` 两种写法恒等。*提示*:积分 ∫exp(−σ(t−tᵢ))dt 给出 (1/σ)(1−exp(−σδ)),那个 1/σ 恰好和前面的 σ 约掉。

2. **(分析题)论证位置编码为何让 MLP 能表达高频。** 分两个层次写清楚:(a) **谱偏差**——为什么裸 ReLU MLP 配低维坐标输入会过度平滑(从"梯度下降优先收敛低频 / NTK 高频特征值小"角度论证);(b) **γ 的作用**——为什么 sin/cos 高频项能(i)拓宽等效核的频谱、(ii)拉开空间相邻两点的编码距离,从而让 MLP 有能力赋予近邻点不同输出。*提示*:核心参考是 Tancik et al.《Fourier Features Let Networks Learn High Frequency Functions》——它直接把 γ 解释成"构造一个含高频的平稳核"。再分析 L 过大为何反而出伪影。

3. **(对比/分析题)NeRF vs 3DGS 取舍量化。** 估算两者渲染一张 800×800 图的计算画像:NeRF 算"光线数×每光线采样数×MLP前向次数"的量级,3DGS 算"高斯排序 M·logM + 逐像素合成"的量级;再估两者单场景的存储(NeRF 的 MLP 参数 vs M 个高斯 ×每高斯字节数)。**结论要落到一句话**:为什么"隐式=慢但省存储"、"显式=快但占存储",根因都在"读一次表示要花多少代价"。*提示*:取 NeRF 8层×256 MLP、N=192;取 3DGS M=10⁶、每高斯 ~59 float。哪些数你拿不准就标「待核」别硬编。

4. **(编码题)实现并自测体渲染。** 照 §2.4 实现 `volume_render`,然后写两个单元测试验证机制:(a) **遮挡测试**——构造一条光线,前面放一个 σ→∞ 的红点、后面放一个绿点,渲染结果应≈纯红(后方被完全遮挡,绿色权重≈0);(b) **透明度测试**——所有点 σ=0,渲染结果应≈背景色(光线全程存活、不取任何前景色)。*提示*:这两个测试能立刻抓出 cumprod 的 exclusive/inclusive 写错(渲染偏暗)和 δ 乘错(几何尺度错)两个头号 bug。

## 8. 小结与承上启下

这一章我们第一次离开 2D,把"从带位姿的多视图照片重建可渲染三维场景"这个反问题讲透了。收束成一句话:**整个神经渲染建立在"让渲染可微、用照片当监督反求场景表示"这一个思想上;体渲染方程 `C=∫Tσc dt` 提供了可微的渲染算子,离散化后就是 alpha 合成 `Ĉ=ΣTᵢαᵢcᵢ`;NeRF 把场景表示成一个连续 MLP(靠位置编码 γ 破解谱偏差才学得到高频、靠分层采样把算力投到表面),省存储但慢;3DGS 换成一堆显式高斯 + GPU 可微光栅化(同一个 alpha 合成公式),用存储换来了实时速度和可编辑性。**

四根支柱串起来,你应当已经能复现:

- **任务设定与相机模型**(§1):反问题的前提是位姿已知、渲染可微;
- **可微体渲染**(§2):★★★ 是 NeRF 和 3DGS 共用的渲染心脏,从物理方程严格离散而来;
- **NeRF**(§3):MLP 当连续场,**位置编码解谱偏差**是它能成立的关键推导,分层采样是效率命门;
- **3DGS**(§4):显式高斯 + splatting,**"隐式 vs 显式"的根本取舍**(算力↔存储↔连续性)是本章最该带走的思维图。

放进整门课的坐标:前面我们学了卷积/ViT 提特征、检测分割做理解、GAN/扩散做二维生成——它们的世界都是平面的。这一章把维度抬到了三维,但仍是**针对单场景过拟合的重建**(每个场景重训一套)。下一步的开放问题是:能不能像 CLIP/扩散那样**跨场景泛化、前馈一次出三维**(generalizable / feed-forward 三维重建)?能不能把二维扩散的生成先验蒸馏进三维、做**文本到三维生成**(如 DreamFusion 的 SDS)?能不能重建**动态三维(4D,带时间)**?当你把本章的"可微渲染 + 场景表示"这把钥匙握牢,这些前沿方向读起来都只是在换"场景表示"和"监督来源"——内核不变。
