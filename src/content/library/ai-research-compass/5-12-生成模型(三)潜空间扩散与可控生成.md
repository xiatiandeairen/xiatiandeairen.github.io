> 这一章把你从"知道扩散模型能在像素上逐步去噪生成图"带到"能在白板上推出 classifier-free guidance 是条件/无条件分布的对数梯度外推、能精确算出潜空间扩散把扩散计算量降了几十倍、能讲清 cross-attention 把一句话注入 U-Net 的每一步去噪、并能照着实现 CFG 采样循环与跨注意力条件注入"。读完你对 Stable Diffusion 这一类系统不应有任何黑箱:它不是"魔法 prompt 引擎",而是**一个在 VAE 压出的低维潜空间里运行、用 cross-attention 接收文本条件、用 CFG 放大条件方向的标准条件扩散模型**,每一个工程选择背后都有可推导的理由。

## 一句话引言

上一章(扩散模型基础)你已经掌握:扩散模型用一个前向加噪过程 `q(xₜ|x₀)` 把数据逐步糊成高斯噪声,再训一个网络 εθ(xₜ, t) 去预测每一步加进去的噪声,采样时从纯噪声逐步去噪还原数据。它生成质量极高、训练极稳(没有 GAN 的对抗博弈),但有两个致命的工程问题:**(1) 在像素空间做扩散太贵**——512×512×3 的图,U-Net 要在 78 万维上反复前向几十上百步,显存和算力都顶不住;**(2) 基础扩散是无条件的**,只会"随机生成一张像训练集的图",你没法叫它"画一只戴帽子的柯基"。这一章就是回答这两个问题:**潜空间扩散(Latent Diffusion / Stable Diffusion)** 用一个 VAE 把图像压到几十倍小的潜空间再做扩散,直接把落地成本砍下来;**条件机制(cross-attention 注入文本)+ 引导(classifier-free guidance)** 让你能用自然语言精确控制生成。这两块拼起来,就是过去几年图像生成落地的主力架构。

## 1. 动机 / 预备:像素空间扩散为什么贵,以及"我们到底在浪费什么算力"

### 1.1 先把上一章的结论压成三行(本章推导的地基)

为了让本章的 guidance 推导自洽,把扩散的三个核心结论复述清楚(细节见上一章):

```
前向(固定,无参数):  q(xₜ | x₀) = N( xₜ ; √(ᾱₜ)·x₀ , (1−ᾱₜ)·I )
   ⟹ 可一步采样:   xₜ = √(ᾱₜ)·x₀ + √(1−ᾱₜ)·ε ,  ε ~ N(0, I)
训练目标(ε-prediction):  L = E_{x₀,ε,t} ‖ ε − εθ(xₜ, t) ‖²
分数(score)联系:    εθ(xₜ,t) ≈ −√(1−ᾱₜ)·∇_{xₜ} log q(xₜ)
```

`ᾱₜ`(读作 alpha-bar-t)是噪声调度的累积量,从 t=0 的 ≈1 单调降到 t=T 的 ≈0;`xₜ` 是第 t 步的带噪样本。第三行是**全章最重要的桥**:网络预测的噪声 εθ 本质上等价于(带一个负的尺度因子的)**数据对数密度的梯度** ∇log q(xₜ),这个量叫**分数(score)**,它指向"概率密度上升最快的方向"。后面 classifier guidance 和 CFG 的全部推导,都建立在"εθ 就是 score"这个等价上。先记牢它。

> 直觉:训一个网络预测"这张糊图里加了多少噪声",等价于训它指出"该往哪个方向擦一点点,这张图就更像真实数据"。前者是工程上好算的形式,后者是概率上有意义的形式,两者只差一个已知常数。

### 1.2 像素空间扩散的成本结构:算一笔账

扩散采样要把 U-Net 在带噪图上**前向 T 次**(T 从 DDPM 的 1000 到 DDIM 的 20~50)。U-Net 的计算量主要由空间分辨率决定。设输入分辨率 H×W,U-Net 在最高分辨率那一级的卷积/注意力计算量大致正比于像素数 H·W(卷积)或 (H·W)²(全局注意力)。

```
设像素空间分辨率 512×512,潜空间下采样因子 f=8(SD 默认),则潜空间 64×64。
卷积层 FLOPs ∝ 空间像素数:
   像素空间 / 潜空间 = (512·512) / (64·64) = 262144 / 4096 = 64×
若某些层用空间全局自注意力(FLOPs ∝ (token 数)² = (H·W)²):
   像素空间 / 潜空间 = (512²)² / (64²)² = 64² = 4096×
```

**结论:把扩散搬到下采样 8 倍的潜空间,卷积主导的层算力降约 64 倍,注意力主导的层降约几千倍。** 这不是优化技巧,这是**让 512×512、乃至 1024×1024 文生图能在单张消费级 GPU 上跑起来的根本原因**。Rombach et al.(2022,"High-Resolution Image Synthesis with Latent Diffusion Models",即 Stable Diffusion 的论文)的核心 thesis 就一句话:**像素空间里大量算力浪费在了人眼根本不在意的高频细节上,先把它们压掉,扩散只在"语义/感知"层面运算。**

### 1.3 "浪费在哪"——感知冗余与率失真直觉

为什么可以压掉?因为自然图像的信息是**高度可压缩**的:JPEG 能把图压到 1/10 大小而人眼几乎看不出区别,说明像素里有大量**感知冗余(perceptual redundancy)**——高频纹理、微小噪点、不影响"这是什么"的细节。

把它放到**率失真(rate-distortion)** 框架看:压缩分两个阶段。

```
阶段一(感知压缩, perceptual compression):去掉高频冗余,大幅降维,
        但保留人眼可辨的全部语义/结构。  ← 由 VAE/autoencoder 承担
阶段二(语义压缩, semantic compression):在已经去冗的紧凑表示上,
        建模"什么样的内容是合理的"这个语义分布。  ← 由扩散模型承担
```

**Latent Diffusion 的设计哲学就是"分工"**:别让一个昂贵的扩散模型同时干"擦掉噪点(感知)"和"想象合理内容(语义)"两件事。先用一个**一次性训练好、之后冻结**的 autoencoder 把图压到感知冗余已基本去除的潜空间,扩散模型只在这个紧凑空间里专注做语义生成。**这是本章第一个、也是最关键的架构决策。**

## 2. Latent Diffusion 的自编码器:把图压到潜空间(感知压缩)

### 2.1 问题:需要一个什么样的编码器

我们要一个 encoder `E: ℝ^{H×W×3} → ℝ^{h×w×c}`(h=H/f, w=W/f,f 是下采样因子,SD 用 f=8,c 是潜通道数,SD-1.x 用 c=4)和一个 decoder `D: ℝ^{h×w×c} → ℝ^{H×W×3}`,满足:

- **重建要好**:`D(E(x)) ≈ x`,而且要在**感知上**好(人眼觉得像),不只是逐像素 MSE 小。
- **潜空间要"规整"**:潜变量 z 的分布不能太离谱(方差不能爆、不能有奇异结构),否则下游扩散难训。
- **下采样要狠但别过头**:f 太小(如 f=2)压缩不够、扩散还是贵;f 太大(如 f=16、32)丢的语义太多、重建糊、生成上限低。**f=8 是 SD 论文实验出的甜点(待核:论文比较了 f∈{1,2,4,8,16,32},f=4/8 在质量与效率上最优)。**

### 2.2 为什么不用纯 MSE 的 VAE:三个损失项的来历

如果只用逐像素 L2 重建损失训 autoencoder,会得到和"普通 VAE 一样糊"的解码器——因为 **L2 损失对高频细节是"平均主义"的**:面对纹理的多种合理可能,最小化 L2 的最优解是取所有可能的均值,均值就是模糊。所以 LDM 的 autoencoder(常被称作 **VQ-VAE 风格的 autoencoder** 或 KL 正则的 autoencoder)用了**三个损失项的组合**(对应论文中的 autoencoder 训练):

```
L_AE =  L_rec(x, D(E(x)))                    # (1) 重建:L1 或 L2 像素损失
      + L_perc(x, D(E(x)))                   # (2) 感知损失:LPIPS,在 VGG 特征空间比距离
      + λ_adv · L_adv(D(E(x)))               # (3) patch 判别器对抗损失(让纹理逼真)
      + λ_reg · L_reg(z)                      # (4) 潜空间正则:KL 项 或 VQ 量化
```

逐条讲清动机:

- **(1) 像素重建** 保证整体结构和颜色对。
- **(2) 感知损失 LPIPS**(Learned Perceptual Image Patch Similarity,Zhang et al. 2018):把 x 和重建图都喂进一个预训练 VGG,**在深层特征上比 L2 距离**而不是在像素上比。直觉:深层特征对"语义/纹理"敏感、对像素级抖动不敏感,所以它鼓励"看起来像"而非"逐像素对齐",这是抑制模糊的关键。
- **(3) Patch 判别器对抗损失**(借用 GAN,见上一章):一个只看局部 patch 真假的判别器,逼解码器生成**逼真的高频纹理**。这是 autoencoder 重建能"锐利"的另一半原因。
- **(4) 潜空间正则**:防止潜空间退化成任意尺度/结构。两种主流做法:
  - **KL 正则(KL-reg)**:像 VAE 一样让后验 q(z|x) 靠近标准高斯先验,加一个**权重极小**的 KL 项(权重小是关键——它只要轻微规整,不要像标准 VAE 那样强压成高斯而牺牲重建)。这给出**连续潜空间**,SD-1.x/2.x 用的就是这种(常记作 `KL-f8`)。
  - **VQ 正则(VQ-reg)**:用一个 codebook 把 z 量化到最近的码字(VQ-VAE 那套),得到**离散潜空间**。

**关键认知:这个 autoencoder 是先单独训好、然后冻结的(stage-one training)。扩散模型(stage-two)训练和采样时,encoder/decoder 全程不更新。** 这把"感知压缩"和"语义生成"在训练流程上也彻底解耦了。

### 2.3 潜空间扩散的完整数据流

把 §1、§2 拼起来,Stable Diffusion 的训练与采样数据流是:

```
训练(stage 2,autoencoder 已冻结):
   x ──E──▶ z₀                              # 把图编码进潜空间(一次)
   z₀ ──加噪──▶ zₜ = √(ᾱₜ)z₀ + √(1−ᾱₜ)ε     # 在潜空间做前向扩散
   训 εθ(zₜ, t, c) 预测 ε                    # U-Net 在潜空间去噪,c 是条件(下一节)
   损失 L = E ‖ε − εθ(zₜ, t, c)‖²            # 注意:在 z 上算,不在像素上算

采样:
   z_T ~ N(0, I)  (在潜空间采纯噪声,尺寸 h×w×c,不是 H×W×3)
   for t = T … 1:  zₜ₋₁ = 去噪一步(zₜ, εθ(zₜ,t,c))   # T 次 U-Net 前向,全在潜空间
   x = D(z₀)                                  # 最后解码回像素(一次)
```

**注意两个"一次"和一个"T 次"**:encoder 训练时每张图编码一次,decoder 采样时解码一次,这两步成本固定且小;**真正重复 T 次的昂贵循环全部发生在低维潜空间**——这就是 §1.2 算的 64×~几千× 节省落地的地方。

## 3. 条件生成的第一性问题:如何把"条件 c"塞进扩散

无条件扩散学的是 `∇log p(x)`(数据分布的 score)。**条件生成要学的是 `∇log p(x|c)`**——给定条件 c(类别标签、文本、边缘图……)下的 score。怎么得到它?有两条历史路线,先讲清第一条(classifier guidance),它会自然引出本章重点(CFG)。

### 3.1 贝叶斯拆解:条件 score = 无条件 score + 似然 score

对条件密度 p(x|c) 取对数再对 x 求梯度,用贝叶斯公式 `p(x|c) = p(c|x)p(x)/p(c)`:

```
log p(x|c) = log p(c|x) + log p(x) − log p(c)
对 x 求梯度(注意 log p(c) 不含 x,梯度为 0):
   ∇ₓ log p(x|c) = ∇ₓ log p(x) + ∇ₓ log p(c|x)
                    └─无条件 score─┘  └─分类器给的"条件方向"─┘
```

**这个分解是一切引导技术的源头**,逐项品:

- `∇ₓ log p(x)`:无条件扩散模型本来就在学的东西(等价于 −εθ(xₜ,t)/√(1−ᾱₜ))。
- `∇ₓ log p(c|x)`:**一个分类器**关于输入 x 的梯度——它指向"让 x 更像类别 c"的方向。给定一个能对带噪图 xₜ 输出 p(c|xₜ) 的分类器,这一项就是反向传播算出的输入梯度。

把两项加起来,就得到条件 score。**这就是 classifier guidance(分类器引导,Dhariwal & Nichol 2021,"Diffusion Models Beat GANs")的原理。**

### 3.2 Classifier Guidance 的采样公式与引导强度

实践中我们还想**控制"听条件听得多狠"**,于是给似然项乘一个**引导强度 s ≥ 1**(guidance scale):

```
∇ₓ log p_s(x|c) = ∇ₓ log p(x) + s · ∇ₓ log p(c|x)
```

翻译成 ε(用 §1.1 的桥 εθ = −√(1−ᾱₜ)·∇log p),把分数梯度换成对噪声预测的修正:

```
ε̃(xₜ,t,c) = εθ(xₜ,t) − s·√(1−ᾱₜ)·∇_{xₜ} log p_φ(c|xₜ)
                                  └── 分类器 p_φ 对带噪输入的梯度 ──┘
```

s=1 是无偏的条件采样;**s>1 放大条件方向,让样本更"典型地"属于类别 c**(更像那个类、但多样性下降)。Dhariwal & Nichol 正是用这个把扩散的 FID 压过了当时的 GAN。

### 3.3 Classifier Guidance 的三个致命缺点(为什么要被 CFG 取代)

```
缺点1:要额外训一个分类器,而且必须能吃"带噪图 xₜ"——
       普通在干净图上训的分类器对 xₜ 完全失效,得专门在各噪声级别 t 上重训。
缺点2:∇log p(c|x) 是分类器的输入梯度,容易被"对抗样本"式地利用——
       采样可能朝"骗过分类器但不真实"的方向走(adversarial 方向),伤害样本质量。
缺点3:对"文本"这种条件,p(c|x) 是一个图→文的分类/打分模型,
       既难训又难提供逐像素稳定梯度。文生图几乎没法用 classifier guidance 落地。
```

**这三个缺点直接催生了 classifier-free guidance:能不能不要额外分类器,只用扩散模型本身,就实现"放大条件方向"?** 这是本章的技术核心。

## 4. 本章核心:Classifier-Free Guidance(CFG)的完整推导

### 4.1 关键一步:用贝叶斯把"分类器 score"换成"两个扩散模型之差"

Ho & Salimans(2021,"Classifier-Free Diffusion Guidance")的洞见:**§3.1 里那个讨厌的 `∇log p(c|x)`,可以用扩散模型自己的条件/无条件 score 之差表示出来,根本不需要单独的分类器。** 把贝叶斯分解反过来用:

```
由 §3.1:  ∇ₓ log p(c|x) = ∇ₓ log p(x|c) − ∇ₓ log p(x)
            └ 分类器 score = 条件 score − 无条件 score ┘
```

只要我有**一个能同时给出条件 score `∇log p(x|c)` 和无条件 score `∇log p(x)`** 的模型,就能算出那个"条件方向" `∇log p(c|x)`,而**完全不碰分类器**。

把这一行代入 §3.2 的引导公式 `∇log p_s(x|c) = ∇log p(x) + s·∇log p(c|x)`:

```
∇ₓ log p_s(x|c) = ∇ₓ log p(x) + s·[ ∇ₓ log p(x|c) − ∇ₓ log p(x) ]
                = (1 − s)·∇ₓ log p(x)  +  s·∇ₓ log p(x|c)
                = ∇ₓ log p(x)  +  s·[ ∇ₓ log p(x|c) − ∇ₓ log p(x) ]   ← 同一式两种写法
```

**这就是 CFG 的灵魂,而且它有清晰的几何意义**:引导后的 score 是**无条件 score 与条件 score 的线性组合**,系数是 (1−s) 和 s。当 s>1 时,(1−s)<0——这是一个**外推(extrapolation)**:从无条件 score 出发,沿着"条件 score − 无条件 score"这个方向**越过条件 score 继续走**。下一小节把它落到 ε 上,并把"外推"讲透。

### 4.2 落到噪声预测:同一个网络,两种条件输入

CFG 要求一个网络**同时**会做条件预测 εθ(xₜ,t,c) 和无条件预测 εθ(xₜ,t,∅)(∅ 是"空条件"/null token)。怎么得到这样一个网络?**训练时随机丢条件**:以概率 p_uncond(典型 10%~20%)把条件 c 替换成空条件 ∅,其余时候用真条件。于是同一组参数 θ 既学会了 `εθ(·,·,c) ≈ 条件 score`,也学会了 `εθ(·,·,∅) ≈ 无条件 score`。**这就是 "classifier-free" 的含义:不需要外部分类器,无条件模型就是把同一个网络的条件置空。**

用 §1.1 的桥把 §4.1 的 score 组合翻译成 ε 组合(`ε ∝ −∇log p`,负号和公因子 √(1−ᾱₜ) 对每一项一致,线性组合的系数原样保留):

```
┌──────────────────────────────────────────────────────────────────┐
│  ε̃θ(xₜ,t,c) = εθ(xₜ,t,∅) + w·[ εθ(xₜ,t,c) − εθ(xₜ,t,∅) ]          │   (CFG)
│             = (1−w)·εθ(xₜ,t,∅) + w·εθ(xₜ,t,c)        ← 等价展开     │
└──────────────────────────────────────────────────────────────────┘
```

这里我把引导强度记为 **w**,它就是 §4.1 那个 **s**——同一个"差值系数"(§4.1 记 s、这里记 w,即 **w = s**),也正是 diffusers 的 `guidance_scale`(代码即 `noise_uncond + w·(noise_text − noise_uncond)`)。⚠ 注意有些论文(如 Ho & Salimans)用的是 **w′ = w − 1**,把公式写成 `(1+w′)·ε_c − w′·ε_∅`,记法差一个常数,务必看清——见 §4.5 的核对。读这个式子:

- `εθ(xₜ,t,∅)` 是基准(无条件预测)。
- `εθ(xₜ,t,c) − εθ(xₜ,t,∅)` 是**"加了条件后,噪声预测发生的变化方向"**——这正是条件 c 带来的信息,几何上对应 §4.1 的"条件方向"。
- `w` 放大这个方向。**w=0 退化成纯无条件采样;w=1 为纯条件(无外推);w>1 沿条件方向外推,prompt 贴合度上升。**

### 4.3 把"外推"讲透:为什么 w 越大越贴 prompt、但保真/多样性下降

**外推 vs 内插的几何**。考虑三个点(都在 ε 空间):无条件预测 `ε_∅`、条件预测 `ε_c`、引导结果 `ε̃`。

```
ε̃ = ε_∅ + w·(ε_c − ε_∅)
   w=0:  ε̃ = ε_∅            (在无条件点)
   w=1:  ε̃ = ε_c            (恰好在条件点)
   0<w<1: ε̃ 在 ε_∅ 与 ε_c 之间   ← 内插(interpolation)
   w>1:  ε̃ 越过 ε_c,在 ε_c 的"远离 ε_∅"一侧 ← 外推(extrapolation)
```

**SD 默认 w 对应的 guidance_scale 通常是 7.5(待核默认值),远大于 1,说明实践用的是强外推。** 外推的概率含义,把 §4.1 的分布形式写出来最清楚。对 score 做线性组合等价于对**对数概率**做线性组合(再归一化),即引导后的分布正比于:

```
p_w(x|c)  ∝  p(x) · [ p(x|c) / p(x) ]^w
                       └─ 这一项就是 (似然比)^w,放大"条件相关"的区域 ─┘
```

(推导:由 ∇log p_w(x|c) = ∇log p(x) + w·[∇log p(x|c) − ∇log p(x)] = ∇log p(x) + w·∇log p(c|x),积分回去得 log p_w ∝ log p(x) + w·log p(c|x) = log p(x) + w·log[p(x|c)/p(x)],省略不含 x 的常数。)

**逐条解读这个分布,把三件事一次说清:**

- **为什么更贴 prompt**:`[p(x|c)/p(x)]^w` 是**似然比的 w 次幂**。在"条件 c 解释力强"的区域(p(x|c) ≫ p(x),即这个 x 高度符合条件),这个因子被 w 次幂放得很大;在"条件无关甚至矛盾"的区域被压得很小。w 越大,概率质量越往"最典型地符合 c"的地方集中。所以画"红色跑车"时,w 大会得到更饱和、更刻板的红色跑车。
- **为什么多样性下降**:把质量往"最典型"区域集中,等价于**把分布变尖(low temperature)**——尾部、非典型但仍合理的样本被指数级压低,采样反复落在少数高似然比模式上。w→∞ 极限下分布塌成"最符合 c 的众数",多样性趋于 0。
- **为什么保真度(图像真实感)也会下降(过度引导伪影)**:`p_w` 已经**不是任何真实数据分布**(它带了 w 次幂的扭曲),score 被推到训练时从没见过的尺度。表现为 ε̃ 的范数被外推得过大,去噪步长过猛,导致**过饱和、对比度爆炸、边缘出现伪影/油画感**。这是 w 取太大(如 >15)的典型翻车现象。

**一句话总结 w 的作用**:w 是一个**"听话程度 ↔ 真实/多样"** 的旋钮。`提高 w → 更贴 prompt、细节更强,但更刻板、更易过曝失真`。工程上 SD 文生图常用 7~8;追求精确控制可更高并配合负面提示;要多样性/自然就调低。

### 4.4 负面提示(negative prompt)——CFG 的免费副产品

CFG 公式里那个"无条件基准" ε_∅,**不一定非得是空条件**。把它换成"你不想要的条件" c_neg(如 "blurry, lowres, extra fingers"),就得到**负面提示**:

```
ε̃ = εθ(xₜ,t,c_neg) + w·[ εθ(xₜ,t,c_pos) − εθ(xₜ,t,c_neg) ]
```

几何上:从"坏方向"出发,沿"好 − 坏"方向外推,**主动远离 c_neg 描述的区域**。这解释了为什么负面提示在 SD 里这么好用——它不是玄学,就是把 CFG 的基准点从"无条件"挪到了"负面条件"。**这是一个非常能体现"理解机制就能解锁用法"的例子。**

### 4.5【待核】记号与 diffusers 实现的对齐

CFG 的 w/s 记法在文献和代码里不统一,**实现时极易差 1**,这里点明需核对的地方:

```
本章/原论文常见:  ε̃ = ε_∅ + w·(ε_c − ε_∅) ,  w=0(scale=0)=纯无条件点
diffusers 实现常见: noise_pred = noise_uncond + guidance_scale·(noise_text − noise_uncond)
                     此处 guidance_scale 就是本章 w(差值系数):gs=1 → w=1 → 纯条件无外推;gs=7.5 → w=7.5 → 强外推
```

**结论:diffusers 的 `guidance_scale` 在常见实现里就等于本章的 w(都是 (ε_c−ε_∅) 前的系数);默认 7.5 即 w=7.5,强外推。只有 Ho & Salimans 那套 w′=w−1 的记法才与之差一个常数。落地时务必读你用的那版代码的具体公式,别凭记忆套。**(具体某个 diffusers 版本的确切系数定义,待核。)

## 5. 文生图:文本怎么进 U-Net——cross-attention 注入

CFG 解决了"如何放大条件",但还没说**条件 c 本身(尤其是一句话)是怎么在 U-Net 里被"读"进去的**。答案是 **cross-attention(交叉注意力)**。

### 5.1 第一步:文本 → 向量序列(text encoder)

先把文本编码成一串向量。SD 用 **CLIP 的文本编码器**(SD-1.x 用 CLIP ViT-L/14 的 text encoder;SD-2.x 用更大的 OpenCLIP;Imagen 等用 **T5** 这种大型纯文本编码器,实验证明文本编码器越强、prompt 理解越好):

```
prompt ──tokenizer──▶ token ids (定长,如 77)
       ──text encoder──▶ c ∈ ℝ^{L×d_c}   # L 个 token,每个 d_c 维(CLIP-L 是 768)
```

**c 是一个序列(L 个 token 向量),不是单个向量**——这点至关重要,因为 cross-attention 要逐 token 地让图像去"查询"文本。

### 5.2 第二步:cross-attention 让图像特征"查询"文本

回忆自注意力(self-attention):Q、K、V 都来自同一组特征。**cross-attention 的唯一区别是 Q 和 K/V 来自不同来源**:

```
设 U-Net 某层的图像特征(空间展平成 token)为  F ∈ ℝ^{(h·w)×d}    # 图像 token,Q 的来源
文本编码                                       c ∈ ℝ^{L×d_c}        # 文本 token,K/V 的来源

Q = F · W_Q          # [(h·w) × d_k]    图像特征做查询(我这个像素该关注哪些词)
K = c · W_K          # [   L    × d_k]    文本做键
V = c · W_V          # [   L    × d_v]    文本做值

Attn(Q,K,V) = softmax( Q·Kᵀ / √d_k ) · V       # [(h·w) × d_v]
              └── 注意力图 A ∈ ℝ^{(h·w)×L} ──┘   每个图像 token 对每个词的权重
输出再投影回 d 维,残差加回 F。
```

**机制讲透**:

- **Q·Kᵀ 的形状是 (h·w)×L**:第 i 行第 j 列 = "图像第 i 个位置" 与 "第 j 个词" 的相关度。softmax 沿 L(词)归一化,得到**每个图像位置对各个词的注意力权重**。
- **乘 V**:每个图像位置据此**加权聚合对应词的语义向量**,把"它最该听的那些词的信息"拉进自己的特征。比如生成"戴帽子的柯基",帽子区域的图像 token 会对 "hat" 这个词的 K 给出高权重,从而把 "hat" 的语义 V 注入帽子区域。
- **Q 来自图像、K/V 来自文本——所以是"图像主动查询文本"**,文本作为可被检索的知识库。维度上 Q 和 K 必须同维 d_k(否则点积无意义),但 V 的维度 d_v 可不同。**`d_c`(文本维)和 `d`(图像维)可以不同**,靠 W_K、W_V、W_Q 这几个投影矩阵把它们对齐到公共的 d_k/d_v——这是新手容易忽略的工程点:cross-attention 的投影矩阵承担了"对齐两个模态维度"的活。

### 5.3 第三步:注入 U-Net 的每个尺度,每一步去噪

U-Net 的多个分辨率级别都插入 cross-attention 块(通常在每个 ResBlock 之后接一个 Transformer 块,内含 self-attn + cross-attn + FFN)。这带来两个性质:

- **多尺度注入**:低分辨率级别(感受野大)让文本影响**全局布局/构图**,高分辨率级别让文本影响**局部细节/纹理**。同一句 prompt 在不同尺度被反复"咨询"。
- **每步去噪都注入**:采样的 T 步里,**每一步** U-Net 前向都重新做一遍 cross-attention,所以文本条件在整个去噪轨迹上**持续施加影响**,而不是开头给一次就放手。

```
完整条件 U-Net 前向(一步):
   zₜ ─▶ [ResBlock(含 t 的 embedding) → SelfAttn → CrossAttn(K,V 来自 c) → FFN] × 多尺度 ─▶ εθ(zₜ,t,c)
                                                      └── 文本 c 在此注入 ──┘
   时间步 t 通过正弦 embedding + MLP 注入(加到 ResBlock 的特征上),与文本注入是两条独立通道。
```

**至此文生图闭环了**:文本→CLIP 编码→cross-attention 注入潜空间 U-Net 的每尺度每步→εθ(zₜ,t,c) 给出条件噪声预测→CFG 放大条件方向→去噪循环→VAE 解码出像素图。**Stable Diffusion = §2(潜空间)+ §4(CFG)+ §5(cross-attention 文本注入)的组装。**

## 6. 更强的可控生成:ControlNet、LoRA、图像编辑

文本是粗粒度条件("画只猫"),但很多场景要**精确的空间/结构控制**(按这个姿势、这条边缘、这个深度图生成),或**轻量定制风格**,或**编辑已有图**。下面讲三类主流机制的核心思想(实现细节留给阅读)。

### 6.1 ControlNet:加一路空间结构条件,且"不破坏原模型"

**问题**:想让生成严格遵循一张**结构图**(Canny 边缘、人体姿态骨架、深度图、分割图)。这种条件是**空间对齐的**(条件图每个像素对应输出每个位置),cross-attention 那种"全局检索"不擅长这种密集逐像素约束。

**ControlNet(Zhang et al. 2023)的设计**:

```
1) 复制一份预训练 U-Net 的 encoder(可训练副本),原 U-Net 全程冻结。
2) 结构条件图先过几层卷积编码,加到这个可训练副本的输入上。
3) 副本各层输出经过"零卷积(zero-conv,初始化为全 0 的 1×1 卷积)"再加回原 U-Net 对应层。
```

**两个精妙的设计点,要讲透**:

- **为什么复制 encoder 而不是从头加分支**:复用预训练权重,新条件分支一开始就站在"已经会生成"的肩膀上,训练快、不容易把原模型的能力搞坏。
- **为什么用 zero-conv(零初始化卷积)**:训练第一步,zero-conv 输出恒为 0,**ControlNet 对原 U-Net 的影响恰好为 0**,即起点等于原模型行为,然后梯度让它**从零平滑地学习注入条件**。这避免了训练初期随机分支输出破坏预训练模型(类似残差/恒等初始化的思想)。**这是 ControlNet 能在小数据上稳定微调出强空间控制的关键。**

代价:推理时多跑一路 encoder 副本,**计算和显存约增加一个 encoder 的量级**;每种结构条件(边缘/姿态/深度)通常训一个独立 ControlNet。

### 6.2 LoRA:用低秩增量轻量微调风格/概念

**问题**:想让模型学会一种**新画风**或**特定角色**,但全量微调整个 U-Net(几亿~十几亿参数)太贵、存一份就是几个 GB。

**LoRA(Low-Rank Adaptation,Hu et al. 2021,本是 LLM 微调方法,迁移到扩散极成功)的核心**:冻结原权重 W₀,只学一个**低秩增量** ΔW = B·A:

```
原线性层:   y = W₀·x                       # W₀ ∈ ℝ^{d_out×d_in},冻结
LoRA 改:    y = W₀·x + ΔW·x = W₀·x + (B·A)·x
            其中 A ∈ ℝ^{r×d_in},B ∈ ℝ^{d_out×r},秩 r ≪ min(d_out,d_in)(常 r=4~64)
            初始化:B=0(故起点 ΔW=0,行为=原模型),A 随机。
```

**量化收益(算一笔)**:全量微调一个 d_out×d_in 的层要更新 `d_out·d_in` 个参数;LoRA 只更新 `r·(d_out+d_in)` 个。设 d_out=d_in=1024,r=8:

```
全量:  1024·1024      = 1,048,576 参数
LoRA:  8·(1024+1024)  =    16,384 参数      ⟹  仅约 1.6%,缩小 ~64×
```

**所以一个 SD 的 LoRA 文件常只有几 MB ~ 几十 MB**,而全量微调要存几个 GB。在扩散里,LoRA 通常加在 **U-Net 的 attention 投影层(W_Q/W_K/W_V/W_O)**(也可加在 cross-attention 上来调"文本如何影响图像",从而学新概念)。推理时把 B·A 合并进 W₀ 即可,**零额外推理开销**(合并后就是普通权重)。这是 LoRA 比 ControlNet 轻得多的根本原因。

### 6.3 图像编辑:img2img 与 inpainting 的统一直觉

**img2img(图生图)**:不从纯噪声 z_T 开始,而是把输入图编码 z₀=E(x),**只加噪到中间某步 t₀**(`z_{t₀} = √(ᾱ_{t₀})z₀ + √(1−ᾱ_{t₀})ε`),再从 t₀ 去噪到 0。

```
强度参数 strength = t₀/T ∈ (0,1):
   strength 小(t₀ 小):只加一点噪声 → 保留原图大量结构,改动小(微调风格/色调)
   strength 大(t₀ 大):加很多噪声 → 接近从头生成,原图只剩大致布局
```

**机制直觉**:加噪是"部分遗忘",t₀ 决定遗忘多少。保留的低频结构 + 文本条件重新去噪 = 在原图骨架上按 prompt 重绘。**这是一个旋钮把"完全保留原图"和"完全重画"连续连起来的优雅设计。**

**inpainting(局部重绘)**:给一张图和一个 mask(要重画的区域),每一步去噪后,**用前向加噪把"已知区域"强行替换回原图对应噪声级别的内容**,只让 mask 内的区域自由去噪:

```
每步 t:
   z_known = √(ᾱₜ)·E(x_orig) + √(1−ᾱₜ)·ε        # 已知区按前向公式贴回(它该有的噪声)
   z_unknown = 去噪一步得到的 zₜ₋₁                # mask 内区域正常去噪
   zₜ₋₁ = mask·z_unknown + (1−mask)·z_known       # 拼接:mask 外用已知,mask 内用生成
```

**直觉**:让模型"在已知上下文的约束下,只想象 mask 里该填什么"。已知区每步被钉死,mask 区在它的语义包围中被 cross-attention(文本)+ 邻域上下文共同引导着长出来。专门的 inpainting 模型还会把 mask 作为额外输入通道喂进 U-Net,效果更好。

## 7. 采样加速:把几十步蒸馏成 1~4 步(一致性模型与蒸馏)

### 7.1 问题:扩散慢的根在哪

扩散采样要 T 次 U-Net 前向(即使 DDIM 把 1000 降到 20~50,也比 GAN 的"1 次前向出图"慢一两个数量级)。CFG 还把每步**翻倍**(要同时算条件和无条件两次前向)。**这是扩散相对 GAN 唯一显著的劣势,加速是落地刚需。**

慢的本质:去噪轨迹是一条**概率流 ODE(probability flow ODE)** 的解曲线(上一章:扩散采样可写成求解一个从噪声到数据的常微分方程),而我们用很多小步去数值积分它。**能不能学一个网络一步就从噪声跳到这条曲线的终点(数据)?**

### 7.2 一致性模型(Consistency Models)的核心思想

**Consistency Models(Song et al. 2023)** 训一个函数 f(xₜ, t),要求它对**同一条 ODE 轨迹上的任意点**,都直接输出该轨迹的**起点 x₀**(数据端):

```
一致性性质(自洽约束):  f(xₜ, t) = f(x_{t'}, t')  对同一轨迹上任意 t, t' 成立,且都 = x₀
边界条件:  f(x₀, 0) = x₀     (在数据端是恒等映射)
```

**几何直觉**:把概率流 ODE 的每条轨迹想成一条从噪声流向数据的河;一致性函数学的是"无论你站在河的哪一点,都直接告诉你这条河的入海口在哪"。**一旦学会,采样只需一步**:采个噪声 x_T,`x₀ = f(x_T, T)`,完事。

两种训法:**(a) 蒸馏(consistency distillation)**——用一个训练好的扩散模型当 teacher 生成 ODE 轨迹上相邻点对 (x_{t}, x_{t'}),逼 f 在这对点上输出一致;**(b) 独立训练(consistency training)**——不用 teacher,直接构造约束训。1~4 步采样质量已接近多步扩散(略有损失)。

### 7.3 其它蒸馏路线(一句话各表)

- **渐进式蒸馏(Progressive Distillation,Salimans & Ho 2022)**:训一个 student,让它**一步**匹配 teacher**两步**的结果;反复蒸馏,步数每轮减半(1000→500→…→4)。
- **对抗蒸馏(如 SDXL-Turbo / ADD)**:在蒸馏里加 GAN 判别器损失,逼 1~4 步输出更真,把上一章的对抗思想拿回来补蒸馏的保真。
- **LCM(Latent Consistency Models)**:把一致性模型用到潜空间 SD 上,配 LCM-LoRA 实现"给任意 SD/LoRA 即插即用的 2~4 步加速"。

**统一认知**:这些方法都在**把"慢但准的多步数值积分"换成"快但近似的少步直跳"**,本质是用一个更强的网络去摊销采样时的迭代成本——**训练时多花算力学轨迹,换采样时的步数**。这与 GAN 的"一步出图"殊途同归,但起点是已经训好的高质量扩散模型,所以质量上限更高。

## 8. 代码:CFG 采样循环 + cross-attention 条件注入

下面给两段可照着实现的 PyTorch 核心片段,**只写体现机制的部分**,U-Net/VAE/text encoder 用占位接口。

### 8.1 cross-attention 块(文本注入图像)

```python
import torch, torch.nn as nn, torch.nn.functional as Fn

class CrossAttention(nn.Module):
    """图像特征做 Q,文本编码做 K/V —— 把一句话注入到图像 token。"""
    def __init__(self, dim_img, dim_ctx, dim_head=64, heads=8):
        super().__init__()
        inner = dim_head * heads
        self.heads, self.scale = heads, dim_head ** -0.5
        # 关键:Q 来自图像(dim_img),K/V 来自文本(dim_ctx),投影到公共 inner 维度对齐两个模态
        self.to_q = nn.Linear(dim_img, inner, bias=False)
        self.to_k = nn.Linear(dim_ctx, inner, bias=False)
        self.to_v = nn.Linear(dim_ctx, inner, bias=False)
        self.to_out = nn.Linear(inner, dim_img)

    def forward(self, x, context):
        # x: 图像 token [B, h*w, dim_img];  context: 文本 token [B, L, dim_ctx]
        B, N, _ = x.shape
        q = self.to_q(x)            # [B, N, inner]   图像查询
        k = self.to_k(context)      # [B, L, inner]   文本键
        v = self.to_v(context)      # [B, L, inner]   文本值

        # 拆多头: [B, heads, seq, dim_head]
        split = lambda t: t.view(B, -1, self.heads, t.size(-1)//self.heads).transpose(1, 2)
        q, k, v = map(split, (q, k, v))

        attn = (q @ k.transpose(-1, -2)) * self.scale   # [B, heads, N, L] 图像位置×词 的相关度
        attn = attn.softmax(dim=-1)                     # 沿 L(词)归一化:每个像素该听哪些词
        out = attn @ v                                  # [B, heads, N, dim_head] 加权聚合文本语义

        out = out.transpose(1, 2).reshape(B, N, -1)     # 合并多头
        return self.to_out(out)                         # 投回图像维,外部再做残差相加
```

**易踩坑**:(1) softmax 必须沿**文本维 L**(`dim=-1`),代表"每个图像位置在所有词上分配注意力";沿错维度语义全错。(2) Q 与 K 投影后必须同维(都 inner),V 可不同维但这里取一致省事。(3) `context` 在无条件分支(CFG 的 ∅)就传"空 prompt 的编码"(全空串/特殊 null token 的 CLIP 编码),保持张量形状不变。

### 8.2 CFG 采样循环(DDIM 风格,含条件/无条件双前向)

```python
@torch.no_grad()
def sample_cfg(unet, vae_decode, text_encode, prompt, neg_prompt="",
               steps=30, guidance_scale=7.5, shape=(1, 4, 64, 64),
               alphas_bar=None, device="cuda"):
    """潜空间 CFG 采样。unet(z,t,c)->ε;返回像素图。alphas_bar: 长度 T 的 ᾱ 调度。"""
    # 1) 文本编码:正向 prompt 与负向/空 prompt 各编一份(K/V 来源)
    c_pos = text_encode([prompt])              # [1, L, d_ctx]
    c_neg = text_encode([neg_prompt])          # [1, L, d_ctx]  空串即标准无条件 ∅

    # 2) 从潜空间纯噪声起步
    z = torch.randn(shape, device=device)

    # 选一个步数子集(DDIM 跳步),时间从大到小
    ts = torch.linspace(len(alphas_bar) - 1, 0, steps).long().to(device)

    for i, t in enumerate(ts):
        ab_t = alphas_bar[t]
        # 3) 关键:同一个 z 上做两次前向 —— 条件预测 和 无条件(负向)预测
        eps_pos = unet(z, t, c_pos)            # ε_c
        eps_neg = unet(z, t, c_neg)            # ε_∅ (或负面提示)
        # 4) CFG 外推:沿 (条件−无条件) 方向放大。guidance_scale=1 即无引导
        eps = eps_neg + guidance_scale * (eps_pos - eps_neg)

        # 5) 由 ε 反推 z₀ 估计,再做一步 DDIM 更新(确定性版,η=0)
        z0_hat = (z - (1 - ab_t).sqrt() * eps) / ab_t.sqrt()
        if i < len(ts) - 1:
            ab_prev = alphas_bar[ts[i + 1]]
            # DDIM 确定性更新:用 z₀ 估计和当前 ε 走到下一个噪声级别
            z = ab_prev.sqrt() * z0_hat + (1 - ab_prev).sqrt() * eps
        else:
            z = z0_hat                          # 最后一步直接取 z₀ 估计

    # 6) 解码回像素(整段采样唯一一次接触像素空间)
    return vae_decode(z)
```

**实现要点回顾**:

- **每步两次 U-Net 前向**(条件 + 无条件)是 CFG 的代价,**实现里通常把它俩拼成 batch 维一次前向**(`unet(cat([z,z]), t, cat([c_neg,c_pos]))` 再切开)以省一次 kernel 启动——这是 diffusers 的标准做法,本片段为清晰分开写。这也解释了 §7.1 说的"CFG 让每步翻倍"。
- `guidance_scale` 的数值含义见 §4.5【待核】——此处写法里 `=1` 是无引导、`=7.5` 是强外推,对应本章 (1+w) 族。
- `z0_hat` 那行是"由当前带噪 z 和预测的 ε 反解干净潜变量"的标准式(就是前向公式 `zₜ=√ᾱₜ z₀+√(1−ᾱₜ)ε` 解出 z₀);DDIM 更新公式见上一章,这里用确定性版(η=0)。
- 全程在潜空间 `[1,4,64,64]` 运算,**只有最后 `vae_decode` 一次进像素空间**——§2.3 数据流的代码兑现。

## 9. 量化分析:成本、显存、参数

### 9.1 潜空间 vs 像素空间(把 §1.2 落实到端到端)

```
SD-1.5 设定:像素 512×512×3,f=8,潜空间 64×64×4。
单次 U-Net 前向 FLOPs ∝ 主导层的空间规模:
   若像素空间扩散,U-Net 在 512² 上跑;潜空间在 64² 上跑。
   卷积主导层:省 (512/64)² = 64×
   全局自注意力层:省 ((512²)/(64²))² = 64² = 4096×
采样总成本 ≈ (U-Net 单次前向) × T × 2(CFG双前向) + 1次 VAE decode + 1次 text encode
   ⟹ 潜空间把 "× T × 2" 这个大头整体压在 64² 而非 512² 上,这是落地的决定性差异。
显存:激活显存同样 ∝ 空间像素数,潜空间使峰值激存降约 64×(卷积部分),
      让 512/768/1024 文生图能在 8~12GB 消费卡上跑。
```

### 9.2 各组件的参数量级(数量级,具体值待核)

```
SD-1.x 组件参数量级(待核确切数字):
   U-Net(条件去噪主体)      ~ 8.6 亿参数      ← 推理算力大头(× T × 2 次)
   VAE(encoder+decoder)     ~ 0.8 亿参数      ← 只前向 1 次(decode),占比小
   CLIP text encoder         ~ 1.2 亿参数      ← 只前向 1 次,占比极小
合计约 10 亿级。SDXL 把 U-Net 扩到约 26 亿 + 双 text encoder,质量升、成本升。
```

**一个量化直觉**:推理成本几乎全在 **U-Net × T × 2(CFG)**。所以加速的两个正交方向很清楚——**降 T**(更好的采样器 DDIM/DPM-Solver,或 §7 的蒸馏到 1~4 步)和**降单次前向**(更小 U-Net、潜空间已经做了一层)。CFG 的"× 2"则是质量与成本的固定 trade-off(也有 "CFG distillation" 把它蒸掉)。

### 9.3 LoRA 的参数与存储节省(已在 §6.2 算过,汇总)

```
单层 d×d=1024²:全量 1.05M 参数;LoRA(r=8) 16K 参数 ⟹ ~1.6%。
整个 SD U-Net 的 attention 层加 LoRA,文件常 4~150 MB,对比全量 ckpt 数 GB。
推理:合并 B·A 进 W₀ 后零额外开销。
```

## 10. 设计权衡与常见坑

- **f(下采样因子)的两难**:f 大(16/32)算力省但重建糊、生成上限低;f 小(2/4)质量高但扩散贵。**f=8 是 SD 选的甜点(待核)**。改 f 必须重训 VAE 和 U-Net,不能只换一个。
- **VAE 是质量天花板**:扩散再强,最终也要过 decoder。**潜空间丢掉的细节扩散救不回来**——SD 早期"小字/手指画不好"部分根源在 f=8 的 VAE 把这些高频结构压没了(以及数据/U-Net 容量)。所以有人专门微调/替换 VAE(如更好的 VAE-ft)提升细节。
- **CFG 的 w 是双刃剑**:w 太小不贴 prompt、构图松散;w 太大过曝、油画感、多样性崩。**典型 7~8;高 w 必须配负面提示和好采样器,否则伪影**。这是 §4.3 推导的直接实践推论。
- **w 与 s 差 1 的记号坑**:文献/代码里 `ε̃=ε_∅+w(ε_c−ε_∅)` 与 `=(1+w)ε_c−wε_∅` 两种写法、guidance_scale 是 w 还是 1+w,**极易差一个 1**。落地以你用的代码实际公式为准(§4.5)。
- **cross-attention 的 softmax 维度**:必须沿**文本 token 维**归一化,代表"每个图像位置在词上的注意力分配"。写错维度图文完全对不上。
- **无条件分支不能省形状**:CFG 的 ∅ 分支要传"空 prompt 的编码",张量形状与正向一致(同样 L 个 token),否则 batch 拼接/广播出错。
- **ControlNet 的 zero-conv 不能随机初始化**:零初始化是它训练稳定的命门,随机初始化会让条件分支在训练初期破坏预训练 U-Net。
- **inpainting 的已知区要"贴回带噪版本"而非干净原图**:每步要把已知区贴成"该噪声级别下的内容"(`√ᾱₜ·z_known+√(1−ᾱₜ)·ε`),直接贴干净 z₀ 会与当前噪声级别不匹配,产生接缝。
- **加速 trade-off**:1~4 步蒸馏模型(LCM/Turbo)极快但**多样性和细节通常略逊**多步,且对 CFG 的支持方式不同(很多蒸馏模型内建固定引导,外部 guidance_scale 行为变了)。要质量上限仍用多步 + DPM-Solver 之类。
- **扩散 vs GAN 的取舍**(承上一章):扩散训练稳(无对抗博弈)、覆盖度好、质量高、可控性强(CFG+cross-attn+ControlNet),但**采样慢**(× T,这是它对 GAN 唯一明显劣势,§7 在补);GAN 一步出图但训练脆、控制难。当代落地图像生成基本由扩散主导,GAN 退守"实时/超分/蒸馏判别器"等场景。

## 13. 小结与承上启下

这一章把"Stable Diffusion 为什么能落地、为什么可控"拆成了三块可推导的机制:

- **潜空间扩散(§1–§2)**:先用一个一次性训好、之后冻结的 autoencoder 做**感知压缩**(LPIPS + patch 判别器 + 轻 KL/VQ 正则,专治 MSE 的模糊),把图压到下采样 8 倍的潜空间;扩散只在低维潜空间专注**语义生成**。算一笔账就知道这把卷积层算力降约 64×、注意力层降几千×——**这是高分辨率文生图能在消费卡上跑的根本原因**,不是工程小技巧。
- **可控生成(§3–§6)**:条件 score 的贝叶斯分解 `∇log p(x|c)=∇log p(x)+∇log p(c|x)` 是一切引导的源头;**CFG** 把那个讨厌的分类器项换成"同一模型的条件/无条件预测之差",采样时 `ε̃=ε_∅+w(ε_c−ε_∅)` 沿条件方向**外推**,w 是"听话 ↔ 真实/多样"的旋钮(对应 `p_w∝p(x|c)·[p(x|c)/p(x)]^w` 的分布变尖)。文本靠 **cross-attention**(图像 Q、文本 K/V)在 U-Net 每尺度每步注入;ControlNet(zero-conv 加空间结构条件)、LoRA(低秩增量轻量定制)、img2img/inpainting(用加噪强度连接"保留↔重画")补齐精确控制与编辑。
- **加速(§7)**:把"慢但准的多步 ODE 积分"蒸馏成"快但近似的 1~4 步直跳"(一致性模型/渐进蒸馏/对抗蒸馏),补上扩散相对 GAN 唯一的劣势。

**它在整门课里的位置**:上一章的扩散基础给了你 `εθ ≈ score` 这把钥匙,本章用它打开了"条件与可控"这扇门——你现在应当能把任何一个 SD 衍生系统(SDXL、ControlNet、LoRA 生态、各种编辑工具)还原成"潜空间 + 条件注入 + 引导 + 采样器"这四个零件的组合,而不是把它当 prompt 玄学。**再往后**,这套"用一个网络估计 score / 用条件信号引导一个生成轨迹"的思想会继续延伸:flow matching / rectified flow 用更直的轨迹进一步加速,DiT(Diffusion Transformer)把 U-Net 换成 Transformer 主干并随算力 scaling,而 cross-attention 这把"跨模态注入"的工具会在视频、3D、多模态生成里被反复复用。**你在本章建立的"把条件生成拆成 score 分解 + 注入机制 + 引导外推"的分析框架,是读懂所有现代条件生成系统的通用骨架。**
