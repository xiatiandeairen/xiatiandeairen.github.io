> 这章把你从"自回归 decode 一次只能吐一个 token、慢得心痛"带到"能亲手推出投机解码为什么数学上严格无损、能算清它什么时候净赚、能写出 draft-then-verify 主循环、能讲清 Medusa/EAGLE 的取舍"。读完你应该能在面试白板上把无损性证明一行不漏地推完。

## 引言:一句话和一个反直觉

自回归大模型生成第 t 个 token,必须先有前 t−1 个 token。这是个串行依赖,看起来没法并行。**投机解码(speculative decoding)的反直觉之处在于:它不打破这个依赖,却能在不改变输出分布的前提下,把多个 token 的生成"折叠"进一次大模型前向。** 而且这个"不改变输出分布"不是"误差很小"的近似,是逐 token 严格相等的数学等价——这是它和量化、剪枝、蒸馏等所有 lossy 加速手段的本质区别。

整章围绕两个问题:**(1) 为什么能并行?**(答案在硬件:decode 是 memory-bound,算力大量闲置,顺手多算几个 token 几乎免费)**(2) 为什么无损?**(答案在数学:一个精心设计的 accept/resample 方案,让最终采样分布的边缘分布恰好等于目标分布 p)。把这两个问题各自推到底,你就吃透了投机解码。

## 一、为什么 decode 慢:memory-bound 的第一性原理

### 1.1 prefill vs decode 的根本差异

LLM 推理分两个阶段。**prefill**(预填充):把整段 prompt 一次喂进去,所有位置并行算,产出第一个 token 和全部 KV cache。**decode**(解码):此后每步只输入上一个 token,算出下一个 token,逐个吐。

关键差异在"一次前向处理多少个 token":prefill 一次处理 S 个(S = prompt 长度,几百上千),decode 一次只处理 1 个。这个 1,正是慢的根源。

### 1.2 算一笔账:decode 的算术强度极低

我们用 **算术强度(arithmetic intensity)** = 浮点运算数 / 访存字节数(单位 FLOP/Byte)来判断一个 kernel 是 compute-bound 还是 memory-bound。判据来自 Roofline 模型:

```
设硬件峰值算力 = P (FLOP/s),峰值显存带宽 = B (Byte/s)
ridge point(脊点)算术强度 I* = P / B
  · 若 kernel 的算术强度 I < I*  → memory-bound(瓶颈是搬数据)
  · 若 I > I*                    → compute-bound(瓶颈是算)
```

对一个稠密 Transformer,记参数量为 N(权重 N 个数,FP16 下占 2N 字节)。decode 处理 1 个 token 时:

```
计算量:每个权重参与一次乘加 = 2 FLOP/参数 → 总计算 ≈ 2N FLOP
访存量:把全部权重从 HBM 读进来一次 → ≈ 2N Byte(FP16)
        (KV cache 读取另算,batch=1 短序列下权重读取占主导)

算术强度 I_decode ≈ 2N FLOP / 2N Byte = 1 FLOP/Byte
```

**decode 单序列的算术强度约等于 1 FLOP/Byte,这是个极低的数。** 对比现代加速器,P/B 的脊点 I* 通常在几十到几百 FLOP/Byte 量级(例如某张卡 FP16 算力约 10² TFLOP/s、HBM 带宽约 10³ GB/s,脊点就在 10² FLOP/Byte 量级,具体数值待核)。**I_decode ≪ I*,所以 decode 是重度 memory-bound:GPU 大部分时间在等权重从 HBM 搬过来,算力(tensor core)严重闲置。**

直觉版总结:**为了算出 1 个 token,你把几百 GB 的权重整个搬了一遍。搬运是瓶颈,真正的乘加只占了一点点时间。**

### 1.3 关键杠杆:多算 K 个 token 几乎免费

现在问:如果一次前向不是处理 1 个 token,而是处理 K 个 token(它们的 KV、位置都已知),耗时会变成 K 倍吗?

**不会。** 因为权重只需要搬一次,就能服务这 K 个位置的计算:

```
处理 K 个 token 一次前向:
  计算量 ≈ 2N · K FLOP     (K 个位置各自过一遍权重)
  访存量 ≈ 2N Byte         (权重还是只读一次!K 个位置共享)
            + KV/激活相关的 O(K) 项(相对 2N 小,先忽略)
  算术强度 I ≈ 2N·K / 2N = K FLOP/Byte
```

只要 K 还没大到让算术强度超过脊点 I*(即 K ≲ I*,通常几十),**这次前向仍然是 memory-bound,耗时由"读一遍权重"决定,和 K=1 时几乎一样。** 换句话说:

> **在 memory-bound 区,大模型一次前向算 1 个 token 和算 K 个 token,墙钟时间几乎相同。多出来的算力本来就是闲置的,白用。**

这正是投机解码的物理基础。问题只剩一个:decode 时第 2…K 个 token 还没生成出来,我哪来 K 个已知 token 喂给大模型做并行前向?

**答案:用一个便宜的小模型先"猜"出来。** 这就是 draft-then-verify。

> ⚠ 一个新手常误解:有人以为投机解码靠"小模型分担计算"省钱。错。小模型的前向也要钱。它真正省的是**把大模型本来串行的 K 次 memory-bound 前向,合并成 1 次**——省的是大模型读权重的次数(从 K 次降到约 1 次)。

## 二、draft-then-verify 机制总览

一轮投机解码的流程:

```
给定:大模型(target)p,小草稿模型(draft)q,当前已生成前缀 x₁..xₜ
1. draft 阶段:用 q 自回归地连续采样 K 个候选 token
     x̃₁ ~ q(·| 前缀)
     x̃₂ ~ q(·| 前缀, x̃₁)
     ...
     x̃_K ~ q(·| 前缀, x̃₁..x̃_{K-1})
   (这是 K 次小模型前向,便宜)
2. verify 阶段:把 [前缀, x̃₁..x̃_K] 拼成一个序列,大模型 p 做一次前向,
   并行得到 K+1 个位置的分布:
     p(·| 前缀), p(·| 前缀,x̃₁), ..., p(·| 前缀,x̃₁..x̃_K)
   (这是 1 次大模型前向,memory-bound,≈ 算 1 个 token 的钱)
3. accept/resample 阶段:从左到右逐位置裁决每个 x̃ᵢ 接受还是拒绝。
   一旦在某位置 j 拒绝,丢弃 x̃_j..x̃_K,从一个"修正分布"重采样一个 token 顶上,
   本轮结束。若全部 K 个都接受,再用第 K+1 个位置的 p 免费多采一个 token。
```

每轮的产出 token 数在 1 到 K+1 之间。**核心断言:无论 q 多烂,只要 accept/resample 规则设计正确,这套流程吐出的 token 序列,其分布严格等于直接从 p 逐个采样。** q 烂只会让接受率低、加速比小,但绝不会污染输出分布。下面证明它。

## 三、无损性证明:核心中的核心

这是整章必须推到一字不差的部分。先把单步问题抽象干净。

### 3.1 问题抽象:对单个位置

固定一个位置和它的前缀。记词表上目标分布为 p(x),草稿分布为 q(x)(都是离散分布,∑ₓ p(x) = ∑ₓ q(x) = 1)。草稿模型先采了一个候选 x ~ q。我们要设计一个规则,**只用一次 p 的求值**(因为 verify 已经免费给了 p),把这个来自 q 的样本"加工"成一个真正服从 p 的样本。

**投机采样规则(Leviathan et al. / Chen et al.):**

```
1. 草稿采样:  x ~ q(x)
2. 接受测试:  采 r ~ Uniform(0,1)
              若 r ≤ min(1, p(x)/q(x))  → 接受,输出 x
3. 拒绝重采样:否则(拒绝),从"残差分布"重新采样输出:
                       (p(x) − q(x))₊
              p'(x) = ─────────────────         其中 (z)₊ = max(z, 0)
                      ∑ₓ' (p(x')−q(x'))₊
              输出 x ~ p'(x)
```

直觉:q 在某点 x 上采样过头了(q(x) > p(x)),我们就以一定概率"退货"它;退货后从"p 比 q 还想要、但 q 没给够"的那部分质量((p−q)的正部)里补一个。下面证明这套规则的输出严格服从 p。

### 3.2 主定理与逐步证明

**定理:** 按上述规则得到的输出 token,其分布恰为 p。即对任意 x,P(输出 = x) = p(x)。

**证明。** 输出 = x 有两条互斥路径:**(A)** 草稿采到 x 且被接受;**(B)** 草稿采到某个(任意)token 但被拒绝,随后重采样恰好采到 x。分别算这两条路径的概率,再相加。

**第一步:接受概率(逐点)。** 草稿在 x 处采样的概率是 q(x);给定采到 x,接受概率是 min(1, p(x)/q(x))。所以"采到 x 且接受"的联合概率:

```
P(采到 x 且接受) = q(x) · min(1, p(x)/q(x))
                 = min(q(x), p(x))          ← 把 q(x) 乘进 min
```

最后一步用了恒等式 a·min(1, b/a) = min(a, b)(对 a>0 成立;a=0 时 q(x)=0 该项也为 0,无问题)。**记住这个干净的中间结果:接受路径在 x 处贡献 min(p(x), q(x))。** 这就是路径 (A) 的概率,记 P_A(x) = min(p(x), q(x))。

**第二步:总拒绝概率 β。** 对路径 (B),先算"发生拒绝"这件事的总概率(不分拒绝时草稿采到了谁)。总接受概率是把第一步对所有 x 求和:

```
P(接受) = ∑ₓ min(p(x), q(x))           记作 α
```

所以总拒绝概率:

```
β = 1 − α = 1 − ∑ₓ min(p(x), q(x))
```

这里用一个恒等式把 β 写成更有用的形式。对每个 x,有 p(x) − min(p(x),q(x)) = (p(x) − q(x))₊(因为若 p≥q 则左边 = p−q = 正部;若 p<q 则左边 = p−p = 0 = 正部)。两边对 x 求和:

```
∑ₓ (p(x) − q(x))₊ = ∑ₓ p(x) − ∑ₓ min(p,q) = 1 − α = β        (★)
```

**(★) 是关键:残差正部的总质量恰好等于总拒绝概率 β。** 这保证了第 3 步里残差分布 p'(x) = (p(x)−q(x))₊ / β 的分母正是 β,p' 是个合法分布(非负且和为 1)。

**第三步:路径 (B) 在 x 处的贡献。** 路径 (B) = 先发生拒绝(概率 β),再从残差分布 p' 采到 x。注意:**重采样和"拒绝前草稿采到了谁"无关**——一旦决定拒绝,就完全从 p' 重新独立采样。所以:

```
P_B(x) = P(拒绝) · p'(x)
       = β · (p(x) − q(x))₊ / β        ← 代入 p' 定义,β 约掉
       = (p(x) − q(x))₊
```

β 完美约掉,这就是为什么分母必须用 β。

**第四步:合并两条路径。**

```
P(输出 = x) = P_A(x) + P_B(x)
            = min(p(x), q(x)) + (p(x) − q(x))₊
```

分两种情况验算这个和:

```
情况 1:p(x) ≥ q(x)
   min(p,q) = q(x),  (p−q)₊ = p(x)−q(x)
   和 = q(x) + p(x) − q(x) = p(x)  ✓

情况 2:p(x) < q(x)
   min(p,q) = p(x),  (p−q)₊ = 0
   和 = p(x) + 0 = p(x)  ✓
```

两种情况都得 P(输出 = x) = p(x)。**证毕。** 输出分布严格等于 p,与 q 完全无关。∎

> 这个证明的精髓:**接受路径捞到的是 min(p,q) 的"两者交集"质量;拒绝后重采样补上 (p−q)₊ 的"p 超出 q"质量。两块拼起来恰好是 p。** q 越接近 p,min(p,q) 越大、α 越高、拒绝越少;但即使 q 乱来,这个加法恒等式照样成立——这是无损性的代数本质。

### 3.3 从单步到 K 步:为什么逐位置裁决也无损

verify 阶段是对 K 个位置同时打分,然后从左到右逐位置应用上述规则。要点:

```
对位置 i(前缀已含 x̃₁..x̃_{i-1},全部已被接受):
   · 大模型给出 pᵢ(·) = p(· | 前缀, x̃₁..x̃_{i-1})
   · 草稿给出 qᵢ(·) = q(· | 前缀, x̃₁..x̃_{i-1}),且 x̃ᵢ ~ qᵢ
   · 用 3.1 规则裁决 x̃ᵢ:接受则进位置 i+1;拒绝则从残差 pᵢ' 采一个 token,本轮终止
```

**关键洞察:第 i 步裁决的前提是 x̃₁..x̃_{i-1} 全部被接受。** 而由 3.2,只要前缀确定,每个被接受的位置吐出的 token 都严格服从该位置的条件分布 pᵢ。因此整条被接受的链 x̃₁..x̃_j 服从 p 的联合条件分布:

```
P(x̃₁..x̃_j | 前缀) = p(x̃₁|前缀)·p(x̃₂|前缀,x̃₁)···p(x̃_j|前缀,x̃₁..x̃_{j-1})
```

这正是自回归从 p 采样 j 个 token 的联合分布。在第一个拒绝位置 j+1,我们从残差 p_{j+1}' 重采一个 token,由 3.2 它服从 p_{j+1}(·),即也是从 p 正确采的下一个 token。**所以每轮产出的 token 链(无论长短),逐 token 都等价于从 p 自回归采样。** 多轮拼接,整个序列分布严格等于纯 p 解码。无损性在序列层面成立。

> ⚠ 实现坑:**第一个拒绝处的 token 必须用残差重采样,绝不能简单地"取大模型在该位置的 argmax/采样"丢掉草稿信息。** 直接从 pᵢ 采会引入双重计数偏差(因为到达位置 i 这个事件本身已经依赖于 x̃ᵢ 被采出来又被拒),破坏无损性。残差 (pᵢ−qᵢ)₊ 正是为了抵消这个条件偏差。这是工业实现里最容易写错、且单测很难发现(分布偏差肉眼看不出)的地方。

### 3.4 贪心(temperature=0)情形

若目标是贪心解码(argmax,等价于 p 退化成 one-hot),规则退化为:**接受当且仅当草稿 token = 大模型 argmax;一旦不等就用大模型 argmax 顶替并终止。** 这个特例很多人凭直觉就会写,它确实是上面规则在 p 为 one-hot 时的极限(此时 min(1, p(x)/q(x)) 对 argmax token 给出接受、对其他 token 接受概率为 0,残差分布退化为 one-hot 于 argmax)。但**带温度/top-p 采样时必须用完整的 accept/resample 规则**,不能图省事。

## 四、加速分析:期望接受长度与净赚条件

无损只是"不变差",还得算"快多少"。

### 4.1 期望每轮接受 token 数

把整个序列上的接受行为简化成一个常数:设每个草稿 token 被接受的概率为 α(称 **acceptance rate**;严格来说它逐位置变化,这里取其平均做一阶分析)。一轮里草稿生成 K 个候选,从左到右验证,**第一次拒绝就停**。设 n 为本轮被接受的草稿 token 数(0 ≤ n ≤ K)。

n 是"在 K 次伯努利试验中,首次失败前的成功数,且最多数到 K"(几何分布的截断)。其分布:

```
P(n = i) = α^i · (1−α)        对 i = 0,1,...,K−1   (前 i 个接受,第 i+1 个拒绝)
P(n = K) = α^K                                       (K 个全接受,没有拒绝)
```

期望被接受的草稿数:

```
E[n] = ∑_{i=0}^{K-1} i·α^i·(1−α) + K·α^K
```

这个和有闭式。先用一个标准技巧:对截断几何,期望接受数等于"在每个位置都还没失败的概率之和"(类似 E[X] = ∑ P(X ≥ k)):

```
位置 i (i=1..K) 被接受,要求前 i 个草稿都被接受 → 概率 α^i
E[n] = ∑_{i=1}^{K} α^i = α·(1 − α^K) / (1 − α)        (等比求和,α<1)     (◆)
```

(可验算:K→∞ 时 E[n] → α/(1−α),即无截断几何分布的均值,符合预期。)

**每轮实际产出的新 token 数 = E[n] + 1**:被接受的 n 个,加上"第一个拒绝处重采样的 1 个"(或全接受时第 K+1 个免费 token)。无论哪种结尾,每轮稳赚 1 个由大模型保证的 token。所以**期望每轮产出:**

```
E[tokens/轮] = E[n] + 1 = 1 + α·(1 − α^K)/(1 − α)             (◆◆)
```

代几个数感受一下(取 K=4):

```
α=0.9: E[n]=0.9(1−0.9⁴)/0.1=0.9·(1−0.6561)/0.1=0.9·3.439=3.095 → 每轮 ≈ 4.10 token
α=0.7: E[n]=0.7(1−0.7⁴)/0.3=0.7·(1−0.2401)/0.3=0.7·2.533=1.773 → 每轮 ≈ 2.77 token
α=0.5: E[n]=0.5(1−0.5⁴)/0.5=1·(1−0.0625)=0.9375     → 每轮 ≈ 1.94 token
α=0.3: E[n]=0.3(1−0.3⁴)/0.7≈0.3·1.417=0.425          → 每轮 ≈ 1.42 token
```

**α 对加速的影响是高度非线性的:从 0.7 提到 0.9,每轮产出从 2.77 跳到 4.10。** 这解释了为什么 draft 模型质量(决定 α)如此关键。

### 4.2 净赚条件:把成本算进去

每轮的墙钟成本(取单序列、batch=1):

```
成本/轮 ≈ K · c_draft + 1 · c_target
   c_draft  = 小模型一次前向耗时
   c_target = 大模型一次前向耗时(verify 这次,memory-bound,≈ 大模型出 1 个 token 的钱)
```

记成本比 c = c_draft / c_target(小模型比大模型快多少的倒数,通常 c ≪ 1,如 1/10~1/30)。基线(纯自回归大模型)出 1 个 token 成本 = c_target。投机解码出 E[tokens/轮] 个 token 花了 (K·c + 1)·c_target。**加速比:**

```
            E[tokens/轮]            1 + α(1−α^K)/(1−α)
speedup ≈ ───────────────────── = ──────────────────────       (◆◆◆)
           成本/轮 / c_target            1 + K·c
```

**净赚(speedup > 1)的条件:** 分子 > 分母,即 E[n] > K·c,也就是:

```
α(1−α^K)/(1−α) > K · c
```

直觉解读:**草稿每多生成一个 token 要花 c(相对单位),换来期望 E[n]/K 的接受率收益。当 α 足够高、c 足够小,就净赚;当 α 太低(草稿太烂)或 c 太大(草稿不够快),投机解码反而更慢。** 这是工程中真实会发生的"负优化":拿个不够小或不够准的草稿模型,加速比掉到 1 以下。

### 4.3 最优 K 的存在性

固定 α 和 c,speedup 关于 K 不是单调的:

```
· K 太小:分子 1+α(1−α^K)/(1−α) 还没涨够(没充分利用并行验证),浪费潜力
· K 太大:分子趋于饱和上限 1 + α/(1−α)(再大 α^K→0,接受不动了),
          但分母 1+K·c 持续线性增长 → speedup 反而下降
```

所以存在一个最优 K*。对 (◆◆◆) 求关于 K 的导数令其为 0 可解(α^K 项使其无简洁闭式,实践中直接网格搜 K=2..8 即可)。**经验法则:α 越高,饱和来得越慢,最优 K 越大;c 越大,惩罚越重,最优 K 越小。** 工业系统(如 vLLM 的 speculative 配置)通常把 K(常叫 num_speculative_tokens)设在 3~5。具体最优值待核于实际模型对。

## 五、代码:draft-then-verify 主循环

下面是单序列、带温度采样的完整投机解码主循环。重点是 verify 后的逐位置 accept/resample,以及残差重采样,这两处是机制核心。省略了 KV cache 管理和 batching 的工程细节,但算法逻辑完整可照实现。

```python
import torch

def sample_from(probs):
    # 从一个一维概率分布(已归一化)采一个 token id
    return torch.multinomial(probs, num_samples=1).item()

def residual_dist(p, q):
    # 残差分布 p'(x) = (p(x)-q(x))_+ / sum(...);用于拒绝后的重采样
    diff = torch.clamp(p - q, min=0.0)        # (p - q)_+,逐元素取正部
    s = diff.sum()
    if s <= 1e-12:                            # 数值兜底:残差几乎为 0
        return p                              # 退化时直接用 p(理论上 s == β > 0)
    return diff / s                           # 分母 sum 恰等于拒绝概率 β,见 (★)

@torch.no_grad()
def speculative_step(prefix_ids, draft_model, target_model, K, temperature):
    """执行一轮投机解码,返回本轮新增的 token 列表(长度 1..K+1)。"""
    # ---- 1. draft 阶段:小模型自回归生成 K 个候选,记下每步草稿分布 q_i ----
    draft_tokens, q_dists = [], []
    cur = prefix_ids
    for _ in range(K):
        logits = draft_model(cur)[:, -1, :]           # 小模型一次前向,取最后位置
        q = torch.softmax(logits / temperature, dim=-1).squeeze(0)
        x = sample_from(q)                            # x_i ~ q_i
        q_dists.append(q)
        draft_tokens.append(x)
        cur = torch.cat([cur, torch.tensor([[x]])], dim=1)

    # ---- 2. verify 阶段:大模型对 [prefix + K 草稿] 做"一次"前向,并行得 K+1 个分布 ----
    full = torch.cat([prefix_ids, torch.tensor([draft_tokens])], dim=1)
    logits = target_model(full)                       # 形状 [1, L+K, V];这是唯一一次大模型前向
    # 取出对应每个草稿位置之前的那个分布:p_i 用于裁决草稿 token x̃_i
    # 位置对齐:prefix 末尾位置的输出预测 x̃_1,以此类推
    start = prefix_ids.shape[1] - 1
    p_dists = [torch.softmax(logits[:, start + i, :].squeeze(0) / temperature, dim=-1)
               for i in range(K + 1)]                 # K+1 个目标分布

    # ---- 3. accept/resample:从左到右逐位置裁决 ----
    accepted = []
    for i in range(K):
        x = draft_tokens[i]
        p_i, q_i = p_dists[i], q_dists[i]
        r = torch.rand(1).item()
        if r <= min(1.0, (p_i[x] / q_i[x]).item()):   # 接受概率 min(1, p(x)/q(x))
            accepted.append(x)                        # 接受,继续验证下一个
        else:
            # 拒绝:从残差分布重采一个 token 顶替,本轮结束 —— 这一步绝不能省/简化
            x_new = sample_from(residual_dist(p_i, q_i))
            accepted.append(x_new)
            return accepted                           # 长度 = i+1 (前 i 个接受 + 1 个重采)
    # ---- 全部 K 个都接受:用第 K+1 个位置的 p 免费再采一个 (bonus token) ----
    accepted.append(sample_from(p_dists[K]))
    return accepted                                   # 长度 = K+1
```

要点复盘:

- `q_dists` 必须存下来,因为 accept 测试要算 p(x)/q(x),需要草稿当时的 q。
- 大模型只在 verify 处前向**一次**(`target_model(full)`),这是省钱的关键——`logits` 一次性给出 K+1 个位置的分布。
- 拒绝分支用 `residual_dist`,**绝不能写成 `sample_from(p_i)`**(回顾 3.3 的坑)。
- `accepted` 长度天然在 1..K+1 之间,对应 (◆◆) 的产出。
- batch、KV cache 复用、把被拒绝草稿对应的 KV 回滚(rollback)——这些是工程量,但不改变算法正确性。

## 六、免独立草稿模型:Medusa 与 EAGLE

3 节证明里 q 是任意的——这给了巨大设计自由:**只要能廉价地造出 K 个候选,无损性都成立。** 业界由此发展出"自草稿(self-drafting)"路线,免去单独训练/部署一个草稿模型。

### 6.1 Medusa:给大模型加多个预测头

**Medusa 的核心思想:在大模型最后一层 hidden state 上,额外接几个轻量预测头(MLP head),第 k 个头直接预测"未来第 k 个 token"。** 这样大模型一次前向就同时吐出位置 t+1、t+2、…、t+K 的候选,不需要独立草稿模型。

```
标准模型: hidden_t ──(LM head)──→ p(x_{t+1})           只预测下一个
Medusa:   hidden_t ──┬─(head 0)→ 候选 x_{t+1}
                     ├─(head 1)→ 候选 x_{t+2}            一次前向出多个未来位置
                     └─(head 2)→ 候选 x_{t+3}
```

但有个问题:这些头是**并行独立预测**的(head 1 不知道 head 0 实际选了哪个),候选之间不连贯,单条候选链接受率不高。Medusa 的应对是 **tree attention(树状候选)**:每个头给出 top-k 个候选,组合成一棵候选树(而非一条链),用一次大模型前向 + 特制的树状 attention mask **并行验证整棵树**,沿树选出最长被接受路径。无损性靠的还是 3 节那套 accept/resample,只是从"一条链"推广到"树上的多条候选路径"同时验证。

代价:多个头要**额外训练**(冻结主干、只训 head,成本低但非零);tree attention 的 mask 构造和 KV 索引是工程难点;头越多、树越大,一次 verify 的算力开销越接近脊点,memory-bound 红利会被吃掉。

### 6.2 EAGLE:在 feature 层做自回归草稿

**EAGLE 的洞察:Medusa 各头独立预测导致不连贯;真正的连贯草稿应该是自回归的,但可以在"特征(feature)空间"而非 token 空间做,且复用大模型自己的表示。** EAGLE 用一个轻量自回归头,在大模型倒数第二层 feature 序列上做预测,且把"上一步真实采样的 token embedding"喂回去,从而生成连贯的草稿链——本质是个极小的、与大模型共享词表/表示的草稿器。配合树状验证,EAGLE 系列在公开报告里把接受长度和加速比推到了比 Medusa 更高的水平(具体数字待核)。

> 三条路线的取舍一句话:**独立草稿模型**(Leviathan/Chen 原始版)灵活、零侵入大模型,但要部署两个模型且草稿与目标可能"性格不合"导致 α 低;**Medusa** 零额外模型、改动小,但并行头不连贯、靠树补救;**EAGLE** 连贯草稿、α 最高,但需要训练专门的 feature 级草稿器、实现最复杂。三者的无损性都建立在同一个 3 节定理上——这是它们能共享"无损"招牌的根本原因。

### 6.3 与其它系统组件的相互作用(常被忽视)

- **和 continuous batching 冲突:** 投机解码的红利来自单序列 memory-bound 时算力闲置。**当 batch 很大、大模型本来就接近 compute-bound(算力被多个序列填满)时,"多算 K 个 token"不再免费**,加速比急剧下降甚至 < 1。所以高吞吐 serving(大 batch)和低延迟 serving(小 batch + 投机)是两种不同制度,投机解码主要利好后者。这是它在生产里最大的适用边界。
- **和 PagedAttention/KV cache 的交互:** 被拒绝的草稿 token 占用的 KV 槽位要能回滚释放(见上一章的 block 管理),否则显存泄漏。树状验证还要求 KV 索引支持非连续/分叉访问。

## 七、设计权衡与常见坑(汇总)

**权衡:**
- **草稿模型选型** = 在 α(越准越大)和 c(越快越小)之间找平衡。极端 1:把目标模型自己当草稿(α=1 但 c=1)→ 毫无收益。极端 2:用一个随机模型(c≈0 但 α≈0)→ 同样无收益。最优在中间:一个"小一两个数量级、但和目标同源/同分布"的模型,常见做法是同系列的小尺寸版本(如大模型 70B 配草稿 7B 或更小)。
- **K 的选择** = 见 4.3,存在最优 K*,过大反被分母 1+K·c 拖累。
- **链 vs 树** = 树用更多算力换更高接受长度,在 memory-bound 富余时划算,batch 大时变负担。

**高频坑:**
1. **拒绝处直接从 p 采样而非残差** → 破坏无损性,且单测发现不了(3.3 的坑,最严重)。
2. **忘了 bonus token** → 全接受时不多采那 1 个,白白损失约 1/(K+1) 的吞吐。
3. **草稿模型和目标模型 tokenizer/词表不一致** → p(x)/q(x) 根本对不齐,直接错。两个模型必须共享词表。
4. **大 batch 下盲目开投机** → 目标已 compute-bound,加速比 <1,负优化。上线前务必按 batch size 分制度测。
5. **温度不一致** → draft 和 target 用不同温度,会系统性压低 α(分布故意被你拉开了)。一般两者用同一采样配置。
6. **被拒草稿的 KV 没回滚** → 显存随生成增长泄漏。
7. **用 α 的全局均值估加速却忽略其随上下文剧烈波动** → 代码段、重复串 α 极高,自由创作 α 低;实测要看分布不是看均值。

## 十、小结与承上启下

这一章的两个支柱:**(1) 物理层** —— decode 单序列 memory-bound(算术强度 ≈1 ≪ 脊点),所以大模型一次前向算 1 个 vs 算 K 个 token 墙钟几乎相同,闲置算力让"并行验证"近乎免费;**(2) 数学层** —— accept(概率 min(1,p/q))+ reject-resample(从 (p−q)₊ 归一化)的组合,使输出边缘分布逐 token 严格等于 p,无损性是代数恒等式而非近似。把这两层钉死,你就能解释清楚"为什么并行验证能无损加速",并算清它在什么 α、c、K、batch 制度下净赚。

在整门课里的位置:上一章(KV Cache 与 PagedAttention)解决的是 decode 的**显存**瓶颈(把 KV 当虚拟内存管),本章解决的是 decode 的**延迟/带宽**瓶颈(用并行验证摊薄读权重成本)——两者都根植于"decode 是 memory-bound"这一共同前提,互补而非替代。下一章(量化:GPTQ/AWQ/SmoothQuant)会从第三个角度打这个瓶颈:**直接把权重做小**(FP16→INT4),让"读一遍权重"的字节数本身减少数倍,从根上抬高算术强度。**KV 分页(省显存)、投机解码(摊薄前向)、量化(缩小权重)——这三章合起来,就是现代低延迟推理引擎的三件套。** 学完它们,你看 vLLM/TensorRT-LLM 这类系统时,就能从第一性原理拆解它们每个优化各打的是哪个瓶颈、彼此怎么叠加。
