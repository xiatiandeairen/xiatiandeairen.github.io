> 这一章把你从"模型是个黑箱,我只能看输入输出和 loss 曲线"带到"能逆向工程出网络内部的具体算法(电路)、能从第一性原理推出诱导头为何是 in-context learning 的雏形、能用稀疏自编码器把纠缠的激活分解成单义特征、能从似然优化的目标函数里推出幻觉的数学根源、并能讲清 Mamba/SSM 这类后注意力架构在用什么数学绕开 O(N²)"。学完你不再把 Transformer 当一团权重,而是当一个可以被拆解、被定位、被验证的计算图——以及能判断哪些"下一代架构"是真突破、哪些是换皮。

## 一句话引言

可解释性的核心赌注是:**训练出来的网络里,藏着人类可理解的算法**;不是一团不可名状的权重,而是若干个各司其职的"电路"叠在一起。本章先把这个赌注的证据(诱导头、KV 记忆、SAE 分解的单义特征)逐个推给你看,再用同一套"网络在做什么计算"的视角去解剖幻觉的成因,最后看向那些试图替换或补强注意力的下一代架构。机制优先,代码点到为止。

## 一、为什么这一章绕不过去:从"它能做"到"它怎么做"

前面十四章你已经能搭一个 Transformer、能算它的复杂度和显存、能做预训练和微调、能从 scaling law 预测它的损失。但有一个问题你始终没法回答:**当模型答对一道题时,它内部到底执行了什么计算?** 当它答错、或者一本正经地编造一个不存在的事实时,**是哪一步出了问题?**

这不是哲学问题,而是三个非常实际的工程需求:

- **安全与对齐**:如果你不知道模型内部有没有一个"在特定触发词下输出有害内容"的电路,你就无法保证它不会在部署后被激活。行为测试(behavioral testing)只能覆盖你想到的输入;机制理解才能覆盖你没想到的。
- **调试与改进**:模型有个系统性错误(比如总是把某两个实体搞混),你想定位是哪几层、哪几个注意力头负责。没有机制视角,你只能盲调超参或加数据,治标不治本。
- **幻觉与可控**:模型为什么会自信地胡说?这是采样的锅、训练目标的锅、还是知识存储的锅?不同的根因对应完全不同的缓解手段。搞错根因,缓解就是无用功。

这一章分两大块。**机理可解释性(mechanistic interpretability,简称 mech interp)**:把网络当成一个待逆向工程的程序,目标是还原出它内部的"源代码"——具体是哪些注意力头、哪些 MLP 神经元、按什么顺序组成了完成某个功能的子图(电路)。**架构前沿**:Transformer 不是终点,有一批工作在试图用状态空间模型、线性注意力、统一多模态等思路替换或补强它;我们要看清这些思路在数学上到底改了什么、代价是什么。

**贯穿全章的判断标准**:任何一个可解释性发现,问两件事——(a) 它是不是**因果**的(把这个头/特征消融掉,行为是否如预测般改变),还是仅仅相关;(b) 它能不能**外推**(在更大模型、更多任务上仍成立,还是过拟合了几个 cherry-picked 例子)。任何一个新架构,也问两件事——它**省下了什么复杂度**、又**牺牲了什么能力**。

预备:本章重度依赖第 02、03 章(注意力、Transformer block 的残差结构)和第 01 章(嵌入与叠加)。第 01 章末尾我承诺过"叠加的完整处理留到第 15 章",这一笔账在第四节还上。

## 二、地基:残差流是一条"信息高速公路"

要逆向工程 Transformer,先要换一个看它的视角。常规视角是"一层一层往上算";mech interp 的视角是**残差流(residual stream)**——这是整个领域的几何基础,务必内化。

### 问题:每个组件的输出去了哪里

回忆 Transformer block 的前向(第 03 章):

```
x ← x + Attn(LN(x))      # 注意力的输出加回 x
x ← x + MLP(LN(x))       # MLP 的输出加回 x
```

注意这两行的结构:**每个子层的输出都是"加"回 x,而不是"替换" x**。这意味着,从第 0 层(token + 位置嵌入)到最后一层,有一条贯穿始终、维度恒为 d_model 的向量 x,所有组件都在**往上面读、往上面写**,但谁也不会清空它。Anthropic 的 transformer-circuits 团队把它命名为 residual stream。

把它展开,最后一层某个位置的残差流是:

```
x_final = x_embed                              # 初始输入(token 嵌入 + 位置)
        + Σ_layers Σ_heads  head_output         # 所有注意力头写入的总和
        + Σ_layers          mlp_output          # 所有 MLP 写入的总和
```

这是一个**精确的线性分解**(因为残差是加法,没有任何近似)。每一个组件的贡献都是这个总和里的一个加数。

### 推导:为什么这个视角让"逆向工程"成为可能

残差流视角给了三个极其有用的结论,它们是后面一切电路分析的支点。

**结论一:残差流是组件间的通信总线。** 由于人人都往同一个 d_model 维向量里读写,**一个早层组件的输出,可以被任意一个晚层组件读到**——只要晚层组件的输入投影矩阵在那个方向上有非零分量。换句话说,层与层之间不是只能"相邻传话",而是可以"隔层直达"。这就是为什么会存在跨多层协作的电路。

**结论二:读写都是低秩投影,所以是"在子空间里操作"。** 每个注意力头的输出投影 W_O 把 head 的 d_head 维结果映射回 d_model;每个 MLP 也是先升维再降维写回。因为 d_head ≪ d_model(比如 64 vs 4096),**每个头只能往残差流的一个最多 d_head 维的子空间里写**。可以把残差流想象成一块共享白板,每个头只用其中一小片区域。不同的头如果用的子空间近似正交,它们就互不干扰、并行工作。

**结论三:可以把任意两个组件"接起来"读。** 这是 mech interp 最锋利的工具,叫**逻辑直读(logit lens)/ 直接路径分析**。考虑"某个注意力头 H 的输出,直接(不经过后续任何非线性)对最终预测某个 token 的 logit 贡献了多少"。因为从 H 的输出到 logit,如果走的是纯线性路径(H 写入残差流 → 残差流经 unembedding 矩阵 W_U 算 logit),这条路径是:

```
H 对 token t 的 logit 直接贡献 = (head_output_H) · W_U[:, t]
```

其中 W_U[:, t] 是 unembedding 矩阵中对应 token t 的那一列(d_model 维)。这是一个**可以直接算出来的内积**,不需要训练、不需要梯度。它告诉你"这个头是在把概率往哪个 token 上推"。把这个工具用熟,你就有了一台显微镜。

### 代码:用 logit lens 看每一层在"想"什么

```python
# 思路:取出每一层之后的残差流,直接用 unembedding 算 logit,
# 看模型在中间层就"想到"了哪个词。机制:残差流加法 → 任意中间态都可投影到词表。
def logit_lens(model, tokens):
    x = model.embed(tokens)                 # [seq, d_model]
    layerwise_top = []
    for layer in model.layers:
        x = layer(x)                        # 残差流被这一层读写后的状态
        normed = model.final_ln(x)          # 借用最终 LN,把中间态摆到 unembed 期望的尺度
        logits = normed @ model.W_U         # [seq, vocab]  直接投影到词表
        layerwise_top.append(logits[-1].argmax(-1))   # 最后位置预测的 token
    return layerwise_top   # 常能看到:答案在中间某层就已浮现,后续层只是放大/微调
```

> 这个实验的经典观察:对很多任务,**正确答案在中间层就已经在残差流里"领先"了**,后面的层只是不断加大它的 logit。这说明计算不是均匀分布在所有层,而是有明确的"分工"——某些层负责"想出答案",某些层负责"把答案写清楚"。这是电路存在的间接证据。

### 分析:这个视角带来的两个量化直觉

- **写入容量是受限的**。L 层、每层 H 个头,共 L·H 个头,每个头往一个 ≤ d_head 维子空间写。L·H·d_head 通常远大于 d_model(例如 32 层 × 32 头 × 128 = 131072 ≫ 4096),所以**各头写入的子空间必然大量重叠**,不可能两两正交。这直接预告了"叠加"(第四节):连组件级别都装不下,神经元级别更装不下。
- **直接路径只是冰山一角**。logit lens 只算了线性直达路径。真正的计算大量发生在"头 A 的输出被头 B 当作 query 读入"这种**组合路径(composition)**上,后者是非线性的、数量随层数组合爆炸。诱导头(下一节)正是一个两步组合电路。

## 三、第一个被完整逆向的算法:诱导头与 in-context learning 的雏形

这是 mech interp 最漂亮、最有说服力的成果。我们要把它推到你能自己复现验证的程度。

### 问题:模型凭什么能"现学现用"

In-context learning(ICL,上下文学习)是大模型最神奇的能力之一:你在 prompt 里给几个例子,模型不更新任何权重,就能照着模式做新题。最简单的 ICL 形态是**复制模式补全**:序列里出现过 `...[A][B]...`,后面又出现 `[A]`,模型倾向于预测 `[B]`。比如上文有 "Mr. Dursley",后文再出现 "Mr.",模型就接 "Dursley"。

问题是:**注意力机制在结构上怎么实现"找到上次 A 后面跟的是什么,然后复制过来"?** 这需要两个能力:(1) 定位——找到上一次出现 A 的位置;(2) 复制——把那个位置的下一个 token 搬到当前输出。单个注意力头做不到这件事(它只能"看"不能"组合两步"),那是怎么实现的?

### 推导:两步组合电路

答案是 Anthropic(Elhage et al. 2021;Olsson et al. 2022)发现的**诱导头(induction head)**,它是一个**跨两层、由两个头协作**的电路。我们按 token 的视角推一遍。设当前要预测的位置是 t,当前 token 是 A(它在更早的位置 s 也出现过,且 s 的下一个 token,即位置 s+1,是 B)。

**第一步:前一 token 头(previous-token head),在较早的层。** 这个头的功能极简单:**每个位置都去注意它的前一个位置,把前一个 token 的信息抄到当前位置的残差流里**。于是在位置 s+1(token B)处,残差流里现在不仅有"我是 B",还附带了一份"我前面是 A"的拷贝。

```
位置 s+1 的残差流(经过 previous-token head 后)≈  [ 我是 B ] + [ 我的前一个是 A ]
```

这一步的注意力模式是固定的"看前一格",和内容无关(可以靠位置编码实现:query 在位置 p,key 在位置 p−1 时打分最高)。

**第二步:诱导头本身(induction head),在较晚的层。** 关键在它怎么构造 query 和 key:

- 它在当前位置 t(token A)发出的 **query**,编码的是"我是 A,我要找:谁的'前一个 token'是 A"。
- 它读取的 **key**,正是第一步那个头写进去的"我的前一个是 X"信息。

于是位置 t 的 query("找前一个是 A 的")和位置 s+1 的 key("我前一个是 A")**高度匹配**——注意力被吸引到位置 s+1。然后这个头的 value 把位置 s+1 的 token 身份(B)搬出来,写进位置 t 的残差流,推高 logit(B)。预测完成。

```
诱导头的查找逻辑(prefix matching):
  query@t  : "find the position whose previous-token == A"   (A = 当前token)
  key@s+1  : "my previous-token == A"                        (第一步头写入的)
  match!  → attend to s+1 → copy its token (B) → boost logit(B)
```

把两步连起来,整条电路实现的算法是:**找到当前 token 上一次出现的地方,输出它后面跟的那个 token**。这正是"复制上文模式"。注意它**完全不依赖具体内容**——A、B 是什么无所谓,它学的是"重复结构"这个抽象规则。这就是它被认为是 ICL 雏形的原因:一个不靠权重更新、纯靠上下文模式做泛化的机制。

### 机制为什么成立:Q/K 组合(K-composition)

诱导头能工作,数学上靠的是**组合(composition)**:第二个头的 key 投影 W_K 读的不是原始 token 嵌入,而是**第一个头写进残差流的那个子空间**。用残差流语言说,前一-token 头往残差流的某个方向写"前驱身份",诱导头的 W_K 恰好在那个方向上有投影,于是读到了它。这叫 **K-composition**(K 投影组合了上游头的输出)。

可以用 QK 电路的"虚拟权重"刻画(第 02/03 章里 Attn 打分是 `qᵀk = xᵀ W_Qᵀ W_K x`)。把 W_Qᵀ W_K 合成一个 d_model × d_model 的矩阵,称为该头的 **QK 电路矩阵 W_QK**。诱导头之所以是诱导头,就在于它的 W_QK 把"当前 token 嵌入"方向和"上游头写入的前驱方向"对齐——这是个可以从权重里直接量出来、可被消融验证的结构。

### 代码:三行验证一个头是不是诱导头

不需要训练,直接拿现成模型测它在**重复随机序列**上的行为。这是该领域的标准探针。

```python
# 机制探针:喂一段随机 token,再把它原样拼接一遍 -> [r1 r2 ... rN r1 r2 ... rN]
# 真正的诱导头会在第二遍出现 rk 时,把注意力强烈投向第一遍 rk 的【下一个】位置 (rk+1)
import torch

def induction_score(model, head, seq_len=50):
    rand = torch.randint(0, model.vocab, (1, seq_len))
    tokens = torch.cat([rand, rand], dim=1)          # 重复序列
    _, attn = model(tokens, return_attn=True)         # attn[layer][batch, head, q, k]
    L, H = head
    A = attn[L][0, H]                                 # [2*seq_len, 2*seq_len]
    # 第二遍位置 i (i>=seq_len) 应当强烈注意第一遍的 (i - seq_len + 1) 位
    diag = torch.tensor([A[seq_len + i, i + 1] for i in range(seq_len - 1)])
    return diag.mean()                                # 越接近 1,越是标准诱导头
```

> Olsson et al.(2022)的核心实证:**诱导头的形成,与模型 ICL 能力的涌现,在训练过程中同时发生**——loss 曲线上有一个明显的"小台阶/拐点",诱导头恰在那一刻成型,ICL 分数同步跃升。这是"某个具体电路 ⟷ 某个宏观能力"少有的强因果证据(他们还做了消融:破坏诱导头,ICL 显著退化)。这是把 emergent ability(第 08 章)和具体机制对上号的标志性工作。

### 分析:它的边界

诱导头只是 ICL 的**最底层骨架**,不是全部。真实的 few-shot ICL(给"问题→答案"对、做情感分类等)远比"复制上一次"复杂,涉及更多头的协作、对抽象类别的匹配。但诱导头给了一个**存在性证明**:复杂能力可以由可定位、可验证的电路实现。这把"模型是不可理解的黑箱"这个默认假设,撕开了一道口子。

## 四、叠加:为什么"一个神经元一个概念"是错的

这是理解可解释性最大障碍的关键概念,也是第 01 章欠下的账。

### 问题:概念比神经元多得多,怎么装下

朴素的期望是**单义性(monosemanticity)**:网络里每个神经元(或每个维度)对应一个人类可理解的概念,你激活它就是"这个概念出现了"。如果成立,可解释性就是查字典。

现实是**绝大多数神经元是多义的(polysemantic)**:同一个神经元会对一堆看似无关的东西激活——比如同时对"猫的图片""学术引用格式""某种 HTML 标签"响应。为什么?一个根本性的数量矛盾:**模型要表示的概念/特征数量,远超它的神经元数量。** 语言里有效特征数轻松上百万,而一层的神经元只有几千到几万。鸽巢原理:装不下,必须共用。

### 推导:叠加的几何——用 m 维表示 n ≫ m 个特征

把"特征"理想化为:网络想表示 n 个特征,每个特征在激活空间里对应一个方向。如果要求这些方向两两正交(互不干扰),那么一个 m 维空间最多容纳 **m** 个正交方向——所以严格无干扰最多 m 个特征。但模型可以**放宽正交性**:允许特征方向之间有小角度(近似正交),用 m 维塞进 n ≫ m 个方向。这就是**叠加(superposition)**——多个特征叠加共享同一组神经元。

它凭什么可行?靠两个条件,缺一不可:

**条件一:特征是稀疏的(sparsity)。** 任意一个输入里,绝大多数特征是"不出现"的(取值 0)。"猫"和"学术引用"几乎不会在同一段文本里同时出现。既然很少同时激活,它们方向上的那点重叠就**几乎不会同时被用到**,干扰被稀疏性"摊薄"了。

**条件二:有非线性来去噪。** 近似正交方向会带来串扰(读特征 i 时,特征 j 的方向贡献一点噪声)。激活函数(ReLU/GELU)的阈值效应能把小噪声压回 0,把真正激活的大值留下。**非线性是叠加能去噪、从而可行的关键**——纯线性网络没法干净地解叠加。

可以做个最小化定量论证。设 n 个特征方向是 m 维空间里的单位向量,两两点积(干扰)约为 ±1/√m 量级(高维随机向量近似正交的标准结果)。读取特征 i 时,来自其他 (n−1) 个特征的串扰方差约为:

```
干扰能量 ≈ (n−1) · p · (1/√m)² = (n−1) · p / m
其中 p 是单个特征被激活的概率(稀疏 → p 很小)
```

只要 (n−1)·p ≪ m,串扰就小于信号,叠加可行。**稀疏(p 小)直接决定了能塞多少特征**:p 越小,n 能比 m 大越多。这就是叠加的"容量公式"骨架(Anthropic "Toy Models of Superposition", Elhage et al. 2022 用一个玩具模型把这件事系统验证了:随着稀疏度增加,模型从"只存最重要的几个特征、一特征一维"切换到"用叠加塞进更多特征",甚至出现特征排布成正多边形等几何结构)。

### 为什么叠加让"一神经元一概念"彻底失效(本节核心)

直接后果:**单个神经元的激活方向 ≠ 单个特征的方向**。因为 n 个特征方向不与 m 个坐标轴对齐(它们是斜的、密集排布的),**任意一个坐标轴(神经元)都会同时落在好几个特征方向上**——所以读单个神经元,读到的是好几个特征的混合。这就是多义性的几何来源。指着第 37 号神经元问"它代表什么"是**问错了问题**:概念不住在神经元里,而住在神经元的**线性组合(方向)**里,而且这些方向比神经元多。

> 一句话锁死:**叠加 = 特征数 > 神经元数,靠稀疏性 + 非线性,把不正交的特征方向压进低维空间。** 它的直接代价就是神经元失去单义性,你不能再逐个神经元解读。第 01 章说的"别把嵌入维度可解释当真",根源就在这里。

### 解法:稀疏自编码器(SAE)把叠加拆开

既然概念住在"方向"里、且比神经元多,自然的想法是:**学一个更高维、被强制稀疏的表示,把纠缠的激活重新分解成一堆单义特征**。这就是**稀疏自编码器(Sparse Autoencoder, SAE)**,当前 mech interp 最主力的工具(Anthropic "Towards Monosemanticity" 2023;后续 scaling 工作 2024)。

机制:取模型某一层的激活 x ∈ ℝ^(d_model),学一个**过完备(overcomplete)**字典——隐层维度 d_sae ≫ d_model(比如 8 倍、32 倍),配 L1 稀疏惩罚:

```
编码:  f = ReLU(W_enc · x + b_enc)        # f ∈ ℝ^(d_sae),d_sae ≫ d_model,被逼稀疏
解码:  x̂ = W_dec · f + b_dec              # 重建回原激活
损失:  L = ‖x − x̂‖²        +    λ · ‖f‖₁
            └ 重建项(别丢信息)      └ 稀疏项(逼 f 大部分为 0)
```

直觉:**叠加是"把多个特征压进少数神经元",SAE 是反操作——把少数神经元的激活展开回多个特征。** 因为维度被放大且被逼稀疏,每个被激活的隐藏单元 fᵢ 倾向于对应**单一**人类可理解的概念(单义特征):有的对应"金门大桥"、有的对应"DNA 序列"、有的对应"代码里的 bug"。L1 项是关键:它逼网络"用尽量少的特征解释每个输入",从而把混在一起的概念拆开。

为什么 L1 能逼稀疏?L1 范数 ‖f‖₁ = Σ|fᵢ| 的次梯度在 0 处是常数(±1),这给每个非零分量一个**固定的下拉力**,与该分量大小无关——所以小分量会被直接压到 0(对比 L2 的 2fᵢ 下拉力随分量缩小而消失,压不到 0)。这是 L1 产生稀疏解的标准原理(同 LASSO)。

```python
# SAE 训练核心:重建 + L1。机制——过完备 + 稀疏,把叠加的激活解成单义特征。
def sae_loss(x, W_enc, b_enc, W_dec, b_dec, lam):
    f = torch.relu(x @ W_enc.T + b_enc)        # [batch, d_sae],d_sae >> d_model
    x_hat = f @ W_dec.T + b_dec                # 重建
    recon = ((x - x_hat) ** 2).sum(-1).mean()
    sparse = f.abs().sum(-1).mean()            # L1:逼绝大多数特征为 0
    return recon + lam * sparse                # λ 权衡:大 -> 更稀疏更可解释但重建变差
```

**验证手段(回到判断标准 a:因果)**:找到一个 SAE 特征后,要证它真有意义,就**钳制(clamp)它**——人为把 fᵢ 设成很大值再解码回去、继续前向,看模型行为是否如预测般改变。Anthropic 的著名 demo:把"金门大桥"特征钳到最大,模型开始在任何话题里都谈论金门大桥("Golden Gate Claude")。这是从相关到因果的关键一跃。

### 分析:SAE 的代价与未竟之处

- **重建-稀疏-可解释的三角权衡**:λ 调大,特征更稀疏更单义,但重建误差变大(丢了信息,意味着你解出的特征没覆盖原激活的全部内容)。这个张力没有免费午餐。
- **特征数量爆炸**:把一个前沿模型的所有层都 SAE 化,会得到**数千万级**的特征(待核,具体量级随 d_sae 和层数),人工标注根本看不过来,需要用大模型自动给特征命名,又引入新的不可靠性。
- **dead features(死特征)** 与重建漏失:训练中部分隐藏单元从不激活(浪费容量);且 SAE 重建总有残差,残差里可能还藏着没解出的结构。这些都是 2024 起的活跃研究点(如门控 SAE、JumpReLU、TopK-SAE 等改进,待核细节)。

## 五、知识存在哪里:MLP 是一组键值记忆

注意力管"在序列里搬运信息",那**事实性知识**("巴黎是法国首都")存在哪?证据指向 **MLP 层(前馈层 FFN)是一组键值记忆(key-value memory)**。

### 问题:FFN 占了 2/3 参数,它在干嘛

回忆 Transformer block 里 FFN 的参数量(第 03/05 章):两个矩阵 W_in(d → 4d)、W_out(4d → d),共 2·d·4d = 8d² 参数,而注意力的 QKVO 共约 4d²。**FFN 占了每个 block 约 2/3 的参数**。这么大的容量在存什么?长期以来它被当成"逐位置的非线性变换",语焉不详。

### 推导:把 FFN 写成"键检索 + 值读出"

Geva et al.(2021)的洞察:把 FFN 重写一下就能看出它是记忆。FFN 前向是(略去偏置):

```
FFN(x) = W_out · σ(W_in · x)          σ 是激活函数(ReLU/GELU)
```

把 W_in 的每一行看成一个**键向量 kᵢ**(共 4d 个),W_out 的每一列看成一个**值向量 vᵢ**(共 4d 个)。展开:

```
中间激活第 i 维:   mᵢ = σ(kᵢ · x)            # 输入 x 与第 i 个键的匹配度(过激活)
FFN 输出:          FFN(x) = Σᵢ mᵢ · vᵢ        # 用匹配度加权,把对应的值向量读出来累加
```

这个形式**和注意力的"软查表"如出一辙**:`kᵢ · x` 是 query(当前残差流 x)与第 i 个 key 的相似度,过激活函数后当权重,去加权值向量 vᵢ。区别在于:注意力的 KV 来自**当前序列**(动态、随输入变),FFN 的 KV 是**训练学到的权重**(静态、存在参数里)。所以:**FFN = 一张固定在权重里的、内容寻址的记忆表;输入是 query,匹配到的键触发对应的值被写回残差流。**

### 证据与可定位性

Geva 等的实证:很多中间神经元 mᵢ(即某个键 kᵢ)对应**人类可识别的输入模式**(比如"句子在描述某个时间段""文本是某种代码"),而对应的值 vᵢ 在 unembedding 空间里**指向一组语义相关的输出 token**。更进一步,**ROME / MEMIT**(Meng et al. 2022/2023)证明了可定位、可编辑:他们用因果中介分析(causal tracing,逐个位置/层做激活打补丁,看哪一步对预测正确事实最关键),定位到**特定中间层的 FFN** 是存储某条事实的关键;然后**直接改 W_out 的某几列(值向量)**,就能把"埃菲尔铁塔在巴黎"改成"在罗马"——而且只动这一条、不太影响别的知识。

> 这是机制视角的实用价值范例:**事实不是弥散在所有参数里不可触碰,而是相当程度上局部化在中间层 FFN 的特定键值对里,可定位、可外科手术式编辑。** 但要诚实:模型编辑的副作用(改一条带歪一片)、能否规模化、是否真"删除"了知识(还是只压住了),都仍是开放问题(待核具体效果数据)。

### 分析

- **为什么是中间层而非首尾**:causal tracing 普遍发现事实存储集中在**中间偏前的层**。直觉上,早层在做 token/句法的局部加工、晚层在把答案"投影到词表",中间层正好是"语义检索"发生的地方。
- **键值记忆与叠加叠加在一起**:同一个 FFN 神经元(键)往往是多义的(叠加),所以"一神经元一事实"同样不成立——这也是为什么编辑会有副作用。SAE 也能用在 FFN 激活上拆这个叠加。

## 六、幻觉的机制:为什么自回归模型会自信地编造

把可解释性的视角(模型在做什么计算、目标在优化什么)对准一个最让人头疼的现象:**幻觉(hallucination)**——模型流畅、自信地输出与事实不符或凭空捏造的内容。新手以为这是"bug",其实它**是训练目标的直接数学后果**。

### 推导:似然最大化里根本没有"我不知道"这个选项

自回归语言模型的训练目标(第 00、06 章)是最大化下一 token 的对数似然,等价于最小化交叉熵:

```
L = − Σₜ log P_θ(xₜ | x_<t)
```

把这个目标拆开看它在奖励什么、惩罚什么:

1. **它只优化"在真实数据分布上,把见过的续写概率推高"**。给定一个前缀,训练数据里若紧跟着某个流畅的事实陈述,模型就学着把那种**流畅、合理的陈述**概率拉高。注意:目标函数衡量的是**和训练分布的拟合度(流畅性/合理性)**,而**不是和真实世界的事实一致性**。语料里"听起来对"的句子,和"事实上对"的句子,在交叉熵眼里没有区别——只要它们在数据里出现得一样频繁。

2. **训练里没有"弃权"信号**。损失函数对每个位置都要求模型输出一个词表上的分布,**从来没有一个标签是"此处我不知道、应当拒答"**。模型从未因为"不知道却硬答"被惩罚过,也从未因为"老实说不知道"被奖励过。预训练语料里几乎全是"自信地陈述某事"的文本(没人写"接下来我也不确定但大概是…"),所以模型学到的先验就是:**任何前缀,都续上一个自信、流畅的陈述**。这就是"自信地编造"的根:自信来自模仿语料的语气,编造来自缺乏事实约束。

3. **泛化的副作用**。模型不是查表,它把训练事实压缩进有限参数(还叠加纠缠在一起,见第四、五节)。面对**没见过或记不准**的查询(罕见实体、长尾事实),它会**按学到的统计规律"插值/外推"出一个最像答案的东西**——形式正确(人名像人名、日期像日期、引用像引用),内容捏造。这不是它"想骗你",而是它的目标函数从没教它"宁缺毋滥"。

```
幻觉的因果链(全部源自目标函数):
  目标 = 最大似然 / 流畅性     ──▶  优化的是"像不像训练分布",不是"是否符合事实"
  无弃权标签                  ──▶  任何前缀都被逼输出一个自信分布,没有"不知道"出口
  有限参数 + 叠加压缩          ──▶  长尾/未见事实只能统计外推 = 形式对、内容编
  采样随机性(top-p/温度)      ──▶  即使正确答案概率最高,采样仍可能抽到次高的错误续写
```

第 4 条单独说:即便模型内部"知道"正确答案概率更高,**解码时的随机采样**(温度 > 0、top-p)也会以一定概率选中概率次高但错误的 token,一旦走偏,自回归会**顺着错误继续编下去**(误差累积,exposure bias 的近亲)。所以采样是幻觉的放大器,但不是根因——根因是目标函数。

### 校准:模型"知道自己不知道"吗

一个关键的算法问题:模型输出的概率,**是不是它真实的置信度**?这叫**校准(calibration)**。理想的完美校准是:模型说"我有 70% 把握"的那批答案里,恰好 70% 是对的。形式化为:

```
完美校准 ⟺  对所有置信度 c,  P( 答对 | 模型置信度 = c ) = c
```

经验结论(待核具体数据,但方向稳健):**预训练后的基础模型在 next-token 预测上往往相当校准**(它的概率确实大致反映对错频率);但**经过 RLHF/指令微调后,校准常被破坏**——对齐训练让模型倾向于输出"自信、确定"的语气(因为人类标注者偏好果断的回答),于是它在错误答案上也给出过高置信度。这是一个被反复观察到的代价:对齐换来了有用性和无害性,却可能牺牲了"概率即置信"的诚实性。

### 缓解:算法侧的三类思路

机制理解直接指向缓解方向。注意没有一个能"根除"幻觉(因为根在目标函数),都是降低概率:

- **给"我不知道"一个出口(训练侧)**:在微调数据里**显式加入拒答/表达不确定的样本**,让模型学到"在低置信时弃权"也是被奖励的行为(R-tuning 等思路)。本质是给损失函数补上原本缺失的"弃权"信号。
- **用外部事实约束生成(架构/系统侧)**:RAG(检索增强,后续 MLSys/应用课详述)——先检索可信文档,让模型**基于给定证据**生成而非凭记忆。把"事实一致性"从模型参数里挪到可控的外部知识源。这是工业界当前最有效的实用手段。
- **重新校准 + 利用内部信号(解码/探针侧)**:① 后处理重标定(温度缩放 temperature scaling 等)让输出概率重新对齐真实对错率;② 用一致性做置信度——同一问题多次采样,**答案高度发散 ⟹ 模型其实没把握**(self-consistency 思想);③ mech interp 的前沿尝试:训练**探针(probe)** 从中间层激活里读出"模型内部是否相信这是真的"信号——若内部表示与输出不一致,可能是在幻觉(待核,属活跃研究,效果未成定论)。

> 锁死结论:**幻觉不是 bug,是"用最大似然训练 + 无弃权信号 + 有限参数压缩 + 随机采样"这套配方的必然产物。** 凡声称"彻底消除幻觉"的方案都该警惕;现实是把它压到可接受范围,且不同根因配不同药——搞清是采样、是缺事实约束、还是校准坏了,再下手。

## 七、架构前沿(一):状态空间模型与对 O(N²) 的围剿

Transformer 不是终点。它有一个结构性痛点(第 02、10 章):**注意力是 O(N²) 计算、KV cache 是 O(N) 显存**,长序列下越来越吃不消。一批工作在试图换掉或补强它,主线是状态空间模型。

### 问题:能不能既要 RNN 的 O(N) 推理、又要 Transformer 的并行训练

回顾两类架构的取舍:

- **RNN**:推理 O(N)(每步只更新一个固定大小的隐状态 h,与历史长度无关,KV cache 等价物是常数大小),但**训练不能并行**(时刻 t 依赖 t−1,必须顺序展开),且长程依赖会因梯度消失而丢失。
- **Transformer**:训练完全并行(所有位置一起算),长程依赖直达,但推理/训练都 O(N²),序列一长就爆。

理想架构:**训练时像 Transformer 一样并行,推理时像 RNN 一样 O(1)/步、显存 O(1)。** 状态空间模型(State Space Model, SSM)就是冲着这个"既要又要"去的。

### 推导:从连续 SSM 到可并行的卷积形式

SSM 的根来自控制论里的线性时不变系统。连续形式:一个输入信号 u(t) 驱动一个隐状态 h(t),再读出 y(t):

```
连续:  h′(t) = A·h(t) + B·u(t)        (状态如何随输入演化,A 是状态转移矩阵)
        y(t)  = C·h(t)                 (从状态读出输出)
```

离散化(把连续时间按步长 Δ 采样,标准做法,如零阶保持)得到一个**线性递归**:

```
离散:  hₜ = Ā·hₜ₋₁ + B̄·uₜ            (Ā, B̄ 由 A, B, Δ 算出)
        yₜ = C·hₜ
```

**这就是一个线性 RNN**——推理时每步只更新固定大小的 h,O(1)/步、显存 O(1)。但线性 RNN 不也得顺序展开吗?关键洞察:**因为递归是线性的(没有非线性激活穿插在 h 的更新里)**,可以把它展开成一个**卷积**:

```
hₜ = Σ_{k=0}^{t} Āᵏ · B̄ · uₜ₋ₖ
yₜ = C·hₜ = Σ_{k=0}^{t} (C·Āᵏ·B̄) · uₜ₋ₖ
        = Σ_{k} K̄ₖ · uₜ₋ₖ          其中  K̄ₖ = C·Āᵏ·B̄   是一个固定的卷积核
```

也就是 **y = K̄ ∗ u**,一个线性卷积!卷积可以用 FFT 在 O(N log N) 并行算出整条序列——**训练并行问题解决**。这就是 S4(Gu et al. 2021)的核心:**同一个模型有两副面孔——训练用卷积形式(并行、O(N log N)),推理用递归形式(O(1)/步)**。线性递归 ⟺ 卷积的等价,是 SSM 能"既要又要"的数学命门。

### Mamba 的突破:选择性(让 SSM 看得懂内容)

S4 有个硬伤:Ā、B̄、C 是**固定的、与输入无关的**(时不变 LTI),所以它对所有 token 一视同仁地处理——**不能根据内容决定"记住这个、忘掉那个"**。而注意力的威力恰恰在于内容相关(query 决定看哪些 key)。这是早期 SSM 打不过 Transformer 的关键。

**Mamba(Gu & Dao 2023)的核心改动:让 B、C、Δ 成为输入 u 的函数**(selective SSM / S6),即参数随每个 token 动态变化:

```
S4(时不变):  Ā, B̄, C 固定         →  每个 token 同等处理,不会"选择性记忆"
Mamba(选择): B, C, Δ = f(uₜ)       →  参数随输入变,能根据内容门控:决定记什么、忘什么
```

直觉:Δ(步长)变大 ≈ "重置/重视当前输入";Δ 变小 ≈ "保持旧状态、忽略当前"。这赋予 SSM **内容相关的门控**(很像 LSTM 的门,但在 SSM 框架里),让它能"专注"重要 token、跳过无关 token——这才追平了注意力的核心能力。

**代价**:参数随输入变后,**卷积形式失效了**(卷积核不再固定,FFT 那条路断了)。Mamba 的工程贡献是一个**硬件感知的并行扫描(parallel scan / 前缀和)算法**:线性递归 hₜ = Āₜ·hₜ₋₁ + B̄ₜ·uₜ 虽然时变,但仍是**结合性(associative)操作的前缀扫描**,可用 Blelloch 扫描在 O(log N) 深度并行,且 Mamba 把它做成在 GPU SRAM 里完成、不实体化巨大中间状态的 kernel(精神上接近 FlashAttention)。**所以它牺牲了卷积,换回了选择性,再用并行扫描补回训练并行性。**

### 分析:复杂度对账与边界

| 维度 | Transformer | Mamba / 选择性 SSM |
|---|---|---|
| 训练复杂度 | O(N²·d) | O(N·d·d_state)(扫描线性于 N) |
| 推理/步 | O(N·d)(要看全部 KV) | O(d·d_state)(只更新固定状态) |
| 推理显存 | O(N·d)(KV cache 随长度涨) | O(d·d_state)(常数,与长度无关) |
| 内容寻址 | 强(任意 query↔key) | 经选择性补回,但是"压进固定状态" |

最关键的**能力差异**:Transformer 在每一步都能**回看全部历史的精确表示**(KV 全留着);SSM 把历史压进一个**固定大小的状态 h**,信息必然有损——这在需要**精确召回任意远处单个 token**的任务(如大海捞针、精确复制长串)上是结构性短板。实证上 SSM 在这类"联想召回"任务上常弱于注意力(待核具体 benchmark)。

**所以现实答案是混合(hybrid)。** 既然 SSM 擅长高效处理长程的"压缩记忆"、注意力擅长精确的"随机访问召回",就**交替堆叠**:大部分层用 Mamba(省钱),少数层插注意力(保召回)。Jamba(AI21, 2024)、以及多个后续模型都走这条路(待核各模型具体配比)。这呼应全章判断标准:新架构不是"取代",而是**用 SSM 省下 O(N²),代价是精确召回,再用少量注意力把代价买回来**。

> 还有一条平行支线:**线性注意力(linear attention)**。它从另一头逼近同一目标——把 `softmax(QKᵀ)V` 里的 softmax 用核函数 φ 近似成 `φ(Q)(φ(K)ᵀV)`,利用矩阵乘法结合律把 O(N²) 降成 O(N)(先算 φ(K)ᵀV 得到一个 d×d 的"状态"再乘 φ(Q))。它和 SSM 在数学上深度相通——都可写成"维护一个固定大小的状态、线性更新"。代价同样是丢了 softmax 的尖锐性/精确召回。这条线(Performer、Linear Transformer、RWKV、RetNet 等)与 SSM 正在收敛成同一个"线性循环 + 状态"的大家族(待核谱系细节)。

## 八、架构前沿(二):统一多模态——把一切 token 化

另一条前沿主线不是改注意力,而是**扩大它处理的对象**:让同一个 Transformer 吃下文本、图像、音频、视频。

### 问题:多模态为什么能"统一"

朴素方案是给每种模态配一个专用编码器再拼接,接口复杂、难扩展。统一架构的赌注是:**只要能把任意模态切成 token 序列,Transformer 就能一视同仁地处理**——因为注意力本身不关心 token 来自哪里,它只看 token 间的相似度。

关键机制是**分词/token 化(tokenization)的推广**:

- **图像**:ViT(Dosovitskiy et al. 2020)的做法——把图像切成固定大小的 patch(如 16×16),每个 patch 拉平、线性投影成一个向量,当成一个"视觉 token"。一张图就是一串 patch token,直接喂进 Transformer。
- **离散化图像/音频**:用 VQ-VAE / VQ-GAN 把连续的图像或音频编码成**离散码本索引**(visual tokens),于是图像变成和文本词表同构的离散 token 序列,可以和文本 token 拼进同一个词表、用同一个自回归目标训练(待核各模型实现差异)。
- **音频**:类似地切成帧或用神经音频 codec 离散化成 token。

一旦所有模态都是 token,就能把它们**拼成一条混合序列**,共享同一个 Transformer 主干、同一套注意力、同一个(自回归或扩散)目标。跨模态的关联(图文对齐)自然由注意力在混合序列上学出来——文本 token 可以注意到图像 patch token,反之亦然。这就是"统一架构"的全部魔法:**把统一难题归约成 token 化难题。**

### 分析:统一的代价与张力

- **信息密度/序列长度不匹配**:一张高清图切成 patch 可能就是上千个 token,远长于对应的文字描述。这把第 02、10 章的长上下文问题又放大了——多模态天然是长序列任务,O(N²) 更痛,所以多模态与高效注意力/SSM 是强耦合的。
- **离散化有损 vs 连续表示**:把图像离散成码本会丢细节(重建有损);保留连续 patch 嵌入则难以用纯自回归目标统一生成。这是当前架构选择的核心张力,不同模型(自回归派 vs 扩散派 vs 混合)在此分道扬镳(待核具体路线)。
- **训练数据与对齐**:统一架构吃跨模态对齐数据(图文对等),数据质量和配比是成败关键,这部分超出本章算法范围。

> 取走的核心洞察:**多模态统一不是发明了新机制,而是发现"注意力对 token 来源不可知"这一性质足够通用,只要把万物 token 化即可复用整套 Transformer 机器。** 难点从"设计跨模态机制"转移到了"如何 token 化、如何对齐、如何承受序列变长"。

## 九、设计权衡与常见坑

把全章的"坑"集中钉死,这些都是新手(和不少论文读者)真实会栽的地方。

- **把相关当因果(可解释性头号坑)**。"某神经元在看到猫时激活"——这只是相关。必须做**消融/钳制**(去掉它行为是否变、强化它是否诱发预期行为)才能下因果结论。logit lens、attention 可视化都只给相关性线索,**单独不能定论**。诱导头、SAE 特征之所以可信,正因为做了因果干预。

- **指着单个神经元问"它代表什么"**。由叠加,这通常无意义——概念住在方向(线性组合)里,且多于神经元。要解读得先 SAE 把叠加拆开,或至少找方向而非找轴。第 01 章"别人肉读单维"与此同源。

- **过度解读 cherry-picked 例子**。给几个漂亮例子说"我找到了 X 电路/特征",不验证它在**大量样本、不同模型规模**上是否稳健(判断标准 b:外推),很可能是过拟合到几个巧合。mech interp 的可复现性危机正源于此。

- **以为 SAE 解出的特征就是"真特征"**。SAE 是一个**有自身归纳偏置**的学习器,λ、d_sae、初始化都会改变解出的特征;重建有残差(没解全)、有 dead features。它给的是"一组可解释的近似分解",不是 ground truth。

- **把幻觉当可根除的 bug**。根在目标函数,只能压不能除;且要分清根因(采样 / 缺事实约束 / 校准坏)再下药。指望一个 prompt 或一个 trick 根治,是方向性错误。

- **把 SSM/Mamba 当"全面替代 Transformer"**。它在精确随机召回上有结构短板;现实是混合架构。看到"线性复杂度吊打 Transformer"的标题,先问它在联想召回类任务上掉了多少。**省下 O(N²) 必有代价,代价就是丢精确召回。**

- **混淆"能力涌现"与"指标假象"(承第 08 章)**。看到"某能力在某规模突然出现",分清是真相变还是评测指标非线性放大。诱导头那种"训练中真有拐点 + 消融可验证"的才是硬证据。

- **逻辑直读时忘了 LayerNorm 的非线性**。logit lens 把中间态投到词表时,LN 的归一化(除以范数)是非线性的,会扭曲幅度;严格分析(如直接 logit 归因)要小心处理 LN,否则归因数值会偏。

## 十、动手练习

**练习 1(分析题,核心)解释叠加为何让"一神经元一概念"失效。** 设网络要表示 n 个特征,激活空间维度 m,且 n ≫ m。提示:(a) 先论证 m 维空间最多 m 个两两正交方向;(b) 说明放宽到"近似正交"后,n 个特征方向必然与 m 个坐标轴都不对齐(斜的);(c) 由此推出"读单个坐标轴(神经元)= 读到多个特征的线性混合",即多义性;(d) 用串扰方差 ≈ (n−1)·p/m 解释为什么必须稀疏(p 小)才行。要求结论:概念住在方向里、且方向数 > 神经元数,所以逐神经元解读是问错问题。

**练习 2(论证题)论证诱导头与 ICL 的联系。** 提示:(a) 写出诱导头实现的算法("找到当前 token 上次出现处、复制其后继");(b) 说明这是一种**不更新权重、纯靠上下文模式做泛化**的机制,正是 ICL 的定义性特征;(c) 解释 Olsson et al. 给的两条因果证据——训练拐点同步、消融使 ICL 退化——为什么比"诱导头与 ICL 相关"更强;(d) 讨论它为何只是 ICL 的**骨架**而非全部(真实 few-shot 需要抽象类别匹配、多头协作)。

**练习 3(分析题)拆解幻觉的根因。** 给定一个事实问答场景,模型对一个罕见实体自信地编了个答案。提示:从目标函数出发,分别论证四条因果链各自的贡献——(a) 最大似然优化的是流畅性而非事实性;(b) 无弃权标签 ⟹ 任何前缀都被逼输出自信分布;(c) 有限参数 + 叠加压缩 ⟹ 长尾事实只能统计外推;(d) 采样随机性如何放大但不构成根因。再为每条根因配一个**对应**的缓解手段,说明为什么"配错药"无效(如:根因是缺事实约束却去调温度,治不了)。

**练习 4(推导/编码题)推导 SSM 的递归-卷积等价,并说明 Mamba 为何牺牲它。** 提示:(a) 从离散递归 hₜ = Ā·hₜ₋₁ + B̄·uₜ、yₜ = C·hₜ 出发,展开成 yₜ = Σₖ (C·Āᵏ·B̄)·uₜ₋ₖ,识别卷积核 K̄ₖ = C·Āᵏ·B̄;(b) 说明卷积可用 FFT O(N log N) 并行,这是训练并行性的来源;(c) 论证一旦 B、C 随输入 uₜ 变化(选择性),卷积核不再固定、FFT 路径失效;(d) 说明为何线性时变递归仍可用结合性前缀扫描 O(log N) 并行,从而补回训练并行性。可选:用 numpy 实现固定 SSM 的递归和卷积两种前向,验证数值相等。

## 十一、源码 / 论文导读

- **残差流 / 电路框架**:Anthropic transformer-circuits 的 "A Mathematical Framework for Transformer Circuits"(Elhage et al. 2021)——读它对残差流、QK/OV 电路、组合(composition)的形式化部分,这是整个 mech interp 的公理系统。
- **诱导头**:同系列 "In-context Learning and Induction Heads"(Olsson et al. 2022)——重点读"诱导头形成与 ICL 涌现同步"的训练动力学曲线、以及消融实验那一节(因果证据)。
- **叠加**:"Toy Models of Superposition"(Elhage et al. 2022)——读它用玩具模型展示"稀疏度↑ ⟹ 从一特征一维切换到叠加"的相变,以及特征排成几何结构的可视化。
- **SAE**:"Towards Monosemanticity: Decomposing Language Models With Dictionary Learning"(Bricken et al. 2023)及其 scaling 续作(2024)——读编码器/L1 损失定义、特征解读流程、特征钳制(因果验证)那部分。
- **MLP 即 KV 记忆**:Geva et al. 2021 "Transformer Feed-Forward Layers Are Key-Value Memories";模型编辑读 ROME(Meng et al. 2022)的 causal tracing 章节(怎么定位事实存哪层)。
- **SSM / Mamba**:S4(Gu et al. 2021,"Efficiently Modeling Long Sequences with Structured State Spaces",读递归-卷积等价与 HiPPO 初始化);Mamba(Gu & Dao 2023,读"selective scan / S6"与硬件感知扫描那一节,这是它打平注意力的关键)。
- **线性注意力家族**:Linear Transformer(Katharopoulos et al. 2020,读核化把 O(N²) 降 O(N) 的结合律推导);可对照 RWKV、RetNet 看"线性循环 + 状态"的不同实现。
- **多模态 token 化**:ViT(Dosovitskiy et al. 2020,读 patch 化那一节)是统一架构的起点;离散视觉 token 读 VQ-VAE/VQ-GAN 的码本机制。
- **开源实现**:看 SAE 训练的轻量实现(社区有多个开源 SAE 库,待核具体仓库名)、`mamba` 官方仓库的 `selective_scan` CUDA kernel(理解硬件感知扫描);`nanoGPT` 仍是对照标准 Transformer 残差结构的最干净参照。

## 十二、小结与承上启下

这一章把"黑箱"撬开了一道缝,并给了你判断下一代架构的标尺。

**机理可解释性**部分,你拿到了一套从第一性原理出发的工具链:残差流(把网络看成一条加法通信总线)→ logit lens / 组合(把任意组件接出来读)→ 诱导头(第一个被完整逆向、且有因果证据的算法,ICL 的骨架)→ 叠加(为什么逐神经元解读注定失败的几何根源)→ SAE(把叠加拆回单义特征的主力工具)→ MLP 即 KV 记忆(事实知识可定位、可编辑)。贯穿其中的铁律是**因果(消融/钳制)+ 外推(多样本、多规模)**,这是区分"真发现"与"漂亮巧合"的唯一标准。

**幻觉**部分,你看到它不是 bug 而是"最大似然 + 无弃权 + 有限压缩 + 随机采样"的数学必然,以及为什么缓解要对症下药、为什么"彻底消除"是伪命题。

**架构前沿**部分,你掌握了判断新架构的两把尺子——省了什么复杂度、丢了什么能力:SSM/Mamba 用线性递归(及递归-卷积等价、选择性扫描)换来 O(N) 与常数推理显存,代价是精确召回,故走向混合;统一多模态把"统一难题"归约为"token 化难题",代价是序列变长与离散化有损。

**在整门课里的位置。** 前十四章你学会了**造**一个大模型(架构、训练、scaling、长上下文、PEFT)并把它跑高效。这一章是两个转向:向内,**看懂**你造的东西内部在算什么(为安全、调试、可控打地基);向外,**看向**Transformer 之后的架构图景(为读前沿论文、为自己改进架构装上判断力)。至此你已具备读懂前沿论文、并对"模型内部机制"和"下一代架构"形成独立判断的专家底座——可解释性会告诉你模型为何这样行为,架构前沿会告诉你它接下来可能长成什么样;两者合起来,才是"既能实现、又能改进"的专家所需的完整视角。
