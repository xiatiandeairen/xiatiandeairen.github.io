## 1. 动机:为什么"准确率很高"和"系统安全"是两件几乎无关的事

前面十几章我们都在追问一个问题:**怎么让模型更准**——更好的表示、更强的序列模型、更对的解码、检索增强。这一章反过来追问:**在分布内很准的模型,在对抗、在边缘、在恶意输入下会怎么崩,以及它在"正常"使用中会悄悄造成什么社会危害**。这不是道德附录,而是把 NLP 当成要部署给真实人类的系统时,绕不过去的工程现实。

把威胁分成两类,贯穿全章:

- **能力性失效(robustness / hallucination)**:模型在没人攻击时也会出错——换个同义词就分类翻转、流畅地编造不存在的事实。根源是**模型学到的决策边界和它"应该"学的边界不一致**,以及**训练目标本身只优化流畅不优化真实**。
- **社会性 / 安全性失效(bias / toxicity / privacy / injection)**:模型忠实地反映了它被训练的数据和目标,而那个数据和目标本身带着社会偏见、有害内容、隐私和被滥用的接口。根源往往不是 bug,而是**目标函数和数据分布的诚实产物**。

这两类失效有一个共同的、反直觉的内核,值得开篇就钉死:

**模型的很多"问题行为"不是缺陷,而是它的训练目标在某个维度上被完美优化的结果。** 语言模型最大化 `P(xₜ | x_{<t})`,于是它学会流畅(包括流畅地编造);它最大化语料的似然,于是它学会语料里的性别职业关联(包括刻板印象);它学会服从指令,于是它也服从藏在检索文档里的恶意指令。**理解这一点,你就不会再期待某个"补丁"能根治这些问题——你会转而问:代价转移到哪去了。**

下面五节,每节锁定一类失效:对抗鲁棒性、偏见、毒性与安全攻击、隐私、幻觉。每节都按"问题 → 机制/推导 → 代码 → 分析与局限"展开。

## 2. 对抗鲁棒性:离散性如何让文本攻击与防御都变难

### 2.1 问题:什么叫"对抗样本",文本场景的特殊性在哪

**对抗样本(adversarial example)**:对一个输入做**人类几乎察觉不到、且不改变其真实语义**的微小扰动,使模型的输出发生人类不认可的改变。图像领域的经典定义最清楚:给定分类器 f、图像 x、真标签 y,找一个扰动 δ 使得

```
maximize   Loss(f(x + δ), y)
subject to ‖δ‖∞ ≤ ε          # 扰动在像素空间足够小,人眼看不出
```

这是一个**连续优化**问题。x + δ 仍是合法图像,δ 可以取任意小的实数,于是可以用梯度:沿 ∇ₓ Loss 方向走一小步就能升高损失。这就是 FGSM(Fast Gradient Sign Method,Goodfellow 等 2014)和 PGD(Projected Gradient Descent)的全部基础:

```
FGSM:  x_adv = x + ε · sign( ∇ₓ Loss(f(x), y) )
```

**文本场景为什么不能照搬?核心是离散性(discreteness)。** 一句话由离散 token 组成,token 来自一个有限词表 V。你不能给单词 "good" 加一个 0.01 的扰动得到 "good + 0.01"——那不是任何一个词。这带来三个连锁的难点,是本节的骨架:

1. **没有可直接用的梯度方向。** 损失对 token 的"梯度"在离散空间里没有几何意义:你能算 ∂Loss/∂embedding,但 embedding 空间里走一小步落点通常**不对应任何真实词**。
2. **扰动的"小"无法用范数定义。** 图像用 ‖δ‖∞;文本里"改一个词算大还是小"取决于语义和语法,不是一个可微的范数。把 "good" 换成 "great" 几乎不改语义,换成 "bad" 完全改了——但在 embedding 距离上未必前者更近。
3. **约束更硬。** 合法的对抗文本必须同时满足:(a) 仍然合乎语法;(b) 保持原义(不能把"我喜欢"改成"我讨厌"还说扰动很小);(c) 人类不易察觉。这是三个难以同时量化的约束。

**一句话定调:图像对抗是"连续空间里的有约束梯度上升",文本对抗是"离散空间里的有约束组合搜索"——后者无梯度可直接用、可行解稀疏、约束难量化,所以本质上更难。**

### 2.2 机制:文本攻击的三个层级与离散搜索

按扰动粒度从粗到细,主流攻击分三层:

**(a) 词级:同义词替换(synonym substitution)。** 最经典、语义保持最好。思路:对句子里每个词,从它的同义词集合里挑一个替换,使模型输出翻转,同时尽量保持语义。代表是 **TextFooler(Jin 等 2020)** 和 **PWWS**。它要解一个组合优化:

```
给定句子 x = (w₁, …, wₙ),分类器 f,真标签 y
找替换方案 x' = (w₁', …, wₙ')  使  f(x') ≠ y
约束:每个 wᵢ' ∈ Syn(wᵢ)(同义词集),且整句语义相似度 sim(x, x') ≥ τ
目标:改动的词数尽量少
```

候选空间是 ∏ᵢ |Syn(wᵢ)|,**随句长指数爆炸**,且 f 不可微地依赖 x'(改词是离散跳变),所以**不能用梯度**,只能搜。TextFooler 用的是贪心 + 重要性排序的启发式:

```
步骤 1(词重要性打分):对每个词 wᵢ,把它删掉(或替成 [UNK]),
        看真标签的预测概率 P(y|x) 掉了多少:
        importance(i) = P(y | x) − P(y | x_{\wi})
        掉得越多说明这个词对模型决策越关键 → 优先攻击。
步骤 2(按重要性降序,逐词尝试替换):
        对当前最重要的词 wᵢ,在 Syn(wᵢ) 里挑一个候选 c,
        满足语义相似度约束,且最大化 P(y|·) 的下降;替上去。
步骤 3:一旦 f 翻转(argmax 变了)就停;否则换下一个词。
```

注意这里**梯度被一个"删词看概率掉多少"的有限差分代理替代了**——这就是离散场景下重要性估计的通用套路:用扰动后的输出变化近似"敏感度",因为真梯度对 token 不可用。

**(b) 字符级:错别字 / 同形字扰动(typo / homoglyph)。** 改字母而非整词:"great" → "gr8" / "graet"(乱序)/ "gｒeat"(用全角或西里尔字母 а 冒充拉丁 a)。代表 **DeepWordBug、HotFlip**。它专打**子词分词(tokenization)**这个软肋:一个微小错拼会让 BPE 把 "great" 切成 ["gr", "ae", "t"] 这种碎片,模型在碎片上的表示完全跑偏。这类攻击人类阅读几乎无感,但对依赖词表/分词的模型杀伤大。

**(c) HotFlip:把"翻哪个字符"用梯度近似排序。** 这是连接"离散"与"梯度"的关键技巧,值得讲透。HotFlip(Ebrahimi 等 2018)不直接在离散空间搜,而是**用 embedding 上的一阶泰勒展开,估计每个可能的字符/词替换会让损失变多少**,从而把指数级搜索压成一次梯度计算 + 排序:

```
设当前 token 的 one-hot 向量是 eᵢ,替换成另一个 token(one-hot e_j)
相当于把输入向量改变 (e_j − eᵢ)。
损失的一阶泰勒近似:
   ΔLoss ≈ ∇_{eᵢ} Loss · (e_j − eᵢ)
对所有候选 j 算这个内积,取使 ΔLoss 最大的 j —— 即"最该翻成的字符"。
```

**为什么这是近似而非精确:** 泰勒展开只在 eᵢ 邻域成立,而 e_j − eᵢ 是一个跨越整个词表的"大跳",不是小扰动。所以 HotFlip 给的是**候选排序的启发式**,真正翻上去还要验证 f 是否真翻转。**这正是文本对抗的典型妥协:借梯度来排序候选,但最终必须回到离散空间做实际验证——梯度只是搜索的指南针,不是解本身。**

### 2.3 代码:一个最小可跑的同义词替换攻击骨架

```python
import torch

@torch.no_grad()
def predict_prob(model, tokenizer, text, target_label):
    """返回模型对 target_label 的预测概率(用作攻击信号,故无需梯度)。"""
    enc = tokenizer(text, return_tensors="pt", truncation=True)
    logits = model(**enc).logits.squeeze(0)          # [num_labels]
    return torch.softmax(logits, dim=-1)[target_label].item()

def word_importance(model, tokenizer, words, y):
    """步骤1:删词法估计每个词对真标签的重要性(梯度的有限差分代理)。"""
    base = predict_prob(model, tokenizer, " ".join(words), y)
    scores = []
    for i in range(len(words)):
        masked = words[:i] + words[i+1:]             # 删掉第 i 个词
        drop = base - predict_prob(model, tokenizer, " ".join(masked), y)
        scores.append(drop)                          # 掉得越多越重要
    return scores

def textfooler_attack(model, tokenizer, words, y, synonyms, sim_fn, tau=0.8):
    """
    words: 原句分词; y: 真标签; synonyms(w): 返回 w 的同义词候选列表;
    sim_fn(orig, cand): 整句语义相似度(如用 sentence-transformer 余弦)。
    返回攻击后的句子和是否成功翻转。
    """
    scores = word_importance(model, tokenizer, words, y)  # 只算一次,避免 O(n²) 重复
    order = sorted(range(len(words)),
                   key=lambda i: scores[i],
                   reverse=True)                       # 按重要性降序攻击
    adv = list(words)
    for i in order:
        best_cand, best_drop = None, 0.0
        for c in synonyms(words[i]):
            trial = adv[:i] + [c] + adv[i+1:]
            if sim_fn(" ".join(words), " ".join(trial)) < tau:
                continue                               # 语义约束不满足,跳过
            drop = predict_prob(model, tokenizer, " ".join(adv), y) \
                 - predict_prob(model, tokenizer, " ".join(trial), y)
            if drop > best_drop:
                best_cand, best_drop = c, drop
        if best_cand is not None:
            adv[i] = best_cand                         # 贪心采纳最优替换
        # 检查是否已翻转
        enc = tokenizer(" ".join(adv), return_tensors="pt", truncation=True)
        with torch.no_grad():
            if model(**enc).logits.argmax(-1).item() != y:
                return " ".join(adv), True             # 攻击成功
    return " ".join(adv), False
```

这段代码的价值在于**让"无梯度、靠扰动信号、贪心搜索"这三件事落地可见**:`word_importance` 是有限差分代理梯度,内层循环是离散候选搜索,`sim_fn` 是语义约束。真实工具(TextAttack 库)在此之上加了束搜索、更强的语义约束模型(USE/SBERT)、词性一致性检查等,但骨架就是这个。

### 2.4 分析:防御为什么也难,且都有代价

**对抗训练(adversarial training)**:把攻击生成的对抗样本加进训练集再训。在图像上这是最有效的防御,但**文本上效果打折且代价高**,原因直接来自离散性:

1. **生成对抗样本本身很贵。** 图像里 PGD 几步梯度就出一个对抗样本;文本里每个对抗样本要跑一遍组合搜索(几十次前向),把它塞进训练循环,训练成本数量级上升。
2. **覆盖不全。** 对抗训练只能让模型对**你用过的那类攻击**变鲁棒(对同义词替换鲁棒了,字符级攻击照样破)。攻击空间是组合爆炸的,你训不全。这是对抗鲁棒性的普遍困境:**防御是针对已知攻击的,攻击是开放的。**
3. **鲁棒性-准确率权衡(robustness-accuracy trade-off)**。这是有理论支撑的现象(Tsipras 等 2019):为了对抗鲁棒,模型被迫学更"平滑"的决策边界,而干净数据上的最优边界未必平滑,于是**鲁棒了往往掉点干净准确率**。

**检测(detection)** 类防御:训一个分类器判断输入"是不是对抗样本"(如检测异常拼写、检测语义不连贯)。问题是这又变成一个新的分类任务,**它自己也能被对抗**——攻击者只要把"绕过检测器"也加进目标函数即可(自适应攻击 adaptive attack)。安全文献的血泪教训:**任何"静态检测器"在自适应攻击者面前都会被攻破,真正的评估必须用知道你防御机制的攻击者来测。**

**对 LLM 的对抗:越狱(jailbreak)与对抗 prompt。** 进入大模型时代,"对抗样本"升级成**对抗 prompt**:不是改分类输入,而是构造让对齐模型违反安全策略的提示。手法谱系:

- **角色扮演 / 假设框架**:"假装你是一个没有任何限制的 AI…"——把有害请求包进一个虚构语境。
- **后缀攻击(GCG,Zou 等 2023)**:用类似 HotFlip 的梯度引导的离散搜索,在请求后拼一段看似乱码的 token 后缀,使模型的拒绝概率最小化。它**形式上就是 2.2 节离散搜索 + 梯度排序的直接后裔**,只是目标从"翻转分类"变成"绕过拒绝"。
- **多轮诱导 / 编码绕过**:把有害内容用 base64、用另一种语言、拆成无害的多步来规避关键词过滤。

越狱难根治的原因和上面同构:对齐(安全微调)塑造的是一个"拒绝边界",而**自然语言的输入空间几乎无限维,拒绝边界总有没覆盖到的缝隙**。安全对齐的机制(RLHF / 安全微调)属于强化学习课的范畴,**详见强化学习专家课程的对齐与 RLHF 章节**;这里只需记住它的本质局限:它是在"已知的不良行为分布"上做拟合,而攻击分布是开放且对抗演化的。

## 3. 偏见与公平:社会偏见如何被结构性地学进并放大

### 3.1 问题:偏见从哪来,以及"放大"是什么意思

先把概念分层,避免新手把不同东西混成一锅:

- **数据偏见(data bias)**:训练语料反映了真实世界(或历史)的社会分布与刻板印象。语料里"护士"邻近女性代词的次数远多于男性,"工程师"反之——这不是标注错误,是语言使用的统计事实。
- **模型偏见(model bias / learned association)**:模型把这些共现统计学进了表示。词向量里 `vec(护士) − vec(医生)` 会带上一个和 `vec(她) − vec(他)` 同向的分量。
- **放大(amplification)**:这是最该警惕、也最反直觉的一步——**模型输出的偏见往往强于训练数据里的偏见**。

**为什么会放大?给一个机制性论证(这是本节最重要的推导)。** 考虑一个简化场景:某职业在语料里 70% 关联女性、30% 关联男性。一个**只优化期望准确率**的模型,在给该职业填代词时,最优策略不是按 70/30 输出,而是**总是输出"她"**——因为对任意单条样本,赌多数类(她)的期望正确率(0.7)高于按真实比例随机(0.7×0.7 + 0.3×0.3 = 0.58)。

```
按真实分布输出:  P(对) = 0.7·0.7 + 0.3·0.3 = 0.58
总赌多数类输出:  P(对) = 0.7·1   + 0.3·0   = 0.70   ← 更高
```

**于是 70/30 的数据偏见,被一个理性的、只看准确率的模型放大成了 100/0 的输出偏见。** 这就是 mode collapse 式的放大:**优化期望损失的模型有把软性多数偏好硬化成绝对偏好的内在倾向。** 解码时再叠加(贪心/低温采样总取最可能 token)会进一步加剧。**记住:偏见放大不是模型"学坏了",而是"最小化期望误差"这个目标在不平衡分布上的数学后果。**

(这和第 2 节幻觉、第 5 节是同一个母题:问题行为是训练目标的诚实产物。)

### 3.2 度量:从 WEAT 到下游公平指标

**度量偏见的第一难点:偏见不是一个标量,你必须先定义"对谁、在什么维度上、用什么基准算不公平"。** 介绍三类由内到外的度量。

**(a) WEAT:词嵌入关联测试(Word Embedding Association Test,Caliskan 等 2017)。** 把心理学的内隐联想测验搬到词向量。给两组**目标词**(如 科学词集 X = {科学, 技术, …} vs 艺术词集 Y = {诗歌, 舞蹈, …})和两组**属性词**(如 男性词集 A = {他, 男人, …} vs 女性词集 B = {她, 女人, …}),问:科学词是不是更"靠近"男性词、艺术词更靠近女性词?

定义单个目标词 w 对两个属性集的**关联差**:

```
s(w, A, B) = mean_{a∈A} cos(w, a) − mean_{b∈B} cos(w, b)
             └─ w 与"男性"的平均相似 ─┘   └─ w 与"女性"的平均相似 ─┘
```

`s(w,A,B) > 0` 表示 w 更偏男性属性。整个测试的效应量(effect size,标准化的偏见强度):

```
            mean_{x∈X} s(x,A,B) − mean_{y∈Y} s(y,A,B)
WEAT_d  =  ──────────────────────────────────────────
              std_{w∈X∪Y} s(w, A, B)
```

分子是"科学词的男性偏向"减"艺术词的男性偏向";分母用所有目标词关联差的标准差做标准化(类似 Cohen's d)。**`d` 越大,偏见越强;`d` 趋 0 表示无系统性关联。** 显著性用置换检验(permutation test):随机打乱 X、Y 的划分,看真实 `d` 落在随机分布的哪个尾部。

**(b) 句级 / 上下文化度量:SEAT、CrowS-Pairs、StereoSet。** WEAT 用的是静态词向量;BERT 这类上下文模型每个词的向量随句子变,于是 **SEAT** 把目标/属性词放进模板句("This is [WORD].")取句向量再跑 WEAT。**CrowS-Pairs / StereoSet** 则改用**最小对(minimal pair)**:一句刻板("The nurse said she…")、一句反刻板("The nurse said he…"),看模型给哪句更高的似然 `P(sentence)`;偏向刻板句的比例就是偏见分。这类度量更贴近"模型实际会生成什么"。

**(c) 下游任务公平指标。** 最终我们关心的是部署后的差异化伤害。借用公平机器学习的标准定义,设受保护属性为 g(如性别),预测为 ŷ,真标签 y:

```
人口均等(Demographic Parity):   P(ŷ=1 | g=0) = P(ŷ=1 | g=1)
                                  正预测率与组别无关
机会均等(Equalized Odds):       P(ŷ=1 | y=1, g=0) = P(ŷ=1 | y=1, g=1)
                                  且 P(ŷ=1 | y=0, ·) 同样跨组相等
                                  (真正例率、假正例率都跨组相等)
```

**关键的不可能性结论(必须知道):校准(calibration)与机会均等(均衡)二者一般不可兼得**(Kleinberg 等 2016);**叠加人口均等后,这三类准则整体互斥**(另见 Chouldechova 2017)——除非各组的基率(base rate)恰好相等或预测完美。**这意味着"公平"没有唯一定义,选哪个指标本身就是价值判断,不是技术问题。** 谁声称"我们的模型是公平的"而不说"按哪个定义",就是在偷换概念。

### 3.3 代码:实现 WEAT 效应量

```python
import numpy as np
from itertools import combinations

def cos(u, v):
    return float(u @ v / (np.linalg.norm(u) * np.linalg.norm(v) + 1e-12))

def assoc(w, A, B):
    """s(w,A,B): w 与属性集 A 的平均相似 减 与 B 的平均相似。"""
    return np.mean([cos(w, a) for a in A]) - np.mean([cos(w, b) for b in B])

def weat_effect_size(X, Y, A, B):
    """X,Y: 目标词向量列表; A,B: 属性词向量列表。返回 Cohen's-d 式效应量。"""
    sx = np.array([assoc(x, A, B) for x in X])
    sy = np.array([assoc(y, A, B) for y in Y])
    all_s = np.concatenate([sx, sy])
    return (sx.mean() - sy.mean()) / (all_s.std(ddof=1) + 1e-12)

def weat_pvalue(X, Y, A, B, n_perm=10000, seed=0):
    """置换检验:随机重分 X∪Y,看真实 d 的稀有度(单尾)。"""
    rng = np.random.default_rng(seed)
    pool = X + Y
    k = len(X)
    observed = sum(assoc(x, A, B) for x in X) - sum(assoc(y, A, B) for y in Y)
    count = 0
    for _ in range(n_perm):
        rng.shuffle(pool)
        diff = sum(assoc(w, A, B) for w in pool[:k]) \
             - sum(assoc(w, A, B) for w in pool[k:])
        if diff >= observed:
            count += 1
    return count / n_perm
```

**实现要点与坑**:(1) WEAT 对**目标/属性词集的选取极度敏感**——换一组词,`d` 能从显著变不显著,所以单次 WEAT 不能当作"模型有/无偏见"的定论;(2) 静态词向量的 WEAT 不代表上下文模型(BERT/GPT)的实际行为,要测后者必须用 SEAT/StereoSet;(3) 置换检验的 `n_perm` 要够大,组合数 C(|X∪Y|, |X|) 小时可枚举精确解。

### 3.4 缓解与权衡:三个介入点,没有免费的午餐

按介入位置(数据 → 训练 → 后处理)分三类,每类都讲清代价:

**(a) 数据层:重采样 / 反事实数据增强(CDA, Counterfactual Data Augmentation)。** 对每个含性别词的句子,生成性别对调的副本一起训("他是护士"⇄"她是护士"),强行打平职业-性别共现。
- 代价:只能打平你**显式枚举**的属性词对(他/她),对隐式偏见(名字、职业暗示)无能为力;反事实句可能不合语用("她是父亲")需过滤;扩大数据=更高训练成本。

**(b) 训练层:加去偏正则 / 投影掉偏见子空间(Bolukbasi 等 2016 的 Hard-Debias)。** 找出 embedding 里的"性别方向"(用 he−she 等词对做 PCA),把中性词(护士、工程师)在这个方向上的投影清零。
```
设性别方向 g(单位向量),对应被去偏的词向量 w:
   w_debiased = w − (w · g) g          # 减掉它在性别轴上的分量
```
- 代价:**Gonen & Goldberg 2019 给出的著名反驳**——这种几何去偏只是"把偏见藏起来"而非消除:去偏后,原本性别相关的词在**聚类结构**上仍紧挨在一起,下游分类器照样能恢复性别信息。**表面度量(WEAT)降了,实质偏见还在。** 这是去偏研究最重要的教训:**度量降低 ≠ 危害消除,小心优化了指标却没改变行为。**

**(c) 后处理层:解码时约束 / 输出过滤 / 阈值按组调整。** 生成时禁止某些关联,或对分类阈值按组单独校准以满足某个公平指标。
- 代价:直接触发 3.2 节的不可能定理——你满足了人口均等就破坏了校准;且"按组调阈值"在很多司法辖区涉及"是否允许用受保护属性"的合规问题。

**贯穿三层的元结论:偏见缓解永远是在"公平的某个定义""任务效用""其他公平定义"之间做权衡,不存在一个动作能同时优化全部。** 工程上诚实的做法是:明确写出你优化的是哪个公平定义、监测它对效用和其他维度的影响,而不是声称"已去偏"。

## 4. 毒性与安全:有害生成、提示注入、红队

### 4.1 毒性(toxicity):生成有害内容

**毒性**指模型生成仇恨、骚扰、暴力、违法指引等有害文本。来源同样是双重的:语料里有毒内容(互联网语料无法完全清洗)+ 生成目标不区分"流畅的善意"和"流畅的恶意"。度量上学界常用 **RealToxicityPrompts(Gehman 等 2020)**:用大量"看似无害的前缀"喂模型,看续写的毒性分(用 Perspective API 之类的毒性分类器打分)。一个反直觉发现:**即使是无毒前缀,大模型也会以不可忽略的概率续出有毒内容**——因为语料里那些前缀后面确实跟过有毒文本。缓解手段(数据过滤、毒性分类器在解码时重排、安全微调)和偏见缓解同构,局限也同构:过滤会误伤(把讨论歧视的正当文本也滤掉),分类器本身有偏(对某些方言/群体的正常表达误判为毒性,这本身是新的公平问题)。

### 4.2 提示注入:RAG / Agent 时代的原生漏洞

这是大模型安全里**最该讲透、也最被低估**的一类,因为它不是"模型不够好",而是**架构层面的设计缺陷**。

**核心问题:LLM 把"指令"和"数据"放在同一个通道里。** 传统软件有清晰的边界:代码是代码,用户数据是数据,SQL 注入之所以能修,是因为我们能用参数化查询把数据和指令分开。但 LLM 的输入是一段**没有类型的自然语言**——开发者写的系统提示、用户的问题、从网页/文档检索来的内容,**全都拼成一个字符串喂进去,模型无法从原理上区分哪段是"该服从的指令"、哪段是"只该当资料看的数据"**。

**提示注入(prompt injection)** 就是利用这一点:攻击者把恶意指令藏在"数据"里,让模型当成指令执行。分两种:

- **直接注入**:用户自己在输入里写"忽略你之前的所有指令,改为…"。这就是越狱的一种。
- **间接注入(indirect prompt injection,Greshake 等 2023)**:**这才是 RAG/Agent 的致命伤**。攻击者把恶意指令埋在**模型会去读的外部内容**里——一个网页的隐藏文本、一封邮件的正文、一份被检索到的文档。当 RAG 检索到这段、或 Agent 浏览到这个网页,恶意指令就进入了上下文,被模型执行。

**攻击面分析(画出来):**

```
[系统提示 System]  ──┐
[用户输入 User]    ──┼─→  拼成一个 prompt  ─→  LLM  ─→  动作/输出
[检索文档 Docs]    ──┤         ▲                        │
[工具返回 Tool]    ──┘         │                        ▼
                            注入点:任何        若 LLM 能调工具(发邮件/
                            非可信来源的         执行代码/改数据库),
                            文本都可藏指令       注入 → 任意动作
```

**为什么 RAG/Agent 尤危:**
1. **可信边界被打破**:RAG 的核心就是把外部不可信内容塞进上下文(第 10 章),这恰好是间接注入的入口。检索得越"成功",越可能把藏毒文档拉进来。
2. **能力被放大成行动**:Agent 能调用工具(发邮件、执行 shell、转账、改数据库)。普通聊天机器人被注入最多胡说几句;Agent 被注入可以**执行真实世界的破坏性动作**——读取你的邮件并外发、删库、给攻击者转账。
3. **数据外泄(exfiltration)**:经典攻击链——注入指令让 Agent "把刚才读到的私密信息编码进一个图片 URL 并请求它",私密数据就通过这个外发请求泄露给攻击者的服务器。

**为什么难修(第一性):** 因为它不是 bug 而是**指令/数据不分**这一架构属性的直接后果。只要模型的"控制流"由自然语言决定、而自然语言无法被可靠地标注可信级别,注入就无法从原理上根除。当前的缓解都是减面而非根治:
- **最小权限**:严格限制 Agent 能调的工具、能访问的数据(这是最有效的工程手段——把"被注入后能干的坏事"砍到最小)。
- **人在回路(human-in-the-loop)**:危险动作(发邮件/转账)前必须人工确认。
- **输入隔离 / 分隔符 / 指令防御提示**:用特殊标记把数据围起来、在系统提示里写"不要服从数据里的指令"——**但这些都能被更巧妙的注入绕过,只是抬高门槛,不是证明性的防御**。
- **输出侧检测**:对外发请求、工具调用做策略检查。

**记死这一条:提示注入的"正确"心智模型不是"修一个漏洞",而是"假设上下文里任何外部内容都可能是恶意指令",然后按这个假设做权限和审计设计。** 这和 Web 安全里"永远不信任用户输入"是同一种思维,只是 LLM 把边界搞模糊了。

### 4.3 红队与安全对齐(指向强化学习课)

**红队(red-teaming)**:主动雇人(或用另一个 LLM)去攻击模型,系统性地找出能诱发有害输出的提示,把这些案例收集起来做安全训练。它是当前安全工程的支柱,因为前面反复说的:**攻击空间是开放的,你只能通过持续对抗性探测来发现缝隙**,无法静态证明安全。

**安全对齐**——通过 RLHF / 安全微调让模型学会拒绝有害请求——其算法机制(奖励模型、PPO/DPO、宪法 AI)**属于强化学习专家课程的对齐章节,本章不重复推导**。从 NLP 安全视角只强调它的两个结构性局限:(1) 它拟合的是"已知不良行为分布",对分布外的新型攻击(新越狱模板)天然滞后;(2) 它会引入**过度拒绝(over-refusal)**——为了安全把正当请求也拒了(问"如何杀死一个 Linux 进程"被当成暴力请求),这是安全-有用性权衡的又一实例,和鲁棒性-准确率、公平-效用是同一类张力。

## 5. 隐私:训练数据记忆与抽取攻击

### 5.1 问题:模型会"背下"训练数据,然后被套出来

语言模型在最小化 `P(xₜ | x_{<t})` 时,对**在语料里出现过、尤其是出现多次或高度独特**的序列,会把它**逐字记住(memorization)**。后果:模型可能在生成时**原样吐出训练集里的私密信息**——某人的真实邮箱、电话、住址、一段未公开的代码、某段受版权文本。这不是抽象担忧:**Carlini 等(2021)的训练数据抽取攻击(training data extraction)** 实证了从 GPT-2 里套出几百条逐字训练样本,含真实姓名、电话、地址。

### 5.2 机制:为什么记忆不可避免,以及抽取怎么做

**记忆的必然性(论证):** 模型容量(参数量)足够大时,降低训练损失的一条捷径就是直接记住稀有样本——因为稀有样本无法靠泛化规律预测,只能背。这与第 3.1 节"放大"、第 2 节"流畅 ≠ 真实"同源:**都是损失最小化在特定数据结构上的诚实后果。** 经验上,**重复次数越多、序列越独特,被记住的概率越高**;模型越大,记忆容量越大(Carlini 等 2022 的 scaling 研究表明记忆量随模型规模、数据重复度增长)。

**抽取攻击怎么做(两步):**
```
步骤1(生成候选):用各种前缀(甚至空前缀)大量采样模型输出,
        得到一堆续写文本。
步骤2(成员判定 / membership inference):判断哪些续写"很可能是
        逐字训练数据"。关键信号:模型对该串的困惑度(perplexity)
        异常低 —— 但低困惑度也可能只是"这串本来就很常见"。
        改进:看"两个模型困惑度之比"或"同一模型不同温度/大小写下
        困惑度的反差"。真实记忆的串往往在大模型上困惑度极低、
        而在小模型或加噪后明显升高。
```

**成员推断攻击(membership inference attack, MIA)** 是更一般的隐私威胁:给一条样本,判断它是否在训练集里。原理:模型对**见过**的样本通常给更高似然 / 更低损失(过拟合的痕迹)。一个最朴素的 MIA 就是阈值化损失:`loss(x) < τ ⇒ 判定为成员`。

### 5.3 缓解:差分隐私及其代价

**差分隐私(Differential Privacy, DP)** 是唯一有**数学保证**的缓解(不像前面那些补丁)。它的定义:一个随机算法 M 满足 (ε, δ)-DP,若对任意只差一条记录的相邻数据集 D、D',以及任意输出集合 S:

```
P(M(D) ∈ S)  ≤  e^ε · P(M(D') ∈ S)  +  δ
```

**直觉:加进或拿掉任何单条训练样本,几乎不改变模型输出的分布(改变被 e^ε 这个因子卡住)。** 既然单条样本对输出影响极小,攻击者就无法从输出反推某条样本是否在训练集里——这同时挡住了记忆抽取和成员推断。实现方式是 **DP-SGD**:训练时对每个样本的梯度先**裁剪(clip)** 到一个范数上界,再**加高斯噪声**,然后才更新:

```
DP-SGD 单步:
  对每个样本 i:  gᵢ ← clip(∇Loss(xᵢ),  C)      # 限制单样本影响 ≤ C
  聚合加噪:      g  ← (1/B)( Σᵢ gᵢ + 𝒩(0, σ²C²I) )   # 注入噪声
  更新:          θ  ← θ − η · g
```

**代价(必然的权衡):** 裁剪 + 加噪直接损害梯度信号,**ε 越小(隐私越强)→ 噪声越大 → 模型质量越差**。这是隐私-效用权衡(privacy-utility trade-off),又一个"没有免费午餐"的实例。此外 DP 的保护是**针对单条记录**的;如果同一私密信息在语料里出现很多次(同一邮箱出现在多份文档),"相邻数据集"的定义就保护不住它——所以**数据去重(deduplication)** 是和 DP 互补的、性价比极高的工程手段(Kandpal 等 2022 表明去重能显著降低抽取成功率)。

**坑:** 别把"我们做了脱敏(去掉显式 PII)"当成隐私保护。模型能从上下文重新关联出身份(去标识化数据可被重识别);只有 DP 这种作用于训练过程的机制才有形式化保证,而它有明码标价的效用损失。

## 6. 幻觉的 NLP 视角:为什么这是目标函数的后果,不能根治

幻觉在第 10 章 RAG 里已从"知识来源"角度谈过,这一节从**语言建模目标本身**把它的不可根治性讲到底。

### 6.1 成因:流畅性优化 + 没有"不知道"的信号

**根本成因一:训练目标优化的是流畅(似然),不是真实。** 语言模型最大化 `Σₜ log P(xₜ | x_{<t})`。这个目标里**没有任何一项度量"这句话在世界上是不是真的"**。一个在语料统计上高概率、语法流畅、风格地道的句子,可以是彻头彻尾编的。模型学的是"什么样的 token 序列像人写的",不是"什么是事实"。**幻觉因此是这个目标的诚实产物——它在干它被训练去干的事(产出像样的文本),只是"像样"和"为真"不是一回事。**

**根本成因二:缺少"不知道"的内建信号与负反馈。** 人在不确定时会说"我不确定";模型的训练数据里几乎全是**自信陈述**(没人写"答案大概是X但我不确定"),且自回归目标对"瞎猜一个具体答案"和"承认不知道"在似然上往往前者更高(具体答案更像训练语料)。**更糟的是对齐阶段的激励错配:** 如果评测/奖励按"答对率"打分而不惩罚"自信地答错",模型学到的最优策略就是**永远猜一个而不是认怂**——这和第 3.1 节"赌多数类"是同一个数学结构。**这是近期对幻觉机制研究的一个核心论点:幻觉部分是评测/奖励设计鼓励"瞎猜得分"的结果。**(待核:此论点的具体出处与表述,凭机制理解写出,建议核对 OpenAI 2025 年前后关于幻觉与评测激励的论文原文。)

### 6.2 类型:事实性 vs 忠实性

务必区分两类,因为它们的缓解手段不同:

- **事实性幻觉(factual hallucination)**:输出与**世界真相**不符(说错了一个历史日期、编了一个不存在的 API)。判定需要外部知识源。
- **忠实性幻觉(faithfulness hallucination)**:输出与**给定的输入/上下文**不符——在摘要里加入原文没有的信息(intrinsic / extrinsic 不忠实)、在 RAG 里无视检索到的证据自说自话。判定**不需要外部世界**,只需对照输入。

这个区分在工程上是分水岭:**忠实性可以靠"对照源文本"自动检测(NLI 蕴含判断、问答一致性);事实性要靠外部知识库或检索,且世界真相本身可能有争议、会变化。**

### 6.3 缓解为什么都是补丁

逐一论证每种手段的边界——**它们都不触碰"目标优化流畅不优化真实"这个根**:

- **RAG(检索增强)**:给模型外部证据。但(第 10 章已说)检索可能召回错的;即便召回对的,模型仍可能不忠实地无视它。RAG 把"事实性"问题转化为"忠实性 + 检索质量"问题,**缓解但不根治**。
- **校准(calibration)/ 不确定性估计**:让模型输出的置信度匹配实际正确率,从而能"在不确定时说不知道"。但语言模型的 token 概率与"事实正确概率"**不是一回事**(模型可以对一个错答案给很高的 token 概率);校准只能改善"知道自己不知道"的程度,不能让它真的知道事实。且自由文本生成的"置信度"如何定义本身是开放问题。
- **事后核查(post-hoc verification / fact-checking)**:生成后用另一个系统(检索 + NLI、或另一个 LLM 当裁判)核对。这只是**在错误产生后过滤**,不改变生成机制,且核查器自己也会错(尤其用 LLM 当裁判时,裁判也会幻觉)。
- **自一致性 / 多次采样投票(self-consistency)**:多次采样取多数。对"模型其实知道但采样抖动"的情况有效,对"模型系统性地错/不知道"无效——它会**一致地、自信地**给出同一个错误答案。

**收束成不可根治性论证:** 只要(a)训练目标是似然(优化流畅)、(b)模型没有可靠的内部真值信号、(c)世界真相在参数之外且会变,那么任何不改这三点的方法都只能在"输出端"做拦截和补救。**RAG 补的是 (c) 的一部分(把知识外置),校准想补 (b)(但做不彻底),核查在 (a) 的下游兜底——没有一个动作能让"最大化似然"这个目标本身去优化真实。** 所以幻觉不能被"根治",只能被工程化地**降低到可接受水平并配合不可信假设来用**——正如你不会因为编译器有 bug 就不用编译器,但你会写测试。

**与对抗、偏见、注入的统一收尾:** 这五类问题共享同一个内核——**它们是训练目标 / 数据分布 / 架构假设的诚实产物,而非可消除的缺陷**。专家级的判断不是"能不能根治"(答案几乎都是不能),而是"代价转移到哪、用什么工程纪律把残余风险控制在可接受范围"。

## 7. 设计权衡与常见坑

把全章的权衡和高频陷阱集中钉一遍:

- **坑:把"分布内准确率高"当成"安全/鲁棒"。** 二者几乎正交。一个 SOTA 分类器可能被一个同义词替换攻破,一个对话模型可能被一句话越狱。**部署前必须单独做对抗/红队评测,而非只看测试集准确率。**
- **坑:用非自适应攻击评估防御。** 拿一个固定攻击测自己的防御,通过了就宣称鲁棒——这是安全研究的头号错误。**正确做法:用知道你防御机制的自适应攻击者来测**,否则你只是测了"防御挡不挡得住它专门防的那一种"。
- **坑:把"度量下降"当成"危害消除"。** Hard-Debias 让 WEAT 降了但偏见还在聚类里(3.4 节);DP 之外的脱敏让显式 PII 没了但可重识别(5.3 节)。**优化指标 ≠ 改变行为,要测行为/下游影响。**
- **权衡:鲁棒性 ↔ 准确率、公平 ↔ 效用、隐私 ↔ 效用、安全 ↔ 有用性。** 这四个是同一类张力的不同面孔——**所有"安全/公平/隐私"的强化都在某处花掉效用**,工程上要显式量化这个交换比,而不是假装没有。
- **权衡:公平指标之间不可兼得(不可能定理)。** 选哪个公平定义是价值判断。**先定义"对谁公平、按哪个准则",再谈缓解。**
- **坑:在 Agent/RAG 里信任检索/工具返回的内容。** 间接注入的入口就在这。**默认假设外部内容含恶意指令,靠最小权限 + 人在回路 + 审计兜底,而不是靠"在 prompt 里礼貌地请模型别听坏话"。**
- **坑:以为更大的模型更安全。** 更大往往**记忆更多**(隐私更糟,5.2)、**更会被复杂越狱诱导**、幻觉得更流畅更难辨。规模解决能力,不解决这一章的任何一类问题。
- **数据纪律:去重 + 来源审计是性价比最高的安全工程。** 训练数据去重同时降低记忆抽取风险和偏见放大;清楚知道语料来源是事后追责和合规的前提。

**练习 3(攻击面分析,思辨):画出一个"能读邮件、能发邮件、能查日历"的 LLM 邮件助手的间接提示注入攻击面,并设计防御。**

**练习 4(机制论证,挑战):论证"幻觉无法被根治",并给出一个你认为最接近根治的方向及其残余局限。**

## 9. 源码 / 论文导读

**对抗鲁棒性:**
- **TextAttack**(开源库):读它的 `attack_recipes`(TextFooler、PWWS、DeepWordBug 的实现)和 `goal_functions` / `constraints` 模块——能看到"目标函数 + 约束 + 搜索方法"如何模块化,正是 2.3 节骨架的工业版。
- **HotFlip**(Ebrahimi 等 2018):看它用 embedding 一阶泰勒近似排序字符翻转的那一节(2.2 节推导的原始来源)。
- **GCG**(Zou 等 2023,*Universal and Transferable Adversarial Attacks on Aligned Language Models*):看它的 token 级贪心坐标梯度搜索——越狱版的 HotFlip。

**偏见与公平:**
- **WEAT**(Caliskan, Bryson, Narayanan 2017, *Science*):看效应量定义与置换检验(3.2 节)。
- **Hard-Debias**(Bolukbasi 等 2016)+ **它的反驳**(Gonen & Goldberg 2019, *Lipstick on a Pig*):必须配对读,理解"度量降了危害没消"。
- **StereoSet / CrowS-Pairs**:看上下文模型的最小对偏见度量怎么构造。
- 公平不可能定理:Kleinberg, Mullainathan, Raghavan 2016。

**安全与注入:**
- **间接提示注入**:Greshake 等 2023(*Not what you've signed up for*)——RAG/Agent 注入的奠基论文,看它的攻击分类与外泄链。
- **RealToxicityPrompts**(Gehman 等 2020):毒性评测基准与"无毒前缀也续出毒性"的发现。
- 安全对齐(RLHF/DPO/宪法 AI)的算法:**详见本库《强化学习专家课程》的对齐章节**,本章不重复。

**隐私:**
- **训练数据抽取**:Carlini 等 2021(*Extracting Training Data from Large Language Models*)——抽取攻击两步法的来源(5.2)。
- **记忆 scaling**:Carlini 等 2022(*Quantifying Memorization*)。
- **DP-SGD**:Abadi 等 2016(*Deep Learning with Differential Privacy*)——看裁剪+加噪和隐私会计(moments accountant)。
- 去重降记忆:Kandpal 等 2022。

**幻觉:** 综述见 Ji 等 2023(*Survey of Hallucination in NLG*),区分 intrinsic/extrinsic 不忠实;事实性 vs 忠实性的工程含义见第 10 章 RAG 评测与本库《评测》章。

## 10. 小结与承上启下

这一章讲透的核心是**一个统一的视角**:NLP 模型的对抗脆弱、社会偏见、毒性、隐私泄露、幻觉,**绝大多数不是可修的 bug,而是训练目标(最大化似然/服从指令)、数据分布(带社会偏见、含私密与有害内容)、架构假设(指令与数据同通道)的诚实产物**。我们把每一类都推到了机制层:

- **对抗鲁棒性**:离散性使梯度不可直接用,攻击退化为有约束的组合搜索(TextFooler 的删词代理梯度、HotFlip/GCG 的泰勒排序);防御受困于"开放攻击 vs 已知防御"和鲁棒-准确率权衡。
- **偏见**:用"赌多数类"的数学论证了**放大**为何是期望损失最小化的必然;给了 WEAT 的完整效应量推导与代码;三层缓解各有代价,且公平指标不可兼得。
- **安全**:提示注入(尤其 RAG/Agent 的间接注入)被定性为**指令/数据不分的架构原生漏洞**,只能减面不能根除;红队是应对开放攻击的支柱;安全对齐指向强化学习课。
- **隐私**:记忆是大模型降损失的捷径,抽取攻击实证可行;只有 DP(DP-SGD 的裁剪+加噪)有形式化保证,代价是效用,去重是高性价比补充。
- **幻觉**:从"似然不优化真实 + 无不知道信号 + 奖励鼓励瞎猜"论证其不可根治,RAG/校准/核查/自一致性都只是补丁。

**承上**:本章的幻觉与忠实性,直接接第 10 章 RAG(检索是缓解事实性幻觉的主路径)和本库《评测》章(忠实性/事实性如何量化);本章的对抗与注入,是把前面所有"如何让模型更准"的努力放到对抗环境里重新审视。

**启下**:理解了"问题源于目标与数据"之后,自然的下一步是问——**能不能改目标本身**?这正是对齐(把人类偏好、安全、真实性写进训练目标)要回答的,其算法机制是**《强化学习专家课程》**的主场。而本章反复出现的"度量 ≠ 行为""指标可被优化但危害仍在",则是任何严肃评测体系必须正面应对的——交给本库的评测章。带着这一章训练出的怀疑精神去读它们:**任何声称"已解决"的安全/公平/真实性主张,先问一句"代价转移到哪、用哪个定义、被谁攻击过"。**
