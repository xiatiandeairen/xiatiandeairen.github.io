> 这一章把你从"会推 PPO/GRPO 的损失函数"带到"能在一个真实的 RLHF/RLVR 训练里，看着曲线判断它正在以哪种方式崩、并把它救回来"。前面的章节告诉你算法在数学上为什么成立；这一章告诉你它在工程上为什么会以你意想不到的方式失败，以及失败前曲线长什么样。

## 1. 为什么需要单独讲"稳定性"

监督学习（SFT）里，loss 下降、val loss 不涨，基本就赢了。RL 不是这样。RL 训练有三个监督学习没有的结构性麻烦，它们叠加在一起，让"loss 在动"和"模型在变好"之间几乎没有可靠的因果关系：

**第一，目标是你自己造的，而且会被钻空子。** SFT 的标签是人给的、固定的。RL 的奖励 `r(x, y)` 通常来自一个学出来的奖励模型（Reward Model, RM）或一段你写的规则代码。策略（policy）被优化的唯一目标就是把这个 `r` 推高——它没有任何动机去"真的把任务做好"，它只想把 `r` 这个数字做大。一旦 `r` 和"真的做好"之间有任何缝隙，优化过程会精确地、不知疲倦地把那条缝撑大。这就是 **reward hacking**。

**第二，数据分布是你自己生成的，而且会漂移。** SFT 的数据是离线的、独立同分布（i.i.d.）采样的。RL 里训练数据来自当前策略自己的 rollout（采样轨迹）。策略一更新，下一批数据的分布就变了。这是一个反馈回路：策略影响数据，数据影响梯度，梯度又改策略。任何正反馈环节（比如某种 hacking 一旦开始就自我强化）都会被这个回路放大，直到崩溃。

**第三，优化对象是高方差的随机梯度，且非平稳。** 策略梯度本身方差就大（后面会算），加上奖励信号稀疏、价值估计有偏，再叠加上面那个非平稳的分布漂移，整个优化过程像在一块一直在变形的地面上做梯度下降。一个在 SFT 里完全无害的学习率，在 RL 里可能让训练在 50 步内熵塌缩到完全确定性、然后输出复读机。

所以"稳定性"在 RL 里不是锦上添花的工程细节，**它是算法能不能用的前提**。一个数学上无懈可击的 PPO 实现，配错一个 KL 系数，产出的就是一个谄媚的复读机。这一章的目标，是把"它正在怎么崩"变成你能从曲线和数学上诊断的东西。

预备约定：策略记 πθ(y|x)，参数 θ；旧策略（采样用的）记 π_old；参考策略（KL 锚点，通常是 SFT 模型）记 π_ref。token 级别时 πθ(y|x) = ∏ₜ πθ(yₜ | x, y_<t)。优势记 Aₜ，价值函数 Vφ(s)。本章默认你已经掌握第 11–13 章的 PPO / GRPO / RLHF 推导，这里只在需要时回顾。

## 2. Reward Hacking：把 Goodhart 定律写成数学

### 2.1 形式化：为什么"优化代理指标"必然偏离

先把直觉变成可推的东西。设存在一个**真实目标** `U(y)`（utility，我们真正想要的"好"，比如"回答正确且有帮助"），但我们无法直接优化它——我们只有一个**代理奖励** `R(y)`（proxy，RM 或规则给的分）。它们的关系写成：

```
R(y) = U(y) + B(y)          # B 是 bias/误差项：R 和 U 不一致的部分
```

策略优化做的是 max_θ E_{y~πθ} [R(y)]。代入：

```
E[R] = E[U] + E[B]
```

优化器只看到 E[R] 在涨，但 E[R] 的上升可以来自 E[U] 上升（我们想要的），也可以来自 E[B] 上升（hacking）。**关键事实**：U 和 B 在"行为空间"里通常是不同的方向，而 B 往往更容易拉高。

为什么 B 更容易？因为 U 难（真正把任务做对需要能力），B 容易（找到 RM 的盲区只需要试错）。在策略空间里，沿着 ∇E[B] 的方向移动比沿着 ∇E[U] 的方向移动"性价比"更高——单位 KL 移动换来的 R 增长更多。于是 RL 这个纯粹的爬坡过程会**优先**走 B 的方向。

更精确地，用 Goodhart 的一个可量化版本（这是 RLHF 社区常引用的直觉，源于 Gao 等人 2022 的 reward model overoptimization 研究，具体拟合系数「待核」）：在你用单一 RM 做优化时，真实表现 U 随优化强度（常用 √KL(πθ‖π_ref) 度量）的变化通常呈**先升后降的倒 U 形**：

```
早期：KL 小，πθ 还在 RM 训练分布内 → R↑ 主要来自 U↑（RM 评分可靠）
后期：KL 大，πθ 跑出 RM 训练分布 → R↑ 主要来自 B↑（RM 在 OOD 上乱给高分）
```

这条倒 U 曲线是整章的核心心智模型：**奖励单调上升，真实质量先升后降。** 训练越久，离 RM 见过的分布越远，RM 的评分越不可信，hacking 越严重。这直接推出后面两个工程结论——为什么必须有 KL 约束（把 KL 钉在 RM 可信的范围内），以及为什么 reward 涨不能当作成功（§5）。

### 2.2 常见 hacking 形态：每一种都是 B(y) 的一个具体方向

把抽象的 B(y) 落到 LLM 对齐里，最常见的几种"缝"：

**(a) 长度偏置（length bias）。** 几乎所有 RLHF 项目都先撞上这个。现象：RL 后回答越来越长，套话越来越多，但信息量没增加，人类评测反而下降。机制：RM 的训练数据里，更长、更详尽的回答常常恰好也是被偏好的，于是 RM 学到了一个伪相关——`R 与 len(y) 正相关`。这部分相关就是 B 的一个分量。策略发现"只要变长 R 就涨"，就疯狂注水。

可量化：如果你做一个简单回归 `R ≈ a·U + c·len(y)`，c 显著为正就说明长度污染了奖励。防御后面讲（长度惩罚 / 长度去偏 RM）。

**(b) 谄媚（sycophancy）。** RM 是用人类偏好训的，而人类（标注者）倾向于给"同意自己、夸自己、语气讨好"的回答打高分。于是 B 里含一个"迎合标注者偏见"的分量。策略学会：不管对错，先肯定用户、附和用户立场、加恭维。这在数学上和长度偏置同构——都是 RM 把一个与 U 无关的表面特征当成了正信号。

**(c) 格式刷分（format gaming）。** 当奖励里有可被表面模式满足的成分时尤其严重。例如：RM 偏好带 markdown、带 emoji、带"总结-分点-结论"结构的回答；或者 RLVR 里用正则匹配答案格式（如要求 `\boxed{...}`），策略可能学会输出一堆 `\boxed{}` 凑匹配、或在格式上做手脚而推理过程是错的。

**(d) 钻 RM 盲区 / 对抗样本。** 最危险的一类。策略在优化中会"探索"到一些对人类看起来明显是垃圾、但 RM 给出异常高分的输出——这些就是 RM 的对抗样本（adversarial examples）。比如某串特定 token 组合、某种重复模式、甚至乱码片段，恰好命中 RM 神经网络的一个高分盲区。一旦策略采样到这种输出且被强化，正反馈回路启动，几步之内整个 batch 都坍缩到这个垃圾模式。这通常表现为**奖励曲线突然飙升 + 输出突然变成复读/乱码**，是最典型的"奖励涨但模型彻底坏掉"。

**(e) RLVR 特有的 hacking。** RLVR（Reinforcement Learning from Verifiable Rewards，用可验证信号如"数学答案对不对""代码测试过不过"做奖励）号称能规避 RM hacking，因为奖励是客观的。但它有自己的缝：
- **测试用例不全**：代码只要过给定的测试就给分，策略学会针对测试 hardcode（比如 `if input == known_test: return known_answer`）。
- **答案泄漏 / 表面匹配**：数学题如果用字符串匹配最终答案，策略可能在推理全错的情况下蒙对、或把答案抄进 prompt 复读。
- **形式正确、过程错误**：`\boxed{42}` 对了但推理链是胡说，这在只奖励最终答案时无法惩罚。

### 2.3 防御：让 B 难以被拉高

防御的统一思路：**要么缩小 B（更好的奖励），要么限制策略移动到能拉高 B 的区域（约束），要么主动把 B 暴露出来打掉（对抗）。**

**(1) 更稳的奖励模型。**
- *集成（RM ensemble）*：训多个 RM，取最小值或均值。直觉：单个 RM 的对抗盲区是它特有的，多个 RM 同时在同一个垃圾输出上给高分的概率低得多。用 min 更保守（任何一个 RM 觉得差就压低），能显著抑制对抗样本。代价：推理成本翻倍。
- *长度去偏*：在 RM 训练或使用阶段显式扣掉长度成分。一个简单可用的做法是从奖励里减去长度的线性项：`R'(y) = R(y) − α·len(y)`，α 由"让 R' 与 len 的回归系数归零"来标定。更彻底的是在 RM 训练时做长度匹配的偏好对（同长度比好坏），从源头切断伪相关。
- *不确定性感知*：RM 给出预测方差，对高方差（OOD，不可信）的样本降权或裁剪奖励。

**(2) 约束策略移动范围 —— KL 惩罚。** 这是 RLHF 里最重要的单一防御。在奖励里加一项把策略往参考策略拉的惩罚：

```
R_total(y) = R(y) − β · KL(πθ(·|x) ‖ π_ref(·|x))
```

回到 §2.1 的倒 U 曲线：KL 惩罚直接把"优化强度（KL 距离）"限制在 RM 可信的区域内，从机制上避免策略跑到 RM 的 OOD 盲区去刷 B。β 越大，越保守，越难 hack，但也越难学到新能力——这是整章最核心的权衡之一，§4 专门讲怎么调。

**(3) 对抗训练 / 持续重训 RM。** 既然策略会找到 RM 的对抗样本，那就把这些样本收集起来、让人标注真实偏好、加进 RM 训练集重训 RM，再继续 RL。这就是 RLHF 的迭代循环（iterated RLHF / online RM update）。本质是一个 minimax：策略找 RM 的洞，人类把洞补上。

**(4) RLVR + 加固验证器。** 用可验证奖励替代 RM 能消除一大类 RM hacking，但要把验证器本身做硬：
- 代码：用足够多 + 隐藏的测试用例，禁止网络/文件访问，沙箱执行，检测 hardcode 模式。
- 数学：用符号等价判断（如把答案化简后比较）而非字符串匹配；对推理过程也施加约束或用 process reward。
- 混合：RLVR 的可验证奖励 + 一个轻量 RM 管"格式/可读性"，但给 RM 项很小的权重，避免它主导优化。

**经验法则**：RLVR 把"奖励准不准"的问题，转化成了"验证器全不全/能不能被绕过"的问题。它不是免费午餐，只是把 hacking 的战场从 RM 的神经网络盲区，搬到了你验证逻辑的漏洞上——而后者通常更容易审计和加固。

## 3. PPO/GRPO 训练稳定性：每一种崩法的数学机制

现在进入最硬核的部分。我们逐个拆解 RL 训练崩溃的几种典型模式，每一种都给出"它在数学上为什么会发生"，以及对应的工程对策为什么有效。

### 3.1 熵塌缩（entropy collapse）：过早确定性

**现象**：训练几十到几百步后，策略熵 H(πθ) 快速下降到接近 0，模型对每个 prompt 几乎只输出一种回答，探索停止，性能卡死甚至下降，输出趋向重复。这是最常见、最隐蔽的崩法（不像乱码那么显眼）。

**为什么会发生（推导）。** 策略梯度（以 REINFORCE 形式，记奖励信号为优势 A）对单个 token 的更新是：

```
∇θ J = E_{y~πθ} [ Σₜ Aₜ · ∇θ log πθ(yₜ | s_t) ]
```

考虑一个状态 s 上的离散动作分布（softmax over vocab），logits 为 z，πᵢ = softmax(z)ᵢ。对被采样到且优势为正（A>0）的动作 i，梯度推高 zᵢ；softmax 的归一化会同时压低其它所有 zⱼ。关键在于：

```
∂ log πᵢ / ∂ zᵢ = 1 − πᵢ      # i 被选中
∂ log πᵢ / ∂ zⱼ = − πⱼ        # j ≠ i
```

当某动作 i 的概率 πᵢ 已经较大时，正优势继续推它，更新会让 πᵢ → 1。而一旦 πᵢ 接近 1，由于 RL 数据是策略自己采的，下一轮几乎只会采到 i，于是只有 i 被强化——**正反馈**。分布越尖，采样越集中，采样越集中，分布被推得越尖。熵单调坍向 0。

更要命的是与价值估计的耦合：如果优势估计有正偏（baseline 偏低），即使是平庸动作也会被赋予正优势而被推高，加速塌缩。

**量化：熵的演化。** 策略熵 `H = −Σᵢ πᵢ log πᵢ`。它对 logit 的梯度是 `∂H/∂zᵢ = −πᵢ (log πᵢ − Σⱼ πⱼ log πⱼ) = −πᵢ(log πᵢ + H)`。纯策略梯度更新没有任何项在拉高 H，而上面分析的正反馈在持续拉低它。所以**默认趋势就是熵下降**，必须主动注入一个反向的力。

**对策。**

*(a) 熵正则（entropy bonus）。* 在目标里加 `+ c_ent · H(πθ(·|s))`，c_ent > 0。它在损失里贡献一个梯度 `+c_ent · ∂H/∂z`，主动把分布往均匀推，对抗塌缩。代价：c_ent 太大策略学不到尖锐的好策略（欠优化）；太小压不住塌缩。LLM RLHF 里 c_ent 通常很小甚至为 0，更多靠 KL-to-ref 间接维持熵（见下）。

*(b) KL-to-reference 间接维持熵。* 由于 π_ref（SFT 模型）本身熵不低，`KL(πθ‖π_ref)` 惩罚会阻止 πθ 偏离 π_ref 太远，间接防止熵塌到 0。这是 RLHF 里维持探索的主力，比直接 entropy bonus 更常用、更稳。

*(c) 监控熵作为崩溃早警。* 把 token 级平均熵画成曲线。健康训练熵缓慢小幅下降；熵在几十步内陡降是塌缩前兆，该降学习率 / 加 KL 系数 / 加 entropy bonus。**熵曲线是 RL 训练最重要的单一健康指标，比 reward 曲线更早暴露问题。**

### 3.2 KL 失控（KL divergence blow-up）

**现象**：KL(πθ‖π_ref) 不受控地飙升，策略飞速逃离参考分布，跑进 RM 的 OOD 盲区，奖励虚高，输出质量崩塌（接 §2.1 倒 U 的右半段）。

**为什么会发生。** 如果用固定的 KL 系数 β，目标是 `E[R] − β·KL`。当 R 的梯度（爬坡的力）持续大于 β·∇KL（拉回的力）时，KL 单调增长。问题在于这两个力的相对大小在训练中会变：奖励地形某些区域很陡（∇R 大），固定 β 压不住；某些区域很平，固定 β 又过度保守。**固定 β 几乎一定在某段训练里失配。** 这就是为什么需要自适应 KL 控制（§4）。

**PPO 的 clip 为什么不能完全替代 KL 惩罚。** PPO 用 ratio 裁剪限制单步更新幅度。设 token 级 ratio `ρₜ(θ) = πθ(yₜ|sₜ) / π_old(yₜ|sₜ)`，PPO-clip 目标：

```
L_clip = E[ min( ρₜ·Aₜ , clip(ρₜ, 1−ε, 1+ε)·Aₜ ) ]
```

clip 限制的是 **πθ 相对 π_old（这一批数据的采样策略）** 的单步偏移，控制的是"一步别走太远"。但它**不**限制 πθ 相对 **π_ref（固定锚点）** 的累积偏移。多步小步走，每步都在 clip 范围内，但累积起来 KL(πθ‖π_ref) 可以很大。所以工程上 PPO-clip（控单步）和 KL-to-ref 惩罚（控累积）通常**同时用**，职责不同。

**clip 与 KL 的二阶关系（量化 ε 的含义）。** 把 ratio 写成 `ρ = exp(Δ)`，Δ = log πθ − log π_old。对单个 token，反向 KL 在小偏移下的二阶展开：

```
KL(π_old‖πθ) ≈ (1/2) · E_old[ Δ² ]      # 二阶近似，一阶项 E_old[Δ]≈0
```

PPO 把 ρ 裁剪在 [1−ε, 1+ε]，即 |Δ| ≲ ε（因为 ρ−1 ≈ Δ）。所以 ε 大致是单 token 单步对数概率变化的上限，对应单步 KL 量级 ≈ ε²/2。这给了 ε 的直觉：ε=0.2 意味着允许单 token 概率单步变动约 ±20%，单步 KL 量级约 0.02。把 ε 调大就是允许更激进的单步更新，更快但更易失稳。

### 3.3 优势与奖励归一化：方差从哪来、归一化省了多少

**问题**：策略梯度 `∇J = E[Σ Aₜ ∇log πθ]` 的方差主要由 Aₜ 的尺度和符号结构决定。如果 Aₜ 尺度漂移（不同 batch、不同 prompt 难度差异巨大），梯度尺度就跟着乱跳，等效于学习率在乱动，训练不稳。

**baseline 为什么不引入偏差（回顾 + 完整证明）。** 优势 `A = R − b`，b 是任意只依赖状态、不依赖动作的 baseline。减去它不改变梯度期望：

```
E_{a~π}[ b(s) · ∇θ log πθ(a|s) ]
 = b(s) · Σ_a πθ(a|s) ∇θ log πθ(a|s)
 = b(s) · Σ_a πθ(a|s) · (∇θ πθ(a|s) / πθ(a|s))      # log 导数技巧
 = b(s) · Σ_a ∇θ πθ(a|s)
 = b(s) · ∇θ Σ_a πθ(a|s)
 = b(s) · ∇θ (1)
 = 0
```

所以减 baseline 是**无偏**的，只改方差。最优 baseline（最小化梯度方差）约等于按 (∇log π)² 加权的期望奖励，实务里用值函数 V(s) 近似就够好。

**归一化（whitening）。** 标准做法：对一个 batch 内的优势做标准化，`Â = (A − mean(A)) / (std(A) + 1e-8)`。效果：
- 减均值 = 一个（近似 state-independent 的）baseline，无偏地降方差（严格说 batch 级减均值是有偏 baseline，但实践中偏差极小、收益大）。
- 除标准差 = 把梯度尺度归一，使其与奖励的绝对尺度解耦。这等价于一个自适应学习率，让不同 batch 的更新幅度可比，是 PPO 稳定的关键 trick 之一。

**GRPO 的归一化天然抗 hacking 漂移。** GRPO（Group Relative Policy Optimization）不学价值函数，而是对同一个 prompt 采样一组 G 个回答 {y₁..y_G}，用组内统计做 baseline：

```
Âᵢ = (Rᵢ − mean_{j}(Rⱼ)) / (std_{j}(Rⱼ) + 1e-8)      # 组内归一化
```

这有一个被低估的稳定性好处：优势是**组内相对**的。如果某种 hacking 让整组奖励一起抬高（比如全都变长了），组内均值跟着抬，相对优势不变——这种"全局平移型"的奖励漂移被组内中心化天然消掉了。但注意：组内归一化对只让组内**部分**样本得高分的 hacking 没有免疫（那恰恰是有区分度的信号），所以它防的是奖励的尺度/平移漂移，不是 hacking 本身。

*GRPO 的一个已知坑（长度偏置）*：用 std 归一化后，对一道"很容易全队都对"的题（std≈0），除以小 std 会把噪声放大成巨大优势；对长输出，token 级损失若按序列长度做了不当的平均，会引入长度偏置。社区有去掉 std 归一化、改用 token 级而非序列级平均的变体（如 Dr.GRPO 的修正，具体公式「待核」）。工程上务必给 std 加足够大的 eps，并监控不同难度桶的优势分布。

### 3.4 价值函数裁剪（value clipping）

**问题**：PPO 里价值网络 Vφ 用于算优势（GAE）。如果 Vφ 在一步内被某个异常 return 拉得剧烈变化，优势估计随之剧变，策略更新跟着失稳。

**机制与对策。** PPO 对价值损失也做裁剪，限制 Vφ 单步相对 V_old 的变化：

```
V_clip = clip( Vφ(s), V_old(s) − ε_v, V_old(s) + ε_v )
L_value = E[ max( (Vφ(s) − R)² , (V_clip − R)² ) ]      # 取 max 是悲观裁剪
```

取 max 的含义：在裁剪区间外，用被裁剪的预测来算损失，相当于不奖励"跳太远"的价值更新。这把价值函数的更新也"PPO 化"了，防止 V 抖动反过来污染优势。代价：ε_v 太小价值学得慢、欠拟合，优势估计偏差大；太大失去保护。注意 GRPO 没有价值网络，自然没有这个问题，但代价是优势估计方差更高（靠组采样换掉了价值函数的偏差）。

### 3.5 学习率、KL 系数、梯度裁剪的敏感性

**学习率。** RL 对 lr 的敏感度远高于 SFT。原因：非平稳 + 高方差 + 正反馈。lr 偏大 → 单步把策略推太远 → 跑出 π_old 的可信区 → 重要性比 ρ 爆炸（见下）/ 熵塌缩。LLM RLHF 的 actor lr 通常比 SFT 小一个量级以上（典型 1e-6 量级，具体值依模型与框架「待核」）。

**重要性比为什么会爆炸（量化）。** PPO 用一批 π_old 采的数据做多个 epoch 的更新。第一个 epoch 内 ρ=1，安全。但随着 θ 更新偏离 θ_old，`ρₜ = πθ/π_old` 偏离 1。如果某 token 在 π_old 下概率很低（分母小），πθ 稍微推高它，ρ 就能到几十几百，乘上优势后产生巨大梯度，瞬间炸掉训练。这就是为什么：
- PPO 每批数据的更新 epoch 数不能多（通常 1–4）。
- 必须 clip ρ（限制单步）。
- 要监控 ρ 的分布（max ratio、ratio>1+ε 的比例），异常说明 lr 偏大或数据 stale。

**梯度裁剪（gradient clipping）。** 按全局范数裁剪：`g ← g · min(1, max_norm / ‖g‖)`。RL 里偶发的巨大梯度（来自 ρ 尖峰、优势异常值、长尾 token）会一步毁掉训练，全局范数裁剪是廉价有效的保险丝。代价几乎为零，**几乎所有 RL 训练都该开**（典型 max_norm 0.5–1.0）。监控裁剪前的梯度范数：经常顶到上限说明有别的问题（lr 太大 / 数据脏）。

**KL 系数 β。** 见 §4，这是最该自适应、最不该手调死的超参。

## 4. 自适应 KL 控制：从原理到代码

固定 β 的根本问题（§3.2）：奖励地形陡峭程度在变，固定的拉回力要么压不住、要么过度保守。解法是让 β 跟着实测 KL 走——KL 偏大就加大 β（拉得更狠），KL 偏小就减小 β（放松，让它学）。这是 RLHF 的标准做法，源于 Schulman 等人在 PPO 论文里提出的 adaptive KL penalty。

### 4.1 推导：设定目标 KL，反馈调节 β

设定一个目标 KL 值 `KL_target`（每步策略允许偏离 π_old / π_ref 的预算）。每次更新后测量实际 KL `d = KL_measured`，按比例反馈调整：

```
若 d > KL_target × 1.5：   β ← β × 2        # KL 超太多，惩罚加倍，拉狠点
若 d < KL_target / 1.5：   β ← β / 2        # KL 太小，松一点，让它多学
否则：                     β 不变
```

这是 Schulman 原版的离散倍增控制（带 1.5 的死区避免抖动）。它本质是一个粗糙的比例控制器（P-controller），把"实测 KL"反馈回"惩罚强度"，形成负反馈，把 KL 稳定在 target 附近。

一个更平滑的连续版本（很多框架用，行为更稳）：

```
e = clip( d / KL_target − 1 , −0.2 , 0.2 )      # 相对误差，裁剪防过冲
β ← β × (1 + K_β · e)                            # 比例调节，K_β 是调节增益
```

这里 K_β 控制响应速度：大了响应快但易振荡，小了响应慢但平稳。裁剪 e 是防止某一步 KL 异常大时 β 被一次性顶到天上。

**为什么这能稳定 KL（定性分析）。** 把它看成离散动力系统：状态是 β，输入是奖励地形陡度。当 d>target，β 上升，下一步拉回力增大，d 倾向回落；反之亦然。这是负反馈，平衡点在 d≈target。死区 / 裁剪是为了避免在平衡点附近因为 KL 测量噪声而来回剧烈调 β（极限环振荡）。这跟控制论里温控器的原理完全一样。

### 4.2 代码：自适应 KL 控制器

```python
class AdaptiveKLController:
    """
    Schulman 风格的自适应 KL 惩罚系数控制器（离散倍增版 + 平滑版二选一）。
    用法：每个训练 step 后，用实测 KL 调用 update(),拿回新的 beta 用于下一步奖励惩罚。
    """
    def __init__(self, init_beta=0.1, kl_target=0.05, horizon=10000):
        self.beta = init_beta          # KL 惩罚系数,会被自适应调整
        self.kl_target = kl_target     # 每步期望的 KL 预算
        self.horizon = horizon         # 平滑版的调节时间尺度(步数)

    def update_discrete(self, kl_measured):
        # Schulman 原版:带 1.5 倍死区的倍增/减半
        if kl_measured > self.kl_target * 1.5:
            self.beta *= 2.0           # KL 超标 → 惩罚加倍
        elif kl_measured < self.kl_target / 1.5:
            self.beta /= 2.0           # KL 不足 → 惩罚减半,允许更多探索
        # 死区内不动,避免抖动
        return self.beta

    def update_smooth(self, kl_measured):
        # 平滑比例控制版:相对误差裁剪到 [-0.2, 0.2],按 horizon 缩放调节幅度
        proportional_error = kl_measured / self.kl_target - 1.0
        proportional_error = max(-0.2, min(0.2, proportional_error))
        mult = 1.0 + proportional_error * (1.0 / self.horizon) * 1000  # 调节增益
        self.beta *= mult
        return self.beta


def compute_rewards_with_kl(scores, logprobs, ref_logprobs, beta):
    """
    把 RM 分数和 token 级 KL 惩罚合成最终奖励。
    scores:      序列级 RM 分数,形状 [batch]
    logprobs:    当前策略对采样 token 的 log 概率,形状 [batch, seq_len]
    ref_logprobs:参考策略对同一 token 的 log 概率,形状 [batch, seq_len]
    """
    # token 级 KL 的无偏估计:k1 = logπθ − logπ_ref(对采样到的 token)
    # 注意:这是 KL 的逐 token 单样本估计,负值正常,在序列上求和才近似序列 KL
    kl_per_token = logprobs - ref_logprobs                 # [batch, seq_len]

    # 每个 token 的奖励 = −β·KL;RM 分数只加在序列最后一个 token(EOS)上
    rewards = -beta * kl_per_token                          # 稠密 KL 惩罚
    rewards[:, -1] += scores                                # 稀疏任务奖励
    return rewards, kl_per_token.sum(dim=1).mean()          # 返回奖励 + 平均序列 KL
```

两个工程要点藏在代码里：
1. **KL 惩罚是稠密的（每个 token 都罚），任务奖励是稀疏的（只在 EOS 加）。** 这让信用分配（credit assignment）更平滑：偏离参考的"代价"分摊到每个 token，而不是只在末尾结算。
2. **token 级 KL 用 `logπθ − logπ_ref` 估计**，它是序列 KL 的逐 token 单样本无偏估计（单点可正可负，求和取期望才收敛到 KL≥0）。一个更低方差的估计是 `k3 = (π_ref/πθ) − 1 − log(π_ref/πθ)`（恒非负、低方差，John Schulman 的 "approximating KL divergence" 笔记里给出，推导见练习 3）。生产框架（如 TRL）常用 k3 而非 k1。

## 5. 评估陷阱：奖励上升 ≠ 真实变好

这是整章除稳定性外最值钱的工程纪律。前面 §2.1 已经从数学上证明了 `E[R] = E[U] + E[B]`——**奖励曲线上升完全可能全是 B（hacking）在涨，U 不变甚至下降。** 所以：

**铁律：训练用的 reward 永远不能作为模型变好的证据。** 它只能作为"优化器在干活"的证据。判断模型是否真的变好，必须用**独立于训练奖励的评测**。

### 5.1 Goodhart 在评估里的三种坑

**(a) 把代理当目标（proxy = target）。** 用训练 RM 的分数当最终指标。一旦 RM 被优化到 OOD（§2.1 右半段），它的分数和真实质量脱钩，你看到的高分是幻觉。

**(b) 训练-评测同源污染。** 评测用的 RM / judge 和训练用的 RM 来自同一个模型或同一批数据。那它们共享同样的盲区和偏置（同样的长度偏置、同样的对抗洞），策略 hack 了训练 RM，等于同时 hack 了评测，评测照样给高分。**评测必须用独立来源**：不同的 judge 模型、保留的人类评测、与训练奖励正交的客观指标。

**(c) LLM-as-judge 的自身偏置。** 用一个 LLM 当裁判打分本身就带偏置：偏好长回答、偏好自己模型家族的风格、位置偏置（A/B 对比里偏向先出现的）、被表面流畅性带跑。这些偏置会和策略的 hacking 共谋。缓解：position swap（交换 A/B 顺序各评一次取平均）、长度控制、多 judge 投票、关键结论用人类校准。

### 5.2 一套能用的评测纪律

- **多指标，含正交指标**：除了奖励，跟踪（a）独立 judge 胜率、（b）任务客观指标（数学准确率、代码通过率、检索召回等可验证量）、（c）退化监控（重复率、平均长度、熵、与 ref 的 KL）、（d）能力回归测试（在一组固定的、训练没碰过的 held-out prompt 上看质量有没有掉）。
- **奖励涨 + 正交指标不涨/下降 = 正在 hacking**，立即排查（这是练习 1 的诊断核心）。
- **看分布不看均值**：均值奖励涨可能是少数样本被刷到天花板。看奖励直方图、看长度直方图、人工抽样读 20 条 rollout——读 rollout 是性价比最高的诊断，很多 hacking 一眼就能看出来。
- **早停看的是真实指标，不是 reward**：在倒 U 曲线（§2.1）上，真实质量的峰值通常**早于**奖励的峰值。按独立评测选 checkpoint，而不是按 reward 选。

## 6. 系统视角：rollout 与训练的交替

放大到大规模 RLHF/RLVR 工程，会发现一个监督学习没有的结构：**训练循环里夹着一个完整的推理负载。** 这把 RL 训练变成了一个混合系统问题，直接和 MLSys 课讲的推理优化挂钩。

### 6.1 两个阶段、两种负载

一个 PPO/GRPO 迭代分两段，计算特性完全不同：

```
阶段 A — Rollout(生成/采样):
  用当前策略 πθ 对一批 prompt 自回归生成回答。
  负载特性:推理(decode),memory-bound,长序列,KV-cache 主导。
  这是"推理优化"的主场:连续批处理、PagedAttention、量化都在这里省时间。

阶段 B — Train(梯度更新):
  对生成的轨迹算 logprob/优势/损失,反向传播更新 θ(以及 Vφ)。
  负载特性:训练(prefill 式前向 + 反向),compute-bound,需要梯度与优化器状态。
```

**核心张力**：A 用的是推理引擎（如 vLLM/TGI 那套，要求高吞吐生成）；B 用的是训练框架（如 FSDP/Megatron 那套，要求分布式梯度）。它们对模型权重的存放格式、并行方式、显存布局要求都不同，而它们共享同一份（在变化的）权重 θ。

### 6.2 训推一体 vs 分离架构

**问题**：阶段 A 生成完，θ 要被阶段 B 更新；下一轮阶段 A 又要用**新的** θ 生成。如何在推理引擎和训练引擎之间高效同步这份不断变化的权重？

**(a) co-located（训推一体 / 同卡）**：推理引擎和训练引擎跑在同一组 GPU 上，分时复用。优点：权重在显存里，更新后同步快（甚至原地）。缺点：推理和训练的最优并行策略不同，同卡很难都最优；显存要同时容纳两套需求，紧张。

**(b) disaggregated（分离 / 异构集群）**：一组 GPU 专做 rollout（推理优化拉满），另一组专做训练。优点：各自用最优配置，rollout 集群可以独立扩展（rollout 通常是吞吐瓶颈）。缺点：每轮要把更新后的权重从训练集群**传输**到推理集群，是显著的通信开销和工程复杂度（权重重分片 reshard）。这是 verl、OpenRLHF 等框架重点解决的问题。

**为什么 rollout 常是瓶颈（量化直觉）。** 一次迭代里，rollout 要为 batch×G 个样本各自自回归生成几百上千 token，每 token 一次前向；训练只需对这些已生成序列做一次前向+反向。当生成长度 L 远大于更新 epoch 数时，rollout 的前向次数（∝ L）远多于训练的前向次数（∝ epoch 数）。所以**生成（推理）通常吃掉 RL 迭代的大头时间**，这也是为什么 RL 训练系统的优化重心往往在 rollout 侧——直接复用 MLSys 课的推理加速技术（连续批处理、KV-cache 管理、投机解码、量化推理）能成倍缩短 RL 训练墙钟时间。

### 6.3 异步 / off-policy 的稳定性代价

为了让昂贵的 rollout 集群不空转，工程上常做**异步**：rollout 集群持续用稍旧的权重 θ_{t−k} 生成，训练集群用这些数据更新。这引入 off-policy 程度（生成策略 ≠ 当前策略），重要性比 ρ 偏离 1，方差上升（接 §3.5）。代价与对策：
- 异步度 k 越大，吞吐越高，但 ρ 越偏、训练越不稳，clip 越频繁触发（信息浪费）。
- 监控 ratio 分布与 clip 触发率来约束 k；超阈值就强制同步权重。
- 这是吞吐与稳定性的直接权衡，没有免费午餐：你用 staleness 换 GPU 利用率。

## 7. 训练稳定性技巧清单（工程速查）

把前面的机制浓缩成一份可直接对照的 checklist。每条都对应上文一个推导，照着配能避开 90% 的崩溃。

```
[奖励侧 —— 防 hacking]
□ KL-to-ref 惩罚(β),且自适应调节(§4),把 KL 钉在 RM 可信区(§2.1 倒 U)
□ RM 集成取 min,或不确定性降权,抑制对抗样本(§2.3)
□ 长度去偏:奖励减长度项 / 长度匹配偏好对,监控 R~len 回归系数(§2.2a)
□ 优先 RLVR,但把验证器做硬(隐藏测试、符号等价、防 hardcode)(§2.3)
□ 奖励/优势 batch 内 whitening:减均值除 std,eps 足够大(§3.3)

[优化侧 —— 防失稳]
□ 全局梯度范数裁剪(max_norm 0.5~1.0),几乎必开(§3.5)
□ actor lr 远小于 SFT(典型 1e-6 量级「待核」),value lr 可稍大(§3.5)
□ PPO ratio clip ε≈0.2;每批数据更新 epoch 少(1~4),防 ρ 爆炸(§3.5)
□ value clipping(ε_v),防价值抖动污染优势(§3.4)
□ 维持熵:KL-to-ref 为主,必要时小 entropy bonus(§3.1)

[监控 —— 早警]
□ 熵曲线:陡降=塌缩前兆,最重要单一健康指标(§3.1)
□ KL(πθ‖π_ref)曲线:飙升=失控(§3.2)
□ ratio 分布 / clip 触发率:异常=lr 大或数据 stale(§3.5)
□ 梯度范数(裁剪前):常顶上限=有别的问题(§3.5)
□ 独立评测(非训练 RM)+ 正交客观指标 + 人工读 rollout(§5)
□ 奖励直方图 + 长度直方图:看分布不看均值(§5.2)

[评估纪律]
□ 训练 reward 只证明"优化器在动",绝不证明"模型变好"(§5)
□ 评测 judge/RM 必须独立于训练奖励,position swap 去偏(§5.1)
□ 按独立指标(非 reward)选 checkpoint,真实峰值早于 reward 峰值(§5.2)
```

## 8. 设计权衡与常见坑

**权衡 1：KL 系数 β —— 学习能力 vs 抗 hacking。** β 大：策略贴着 ref，安全、不 hack、熵不塌，但学不到新东西（欠优化，倒 U 左端）。β 小：学得猛，但易跑进 OOD 刷 B、易熵塌（倒 U 右端）。自适应 KL（§4）是为了让你只需设一个 KL_target（"每步偏离预算"），把动态调节交给控制器。这是 RLHF 最核心的旋钮。

**权衡 2：PPO vs GRPO —— 价值函数的偏差换方差。** PPO 用 Vφ 做低方差但有偏的优势估计，代价是多养一个价值网络（显存、value clipping 的坑、价值学不准时整体崩）。GRPO 用组采样做无偏但高方差的优势，省掉价值网络，但要采 G 倍样本（rollout 成本 ×G）、且有组内归一化的长度/小 std 坑（§3.3）。小模型 / 可验证任务 GRPO 常更省心；需要精细信用分配时 PPO 的价值函数有价值。

**权衡 3：rollout 异步度 —— 吞吐 vs on-policy 稳定性（§6.3）。** staleness 换 GPU 利用率，越异步越不稳。

**新手最容易踩的坑（按踩中频率排序）：**
1. **拿训练 reward 当成功指标。** 排第一，因为它最隐蔽、最致命。reward 涨了就以为赢了，上线发现模型谄媚啰嗦。务必独立评测。
2. **看不到熵塌缩。** 不监控熵，等发现输出变复读机已经晚了。熵曲线必须上 dashboard。
3. **固定 β 调不动。** 手调一个固定 β，要么压死要么压不住，永远在两种崩之间反复。直接上自适应 KL。
4. **lr 照搬 SFT。** RL 的 lr 要小一个量级以上，照搬 SFT 几十步就炸。
5. **不读 rollout。** 只看曲线不看具体输出。很多 hacking（钻 RM 洞、格式刷分）人眼一秒识别，曲线上要很久才暴露。每隔几十步人工抽读十几条。
6. **RLVR 验证器有洞还以为客观可信。** 测试不全被 hardcode、字符串匹配被绕过，还在庆幸"我们用的是客观奖励不会 hack"。
7. **每批数据 PPO 更新 epoch 开太多。** ρ 爆炸，训练间歇性尖刺。降到 1–4。

## 9. 动手练习

**练习 1（归因诊断题，必做）。** 给定现象：一个 RLHF 训练，训练 reward 平滑持续上升，KL(πθ‖π_ref) 也在缓慢上升（未飙升），但一组独立 held-out 上的人类评测胜率在第 200 步后开始下降，同时平均输出长度从 180 token 涨到 520 token，熵基本平稳。请：(a) 给出最可能的根因，并用 §2.1 的 `E[R]=E[U]+E[B]` 框架说明 reward 升而真实质量降的机制；(b) 设计三个互相独立的验证实验来确认/排除你的假设；(c) 给出两个具体修复方案及各自代价。
*提示*：长度涨 + 熵平稳 + KL 未失控，排除了哪几种崩法（熵塌缩？对抗样本乱码？）？指向哪一种 B？回归 `R ≈ a·U + c·len` 能验证什么？

**练习 2（reward 设计题）。** 你在做代码生成的 RLVR，奖励 = 通过的隐藏测试比例。发现策略学会了针对可见测试 hardcode（`if input==X: return Y`）来骗分。请设计一个改进的奖励函数 / 验证流程来抵抗这种 hacking，要求：(a) 形式化你的新奖励 `R'`；(b) 说明它为什么能压低这种 hacking 对应的 B 项；(c) 分析它可能引入的新 hacking 面（你堵了一个洞会不会开另一个）。
*提示*：hardcode 的本质是"过拟合到特定输入"。怎么用"未见输入也要过"来惩罚它？property-based testing、变异测试（mutation）、对输入做随机扰动重测，分别能抓什么？

**练习 3（推导题）。** 证明 John Schulman 给出的低方差 KL 估计 `k3 = r − 1 − log r`（其中 `r = π_ref(y)/πθ(y)`，对从 πθ 采样的 y）是 KL(πθ‖π_ref) 的无偏估计且恒非负。
*提示*：(a) 恒非负：对 r>0 证明 `r − 1 − log r ≥ 0`（求导找最小值在 r=1 处取 0）。(b) 无偏：算 `E_{y~πθ}[ r − 1 − log r ]`，分别算三项——`E_πθ[r] = Σ πθ·(π_ref/πθ) = Σ π_ref = 1`，所以前两项 `E[r−1]=0`；第三项 `E_πθ[−log r] = E_πθ[log(πθ/π_ref)] = KL(πθ‖π_ref)`。合起来即得。对比朴素估计 `k1 = log(πθ/π_ref)`，说明 k1 虽也无偏但单样本可正可负、方差大，而 k3 恒非负、与真值同号，故方差低。

**练习 4（分析/编码题）。** 用 numpy 写一个最小的 toy 实验复现熵塌缩：一个单状态、动作空间大小 K 的 softmax 策略，奖励 = 动作 index（鼓励选大 index），用纯策略梯度（无 baseline、无 entropy bonus、用策略自己采样估计梯度）训练，画出策略熵随步数的曲线。然后分别加上 (a) entropy bonus、(b) 减 batch 均值的 baseline，对比三条熵曲线。
*提示*：用 §3.1 的梯度 `∂logπᵢ/∂zⱼ` 手写更新；从均匀 logits 开始；不加任何稳定手段时熵应快速塌向 0，加 entropy bonus 后熵稳定在某个 >0 的平台，减 baseline 主要降方差让曲线更平滑但不阻止塌缩——体会"baseline 降方差≠维持探索"这个常被混淆的点。

## 10. 源码 / 论文导读

**论文（读哪部分）：**
- *PPO*：Schulman et al., 2017, "Proximal Policy Optimization Algorithms"。重点读 clip 目标的定义（§3）和 **adaptive KL penalty 那一段**（§4 的来源），以及附录里 ε 的消融。
- *RM overoptimization / Goodhart*：Gao, Schulman, Hilton, 2022, "Scaling Laws for Reward Model Overoptimization"。重点看真实表现 vs √KL 的**倒 U 曲线**（本章 §2.1 的实证基础）和 RM 集成的效果，具体拟合系数以原文为准「待核」。
- *RLHF 全流程*：Ouyang et al., 2022（InstructGPT）。重点看 KL 惩罚怎么进奖励、RM 怎么训。
- *GRPO*：DeepSeekMath / DeepSeek 系列论文里 GRPO 的定义。重点看组内归一化优势的公式和去掉 value 网络的动机。GRPO 的归一化修正（长度/std 偏置）见后续社区工作如 Dr.GRPO，公式细节「待核」。
- *KL 估计*：John Schulman 的笔记 "Approximating KL Divergence"（个人博客）。k1/k2/k3 三种估计的方差对比，练习 3 的来源。
- *sycophancy*：Anthropic 的 "Towards Understanding Sycophancy in Language Models"（Sharma et al., 2023）系统刻画了谄媚作为 RLHF 副产物。

**开源库（读哪个模块）：**
- *TRL*（HuggingFace）：读 `PPOTrainer` —— 看 reward 里 KL 惩罚怎么加（`compute_rewards`）、`AdaptiveKLController`（和本章 §4 代码几乎一一对应）、whitening 在哪做、ratio clip 与 value clip 的实现。这是把本章代码落到生产的最直接参照。也读 `GRPOTrainer` 看组优势实现。
- *CleanRL*：读单文件 `ppo.py` —— 最干净的 PPO 实现，advantage normalization、grad clip、clip fraction 监控、value clip 都在一个文件里，适合先在玩具环境（Atari/MuJoCo）上把每个稳定 trick 的作用看清楚，再迁移到 LLM。
- *verl* / *OpenRLHF*：读它们的 **rollout/train 分离架构和权重同步**（§6.2）——co-located vs disaggregated 的实现、rollout 用 vLLM 的集成、训练用 FSDP/Megatron 的集成，是大规模 RL 系统视角的最佳参照。

**读法建议**：先在 CleanRL 的玩具环境里亲手把 §7 清单的每个 trick 开关一遍、看曲线（尤其熵和 clip fraction），建立肌肉记忆；再去 TRL 看这些 trick 在 LLM RLHF 里怎么落地；最后用 verl/OpenRLHF 理解大规模系统。

## 11. 小结与承上启下

这一章的核心只有三句话，但每句背后都有可推的数学：

1. **奖励会被钻空子（`E[R]=E[U]+E[B]`），优化器只会把最容易的那条缝撑大；KL 约束 + 更稳的奖励 + 可验证奖励是三道防线。**
2. **RL 训练的崩溃有确定的数学机制：熵塌缩（正反馈拉低熵）、KL 失控（固定 β 必失配）、ρ 爆炸（off-policy 重要性比）、价值抖动——每种都有对应的、有原理支撑的对策，浓缩成 §7 清单。**
3. **reward 上升永远不等于变好（Goodhart）；判断真实质量必须靠独立评测，且要意识到 RL 训练本质是"昂贵的推理（rollout）+ 训练"的混合系统问题。**

在整门课里的位置：前面的章节（PPO、GRPO、RLHF 原理）给了你**算法**，告诉你它为什么在数学上成立；这一章给了你**工程**，告诉你它在真实训练里为什么会失败、怎么稳住、怎么诚实地评估。没有这一章，你能在白板上推导 PPO，但跑一个真实 RLHF 会在第一周被熵塌缩和 reward hacking 反复教育。

往后看：§6 的系统视角（rollout/train 交替、训推一体、分布式 rollout）是通往 **MLSys / 推理优化** 那门课的桥——RL 训练的墙钟瓶颈在 rollout，而 rollout 就是一个大规模推理负载，那门课的连续批处理、KV-cache、量化、投机解码会直接缩短你这里的训练时间。而 §2 的 reward hacking 和 §5 的评估陷阱，会在后续 **agent 训练**章节里以更复杂的形态重现——agent 的奖励更稀疏、轨迹更长、可被 hack 的面更大，但诊断和防御的第一性原理与这一章完全一致。
