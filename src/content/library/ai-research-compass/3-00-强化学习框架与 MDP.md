> 这章把你从"会训分类器、会推公式但没碰过 RL",带到能把任意试错型任务严格写成一个 MDP 五元组,能从第一性原理推出回报为什么收敛、状态价值与动作价值为什么这样定义、它们彼此怎么换算,并能亲手写一个小环境 + 随机策略 rollout 把回报算出来。后面贝尔曼方程、策略梯度、PPO、DPO、GRPO 全部是在这一章定义的对象上做文章——地基歪一寸,上面塌一丈。

## 一、为什么需要一套新框架:RL 不是监督学习换个皮

你已经熟练监督学习。给一堆 `(x, y)`,定义损失 `L(f(x), y)`,对参数求梯度下降,完事。强化学习不能这么干,而且**四个根本性的差异**让你过去的全部直觉都需要重新校准。把这四点想透,你就理解了为什么后面要绕那么大一圈。

**差异一:没有标注 `y`,只有标量反馈 `r`。** 监督学习里每个输入都配一个"正确答案"。RL 里没有人告诉你"在这个状态下正确动作是 a*"。环境只在你做完动作后给一个**标量奖励**(reward),比如 +1 / −1 / 0。这个反馈信息量远低于标注:它只说"这步好不好",不说"什么才是最好"。围棋下完赢了给 +1,但没人告诉你第 37 手该下哪。

**差异二:奖励延迟且稀疏。** 关键反馈往往在很久之后才出现。下棋走 200 步,只在终局拿到一个胜负信号;迷宫走 50 步才碰到出口。问题随之而来——**这 200 步里哪几步是功臣、哪几步是罪人?** 这叫**信用分配问题**(credit assignment),是 RL 最硬的难题之一(本课 13 章专门攻它)。监督学习没有这个问题,因为每个样本的损失当场结清。

**差异三:数据分布随策略改变,不是 i.i.d。** 这是最容易被工程师忽略、却最致命的一点。监督学习假设训练样本独立同分布(i.i.d),而且数据集是**固定的**。RL 里,你看到什么数据**取决于你当前的策略**:策略爱往左走,你就只见到左边的状态;策略一更新,数据分布立刻跟着变。**优化目标和数据生成过程被同一组参数耦合在一起。** 这个反馈环是 RL 训练极不稳定的根源——你改进策略 → 数据分布漂移 → 之前学到的价值估计失效 → 训练震荡甚至发散。监督学习里你绝不会遇到"优化着优化着训练集自己变了"。

**差异四:必须主动探索。** 既然数据由策略生成,而你一开始的策略很烂,你就有可能**永远见不到好的状态**。一家你从没点过的餐厅可能是全城最好吃的,但你不点就永远不知道。这逼出**探索 vs 利用**(exploration vs exploitation)的根本张力:**利用**当前认为最优的动作拿确定的收益,还是**探索**没试过的动作博取可能更高的未知收益?监督学习里数据已经摆在那,无所谓探索。这个张力贯穿全部 RL,02 章会用多臂老虎机正面展开。

把这四点合起来看,RL 的处境是:**在一个会因你的行为而改变、反馈延迟且信息稀薄、还必须自己去采样数据的环境里,学一个能最大化长期累积收益的决策规则。** 这是一个全新的问题类别,需要全新的数学语言。这个语言就是**马尔可夫决策过程**(MDP)。

> 一句话锚定全课:**RL 研究的是 agent 通过与环境反复交互、在试错中学习一个策略,以最大化长期累积奖励的期望。** 这门课从这一句话出发,一路推到能在工业界训 RLHF、推理模型和 agent。

## 二、交互的形式语言:智能体–环境循环

先把对象摆清楚。RL 把世界切成两半:**智能体**(agent,你要训练的决策者)和**环境**(environment,其余一切)。两者按离散时间步循环交互:

```
在时间步 t:
  1. agent 观察到环境状态   s_t ∈ S
  2. agent 依策略选一个动作  a_t ∈ A,  a_t ~ π(·|s_t)
  3. 环境转移到新状态并吐出奖励:
        s_{t+1} ~ P(·|s_t, a_t),   r_{t+1} = R(s_t, a_t, s_{t+1})
  4. t ← t+1,回到第 1 步
```

注意奖励的下标约定:**在 s_t 做了 a_t,得到的奖励记作 r_{t+1}**(这是 Sutton & Barto 教科书的标准约定,本课沿用)。直觉是"奖励和它带你去的下一个状态同时到达"。把这个循环连起来,得到一条**轨迹**(trajectory,也叫 episode、rollout 或 trace):

```
τ = (s₀, a₀, r₁, s₁, a₁, r₂, s₂, a₂, r₃, …)
```

**马尔可夫性质**(Markov property)是整个框架的核心假设,务必理解到位:**未来只依赖当前状态,与历史无关。** 形式化:

```
P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, …, s₀, a₀)  =  P(s_{t+1} | s_t, a_t)
```

直觉解释:**状态 s_t 是历史的一个"充分统计量"**——只要你知道现在的状态,过去怎么走到这里的对预测未来毫无额外帮助。下棋时当前棋盘布局就是状态,你不需要知道落子顺序;但如果状态只给你棋子位置而漏掉"轮到谁走",它就不满足马尔可夫性。**马尔可夫性不是环境的固有属性,而是你怎么定义状态的结果。** 把它做满足是 RL 建模的一项核心手艺:状态信息不全 → 非马尔可夫 → 后面所有理论(贝尔曼方程、价值函数)都站不住。这点新手极易栽跟头——他们把"看得见的观测"直接当状态,却没意识到观测里缺了决定未来的关键变量。

## 三、MDP 五元组:把问题钉死

一个**马尔可夫决策过程**由五个对象唯一确定,写作 `(S, A, P, R, γ)`:

- **S — 状态空间(state space):** 所有可能状态的集合。可以有限(gridworld 的格子)、可数无限、或连续(机器人关节角度 ∈ ℝⁿ)。
- **A — 动作空间(action space):** agent 可选动作的集合。离散(上下左右)或连续(方向盘转角)。严格说应记 `A(s)`(不同状态可选动作不同),多数情况简化为统一的 A。
- **P — 状态转移概率(transition dynamics):** `P(s′ | s, a) = Pr(s_{t+1}=s′ | s_t=s, a_t=a)`,描述环境的动力学。它对每个 `(s,a)` 给出下一状态的概率分布,满足 `Σ_{s′} P(s′|s,a) = 1`。**这是环境的"物理定律",agent 通常不知道它**——知道叫 model-based(07 章),不知道叫 model-free(02–06 章主线)。更一般地可写联合分布 `p(s′, r | s, a)`,把奖励的随机性也包进去。
- **R — 奖励函数(reward function):** 给出每步的标量奖励。常见三种等价写法:`R(s)`、`R(s,a)`、`R(s,a,s′)`,或直接定义期望奖励 `r(s,a) = E[r_{t+1} | s_t=s, a_t=a]`。**奖励是 RL 唯一的优化目标信号,设计它是一门危险的艺术**——设计歪了,agent 会找到你没料到的捷径来刷高分,这就是贯穿全课的 **reward hacking**(14 章详述)。
- **γ — 折扣因子(discount factor):** `γ ∈ [0, 1]`,衡量未来奖励相对当前的折价。下一节专门推它为什么必须存在。

**奖励假设**(reward hypothesis,Sutton 的核心信条):**任何目标都可以被表达为"最大化一个标量奖励信号的累积期望"。** 这是一个很强、也常被质疑的假设(真有那么多目标能压缩成一个标量吗?),但它是整个 RL 大厦的公理。你接受它,后面一切才成立。

**老虎机 = 单状态 MDP。** 多臂老虎机(multi-armed bandit)是 MDP 的退化特例:只有一个状态 `|S|=1`,没有状态转移(拉哪个臂都回到同一状态),`γ` 无关紧要。它剥离了"序贯"和"信用分配",只留下纯粹的探索 vs 利用,所以是研究探索的最干净实验台(02 章主角)。**先在老虎机里把探索想清楚,再回到完整 MDP,是经典的学习路径。**

**POMDP(部分可观马尔可夫决策过程)一句话带过:** 现实中 agent 常常看不到完整状态,只能拿到一个**观测** `o`,而 `o` 不满足马尔可夫性(比如只看一帧画面看不出物体速度)。这就是 POMDP——理论上需要对历史/信念状态建模,工程上常用堆叠多帧、RNN/Transformer 编码历史来近似恢复马尔可夫性。**LLM agent 本质就是 POMDP**(看不到环境全貌),12 章会回到这点。

## 四、回报与折扣:为什么是 Σ γᵏ r,以及 γ 到底在干嘛

agent 要最大化的不是单步奖励,而是**长期累积奖励**。把从时刻 t 起的累积(打折)奖励定义为**回报**(return):

```
G_t  =  r_{t+1} + γ·r_{t+2} + γ²·r_{t+3} + γ³·r_{t+4} + …
     =  Σ_{k=0}^∞  γᵏ · r_{t+k+1}                          ……(定义式)
```

这个看似简单的式子,每一个设计选择都有硬道理。逐条拆开。

**(1) 为什么要折扣 γ?第一个理由:数学收敛。** 如果不打折(γ=1)且任务永不终止(continuing task,无限 horizon),`G_t = Σ r` 是无穷多项求和,完全可能发散到 +∞,那"最大化回报"就失去意义——无穷大没法比较大小。引入 `γ < 1` 后,只要奖励有界,回报必然有限。给出严格界:

```
设奖励有界:对所有步 |r| ≤ R_max。则对 γ ∈ [0,1):

   |G_t|  =  | Σ_{k=0}^∞ γᵏ r_{t+k+1} |
          ≤  Σ_{k=0}^∞ γᵏ |r_{t+k+1}|        (三角不等式)
          ≤  Σ_{k=0}^∞ γᵏ · R_max            (逐项放大到 R_max)
          =  R_max · Σ_{k=0}^∞ γᵏ
          =  R_max · 1/(1−γ)                  (几何级数求和,γ<1 时收敛)

   ∴  |G_t| ≤ R_max / (1−γ)   <  ∞           (回报有界,得证)
```

这里用到几何级数 `Σ_{k=0}^∞ γᵏ = 1/(1−γ)`(`|γ|<1`)。**这个上界 R_max/(1−γ) 极其有用**:它是价值函数的天然量纲,后面分析贝尔曼算子是 γ-压缩映射(01 章)、给价值估计做归一化(14 章)都要用它。记住它。

**(2) 第二个理由:时间偏好。** 即便能收敛,我们通常也更看重眼前的奖励(像金融里的贴现:今天的 100 块比明年的 100 块值钱)。`γ` 量化了这种"耐心程度"。

**(3) γ 是一个把 horizon 调成连续旋钮的参数。** 这是理解 γ 最深的一层。看权重序列 `1, γ, γ², γ³, …`,它呈几何衰减。定义**有效 horizon**(effective horizon)为权重衰减到可忽略的步数,一个常用的刻画是 `1/(1−γ)`:

```
   γ = 0     → 权重 (1,0,0,…),只看眼前一步,有效 horizon ≈ 1   (完全短视 myopic)
   γ = 0.9   → 有效 horizon ≈ 1/(1−0.9) = 10    步
   γ = 0.99  → 有效 horizon ≈ 1/(1−0.99) = 100  步
   γ = 0.999 → 有效 horizon ≈ 1000 步
   γ → 1     → 有效 horizon → ∞,几乎一视同仁地看到无限远  (完全远视 far-sighted)
```

**两个极端的行为差异(这是本章练习要你解释的核心):**

- **γ → 0(短视):** agent 只在乎 `r_{t+1}`,即下一步的即时奖励,完全不管长远后果。它会做出**贪图眼前、牺牲未来**的决策——比如为了立刻吃到旁边一颗小糖,放弃走两步去拿大蛋糕。优化目标退化成"逐步最大化即时奖励",几乎不需要规划。
- **γ → 1(远视):** agent 把遥远的未来奖励看得几乎和眼前一样重。它愿意为了终局的大奖**忍受长期的零奖励甚至负奖励**(比如迷宫里走很多步空步只为最后碰到出口)。但代价是:回报方差变大(累加了更多随机项)、学习更慢更难(信用要分配到很远),且 γ=1 时无限任务下回报可能不收敛。

**这就是 γ 隐藏的工程杠杆:它不只是个超参,它在调节问题的难度本身。** 调大 γ 意味着要求 agent 做更长程的规划,而长程规划的方差和信用分配难度都急剧上升(本课 05 章 GAE、13 章信用分配反复处理这个张力)。新手常把 γ 当成"随便设个 0.99",其实它和任务的内在 horizon 必须匹配:稀疏奖励的长任务设 γ 太小会让 agent 根本"看不到"终点的奖励。

**有限 horizon 任务**(episodic task,如一局游戏)会在某个**终止状态**(terminal state)结束。技巧上把终止状态设成"吸收态"(转移只能转回自己、奖励恒为 0),就能用同一个无限求和的定义式统一处理 episodic 和 continuing 两种情况——终止后那些恒为 0 的奖励项不影响求和。

最后,回报的**递归结构**——后面整个动态规划和贝尔曼方程的引擎,先在这里埋下:

```
   G_t  =  r_{t+1} + γ·r_{t+2} + γ²·r_{t+3} + …
        =  r_{t+1} + γ·( r_{t+2} + γ·r_{t+3} + … )
        =  r_{t+1} + γ·G_{t+1}                          ……(回报的一步递归)
```

**一句话:此刻的回报 = 即时奖励 + 折扣后的下一刻回报。** 这个朴素的拆分是 01 章贝尔曼方程的全部秘密——对它两边取期望,就得到价值的递归方程。先把它刻进脑子。

## 五、策略:agent 的决策规则

**策略**(policy)`π` 是 agent 的大脑——一个从状态到动作的映射。两种形式:

- **随机策略(stochastic):** `π(a|s) = Pr(a_t=a | s_t=s)`,对每个状态给出动作的概率分布,满足 `Σ_a π(a|s)=1`。
- **确定性策略(deterministic):** `a = μ(s)`,直接输出一个动作,是随机策略的特例(某个动作概率为 1)。

**为什么 RL 里默认用随机策略?** 三个理由,层层递进:① **探索**——随机性天然带来尝试不同动作的能力;② **可微优化**——`π_θ(a|s)` 对参数 θ 可导,才能跑策略梯度(04 章的全部基础);③ **博弈/部分可观下的最优性**——在 POMDP 或对抗博弈(石头剪刀布)里,最优策略本身就是随机的,确定性策略会被对手利用。**记住:在完整可观的 MDP 里存在确定性最优策略(01 章会证),但训练过程几乎总用随机策略,这两件事不矛盾。**

**关键澄清:策略只依赖当前状态,不依赖时间步 t。** 这叫**平稳策略**(stationary policy)。能这么做正是因为马尔可夫性——状态已经是充分统计量,知道"现在在哪"就够了,不需要知道"现在是第几步"。这一步把策略从"对整段历史的函数"压缩成"对单个状态的函数",是 RL 可解的关键简化。

策略 `π` 一旦固定,它和环境动力学 `P` 一起,就在轨迹空间上诱导出一个概率分布 `P(τ)`(给定 s₀):

```
P(τ | π)  =  ∏_{t=0}^{T-1}  π(a_t | s_t) · P(s_{t+1} | s_t, a_t)
```

**这就是差异三的数学根源:轨迹分布 P(τ|π) 里含着 π,改 π 就改了数据分布。** 04 章对 `J(θ) = E_{τ~P(τ|π_θ)}[G(τ)]` 求梯度时,难点正是梯度要穿过这个含 θ 的分布——log-derivative trick 就是为破这个局而生。

## 六、价值函数:衡量"处境有多好"

奖励 `r` 是即时的、短视的。我们真正想知道的是:**从某个状态出发,长期看能拿多少回报?** 这就是**价值函数**(value function)。注意——价值永远是相对于某个策略 π 定义的,**脱离策略谈价值毫无意义**(同一个状态,激进策略和保守策略下价值天差地别)。

**状态价值函数(state-value function)** `V^π(s)`:从状态 s 出发、之后一直遵循策略 π,所能得到的回报的期望。

```
V^π(s)  =  E_π [ G_t | s_t = s ]
        =  E_π [ Σ_{k=0}^∞ γᵏ r_{t+k+1}  |  s_t = s ]
```

这里期望 `E_π[·]` 是对"在 π 和 P 下展开的所有可能未来轨迹"求平均(动作的随机性来自 π,状态转移的随机性来自 P)。直觉:`V^π(s)` 回答"在策略 π 下,处在 s 这个局面值多少分"。

**动作价值函数(action-value function)** `Q^π(s,a)`:在状态 s **先强制执行动作 a**(哪怕 π 未必选它)、**之后**再遵循 π,所得回报的期望。

```
Q^π(s,a)  =  E_π [ G_t | s_t = s, a_t = a ]
```

`Q` 比 `V` 多固定了"第一步动作",这一点差别极其重要——**它让你能比较同一状态下不同动作的好坏,从而知道怎么改进策略**(02 章 Q-learning、04 章 advantage 全靠它)。`V` 只告诉你"这个局面值多少",`Q` 告诉你"在这个局面下,各个动作分别值多少"。

**两者的换算关系(必须烂熟,后面天天用):**

`V` 是 `Q` 对策略求期望——`V` 看的是"按 π 随机选动作后的平均价值":

```
   V^π(s)  =  Σ_a  π(a|s) · Q^π(s,a)        ……(V 由 Q 求策略期望得到)
```

反过来,`Q` 由 `V` 通过一步前瞻(one-step lookahead)得到——先吃即时奖励,再加折扣后下一状态的价值:

```
   Q^π(s,a)  =  E[ r_{t+1} + γ·V^π(s_{t+1}) | s_t=s, a_t=a ]
             =  Σ_{s′}  P(s′|s,a) · [ r(s,a,s′) + γ·V^π(s′) ]   ……(Q 由 V 一步前瞻得到)
```

**把这两式互相代入,就得到 01 章的贝尔曼期望方程**——所以贝尔曼方程不是天上掉下来的,它就是"`V↔Q` 换算 + 回报递归 `G_t=r+γG_{t+1}`"三者扣在一起的必然结果。我把这个推导在这里走一遍,让你看到它的有机性,01 章再正式严密化:

```
   V^π(s)  =  Σ_a π(a|s) Q^π(s,a)                              (用 V↔Q)
           =  Σ_a π(a|s) Σ_{s′} P(s′|s,a) [ r(s,a,s′) + γ V^π(s′) ]   (代入 Q 的前瞻式)
```

看最后一行:**`V^π` 用它自己(在下一状态 s′ 处的取值)来定义自己**——这就是价值函数的自指/递归本质,也是为什么 RL 能用"自举"(bootstrapping)来学习(02 章 TD)。

## 七、最优:π*、V*、Q* 与 RL 的终极目标

到此我们能评估任意给定策略了。但 RL 的目标是**找最好的策略**。

**策略的偏序关系:** 定义 `π ≥ π′` 当且仅当对**所有**状态 s 都有 `V^π(s) ≥ V^{π′}(s)`。注意这是个**偏序**(partial order)——可能两个策略各有所长(在不同状态上互有胜负),没法比较。但有限 MDP 的一个**深刻定理**(01 章证明)保证了好结局:

> **最优策略存在定理:** 任何有限 MDP 都存在至少一个策略 π*,它**同时**在所有状态上不劣于其他任何策略。而且,**总存在一个确定性的最优策略**。

这个定理是 RL 的"地基中的地基"——它保证了"寻找单一最优策略"这件事是良定义的(而不是陷入"东边最优西边不优"的死结)。**最优状态价值函数**与**最优动作价值函数**:

```
   V*(s)  =  max_π  V^π(s)         (所有策略中,该状态能达到的最高价值)
   Q*(s,a) =  max_π  Q^π(s,a)
```

它们之间有个漂亮的关系——**最优状态价值 = 在该状态选最优动作的 Q 值**:

```
   V*(s)  =  max_a  Q*(s,a)
```

直觉:最优策略在每个状态都会挑 Q* 最大的那个动作,所以状态的最优价值就等于其最佳动作的最优动作价值。**而且这直接给出了从 Q* 提取最优策略的方法**——这是 RL 最实用的结论之一:

```
   π*(s)  =  argmax_a  Q*(s,a)        ……(贪心于 Q* 即得最优策略)
```

**这一式的工程意义怎么强调都不过分:只要你拿到了 Q*,最优策略就是免费的——对每个状态取 argmax 即可,根本不需要环境模型 P。** 这正是整个价值类方法(02 章 Q-learning、03 章 DQN)的立身之本:它们绕过对 π 的显式建模,直接学 Q*,再贪心读出策略。而策略类方法(04–06 章)则反其道,直接参数化并优化 π。这两条技术路线的分野,根子就在这里。

把 `V* = max_a Q*` 和 `Q*` 的一步前瞻式扣起来,会得到**贝尔曼最优方程**——它和期望方程的唯一区别是把"对 π 求期望 `Σ_a π(a|s)`"换成了"取最大 `max_a`"。这个 `max` 带来了非线性,也带来了 01 章动态规划的全部精妙(以及 02 章 Q-learning 用随机近似求解它的思路)。本章先把对象定义清楚,方程的求解留给 01 章。

## 八、代码:从零搭一个 MDP + 随机策略 rollout

理论说够了,动手。我们实现两个最小可跑的 MDP,并用随机策略采样轨迹、按定义式算回报。**目的不是炫技,是让你亲眼看到 `S/A/P/R/γ`、轨迹 τ、回报 `G_t` 这些抽象对象在代码里长什么样。**

### 8.1 多臂老虎机(单状态 MDP):隔离探索问题

```python
import numpy as np

class BernoulliBandit:
    """k 臂老虎机 = 单状态 MDP。拉臂 i 以概率 p[i] 给奖励 1,否则 0。
    没有状态转移、没有 γ 的事——这正是它用来研究纯探索的原因。"""
    def __init__(self, probs):
        self.probs = np.array(probs)        # 各臂的真实中奖概率(agent 不知道)
        self.k = len(probs)

    def pull(self, arm):
        return float(np.random.rand() < self.probs[arm])   # 伯努利采样,返回 0/1 奖励

def random_policy_bandit(bandit, steps=1000, seed=0):
    """随机策略:每步均匀随机拉一个臂,统计平均奖励。"""
    rng = np.random.default_rng(seed)
    total = 0.0
    for _ in range(steps):
        arm = rng.integers(bandit.k)        # π(a|s) = 1/k,均匀随机
        total += bandit.pull(arm)
    return total / steps                    # 单状态、γ 无关,平均奖励即可衡量策略好坏

bandit = BernoulliBandit([0.2, 0.5, 0.75])  # 第 3 臂最优(0.75)
avg = random_policy_bandit(bandit, steps=20000)
print(f"随机策略平均奖励 ≈ {avg:.3f}")   # 理论值 = mean([0.2,0.5,0.75]) ≈ 0.483
# 关键观察:随机策略只能拿到三臂的"平均"≈0.483,
# 而最优策略(永远拉第 3 臂)能拿 0.75。这个 0.75−0.483 的差距
# 就是"不探索/不利用所付出的代价",02 章要用更聪明的策略把它逼近 0。
```

**这段代码隔离出的真相:** 老虎机里没有"序贯"也没有"信用分配"(每步奖励当场结清),剩下的纯粹是"我该把有限的尝试机会分给哪个臂"——这就是探索 vs 利用的最干净形态。`mean([0.2, 0.5, 0.75]) = 0.483…` 可手算核对,代码跑出来应该非常接近。

### 8.2 Gridworld(多状态 MDP):序贯决策登场

```python
import numpy as np

class GridWorld:
    """4×4 网格。从 start 出发,目标格 +1 并终止,陷阱格 −1 并终止,其余每步 0。
    动作:0=上 1=下 2=左 3=右。撞墙则原地不动。这是确定性转移的 episodic MDP。"""
    def __init__(self, gamma=0.9):
        self.n = 4
        self.gamma = gamma                          # ← MDP 五元组里的 γ
        self.goal = (0, 3)                          # 目标:+1
        self.trap = (1, 3)                          # 陷阱:−1
        self.start = (3, 0)
        self.actions = [(-1,0), (1,0), (0,-1), (0,1)]   # 上下左右 = 动作空间 A

    def is_terminal(self, s):                       # 终止状态(吸收态)
        return s == self.goal or s == self.trap

    def step(self, s, a):
        """环境动力学 P 与奖励 R。返回 (s', r, done)。本例 P 是确定性的。"""
        if self.is_terminal(s):
            return s, 0.0, True
        dr, dc = self.actions[a]
        nr, nc = s[0] + dr, s[1] + dc
        if not (0 <= nr < self.n and 0 <= nc < self.n):  # 撞墙:原地不动
            nr, nc = s
        s_next = (nr, nc)
        if s_next == self.goal:  return s_next, 1.0, True    # R: 到目标 +1
        if s_next == self.trap:  return s_next, -1.0, True   # R: 到陷阱 −1
        return s_next, 0.0, False                            # R: 其余每步 0

def rollout_return(env, max_steps=100, seed=0):
    """随机策略 rollout 一条轨迹 τ,并严格按定义式 G_0 = Σ γᵏ r_{k+1} 算回报。"""
    rng = np.random.default_rng(seed)
    s, traj, done = env.start, [], False
    for _ in range(max_steps):
        if done: break
        a = rng.integers(4)                 # 随机策略 π(a|s) = 1/4
        s_next, r, done = env.step(s, a)
        traj.append((s, a, r))              # 记录 (s_t, a_t, r_{t+1})
        s = s_next
    # 按定义式累加折扣回报:G_0 = r_1 + γ r_2 + γ² r_3 + …
    G, discount = 0.0, 1.0
    for (_, _, r) in traj:
        G += discount * r                   # 第 t 步贡献 γᵗ · r_{t+1}
        discount *= env.gamma               # 折扣逐步乘 γ
    return G, len(traj)

env = GridWorld(gamma=0.9)
returns = [rollout_return(env, seed=i)[0] for i in range(2000)]
print(f"随机策略下 V^π(start) 的蒙特卡洛估计 ≈ {np.mean(returns):.3f}")
# 这个均值就是 V^π(start) 的蒙特卡洛估计(02 章正式叫"蒙特卡洛预测"):
# 用大量轨迹回报的样本均值去逼近 E_π[G_0 | s_0=start]。
# 注意它通常 < 0:随机乱走很容易先撞进陷阱(−1),体现了随机策略之烂。
```

**这段代码把全部抽象对象落了地:** `env.step` 就是 `P` 和 `R`;`rng.integers(4)` 就是随机策略 `π`;`traj` 就是轨迹 `τ`;那个 `G += discount * r; discount *= gamma` 的循环**逐字实现了回报定义式 `G_t = Σ γᵏ r_{t+k+1}`**;最后对 2000 条轨迹取均值,就是 `V^π(start) = E_π[G_0|s_0=start]` 的蒙特卡洛估计。**把这两份代码跑通、改改 γ 看回报怎么变,你对本章全部概念的理解会从"看懂"变成"摸过"。**

## 九、设计权衡与常见坑

把抽象框架落到真实问题时,下面这些坑会反复咬人。它们多数不是算法问题,而是**建模问题**——而建模错了,再好的算法也救不回来。

- **坑 1:状态定义不满足马尔可夫性(头号杀手)。** 新手把"能观测到的东西"直接当状态,漏掉决定未来的关键变量。例:用单帧图像当状态去玩 Atari——单帧看不出小球速度方向,违反马尔可夫性,价值函数学不出来。**自检法:问自己"光看当前状态,能不能预测未来分布?如果还需要知道历史,那状态就缺信息。"** 修法:堆叠多帧、加入速度/计数器、用 RNN 编码历史。

- **坑 2:γ 当成无所谓的超参乱设。** γ 在悄悄定义问题的有效 horizon(第四节)。稀疏奖励的长任务里 γ 设太小(如 0.9,有效 horizon 才 10 步),agent 根本"看不到"几十步外的终点奖励,学不出长程行为。反过来 γ 太接近 1 会放大方差、拖慢收敛。**γ 必须和任务的内在 horizon 匹配,不是默认 0.99 拉倒。**

- **坑 3:奖励设计引发 reward hacking(最危险、最普遍)。** 奖励是 agent 唯一的指南针,它会**精确地优化你写下的东西,而不是你心里想的东西**。经典案例:训赛艇游戏 agent,本想让它赢得比赛,却按"得分"给奖励,结果它发现原地绕圈刷分箱比完赛得分更高,于是放弃比赛、原地打转刷分(OpenAI 的 CoastRunners 案例「待核:具体出处与细节」)。**这是 Goodhart 定律的 RL 版——"当一个度量变成目标,它就不再是好度量"。** 设奖励时务必反问:"有没有一条我没料到、能刷高分却违背我真实意图的捷径?"本课 08、10、14 章会从 RLHF/RLVR 角度反复回到这个问题。

- **坑 4:把延迟/稀疏奖励当成理所当然能学。** 只在终局给一个奖励、中间全是 0(如只在"赢了"才 +1),信用分配会极其困难——成百上千步里到底哪步是功臣?稀疏奖励是 RL 训练失败的常见原因(13 章用奖励塑形、HER、好奇心来攻它)。**建模时先评估你的奖励有多稀疏,这直接决定问题难度。**

- **坑 5:误以为"奖励上升 = 任务真的变好"。** 由于 reward hacking 和 Goodhart,训练曲线上的奖励涨,完全可能对应着模型钻了空子而真实能力没变甚至变差。**必须用一个独立的、agent 无法直接优化的指标来评测真实表现**(14 章评估纪律的核心)。

- **权衡:状态/动作空间的粒度。** 粒度越细,马尔可夫性越容易满足、表达越精确,但空间越大、样本需求越高(组合爆炸)。粒度越粗越省样本,但可能丢掉关键信息破坏马尔可夫性。这是建模时绕不开的取舍,没有银弹,取决于任务。

### 练习 1:学生 MDP 在给定策略下的价值(手算)

先把题目里的数据钉死。状态 `S = {学习, 玩手机, 睡觉}`,`睡觉` 是终止态,`γ = 0.5`。奖励算在**进入某状态**上(即转移后立刻拿到的 `r_{t+1}`):进入"学习"得 +2、进入"玩手机"得 −1、进入"睡觉"得 0。转移由策略选的动作确定性决定。策略 π:

- 在"学习"态:0.5 转回"学习",0.5 转到"玩手机"。
- 在"玩手机"态:0.5 转回"玩手机",0.5 转到"睡觉"。

**(a) 终止态价值**

终止态之后再无任何奖励,按定义 `V^π(睡觉) = 0`。

```
V^π(睡觉) = 0
```

**(b) 列方程组**

套用第六节末尾的式子 `V^π(s) = Σ_a π(a|s) Σ_{s′} P(s′|s,a)[ r(s,a,s′) + γ V^π(s′) ]`。这里转移确定,每个动作只对应一个 s′,所以内层 `Σ_{s′}` 退化成一项。为简洁记 `Vs = V^π(学习)`、`Vp = V^π(玩手机)`,`γ = 0.5`。

"玩手机"态(0.5 转回玩手机 r=−1,0.5 转去睡觉 r=0):

```
Vp = 0.5·[ (−1) + 0.5·Vp ] + 0.5·[ 0 + 0.5·V(睡觉) ]
   = 0.5·(−1 + 0.5·Vp) + 0.5·(0 + 0.5·0)
```

"学习"态(0.5 转回学习 r=+2,0.5 转去玩手机 r=−1):

```
Vs = 0.5·[ (+2) + 0.5·Vs ] + 0.5·[ (−1) + 0.5·Vp ]
```

这就是关于 `Vs, Vp` 的二元一次方程组。

**(c) 求解(从靠近终止的"玩手机"倒推)**

先解只含 `Vp` 的玩手机方程:

```
Vp = 0.5·(−1 + 0.5·Vp)
Vp = −0.5 + 0.25·Vp
Vp − 0.25·Vp = −0.5
0.75·Vp = −0.5
Vp = −0.5 / 0.75 = −2/3 ≈ −0.667
```

把 `Vp = −2/3` 回代学习方程:

```
Vs = 0.5·(2 + 0.5·Vs) + 0.5·(−1 + 0.5·(−2/3))
   = (1 + 0.25·Vs) + 0.5·(−1 − 1/3)
   = 1 + 0.25·Vs + 0.5·(−4/3)
   = 1 + 0.25·Vs − 2/3
0.75·Vs = 1 − 2/3 = 1/3
Vs = (1/3) / 0.75 = 4/9 ≈ 0.444
```

**结论:V^π(睡觉) = 0,V^π(玩手机) = −2/3 ≈ −0.667,V^π(学习) = 4/9 ≈ 0.444。**

验算(把解代回两条原方程):
```
玩手机:0.5·(−1 + 0.5·(−2/3)) + 0.5·0 = 0.5·(−4/3) = −2/3 ✓
学习:  0.5·(2 + 0.5·(4/9)) + 0.5·(−1 + 0.5·(−2/3))
      = 0.5·(2 + 2/9) + 0.5·(−4/3)
      = 0.5·(20/9) − 2/3 = 10/9 − 6/9 = 4/9 ✓
```

直觉:"玩手机"是个负价值的吸收漩涡(只会刷 −1 直到睡觉),所以为负;"学习"虽然有一半概率掉进玩手机,但另一半能持续刷 +2,净下来仍为正。

### 练习 2:γ→0 与 γ→1 的行为差异

**(a) 折现计算**

把"到达目标所需步数"对应的 +1 奖励折现到 t=0(走 n 步后到达目标,该 +1 出现在第 n 个奖励位上,折现权重为 `γ^(n−1)`;为和题目/提示口径一致,这里按提示直接算 `γ^n`,即把它当作"n 步之后的奖励"近似比较,核心是相对大小不是绝对位):

```
γ = 0.99:  0.99^6 ≈ 0.9415     0.99^8 ≈ 0.9227
γ = 0.5 :  0.5^6  = 0.015625    0.5^8  = 0.003906
```

关键看的是**两条路径的折现比** `γ^6 / γ^8 = 1/γ²`(近路 6 步比远路 8 步多赚的折现倍率):

```
γ = 0.99:  6步现值 / 8步现值 = 1/0.99² ≈ 1.020   (近路只比远路值钱约 2%)
γ = 0.5 :  6步现值 / 8步现值 = 1/0.5²  = 4.0     (近路是远路的 4 倍值钱)
```

**(b) 论证:为什么小 γ 偏好"快速到达"甚至冒险,大 γ 更有耐心绕远路**

- **小 γ(如 0.5):** 折现是几何衰减且衰减极快。同一个 +1,放在第 8 步只值 0.0039,放在第 6 步值 0.0156——晚两步价值就掉到原来的 1/4。于是 agent 对"早一步拿到奖励"极度敏感:6 步近路的折现回报远高于 8 步远路,即便近路紧贴 −1 陷阱、有踩坑风险,只要这风险带来的期望损失抵不过"早 2 步到达"的折现增益,它就会选近路冒险。再加上 −1 陷阱本身若发生也被强烈折现(踩坑越晚越不痛),进一步弱化了它对风险的顾忌。**小 γ = 短视 = 急着兑现 = 愿意为快冒险。**

- **大 γ(如 0.99):** 折现几乎不衰减。6 步和 8 步到达的 +1 现值只差约 2%(0.9415 vs 0.9227),"早到 2 步"带来的好处微乎其微。这点蝇头小利完全压不住踩进 −1 陷阱的期望损失(−1 的现值同样几乎不被折掉,该损失被"看得很重")。于是 agent 宁可多走 2 步选远离陷阱的安全路线,用一点点时间成本换取大幅降低的踩坑概率。**大 γ = 远视 = 不急于兑现 = 愿意为安全付出时间。**

一句话:**γ 控制了"时间成本"相对"风险成本"的兑换率——γ 小则时间贵、风险贱,催人冒进抄近路;γ 大则时间贱、风险贵,容人耐心绕安全路。**

**(c) 代码实验:把 gamma 改成 0.1 / 0.999,各跑 2000 条轨迹**

直接改 8.2 里 `GridWorld(gamma=...)`,跑 2000 条随机策略轨迹取均值。实测结果(`np.random.default_rng(seed=i)`,i=0..1999):

```
gamma = 0.1   →  V^π(start) 蒙特卡洛估计 ≈ −0.000   (实测约 -0.0000,极接近 0)
gamma = 0.9   →  V^π(start) 蒙特卡洛估计 ≈ −0.066
gamma = 0.999 →  V^π(start) 蒙特卡洛估计 ≈ −0.40
```

(注:具体小数随机种子不同会有波动,但量级与单调趋势稳定。)

可直接运行的实验脚本:

```python
import numpy as np

class GridWorld:
    """4×4 网格;goal=(0,3) 给 +1,trap=(1,3) 给 −1,其余每步 0;撞墙原地不动。"""
    def __init__(self, gamma=0.9):
        self.n = 4
        self.gamma = gamma                                  # ← 唯一要改的旋钮
        self.goal, self.trap, self.start = (0, 3), (1, 3), (3, 0)
        self.actions = [(-1, 0), (1, 0), (0, -1), (0, 1)]   # 上下左右

    def is_terminal(self, s):
        return s == self.goal or s == self.trap

    def step(self, s, a):
        if self.is_terminal(s):
            return s, 0.0, True
        dr, dc = self.actions[a]
        nr, nc = s[0] + dr, s[1] + dc
        if not (0 <= nr < self.n and 0 <= nc < self.n):     # 撞墙:原地不动
            nr, nc = s
        s_next = (nr, nc)
        if s_next == self.goal: return s_next, 1.0, True
        if s_next == self.trap: return s_next, -1.0, True
        return s_next, 0.0, False

def rollout_return(env, max_steps=100, seed=0):
    rng = np.random.default_rng(seed)
    s, traj, done = env.start, [], False
    for _ in range(max_steps):
        if done: break
        a = rng.integers(4)                 # 随机策略 π(a|s)=1/4
        s, r, done = env.step(s, a)[0], *env.step(s, a)[1:]  # 见下方说明,实际请用单次 step
        # —— 上一行只为示意;正确写法是单次调用:——
    # 正确实现:
    return _rollout(env, max_steps, seed)

def _rollout(env, max_steps, seed):
    rng = np.random.default_rng(seed)
    s, traj, done = env.start, [], False
    for _ in range(max_steps):
        if done: break
        a = rng.integers(4)
        s_next, r, done = env.step(s, a)    # 每步只调一次 step
        traj.append((s, a, r))
        s = s_next
    G, discount = 0.0, 1.0
    for (_, _, r) in traj:                  # G_0 = Σ γᵏ r_{k+1}
        G += discount * r
        discount *= env.gamma
    return G, len(traj)

for g in [0.1, 0.9, 0.999]:
    env = GridWorld(gamma=g)
    returns = [_rollout(env, 100, seed=i)[0] for i in range(2000)]
    print(f"gamma={g}: V^π(start) MC估计 ≈ {np.mean(returns):.4f}")
```

(说明:为避免 `env.step` 被调用两次的笔误,实际请直接用上面 `_rollout` 那段单次 step 的逻辑——它就是 8.2 原代码,只是把 gamma 参数化。)

**现象解释:**

- **γ = 0.1(≈0):** 折现极猛,有效 horizon ≈ 1/(1−0.1) ≈ 1.1 步。随机策略从 start 出发,绝大多数轨迹要走很多步才偶然撞到 ±1 终止格,而那个终局奖励被 `0.1^n`(n 通常 ≥5)折到几乎为 0(`0.1^5 = 1e-5`)。所以无论最后是 +1 还是 −1,折现后都≈0,均值自然趋近 0。**γ 太小让 agent"看不见"几步之外的终点奖励。**

- **γ = 0.9:** 有效 horizon ≈ 10 步,终局奖励能被部分"看见"。随机乱走从 (3,0) 出发,撞进陷阱 (1,3) 通常比绕到目标 (0,3) 更近更易发生,所以净估计为小负值(≈ −0.066)。

- **γ = 0.999:** 有效 horizon ≈ 1000 步,几乎不折现,终局的 ±1 被近乎原值地累计进回报。随机策略"先撞陷阱"的倾向被充分暴露,负值被放大(≈ −0.40);同时方差也明显变大(每条轨迹的 ±1 几乎不衰减地计入)。

**结论:随 γ 从 0.1→0.9→0.999,V^π(start) 的 MC 估计从 ≈0 单调走向更负,印证了第四节的论断——γ 越大,远处奖励(这里随机策略下更多是陷阱的 −1)被看得越重,且回报方差越大。**

### 练习 3:回报有界界 `|G_t| ≤ R_max/(1−γ)` 的紧致性(推导)

**(a) 构造使上界可达的 MDP 与策略**

让**每一步奖励都恒等于 R_max**,界即被取到。最简单的构造:

- 单状态 MDP:`S = {s}`,`A = {a}`(只有一个动作),转移 `P(s|s,a) = 1`(自环,永不终止,continuing task)。
- 奖励函数 `R(s,a,s) = R_max`(每步必拿满额 R_max)。
- 策略 π 只能选唯一动作 a(确定性,平凡)。

此时回报:

```
G_t = R_max + γ·R_max + γ²·R_max + …
    = R_max · Σ_{k=0}^∞ γᵏ
    = R_max · 1/(1−γ)
    = R_max/(1−γ)
```

**恰好等于上界,故界可达、是紧的。** 证毕。(界的推导本身在第四节用三角不等式+逐项放大到 R_max+几何级数得到;这里的构造让每一步不等式都取等号:`|r_{t+k+1}| = R_max` 处取等,所以整体取等。)

更现实一点的等价构造:任何"在所有可达 `(s,a,s′)` 上奖励恒为 R_max 的常数奖励 MDP",在该上界处取等的本质都是——每一步都拿到最大可能奖励、且任务无限延续。

**(b) 奖励无界(如 `r_t = t`)时使 G_t 仍收敛的 γ 条件**

奖励无界后,第四节"逐项放大到 R_max"那步失效,需要直接判级数

```
G_t = Σ_{k=0}^∞ γᵏ · r_{t+k+1}
```

的收敛性。取最坏情形 `r_{t+k+1} = t+k+1`(随 t 线性增长,无界),其量级与 `Σ_{k=0}^∞ γᵏ·(k+1)` 同阶。用**比值判别法**(ratio test),令 `a_k = γᵏ·(k+1)`:

```
|a_{k+1} / a_k| = γ^{k+1}(k+2) / [γᵏ(k+1)]
               = γ · (k+2)/(k+1)
               → γ        (当 k → ∞,因 (k+2)/(k+1) → 1)
```

比值极限为 γ。比值判别法:极限 < 1 则绝对收敛。故

```
只要 γ < 1,级数收敛 ⟹ G_t 有限。
```

**结论:对线性增长(乃至任意多项式增长 `r_t = poly(t)`)的无界奖励,γ < 1 这一条件依旧足以保证 G_t 收敛——几何衰减 γᵏ 压得住任何多项式增长。** 验算(闭式与数值一致):

```
Σ_{k=0}^∞ (k+1)·γᵏ = 1/(1−γ)²
   γ=0.5 → 4,   γ=0.9 → 100,   γ=0.99 → 10000   (数值求和完全吻合)
```

**附注(边界):** 比值判别法只在 `γ < 1` 给出收敛、在 `γ > 1` 给出发散;`γ = 1` 时判别法失效,而此时 `Σ(k+1)` 显然发散。所以对多项式增长奖励,收敛的充要条件就是 **`0 ≤ γ < 1`**。但要注意:若奖励是**指数增长** `r_t = b^t`(b>1),则 `a_k ∝ (γb)^k`,收敛条件变成 `γ·b < 1` 即 `γ < 1/b`,此时光有 `γ < 1` 不够——几何衰减只能压住"比 1/γ 慢的增长"。题中线性增长属于多项式,`γ < 1` 足矣。

### 练习 4:把"红绿灯路口"建模成 MDP(建模 + 概念)

**MDP 五元组设计**

- **S(状态空间):** 只有"当前位置"**不够**(回到坑 1:位置不是历史的充分统计量)。一个满足马尔可夫性的合理状态至少应包含:
  - `pos` — 车在路段上的位置(或"距下一个路口的距离");
  - `v` — 当前车速(决定下一秒位置怎么变,是关键的动力学变量);
  - `light` — 前方最近路口的当前灯色 {红, 黄, 绿};
  - `t_light` — 该灯色的剩余持续时间 / 配时相位(灯何时变,直接决定"现在能不能闯过去");
  - (可选,增强马尔可夫性)`idx` — 当前是第几个路口 / 是否已通过终点。
  即 `s = (pos, v, light, t_light, idx)`。**直觉判据:给定这个 s,下一秒的 (位置、车速、灯色) 分布就完全确定了,不需要再翻历史。**

- **A(动作空间):** 纵向控制即可。离散版 `A = {加速, 保持, 减速/刹车}`;连续版 `A = 加速度 a ∈ [a_min, a_max]`(含负值表示刹车)。

- **P(转移动力学):** 由简单运动学给出 `pos' = pos + v·Δt`、`v' = clip(v + a·Δt, 0, v_max)`;灯色按固定配时确定性推进(`t_light` 递减,到 0 切换 light 并重置)。可加噪声(如车速执行误差)使其随机。

- **R(奖励函数):** 编码"快速且安全通过"+"重罚闯红灯":
  - 每步给一个小的**时间惩罚** `−c_time`(c_time>0),鼓励尽快通过(等价于鼓励高效率,不要磨蹭);
  - 成功通过一个路口 / 到达终点给 **+1(或正奖励)**;
  - **闯红灯**(在 `light=红` 时位置越过停止线)给一个**大负奖励** `−C_runred`(C_runred ≫ 其他项,体现"安全是硬约束");
  - 可选:急刹/急加速给小负的**舒适性惩罚** `−c_jerk·|a|`,鼓励平顺驾驶;超速给负奖励。
  - 形如:`R = −c_time − c_jerk·|a| + (+1 若通过) + (−C_runred 若闯红灯)`。

- **γ:** 通过一串路口是中等 horizon 的 episodic 任务,取 `γ ∈ [0.95, 0.99]` 量级,使 agent 既看得到几个路口外的累计收益、又不过分放大方差。

**漏掉车速会不会破坏马尔可夫性?会。**

判据永远是"光看状态能否预测下一步的转移分布"。设想两辆车**位置完全相同**,但一辆以 60 km/h 行驶、一辆已刹停(0 km/h):

- 下一秒,高速那辆已冲过路口(可能恰好闯了红灯),刹停那辆还在原地。
- 即:**相同的"位置"对应了不同的下一状态分布**——`P(s' | pos)` 不是良定义的,必须额外知道车速才能确定 `s'`。

因此"位置"单独**不是历史的充分统计量**,状态缺了车速 → 转移不再只依赖当前状态 → 违反马尔可夫性 → 价值函数 / 贝尔曼方程全部失去依据。**修法:把车速 v(以及灯色剩余时间 t_light,同理它也影响"现在能否安全通过")纳入状态**,让 `s = (pos, v, light, t_light, …)` 重新成为充分统计量。

**结论:状态必须包含 `(位置, 车速, 灯色, 灯色剩余时间)` 这一组——任何一个能改变"下一秒转移分布"却被漏掉的变量(车速是最典型的),都会把问题悄悄变成非马尔可夫,使后续全部 RL 理论失效。这正是坑 1"把可观测量直接当状态"的典型翻车点。**
