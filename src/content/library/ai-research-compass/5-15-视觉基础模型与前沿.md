> 这一章把你从"知道有 SAM、知道能用文本检测物体"带到"能讲清可提示分割在数学上是什么、为什么数据引擎(而非模型)才是 SA-1B 的真正秘密、开放词表检测如何把 CLIP 的对齐塞进检测头、为什么视觉的'GPT 时刻'比 NLP 难得多(并能从离散性、任务异构性、监督结构三个角度论证)、以及对抗鲁棒性/长尾/域偏移这些公认难点的数学成因与防御思路在哪里失效"。这是计算机视觉专家课程的收束章,目标是让你站在 2024–2025 的前沿位置,知道哪些是已解的、哪些是没解的、以及没解的难在数学的哪一处。

## 1. 一句话引言

视觉的"基础模型"(foundation model)这个词,核心不是"模型很大",而是**一个预训练好的模型,无需为新任务重新训练,就能通过提示(prompt)或轻量适配迁移到大量下游任务**。本章的主线是:NLP 靠"下一个 token 预测 + 海量纯文本"自然长出了基础模型,视觉却没有等价物——**图像没有天然的离散 token,视觉任务比 NLP 异构得多,监督信号也贵得多**。SAM(Segment Anything Model)给出的答案是"换一个能规模化的任务(可提示分割)+ 一个能自我滚雪球的数据引擎";开放词表检测(Grounding DINO 等)给出的答案是"把 CLIP 的图文对齐嫁接到检测/分割的定位头上"。把这两条路讲透,再把"视觉为什么难统一"和"公认难点的数学根因"讲透,你就站到了这门课的终点,也是研究的起点。

## 2. 动机:NLP 已经有了基础模型,视觉为什么还没有

### 2.1 先把"基础模型"这个词拆干净

Bommasani 等人(2021,斯坦福 CRFM)把 foundation model 定义为"在大规模数据上训练、可适配(adapt)到广泛下游任务的模型"。落到可操作的层面,一个真正的视觉基础模型应当同时满足三件事:

1. **任务通用**:一个权重,覆盖分类、检测、分割、深度、跟踪……而不是每个任务一个 head 一套训练。
2. **零样本/少样本迁移**:遇到训练时没见过的概念或任务,不重训就能做(像 GPT 的 in-context learning,像 CLIP 的零样本分类)。
3. **可提示(promptable)**:用户用一句话、一个点、一个框告诉模型"我想要什么",模型即时响应。

NLP 在 2018–2020 间(BERT→GPT-3)一次性拿齐了这三件事。**视觉到今天(2025)都没有一个模型同时满足三条**——这就是本章要解释的"缺口"。

### 2.2 NLP 的基础模型为什么"自然"长出来

把 NLP 的成功拆成机制,你会发现它依赖三个恰好都成立的前提:

- **天然离散的 token**。文本本身就是离散符号序列,分词(BPE)只是再压缩一下,**自带一个有限词表**(GPT-2 约 5 万)。这让"预测下一个符号"成为一个**良定义的、闭集 softmax 分类问题**,可以用交叉熵直接优化。
- **统一的任务接口**。翻译、问答、摘要、分类——**几乎所有 NLP 任务都能写成"给一段文本,生成一段文本"**(text-to-text,T5 把这点讲到极致)。任务异构性被语言这个通用接口抹平了。
- **海量免费监督**。互联网文本本身就是"下一个词预测"的标注——**每段文本都是它自己的标签**(自监督),不需要人去标。规模可以无限堆。

三者合一,基础模型几乎是"数据 + 算力 + Transformer"的自动结果。

### 2.3 视觉同样三条,全都不成立(本章的核心论断)

把同样三条搬到视觉,**逐条崩溃**——这是理解"视觉为什么难"的钥匙,后面 §6 会再深挖:

```
NLP 前提                视觉对应物的状态
─────────────────────────────────────────────────────────
1. 天然离散 token   →   图像是连续像素栅格,没有天然词表。
                        要"token 化"必须人为切 patch(ViT)或
                        学一个离散码本(VQ-VAE),都是有损/人造的。
2. 统一任务接口     →   分类给一个标签、检测给框+类、分割给逐像素
                        掩码、深度给稠密实数图、跟踪给跨帧关联……
                        输出空间维度/类型天差地别,无单一接口。
3. 海量免费监督     →   "这张图配哪句话"是免费的(CLIP 走这条),
                        但"这张图里每个像素属于哪个物体"是天价
                        人工标注。任务越精细,标注越贵、越不可规模化。
```

**结论先行**:NLP 的基础模型是"任务统一 + 监督免费 + 离散接口"三个红利叠加的产物;视觉这三个红利**一个都不是免费的**,必须靠工程(数据引擎)和迁移(借 CLIP 的语言对齐)去人造出来。SAM 解的是第 2、3 条(造一个能规模化的统一任务 + 造数据),开放词表检测解的是借语言把第 1、2 条的"开集"打通。理解了这个映射,本章后面全是它的展开。

## 3. SAM 之一:把"分割"重新定义成一个可提示的任务

### 3.1 问题:分割为什么一直无法"基础模型化"

回忆分割那章:语义分割给每像素一个类标签,实例分割给每个物体一个掩码 + 类。两者都**绑死在一个固定类表上**(COCO 80 类、ADE20k 150 类),且**标注极贵**(逐像素描边,一张复杂图可能要几十分钟)。这正是 §2.3 第 2、3 条的叠加:任务被类表钉死(不通用),监督是天价(不可规模化)。

Kirillov 等人(Meta,2023,Segment Anything)的关键一招是**重新定义任务本身**:不要"分割出第几号类",而是做**可提示分割(promptable segmentation)**——

> 给定一张图和一个"提示"(一个点、一个框、一团粗掩码,或一段文本),输出**提示所指向的那个/那些区域的掩码**,而**完全不关心这个区域叫什么类**。

这一步重定义解掉了两个死结:

- **甩掉了类表**。任务不再是"分到 N 个固定类",而是"把提示指的东西抠出来"。**类别这个维度被彻底移除**,于是不再有闭集天花板——这是它能"零样本"到任意新物体的根本原因(它根本不认类,只认"这一坨")。
- **任务变得可规模化标注**。"点一下、出个掩码"是个能让模型辅助、能让标注员秒级确认的交互,这是 §4 数据引擎能滚起来的前提。

**这是本章第一个核心洞察:SAM 的突破首先是一次"任务重定义",把不可规模化的'闭集逐像素分类'换成了可规模化的'开集可提示抠图'。架构和数据引擎都是在为这个新任务服务。**

### 3.2 可提示分割的歧义性,以及它如何被建模

可提示分割有一个 NLP 里不存在的内在麻烦:**提示天然歧义**。你在一个人身上点一个点——你要的是这个人?他的衣服?还是他手里的杯子?**一个点对应多个语义上都合理的掩码**。

SAM 的处理方式很漂亮,直接把歧义建进输出:**对单个提示,模型输出 3 个掩码**(经验上对应"整体/部分/子部分"三个粒度,例如人/衣服/口袋),外加每个掩码的**置信度(IoU 预测)**。训练时用 **min-over-masks** 损失:

```
对一个歧义提示,模型给出 K=3 个候选掩码 {m̂₁,m̂₂,m̂₃} 和预测 IoU {ŝ₁,ŝ₂,ŝ₃}。
设该提示的真值掩码为 m*。
mask 损失只对"和真值最匹配的那一个"候选回传:
    k* = argmin_k  L_mask(m̂_k, m*)            # 只惩罚最接近的那个
    L = L_mask(m̂_{k*}, m*) + λ · L_iou(ŝ_{k*}, IoU(m̂_{k*}, m*))
其中 L_mask = 20·focal + dice (SAM 的配比,具体权重「待核」),
     L_iou  = 候选与真值 IoU 的回归(MSE)。
```

为什么这样能 work?**min-over-masks 是一种隐式的"多假设竞争"机制**:三个候选 head 会自发地分工去覆盖不同粒度,因为损失只奖励"最接近真值的那个",于是每个 head 倾向于专精一种解释,避免三个 head 互相平均成一团糊。这和检测里的多 anchor、集合预测里的匈牙利匹配是同一种思想——**用"赢者通吃"的稀疏监督逼出多样化的假设**。推理时,模型用预测的 IoU 分挑出最自信的那个掩码返回。

### 3.3 架构:为什么"重图像编码器 + 极轻提示解码器"是刻意设计

SAM 的三块结构,每一块的"重/轻"都是为可提示这个目标算计过的:

```
                ┌─────────────────────┐
   图像 (1024²) │  Image Encoder       │  ViT-H/L/B,MAE 预训练
        ───────▶│  (重,~632M @ ViT-H) │──▶ 图像嵌入 64×64×256
                └─────────────────────┘     (一张图算一次,可缓存!)
                ┌─────────────────────┐
   提示(点/框/ │  Prompt Encoder      │  点/框→位置编码;
   掩码/文本)  │  (极轻)              │  掩码→卷积下采样;文本→CLIP
        ───────▶└─────────────────────┘──▶ 提示嵌入
                ┌─────────────────────┐
                │  Mask Decoder        │  双向 cross-attention,
   图像嵌入 ───▶│  (极轻,2 层)        │  仅 ~ms 级
   提示嵌入 ───▶└─────────────────────┘──▶ 3 个掩码 + 3 个 IoU 分
```

**关键设计:把昂贵计算(图像编码)和廉价计算(提示解码)切开**。理由是交互式使用场景——用户会在**同一张图上反复点很多次**。如果每次点击都要重跑 ViT-H,延迟无法接受。SAM 的做法是:

- 图像编码器对一张图**只跑一次**(ViT-H 在 GPU 上约几百 ms),结果(64×64×256 的嵌入)缓存住;
- 之后每次提示,只跑**极轻量的 mask decoder**(双向 attention,两层),在浏览器/CPU 上都能做到约 50 ms 的实时响应(SAM 论文宣称 mask decoder 在 web 上 ~50ms,具体数「待核」)。

**这是本章第二个洞察:架构的"重头轻尾"不是为了精度,是为了把交互延迟从'每次点击重算 backbone'降到'每次点击只重算解码器'。这种"算一次重的、反复跑轻的"切分,是一切交互式大模型(也包括 LLM 的 KV-cache)的共同工程母题。**

mask decoder 的内部机制值得点一下:它在图像嵌入和提示 token(外加可学习的 output token、IoU token)之间做**双向 cross-attention**——既让提示去"看"图像,也让图像 token 被提示"调制"。这其实是一个极小的 DETR 式集合解码器(参见检测那章),output token 经 MLP 投影成掩码的动态卷积核,与上采样后的图像嵌入做点积得到掩码。**整个解码器之所以能这么小,是因为重活已经被 ViT-H 干完了,解码器只需做"提示 × 图像"的轻量匹配。**

### 3.4 量化:延迟从哪里省下来

设图像编码器一次前向耗时 T_enc(ViT-H 约数百 ms),解码器一次 T_dec(约 50 ms)。用户在一张图上点 n 次:

```
朴素方案(每次点击重跑全模型):    总耗时 = n · (T_enc + T_dec)
SAM 方案(编码缓存,只重跑解码):  总耗时 = T_enc + n · T_dec
当 T_enc ≫ T_dec(这里约 10×)且 n 较大时:
    加速比 ≈ n·(T_enc+T_dec) / (T_enc + n·T_dec)
           ──n→∞──▶  (T_enc+T_dec)/T_dec ≈ 11×    # 渐近上界
单看"第 2 次起的每次点击",延迟从 ~550ms 降到 ~50ms,约 11× 更快。
```

参数量级:ViT-H 图像编码器约 632M(占全模型的绝大部分),prompt encoder + mask decoder 合计仅约 4M（量级「待核」)。**这个 ~150:1 的参数失衡,正是"重图轻提示"设计的直接体现。**

## 4. SAM 之二:数据引擎——为什么数据比模型重要

### 4.1 鸡生蛋问题与"模型在环"的破法

要训一个能分割任何东西的模型,需要海量掩码标注;但海量掩码标注本身就是个天价工程(逐像素描边)。**这是个死循环:没好模型→标不快→没数据→训不出好模型。**

SAM 的 data engine 用**模型在环(model-in-the-loop)的滚雪球**破这个循环,分三阶段,每阶段模型越强、人越省力:

```
阶段 1  辅助手动 (assisted-manual)
   标注员用早期 SAM 在图上点/框,SAM 出掩码,人修正确认。
   人主导,模型打草稿。随着数据积累,模型每隔一阵重训一次。
   产出量级:~4.3M 掩码 / 12 万图(量级「待核」)

阶段 2  半自动 (semi-automatic)
   SAM 先自动分出它"有把握"的物体(置信掩码),
   人只去补标"模型漏掉的"那些物体 → 提升掩码多样性。
   人只补缺口,模型包圆容易的。
   产出量级:再加 ~5.9M 掩码（量级「待核」)

阶段 3  全自动 (fully-automatic)
   在图上撒规则网格点(如 32×32),每点都当提示跑 SAM,
   靠 IoU 预测 + 稳定性过滤 + NMS 去重,全自动产出掩码,无人参与。
   模型完全自主。
   产出量级:对 1100 万张图全自动标注 → SA-1B 共 ~11 亿 (1.1B) 掩码
```

最终 **SA-1B = 约 1100 万张授权高分辨率图 + 约 11 亿掩码**(这两个数是论文公开的核心数字),其中绝大部分掩码来自阶段 3 全自动,人完全没碰。

### 4.2 为什么"数据引擎"是范式级的关键,而不只是工程细节

把这件事讲透,需要看到它在**信息论和飞轮(flywheel)层面**的意义,而不仅是"标得快":

**(1) 它制造了一个正反馈飞轮。** 模型越强→自动标得越准越多→数据越多越广→重训后模型更强。**数据和模型互为燃料**,这正是基础模型"规模化"的本质:不是有了大数据才训大模型,而是**用模型去生产训自己的数据**。这与 LLM 的"用强模型生成合成数据训弱模型(蒸馏/self-instruct)"是同构的范式。

**(2) 它把人的角色从"生产者"降级为"校验者",这是规模化的数学前提。** 人工从零描一个掩码是 O(物体边界周长) 的操作;确认/否决一个模型给的掩码是 O(1) 的操作。**数据引擎的本质是把标注的人力复杂度从 O(N·边界) 压到 O(N) 甚至(阶段 3)O(0)**——只有把单位标注成本压到趋近于零,十亿级才可能。这呼应了你 know 里那条"数据引擎为什么关键":**它改变的不是标注速度,是标注的复杂度阶。**

**(3) 它用"覆盖度"而非"精标"对冲噪声。** 全自动产出的掩码必然有噪声(IoU 不完美、偶有错分)。但 SA-1B 赌的是:**11 亿个"大致对"的掩码,在覆盖度上的价值,远超 100 万个"完美但稀少"的掩码**。这和扩散模型那章"期望强制覆盖"是同一种信仰——**规模带来的分布覆盖,能压过单样本的标注噪声**。Kirillov 团队也做了人工质检抽样,自动掩码质量被评为相当高(具体 IoU 阈值与抽样比例「待核」)。

**这是本章第三个、也是最重要的洞察:SAM 之所以是"视觉走向基础模型"的范式样本,不在它的网络(就是个 ViT + 小解码器),而在它证明了"可规模化任务 + 模型在环数据引擎"能把视觉里最贵的稠密标注问题,转化成一个能自我滚雪球到十亿级的飞轮。范式 = 可提示任务(让模型能辅助) + 数据引擎(让数据能自产)。模型只是飞轮的一个零件。**

### 4.3 SAM 的零样本能力来自哪里,以及它不是什么

SAM 的"零样本"和 CLIP 的"零样本"是**两种不同的零样本**,务必分清:

- **CLIP 的零样本**是**语义零样本**:它能识别训练时没见过的**类别名**(因为类别变成了可比较的文本嵌入)。
- **SAM 的零样本**是**几何零样本**:它能分割训练时没见过的**物体/领域**(显微图、水下、卫星图……),因为它**根本不做语义分类**,只学"什么是一个连通的、有边界的物体"这一**类无关(class-agnostic)的几何先验**。它对你点的东西叫什么一无所知。

所以 SAM **不能**直接告诉你"这是猫还是狗"。**它是个分割引擎,不是识别引擎。** 这个边界极其重要:它解释了为什么 SAM 必须和别的东西(CLIP、检测器、人)组合才能做语义任务——下一节的开放词表分割,正是"SAM 出掩码 + CLIP/检测器赋予语义"的拼装。

## 5. 开放词表:用语言打通检测与分割的"开集"

### 5.1 问题:检测/分割如何摆脱固定类表

检测那章的检测头最后是 `cls_logits = W·feature`,`W ∈ ℝ^{C×d}`,C 是固定类数(COCO 80)。这和 CLIP 那章批判的"闭集分类头"是同一个病:**类数建图时钉死,新类要重标重训**。开放词表(open-vocabulary)检测/分割要解的就是:**让检测/分割能识别任意文本描述的类别,包括训练时没标注过的类**。

破法和 CLIP 一脉相承——**把固定的分类权重 W 换成"文本编码器现算的类别嵌入"**:

```
闭集检测头:    cls_logits[c] = w_c · region_feat        # w_c 是学出来的固定权重
开放词表检测头: cls_logits[c] = E_text("a photo of {类名_c}") · region_feat / τ
                                └── 用 CLIP 文本编码器现场算 ──┘
```

一旦分类权重来自文本编码器,**类表就活了**:推理时你给什么文本(哪怕训练时没标过的"柯基""路由器"),就能检测什么。这正是 CLIP 那章末尾埋的接口——**CLIP 把"类别"变成了可比较的向量,检测/分割只要把自己的分类头接到这个向量空间上即可继承开集能力。**

### 5.2 两条技术路线:后融合 vs 早融合

把语言注入定位,工程上有两条主线,理解其 trade-off 是这一节的重点:

**路线 A:区域 × 文本的"后融合"(以 ViLD、RegionCLIP、F-VLM 为代表)。**

```
1. 用一个 class-agnostic 的 proposal 网络(RPN/Mask2Former)产生候选区域(只定位,不分类)。
2. 对每个区域裁出特征,投影到 CLIP 视觉空间,得 region_emb。
3. 对每个候选类名,用 CLIP 文本编码器得 text_emb。
4. 分类 = region_emb 与各 text_emb 的余弦相似度(+ 蒸馏:让 region_emb 对齐
   CLIP 对该区域 crop 的图像嵌入,把 CLIP 的开集知识"蒸"进检测器)。
```

- 优点:**复用现成 CLIP,简单,定位与识别解耦**。SAM + CLIP 做开放词表分割就是这个套路(SAM 出掩码当 region,CLIP 给掩码区域打文本标签)。
- 缺点:**区域特征和文本是"事后"对齐的**,图文之间没有早期交互,对**指代性、关系性**的文本(如"左边那个穿红衣服的人")无能为力——它只会做"这块像不像'人'"这种类级匹配。

**路线 B:图文"早融合 + 定位即对齐"(以 GLIP、Grounding DINO 为代表)。**

GLIP(Li et al., 2022)的关键洞察是:**检测 = phrase grounding(短语定位)**。把"检测 COCO 80 类"重写成"在图里定位这句话提到的所有名词短语":输入图 + 文本提示(把所有类名拼成一句 prompt,如 `person. bicycle. car. ...`),模型输出每个区域和文本里**哪个词**对齐。这样:

```
检测 logits 不再是 region 对 C 个固定类,
而是 region 对"文本里的每个 token"的对齐分:
    alignment[i, j] = region_emb_i · token_emb_j        # i:区域, j:文本token
```

Grounding DINO(Liu et al., 2023)在此之上做**深层的跨模态早融合**:在 DINO(DETR 的强化版)的特征提取、query 选择、解码三个阶段都插入**图文 cross-attention**,让文本从一开始就参与视觉特征的形成。

- 优点:因为图文**早期就交互**,它能处理**自由形式、带属性和关系的文本**("the dog on the left"),这是后融合做不到的。Grounding DINO 在零样本 COCO 检测上达到很强的水平(具体 AP 数「待核」)。
- 缺点:**架构更复杂、训练更重**(要在多个阶段做跨模态注意力),且依赖大量 grounding 标注数据(图 + 带框的短语)。

**对比表(本节核心 trade-off):**

| 维度 | 后融合(ViLD / F-VLM / SAM+CLIP) | 早融合(GLIP / Grounding DINO) |
|---|---|---|
| 图文交互时机 | 事后(区域特征算完再比文本) | 全程(从特征提取就交互) |
| 能处理的文本 | 类级名词("cat") | 自由短语+属性+关系("左边红衣服的人") |
| 复用现成模型 | 高(直接用 CLIP) | 低(需联合训练) |
| 训练成本 | 低 | 高 |
| 典型用途 | 给 SAM 掩码贴语义标签 | 文本指代检测/分割、Grounded SAM |

### 5.3 Grounded-SAM:把三件事拼成开放词表分割的事实标准

2023 年社区里最实用的开放词表分割管线,是把前面所有零件拼起来——**Grounded-SAM**:

```
文本 "棕色的狗"
      │
      ▼
[Grounding DINO]  ──▶  根据文本输出"棕色的狗"的检测框(开集、带语义)
      │  框
      ▼
[SAM]  ──框当提示──▶  把框内那只狗精确抠成像素级掩码(几何精确、类无关)
      │
      ▼
   带语义的精确掩码 = 开放词表实例分割
```

这个拼装精确对应 §4.3 的洞察:**SAM 负责"在哪、什么形状"(几何),Grounding DINO/CLIP 负责"是什么"(语义)**。两种零样本(几何零样本 + 语义零样本)正交互补,拼起来才是完整的开放词表能力。**记住这个分工,它是当前(2024–2025)绝大多数"用文本分割任意物体"系统的骨架。**

### 5.4 SAM 2:把"可提示"扩展到视频(时间维)

SAM 2(Meta,2024)把可提示分割推广到**视频**:在任意一帧给提示,模型在整段视频里跟踪并分割那个对象(promptable visual segmentation,PVS)。机制上加了一个**记忆库(memory bank)+ 记忆注意力**:把过去帧的对象特征存进记忆,当前帧的特征通过 cross-attention 去查记忆,实现跨帧的对象一致性。这正是把 §3 的"图像嵌入"沿时间轴扩成"带记忆的流式嵌入"。它顺带说明了 §2.3 第 1 条的延伸难点——**时间维让"token 化"更难,视频既没有空间离散 token 也没有时间离散单元**(具体架构与 benchmark 数「待核」)。

## 6. 视觉的"GPT 时刻":会不会、难在哪、VLA

### 6.1 先把问题问准

"视觉会不会有 GPT 时刻"这个问题,容易被问成"会不会有个超大视觉模型"。但 GPT 的意义不在大,在**一个自回归目标(预测下一个 token)统一了几乎所有 NLP 任务、且能 in-context 学习新任务**。所以准确的问题是:**视觉能不能找到一个像"下一个 token 预测"那样,既能规模化自监督、又能统一所有视觉任务的单一目标?**

本章的判断(可证伪):**短期(到 2025)没有单一模型统一视觉;中长期方向上,统一会发生但形态和 NLP 不同——更可能是"少数几个可提示的专长基础模型(SAM 这类几何引擎 + CLIP 这类语义引擎 + 扩散这类生成引擎)+ 一个 VLM/VLA 做编排"的组合,而不是一个 GPT 式的单体。** 下面从三个角度论证"难在哪"。

### 6.2 难点一:图像没有天然的离散 token(最根本的一条)

这是 §2.3 第 1 条的深挖,也是最数学化的一条。GPT 的自回归 `P(xₜ | x₁..xₜ₋₁)` 要求把数据切成**离散符号序列**,因为它最后是个 softmax over 词表。文本天生离散,图像不是。要把图像变成 token,只有两条路,各有不可调和的代价:

**(a) 切 patch(ViT 路线):有顺序但不离散。** 把图切成 16×16 的 patch,每个 patch 线性投影成一个连续向量。问题:**这些 token 是连续向量,不是离散符号,无法直接做"预测下一个 token 的 softmax 分类"**。要在 patch 上做自回归生成,得回到回归(预测连续像素,损失病态、易糊)或额外离散化。

**(b) 学离散码本(VQ-VAE/VQGAN 路线):离散但有损且无自然序。** 用一个编码器把图像区域映射到一个**学出来的有限码本**(如 8192 个码字)里最近的那个,图像就变成一串离散码 id,可以像文本一样做自回归(DALL·E、Parti、LlamaGen 走这条)。但有两个硬伤:

```
硬伤 1:量化即有损。把连续特征映到最近码字,有量化误差;
        码本大小 K 决定上界——
        每 token 信息量 ≤ log₂K bit(K=8192 时 = 13 bit/token)。
        要无损表达高分辨率图,K 和 token 数都得爆炸。
        且最近邻查找不可导,要靠 straight-through 估计或 commitment loss 硬接梯度。
硬伤 2:没有自然的生成顺序。文本的顺序是语义因果的(从左到右读),
        图像的"光栅扫描顺序(raster order)"是人为强加的——
        左上角像素和右下角像素没有"先因后果"的关系。
        强行 raster-order 自回归 = 给图像强加一个不存在的因果结构。
```

**结论:NLP 的离散 token 是"免费的真实结构",视觉的离散 token 是"人造的、有损的、序错的"。这是视觉难以照搬 GPT 的最深层数学障碍。** 扩散模型那章其实给了另一条出路——**不离散化,直接在连续空间建模分布**(score/扩散),这也是为什么"视觉的生成基础模型"是扩散赢了自回归,而 NLP 是自回归赢了:**底层数据是否天然离散,决定了哪种范式更自然。**

### 6.3 难点二:视觉任务异构,输出空间无法统一

§2.3 第 2 条。NLP 任务能统一成 text-to-text,因为输出都是 token 序列。视觉的输出空间是**异构的几何对象**:

```
任务        输出空间                            维度/类型
─────────────────────────────────────────────────────────
分类        一个类别                            离散标量
检测        N 个 (框, 类, 分)                   变长集合,框是连续4维
分割        H×W 个像素标签 / N 个二值掩码        稠密栅格
深度估计    H×W 个连续实数                       稠密实数场
姿态        K 个关键点坐标                       连续点集
跟踪        跨帧的对象关联                       时序图结构
```

要"统一",有人尝试**把一切都序列化成 token**(Pix2Seq 把框序列化成离散坐标 token;OFA、Unified-IO 把检测/分割/深度全转成 token 序列;Painter/SegGPT 把任务统一成"图像 in-context 修复")。这条路在涨,但代价是:**把稠密/几何输出硬塞进离散 token 序列,要么精度损失(坐标量化),要么序列极长(逐像素 token 数 = H×W,二次注意力直接爆显存)。** 统一接口和精度/效率之间,目前没有免费午餐。

### 6.4 难点三:监督结构不对称

§2.3 第 3 条。NLP 的"下一个 token 预测"是**任务即监督**——每段文本免费自带标签。视觉里**唯一同等免费的自监督信号是"图文配对"(CLIP)和"图像重建/掩码补全"(MAE、MoCo)**,但这两类自监督**只给得出全局语义或低层结构,给不出"实例/像素/3D"这种结构化标注**。也就是说:**视觉里"免费的自监督"和"我们真正想要的稠密结构化能力"之间有一道鸿沟**。SAM 的数据引擎(§4)正是为了**人造**地跨过这道鸿沟——它本质上是在"用模型生产稠密监督",因为这种监督在自然界里不像文本那样免费存在。

### 6.5 VLA:视觉-语言-动作,以及它把难点又抬高了一层

VLA(Vision-Language-Action)是把 VLM(视觉-语言模型,如 LLaVA = CLIP 视觉编码器 + LLM)再接上**动作输出**,让模型直接从"看图 + 听指令"产生机器人/智能体的动作(RT-2、OpenVLA 等)。机制上常见做法是**把连续动作也离散化成 token**,接在语言 token 之后自回归生成:

```
输入:  图像 token  +  指令 "把红色积木放进碗里"  
输出:  动作 token 序列(末端执行器的 Δx,Δy,Δz,Δroll,...,gripper 离散化成 bin)
       → 反量化 → 机器人控制指令
```

VLA 把视觉基础模型的难点又叠加了**具身(embodiment)层面**的新难点:

- **动作的离散化同样有损**,且动作误差会**沿时间累积(compounding error)**——预测一步偏一点,下一步状态就偏,误差滚雪球。这比静态识别错一帧严重得多。
- **数据极度稀缺且不可从互联网爬**。互联网有海量图文,但**几乎没有"图像→正确动作"的配对**;机器人数据要在真实/仿真里一条条采,**这是比稠密标注更贵的监督**。Open X-Embodiment 这类数据集就是在凑这个"动作数据引擎"(具体规模「待核」)。
- **物理世界没有"撤销键"**,sim-to-real gap、安全性、闭环反馈让评估和训练都远难于纯感知。

**洞察:VLA 是"视觉 GPT 时刻"问题的最难版本——它要求模型不仅理解世界(感知),还要在世界里正确行动(控制),而后者的监督比前者更稀缺、误差更不可逆。如果说 SAM 解决了"感知的数据引擎",VLA 的开放问题是"行动的数据引擎"还没找到等价的滚雪球机制。**

## 7. 公认难点(一):对抗鲁棒性——微小扰动为何能骗过模型

### 7.1 现象与第一性成因

给一张被正确分类为"熊猫"的图,加上一个**人眼几乎不可见**的扰动 δ(每像素改动 ≤ 几个灰度级),模型就以高置信度判成"长臂猿"(Szegedy et al., 2013;Goodfellow et al., 2014/2015 的经典例子)。这不是个别 bug,是**几乎所有标准训练的神经网络的普遍脆弱性**。

为什么?Goodfellow 的**线性假说(linear explanation)**给出了最干净的第一性解释。考虑一个线性分类器对输入做内积 `wᵀx`。加扰动 δ 后:

```
wᵀ(x + δ) = wᵀx + wᵀδ
要在 ‖δ‖_∞ ≤ ε 约束下最大化 wᵀδ(最大化对 logit 的推动):
    最优 δ = ε · sign(w)          # 每个维度都顶到边界,且方向与 w 同号
此时扰动带来的 logit 变化:
    wᵀδ = ε · Σᵢ |wᵢ| = ε · ‖w‖₁ = ε · d · mean(|wᵢ|)
                                        └── 随维度 d 线性增长!
```

**关键在最后一步:扰动对 logit 的推动 ∝ d(输入维度)。** 每个像素只动 ε(肉眼不可见),但有 d = 几十万个像素,**每个像素的微小推动在 d 个维度上线性累加,合起来就足以翻转决策**。`‖δ‖_∞ = ε` 保证单像素改动不可见,而 `wᵀδ = ε‖w‖₁` 在高维下却很大——**人眼按 L∞(单像素幅度)感知,模型按 L1/L2(全维度累加)响应,这个范数错配就是对抗样本的几何根因。**

**这是本章关于鲁棒性的核心洞察:对抗脆弱性不主要是"模型太非线性、有诡异坑洞",恰恰相反,是模型在高维输入上"太线性"——线性响应在高维下被维度灾难放大。深网络在局部足够线性,使这个线性分析惊人地适用。** 这也解释了为什么对抗样本能**跨模型迁移**:不同模型学到的决策边界方向相近,同一个 δ 对它们都推得动。

### 7.2 怎么造:FGSM 与 PGD

把上面的线性分析直接变成攻击,就是 **FGSM(Fast Gradient Sign Method)**:不再用 w(线性才有),而用损失对输入的梯度方向:

```python
def fgsm(model, x, y, eps):
    x = x.clone().requires_grad_(True)
    loss = F.cross_entropy(model(x), y)
    loss.backward()                       # 拿到 ∂loss/∂x
    delta = eps * x.grad.sign()           # 每像素顶到 ±eps,方向=让loss增大
    return (x + delta).clamp(0, 1).detach()
```

机制:`sign(∂loss/∂x)` 是在 L∞ 球内**让损失上升最快**的方向(一阶近似),完全对应 §7.1 的 `ε·sign(w)`。**PGD(Projected Gradient Descent)**是它的多步强化版——小步走、每步投影回 ε 球,是目前评估鲁棒性的"标准强攻击":

```python
def pgd(model, x, y, eps, alpha, steps):
    delta = torch.empty_like(x).uniform_(-eps, eps)   # 随机起点(逃局部)
    for _ in range(steps):
        delta.requires_grad_(True)
        loss = F.cross_entropy(model(x + delta), y)
        g = torch.autograd.grad(loss, delta)[0]
        delta = (delta + alpha * g.sign()).clamp(-eps, eps)  # 走一步并投影回L∞球
        delta = (x + delta).clamp(0, 1) - x                  # 保证像素合法
        delta = delta.detach()
    return (x + delta).detach()
```

### 7.3 防御:对抗训练为何有效、代价是什么、为何仍不够

最有效、最经得起检验的防御是**对抗训练(adversarial training,Madry et al., 2018)**——把训练写成一个 **min-max(鞍点)问题**:

```
min_θ  E_{(x,y)~data} [ max_{‖δ‖≤ε}  L(f_θ(x+δ), y) ]
       └─ 外层:学参数 ─┘ └─ 内层:找最坏扰动(用PGD近似) ─┘
直觉:不再要求"在 x 上分对",而要求"在 x 周围整个 ε 球内都分对"。
实现:每个 batch 先用 PGD 生成对抗样本,再用对抗样本算损失更新 θ。
```

为什么有效:它**显式地把决策边界从数据点推远 ε**,直接消除 §7.1 那个"沿梯度方向稍走就翻转"的脆弱性。但代价极其诚实,且暴露了一个更深的开放问题:

- **算力贵 N 倍**:内层每步 PGD 要 K 次前向+反向,训练慢 K 倍(常 K=7~10)。
- **鲁棒性-精度权衡(robustness-accuracy trade-off)**:对抗训练后的模型,**干净样本的准确率会下降**(Tsipras et al., 2019 论证这种 trade-off 可能是内禀的——鲁棒特征和高精度特征在某些分布下本质冲突)。
- **只对你训练时用的那种扰动鲁棒**:对 L∞ 训练的模型,换 L2 攻击或空间变换、色彩攻击就可能再次失效。**没有"对一切扰动都鲁棒"的免费方法**——这是个仍然敞开的难题。

**坑(极其重要):梯度遮蔽(gradient masking / obfuscated gradients)。** 很多看似有效的防御(加随机噪声、不可导预处理、梯度截断)只是**让攻击者的梯度失真**,而非真的鲁棒。它们能挡住依赖梯度的 FGSM/PGD,却挡不住更强的自适应攻击(BPDA、EOT)。Athalye et al.(2018)系统性地打穿了一批这类防御。**评估鲁棒性必须用自适应的强攻击(如 AutoAttack),否则数字是假的**——这是 know 里"N=1 不算 feasibility"在安全领域的镜像:**单一弱攻击下的高鲁棒分,极可能是梯度遮蔽的假象。**

## 8. 公认难点(二):长尾、域偏移、3D/视频、可解释性、组合理解

这些难点都共享一个母题:**深度模型学的是"训练分布上的统计相关性",而非"因果/组合的世界结构"**。逐个点透成因与现状。

### 8.1 长尾分布与类别不均衡

真实世界类别频率服从**长尾(幂律)分布**:少数 head 类(人、车)样本极多,大量 tail 类(某种罕见鸟)样本极少。标准交叉熵的问题在梯度层面:

```
softmax 交叉熵对类 c 的 logit 梯度 ∝ (p_c − y_c)。
head 类在每个 batch 频繁出现 → 梯度被它们主导;
tail 类罕见 → 梯度信号被淹没,且分类器权重范数偏小,预测时被压制。
```

应对思路分三类,各有局限:(1) **重采样/重加权**(按频率反比加权,如 class-balanced loss、focal loss 压低易分样本权重)——简单但易过拟合 tail;(2) **解耦表示与分类器**(Kang et al., 2020:先正常训表示,再单独平衡地重训分类头)——发现"表示其实学得还行,坏的是分类器的偏置";(3) **logit adjustment**(按类别先验 log πc 调整 logit,有贝叶斯最优解释)。**核心洞察:长尾的病灶常在分类器的 decision boundary 偏置,而非表示本身——这解释了为何"只重训分类头"这种便宜方法常常有效。**

### 8.2 域泛化与分布偏移(OOD)

模型在训练分布(source)上学好,部署到不同分布(target:不同相机、光照、画风、地域)就掉点——**分布偏移(distribution shift)**。根因是 §8 的母题:模型抓的是**伪相关(spurious correlation)**(如"牛多在草地上"→把草地当成牛的特征),分布一变,伪相关失效。

```
偏移的两种典型:
  协变量偏移 covariate shift:  P(x) 变,P(y|x) 不变(换相机、换画风)
  标签/先验偏移 label shift:    P(y) 变,P(x|y) 不变(部署地类别比例不同)
```

应对:数据增强(让模型见更多变体)、领域自适应(用 target 无标签数据对齐特征分布,如 DANN 的对抗对齐)、不变风险最小化(IRM,学跨环境不变的特征)。**但没有一个方法在所有 shift 上稳健**——DomainBed 这类基准的扎心结论是:**做足数据增强的朴素 ERM(经验风险最小化)常常打平甚至超过花哨的域泛化算法**。这本身就是个警示:**很多"域泛化方法"的增益经不起公平基准的检验。**

### 8.3 3D 与视频:多了一个维度,监督和建模都更难

2D 图像到 3D/视频,难点是**结构化 + 维度爆炸 + 标注更贵**:

- **3D 表示无统一形态**:点云(无序、稀疏)、体素(规则但 O(N³) 显存爆炸)、网格(拓扑可变)、隐式场(NeRF/SDF,用 MLP 表示连续场)——没有像 2D 像素栅格那样的统一表示,每种表示要不同的网络(PointNet 对点云、稀疏卷积对体素)。
- **多视图几何约束**:3D 重建要服从对极几何、相机投影等**硬几何约束**,纯数据驱动易违反物理(穿模、尺度漂移)。NeRF 用可微体渲染把 2D 监督反传到 3D 场,是把几何约束嵌进损失的范例。
- **视频 = 空间 × 时间**:朴素 3D 卷积/时空注意力的复杂度对帧数 T **再乘一层**;时序冗余高(相邻帧几乎一样)又导致信息密度低、标注更贵(逐帧标天价)。这正是 §6.2 离散化难题在时间轴的延伸。

### 8.4 可解释性

深网络是**高维非线性黑箱**,"它为什么这么判"难以保证忠实(faithful)的解释。常见工具——saliency map/Grad-CAM(看哪些像素梯度大)、注意力可视化、概念激活向量——都有一个共同陷阱:**它们解释的是"模型对输入的局部敏感度",不等于"模型的因果决策依据"**。Grad-CAM 高亮的区域可能只是相关而非因果;注意力权重高不代表那里"重要"(注意力可被重新分配而不改输出)。**洞察:目前的可解释性工具多是"事后的、近似的、局部的",缺乏忠实性保证——这是一个方法论尚未成熟的开放领域,不要把热力图当成模型的"真实理由"。**

### 8.5 细粒度、计数与组合理解的脆弱

这是 CLIP 那章就埋下的伏笔,也是当前 VLM 最尴尬的短板。表现:

- **细粒度**:区分相似子类(犬种、鸟种)需要局部判别特征,而对比预训练倾向学全局语义,**细节被全局表示平均掉**。
- **计数**:数清"图里有几只羊"——模型常给出"差不多但不对"的数,因为它学的是**纹理/存在性**("有没有羊"),而非**离散的、需要逐个体定位+累加的计数过程**。计数要求一种**组合的、符号式的**操作,与统计纹理匹配的归纳偏置冲突。
- **组合性(compositionality)**:把熟悉的概念以新方式组合("红色立方体在蓝色球的左边")。CLIP 类模型被诟病像**词袋(bag of words)**——它知道有"红""立方体""蓝""球",却常常分不清谁修饰谁、谁在谁左边(Winoground、ARO 这类基准专门戳这个,模型表现常接近随机)。**根因:对比/统计学习抓的是概念的共现频率,而组合理解要求显式的关系/绑定结构(谁绑定谁的属性、谁与谁的空间关系),这种结构没有被对比损失显式监督。**

**把 §8 收成一句:这些难点是同一个病的不同症状——深度模型擅长插值训练分布内的统计相关,不擅长外推、计数、绑定、因果这些需要显式结构与组合的能力。这正是当前视觉(乃至多模态)智能与"真正理解"之间的鸿沟,也是研究最前沿的开放战场。**

## 9. 设计权衡与常见坑

- **别把 SAM 当识别器。** SAM 只出几何掩码、不知语义(§4.3)。要语义必须配 CLIP/检测器。把"SAM 零样本"误当"能认识任何东西"是新手最常见的误解。
- **用 SAM 务必缓存图像嵌入。** 在同一张图反复点提示时,只重跑 mask decoder(§3.4),否则白白浪费 10× 算力。torchvision/官方 `SamPredictor.set_image()` 就是在做这件缓存。
- **开放词表选路线看需求。** 只需"类级开集标签"用后融合(SAM+CLIP,便宜);需要"带属性/关系的指代"必须用早融合(Grounding DINO),别指望后融合处理"左边那个红色的"。
- **prompt 工程对开放词表同样敏感。** 类名写法("dog" vs "a photo of a dog" vs "棕色的狗")显著影响文本嵌入,进而影响检测/分割结果——继承自 CLIP 的 prompt 敏感性。
- **评估对抗鲁棒性只用单一弱攻击 = 自欺。** 必用 PGD/AutoAttack 等强自适应攻击,警惕梯度遮蔽给出的假鲁棒分(§7.3)。这是安全领域最大的坑。
- **对抗训练的干净精度会掉,要事先接受这个 trade-off**(§7.3),不要指望既最鲁棒又最高精度。
- **域泛化别迷信花哨算法。** 先把数据增强 + ERM 这个强基线做扎实,很多论文方法在公平基准(DomainBed)上并不超过它(§8.2)。
- **别把 Grad-CAM/注意力热力图当"模型的真实理由"。** 它们是局部敏感度的近似,不保证因果忠实(§8.4)。用来 debug 可以,用来"证明模型理解了"不行。
- **VLA 里动作误差会累积。** 静态感知错一帧无所谓,闭环控制里一步偏会滚雪球(§6.5),评估必须看闭环成功率而非单步精度。

## 10. 动手练习

1. **(分析/论证题)论证"视觉为何比 NLP 更难统一成一个基础模型"。** 不要泛泛而谈"视觉更复杂"。要求你严格沿 §6.2–6.4 的三条主线,各给一个**可量化或可形式化**的论据:(a) 离散性——写出 VQ 码本每 token 的信息上界 log₂K,并论证为何 raster-order 自回归给图像强加了不存在的因果序;(b) 任务异构——列出至少 4 种视觉任务的输出空间维度/类型,说明为何无法像 text-to-text 那样统一接口而不损精度;(c) 监督结构——论证为何"图文配对"这种免费自监督给不出实例/像素级监督。*提示*:落点是"NLP 的三个红利(离散接口/统一任务/免费稠密监督)视觉一个都不免费",并指出 SAM 的数据引擎和 CLIP 的语言对齐分别是在人造哪一个红利。

2. **(推导题)从线性假说推对抗扰动的维度依赖,并验算。** (a) 对线性分类器 `wᵀx`,在 `‖δ‖_∞ ≤ ε` 下推出最优扰动 `δ* = ε·sign(w)` 及 logit 变化 `wᵀδ* = ε‖w‖₁`;(b) 设 w 各分量独立同分布、均值 0 方差 σ²,论证 `E[‖w‖₁] = d·σ·√(2/π)`,从而扰动对 logit 的推动随维度 d **线性增长**,而 `‖δ‖_∞=ε` 与 d 无关——这就是"肉眼不可见却能翻转决策"的数学根因;(c) 用 numpy 取 d=10⁴、ε=0.01、w~N(0,1),数值验证 `wᵀδ*` 的均值约等于 `d·ε·√(2/π) ≈ 80`,远大于 logit 翻转所需的量级。*提示*:核心是分清"人眼按 L∞ 感知、模型按 L1 累加响应"这个范数错配。

3. **(编码题)实现 PGD 并观察鲁棒性-精度权衡。** 在 CIFAR-10 上:(a) 实现 §7.2 的 `pgd`,对一个正常训练的小 ResNet 测干净精度 vs PGD(ε=8/255, steps=10)精度,应看到鲁棒精度暴跌(常 <5%);(b) 做一轮对抗训练(每 batch 用 PGD 生成对抗样本再更新),对比训练后干净精度(应略降)和 PGD 精度(应大幅回升);(c) 把内层 PGD 换成单步 FGSM 训练,观察是否出现"对单步攻击鲁棒、对多步 PGD 仍脆弱"的梯度遮蔽假象。*提示*:务必用与训练**不同**的攻击(如换 ε 或换 AutoAttack)做最终评估,防止自欺;这道题就是 know 里"N=1 不算 feasibility"的安全版实验。

4. **(分析题)讨论 SAM 的数据引擎为何关键,并设计 VLA 的等价物。** (a) 用复杂度论证:为何"模型在环"把标注的人力复杂度从 O(N·边界) 降到 O(N)→O(0) 是十亿级掩码的前提(§4.2);(b) 论证为何全自动阶段"用覆盖度对冲噪声"成立(类比扩散的期望强制覆盖);(c) 思考题——VLA(§6.5)缺的是"动作数据引擎"。仿照 SAM 三阶段,设计一个机器人动作数据的滚雪球方案,并指出它比 SAM 难在哪(提示:动作的"确认"是否像掩码那样 O(1)?物理世界能否像点击那样廉价试错?sim-to-real 这一关 SAM 有没有对应物?)。

## 12. 小结与承上启下

这一章站到了整门课的终点,也是研究的起点。把全章收成一条主线:**NLP 的基础模型是"天然离散 token + 统一任务接口 + 免费稠密监督"三个红利叠加的自动产物;视觉这三个红利一个都不免费,于是视觉的基础模型必须靠工程和迁移去人造它们。**

- **SAM** 人造了"统一任务 + 稠密监督":用"可提示分割"这个**类无关、可规模化标注**的新任务,配上"模型在环数据引擎"的飞轮,把视觉里最贵的稠密标注滚到了十亿级。它的范式价值不在网络,在**证明了"可提示任务 + 自产数据"能让视觉走通基础模型之路**(§3–4)。
- **开放词表检测/分割** 借 CLIP 人造了"开集语义接口":把固定分类头换成文本编码器的现算嵌入,后融合(SAM+CLIP)给类级标签、早融合(Grounding DINO)给指代关系,Grounded-SAM 把"几何零样本(SAM)+ 语义零样本(CLIP/DINO)"拼成事实标准(§5)。
- **视觉的 GPT 时刻** 短期不会以单体形态到来:**图像没有天然离散 token(最深的数学障碍)、任务输出异构、稠密监督不免费**三道坎,使统一更可能是"几个可提示专长引擎 + VLM/VLA 编排"的组合;VLA 把难度推到具身,缺的是"动作数据引擎"(§6)。
- **公认难点** 共享一个母题——**深度模型擅长插值训练分布内的统计相关,不擅长外推、计数、绑定、因果**:对抗脆弱源于高维线性响应的范数错配(§7),长尾病在分类器偏置,域偏移病在伪相关,3D/视频病在维度与几何约束,可解释性缺忠实保证,组合理解缺显式绑定结构(§8)。

回望整门课:从**视觉的根本问题**(语义鸿沟)出发,经**卷积/ResNet** 建起归纳偏置,**检测/分割** 把任务做精,**ViT** 引入注意力打破局部性,**CLIP** 用语言打破闭集,**GAN/扩散** 学会生成——到这一章,这些零件被**组装成基础模型**,并直面"统一视觉"这个尚未解决的终极问题。你现在应当能做三件事:**读懂前沿论文的机制设计、亲手实现并改进这些模型、并对"哪些是已解的、哪些是开放的、开放的难在数学哪一处"有清醒判断。** 这正是从"会用"到"能做研究"的分界线——这门课到此交付,但视觉作为一个领域,最有意思的问题恰恰都还开着。
