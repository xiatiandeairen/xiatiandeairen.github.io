> 这章把你从"会用 LLM 解决主流 NLP 任务"带到能看清**这个领域还有哪些真问题没被解决、为什么没被解决、增量研究该往哪扎**。前十四章你学的是"怎么把 NLP 做出来";这一章学的是"作为一个研究者/高级工程师,当所有任务都坍缩成 prompt 之后,你还能在哪里做出别人做不出的东西"。读完你应当能:判断一个 NLP 研究方向是"被统一吞掉了"还是"统一暴露出的新难题";理解低资源、组合泛化、长上下文、agent、可信这五条主线各自的第一性障碍;并能为一门冷门语言或一个组合泛化问题写出像样的方案草稿。

## 1. 动机:统一之后,"NLP 研究"的坐标系塌了一半

第 09 章讲过一次坐标系旋转:LLM 把"任务定义"从模型参数 θ 搬到了输入 x,分类/抽取/翻译/摘要/QA 在使用层面坍缩成"给一段 prompt 让模型续写"。这件事对**做产品**是天大的好消息,但对**做 NLP 研究**是一次身份危机。

危机的具体形态是这样的。2018 年以前,NLP 顶会的论文版图大致按任务切分:一拨人做句法分析,一拨人做指代消解,一拨人做关系抽取,各有各的数据集、各有各的 SOTA 榜单、各有各的专用模型(CRF、指针网络、树形 LSTM……)。一篇论文的增量很清晰:"我在 CoNLL-2003 NER 上把 F1 从 92.4 提到 93.1"。到 2023 年,这套版图大面积失效:一个被指令微调过的通用模型,zero-shot 就能在大多数这类任务上做到"够用甚至接近专用模型",而且**不需要那个任务的训练数据**。于是"我提了 0.7 个 F1"这种增量,要么被通用模型一夜抹平,要么变得没人关心。

**这里要诚实地区分两种"被统一"**:

- **第一种:任务被吞掉了。** 这个任务的难度本质上是"把输入文本映射到某种结构化输出",而这种映射恰好是大模型预训练时大量见过的模式。情感分类、主题分类、简单 NER、释义判断,基本属于此类。对这些任务,继续做"专用模型 + 刷榜"的增量研究,边际价值确实趋近于零——这不是悲观,是事实。
- **第二种:任务没被解决,只是被遮蔽了。** 表面上 LLM"能做",但它做得**不可靠、不可控、不可验证、不可解释**,或者在**分布之外**就垮。统一恰恰让这些问题从"某个小任务的局部困难"上升成"通用系统的全局困难"。这才是增量研究的金矿。

**所以本章的纲领是:不要问"还有什么任务没被 LLM 做",要问"LLM 在什么维度上系统性地不可靠,而这种不可靠又无法靠把模型做大来根治"。** 后者才是 NLP 作为一门独立学科在 LLM 时代的立身之本。本章按五条这样的主线展开:低资源/多语言、长文档/长上下文、组合泛化、agent 化、可解释与可信,最后正面回答"NLP 研究增量在哪"。

按本课分工:Transformer/注意力/长上下文外推/解码的**内部机制**属于算法课(02-05、13-14),RLHF/对齐的**算法细节**属于强化学习课,本章只在需要时点到并注明出处,重心放在 NLP 特有的问题结构上。

一个贯穿全章的判据,先立在这里:

```
某方向是否值得做增量研究的粗判据:
  1. 把模型规模 ×10、数据 ×10,这个问题会自动消失吗?
     —— 会消失 → 大概率是"被吞掉"的任务,别在这做学术增量
     —— 不会消失 → 它触及了某种结构性障碍,值得研究
  2. 这个失败是否可量化、可复现、可在受控数据上构造?
     —— 不能 → 你连"有没有进步"都说不清,先做评测再说
     —— 能   → 它具备成为研究问题的最低条件
```

判据 1 的"做大就消失"在 scaling law 上有依据:很多能力随算力呈幂律平滑提升(算法课讲过 scaling law 的形式)。但**幂律平滑提升 ≠ 任意能力都会涌现**——后面会看到组合泛化、长尾事实、忠实性这几样,经验上并不随规模平滑变好,这正是它们成为开放问题的原因。

## 2. 低资源与多语言:数据的长尾,以及"以英语为中心"的结构性不公

### 2.1 问题:世界有约 7000 种语言,NLP 只服务其中几十种

把"NLP 在 LLM 时代很成熟了"这句话放到全球语言版图上,立刻破功。地球上活着的语言约 7000 种(Ethnologue 的常引数字,待核具体值),但**绝大多数语言在网络上几乎没有可用文本**。一个常被引用的分层框架(Joshi et al., 2020,*The State and Fate of Linguistic Diversity*)把语言按资源量分成 0~5 级:

```
Class 5: 英语等极少数         —— 数据海量,几乎所有 NLP 资源都为它造
Class 4: 中/法/德/西/日 等     —— 资源丰富
Class 3: 印地语/越南语 等       —— 中等,够训像样模型
Class 2/1: 大量语言            —— 零星标注,基本只有圣经译本/维基条目
Class 0: 数千种                —— 几乎无任何数字化文本,连分词工具都没有
```

关键事实:**全球大多数语言落在 0/1 级,而它们对应着相当比例的人口。** 这不是"再等几年数据自然就有了"的问题——很多 0 级语言没有标准书写系统、说话者老龄化、文本根本不会被产生出来。**数据稀缺对它们是结构性的,不是暂时的。** 这就是判据 1 的反例:你把英语模型做到一万倍大,对一门只有几千文本的语言毫无帮助,因为瓶颈在数据侧而非模型侧。

更隐蔽的是**以英语为中心(Anglocentric)的设计偏置**。我们的整套技术栈——子词分词、评测基准、甚至"什么叫一个词"的假设——都是围着英语和少数高资源语言长出来的。举一个能算清楚的具体伤害:**分词的语言不公**。

### 2.2 推导:为什么同一个 BPE 分词器对低资源语言"收更多 token 税"

第 01 章讲过 BPE(byte-pair encoding):从字节/字符开始,反复合并语料里最高频的相邻对,直到词表达到设定大小。合并规则的频率统计**主要来自训练语料**,而训练语料以英语为主。后果是:

- 英语常见词("the""ing""tion")很早就被合并成单个 token;
- 一门低资源语言的词,因为在语料里出现少,**很少有合并规则覆盖它**,只能退回到按字节/字符切。

设同一段语义内容,英语写法占 `n_en` 个 token,某低资源语言写法占 `n_lo` 个 token。定义**分词膨胀率(tokenizer fertility)**:

```
fertility(lang) = (该语言每个"词"平均切出的 token 数)
等价的内容级度量:
   ρ = n_lo / n_en   (表达同一语义,低资源语言比英语多用的 token 倍数)
```

经验上 ρ 常常达到 2~5 甚至更高(具体值随语言和分词器,待核)。这个倍数直接转化成三重不公,而且每一重都能定量:

```
设 LLM 上下文窗口 = C tokens,API 按 token 计费 = p 元/token

(1) 有效上下文不公:
    英语能放进窗口的内容量 ∝ C
    低资源语言能放进的内容量 ∝ C / ρ      ← 同样窗口只装 1/ρ 的信息

(2) 成本不公:
    处理同一份内容,低资源语言费用 = ρ × 英语费用

(3) 延迟/能力不公:
    自回归解码逐 token 生成,生成同等信息需多 ρ 倍步数;
    且更长的 token 序列意味着更长的依赖跨度 → 在长程建模上天然吃亏
```

**这是一个纯工程后果,但它把"语言公平"从口号变成了可测量的量。** 你可以用任意分词器(如 HF `tokenizers`)对一组平行句子(同义、多语言,如 FLORES 平行语料)算 ρ,直接画出"哪些语言在交税"。这是低资源 NLP 一个干净、可发表的小研究切口。

### 2.3 机制:三条让低资源语言"借力"的路线

既然本语言数据不够,核心思路就是**从别处借**。三条主路线:

**路线一:跨语言迁移(cross-lingual transfer)。** 在高资源语言上学到的东西,迁移到低资源语言。最成功的实现是**多语言预训练**:用上百种语言的混合语料训一个共享模型(mBERT、XLM-R,Conneau et al., 2020)。关键机制是**共享子词词表 + 共享参数**让不同语言映射进同一个表示空间,从而:

```
zero-shot 跨语言迁移(典型范式):
  1. 在多语言语料上预训练共享编码器  fθ
  2. 仅用「高资源语言(如英语)」的标注数据微调下游任务头
  3. 直接在「低资源语言」上推理 —— 无该语言任何标注

为什么有时work:若 fθ 把"语义相近的不同语言句子"映射到邻近向量,
              那么在英语上学的分类边界,对低资源语言的同义句也近似成立。
```

但这里有个**著名的悖论,必须讲透**:**多语言诅咒(curse of multilinguality)**。固定模型容量,往里塞越来越多语言:

```
语言数 ↑(模型容量固定):
  少量语言时 —— 低资源语言"搭便车",受益于跨语言共享,性能↑(正迁移)
  语言过多时 —— 有限容量被瓜分,每种语言分到的"表示预算"↓,
                高资源语言开始掉点,低资源语言收益也见顶(负迁移/容量竞争)
```

这是一个清晰的**容量-覆盖权衡**:你想覆盖更多语言,就得给更多容量,否则就是拆东墙补西墙。XLM-R 论文正是用"加大模型 + 加大数据"把这条曲线整体抬高的。这条权衡又一次说明:多语言不是"免费午餐",它有可量化的代价。

**路线二:数据增强与合成(data augmentation)。** 本语言没数据,就造。NLP 的数据增强比 CV 难得多——CV 里翻转/裁剪图像不改语义,文本里随便改个词可能就改了意思或变不通顺。NLP 特有的几招:

```
回译(back-translation,最常用,Sennrich et al., 2016):
  单语低资源句 y  →(用一个 y→en 模型翻成)→  伪英语 x'
  得到伪平行对 (x', y),用来训 en→y 的翻译/任务模型
  直觉:y 是真的(人写的),只有 x' 是机器造的,目标端质量有保证

token 级扰动:同义词替换 / 随机删除 / 随机交换(EDA, Wei & Zou, 2019)
  —— 廉价但风险高:可能破坏语法或翻转标签,需任务上验证

LLM 合成:用强多语言 LLM 直接生成该语言的标注样本
  —— 当下最强但有隐患:LLM 对低资源语言本身就弱,
     合成数据可能系统性地带着"翻译腔"和英语句法投影 → 污染
```

**路线三:以语言为中心的本地建设(language-centric)。** 这是最不性感但最根本的一条:为目标语言**社区驱动地**采集语料、制定书写规范、标注种子数据、训练本地分词器。Masakhane(非洲语言)、AmericasNLP(美洲原住民语言)等是代表性的社区项目。这条路线的研究增量不在"模型多新",而在**如何用最少的人工标注撬动最大的能力**——主动学习(active learning,优先标注模型最不确定的样本)、少样本提示在低资源语言上的可靠性、用语言学先验(形态学规则)替代数据,都是真问题。

### 2.4 分析:低资源 NLP 的增量金矿与陷阱

- **金矿**:跨语言迁移的**失败分析**(为什么某些语系迁移好、某些差?和语序、形态丰富度的关系)、低资源**评测基准的建设本身**(没有测试集就无法度量进步,造一个高质量测试集是实打实的贡献)、**形态丰富语言**(芬兰语、土耳其语这类一个词能黏上十几个语素的)的子词建模。
- **陷阱一:用英语测试集的翻译版当低资源测试集。** 翻译过来的句子带着英语的话题分布和句法投影,**不代表该语言母语者真实的语言使用**,测出来的分数虚高且有偏。
- **陷阱二:把"LLM 在低资源语言上能输出该语言文字"误当成"能力达标"。** 它可能语法通顺但事实全错、或暗中用英语思维直译。低资源能力的评测必须有**母语者参与**,这是 NLP 公平性研究绕不开的成本。

## 3. 长文档与长上下文:窗口变长不等于会用长

### 3.1 问题:篇章级理解,不是把句子拼起来那么简单

前面的任务大多在句子或短段落上。但现实文本是**篇章(discourse)**:一篇合同、一份病历、一本小说、一个代码仓库。篇章级理解有句子级没有的结构:

- **指代与共指(coreference)**:第 8 页的"该公司"指的是第 1 页定义的哪个实体?
- **篇章关系(discourse relations)**:这一段和上一段是"因果""转折"还是"举例"?(RST、PDTB 这类篇章理论刻画的就是这个。)
- **长程依赖**:答案的前提在文档开头,问题在结尾,中间隔着上万词。
- **全局一致性**:摘要不能自相矛盾,长文生成不能前后人设崩塌。

这些都不是"上下文窗口够长就自动会"的。我们分两层看:**能不能放进去(容量)** 和 **放进去会不会用(利用)**。

### 3.2 推导:为什么"放得进"很贵——注意力的二次复杂度

标准自注意力(算法课细推过,这里只取结论用于算账)对长度 L 的序列,要算一个 L×L 的注意力矩阵:

```
注意力计算/显存复杂度:O(L²·d)        d = 表示维度
  L = 1k   → L² = 10⁶
  L = 100k → L² = 10¹⁰      ← 增长 10000 倍,而 L 只增长 100 倍
```

二次项是把上下文做长的根本障碍。这催生了一整片**高效注意力**研究(稀疏注意力、线性注意力、状态空间模型如 Mamba……),**机制属于算法课**,本章只需记住结论:**这些方法用"近似全局注意力"换"次二次复杂度",几乎都在某种长程精度上有损**——天下没有免费的长上下文。从 NLP 角度,你要问的不是"哪个高效注意力 FLOPs 更低",而是"它在我的篇章任务(长文档 QA、跨段共指)上损失了什么"。

工程上还有一条不碰二次复杂度的路:**检索式记忆**——不把全文塞进窗口,而是存进外部库,按需检索相关片段进窗口。这正是第 10 章 RAG 的思路用在"长文档"上(把一篇长文切块、检索块),也是"长期记忆"型 agent 的基础。**长上下文 vs 检索,是同一个问题的两种解法**,后面权衡一节细谈。

### 3.3 机制:"放得进"也不代表"用得好"——Lost in the Middle

即便你有了能塞十万 token 的窗口,还有一个反直觉且后果严重的经验现象:**Lost in the Middle**(Liu et al., 2023)。

```
实验设定(多文档 QA):把含答案的那一篇文档放在 k 篇文档里的不同位置,
                     测模型答对率随"答案位置"的变化。
观察到的曲线:大致呈 U 形 ——
  答案在开头   → 准确率高
  答案在结尾   → 准确率较高
  答案在中间   → 准确率显著下降(谷底)
```

**这是判据 1 的又一个反例:它不随窗口变长而消失,把窗口做到 128k 仍然存在。** 它说明"模型支持的上下文长度"和"模型能有效利用的上下文长度"是两个数,后者往往远小于前者。

为什么会这样?这是个仍在研究的开放问题,主流的几种解释(都属算法课更细的机制,这里点到):位置编码的外推性质让模型对中段位置不敏感;预训练数据里"关键信息出现在开头/结尾"的统计先验被学进去了(文档开头常是摘要、结尾常是结论);以及训练时极少见过"答案恰在超长上下文正中"的样本。无论哪种,**对 NLP 实践的直接 takeaway 是可操作的**:

```
利用长上下文的工程对策(可量化验证):
  1. 重要证据放在 prompt 的首/尾,别埋在正中
  2. 检索 + 重排:与其塞 50 篇,不如检索精排出 5 篇放进去(降低 k)
  3. 显式评测"有效上下文长度":固定任务,逐步加长无关上下文(干扰),
     看准确率从哪个长度开始掉 —— 这才是你模型真正能用的长度
```

第 3 点是一个干净的可发表方法论:**用"针在草堆"(needle-in-a-haystack)式探针,把"宣称的窗口长度"和"有效长度"分开测量。** 不要相信厂商标的窗口数字,要自己测。

### 3.4 代码:一个最小的"有效上下文长度"探针

```python
import random

def make_haystack(needle: str, distractors: list[str], pos: float, length: int) -> str:
    """把一句关键信息(needle)埋进一堆无关文本(distractors)的指定相对位置。
    pos ∈ [0,1]: 0=开头, 0.5=正中, 1=结尾;length: 目标干扰文本块数。"""
    filler = [random.choice(distractors) for _ in range(length)]
    insert_at = int(pos * len(filler))
    filler.insert(insert_at, needle)          # 在相对位置插入"针"
    return "\n".join(filler)

def probe_effective_context(model_answer_fn, needle, question, gold,
                            distractors, lengths, positions, trials=5):
    """扫描 (上下文长度 × 针的位置),统计答对率。
    model_answer_fn(context, question) -> str 是你要测的 LLM。
    返回一张 {(length,pos): accuracy} 表,据此画出 U 形曲线和'有效长度'拐点。"""
    result = {}
    for L in lengths:
        for pos in positions:
            hit = 0
            for _ in range(trials):                       # 多次重复,抵消采样噪声
                ctx = make_haystack(needle, distractors, pos, L)
                ans = model_answer_fn(ctx, question)
                hit += int(gold.lower() in ans.lower())   # 简化的命中判定
            result[(L, pos)] = hit / trials
    return result

# 用法骨架(model_answer_fn 接任意 LLM API):
# needle = "马里亚纳海沟最深处约为 10994 米。"
# question = "马里亚纳海沟最深处大约多少米?"; gold = "10994"
# 扫 lengths=[10,50,200,1000], positions=[0.0,0.25,0.5,0.75,1.0]
# 看 accuracy 在哪个 L 开始崩、哪个 pos 是谷底 —— 那就是你模型的真实可用区
```

这段代码的价值不在复杂,而在它把一个口头断言("长上下文不好使")变成了**可复现的曲线**。判据 2(可量化、可构造)在这里落地。

## 4. 组合泛化与系统性:语言学的老问题,大模型的新软肋

### 4.1 问题:学会"跳"和"两次",能不能零样本会"跳两次"?

这是本章最深的一节,也是最能体现"NLP 作为认知科学交叉学科"价值的一节。

人类语言能力的一个核心特征叫**系统性(systematicity)**,Fodor & Pylyshyn 1988 年就提出了:**如果你理解"约翰爱玛丽",你必然也理解"玛丽爱约翰"。** 因为你掌握的是"X 爱 Y"这个**可复用的结构**和"约翰/玛丽"这些**可替换的部件**,而不是把每个完整句子当成一个孤立的整体死记。语言因此能**用有限的部件组合出无限的、从没见过的句子**——这就是**组合性(compositionality)**。

**组合泛化(compositional generalization)** 就是问:模型能不能像人一样,**把训练中见过的部件(原子概念)系统地组合成训练中从没见过的新组合,并正确处理?** 这正是判据 1 的核心反例:它**不随数据规模平滑变好**,因为问题不在"见得不够多",而在"模型学到的是表层共现统计,还是底层组合规则"。

### 4.2 机制:SCAN——一个把组合泛化逼到墙角的诊断数据集

Lake & Baroni 2018 年的 **SCAN** 数据集是这个领域的标尺,它的设计极其干净,值得讲透其构造逻辑。

任务:把自然语言指令翻译成动作序列。

```
输入(指令)              →  输出(动作序列)
"jump"                   →  JUMP
"jump twice"             →  JUMP JUMP
"walk left"              →  LTURN WALK
"jump twice and walk"    →  JUMP JUMP WALK
```

规则是完全组合的、确定性的(像一个小语法)。关键在**怎么切训练/测试集**,不同切法考不同的泛化:

```
切法一(随机划分,baseline):
  train/test 随机分 —— 模型刷到接近 100%。说明它能学这个映射。

切法二(length split,长度泛化):
  train 只含"短指令"(动作序列 ≤ k),test 全是"更长指令"。
  → 模型准确率暴跌。它没学会"组合规则可以无限套用",
    只学会了"我见过的长度范围内的模式"。

切法三(primitive split,原语泛化,最致命):
  "jump" 这个原语在训练里只单独出现过("jump"→JUMP),
  从没和 twice/and/left 等组合过;
  但 "walk/run/look" 等其他原语在训练里见过各种组合。
  test:"jump twice""jump left"——把 jump 放进见过的组合框架。
  人类一眼就会:框架我懂、jump 我懂,拼起来即可。
  → 早期 seq2seq 模型准确率接近 0。
```

**切法三是整个领域的"诛心之问":模型究竟是学到了"twice = 重复两次"这个可作用于任意原语的算子,还是只记住了"walk twice→WALK WALK"这类具体串?** SCAN 的实验给出了一个令早期研究界沮丧的答案:标准 seq2seq(LSTM、甚至 Transformer)在 primitive split 上几乎完全失败。它们学到的是**表层共现**,不是**组合算子**。

### 4.3 推导:为什么"插值"容易、"外推到新组合"难

用一点几何直觉把这件事说清楚。设模型把输入映射到表示空间,在表示上学一个映射到输出。训练样本在输入组合空间里占据若干"点":

```
组合空间(示意,二维代指"原语 × 修饰结构"):
        twice   and    left   thrice
 walk    ●       ●      ●       ●        ← walk 行:训练见过全部组合
 run     ●       ●      ●       ●
 look    ●       ●      ●       ●
 jump    ●       ○      ○       ○        ← jump 行:只见过 jump 本身(●),
                                            其余组合(○)从未见过 = 测试点
```

- **i.i.d. 泛化(插值)**:测试点落在训练点的"包络/凸包"内部。模型只要学一个在训练点附近**光滑**的函数,插值即可,神经网络极擅长此事。随机划分就是这种。
- **组合泛化(外推)**:测试点(jump 行的 ○)落在训练分布**没覆盖的方向**上。要答对,模型必须把"twice 这个算子"和"jump 这个原语"**解耦**成两个可独立操作的因子,再在新坐标上重新组合。

```
本质区别:
  插值  —— 在数据流形(manifold)内部内插,学光滑函数就够
  组合外推 —— 要求模型把表示"因子化(factorize)"成可独立重组的部件,
             这等价于发现数据背后的生成语法,而非拟合数据点分布
```

**纯似然训练没有任何压力逼模型去做这种因子化。** 最大化 `P(动作序列 | 指令)`,只要在训练分布上拟合得好就行;把"twice"和具体原语纠缠在一起记,在训练集上一样能拿满分。**于是模型默认走捷径(shortcut)——学共现,不学规则。** 这是组合泛化难的第一性原因:**目标函数不奖励组合性,数据分布又不强制它。**

### 4.4 大模型把这条线推到哪了,以及为什么仍是开放问题

必须更新到当下:大语言模型 + in-context learning + chain-of-thought,在很多组合泛化基准上**比早期 seq2seq 好得多**,有些 SCAN 变体几乎被攻克。但下结论"组合泛化已解决"是错的,理由有三:

- **测试集污染**:SCAN 这类经典基准很可能已进入大模型预训练语料,"会做"可能是"见过"。这把我们直接带到第 6 节的评测元危机。
- **更难的基准仍未饱和**:COGS(Kim & Linzen, 2020,语义解析上的组合泛化)、CFQ(Keysers et al., 2020,用"复合发散度"系统控制训练/测试组合分布差异)等更精细的基准上,模型仍有明显且系统的缺口(具体数字随模型,待核)。
- **脆弱性证据普遍**:换个表述、加点无关前提、把数字变大,模型表现就抖。这种对表层形式的敏感,正是"学了共现没学规则"的指纹。

**所以正确的判断是:规模和 CoT 缓解了组合泛化,但没有从机制上根治。** 缓解的来源(更大数据让更多组合变成了"见过的插值";CoT 把一步组合拆成多步,每步都落在见过的范围内)恰恰说明根子还在——一旦真正超出训练分布的组合方向,脆弱性就回来。

这一节为什么对 NLP 研究者尤其重要?因为它是少数**逼问"模型到底理解了什么"** 的实验范式,把抽象的"理解 vs 模仿"之争,变成了可在受控数据上量化的命题(判据 2 的典范)。**会设计一个能干净区分"组合 vs 共现"的数据集切法,本身就是一项研究能力。**

## 5. Agent 化的 NLP:从"理解文本"到"用语言行动"

### 5.1 问题:NLP 的疆界从"文本→文本"扩张到"文本→世界"

前十四章的 NLP,输入输出都是文本:给文本、出文本。**Agent 化是一次疆界扩张:让语言成为"感知—规划—行动"的接口,模型不只描述世界,还通过语言去改变世界。** 模型生成的不再只是答案,而是**动作**:调用一个 API、执行一段代码、在网页上点一个按钮、对另一个 agent 下一条指令。

这把强化学习课的"序贯决策"和 NLP 焊在了一起。一个语言 agent 的循环:

```
agent loop(语言作为策略接口):
  loop:
    obs   = 环境观察(网页 DOM / 工具返回 / 报错信息),以文本形式给模型
    think = LLM 基于历史和 obs 生成推理(可选,CoT 风格)
    act   = LLM 生成一个"动作":{工具名, 参数} 或一段代码
    result= 在真实环境执行 act,得到新观察
    若 act = "终止" 或 目标达成:break
```

**这里语言扮演三个全新角色**,每个都是经典 NLP 没正面处理过的:

- **语言作为策略(policy)**:`act = π(history)`,策略 π 就是个 LLM,动作空间是"自然语言/代码"——一个无穷大、组合式的离散动作空间。这与强化学习课里动作空间通常有限/连续的设定**根本不同**,经典 RL 算法不能直接搬。
- **语言作为推理草稿(scratchpad)**:ReAct(Yao et al., 2023)的核心洞见——让模型**交替**产出"推理(reason)"和"行动(act)",推理用来规划、消化观察、纠错。这是 CoT 从"纯文本推理"到"与环境交互的推理"的推广。
- **语言作为 agent 间协议**:多 agent 系统里,agent 之间用自然语言通信、分工、辩论。通信协议本身成了研究对象。

### 5.2 机制:工具使用的接口,与 NLP 该负责的那一半

工具使用(tool use / function calling)是 agent 最落地的形态。其接口本质是把"调用工具"变成"生成一段受约束的结构化文本":

```
工具使用的数据流:
  1. 工具定义(name, 参数 schema, 自然语言描述)放进上下文
  2. 模型决策:这一步要不要用工具?用哪个?参数填什么?
  3. 模型生成结构化调用,如 {"tool":"search","args":{"q":"..."}}
  4. 系统解析并真实执行 → 把返回结果(文本)拼回上下文
  5. 模型基于结果继续(可能再调工具,或给最终答案)
```

**注意第 3 步要求模型生成严格合法的结构化输出**——这正是第 09 章埋下的"结构化输出/约束解码"接口的用武之地,也是 RAG(第 10 章)作为"检索工具"接入 agent 的衔接点。**RAG 可以看成 agent 只用一种工具(检索器)的特例。**

按本课分工划清边界:**怎么训练 agent 把任务做成(策略优化、奖励设计、信用分配)是强化学习课;NLP 这一侧要负责的是另一半**,而且这半同样困难且开放:

- **指令到动作的接地(grounding)**:把"帮我订一张明天去北京的票"这句模糊的自然语言,可靠地解析成一串正确的工具调用。这是组合泛化(第 4 节)在动作空间上的直接复现——见过订机票、见过查天气,能不能零样本组合出"订票前先查天气决定要不要去"?经验上同样脆弱。
- **长程一致性与错误累积**:agent 跑十几步,每步小错都会**沿轨迹累积**。设单步成功率 p,k 步全对的朴素上界 `≈ p^k`(各步独立的粗估):

```
单步成功率 p,任务需 k 步顺次全对(独立近似):
  p=0.95, k=10  → 0.95¹⁰ ≈ 0.60     ← 已经只剩六成
  p=0.95, k=30  → 0.95³⁰ ≈ 0.21     ← 长任务雪崩
  p=0.90, k=10  → 0.90¹⁰ ≈ 0.35
```

这个 `p^k` 是粗估(真实中各步并不独立,且模型能从报错里恢复,所以实际不会衰减得这么干脆),但它点出了 agent 的命门:**单步可靠性的微小提升,在长程上被指数放大。** 这解释了为什么 agent 系统在 demo 里惊艳、在长任务上拉胯。NLP 侧的对策——更可靠的指令解析、从环境反馈中**自我纠错(self-correction / reflection)**、把长任务**分解(decomposition)** 成短可验证子任务——全是开放问题。

- **评测的爆炸**:agent 的输出是一条**轨迹(trajectory)**,不是一个字符串。怎么评一条轨迹好不好?(只看最终成败,还是看过程效率/是否走弯路/是否调错工具?)这把第 6 节的评测危机又抬高了一个维度。

### 5.3 分析:agent 化对"NLP 研究"意味着什么

- **它扩大了 NLP 的疆界**:语言第一次成为**行动的接口**而非仅仅是分析的对象。这给了 NLP 一块强化学习、HCI、软件工程都插不进来填满的独特地盘——**自然语言到可执行意图的可靠映射**。
- **它把组合泛化和长程一致性从"学术诊断"变成"产品死因"**:第 4 节的脆弱性,在 agent 里直接表现为"任务做到一半翻车"。
- **坑**:别把"demo 能跑"当成"能力达标"。agent 的真实能力必须在**有真实失败、可复现的环境**(WebArena、SWE-bench 等基准)里测,而不是精心挑选的几个成功案例。**会搭一个公平、不可被刷的 agent 评测环境,是这条线最稀缺的能力。**

## 6. 可解释、可控、可信:统一暴露出的"黑箱"总账,与评测的元危机

LLM 把能力统一了,也把**一笔"不可信"的总账**一起放大了。这笔账分三块:可解释、可控、可评测。

### 6.1 可解释性:从"特征归因"到"机理可解释"

第一层是**归因(attribution)**:输出的这个词,是输入的哪些部分导致的?经典方法有梯度类(saliency,看 ∂output/∂input)、扰动类(遮住输入某部分看输出变化)、注意力权重(但**注意力权重 ≠ 解释**,Jain & Wallace 2019 专门论证过:注意力高的位置未必是因果上重要的位置,新手极易在这里误用)。

第二层、也是当下前沿的是**机理可解释性(mechanistic interpretability)**:不满足于"哪个输入重要",而要**逆向工程**模型内部的计算回路——某个注意力头在干什么(比如"归纳头 induction head"负责复制重复模式)、某个神经元/方向编码了什么概念、某种能力对应哪条信息通路。**这部分的具体技术(探针 probing、激活修补 activation patching、稀疏自编码器找可解释特征、电路分析)属于算法课 15**,本章只给定位:

**为什么这对 NLP 不是可有可无的"事后解释",而是核心研究?** 因为它是回答第 4 节那个终极问题——"模型到底学到了组合规则还是共现统计"——的**唯一直接手段**。行为测试(SCAN 那种)只能从外部观察"行不行";机理可解释能打开盒子看"它内部到底是怎么算的"。**外部行为 + 内部机制双管齐下,才能真正回答"理解 vs 模仿"。**

### 6.2 可控生成:让"想要的属性"出现、"不想要的"消失

可控生成(controllable generation)问:能不能在不破坏流畅性的前提下,**指定输出的某个属性**(情感正负、正式程度、不含某些内容、遵循某种格式、忠实于给定证据)?三类思路,复杂度递增:

```
(1) 提示/指令控制:prompt 里直接说"用正式语气" —— 最简单,但软约束,可能不遵守
(2) 解码期控制:在生成时干预概率分布
      —— 约束解码(FSM 强制合法格式,硬约束;算法课 13)
      —— 分类器引导(用一个属性分类器的梯度推 logits 偏向目标属性)
(3) 训练期控制:用偏好数据把"要的属性"训进模型(对齐;强化学习课)
```

可控生成与**安全**直接接壤:"不生成有害内容"就是一种属性控制。其难点是**控制与质量的张力**:约束越硬,流畅性/信息量往往越受损;且属性常常**纠缠**(把语气调正式可能连带改了语义)。这是个有清晰权衡、可量化(控制成功率 vs 流畅度 PPL vs 语义保真)的开放问题。

### 6.3 评测的元危机:当尺子本身坏了

这是本章给所有做 NLP 的人的**最重要警告**,因为它动摇的是判据 2 的根基——**如果连"有没有进步"都测不准,整个领域的进步叙事就建立在流沙上。** 三个相互叠加的危机:

**危机一:基准饱和(saturation)。** 一个基准被刷到接近人类/接近 100%,它就**失去区分力**——再好的新模型也只能挤出零点几个点,而这零点几点可能纯属噪声或对该基准特定瑕疵的过拟合。NLP 历史就是一部"造基准→刷爆→换更难的基准"的循环(SQuAD→SQuAD 2.0,GLUE→SuperGLUE,……)。**饱和意味着这把尺子该退役了**,但人们常常继续用它报喜。

**危机二:数据污染(contamination)。** LLM 预训练语料是从全网爬的,**测试集极可能已经在训练数据里**。那么高分到底是"会做"还是"背过答案"?这从根本上动摇了一切公开基准的可信度——这正是第 4 节说"SCAN 被攻克"要打问号的原因。诊断污染很难(你往往拿不到、也搜不全模型的训练数据),缓解手段(用模型训练截止日之后才出现的全新数据、私有保留测试集、给题目加扰动看分数掉多少)各有局限,**这本身是当下一个活跃的研究方向**。

**危机三:生成任务的评测内在困难。** 分类有标准答案,F1 算得明白。但摘要、翻译、开放对话**没有唯一正确答案**——一段文本有无数种好摘要。于是:

```
评测演进与各自的病:
  n-gram 重叠(BLEU/ROUGE):快、可复现,但只测"词面重合",
    一个换词不换义的好摘要可能得低分,一个抄词但语义错的可能得高分
  → 与人类判断相关性有限,尤其对高质量系统区分力差
  
  模型化评测(BERTScore 等用嵌入算语义相似;LLM-as-a-judge 用强模型打分):
    与人类相关性更高,但引入新病——
    LLM 评委有偏(偏好长答案、偏好自己家模型的风格、对位置敏感),
    且评委本身会被钻空子(对抗性输出骗高分)
```

**"用 LLM 评 LLM"是当下主流又危险的范式**:它高效,但评委和被评者共享同源偏置,可能形成"自说自话"的闭环。文本生成评测、忠实性(faithfulness:生成内容是否真的被给定证据支持)的细致方法,是本课**生成评测专章与忠实性专章**正面攻克的硬骨头;本章只把它定位成**统一之后被全面放大的元问题**:

**当所有任务都坍缩成"生成",而"生成的好坏"又难以可靠自动度量时,整个领域面临"我们其实不知道自己有没有在进步"的元危机。** 谁能造出"抗饱和、抗污染、对生成质量有高人类相关性、且不易被刷"的评测,谁就握住了 LLM 时代 NLP 研究的命脉——**因为没有可信的尺子,所有其他研究的"进步"都无法被确认。**

## 7. 设计权衡与常见坑

把前六节的权衡收成一张可操作的表,外加几个高发的认知陷阱。

**长上下文 vs 检索(第 3、5 节):**
```
            长上下文(全塞窗口)         检索式记忆(RAG)
更新知识     需进上下文/重训             改库即可,最灵活
成本         O(L²),长则爆贵            检索 O(log N)~线性,生成只付 top-k 的钱
信息利用     有 lost-in-the-middle      取决于检索质量(召回错=全错)
可溯源       弱                         强(直接给引用片段)
适用         强连贯的整体推理(读懂一篇) 海量、可切块、需引用的知识
```
**坑**:默认"窗口越长越好"。窗口长 → 二次成本爆炸 + 中段信息被忽略。很多场景"检索精排出 5 篇"远胜"硬塞 50 篇"。

**多语言:覆盖 vs 容量(第 2 节)。** 想覆盖更多语言又不掉点,只能加容量,否则是多语言诅咒下的拆东补西。**坑**:用英语测试集的机翻版评低资源语言——分数虚高且有系统偏差。

**组合泛化:似然拟合 vs 规则学习(第 4 节)。** 纯似然训练默认走"学共现"的捷径。**坑(最致命)**:在 i.i.d. 测试集上刷出高分就宣称"模型理解了"——i.i.d. 高分只证明会插值,不证明会组合外推。要换 primitive/length split 才看得出真本事。

**agent:单步可靠 vs 长程累积(第 5 节)。** 单步 p 的微小提升被 p^k 指数放大。**坑**:拿几个精挑的成功 demo 当能力证据,不在有真实失败的环境里跑。

**评测:省事 vs 可信(第 6 节)。** BLEU/ROUGE 省事但与人类相关性弱;LLM-judge 相关性高但有同源偏置和可被刷风险。**坑(贯穿全章)**:不质疑基准就报喜——基准可能已饱和、已被污染、或根本测不准生成质量。**任何一个"SOTA"结论,先问这把尺子还准不准。**

**横跨全章的认知坑**:把"LLM 表面能做某事"等同于"该问题已解决"。本章五条主线(低资源公平、长上下文利用、组合泛化、agent 长程可靠、可信评测)的共同点就是——**表面能做,底下不可靠、不可控、不可信、不可验证,且都不随把模型做大而自动消失。** 这正是"被吞掉的任务"和"被遮蔽的真问题"的分水岭(回到第 1 节判据)。

## 8. 动手练习

**练习 1(立论,呼应第 1、6 节):NLP 被 LLM 统一后,还剩哪些独立研究价值?**
任选一个曾经的经典 NLP 子任务(如指代消解、句法分析、关系抽取),论证它属于"被吞掉"还是"被遮蔽"。
- 提示:套第 1 节的两条判据。判据 1——把模型/数据 ×10 它会消失吗?判据 2——它的失败可量化、可在受控数据上构造吗?
- 提示:对"被遮蔽"类,具体说出 LLM 在该任务上**哪个维度**系统不可靠(可靠性/可控/可解释/分布外/可验证),并设计一个能暴露该不可靠的最小实验。
- 提示:好答案不是"还有价值/没价值"的口号,而是给出一条**别人靠把模型做大也走不到**的研究路径。

**练习 2(机制分析,呼应第 4 节):组合泛化为什么难?设计一个诊断切法。**
不借助现成数据集,为"判断两个商品评论是否表达相同情感倾向"这类任务,**自己设计一个能区分"模型学了组合规则 vs 学了表层共现"的训练/测试集切法**。
- 提示:照搬 SCAN 的 primitive split 思想——让某个"原语"(某个领域词、某种否定结构)在训练里只以受限方式出现,测试时把它放进训练中见过、但与该原语没共现过的组合框架里。
- 提示:再从第 4.3 节的几何视角解释,为什么你的测试点落在了训练分布"没覆盖的方向"上,以及纯似然训练为何没有压力去做正确的因子化。
- 提示:补一个反事实——什么样的切法**只能**测插值、测不出组合外推?(答:随机划分。说清为什么。)

**练习 3(系统设计,呼应第 2 节):为一门低资源语言设计 NLP 方案。**
任选一门你略知一二的低资源语言(或假想一门:无标注数据、有少量单语文本、属于某高资源语言的语系近亲),设计一个"情感分类"系统的完整方案。
- 提示:覆盖三条路线——跨语言迁移(选哪个多语言底座?用哪个高资源近亲语言迁移,为什么)、数据增强(回译可行吗?LLM 合成的污染风险怎么控)、本地建设(用主动学习先标哪些样本最划算)。
- 提示:先算一笔分词税——用第 2.2 节的 ρ 估一下你的语言相对英语的 token 膨胀,讨论它对成本和有效上下文的影响。
- 提示:重点写**评测怎么做**(没有现成测试集时,如何用最少母语者人工撬动一个可信的评测集),并说明为什么不能用英语测试集的机翻版。

**练习 4(可选,呼应第 3 节):动手测"有效上下文长度"。**
用第 3.4 节的探针骨架接一个你能访问的 LLM,扫描"上下文长度 × 针位置",画出准确率曲线。
- 提示:重点观察两件事——(a) 准确率从哪个长度开始崩(真实可用长度 vs 宣称窗口);(b) 是否出现 U 形(lost-in-the-middle)。
- 提示:把"针"和"干扰文本"设计成话题相近的(更难)和完全无关的(更易)两组,对比模型抗干扰能力——这比单纯加长更接近真实长文档场景。

## 9. 源码 / 论文导读

**低资源/多语言:**
- **Joshi et al., 2020**(*The State and Fate of Linguistic Diversity and Inclusion in the NLP World*):读 0~5 级语言分层那张图和讨论——一次看清"NLP 为谁服务"的不公全貌。
- **XLM-R**(Conneau et al., 2020,*Unsupervised Cross-lingual Representation Learning at Scale*):读"curse of multilinguality"那一节——理解覆盖 vs 容量的权衡曲线。
- **回译**(Sennrich et al., 2016,*Improving Neural Machine Translation Models with Monolingual Data*):读方法节,这是低资源数据增强的看家招。
- 开源/社区:HF `tokenizers`(自己对平行语料算分词膨胀率 ρ)、Masakhane / FLORES(多语言平行评测集)。

**长上下文:**
- **Lost in the Middle**(Liu et al., 2023,*How Language Models Use Long Contexts*):读那张 U 形准确率-位置图,这是"宣称窗口 ≠ 有效窗口"的实证。高效注意力的机制去算法课长上下文章。

**组合泛化:**
- **SCAN**(Lake & Baroni, 2018,*Generalization without Systematicity*):重点读数据集构造与 primitive/length split 的设计动机——理解怎么用数据切法把"组合 vs 共现"逼到墙角。
- **COGS**(Kim & Linzen, 2020)与 **CFQ**(Keysers et al., 2020,提出用 compound divergence 系统控制组合分布差异):看它们如何把 SCAN 的思想推广到更真实的语义解析。
- 理论源头:**Fodor & Pylyshyn, 1988** 关于系统性的论证(认知科学经典,选读,理解问题的哲学根)。

**agent:**
- **ReAct**(Yao et al., 2023,*Synergizing Reasoning and Acting in Language Models*):读"reason 与 act 交替"的核心设定——理解语言作为推理草稿 + 行动接口。策略优化去强化学习课。
- 评测环境:**WebArena**(网页 agent)、**SWE-bench**(真实 GitHub issue 修复)——看它们怎么构造"有真实失败、不易被刷"的 agent 基准(具体指标与版本待核)。

**可解释/可信:**
- **Attention is not Explanation**(Jain & Wallace, 2019):读完根治"拿注意力权重当解释"的新手通病。机理可解释性(探针/激活修补/电路/稀疏自编码器)系统学去**算法课 15**。
- 评测危机:NLP 评测综述类工作(基准饱和、污染、LLM-as-judge 的偏置;具体篇目随时效,待核)。LLM-judge 的偏置与忠实性的细致评测方法,去本课**生成评测专章 / 忠实性专章**。

## 10. 小结与承上启下

这一章不教新模型,教的是**判断力**:在所有任务都坍缩成 prompt 之后,如何分辨"被 LLM 吞掉、不值得做学术增量的任务"和"被统一遮蔽、恰恰成了金矿的真问题"。一条判据贯穿始终——**把模型/数据 ×10 它会自动消失吗?会消失的别做,不会消失的才触及结构性障碍。**

按这条判据,我们讲透了五条仍然开放、且不随规模平滑变好的主线:

- **低资源/多语言**:数据稀缺对多数语言是**结构性**的,不会随英语模型变大而缓解;分词税(ρ 倍 token 膨胀)把"语言不公"变成可测量的量;跨语言迁移有"多语言诅咒"的容量-覆盖硬权衡。
- **长文档/长上下文**:窗口能放进去(O(L²) 的贵)和模型用得好(lost-in-the-middle)是两码事;"宣称窗口 ≠ 有效窗口",必须用探针自测。
- **组合泛化**:纯似然训练不奖励组合性,默认走"学共现"捷径;i.i.d. 高分只证明会插值,primitive split 才暴露会不会组合外推;规模和 CoT 缓解但未根治。
- **agent 化**:语言成为策略/推理草稿/行动接口,把 NLP 疆界从"文本→文本"扩到"文本→世界";单步可靠性的微小提升被 p^k 指数放大,长程一致性是命门。
- **可解释/可控/可信**:机理可解释是回答"理解 vs 模仿"的唯一直接手段;而**评测的元危机(饱和、污染、生成难评)动摇的是整个领域"有没有进步"的根基**——没有可信的尺子,一切 SOTA 都是流沙上的城堡。

**承上**:这五条线把前十四章串成了一个有机整体——文本表示(01)给了"语义空间"这个看组合泛化的几何视角;序列模型(03-05)是 SCAN 上最早暴露脆弱性的被试;LLM 统一(09)制造了本章的身份危机;RAG(10)是长上下文与 agent 的共同底层;而生成评测与忠实性专章正是第 6 节元危机的正面战场。

**启下(也是出师)**:这门 NLP 课到此为止,给你的不是一套"做完即弃"的任务配方,而是**一副在飞速变化的领域里持续判断"什么是真问题"的眼睛**。当下一个新模型、新基准、新"SOTA"出现时,你会本能地问三件事:**它把哪些东西做大就消失了?它在哪个维度仍然系统不可靠?用来证明它的那把尺子,还准吗?** 能稳定问出并回答这三个问题,你就从"会用 NLP 的工程师"长成了"能推动 NLP 的研究者"——这,正是这门课要把你带到的地方。
