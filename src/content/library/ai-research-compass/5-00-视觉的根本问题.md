> 这一章把你从"知道图像就是一堆数字、CV 现在都用深度学习"带到"能从第一性原理说清楚像素和语义之间的鸿沟到底难在哪、为什么手工特征必然撞墙、为什么全连接网络处理图像在数学上就不该用、以及卷积的三大归纳偏置各自精确地编码了自然图像的哪一条统计先验"。这是整门课的地基:后面所有的 CNN 架构、检测分割头、ViT、扩散模型,本质上都是在用不同的工程手段去跨越同一道鸿沟。读完你应该能亲手算出"全连接 vs 卷积处理一张图的参数量相差几个数量级",能默写出局部性 / 平移等变 / 参数共享分别对应图像的什么先验,并能用 PyTorch 加载一张图、看清它的张量结构、用一个手写卷积核把边缘"抠"出来——从而真正理解后面所有架构都在优化的那个对象到底长什么样。

## 一、动机:把"看见"翻译成一个机器能优化的问题

你每天睁眼就在做计算机视觉:一瞥之下就知道桌上有杯咖啡、它离你多远、能不能一把抓住。这件事对你毫不费力,以至于很容易低估它的难度。但当你要让一台机器做同样的事,你拿到的不是"一杯咖啡",而是一个形如 `1080 × 1920 × 3` 的整数数组——两百多万个 0 到 255 之间的数字。**机器的全部输入就是这堆数字,它对"咖啡""杯子""距离"一无所知。从这堆数字到"这是一杯咖啡"之间,没有任何直接的、连续的数学公式可写。** 这道沟,就是本章的母题:**语义鸿沟(semantic gap)**。

做架构研究的人必须把这个"难"从一句感叹拆成**精确的、可被数学刻画的几个性质**,否则你无法理解为什么某个网络结构能跨越它、某个不能。本章的全部任务,就是做三件事:

1. **把鸿沟讲透**——它不是"数据多就行"那么浅,而是源于像素表示和语义标签之间几个根本性的不匹配(高维、对无关变化极度敏感、对相关变化又必须敏感)。
2. **论证为什么是深度学习**——为什么人手写特征(SIFT/HOG)这条路在数学上注定有天花板,而"让特征从数据里自己长出来"为什么能突破它。
3. **推导为什么是卷积**——为什么把图像直接喂全连接网络在参数量上就是灾难,以及卷积凭哪三条归纳偏置(局部性、平移等变、参数共享)恰好契合自然图像的统计结构。

**给做过工程和数学、但视觉深度学习从零的读者的迁移提示:** 你已经熟悉"定义损失、梯度下降优化"这套范式。视觉深度学习没有跳出它——陌生的地方只在于**输入是一个带强空间结构的高维张量**,而这个结构(相邻像素强相关、物体可以平移、同一模式可出现在任意位置)如果不被架构显式利用,模型要么参数爆炸、要么学不动。理解了"卷积是如何把图像的空间先验编码进网络结构"的,你才能理解后面 ViT 为什么敢部分丢掉这些先验、又要付出什么代价。

我们要建立的核心直觉链条是:

```
像素 = 低层强度数组(H×W×C 张量)
  → 语义 = 高层离散概念(标签/框/掩码/三维结构)
  → 鸿沟:二者之间无直接映射,且像素对"无关变化"极度敏感   （§二:刻画难度）
  → 手工特征:人写算子去抹掉无关变化 → 靠人脑先验,脆弱、不可扩展   （§三:为何撞墙）
  → 端到端学习:让特征从数据中自动涌现,层次化逼近语义   （§三:为何突破）
  → 但全连接处理图像 = 参数爆炸 + 丢掉空间结构   （§四:为何不能硬上 FC）
  → 卷积 = 用三大归纳偏置(局部/平移等变/参数共享)契合图像统计先验   （§五:为何是卷积）
```

这一章会把这条链上的每个箭头都补上推导或论证。读完你会明白:**"用深度卷积网络做视觉"不是一句口号,而是一连串被数据结构和优化现实逼出来的、几乎别无选择的工程决策。**

## 二、把"难"讲精确:语义鸿沟到底由什么构成

很多人对 CV 的难度只有"光照变了就认不出"这种零散感受。我们要把它拆成四条**可量化、可分别攻击**的性质。后面整门课的每个技术,几乎都能对应到它在缓解其中某一条。

### 2.1 维度灾难:输入空间大到无法被"覆盖"

一张 `224×224×3`(后面会反复出现的标准输入尺寸)的图像,是 224·224·3 = 150528 维的向量。每一维取值 0–255,**理论上的图像空间大小是 256^150528**——这是一个连"宇宙原子数(约 10^80)"都微不足道的天文数字。

这意味着什么?**你能采集到的任何数据集,在这个空间里都是一个测度几乎为零的稀疏点云。** ImageNet 的一百多万张图,相对 256^150528 这个空间,稀疏到可以忽略。所以视觉模型的核心任务从来不是"记住见过的图",而是**在一个高到荒谬的维度里,从极稀疏的样本中归纳出能泛化到没见过的点的规律**。这一条直接决定了:任何不带强先验(归纳偏置)的模型,在视觉上都需要海量数据才能勉强工作——这正是 §五卷积、以及后续 ViT 与数据量权衡的根。

> 术语直觉:**归纳偏置(inductive bias)** 是模型在见数据之前就自带的假设。比如"相邻像素更可能相关""物体平移后还是同一物体"。偏置越强,需要的数据越少(因为模型不用从头学这些先验),但偏置如果和真实世界不符,会限制上限。这是贯穿全课的一个核心权衡。

### 2.2 表示与语义的不匹配:像素的"近"不是语义的"近"

这是鸿沟最尖锐的一条。在像素空间里度量两张图的距离(比如逐像素差的平方和,L2 距离),**这个距离和语义距离几乎无关**。

举两个互补的例子,把问题钉死:

- **像素远但语义同:** 同一只猫,光照从白天变黄昏,每个像素值都大幅改变,两张图的 L2 距离巨大;但语义标签都是"猫"。把猫平移 50 个像素,几乎每个位置的像素都变了,L2 距离同样巨大,语义却不变。
- **像素近但语义异:** 给一张猫的图加一层人眼几乎看不见的、精心构造的微小噪声(对抗扰动,adversarial perturbation),像素 L2 距离极小,模型却可能以高置信度判成"长臂猿"。

**结论:语义不是像素的连续函数。** 我们想要的理想表示 φ(x),要满足一个苛刻的双重要求:

```
对"语义无关变化"(光照、视角、平移、遮挡、形变)：  φ 要尽量不变(invariant)
对"语义相关变化"(猫→狗、有物体→无物体)：          φ 要敏感(discriminative)
```

**整部 CV 表示学习史,就是在逼近这个"对无关变化不变、对相关变化敏感"的 φ。** 手工特征是人去猜 φ 该长什么样,深度学习是让 φ 从数据里学出来。记住这个 φ 的双重要求,你就有了评判任何视觉表示好坏的统一标尺。

### 2.3 不变性的种类:列出我们要对抗的"无关变化"

把 2.2 里"语义无关变化"展开,这是 CV 反复出现的一张清单(后续每章会针对性处理,本章只建立全景):

- **光照(illumination):** 亮度、色温、阴影改变像素值,语义不变。
- **视角 / 姿态(viewpoint / pose):** 同一物体从不同角度看,二维投影差异巨大(一个三维物体在二维上的投影有无穷多种)。
- **尺度(scale):** 物体远近不同,在图上大小差几十倍。
- **遮挡(occlusion):** 物体被挡住一部分,信息缺失,仍需识别。
- **形变(deformation):** 非刚体(猫可以蜷成一团也可以伸长),同一类内部形状差异巨大(类内方差,intra-class variance)。
- **背景杂乱(clutter):** 物体嵌在复杂背景里,要从干扰中分离。

**一个关键认知:其中视角和尺度的难度,本质来自"三维世界投影到二维像素时丢失了信息"。** 深度、被遮挡的背面,这些信息在单张二维图里根本不存在,模型只能靠先验和上下文去"猜"。这就是为什么三维重建(后续章)是 CV 里最硬的方向之一——它要逆转一个本质上不可逆(信息有损)的投影过程。

### 2.4 长尾、域偏移、标注成本:工程现实的三道附加难度

前三条是"物理/数学"层面的难,这一条是"数据/工程"层面的难,同样决定方向走势:

- **长尾分布(long-tail):** 真实世界里类别频率极不均衡——"人""车"海量,"某种罕见鸟"极少。模型容易只学好头部、忽略尾部。
- **域偏移(domain shift):** 训练数据(比如晴天城市街景)和部署环境(雨夜、另一个国家)分布不同,性能会塌。这是自动驾驶等落地场景的头号杀手。
- **标注成本(annotation cost):** 这是贯穿全课的隐藏主线。**任务粒度越细,标注越贵**:整图一个标签(分类)便宜;框出每个物体(检测)贵;逐像素标注(分割)极贵。**正是标注成本的压力,逼出了后期的自监督学习和图文预训练**(用网上免费的图文对替代人工标注)——这是理解 CLIP、MAE 等为何兴起的内在动机。

> 把 §二收个口:语义鸿沟 = 维度灾难(§2.1)+ 表示语义不匹配(§2.2)+ 多种不变性需求(§2.3)+ 数据工程难题(§2.4)。**后面这门课里每个技术,你都可以问一句"它在缓解这四条里的哪几条",答案往往非常清晰。** 这就是本章给你的诊断框架。

## 三、为什么是深度学习:手工特征的天花板与端到端学习的突破

### 3.1 手工特征范式:人脑当特征工程师

在 2012 年之前,主流 CV 是**两段式流水线**:

```
图像 → [人手工设计的特征提取器 φ_hand] → 特征向量 → [浅层分类器(SVM / boosting)] → 标签
```

特征提取器 φ_hand 是人用数学直觉**手写死**的算子。两个最有代表性的(后续可单独成章,这里讲清它们体现的思想):

- **SIFT(Scale-Invariant Feature Transform,尺度不变特征变换,Lowe,1999/2004):** 核心思路是**显式构造不变性**。它在不同尺度的高斯模糊图(高斯金字塔)里找极值点(对尺度不变),再以该点的主导梯度方向为基准旋转坐标系(对旋转不变),最后用周围区域的**梯度方向直方图**作为描述子(对光照的线性变化不变,因为梯度抵消了整体亮度)。**SIFT 的每一步都是在 §2.2 那个 φ 的双重要求里,人手工地往"对某种无关变化不变"上凑。**
- **HOG(Histogram of Oriented Gradients,方向梯度直方图,Dalal & Triggs,2005):** 把图像分成小格(cell),统计每格内像素梯度的方向分布做直方图,再做块内归一化(对光照不变)。它抓的是"物体的边缘朝向分布",在行人检测上曾是 SOTA。

这套范式能工作,是因为它**把人对视觉不变性的理解,直接编码进了算子**。但它有一个数学上不可逾越的天花板。

### 3.2 为什么手工特征必然撞墙:三个结构性缺陷

**缺陷一:表达力被人脑先验封顶。** φ_hand 是人能想出来、能用解析公式写出来的函数。但 §2.3 列的那些不变性(尤其形变、类内方差、复杂上下文)的真实结构,**远比任何人能手写的公式复杂**。"猫"这个概念对应的像素流形(manifold)极其扭曲,SIFT/HOG 这种基于局部梯度统计的算子,只能捕获它的很浅一层(边缘、纹理),抓不到"部件如何组合成猫"这种高层结构。**人脑能想出的特征,是真实视觉规律的一个低维、粗糙的近似。**

**缺陷二:不可端到端优化,两段割裂。** φ_hand 是固定的,不随任务/数据调整;只有后面的分类器在学。这意味着**特征提取和最终目标(分类准确率)之间没有梯度通路**——你无法用"分错了"这个信号去改进特征本身。特征是"盲"的,它不知道下游要干什么。

**缺陷三:不可扩展(not scalable)。** 给你更多数据、更强算力,SIFT 还是那个 SIFT,性能不会随之提升——**手工特征对数据规模和算力不敏感**。这在"数据和算力即将指数增长"的时代是致命伤:它吃不到红利。事实上,ImageNet 上手工特征 + 浅分类器的 top-5 错误率长期卡在约 26%(具体数字「待核」),怎么调都下不去——不是没调好,是**范式的上限到了**。

### 3.3 端到端学习:让 φ 从数据里"长"出来

深度学习的革命性转变,一句话:**不再手写 φ,而是把 φ 也参数化成一个可学习的网络 φ_θ,和分类器一起,用同一个损失端到端地训。**

```
图像 → [可学习特征 φ_θ(深度网络)] → 特征 → [分类器(也可学习)] → 标签
                          ↑________________ 梯度回传,联合优化 ________________↓
```

这一步同时解掉了 3.2 的三个缺陷,我们逐个对应,讲清**为什么数学上必然更强**:

**解掉缺陷一(表达力):** 深网是**通用函数逼近器**(universal function approximator)——足够宽/深的神经网络可以逼近任意连续函数(Cybenko 1989 / Hornik 1991 的万能逼近定理,针对前馈网络)。所以 φ_θ 的假设空间不再被"人能写出的公式"限制,理论上能逼近那个真实的、极度扭曲的理想 φ。**人脑先验的天花板被拆掉了,换成了"只要数据和容量够,就能学到"的天花板。**

**解掉缺陷二(端到端):** 因为 φ_θ 处处可微,损失对特征参数的梯度可以一路回传(反向传播),**特征提取器现在"知道"下游目标了**——分错一张猫,梯度会去调整"提取猫相关特征"的那些权重。特征不再是盲的,而是为最终任务量身学出来的。

**解掉缺陷三(可扩展):** φ_θ 的能力随参数量、数据量、算力一起增长。给它更多数据,它学到更细的规律;给它更大容量,它能表示更复杂的 φ。**它能吃到数据和算力增长的红利**——这正是过去十几年视觉进步的引擎。

### 3.4 层次化特征:深度为什么不可或缺,以及"涌现"的含义

为什么要"深"?因为视觉概念本身是**组合层次结构(compositional hierarchy)**:像素 → 边缘 → 纹理/角点 → 物体部件 → 完整物体 → 场景。深网的每一层在前一层基础上做非线性组合,**天然匹配这个层次**:

- 浅层学到的是 **边缘、颜色斑块、简单纹理**(类似 Gabor 滤波器,这一点在 AlexNet 第一层卷积核的可视化里被反复观察到)。
- 中层把边缘组合成 **角点、纹理基元、简单形状**。
- 深层把这些组合成 **物体部件(眼睛、车轮)乃至整个物体**。

**关键认知:这种层次不是人设计的,是端到端训练后"涌现"出来的——你给的监督信号只有最终标签,但网络为了分对,自发地在中间层组织出了这个由简到繁的特征金字塔。** 这就是 3.3 那句"让特征从数据中自动涌现"的精确含义。

> 一句话锚定深度的价值:**用同样的参数量,深而窄的网络能表示的函数复杂度,远高于浅而宽的网络。** 直觉上,组合(深度方向的函数嵌套 f∘f∘…)带来的表达力是指数级的,而单纯加宽(同一层堆神经元)只是线性增加。视觉概念是高度组合的,所以"深"在视觉上尤其划算。(严格的深度分离定理是一个活跃的理论方向,具体定理与界「待核」,但"深比浅在表达组合结构上指数级高效"这个定性结论是稳固的。)

### 3.5 转折点:ImageNet + AlexNet(2012)

这一切在 2012 年由两个条件凑齐而引爆:

1. **数据到位:ImageNet(Deng、李飞飞等,2009 起)** 提供了百万级、上千类的标注图像。§2.1 说过视觉需要海量数据来对抗维度灾难,ImageNet 第一次把数据量推到了让深网能学起来的量级。
2. **算力到位:GPU。** 卷积是高度并行的矩阵运算,GPU 的并行架构恰好适配,使训练大型 CNN 在工程上变得可行。

**AlexNet(Krizhevsky、Sutskever、Hinton,2012)** 在这两个条件上,用一个 8 层 CNN(5 卷积 + 3 全连接),配 ReLU 激活、Dropout、双 GPU 训练、数据增强,把 ImageNet top-5 错误率从手工特征的约 26% 一举降到约 16%(具体数字「待核」),把第二名甩开十几个百分点。**这不是渐进改良,是范式的断崖式更替**——一年内整个学界转向 CNN。

它为什么赢,正是 3.3 三条的现实兑现:**端到端学的特征(而非手写 SIFT)+ 吃到了 ImageNet 数据规模 + 吃到了 GPU 算力**。AlexNet 的具体三件法宝(ReLU 破解梯度饱和、Dropout 正则、数据增强)是后续章节的内容,本章你只需记住它的历史地位:**它是"手工特征 → 学习特征"这次范式断裂的引爆点,也是这门课真正的起点。**

## 四、图像作为张量:为什么不能直接用全连接

确立了"用可学习的深网"之后,下一个问题是:**用什么结构的网络?** 最朴素的想法是把图像拉平成一个长向量,喂给全连接网络(fully-connected,FC,也叫多层感知机 MLP)。这一节用两个硬核论证说明:**这样做在参数量和归纳偏置上都是灾难。** 这个"反面教材"正是卷积存在的理由。

### 4.1 图像的张量结构:H × W × C

先把对象定义清楚。一张数字图像是一个三维张量(tensor):

```
形状 (H, W, C)：
  H = 高度(行数,像素)
  W = 宽度(列数,像素)
  C = 通道数(channel)：灰度图 C=1，彩色 RGB 图 C=3(红/绿/蓝三个强度面)
每个元素是一个像素强度,通常 uint8 ∈ [0,255]，或归一化到浮点 [0,1] / [−1,1]
```

> 注意框架约定的坑(现在记住,后面写代码必踩):PyTorch 的卷积层默认用 **(C, H, W)** 顺序(通道在前,channels-first);而常见图像库(PIL、matplotlib、OpenCV)用 **(H, W, C)**(通道在后);训练时再加一个 batch 维 N,变成 **(N, C, H, W)**。读图后几乎总要做一次 `(H,W,C) → (C,H,W)` 的维度转置。新手最常见的 bug 就是维度顺序搞错,导致"把通道当成了高度"。

**为什么是 (H, W, C) 这个结构,而不是一个无结构的长向量?** 因为这个张量带着关键的**空间结构信息**:第 (i, j) 个像素和第 (i+1, j) 个像素在物理上是**相邻**的、强相关的;通道维度 C 是同一空间位置的不同颜色分量。**这个空间邻接关系,是自然图像最重要的先验,也是全连接网络会粗暴丢弃、卷积会精心利用的东西。** 记住这句话,它是 §四和 §五的连接点。

### 4.2 致命问题一:参数量爆炸(给出精确计算)

把图像拉平喂 FC,意味着输入层有 H·W·C 个神经元,全连接到下一层。我们来算参数量——**这是本章必须亲手算的量化分析之一。**

设输入是 `224 × 224 × 3` 的图(ImageNet 标准尺寸),第一隐藏层有 1000 个神经元。全连接层的权重矩阵形状是 `(输入维度) × (输出维度)`:

```
输入维度 = 224 × 224 × 3 = 150528
输出维度 = 1000
权重参数量 = 150528 × 1000 = 150,528,000 ≈ 1.5 亿
(再加 1000 个偏置,可忽略)
```

**仅仅第一层、仅仅 1000 个隐藏单元,就要 1.5 亿个参数。** 作为对照:整个 AlexNet 约 6000 万参数,整个 ResNet-50 约 2500 万参数。**一个全连接的第一层就吃掉了比整个现代主流网络还多几倍的参数。** 如果输入分辨率提到 `1024×1024×3`,这一层参数量直接破 30 亿。

参数量爆炸的直接后果:

- **存储与计算不可承受**:这还只是一层。
- **几乎必然过拟合**:参数量远超任何数据集的样本量,模型有足够容量去"背下"训练集而非泛化(回到 §2.1 维度灾难——稀疏样本配超大容量,泛化无从谈起)。
- **优化困难**:如此巨大的参数空间,梯度下降难以找到好解。

### 4.3 致命问题二:丢掉空间结构(更本质的缺陷)

参数量还只是表象,**更本质的问题是 FC 在数学上不尊重图像的空间结构。**

当你把 `(H, W, C)` 拉平成一维向量再全连接,**像素的二维邻接关系被彻底抹掉了**:在 FC 看来,输入只是 150528 个无序的标量,第 (0,0) 个像素和它右边的 (0,1) 像素,与它和图像另一角 (223,223) 像素,地位完全一样——**FC 不知道哪些像素在空间上挨着。** 它得从数据里把"哪些输入维度是空间相邻的"这件本应免费的先验,从零学起。

还有一个致命点:**FC 没有平移不变性。** 假设网络已经学会了识别图像左上角的一只猫(靠左上角那批输入连接的特定权重)。现在把同一只猫平移到右下角——对 FC 来说,激活的是**完全不同的另一批输入维度、另一批权重**,它得**重新、独立地**学一遍"右下角的猫长什么样"。**同一个模式出现在不同位置,FC 要为每个位置分别学一套参数。** 这是对参数和数据的巨大浪费,且与 §2.3"平移不应改变语义"的先验直接冲突。

**§四的结论:全连接处理图像,既参数爆炸(4.2),又丢掉空间结构、无平移不变性(4.3)。这两个缺陷恰好对应了卷积要解决的两件事——参数共享省参数,局部 + 平移等变保结构。** 卷积不是凭空发明的技巧,而是针对 FC 这两个缺陷的精确解药。

## 五、卷积:用三大归纳偏置契合自然图像的统计先验

现在正面推导卷积。我们会先定义卷积操作并算清它的参数量(对比 §4.2 的 FC),再把它的三大归纳偏置——**局部性、平移等变、参数共享**——逐个讲透:每一个对应自然图像的哪条统计先验、在数学上是什么、解决了 FC 的哪个缺陷。

### 5.1 卷积操作:一个小核在整张图上滑动

卷积层的核心是一个小的**卷积核(kernel / filter)**,比如一个 `3×3×C` 的权重张量。它在输入图上**滑动(slide)**,每滑到一个位置,就和该位置覆盖的局部区域做**逐元素相乘再求和**(点积),输出一个标量;滑遍整张图,得到一张**特征图(feature map)**。

```
设输入特征图 X 形状 (C_in, H, W)，一个卷积核 W 形状 (C_in, K, K)（K 是核边长，如 3）
输出特征图上位置 (i, j) 的值（单个输出通道、暂略 stride/padding）：

  Y[i, j] = Σ_{c=0}^{C_in−1} Σ_{u=0}^{K−1} Σ_{v=0}^{K−1}  X[c, i+u, j+v] · W[c, u, v]  + b

  含义：把核 W 对齐到输入的 (i,j) 处那个 K×K×C_in 的小立方体,逐元素相乘再全部加起来,得一个数。
```

要输出多个特征(检测多种模式),就用 **C_out 个不同的卷积核**,得到 C_out 张特征图,堆叠成输出张量 `(C_out, H', W')`。

输出空间尺寸由一个必须背熟的公式给出(K 核边长,P padding 填充,S stride 步幅):

```
输出边长 H' = ⌊ (H − K + 2P) / S ⌋ + 1     ←（W' 同理）
直觉验证：S=1, P=0, K=3, H=5 → (5−3+0)/1+1 = 3，核能放 3 个位置，对。
          要让输出和输入同尺寸（常用）：S=1 时取 P = (K−1)/2，如 K=3 → P=1。
```

> 术语直觉:**stride(步幅)** 是核每次滑动跳几格,S>1 会下采样(输出变小);**padding(填充)** 是在输入四周补几圈(通常补 0),用来控制输出尺寸、避免边缘信息被忽略。这两个旋钮控制特征图分辨率怎么变,是后续设计网络下采样路径的基本工具。

### 5.2 算参数量:卷积比 FC 省几个数量级(核心量化分析)

把 §4.2 的场景换成卷积来算,直接看差距。输入 `224×224×3`,我们要第一层输出 64 张特征图(C_out=64),用 `3×3` 卷积核:

```
单个卷积核参数量 = C_in × K × K = 3 × 3 × 3 = 27
共 C_out 个核        = 64 × 27 = 1728
加偏置 C_out 个      = + 64
卷积层总参数量 ≈ 1792
```

**对比 §4.2 的全连接第一层:约 1.5 亿参数。卷积层:约 1792 参数。差了约 8 万倍(接近 5 个数量级)。**

请务必看清这个惊人差距**从哪来**:

```
FC 参数量  = (H·W·C_in) × (输出神经元数)         ← 参数量随输入分辨率 H·W 线性暴涨
卷积参数量 = C_in × K × K × C_out                ← 完全不含 H、W！与输入分辨率无关
```

**卷积层的参数量与输入图像的分辨率 H、W 完全无关**——把图从 224 换成 4096,FC 第一层参数涨几百倍,卷积层参数**一个不变**(只是特征图算得更多次)。这个"参数量与分辨率解耦"的性质,正是下面"参数共享"归纳偏置的直接结果。**这就是卷积让深度视觉网络在工程上成为可能的根本原因。**

> 补一个量化直觉:虽然卷积参数极少,但它的**计算量(FLOPs)不小**——每个输出位置都要做一次 K·K·C_in 次乘加,输出有 H'·W'·C_out 个位置,所以这一层 FLOPs ≈ H'·W'·C_out·C_in·K·K·2(乘加各算一次)。即 `参数少但算力可能大`,这是卷积的典型画像:**它用"在很多位置重复使用同一组少量参数"来换表达力,参数省在共享,算力花在重复。** 后续讲高效网络(如深度可分离卷积)就是在动这个 FLOPs 公式。

### 5.3 归纳偏置一:局部性(locality)—— 对应"图像的相关性随距离衰减"

**操作上**:卷积核只看一个 K×K 的局部窗口,而非像 FC 那样连接整张图。每个输出只由输入的一小块决定。这块输入区域叫该输出神经元的**感受野(receptive field)**。

**它编码的图像先验是:自然图像中,像素间的相关性随空间距离迅速衰减——挨得近的像素强相关(同属一条边、一个纹理、一个物体表面),隔得远的像素近似无关。** 这是自然图像最稳健的统计规律之一(可以从自然图像的空间自相关函数测出来,它随距离单调下降)。

**所以局部连接不是信息损失,而是匹配先验:** 既然远处像素本就基本无关,FC 那种"每个输出都连接到远处每个像素"的全局连接,绝大部分连接权重是在拟合噪声、浪费参数。局部性把这些注定无用的远程连接直接砍掉,**是基于图像统计结构的、有理有据的剪枝**。

> 这里埋一个贯穿全课的伏笔:**单层卷积感受野很小(K×K),但层层堆叠,感受野会逐层扩大**——堆 L 层 3×3 卷积(stride 1),感受野约为 `1 + L·(K−1) = 1 + 2L`,线性增长;若中间有 stride 2 的下采样,感受野指数级扩大。**于是网络浅层看局部细节、深层看全局语义,正好对应 §3.4 的层次化特征。** "如何用最少的层/参数把感受野做大到覆盖整个物体"是 CNN 架构设计(空洞卷积、下采样策略)的核心母题之一,后续会反复回到这条。

### 5.4 归纳偏置二:平移等变(translation equivariance)—— 对应"模式可以出现在任何位置"

先严格区分两个常被混淆的词:

```
平移不变(invariance)：    输入平移,输出不变。       f(shift(x)) = f(x)
平移等变(equivariance)：   输入平移,输出跟着平移同样的量。 f(shift(x)) = shift(f(x))
```

**卷积本身是平移等变的,不是平移不变的。** 用 §5.1 的定义可以直接证明:若把输入整体平移 (Δi, Δj),由于卷积核在每个位置做的是同一套运算,输出特征图也会整体平移 (Δi, Δj),特征值不变只是位置搬了家——这就是 `卷积(平移(x)) = 平移(卷积(x))`,定义上的等变。

**它编码的图像先验是:同一个视觉模式(一条边、一个车轮、一只眼睛),可以出现在图像的任何位置,而它"是什么"与"在哪"无关。** 一只猫在左上角还是右下角,都是猫。这条先验,正是 §4.3 中 FC 缺失、要为每个位置重学一遍的东西。

**等变 vs 不变,谁对?分两步看,这是个常被讲糊的点:**

- **特征提取阶段要等变(而非不变):** 我们希望"猫的特征"无论猫在哪都能被同样的核检测到(等变保证了这点——核滑到哪都用同一套权重),**同时保留"它在哪"的位置信息**(等变让位置信息以特征图坐标的形式留存)。位置信息对检测、分割(要定位)至关重要,不能在特征层就丢。
- **不变性是后续逐步、有控制地引入的:** 真正要"猫在哪都判成猫"的不变性,是靠后面的**池化(pooling)** 和**最终的全局聚合**逐步实现的。比如最大池化在小邻域取最大,带来对小幅平移的局部不变;网络末端的全局平均池化把整张特征图压成一个向量,带来对大幅平移的不变。

**所以正确的设计哲学是:卷积层保持等变(留住位置),到了需要做整图判断时,再用池化/全局聚合换取不变。** 这个"先等变、后按需换不变"的分工,是理解 CNN 为什么这样堆叠的关键,也直接决定了分类(末端要强不变)和检测/分割(中间要保位置)在网络结构上的差异。

### 5.5 归纳偏置三:参数共享(parameter sharing)—— 等变的实现机制 + 省参数的来源

**操作上**:同一个卷积核的那组权重,在滑过整张图的**所有位置时是同一套**——核滑到 (0,0) 用的权重,和滑到 (100,100) 用的,完全相同。这叫**权重共享(weight sharing / weight tying)**。

参数共享是上面两条偏置在工程上的**实现机制和直接收益来源**,它一举做成三件事:

1. **它就是平移等变的来源(§5.4):** 正因为每个位置用同一套权重,平移输入才会得到平移的、值不变的输出。如果每个位置一套独立权重(那叫局部连接但不共享),就只有局部性、没有等变性了。**等变来自参数共享(同一套权重滑遍各位置);局部性负责省参数。二者合起来才是卷积,但等变的根本来源是共享。**
2. **它是 §5.2 参数量爆炸式下降的来源:** FC 是每个输出位置一套独立权重;卷积是所有位置共用一套核。`参数量 = C_in·K·K·C_out`,与位置数(H·W)无关——省掉的就是"为每个位置重学一遍"的那部分。这也是为什么参数量与分辨率解耦。
3. **它带来更好的泛化(统计角度):** 共享权重等于对"同一模式在不同位置应被同样处理"这一先验施加了**硬约束(hard constraint)**,极大缩小了假设空间。假设空间越小、越贴合真实先验,所需样本越少、越不易过拟合(回到 §2.1 维度灾难——参数共享是对抗它的核心武器)。一个数据样本里物体出现的那个位置,通过共享,等价于为所有位置都提供了训练信号。

> 把三大偏置收口成一句话:**局部性砍掉了远程无用连接(对应"相关性随距离衰减"),参数共享让同一套权重滑遍全图(对应"模式可出现在任何位置"),二者合起来给出平移等变(对应"是什么与在哪无关")——这三条恰好精确编码了自然图像的核心统计结构,既把参数量压低 5 个数量级,又把图像的空间先验注入网络。这就是为什么卷积是视觉的默认起点,而不是某个偶然好用的技巧。**

### 5.6 一个清醒的边界:归纳偏置是双刃剑

最后必须诚实地补一句,否则你会以为"卷积永远最优":**归纳偏置强,在数据少时是优势(省数据、防过拟合),但它也是对模型的约束,在数据极多时可能成为上限。**

- 卷积假设"局部 + 平移等变",这在大多数自然图像上成立。但当数据规模大到模型可以"自己从数据里学出这些先验"时,**硬编码这些偏置反而限制了灵活性**。
- 这正是 **Vision Transformer(ViT,2020)** 的故事:它几乎丢掉卷积的局部性和平移等变(改用注意力让每个图块直接关注所有图块),归纳偏置弱得多。**代价是数据需求陡增**——小数据上 ViT 打不过 CNN,但喂到足够大的数据(如三亿张图的 JFT,具体数据集与规模「待核」)时,ViT 反超。

**这个"归纳偏置 vs 数据规模"的权衡,是贯穿整门课的主旋律之一(§2.1 已埋下),也是你读任何架构论文时该问的第一个问题:它假设了什么先验?这个先验在我的数据规模下是帮忙还是碍事?** 本章先把卷积这一端讲透,ViT 那一端留到后续章节。

## 六、CV 任务谱:同一道鸿沟,六种跨法

前面讲的"像素→语义"是抽象母题。落到具体,CV 是一族任务,**区别只在于"语义"被定义成什么形状的输出**,以及由此决定的难度和监督成本。这张谱是你给整门课定位的坐标系。

```
任务            输入            输出(语义的形状)                     难度/标注成本   本课位置
─────────────────────────────────────────────────────────────────────────────────────
图像分类        一张图          一个类别标签(整图级)                 低 / 低         最基础,先讲
目标检测        一张图          若干 (框坐标 + 类别)                 中 / 中         分类→定位
语义分割        一张图          逐像素类别图 (H×W 每点一个标签)       高 / 高         密集预测
实例分割        一张图          逐像素 + 区分同类不同个体             高 / 很高       检测+分割
图像生成        噪声/文本       一张全新的、符合分布的图              高 / 看范式     GAN/扩散
三维重建        一/多张图       三维结构(深度/网格/辐射场)           很高 / 很高     逆投影,最硬
视频理解        帧序列          时空标签(动作/跟踪/时序事件)         高 / 高         加时间维
```

读这张谱的三个关键认知:

1. **输出空间越大,任务越难、标注越贵。** 分类输出是一个标量(从 |类别| 个里选一个);分割输出是 H×W 个标量;三维重建输出是连续的三维结构。**输出维度的暴涨,直接对应难度和标注成本的暴涨**(回到 §2.4 标注成本主线)——这是后期自监督、生成式方法兴起的内在压力。
2. **大多数任务可分解为"在分类骨干上加不同的头(head)"。** 一个在 ImageNet 上学好特征的卷积骨干(backbone),换上分类头做分类、换上框回归头做检测、换上逐像素预测头做分割。**这就是为什么本课从分类讲起——它训练出的骨干和表示,是几乎所有下游任务的共同基础**(迁移学习的根)。
3. **三维重建和视频理解多了"被丢失的维度"。** 三维重建要逆转 §2.3 那个有损的二维投影(恢复被丢的深度);视频要处理多出来的时间维(运动、时序因果)。**它们之所以最硬,是因为要从二维/单帧里恢复出本不在输入里的信息**,只能靠强先验和多视角/多帧约束。

**这张谱的用法:** 后面每学一个模型,先在谱里定位它解决哪个任务、输出是什么形状、监督从哪来,你就不会在细节里迷失方向。

## 七、设计权衡与常见坑

**坑 1:用像素 L2 距离判断图像"像不像"。** §2.2 已证像素距离与语义距离基本无关。算图像相似度、做检索、做损失,几乎从不用裸像素 L2(它对光照/平移极度敏感)——而是在**学出的特征空间**里度量(感知损失 perceptual loss、特征距离)。把这条忘了,会写出一个"对人眼几乎一样的两张图却给出巨大 loss"的错误目标。

**坑 2:把图像拉平喂 MLP。** §四的反面教材。除了极小图(如 28×28 的 MNIST,FC 尚可勉强跑)或已被骨干提取成低维特征之后,**绝不要把原始高分辨率图拉平进全连接**——参数爆炸 + 丢空间结构,两头不讨好。原始图像进网络,第一步几乎总是卷积(或 ViT 的 patch embedding)。

**坑 3:混淆平移等变与平移不变。** §5.4 的核心区分。卷积是等变,池化/全局聚合才逐步带来不变。设计检测/分割网络时若在中间层过早做强不变操作(如过激的池化),会丢掉定位所需的位置信息,导致框/掩码定位变差。**要定位的任务,中间层要保等变、留位置。**

**坑 4:维度顺序 (H,W,C) vs (C,H,W) vs (N,C,H,W) 搞错。** §4.1 的工程坑。PIL/OpenCV 读出来是 (H,W,C),PyTorch 卷积要 (N,C,H,W)。漏掉转置或 batch 维,轻则报错,重则把通道当成空间维静默地算错。读图后第一件事就是确认并调整维度顺序。

**坑 5:以为"归纳偏置越强越好"或"越弱越好"。** §5.6 的权衡。没有绝对优劣,只有"和你的数据规模匹不匹配"。小数据用强偏置的 CNN,大数据可考虑弱偏置的 ViT。读论文、做选型时,先问"它假设了什么先验、我的数据撑不撑得起这个假设"。

**坑 6:忽视域偏移,只看训练集指标。** §2.4。训练集准确率高不代表部署能用——分布一变就塌。任何要落地的视觉系统,都要单独评估目标域(乃至 OOD,分布外)表现,而不是只盯训练/验证集那一个分布。

## 八、动手练习

**练习 1(量化,核心):全连接 vs 卷积处理一张图的参数量差。**
输入图像 `256 × 256 × 3`。
(a) 若第一层是全连接、输出 512 个神经元,算权重参数量。
(b) 若第一层是卷积、输出 512 张特征图、用 `5×5` 卷积核(stride 1, padding 2),算这一层的参数量,并用 §5.1 公式算输出特征图的空间尺寸 H'×W'。
(c) 两者参数量相差几个数量级?用 §5.2 的两条公式解释"为什么卷积参数量与输入分辨率 256 无关,而全连接与它成正比"。
*提示:(a) 256·256·3·512。(b) 卷积参数 = C_in·K·K·C_out + C_out = 3·5·5·512 + 512;输出尺寸 (256−5+2·2)/1+1 = 256(same padding)。(c) 量级差约 3~4 个数量级(具体约 2587 倍,≈10^3.4;因本题 C_out=512、5×5 核使卷积参数比 §5.2 大,故比值小于 §5.2 的 5 个数量级,但"参数量与分辨率解耦"这一定性结论不变),根因是参数共享让权重与位置数 H·W 解耦。*

**练习 2(分析,本章题眼):三大归纳偏置各对应图像的什么先验。**
分别针对**局部性、平移等变、参数共享**,各写清楚三件事:(i) 它在卷积操作上具体是什么;(ii) 它编码了自然图像的哪一条统计先验(要能举出一个图像层面的具体现象作为该先验的证据);(iii) 它解决了全连接处理图像时的哪个缺陷(对应 §4.2 参数爆炸 或 §4.3 丢结构/无平移不变中的哪条)。最后用一句话说明为什么平移等变的根本来源是参数共享(而局部性只负责省参数、并非等变的必要成分)。
*提示:局部性↔相关性随距离衰减↔砍远程无用连接;平移等变↔模式可出现在任意位置↔FC 要为每个位置重学;参数共享↔同一模式同样处理↔参数爆炸 + 是等变的实现机制。最后一问回到 §5.5 第 1 点。*

**练习 3(编码):加载一张图、看张量结构、手写边缘滤波器做卷积。**
用 PyTorch 完成:(a) 读入任意一张图(用 PIL 或 torchvision),打印它转成张量后的 `dtype` 和 `shape`,确认通道顺序,并把它整理成 `(N, C, H, W)`、归一化到 [0,1];(b) 手工构造一个 Sobel 横向边缘核(检测竖直边缘,见下方提示),用 `torch.nn.functional.conv2d` 作用在灰度图上,得到边缘响应图;(c) 把原图和边缘图可视化对比,观察"卷积如何把人没标注过的边缘抠出来",并思考:这个核里的 9 个数是人手工设定的,而 CNN 训练让这些数从数据里学出来——这正是 §三"手工特征 vs 学习特征"的微缩对照。
*提示:Sobel 横向核 = [[−1,0,1],[−2,0,2],[−1,0,1]];conv2d 要求权重形状 (C_out, C_in, K, K),灰度图 C_in=1,所以 reshape 成 (1,1,3,3);记得 padding=1 保持尺寸。下方 §九前的代码块可直接作为起点。*

**练习 4(推导,选做):感受野如何随深度增长。**
只用 `3×3`、stride 1、padding 1 的卷积层堆叠。(a) 推导堆 L 层后,最深层一个神经元的感受野边长是多少(用 L 表示)。(b) 若在第 3 层后插入一个 stride 2 的下采样,后续层的感受野(换算回原图尺度)如何变化?定性说明"为什么下采样能让感受野指数级扩大"。(c) 据此论证 §5.3 的伏笔:为什么"浅层看局部细节、深层看全局语义"是卷积堆叠的自然结果。
*提示:(a) 每层 3×3 让感受野边长 +2,L 层后约 1+2L。(b) stride 2 之后每个像素对应原图 2×2 区域,再往后每层在"已被压缩的尺度"上扩 2,换算回原图是翻倍式增长。(c) 把 (a)(b) 和 §3.4 层次化特征对起来。*

下面是练习 3 可直接起步的代码骨架,它同时演示了 §4.1 的张量结构和 §5.1 的卷积机制:

```python
import torch
import torch.nn.functional as F
from PIL import Image
import torchvision.transforms.functional as TF

# (a) 读图 → 看张量结构 → 整理成 (N, C, H, W)
img = Image.open("some_image.jpg").convert("L")   # 转灰度,C=1,方便演示
x = TF.to_tensor(img)        # to_tensor: (H,W,C) uint8[0,255] → (C,H,W) float[0,1]
print("dtype:", x.dtype, "shape (C,H,W):", x.shape)   # 确认通道在前、已归一化
x = x.unsqueeze(0)           # 加 batch 维 → (N=1, C=1, H, W),conv2d 要求的格式

# (b) 手工构造 Sobel 横向边缘核(检测竖直边缘) —— 这 9 个数是"手工特征"
sobel_x = torch.tensor([[-1., 0., 1.],
                        [-2., 0., 2.],
                        [-1., 0., 1.]])
# conv2d 权重要求形状 (C_out, C_in, K, K);这里 C_out=1, C_in=1
weight = sobel_x.view(1, 1, 3, 3)

# 卷积:padding=1 配 3×3 核保持空间尺寸(对应 §5.1 的 P=(K-1)/2)
edges = F.conv2d(x, weight, padding=1)   # 输出 (1,1,H,W),每点是该处的横向梯度强度
print("edge map shape:", edges.shape)

# (c) 可视化对比(留给练习):把 x[0,0] 和 edges[0,0].abs() 并排画出来
#     观察:没有任何标注,纯靠一个固定的小核滑过全图,边缘就被"抠"了出来。
#     思考:CNN 训练 = 让这 9 个数(及成百上千个核)从数据里自动学出来。
```

## 九、源码 / 论文导读

按"先建立直觉、再啃原文"的顺序:

- **手工特征范式(理解被取代的是什么):** Lowe, "Distinctive Image Features from Scale-Invariant Keypoints"(SIFT, 2004);Dalal & Triggs, "Histograms of Oriented Gradients for Human Detection"(HOG, 2005)。读它们如何**显式手工地构造尺度/旋转/光照不变性**,对照 §3.1–3.2,你会切身体会"人写不变性"的精巧与天花板。
- **范式转折点(本课起点):** Krizhevsky、Sutskever、Hinton, "ImageNet Classification with Deep Convolutional Neural Networks"(AlexNet, 2012)。重点看它如何用端到端学习的特征碾压手工特征(对照 §3.3、§3.5),以及第一层卷积核可视化呈现的边缘/Gabor 模式(对照 §3.4 层次化特征)。ImageNet 数据集本身参见 Deng 等(2009)。
- **卷积与归纳偏置的奠基:** LeCun 等, "Gradient-Based Learning Applied to Document Recognition"(LeNet-5, 1998)——卷积网络的源头,看它如何用局部连接 + 权重共享处理图像(§五三大偏置的原始出处)。理论侧:Goodfellow、Bengio、Courville《Deep Learning》第 9 章(Convolutional Networks)对局部连接、参数共享、等变性的论述,与本章 §5.3–5.5 一一对应,是把本章数学讲严的最佳教材章节。
- **归纳偏置的另一端(为后续埋线):** Dosovitskiy 等, "An Image Is Worth 16×16 Words"(ViT, 2020)——读它的引言和数据规模实验部分,对照本章 §5.6 的权衡:弱归纳偏置 + 大数据如何反超 CNN。现在只需读个所以然,细节留到后续 ViT 章。
- **开源实现(把数学落成代码):** **torchvision.models**——`torchvision.models.resnet50()` 看一个真实卷积骨干怎么堆叠(关键词 `Conv2d`、`stride`、`padding`、`downsample`),`torchvision.transforms` 看图像如何被读入、转张量、归一化(对应 §4.1 和练习 3)。`torch.nn.Conv2d` 的文档参数(`in_channels, out_channels, kernel_size, stride, padding`)与本章 §5.1–5.2 的参数量/尺寸公式逐一对应,是验证你算的参数量是否正确的最快途径。进阶可看 **timm** 库里的卷积骨干实现,工业级地展示了感受野/下采样的设计。

## 十、小结与承上启下

这一章把整门课的**问题框架**立住了。你现在应该能回答最根本的几个问题:

- **CV 的母题是什么?** 跨越**语义鸿沟**——从低层像素强度数组(H×W×C 张量)到高层离散语义概念之间没有直接映射的那道沟。它由维度灾难、像素/语义不匹配、多种不变性需求、数据工程难题四条构成(§二),给了你诊断任何视觉技术的统一框架。
- **为什么是深度学习?** 手工特征(SIFT/HOG)靠人脑先验,表达力被封顶、不可端到端、不可扩展(§3.2);端到端学习让特征 φ_θ 从数据中自动涌现成层次化结构,拆掉了这三个天花板(§3.3–3.4);ImageNet + AlexNet(2012)是这次范式断裂的引爆点(§3.5)。
- **为什么是卷积、不是全连接?** 全连接处理图像参数爆炸(1.5 亿 vs 1792,约 5 个数量级)且丢掉空间结构、无平移不变(§四);卷积用三大归纳偏置——**局部性**(对应相关性随距离衰减)、**平移等变**(对应模式可出现在任意位置)、**参数共享**(等变的实现 + 省参数的来源)——恰好契合自然图像的统计先验(§五)。而归纳偏置是双刃剑,强偏置省数据但限上限,这个"偏置 vs 数据规模"的权衡贯穿全课(§5.6)。
- **CV 是一族什么任务?** 分类、检测、分割、生成、三维重建、视频理解,区别在"语义被定义成什么形状的输出",输出空间越大越难越贵;多数任务 = 共享骨干 + 不同的头,这是从分类讲起的理由(§六)。

**承上启下:** 我们已经知道"要跨越语义鸿沟、要用可学习的卷积特征、卷积凭三大偏置契合图像先验"。但还没回答**"如何把卷积层组织成一个真正能学、能训深、能跨越鸿沟的网络"**——单层卷积感受野太小、堆深了会有梯度和退化问题、特征图的分辨率与通道数如何随深度变化、用什么激活和归一化让它训得动。**下一章起,我们将从最基础的卷积层、激活、池化讲起,沿着 LeNet → AlexNet → VGG → ResNet 这条主线,一层层看清"卷积网络是如何被一步步设计成能跨越鸿沟的"**;再往后是检测/分割如何在骨干上加头(对应 §六的任务谱)、ViT 如何走向 §5.6 那条弱偏置 + 大数据的路线、以及生成与三维。本章建立的"语义鸿沟四要素 + 归纳偏置 vs 数据规模"这套语言,将是后面评判每一个架构改进"到底在跨越鸿沟的哪一段、付出了什么代价"的统一标尺。
