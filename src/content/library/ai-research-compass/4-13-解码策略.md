> 这一章把你从"知道模型每步吐出一个 softmax 分布"带到"能从第一性原理推导 temperature 对分布熵的影响、能讲清 top-k/top-p 的自适应差异、能徒手实现一个工业级采样函数、能解释为什么最大化概率反而生成空洞重复文本"。**训练决定了模型有多好,解码决定了你能把它用出多好。** 同一组权重,greedy 和 nucleus 采样可以产生天差地别的文本质量——这一章要让你对"从分布到文本"这最后一公里没有任何黑箱。

## 一句话引言

推理时,自回归模型在每个位置给出词表上的一个分布 `P(xₜ | x_<t)`。**解码策略就是一个把这串分布"具体化"成一个 token 序列的算法。** 本章从"为什么不能简单地每步取最大"讲起,推透确定性方法(greedy / beam search)、随机采样(temperature / top-k / top-p)的数学,剖开"似然最大化导致退化"这个反直觉现象,再串起重复惩罚、约束解码、投机解码这些工程手段。

## 一、动机:训练目标和推理目标不是同一件事

在第 06 章我们确立了:decoder-only 自回归模型用最大似然(MLE)训练,目标是让模型分布 `P_θ(xₜ|x_<t)` 逼近真实数据分布。训练完之后,模型就是一个**函数**:输入一段前缀 `x_<t`,输出词表 V 上的一个 logits 向量 `z ∈ ℝ^|V|`,过 softmax 得到概率分布。

但这里有一个**绝大多数初学者忽略的鸿沟**:训练只教会了模型"给定前缀,每个词出现的概率是多少";它**没有教模型"在推理时具体该选哪个词、怎么把整段话生成出来"**。从"逐位置的条件分布"到"一整段文本",中间隔着一个完全独立的算法决策——这就是**解码(decoding)**,也叫**生成(generation)** 或**搜索(search)**。

为什么这件事不平凡?三个层层递进的理由:

**理由一:逐词贪心 ≠ 整句最优。** 你可能想:既然要"概率最大的句子",那每步取概率最大的词不就行了?**错。** 局部最优不等于全局最优。设我们要找的是使整句联合概率 `P(x_1...x_T) = ∏ₜ P(xₜ|x_<t)` 最大的序列。每步取 argmax(贪心)只看当前一步,完全可能因为第 1 步贪了一个 0.6 的词,而错过"第 1 步选 0.4 但后续都是 0.9"的更优路径。这是一个**指数大的搜索空间**(|V|^T 条候选序列),精确求最优在计算上不可行——所有解码策略本质都是在这个指数空间里做的**近似搜索或采样**。

**理由二:训练分布 ≠ 你想要的分布。** 这是第 06 章 exposure bias 的延续。MLE 让模型去拟合"人类文本的平均统计",但你在具体任务里想要的,未必是"平均"。翻译任务你想要**唯一正确**的译文(应该偏确定性);开放写故事你想要**多样、有惊喜**的文本(应该偏随机性)。同一个模型,不同任务,需要的解码行为完全不同。解码策略就是你在推理时**注入任务先验**的旋钮。

**理由三:最大化概率本身会出问题。** 这是本章最反直觉、也最重要的发现——**人类语言并不总是走高概率路径**。一味追求"最可能的句子"(greedy / beam search)会掉进**神经文本退化(neural text degeneration)** 的坑:输出乏味、空洞、不断重复。我们会在第四节把这件事的机制讲透。

**给做过工程的读者的迁移提示:** 把解码理解成一个**在概率图上的搜索/采样问题**会很顺。greedy 是贪心搜索,beam search 是有限宽度的 best-first 搜索,采样方法是从分布里抽样的蒙特卡洛。你熟悉的"搜索 vs 采样""利用 vs 探索(exploitation vs exploration)"的全部直觉都能迁移过来:确定性方法是纯利用(总选最可能的),会陷入局部模式;随机方法引入探索,能跳出来但可能采到坏 token。

本章的核心地图:

```
模型给出每步分布 P(xₜ|x_<t)
  │
  ├─ 确定性(最大化似然)── greedy（贪心，每步 argmax）
  │                      └ beam search（维护 k 条束，近似找最高概率序列）
  │        ↓ 但最大化似然在开放生成上会"退化"（第四节）
  │
  └─ 随机采样 ─── temperature（缩放 logits 调锐度，第三节推导）
                ├ top-k（截断到概率最高 k 个）
                └ top-p / nucleus（截断到累积概率 ≥ p 的最小集合，自适应）
  ↓ 叠加约束
  重复惩罚 / no-repeat-ngram / 语法约束 / 投机解码（无损加速）
```

## 二、确定性解码:greedy 与 beam search

### 2.1 greedy:每步取 argmax

最朴素的策略:每一步选当前条件概率最大的词,把它当作已生成的前缀,继续下一步。

```
xₜ = argmax_w P(w | x_<t),   t = 1, 2, ..., 直到生成 <eos> 或达到最大长度
```

**算法是确定性的**(同一前缀永远输出同一结果),实现只需一行 `argmax`,零额外开销。它求的是"逐步局部最优"的序列,**不保证**得到全局联合概率最高的序列。

greedy 的致命问题是**重复退化(repetition)**。一旦模型进入某个状态,它给某个词很高的概率,选了它之后又回到类似状态,又给同一个词高概率……于是陷入死循环:

```
The cat sat on the mat. The cat sat on the mat. The cat sat on the mat. ...
```

为什么会这样?直觉是:**高概率 token 一旦被选中并喂回上下文,会进一步强化模型对同类 token 的偏好**——这是一个正反馈环。我们在第四节给出更定量的解释。

### 2.2 beam search:维护 k 条候选束

既然贪心会因为"一步走错满盘皆输",一个自然的改进是**同时保留多条候选**。beam search(束搜索)维护宽度为 k(beam width / beam size)的候选集:每一步,对当前 k 条部分序列各自扩展所有可能的下一个词,得到 k·|V| 条新序列,**按累积对数概率排序,只保留 top-k 条**,丢弃其余。重复直到所有束都生成 <eos>。

用对数概率(log-prob)而非概率累乘,是因为概率连乘会迅速下溢到 0(几百个 0.x 相乘),取对数把乘法变加法,数值稳定:

```
score(x_1...x_t) = Σᵢ log P(xᵢ | x_<i)        # 累积对数概率，越大越好
```

beam search 的完整一步(伪代码):

```python
# beams: List of (token_sequence, cumulative_log_prob)，初始 [([], 0.0)]
def beam_step(beams, model, k):
    candidates = []
    for seq, score in beams:
        logp = log_softmax(model(seq))          # 当前束的下一词 log 概率，形状 [|V|]
        topk_logp, topk_idx = logp.topk(k)      # 只需各束的前 k 个，剪枝
        for lp, w in zip(topk_logp, topk_idx):
            candidates.append((seq + [w], score + lp.item()))
    # 全局按累积 log-prob 排序，保留最优 k 条 —— 这是 beam search 的核心剪枝
    candidates.sort(key=lambda x: x[1], reverse=True)
    return candidates[:k]
```

**为什么 beam search 适合翻译/摘要这类"有正确答案"的任务?** 这类任务存在一个或少数几个**高质量、高概率的目标序列**,你的目标确实是"找到模型认为最可能的那条译文"。beam search 是对"找最高概率序列"这个 argmax 问题的有效近似搜索:k 越大,搜到的序列联合概率通常越高,越接近全局最优。机器翻译里 beam=4~8 是经典配置,常比 greedy(beam=1)有稳定的 BLEU 提升。

**长度归一化(length normalization)是 beam search 的必要补丁。** 注意 `score` 是 log-prob 之和,而 log P ≤ 0,所以**序列越长,累积分数越负(越小)**。直接用原始 score 排序,beam search 会**系统性偏好短序列**(早早生成 <eos>),输出残缺。修正办法是除以长度的某个幂:

```
score_norm = (Σᵢ log P(xᵢ|x_<i)) / (T^α),   α ∈ [0.6, 1.0] 常见
```

α=1 是按平均每词 log-prob 排序;α<1 是温和地补偿长序列。**这是新手用 beam search 时最常见的坑之一:不做长度归一化,输出莫名其妙地短。** (具体 α 默认值随实现而异,**待核**:不同框架/论文取值不同,需查你用的实现。)

### 2.3 beam search 在开放生成上反而更差

这里是关键的反直觉点:**beam search 在翻译上是利器,在开放式生成(讲故事、闲聊、续写)上却往往产出比随机采样更乏味、更重复的文本。** 因为它本质是在**更努力地最大化似然**——而我们将在第四节论证,在开放生成场景里,**似然最大化本身就是错的目标**。beam=1(greedy)已经会退化,beam search 用更大的搜索宽度把"找最高概率序列"这件事做得更彻底,结果是退化得更整齐:更长的精确重复片段、更安全也更无聊的措辞。**搜索做得越好,离"像人写的"越远**——这正是把退化问题逼出来的钥匙,放到第四节展开。

**复杂度。** 每步要对 k 条束各跑一次前向(或 batch 成一次 k 倍的前向),并做 k·|V| 的排序。相比 greedy,时间和显存都是约 **k 倍**(KV cache 也要存 k 份)。这是 beam search 的实打实成本——为有限的似然提升付 k 倍算力。

## 三、随机采样:temperature、top-k、top-p

开放生成需要**多样性**,所以我们不再取 argmax,而是**从分布里抽样**:`xₜ ~ P(· | x_<t)`。直接按原始 softmax 概率采样(常称 ancestral sampling / 纯采样)是最朴素的随机解码。但纯采样有个大问题:**长尾**。词表有几万个词,即便每个低概率词只有万分之一,几万个加起来的尾部总概率可观,采样时总会偶尔抽到一个完全不合适的词,**一个坏 token 就能毁掉整段连贯性**。三个旋钮——temperature、top-k、top-p——就是用来驯服这件事的。

### 3.1 Temperature:缩放 logits 调节锐度

**Temperature(温度,记 T)** 把 logits 除以 T 再做 softmax:

```
P_T(i) = softmax(z / T)_i = exp(zᵢ / T) / Σⱼ exp(zⱼ / T)
```

直觉:T 是一个"锐度旋钮"。**T < 1 让分布更尖锐(放大 logits 差距,更偏向高概率词);T > 1 让分布更平坦(更接近均匀,更随机)。** T=1 就是原始分布。下面把两个极限严格推出来。

**极限一:T → 0⁺,趋向 greedy(argmax)。** 设最大 logit 是 `z_max`(下标 m,假设唯一最大)。把分子分母同除以 `exp(z_max/T)`:

```
              exp((zᵢ - z_max)/T)
P_T(i) = ─────────────────────────────────
          Σⱼ exp((zⱼ - z_max)/T)

对 i = m：分子 = exp(0) = 1
对 i ≠ m：zᵢ - z_max < 0，所以 (zᵢ - z_max)/T → -∞（当 T→0⁺），exp(...) → 0

⟹ 分母 → 1 + 0 + ... = 1，  P_T(m) → 1，其余 P_T(i) → 0
```

所以 **T→0 时分布塌缩到一个 one-hot,等价于 argmax,也就是 greedy。** 实践里 T 不能真取 0(除零),工程上 T→0 直接走 argmax 分支。

**极限二:T → ∞,趋向均匀分布。** 当 T → ∞,`zᵢ/T → 0`,于是 `exp(zᵢ/T) → exp(0) = 1` 对所有 i 成立:

```
P_T(i) → 1 / Σⱼ 1 = 1 / |V|   （对所有 i 相等）
```

**分布趋向均匀,模型几乎在词表上瞎猜,文本完全失控。** 这两个极限框定了 T 的语义:T 是在"确定性"(0)和"完全随机"(∞)之间连续滑动的旋钮。

**定量结论:T 单调增大,分布的熵单调不减。** 这是 temperature 最该被讲透的性质。熵 `H(P_T) = -Σᵢ P_T(i) log P_T(i)` 衡量分布的"不确定性/多样性"。我们证明 **dH/dT ≥ 0**(熵随温度单调不减),从而 T 越大文本越多样、越不可预测。

引入 inverse temperature `β = 1/T`,这样 `P_β(i) = exp(β zᵢ)/Σⱼexp(β zⱼ)`(这就是统计物理里的玻尔兹曼/吉布斯分布,T 正是物理"温度"——这个名字不是比喻)。我们证更顺手的形式:**熵 H 对 β 单调不增**(等价于对 T 单调不减,因为 β=1/T 是 T 的减函数)。

```
设 logZ(β) = log Σⱼ exp(β zⱼ)  （配分函数的对数，cumulant generating function）

【步骤 1】配分函数对数的一阶导 = logits 在分布下的均值
  d(logZ)/dβ = Σⱼ zⱼ exp(β zⱼ) / Σₖ exp(β zₖ) = Σⱼ zⱼ P_β(j) = E[z]

【步骤 2】二阶导 = 方差
  d²(logZ)/dβ² = E[z²] - (E[z])² = Var[z] ≥ 0     （配分函数对数是凸函数）

【步骤 3】把熵写成 logZ 和 E[z] 的组合
  log P_β(i) = β zᵢ - logZ(β)
  H = -Σᵢ P_β(i) log P_β(i)
    = -Σᵢ P_β(i) (β zᵢ - logZ)
    = -β E[z] + logZ                                 （因 Σᵢ P_β(i) = 1）

【步骤 4】对 β 求导
  dH/dβ = -E[z] - β d(E[z])/dβ + d(logZ)/dβ
        = -E[z] - β · Var[z] + E[z]                  （步骤1：d(logZ)/dβ=E[z]；步骤2：d(E[z])/dβ=d²logZ/dβ²=Var[z]）
        = -β · Var[z]

【结论】β = 1/T > 0，Var[z] ≥ 0  ⟹  dH/dβ = -β·Var[z] ≤ 0
        即熵对 β 单调不增；因 β=1/T 是 T 的减函数，β 减则 T 增，故 H 对 T 单调不减。 ∎
```

**这个推导给你三个超出"背结论"的洞察:**
1. **熵随 T 单调上升有严格保证**,不是经验观察。T 大 ⟹ 熵大 ⟹ 文本更不可预测、更多样。
2. **变化率正比于 logits 的方差 Var[z]**。当模型对下一个词"很有把握"(几个 logit 远高于其余,方差大),调 T 的效果剧烈;当模型"很犹豫"(logits 都差不多,方差小),调 T 几乎没用。**所以 temperature 的实际敏感度是上下文相关的**——这是很多人调参时困惑"同样改 0.2,有时天翻地覆有时毫无变化"的根因。
3. **等号 Var[z]=0 当且仅当所有 logit 相等**,此时分布恒为均匀,T 完全失效。

### 3.2 Top-k 采样:截断到概率最高的 k 个

temperature 重塑了整个分布,但**长尾依然在**(只是被压扁/拉高)。**top-k 采样**直接动刀:只保留概率最高的 k 个词,把其余全部置零,在这 k 个里重新归一化后采样。

```
V_k = {概率最高的 k 个 token}
P'(i) = P(i) / Σ_{j∈V_k} P(j)  （若 i ∈ V_k，否则 0），然后从 P' 采样
```

top-k 的好处是**硬性切断长尾**,坏处是 **k 是固定的,不随分布形状自适应**——这是它的核心缺陷,也是 nucleus 采样要解决的问题。看两个极端上下文:

- **分布很尖(模型很确定),比如下个词几乎必然是 "the"**:概率可能是 the=0.95,后面全是 0.001 量级。此时若 k=40,你会把 39 个**几乎不该出现的低概率词**纳入候选,采样时有不小概率抽到垃圾,**该确定的地方反被引入噪声**。
- **分布很平(模型很开放),比如句子开头可以接很多词**:可能有几百个词概率都在 0.005 上下。此时 k=40 又**太小**,把大量同样合理的候选粗暴砍掉,**该多样的地方被人为限制**。

**一个固定的 k 没法同时适配这两种情况。** top-k 的"截断位置"是按**排名**定的,而排名和分布的实际形状脱钩。

### 3.3 Top-p / nucleus 采样:截断到累积概率 ≥ p 的最小集合

**Top-p 采样(又称 nucleus sampling,核采样;Holtzman et al., 2020)** 换了一个截断准则:不按"取前 k 个",而按"取累积概率刚好达到 p 的最小集合"。把 token 按概率从高到低排序,从最高的开始累加,直到累积概率首次 **≥ p**,这一组词就是 **nucleus(核)**,在核内归一化后采样。

```
按概率降序排列 token：P(w_(1)) ≥ P(w_(2)) ≥ ...
找最小的 n 使得  Σ_{i=1}^{n} P(w_(i)) ≥ p     # 例如 p=0.9
核 V_p = {w_(1), ..., w_(n)}
P'(i) = P(i) / Σ_{j∈V_p} P(j)  （i∈V_p 否则 0），从 P' 采样
```

**为什么 nucleus 比固定 k 更自适应?核心在于它的截断阈值是按"概率质量"而非"排名"定的,核的大小 n 随分布形状自动伸缩。** 回到上面两个上下文(取 p=0.9):

- **分布很尖(the=0.95)**:第一个词就已经累积 0.95 ≥ 0.9,核里**只有 1 个词**。nucleus 自动塌缩成 greedy,该确定时绝不引入噪声。
- **分布很平(几百个词各 0.005)**:要累加近 180 个词才到 0.9,核里**有上百个词**。nucleus 自动放开,该多样时给足候选。

**同一个 p=0.9,核大小从 1 自适应到上百——这就是 nucleus 的"自适应性"。** 它截掉的永远是"概率分布的尾巴(那 1-p 的低质量长尾)",而**保留的高质量主体大小随模型的确定程度自动变化**。这正好对症"模型在不同位置确定程度差异巨大"这一事实:在 the 之后该确定,在句子开头该发散,top-p 用一个固定旋钮自动处理了两种情形,而 top-k 做不到。

**实践组合与典型值。** 三个旋钮常一起用,且**先 temperature,后 top-k/top-p 截断,再归一化采样**(温度改变分布锐度,会改变累积概率的分布,因而影响 top-p 的核大小;顺序不能错)。开放生成的常见档位是 **T≈0.7~1.0,top-p≈0.9~0.95**,或 top-k≈40;事实问答/代码生成偏确定,常用低温甚至 greedy。(这些是社区经验区间,**待核**:不同模型/任务的最优值需实测,不存在普适默认。)

### 3.4 一份可运行的采样实现(PyTorch)

把 temperature + top-k + top-p 合在一个函数里。这是 HF transformers 里 `TemperatureLogitsWarper` / `TopKLogitsWarper` / `TopPLogitsWarper` 的精简自洽版,逐行注释机制:

```python
import torch
import torch.nn.functional as F

def sample_next_token(logits, temperature=1.0, top_k=0, top_p=1.0):
    """
    logits: [vocab_size]  当前步未归一化的 logits（模型直接输出）
    返回: 采样得到的 token id（标量 LongTensor）
    顺序：温度 → top-k 截断 → top-p 截断 → 归一化采样
    """
    logits = logits.clone()

    # ---- 1. temperature：除以 T 改变锐度（见 3.1 推导）----
    if temperature <= 0:                 # T→0 退化为 greedy，避免除零
        return torch.argmax(logits)
    logits = logits / temperature

    # ---- 2. top-k：把第 k 大以外的 logit 置 -inf（softmax 后变 0）----
    if top_k > 0:
        k = min(top_k, logits.size(-1))
        # 第 k 大的值作为阈值；严格小于它的全部屏蔽
        kth_value = torch.topk(logits, k)[0][..., -1, None]
        logits = logits.masked_fill(logits < kth_value, float('-inf'))

    # ---- 3. top-p / nucleus：保留累积概率刚好 ≥ p 的最小集合 ----
    if top_p < 1.0:
        sorted_logits, sorted_idx = torch.sort(logits, descending=True)
        cum_probs = torch.cumsum(F.softmax(sorted_logits, dim=-1), dim=-1)
        # 标记“累积已超过 p”之后的位置（这些要被移除）
        remove = cum_probs > top_p
        # 右移一位：保证至少保留第 1 个 token，且核内首次达到 p 的那个词被保留
        remove[..., 1:] = remove[..., :-1].clone()
        remove[..., 0] = False
        # 映射回原始 token 顺序再屏蔽
        remove_orig = remove.scatter(-1, sorted_idx, remove)
        logits = logits.masked_fill(remove_orig, float('-inf'))

    # ---- 4. 归一化为概率并按概率多项式采样 ----
    probs = F.softmax(logits, dim=-1)
    return torch.multinomial(probs, num_samples=1).squeeze(-1)
```

**三个易错点(代码里都已处理):**
- **置 -inf 而非乘 0**:截断作用在 softmax **之前**的 logits 上(加 -inf,exp 后为 0);若在 softmax 后乘 0,剩余概率和不为 1,破坏归一化(和第 03 章因果掩码同一个坑)。
- **top-p 的"右移一位"**:目的是**保留"首次让累积 ≥ p 的那个词"**(它属于核),并保证 `top_p` 很小时**至少留 1 个 token**(否则可能一个都不留,采样崩溃)。
- **顺序**:温度在前,截断在后;温度会改变 softmax 后的累积概率,从而改变 top-p 的核大小。

## 四、退化问题:为什么最大化概率反而生成空洞重复文本

这是本章的"灵魂拷问",也是 nucleus 采样那篇论文(标题就叫 *The Curious Case of Neural Text Degeneration*)的核心。**一个训练得很好(困惑度很低)的语言模型,用 greedy/beam search 解码,会产出大量重复、乏味、空洞的文本。** 为什么?

### 4.1 现象:似然最大化 → 退化

把现象先钉死:beam search/greedy 这类**最大化似然**的解码,在开放生成上典型失败模式是:
- **重复**:整句、整段循环复读("I don't know. I don't know. I don't know...")。
- **乏味/空洞**:大量高频套话、安全但无信息的措辞,缺乏惊喜。

而**纯随机采样**(尤其高温)走另一个极端:**不连贯、跑题、胡言乱语**(因为采到长尾坏 token)。两个极端之间,nucleus 采样找到了"既连贯又不退化"的甜区。

### 4.2 机制一:人类语言不走"最高概率路径"

最深刻的解释来自一个对人类文本的观测:**人类说话/写作,下一个词的概率(在一个好模型下度量)是剧烈起伏的,而不是一路走高。** 人类会不断引入**信息量高(即概率低)** 的词——这正是"信息"的定义:`信息量 I(w) = -log P(w)`,低概率词携带高信息。一段全是高概率词的文本,信息量极低,读起来就是空话。

Holtzman 等人(2020)的关键图示(定性复述,**待核**:具体数字以原文为准)显示:beam search 生成文本的 per-token 概率**长期居高不下且平稳**,而人类文本的 per-token 概率**剧烈震荡、频繁探入低概率区**。**这意味着"最可能的句子"恰恰是"最不像人写的句子"——最大化似然这个目标,在开放生成上和"像人"是冲突的。** 这是 exposure bias(第 06 章)之外,解码层面独立的一个深刻问题:**即便模型完美拟合了 P(human text),argmax/beam 这种解码方式也会系统性偏离人类文本的真实"形状"**,因为人类文本本身不是 P 的众数(mode),而是从 P 里"采"出来的、带高信息起伏的样本。

**一句话钉住:开放生成要的是"从分布里采一个典型样本",不是"找分布的众数"。** 高维分布的众数往往是个**退化的、不典型的点**(类似高维高斯里,概率密度最大的是原点,但随机采的样本几乎都落在远离原点的薄球壳上——众数不代表典型样本)。beam search 在找众数,所以拿到的是不典型的退化文本。

### 4.3 机制二:重复的正反馈环

为什么具体表现为**重复**?给一个机制性解释:

```
设模型已生成片段 s。把 s 重新喂回当前上下文后：
- 自回归注意力会强烈关注近期 token；若上下文里已出现 "A B A B"，
  模型对"接着出 A"赋予的概率会因为这个局部模式被进一步抬高。
- 选了高概率的 A → 上下文变成 "...A B A B A" → 模型对"接着 B"概率更高 → ...
  形成自我强化的正反馈：P(重复 | 已重复) 随重复次数单调上升。
```

实证上,研究观察到**一个 n-gram 重复出现的概率会随其已出现次数增加而上升**(模型"越复读越想复读")。greedy/beam 总选最高概率,一旦踏入这个环就**无法自拔**(确定性策略没有跳出机制);而随机采样有一定概率选别的词,**天然能打断正反馈**——这是采样方法比贪心更不易重复的根本原因之一。

### 4.4 为什么 beam search 在翻译上没这个问题

回到第二节的悬念。翻译/摘要这类任务**没有退化问题**,因为:
- **目标分布很尖**:给定源句,正确译文是少数几个高概率序列,**众数恰好就是好答案**,argmax/beam 找众数找对了。
- **长度和内容被输入强约束**:不能随便重复(会偏离源句语义),模型的高概率路径本身就受源句牵制,不容易陷入空转。

**而开放生成的目标分布很"宽"且"多峰"**(同一个开头有无数种合理续写),众数不再对应好文本,采样才是对的。**结论:解码策略的选择本质取决于"任务的目标分布是尖还是宽、众数是否等于好答案"。尖→确定性(beam),宽→采样(nucleus)。** 这是贯穿全章的判据。

## 五、约束与增强:重复惩罚、no-repeat-ngram、语法约束、投机解码

退化和长尾之外,工程上还要给解码加各种约束。这些都可以统一理解为**在采样前对 logits 做一次变换(logits processor)**,或对搜索空间做硬性限制。

### 5.1 重复惩罚(repetition penalty)

直接对**已经出现过的 token** 压低 logits,从概率层面抑制重复(Keskar et al., 2019, CTRL 提出的形式):

```
对已生成序列中出现过的 token i：
   zᵢ ← zᵢ / r        若 zᵢ > 0     （r > 1，如 1.1~1.3）
   zᵢ ← zᵢ · r        若 zᵢ ≤ 0
（正 logit 除以 r 变小、负 logit 乘 r 更负 —— 两种情况都让该 token 概率下降）
```

注意正负分开处理:因为除法对正数是"缩小",对负数却是"放大"(更接近 0),所以负 logit 要用乘法才能进一步压低。**r=1 即不惩罚;r 太大(如 >1.5)会过度抑制,导致模型不敢复用必要的词(代词、专有名词),文本变得别扭。** 这是个需要克制的旋钮。

### 5.2 no-repeat-ngram:硬性禁止 n-gram 重复

更强硬的做法:**禁止任何长度为 n 的 token 序列出现第二次**。实现上,在每步生成前,检查"当前末尾的 (n-1) 个 token + 候选词"是否构成一个已出现过的 n-gram,若是就把该候选词的 logit 置 -inf。

```python
def banned_tokens_by_ngram(prev_tokens, n):
    # 返回当前步必须禁止的 token 集合：使得任一 n-gram 不重复出现
    if len(prev_tokens) < n - 1:
        return set()
    prefix = tuple(prev_tokens[-(n-1):])         # 当前末尾 n-1 个 token
    banned = set()
    seen = {}                                    # 记录历史上每个 (n-1)-prefix 后面跟过哪些 token
    for i in range(len(prev_tokens) - n + 1):
        ng = tuple(prev_tokens[i:i+n])
        seen.setdefault(ng[:-1], set()).add(ng[-1])
    return seen.get(prefix, set())               # 若当前 prefix 历史上跟过某些 token，全禁
```

**典型用 n=3(no_repeat_ngram_size=3),即不允许任何 3-gram 重复。** 它能干净利落地消除复读,但**对天然需要重复的内容(诗歌叠句、代码里反复出现的 `for i in range`、列表项)会误伤**。这是硬约束的通病:简单粗暴但不分青红皂白。

### 5.3 约束解码(constrained decoding):JSON / 语法约束

很多场景要求输出**符合特定结构**:合法 JSON、匹配某个正则、符合一段 BNF 文法(比如让模型只输出特定枚举值,或生成可解析的 SQL)。约束解码的核心思想:**在每一步,根据"当前已生成的前缀 + 目标文法",计算出哪些 token 是合法的下一步,把所有非法 token 的 logit 置 -inf,只在合法集合内采样。**

```
状态机/语法驱动的掩码：
  维护一个解析器状态（如 JSON 解析到 "期待一个 value" 或 "期待逗号或 }"）
  每步：合法下一 token 集合 = 文法在当前状态允许的转移
        logits[非法 token] = -inf
        采样 → 更新解析器状态
```

这保证**输出 100% 满足约束**(而不是靠 prompt 祈祷模型守规矩)。代表性工具/方法:基于有限状态机把正则/JSON schema 编译成 token 级掩码(如 Outlines/guidance 类库,**待核**:具体库的实现细节)。

**关键工程坑:文法约束作用在 token 上,但文法通常定义在字符上。** 一个 token 可能跨越文法边界(比如 token `"},` 同时含结构符号),token 化和文法的对齐很麻烦——这是约束解码工程上最硬的部分,需要预编译"token→是否能推进文法状态"的映射。**约束解码不改变模型对合法 token 之间的相对偏好,只是把非法选项清零**;因此它能保证格式,但不能凭空让模型"内容更对"。

### 5.4 投机解码(speculative decoding):从算法视角看无损加速

自回归生成天然是**串行**的:生成第 t 个 token 必须等第 t-1 个生成完。这导致大模型推理慢——每个 token 都要完整跑一次几百亿参数的前向。**投机解码(Leviathan et al., 2023;Chen et al., 2023)用一个巧妙的算法把它部分并行化,且严格无损**(输出分布与原模型逐 token 采样完全一致)。

核心思想:**小模型起草,大模型并行验证。**

```
设 P = 大模型(target)分布，Q = 小模型(draft)分布（同词表，便宜很多）

1. 起草：用小模型 Q 自回归快速生成 γ 个候选 token  x̃₁...x̃_γ（串行，但小模型快）
2. 验证：用大模型 P 对这 γ 个位置做【一次并行前向】，同时拿到 P 在每个位置的分布
        （关键：验证 γ 个已知 token 只需一次前向，因为 token 已给定，可并行算 logits）
3. 接受/拒绝（逐位置，保证无损）：对位置 i 的草稿 token x̃ᵢ：
     以概率 min(1, P(x̃ᵢ|·) / Q(x̃ᵢ|·)) 接受它；
     若拒绝，则从修正分布 norm(max(0, P(·) - Q(·))) 重采一个 token，并丢弃 i 之后所有草稿。
4. 若全部 γ 个都被接受，再从 P 额外采 1 个 → 一次大模型前向最多前进 γ+1 个 token。
```

**为什么无损?** 这里的接受-拒绝是一个精心设计的采样方案,可以**证明**:经过"以 min(1, P/Q) 接受、否则从 norm(max(0,P-Q)) 重采"后,最终每个位置吐出的 token 其边际分布**恰好等于 P**——和直接用大模型逐 token 采样的分布逐位完全相同。直觉:小模型猜得准(Q≈P)的地方,接受率 P/Q≈1,几乎都接受,省下大模型的串行步;小模型猜错的地方被拒绝并用残差分布 max(0,P-Q) 补偿,残差恰好填平 Q 与 P 的差。**它不是近似、不是质量换速度——输出分布严格不变,纯赚吞吐。**

```
为什么 min(1, P/Q) + 残差重采 = 从 P 采样（无损性证明骨架）：
某个 token x 被最终输出的总概率
  = P[草稿是 x 且被接受] + P[草稿是 y≠x 被拒后残差重采到 x]
  接受项 = Q(x)·min(1, P(x)/Q(x)) = min(Q(x), P(x))
  残差重采到 x 的概率 = (整体拒绝概率) × norm_resid(x)
     其中 残差分布 ∝ max(0, P(x)-Q(x))，整体拒绝概率 = Σ_y (Q(y)-min(Q(y),P(y))) = Σ_y max(0,Q(y)-P(y))
     可证 残差归一化常数 = Σ_y max(0,P(y)-Q(y)) = 整体拒绝概率（两者相等，因 ΣP=ΣQ=1）
     ⟹ 残差项 = max(0, P(x)-Q(x))
  合计 = min(Q(x),P(x)) + max(0, P(x)-Q(x)) = P(x)    ∎  （对 P>Q 和 P≤Q 两种情况各自验证相等）
```

**加速比取决于"接受率"** ——小模型与大模型分布的一致程度。一致性越高,平均每次大模型前向能"放行"越多 token。**算法视角到此为止;为什么验证 γ 个 token 只要一次前向、KV cache 如何复用、draft 模型怎么选——这些系统实现细节在 MLSys 课展开。** 这里你只需牢牢记住两点:**(1) 串行瓶颈来自自回归的逐 token 依赖;(2) 投机解码用"草稿 + 并行验证 + 接受拒绝"在不改变输出分布的前提下把串行步数摊薄。**

## 六、设计权衡与常见坑

**权衡总览:确定性 ↔ 随机性,本质是"任务目标分布是尖还是宽"。** 把它内化成一张决策表:

```
任务                目标分布   众数=好答案?   推荐解码
机器翻译/摘要        尖、单峰    是            beam search (k=4~8) + 长度归一化
事实问答/抽取        尖          是            greedy 或 低温(T≈0.2)
代码生成            较尖        多数情况是      低温 + 适度 top-p（要确定但偶尔需变体）
开放写作/对话/头脑风暴 宽、多峰   否(要采典型样本) nucleus (T≈0.8, top-p≈0.9)
```

**坑 1:在开放生成里用 beam search 求"更好"的文本。** 反了。beam 越宽,越逼近众数,越退化(更长的重复、更无聊的安全话)。**开放生成默认用采样,不是搜索。**

**坑 2:以为 temperature 在所有上下文里效果一致。** 由 3.1 推导,温度的边际效果正比于 logits 方差 `Var[z]`:模型确定时(方差大)调 T 很灵,犹豫时(方差小)调 T 几乎无效。别用一个固定 T 期待全程一致的"创造力"。

**坑 3:top-k 在尖分布上引噪、在平分布上砍多。** 固定 k 不自适应。**默认优先用 top-p**;要更稳可 top-k 和 top-p 叠加(取两者交集),把"既不超过 k 个、又不超过累积 p"作为候选集。

**坑 4:截断后忘记重新归一化,或在 softmax 之后乘 0。** 必须在 softmax **之前**置 -inf,再 softmax 才能得到合法分布。softmax 后乘 0 会让概率和 < 1,`multinomial` 行为未定义。

**坑 5:beam search 不做长度归一化 → 输出过短。** log-prob 之和天然偏向短序列(每多一个词就再减一个非负量)。必做长度归一化(除以 T^α)。

**坑 6:重复惩罚/no-repeat-ngram 一刀切误伤。** 对需要重复的内容(代码、列表、诗歌叠句)硬禁 n-gram 会破坏正确性。**这类结构化生成宁可用约束解码(保证格式)也别用 no-repeat-ngram(误伤内容)。**

**坑 7:T=0 在代码里除零。** T→0 是 greedy 的极限但不能真取 0,工程上 T≤0 直接走 argmax 分支(见 3.4 代码)。

**坑 8:把投机解码当成"质量换速度"。** 它**无损**,输出分布严格等于原模型。如果你观察到用了投机解码后输出"变差",几乎一定是实现 bug(接受-拒绝逻辑错),而非算法本身的代价。

**坑 9:seed 与可复现。** 采样是随机的,**不固定随机种子,同一 prompt 每次输出都不同**;调试/评测时务必固定 seed,否则你分不清"改了策略"和"只是抽样波动"。greedy/beam 是确定性的,不受此影响。

## 七、动手练习

## 九、小结与承上启下

这一章我们把"模型算完概率之后到底怎么变成文本"这件常被忽略、却直接决定产品体验的事讲透了:

1. **解码是独立于训练的算法决策**。训练只给出逐位置分布 `P(xₜ|x_<t)`;从分布到序列是一个指数空间(|V|^T)里的近似搜索/采样问题,你在这里注入任务先验。
2. **确定性方法(greedy/beam)在最大化似然**。beam search 用 k 条束近似找最高概率序列,适合**目标分布尖、众数=好答案**的任务(翻译/摘要),必配长度归一化;但它**不保证全局最优**,且在开放生成上越努力越退化。
3. **随机采样三旋钮**:temperature 缩放 logits 调锐度(**严格证明熵对 T 单调不减,变化率 ∝ Var[z]**);top-k 按排名硬截断(不自适应);**top-p/nucleus 按累积概率质量截断,核大小随分布形状自适应**(尖→塌缩成 greedy,宽→放开),这是它优于固定 k 的根因。
4. **退化问题是解码层独立于 exposure bias 的深刻现象**:人类文本靠低概率词承载信息、不走最高概率路径,而高维分布的众数是不典型的退化点——**所以开放生成要"采典型样本"而非"找众数",beam/greedy 找众数就掉进重复空洞**。重复还有"正反馈环"的机制,采样能打断它。
5. **工程约束**:重复惩罚/no-repeat-ngram(简单但会误伤)、约束解码(token 级文法掩码,保证格式)、**投机解码(草稿+并行验证+接受拒绝,严格无损加速,不是质量换速度)**。

**贯穿全章的判据一句话:看任务的目标分布是尖还是宽、众数是不是好答案——尖且众数好就用确定性搜索,宽且要典型样本就用采样。** 把这句话刻进直觉,你就掌握了解码策略选择的全部要义。

**这一章在整门课里的位置**:它是**第 06 章(预训练目标)埋下的伏笔的收口**——decoder-only AR 留下的 exposure bias 缺口,一半靠后训练(RLHF)补,一半靠解码策略补;它也是连接**算法与系统**的关节:本章讲清了投机解码"为什么无损"的算法本质,而"为什么快、KV cache 怎么复用"的系统实现,正是 MLSys 课的内容。**更重要的是,它把你的视角从"模型内部"拉到了"模型作为一个采样器如何被使用"** ——后面的对齐(RLHF)、推理(reasoning,链式思考其实也是一种解码时的探索)、Agent 工具调用(常需约束解码保证可解析),都建立在你这一章对"从分布到文本"的透彻理解之上。**记住:再强的模型,解码用错了也发挥不出来;解码策略是你手里最便宜、最即时的质量旋钮。**
