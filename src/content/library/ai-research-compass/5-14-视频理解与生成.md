> 这一章把你从"会做单帧图像的分类/检测/生成、知道 ViT 和扩散"带到"能从第一性原理说清在空间之上为什么必须显式建时间、能徒手写 3D 卷积与分解时空注意力并算清它们的 FLOPs/显存、能定量比较 3D 卷积 / 双流 / 时空注意力三条路线的算力与建模能力、能讲透视频扩散里'逐帧独立必闪烁'的数学根因和时间注意力如何把它治好、并理解 VLA/世界模型为什么是把视觉从'看'推向'行动'的接口"。本章不重讲 2D 卷积、ViT、扩散——那是 01/07/11 章的事,这里默认你会。本章只回答一个问题:**多了一根时间轴 t,模型该怎么变,代价是什么。**

## 一句话引言

视频不是"一堆图像",而是**一段在时间上高度相关、且承载了运动这一全新信息的信号**。加上时间维带来两件事:一件是机会——**运动(motion)是图像里没有的、对"发生了什么"极具判别力的信号**;一件是代价——**时间轴让数据量、算力、显存全部乘上帧数 T,且暴力地把时间当成"第四个空间维"会让复杂度爆炸**。视频建模的全部工程,本质是在"如何高效抽取时序信息"和"如何不被 T 这个乘数压垮"之间反复权衡。理解了这条主线,从 C3D 到 TimeSformer 到 Sora 类视频扩散,你看到的都是同一道题的不同解法。

## 1. 动机:为什么"时间"不能当成又一个空间维

最朴素的想法:图像是 H×W,视频是 T×H×W,那把单帧那套 2D 操作直接升成 3D(在时间上也卷、也注意力)不就行了?这想法**方向对、但忽略了三个本质差异**,正是这三点定义了整个领域的难度。

**差异一:时间轴的语义和空间轴不同,但又不完全独立。** 空间上 x/y 两个方向是对称的(图像转 90° 还是合理图像)。时间则有方向性(箭头):一个人坐下和起立是时间倒放的关系,但语义完全相反。更关键的是,**空间相关性和时间相关性的尺度不同**——相邻像素几乎总是强相关,相邻帧大部分像素也几乎不变(背景静止),真正变化的是局部运动区域。这意味着把时间当成对称的第三个空间维去做各向同性卷积,既浪费(大量算力花在不变的背景上),又错失方向性信息。

**差异二:运动是独立于外观的判别信号。** 判断"这是在跑步还是在走路",单看任何一帧的外观(appearance,物体长什么样)可能都分不清,**信息藏在帧间的位移里**。光流(optical flow)——每个像素从这一帧到下一帧的二维位移场——把这个运动信号显式地抽出来。一个核心经验事实:**外观和运动是两类互补的信号**,这直接催生了双流网络(§3)。

**差异三:T 是个残暴的乘数。** 这是最硬的约束,先把账算清。一段视频按 16 帧、224×224 的常见输入:

```
单帧像素:    224 × 224 = 50,176
16 帧:       16 × 50,176 = 802,816 个像素位置
ViT patch 化(16×16):
  单帧 token: (224/16)² = 196
  16 帧:      16 × 196 = 3,136 个 token
全局时空自注意力复杂度:O(n²),n=3136 → n² ≈ 9.83×10⁶  (单帧 196²≈3.8×10⁴,差 256 倍)
```

**时间维让 token 数线性乘 T,让自注意力复杂度乘 T²。** 一段 10 秒、30fps 的视频有 300 帧,token 数 5.88 万,全局时空注意力的 n² 是 34.6 亿——和当年 ViT 拒绝"逐像素 token"时面对的数字同量级。**于是整个视频领域,无论理解还是生成,第一性的设计压力都是:在不丢时序建模能力的前提下,把这个 T(或 T²)的乘数想办法压下去。** 记住这句话,后面每个架构都是它的注脚。

## 2. 3D 卷积:把卷积核也"加厚"一维

### 2.1 从 2D 到 3D:机制与等价形式

2D 卷积核是 `Cᵢₙ × K × K`,在 H、W 两个空间维上滑动。**3D 卷积**(Tran 等人 C3D,2015)只是把核加厚一个时间维,变成 `Cᵢₙ × k_t × K × K`,在 T、H、W 三个维上同时滑动。一个 3D 卷积层的前向(忽略偏置)对输出位置 (t, y, x) 的某个输出通道 c_out:

```
out[c_out, t, y, x] = Σ_{cᵢₙ} Σ_{i=0}^{k_t−1} Σ_{j=0}^{K−1} Σ_{l=0}^{K−1}
                        W[c_out, cᵢₙ, i, j, l] · in[cᵢₙ, t+i, y+j, x+l]
```

**机制直觉**:这个核同时看 k_t 帧、每帧 K×K 的空间邻域,因此一次操作就能捕捉"局部时空模式"——比如"一只手在 3 帧内向上移动 K 个像素"这种短时运动。这是 3D 卷积相对 2D 的根本增益:**时序信息在每一层、每个核里被局部地、隐式地建模**,不需要外接光流。

### 2.2 关键账:参数量、FLOPs、感受野

设一层 3D 卷积:输入 `Cᵢₙ` 通道、输出 `C_out` 通道、时间核 `k_t`、空间核 `K×K`,输出时空尺寸 `T'×H'×W'`。

**参数量**(决定模型大小、过拟合风险):
```
params = C_out × Cᵢₙ × k_t × K × K   (+ C_out 偏置)
对比 2D 同配置:params_2D = C_out × Cᵢₙ × K × K
比值 = k_t   →  时间核 k_t=3 时,参数量正好是 2D 的 3 倍
```

**FLOPs**(决定算力/延迟;一次乘加算 2 FLOPs):
```
FLOPs ≈ 2 × (C_out × T' × H' × W') × (Cᵢₙ × k_t × K × K)
                └────输出元素数────┘   └──每个输出的乘加数──┘
```
对比 2D 卷积处理同样 T' 帧(逐帧独立)的总 FLOPs 是 `2 × C_out × T' × H' × W' × Cᵢₙ × K × K`,**3D 比逐帧 2D 多出 k_t 倍**。这就是 3D 卷积"贵"的根因——核厚了 k_t 倍,参数和计算都乘 k_t。

**时间感受野**(决定能建多长的时序依赖):一层时间核 k_t 的时间感受野是 k_t。L 层堆叠(时间步长为 1),时间感受野线性增长:
```
RF_t(L) = 1 + L × (k_t − 1)        # 与 01 章空间感受野同一公式,只换到时间轴
k_t=3、L=8:  RF_t = 1 + 8×2 = 17 帧
```
**这是 3D 卷积的天花板之一:要"看到"长程时间依赖(比如一个持续 5 秒、150 帧的动作),要么堆很深,要么加时间步长/池化下采样。** 局部性先验在时间轴上同样是双刃剑——高效但近视。

### 2.3 I3D 的关键招:用 2D 预训练权重"膨胀"出 3D

3D 卷积有个现实痛点:**参数多 k_t 倍,而视频标注数据远比图像稀缺**(ImageNet 有 128 万张图,早期动作识别数据集只有几千到几万段视频),从头训 3D 网络极易过拟合且烧钱。

I3D(Inflated 3D ConvNet,Carreira & Zisserman,2017)的核心 trick 叫**膨胀(inflation)**:把一个在 ImageNet 上训好的 2D 网络(如 Inception),逐层"加厚"成 3D,并用 2D 权重初始化 3D 权重。机制是:

> 把一张 2D 核 `W_2D ∈ ℝ^{C_out×Cᵢₙ×K×K}` 沿时间维复制 k_t 份,**再除以 k_t**,得到 3D 核 `W_3D[:,:,i,:,:] = W_2D / k_t`(对所有 i)。

为什么除以 k_t?这是个可以验算的关键设计,**目的是让"把一段静止视频(所有帧相同)喂进 3D 网络"得到的逐帧响应,等于"把单帧喂进原 2D 网络"的响应**——即膨胀后初始行为与 2D 预训练完全一致,不浪费预训练知识。验算:设输入是常数帧 in[:, t+i, y+j, x+l] = X[:, y+j, x+l](与 t、i 无关),则

```
out_3D = Σ_cᵢₙ Σ_i Σ_j Σ_l (W_2D[c,cᵢₙ,j,l]/k_t) · X[cᵢₙ, y+j, x+l]
       = Σ_i (1/k_t) · [ Σ_cᵢₙ Σ_j Σ_l W_2D[c,cᵢₙ,j,l] · X[cᵢₙ,y+j,x+l] ]
       = Σ_i (1/k_t) · out_2D            # 内层方括号与 i 无关
       = k_t × (1/k_t) × out_2D = out_2D  ✓
```
完美等价。**膨胀让 3D 网络站在 2D 预训练的肩膀上起步**,这是 I3D 把动作识别精度大幅推高的关键工程突破,也是"如何把图像先验迁移到视频"这一母题的第一个经典答案——你会发现视频扩散(§6)解决"数据稀缺 + 充分利用图像模型"时用的是同一思想。

### 2.4 代码:3D 卷积块与膨胀初始化

```python
import torch
import torch.nn as nn

class Conv3DBlock(nn.Module):
    """一个最小的 3D 卷积块:时空卷积 + BN + ReLU。
       kernel=(k_t, K, K),在 (T,H,W) 上同时滑动。"""
    def __init__(self, c_in, c_out, k_t=3, k_s=3, stride=(1,1,1)):
        super().__init__()
        pad = (k_t//2, k_s//2, k_s//2)          # same padding,保持时空尺寸(stride=1 时)
        self.conv = nn.Conv3d(c_in, c_out, kernel_size=(k_t, k_s, k_s),
                              stride=stride, padding=pad, bias=False)
        self.bn = nn.BatchNorm3d(c_out)          # 注意是 3D 版 BN,统计量也在 T 维上聚合
        self.act = nn.ReLU(inplace=True)
    def forward(self, x):                        # x: (B, C, T, H, W)
        return self.act(self.bn(self.conv(x)))

@torch.no_grad()
def inflate_2d_to_3d(conv2d: nn.Conv2d, k_t: int) -> nn.Conv3d:
    """I3D 膨胀:把 2D 卷积权重沿时间维复制 k_t 份并 /k_t,初始化 3D 卷积。"""
    c_out, c_in, K, _ = conv2d.weight.shape
    conv3d = nn.Conv3d(c_in, c_out, kernel_size=(k_t, K, K),
                       stride=(1, conv2d.stride[0], conv2d.stride[1]),
                       padding=(k_t//2, conv2d.padding[0], conv2d.padding[1]),
                       bias=conv2d.bias is not None)
    # 关键三步:复制到时间维 → 除以 k_t → 保证静止视频响应 == 2D 响应
    w2d = conv2d.weight                          # (c_out, c_in, K, K)
    w3d = w2d.unsqueeze(2).repeat(1, 1, k_t, 1, 1) / k_t   # (c_out, c_in, k_t, K, K)
    conv3d.weight.copy_(w3d)
    if conv2d.bias is not None:
        conv3d.bias.copy_(conv2d.bias)
    return conv3d
```

**坑提示**:`BatchNorm3d` 的统计量是在 (B, T, H, W) 上聚合的,batch 里帧数多会让有效样本数变大、统计更稳,但也意味着小 batch 时时间维的引入反而让 BN 统计更不稳;这是 3D 网络常用 GroupNorm 替代 BN 的原因之一。

### 2.5 把 3D 卷积分解:(2+1)D —— 用更少参数逼近 3D

3D 卷积贵在那个 `k_t×K×K` 的实心立方核。R(2+1)D(Tran 等人,2018)提出**把一个 3D 卷积分解成"一个 2D 空间卷积 + 一个 1D 时间卷积"**串联:

```
3D 核 (k_t × K × K)  ≈  空间核 (1 × K × K)  然后  时间核 (k_t × 1 × 1)
```
参数量对比(中间通道设为 M):
```
3D:        C_out × Cᵢₙ × k_t × K × K
(2+1)D:    [M × Cᵢₙ × 1 × K × K] + [C_out × M × k_t × 1 × 1]
```
当中间通道 M 选得合适(论文取 M 使两者参数量大致相等以公平对比),**(2+1)D 在相同参数预算下表达力更强**——因为它在空间卷积和时间卷积之间多插了一个非线性激活,等价于把一个线性时空滤波拆成两段非线性变换,**网络更深、非线性更多**;同时把时空解耦成"先看空间、再看时间",优化更容易。这是"时空可分离"思想的代表,和我们下一节要讲的"分解时空注意力"在动机上一模一样:**全 3D/全时空联合太贵,拆成空间×时间两步既省又往往更好。**

```python
class R2Plus1DBlock(nn.Module):
    """(2+1)D 卷积:空间 (1,K,K) -> 激活 -> 时间 (k_t,1,1)。"""
    def __init__(self, c_in, c_out, k_t=3, K=3, mid=None):
        super().__init__()
        # 取中间通道使参数量≈对应3D核(论文做法,这里给一个简化估计)
        mid = mid or (c_in * c_out * 3 * K * K) // (c_in * K * K + 3 * c_out)
        self.spatial  = nn.Conv3d(c_in, mid, (1, K, K), padding=(0, K//2, K//2), bias=False)
        self.bn1, self.act = nn.BatchNorm3d(mid), nn.ReLU(inplace=True)
        self.temporal = nn.Conv3d(mid, c_out, (k_t, 1, 1), padding=(k_t//2, 0, 0), bias=False)
        self.bn2 = nn.BatchNorm3d(c_out)
    def forward(self, x):
        x = self.act(self.bn1(self.spatial(x)))
        return self.act(self.bn2(self.temporal(x)))
```

## 3. 双流网络:把外观和运动分开喂

### 3.1 动机与机制

双流网络(Two-Stream,Simonyan & Zisserman,2014)直面差异二:**外观和运动是两类互补信号,那就用两个网络分别处理**。

- **空间流(spatial stream)**:输入是**单帧 RGB 图像**,就是一个普通的 2D CNN 分类网络。它学的是"画面里有什么"——外观。一帧就能判断"这是个篮球场",这类静态线索对很多动作类别已经很有信息量。
- **时间流(temporal stream)**:输入是**堆叠的光流场**。把相邻帧的光流(每点一个二维位移 (Δx, Δy))在时间上堆 L 帧,得到一个 `2L × H × W` 的输入(2 是 x/y 两个分量),也喂给一个 2D CNN。它学的是"东西在怎么动"——运动。

两流各自输出类别分数,最后**融合**(最简单是分数平均,或在某中间层做特征级融合)。

**为什么显式光流有用?** 这是双流最深刻的设计点:光流把"运动"这个本来藏在 RGB 帧间差里的信号,**预先、显式地解算出来**,网络不必再从原始像素里费力地学"如何感知运动"。等价于给网络喂了一个强力的、手工设计的运动特征。代价是:**光流计算本身昂贵**(传统 TV-L1 光流是迭代优化,慢;且需离线预算并存盘),这也是双流路线后来逐渐被"端到端学运动"的 3D 卷积/注意力取代的主因之一。

### 3.2 光流是什么:亮度恒定假设与光流方程

光流的根本假设是**亮度恒定(brightness constancy)**:同一个物理点在相邻帧亮度不变。设图像亮度 I(x, y, t),点 (x,y) 在 dt 内移动 (dx, dy):
```
I(x, y, t) = I(x + dx, y + dy, t + dt)
一阶泰勒展开右边:
I(x+dx, y+dy, t+dt) ≈ I(x,y,t) + Iₓ·dx + I_y·dy + I_t·dt
两边相减 ⇒  Iₓ·dx + I_y·dy + I_t·dt = 0
两边除以 dt,记 (u,v)=(dx/dt, dy/dt) 为光流:
        Iₓ·u + I_y·v + I_t = 0          ← 光流约束方程(一个方程,两个未知数)
```
**一个方程解不出两个未知数 u、v——这就是著名的"孔径问题(aperture problem)":只看局部,你只能确定运动在梯度方向上的分量,无法确定沿边缘切向的分量。** 经典解法 Lucas-Kanade 假设"局部邻域内光流相同",凑出多个方程做最小二乘;Horn-Schunck 加全局平滑正则。这些细节属于经典视觉,这里点到为止——**关键 take-away:光流是个欠定的、需要额外假设/正则才能解的逆问题,所以才贵、才有噪声。** 现代做法(FlowNet、RAFT)直接用 CNN 端到端学光流,快且准。

### 3.3 复杂度对比与定位

双流的算力 ≈ 两个 2D CNN(空间流处理少量帧、时间流处理 L 帧光流),**比 3D 卷积省**(没有那个 k_t 乘数),但**总成本要算上光流预计算**——这部分常被忽略,是双流"看着便宜其实不便宜"的隐藏账。建模能力上,双流的时序窗口受限于堆叠的光流帧数 L,且两流融合得晚,**长程时序建模和时空联合建模都不如 3D / 注意力**。它的历史地位是:**第一个明确把"运动"作为独立信号工程化的框架**,这个"外观/运动分治"的洞见至今影响深远。

## 4. 时空注意力:把 ViT 扩到视频

### 4.1 问题:联合时空注意力为什么不可行

最直接的把 ViT 扩到视频:把视频切成时空 patch(或叫 tubelet,小立方块),拉平成 N = T·N_s 个 token(N_s 是每帧 patch 数),做标准全局自注意力。这叫**联合时空注意力(joint space-time attention)**。问题在 §1 已经算过:复杂度是 O((T·N_s)²)。

定量(每层自注意力的 token-pair 数,即注意力矩阵大小):
```
联合:  (T·N_s)² = T² · N_s²
单帧:  N_s²
比值:  T²          # T=16 时,注意力计算量是单帧的 256 倍
具体:  T=16, N_s=196 → (16×196)² = 3136² ≈ 9.83×10⁶ token-pair / 层 / 头
```
**T² 的乘数让联合时空注意力对稍长的视频直接不可行。** TimeSformer 和 ViViT 的核心贡献,就是**把这个联合注意力分解(factorize)成"空间注意力 + 时间注意力"两步,把 T²·N_s² 降到 T·N_s² + T²·N_s。**

### 4.2 分解时空注意力:机制与复杂度

**分解(divided / factorized)时空注意力**(TimeSformer,Bertasius 等人,2021)把每个 Transformer block 里的一次注意力拆成串联的两次:

- **时间注意力(temporal attention)**:对每个空间位置,只在它**跨所有帧的 T 个 token**之间做注意力(同一个 patch 位置在不同时刻互相看)。复杂度 O(T²) per 空间位置,共 N_s 个位置 → **N_s · T²**。
- **空间注意力(spatial attention)**:对每一帧,只在它**帧内的 N_s 个 token**之间做注意力(标准 ViT 单帧注意力)。复杂度 O(N_s²) per 帧,共 T 帧 → **T · N_s²**。

复杂度对比:
```
联合时空:     O(T² · N_s²)
分解时空:     O(N_s · T²  +  T · N_s²)  =  O(T·N_s·(T + N_s))
T=16, N_s=196:
  联合:  T²·N_s² = 256 × 38416 ≈ 9.83×10⁶
  分解:  N_s·T² + T·N_s² = 196×256 + 16×38416 = 50,176 + 614,656 ≈ 6.65×10⁵
  加速:  约 14.8×            # 且 T 越大,优势越夸张(联合 ∝T²,分解 ∝T)
```
**这就是分解的全部价值:用两次"瘦"注意力近似一次"胖"注意力,把乘积 T²·N_s² 降为 T·N_s·(T+N_s)——提出公共因子 T·N_s 后,括号内由乘积 N_s 退化为和 (T+N_s);复杂度从对 T 和 N_s 都二次的乘积,降到(就 T 而言的)二次与一次的和。** 代价是**表达力的损失**:联合注意力里"第 3 帧左上角的 patch"可以直接注意到"第 10 帧右下角的 patch";分解后,这种"时空对角"的依赖必须**先经时间注意力传到同位置、再经空间注意力传到异位置**,要两层才能间接建模。TimeSformer 论文的消融显示,分解时空在精度上**几乎不输甚至略优于**联合(同时省一个数量级算力)——因为视频里真正重要的长程依赖,大多确实可以分解成"先时间对齐、再空间关联"。

ViViT(Arnab 等人,2021)系统化了这一族,给出四种分解变体(从联合、到分解 encoder、到分解 self-attention、到分解 dot-product),并提出 **tubelet embedding**——不是逐帧切 patch,而是直接切 `t×P×P` 的时空小立方块做 embedding,**在输入端就融合了一点时序**,比逐帧切更高效。

### 4.3 代码:分解时空注意力 block

```python
import torch
import torch.nn as nn

class DividedSpaceTimeBlock(nn.Module):
    """TimeSformer 式分解时空注意力 block。
       输入 token 组织为 (B, T, N_s, D):T 帧,每帧 N_s 个空间 patch,D 维。
       一次 block = 时间注意力(跨帧同位置) -> 空间注意力(帧内) -> MLP,各带残差。"""
    def __init__(self, dim, n_heads, mlp_ratio=4.0):
        super().__init__()
        self.t_norm = nn.LayerNorm(dim)
        self.t_attn = nn.MultiheadAttention(dim, n_heads, batch_first=True)
        self.s_norm = nn.LayerNorm(dim)
        self.s_attn = nn.MultiheadAttention(dim, n_heads, batch_first=True)
        self.m_norm = nn.LayerNorm(dim)
        hidden = int(dim * mlp_ratio)
        self.mlp = nn.Sequential(nn.Linear(dim, hidden), nn.GELU(), nn.Linear(hidden, dim))

    def forward(self, x):                      # x: (B, T, N_s, D)
        B, T, N, D = x.shape

        # ---- 时间注意力:把"帧"当序列,对每个空间位置独立做 T 之间的注意力 ----
        # reshape 使空间位置进 batch 维、时间 T 进序列维
        xt = x.permute(0, 2, 1, 3).reshape(B * N, T, D)   # (B*N_s, T, D)
        h  = self.t_norm(xt)
        xt = xt + self.t_attn(h, h, h, need_weights=False)[0]   # 复杂度 ∝ T²
        x  = xt.reshape(B, N, T, D).permute(0, 2, 1, 3)         # 回到 (B, T, N_s, D)

        # ---- 空间注意力:把"patch"当序列,对每一帧独立做 N_s 之间的注意力 ----
        xs = x.reshape(B * T, N, D)                        # (B*T, N_s, D)
        h  = self.s_norm(xs)
        xs = xs + self.s_attn(h, h, h, need_weights=False)[0]   # 复杂度 ∝ N_s²
        x  = xs.reshape(B, T, N, D)

        # ---- 逐 token MLP ----
        x = x + self.mlp(self.m_norm(x))
        return x
```

注意代码里两次注意力的关键差别**全在 reshape**:时间注意力把 N_s 折进 batch、让 T 当序列;空间注意力把 T 折进 batch、让 N_s 当序列。**这是分解时空注意力的整个工程实现精髓——没有新算子,只是用 reshape 切换"谁是序列、谁是批"。** 这也提示了一个实现坑:reshape/permute 频繁会带来内存非连续(需要 `.contiguous()`)和额外拷贝开销,大规模训练时要留意。

### 4.4 一个常被忽略的点:位置编码要补时间

ViT 的位置编码补的是"二维空间里 patch 的位置"。视频里 token 多了时间下标,**必须额外告诉模型每个 token 在哪一帧**。常见做法是**分解的位置编码**:空间位置编码(N_s 个)+ 时间位置编码(T 个)相加。漏掉时间位置编码,模型分不清帧序,时序建模直接失效——这是把 ViT 改成视频模型时最容易漏的一步。

## 5. 视频理解的任务与评测:把"目标"定义清楚

模型再好,得有任务和指标来锚定。视频理解两大基础任务:

**动作识别(action recognition)**:给一段已剪辑好的短视频(trimmed,通常几秒、只含一个动作),分类出动作类别。本质是"视频级分类",指标就是 **Top-1 / Top-5 准确率**。代表数据集:Kinetics(大规模、约几百类、几十万段,「待核:Kinetics-400/600/700 的确切规模随版本不同」)、Something-Something(强调时序——很多类别如"把 X 推到 Y 左边"必须看运动方向才能分对,是检验模型真在用时序而非外观蒙的关键基准)、UCF-101 / HMDB-51(早期小数据集)。

**时序动作定位(temporal action localization)**:给一段**未剪辑的长视频**(untrimmed,含大量背景/无关片段),不仅要分类,还要**预测每个动作在时间轴上的起止区间** [t_start, t_end]。这是"时间轴上的目标检测",和 04/05 章空间检测同构:把"空间边界框"换成"时间区间",把空间 IoU 换成**时间 IoU(temporal IoU)**:
```
tIoU = |[s₁,e₁] ∩ [s₂,e₂]| / |[s₁,e₁] ∪ [s₂,e₂]|     # 两个时间区间的交并比
```
指标也对齐检测:**mAP@tIoU**(在某 tIoU 阈值下的平均精度,如 mAP@0.5),通常报多个阈值的平均。代表数据集:THUMOS、ActivityNet。

**评测设计的坑**:动作识别推理时常用 **multi-crop / multi-clip 测试**——从长视频里采多个时间片段、每片段多个空间裁剪,分数平均。报告精度时若不说清测试时采了几个 view,数字不可比(单 view 和 30 view 能差好几个点)。读论文务必看清测试协议。

## 6. 视频生成:在扩散上接时间,核心是时序一致性

### 6.1 第一性的难题:为什么逐帧独立生成必然闪烁

把 11 章的图像扩散直接拿来生成视频,最朴素做法:**逐帧独立地跑一遍图像扩散**,生成 T 张图拼成视频。结果必然是**严重闪烁(flickering)、内容漂移**——同一个物体在相邻帧颜色、纹理、位置乱跳。

这不是工程 bug,是**数学上的必然**,值得讲透。图像扩散建模的是单帧分布 p(x)。逐帧独立生成,等价于假设视频的联合分布可分解为各帧边缘的乘积:
```
p(x₁, x₂, ..., x_T) = Π_t p(xₜ)          ← 逐帧独立的隐含假设(错误!)
```
而真实视频里相邻帧**极强相关**,真实联合分布根本不是边缘的乘积。更要命的是采样的随机性:扩散采样从独立的高斯噪声 x_T ~ N(0,I) 起步,**每一帧用的是独立采样的初始噪声和独立的去噪随机性**。即便条件(文本 prompt)相同,两次独立采样会落到数据流形上**相距很远的两个合理样本**——都"像猫",但是不同的猫、不同的姿态。逐帧独立 = 让每一帧各自随机落到流形某处,**帧间没有任何约束把它们拉到"同一个物体的连续运动"这条细线上**。这就是闪烁的根:
```
逐帧独立:每帧 ε_t 独立采样 → 每帧落到 p(x) 流形上随机一点 → 帧间无相关 → 闪烁
我们要的:p(x₁:T) 上的一条"时间连贯"的轨迹 → 必须显式建模帧间依赖
```
**结论:视频生成的核心,就是把对帧间相关性的建模显式地加进扩散模型。** 一切视频扩散技术都是为这一件事服务。

### 6.2 主流解法:在图像扩散里插入时间层

最成功也最主流的范式(Video Diffusion Models、Make-A-Video、AnimateDiff、以及 Sora 类时空 latent 扩散的思想源头):**拿一个预训练好的图像扩散 U-Net(或 DiT),在它原有的空间层之间插入"时间层(temporal layer)",只让这些时间层负责跨帧建模,空间层尽量复用图像先验。** 这与 §2.3 的 I3D 膨胀是同一思想的回响:**充分利用海量图像数据训出的图像模型,只额外学"时间"这一新维度。**

具体地,把图像 U-Net 的特征图从 `(B, C, H, W)` 扩成视频的 `(B, C, T, H, W)`:

- **空间层(原有的 2D 卷积、空间自注意力、交叉注意力)**:把 T 折进 batch 维 `(B·T, C, H, W)`,**逐帧独立**地跑——直接复用图像模型权重,不需重学。
- **时间层(新插入)**:把空间位置折进 batch 维,只在 T 维上做**时间注意力**或 **1D 时间卷积**,建模帧间依赖。这正是 §4.2 分解时空注意力的"时间注意力"那一半,搬到了生成里。

```python
import torch
import torch.nn as nn

class TemporalAttention(nn.Module):
    """插进图像扩散 U-Net 的时间注意力层:只在 T 维做注意力,空间位置当 batch。
       输入特征 (B, C, T, H, W) —— 视频化后的 U-Net 特征图。"""
    def __init__(self, dim, n_heads):
        super().__init__()
        self.norm = nn.LayerNorm(dim)
        self.attn = nn.MultiheadAttention(dim, n_heads, batch_first=True)
        # 时间位置编码:告诉模型每个 token 属于第几帧(漏了就没法保证时序连贯)
        self.t_pos = nn.Parameter(torch.zeros(1, 64, dim))   # 支持最多64帧,按需扩
    def forward(self, x):                       # x: (B, C, T, H, W)
        B, C, T, H, W = x.shape
        # 把空间位置 (H,W) 折进 batch,时间 T 当序列
        h0 = x.permute(0, 3, 4, 2, 1).reshape(B * H * W, T, C)  # (B*H*W, T, C)
        h  = self.norm(h0) + self.t_pos[:, :T]                  # 归一化+位置编码只喂给注意力
        h0 = h0 + self.attn(h, h, h, need_weights=False)[0]     # 残差加回未归一化的 h0,复杂度 ∝ T²
        return h0.reshape(B, H, W, T, C).permute(0, 4, 3, 1, 2) # 回 (B,C,T,H,W)
```

**为什么这招治得了闪烁**:时间注意力让每个空间位置的 T 帧特征**互相可见、互相约束**,去噪时第 t 帧的预测会参考其余帧,迫使整段序列收敛到一条连贯轨迹,而非各帧自由落体。**关键工程做法**:训练时常**冻结(freeze)空间层、只训时间层**,既省算力又防止破坏图像先验——这是 AnimateDiff 等"给任意图像模型加运动"方法的核心。

### 6.3 时空一致性的其他抓手与代价

- **跨帧共享噪声 / 噪声相关性**:让相邻帧的初始噪声 x_T 不完全独立(部分共享或加相关),从源头减少帧间随机漂移。代价是多样性下降。
- **3D 卷积/时空 latent**:Sora 类思路是先用一个**时空自编码器**把视频压成时空 latent(同时在 T、H、W 上下采样),再在这个紧凑 latent 上做扩散(对应 11 章 Latent Diffusion 的视频版),**省下几十倍算力**——这是长视频生成可行的关键。
- **代价与开放问题**(诚实地讲):①**算力/显存**——视频 latent 仍带 T 乘数,长视频训练和采样极贵;②**长程一致性**——几秒尚可,几十秒后物体身份、场景几何容易漂(物体凭空出现/消失);③**物理合理性**——生成的运动常违反物理(水流、碰撞、刚体),因为模型学的是像素统计而非物理定律,这正是世界模型(§7)想补的;④**可控性与评测**——视频生成质量缺乏像 FID 那样公认稳的单一指标(FVD 是常用的视频版,「待核:FVD 的具体实现/特征提取器细节」),时序一致性、运动自然度难量化。

## 7. 具身与世界模型:视觉作为"感知 + 行动"的接口

视频建模的终点不止于"看懂"和"生成好看",而是**让视觉成为智能体在世界里感知并行动的接口**。这把视觉课和强化学习课接了起来。

**世界模型(world model)**:学一个能**预测未来**的模型——给定当前观测(帧)和将采取的动作 a,预测下一帧(或下一个 latent 状态)。形式上就是学一个条件分布:
```
p(x_{t+1} | x_{≤t}, aₜ)        # 给定历史观测和动作,预测下一观测
```
这和视频生成的差别**只在一个条件 aₜ**:无条件视频生成预测"接下来会发生什么",世界模型预测"我做了动作 a 之后会发生什么"——**可交互**。一旦有了这个模型,智能体就能在"脑子里"(latent 空间)推演不同动作的后果,做规划,而不必在真实世界里试错——**这是 model-based RL 的核心,呼应强化学习课:世界模型 = 学到的环境模拟器,在它内部跑 planning/policy 学习极大提升样本效率。** Dreamer 系列是把"在 latent 世界模型里学策略"做通的代表。

**VLA(Vision-Language-Action,视觉-语言-动作模型)**:把"看(视觉)+ 听懂指令(语言)+ 干活(动作)"统一进一个模型,直接从图像观测和语言指令输出机器人动作。它把 09 章的视觉-语言对齐(CLIP 那套把图像和文本拉进同一语义空间)再往前推一步——**输出端从'理解'扩到'控制'**:不只是说出"画面里有个杯子",而是输出"把手移到杯子上方、合拢夹爪"的动作序列。RT 系列、OpenVLA 等是代表。VLA 的训练数据是"(观测, 指令, 动作)"三元组(机器人遥操作演示),本质是把动作当成另一种"待预测的 token 序列",用类似语言模型/序列建模的方式自回归地生成动作。

**为什么这是视频建模的自然归宿**:动作发生在时间里,理解动作的后果需要预测时间上的演化——**时序建模(本章主线)正是世界模型和 VLA 的地基**。你在本章学的时空注意力、时间一致性、把图像模型扩到视频的思想,几乎原样迁移到世界模型(预测未来帧)和 VLA(基于视频流的策略)。视觉至此从"被动地看"变成"主动地在世界里行动的眼睛"。

## 8. 设计权衡与常见坑

把三条理解路线和生成的关键权衡收成一张账,这是本章最该带走的判断力:

| 路线 | 时序建模方式 | 相对算力 | 长程时序 | 关键代价/坑 |
|---|---|---|---|---|
| **3D 卷积 (C3D/I3D)** | 局部时空核,隐式 | 高(×k_t) | 弱(感受野线性) | 参数多易过拟合 → 靠膨胀(§2.3);时间感受野近视 |
| **(2+1)D** | 空间卷积+时间卷积 | 中 | 中 | 同参数下表达力优于3D,但仍是局部 |
| **双流 (Two-Stream)** | 光流显式编码运动 | 中(+光流预算) | 弱(受堆叠帧数限) | 光流离线计算贵、有噪;融合晚;隐藏成本常被低估 |
| **时空注意力 (TimeSformer/ViViT)** | 全局,分解时空 | 中-高(分解后) | 强(全局) | 漏时间位置编码即失效;reshape 内存开销;小数据吃亏(ViT 的数据饥渴遗传到视频) |

**通用坑(贯穿理解与生成)**:

- **算力是第一约束,T 是乘数。** 任何"看着合理"的暴力时空操作,先把 T(或 T²)的乘数算进去再决定可行性。这是视频区别于图像的第一性约束。
- **必须验证模型真在用时序,而非靠单帧外观蒙。** 很多动作类别单帧外观就能猜对(篮球场→打篮球),模型可能根本没学时序。**用 Something-Something 这类"时序敏感"基准、或做"打乱帧序看掉多少分"的消融**,才能证明时序建模真起作用——这是视频研究最常见的自欺。
- **时间位置编码 / 时间维归一化别漏。** 注意力模型漏时间位置编码、卷积模型用 BatchNorm3d 在小 batch 上统计不稳,都是隐蔽的性能杀手。
- **生成端别用图像指标骗自己。** 单帧 FID 高不代表视频好——它对闪烁完全不敏感。必须看时序一致性相关的指标(FVD 等)和肉眼看连贯性。
- **数据稀缺 → 优先迁移图像先验。** 视频标注昂贵稀少,从头训大视频模型几乎总是亏的。膨胀(I3D)、插时间层冻空间层(视频扩散)都是同一招:**让图像模型出力,只额外学时间。**

## 9. 动手练习

1. **(分析 + 计算题)三条路线的算力与建模能力对账。** 取输入 16 帧、224×224、patch 16×16(N_s=196)。(a) 算一层 3D 卷积(Cᵢₙ=C_out=256, k_t=3, K=3)的参数量和 FLOPs,与同配置逐帧 2D 卷积比,验证差 k_t 倍;(b) 算联合时空注意力 vs 分解时空注意力每层每头的 token-pair 数,验证 §4.2 的约 14.8× 加速;(c) 用一句话各自总结三条路线"用什么换什么"。*提示*:FLOPs 用 §2.2 公式;注意力 token-pair 数就是注意力矩阵元素数;(c) 答案形如"3D 卷积用算力换端到端学时序,双流用光流预计算换运动信号,注意力用分解近似换全局建模"。

2. **(推导题)推导 3D 卷积的时间感受野,并设计达到 64 帧时间感受野的最省方案。** (a) 证明 L 层时间核 k_t、时间步长 1 的堆叠时间感受野是 `1+L(k_t−1)`;(b) 若允许在某些层用时间步长 s>1(时间下采样),重推感受野递推式;(c) 要达到 RF_t ≥ 64,对比"纯堆 k_t=3 层"和"穿插时间步长"两种方案的层数/参数量。*提示*:(b) 感受野递推 `RF_new = RF_old + (k_t−1)×(累计时间步长乘积)`,步长会让后续层的感受野按累计步长放大,和 01 章空间感受野递推同构。

3. **(编码题)实证"逐帧独立必闪烁 vs 时间注意力治闪烁"。** 用一个**2D toy**:把"视频"定义为一个小球在 H×W 网格上做匀速直线运动的 T 帧序列(每帧是小球位置的高斯亮点)。(a) 训一个无条件图像扩散(复用 11 章 toy)逐帧独立采样 T 帧,度量帧间小球位置的抖动(相邻帧位置差的方差);(b) 给 U-Net 插入 §6.2 的 TemporalAttention,在"小球轨迹"数据上训练,再采样,度量同一抖动指标。*提示*:不必真实视频,2D 小球轨迹就能让"闪烁(位置乱跳)vs 连贯(匀速直线)"肉眼可见、可量化;这是 §6.1 论断的最小验证实验,和 11 章"2D 八高斯验扩散覆盖"是同一套方法论。

4. **(分析题)解释分解时空注意力丢了什么,何时会出问题。** 论证:(a) 为什么联合时空注意力能一层建模"第3帧某处 ↔ 第10帧另一处"的依赖,而分解需要两层才能间接建模;(b) 举一类**分解会明显吃亏**的视频场景(提示:运动 + 空间位置同时剧变、需要"时空对角"长程关联的情形,如快速移动物体的跨帧跨位置关联),并说明为什么 TimeSformer 在多数动作识别基准上仍几乎不输联合(提示:从"真实视频长程依赖大多可分解为先时间对齐再空间关联"论证)。

## 10. 源码 / 论文导读

- **C3D(3D 卷积起点)**:Tran et al.《Learning Spatiotemporal Features with 3D Convolutional Networks》(2015)——看它确立 3×3×3 时空核的实验论证。
- **I3D(膨胀,必读)**:Carreira & Zisserman《Quo Vadis, Action Recognition?》(2017)——重点读"Inflating 2D ConvNets into 3D"那节,以及为什么除以时间维长度做初始化(对应本章 §2.3 的验算);它同时发布了 Kinetics。
- **R(2+1)D(时空可分离)**:Tran et al.《A Closer Look at Spatiotemporal Convolutions for Action Recognition》(2018)——看 (2+1)D 与 3D 的受控对比,理解"同参数下分解为何更强"(对应 §2.5)。
- **双流(运动流的源头)**:Simonyan & Zisserman《Two-Stream Convolutional Networks for Action Recognition in Videos》(2014)——奠基"外观/运动分治";光流的现代学法看 RAFT(Teed & Deng, 2020)。
- **TimeSformer(分解时空注意力,必读)**:Bertasius et al.《Is Space-Time Attention All You Need for Video Understanding?》(2021)——重点读它对五种注意力方案(含联合、分解)的复杂度与精度消融,直接对应本章 §4.2;**Divided Space-Time** 那一版就是 §4.3 代码所实现的。
- **ViViT(系统化时空 ViT)**:Arnab et al.《ViViT: A Video Vision Transformer》(2021)——读四种分解变体和 tubelet embedding(§4.2)。
- **视频扩散(生成主线)**:Ho et al.《Video Diffusion Models》(2022)给"factorized space-time U-Net + 时间注意力"范式;Singer et al.《Make-A-Video》(2022)、Guo et al.《AnimateDiff》(2023,**冻空间训时间**的代表,对应 §6.2)是把图像扩散扩成视频扩散的关键读物。
- **世界模型 / VLA(承接 RL)**:Ha & Schmidhuber《World Models》(2018)是概念奠基;Hafner et al. 的 **Dreamer** 系列把"latent 世界模型 + 内部 planning"做通;VLA 看 RT-2 / OpenVLA(把动作当 token 自回归,对应 §7)。
- **开源实现**:
  - **torchvision** `torchvision.models.video`:`r3d_18`、`mc3_18`、`r2plus1d_18`、`s3d` 等 3D/2+1D 网络的标准实现,是照着读 3D 卷积网络结构的最佳入口(对应 §2)。
  - **PyTorchVideo**(Meta):视频理解的工业级库,含 SlowFast、I3D、X3D、MViT 及数据加载/multi-clip 测试协议(对应 §5 的评测坑),是做视频理解最该用的脚手架。
  - **`diffusers`**:`models/unets/unet_3d_condition.py`、`unet_spatio_temporal_condition.py` 是视频扩散 U-Net(空间层 + 时间层)的干净实现,对照 §6.2 的 TemporalAttention 逐行看插时间层怎么做;`AnimateDiff` pipeline 看冻空间训时间的接法。
  - **timm**:虽以图像为主,但视频 ViT 的 patch/attention 积木与其一致,可对照 §4 实现。

## 11. 小结与承上启下

这一章我们只做了一件事:**给视觉模型加一根时间轴 t,并反复回答"多了 T 这个乘数,怎么既不丢时序、又不被算力压垮"**。把整章逻辑收成一句话:**时间维带来运动这一新信号(机会)和 T/T² 这个乘数(代价);理解侧用三条路线抽时序——3D 卷积(局部时空核,靠膨胀借图像先验)、双流(显式光流分治外观与运动)、时空注意力(全局,靠分解把 T²·N_s² 降成 T²+N_s²);生成侧的命门是时序一致性,逐帧独立因建模了错误的"边缘乘积"分布而必然闪烁,治法是往图像扩散里插时间层显式建帧间依赖(冻空间训时间,呼应膨胀);终点是世界模型/VLA,把视觉从'看'扩成'预测后果并行动'的接口,接上强化学习。**

四个判断你应当带走:

- **T 是第一性约束**:视频的一切设计压力都来自"时间是乘数",分解(可分离卷积、分解注意力、空间层/时间层解耦)是反复出现的破局母题;
- **运动 vs 外观是互补信号**:从双流的显式光流到 3D/注意力的隐式学习,如何抽运动是理解侧的核心分歧;
- **时序一致性是生成侧的命门**:它不是工程瑕疵而是"逐帧独立 = 错误独立性假设"的数学必然,显式建帧间依赖是唯一解;
- **迁移图像先验是穿透全章的工程铁律**:I3D 膨胀和视频扩散"冻空间训时间"是同一招,根因是视频数据相对图像永远稀缺。

放到整门课的坐标里:01-07 章你建好了**空间**的全部武器(卷积、感受野、ResNet、检测、分割、ViT),09 章给了**视觉-语言对齐**,10-11 章给了**生成**(GAN、扩散)。本章把这些武器**沿时间轴展开一遍**——你会清楚地看到,视频里没有全新的原理,全是"空间那套 + 一根时间轴 + 如何省 T"的再组合。而本章末尾的世界模型/VLA,正是通向**强化学习课**和**具身智能**的桥:当视觉不再只是输入、而成为智能体感知与行动的闭环接口时,"看"和"做"就合一了。下一步,无论你走向多模态大模型(把视频 token 喂进 LLM)还是具身智能(VLA + 真实机器人),本章这根"时间轴"都是你绕不开的地基。
