> 这一章把你从"知道 Transformer 用了注意力"带到"能在白板上从信息瓶颈推出 `softmax(QKᵀ/√dₖ)V` 的每一个符号、能徒手实现单头注意力、能算清它的 FLOPs 和显存"。注意力是 Transformer 的心脏,这章要求你对它没有任何黑箱。

## 一句话引言

注意力本质是一个**可微的、内容寻址的软查表**:用 query 去和每个 key 算相似度,把相似度归一化成权重,再对 value 加权求和。本章从 seq2seq 的信息瓶颈讲起,推到加性/乘性注意力,最后把 scaled dot-product attention 的公式、缩放因子 √dₖ 的来历、softmax 的雅可比、以及 O(n²) 的复杂度根源全部推透。

## 1. 动机:固定向量是个信息瓶颈

### 1.1 seq2seq 的结构性缺陷

考虑机器翻译。经典 seq2seq(Sutskever et al., 2014;Cho et al., 2014)的做法是:

```
encoder:  x₁ x₂ ... xₙ  ──RNN──▶  c  (最后一个隐藏状态,固定维度 d)
decoder:  c  ──RNN──▶  y₁ y₂ ... yₘ
```

整个源句子 `x₁...xₙ` 被压缩成**一个固定长度的向量 c**,解码器只能看着这一个 c 生成全部目标词。这就是**信息瓶颈(information bottleneck)**:

- **容量不随输入增长**。c 的维度是固定的 d(比如 512)。源句子是 5 个词还是 50 个词,都得塞进同一个 512 维向量。句子越长,平均每个词能分到的"带宽"越少。
- **远距离信息衰减**。RNN 顺序更新隐藏状态,`xₙ` 离 c 近、`x₁` 离 c 远。即使用 LSTM/GRU 缓解梯度消失,早期 token 的信息在抵达 c 时也已被反复覆写。Sutskever 等人当年甚至把输入句子**反转**输入来缓解这个问题——这本身就是瓶颈存在的强证据。
- **解码每一步的需求不同**。生成目标第 1 个词时,你最想看源句子开头;生成第 5 个词时,你想看的是源句子对应的那一段。但固定的 c 对每一步解码提供的信息是**完全一样的**,无法"按需"提供。

**核心矛盾**:翻译是一个 token 级别的对齐问题(目标词 ↔ 源词的局部对应),却被强行用一个全局固定向量来承载。

### 1.2 解决思路:不要压缩,要检索

Bahdanau, Cho & Bengio(2015,ICLR)提出的破局点很朴素:**别再把源句子压成一个向量了**。把编码器每个位置的隐藏状态 `h₁, h₂, ..., hₙ` 全部保留下来;解码到第 t 步时,根据当前解码状态 `sₜ`,动态地决定"这一步该重点看源句子的哪些位置",算出一个**针对这一步定制的上下文向量 cₜ**。

```
cₜ = Σⱼ αₜⱼ · hⱼ ,   其中 Σⱼ αₜⱼ = 1, αₜⱼ ≥ 0
```

`αₜⱼ` 是第 t 步解码对源位置 j 的**注意力权重**——它衡量"生成第 t 个目标词时,源位置 j 有多重要"。这组权重不是固定的,而是**算出来的、可微的、随每一步变化的**。

这一步在概念上做了三件事,后面所有注意力变体都没逃出这三件事:

1. **打分(score)**:对每个源位置 j,算一个相关性分数 `eₜⱼ = score(sₜ, hⱼ)`。
2. **归一化(normalize)**:`αₜⱼ = softmax_j(eₜⱼ)`,把分数变成和为 1 的权重(软对齐,soft alignment)。
3. **聚合(aggregate)**:`cₜ = Σⱼ αₜⱼ hⱼ`,按权重把源信息加权求和。

**为什么瓶颈被打破了**:上下文向量 cₜ 现在直接访问全部 n 个源状态,信息通路从 `xⱼ → c → yₜ`(经过 n-j 步 RNN 压缩)变成 `hⱼ → cₜ`(一步加权)。任意源位置到任意解码步的路径长度变成 O(1),且容量随 n 线性增长。

> 直觉:把 seq2seq 的"先读完整本书再凭记忆复述"换成"边复述边随时翻回原书查相关页"。

## 2. 从加性到乘性:打分函数怎么选

注意力机制的全部"花样"集中在第 1 步的打分函数 `score(s, h)` 上。两个历史上最重要的选择:

### 2.1 加性注意力(Bahdanau, additive / concat)

```
eₜⱼ = vᵀ · tanh(W_s sₜ + W_h hⱼ)
```

- `sₜ ∈ ℝ^{dₛ}`(解码状态),`hⱼ ∈ ℝ^{dₕ}`(编码状态),可以维度不同。
- `W_s ∈ ℝ^{dₐ×dₛ}`、`W_h ∈ ℝ^{dₐ×dₕ}` 把两者投影到同一个 dₐ 维空间相加,过 tanh 非线性,再用向量 `v ∈ ℝ^{dₐ}` 投到标量。
- 它本质是一个**单隐层小 MLP** 在给 (s, h) 对打分。表达力强(有非线性),但每个 (t, j) 对都要过一遍这个 MLP。

### 2.2 乘性 / 点积注意力(Luong, multiplicative / dot)

Luong, Pham & Manning(2015,EMNLP)提出更简洁的几种打分,核心是**点积**:

```
dot:      eₜⱼ = sₜᵀ hⱼ                (要求 dₛ = dₕ)
general:  eₜⱼ = sₜᵀ W hⱼ              (用 W 对齐维度并引入可学习交互)
```

点积的物理含义:**两个向量越"指向同一方向",点积越大,相关性越高**。它没有非线性,但有一个加性注意力没有的巨大优势——**可以写成大矩阵乘法,在 GPU 上极快**。

### 2.3 两者的权衡(为什么后来点积赢了)

| 维度 | 加性(Bahdanau) | 乘性/点积(Luong) |
|---|---|---|
| 计算形式 | 逐对过 MLP,难批成单个 matmul | 整体一个 matmul `SHᵀ` |
| 非线性 | 有(tanh) | 无 |
| 参数量 | W_s, W_h, v | 0(dot)或一个 W(general) |
| 大 dₖ 下数值 | 相对稳健 | 点积方差随 dₖ 膨胀(见 §3.3) |
| 硬件友好 | 差 | 极好(GEMM) |

Vaswani et al.(2017)在《Attention Is All You Need》里明确指出:dₖ 较小时两者表现接近;dₖ 较大时**未缩放的点积会显著变差**,他们怀疑是点积量级变大把 softmax 推入饱和区——于是给点积加了个 `1/√dₖ` 的缩放,得到 **scaled dot-product attention**。点积因为能吃满矩阵乘法硬件,成了 Transformer 的默认选择。下一节把这个公式彻底推开。

## 3. Scaled Dot-Product Attention 完整推导

### 3.1 从"软对齐"到 Q/K/V 抽象

Transformer 把 §1.2 的三步骤抽象成三个角色,并彻底参数化。给定一组待编码的 token 表示,我们为每个 token 生成三个向量:

- **Query(查询 q)**:"我想找什么样的信息?"——主动方,拿着它去匹配。
- **Key(键 k)**:"我这里有什么样的信息?"——被检索方的索引/标签。
- **Value(值 v)**:"如果你匹配上我,你实际取走的内容是什么?"——真正被聚合的载荷。

把 §1.2 的对应关系翻译过来:`sₜ → query`,`hⱼ → key`(用于打分)同时 `hⱼ → value`(用于聚合)。Transformer 的关键一步是**把 key 和 value 解耦**:打分用的表示和被取走的内容不必相同,各自由独立的线性投影生成。

形式化。设有 n 个 query(组成矩阵 `Q ∈ ℝ^{n×dₖ}`)、m 个 key/value 对(`K ∈ ℝ^{m×dₖ}`,`V ∈ ℝ^{m×dᵥ}`)。注意 query 和 key 必须同维 dₖ(才能点积),value 维度 dᵥ 可以不同。

**单个 query 的情形**先讲清楚。取第 i 个 query `qᵢ ∈ ℝ^{dₖ}`:

```
第1步 打分:   对每个 key j,  eᵢⱼ = qᵢ · kⱼ = qᵢᵀ kⱼ        (标量, j=1..m)
第2步 缩放:   eᵢⱼ ← eᵢⱼ / √dₖ
第3步 归一化: αᵢ = softmax(eᵢ/√dₖ) ∈ ℝ^m,  Σⱼ αᵢⱼ = 1
第4步 聚合:   outᵢ = Σⱼ αᵢⱼ · vⱼ ∈ ℝ^{dᵥ}
```

`outᵢ` 是 m 个 value 的**凸组合**(权重非负且和为 1),它一定落在这些 value 张成的凸包内。这点很重要:注意力的输出是 value 的"加权平均检索结果",不会外推到 value 之外。

### 3.2 向量化到矩阵公式

把 n 个 query 堆叠,上述四步可以写成一个紧凑的矩阵表达式。逐步推:

```
打分矩阵:  S = Q Kᵀ        S ∈ ℝ^{n×m},  Sᵢⱼ = qᵢᵀ kⱼ
缩放:       S ← S / √dₖ
按行 softmax: A = softmax_row(S/√dₖ)    A ∈ ℝ^{n×m},  每行和为 1
聚合:       O = A V         O ∈ ℝ^{n×dᵥ}
```

合并成一行,就是那个著名公式:

```
Attention(Q, K, V) = softmax( Q Kᵀ / √dₖ ) V
```

**逐符号确认形状**(初学者最容易在这里绕晕,务必对齐):

```
Q:  n × dₖ
Kᵀ: dₖ × m        =>  QKᵀ : n × m        (每行 i = query i 对所有 key 的分数)
softmax 沿 m 维(每行内归一化),形状不变:  A : n × m
V:  m × dᵥ        =>  AV : n × dᵥ          (每行 i = query i 的输出)
```

关键点:**softmax 沿 key 维(行内)做**。第 i 行 `Aᵢ:` 是 query i 在 m 个 key 上的概率分布。如果你把 softmax 做错了维度(沿 query 列做),整个语义就崩了——这是实现时的高频 bug。

> 一句话记忆:`QKᵀ` 给"相似度矩阵",行 softmax 给"每个 query 的注意力分布",乘 `V` 给"按分布检索出的内容"。

### 3.3 为什么除以 √dₖ:点积方差的完整推导

这是本章最该背下来的推导。问题:打分前为什么要除以 √dₖ?

**建模假设**。假设 q 和 k 的各分量是**独立、零均值、单位方差**的随机变量(初始化时这是合理近似,例如标准正态初始化后):

```
对所有 a:  E[qₐ] = 0,  Var(qₐ) = 1,  E[kₐ] = 0,  Var(kₐ) = 1
q 与 k 相互独立,各分量也相互独立
```

考察未缩放点积 `e = q·k = Σₐ qₐ kₐ`(a 从 1 到 dₖ)。

**第一步:单项的均值与方差。** 看每一项 `qₐ kₐ`。因为 q、k 独立:

```
E[qₐ kₐ] = E[qₐ] · E[kₐ] = 0 · 0 = 0
Var(qₐ kₐ) = E[(qₐ kₐ)²] − (E[qₐ kₐ])²
           = E[qₐ²] · E[kₐ²] − 0           (独立 ⇒ 乘积期望=期望乘积)
           = Var(qₐ) · Var(kₐ)             (因为均值为0, E[X²]=Var(X))
           = 1 · 1 = 1
```

**第二步:求和的均值与方差。** dₖ 个项 `qₐkₐ` 两两独立(不同 a 之间独立):

```
E[e] = Σₐ E[qₐ kₐ] = 0
Var(e) = Σₐ Var(qₐ kₐ)        (独立随机变量之和,方差可加)
       = dₖ · 1 = dₖ
```

**结论**:`E[q·k] = 0`,`Var(q·k) = dₖ`,标准差 `σ = √dₖ`。点积的典型量级随维度按 √dₖ **增长**。dₖ=64 时,点积的标准差是 8;dₖ=512 时是约 22.6。

我在数值上验证过这个结论:dₖ=64、20 万对随机正态向量,点积的经验方差是 63.99,与理论值 64 吻合到两位有效数字。

**第三步:缩放把方差打回 1。** 除以 √dₖ 后:

```
Var( e / √dₖ ) = Var(e) / (√dₖ)² = dₖ / dₖ = 1
```

于是无论 dₖ 多大,送进 softmax 的 logits 的方差都被稳定在 1 的量级。**这正是 √dₖ 的全部来历——它是把点积方差归一化到 1 的标准差缩放因子。**

注意一个微妙处:这里假设 q、k 是"独立随机"的,这在**训练初期**(刚初始化、还没学到对齐)最准。训练后 q、k 会学到相关性,严格的方差不再恰好是 dₖ。但缩放的目的恰恰是**保证训练能顺利启动**——如果一开始 logits 就在饱和区,梯度上不去,模型根本学不起来。缩放是一个"让优化能进行"的工程保险,不是对收敛态的精确刻画。

### 3.4 不缩放为什么坏:softmax 饱和与梯度消失

光知道"方差变大"还不够,要讲清**方差大到底怎么伤害训练**。链条是:大 logits → softmax 接近 one-hot(饱和)→ softmax 的雅可比塌缩 → 反向传播梯度趋于 0。

先把 softmax 的雅可比推出来(反向传播必用,§3.5 还要再用)。

**softmax 雅可比推导。** 设 `p = softmax(z)`,即 `pᵢ = exp(zᵢ) / Σₗ exp(zₗ)`。求 `∂pᵢ/∂zⱼ`。

```
记 Z = Σₗ exp(zₗ)。   pᵢ = exp(zᵢ) / Z

情形 i = j:
  ∂pᵢ/∂zᵢ = [exp(zᵢ)·Z − exp(zᵢ)·exp(zᵢ)] / Z²       (商的导数)
          = exp(zᵢ)/Z − (exp(zᵢ)/Z)²
          = pᵢ − pᵢ² = pᵢ(1 − pᵢ)

情形 i ≠ j:
  ∂pᵢ/∂zⱼ = [0·Z − exp(zᵢ)·exp(zⱼ)] / Z²
          = − (exp(zᵢ)/Z)(exp(zⱼ)/Z)
          = − pᵢ pⱼ
```

合并成矩阵形式(J 的第 (i,j) 元是 ∂pᵢ/∂zⱼ):

```
J = diag(p) − p pᵀ
```

我数值验证过:对随机 logits,有限差分算出的雅可比与 `diag(p) − ppᵀ` 的最大逐元素误差约 6×10⁻¹¹,完全吻合。

**饱和分析。** 现在看当 logits 量级很大时雅可比发生什么。假设某个 `pₖ → 1`(softmax 几乎把全部概率压到一个位置,即接近 one-hot),其余 `pᵢ → 0`:

```
对角项 pᵢ(1−pᵢ):  当 pᵢ→1 时 → 1·0 = 0;  当 pᵢ→0 时 → 0·1 = 0
非对角项 −pᵢpⱼ:    至少一个因子 →0,故 → 0
```

**整个雅可比 J → 0**。也就是说,softmax 一旦饱和(逼近 one-hot),它对输入 z 的局部导数几乎全为零——**梯度无法穿过 softmax 回传**。这是经典的饱和型梯度消失,和 sigmoid 在两端饱和是同一类问题。

把这个和 §3.3 连起来:不缩放时 logits 标准差 √dₖ(dₖ=512 时约 22.6),softmax 的输入动辄相差几十,exp 之后差几个数量级,分布必然接近 one-hot → 饱和 → 梯度消失 → **训练初期就卡死**。

我做了个直接的数值实验佐证。固定一组 logits,改变其标准差,观察雅可比的 Frobenius 范数(范数越大代表梯度通道越"通畅"):

```
logit 标准差 = 1.0   max(p)=0.38   ‖J‖_F = 0.415
logit 标准差 = 4.0   max(p)=0.78   ‖J‖_F = 0.333
logit 标准差 = 8.0   max(p)=0.86   ‖J‖_F = 0.221
```

上表是特定 logit 集下的示例数值,并非对任意输入都成立:‖J‖_F 随 logit 标准差其实是先升后降(峰值约在中等 std),只有当标准差进一步增大、分布趋于 one-hot 后,雅可比范数才随之单调塌缩趋 0。把标准差摁回 1,正是让 softmax 工作在梯度健康的区间。**这就是"不缩放 → softmax 饱和 → 梯度消失 → 学不起来"的完整因果链。**

### 3.5 完整反向传播(把雅可比用起来)

为了让你真的能自己实现,把 scaled dot-product attention 的反向也推一遍。前向记号:`S = QKᵀ/√dₖ`,`A = softmax_row(S)`,`O = AV`。设上游传来 `dO = ∂L/∂O`(形状 n×dᵥ)。

```
对 V:    dV = Aᵀ dO                              (O = AV 的标准 matmul 反传)
对 A:    dA = dO Vᵀ                              (形状 n×m)
对 S:    逐行用 softmax 雅可比。对第 i 行:
         dSᵢ = Jᵢ dAᵢ = (diag(Aᵢ) − AᵢAᵢᵀ) dAᵢ
         向量化技巧:dS = A ⊙ (dA − rowsum(A ⊙ dA))   (⊙ 逐元素, rowsum 按行求和后广播)
对 Q,K:  S = QKᵀ/√dₖ  ⇒  dQ = (dS) K / √dₖ ,  dK = (dS)ᵀ Q / √dₖ
```

那个 `dS = A ⊙ (dA − rowsum(A⊙dA))` 是把 `diag(p)−ppᵀ` 作用在向量上的高效写法,展开验证:第 i 个分量 `Σⱼ Jᵢⱼ dAⱼ = pᵢ dAᵢ − pᵢ Σⱼ pⱼ dAⱼ = pᵢ(dAᵢ − ⟨p, dA⟩)`,正好对应逐元素乘减去行内加权和。FlashAttention 等高效实现的反向核,本质就是把这套公式融合(fuse)进一个不落地 n×m 中间矩阵的 kernel。

## 4. 代码:从零实现单头注意力(含 mask)

下面是能体现全部机制的 PyTorch 实现。重点看 mask 怎么加、softmax 沿哪个维、缩放放在哪。

```python
import torch
import torch.nn.functional as F
import math

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Q: (..., n, d_k)   queries
    K: (..., m, d_k)   keys
    V: (..., m, d_v)   values
    mask: (..., n, m)  布尔张量, True=允许 attend, False=屏蔽 (例如因果掩码或 padding)
    返回: out (..., n, d_v),  attn (..., n, m)
    """
    d_k = Q.size(-1)

    # 第1步: 打分 + 缩放。 K.transpose(-2,-1): (..., d_k, m)
    # scores: (..., n, m), 每行 i = query i 对所有 key 的分数
    scores = (Q @ K.transpose(-2, -1)) / math.sqrt(d_k)   # 关键: 除以 sqrt(d_k)

    # 第2步: 掩码。被屏蔽的位置填 -inf, 经 softmax 后权重→0
    if mask is not None:
        # 用一个大负数而非 float('-inf'), 避免某行全屏蔽时出现 NaN
        scores = scores.masked_fill(~mask, -1e9)

    # 第3步: 沿 key 维 (最后一维 m) 做 softmax —— 务必是 dim=-1, 不是 dim=-2
    attn = F.softmax(scores, dim=-1)        # (..., n, m), 每行和为 1

    # 第4步: 聚合
    out = attn @ V                          # (..., n, d_v)
    return out, attn


def causal_mask(n, device=None):
    """下三角掩码: 位置 i 只能看到 j<=i (自回归解码用)。True=可见。"""
    return torch.tril(torch.ones(n, n, dtype=torch.bool, device=device))


# --- 自检 ---
torch.manual_seed(0)
n, m, d_k, d_v = 4, 4, 8, 16
Q = torch.randn(n, d_k); K = torch.randn(m, d_k); V = torch.randn(m, d_v)

out, attn = scaled_dot_product_attention(Q, K, V, mask=causal_mask(n))
print(attn.round(decimals=3))     # 应是下三角, 每行和为1
assert torch.allclose(attn.sum(-1), torch.ones(n), atol=1e-5)
# 因果性: 第0行只有第0列非零
assert (attn[0, 1:].abs() < 1e-8).all()
```

几个实现要点,每个都对应前面的推导:

- **缩放写在 matmul 之后、mask 之前**:对 logits 整体缩放,mask 的 -1e9 不应被 √dₖ 改变量级(它要足够负,使 exp 后为 0)。
- **mask 用大负数 -1e9 而非真 -inf**:若某一行被全部屏蔽(padding 行),`-inf` 经 softmax 会得到 `0/0 = NaN`;-1e9 退化成近似均匀分布,数值安全。这是工业实现里的标准技巧。
- **softmax `dim=-1`**:沿 key 维归一化。错成 `dim=-2` 是经典 bug,代码不会报错但语义全错。
- **因果掩码**用下三角,保证位置 i 不偷看未来——这是 decoder 自注意力(以及 GPT)的关键约束。

HuggingFace 和 nanoGPT 的现代实现里,这一整段通常被一行 `F.scaled_dot_product_attention(q, k, v, is_causal=True)`(PyTorch 2.0+ 内置,底层走 FlashAttention)替代,但**它的数学就是上面这四步**。手写一遍再切换到内置算子,你才知道那一行在算什么。

## 5. 复杂度分析:O(n²) 的根源

这一节决定了你能不能理解后续所有"高效注意力""长上下文"的工作——它们全是在和这里的 n² 搏斗。设序列长 n(取 m=n,自注意力),维度 dₖ ≈ dᵥ ≈ d。

### 5.1 计算量(FLOPs)

```
QKᵀ:    Q(n×d) × Kᵀ(d×n) → S(n×n)
        每个输出元素是长度 d 的点积 = d 次乘加 ≈ 2d FLOPs
        共 n×n 个元素  ⇒  ≈ 2 n² d FLOPs           → O(n²d)

softmax: 对 n×n 矩阵逐行做, 每元素常数次运算  ⇒  O(n²)

AV:     A(n×n) × V(n×d) → O(n×d)
        同理  ≈ 2 n² d FLOPs                        → O(n²d)
```

**总计 O(n²d)**。两个 matmul 各贡献 ≈ 2n²d,主导项是 `n²d`。注意:**计算量对序列长度是平方级 n²,对维度只是线性 d**。这就是为什么"把序列拉长"远比"把模型加宽"更致命。

(若把生成 Q/K/V 的三个投影 `XW`(各 ≈ 2nd² FLOPs)和输出投影也算进去,它们是 O(nd²)。当 n < d 时投影占主导;当 n ≫ d 时,n²d 的注意力项主导——长上下文场景正是后者。)

### 5.2 显存(memory)

显存是比 FLOPs 更硬的墙。朴素实现必须把**注意力分数矩阵 S(以及 A)实例化在显存里**:

```
S, A 的形状: n × n
单头单样本:  n² 个浮点数  ⇒  O(n²) 显存
```

加上 batch size b 和头数 h,中间注意力矩阵的显存是 **O(b·h·n²)**。具体感受一下量级(fp16,每个数 2 字节,单头单样本):

```
n = 1024:    1024²×2 B ≈ 2 MB
n = 8192:    8192²×2 B ≈ 128 MB
n = 32768:   32768²×2 B ≈ 2 GB        ← 单头单样本就 2 GB
```

乘上 batch 和多头,几十 GB 的显卡瞬间被一个 n×n 矩阵吃光。**这就是长上下文的显存墙,也是 FlashAttention(Dao et al., 2022)的出发点**——它通过分块(tiling)和在线 softmax(online softmax)**不把完整 n×n 矩阵落地到显存**,把注意力的显存从 O(n²) 降到 O(n),计算量仍是 O(n²d) 但因避免反复读写 HBM 而大幅加速。注意 FlashAttention 优化的是**显存与访存**,没有改变 O(n²) 的计算复杂度;真正想降计算复杂度要靠稀疏注意力、线性注意力等(后续章节)。

### 5.3 一张表收尾

| 量 | 朴素全注意力 | 主导来源 |
|---|---|---|
| 计算量 FLOPs | O(n²d) | QKᵀ 与 AV 两个 matmul |
| 中间显存 | O(n²)(每头每样本) | 实例化的 S/A 矩阵 |
| 最长依赖路径 | O(1) | 任意位置一步直连(对比 RNN 的 O(n)) |
| 参数量(单头) | O(d²) | W_Q, W_K, W_V, W_O 各 d×d |

最后一行点出注意力的灵魂卖点:**最大路径长度 O(1)**。RNN 里两个相距 n 的 token 要经过 n 步才能交互,注意力一步就连上。这是 Transformer 能建模长程依赖、且高度并行的根本原因——代价就是 §5.1/5.2 的 O(n²)。**这个 trade-off(用 n² 的计算/显存换 O(1) 的路径长度和完全并行)是理解整个 Transformer 时代的主线之一。**

## 6. 设计权衡与常见坑

- **softmax 维度搞反**(`dim=-2` 当成 `dim=-1`)。代码照跑、loss 照降一点,但模型废了。永远显式确认 softmax 沿 **key 维**。
- **缩放因子用错或漏掉**。漏掉 √dₖ 在小模型小 dₖ 上可能看不出问题,一旦 dₖ 上到 64/128 训练就不稳。另外缩放是 `1/√dₖ` 不是 `1/dₖ`——前者归一化标准差,后者会把 logits 压得过小、softmax 过于平坦、注意力学不出尖锐对齐。
- **mask 用 `-inf` 导致 NaN**。padding 行被全屏蔽时 `softmax(-inf,...,-inf)=NaN`,污染整个 batch 还很难定位。用 -1e9(或 fp16 下用 -65504 附近的安全大负数,真 -inf 在某些 fused kernel 里反而被特殊处理——以具体框架为准)。
- **把 query 维当成归一化维**。注意力分布是"每个 query 在所有 key 上的分布",不是"每个 key 在所有 query 上"。前向画错维度,反向公式(§3.5)全错。
- **以为 FlashAttention 降了复杂度**。它降的是显存/访存常数,计算仍是 O(n²d)。想突破 n² 计算量得换注意力形式(稀疏/线性/低秩),那是另一类近似,通常有精度代价。
- **数值精度**:fp16/bf16 下 `QKᵀ` 容易溢出,稳妥做法是 matmul 在更高精度累加,或先缩放再乘。softmax 内部务必做 `z − max(z)` 的稳定化(PyTorch 内置 `F.softmax` 已处理,自己手写 exp 时别忘)。
- **value 的凸组合性质**:输出永远在 value 凸包内,注意力**不能外推**到 value 之外。需要"放大/变换"信息时,靠的是后续的 W_O 投影和 FFN,而非注意力本身。

### 练习 1:点积方差,从头推 E[q·k]=0、Var(q·k)=dₖ 与 √dₖ 缩放

**建模假设。** q、k 的各分量独立、零均值、单位方差,且 q 与 k 相互独立:

```
对所有 a:  E[qₐ]=0, Var(qₐ)=1, E[kₐ]=0, Var(kₐ)=1
不同分量之间独立, q 与 k 之间也独立
```

考察未缩放点积 `e = q·k = Σₐ qₐkₐ`(a 从 1 到 dₖ)。

**第一步:单项 qₐkₐ 的均值与方差。**

均值(用独立性,乘积的期望=期望的乘积):

```
E[qₐkₐ] = E[qₐ]·E[kₐ] = 0·0 = 0
```

方差(零均值时 E[X²]=Var(X)):

```
Var(qₐkₐ) = E[(qₐkₐ)²] − (E[qₐkₐ])²
          = E[qₐ²kₐ²] − 0
          = E[qₐ²]·E[kₐ²]        (qₐ²、kₐ² 仍独立 ⇒ 乘积期望=期望乘积)
          = Var(qₐ)·Var(kₐ)       (零均值: E[qₐ²]=Var(qₐ), E[kₐ²]=Var(kₐ))
          = 1·1 = 1
```

**第二步:求和 e = Σₐ qₐkₐ。** dₖ 个项两两独立(不同 a 对应的 qₐkₐ 相互独立),均值可加、独立时方差可加:

```
E[e]   = Σₐ E[qₐkₐ] = Σₐ 0 = 0
Var(e) = Σₐ Var(qₐkₐ) = dₖ · 1 = dₖ
```

**所以 E[q·k]=0,Var(q·k)=dₖ,标准差 σ=√dₖ。** 点积的典型量级随维度按 √dₖ 增长(dₖ=64 时 σ=8,dₖ=512 时 σ≈22.6)。

**第三步:为什么除以 √dₖ 后方差变 1。** 对随机变量乘以常数 c,方差乘以 c²。取 c = 1/√dₖ:

```
Var(e/√dₖ) = Var(e)/(√dₖ)² = dₖ/dₖ = 1
```

**结论:除以 √dₖ 恰好把点积方差从 dₖ 归一化回 1,使送进 softmax 的 logits 量级不随维度膨胀。** 注意必须是 1/√dₖ 而非 1/dₖ——前者归一化标准差(方差→1),后者会把方差压成 1/dₖ、logits 过小、softmax 过平,学不出尖锐对齐。

**数值验算。** dₖ=64、20 万对随机正态向量:经验 `Var(q·k) = 63.99`(理论 64),除以 √dₖ 后经验方差 ≈ 0.9998,与推导吻合。

### 练习 2:softmax 雅可比 J=diag(p)−ppᵀ,以及不缩放→饱和→梯度消失

**第一部分:推导 softmax 雅可比。** 设 `p = softmax(z)`,即 `pᵢ = exp(zᵢ)/Z`,`Z = Σₗ exp(zₗ)`。求 `∂pᵢ/∂zⱼ`,分两种情形。

对角(i = j),用商的求导法则:

```
∂pᵢ/∂zᵢ = [exp(zᵢ)·Z − exp(zᵢ)·exp(zᵢ)] / Z²
        = exp(zᵢ)/Z − (exp(zᵢ)/Z)²
        = pᵢ − pᵢ² = pᵢ(1 − pᵢ)
```

非对角(i ≠ j),分子里 exp(zᵢ) 不含 zⱼ,只有 Z 对 zⱼ 求导出 exp(zⱼ):

```
∂pᵢ/∂zⱼ = [0·Z − exp(zᵢ)·exp(zⱼ)] / Z²
        = −(exp(zᵢ)/Z)(exp(zⱼ)/Z)
        = − pᵢpⱼ
```

把两种情形拼成矩阵(第 (i,j) 元为 ∂pᵢ/∂zⱼ):

```
J = diag(p) − p pᵀ
```

(验证:对角元 pᵢ − pᵢ·pᵢ = pᵢ(1−pᵢ) ✓,非对角元 0 − pᵢpⱼ = −pᵢpⱼ ✓。)

**第二部分:某个 pₖ→1(分布趋于 one-hot)时 J→0。** 此时 pₖ→1,其余 pᵢ→0。逐项取极限:

```
对角项 pᵢ(1−pᵢ):  pₖ→1 时 → 1·(1−1)=0
                   其余 pᵢ→0 时 → 0·(1−0)=0
非对角项 −pᵢpⱼ:    任意 i≠j, 至少一个因子 →0  ⇒  → 0
```

**所以整个雅可比 J→0。** 反向传播时 `dz = J·dp`,J 全为 0 意味着无论上游梯度 dp 多大,回传到 z 的梯度 dz≈0——**梯度穿不过饱和的 softmax**。这与 sigmoid 在两端饱和、导数 σ(1−σ)→0 是同一类问题。

**第三部分:为什么不缩放会让训练初期就饱和(结合练习 1)。** 由练习 1,未缩放 logits 的标准差是 √dₖ。dₖ=512 时 σ≈22.6,意味着各 logit 之间动辄相差几十:

```
两个 logit 相差 Δ → softmax 权重之比 = exp(Δ)
Δ=10 时比值 ≈ 2.2万倍;  Δ=20 时 ≈ 4.9亿倍
```

exp 把量级差放大成几个数量级,分布在初始化时就已逼近 one-hot,落入饱和区 → J→0 → 梯度消失 → **训练根本启动不了**。除以 √dₖ 把标准差摁回 1,logit 之间典型差距是 O(1),softmax 工作在分布不尖锐、雅可比范数健康的区间,梯度才能有效回传。

**数值验算。** 对随机 logits,有限差分算出的雅可比与 `diag(p)−ppᵀ` 最大逐元素误差 ≈2×10⁻¹¹,完全吻合;同时随 logit 标准差从 1→4→8 增大,max(p) 单调上升、‖J‖_F 随分布变尖而下降(具体数值依 logit 集而定,章节正文也注明该表是特定输入下的示例,趋势为分布越接近 one-hot、雅可比范数越塌缩趋 0)。

### 练习 3:numpy 实现前向 + 手写反向 + 有限差分校验

下面是纯 numpy 的完整实现:前向按 §3.2 四步,反向按 §3.5 公式,末尾用中心差分校验解析梯度。CPU 即可运行,无需 GPU 或额外依赖。

```python
import numpy as np

# ---------- 前向 ----------
def softmax_row(S):
    # 沿 key 维 (axis=1, 最后一维) 做 softmax; 减行最大值做数值稳定化
    S = S - S.max(axis=1, keepdims=True)
    E = np.exp(S)
    return E / E.sum(axis=1, keepdims=True)

def forward(Q, K, V):
    """Q:(n,dk) K:(m,dk) V:(m,dv) -> O:(n,dv), A:(n,m)"""
    dk = Q.shape[1]
    S = Q @ K.T / np.sqrt(dk)      # 打分 + 缩放 1/sqrt(dk)
    A = softmax_row(S)            # 行 softmax, 每行和为 1
    O = A @ V                      # 聚合: value 的凸组合
    return O, A

# ---------- 反向 (§3.5) ----------
def backward(Q, K, V, A, dO):
    """给定上游 dO=∂L/∂O, 返回 dQ, dK, dV"""
    dk = Q.shape[1]
    dV = A.T @ dO                                       # O=AV
    dA = dO @ V.T                                       # O=AV
    # dS = A ⊙ (dA − rowsum(A⊙dA)) —— 把 diag(p)−ppᵀ 作用到向量的高效写法
    dS = A * (dA - (A * dA).sum(axis=1, keepdims=True))
    dQ = dS @ K / np.sqrt(dk)                           # S=QKᵀ/√dk
    dK = dS.T @ Q / np.sqrt(dk)                         # S=QKᵀ/√dk
    return dQ, dK, dV

# ---------- 先验证 dS 高效写法 == 逐行 (diag(A_i)−A_iA_iᵀ)dA_i ----------
np.random.seed(0)
n, m, dk, dv = 4, 5, 8, 6
Q = np.random.randn(n, dk); K = np.random.randn(m, dk); V = np.random.randn(m, dv)
O, A = forward(Q, K, V)
dO = np.ones_like(O)                  # loss = O.sum() => dO 全 1

dA = dO @ V.T
dS_fast = A * (dA - (A * dA).sum(axis=1, keepdims=True))
dS_loop = np.zeros_like(dS_fast)
for i in range(n):
    p = A[i]
    J = np.diag(p) - np.outer(p, p)   # softmax 雅可比
    dS_loop[i] = J @ dA[i]
print("dS 高效写法 vs 逐行雅可比 max diff:", np.abs(dS_fast - dS_loop).max())  # ~1e-16

# ---------- 有限差分校验解析梯度 ----------
def loss(Q, K, V):
    O, _ = forward(Q, K, V)
    return O.sum()                    # 最简单的标量 loss

def num_grad(X, fn, eps=1e-6):        # 中心差分
    g = np.zeros_like(X)
    it = np.nditer(X, flags=['multi_index'])
    while not it.finished:
        idx = it.multi_index
        old = X[idx]
        X[idx] = old + eps; lp = fn()
        X[idx] = old - eps; lm = fn()
        X[idx] = old
        g[idx] = (lp - lm) / (2 * eps)
        it.iternext()
    return g

dQ, dK, dV = backward(Q, K, V, A, dO)
ngQ = num_grad(Q, lambda: loss(Q, K, V))
ngK = num_grad(K, lambda: loss(Q, K, V))
ngV = num_grad(V, lambda: loss(Q, K, V))

def rel_err(a, b):
    return np.abs(a - b).max() / (np.abs(a).max() + np.abs(b).max() + 1e-12)

print("rel err dQ:", rel_err(dQ, ngQ))
print("rel err dK:", rel_err(dK, ngK))
print("rel err dV:", rel_err(dV, ngV))
assert max(rel_err(dQ, ngQ), rel_err(dK, ngK), rel_err(dV, ngV)) < 1e-4
print("PASS: 解析梯度与有限差分一致, 最大相对误差 < 1e-4")
```

**实际运行输出(已验算):**

```
dS 高效写法 vs 逐行雅可比 max diff: 1.11e-16     # 机器精度, 两种写法等价
rel err dQ: 3.6e-10
rel err dK: 3.1e-10
rel err dV: 1.5e-10
PASS: 解析梯度与有限差分一致, 最大相对误差 < 1e-4
```

**要点说明:**
- `loss = O.sum()` ⇒ `dO = ∂L/∂O` 是全 1 矩阵,反向入口最简单。
- 关键的 `dS = A ⊙ (dA − rowsum(A⊙dA))`:第 i 分量展开为 `Σⱼ Jᵢⱼ dAⱼ = pᵢ dAᵢ − pᵢ Σⱼ pⱼ dAⱼ = pᵢ(dAᵢ − ⟨p,dA⟩)`,正是"逐元素乘 减去 行内加权和"——代码里先验证它逐行等于 `(diag(Aᵢ)−AᵢAᵢᵀ)dAᵢ`,差 ~1e-16。
- 中心差分误差是 O(ε²),用 ε=1e-6 配 float64,得到 ~1e-10 的相对误差,远优于要求的 1e-4。

### 练习 4:FLOPs 与显存随 n 的增长,及长上下文障碍

设序列长 n(自注意力取 m=n)、维度 dₖ≈dᵥ≈d。

**第一部分:推导单头前向 FLOPs ≈ 2n²d。** 把 §5.1 两个 matmul 分开算(一次乘加按 ≈2 FLOPs 计):

```
QKᵀ:  Q(n×d) × Kᵀ(d×n) → S(n×n)
      每个输出元素 = 长度 d 的点积 = d 次乘加 ≈ 2d FLOPs
      共 n×n 个元素  ⇒  ≈ 2n²d FLOPs

AV:   A(n×n) × V(n×d) → O(n×d)
      每个输出元素 = 长度 n 的点积 = n 次乘加 ≈ 2n FLOPs
      共 n×d 个元素  ⇒  ≈ 2n²d FLOPs

softmax: n×n 矩阵逐元素常数运算  ⇒  O(n²), 低阶项可忽略
```

**合计 ≈ 2n²d + 2n²d = 4n²d,主导项 O(n²d)。** 单看"分别给 QKᵀ 和 AV 的项"则各 ≈ 2n²d——这就是题面里 2n²d 的来源。关键观察:**对序列长 n 是平方级,对维度 d 只是线性**,所以"拉长序列"远比"加宽模型"更致命。

**第二部分:显存(中间注意力矩阵实例化)。** 朴素实现要把 S(以及 A)的 n×n 矩阵落地。加上 batch=b、heads=h,fp16 每个数 2 字节:

```
中间显存 ≈ b · h · n² · 2  字节
```

代入 b=8、h=12:

```
n = 1024:   8·12·1024²·2  = 2.01e8  字节 ≈ 192 MiB  (≈ 0.19 GiB)
n = 8192:   8·12·8192²·2  = 1.29e10 字节 ≈ 12 GiB
n = 32768:  8·12·32768²·2 = 2.06e11 字节 ≈ 192 GiB
```

(单头单样本对照,与正文 §5.2 一致:n=1024→2 MiB,n=8192→128 MiB,n=32768→2 GiB。)

**第三部分:从 n=8k 到 n=32k 的增长倍数。**

```
n 从 8192 → 32768, 翻 4 倍 (32768/8192 = 4)
计算量 ∝ n²d  ⇒  涨 4² = 16 倍   (≈4n²d 项)
显存   ∝ n²   ⇒  涨 4² = 16 倍   (12 GiB → 192 GiB, 正好 16×)
```

**n 翻 4 倍,计算量和显存都涨 16 倍。**

**第四部分:为什么这是长上下文的核心障碍。**
- **平方级缩放**:序列翻倍,计算和显存各翻 4 倍;翻 4 倍就各涨 16 倍。上下文从 8k 拉到 32k,光中间注意力矩阵就从 12 GiB 涨到 192 GiB,几十 GB 的显卡被一个 n×n 矩阵直接吃光——**显存是比算力更硬的墙**。
- **代价的本质**:这个 O(n²) 正是注意力换来 O(1) 最长依赖路径与完全并行所付的价(§5.3),长上下文等于把这个代价不断放大。
- **缓解方向各有边界**:FlashAttention 用分块 + 在线 softmax 让 n×n 矩阵不落地,把显存降到 O(n)、并因减少 HBM 读写而加速,但**计算量仍是 O(n²d)**;要真正降计算复杂度,只能换注意力形式(稀疏 / 线性 / 低秩),那是带精度代价的近似。所以"长上下文"始终在和这个 n² 搏斗——这也是后续所有高效注意力工作的共同出发点。
