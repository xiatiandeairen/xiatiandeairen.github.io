> 这一章把你从"会推 PPO、会写策略梯度"带到"能把一个只会续写的预训练语言模型，沿着监督微调 → 奖励建模 → 强化优化三步，真正调成一个有用、诚实、无害的助手"。我们要把这条产线上的每一个数学决策都拆到能自己复现推导、能照着写代码的程度。

## 0. 母题：写不出损失函数的目标，怎么优化

前面几章我们一直假设奖励 `r(s,a)` 是给定的——围棋赢了给 +1，游戏过关给分数。但**对齐（alignment）这件事最刺人的地方在于：它的目标根本写不出损失函数**。

"这个回答更有用""这个更诚实""这个更无害"——你没法写一个 `loss = f(回答)` 的解析式，因为"有用"依赖语境、依赖人的偏好、依赖无数说不清的常识。监督学习需要标准答案 `y*`，可对开放式生成，"标准答案"不存在：同一个问题有无数个好回答。

但人类有一个能力是确定的：**给两个回答，人一眼能比出 A 比 B 好。** 说不出"好的绝对分数"，但能给出"相对偏好"。RLHF（Reinforcement Learning from Human Feedback，基于人类反馈的强化学习）整套方法论就建立在这一个支点上——**把"人能比较"这件廉价的事，撬动成"机器能优化"这件昂贵的事**。

撬动分三步，这也是本章的骨架：

1. **SFT（Supervised Fine-Tuning，监督微调）**：先用人写的高质量示范，把预训练模型从"会续写"调成"会按指令对话"。这一步给后面两步一个像样的起点。
2. **RM（Reward Modeling，奖励建模）**：收集人对成对回答的偏好标注，训一个神经网络奖励模型 `r_φ(x,y)`，把"说不清的好"变成"可优化的标量分"。
3. **RL（这里用 PPO）**：把生成建模成序贯决策，用奖励模型当奖励信号、用 PPO 优化语言模型策略，同时用一个 KL 惩罚项拴住它别跑偏。

这三步对应 InstructGPT（Ouyang et al., 2022）确立的经典流程，也是 ChatGPT 第一版的训练范式。**本章的核心不是流程本身（流程谁都背得出），而是三个数学决策为什么成立**：偏好为什么能用 Bradley-Terry 模型转成损失、KL 惩罚为什么是必需品而非可选项、PPO 在 token 级序贯决策上到底在更新什么。把这三个讲透，你才算真正理解 RLHF，而不是会调库。

**前置假设**：你已经掌握策略梯度（REINFORCE）、优势函数 `A(s,a)`、PPO 的裁剪目标与 GAE（前面的 PPO 章节）。本章不重推 PPO 本体，只讲它落到 LLM 上的特化。

## 1. 第一步 SFT：为什么 RL 之前必须先监督微调

**问题**：能不能跳过 SFT，直接拿预训练模型上 RM + PPO？答案是不能，理由是数学的，不是工程偏好。

预训练模型 `π_base` 的分布是"互联网文本续写分布"。你给它 "如何用 Python 读取文件？"，它很可能续写成 "如何用 Java 读取文件？如何用 Go……"（因为训练语料里这种列表很常见），而不是回答问题。**这个分布离"对话助手分布"太远**，三个致命后果：

- **采样质量太低**：PPO 是 on-policy（在线策略）的，它只能从当前策略采样到的回答里学。如果策略压根采不出像样的回答，奖励模型给的全是低分，梯度信号几乎为零，学不动。
- **KL 锚点错位**：第 3 步要用 KL 把策略拴在一个参考分布 `π_ref` 上。这个锚点必须本身就是个像样的对话模型，否则你把策略拴在"乱续写"上，越拴越糟。
- **探索空间爆炸**：语言的动作空间是整个词表（几万到十几万 token），序列长度上百。从一个糟糕初始点出发，在这么大的空间里靠 RL 探索出好策略，样本效率低到不可接受。

SFT 干的事很朴素：拿人工写的 (prompt, 高质量回答) 对，做标准的**最大似然 / 交叉熵**微调。

```
L_SFT(θ) = − E_{(x, y*) ~ D_demo} [ Σₜ log π_θ(y*ₜ | x, y*_<t) ]
```

逐 token 的负对数似然，和预训练同一个损失，只是数据换成了示范对话。`y*_<t` 是 `y*` 的前 t-1 个 token（teacher forcing，用真值做条件）。

**SFT 在 RLHF 里的真正角色，是把策略"挪到"对话分布的一个好邻域里**，让后面的 RM 打分有区分度、让 PPO 的采样有质量、让 KL 有个合理的锚。它不追求最优，只追求"足够好的起点"。这就是为什么 SFT 数据量不需要很大（几千到几万条高质量示范，远少于预训练），但质量要求极高——**SFT 教的是"格式与风格"，不是"知识"**，知识在预训练里已经有了。

> 一个常被忽略的点：SFT 后的模型，通常就是后面要用的参考模型 `π_ref`。也就是说 `π_ref = π_SFT`。记住这一点，第 3 步的 KL 项才好理解。

## 2. 第二步 RM：从成对偏好到一个可优化的标量

这是本章数学密度最高的一节，也是 RLHF 真正的地基。

### 2.1 问题形式化

我们手里有偏好数据集 `D_pref = {(x, y_w, y_l)}`，每条是：一个 prompt `x`、一个人类标注者**更偏好的回答 y_w**（winner）、一个**较差的回答 y_l**（loser）。注意我们**只有相对序，没有绝对分**——标注者只说"w 比 l 好"，没说好多少、没说 w 值几分。

我们想训一个奖励模型 `r_φ(x,y)`，给任意 (prompt, 回答) 输出一个标量分。要求：`r_φ` 给 `y_w` 的分应该高于 `y_l`。问题是——**只知道序，怎么定义一个让分数有意义的损失？**

### 2.2 Bradley-Terry 模型：把"谁更好"翻译成概率

答案来自一个 1952 年的统计模型——**Bradley-Terry 模型**（Bradley & Terry, 1952，原本用于体育比赛排名）。它的核心假设：

**每个对象 i 有一个潜在的实数"实力分" s_i；i 在与 j 的两两比较中胜出的概率，只由两者实力分之差决定。** 具体地：

```
P(i ≻ j) = exp(sᵢ) / ( exp(sᵢ) + exp(sⱼ) )
```

这就是一个 softmax over 两个选项。我们把"实力分"换成奖励模型的输出 `s = r_φ(x,y)`，就得到 RLHF 里的偏好概率：

```
P(y_w ≻ y_l | x) = exp(r_φ(x,y_w)) / ( exp(r_φ(x,y_w)) + exp(r_φ(x,y_l)) )
```

**为什么这个形式是对的、不是随便凑的？** 三个性质值得逐个验算，它们是 Bradley-Terry 能成立的全部理由：

1. **只依赖差值，不依赖绝对值**。把分子分母同除 `exp(r(x,y_w))`，立刻得到下面的化简，可见 P 只是 `r(x,y_w) − r(x,y_l)` 的函数。这恰好匹配数据——我们本来就只有相对信息，模型也只应该从相对信息里学。绝对零点是自由的（整体加常数 P 不变），这正确反映了"偏好数据无法确定绝对分"这一事实。
2. **概率随差值单调**：差越大，胜率越接近 1。符合直觉。
3. **传递性的概率版本**：如果 a 普遍胜 b、b 普遍胜 c，模型会自动推出 a 大概率胜 c（因为实力分 `sₐ > s_b > s_c`）。这让我们能从**局部成对比较**外推到**全局排序**——这是把"廉价的成对标注"撬动成"全局奖励函数"的数学关键。

### 2.3 推导 RM 损失（完整，逐行）

有了概率模型，损失就是**最大化观测到的偏好的似然**（等价于最小化负对数似然）。这是标准的极大似然估计（MLE）。逐行推：

```
第1步  把 Bradley-Terry 概率化简成 sigmoid。
       记差值 Δ = r_φ(x, y_w) − r_φ(x, y_l)。
       P(y_w ≻ y_l | x)
         = exp(r_w) / (exp(r_w) + exp(r_l))           （定义，记 r_w=r_φ(x,y_w)）
         = 1 / (1 + exp(r_l − r_w))                    （分子分母同除 exp(r_w)）
         = 1 / (1 + exp(−Δ))
         = σ(Δ)                                        （σ 是 logistic sigmoid）

       —— 这一步是整节的枢纽：成对偏好概率 = sigmoid(奖励差)。

第2步  写单条样本的对数似然。每条标注是"w 胜 l"这一个事件，
       它的概率就是 σ(Δ)，所以
       log-likelihood(单条) = log σ(Δ) = log σ( r_φ(x,y_w) − r_φ(x,y_l) )

第3步  整个数据集做 MLE：最大化所有样本对数似然之和，
       等价于最小化其负值的期望（除以样本数）：

       L_RM(φ) = − E_{(x, y_w, y_l) ~ D_pref} [ log σ( r_φ(x,y_w) − r_φ(x,y_l) ) ]
```

**这就是 RLHF 奖励模型的训练损失**，InstructGPT 用的就是它。它本质是一个**成对的 logistic 回归 / 二分类交叉熵**——把"w 胜 l"当成标签为 1 的二分类，logit 是奖励差 Δ。

来验算一下它的合理性（这是判断损失对不对的好习惯）：

- 当模型已经做对（`r_w ≫ r_l`，Δ → +∞），σ(Δ) → 1，`log σ(Δ) → 0`，损失趋于 0。✓ 做对不罚。
- 当模型做反（`r_w ≪ r_l`，Δ → −∞），σ(Δ) → 0，`log σ(Δ) → −∞`，损失趋于 +∞。✓ 做反重罚。
- 当模型没主意（Δ = 0），σ(0) = 0.5，损失 = `−log 0.5 ≈ 0.693 nats`。✓ 这是无信息基线。

### 2.4 梯度长什么样：它在"推开"什么

把损失对奖励差求导，能看清 RM 训练到底在干什么。利用 `d/dΔ [ −log σ(Δ) ] = −(1−σ(Δ)) = σ(Δ) − 1`（这是 sigmoid 的标准结论，可自己验：`σ' = σ(1−σ)`，`d log σ/dΔ = σ'/σ = 1−σ`），再用链式法则：

```
∂L/∂φ = E[ (σ(Δ) − 1) · ( ∇φ r_φ(x,y_w) − ∇φ r_φ(x,y_l) ) ]

其中 (σ(Δ) − 1) ∈ (−1, 0)，恒为负。
```

把负号吸收进去看梯度下降一步（φ ← φ − η·∂L/∂φ）的方向：参数朝着**抬高 r(x,y_w)、压低 r(x,y_l)** 的方向移动，**移动幅度由 (1 − σ(Δ)) 加权**。这个权重是关键设计——

**它是一种自动的难例聚焦（hard-example weighting）**：当模型已经把 w 排得远高于 l（σ(Δ)→1），权重 (1−σ(Δ))→0，几乎不更新（这条已经学会了，别浪费梯度）；当模型还分不清甚至排反（σ(Δ)≤0.5），权重接近或大于 0.5，狠狠更新。**梯度自动把算力投到模型还没学会的成对比较上**，这正是 logistic 损失优于"硬 margin"（如固定间隔 hinge）的地方——它平滑、自带难度自适应，不需要手调 margin。

### 2.5 RM 的网络结构与一个隐蔽的工程细节

`r_φ` 通常就是**拿 SFT 模型，去掉最后的 token 预测头（unembedding），换上一个输出单个标量的线性头**（value head / reward head）。输入整段 (x, y)，取**最后一个 token 位置**的隐状态过线性头，得到一个标量分 `r_φ(x,y)`。复用 SFT 权重是因为它已经"懂语言"，只需学"打分"这个新任务。

一个新手必踩、论文里却轻描淡写的坑：**奖励的绝对尺度和零点是不可辨识的（unidentifiable）**。前面证过 Bradley-Terry 整体加常数 P 不变，所以 RM 训完，`r_φ` 的绝对数值没有意义，只有差值有意义。实践中：
- 训练后常对 RM 输出做**标准化**（减去某个参考集上的均值），让奖励大致零均值，便于第 3 步 PPO 的优势估计稳定。
- 不要去解读"这个回答得了 3.7 分"的绝对含义——3.7 相对什么才有意义。

### 2.6 代码：RM 损失（核心机制片段）

```python
import torch
import torch.nn.functional as F

def reward_model_loss(reward_model, input_ids_w, input_ids_l, attn_w, attn_l):
    """
    Bradley-Terry / pairwise logistic 损失。
    reward_model: 输入一段 token，返回每个样本一个标量分 (取最后非 pad token 的 value-head 输出)
    _w 后缀 = 更优回答 (winner)，_l = 较差回答 (loser)
    """
    r_w = reward_model(input_ids_w, attn_w)   # shape: (batch,)
    r_l = reward_model(input_ids_l, attn_l)   # shape: (batch,)

    # Δ = r_w - r_l；损失 = -log σ(Δ)
    # 用 softplus 数值稳定地实现 -log σ(Δ) = log(1 + exp(-Δ)) = softplus(-Δ)
    loss = F.softplus(-(r_w - r_l)).mean()

    # 监控指标：成对准确率 (模型把 w 排在 l 前面的比例)，这是 RM 唯一真正可信的健康度指标
    accuracy = (r_w > r_l).float().mean()
    return loss, accuracy
```

注意用 `softplus(-Δ)` 而不是 `-torch.log(torch.sigmoid(Δ))`：前者数值稳定（避免 σ 下溢到 0 后 log 出 -inf）。这是个工程细节但很重要——**RLHF 全程对数值稳定极度敏感**，本章后面会反复看到。

### 2.7 RM 的样本复杂度与方差直觉

RM 学的是一个标量回归（差值回归），它的有效监督信号是**成对比较数**，不是回答数。N 个回答两两配对可产生 O(N²) 个对，但实践中标注预算有限。一个粗略的方差直觉：每个 prompt 多采几个回答、构造多个成对比较，能在不额外采样的情况下放大监督密度（这是后面 best-of-n 和 GRPO 思想的种子）。RM 准确率（成对准确率）通常做到 65%–75% 区间就已经能驱动有效的 PPO（具体数值随任务和数据规模差异很大，**待核**），**不需要也不可能做到接近 100%**——人类标注本身就有内部不一致（同一对，不同人甚至同一人不同时刻判断会变），这构成偏好数据的不可约噪声（贝叶斯误差），是 RM 准确率的天花板。

## 3. 第三步 RL：把生成建模成序贯决策

现在我们有了一个会打分的 `r_φ`。怎么用它来"优化"语言模型？这一节先建模，下一节讲为什么必须加 KL，再下一节落到 PPO。

### 3.1 把自回归生成翻译成 MDP

语言模型生成是逐 token 的，天然是序贯决策。映射如下（这是把 RL 搬到 LLM 的第一性翻译，务必记牢）：

```
状态 sₜ  = (prompt x, 已生成的前 t-1 个 token y_<t)     —— 上下文
动作 aₜ  = 第 t 个 token yₜ ∈ 词表 V                      —— 选下一个词
策略 π_θ(aₜ | sₜ) = π_θ(yₜ | x, y_<t)                     —— 就是 LM 的 softmax 输出
转移   sₜ → sₜ₊₁ = sₜ ⊕ aₜ                                 —— 确定性！把新 token 拼上去
奖励   除终止外全程 0，序列结束时给一次 r_φ(x, y)         —— 极度稀疏的终局奖励
```

三个特征决定了 LLM-RL 和经典 RL 不一样：

- **转移是确定的**：拼 token 没有随机性，环境随机性为零。这简化了一部分理论，但不改变信用分配的难度。
- **奖励极度稀疏且只在终局**：整段话生成完才打一个分，中间所有 token 奖励为 0。这是个**长 horizon、稀疏奖励**的信用分配难题——哪个 token 该为最终的好/坏负责？（这正是后续 PRM 过程奖励、GRPO 等方法想改进的痛点，本课后面的章节会展开。）
- **动作空间巨大**：词表几万到十几万，序列上百步。

### 3.2 朴素目标和它的致命缺陷

最朴素的想法：直接最大化期望奖励。

```
max_θ  E_{x ~ D, y ~ π_θ(·|x)} [ r_φ(x, y) ]
```

**这个目标是错的，会直接导致灾难。** 错在哪？因为 `r_φ` 不是真实奖励，**它是一个在有限偏好数据上训出来的、有缺陷的代理（proxy）**。如果你不加任何约束地往死里优化这个代理，会发生：

- **Reward hacking（奖励黑客）**：策略发现 RM 的盲区，生成一些 RM 给高分、但人类觉得糟糕的输出。比如 RM 学到"长回答通常更好"，策略就学会**无脑灌水拉长**；RM 偏爱礼貌用语，策略就满篇 "Certainly! I'd be happy to help!" 的空话。
- **分布崩塌 / 退化**：策略可能塌缩到几个高分模板，**丧失多样性**，甚至输出语法崩坏但恰好骗过 RM 的乱码。
- **离开 RM 的可信域**：RM 只在"接近 SFT 分布"的回答上被训练过、打分才准。策略一旦漂远，RM 在那片区域是**外推（extrapolation）**，打分完全不可信，于是策略追着 RM 的幻觉越跑越偏。

这是 **Goodhart 定律**的教科书现身："当一个度量变成优化目标，它就不再是好度量。"`r_φ` 一旦成为被死命优化的对象，它和真实人类偏好的相关性就会在优化压力下断裂。

### 3.3 KL 惩罚：为什么是必需品（讲透三重作用）

修正方法是在目标里加一个**对参考策略的 KL 散度惩罚**，把策略"拴"在 SFT 模型 `π_ref = π_SFT` 附近：

```
max_θ  E_{x ~ D, y ~ π_θ(·|x)} [ r_φ(x, y) − β · KL( π_θ(·|x) ‖ π_ref(·|x) ) ]
```

其中 β 是 KL 系数（拉皮筋的松紧），KL 散度 `KL(π_θ ‖ π_ref) = E_{y~π_θ}[ log(π_θ(y|x) / π_ref(y|x)) ]` 衡量新策略相对参考策略偏离了多少。**这个 KL 项不是正则化的可选装饰，是 RLHF 能成立的结构性必需品**，它同时干三件事：

**作用一：防止 reward hacking 与分布漂移（最重要）。** RM 只在 `π_ref` 附近可信。KL 惩罚显式地让策略"不准跑太远离 `π_ref`"，于是策略被限制在 **RM 打分可信的那片区域里**做优化。皮筋越紧（β 越大），越不容易 hack，但能榨取的奖励也越少。本质是**在"代理奖励能信"的信任域内最大化奖励**，而不是在整个空间里盲目最大化一个会骗你的代理。

**作用二：保持语言流畅性与知识。** `π_ref`（=SFT 模型）已经是个流畅、有知识的语言模型。KL 把策略拴在它附近，等于**继承了 SFT 模型的语言能力**，防止 RL 把模型练成只会刷分的复读机、防止退化成语法崩坏的乱码。没有 KL，纯奖励优化几乎必然摧毁语言质量。

**作用三：保持探索 / 防止熵塌缩（数学上有精确刻画）。** 这一点最被低估，但有干净的数学。考虑带 KL 的目标在**单步、忽略序贯结构**时的最优解。固定 prompt，把它当成一个对分布 `π` 的优化问题（用变分法 / 拉格朗日乘子，约束 `Σ_y π(y)=1`）：

```
目标：max_π  Σ_y π(y) r(y) − β Σ_y π(y) log(π(y)/π_ref(y))
约束：Σ_y π(y) = 1

构造拉格朗日量 L = Σ π(y)[ r(y) − β log(π(y)/π_ref(y)) ] + ν(Σπ(y) − 1)
对 π(y) 求偏导并令为 0：
  r(y) − β( log(π(y)/π_ref(y)) + 1 ) + ν = 0
  log(π(y)/π_ref(y)) = ( r(y) + ν − β ) / β
  π(y) = π_ref(y) · exp( r(y)/β ) · exp((ν−β)/β)
归一化吸收常数项，得闭式最优策略：

  π*(y | x) = (1/Z(x)) · π_ref(y | x) · exp( r(x,y) / β )
            其中 Z(x) = Σ_y π_ref(y|x) · exp(r(x,y)/β)
```

**这个解极其重要，把它记牢——它是理解整个偏好优化家族（含后面 DPO）的钥匙。** 它告诉你：

- 带 KL 的 RLHF，**最优策略不是"把奖励顶满的确定性策略"，而是"参考策略经奖励指数加权重塑后的分布"**。它仍是个分布（有熵、有探索），不是退化的 one-hot。这就是"防熵塌缩"的精确含义。
- **β 是温度旋钮**：β → ∞，`exp(r/β) → 1`，π* → π_ref（完全不优化，只复制 SFT）；β → 0，指数项把全部质量堆到 `argmax_y r`（退化成纯奖励最大化，回到 3.2 的灾难）。β 在两者间插值，控制"奖励 vs 忠于参考"的权衡。
- 这个闭式解之所以漂亮，正因为 KL 项里的 `log π` 和奖励里没有 log——变分一求导，`log π` 项自然冒出来，解出来就是指数族。**KL 不是随便选的距离，它是唯一能给出这种解析最优策略的选择**，这也是为什么整个领域都用 KL 而不用别的散度。

> 把三重作用串成一句话记忆：**KL 惩罚 = 把优化关在"代理奖励可信、语言能力还在、分布没塌"的信任域里**。少了它，RLHF 不是"效果差一点"，而是"根本不 work，会崩"。

### 3.4 KL 怎么落到逐 token 的奖励里（工程实现的关键）

3.3 的目标是序列级的，但 PPO 需要**每个 token 一个奖励**才能做时序信用分配。实现上的标准做法是把 KL 惩罚**摊到每一步**，构造一个"塑形后的逐 token 奖励"：

```
对生成序列 y = (y₁, ..., y_T)：
  中间 token (t < T)：  R̃ₜ = − β · ( log π_θ(yₜ|sₜ) − log π_ref(yₜ|sₜ) )
  最后 token (t = T)：  R̃_T = r_φ(x, y) − β · ( log π_θ(y_T|s_T) − log π_ref(y_T|s_T) )
```

也就是：**每一步都根据"当前策略在这个 token 上比参考策略激进了多少"扣一点分（per-token KL 惩罚），只有最后一步额外加上 RM 给的整段奖励。** 把这串 `R̃ₜ` 当作环境奖励喂给 PPO 即可。

几个要点：
- 这里用的是 **KL 的单样本估计**：`log π_θ(yₜ) − log π_ref(yₜ)` 是 `log(π_θ/π_ref)` 在采样到的 yₜ 上的取值，其在 `y~π_θ` 下的期望正是逐 token KL。所以它是序列 KL 的无偏蒙特卡洛估计（实践中也有人用 `k3` 估计器降方差，**待核**具体收益）。
- 因为转移确定、奖励稀疏，**真正承载 RM 信号的只有最后一步**，中间步全靠 KL 惩罚和价值函数 bootstrap 来分配信用——这就是为什么价值头（critic）在这里很重要。
- β 在实践中常用 **adaptive KL controller**（自适应 KL 控制器，InstructGPT 用了）：设一个目标 KL，实测 KL 偏高就调大 β（拉紧），偏低就调小 β（放松）。手动固定 β 很难调，太大学不动、太小会 hack（见第 5 节坑）。

### 3.5 用 PPO 优化：落到 LLM 的特化

到这一步，问题已经化归成一个标准的"给定逐 token 奖励 `R̃ₜ`、最大化折扣回报"的 RL 问题，直接上 PPO（你已熟悉）。这里只讲 **LLM 特化的部分**，PPO 本体不重复：

- **Actor**：就是要训的语言模型 `π_θ`（从 SFT 初始化）。
- **Critic（价值头）**：在语言模型主干上**加一个标量价值头** `V_ψ(sₜ)`，估计状态价值。它和 RM 结构几乎一样（都是主干+标量头），但**职责完全不同**：RM 估"整段回答好不好"（在偏好数据上训、训完冻结），critic 估"从当前 token 状态出发未来的期望回报"（PPO 训练中和 actor 一起更新）。新手极易混淆这两个标量头。
- **Reference model**：`π_ref`（=SFT，冻结），只用来算 KL，不更新。
- **优势估计**：用 GAE（你已熟悉），基于逐 token 奖励 `R̃ₜ` 和 critic 的价值估计算每个 token 的优势 `Âₜ`。
- **PPO 裁剪目标**（token 级）：

```
L_PPO(θ) = − E_t [ min( ρₜ(θ)·Âₜ ,  clip(ρₜ(θ), 1−ε, 1+ε)·Âₜ ) ]
其中重要性比 ρₜ(θ) = π_θ(yₜ|sₜ) / π_θ_old(yₜ|sₜ)
```

`π_θ_old` 是采样时那一版策略（每个 PPO 迭代采一批样本，然后在这批上做几个 epoch 的小步更新，故需要 old/new 之分）。裁剪 ε（典型 0.2，**待核**具体值随实现而异）限制单步更新幅度，防止策略一步迈崩——这在 LLM 上尤其要命，因为崩了的语言模型几乎救不回来。

**注意 RLHF 里有两个 KL，别搞混**：(1) 目标里对 `π_ref` 的 **behavioral KL**（β 那个，3.3 的主角，拴住别漂离 SFT）；(2) PPO 裁剪隐含限制的对 `π_θ_old` 的 **trust-region KL**（防单步更新过大，PPO 本体的机制）。前者是 RLHF 特有的对齐约束，后者是 PPO 通用的稳定性机制。

### 3.6 代码：RLHF-PPO 训练循环骨架

下面是体现完整机制的骨架（生成 → 打分 → 加 KL → 算优势 → PPO 更新），省略了分布式、采样缓存等工程噪声，保留所有关键数学步骤：

```python
# ============ RLHF 第三阶段：PPO 训练循环骨架 ============
# 四个模型：policy(actor,训) / value(critic,训) / reward_model(冻结) / ref_model(冻结)

def rlhf_ppo_step(prompts, policy, value, reward_model, ref_model,
                  beta=0.05, clip_eps=0.2, gamma=1.0, lam=0.95, ppo_epochs=4):

    # ---- 1. 生成 (rollout)：用当前策略对每个 prompt 采样回答 ----
    with torch.no_grad():
        responses = policy.generate(prompts)            # 自回归采样出 token 序列
        seq = concat(prompts, responses)                # 完整 (x, y)

        # 记录采样时的 log prob（即 π_θ_old），PPO 重要性采样要用
        logp_old = policy.log_probs(seq)                # 每个 response token 的 log π_old
        logp_ref = ref_model.log_probs(seq)             # 参考模型 log π_ref（算 KL 用）
        values   = value(seq)                           # critic 对每个 token 状态的价值估计

        # RM 给整段回答打一个标量分（只在序列末尾）
        scores   = reward_model(seq)                    # shape: (batch,)

    # ---- 2. 构造逐 token 奖励：per-token KL 惩罚 + 末尾 RM 分 ----
    #  R̃ₜ = -β·(log π_θ_old - log π_ref)  ；末尾再 + RM 分
    kl_per_token = logp_old - logp_ref                  # 单样本 KL 估计 (response 位置)
    rewards = -beta * kl_per_token                      # 每步都扣 KL 惩罚
    rewards[:, -1] += scores                            # 最后一个 token 加上 RM 整段分

    # ---- 3. GAE 计算优势 Âₜ 和回报 (你已熟悉的标准 GAE) ----
    advantages, returns = compute_gae(rewards, values, gamma, lam)
    advantages = (advantages - advantages.mean()) / (advantages.std() + 1e-8)  # 标准化降方差

    # ---- 4. PPO 多 epoch 更新（在同一批 rollout 上做几次小步更新）----
    for _ in range(ppo_epochs):
        logp_new = policy.log_probs(seq)                # 当前(更新中)策略的 log π_θ
        ratio = torch.exp(logp_new - logp_old)          # 重要性比 ρₜ = π_θ / π_θ_old

        # 裁剪式策略损失
        surr1 = ratio * advantages
        surr2 = torch.clamp(ratio, 1 - clip_eps, 1 + clip_eps) * advantages
        policy_loss = -torch.min(surr1, surr2).mean()

        # critic 回归损失（拟合 GAE 回报）
        value_loss = F.mse_loss(value(seq), returns)

        # 可选：策略熵奖励，进一步防熵塌缩（系数小）
        loss = policy_loss + 0.5 * value_loss  # - ent_coef * entropy

        loss.backward(); optimizer.step(); optimizer.zero_grad()

    # 监控：实测 KL（防漂移）、平均奖励、价值损失——RLHF 三个必看仪表盘
    # 注意用更新后策略 logp_new 对 ref 算 KL：kl_per_token 是采样时刻 π_θ_old 的 KL（更新前），
    # 经过多个 PPO epoch 后会低估真实漂移，漂移监控要看当前策略
    approx_kl = (logp_new - logp_ref).mean().item()
    return {"reward": scores.mean().item(), "kl": approx_kl}
```

读这段代码要抓住四个"机制锚点"：(1) **逐 token 奖励 = KL 惩罚 + 末尾 RM 分**，这是 3.4 的落地；(2) **ratio = π_θ/π_θ_old**，PPO 的重要性采样，让一批 rollout 能复用几个 epoch，省采样成本；(3) **优势标准化**，降方差的标准手段；(4) **三个仪表盘（reward / KL / value loss）**，下一节讲为什么必须盯着 KL。

## 4. 量化分析：方差、KL 预算、与"奖励 vs KL"的帕累托前沿

### 4.1 为什么 RLHF 的 PPO 方差天然大

策略梯度的方差随 horizon 增长、随奖励稀疏程度恶化。LLM-RL 同时占了两个最坏因素：**horizon 长（上百 token）+ 奖励稀疏（只在终局）**。回忆策略梯度方差的来源——回报的随机性通过 `∇log π · R` 注入，序列越长、累积随机性越大。RLHF 的三道减方差防线，对应代码里三个地方：

- **价值基线（critic baseline）**：用 `Âₜ = Rₜ − V(sₜ)` 替代裸回报，扣掉与动作无关的基线，不改变期望（无偏）但大幅降方差。这是 critic 存在的核心理由之一。
- **GAE 的 λ**：在"一步 TD（低方差高偏差）"和"蒙特卡洛回报（低偏差高方差）"间用 λ 指数加权插值，找偏差-方差甜点（你已熟悉）。
- **优势标准化**：把一批 `Â` 归一到零均值单位方差，稳定梯度尺度。

### 4.2 KL 预算：把"漂移"量化成可监控的数

实测 KL（代码里的 `approx_kl`）是 RLHF **最重要的单一健康指标**，没有之一。它直接量化"策略漂离 SFT 多远"。经验规律（数值因任务而异，**待核**精确阈值）：

- KL 缓慢上升、reward 同步上升 → **健康**，正在学。
- KL 飙升、reward 也飙升 → **大概率在 reward hacking**：策略找到了 RM 盲区，正快速漂出可信域。这时 reward 涨是假象，人评会变差。**必须警惕"reward 涨得太爽"**。
- KL 上升、reward 不动或下降 → 训练发散/不稳，该回滚或调小学习率/调大 β。

一个有用的心智模型：**RLHF 是在花 KL 预算买 reward**。你只有有限的 KL 预算（漂太远 RM 就不可信了），目标是用这点预算买到尽可能多的"真实质量提升"。**β 决定汇率，RM 质量决定买到的是真货还是假货。**

### 4.3 奖励-KL 帕累托前沿（理解对齐税的框架）

把训练过程中的 (实测 KL, RM reward) 描点，会得到一条向上的曲线——KL 越大、能榨的 reward 越高，但边际递减。这条**奖励-KL 帕累托前沿**是评估 RLHF 的标准视角（业界常用，源于对过优化的实证研究，Gao et al. 2023 *Scaling Laws for Reward Model Overoptimization* 是代表，**待核**具体函数形式）。

关键现象——**用 RM 分（proxy reward）画的曲线一路上升，但用真实人评（gold reward）画的曲线先升后降**：KL 小时两者都涨（学到真东西）；KL 超过某拐点后，proxy 还在涨、gold 开始跌——**这就是 RM 过优化（overoptimization）的直接观测，是 Goodhart 定律的曲线形态**。最优 KL 大致落在 gold 曲线的峰值附近。这条曲线也解释了为什么 RLHF 不能"练到 reward 不涨为止"——那几乎注定已经过优化了。

### 4.4 复杂度账：为什么 RLHF 贵

PPO-RLHF 一步要同时持有**四个模型**：actor（训，含梯度+优化器态）、critic（训，含梯度+优化器态）、reward model（冻结，前向）、reference model（冻结，前向）。其中 actor 和 critic 各自规模与 SFT 模型相当。

- **显存**：约 = 2 个训练态模型（actor+critic，每个按"参数+梯度+Adam 双矩+激活"，回忆 MLSys 课每参数约 16 字节那本账）+ 2 个推理态模型（RM+ref，约每参数 2 字节 fp16）。**这是 RLHF 显存爆炸的根因**，也是 GRPO 想干掉 critic、DPO 想干掉 RL 全程的直接动机（后续章节）。
- **计算**：每步要做 1 次自回归生成（最贵，逐 token 解码，无法并行）+ 4 个模型的若干次前向 + actor/critic 的反向。生成是吞吐瓶颈。
- **不稳定性**：四个模型耦合 + on-policy + 高方差 + 对超参敏感，使 RLHF-PPO 成为出了名难复现、难调的训练任务。**这是后续离线方法（DPO 家族）最大的卖点来源。**

## 5. 设计权衡与常见坑

**1. Reward hacking（最致命，必然发生，只能缓解）。** 只要 RM 是学出来的代理，策略就会找它的空子。表现：无脑拉长、堆礼貌套话、复读高分模板、生成"看起来对但其实空"的内容。缓解手段叠加使用：KL 惩罚（核心）、盯 KL 预算别超拐点、定期用新策略采样补充偏好数据重训 RM（让 RM 跟上策略的新花招）、人评抽查别只信 RM 分。**记住：reward 涨得异常顺滑往往是 hacking 的信号，不是好消息。**

**2. RM 过优化 / Goodhart（第 4.3 节）。** proxy reward 涨 ≠ 真实质量涨。务必维护一个 gold（人评或更强模型评）信号画 KL-gold 曲线，在 gold 峰值附近停。别拿 RM 分当最终 KPI。

**3. 对齐税（alignment tax）。** RLHF 让模型更对齐（更有用/无害），但常以**某些能力指标下降**为代价——比如在一些标准 benchmark（尤其需要"原始"分布能力的任务）上分数掉一点。直觉：KL 把策略拉离了纯预训练分布，而那个分布在某些任务上恰好更优。缓解：在 RL 阶段混入少量预训练/SFT 数据的损失（pretraining mix，InstructGPT 的 PPO-ptx 变体就是干这个的），或调小 β 控制偏离。**对齐税是真实存在的权衡，不是 bug，要主动测量并接受/折中。**

**4. KL 系数 β 调参（高频翻车点）。** β 太大 → 策略几乎不动，学不到东西（皮筋太紧）；β 太小 → 快速 reward hacking、漂出可信域（皮筋太松）。固定 β 极难调，强烈建议用 adaptive KL controller（设目标 KL 自动调 β）。**新手常见错误：β 设 0 或极小图省事，然后看着 reward 一路飙升以为大获成功，实际模型已经废了。**

**5. 两个标量头混淆（reward model vs value head）。** RM 估"整段好不好"（偏好数据训、冻结用），critic value 估"从此状态起的未来回报"（PPO 中训）。结构像、职责天差地别。代码里它们是不同的对象，别共享、别搞反。

**6. 价值头初始化与冷启动。** critic 一开始价值估计很烂，会给出垃圾优势，早期训练抖动大。常见做法：critic 也从 SFT/RM 初始化、给若干 warmup 步、或 value loss 加 clip。critic 不稳是 PPO-RLHF 早期发散的常见诱因。

**7. 生成与训练的 KV/数值不一致。** 采样（generate，常 fp16/bf16、可能用专门推理引擎）和训练前向（算 logp）若在精度、KV cache、padding 处理上不一致，会让 `logp_old` 与实际采样分布对不上，污染重要性比 ρ。**这是工程上极隐蔽的坑**，会让 PPO 莫名其妙学不动或发散。务必保证 `logp_old` 来自与采样一致的计算。

**8. 偏好数据的位置/长度偏置。** 标注者倾向选更长、排版更漂亮的回答，RM 会把这些表面特征学成"好"的信号，于是策略学会刷长度/排版。构造偏好数据和训 RM 时要警惕这类 spurious correlation（可做长度去偏，**待核**具体方法）。

## 6. 动手练习

**练习 1（推导题）：从 Bradley-Terry 完整推出 RM 损失，并推它的梯度。**
(a) 从 `P(y_w ≻ y_l|x) = exp(r_w)/(exp(r_w)+exp(r_l))` 出发，证明它等于 `σ(r_w − r_l)`，写出每一步代数化简。(b) 写出数据集上的负对数似然 `L_RM`。(c) 求 `∂L_RM/∂φ`，并解释其中权重 `(1−σ(Δ))` 为什么实现了"难例自动聚焦"。
> 提示：(a) 分子分母同除 `exp(r_w)`。(c) 用 `d log σ(Δ)/dΔ = 1 − σ(Δ)` 和 `σ'=σ(1−σ)`；分析 Δ→+∞ 和 Δ→0 两端权重的极限，对应"已学会/还没学会"。

**练习 2（推导题）：推出带 KL 惩罚的最优策略闭式解，并讨论 β 的两个极限。**
对单 prompt、忽略序贯结构的目标 `max_π Σ_y π(y)r(y) − β·KL(π‖π_ref)`（约束 `Σπ=1`），用拉格朗日乘子法推出 `π*(y) ∝ π_ref(y)·exp(r(y)/β)`，写出归一化常数 Z。然后分别取 β→∞ 和 β→0，说明 π* 退化成什么、对应 RLHF 的什么行为。
> 提示：对 π(y) 求偏导时记得 KL 项里 `d[π log(π/π_ref)]/dπ = log(π/π_ref) + 1`。β→∞ 时 `exp(r/β)→1`；β→0 时指数把质量全堆到 argmax r。把结论和第 3.3 节、第 4.3 节的过优化现象联系起来。

**练习 3（分析题）：解释 KL 项的三重作用，并设计一个实验观测 reward hacking。**
(a) 用自己的话说清 KL 惩罚的三个作用（防漂移/保流畅/防熵塌缩），并指出哪个有 2 中的闭式解作为数学支撑。(b) 设计一个实验：固定 RM 和 SFT，扫一组 β（含 β=0 或极小），训练并记录 (实测 KL, RM reward, 人评/gold reward) 三条曲线，画出 KL-reward 帕累托前沿。预期看到什么？过优化拐点大致在哪？
> 提示：(b) 预期 RM-reward 随 KL 单调升、gold-reward 先升后降；β 越小 KL 漂得越远越易 hacking。把"reward 涨但 gold 跌"标成 Goodhart 区。

**练习 4（编码题）：实现 per-token 奖励塑形并验证 KL 估计无偏。**
基于第 3.6 节骨架，实现 `build_token_rewards(logp_policy, logp_ref, rm_score, beta)`：中间 token 给 `−β·(logp_policy − logp_ref)`，末尾再加 `rm_score`。然后写一个数值验证：对一个小词表的玩具策略，蒙特卡洛采样大量序列，验证 `E[Σₜ(logp_policy − logp_ref)]` 收敛到解析 KL `Σ_y π_θ(y)log(π_θ(y)/π_ref(y))`。
> 提示：玩具设定取词表大小 5、序列长 3，π_θ 和 π_ref 都用固定 softmax 分布即可解析算 KL；采样 10⁴~10⁵ 条，比较蒙特卡洛均值与解析值，验证逐 token log-ratio 之和是序列 KL 的无偏估计。

## 7. 源码 / 论文导读

- **InstructGPT（Ouyang et al., 2022，《Training language models to follow instructions with human feedback》）**：**本章的母本，必读**。重点读：三阶段流程图（Fig 2）、RM 损失（成对 logistic）、PPO 目标里的 KL 项与 PPO-ptx（pretraining mix）变体、对齐税的实证。读懂这篇，本章的工程决策就都有出处了。
- **RLHF 的偏好学习源头**：Christiano et al., 2017《Deep Reinforcement Learning from Human Preferences》——RLHF 思想的开创工作（先在控制任务上），Bradley-Terry 偏好建模 + 学习型奖励的范式从这里来。Stiennon et al., 2020《Learning to summarize from human feedback》是把它用到语言生成（摘要）的关键过渡。
- **PPO 本体**：Schulman et al., 2017《Proximal Policy Optimization》+ GAE 论文 Schulman et al., 2015。本章只讲了 LLM 特化，裁剪目标和 GAE 的推导回到 PPO 章节和这两篇。
- **RM 过优化 / Goodhart**：Gao et al., 2023《Scaling Laws for Reward Model Overoptimization》——**理解第 4.3 节帕累托前沿、proxy vs gold reward 的权威实证**，给出了过优化随 KL 和 RM 规模变化的标度规律。
- **Bradley-Terry 原始模型**：Bradley & Terry, 1952（统计学经典），了解成对比较模型的统计学根源即可，不必深读原文。
- **开源实现导读**：
  - **TRL（Hugging Face transformers 强化学习库）**：读 `PPOTrainer`（旧版）/ `PPOv2Trainer`，看四个模型（policy/value/reward/ref）如何组织、per-token KL 奖励如何构造、generate→score→compute_advantages→ppo_step 的循环——**和本章 3.6 骨架一一对应，最适合对照阅读**。`RewardTrainer` 对应第 2 节 RM 损失。
  - **OpenRLHF / verl**：面向大规模、分布式的 RLHF 框架，看它们怎么把四模型、生成引擎（如 vLLM）、训练后端拆开做工程优化——理解第 4.4 节的复杂度账如何在真实系统里被拆解。（具体模块名随版本变化，**待核**。）
  - **CleanRL** 的 PPO（`ppo.py`）：虽是经典控制环境的 PPO，但裁剪目标、GAE、优势标准化的最小可读实现，建议先在这里把 PPO 跑通再看 LLM 版。

## 8. 小结与承上启下

这一章我们把"人类偏好"一路优化进了大模型，每一步都落到了数学：

- **母题**：有用/诚实/无害写不出损失函数，但人能比较。RLHF 用三步把"能比较"撬成"能优化"——**SFT 给好起点 → RM 把偏好变标量 → PPO 在 KL 信任域里优化**。
- **RM 的地基是 Bradley-Terry**：成对偏好概率 = `σ(奖励差)`，损失 = `−E[log σ(r_w − r_l)]`（成对 logistic 回归），梯度自带难例聚焦。奖励的绝对尺度不可辨识，只有差值有意义。
- **KL 惩罚是结构性必需品，不是可选正则**：它把优化关在"代理奖励可信 + 语言能力还在 + 分布没塌"的信任域里，三重作用都有数学支撑——尤其那个闭式最优解 `π* ∝ π_ref·exp(r/β)`，是整个偏好优化家族的钥匙。
- **量化**：RLHF 的 PPO 天然高方差（长 horizon + 稀疏奖励），靠 critic 基线 / GAE / 优势标准化压；KL 是最重要的健康仪表盘；reward-KL 帕累托前沿上 proxy 升而 gold 先升后降，就是 RM 过优化（Goodhart）的现身。代价是四模型耦合、显存与不稳定性的高昂账单。

**承上**：本章的 PPO 裁剪目标、GAE、优势-价值基线全部来自前面的策略梯度与 PPO 章节——RLHF 没有发明新的 RL 算法，它发明的是**奖励从哪来（学习型 RM）和怎么不被代理奖励带偏（KL 信任域）**。MLSys 课里"每参数约 16 字节"那本显存账，正是第 4.4 节四模型显存爆炸的来源。

**启下**：这章也暴露了 RLHF-PPO 的三个痛点，每一个都开出了后续章节的主线——
- **太重太不稳**（四模型、对超参极敏感、难复现）→ 下一条线 **DPO / 离线偏好优化家族**：它直接用第 3.3 节那个闭式解 `π* ∝ π_ref·exp(r/β)` 做代数反解，把"训 RM + 跑 PPO"两步合并成**一个监督式损失**，干掉在线 RL 全程。你会看到本章推的最优策略解，在 DPO 里变成损失函数本身。
- **奖励稀疏、信用分配难**（只在终局打一个分）→ **过程奖励模型 PRM / 可验证奖励 RLVR**：给推理的每一步打分，或在数学/代码这类可验证领域直接用验证器当奖励，绕开学习型 RM 的过优化。
- **critic 太贵**（和策略一样大）→ **GRPO**：用组内奖励均值当基线，彻底删掉 critic。

换句话说，本章是整个"RL 优化大模型"的**第一座、也是最完整的一座桥**；后面所有方法，都是在拆这座桥的某一块石头——要么换掉奖励来源，要么换掉优化方式，要么砍掉某个模型。理解了 RLHF 的完整三阶段和它的三个痛点，你就拿到了读懂后续所有对齐与推理训练方法的坐标系。
