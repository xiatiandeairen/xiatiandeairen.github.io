> 这一章把你从"知道两阶段检测器(Faster R-CNN)怎么工作"带到"能在白板上推出 Focal Loss 的梯度、说清单阶段为什么会被背景淹没、能徒手实现 RetinaNet 的分类损失、能讲透 DETR 用匈牙利匹配做集合预测从而干掉 NMS 的全部机制"。检测从这一章开始进入"快"和"无后处理"两个方向,前沿论文的多数招式都从这里发散。

## 一句话引言

单阶段检测把"提议 + 分类"两步压成一步,代价是**正负样本极度不均衡**;Focal Loss 用一个调制因子 `(1−pₜ)ᵞ` 让损失自动聚焦难样本,解决了这个均衡问题;DETR 则更激进,把检测重新定义为**集合预测**,用匈牙利匹配让每个预测和每个真值一一对应,从而连 anchor 和 NMS 都不要了。本章把这三件事的动机、数学和实现全部推开。

## 1. 动机:两阶段的"慢"和单阶段的"乱"

上一章你已经掌握了 Faster R-CNN 的范式:**RPN 先产生几百到几千个候选框(region proposal),再由第二阶段对每个候选框做精细分类和回归**。这套"先粗筛再精修"的设计精度高,但有两个结构性代价。

**代价一:速度。** 两阶段意味着特征要被"用两次"。RPN 在特征图上滑窗给出 proposal,然后每个 proposal 要做一次 RoIAlign 把对应区域的特征抠出来、再过一遍检测头。proposal 数量动辄上千,RoI 级别的计算无法完全共享,这条串行链路天然慢。在需要实时(≥30 FPS)的场景——自动驾驶、视频分析、移动端——两阶段难以满足延迟预算。

**代价二:复杂。** RPN 本身就是一个小检测器,有自己的 anchor、自己的正负样本采样、自己的损失。整个系统是"检测器套检测器",训练流程和超参都多。

单阶段(one-stage / single-shot)检测器的破局点很直接:**去掉 proposal 这一步,直接在特征图上,对每个位置(grid cell / anchor)同时预测"这里有没有物体、是什么类、框在哪"**。一次前向就出结果,没有 RoI 级别的串行计算,因此快。

但这个"直接"带来一个新的、致命的问题——**类别极度不均衡(class imbalance)**。两阶段里,RPN 已经帮你把绝大多数背景框过滤掉了,送进第二阶段的几百个 proposal 里前景比例不算太低(还会做正负采样,比如 1:3)。单阶段没有这个过滤器:你在密集的特征图上铺 anchor,一张图可能有 **10⁴ ~ 10⁵ 个候选位置,其中真正含物体的可能只有几十个**,正负比能到 1:1000 甚至更悬殊。这就是单阶段"乱"的根源——海量背景样本会在损失里淹没稀少的前景。

本章的逻辑主线就是:**先看单阶段怎么把检测变成回归(YOLO / SSD)→ 直面不均衡这个拦路虎(Focal Loss / RetinaNet)→ 进一步去掉 anchor 超参(FCOS / CenterNet)→ 最后看 DETR 如何用集合预测重写整个范式。**

## 2. 单阶段范式:把检测变成网格回归

### 2.1 YOLO:每个网格格子预测框和类

YOLO(You Only Look Once,Redmon et al., 2016,CVPR)的核心洞察:**把整张图划成 S×S 网格,让每个格子负责预测"中心落在该格子里的物体"**。

```
输入图像 ──CNN──▶  S×S 特征网格
每个格子预测:
  B 个边界框,每个框 = (x, y, w, h, confidence)
  C 个类别条件概率 P(class | object)
输出张量形状: S × S × (B·5 + C)
```

逐符号拆开(以 YOLOv1 为例):

- `(x, y)`:框中心相对**该格子左上角**的偏移,归一化到 [0, 1]。注意是相对格子,不是相对全图——这把回归目标限制在一个格子内,数值范围小、好学。
- `(w, h)`:框宽高相对**全图**的比例,归一化到 [0, 1]。
- `confidence`:定义为 `Pr(object) · IoU(pred, truth)`。它同时编码了"这里有没有物体"和"框预测得准不准"。训练时,含物体的格子其目标 confidence 是预测框与真值的 IoU,不含物体的格子目标是 0。
- `C` 个类别概率是**条件概率** `Pr(classᵢ | object)`,整个格子共享(YOLOv1 一个格子只预测一套类别概率,不是每个框一套)。

测试时某框对某类的得分 = `Pr(classᵢ | object) · Pr(object) · IoU = Pr(classᵢ) · IoU`,直接作为该类的检测置信度。

**YOLOv1 的损失**是一个纯 sum-of-squared-error(平方损失),但做了几处关键工程修正:

```
L = λ_coord · Σ_{含物体格子} [ (x−x̂)² + (y−ŷ)²  + (√w−√ŵ)² + (√h−√ĥ)² ]      # 定位
  +           Σ_{含物体格子} (C−Ĉ)²                                          # 含物体的置信度
  + λ_noobj · Σ_{不含物体格子} (C−Ĉ)²                                        # 不含物体的置信度
  +           Σ_{含物体格子} Σ_{classes} (p−p̂)²                              # 分类
```

这里有两个**直击不均衡的早期补丁**,值得记住:

1. `λ_coord`(论文取 5):定位误差权重调高。因为大量格子是背景,定位项只在少数格子上有,得加权放大,否则被分类/置信度项压过。
2. `λ_noobj`(论文取 0.5):背景格子的置信度损失权重调低。这是**YOLO 对类别不均衡的朴素回应**——直接给海量背景样本的损失乘一个小系数。这其实是后面 Focal Loss 的"原始版":都是想压低背景样本的损失贡献,只不过 λ_noobj 是一个**对所有背景一视同仁的固定常数**,而 Focal Loss 会做到**按难易自适应**。

3. `√w, √h`:对宽高开根号再算平方差。**动机**:同样 5 像素的误差,对一个 20 像素的小框是致命的,对一个 300 像素的大框无所谓。直接对 w 算 (w−ŵ)² 会让大框的误差主导梯度。开根号后,小框的相对误差被放大——`d(√w) = dw/(2√w)`,w 越小,同样 dw 对应的 √w 变化越大,梯度越大。这是一个"用非线性变换重新加权误差尺度"的经典技巧。

**YOLO 的代价**:每个格子只预测固定数量的框,且类别在格子内共享。如果两个物体中心落在同一格子,或一群小物体密集排列(鸟群),YOLOv1 会漏检。这是"网格"这个空间离散化的内在限制。后续版本(YOLOv2 引入 anchor、v3 引入多尺度、v4/v5/v8 大量工程优化)逐步缓解,但"密集网格回归"的内核没变。

### 2.2 SSD:多尺度特征图上预测

SSD(Single Shot MultiBox Detector,Liu et al., 2016,ECCV)与 YOLO 同期,核心改进是**多尺度**:不只在最后一层特征图预测,而是在 backbone 的**多个不同分辨率的特征层上都铺 anchor(论文叫 default box)做预测**。

```
backbone (VGG) ──▶ Conv4_3 (38×38) ──┐
                ──▶ Conv7   (19×19) ──┤
                ──▶ Conv8_2 (10×10) ──┼──▶ 每层都接 3×3 卷积预测头
                ──▶ Conv9_2 (5×5)   ──┤    输出 (类别 + 4 偏移) × k 个 default box
                ──▶ Conv10_2(3×3)   ──┤
                ──▶ Conv11_2(1×1)   ──┘
```

**为什么多尺度有效——感受野与物体尺度的匹配。** 浅层特征图分辨率高、感受野小、空间细节多,适合检测**小物体**;深层特征图分辨率低、感受野大、语义强,适合检测**大物体**。SSD 让不同层负责不同尺度,正是让"特征的感受野"和"要检测的物体尺度"对齐。具体地,SSD 给第 k 个特征层设计的 default box 基准尺度按线性插值取:

```
sₖ = s_min + (s_max − s_min)/(m−1) · (k−1),  k = 1..m
```

s_min ≈ 0.2、s_max ≈ 0.9(相对图像尺寸),m 是预测层数。浅层 sₖ 小(检测小物体),深层 sₖ 大。每个尺度再配几种长宽比 {1, 2, 3, 1/2, 1/3},覆盖不同形状。

SSD 同样直面不均衡,它的回应是 **hard negative mining(难负样本挖掘)**:对所有负样本(背景 default box)按分类损失从大到小排序,只取损失最大的那批,使**负:正 ≈ 3:1**,其余背景直接丢弃不计损失。这是另一种压制背景的思路——不是降权,而是**直接采样、只留难的**。记住这点,它和 Focal Loss 是同一问题的两条不同技术路线。

### 2.3 单阶段 vs 两阶段:速度-精度权衡的本质

把两条路线放在一起对比:

| 维度 | 两阶段(Faster R-CNN) | 单阶段(YOLO / SSD / RetinaNet) |
|---|---|---|
| proposal 步骤 | 有(RPN) | 无,直接密集预测 |
| RoI 级计算 | 有(RoIAlign + 每框过头) | 无,全卷积一次出 |
| 速度 | 慢(串行、RoI 数多) | 快(单次前向) |
| 正负不均衡 | 被 RPN + 采样缓解 | 极严重(1:1000+),需专门处理 |
| 历史精度 | 高 | 早期略低,RetinaNet 后追平/反超 |
| 后处理 | NMS | NMS(DETR 之前都要) |

**核心权衡的数学直觉**:两阶段用 RPN 把候选数从 ~10⁵ 降到 ~10³,相当于在送进精修头之前先做了一次"难度过滤",所以第二阶段面对的样本分布相对均衡、精度好做;但这次过滤本身是串行计算,是速度瓶颈。单阶段省掉过滤、全卷积并行,所以快;但面对的是原始的、极度不均衡的 ~10⁵ 个样本,**精度上不去的根本原因就是这个不均衡**——这正是 Focal Loss 要解决的问题。Lin 等人(2017)的关键论断是:**只要解决了不均衡,单阶段就能在保持速度的同时达到两阶段的精度。** 下一节我们把这件事推透。

## 3. 类别不均衡与 Focal Loss

### 3.1 问题:海量易分背景淹没损失

设二分类(前景 vs 背景)交叉熵。对一个样本,设模型预测它"是前景"的概率为 p ∈ (0, 1),真值标签 y ∈ {0, 1}。标准交叉熵:

```
CE(p, y) = −log(p)      if y = 1   (正样本/前景)
         = −log(1−p)    if y = 0   (负样本/背景)
```

为了把两种情况写成一个式子,定义 pₜ("对正确类别预测的概率"):

```
pₜ = p        if y = 1
   = 1−p      if y = 0
```

于是 `CE(p, y) = CE(pₜ) = −log(pₜ)`。pₜ 越接近 1,说明分类越正确、越"容易",损失越小;pₜ 越接近 0,样本越"难"。

**关键观察(Focal Loss 论文的核心实验事实)**:即便一个背景样本被分得很好(比如 p = 0.01,即模型很确信它是背景,pₜ = 1−p = 0.99),它的交叉熵 `−log(0.99) ≈ 0.01` 也不是 0。单独看微不足道,但**当你有 10⁵ 个这样的易分背景**,它们的损失之和 ≈ 10⁵ × 0.01 = 1000,而你只有几十个前景样本,前景总损失可能只有几十。**梯度被海量易分背景的微小损失之和主导,前景样本的信号被淹没。** 模型最终学成"对什么都预测背景"也能让总损失很低。

这就是单阶段精度上不去的数学本质。注意:问题不只是"正负数量不均衡",更是"**易分样本(无论正负,但主要是背景)的损失虽小但数量巨大,累加后压过难样本**"。所以解法不能只是简单地按类别数量加权(那只解决数量,不解决"易分"),而要**按难易程度降权**。

### 3.2 推导 Focal Loss

我们想要一个损失,满足:**易分样本(pₜ → 1)的损失被大幅压低,难分样本(pₜ → 0)的损失基本不变。** 最自然的做法——在交叉熵前乘一个随 pₜ 单调递减、且在 pₜ→1 时趋于 0 的"调制因子(modulating factor)"。Lin 等人选了最简形式:

```
调制因子 = (1 − pₜ)ᵞ ,   γ ≥ 0 (focusing parameter,聚焦参数)

Focal Loss:  FL(pₜ) = −(1 − pₜ)ᵞ · log(pₜ)
```

逐项验证它满足设计目标:

```
当 pₜ → 1 (易分):  (1−pₜ)ᵞ → 0 ,  整体损失 → 0       ✅ 易分样本被强力压低
当 pₜ → 0 (难分):  (1−pₜ)ᵞ → 1 ,  FL → −log(pₜ) = CE  ✅ 难分样本损失近似不变
当 γ = 0:          (1−pₜ)⁰ = 1 , FL 退化为标准 CE      ✅ γ 控制"聚焦强度"
```

**γ 的几何意义——把易分样本"按"下去多狠。** 取一个易分样本 pₜ = 0.9:

```
γ = 0:  调制因子 = (1−0.9)⁰ = 1         → 损失不变
γ = 1:  调制因子 = (0.1)¹   = 0.1       → 损失降到 1/10
γ = 2:  调制因子 = (0.1)²   = 0.01      → 损失降到 1/100
γ = 5:  调制因子 = (0.1)⁵   = 1e-5      → 损失几乎归零
```

而对一个难样本 pₜ = 0.1:

```
γ = 2:  调制因子 = (1−0.1)² = 0.9²= 0.81 → 损失只降到 0.81 倍,基本保留
```

对比:**易分样本(pₜ=0.9)在 γ=2 时损失被压到 1/100,难样本(pₜ=0.1)只被压到 0.81 倍。** 两者损失的相对差距被拉大了约 80 倍。模型的梯度注意力因此自动转向难样本。论文实验发现 **γ = 2 是综合最优**(待核:这是 COCO 上的经验值,不同数据集可能略有差异)。

实际用的是 **α-balanced Focal Loss**,再叠一个类别权重 α 处理"正负数量"这一维:

```
FL(pₜ) = −αₜ · (1 − pₜ)ᵞ · log(pₜ)

其中  αₜ = α      if y = 1
        = 1−α    if y = 0
```

α ∈ [0, 1],给稀少的前景一个略大的权重(论文 α = 0.25,配合 γ=2)。**注意一个反直觉的点**:α=0.25 是给前景 0.25、背景 0.75,看起来反而压低了前景?这是因为 γ 这一项已经把背景压得极狠,如果 α 再给前景大权重会过度补偿,所以 α 和 γ 要联合调,最优组合下 α 反而偏小。这正是上面 know 提到的"语义负载字段"陷阱——别脱离 γ 单独理解 α。

### 3.3 Focal Loss 的梯度(为什么它"自动聚焦")

光看损失值还不够,**真正驱动学习的是梯度**。我们对模型输出的 logit(记为 x,p = σ(x) = sigmoid)求导,看 Focal Loss 的梯度如何随难易变化。先给出标准交叉熵对 logit 的梯度(经典结果):

```
对正样本 (y=1):  d CE/dx = σ(x) − 1 = p − 1 = −(1 − p) = −(1 − pₜ)
对负样本 (y=0):  d CE/dx = σ(x) − 0 = p     =  1 − pₜ
统一写:         |d CE/dx| = (1 − pₜ)
```

**交叉熵梯度的幅度就是 (1−pₜ)**。这已经有一点"难样本梯度大"的味道(pₜ 小则 |梯度| 大),但它是**线性**的、太弱,压不住数量。下面推 Focal Loss 的梯度。

以正样本为例,FL = −(1−p)ᵞ log(p),对 x 求导(链式法则,记 σ′ = σ(1−σ) = p(1−p)):

```
设 f(p) = −(1−p)ᵞ · log(p)
df/dp = −[ d/dp (1−p)ᵞ · log p ]
      = −[ −γ(1−p)^{γ−1} · log p  +  (1−p)ᵞ · (1/p) ]
      =  γ(1−p)^{γ−1} log p  −  (1−p)ᵞ / p

dFL/dx = df/dp · dp/dx = df/dp · p(1−p)
       = γ(1−p)^{γ−1} log p · p(1−p)  −  (1−p)ᵞ/p · p(1−p)
       = γ p (1−p)ᵞ log p  −  (1−p)^{γ+1}
       = (1−p)ᵞ · [ γ p log p − (1−p) ]
```

得到正样本梯度:

```
dFL/dx = (1−p)ᵞ · [ γ·p·log(p) − (1−p) ]
```

**怎么读这个式子。** 把它和交叉熵梯度 `−(1−p)` 对比,Focal Loss 在前面多乘了一个因子 `(1−p)ᵞ`,并在方括号里加了一项 `γ·p·log(p)`。核心是前面那个 `(1−p)ᵞ`:

```
易分正样本 p → 1:  (1−p)ᵞ → 0,  整个梯度 → 0     ⟹ 几乎不更新(已经学好了,别浪费梯度)
难分正样本 p → 0:  (1−p)ᵞ → 1,  方括号 → [0 − 1] = −1,  梯度 → −1  ⟹ 梯度幅度接近 CE 的最大值,全力更新
```

**这就是"自动聚焦"的数学机制**:Focal Loss 通过 `(1−pₜ)ᵞ` 这个因子,把易分样本的梯度幅度按 γ 次幂衰减到几乎为零,而难样本梯度近乎完整保留。于是即便背景样本数量是前景的 1000 倍,只要它们大多是易分的(pₜ→1),它们的**梯度贡献**也被 (1−pₜ)ᵞ 压到可忽略,前景的梯度信号重新浮现出来。**注意 Focal Loss 解决的是梯度层面的不均衡,不是简单的样本计数。**

### 3.4 RetinaNet:Focal Loss 的载体

Focal Loss 只是一个损失函数,需要一个网络承载。RetinaNet(同篇论文)= **ResNet + FPN(特征金字塔)+ 两个并行子网(分类头、回归头)+ Focal Loss**。

```
ResNet backbone ──▶ FPN (P3..P7 多尺度特征) ──┬──▶ 分类子网 (4×Conv + 输出 K·A 通道)  ← Focal Loss
                                              └──▶ 回归子网 (4×Conv + 输出 4·A 通道)   ← Smooth L1
A = 每位置 anchor 数(=9),K = 类别数
```

两个工程细节体现了对不均衡的深度理解:

1. **分类头偏置初始化**。训练初期模型随机,如果分类头偏置为 0,则每个 anchor 预测前景概率 ≈ 0.5,海量背景 anchor 一上来就贡献巨大损失,训练不稳定甚至 NaN。RetinaNet 把分类头最后一层的偏置初始化为 `b = −log((1−π)/π)`,取 π = 0.01,使得**初始时每个 anchor 预测前景的先验概率就是 0.01**(符合"绝大多数 anchor 是背景"的真相)。推导:sigmoid(b) = π ⟹ b = log(π/(1−π)) = −log((1−π)/π)。这让初始损失从背景项主导变得可控。

2. **损失归一化**。Focal Loss 的总损失除以**被分配为前景的 anchor 数**(不是总 anchor 数),因为绝大多数 anchor 是背景且损失被压到接近 0,用总数归一会把损失稀释得没意义。

### 3.5 Focal Loss 的 PyTorch 实现

下面是逐位置 sigmoid focal loss 的标准实现(对每个 anchor 的每个类别独立做二分类,这是 RetinaNet 的做法——多标签式,不是 softmax 多分类)。

```python
import torch
import torch.nn.functional as F

def sigmoid_focal_loss(
    logits: torch.Tensor,      # (N, K) 每个样本对 K 个类别的原始 logit
    targets: torch.Tensor,     # (N, K) one-hot,前景类对应位为 1,其余 0;背景 anchor 全 0
    alpha: float = 0.25,
    gamma: float = 2.0,
    reduction: str = "sum",
) -> torch.Tensor:
    # p = sigmoid(logit),每个 (anchor, class) 独立的前景概率
    p = torch.sigmoid(logits)

    # 先算逐元素标准交叉熵(数值稳定版,内部用 logits 避免 log(0))
    # ce = -[ y·log(p) + (1-y)·log(1-p) ]
    ce = F.binary_cross_entropy_with_logits(logits, targets, reduction="none")

    # pₜ:对"正确类别"的预测概率
    #   y=1 时 pₜ = p ;  y=0 时 pₜ = 1-p
    #   下式用 targets 做选择,等价于上面的分段定义
    p_t = p * targets + (1 - p) * (1 - targets)

    # 调制因子 (1 - pₜ)^γ —— 把易分样本(pₜ→1)压向 0
    modulating = (1.0 - p_t) ** gamma

    loss = modulating * ce

    # α 平衡:前景 α,背景 1-α
    if alpha >= 0:
        alpha_t = alpha * targets + (1 - alpha) * (1 - targets)
        loss = alpha_t * loss

    if reduction == "sum":
        return loss.sum()
    elif reduction == "mean":
        return loss.mean()
    return loss  # "none"
```

**实现要点与坑**:

- 必须用 `binary_cross_entropy_with_logits`(传 logit)而不是先 sigmoid 再 BCE,否则 p 取到 0 或 1 时 log 会 NaN。内部用 log-sum-exp 技巧保证数值稳定。
- `p_t = p·y + (1−p)·(1−y)` 这一行是把分段定义向量化的标准写法,务必理解:y=1 处取 p,y=0 处取 1−p。
- 归一化:外层调用时 `loss.sum() / num_foreground`,而不是 `.mean()`(否则被海量背景稀释)。
- 这是**多标签 sigmoid** 版本(每类独立二分类)。如果用 softmax 多分类版本,pₜ 的定义和实现要相应改。torchvision 的 `torchvision.ops.sigmoid_focal_loss` 就是这个 sigmoid 版本,可对照阅读。

### 3.6 复杂度小结

Focal Loss 相对交叉熵的**额外计算**:每个样本多一次幂运算 `(1−pₜ)ᵞ` 和一次乘法,即 O(N·K) 的逐元素开销,N 是 anchor 数、K 是类别数。**没有改变量级**,几乎不增加训练成本,却把单阶段精度拉到两阶段水平——这是它影响力大的关键:**纯损失函数改进,零额外推理成本,即插即用**。

## 4. Anchor-free:去掉 anchor 超参

无论 YOLOv2+/SSD/RetinaNet,都依赖 **anchor(预设框)**:你得手工设定 anchor 的尺度、长宽比、每位置数量。这些超参敏感、与数据集强相关、还引入大量待匹配的框。Anchor-free 这一派想彻底去掉它们。

### 4.1 FCOS:逐像素回归到四条边的距离

FCOS(Fully Convolutional One-Stage,Tian et al., 2019,ICCV)的思路:**特征图上的每个像素点,如果落在某个真值框内部,就让它直接回归到该框四条边的距离**。

```
对落在真值框内的像素位置 (x, y),回归一个 4 维向量 (l, t, r, b):
  l = x − x₀   (到左边界距离)      t = y − y₀   (到上边界距离)
  r = x₁ − x   (到右边界距离)      b = y₁ − y   (到下边界距离)
其中 (x₀,y₀,x₁,y₁) 是该真值框的左上、右下角
预测框可由 (l,t,r,b) 反算出来
```

这把检测变成了**逐像素的稠密回归**,完全没有 anchor。但带来两个新问题,FCOS 各有对策:

**问题一:一个像素落在多个重叠框内,该回归哪个?** FCOS 规则:回归**面积最小**的那个框(小物体优先,否则小物体永远被大物体覆盖)。再配合 FPN 多尺度——不同大小的框分配到不同 FPN 层(按 l,t,r,b 的最大值落在哪个区间),大幅减少同层内的歧义。

**问题二:远离物体中心的像素会预测出低质量框。** 框内靠近边缘的像素(比如紧贴边界)虽然也在框内,但它"看到"的物体信息少,预测的框往往偏。FCOS 引入 **centerness(中心度)** 分支,预测当前像素离物体中心有多近:

```
centerness = √[ (min(l,r)/max(l,r)) · (min(t,b)/max(t,b)) ]
```

逐项理解:`min(l,r)/max(l,r)` 衡量像素在水平方向上有多居中(正中心时 l=r,比值=1;贴边时比值→0),垂直方向同理,两者相乘开根号得到一个 [0,1] 的居中度。**测试时把 centerness 乘到分类得分上**,自动压低那些远离中心、框质量差的预测,起到类似"内置 NMS 前置过滤"的作用。它不去掉 NMS,但让 NMS 前的候选质量更高。

### 4.2 CenterNet:把物体表示为一个中心点

CenterNet(Objects as Points,Zhou et al., 2019)更激进:**一个物体 = 它的中心点**。检测变成"在一张热图(heatmap)上找峰值"。

```
预测三个东西:
  1. 中心点热图 Ŷ ∈ [0,1]^{H×W×C}:每个类别一张热图,峰值处=物体中心
  2. 中心点偏移 (offset):热图下采样取整带来的量化误差补偿
  3. 物体尺寸 (w, h):在中心点位置回归框的宽高
```

**训练目标怎么造——高斯溅射(Gaussian splatting)。** 真值中心点不是一个孤立的 1、周围全 0(那样太稀疏、难学),而是用一个 2D 高斯核把"是中心"的信号在周围铺开:

```
Y_{xyc} = exp( −[(x−px)² + (y−py)²] / (2σ_p²) )
```

(px,py) 是某类 c 物体的真值中心,σ_p 与物体尺寸相关(大物体核大)。这样靠近中心的位置也有部分监督信号,梯度更平滑。

**损失用 Focal Loss 的变体(penalty-reduced focal loss)**,因为热图上正样本(峰值)极少、负样本(背景)海量——又是不均衡问题:

```
                  ⎧ (1−Ŷ)ᵅ · log(Ŷ)                      if Y = 1
L_heatmap = −1/N · Σ ⎨
                  ⎩ (1−Y)ᵝ · Ŷᵅ · log(1−Ŷ)               otherwise
```

注意它在标准 Focal 上额外乘了 `(1−Y)ᵝ`:对靠近真值中心(Y 接近 1,即高斯核内)的负样本**减轻惩罚**,因为它们"接近正确",不该被当成纯背景狠罚。这是 Focal Loss 思想在热图上的延伸。

**CenterNet 的杀手锏:理论上不需要 NMS。** 因为它直接在热图上取峰值——具体用一个 3×3 的 max pooling 找局部极大值(等价于一个极轻量的峰值提取),**这一步本身就起到了去重作用**。这是 anchor-based 方法做不到的:anchor 方法会对同一物体产生多个高度重叠的高分框,必须靠 NMS 去重;CenterNet 一个物体只有一个峰值,天然无重叠。这点和下一节 DETR 的"无 NMS"动机相通,但实现路径完全不同。

## 5. DETR:把检测重写成集合预测

到这里,所有方法——无论单/双阶段、anchor-based/free——都共享一套**手工设计的流水线**:铺密集候选(anchor 或像素)→ 给每个候选独立打分回归 → **用 NMS 去掉重复框**。DETR(DEtection TRansformer,Carion et al., 2020,ECCV,Facebook AI)第一次把这套流水线全部干掉,用一个端到端的 Transformer 直接输出一组框。

### 5.1 核心思想:检测就是集合预测

DETR 的视角转换:**目标检测 = 预测一个无序的集合 {(类别, 框)}**。一张图的标注本来就是一个集合(框之间没有顺序),那就让模型直接吐出一个**固定大小 N 的预测集合**(N 取得足够大,比如 100,远超图中物体数),多出来的预测就标成特殊类 ∅(no object,"没有物体")。

```
Transformer 输出 N 个预测: { (ĉᵢ, b̂ᵢ) }, i = 1..N
  ĉᵢ:类别概率分布(含 ∅ 类)
  b̂ᵢ:框 (cx, cy, w, h),归一化到 [0,1]
真值: { (cⱼ, bⱼ) }, j = 1..M,  M ≤ N,用 ∅ 补齐到 N
```

**这个视角立刻引出一个根本问题:N 个预测和 M 个真值,谁对应谁?** 在 anchor 方法里,对应关系由"哪个 anchor 和哪个 GT 的 IoU 最大"硬性决定,所以会有多个 anchor 匹配同一 GT → 多个重复框 → 需要 NMS。DETR 不要这种多对一,它要**预测和真值之间一对一的最优匹配**。这就是匈牙利匹配登场的地方。

### 5.2 架构:CNN + Transformer encoder-decoder + object queries

```
图像 ──CNN(ResNet)──▶ 特征图 (C×H×W)
   ──1×1 conv 降维──▶ (d×H×W) ──展平──▶ d × HW 的序列 (+ 位置编码)
        │
        ▼
  Transformer Encoder (标准自注意力,在 HW 个空间位置间做全局交互)
        │  memory (d × HW)
        ▼
  Transformer Decoder
        ▲
   N 个 object queries (d 维可学习向量,随机初始化,训练得到)
        │
        ▼  N × d
  前馈头:每个 query → (类别分布 ĉᵢ , 框 b̂ᵢ)
```

**object query 是 DETR 最精妙也最反直觉的设计。** 它是 N 个可学习的嵌入向量(不依赖输入图像,训练后固定),作为 decoder 的输入"插槽"。每个 query 通过 decoder 的**交叉注意力**去 encoder 输出的图像特征里"查询"——可以理解为每个 query 学会了去图像的某个区域、某种尺度找物体("我负责找左上区域的中等物体")。decoder 的**自注意力**让 N 个 query 之间互相通信——这是去重的关键:query 们彼此知道对方要预测什么,从而**避免两个 query 报同一个物体**。这是 DETR 不需要 NMS 的结构性原因之一(另一个是匈牙利匹配的一对一约束)。

### 5.3 匈牙利匹配:二分图最优分配

这是 DETR 的数学核心。我们要在 N 个预测和(补齐到 N 的)真值之间找一个**一一对应(置换)σ**,使得总匹配代价最小。这是经典的**二分图最优分配问题(bipartite matching / assignment problem)**。

设置代价矩阵。对"预测 i 匹配真值 j"定义匹配代价 `L_match(i, j)`:

```
L_match(i, j) = −𝟙{cⱼ≠∅} · p̂ᵢ(cⱼ)                          # 类别项:预测 i 给真值类别 cⱼ 的概率越高,代价越低(取负)
              + 𝟙{cⱼ≠∅} · L_box(bⱼ, b̂ᵢ)                    # 框项:框越接近代价越低
```

- 类别项用的是**概率 p̂ᵢ(cⱼ)**(不是 log),论文说这样和框项数值尺度更协调。前面取负号:预测对真值类别越自信,匹配代价越小。
- 框项 `L_box = λ_iou · L_giou(bⱼ, b̂ᵢ) + λ_L1 · ‖bⱼ − b̂ᵢ‖₁`,**GIoU(Generalized IoU)+ L1 的组合**。为什么不能只用 L1?因为 L1 对框的绝对尺度敏感(大框 L1 天然大),GIoU 是尺度不变的;但只用 GIoU 在框不重叠时梯度信息弱,L1 补上。两者互补。

然后求最优置换:

```
σ̂ = argmin_{σ ∈ 置换群 S_N}  Σᵢ L_match( i, σ(i) )
```

**这个最小化用匈牙利算法(Hungarian algorithm / Kuhn-Munkres)精确求解,复杂度 O(N³)。** N=100 时 N³=10⁶,在 CPU 上微秒级,完全可接受。注意:**匹配只在算损失时做、不参与梯度**(它输出的是"哪个预测对哪个真值"的离散对应,由 scipy.optimize.linear_sum_assignment 给出),梯度只通过下面的损失流回网络。

### 5.4 匈牙利损失:用最优匹配算最终损失

拿到最优匹配 σ̂ 后,真正的训练损失对每对匹配求和:

```
L_Hungarian = Σᵢ [ −log p̂ᵢ(c_{σ̂(i)})  +  𝟙{c_{σ̂(i)}≠∅} · L_box( b_{σ̂(i)}, b̂ᵢ ) ]
```

- 分类用标准负对数似然(注意这里用 log,和匹配代价用概率不同——匹配阶段为了和框项尺度协调用概率,损失阶段为了梯度性质好用 log)。
- 匹配到 ∅ 的预测(大多数):只算分类损失(学会预测"无物体"),不算框损失。论文还把 ∅ 类的分类损失**降权(÷10)**,因为匹配到 ∅ 的预测数量远多于匹配到真实物体的——**又是不均衡!** DETR 用的是简单降权,呼应前面 YOLO 的 λ_noobj。

### 5.5 为什么 DETR 不需要 NMS——把机制说死

把整条逻辑链收紧,这是本章必须讲透的点:

1. **一对一匹配约束**。匈牙利匹配保证 N 个预测中,**每个真值只被分配给一个预测**(置换是双射)。所以网络被训练成"对一个物体只产生一个高分框",而不是像 anchor 方法那样多个 anchor 命中同一物体。
2. **去重信号来自损失,不是后处理**。anchor 方法的去重是事后的(NMS 在推理时按 IoU 抑制重复);DETR 的去重是**训练时学会的**——如果两个 query 都想报同一物体,匈牙利匹配只会让其中一个匹配到真值、另一个被迫匹配到 ∅(从而被分类损失推向"预测无物体")。久而久之,网络学会让 query 们分工,不重复。
3. **decoder 自注意力提供通信渠道**。N 个 query 通过自注意力互相看到对方的预测倾向,这是它们能"协商分工、不撞车"的结构基础。

**结论**:DETR 把"去重"从一个手工的、不可微的推理后处理(NMS),变成了一个**可微的、在训练中习得的集合级约束**。这是端到端的真正含义——整条流水线没有任何手工设计的后处理,全部参数从数据中学。

### 5.6 代价:收敛慢

DETR 不是没有代价,而且代价很显著:**训练收敛极慢——原始 DETR 要训 500 epochs**(对比 Faster R-CNN 的几十 epochs),且**小物体性能偏弱**。原因有两层,都值得理解:

1. **匹配不稳定导致优化困难**。训练早期网络是随机的,匈牙利匹配给出的"预测↔真值"对应关系**每个 iteration 都在变**(这个 epoch 预测 i 匹配真值 j,下个 epoch 可能匹配真值 k)。监督目标不稳定,query 的"分工"要很久才能稳定下来,优化信号噪声大。这是收敛慢的主因(后续 DN-DETR、DINO 用"去噪查询"稳定匹配来加速,把 500 epochs 降到 ~12-36)。
2. **注意力的 O((HW)²) 复杂度限制了特征分辨率**。encoder 自注意力在 HW 个空间位置间是平方复杂度。为控算力,DETR 用的特征图分辨率较低(一般取 C5,下采样 32 倍),空间太粗 → 小物体信息丢失 → 小物体检测弱。Deformable DETR(2021)用稀疏的可变形注意力(每个 query 只关注少数采样点)把复杂度降下来,既能用高分辨率/多尺度特征,又把收敛压到 ~50 epochs。

**这是一个清晰的范式权衡**:DETR 用"概念上的极简(无 anchor、无 NMS、端到端)"换来了"优化上的困难(慢、调参敏感)"。是否值得,取决于你更看重部署简洁还是训练成本——这也是 2020 年后检测领域的主线之一。

### 5.7 DETR 匈牙利匹配伪代码

```python
# 依赖:scipy.optimize.linear_sum_assignment 解二分图最优分配(匈牙利算法)
import torch
from scipy.optimize import linear_sum_assignment

@torch.no_grad()  # 匹配过程不需要梯度
def hungarian_match(pred_logits, pred_boxes, gt_labels, gt_boxes,
                    cost_class=1.0, cost_l1=5.0, cost_giou=2.0):
    """
    pred_logits: (N, num_classes+1)  N 个预测的类别 logit(含 ∅ 类)
    pred_boxes : (N, 4)              N 个预测框 (cx,cy,w,h),已归一化
    gt_labels  : (M,)                M 个真值类别
    gt_boxes   : (M, 4)              M 个真值框
    返回 (idx_pred, idx_gt):一一对应的索引,len == M
    """
    # ---- 1. 构造代价矩阵 C ∈ R^{N×M} ----
    prob = pred_logits.softmax(-1)                 # (N, num_classes+1)

    # 类别代价:取负的"预测给真值类别的概率"。prob[:, gt_labels] → (N, M)
    cost_cls = -prob[:, gt_labels]                 # (N, M)

    # L1 框代价:每个 pred 框 vs 每个 gt 框的 L1 距离 → (N, M)
    cost_box = torch.cdist(pred_boxes, gt_boxes, p=1)   # (N, M)

    # GIoU 代价:1 - GIoU(pred, gt),需先把 (cx,cy,w,h) 转 (x0,y0,x1,y1)
    cost_giou_mat = -generalized_iou(
        box_cxcywh_to_xyxy(pred_boxes),
        box_cxcywh_to_xyxy(gt_boxes),
    )                                              # (N, M),GIoU 越大代价越小

    # 加权合并:这三个权重要和 5.4 损失阶段对齐
    C = cost_class * cost_cls + cost_l1 * cost_box + cost_giou * cost_giou_mat
    C = C.cpu().numpy()                            # (N, M)

    # ---- 2. 匈牙利算法求最优一一对应,O(N^3) ----
    idx_pred, idx_gt = linear_sum_assignment(C)    # 各返回长度 M 的索引数组
    return torch.as_tensor(idx_pred), torch.as_tensor(idx_gt)


def detr_loss(pred_logits, pred_boxes, gt_labels, gt_boxes):
    # 1) 先做匹配(no_grad)
    idx_pred, idx_gt = hungarian_match(pred_logits, pred_boxes, gt_labels, gt_boxes)

    # 2) 分类损失:所有 N 个预测都算
    #    匹配上的预测 → 目标=对应真值类;未匹配的 → 目标=∅ 类
    N = pred_logits.shape[0]
    target_classes = torch.full((N,), fill_value=NO_OBJECT_IDX, dtype=torch.long)
    target_classes[idx_pred] = gt_labels[idx_gt]
    # ∅ 类降权(论文 ÷10)缓解 no-object 主导
    class_weight = torch.ones(num_classes + 1); class_weight[NO_OBJECT_IDX] = 0.1
    loss_cls = F.cross_entropy(pred_logits, target_classes, weight=class_weight)

    # 3) 框损失:只对"匹配到真实物体"的预测算 L1 + GIoU
    matched_pred_boxes = pred_boxes[idx_pred]
    matched_gt_boxes   = gt_boxes[idx_gt]
    loss_l1   = F.l1_loss(matched_pred_boxes, matched_gt_boxes, reduction="sum")
    loss_giou = (1 - generalized_iou_diag(
        box_cxcywh_to_xyxy(matched_pred_boxes),
        box_cxcywh_to_xyxy(matched_gt_boxes))).sum()

    num_boxes = max(len(idx_gt), 1)  # 用真值物体数归一化
    return loss_cls + (5.0 * loss_l1 + 2.0 * loss_giou) / num_boxes
```

**伪代码要点与坑**:
- 匹配阶段类别用 `softmax` 概率(取负),损失阶段用 `cross_entropy`(即 log)——这个不一致是论文有意为之,别"统一"它。
- 匹配在 `@torch.no_grad()` 下做,匈牙利算法本身不可微;梯度只走 `detr_loss` 里那些张量运算。
- `linear_sum_assignment` 跑在 CPU/numpy 上,要 `.cpu().numpy()`。
- ∅ 类降权 0.1 容易被忽略,漏了会导致模型偏向预测"无物体"。
- 实际 DETR 还在 decoder 每一层都接预测头算辅助损失(auxiliary loss),加速收敛,这里略去。

## 6. 设计权衡与常见坑

**1. Focal Loss 不是万能开关。** 它假设"难样本值得学";但如果数据集里有大量**标注错误(噪声标签)**,这些样本恰好也是 pₜ 很低的"难样本",Focal Loss 会把梯度狠狠聚焦到错误标签上,反而有害。脏数据集上用大 γ 要谨慎。

**2. α 和 γ 必须联调。** 见 3.2,α 单独看是反直觉的(前景反而权重小)。新手常犯的错是"前景少所以 α 给大",忽略了 γ 已经把背景压狠了。论文最优是 (α=0.25, γ=2),它们是一对耦合超参。

**3. anchor-free 不等于无超参。** FCOS 把 anchor 超参换成了"FPN 各层负责的尺度区间""centerness 的定义""哪个框优先(面积最小)"等规则;CenterNet 引入了高斯核 σ、热图分辨率。**超参没消失,只是换了形式**。别以为 anchor-free 就免调参。

**4. DETR 的 N(query 数)要远超图中最大物体数。** N=100 对 COCO(单图通常 <100 物体)够用,但密集场景(人群、商品货架)可能爆 N,导致漏检。N 也不能太大(每个 query 都参与自注意力,O(N²),且空 query 太多稀释训练)。

**5. DETR 训练不稳定的典型症状**:前期 loss 下降后长时间平台、各类 AP 长期为 0 然后某个 epoch 突然起来。这是匹配在"洗牌"。**别在 50 epoch 就判死刑**(原始配方要 500)。预算有限直接上 Deformable DETR / DINO 这类改进版。

**6. NMS 仍是 anchor-based 单阶段的标配。** YOLO/SSD/RetinaNet/FCOS 推理时都要 NMS;只有 CenterNet(峰值提取)和 DETR(集合预测)系真正免 NMS。别把"单阶段"和"无 NMS"混为一谈,二者正交。

**7. 单阶段的延迟不只看 backbone。** NMS 本身在物体多时也耗时(O(框数²) 的朴素实现),密集场景下 NMS 可能成为瓶颈——这也是无 NMS 方法的一个实际卖点。

## 7. 动手练习

**练习 1(推导题):Focal Loss 对易/难样本的梯度比。**
对负样本(y=0,p=σ(x)),仿照 3.3 推出 `dFL/dx`(提示:负样本 pₜ=1−p,CE=−log(1−p),按链式法则一步步来,你会得到形如 `(1−pₜ)ᵞ·[...]` 的结果,注意符号)。然后取易分负样本 p=0.05(pₜ=0.95)和难分负样本 p=0.4(pₜ=0.6),在 γ=2 下分别算出梯度幅度,求两者之比,并与标准交叉熵(γ=0)下的同一比值对比,定量说明 Focal Loss 把多少梯度从易分样本转移到了难样本。

**练习 2(编码题):跑通 sigmoid focal loss 并验证调制效果。**
基于 3.5 的实现,构造一个 batch:10 个前景样本(targets=1)+ 10000 个背景样本(targets=0),让背景样本的 logit 服从"大多易分"(比如 logit ~ −5,即 p≈0.0067)。分别用标准 BCE(γ=0)和 Focal(γ=2)计算总损失,打印**前景损失占总损失的比例**。验证:γ=0 时该比例极低(背景淹没),γ=2 时显著提升。提示:用 `reduction="none"` 拿到逐样本损失再分组求和。

**练习 3(分析题):anchor-based vs anchor-free 的正负样本分配对比。**
写一段文字(配公式)对比三者如何决定"哪个预测位置负责哪个真值":(a) RetinaNet(anchor,按 IoU 阈值);(b) FCOS(像素落在框内 + 面积最小 + FPN 分层);(c) DETR(匈牙利全局最优匹配)。重点分析:为什么 (a)(b) 会产生"一个真值对多个正样本"从而必须 NMS,而 (c) 强制一对一所以不需要 NMS?把"多对一→需 NMS""一对一→免 NMS"这条因果链讲清楚。

**练习 4(分析+推导题):为什么 DETR 收敛慢?**
(a) 解释"匹配不稳定"如何导致监督目标在 iteration 间跳变(举一个 2 个 query、2 个真值的最小例子,说明早期一次代价矩阵微小扰动就能翻转匹配)。(b) 算一下:输入 800×800、下采样 32 倍得 25×25 特征图,encoder 自注意力的复杂度是多少(以 HW 计)?如果换成下采样 8 倍(100×100),复杂度涨几倍?用这个数说明为什么原始 DETR 不敢用高分辨率特征、从而小物体弱。

## 8. 源码 / 论文导读

**论文(按阅读顺序):**
- **YOLOv1**:Redmon et al., *You Only Look Once*(CVPR 2016)。重点读第 2 节(网络输出张量定义)和损失公式(理解 λ_coord / λ_noobj / √w 三个补丁)。

- **CenterNet**:Zhou et al., *Objects as Points*(arXiv 2019)。读热图高斯目标的构造、penalty-reduced focal loss、用 maxpool 取峰值替代 NMS。

## 9. 小结与承上启下

这一章你掌握了三件事,它们构成现代检测的三条主干:

1. **单阶段范式**(YOLO 网格回归 / SSD 多尺度):去掉 proposal 换速度,代价是直面极端类别不均衡。
2. **Focal Loss**:用调制因子 `(1−pₜ)ᵞ` 在**梯度层面**自动聚焦难样本,纯损失改进、零推理成本地解决不均衡,让单阶段精度追平两阶段。其梯度 `(1−p)ᵞ·[γp·log p −(1−p)]` 的 `(1−p)ᵞ` 因子是"自动聚焦"的数学心脏。anchor-free(FCOS/CenterNet)进一步把 anchor 超参也去掉。
3. **DETR**:把检测重写成**集合预测**,用匈牙利匹配做一对一最优分配,把"去重"从不可微的 NMS 变成训练中习得的可微约束,实现真正端到端;代价是收敛慢、小物体弱。

**在整门课里的位置**:上一章(两阶段)+ 这一章(单阶段 + DETR)合起来,给了你检测的完整版图——你现在能读懂从 R-CNN 到 DINO 几乎所有检测论文的核心机制。**DETR 引入的 Transformer + 集合预测,是检测通向分割、跟踪、乃至多模态(检测作为"视觉接口")的桥梁**:下一章(实例/全景分割)你会看到 Mask R-CNN 如何在检测框基础上加分割分支,以及 MaskFormer / Mask2Former 如何把 DETR 的"集合预测 + query"思想直接搬到分割——届时你会发现,本章学的 object query 和匈牙利匹配,在分割里几乎原样复用。检测和分割,从 DETR 之后开始走向统一。
