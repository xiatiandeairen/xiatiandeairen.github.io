> 这一章把你从"知道 CLIP 能做零样本分类"带到"能在白板上推出双向对称 InfoNCE 的每一项梯度、能徒手写出对比损失和零样本推理流程、能精确说清零样本为什么不需要微调、以及 CLIP 在细粒度/计数/空间关系上弱在哪里、为什么弱"。CLIP 是开放词表、VLM、文生图三条主线的共同地基,这章要求你对"自然语言监督 + 对比对齐"这套机制没有任何黑箱。

## 一句话引言

CLIP(Contrastive Language–Image Pre-training,Radford et al., 2021)的全部魔法可以压成一句话:**用海量(图,文)对做对比学习,把图像和文本嵌入硬塞进同一个单位球面上,让配对的近、不配对的远;塞好之后,任何一句话都成了一个可比较的"分类器权重",于是分类不再需要训练**。本章从"闭集监督的诅咒"讲起,推到 batch 内 N×N 相似度矩阵上的双向对称 InfoNCE,把温度、L2 归一化、logit_scale 的作用全部推开,再讲透零样本推理为什么 work、prompt 为什么敏感、以及它通向 LLaVA 和文生图的接口在哪。

## 1. 动机:闭集监督的诅咒

### 1.1 一个 softmax 分类头能认识多少东西

回忆前面章节里标准的图像分类:一个 backbone(ResNet / ViT)输出特征 `z ∈ ℝ^d`,接一个线性分类头 `W ∈ ℝ^{C×d}`,算 `logits = Wz`,过 softmax,和 one-hot 标签算交叉熵。

这套范式有一个**结构性的天花板,藏在 C 里**:

- **类别数 C 在建图时就被钉死**。ImageNet-1k 训出来的模型,分类头永远只有 1000 行。想认识"柯基犬"以外的"边境牧羊犬",得加一行 `W`、重新标数据、重新训练。
- **标签是无语义的整数**。第 207 类和第 208 类在 one-hot 监督下是**正交的、等距的**——模型无从知道"金毛"和"拉布拉多"比"金毛"和"卡车"更像。所有语义关系都被 one-hot 抹平了。
- **监督信号极贵**。每张图要人工指派一个固定类别。ImageNet 那 128 万张标注耗费了巨大人力,而它只覆盖 1000 个概念。

**核心矛盾**:视觉世界的概念是开放的、连续的、有语义结构的,却被强行塞进一个"固定 C 类、类间正交、逐图人工标注"的离散框里。这就是**闭集(closed-set)监督的诅咒**。

### 1.2 破局点:把"标签"换成"一句话"

CLIP 的破局思路和注意力打破信息瓶颈一样朴素:**别再用整数当标签,用自然语言当标签**。

互联网上有海量天然的(图,文)对——网页里图片的 alt 文本、社交媒体的图配文、图片站的 caption。CLIP 收集了约 **4 亿对**(图,文)(论文称 WIT,WebImageText;具体规模 400M 来自论文,数据集未公开,细节「待核」)。这些文本不是 1000 个固定类别,而是任意自由文本:"a black dog running on the beach"、"diagram of a transformer architecture"、"两只猫在沙发上"。

用自然语言当监督,一举解掉 §1.1 的三个问题:

1. **没有固定 C**。监督目标是"这张图配哪句话",句子空间是开放的,天然支持任意新概念。
2. **标签自带语义结构**。"金毛"和"拉布拉多"两句话在文本编码器里本就靠得近,正交性消失了,语义关系被保留。
3. **监督几乎免费**。(图,文)对是互联网的副产品,不需要逐图人工指派类别——这是能上到 4 亿规模的前提。

> 直觉:闭集分类是"指着图说它是第几号";自然语言监督是"指着图说它配哪句话"。前者的答案空间是 1000 个抽屉,后者的答案空间是整个语言。**答案空间从离散有限变成开放连续,零样本能力就是这个转变的直接后果。**

### 1.3 为什么是"对比",不是"生成"

有了(图,文)对,怎么学?两条路:

- **生成式**:看图生成 caption(image captioning)。逐词预测,要建模文本的完整联合分布,**训练慢、且把算力浪费在学习语言的语法细节上**。早期工作(如 VirTex)证明可行但样本效率低。
- **对比式**:不要求生成具体文字,只要求"判断哪张图配哪句话"。这是一个**判别任务**,比生成简单得多——你不需要知道怎么写出那句话,只需要知道图和句子配不配。

CLIP 论文里有一张关键消融:把训练目标从"预测 caption 的每个词"换成"对比式地预测整句 caption 与图配不配",**达到同样 zero-shot 精度所需的训练数据量少了约 4 倍**(数值「待核」,但"对比比生成样本效率高数倍"是论文核心结论之一)。原因:**对比任务把建模目标从"P(文本 | 图像) 的完整分布"放松到"图文是否匹配的相对排序",难度骤降,样本效率骤升**。

这就引出本章的技术核心——对比学习怎么在数学上把图文对齐。

## 2. 双塔架构:把图和文映到同一个球面

### 2.1 两个编码器 + 两个投影头

CLIP 是**双塔(two-tower / dual-encoder)**结构,图像和文本各走一条独立的塔,最后投到同一个 d 维空间:

```
图像 I ──[图像编码器 f_img]──▶ h_img ∈ ℝ^{d_img} ──[W_img]──▶ ──L2归一化──▶ uᵢ ∈ ℝ^d, ‖uᵢ‖=1
文本 T ──[文本编码器 f_txt]──▶ h_txt ∈ ℝ^{d_txt} ──[W_txt]──▶ ──L2归一化──▶ vᵢ ∈ ℝ^d, ‖vᵢ‖=1
```

- **图像编码器** `f_img`:CLIP 试了两种主干——改造的 ResNet 和 ViT。取全局表示(ResNet 用注意力池化,ViT 用 [CLS] token 或池化),得到 `h_img`。
- **文本编码器** `f_txt`:一个标准的 Transformer(GPT 式,causal,论文用 12 层、宽度 512、8 头的版本「待核具体超参」)。文本加 [SOS]/[EOS],**取 [EOS] 位置的最后一层激活**作为整句表示 `h_txt`。
- **投影头** `W_img ∈ ℝ^{d×d_img}`、`W_txt ∈ ℝ^{d×d_txt}`:把两塔各自的输出**线性投影到同一个共享维度 d**(CLIP 里是无偏置的线性层)。投影是把两个异构空间拉到可比较空间的桥。
- **L2 归一化**:把投影后的向量除以自身模长,**压到单位超球面 Sᵈ⁻¹ 上**。这一步是后面温度、余弦相似度成立的前提,§3.2 专门推它的作用。

**双塔的关键性质:图、文两条路完全解耦,不在中途交互**。图像编码器的输出不依赖任何文本,反之亦然。这带来一个工程上的巨大好处,也是 CLIP 大规模检索/零样本可行的根本:**文本侧的嵌入可以离线预计算并缓存**。零样本分类时,1000 个类别的文本向量算一次存起来,之后每张新图只过一次图像塔再做点积——这就是开放词表检索能 scale 的原因(对比"图文必须一起进一个 cross-attention 网络"的单塔结构,后者每对都要重算,无法缓存)。

### 2.2 相似度:为什么是余弦

L2 归一化之后,两个向量的点积就是**余弦相似度**:

```
对单位向量 ‖u‖=‖v‖=1:
  u · v = ‖u‖‖v‖ cos∠(u,v) = cos∠(u,v) ∈ [−1, 1]
```

CLIP 用余弦相似度(归一化点积)而非裸点积衡量图文匹配度,理由在 §3.2 推。直觉先记住:**归一化后,相似度只和"方向"有关,和"模长"无关**——模型不能靠把某些样本的向量拉长来作弊,所有比较都在同一个尺度上。

## 3. 对比损失:从 N×N 矩阵到双向对称 InfoNCE

这是全章最硬的一节,也是 CLIP 的灵魂。我们要把"配对的近、不配对的远"翻译成一个可微、可批量、可反传的损失,并把它的每一项梯度推开。

### 3.1 一个 batch 构造出 N² 个判断

取一个大小为 N 的 batch:N 个(图,文)对 `(I₁,T₁), (I₂,T₂), ..., (Iₙ,Tₙ)`。各自过塔、投影、归一化,得到:

```
图像嵌入: u₁, u₂, ..., uₙ ∈ ℝ^d   (每个 ‖uᵢ‖=1)
文本嵌入: v₁, v₂, ..., vₙ ∈ ℝ^d   (每个 ‖vⱼ‖=1)
```

构造一个 **N×N 的相似度矩阵 L**(logits):

```
Lᵢⱼ = (uᵢ · vⱼ) · exp(τ)        i,j ∈ {1..N}
```

- `Lᵢⱼ` 是第 i 张图和第 j 句文的相似度(乘了温度缩放,下面解释 τ)。
- **对角线 Lᵢᵢ 是 N 个真实配对(正样本)**。
- **每一行/每一列的非对角元素是负样本**:第 i 张图配第 j≠i 句文,是同 batch 里"别人的文本",天然就是负例。

**这是 CLIP 最聪明的地方:batch 自己提供负样本**。不需要像 MoCo 那样维护一个 momentum 队列、像 SimCLR 那样精心设计数据增强生成正负对。一个 N 的 batch 直接给出 N 个正例 + N²−N 个负例。**于是 batch 越大,每个正例对比的负例越多,对比信号越强**——这直接解释了为什么 CLIP 要用极大 batch(原论文 batch size 32768「待核」)。

### 3.2 温度与归一化:logit_scale 在干什么

`L = (U Vᵀ) · exp(τ)` 里那个 `exp(τ)` 是 CLIP 的**可学习温度(temperature)**,代码里叫 `logit_scale`。逐层拆它为什么这么设计:

**第一,为什么要 L2 归一化再点积。** 设想不归一化,直接用裸点积 `hᵢ · hⱼ` 当 logit。那么模型可以通过**放大正样本对的向量模长**来无限增大正样本 logit、压低损失,而不真正改善"方向对齐"。这是一种退化捷径(collapse)。L2 归一化把所有向量钉在单位球面上,**模长这个自由度被消掉,模型只能靠调整方向来降损失**——而方向对齐才是我们要的语义对齐。

**第二,归一化后相似度被限死在 [−1,1],softmax 会过于平坦。** 余弦值域只有 [−1,1],正负样本的 logit 差最多差 2。把这么挤的 logits 喂给 softmax,分布会非常平,几乎均匀,**梯度软弱、学不出尖锐的对齐**。需要一个缩放因子把 logits 拉开。

**第三,这个缩放因子设成可学习的。** 记缩放为 1/temperature。CLIP 不固定它,而是把 `log(1/temperature) = logit_scale = τ` 作为一个**可训练标量参数**,前向时乘 `exp(τ)`。为什么用 exp:保证缩放恒为正(`exp(τ)>0`),且让优化在对数空间进行更稳。论文还把 `exp(τ)` **裁剪上限到 100**(防止温度过低、logits 爆炸导致训练不稳)。可学习温度让模型自己找到"该把分布拉多尖"。

```
有效 logit:  Lᵢⱼ = cos∠(uᵢ,vⱼ) · exp(τ)
            值域从 [−1,1] 被放大到 [−exp(τ), exp(τ)]  (典型 exp(τ)≈14~100)
```

一句话:**L2 归一化保证学的是方向(语义),可学习温度 exp(τ) 把被归一化压扁的 logits 重新拉开到合适锐度**。两者缺一不可,这是新手最容易漏掉的细节。

### 3.3 双向对称 InfoNCE 的完整推导

现在把 N×N 矩阵变成损失。CLIP 用的是 **InfoNCE**(也叫 NT-Xent / 多类对比交叉熵),而且是**双向对称**的——既要"图找对文",也要"文找对图"。

**先看图 → 文方向(image-to-text)。** 固定第 i 张图,它的 logits 是矩阵 L 的**第 i 行** `Lᵢ· = (Lᵢ₁, ..., Lᵢₙ)`。在这 N 个候选文本里,正确答案是 j=i(它自己配的那句)。这是一个 **N 选 1 的分类问题**,用交叉熵:

```
图→文 第 i 个样本的损失(对第 i 行做 softmax,目标类是 i):
  pᵢ→ⱼ = exp(Lᵢⱼ) / Σ_{k=1..N} exp(Lᵢₖ)        (第 i 行的 softmax)
  ℓ_i^{i2t} = −log pᵢ→ᵢ = −log [ exp(Lᵢᵢ) / Σₖ exp(Lᵢₖ) ]
```

直觉:**最小化它 = 让第 i 行里对角元 Lᵢᵢ 在所有列中最大**,即第 i 张图和它真正的文本最像、和别人的文本不像。

**再看文 → 图方向(text-to-image)。** 完全对称:固定第 j 句文,它的 logits 是矩阵 L 的**第 j 列** `L·ⱼ`,正确答案是 i=j,对列做 softmax:

```
文→图 第 j 个样本的损失(对第 j 列做 softmax,目标类是 j):
  qⱼ→ᵢ = exp(Lᵢⱼ) / Σ_{k=1..N} exp(Lₖⱼ)        (第 j 列的 softmax)
  ℓ_j^{t2i} = −log qⱼ→ⱼ = −log [ exp(Lⱼⱼ) / Σₖ exp(Lₖⱼ) ]
```

**两个方向取平均,得到 CLIP 的总损失:**

```
L_CLIP = (1/2) · [ (1/N) Σᵢ ℓ_i^{i2t}  +  (1/N) Σⱼ ℓ_j^{t2i} ]
       = (1/2) · [ CE(行 softmax, 对角标签)  +  CE(列 softmax, 对角标签) ]
```

**为什么必须双向对称?** 只做图→文(行 softmax)只保证"每张图能找对文",但不保证"每句文能找对图"——可能出现一句话被多张图同时认领、而该句对应的真图反而没认领它。检索是双向的(以文搜图、以图搜文都要 work),对齐也必须双向约束。**对称损失同时把行和列都逼成"对角线最大"**,得到的相似度矩阵在两个方向上都接近置换矩阵(理想情况是对角线极大、其余极小)。

把这套写成最干净的形式:**标签就是 `0,1,2,...,N−1`(对角线的下标),损失就是对 L 的行做一次交叉熵、对 Lᵀ 的行(=L 的列)做一次交叉熵,取平均。** 实现上这甚至只有几行(§4)。

### 3.4 梯度推导:对比损失到底在拉什么

把损失对 logits 求导,看清它在几何上对每个嵌入做了什么。只推图→文方向第 i 个样本(文→图对称)。

```
设第 i 行 softmax 概率 pᵢₖ = exp(Lᵢₖ)/Σₘ exp(Lᵢₘ),目标类是 i。
标准 softmax + 交叉熵的 logit 梯度(经典结论,可对 ℓ=−log p_target 直接求导得到):

  ∂ℓ_i^{i2t}/∂Lᵢⱼ = pᵢⱼ − 𝟙[j=i]

即:
  对正确列 j=i:   ∂ℓ/∂Lᵢᵢ = pᵢᵢ − 1  ≤ 0   (负梯度 → 梯度下降会增大 Lᵢᵢ)
  对错误列 j≠i:   ∂ℓ/∂Lᵢⱼ = pᵢⱼ    ≥ 0   (正梯度 → 梯度下降会减小 Lᵢⱼ)
```

**几何解读非常干净**:梯度下降一步,会**增大对角相似度 Lᵢᵢ(把 uᵢ 往 vᵢ 方向拉)**,同时**减小所有非对角 Lᵢⱼ(把 uᵢ 往远离其他 vⱼ 方向推)**,而且推开的力度正比于当前的"误认概率" pᵢⱼ——**模型现在越容易把 uᵢ 误认成 vⱼ,对它的排斥就越强(hardest negatives 得到最大梯度)**。这就是对比学习"拉近正样本、推开难负样本"的数学本体。

再往下一层,经 logit 传到嵌入。由 `Lᵢⱼ = exp(τ)·(uᵢ·vⱼ)`,对 uᵢ 的梯度:

```
∂Lᵢⱼ/∂uᵢ = exp(τ) · vⱼ           (注:uᵢ 还有 ‖uᵢ‖=1 的约束,真实梯度需投影到球面切空间)
⇒ ∂ℓ_i^{i2t}/∂uᵢ = exp(τ) · Σⱼ (pᵢⱼ − 𝟙[j=i]) vⱼ
                = exp(τ) · [ (pᵢᵢ−1) vᵢ + Σ_{j≠i} pᵢⱼ vⱼ ]
```

读这个式子:**对 uᵢ 的更新方向 = 朝正样本 vᵢ 走(系数 1−pᵢᵢ>0),被各负样本 vⱼ 以权重 pᵢⱼ 拉走**。缩放 exp(τ) 是整体的梯度增益——**exp(τ) 越大(等价于温度越低、分布越尖),正负样本概率越极端,梯度越集中在难样本上**。这也解释了为什么 exp(τ) 不能太大(温度不能太低):太大则只有极少数最难负样本有梯度,训练不稳;太小则梯度平摊到所有负样本、信号糊。可学习温度让模型自己在这两者间找平衡。

### 3.5 复杂度与"大 batch"的代价

设 batch 为 N,共享维度为 d。对比损失部分(不含两塔前向)的开销:

```
相似度矩阵 L = U Vᵀ:   U(N×d) × Vᵀ(d×N) → L(N×N)
  ≈ 2 N² d FLOPs                              → O(N²d)
两次 softmax + 两次交叉熵:  对 N×N 矩阵逐行/逐列, O(N²)
显存:  必须实例化 L(N×N)                      → O(N²)
```

**注意 N² 这一项**——它和注意力的序列长度 n² 是同构的难题。batch 翻倍,对比矩阵的计算和显存涨 4 倍。但 CLIP 偏要大 batch(§3.1:负样本越多对比越强),于是 batch 是分布式切到几百张 GPU 上的,**每张卡只算自己那块 logits,再 all-gather 嵌入拼出全局 N×N 矩阵**(这就是工程上的 distributed InfoNCE / gather negatives,实现细节是 CLIP/OpenCLIP 大规模训练的关键)。两塔的前向才是主要算力(O(N) 张图过 ViT/ResNet、O(N) 句话过 Transformer),对比矩阵 O(N²d) 在 d 不大、N 几万时通常不是瓶颈,但显存上的 N×N 是要小心的。

## 4. 代码:对称对比损失 + 零样本分类

### 4.1 双向对称 InfoNCE(训练核心)

把 §3.3 直接落成 PyTorch。**核心只有四步:归一化 → 算缩放后的 logits → 行/列两次交叉熵 → 取平均**。

```python
import torch
import torch.nn.functional as F

def clip_contrastive_loss(img_emb, txt_emb, logit_scale):
    """
    img_emb: [N, d]  图像投影后的嵌入(未归一化)
    txt_emb: [N, d]  文本投影后的嵌入(未归一化)
    logit_scale: 标量参数 exp(τ),前向前已做 .exp();通常 clamp 到 <=100
    """
    N = img_emb.shape[0]

    # 1) L2 归一化,压到单位球面 —— 让相似度=余弦,消掉模长自由度(§3.2)
    img = F.normalize(img_emb, dim=-1)          # [N, d], 每行 ‖·‖=1
    txt = F.normalize(txt_emb, dim=-1)          # [N, d]

    # 2) N×N 相似度矩阵,乘可学习温度把 [-1,1] 拉开(§3.2)
    #    logits_per_image[i,j] = scale * (img_i · txt_j)
    logits_per_image = logit_scale * img @ txt.t()   # [N, N], 行=图, 列=文
    logits_per_text  = logits_per_image.t()          # [N, N], 行=文, 列=图(= L 的转置)

    # 3) 正样本在对角线 → 标签就是 0,1,...,N-1
    labels = torch.arange(N, device=img.device)

    # 4) 双向对称:对行做 CE(图→文) + 对列做 CE(文→图),取平均(§3.3)
    loss_i2t = F.cross_entropy(logits_per_image, labels)  # 每行 softmax,目标=对角
    loss_t2i = F.cross_entropy(logits_per_text,  labels)  # 每列 softmax,目标=对角
    return (loss_i2t + loss_t2i) / 2
```

把这段对照 §3.3 的公式逐行验证:`cross_entropy(logits, labels)` 内部就是先 `log_softmax` 再取目标类的负对数——正是 `−log[exp(Lᵢᵢ)/Σ exp(Lᵢₖ)]`。`labels = arange(N)` 把"第 i 行的目标是第 i 列"直接编码成对角下标。**全部对比逻辑就这么多;CLIP 的难度从来不在损失公式,而在数据规模、大 batch 工程和两塔的预训练。**

logit_scale 的可学习封装(训练时):

```python
import numpy as np, torch.nn as nn
# 初始化成 log(1/0.07) ≈ 2.659,即初始温度 0.07(CLIP 默认值)
logit_scale = nn.Parameter(torch.tensor(np.log(1 / 0.07)))
# 前向:scale = logit_scale.exp().clamp(max=100)   # 上限防爆炸(§3.2)
```

### 4.2 零样本分类(推理流程)

零样本分类的精髓:**把每个类名写成一句话,编码成文本向量,这些文本向量就直接当分类头的权重用**。

```python
@torch.no_grad()
def zero_shot_classify(model, image, class_names, templates, device):
    """
    无需在目标数据集上做任何训练 —— 这就是 zero-shot(§5 解释为什么)
    class_names: 如 ["cat", "dog", "car", ...]  (C 个类)
    templates:   prompt 模板,如 ["a photo of a {}.", "a blurry photo of a {}."]
    """
    # 1) 为每个类构造文本嵌入 = 该类的"分类器权重向量"
    zeroshot_weights = []
    for cname in class_names:
        texts = [t.format(cname) for t in templates]          # prompt ensemble
        tokens = model.tokenize(texts).to(device)             # [T, L]
        txt = model.encode_text(tokens)                       # [T, d]
        txt = F.normalize(txt, dim=-1)                        # 各模板归一化
        txt = txt.mean(dim=0)                                 # 跨模板平均(集成)
        txt = F.normalize(txt, dim=-1)                        # 平均后再归一化
        zeroshot_weights.append(txt)
    W = torch.stack(zeroshot_weights, dim=1)                  # [d, C], 第 c 列=类 c

    # 2) 图像编码 + 归一化
    img = F.normalize(model.encode_image(image), dim=-1)      # [B, d]

    # 3) 相似度 = 图像向量与每个类文本向量的余弦,取最大 → 预测类别
    logits = img @ W                                          # [B, C], 余弦相似度
    return logits.argmax(dim=-1)                              # [B]
```

**第 1 步是整个零样本魔法的物理位置**:`W ∈ ℝ^{d×C}` 的第 c 列就是"a photo of a {类c}"的文本嵌入。它在形状和用法上**和监督训练里那个线性分类头 `W ∈ ℝ^{C×d}` 一模一样**(转置而已)。区别只在于——**监督分类头是反向传播训出来的,CLIP 的"分类头"是文本编码器现场算出来的**。改类别集只需改 `class_names` 重算一次 W,模型一个参数都不动。这就是开放词表的实现机制。

## 5. 为什么零样本不需要微调(核心追问)

这是本章最该想透的一个问题。新手常觉得"没训练过这个数据集还能分类"很玄,其实拆开看一点都不玄。

**关键认知一:CLIP 预训练学的不是"某些类",而是"一个图文共享语义空间"。** 预训练阶段,4 亿对图文把图像球面和文本球面对齐成同一个球面:语义相近的图和文落在相近方向。模型从没见过 ImageNet 的 1000 个类的标签划分,但它见过海量"狗的图 + 含 dog 的文本"——于是**"狗"这个概念在球面上的位置已经被学好了**,无论你后来叫它第几类。

**关键认知二:推理时的"分类头"是临时用文本拼出来的,不需要训练。** 监督范式里分类头 `W` 必须见过目标类的标注图、靠梯度训出来。CLIP 把这一步**外包给了文本编码器**:类名 → 一句话 → 文本向量,**这个向量天然就在已对齐好的共享空间里**,可以直接和图像向量比余弦。**没有"针对目标数据集的参数"需要学,因为分类边界是由文本现场定义的,不是由训练数据拟合的**。

**关键认知三:从信息论角度,自然语言把"标签"做了语义压缩与泛化。** one-hot 标签 207 不携带任何可迁移信息;而文本"a photo of a golden retriever"携带了"golden""retriever""dog(隐含)""a photo of(摄影场景)"等可组合、可泛化的语义碎片。**这些碎片在预训练里都被见过、被对齐过**,所以哪怕"golden retriever"作为整体没在某个评测集出现,它的组成部分早已在球面上各就各位。**零样本能力 = 语义碎片的组合泛化能力**,这是自然语言监督相对 one-hot 监督的本质增益。

形式化地说:监督分类在学 `P(y | x)`,其中 y 是固定 C 类;CLIP 在学一个**相似度函数 `s(图, 文)`**。分类不过是这个相似度函数的一个**下游应用**:

```
零样本预测 = argmax_c  s(图, prompt(类c))
           = argmax_c  uᵢ · v_c          (余弦)
```

**分类被重新表述成"在候选文本里检索最相似的那个"**。候选文本可以是任意类名集合——这就是为什么同一个 CLIP 不动一个参数,既能在 ImageNet 上分 1000 类,也能在卫星图、医学、卡通数据集上零样本分类(精度各异,但机制相同)。

> 一句话收口:**监督模型把"答案"烧进了权重,CLIP 把"答案"留在了语言里。所以换问题(换类别集)不用重训,只需换一句话。**

## 6. 影响、局限与去向

### 6.1 规模与数据质量是第一性变量

CLIP 的能力高度依赖**数据规模与多样性**,而非精巧的损失设计(损失就是 §4 那几行)。论文里 zero-shot 精度随数据量、模型规模平滑上升,呈现 scaling 行为。后续 OpenCLIP / LAION 的工作进一步确认:**在固定算法下,换更大更干净的数据集(如 LAION-2B、DataComp 筛选过的数据)就能稳定涨点**。这条经验对你做任何对比预训练都成立——**先怀疑数据,再怀疑算法**。

### 6.2 CLIP 弱在哪里,为什么弱(必须讲透)

CLIP 不是万能的,它的弱点**直接来自"全局对比 + 网络弱文本监督"这套机制**,不是实现 bug:

| 弱项 | 现象 | 机制层面的根因 |
|---|---|---|
| **细粒度区分** | 难分近似品种(specific 鸟种/机型/车款) | 全局图文对比只逼"整体语义对齐",**不提供区分细粒度所需的局部判别梯度**;且网络 caption 很少精确到细粒度名 |
| **计数** | "三只猫"常被当成"猫" | 文本编码器把句子压成一个全局向量,**数量信息在池化中被稀释**;训练中"猫"出现与数量无强对齐监督 |
| **空间关系** | "杯子在书的左边" vs "右边"难区分 | 同上,全局向量是**词袋式(bag-of-words)倾向**的——CLIP 文本表示对词序/空间介词不敏感,常把句子当无序词集 |
| **OCR/精确文字** | 长文本、罕见字符读不准 | 视觉编码器学的是语义级特征,**非字符级**;网络图文对里文字内容与 caption 对齐稀疏 |
| **抽象/合成推理** | 反事实、属性绑定(红色立方体 vs 蓝色球) | 对比目标只要求"整体配得上",**不强制属性-对象的正确绑定**(binding problem) |

**一条贯穿性的根因:CLIP 的训练目标是"图与整句话的全局匹配",这必然把表示推向"够用来全局区分即可"的捷径,而抛弃计数、空间、属性绑定这些需要结构化/局部理解的信息**——因为这些信息对降低对比损失的边际帮助不大(batch 里的负样本大多在粗语义上就已经能区分开)。理解这一点,你就能预判:**任何需要细粒度/结构化视觉理解的下游任务,纯 CLIP zero-shot 都会偏弱,需要后续的局部对齐(如 region-text)或接生成式 VLM 来补。**

### 6.3 prompt 敏感性

零样本精度对 prompt 写法敏感——`"{class}"` vs `"a photo of a {class}."` 在 ImageNet 上能差好几个点(具体幅度「待核」,但"prompt 工程显著影响 zero-shot"是论文明确结论)。原因:**文本编码器是在"自然句子"分布上训练的,孤零零一个类名是分布外(OOD)的输入,而 "a photo of a ___" 把它拉回到训练时见过的句式分布**。实践上用 **prompt ensemble**(§4.2 里平均多个模板的文本向量)稳定且涨点——本质是在文本嵌入空间做平滑/降方差。这是 CLIP 工程化的标准动作,也是一个值得警惕的脆弱性:**你的零样本"性能"一部分是 prompt 调出来的,报告结果时必须说明用了什么 prompt。**

### 6.4 去向:CLIP 是三条主线的地基

CLIP 的视觉-语言对齐空间是后续多个方向的入口:

- **开放词表检测/分割**:把"类名文本向量"当检测/分割头的查询(GLIP、Grounded-SAM、OWL-ViT 等),实现"用文字检测/分割任意概念"。机制延续本章:**文本当可替换的类别原型**。
- **VLM(视觉语言模型,如 LLaVA 范式)**:**把 CLIP 的图像编码器冻住或微调,接一个投影层把图像 token 投到 LLM 的词嵌入空间,喂给 LLM**。CLIP 视觉特征已经"语言对齐",所以 LLM 接得上、少量指令数据就能学会"看图对话"。这是当前多模态大模型的主流接法之一(细节后续章节)。
- **文生图(text-to-image)**:CLIP 的文本编码器(或其 image/text 对齐空间)被用作**文本条件**——早期 DALL·E 2 用 CLIP 文本嵌入做条件并训一个 prior;Stable Diffusion 用 CLIP/OpenCLIP 文本编码器给 U-Net 提供 cross-attention 条件。**"文本指导生成"的那个文本向量,常常就来自 CLIP 谱系的文本塔**(扩散模型那章细讲)。

一句话:**CLIP 把"图像"和"语言"焊到同一个空间后,语言就成了视觉系统的通用遥控器——检测、分割、对话、生成都可以用一句话来驱动。**

## 7. 设计权衡与常见坑

- **L2 归一化漏了或只归一边**。两塔嵌入必须都归一化再点积,否则相似度退化成裸点积,模型靠拉模长作弊、训练 collapse。新手常忘 `F.normalize`。
- **温度当超参写死,或方向搞反**。CLIP 是**可学习温度**且用 `exp(τ)` 做正向缩放;别和 SimCLR 那种"除以固定 τ"混用方向。记住:**乘 exp(τ) 是放大 logits(等价于除以小温度)**。还要 clamp 上限(默认 100)防爆炸。
- **只做单向损失**。只算图→文(行 CE)会让检索一个方向好、另一个方向塌。**必须行、列各一次 CE 取平均**(§3.3)。
- **batch 太小**。负样本数 ≈ N−1,batch 一小对比信号就弱,零样本质量塌方。CLIP 级别要大 batch + 分布式 gather 负样本;小卡复现请用 gradient checkpointing + gather,而不是只在本卡内算 logits(否则负样本只有本卡那一小撮)。
- **文本表示取错位置**。CLIP 取 **[EOS] token** 的最后层激活当整句表示(配合 causal mask,EOS 能看到全句);取 mean-pooling 或 [CLS] 是另一种约定,**和训练时不一致就废**。复现别想当然。
- **prompt 不一致**。训练/评测/部署用不同 prompt 会让数字不可比。固定 prompt(或 prompt ensemble)并写进报告。
- **拿 zero-shot 当上限**。同一 CLIP,**linear probe(冻结特征训个线性头)或 fine-tune 通常显著高于 zero-shot**;但 fine-tune 会损失部分鲁棒性/开放词表能力(分布偏移下反而可能变差,论文有讨论)。按任务选,别默认 zero-shot 是天花板。
- **误以为 CLIP 懂计数/空间/属性绑定**(§6.2)。这是机制性短板,别用 prompt 硬怼;需要结构化理解就换/接更强的模型。
- **数据泄漏与评测可信度**。4 亿网络数据可能已包含某些评测集的近似样本,**"零样本"未必对评测集完全 unseen**。读 benchmark 数字要带这层警惕。

## 8. 动手练习

1. **(推导题)双向对称 InfoNCE 的梯度。** 从 `L_CLIP = ½[CE_row + CE_col]` 出发,推出对 logit `Lᵢⱼ` 的总梯度 `∂L_CLIP/∂Lᵢⱼ`(提示:行 CE 贡献 `½·(1/N)·(pᵢⱼ−𝟙[j=i])`,其中 `p` 是第 i 行 softmax;列 CE 贡献 `½·(1/N)·(qᵢⱼ−𝟙[i=j])`,其中 `q` 是第 j 列 softmax;两者相加)。再说明:对角项梯度为负(被拉大)、非对角为正(被压小),据此论证为什么收敛后相似度矩阵趋于"对角线极大、其余极小"。

2. **(推导题)温度的作用与上限。** 设余弦相似度 ∈ [−1,1],缩放后 logit = s·exp(τ)。(a) 论证当 exp(τ)→0 时,任意 batch 的 softmax → 均匀分布、交叉熵 → log N、梯度趋零(学不动);(b) 论证 exp(τ) 过大时,softmax 几乎 one-hot,梯度集中在极个别最难负样本上、方差大、训练不稳。(c) 由此解释 CLIP 为什么用可学习 τ 且 clamp 上限。*提示*:把 softmax 写成 `exp(scale·sₖ)/Σ`,对 scale 取两个极限。

3. **(编码题)从零搭一个 mini-CLIP 并验证零样本。** 用 torchvision 的小 ResNet 当图像塔、一个浅层 Transformer 当文本塔(或直接用 HuggingFace tokenizer + nn.Embedding + 几层 TransformerEncoder),在一个小图文数据集(如 Flickr8k 子集)上跑 §4.1 的对称对比损失训几百步;然后用 §4.2 流程在一组类名上做零样本分类。*提示*:先在玩具上确认"对角线相似度随训练上升、非对角下降"(把相似度矩阵可视化成热力图),再谈精度;batch 尽量开大;别忘 normalize 和可学习 logit_scale。

4. **(分析题)定位 CLIP 的弱项。** 取一个预训练 CLIP(open_clip),构造三组对照探针:(a) 细粒度——同一物种的近似子类名;(b) 计数——"one/two/three {obj}"配真实数量不同的图;(c) 空间——"A left of B" vs "A right of B"配同一对物体左右互换的图。报告各组 zero-shot 准确率,并用 §6.2 的机制(全局对比 + 词袋倾向)解释观察到的失败模式。*提示*:计数/空间组要控制变量,只改数量/方位,其余尽量一致,才能把失败归因到机制而非图像差异。

## 10. 小结与承上启下

这章把 CLIP 从**闭集监督的诅咒**讲起,推到 **batch 内 N×N 相似度矩阵上的双向对称 InfoNCE**:你现在应该能在白板上写出对称对比损失、推出"对角拉大、非对角推开、难负样本梯度最大"的梯度结构、说清 L2 归一化和可学习温度 exp(τ) 各自消掉/补回了什么自由度;能徒手实现对比损失和零样本推理;能从"答案在权重 vs 答案在语言"的角度讲透零样本为什么不用微调;也能用"全局对比 + 词袋倾向"这一条机制根因预判 CLIP 在细粒度、计数、空间关系上的系统性短板。

**这章在整门课里的位置**:前面你已经吃透了 CNN 主干、ViT、以及自监督的对比学习(MoCo/SimCLR);CLIP 是把**对比学习从"图-图"扩展到"图-文"**的临门一脚——它用自然语言监督打通了视觉与语言两个模态,得到的对齐空间正是**开放词表(检测/分割)、VLM(看图对话)、文生图(文本条件)**这三条后续主线的共同地基。

**承上启下**:本章的对齐空间是"静态"的——图和文各编码一次、点积一下。但很多任务需要**图文在网络内部深度交互**(看图回答细粒度问题、定位文字指代的具体区域),纯双塔的全局向量不够(§6.2 的根因)。下一阶段进入**视觉语言模型(VLM)**:如何把 CLIP 的视觉编码器接到 LLM 上、投影层在做什么、为什么"CLIP 视觉特征已语言对齐"是这套接法能 work 的前提。把本章的"图文共享空间"吃透,VLM 不过是"在这个空间的图像端,换一个更强的、能交互推理的语言解码器"——而那个空间能存在,正是你此刻已经推透的这套对比对齐机制的功劳。
