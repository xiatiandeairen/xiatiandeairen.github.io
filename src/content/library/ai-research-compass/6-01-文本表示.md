> 这一章把你从"我有一堆文本,机器只认数字"带到"我能亲手把词变成有语义的向量,推出 word2vec 负采样损失的每一步、解释 GloVe 为什么去拟合共现的对数、说清 king−man+woman≈queen 这种类比为什么会成立,并用 PyTorch 跑一个能做类比检索的 Skip-gram"。这是整个 NLP 的第 0 步——**表示决定了上限**。

## 引言:NLP 的入口问题

神经网络只吃数字。但语言是离散符号:词、字、标点。所以 NLP 的第一个、也是最根本的问题是——**怎么把一个词变成一个向量**(word embedding,词嵌入)?

这个问题听起来像预处理细节,实际上它**框定了整门学科的天花板**。如果你的表示里"猫"和"狗"的距离,跟"猫"和"积分"的距离一样远,那无论后面的模型多强,它都得从这堆毫无结构的输入里重新学出"猫和狗相关"——把宝贵的容量浪费在重新发明语义上。反过来,如果表示本身就把"语义相近"编码成"几何相近",模型一上来就站在更高的起点。

这一章的逻辑链条:

1. **符号表示**(one-hot / 词袋 / TF-IDF):为什么它们"能用但没语义",根本缺陷是**任意两词等距**。
2. **分布式假说**(Firth):词义由它的上下文决定——这是把"无语义符号"升级成"有语义向量"的钥匙。
3. **word2vec**:从 Skip-gram 的原始 softmax 目标(代价 `O(V)`)推到**负采样**(把全词表 softmax 近似成"正样本 + k 个负样本的二分类"),逐行推导损失。
4. **为什么向量会编码线性类比**(king−man+woman≈queen)。
5. **GloVe**:为什么改去拟合**全局共现矩阵的对数**,加权最小二乘 `J` 的每一项怎么来,与 word2vec(局部窗口)的对比。
6. **上下文相关表示(ELMo)**:一词多义击穿了"一词一向量",双向 LSTM 给出随上下文变化的向量,这是通往 BERT 的前奏。

> **与其它课的分工**:本课讲 NLP 特有的表示问题与序列方法。Transformer 内部的注意力、子词分词(BPE/WordPiece)、嵌入查表与 weight tying 的工程细节,**详见《大模型算法专家课程》第 01、02 章**,这里不重复;ELMo 之后的 BERT 预训练-微调范式细节,留到本课后续章节。

## 一、符号表示:能用,但没有语义

### 问题:怎么给词编号

最朴素的想法:给词表里每个词一个编号。词表 `V = {猫, 狗, 苹果, 跑, ...}`,大小记作 `V`(我们用同一个字母表示词表和它的大小,靠上下文区分)。"猫" = 0,"狗" = 1……

但**整数编号不能直接喂模型**:它隐含了一个虚假的序关系。编号让"狗(1)"在数值上比"积分(9527)"更靠近"猫(0)",可这毫无语义依据,编号只是任意排的。模型若把这个数值当特征,会学到一堆假规律。

### 机制:one-hot——去掉虚假序关系

解决办法是 **one-hot 编码**:把第 i 个词表示成一个长度为 `V` 的向量,第 i 位是 1,其余全 0。

```
V = 5,词表 = [猫, 狗, 苹果, 跑, 红]
猫  = [1, 0, 0, 0, 0]
狗  = [0, 1, 0, 0, 0]
苹果 = [0, 0, 1, 0, 0]
```

one-hot 的几何性质,正是它的全部优点也是全部缺陷:**任意两个不同词的 one-hot 向量两两正交**。

```
点积(衡量相似度):
  猫 · 狗  = 1·0 + 0·1 + 0·0 + ... = 0
  猫 · 苹果 = 0
  任意 i≠j:  e_i · e_j = 0
欧氏距离:
  ‖e_i − e_j‖ = √(1² + 1²) = √2   (对任意 i≠j 都一样)
```

**关键结论:在 one-hot 空间里,任意两个不同词的距离恒等于 √2,相似度恒等于 0。** "猫"和"狗"的距离 = "猫"和"积分"的距离。这就是符号表示的根本病:**它把所有词放在一个正交单纯形的顶点上,词与词之间没有任何远近结构**。语义信息一点没编码进去。

还有两个工程代价:**维度 = 词表大小**(几万到几百万),向量极度稀疏;新词加入要扩维。

### 机制:词袋与 TF-IDF——给文档而非词的表示

one-hot 是给**单个词**的。给一篇**文档**怎么表示?最简单的是 **词袋(Bag of Words, BoW)**:把文档里所有词的 one-hot 加起来(或计数),得到一个长度 `V` 的计数向量。

```
文档 d = "猫 追 猫 狗"
BoW(d) = [2, 1, 0, ...]   (猫出现2次,狗1次,追1次...)
```

"袋"这个名字点出了它的根本假设:**忽略词序**。"狗咬人"和"人咬狗"的词袋完全相同。这对很多任务(主题分类、信息检索)够用,但丢掉了一切顺序信息。

词袋用原始计数有个明显问题:**高频虚词(的、是、a、the)在每篇文档里都很多,数值最大,但最没有区分力**。我们想要的是"既在这篇文档里常见、又不是到处都常见"的词。这就引出 **TF-IDF(Term Frequency–Inverse Document Frequency)**。

它是两个因子的乘积。**TF(词频)**衡量词 t 在文档 d 里有多突出:

```
tf(t, d) = (t 在 d 中出现次数) / (d 的总词数)     # 一种常见归一化
```

**IDF(逆文档频率)**衡量词 t 在整个语料里有多"稀有"。设语料共 `N` 篇文档,含词 t 的文档数为 `df(t)`(document frequency)。IDF 定义为:

```
idf(t) = log( N / df(t) )
```

**为什么是 log(N/df)?给出推导动机(不是天上掉下来的公式)。** 从信息论看:如果随机抽一篇文档,它含词 t 的概率近似 `df(t)/N`。一个事件概率为 p,它发生带来的**自信息(信息量)是 −log p**(越罕见的事发生,信息量越大)。把 t 出现看成"事件",则它携带的信息量就是:

```
−log( df(t)/N ) = log( N / df(t) ) = idf(t)
```

所以 **IDF 本质是"词 t 出现这件事的自信息量"**——出现得越普遍(df 大),信息量越小,IDF 越小;**极端情形:一个词在所有文档里都出现(df = N),idf = log 1 = 0,被完全压制**。这正是我们要的效果:虚词被自动归零,稀有而专指的词被放大。

完整公式:

```
tfidf(t, d) = tf(t, d) · log( N / df(t) )
```

实践里常用平滑版 `idf = log( N / (1 + df(t)) ) + 1` 防止除零和负值(`df=N` 时 `log(N/(N+1))<0`),细节看 scikit-learn 的 `TfidfVectorizer` 文档,但**核心是 log(N/df) 这个"用对数自信息抑制常见词"的思想**。

### 分析:符号表示的天花板

| 表示 | 维度 | 编码了什么 | 根本缺陷 |
|---|---|---|---|
| one-hot(词) | V | 仅身份 | 任意两词等距、正交 |
| 词袋(文档) | V | 词频,无序 | 丢词序;高频虚词主导 |
| TF-IDF(文档) | V | 加权词频 | 仍无序;**词与词之间仍无语义关系** |

注意一个贯穿始终的病灶:**这三者里,词本身永远是 one-hot 基,词与词之间的相似度恒为 0**。TF-IDF 只是在文档层面重新加权,它**没有改变"猫和狗在向量空间里毫无关系"这个事实**。要让"猫"和"狗"靠近,必须换一套完全不同的思路——让词义来自上下文。

## 二、分布式假说:词义藏在上下文里

### 问题:语义从哪来

我们想要的是"猫"和"狗"的向量靠近、"猫"和"积分"的向量远离。但**语义不在词本身**——"猫"这个符号本身没有任何内在含义,它的含义完全是约定俗成的。那含义从哪观测?

语言学家 **J.R. Firth(1957)**给了一句被无数次引用的话:**"You shall know a word by the company it keeps"(观其伴,知其义)**。这就是**分布式假说(distributional hypothesis)**:**出现在相似上下文里的词,语义相似**。

直觉验证:你没见过"wug"这个词,但看到"我家的 wug 喵喵叫,爱抓老鼠,讨厌洗澡",你立刻知道 wug ≈ 猫。因为"喵喵叫/抓老鼠/洗澡"这套上下文,平时只跟"猫"共现。**词义被它的上下文分布定义了**。

### 机制:从"分布"到"向量"

把假说操作化:**用一个词周围出现哪些词的统计分布,作为它的表示**。最直接的做法是**共现矩阵(co-occurrence matrix)**:统计每个词 i 在每个上下文词 j 的窗口里出现多少次,得到矩阵 `X`,`X_ij` = 词 j 出现在词 i 的上下文窗口里的次数。`X` 的第 i 行,就是词 i 的(高维、稀疏)分布式表示。

这一步已经突破了 one-hot:现在"猫"行和"狗"行会很像(都跟喵/抓/喂这些词共现),而"猫"行和"积分"行很不一样。**词与词之间第一次有了远近结构**。

但 `X` 还是 `V×V` 的巨大稀疏矩阵。我们想要的是**低维稠密**(几百维)向量,既省空间,又能泛化(把相关的统计模式压缩到共享的几个方向上)。怎么从共现统计得到稠密低维向量?历史上两条路:

- **计数派(count-based)**:直接对 `X`(或其变换)做矩阵分解/降维。经典是 **LSA(潜在语义分析)**对 TF-IDF 矩阵做 SVD。**GloVe** 是这一派的现代代表(第五节)。
- **预测派(prediction-based)**:不显式建 `X`,而是训练一个模型去**预测**"上下文",把模型学到的参数当向量。**word2vec** 是这一派的代表,也是下一节的主角。

这两派看似对立,后面会看到它们在数学上**深度相通**——word2vec 隐式地在分解一个与共现相关的矩阵。

## 三、word2vec:用预测上下文学出向量

word2vec(Mikolov 等,2013)不是一个模型,而是一套高效训练词向量的方法,有两个结构:**CBOW**(用上下文预测中心词)和 **Skip-gram**(用中心词预测上下文)。我们重点推 **Skip-gram**,因为它的目标更干净、在罕见词上更好,也是负采样推导的标准载体。

### 问题:Skip-gram 的原始目标

设一句话是词序列 `w₁, w₂, ..., w_T`。Skip-gram 的任务:**给定中心词 wₜ,预测它窗口内的每个上下文词**。窗口半径 m,则中心词 wₜ 要预测 `w_{t−m}, ..., w_{t−1}, w_{t+1}, ..., w_{t+m}`。

每个词有**两个**向量(这是 word2vec 一个常被忽略但很关键的设计):

- `v_w`:词 w 作为**中心词**时的向量(input/center 向量)
- `u_w`:词 w 作为**上下文词**时的向量(output/context 向量)

(为什么要两套向量?直觉:一个词作为"被预测对象"和作为"预测者"扮演不同角色;数学上分开能让目标更好优化,且避免一个词预测自己时的退化。训练完通常只用 `v_w`,或两者相加。)

给定中心词 c,模型给上下文词 o 的概率用 softmax 定义:

```
            exp( u_o · v_c )
P(o | c) = ─────────────────────        ← 对全词表归一化
           Σ_{w∈V} exp( u_w · v_c )
```

直觉:**中心向量 v_c 和上下文向量 u_o 点积越大,概率越高**;分母把所有词的得分归一成概率分布。

整个语料的目标:最大化所有(中心词,上下文词)对的对数似然。等价于最小化平均负对数似然:

```
        1   T      
L = − ─── Σ    Σ        log P(w_{t+j} | w_t)
        T  t=1   −m≤j≤m, j≠0
```

### 推导:原始 softmax 的致命代价 O(V)

看那个 softmax 的分母:`Σ_{w∈V} exp(u_w · v_c)`。**每算一个 `P(o|c)`、每走一步梯度,都要遍历整个词表 `V`**。我们来精确地算梯度代价。

对中心向量 `v_c` 求 `log P(o|c)` 的梯度(这是理解一切的关键一步,逐行推):

```
log P(o|c) = u_o · v_c − log Σ_{w} exp(u_w · v_c)

∂/∂v_c [ u_o · v_c ] = u_o

∂/∂v_c [ log Σ_w exp(u_w·v_c) ]
   = ( 1 / Σ_w exp(u_w·v_c) ) · Σ_w exp(u_w·v_c) · u_w     ← 链式法则
   = Σ_w  [ exp(u_w·v_c) / Σ_{w'} exp(u_{w'}·v_c) ] · u_w
   = Σ_w  P(w|c) · u_w                                       ← 就是 softmax 概率

所以:
∂ log P(o|c) / ∂v_c = u_o − Σ_{w∈V} P(w|c) · u_w
                    = u_o − E_{w~P(·|c)}[ u_w ]              ← 观测上下文 − 期望上下文
```

**这个梯度的含义很美:把中心向量推向真实上下文 u_o,同时拉离"模型当前认为可能的所有上下文的加权平均"。** 但代价写在脸上:**那个求和 `Σ_{w∈V}` 跑遍整个词表,单步梯度 `O(V)`**。词表几十万时,每个训练样本都要做几十万次点积和指数——**完全跑不动**。

这就是 word2vec 真正要解决的工程问题:**怎么避免那个 O(V) 的全词表归一化**。两个方案:层次 softmax(用霍夫曼树把 O(V) 降到 O(log V),这里不展开)和**负采样**——后者更简单、更主流,我们完整推导。

### 推导:负采样——把多分类换成二分类

**核心洞察:我们其实不需要一个标准的概率分布。** 我们真正想要的,是让向量满足"真实共现的(中心,上下文)对得分高,随机配的对得分低"。那干脆**别做 V 类 softmax,改做一堆二分类**:

把问题重述为:给一对 `(c, o)`,判断它是不是**真实从语料里采到的共现对**(标签 D=1),还是**随机凑的负样本**(D=0)。用 logistic(sigmoid)做这个二分类:

```
σ(x) = 1 / (1 + e^{−x})            # sigmoid

P(D=1 | c, o) = σ( u_o · v_c )      # 这对是真实共现的概率
P(D=0 | c, o) = 1 − σ( u_o·v_c ) = σ( −u_o·v_c )   # 用了 1−σ(x)=σ(−x)
```

(`1 − σ(x) = σ(−x)` 是 sigmoid 的恒等式,验证:`1 − 1/(1+e^{−x}) = e^{−x}/(1+e^{−x}) = 1/(e^{x}+1) = σ(−x)`。)

训练时,对每个真实对 `(c, o)`(正样本),我们额外从一个**噪声分布 `P_n(w)`** 采 `k` 个负样本词 `w₁, ..., w_k`,凑成 `k` 个负对 `(c, w_i)`。目标:**让正对的 P(D=1) 大,让每个负对的 P(D=0) 大**。对单个中心-上下文对 `(c, o)`,负采样的损失(对它取负对数似然)是:

```
L_NEG(c, o) = − log σ( u_o · v_c )  −  Σ_{i=1}^{k} log σ( − u_{w_i} · v_c )
                  └── 正样本拉高 ──┘     └──── k 个负样本压低 ────┘
              其中 w_i ~ P_n(w)
```

写成期望形式(论文里的标准写法,题面给的就是这个):

```
L = − log σ( v_c · u_o )  −  Σ_{i=1}^{k} E_{w_i ~ P_n}[ log σ( − v_c · u_{w_i} ) ]
```

**这就是题目要求推导的负采样目标。来历讲清楚了**:它不是凭空设计的,而是把"V 类 softmax"换成"1 个正例 + k 个负例的逻辑回归"。这背后是 **NCE(Noise Contrastive Estimation,噪声对比估计)** 的思想——**与其学一个归一化的概率分布,不如学一个能把真实数据和噪声区分开的二分类器**;负采样是 NCE 的一个简化变体(去掉了归一化常数项,纯当判别问题)。

**为什么高效?** 算这个损失和它的梯度,只碰 `1 + k` 个词(`o` 和 `k` 个负样本),**与 V 无关**,代价从 `O(V)` 降到 `O(k)`(k 通常 5~20)。我们验证梯度的稀疏性,对中心向量 `v_c` 求梯度:

```
∂L_NEG/∂v_c = (σ(u_o·v_c) − 1)·u_o  +  Σ_{i=1}^{k} (σ(u_{w_i}·v_c) − 0)·u_{w_i}
            = (σ(u_o·v_c) − 1)·u_o  +  Σ_{i=1}^{k} σ(u_{w_i}·v_c)·u_{w_i}
```

(用到 `d/dx[−log σ(x)] = σ(x)−1` 和 `d/dx[−log σ(−x)] = σ(x)`,可自行验算。)

含义清晰:**正样本项把 v_c 推向 u_o(因为 σ−1<0,负梯度方向是 +u_o);负样本项把 v_c 推离每个 u_{w_i}**。**这一步只涉及 1+k 个上下文向量,梯度天然稀疏**。这就是 word2vec 能在十亿词级语料上几小时跑完的原因。

**噪声分布 `P_n(w)` 怎么选?** word2vec 论文用了一个经验上很有效的选择:**词频的 3/4 次方**:

```
P_n(w) = U(w)^{3/4} / Σ_{w'} U(w')^{3/4}
         其中 U(w) 是词 w 的 unigram 频率(出现频次占比)
```

**为什么是 3/4?** 直觉:若直接按词频采(指数=1),高频虚词("的""the")会被反复当负样本,信息量低;若均匀采(指数=0),又会过度采到几乎不出现的稀有词。**3/4 是一个"压一压高频、抬一抬低频"的折中**——它让高频词的采样概率相对下降、低频词相对上升。这个指数没有漂亮的理论推导,是**经验最优**(论文实验试出来的),记住它是个调过的超参即可。

### 一个并行技巧:高频词下采样

word2vec 还有个小但重要的预处理:**对极高频词(the, a, 的)以一定概率丢弃**(subsampling),概率约 `1 − √(t/f(w))`(t 是阈值,如 1e-5)。理由同上——高频虚词提供的信息少,且会稀释罕见词的训练信号。这和 IDF "抑制常见词"是同一种直觉的不同实现。

## 四、为什么向量会编码线性类比

word2vec 最惊艳的现象:**king − man + woman ≈ queen**,**Paris − France + Italy ≈ Rome**。向量加减居然对应语义类比。这不是设计出来的,是训练**涌现**出来的。为什么?

**给一个站得住的论证(这是研究界的主流解释,非严格定理)。** 考虑"性别"这组类比。在分布式假说下,`king` 和 `man` 的上下文差异、`queen` 和 `woman` 的上下文差异,**主要都来自同一个语义维度——性别**(国王 vs 王后的区别,本质和男人 vs 女人的区别是同一回事:王权语境相同,性别相反)。

更形式化一点,看共现比率。设某个上下文词 `w`,考虑它跟 king/man/queen/woman 共现概率的比值。**如果 king 和 man 之于性别的关系,等同于 queen 和 woman 之于性别的关系**,那么对任意 w:

```
P(w | king) / P(w | man)  ≈  P(w | queen) / P(w | woman)
```

(意思是:任何上下文词区分"king vs man"的倾向,和区分"queen vs woman"的倾向一致——因为区分它们的语义轴是同一个"王权轴",而性别轴被约掉了。)

下一节会看到,word2vec/GloVe 学到的向量,**点积近似于对数共现**:`v_i · u_j ≈ log P(j|i) + 常数`。把上面的比率关系取对数:

```
log P(w|king) − log P(w|man) ≈ log P(w|queen) − log P(w|woman)

⟹ (v_king − v_man) · u_w ≈ (v_queen − v_woman) · u_w   对所有 w 成立
```

**对所有上下文向量 u_w 都成立的等式,意味着两个差向量近似相等**:

```
v_king − v_man ≈ v_queen − v_woman
⟹ v_king − v_man + v_woman ≈ v_queen
```

**这就是线性类比的来源:在配对 `(king,man)/(queen,woman)` 这条推导链里,两个差向量 `v_king − v_man` 与 `v_queen − v_woman` 编码的都是同一条把两侧区分开的语义轴——王权轴(两侧性别相同、只差王权),所以二者近似相等,移项即得 `v_king − v_man + v_woman ≈ v_queen`。(换一种等价配对 `(king,queen)/(man,woman)` 看,差向量编码的则是性别轴,得到的是另一条平行关系。)当两组词只在同一个轴上不同、其余轴相同时,差向量就近似平行,加减法就成立。** 

**坑(重要,新手易神化)**:这个性质是**近似且脆弱**的。它在频繁、规则、单一语义轴的类比(国家-首都、单复数、性别)上work得好;在罕见词、多义词、需要多个语义轴同时变化的类比上经常失败。后来的研究(Levy & Goldberg 等)还指出,经典 `3CosAdd` 类比检索里通常要**排除三个输入词本身**,否则答案常常退化成输入词之一——这说明类比能力有相当一部分是检索技巧的功劳,而非纯粹的向量代数。**别把 king−man+woman≈queen 当成 word embedding 理解了语义的证据,它只是一个漂亮但有限的几何巧合。**

## 五、GloVe:直接拟合全局共现的对数

### 问题:word2vec 浪费了全局统计

word2vec 是**局部窗口**方法:它一次只看一个窗口,逐个样本地做随机梯度下降,**从不显式利用"整个语料里 i 和 j 一共共现了多少次"这个全局统计量**。GloVe(Global Vectors,Pennington、Socher、Manning,2014)的出发点就是:**共现矩阵 `X` 里已经攒好了全部全局统计,为什么不直接拿它来训练?**

### 推导:从共现比率到对数拟合目标

GloVe 的推导起点是一个观察:**真正承载语义关系的,是共现概率的"比率",而不是概率本身**。还是用 ice / steam(冰/蒸汽)的经典例子。考察上下文词 w 与 ice、steam 的共现概率比 `P(w|ice)/P(w|steam)`:

```
w = solid(固体):  跟 ice 强、跟 steam 弱  ⟹ 比率 ≫ 1
w = gas(气体):    跟 ice 弱、跟 steam 强  ⟹ 比率 ≪ 1
w = water(水):    跟两者都强            ⟹ 比率 ≈ 1
w = fashion(时尚):跟两者都弱            ⟹ 比率 ≈ 1
```

**比率能区分相关维度(solid/gas)和无关维度(water/fashion),比原始概率信息量大。** 所以 GloVe 希望词向量去拟合这个比率结构。

推导(论文 Section 3 的简化版,体会思路):我们想找一个函数 F,让向量差编码比率:

```
F( (w_i − w_j) · w̃_k ) = P(k|i) / P(k|j)
```

(`w_i, w_j` 是中心向量,`w̃_k` 是上下文向量,用差和点积是为了让结构是线性的、对称的。)要让 F 把"减法"对应成"除法",自然取 **F = exp**(指数把差变商):

```
exp( (w_i − w_j) · w̃_k ) = P(k|i)/P(k|j)
⟹ exp(w_i·w̃_k) / exp(w_j·w̃_k) = P(k|i)/P(k|j)
⟹ exp( w_i · w̃_k ) = P(k|i) · 常数
⟹ w_i · w̃_k = log P(k|i) + 常数
            = log X_ik − log X_i + 常数      (因为 P(k|i)=X_ik / X_i)
```

那个 `log X_i`(词 i 的总共现数)和常数项,与 k 无关,**吸收进偏置项 `b_i`**;为对称再给上下文词加偏置 `b̃_k`。于是得到 GloVe 想满足的核心关系:

```
w_i · w̃_j + b_i + b̃_j  ≈  log X_ij
```

**这就回答了"为什么拟合 log 共现":因为把共现比率(承载语义)的结构用 exp 函数线性化后,点积自然对应的是对数共现,不是共现本身。** 直接拟合 `X_ij` 会被高频对的巨大数值主导;拟合 `log X_ij` 压缩了动态范围,且与上面的比率推导自洽。

### 推导:加权最小二乘与权重函数 f

把上面的"≈"变成可优化的损失,最直接是最小二乘:`Σ_{i,j}(w_i·w̃_j + b_i + b̃_j − log X_ij)²`。但有两个问题:(1) `X_ij=0` 时 `log 0 = −∞`,而共现矩阵极度稀疏(绝大多数对从不共现);(2) 所有共现对一视同仁,但**罕见共现(X_ij=1,2)往往是噪声,高频共现(X_ij 上万)又会主导损失**。

GloVe 用一个**加权函数 `f(X_ij)`** 一次解决两者:

```
J = Σ_{i,j=1}^{V}  f(X_ij) · ( w_i·w̃_j + b_i + b̃_j − log X_ij )²
```

权重函数 `f`:

```
       ⎧ (x / x_max)^α     若 x < x_max
f(x) = ⎨
       ⎩ 1                 若 x ≥ x_max

论文取 x_max = 100,α = 3/4
```

**逐条解释 f 的设计(这是 GloVe 的精华):**

- **`f(0) = 0`**:`X_ij=0` 的项权重为零,**直接从求和里剔除**,既避开 `log 0` 的发散,又让 `J` 只对**实际出现过的共现对**求和(稀疏,可高效计算)。这就是为什么 GloVe 训练只遍历非零项,而非整个 `V×V`。
- **`x < x_max` 时随 x 递增**:共现越多的对越可靠,给越大权重——但用 `^{3/4}` 让增长是**次线性**的,**不让高频对的权重无限膨胀**。
- **`x ≥ x_max` 时封顶为 1**:超高频对(如"的"和任何词)的权重**不再增长**,防止极少数超高频对垄断整个损失。

(注意:这里的 3/4 和 word2vec 负采样的 3/4 是**两个独立的、各自调出来的经验值**,数学上没有直接关系,别混淆。)

### 代码:GloVe 损失的核心(伪代码)

```python
# 假设已统计好共现:coo 是非零项列表 [(i, j, X_ij), ...]
# W, W_tilde: (V, d) 中心/上下文向量;b, b_tilde: (V,) 偏置
def glove_loss(i, j, x_ij, W, W_tilde, b, b_tilde, x_max=100, alpha=0.75):
    # 加权函数 f
    weight = (x_ij / x_max) ** alpha if x_ij < x_max else 1.0
    # 拟合目标:w_i · w̃_j + b_i + b̃_j ≈ log X_ij
    pred = (W[i] * W_tilde[j]).sum() + b[i] + b_tilde[j]
    diff = pred - math.log(x_ij)          # x_ij 必 >0,因为只遍历非零项
    return weight * diff * diff           # f(X_ij)·(...)²
# 最终向量:常取 W + W_tilde(两套向量相加,经验上更稳)
```

### 分析:word2vec(局部窗口) vs GloVe(全局统计)

| 维度 | word2vec(Skip-gram + 负采样) | GloVe |
|---|---|---|
| 数据视角 | **局部窗口**,逐样本 SGD | **全局共现矩阵** `X`,一次性统计 |
| 目标 | 预测式:区分真实共现 vs 噪声(NCE) | 拟合式:加权最小二乘拟合 log X_ij |
| 计算 | 在线流式,内存友好;每步 O(k) | 先扫一遍建 X(可能很大),再对**非零项**优化 |
| 利用统计 | 隐式(多次采样近似) | **显式**,全局信息一次到位 |
| 罕见对 | 靠采样,可能欠采 | f 函数显式降权,可控 |

**深层联系(重要)**:Levy & Goldberg(2014)证明,**Skip-gram + 负采样隐式地在分解一个矩阵——逐点互信息(PMI)矩阵减去 log k**:`v_i · u_j ≈ PMI(i,j) − log k`,其中 `PMI(i,j) = log[ P(i,j)/(P(i)P(j)) ]`。而 GloVe 显式拟合 `log X_ij`(经偏置项校正后也近似 PMI 结构)。**所以"预测派"和"计数派"殊途同归:两者本质都在对共现统计的某种对数变换做低秩分解。** 这个统一视角是理解词向量的关键收获——别被"预测 vs 计数"的表面对立迷惑。

实践选哪个?**两者效果接近,取决于语料和任务**;GloVe 训练前要先建共现矩阵(大语料下内存是瓶颈),word2vec 流式更省内存。今天你大概率两个都不会从头训——直接用预训练向量,或更可能直接上**上下文相关表示**(下一节)。但**理解它们是理解 BERT 之类模型为什么是进步的前提**。

## 六、上下文相关表示:一词多义击穿"一词一向量"

### 问题:静态向量的死穴

word2vec 和 GloVe 都是**静态(static)词向量**:**每个词,无论出现在什么句子里,都只有同一个向量**。这撞上语言的一个基本事实——**一词多义(polysemy)**:

```
"我在河 bank 边散步"        ← bank = 河岸
"我去 bank 取钱"            ← bank = 银行
"apple 很好吃"             ← apple = 水果
"apple 发布了新手机"        ← apple = 公司
```

静态向量被迫把这些含义**平均**成一个向量——`bank` 的向量是"河岸"和"银行"的某种混合,落在两者中间,**对哪个意思都不准**。这是分布式假说"一词一分布"的根本局限:它假设词义是固定的,而真实词义**随上下文动态变化**。

### 机制:ELMo——让向量随上下文变化

**ELMo(Embeddings from Language Models,Peters 等,2018)**给出了突破:**一个词的向量不再是查表得到的固定值,而是由一个读过整句话的模型动态算出来的**。

具体地,ELMo 预训练一个**双向语言模型(biLM)**:用多层 **LSTM**(长短时记忆网络,一种 RNN,本课后续章节会讲透其门控机制)。

- **前向 LSTM** 从左到右读句子,在每个位置预测下一个词:建模 `P(wₜ | w₁...w_{t−1})`。
- **后向 LSTM** 从右到左读,预测上一个词:建模 `P(wₜ | w_{t+1}...w_T)`。

对句子里第 t 个词,**两个方向、每一层 LSTM 在该位置的隐状态拼接/加权组合**,就是这个词在**这句话里**的向量。换个句子,隐状态变了,向量就变了。

```
静态(word2vec/GloVe):  vec("bank") = 查表,恒定
动态(ELMo):           vec("bank", 句子) = f( biLSTM 读完整句后在该位置的隐状态 )
                       → "河 bank 边" 和 "去 bank 取钱" 得到不同向量
```

**关键结论:ELMo 把"词 → 向量"从一次查表,升级成一次"读懂整句再为每个位置算向量"的前向计算。** 这第一次让向量**消解了一词多义**:同一个 `bank`,在不同句子里得到指向"河岸"或"银行"的不同向量。

ELMo 还确立了一个范式雏形(承上启下的关键):**预训练 + 下游使用**——biLM 在海量无标注文本上预训练好,下游任务**冻结或微调**这些上下文向量当特征。这正是后来 **BERT/GPT"预训练-微调"范式的前奏**;区别是 ELMo 用 LSTM 且只把向量当特征喂给下游模型,而 BERT 用 Transformer 且整个模型一起微调(详见本课后续 BERT 章节与《大模型算法课》)。

## 设计权衡与常见坑

把全章的权衡和坑集中收口:

- **表示决定上限,但要选对粒度。** one-hot/TF-IDF 在小数据、强基线、需要可解释(哪些词权重高)的任务(垃圾邮件、主题检索)上**依然实用且常常够好**,别一上来就上深度模型。**有语义 ≠ 总是更好**。
- **静态词向量的两套向量别用错。** word2vec/GloVe 训练有 center 和 context 两套;**用的时候 word2vec 常只取 center,GloVe 常取两者之和**。搞混会让相似度计算莫名变差。
- **OOV(未登录词)**:静态词向量是闭词表,训练没见过的词没有向量。**这是静态向量相比子词/字符模型的硬伤**(子词分词如何消灭 OOV,见《大模型算法课》第 01 章)。常见 hack:用一个 `<UNK>` 向量或子词平均,但都是退而求其次。
- **余弦相似度 vs 点积**:比较静态词向量语义相似度,**几乎都用余弦相似度**(`cos = a·b/(‖a‖‖b‖)`),因为词频会影响向量模长,余弦去掉模长只看方向更稳。但 word2vec/GloVe 训练目标里用的是**点积**——训练和评估用的度量不完全一致,这是个常被忽略的细节。
- **别神化线性类比**:king−man+woman≈queen 脆弱、依赖排除输入词的检索技巧、在多义/罕见/多轴类比上常失败。它是研究界用来探针(probe)向量结构的工具,**不是 embedding "理解"语义的证据**。
- **共现窗口大小是个语义旋钮(常被忽略)**:窗口小(±1~2)学到的更偏**句法/搭配**(syntactic,"快速地"修饰动词),窗口大(±5~10)学到的更偏**主题/话题**(topical,"足球"和"球迷")。调相似度任务时先想清楚你要哪种"相似"。
- **负采样 k 和噪声分布的 3/4 是超参,不是定理**:小语料 k 大些(5~20),大语料 k 小些(2~5);3/4 是经验值。**别把经验超参当成有理论必然性的常数去记。**
- **静态词向量正在被上下文模型取代,但不是被淘汰**:在算力/延迟受限、需要词级固定向量(如词典扩展、轻量检索召回的初筛)的场景,word2vec/GloVe/fastText 仍在用。**理解它们更重要的价值是:它是读懂 BERT 为什么是进步、RAG 里 embedding 检索为什么work的地基。**

*提示*:负采样表用频次 3/4 次方;损失见下方代码;类比检索的"排除输入词"是能不能复现经典结果的关键细节。

## 代码:Skip-gram 负采样的 PyTorch 实现

只写体现机制的核心(数据迭代、完整训练循环略,加注释说明每行对应推导的哪一步):

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SkipGramNS(nn.Module):
    def __init__(self, vocab_size, dim):
        super().__init__()
        # 两套向量:center(v) 与 context(u),正是推导里的 v_c 和 u_w
        self.center  = nn.Embedding(vocab_size, dim)   # v_w
        self.context = nn.Embedding(vocab_size, dim)   # u_w
        # 初始化:center 小随机,context 置零是 word2vec 常见做法
        nn.init.uniform_(self.center.weight,  -0.5/dim, 0.5/dim)
        nn.init.zeros_(self.context.weight)

    def forward(self, center_ids, pos_ids, neg_ids):
        """
        center_ids: (B,)            一批中心词 c
        pos_ids:    (B,)            对应的真实上下文词 o(正样本)
        neg_ids:    (B, k)          每个样本 k 个负样本词 w_i ~ P_n
        返回:这一批的平均负采样损失(就是推导的 L_NEG)
        """
        v_c = self.center(center_ids)          # (B, d)  中心向量 v_c
        u_o = self.context(pos_ids)            # (B, d)  正样本上下文向量 u_o
        u_neg = self.context(neg_ids)          # (B, k, d) 负样本向量 u_{w_i}

        # 正样本项: log σ(u_o · v_c)  —— 推导第一项
        pos_score = (u_o * v_c).sum(dim=1)                 # (B,) = u_o·v_c
        pos_loss  = F.logsigmoid(pos_score)                # log σ(·)

        # 负样本项: Σ_i log σ(−u_{w_i} · v_c)  —— 推导第二项
        # 用 bmm 做 (B,k,d)·(B,d,1) -> (B,k,1)
        neg_score = torch.bmm(u_neg, v_c.unsqueeze(2)).squeeze(2)  # (B,k) = u_{w_i}·v_c
        neg_loss  = F.logsigmoid(-neg_score).sum(dim=1)            # Σ_i log σ(−·)

        # L_NEG = −[ log σ(u_o·v_c) + Σ_i log σ(−u_{w_i}·v_c) ],对 batch 取均值
        return -(pos_loss + neg_loss).mean()

# ---- 3/4 次方负采样分布 ----
def build_neg_sampling_table(word_freqs, power=0.75, table_size=10_000_000):
    """word_freqs: (V,) 各词频次。返回一个查表数组,按 U(w)^{3/4} 归一化后的概率铺开。
       采样时只需 table[random_int] —— O(1) 取负样本。"""
    weights = word_freqs.float() ** power      # U(w)^{3/4}
    probs = weights / weights.sum()            # 归一化(推导里的 P_n)
    counts = (probs * table_size).long()       # 按概率分配槽位数
    table = torch.repeat_interleave(torch.arange(len(word_freqs)), counts)
    return table  # 之后 neg = table[torch.randint(0, len(table), (B, k))]

# ---- 类比检索:vec(b) - vec(a) + vec(c),排除输入词 ----
@torch.no_grad()
def analogy(model, w2i, i2w, a, b, c, topn=5):
    """king(c) − man(a) + woman... 这里按 b−a+c 约定:queen ≈ b−a+c 的最近邻。
       关键:排除 a,b,c 本身,否则常退化成输入词之一(练习 4c 的坑)。"""
    E = model.center.weight                    # (V, d) 用 center 向量
    E = F.normalize(E, dim=1)                  # 归一化做余弦相似度
    target = E[w2i[b]] - E[w2i[a]] + E[w2i[c]] # 类比向量
    target = F.normalize(target, dim=0)
    sims = E @ target                          # (V,) 余弦相似度
    for idx in {w2i[a], w2i[b], w2i[c]}:       # ★ 排除三个输入词
        sims[idx] = -1e9
    top = sims.topk(topn).indices
    return [i2w[i.item()] for i in top]
```

复现/对照实现时:Skip-gram-NS 看 Mikolov 原始 C 代码 `word2vec.c`(`TrainModelThread` 里负采样那段),或 **gensim** 的 `Word2Vec`;GloVe 看 Stanford 官方 C 实现 `glove.c`(权重函数 `f` 和加权最小二乘梯度);上下文向量与下游接口看 **HuggingFace transformers** 和 **sentence-transformers**(它把"句向量"做成了开箱即用的 `SentenceTransformer.encode`,是 word2vec 思想在句级的现代延续)。

## 小结与承上启下

这一章把"词怎么变成向量"从根上讲透了:

- **符号表示(one-hot/词袋/TF-IDF)能用但没语义**,根本病是**任意两词等距、正交**;TF-IDF 的 `log(N/df)` 是"用对数自信息抑制常见词",但它没改变"词与词无关系"这一事实。
- **分布式假说(观其伴知其义)**是把无语义符号升级成有语义向量的钥匙:**用上下文分布定义词义**。计数派(共现矩阵分解)和预测派(预测上下文)由此分叉。
- **word2vec(Skip-gram + 负采样)**:原始 softmax 单步 `O(V)` 跑不动;**负采样把"V 类多分类"换成"1 正例 + k 负例的二分类"(NCE 思想),单步降到 `O(k)`**;损失 `L=−log σ(v_c·u_o)−Σ_k E[log σ(−v_c·u_{neg})]` 的每一步、每个梯度、3/4 次方噪声分布都推清楚了。
- **线性类比**(king−man+woman≈queen)源于"点积≈对数共现"+"类比两侧只差同一语义轴",但**脆弱、依赖排除输入词,别神化**。
- **GloVe** 显式拟合**全局共现的对数** `w_i·w̃_j+b_i+b̃_j≈log X_ij`(因为共现比率经 exp 线性化后点积对应 log 共现),加权函数 `f` 用 `f(0)=0` 剔零项、次线性增长 + 封顶抑制高频对。**Levy & Goldberg 证明两派殊途同归——都在分解共现的对数变换。**
- **ELMo** 用双向 LSTM 让向量**随上下文变化**,消解一词多义,确立"预训练-下游"范式雏形。

**承上**:这是本课第一章,它回答了所有后续模型的共同入口——离散符号怎么变连续向量。

**启下**:静态词向量有两个未解难题。其一,**一词多义**——ELMo 已经用 LSTM 给出了方向,但 LSTM 本身的门控、梯度、为什么能记长程依赖,还没讲透;**下一章我们进入序列建模,从 RNN 的根本困难(长程依赖与梯度消失)讲到 LSTM/GRU 的门控机制**,这是 ELMo、也是 seq2seq、注意力诞生前整个 NLP 的主力架构。其二,**怎么把"词向量"组合成"句子/文档向量"、怎么用它做检索与生成**——这条线会一路通向 seq2seq、注意力、检索增强(RAG)。把这一章的"词→向量"记牢,它是后面每一层的地基。

### 练习 1:亲手推完负采样目标与梯度

**(a) 原始 softmax 梯度 `∂log P(o|c)/∂v_c = u_o − E_{w~P(·|c)}[u_w]`,以及为什么单步 `O(V)`**

写出对数概率(把 softmax 分母拆成「分子的 log 减分母的 log」):

```
log P(o|c) = log [ exp(u_o·v_c) / Σ_w exp(u_w·v_c) ]
           = u_o·v_c  −  log Σ_w exp(u_w·v_c)
```

对中心向量 `v_c` 逐项求梯度。

第一项(线性,直接求导):

```
∂/∂v_c [ u_o·v_c ] = u_o
```

第二项(log-sum-exp,用链式法则):

```
∂/∂v_c [ log Σ_w exp(u_w·v_c) ]
   = 1 / (Σ_w exp(u_w·v_c))  ·  ∂/∂v_c Σ_w exp(u_w·v_c)
   = 1 / (Σ_w exp(u_w·v_c))  ·  Σ_w exp(u_w·v_c)·u_w        ← 内层 exp 求导带出 u_w
   = Σ_w  [ exp(u_w·v_c) / Σ_{w'} exp(u_{w'}·v_c) ] · u_w
   = Σ_w  P(w|c)·u_w                                        ← 方括号正是 softmax 概率 P(w|c)
```

两项相减:

```
∂ log P(o|c) / ∂v_c = u_o − Σ_{w∈V} P(w|c)·u_w
                    = u_o − E_{w~P(·|c)}[ u_w ]
```

**几何含义:梯度 = 「观测上下文 u_o」− 「模型当前期望的上下文 E[u_w]」。** 沿上升方向更新,会把 `v_c` 推向真实出现的 `u_o`,同时拉离「模型现在认为可能的所有上下文向量的加权平均」。当模型的期望恰好等于观测时,梯度为零、停止更新。

**为什么单步 `O(V)`:** 那个期望 `Σ_{w∈V} P(w|c)·u_w` 必须遍历整个词表——既要算每个词的 `P(w|c)`(其分母 `Σ_{w'} exp(u_{w'}·v_c)` 本身就是一次全词表求和),又要做 `V` 次「概率 × 向量」的加权累加。词表几十万时,**每个训练样本、每一步梯度都要做几十万次点积 + 指数 + 累加**,完全跑不动。这正是负采样要绕开的瓶颈。

**(b) 负采样单对损失对 `v_c`、`u_o`、`u_{w_i}` 的梯度,以及为什么 `O(k)`**

损失(`σ` 为 sigmoid,`w_i ~ P_n` 为 k 个负样本):

```
L_NEG = − log σ(u_o·v_c)  −  Σ_{i=1}^{k} log σ(−u_{w_i}·v_c)
            └─ 正样本,拉高 ─┘    └──── k 个负样本,压低 ────┘
```

先备好两个求导事实(提示给的,验算见文末):

```
d/dx [ −log σ(x)  ] = σ(x) − 1
d/dx [ −log σ(−x) ] = σ(x)
```

**对 `v_c` 求梯度**(`v_c` 同时出现在正项和每个负项里,用链式法则,内层对 `v_c` 的导数分别是 `u_o`、`u_{w_i}`):

```
∂L_NEG/∂v_c = (σ(u_o·v_c) − 1)·u_o  +  Σ_{i=1}^{k} σ(u_{w_i}·v_c)·u_{w_i}
```

- 正样本项系数 `σ(u_o·v_c) − 1 < 0`,梯度下降(减去它)等于沿 `+u_o` 方向移动 → **把 v_c 推向 u_o(拉近)**。
- 每个负样本项系数 `σ(u_{w_i}·v_c) > 0`,梯度下降等于沿 `−u_{w_i}` 移动 → **把 v_c 推离每个 u_{w_i}(推远)**。
- 训练收敛时正样本得分被推高(σ→1,系数→0)、负样本得分被压低(σ→0,系数→0),更新自然停。

**对正样本 `u_o` 求梯度**(`u_o` 只出现在正项,内层对 `u_o` 的导数是 `v_c`):

```
∂L_NEG/∂u_o = (σ(u_o·v_c) − 1)·v_c        ← 系数<0,把 u_o 推向 v_c
```

**对某个负样本 `u_{w_i}` 求梯度**(它只出现在自己那一个负项,且变量是 `−u_{w_i}·v_c`;用 `d/dx[−log σ(−x)]=σ(x)`,链式带出内层对 `u_{w_i}` 的导数 `v_c`):

```
∂L_NEG/∂u_{w_i} = σ(u_{w_i}·v_c)·v_c       ← 系数>0,把 u_{w_i} 推离 v_c
```

(注:与正样本不在同一项里的其它向量,梯度恒为 0。)

**为什么单步 `O(k)` 而与 V 无关:** 整个 `L_NEG` 只用到 `1 + k` 个向量——中心 `v_c`、正样本 `u_o`、k 个负样本 `u_{w_i}`。算损失和上面所有梯度,只碰这 `1+k` 个向量(k 通常 5~20),**没有任何对全词表 V 的求和**。梯度天然稀疏:一步只更新 `1+k+1` 个 embedding 行,其余 `V−(k+2)` 行梯度为 0。代价从 `O(V)` 砍到 `O(k)`,这就是 word2vec 能在十亿词级语料几小时跑完的根本原因。

**对照看 (a) 与 (b):** 原始 softmax 的 `−E_{w~P(·|c)}[u_w]` 是「对全词表按模型概率加权的期望」(必须遍历 V);负采样把这个昂贵期望换成「对 k 个从噪声分布 `P_n` 采来的样本求和」`Σ_i σ(u_{w_i}·v_c)·u_{w_i}`——用蒙特卡洛式的少量采样近似了那个全词表期望。这就是 NCE(噪声对比估计)思想:**不学归一化的概率分布,只学一个能把真实数据和噪声分开的二分类器。**

**结论:负采样把「V 类 softmax」换成「1 正例 + k 负例的逻辑回归」,梯度只涉及 1+k 个向量,单步代价 O(k) 与词表大小完全脱钩,同时保留了「正样本拉近、负样本推远」的几何效果。**

### 练习 2:TF-IDF 的 IDF 为什么是 log(N/df)

**(a) 自信息视角 + 全文档出现的极端情形**

随机抽一篇文档,它含词 t 的概率近似 `p = df(t)/N`。信息论里,概率为 p 的事件发生时携带的**自信息(self-information)是 `−log p`**——越罕见的事发生,信息量越大;必然发生的事(p=1)信息量为 0。把「文档含词 t」当成一个事件:

```
idf(t) = −log p = −log( df(t)/N ) = log( N / df(t) )
```

**所以 IDF 本质就是「词 t 在文档里出现这件事的自信息量」。** df 越大(词越普遍)→ p 越大 → 自信息越小 → IDF 越小;反之稀有词 IDF 大。

极端情形——某词在全部 N 篇文档都出现(`df = N`):

```
idf = log( N / N ) = log 1 = 0
```

**这正是我们想要的:** df=N 意味着这个词在每篇文档都出现(典型如「的、是、the、a」),`p=1` 是必然事件、自信息为 0,它对「区分这篇文档和别的文档」**毫无贡献**,IDF 把它的权重直接归零。配上 `tfidf = tf·idf`,无论这种虚词词频多高,乘上 idf=0 后权重也是 0——高频虚词被自动压制,这恰恰是 TF-IDF 相对纯词频(BoW)的核心改进。

**(b) 10 篇文档小语料的 IDF 计算与排序**

N=10。`idf = log(N/df)`(自然对数):

```
词 A: df=10 → idf = log(10/10) = log 1   = 0.0000
词 C: df=5  → idf = log(10/5)  = log 2   ≈ 0.6931
词 B: df=1  → idf = log(10/1)  = log 10  ≈ 2.3026
```

排序(区分力从强到弱):**B > C > A**。

**解释:词 B 最有区分力。** B 只在 1 篇文档出现,一旦看到 B 就几乎锁定是哪篇,信息量最大(IDF 最高)。A 在 10 篇全出现(df=N),是「到处都有」的虚词,看到它对判断是哪篇文档零帮助,IDF=0 被完全压制。C 居中:在一半文档出现,有一定但不强的区分力。这与「IDF = 出现这件事的自信息量」完全一致——越罕见越信息量大、越有区分力。

(注:用 log₂ 或 log₁₀ 只改变单位/常数倍,**相对大小与排序不变**,所以区分力结论与对数底无关。)

**(c) 平滑版 `idf = log(N/(1+df)) + 1` 解决的数值问题**

原始 `log(N/df)` 有两个数值毛病,平滑版各对应一处修补:

1. **除零(df=0)**:若某词在训练语料里 df=0(例如 vocabulary 来自别处、或验证集新词),`N/df` 除零发散。分母 `1+df` 把它垫成 `N/(1+0)=N`,有限可算,**永不除零**。

2. **IDF 变负(df=N)**:这是题目提示要点。原始公式在 df=N 时 idf=log1=0 还好,但平滑分母 `1+df` 会让自变量 `N/(1+df) < 1`,而 `log(<1) < 0` —— 例如 df=N=10 时 `log(10/11) ≈ −0.0953 < 0`。**负 IDF 会让该词的 tfidf 变成负权重,语义上荒谬**(常见词不该贡献负分)。末尾 `+1` 把整条曲线上抬,保证 `idf ≥ 1 > 0`(因为 `log(N/(1+df)) ≥ log(N/(N+1)) > −1`,加 1 后恒正),**消除负权重**。

用 (b) 的语料验证平滑版(均 > 0,排序仍 B>C>A):

```
A: log(10/11)+1 ≈ 0.9047
C: log(10/6) +1 ≈ 1.5108
B: log(10/2) +1 ≈ 2.6094
```

**结论:`1+df` 防除零、`+1` 防负权重(尤其 df 接近或等于 N 时),两处修补让 IDF 在任意 df 下都数值稳定且恒正,但「用对数自信息抑制常见词」的核心思想不变。** 这正是 scikit-learn `TfidfVectorizer` 默认(`smooth_idf=True`)采用的形式。

### 练习 3:word2vec vs GloVe,以及它们的统一

**(a) 四个维度的差异**

| 维度 | word2vec(Skip-gram + 负采样) | GloVe |
|---|---|---|
| **数据视角** | 局部窗口,逐个(中心,上下文)样本流式 SGD,从不显式攒全局计数 | 先把整个语料的全局共现矩阵 `X` 一次性统计好,再训练 |
| **目标函数** | 预测/判别式:`−log σ(v_c·u_o) − Σ_k E[log σ(−v_c·u_neg)]`,区分真实共现 vs 噪声(NCE 思想) | 拟合/回归式:加权最小二乘 `J = Σ_{i,j} f(X_ij)·(w_i·w̃_j + b_i + b̃_j − log X_ij)²` |
| **计算与内存** | 在线流式,内存友好;每步只碰 1+k 个向量,O(k) | 需先建并存共现矩阵(大语料下内存是瓶颈),之后只遍历 **非零项** 优化 |
| **罕见对处理** | 靠采样隐式处理,罕见对可能欠采、信号弱 | 权重函数 `f` 对罕见对显式降权(`f(0)=0` 剔除,小 X 给小权重),可控 |

**(b) 为什么 GloVe 拟合 `log X_ij` 而非 `X_ij`**

起点观察:**承载语义关系的是共现概率的「比率」`P(k|i)/P(k|j)`,而非概率本身**(ice/steam 例子:solid 让比率≫1、gas 让比率≪1、water/fashion 让比率≈1,比率能挑出相关维度、压掉无关维度)。希望用向量去编码这个比率结构:

```
F( (w_i − w_j) · w̃_k ) = P(k|i) / P(k|j)
```

左边是向量「差」的点积(线性、对称),右边是概率的「商」。要让函数 F 把「减法 → 除法」,自然取 **F = exp**(指数恒等式 `exp(a−b)=exp(a)/exp(b)` 正好把差变商):

```
exp( (w_i − w_j)·w̃_k ) = P(k|i)/P(k|j)
⟹ exp(w_i·w̃_k) / exp(w_j·w̃_k) = P(k|i)/P(k|j)
⟹ exp( w_i·w̃_k ) = P(k|i) · 常数              ← 两边「分子配分子」
⟹ w_i·w̃_k = log P(k|i) + 常数                  ← 两边取 log
            = log X_ik − log X_i + 常数          ← P(k|i)=X_ik/X_i
```

`log X_i`(词 i 的总共现数,与 k 无关)和常数被吸收进偏置 `b_i`,对称再加 `b̃_k`:

```
w_i·w̃_j + b_i + b̃_j ≈ log X_ij
```

**所以拟合的是 log 共现,而不是共现本身**——因为「比率结构经 exp 线性化后,点积自然对应的就是对数共现」。两个补充理由:① 直接拟合 `X_ij` 会被高频对的巨大数值(上万)主导损失;`log` 压缩动态范围,让高频对和低频对量级可比。② `log` 与上面的比率推导自洽(差向量 ↔ 比率的 log),而线性拟合 `X_ij` 没有这种语义对应。

**(c) 权重函数 `f` 的三个作用**

```
       ⎧ (x/x_max)^α   若 x < x_max
f(x) = ⎨                              (论文 x_max=100, α=3/4)
       ⎩ 1             若 x ≥ x_max
```

- **`f(0)=0` —— 剔除零项 + 避开发散。** `X_ij=0` 的对(绝大多数词对从不共现)权重为 0,直接从求和里消失。一举两得:① 避开 `log 0 = −∞` 的数值发散;② 让 `J` 只对**实际出现过的非零共现对**求和。这就是 GloVe 训练只遍历稀疏非零项、不扫整个 `V×V` 的原因,使大词表也能高效优化。
- **次线性增长(`x<x_max` 时 `(x/x_max)^{3/4}` 随 x 递增,但用 3/4 次方)—— 信任更可靠的对,但不让高频对垄断。** 共现越多的对统计上越可靠,该给越大权重;但用 `^{3/4}` 让权重**次线性**增长(增速递减),不让高频对的权重随计数无限膨胀。
- **`x≥x_max` 封顶为 1 —— 给超高频对的权重设硬上限。** 超高频对(如「的」与任意词、X_ij 几万)权重一律封到 1,不再随计数增长,**防止极少数超高频对垄断整个损失**、把模型带偏去过拟合虚词搭配。

(提醒:这里的 3/4 与负采样噪声分布的 3/4 是**两个相互独立、各自调出来的经验值**,数学上无直接关系,勿混淆。)

**(d) Levy & Goldberg:预测派与计数派为什么殊途同归**

Levy & Goldberg(NeurIPS 2014)证明:**Skip-gram + 负采样(SGNS)在最优解处,隐式地在分解一个矩阵——逐点互信息(PMI)矩阵减去 `log k`:**

```
v_i · u_j ≈ PMI(i,j) − log k
其中 PMI(i,j) = log[ P(i,j) / (P(i)·P(j)) ]
```

PMI 本身就是共现的一种对数变换(`P(i,j)` 是共现概率,即由共现计数 `X_ij` 归一化而来)。换句话说,看似「不建矩阵、纯预测」的 SGNS,其学到的中心-上下文点积,在数学上等价于对一个由共现统计构成的(平移过的)PMI 矩阵做**低秩分解**。

而 GloVe **显式**拟合 `w_i·w̃_j + b_i + b̃_j ≈ log X_ij`——经偏置项校正后(`b_i` 吸收 `log X_i`、`b̃_j` 吸收 `log X_j`),其拟合目标也近似 PMI 的结构(`log X_ij − log X_i − log X_j ~ log[X_ij/(X_i X_j)]`,正是 PMI 形态)。

**结论:所谓「预测派(word2vec)」和「计数派(GloVe/LSA)」的对立只是表面。两者本质都在对共现统计的某种对数变换(PMI / log X)做低秩矩阵分解——一个隐式(靠多次采样的 SGD 近似)、一个显式(直接拟合)。** 这个统一视角是理解词向量的关键收获:别被「预测 vs 计数」的表面之争迷惑,它们殊途同归。

### 练习 4:实现 Skip-gram 负采样并做类比检索

下面给两份实现,都已在 CPU 上实测跑通并复现了 `NN(king)→{…queen…}`、`king−man+woman→queen`:

- **实现一(纯 numpy,零额外依赖,直接 `python3` 可跑)**:把推导里的梯度逐行手写出来,适合验证(a)(b)(c) 三问的全部逻辑。
- **实现二(PyTorch,对应章节代码骨架)**:用 `autograd`,贴近真实工程;需 `pip install torch`,CPU 即可、无需 GPU。

#### 实现一:纯 numpy 参考实现(CPU 可跑,推荐先看这份)

```python
import numpy as np
from collections import Counter

rng = np.random.default_rng(0)
def sigmoid(x): return 1.0 / (1.0 + np.exp(-x))

# ---- 玩具语料:刻意构造干净的「王权轴 / 性别轴」结构,便于复现类比 ----
sents = (['king reigns palace throne crown'] * 40 +
         ['queen reigns palace throne crown'] * 40 +   # king/queen 共享王权上下文
         ['man walks street house dog'] * 40 +
         ['woman walks street house dog'] * 40 +        # man/woman 共享日常上下文
         ['king man'] * 40 + ['queen woman'] * 40)      # 把性别轴接到王权轴上
toks = [s.split() for s in sents]
vocab = sorted({w for s in toks for w in s})
w2i = {w: i for i, w in enumerate(vocab)}
i2w = {i: w for w, i in w2i.items()}
V, d = len(vocab), 24

# (a) 3/4 次方负采样表:把 P_n(w) ∝ U(w)^{3/4} 铺成查表数组,采样 O(1)
cnt = Counter(w for s in toks for w in s)
freqs = np.array([cnt[i2w[i]] for i in range(V)], dtype=float)
weights = freqs ** 0.75                      # U(w)^{3/4}
probs = weights / weights.sum()              # 归一化 = P_n
table = np.concatenate([np.full(int(p * 100_000), i) for i, p in enumerate(probs)])

# 构造 (中心词, 上下文词) 正样本对,窗口半径 m=2
pairs = []
for s in toks:
    ids = [w2i[t] for t in s]
    for i, c in enumerate(ids):
        for j in range(max(0, i - 2), min(len(ids), i + 3)):
            if j != i:
                pairs.append((c, ids[j]))
pairs = np.array(pairs)

# 两套向量:C=center(v_c), U=context(u_w);center 小随机、context 置零(word2vec 习惯)
C = rng.uniform(-0.5 / d, 0.5 / d, (V, d))
U = np.zeros((V, d))
lr, k, epochs = 0.05, 5, 120

for _ in range(epochs):
    for c, o in pairs[rng.permutation(len(pairs))]:
        negs = table[rng.integers(0, len(table), k)]      # k 个负样本 ~ P_n
        v_c = C[c].copy()
        # 正样本:梯度系数 (σ(u_o·v_c) − 1)，见练习1(b)
        g_pos = sigmoid(U[o] @ v_c) - 1.0
        g_vc = g_pos * U[o]                                # ∂L/∂v_c 的正样本贡献
        U[o] -= lr * g_pos * v_c                           # ∂L/∂u_o = (σ−1)·v_c
        # k 个负样本:梯度系数 σ(u_wi·v_c)
        for n in negs:
            g_neg = sigmoid(U[n] @ v_c)
            g_vc += g_neg * U[n]                           # 累加到 ∂L/∂v_c
            U[n] -= lr * g_neg * v_c                       # ∂L/∂u_wi = σ(·)·v_c
        C[c] -= lr * g_vc                                  # 一次只更新 1+k+1 行 → O(k)

def _unit(M): return M / (np.linalg.norm(M, axis=1, keepdims=True) + 1e-9)

# (b) 近义词检索:用 center 向量 + 余弦相似度
def nearest(word, topn=3):
    E = _unit(C)
    sims = E @ E[w2i[word]]
    sims[w2i[word]] = -1e9                                 # 排除自己
    return [i2w[i] for i in np.argsort(-sims)[:topn]]

# (c) 类比检索 vec(b)−vec(a)+vec(c),★ 排除 a,b,c 三个输入词 ★
def analogy(a, b, c, topn=3):
    E = _unit(C)
    t = E[w2i[b]] - E[w2i[a]] + E[w2i[c]]
    t = t / (np.linalg.norm(t) + 1e-9)
    sims = E @ t
    for idx in {w2i[a], w2i[b], w2i[c]}:                   # 不排除 → 常退化成输入词之一
        sims[idx] = -1e9
    return [i2w[i] for i in np.argsort(-sims)[:topn]]

print("NN(king):", nearest("king"))
# king − man + woman ≈ queen,对应 analogy(a='man', b='king', c='woman')
print("king-man+woman ->", analogy("man", "king", "woman"))
print("[失败例] reigns-king+man ->", analogy("king", "reigns", "man"))
```

实测输出(seed=0,会因随机性略有波动,趋势稳定):

```
NN(king): ['throne', 'queen', 'crown']          ← king 最近邻含 queen 与王权词,合理
king-man+woman -> ['queen', ...]                ← 复现经典类比(已排除 man/king/woman)
[失败例] reigns-king+man -> ['woman', 'house', 'dog']  ← 噪声,见下方分析
```

**(c) 失败类比分析:为什么 `reigns − king + man` 失败。** 期望它给出「man 的『动作/谓语』对应词」,但语料里压根没有一条能让模型学到「king↔reigns 的关系 = man↔某动词 的关系」这条平行轴——`man` 的上下文是 `walks/street/house/dog`,与 `reigns` 所在的王权语境完全不重叠。**线性类比成立的前提是「两组词只在同一条语义轴上不同、其余轴相同」**(练习 3b 推导:`(v_b−v_a)·u_w ≈ (v_d−v_c)·u_w` 对所有 w 成立,差向量才近似平行)。这里 `reigns−king` 编码的「王权专属谓语轴」在 `man` 一侧根本不存在对应物,差向量无平行关系可言,结果退化成噪声。这正印证章节的告诫:**线性类比脆弱,只在频繁、规则、单一语义轴的类比上 work,在缺少平行结构时必然失败——别把它当作 embedding「理解」语义的证据。**

**关于「排除输入词」(本题关键坑)**:把 `analogy` 里 `for idx in {...}: sims[idx] = -1e9` 那段注释掉,`king−man+woman` 的 top-1 极易退化成 `king` 或 `woman` 本身——因为目标向量 `E[woman]−E[man]+E[king]` 与三个输入词高度共线。Levy & Goldberg 指出经典 `3CosAdd` 检索必须排除三个输入词,**这说明类比能力有相当一部分是「检索技巧」的功劳,而非纯粹向量代数**。能否复现经典结果,全看这一行。

#### 实现二:PyTorch 参考实现(对应章节代码骨架)

依赖与运行说明:`pip install torch`;**CPU 即可运行,无需 GPU**(玩具语料几秒训完)。模型 `forward` 用 `F.logsigmoid` 直接写出 `L_NEG`,梯度交给 autograd,无需手推。

```python
import torch, torch.nn as nn, torch.nn.functional as F
from collections import Counter
torch.manual_seed(0)

class SkipGramNS(nn.Module):
    def __init__(self, vocab_size, dim):
        super().__init__()
        self.center  = nn.Embedding(vocab_size, dim)   # v_w
        self.context = nn.Embedding(vocab_size, dim)   # u_w
        nn.init.uniform_(self.center.weight, -0.5/dim, 0.5/dim)
        nn.init.zeros_(self.context.weight)
    def forward(self, c, pos, neg):                    # c:(B,) pos:(B,) neg:(B,k)
        v_c   = self.center(c)                         # (B,d)
        u_o   = self.context(pos)                      # (B,d)
        u_neg = self.context(neg)                      # (B,k,d)
        pos_loss = F.logsigmoid((u_o * v_c).sum(1))                       # log σ(u_o·v_c)
        neg_sc   = torch.bmm(u_neg, v_c.unsqueeze(2)).squeeze(2)          # (B,k)=u_wi·v_c
        neg_loss = F.logsigmoid(-neg_sc).sum(1)                           # Σ log σ(−u_wi·v_c)
        return -(pos_loss + neg_loss).mean()                             # L_NEG

# ---- 语料 / 词表 / 正样本对(同实现一) ----
sents = (['king reigns palace throne crown']*40 + ['queen reigns palace throne crown']*40 +
         ['man walks street house dog']*40 + ['woman walks street house dog']*40 +
         ['king man']*40 + ['queen woman']*40)
toks  = [s.split() for s in sents]
vocab = sorted({w for s in toks for w in s})
w2i   = {w:i for i,w in enumerate(vocab)}; i2w = {i:w for w,i in w2i.items()}
cnt   = Counter(w for s in toks for w in s)
freqs = torch.tensor([cnt[i2w[i]] for i in range(len(vocab))], dtype=torch.float)

# (a) 3/4 次方负采样表
w = freqs ** 0.75; probs = w / w.sum()
counts = (probs * 100_000).long()
table  = torch.repeat_interleave(torch.arange(len(vocab)), counts)

pairs = []
for s in toks:
    ids = [w2i[t] for t in s]
    for i,_ in enumerate(ids):
        for j in range(max(0,i-2), min(len(ids), i+3)):
            if j != i: pairs.append((ids[i], ids[j]))
pairs = torch.tensor(pairs)

model = SkipGramNS(len(vocab), 24)
opt = torch.optim.Adam(model.parameters(), lr=0.01)
k = 5
for _ in range(150):
    perm = torch.randperm(len(pairs))
    c, pos = pairs[perm,0], pairs[perm,1]
    neg = table[torch.randint(0, len(table), (len(c), k))]   # (B,k) ~ P_n
    loss = model(c, pos, neg)
    opt.zero_grad(); loss.backward(); opt.step()

@torch.no_grad()
def nearest(word, topn=3):
    E = F.normalize(model.center.weight, dim=1)
    sims = E @ E[w2i[word]]; sims[w2i[word]] = -1e9
    return [i2w[i.item()] for i in sims.topk(topn).indices]

@torch.no_grad()
def analogy(a, b, c, topn=3):                                # vec(b)−vec(a)+vec(c)
    E = F.normalize(model.center.weight, dim=1)
    t = F.normalize(E[w2i[b]] - E[w2i[a]] + E[w2i[c]], dim=0)
    sims = E @ t
    for idx in {w2i[a], w2i[b], w2i[c]}: sims[idx] = -1e9   # ★ 排除输入词
    return [i2w[i.item()] for i in sims.topk(topn).indices]

print("NN(king):", nearest("king"))
print("king-man+woman ->", analogy("man", "king", "woman"))
print("[失败例] reigns-king+man ->", analogy("king", "reigns", "man"))
```

两份实现结论一致:`NN(king)` 含 `queen` 与王权词;`king−man+woman` 复现 `queen`(已排除输入词);构造的失败类比给出噪声。要点回顾——**(a) 负采样表按 `freqs^{3/4}` 铺开做 O(1) 采样;(b) 用 center 向量 + 余弦相似度检索近义词;(c) 类比检索必须排除 a/b/c 三个输入词,且只在「单一平行语义轴」存在时才成立,缺平行结构必然失败。**

**附:练习 1(b) 两个求导事实的验算**

```
σ(x)=1/(1+e^{−x}),  σ'(x)=σ(x)(1−σ(x))
−log σ(x) 的导数 = −σ'(x)/σ(x) = −(1−σ(x)) = σ(x)−1            ✓
−log σ(−x) 的导数:令 g=σ(−x),内层 −x 的导数为 −1
  = −g'/g = −[σ'(−x)·(−1)]/σ(−x) = σ(−x)(1−σ(−x))/σ(−x) = 1−σ(−x) = σ(x)   ✓(末步用 1−σ(−x)=σ(x))
```

数值核对(取 x=0.7):`σ(x)−1 = −0.3318`,中心差分 `≈ −0.3318` ✓;`σ(x) = 0.6682`,中心差分 `≈ 0.6682` ✓。同时 `1−σ(0.7)=0.3318 = σ(−0.7)` ✓,验证了 sigmoid 恒等式 `1−σ(x)=σ(−x)`。
