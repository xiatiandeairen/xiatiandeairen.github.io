## 一句话结论(先给答案)

- **想最快变成"有竞争力的人"、且能复用你的系统/Rust 功底 → 大模型系统 MLSys 为主线,推理优化做延伸。** 这条路对学历(PhD)要求最低,你已有的并发、内存、调度、缓存直觉能直接迁移,到"大佬"的距离最短。
- **想站在最前沿、愿意啃数学和理论 → 强化学习/推理模型 或 大模型算法与架构。** 曝光高、天花板高,但顶尖岗多需 PhD,竞争最激烈。
- **CV / NLP 是深度学习的两块地基。** 想真正读懂"现代大模型是怎么长出来的",这两章按时间线读一遍最划算;但作为**今天的主攻方向**,纯 CV/NLP 研究岗已大量被多模态/LLM 收编,更适合当**地基**而非**终点**。

## 分维度细看

### 1. 就业与门槛(你最关心的"有价值")

- **MLSys / 推理优化**:缺口最大、对学历最宽容。一个强系统工程师能直接以"推理/训练框架工程师"切入,不被博士门槛卡死。**这是你启动最快的一条。**
- **大模型算法**:岗位多但分层极陡——应用层(调模型、做数据)门槛中等,**真正的"预训练/架构研究岗"几乎只招顶尖 PhD**。想当这一类"大佬",要做好读博或长期重投入的准备。
- **强化学习/推理模型**:正处风口(o1/R1 之后),岗位在涨,但同样偏研究、偏 PhD;工程落地的 agent 岗相对更友好。
- **CV**:最成熟、岗位最多最稳(安防、自动驾驶、工业质检、医疗影像),门槛中等;缺点是"红海",增量不如大模型方向。
- **NLP**:纯 NLP 岗已大量并入 LLM 岗;作为独立方向求职面变窄,但作为通往 LLM 的能力底座非常值钱。

> 薪资具体数字「待核」——不同公司/城市/级别差异极大,这里只给相对排序,别拿去当 offer 依据。

### 2. 论文时间线的厚度 & 是否仍在爆发

- **最厚且仍在爆发**:大模型算法(2017 Transformer 起,每年都有范式级工作)、强化学习/推理(2013 DQN 起,2024-25 因推理模型再次爆发)。
- **全是"优化技术"、最贴你那条"近 10 年优化"要求**:MLSys(从 Megatron/ZeRO 到 FlashAttention、vLLM、投机解码,几乎每个里程碑都是一次优化突破)。
- **成熟但仍在演进**:CV(2012 AlexNet 起,近年转向自监督、3D、生成)。
- **范式已被统一**:NLP(从 word2vec 到 BERT 的演进史极完整,但 2022 后主线被 LLM 吸收)。

### 3. 与你(系统/Rust 背景)的契合度

- **MLSys = 头部优势**:并行、显存/带宽、调度、缓存(KV cache 像极了操作系统的页表与缓存)——你的存量直觉几乎全部可迁移,别人补系统课,你补的只是上层 ML。
- **CV / 强化学习 / 算法**:契合中等,需要补的数学和领域直觉更多(概率、优化、信号、博弈)。
- **NLP**:契合中等,但它是理解 LLM 的最短认知路径。

### 4. 天花板 / 护城河 / 过热风险

- **天花板最高、也最热**:大模型算法、强化学习/推理——但热意味着竞争与泡沫风险并存,且高度依赖算力与数据资源(个人很难复现前沿)。
- **护城河最稳、最少被颠覆**:MLSys、CV 工程——硬技能、可积累、不追热点也饿不死。
- **被收编风险**:纯 NLP、纯单模态 CV 的部分子方向正被多模态/LLM 吞并,选子方向时要挑"还在长"的(如多模态、3D、具身、视频)。

## 决策树(看重什么 → 选什么)

```
你最看重 ——
├─ 免 PhD、最快有竞争力、复用系统功底 ……… 大模型系统 MLSys(+ 推理优化)   ← 给你的默认
├─ 站最前沿、愿读博/重投理论 …………………… 大模型算法 或 强化学习/推理
├─ 和你正在做的 AI 产品(agent)强绑定 ……… 强化学习/推理/Agent
├─ 岗位多而稳、应用扎实、不追热点 …………… 计算机视觉 CV
└─ 先把深度学习地基与"LLM 怎么来的"打通 … NLP + CV(按时间线读,作为入口而非终点)
```

## 几种组合走法(主线 + 支撑)

- **系统大佬路(最贴你)**:`MLSys 主线` + `推理优化 / CV 工程` 支撑。硬技能、护城河深、启动快。
- **算法大佬路(天花板最高)**:`大模型算法 主线` + `强化学习/推理` 支撑。需重投数学与算力资源。
- **理解打底路(适合先补课)**:先 `CV + NLP` 打深度学习地基 → 再 `大模型算法` → 最后 `MLSys` 落到工程。

> 别同时主攻 3 个互不相关的方向——"什么都想会"是成不了大佬的最大陷阱。挑 1 条主线扎到底,其余作支撑泛读。

## 怎么用这个库

1. **还没想好选哪个** → 先读本篇 → 再把每章的「方向全景」和「公认难点」两段速览一遍 → 选定后精读那一章。
2. **已偏向系统** → 精读 MLSys 章,其余按时间线泛读建立全局观。
3. **想补深度学习史** → CV、NLP 两章按里程碑时间线读,会发现 Transformer/注意力是怎么从翻译任务一路长成今天的 LLM 的。
4. 每章结构一致(方向全景 → 脉络地图 → 里程碑精讲 → 优化技术 → 前沿 → 难点 → 就业 → 必读),方便你横向对照着读。
