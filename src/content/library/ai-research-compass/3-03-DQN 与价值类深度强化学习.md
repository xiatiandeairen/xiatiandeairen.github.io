> 这一章把你从「能用一张表跑 Q-learning」带到「能解释清楚为什么把表换成神经网络会发散、并亲手实现一个不发散的 DQN」。你会拿到致命三要素的机制级拆解、一个能在白板上验算的发散反例、DQN 两个稳定器的数学动机,以及从 Double DQN 到 Rainbow 的改进谱系——每一步都讲到能复现的程度。

## 为什么这一章不可跳过

第 02 章里,你用 `Q(s,a) ← Q(s,a) + α[r + γ max_{a'} Q(s',a') − Q(s,a)]` 解了 gridworld。那张 Q 表有个隐含前提:**状态空间小到能逐格存一个数**。Atari 的一帧是 210×160×3 的像素,状态空间是 256^(210·160·3) 量级——别说存表,连枚举都不可能。

很自然的想法:**用一个带参数 θ 的函数 `Q(s,a;θ)` 去逼近那张表**,θ 可以是线性权重,也可以是卷积网络。这一步叫**函数逼近(function approximation)**。它带来泛化(没见过的状态也能给出估值),代价却是把第 01 章那个干净的收敛保证彻底打破。

第 01 章我们证过:Bellman 最优算子 `T*` 在 max-norm 下是 γ-压缩映射,Banach 不动点定理保证价值迭代必然收敛到唯一的 `Q*`。**这个保证在表格情形成立,在函数逼近下不再成立**。2013 年 DeepMind 用 DQN 让一个网络直接从像素学会打 Atari、2015 年发到 Nature(Mnih et al., *Human-level control through deep reinforcement learning*),靠的不是更深的网络,而是两个让训练「不发散」的工程稳定器。

本章的核心问题就是:**为什么会发散,以及怎么稳住**。理解这一点不是为了打游戏——你后面做 RLHF / 推理 RL 时,actor-critic 里的 critic、PPO 里的 value head,本质都是在做带自举的价值逼近,踩的是同一类坑。

## 一、致命三要素:发散从哪来

### 1.1 先把表格收敛的根基说清楚

第 01 章的收敛靠两件事咬合:

```
1. T* 是压缩:‖T*Q₁ − T*Q₂‖_∞ ≤ γ‖Q₁ − Q₂‖_∞   (max-norm 下系数 γ<1)
2. 表格备份是"投影到自己":更新 Q(s,a) 时,改的就是 Q(s,a) 那一格,
   不影响其它格子如何被表示 → 等价于在精确的函数空间里做 T*
```

关键在第 2 点:**表格的每一格是独立自由参数**。你把 `Q(s,a)` 朝 `T*Q(s,a)` 推一点,不会动到 `Q(s'',a'')`。所以整个迭代就是在「能精确表示任意 Q 函数的空间」里反复施加压缩算子,Banach 定理直接适用。

函数逼近破坏的正是这个独立性。`Q(s,a;θ)` 共享参数 θ:你为了让 `Q(s,a;θ)` 逼近目标,调了 θ,**所有状态的估值都会跟着变**。备份不再是「投影到自己」,而是先做一次 Bellman 备份、再用网络去拟合(投影回网络能表示的函数子空间)。这一投影,可能把压缩性质毁掉。

### 1.2 三个因素,缺一不发散

Sutton & Barto 在《Reinforcement Learning》第二版 11.3 节给这组组合起了名字:**致命三要素(deadly triad)**。三个因素是:

**① 函数逼近(Function Approximation)。** 用参数化函数(尤其是会泛化的)逼近价值,参数共享,一处更新牵动全局。单独它不致命——监督学习天天用神经网络拟合,不发散。

**② 自举(Bootstrapping)。** 用「当前估计」去构造更新目标,而不是用真实采样回报。TD 的目标 `r + γ max_{a'} Q(s',a')` 里的 `Q(s',a')` 是网络自己当前的输出,是个估计值,不是 ground truth。蒙特卡洛不自举(用完整回报 `G_t`),所以 MC + 函数逼近不会因这条发散。

**③ 离策略(Off-policy)。** 更新用的数据分布,和你正在评估/改进的目标策略所诱导的分布**不一致**。Q-learning 是离策略的:无论你用什么行为策略探索(ε-greedy),目标里都取 `max`(贪心策略的值)。on-policy 的 SARSA 用实际走的下一动作,数据分布和被评估策略一致。

**三者同时出现,才致命。** 任意去掉一个,通常都能恢复稳定:

```
去掉 ①(回到表格):第 01 章的收敛保证成立。
去掉 ②(用 MC 而非 TD):梯度是真梯度,等价于在做监督回归,稳。
去掉 ③(用 on-policy,如 SARSA + FA):数据分布与评估策略一致,
        更新方向在期望意义下"指向"一个合理目标,通常稳。
```

记住这句话:**致命三要素是「函数逼近 + 自举 + 离策略」的化学反应,不是任一单体的毒性。**

### 1.3 机制:为什么这三者凑齐就炸

把抽象说法落到机制上。设网络对参数 θ 是线性的(便于分析,结论可推广):`Q(s,a;θ) = φ(s,a)ᵀθ`,φ 是特征向量。TD 更新(半梯度,1.4 节会解释为什么是「半」)是:

```
θ ← θ + α · δ · φ(s,a)
其中 TD 误差  δ = r + γ φ(s',a*)ᵀθ − φ(s,a)ᵀθ
```

在期望意义下,这个迭代的「平均行为」由一个矩阵 `A` 决定:

```
E[Δθ] ∝ −A θ + b
A = E_{(s,a)~d}[ φ(s,a) ( φ(s,a) − γ φ(s',a*) )ᵀ ]
```

- `d` 是**采样**这些转移的分布(由行为策略决定)。
- `φ(s',a*)` 里的 `a*` 是**目标策略**(贪心)选的动作。

迭代 `θ ← θ − αAθ + αb` 收敛的充要条件是 **A 的所有特征值实部为正**(这样 −A 是稳定的,系统朝不动点收缩)。

- **on-policy** 时:采样分布 `d` 恰好是目标策略的稳态分布,可以证明 `A` 是正定的(`A + Aᵀ ≻ 0`),特征值实部全正,**收敛**。这是 on-policy 线性 TD 的经典稳定性结果。
- **off-policy** 时:`d`(行为策略采的)和目标策略的分布错位。这个错位会让 `A` 失去正定性,**特征值实部可能为负**——一旦为负,`θ` 沿那个方向指数发散。

直觉版本:**自举让你「用估计追估计」**,**离策略让你「在 A 区域反复采样、却去修正 B 区域(目标策略常去但你采样不足)的估值」**,**函数逼近让 A、B 共享参数**——你为了修 B 调 θ,顺手把 A 的估值推高了一点;下一轮自举目标因此更高,你又往上调……正反馈成环,误差被 γ 放大不及参数耦合放大快,于是发散。

> 写代码时记住:**梯度裁剪、target 网络、归一化都是在掐断这个正反馈环**,而不是在「优化得更准」。这是后面所有稳定技巧的统一视角。

### 1.4 一个能验算的发散反例(Baird 风格)

光说「可能发散」不够,给个能在白板上算的最小构造。Baird(1995)的反例是经典,这里给一个精神一致、更易手算的两状态版本。

设两个状态 s₁、s₂,只有一个动作(纯预测问题,直接看 V 的发散,Q 同理),奖励恒为 0,γ = 0.99。用线性逼近,但**特征故意设计成共享一个被过度参数化的权重**:

```
V(s₁;w) = 2w        V(s₂;w) = w        (单参数 w,特征分别是 2 和 1)
转移:总是 s₁ → s₂(off-policy:行为策略让我们几乎只在 s₁ 上更新)
```

TD(0) 半梯度更新(只在 s₁ 上更新,r=0):

```
δ = r + γ V(s₂;w) − V(s₁;w) = 0 + 0.99·w − 2w = (0.99 − 2)w = −1.01 w
∇_w V(s₁;w) = 2
w ← w + α · δ · 2 = w + α·(−1.01w)·2 = w (1 − 2.02 α)
```

看着像收敛(系数 < 1)?注意我们 off-policy 地**几乎不在 s₂ 上更新**。真正的 Baird 反例把特征和分布配置成:更新 s₁ 抬高 w → `V(s₂)` 也被抬高(因为共享 w)→ 下一次 `V(s₂)` 当目标时更大 → δ 反而把 w 越推越远。我把符号配成发散需要换一组特征,这里给出**会发散的配置**:

```
设 V(s₁;w)=w,  V(s₂;w)=2w  (特征 1 和 2),仍 s₁→s₂,r=0,γ=0.99
δ = γ·V(s₂) − V(s₁) = 0.99·2w − w = (1.98 − 1)w = 0.98 w   ← 正!
∇_w V(s₁) = 1
w ← w + α·δ·1 = w + α·0.98w = w(1 + 0.98α)   ← 系数 > 1,指数发散!
```

逐行读这个推导:**目标 `γV(s₂)=1.98w` 比当前估计 `V(s₁)=w` 大**(因为 s₂ 的特征更大、共享同一个被抬高的 w),于是 δ>0,更新把 w 调大;w 一大,`V(s₂)=2w` 跟着更大,下一轮目标更高,δ 更大……正反馈,`w → ∞`。真实回报明明是 0,估值却炸到无穷。

这就是致命三要素的浓缩演示:**函数逼近**(s₁、s₂ 共享 w)+ **自举**(用 `V(s₂)` 当目标)+ **离策略**(只在 s₁ 更新、不去 s₂ 修正)。注意这里的发散系数其实是 `1+α(2γ−1)`(γ=0.99 时 `2γ−1=0.98`,故得 `1+0.98α`):当 `2γ−1>0`(即 γ>0.5)时发散——**γ 仍在起作用,只是不像表格情形那样「γ<1 就保证收敛」**;是特征几何 + 离策略分布让压缩失效,而非 γ 失灵。要做出「发散与 γ 无关」的强论断,得换用真正的 Baird 反例(多状态 + 过参数化特征 + 特定 off-policy 分布),而不是这个两状态玩具。

> **新手坑 #1**:很多人以为「调小学习率 α 就能稳」。看上面系数 `(1+0.98α)`:α 再小也 > 1,只是发散变慢,不会变收敛。**发散是结构性的,不是步长问题**。这正是为什么需要 target 网络这种结构性修复,而不是调参。

### 1.5 半梯度:为什么 DQN 的梯度是「假的」

补一个关键概念,后面反复用到。TD 的损失看起来是:

```
L(θ) = E[ ( y − Q(s,a;θ) )² ] ,  y = r + γ max_{a'} Q(s',a';θ)
```

如果对 θ 求**真梯度**,目标 y 里也含 θ,要一起求导:

```
∇_θ L = E[ ( y − Q ) · ( ∇_θ y − ∇_θ Q ) ]      ← 真梯度,y 也求导
```

但 TD 实际用的是:

```
∇_θ L ≈ E[ ( y − Q ) · ( − ∇_θ Q ) ] = −E[ δ · ∇_θ Q ]   ← 半梯度,把 y 当常数
```

**把自举目标 y 当成不依赖 θ 的常数**(stop-gradient),只对 `Q(s,a;θ)` 求导。这叫**半梯度(semi-gradient)**。它不是任何函数的真梯度(所以收敛理论失效),但实践中表现远好于真梯度版本(真梯度版本叫残差梯度 / residual gradient,慢且有偏向)。

**DQN 的 target 网络,本质就是把「把 y 当常数」这件事用一个真的冻结网络做实**——3.2 节会看到,这同时还顺手解决了「移动靶」问题。

## 二、DQN:用两个稳定器掐断正反馈

DQN 不发明新的学习目标——它还是 Q-learning 的半梯度版。它做对的是**两个工程稳定器**,正好打在 1.3 节那个正反馈环的两个环节上。

### 2.1 DQN 的损失函数

先把目标写清楚。在线网络参数 θ,目标网络参数 θ⁻:

```
L(θ) = E_{(s,a,r,s') ~ D} [ ( r + γ · max_{a'} Q(s',a'; θ⁻) − Q(s,a; θ) )² ]
                            └────────── 目标 y(用 θ⁻) ──────────┘   └─ 当前估计 ─┘
```

三个要点,逐个看:

- **D 是 replay buffer**(经验回放池),从过去存的转移里**均匀随机采**一个 minibatch,而不是用刚刚走的连续几步。
- **目标用 θ⁻ 算**,不是 θ。θ⁻ 是 θ 的一个**滞后副本**,隔一段时间才同步一次。
- 对 θ 求**半梯度**(y 里的 θ⁻ 视为常数,本来就是冻结的):

```
∇_θ L = −E_{(s,a,r,s')~D} [ δ · ∇_θ Q(s,a;θ) ]
δ = r + γ max_{a'} Q(s',a';θ⁻) − Q(s,a;θ)        ← TD 误差
```

### 2.2 稳定器一:经验回放(Experience Replay)

**它解决两个问题。**

**(a) 打破样本时间相关性。** 在线 RL 里,连续几步的转移高度相关(你在迷宫一个角落连走 5 步,5 个样本几乎一样)。SGD 的收敛分析建立在**样本近似 i.i.d.** 上;喂强相关的连续样本,梯度估计偏向当前局部、方差被低估,网络会对最近的轨迹过拟合、灾难性遗忘早先学到的。把转移存进一个大池子(典型 1e5~1e6 条),每次**均匀随机抽 minibatch**,就把时间相关性打散,逼近 i.i.d.,梯度估计的偏差大幅下降。

**(b) 提升样本复用率(sample efficiency)。** 与环境交互(尤其真实机器人 / 慢模拟器)是最贵的。on-policy 方法用完一批数据就扔(数据一旦更新策略就「过期」)。**replay 让每条转移被复用很多次**——这正是「离策略」赋予的能力:旧策略采的数据,对学 `Q*`(贪心目标)依然有效。这把样本效率提高一个数量级。

**代价**:这正是 1.2 节说的「③ 离策略」的来源——你刻意用旧分布的数据训当前网络。所以 replay 是一把双刃剑:**它用「制造离策略」换「打破相关性 + 复用样本」**。能这么换而不炸,全靠下一个稳定器顶住。

### 2.3 稳定器二:目标网络(Target Network)

回到 1.3 节的正反馈环:`Q(s,a;θ)` 和 `Q(s',a';θ)` 共享 θ,你更新 `Q(s,a)`,目标 `γ max Q(s',·)` 也跟着动——**你在追一个自己每步都在挪的靶子**。这叫**移动靶(moving target)** 问题。

目标网络的修法极简单:**算目标 y 时,用一个冻结的旧参数 θ⁻**。

```
y = r + γ max_{a'} Q(s',a'; θ⁻)      ← θ⁻ 在一段时间内不变
```

在 θ⁻ 冻结的这段时间里,**目标 y 是固定的**,于是 `min_θ E[(y − Q(s,a;θ))²]` 退化成一个**标准的监督回归**——靶子不动,SGD 当然稳。每隔 C 步(DQN 原文 Atari 用 C≈10000 梯度步),把 θ⁻ ← θ 同步一次,让靶子「跳」到新位置,再固定一段。

**这同时把 1.4 节的半梯度落了地**:目标用 θ⁻ 算,θ⁻ 对 θ 求导本就是零,半梯度从「人为 stop-gradient」变成「靶子真的是另一个不参与求导的网络」,逻辑上更干净。

两种同步方式:

```
硬更新(hard update):每 C 步令 θ⁻ ← θ          (DQN 原版)
软更新(soft update):每步令 θ⁻ ← τθ + (1−τ)θ⁻   (τ 很小,如 0.005,DDPG/SAC 常用)
```

软更新让 θ⁻ 以指数滑动平均(EMA)缓慢跟随 θ,等效「时间常数」约 `1/τ` 步。**硬更新省算力但靶子会突跳(同步那一刻 loss 可能抖一下);软更新更平滑但每步多一次参数 EMA**。本章代码用软更新。

> **新手坑 #2**:目标网络更新频率是个**敏感且反直觉**的超参。C 太小(或 τ 太大)→ 靶子追得太紧,退回移动靶,发散风险回来;C 太大(或 τ 太小)→ 靶子太陈旧,`Q(·;θ⁻)` 长期低估真实 `Q*`,学习极慢、甚至卡在次优。**没有「越稳越好」**,要在「稳」和「跟得上」之间折中。

### 2.4 把两个稳定器对回致命三要素

这是本章最该记住的一张对应表:

```
致命三要素           DQN 的应对
─────────────────────────────────────────────────
① 函数逼近   →  保留(就是要用 CNN,这是泛化的来源,不消除)
② 自举       →  保留,但用 target 网络让自举目标在一段时间内"冻住"
③ 离策略     →  replay 制造的,但用 target 网络 + 均匀采样把它的危害压住
```

注意:**DQN 没有消除三要素中的任何一个**(它仍是 FA + bootstrap + off-policy)。它做的是**让三者的正反馈环路足够「钝」**——靶子冻住(切断自举的即时正反馈)、样本打散(削弱离策略 + 相关性的危害)——把发散概率压到工程可用。所以 **DQN 从来没有收敛性证明,它是经验上稳、理论上仍可能发散的方法**。这个诚实的认识,会让你在它真的发散时(确实会)知道往哪调,而不是怀疑代码写错。

### 2.5 DQN 核心代码

下面是去掉环境噪声、只留机制的核心实现(PyTorch 风格,可照着补全成可跑代码)。

```python
import torch, torch.nn as nn, torch.nn.functional as F
import random, collections

class QNet(nn.Module):                       # 离散动作:输出每个动作的 Q 值
    def __init__(self, obs_dim, n_act):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(obs_dim, 128), nn.ReLU(),
            nn.Linear(128, 128),     nn.ReLU(),
            nn.Linear(128, n_act))           # 输出维度 = 动作数,而非 1
    def forward(self, s):
        return self.net(s)                   # 形状 (batch, n_act)

class ReplayBuffer:                          # 稳定器一:打破相关性 + 复用样本
    def __init__(self, cap):  self.buf = collections.deque(maxlen=cap)
    def push(self, *t):       self.buf.append(t)            # (s,a,r,s',done)
    def sample(self, bs):
        batch = random.sample(self.buf, bs)                # 均匀随机 → 近似 i.i.d.
        s,a,r,s2,d = zip(*batch)
        return (torch.tensor(s, dtype=torch.float32),
                torch.tensor(a),                            # long
                torch.tensor(r, dtype=torch.float32),
                torch.tensor(s2, dtype=torch.float32),
                torch.tensor(d, dtype=torch.float32))
    def __len__(self): return len(self.buf)

def soft_update(target, online, tau):        # 稳定器二:目标网软更新
    for pt, po in zip(target.parameters(), online.parameters()):
        pt.data.mul_(1 - tau).add_(tau * po.data)          # θ⁻ ← τθ + (1−τ)θ⁻

def train_step(q, q_target, buf, opt, gamma, bs, tau):
    if len(buf) < bs: return
    s, a, r, s2, done = buf.sample(bs)

    # 当前估计 Q(s,a;θ):按实际采取的动作 a 取出对应列
    q_sa = q(s).gather(1, a.unsqueeze(1)).squeeze(1)        # (batch,)

    # TD 目标 y = r + γ·max_a' Q(s',a';θ⁻),终止状态不自举(done=1 时去掉未来)
    with torch.no_grad():                                  # 关键:目标不回传梯度(半梯度)
        max_q2 = q_target(s2).max(dim=1).values            # max over a' on θ⁻
        y = r + gamma * (1 - done) * max_q2

    loss = F.smooth_l1_loss(q_sa, y)                       # Huber,对离群 δ 更稳(见 3.5)
    opt.zero_grad(); loss.backward()
    nn.utils.clip_grad_norm_(q.parameters(), 10.0)         # 梯度裁剪:再加一道掐断发散
    opt.step()
    soft_update(q_target, q, tau)                          # 每步缓慢同步靶网
    return loss.item()
```

主循环(行为策略 ε-greedy,体现「采集 → 存 → 抽样训练」的离策略节奏):

```python
def act(q, s, eps, n_act):                   # ε-greedy 行为策略(off-policy 的"行为"侧)
    if random.random() < eps: return random.randrange(n_act)
    with torch.no_grad():
        return int(q(torch.tensor(s, dtype=torch.float32)).argmax())

# 训练骨架
q, q_target = QNet(obs_dim, n_act), QNet(obs_dim, n_act)
q_target.load_state_dict(q.state_dict())     # 初始同步
buf = ReplayBuffer(100_000)
opt = torch.optim.Adam(q.parameters(), lr=1e-3)

for ep in range(N_EPISODES):
    s, eps = env.reset(), max(0.05, 1.0 - ep / 500)        # ε 线性退火,逐步少探索
    done = False
    while not done:
        a = act(q, s, eps, n_act)
        s2, r, done, _ = env.step(a)
        buf.push(s, a, r, s2, float(done))                 # 存进 replay,不立刻用
        s = s2
        train_step(q, q_target, buf, opt, gamma=0.99,
                   bs=64, tau=0.005)                       # 从 replay 抽样训练
```

读这段代码,把它和理论一一对上:`with torch.no_grad()` 包住的 y 就是半梯度的 stop-gradient;`q_target` 就是冻结靶;`random.sample` 就是打散相关性;`gather(1, a)` 体现「Q 网对离散动作一次前向出全部动作值,再按实际动作取一列」——这是离散动作价值方法的标准写法,也正是它后面遇到连续动作时栽跟头的根源(3.6 节)。

## 三、改进谱系:每一个都在补一个具体的洞

DQN 之后几年的工作,几乎都是「发现一个具体缺陷 → 一个针对性补丁」。理解每个补丁补的是哪个洞,比记住名字重要得多。

### 3.1 max 算子的高估偏差(Double DQN 要治的病)

**先证明病根**。DQN 目标里有个 `max_{a'} Q(s',a';θ⁻)`。问题:`Q(s',a';θ⁻)` 是带噪声的估计(网络没收敛、采样有限)。**对一堆有噪估计取 max,会系统性地偏高**。

形式化。设每个动作的真值都相等 `Q*(s',a) = q` 对所有 a;但估计有零均值噪声 `Q̂(s',a) = q + ε_a`,`E[ε_a]=0`。那么:

```
E[ max_a Q̂(s',a) ] = E[ max_a (q + ε_a) ] = q + E[ max_a ε_a ]
```

由 max 是凸函数,Jensen 给 `E[max ε]≥0`;且只要这些零均值变量不恒相等,严格有 `E[max ε]>0`(可直接计算验证)。所以:

```
E[ max_a Q̂(s',a) ]  =  q + E[max_a ε_a]  >  q  =  max_a Q*(s',a)
└──── 我们用作目标的量 ────┘                    └── 真实该用的量 ──┘
```

**结论(加粗记住):用 max 估计去近似「真值的 max」,在有噪声时系统性高估**。误差量级随**动作数增多而增大**(更多噪声变量取 max,最大值期望更高),也随噪声方差增大而增大。这个偏差还会被自举层层传播放大:高估的 `Q(s')` 进入 `Q(s)` 的目标,使 `Q(s)` 也高估……

直觉一句话:**max 同时干了「选哪个动作」和「这个动作值多少」两件事,而且用的是同一组有噪声的估计——它倾向于选中那些「恰好被噪声抬高」的动作,于是估值被噪声的上偏部分污染**。

### 3.2 Double DQN:用两个网络拆开「选」与「评」

van Hasselt、Guez、Silver(AAAI 2016)的洞察:**把「选动作」和「评估那个动作的值」用两套参数分开**,噪声就不再同向耦合。

原始 Double Q-learning(van Hasselt 2010)要两张独立 Q 表。Double DQN 巧在:**DQN 现成就有两个网络**(在线 θ 和目标 θ⁻),直接拿来用,几乎零额外成本。

```
DQN 目标:         y = r + γ · Q( s', argmax_{a'} Q(s',a';θ⁻) ; θ⁻ )
                                         └── 选 ──┘            └ 评 ┘   都用 θ⁻ → 同向高估

Double DQN 目标:  y = r + γ · Q( s', argmax_{a'} Q(s',a';θ ) ; θ⁻ )
                                         └ 选(θ)┘            └评(θ⁻)┘  选用 θ、评用 θ⁻
```

**为什么这能缓解高估?** 高估来自「同一组估计既被用来挑出最大动作、又被用来报告那个最大值」。现在:**用在线网 θ 挑出它认为最好的动作 `a* = argmax Q(s',·;θ)`,再用目标网 θ⁻ 去报告 `a*` 的值 `Q(s',a*;θ⁻)`**。θ 和 θ⁻ 的噪声不完全相关——θ 因为某个动作的正噪声而挑中它,θ⁻ 对同一动作未必也是正噪声,**报告的值因此不再系统性偏高**。理想情况(两网噪声独立)下偏差可降到接近 0;实际两网相关(θ⁻ 是 θ 的滞后副本),只是显著减小、不归零。

代码改动只有一行——把 train_step 里目标那段换成:

```python
with torch.no_grad():
    a_star = q(s2).argmax(dim=1, keepdim=True)             # 选:用在线网 θ
    max_q2 = q_target(s2).gather(1, a_star).squeeze(1)     # 评:用目标网 θ⁻
    y = r + gamma * (1 - done) * max_q2
```

> **代价与坑**:Double DQN 几乎免费且通常更稳,所以现在它基本是默认项。但要清楚它**不保证消除高估**(两网相关),在某些环境甚至轻微低估;它治的是「max 高估」这一个洞,不治发散本身(发散还是靠 replay + target)。

### 3.3 Dueling 架构:把 Q 拆成 V + A

Wang et al.(ICML 2016)的观察:**很多状态下,选哪个动作根本无所谓**(比如赛车在直道上,左微调右微调价值几乎一样)。这种状态,网络没必要为每个动作单独学一个绝对 Q 值;它更该先学「这个状态本身好不好」(状态价值 V),再学「各动作相对好多少」(优势 advantage `A(s,a) = Q(s,a) − V(s)`)。

Dueling 把网络最后分成两个头:

```
Q(s,a) = V(s) + ( A(s,a) − mean_{a'} A(s,a') )
                   └────── 减均值,解决可辨识性 ──────┘
```

**为什么要减均值?** `Q = V + A` 有冗余:给 V 加常数 c、给所有 A 减 c,Q 不变——网络分不清该把基准算给 V 还是 A(不可辨识)。强制 `mean_a A(s,a) = 0`(减掉均值)就**钉死了基准**:V 必须承担「平均价值」,A 只表达「相对偏离」。(也可用 `max` 替代 `mean`,原文讨论过,`mean` 更稳。)

**收益**:在动作多、但多数动作价值接近的环境,V 这一支可以被**所有动作的样本共同更新**(每次更新都改 V),学得更快、更稳;A 只需学细微差异。本质是个**更好的归纳偏置**,不改损失、不改训练流程,只换网络头,可与 Double / PER 自由叠加。

```python
class DuelingQNet(nn.Module):
    def __init__(self, obs_dim, n_act):
        super().__init__()
        self.body = nn.Sequential(nn.Linear(obs_dim,128), nn.ReLU())
        self.V = nn.Linear(128, 1)           # 状态价值头,标量
        self.A = nn.Linear(128, n_act)       # 优势头,每动作一个
    def forward(self, s):
        h = self.body(s)
        v, a = self.V(h), self.A(h)
        return v + (a - a.mean(dim=1, keepdim=True))       # 减均值,钉死基准
```

### 3.4 Prioritized Experience Replay:不是所有样本一样有用

Schaul et al.(ICLR 2016)。均匀采样的浪费:**多数转移网络已经预测得很准(δ≈0),从它们身上几乎学不到东西**;少数「意外」转移(δ 大)才含新信息。PER 的想法:**按 TD 误差大小给样本定优先级,误差大的更常被抽到**。

```
样本 i 的优先级:  p_i = |δ_i| + ε              (ε>0 防止 δ=0 的样本永不被抽)
采样概率:         P(i) = p_iᵃ / Σ_k p_kᵃ        (α∈[0,1] 调"偏好程度",α=0 退回均匀)
```

但**有偏采样会让梯度估计有偏**(你系统性多看大误差样本,改变了期望)。PER 用**重要性采样权重(importance sampling, IS)** 校正:

```
w_i = ( 1 / (N · P(i)) )ᵝ ,  再除以 max_i w_i 归一化
更新时:用 w_i 给每个样本的 loss 加权 →  L = Σ_i w_i · δ_i²
```

逐项解释:`1/(N·P(i))` 是「均匀概率 / 实际采样概率」的比值——被过度采样(P(i) 大)的样本权重被压小,把分布偏差「除回去」。β 从一个小值(如 0.4)**退火到 1**:训练早期允许有偏(换取学得快),后期 β→1 完全校正(保证渐近无偏)。

```
偏差-方差权衡:
  α 大 → 越偏向大误差样本 → 信息密度高、收敛快,但采样分布越偏、方差越大、越依赖 IS 校正
  β 大 → IS 校正越彻底 → 偏差越小,但权重方差越大(早期不稳),故退火 β: 0.4 → 1.0
```

> **新手坑 #3**:实现 PER 必须用 **sum-tree**(线段树)数据结构,才能做到 O(log N) 的「按优先级采样」和「更新优先级」;用线性扫描是 O(N),会让 PER 慢到得不偿失。这是 PER 工程上最大的坑,也是它没成为默认项的原因之一。

### 3.5 一个小但关键的稳定技巧:Huber 损失 / 误差裁剪

代码里用了 `smooth_l1_loss`(Huber)而非纯 MSE,值得单独点出。MSE 的梯度 `∝ δ`,**δ 很大时梯度爆炸**(replay 里偶尔有奖励量级异常的转移,或网络早期 δ 巨大)。Huber 在 |δ|≤1 时是二次(梯度 ∝ δ,精细),|δ|>1 时转线性(梯度被截断为常数 ±1):

```
Huber(δ) = ½δ²            if |δ| ≤ 1
         = |δ| − ½        if |δ| > 1     → 梯度 = sign(δ),对离群 δ 不敏感
```

DQN 原文用的就是「把 TD 误差裁剪到 [−1,1]」,数学上与 Huber 等价。**这是又一道掐断发散的保险**——别让一个异常 δ 一步把网络踹飞。

### 3.6 Rainbow:把六个互补改进合起来

Hessel et al.(AAAI 2018)做的事很务实:**把当时六个独立有效、且机制互补的 DQN 改进装进同一个 agent**,验证它们叠加后的协同效果(在 Atari 上显著超过任一单项)。六个组件:

```
1. Double DQN          → 治 max 高估(3.2)
2. Dueling             → V/A 分解,更好的归纳偏置(3.3)
3. Prioritized Replay  → 按 TD 误差采样,信息密度(3.4)
4. Multi-step (n-step) → 用 n 步回报,偏差-方差插值(第 02 章 n-step return)
5. Distributional (C51)→ 学回报的"整个分布"而非期望,信息更丰富
6. Noisy Nets          → 在权重上加可学习噪声做探索,替代 ε-greedy
```

第 5、6 项本章未单独展开(分布式 RL 和参数空间探索是独立话题),但你已具备读懂它们的全部基础。**Rainbow 的价值不在某个新算法,而在于「这些补丁机制正交、可叠加」这个工程事实**——它告诉你做价值类 RL 时这些技巧基本都该用上,且彼此不冲突。

### 3.7 离散动作的天花板:为什么需要 DDPG / SAC

DQN 全家的结构里藏着一个硬约束:**目标里的 `max_{a'} Q(s',a')` 要求能枚举所有动作**。Q 网一次前向输出「每个动作一个值」,再取 max——这只在**离散且不太多**的动作上可行。

连续动作(机械臂关节力矩 ∈ ℝ、油门开度 ∈ [0,1])下,`max_{a'}` 是对连续空间的优化,**无法逐个枚举**。两条出路(下一章策略类会铺垫,这里只给定位):

- **DDPG**(Lillicrap et al., ICLR 2016):再训一个**确定性策略网络 μ(s)** 专门「输出那个 argmax 动作」,用它替代 `max`,即目标变 `r + γ Q(s', μ(s'); θ⁻)`。它把 DQN 的 replay + target 网络思想搬到连续控制(soft update 的 τ 就是 DDPG 推广开的)。
- **SAC**(Haarnoja et al., ICML 2018):在目标里加**最大熵正则**(鼓励策略保持随机性),用随机策略 + 两个 Q 网取 min(治高估,思想和 Double 一脉相承)做到样本高效且稳,是当前连续控制的主力之一。

**一句话定位**:DQN 是「离散动作 + 价值类」的标杆;连续动作必须借助一个**策略网络**来回避对动作空间求 max——这恰好把你引向第 04 章的策略梯度。**价值类和策略类不是对立,actor-critic(第 05 章)正是把两者缝起来**:critic 学价值(本章这套),actor 学策略(下章那套)。

## 四、设计权衡与常见坑(汇总)

把分散在各节的坑收拢成一份排错清单——这些是真做 DQN 时会反复遇到的:

**① 发散是结构性的,不是 bug。** DQN 无收敛证明。Q 值随训练**单调爆炸到 1e5+**,先查:target 网络更新是不是太频(C 太小 / τ 太大)、有没有梯度裁剪、有没有 Huber、奖励是否需要 clip。别先怀疑代码错(1.4 节)。

**② 「越稳越好」是错觉。** target 网络更新太慢、replay 太大、ε 退火太慢,都会让 agent **学得极慢或卡在次优**——稳和「跟得上 + 探索够」是对立面,要折中(2.3 节坑 #2)。

**③ Q 值绝对数值会持续偏高,别用它判断收敛。** 即便用了 Double DQN,自举 + max 仍倾向高估(3.1)。判断训练好坏看**回合回报(episode return)**,不是看 loss、更不是看 Q 值大小——**DQN 的 loss 不单调下降是正常的**(靶子每 C 步跳一次,loss 必然抖)。

**④ replay buffer 容量与 warmup。** 太小退回相关性问题;太大则混入大量「远古陈旧策略」的数据,拖慢。开训前要**先随机填一批转移(warmup)** 再开始更新,否则前期 minibatch 重复采样极少的几条,严重过拟合。

**⑤ reward 尺度。** 奖励量级过大直接放大 δ → 放大梯度 → 助推发散。DQN 原文在 Atari 把奖励 clip 到 {−1,0,+1};一般环境也常对奖励做归一化/缩放。**这是 RL 里最常被忽视、又最影响稳定性的一环**。

**⑥ off-by-one:终止状态不要自举。** 代码里 `(1 - done)` 那项漏了,会让终止状态错误地加上「未来价值」,污染整条回传链。这是新手最常见的实现 bug。

- **Mnih et al., 2015, Nature, *Human-level control through deep reinforcement learning***(DQN 正式版)。重点读 Methods 里的算法框(Algorithm 1)和「Training details」——经验回放容量、target 网络更新频率 C、奖励 clip 都在那。开头的 NIPS 2013 workshop 版(*Playing Atari with Deep RL*)更短,适合先看主干。
- **van Hasselt, Guez, Silver, 2016, AAAI, *Deep Reinforcement Learning with Double Q-learning***。重点读第 3 节高估偏差的分析和那张「DQN 高估、Double DQN 贴近真值」的对比图——3.1 的证明直觉就来自这里。
- **Wang et al., 2016, ICML, *Dueling Network Architectures***:看「为什么减均值而非直接 V+A」那段(可辨识性论证)。
- **Schaul et al., 2016, ICLR, *Prioritized Experience Replay***:重点是 IS 权重那一节和 sum-tree 实现细节。
- **Hessel et al., 2018, AAAI, *Rainbow***:看消融实验那张图——它逐个拿掉组件,直观告诉你哪个改进贡献最大(经验上 PER 和 multi-step 贡献突出)。
- **Sutton & Barto, 2nd ed, 第 11 章**:致命三要素(11.3)和 Baird 反例(11.2)的权威叙述,1.2–1.4 节的理论根在这。

**开源库(读哪个模块)**:

- **CleanRL** 的 `dqn.py`(单文件实现):本章首选精读对象——一个文件含完整 DQN,变量名清晰,和本章代码骨架几乎一一对应。它还有 `dqn_atari.py`(带 CNN 和帧堆叠)、`ddpg_continuous_action.py`(看 3.7 的连续动作怎么处理)。**「单文件、无层层抽象」正是 CleanRL 适合学机制的原因**。
- **Stable-Baselines3** 的 `dqn` 模块:工程化更完整(回调、日志、向量化环境),适合在读懂 CleanRL 后看「生产级实现多了哪些细节」(如 `target_update_interval`、`exploration_fraction` 的默认值)。
- (说明:**TRL / verl 这类库面向 LLM 的 RLHF / PPO,不含 DQN**——价值类离散方法和后面的 LLM 对齐用的是不同工具链,别去那里找 DQN。)

## 七、小结与承上启下

这一章你拿到了三样东西:

1. **致命三要素的机制级理解**——函数逼近 + 自举 + 离策略的正反馈环,以及一个能在白板上算出 `w→∞` 的发散反例。这是价值类深度 RL 一切不稳定的根。
2. **DQN 两个稳定器的数学动机**——replay(打散相关性 + 复用样本,代价是制造离策略)和 target 网络(冻住自举靶子,切断即时正反馈),以及「它们没消除三要素、只是把环路钝化、所以 DQN 仍无收敛证明」这个诚实认识。
3. **改进谱系的「每个补丁补哪个洞」**——Double 治 max 高估(可证)、Dueling 给 V/A 归纳偏置、PER 按信息密度采样(IS 校正偏差)、Rainbow 验证它们正交可叠加;以及离散动作的硬天花板把我们推向连续控制的 DDPG/SAC。

**在整门课里的位置**:00–02 章建立了「MDP → Bellman → 表格 TD/Q-learning」的无模型基础;本章是**把那套接上深度网络的第一道关**,核心是「稳定性」而非「新目标」。但价值类有个绕不开的结构限制——**它要对动作空间取 max**,离散动作还行,连续动作就卡死;而且它优化的是「价值」,策略是从价值里**间接**导出的(取 argmax)。

下一章(第 04 章,策略梯度定理)换一条完全不同的路:**不学价值、直接对策略参数求梯度、直接优化策略**——`max` 的枚举难题在那里自然消失。再下一章(05,Actor-Critic + GAE)会把今天学的「价值估计」当作 critic、把策略梯度当作 actor,**两条路在那里合流**。而你在本章反复见到的高估、自举不稳、靶子要冻、归一化、梯度裁剪这些坑,会在 PPO(06)、RLHF(08)、推理 RL 的 GRPO(11)里**以新的面貌重复出现**——那时 critic 就是这章这套价值逼近,你今天对致命三要素的理解,会直接变成调那些大模型 RL 训练的直觉。
