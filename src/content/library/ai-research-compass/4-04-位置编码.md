> 这一章把你从"知道 attention = softmax(QKᵀ/√d)·V"带到"能证明纯注意力对 token 顺序完全失明、能从第一性原理推出 RoPE 为什么把相对位置天然编进点积、能照着公式写出一个 RoPE 的 PyTorch 实现并讲清它为什么有一定外推能力"。读完你应当能向别人证明三件事:自注意力是置换等变的(打乱输入顺序,输出只是相应打乱),所以**必须**外部注入位置信息;正弦编码、可学习编码、相对编码、RoPE、ALiBi 各自把位置塞在哪、代价是什么;以及为什么 RoPE 成了今天几乎所有主流大模型(LLaMA / Qwen / Mistral / DeepSeek 等)的默认选择。

## 一、动机:注意力为什么"看不见顺序"

上一章我们把注意力机制拆透了:给定 query Q、key K、value V,注意力做的是

```
Attn(Q, K, V) = softmax(Q Kᵀ / √d) · V
```

这个式子有一个被很多人忽略、但**决定了整个 Transformer 必须有位置编码**的性质:**它对 token 的排列顺序是失明的**。直觉上讲,注意力是一个"加权求和池化"——它根据 query 和每个 key 的相似度给 value 加权平均。而"加权平均"这件事,和你把那些 value 按什么顺序摆进来,没有任何关系。把第 3 个 token 和第 7 个 token 调换位置,每个 token 算出来的输出向量原封不动,只是整体跟着调换了位置而已。

这意味着:**在没有额外信息的情况下,Transformer 眼里的 "the cat sat on the mat" 和 "mat the on sat cat the" 是同一个东西**(只要词表里的词一样、个数一样)。这对语言、代码、时间序列这些**顺序即语义**的数据是致命的——"狗咬人"和"人咬狗"必须是两件事。

所以位置编码不是一个锦上添花的 trick,而是**给一个天生无序的运算补上"序"这一维信息的刚需**。这一章要回答三个递进的问题:

1. 这个"失明"到底有多严格?——我们会**严格证明**自注意力(乃至整个不含位置信息的 Transformer 层)是置换等变的。这是后面一切设计的逻辑起点。
2. 怎么把位置信息注入回去?——从最早的**正弦绝对编码**,到**可学习绝对编码**,到**相对位置编码**,理解每一步在解决前一步的什么毛病。
3. 为什么现在大家都用 RoPE?——把**旋转位置编码(RoPE)** 从二维旋转的第一性原理推到尾,证明它的核心性质:旋转后的 query·key 内积**只依赖相对位置 m−n**。再用 **ALiBi** 作为对照,讲清"外推"这个工程上最关心的指标各方案差在哪。

### 预备:记号与一个要反复用的事实

沿用上一章记号。序列长度 N,模型维度(或单头维度)d。token i 经过 embedding 后是向量 xᵢ ∈ ℝ^d。注意力把每个 xᵢ 线性投影成 query/key/value:qᵢ = W_Q xᵢ,kᵢ = W_K xᵢ,vᵢ = W_V xᵢ。

要反复用的一个事实:**注意力分数只是 query 和 key 的内积(再缩放、softmax)**,即第 i 个 token 对第 j 个 token 的注意力权重正比于 `exp(qᵢ · kⱼ / √d)`。所有位置编码方案的分歧,归根到底就是一个问题:**位置信息从哪进入 `qᵢ · kⱼ` 这个内积**。把这条主线抓住,后面五种方案就是同一道题的五种解法。

## 二、严格证明:自注意力对 token 排列是等变的

我们要证的命题,精确地说是**置换等变(permutation equivariant)**,而不是"置换不变(invariant)"。两者差一个字,意思差很远,先讲清:

- **置换不变**:打乱输入,输出**完全不变**(比如对一个集合求和)。
- **置换等变**:打乱输入,输出**跟着以同样方式打乱**,但每个元素的值不变。

自注意力是**等变**的:把第 i、j 行换位,输出矩阵的第 i、j 行也跟着换位,但每一行的内容本身不变。下面给出完整证明。

### 2.1 把"打乱顺序"写成矩阵

设输入是 X ∈ ℝ^{N×d}(第 i 行是 token i 的向量)。"打乱 N 个 token 的顺序"在线性代数里就是左乘一个**置换矩阵(permutation matrix)** P ∈ {0,1}^{N×N}:P 每行每列恰好一个 1,其余为 0。`PX` 就是把 X 的行按 P 重新排列。

置换矩阵有两条我们要用的性质:

```
(1) 正交:Pᵀ P = P Pᵀ = I,即 Pᵀ = P⁻¹  （把行换回去就是逆置换）
(2) 它只搬行/列,不改元素值
```

### 2.2 逐步推导:打乱输入,注意力输出只是被同样打乱

注意力(为干净起见省去 1/√d,它是常数不影响结论)逐步是:

```
Q = X W_Q,  K = X W_K,  V = X W_V          # 投影是逐行(per-token)的线性映射
S = Q Kᵀ                                    # 打分矩阵
A = softmax(S)                              # 按行 softmax
O = A V                                     # 输出
```

现在把输入换成打乱后的 `X' = P X`,逐步推每个量怎么变:

```
第1步  投影逐行作用,行被 P 搬走,投影结果也跟着搬:
       Q' = X' W_Q = P X W_Q = P Q         （同理 K' = P K, V' = P V）

第2步  打分矩阵:
       S' = Q' K'ᵀ = (P Q)(P K)ᵀ
          = P Q Kᵀ Pᵀ                       （转置反序:(PK)ᵀ = Kᵀ Pᵀ）
          = P S Pᵀ
       含义:S 的第(i,j)元被搬到 (π(i), π(j))，即"行列同时按 P 置换"

第3步  按行 softmax。关键:softmax 是逐行独立算的，且 P S Pᵀ 这种
       "行列同置换"等价于"把每一行整体搬走、行内元素也同步重排"。
       softmax 对"行内重排"是等变的（softmax 后再排 = 先排再 softmax），所以:
       A' = softmax(P S Pᵀ) = P · softmax(S) · Pᵀ = P A Pᵀ

第4步  输出:
       O' = A' V' = (P A Pᵀ)(P V)
          = P A (Pᵀ P) V                    （结合律）
          = P A V                            （Pᵀ P = I，逆置换抵消！）
          = P O
```

**结论(置换等变定理):`Attn(PX) = P · Attn(X)`。打乱输入的顺序,输出只是被同一个置换 P 重排,每一行的数值分毫不变。** 第 4 步那个 `Pᵀ P = I` 是整个证明的命门——它说明:S 里被引入的列置换 Pᵀ,在和被同样置换的 V 相乘时正好抵消,于是顺序信息"自我湮灭"了。注意力压根没有保留任何关于"谁在前谁在后"的痕迹。

### 2.3 推广到整个 Transformer 层与一个重要例外

上面只证了单头注意力。再看 Transformer 层里其他算子:

- **逐位置前馈网络(FFN / MLP)**:对每个 token 独立作用,显然逐行,满足 `FFN(PX) = P·FFN(X)`。
- **LayerNorm / RMSNorm**:也是逐行(对每个 token 的特征维归一化),等变。
- **残差连接** `X + Sublayer(X)`:两项都等变,和仍等变。

所以:**不含位置信息时,整个 Transformer 编码器(任意多层堆叠)对 token 排列都是等变的。** 这就把"必须注入位置"从直觉升级成了定理。

**一个重要例外要点明:带因果 mask 的解码器(decoder)并非对任意置换等变。** 因为因果 mask 显式规定了"第 i 个位置只能看 ≤ i 的位置",mask 矩阵本身就编码了顺序。但要小心:**mask 提供的只是"谁能看谁"的偏序,不是"相隔多远"的距离信息**——它知道 j 在 i 前面,却不知道隔了 1 个还是 100 个 token。所以即使是因果解码器,**仍然需要位置编码来提供距离/绝对位置信息**(GPT 系列就既有因果 mask 又有位置编码)。把"因果 mask 能不能替代位置编码"想清楚,是这一节最容易被面试问倒的点:不能,它们提供的是不同维度的信息。

## 三、第一代:正弦绝对位置编码(Sinusoidal)

既然必须注入位置,最朴素的想法是:给每个位置 pos 造一个向量 PE(pos) ∈ ℝ^d,**加到** 该位置的词向量上,`x'ᵢ = xᵢ + PE(i)`,然后照常走注意力。这就是原始 Transformer(Vaswani et al., 2017)的做法。问题只剩:PE(pos) 怎么造?

### 3.1 设计:用一组不同频率的 sin/cos

原始论文给出的公式是:把 d 维向量两两配对,第 2i 维放 sin、第 2i+1 维放 cos,频率随维度指数衰减:

```
PE(pos, 2i)   = sin( pos / 10000^(2i/d) )
PE(pos, 2i+1) = cos( pos / 10000^(2i/d) )
        其中 i = 0, 1, …, d/2−1,  pos 是位置(0,1,2,…)
```

记每一对(维度 2i, 2i+1)的角频率为 ωᵢ = 1 / 10000^(2i/d)。i 越大,ωᵢ 越小,波长越长。所以**这组编码本质是一个"多分辨率时钟":低维(i 小)是高频快针,相邻位置就明显不同,刻画局部精细位置;高维(i 大)是低频慢针,波长可达上万,刻画全局的粗位置。** 几何上,每个位置 pos 在每个二维子空间里都是单位圆上一个由 ωᵢ·pos 决定的相位点;把 d/2 个子空间的相位拼起来,就唯一编码了这个位置——这和钟表用时/分/秒三根不同速度的指针唯一确定时刻是同一个原理。

为什么基数取 10000?它决定了最长波长(最低频维度的周期约 2π·10000),保证在常见序列长度(几千)内,最慢那根针也不会绕完一圈造成位置混叠。这个 10000 是经验常数,不是推出来的,可换。

### 3.2 关键性质:相对位移是一个线性(旋转)变换

正弦编码有一个常被引用的优良性质:**PE(pos+k) 可以表示成 PE(pos) 的一个只依赖 k 的线性变换。** 这给后面 RoPE 埋了伏笔,值得算清楚。

只看一个二维子空间 (2i, 2i+1),记 ω = ωᵢ。该子空间里 PE(pos) = (sin(ω·pos), cos(ω·pos))。用三角和角公式:

```
sin(ω(pos+k)) = sin(ω·pos)cos(ω·k) + cos(ω·pos)sin(ω·k)
cos(ω(pos+k)) = cos(ω·pos)cos(ω·k) − sin(ω·pos)sin(ω·k)
```

写成矩阵(对这个二维子空间):

```
⎡ sin(ω(pos+k)) ⎤   ⎡ cos(ωk)   sin(ωk) ⎤ ⎡ sin(ω·pos) ⎤
⎢               ⎥ = ⎢                   ⎥ ⎢            ⎥
⎣ cos(ω(pos+k)) ⎦   ⎣ −sin(ωk)  cos(ωk) ⎦ ⎣ cos(ω·pos) ⎦
```

右边那个 2×2 矩阵**只依赖 k(位移),不依赖 pos**,而且它恰好是一个**旋转矩阵**(行列式为 1、正交)。所以:**相对位移 k 在正弦编码的每个二维子空间里,就是旋转一个固定角度 ω·k。** 这说明正弦编码"内禀地"携带了相对位置的线性结构——记住这个"位移=旋转"的事实,第五节 RoPE 会把它从"加在输入上"改成"直接旋转 Q/K",一下子变干净。

### 3.3 这套方案的代价:相对性被埋没在加法里

正弦编码理论很漂亮,但有个实践上的硬伤,直接催生了后续所有改进。问题出在它是**加在输入上**的:`x'ᵢ = xᵢ + PE(i)`。我们关心的是注意力分数 `qᵢ · kⱼ`,展开看位置信息进去后变成什么(为聚焦,设投影近似恒等,直接看 (x+PE) 的内积):

```
(xᵢ + PE(i)) · (xⱼ + PE(j))
 = xᵢ·xⱼ              ← 纯语义项（和位置无关）
 + xᵢ·PE(j) + PE(i)·xⱼ ← 语义×位置 的交叉项（这两项让"内容"和"绝对位置"纠缠在一起）
 + PE(i)·PE(j)         ← 纯位置项
```

理论上 `PE(i)·PE(j)` 这一项是 i−j 的函数(由 3.2 的旋转性质可推),携带相对位置。**但它和另外三项混在一个标量里,模型拿不到"干净的相对位置信号",还得自己从这一锅交叉项里学着分离出来。** 而且经过 W_Q、W_K 投影后,上面这种干净分解还会被进一步打散。**根子上的别扭是:我们想要的是相对位置 i−j,但绝对编码把绝对位置塞进了输入,相对性只能作为副产品被动浮现。** 这就引出了下一代思路:与其指望相对性"涌现",不如**直接把相对位置编进注意力分数**。

### 3.4 代码:正弦编码

```python
import torch, math

def sinusoidal_pe(n_pos: int, d: int) -> torch.Tensor:
    """返回 [n_pos, d] 的正弦位置编码表，直接加到 token embedding 上。"""
    pe = torch.zeros(n_pos, d)
    pos = torch.arange(n_pos).unsqueeze(1).float()          # [n_pos, 1]
    # 频率 ω_i = 1 / 10000^(2i/d)，用 exp(log) 形式数值更稳
    i2 = torch.arange(0, d, 2).float()                      # 0,2,4,... 对应每一对
    omega = torch.exp(-(math.log(10000.0)) * i2 / d)        # [d/2]
    pe[:, 0::2] = torch.sin(pos * omega)                    # 偶数维放 sin
    pe[:, 1::2] = torch.cos(pos * omega)                    # 奇数维放 cos
    return pe                                               # 无可学习参数

# 用法：x = token_emb + sinusoidal_pe(seq_len, d)[:seq_len]
```

**分析(参数量/外推)**:正弦编码**零参数**,且因为是个解析函数,理论上能给任意大的 pos 算出编码——所以它**名义上支持外推**到训练长度之外。但实践中外推效果有限:模型在训练时只见过短序列对应的那段编码,W_Q/W_K 学到的处理方式未必能泛化到没见过的长波长相位组合,推到远超训练长度时注意力往往退化(具体退化点和模型相关,「待核」)。

## 四、第二代:可学习绝对位置编码(Learned Absolute)

更工程化、被 BERT / 原始 GPT 等大量采用的方案,干脆放弃解析公式:**给每个位置配一个可训练的向量,当成参数学**。

```python
class LearnedAbsolutePE(torch.nn.Module):
    def __init__(self, max_len: int, d: int):
        super().__init__()
        self.pe = torch.nn.Embedding(max_len, d)            # [max_len, d] 可训练
    def forward(self, x):                                   # x: [B, L, d]
        pos = torch.arange(x.size(1), device=x.device)
        return x + self.pe(pos)                             # 加到输入
```

**优点**:简单、灵活,让模型自己学位置该长什么样,不预设任何结构,在训练长度内往往比正弦略好(数据自己说话)。

**致命缺点——不能外推**:`Embedding` 表大小是 `max_len`,**位置 ≥ max_len 的向量根本不存在**,模型在那些位置上没有任何可用编码,只能报错或截断。这是个**硬上界**:训练时设 max_len=2048,推理时给 4096 长度直接崩。这一条**直接淘汰了它用于需要长上下文外推的现代大模型**。

**参数量**:max_len × d 个参数(如 2048 × 4096 ≈ 8.4M),不算大,但纯属"记忆"性质、不可外推,性价比在长上下文时代变差。

**一个常被忽略的坑**:可学习绝对编码下,模型可能学到"利用绝对位置作弊"的捷径(比如"答案常出现在第 5 个位置"),在分布变化时脆。相对编码因为只看相对距离,对这种平移天然更鲁棒。

## 五、第三代主线:相对位置编码与 RoPE(现代主流)

前两代都把"绝对位置"塞进输入,相对性只能被动涌现。第三代的中心思想是:**直接把"相对距离"编进注意力分数 `qᵢ·kⱼ`。** 这条线上有两个里程碑:Shaw et al.(2018)首次把相对位置作为可学习偏置加进 attention;T5(2020)用了一个简化的相对位置桶 bias。但真正统一并主导今天大模型的,是 **RoPE(Rotary Position Embedding,Su et al., 2021,RoFormer)**。我们把它推到尾。

### 5.1 目标:我们到底想要什么

把诉求写成一个明确的数学目标。我们希望找到一种对 query、key 的位置变换:把位置 m 处的 query 记成 `f(q, m)`、位置 n 处的 key 记成 `f(k, n)`,使得它们的内积**只依赖相对位置 m−n**,而不单独依赖 m 或 n:

```
⟨ f(q, m), f(k, n) ⟩ = g(q, k, m − n)     （目标方程）
```

只要做到这点,注意力分数 `qᵢ·kⱼ` 就**天然是相对位置的函数**,模型不必从交叉项里费劲分离——相对性是构造保证的,不是涌现的。**RoPE 给出的答案是:让 f 是"旋转"——把 q、k 按它们的位置旋转一个正比于位置的角度。** 下面证明旋转恰好满足目标方程。

### 5.2 二维情形:旋转为什么天然满足"只依赖 m−n"

先看最小的二维(d=2)情形,这是 RoPE 全部数学的核心,二维讲透了高维只是堆叠。

定义二维旋转矩阵(逆时针转角度 α):

```
        ⎡ cos α   −sin α ⎤
R(α) =  ⎢                ⎥
        ⎣ sin α    cos α ⎦
```

RoPE 的做法:位置 m 处的 query 旋转角度 m·θ,即 `f(q, m) = R(mθ)·q`;位置 n 处的 key 旋转 n·θ,即 `f(k, n) = R(nθ)·k`。这里 θ 是这个二维子空间的固定频率。现在算它们的内积:

```
⟨ R(mθ)q, R(nθ)k ⟩
 = (R(mθ)q)ᵀ (R(nθ)k)              ← 内积写成 行向量·列向量
 = qᵀ R(mθ)ᵀ R(nθ) k              ← 转置反序 (AB)ᵀ=BᵀAᵀ
 = qᵀ R(−mθ) R(nθ) k              ← 旋转矩阵正交：R(α)ᵀ = R(−α)
 = qᵀ R(nθ − mθ) k                ← 旋转可加：R(α)R(β) = R(α+β)
 = qᵀ R((n−m)θ) k
```

最后一行**只含 (n−m)**,query、key 各自的绝对位置 m、n 消失了!这就证明了目标方程:`⟨R(mθ)q, R(nθ)k⟩ = g(q, k, n−m)`。

这里用到的两个旋转矩阵基本性质,各一行就能验:

```
正交性  R(α)ᵀ = R(−α):  转置把 sinα 的符号位置对调，等价于 α→−α。
可加性  R(α)R(β) = R(α+β):  两次旋转相加，等价于把两个旋转矩阵相乘后用和角公式化简，
        得到的正是 R(α+β)（可手算 2×2 矩阵乘法 + cos(α+β)=cosαcosβ−sinαsinβ 验证）。
```

**这就是 RoPE 的全部魔法:旋转矩阵的"转置=反向旋转"+"相乘=角度相加",让两个旋转后向量的内积自动塌缩成相对角度 (n−m)θ 的函数。** 它把"相对位置"从一个需要模型去学的东西,变成了点积运算的代数恒等式。注意一个优雅之处:**RoPE 是乘性的(旋转/乘 q、k),不是加性的(像正弦那样加到输入)**——正因为是乘在 q、k 上、且旋转正交保范,绝对位置才能在内积里被干净地约掉。

### 5.3 高维:把 d 维拆成 d/2 个二维子空间,各转各的

真实 q、k 是 d 维(d 偶数)。RoPE 把它们**两两配对成 d/2 个二维子空间**,第 i 对用频率 θᵢ。整体旋转矩阵是分块对角的:

```
        ⎡ R(m θ₀)                       ⎤
        ⎢        R(m θ₁)                ⎥
R_m  =  ⎢               ⋱               ⎥     ∈ ℝ^{d×d}，分块对角，d/2 个 2×2 旋转块
        ⎢                  R(m θ_{d/2−1})⎥
        ⎣                               ⎦
```

频率沿用正弦编码的同款指数衰减:**θᵢ = 10000^(−2i/d)**,i = 0,…,d/2−1。低维(i 小)θ 大、转得快(高频,刻画局部),高维(i 大)θ 小、转得慢(低频,刻画长程)。因为每个块都是旋转,5.2 的推导对每一块独立成立,整体内积是各块内积之和:

```
⟨ R_m q, R_n k ⟩ = Σ_{i=0}^{d/2−1} ⟨ R(mθᵢ)q⁽ⁱ⁾, R(nθᵢ)k⁽ⁱ⁾ ⟩
                 = Σ_{i} qᵢᵀ R((n−m)θᵢ) kᵢ        ← 每块都只依赖 n−m
                 = h(q, k, n−m)
```

(q⁽ⁱ⁾ 是 q 的第 i 个二维分量。)**所以高维 RoPE 的内积仍只依赖相对位置 n−m,只不过是各频率子空间相对贡献的叠加。** 这正好对应一个直觉:近距离(n−m 小)时各频率都还没转开多少,内积保留得多;远距离时高频块已转过很多圈、贡献快速振荡相消,低频块还能携带粗距离信息——**模型因此天然获得了"近处看得细、远处看得粗"的距离感**。

### 5.4 为什么 RoPE 有一定外推能力(而不是无限)

RoPE 比可学习绝对编码外推好,原因有二,但要诚实说清它**不是无限外推**:

1. **它编的是相对量,函数对任意 m、n 都有定义。** 旋转角 m·θᵢ 对任意大的 m 都能算(不像 Embedding 表会越界),所以**形式上**能处理任意长序列,不会像可学习绝对编码那样硬崩。
2. **近距离行为在训练/测试间一致。** 注意力主要由小的 n−m(局部)主导,而小相对距离对应的旋转角 (n−m)·θᵢ 在训练时被充分见过,所以局部注意力模式能迁移。

**但外推有上限,原因是高频维度的"绕圈混叠"。** 高频块(θᵢ 大)在训练时,最大相对距离 N_train 对应的角度 N_train·θᵢ 可能还没绕满一圈;推到 N_test ≫ N_train 时,这些角度进入了训练中从未出现过的区间(绕了更多圈、相位组合是新的),模型没学过怎么处理,注意力分数分布漂移,性能下降。**所以朴素 RoPE 直接外推到远超训练长度通常会退化。** 这正是 NTK-aware scaling、Position Interpolation(PI)、YaRN 等"长度扩展"方法要解决的(它们或缩放位置、或调整 θ 的频率基数,把测试时的角度压回训练见过的范围)——这些是后续章节/进阶专题的内容,这里点到为止:**记住 RoPE 的外推是"有结构因而可被进一步扩展"的,而不是开箱即用的无限外推。**

### 5.5 代码:RoPE 的 PyTorch 实现

RoPE 落地有一个高效技巧:不去显式构造 d×d 的分块对角矩阵(那样浪费),而是用 sin/cos 向量配合一个"把每对维度交换并取负"的辅助操作,直接对 q、k 逐元素算。推导:对一个二维块 (x₀, x₁) 旋转角 α,结果是

```
旋转后 = (x₀ cosα − x₁ sinα,  x₁ cosα + x₀ sinα)
```

把整个 d 维向量的旋转写成逐元素形式:`x ⊙ cos + rotate_half(x) ⊙ sin`,其中 `rotate_half` 把后一半维度搬到前面并取负(对应"前后半配对":第 j 维与第 j+d/2 维构成一对，对每对做 (x_j,x_{j+d/2})→(−x_{j+d/2},x_j))。这是 HF transformers / LLaMA 的标准实现写法。

```python
import torch

def build_rope_cache(seq_len: int, d: int, base: float = 10000.0, device=None):
    """预计算 cos/sin 缓存。d 为 head_dim（必须偶数）。"""
    # θ_i = base^(-2i/d)，i=0..d/2-1
    inv_freq = 1.0 / (base ** (torch.arange(0, d, 2, device=device).float() / d))  # [d/2]
    pos = torch.arange(seq_len, device=device).float()                            # [L]
    freqs = torch.outer(pos, inv_freq)            # [L, d/2]，元素 = pos * θ_i = 旋转角
    emb = torch.cat([freqs, freqs], dim=-1)       # [L, d]，复制成前后两半（配合 rotate_half）
    return emb.cos(), emb.sin()                   # 各 [L, d]

def rotate_half(x):
    """把后一半维度取负搬到前面：实现每个二维块内 (x0,x1)->(-x1,x0)。"""
    x1, x2 = x.chunk(2, dim=-1)                    # 前半 / 后半
    return torch.cat([-x2, x1], dim=-1)

def apply_rope(q, k, cos, sin):
    """q,k: [B, n_head, L, d]；cos,sin: [L, d]，广播到前两维。"""
    cos = cos[None, None, :, :]                    # [1,1,L,d]
    sin = sin[None, None, :, :]
    q_rot = q * cos + rotate_half(q) * sin         # 对 q 每个二维块旋转 (pos*θ_i)
    k_rot = k * cos + rotate_half(k) * sin         # 对 k 同样旋转
    return q_rot, k_rot

# 用法（注意：RoPE 只作用于 Q、K，不动 V！value 不旋转，因为我们只想让"分数"携带相对位置）
# cos, sin = build_rope_cache(L, head_dim)
# q, k = apply_rope(q, k, cos, sin)
# attn = softmax(q @ k.transpose(-1,-2) / head_dim**0.5) @ v
```

**实现要点(也是新手坑)**：
- **RoPE 只旋转 Q 和 K,绝不旋转 V**。目标方程只约束 `⟨f(q,m), f(k,n)⟩`(即分数),value 的内容不应被位置扰动,旋转 V 会破坏语义。
- `rotate_half` 那个"取负+换半"等价于分块旋转,前提是 **cos/sin 也按相同方式复制成前后两半**(代码里 `torch.cat([freqs, freqs])`)。这套"前后半"配对方式和"相邻两维"配对方式数学等价、效果一样,但**两者不能混用**——cos/sin 的排列必须和 rotate_half 的拆分方式严格对应,否则旋转错乱,这是最常见的实现 bug。
- **不消耗显存存大矩阵**:cos/sin 缓存是 [L, d],可全模型/全层共享,RoPE **零可学习参数**。

**复杂度/参数量分析**:RoPE 给 q、k 各做一次逐元素乘加,额外开销 O(N·d)(相比注意力本身的 O(N²·d) 可忽略),**零新增参数**,cos/sin 缓存 O(N·d) 显存且可共享。这是它能无痛替换进任何注意力的工程原因之一。

### 5.6 对照组:ALiBi——不动 Q/K,直接给分数加距离惩罚

RoPE 把相对位置编进 q、k 的旋转;**ALiBi(Attention with Linear Biases,Press et al., 2021)** 走另一条更直接的路:**完全不碰 q、k,而是在 softmax 之前,给注意力分数按相对距离加一个线性惩罚偏置**:

```
score(i, j) = qᵢ · kⱼ / √d  −  s_h · |i − j|        # 因果场景常用 -s_h·(i-j)，i≥j
```

其中 `s_h > 0` 是每个注意力头固定的斜率(slope,不同头取不同值,按几何级数设定,如头 h 的斜率约 1/2^h「待核」),`|i−j|` 是相对距离。**含义极直白:两个 token 离得越远,这个分数就被减得越多,注意力越被压低——一个内建的"越远越不关注"的先验。** 注意几个特点:

- **它是加性偏置,加在分数上,不引入任何对 q/k 的变换、零参数**(斜率是预设常数,不训练)。
- **天然只依赖相对距离 |i−j|**,所以和 RoPE 一样满足"相对性",但实现上简单到只是给分数矩阵加一个由距离决定的下三角偏置矩阵。
- **外推更直接、更强**:因为偏置就是关于距离的简单单调线性函数,推到任意长度时这个函数定义良好且行为一致(不像 RoPE 有高频绕圈混叠问题)。ALiBi 论文主打的卖点正是**在短序列训练、长序列推理上外推性能优异**。

**ALiBi 的代价**:这个"越远越弱"是一个**很强且固定的归纳偏置**——它假设注意力应当随距离单调衰减。对很多语言任务这个先验对(局部更重要),但它**牺牲了表达力**:模型无法表达"我要重点关注很远处某个特定 token"(比如长文档里回指开头的实体),因为远处一律被线性压制。RoPE 没有这种单调衰减的硬约束,远近的相对关系是模型自己学的,表达力更强。**这就是两者的核心权衡:ALiBi 用更强的距离先验换更好的外推和更简实现;RoPE 用更弱的先验换更强的表达力,外推靠后续 scaling 技术补。** 现代主流(LLaMA 系等)选了 RoPE,但 ALiBi 在一些强调外推的模型(如早期 BLOOM、MPT「待核」)里被采用。

## 六、设计权衡与常见坑

- **位置编码注入的位置(加在输入 vs 乘在 Q/K vs 加在分数)决定了一切。** 绝对编码(正弦/可学习)加在输入,相对性是涌现的、被交叉项稀释;RoPE 乘在 Q/K 的旋转上,相对性是代数恒等式保证的;ALiBi 加在分数上,相对性是显式偏置。**记住这条"注入位置"的分类法,任何新位置编码方案都能往里套。**
- **"加性 vs 乘性"不是细节,是本质。** 正弦是加性(x+PE),绝对位置会污染 value 通路且和语义纠缠;RoPE 是乘性旋转且只作用 Q/K、保范,绝对量在内积里干净约掉。乘性 + 只动 Q/K + 保范,三者缺一,相对性就不再是恒等式。
- **RoPE 只转 Q/K 不转 V**(已在代码处强调,因太常踩再列一次):转了 V 等于给值向量注入位置噪声,破坏语义,且破坏"内积只依赖 m−n"的推导前提。
- **rotate_half 的配对方式必须和 cos/sin 排列严格一致。** "前后半配对"和"相邻两维配对"两种实现都对、数学等价,但混用必错。读不同代码库时先确认它用哪种,迁移权重时尤其要小心(配对方式不同的两份实现权重不能直接互换)。
- **可学习绝对编码不能外推,是硬上界,不是性能问题。** 位置 ≥ max_len 直接没有向量。需要长上下文/外推的场景从设计起就不该选它。
- **RoPE 不是开箱即用的无限外推。** 直接把训练 4K 的 RoPE 模型喂 32K 通常退化(高频混叠)。要长外推得上 PI / NTK / YaRN 等 scaling(后续专题)。把"RoPE 自带无限外推"当结论会出事。
- **θ 的基数(默认 10000)是个旋钮。** 长上下文模型常调大这个 base(如调到 1e6 量级)来拉长低频波长、缓解长程混叠——这本质就是 NTK-aware 思路的一种。改 base 要同步改训练和推理,不一致会崩。
- **数值与精度**:cos/sin 缓存建议用 FP32 算好再转低精度;在很长序列、低精度下,pos·θ 的相位精度损失可能让远距离旋转角失真,长上下文推理要留意(「待核」具体阈值)。

## 七、动手练习

1.(推导题,必做)**亲手证明自注意力的置换等变性 `Attn(PX)=P·Attn(X)`。** 不看正文,从 `X'=PX` 出发,逐步推 Q'、K'、S'、A'、O',显式写出每一步用到的性质(投影逐行、(PK)ᵀ=KᵀPᵀ、softmax 对行内重排等变、Pᵀ P=I)。然后回答:**带因果 mask 的解码器还满足这个等式吗?它为什么仍需要位置编码?** *提示:命门是第 4 步 `Pᵀ P=I` 让列置换和 V 的行置换抵消;因果 mask 提供"偏序"但不提供"距离",这两点要分开说。*

2.(推导题,核心)**推导 RoPE 的相对性 `⟨R(mθ)q, R(nθ)k⟩ = g(q,k,m−n)`。** 先在二维用 `R(α)ᵀ=R(−α)` 和 `R(α)R(β)=R(α+β)` 把内积化简到只含 (n−m)(这两条性质也要各自验证:前者看转置,后者用和角公式手算 2×2 矩阵乘法)。再推广到 d 维分块对角,说明为什么各二维块的相对性可以独立叠加,以及"高频块远距离贡献振荡相消、低频块携带粗距离"对应什么直觉。

3.(分析题)**对比正弦 / 可学习绝对 / RoPE / ALiBi 在外推上的表现,并排出强弱、给出机制原因。** 至少覆盖:(a) 可学习为什么是硬上界(Embedding 越界);(b) 正弦为什么名义可外推但实践有限(没见过的长波长相位组合);(c) RoPE 为什么"有结构的有限外推"(局部一致 + 高频绕圈混叠),需要什么技术扩展;(d) ALiBi 为什么外推最直接(距离的简单单调函数,处处定义良好),代价是什么(单调衰减先验牺牲表达力)。*提示:把每种方案归到"注入位置"分类(输入/Q-K/分数)+"加性 vs 乘性",外推差异基本由此决定。*

4.(编码题)**实现并对拍 RoPE 的相对性。** 用正文 `apply_rope` 实现,然后**数值验证**:随机取 q、k 和两组位置 (m,n) 与 (m+t, n+t)(平移同一个 t),对每组算 `(R_m q)·(R_n k)`,用 `torch.allclose` 确认两组内积相等(因为相对位置 n−m 不变)。再做一个反例:改成"也旋转 V"或"q/k 用不同 θ",观察相对性被破坏。*提示:这是把第二节目标方程落成代码的最短闭环,平移不变的内积就是 `⟨R_m q,R_n k⟩` 只依赖 m−n 的直接证据。*

## 八、源码 / 论文导读

- **原始 Transformer(正弦编码)**:Vaswani et al., *"Attention Is All You Need"*(NeurIPS 2017)。读 **Section 3.5 "Positional Encoding"**——就那几行 sin/cos 公式和"为什么选它(能表达相对位移、可外推)"的论证;它对相对位移=线性变换的说法,对应本章 3.2 的推导。
- **可学习绝对编码**:见 BERT(Devlin et al., 2018)与 GPT 系实现——位置就是一个 `nn.Embedding(max_len, d)` 加到 token embedding 上;重点体会 max_len 这个硬上界。
- **相对位置编码的开端**:Shaw et al., *"Self-Attention with Relative Position Representations"*(2018,首次把相对位置作为可学习项加进 attention);以及 **T5**(Raffel et al., 2020)的相对位置桶(relative position bucket)bias——读它怎么把相对距离分桶成可学习标量偏置。
- **RoPE(重点)**:Su et al., *"RoFormer: Enhanced Transformer with Rotary Position Embedding"*(2021)。重点读**第 3 节的"目标方程"(要求内积只依赖相对位置)和二维旋转的构造与证明**——和本章第五节逐步对得上;附录有从二维推广到高维的完整形式。
- **ALiBi**:Press et al., *"Train Short, Test Long: Attention with Linear Biases Enables Input Length Extrapolation"*(2021)。重点看**斜率 s_h 的几何级数设定**和**短训练长推理的外推实验**,这是 ALiBi 主打的卖点。
- **开源实现(最值得读)**:
  - **nanoGPT**(karpathy)用的是简单的可学习/正弦位置编码,适合先看最朴素的注入方式;
  - **HF transformers 的 LLaMA 实现**(`modeling_llama.py` 里的 `LlamaRotaryEmbedding`、`rotate_half`、`apply_rotary_pos_emb`)是 RoPE 的工业标准写法,本章 5.5 代码即按它组织,务必对照精读「文件名以仓库版本为准,待核」;
  - **x-transformers**(lucidrains)把正弦 / 可学习 / RoPE / ALiBi 都实现了一遍且高度模块化,是横向对比各方案最方便的单一仓库,强烈建议拉下来逐个开关跑。
- **长度扩展(进阶,知道它存在)**:Position Interpolation(Chen et al., 2023)、NTK-aware scaling(社区,2023)、YaRN(Peng et al., 2023)——都是在 RoPE 基础上做长外推,留待后续专题。
- 建议路径:Vaswani 3.5(正弦)→ 看一眼可学习编码的 max_len 上界 → RoFormer 第 3 节(RoPE 推导)+ LLaMA 实现精读 → ALiBi 论文(对照外推)→ 用 x-transformers 把四种开关跑一遍对比。

## 九、小结与承上启下

这一章我们做了一件具体的事:**把"Transformer 为什么必须有位置编码"从一句口号,变成一条能验算的推理链。**

```
自注意力对 token 排列是【置换等变】的（Attn(PX)=P·Attn(X)，命门 PᵀP=I）
  → 纯注意力对顺序失明，必须外部注入位置信息
  → 正弦绝对编码（多频 sin/cos，加在输入；相对性是涌现的、被交叉项稀释；名义可外推但有限）
  → 可学习绝对编码（每位置一个可训练向量；简单但 max_len 是硬上界，不能外推）
  → 相对位置编码主线：直接把 m−n 编进分数
  → RoPE（按位置旋转 Q/K：⟨R_m q,R_n k⟩=g(q,k,n−m)，靠旋转矩阵 R(α)ᵀ=R(−α)、R(α)R(β)=R(α+β)）
       · 乘性 + 只动 Q/K + 保范 → 相对性是代数恒等式，不是涌现
       · 有结构的有限外推（局部一致 + 高频绕圈混叠 → 需 PI/NTK/YaRN 扩展）
  → ALiBi（不动 Q/K，分数减 s_h·|i−j| 线性惩罚）：外推最直接，代价是单调衰减强先验、牺牲表达力
```

**最该带走的一句话:位置编码的本质,是回答"位置信息从哪进入 `qᵢ·kⱼ` 这个内积";谁能让相对位置成为内积运算的恒等式(RoPE)而非涌现的副产品,谁就赢了这一代。** "把想要的性质做成构造保证,而不是指望模型学出来"——这个设计哲学会在本课后续反复出现。

放到整门课的坐标里:上一章我们拆透了注意力机制本身(`softmax(QKᵀ/√d)V` 怎么算、为什么这么算);这一章给它补上了它天生缺失的"序"。**下一步**,我们会把视角从"单层注意力 + 位置编码"拉远到**完整的 Transformer 块与整体架构**(多头、残差、归一化位置 Pre-LN/Post-LN、FFN 设计),看这些组件怎么拼成一个能稳定训练到几百层、几千亿参数的整体——而你这一章建立的"等变性 / 注入位置 / 构造保证相对性"的视角,会是理解后面 decoder-only 架构、长上下文、乃至 KV cache(那里 RoPE 的旋转和缓存如何交互是个关键工程点)的地基。

### 练习 1:证明自注意力置换等变 + 因果 mask 为何仍需位置编码

**第一部分:完整推导 `Attn(PX) = P·Attn(X)`**

记输入 X ∈ ℝ^{N×d}(第 i 行是 token i)。P ∈ {0,1}^{N×N} 是置换矩阵,左乘 `PX` 把 X 的行重排。要用到 P 的两条性质:正交 `Pᵀ P = I`(即 `Pᵀ = P⁻¹`),且它只搬行列、不改元素值。

把打乱后的输入记为 `X' = PX`,逐步推导(省去常数 1/√d):

```
第1步  投影逐行(per-token)作用，行被 P 搬走，投影结果也跟着搬：
       Q' = X'W_Q = (PX)W_Q = P(XW_Q) = P Q       （矩阵乘法结合律）
       同理  K' = P K,  V' = P V
       用到的性质：投影是逐行线性映射，左乘 P 只换行不动列方向的运算。

第2步  打分矩阵：
       S' = Q'K'ᵀ = (PQ)(PK)ᵀ
          = (PQ)(Kᵀ Pᵀ)                            （转置反序 (PK)ᵀ = Kᵀ Pᵀ）
          = P (Q Kᵀ) Pᵀ
          = P S Pᵀ
       含义：S 的第 (i,j) 元被搬到 (π(i), π(j))，行、列同时按 P 置换。

第3步  按行 softmax。softmax 是逐行独立计算的；P S Pᵀ 相当于
       "把每一行整体搬到新位置，且行内元素也按同一置换重排"。
       softmax 对"行内重排"等变（先排再 softmax = 先 softmax 再排），故：
       A' = softmax(P S Pᵀ) = P · softmax(S) · Pᵀ = P A Pᵀ
       用到的性质：softmax 对行内元素重排等变。

第4步  输出（命门）：
       O' = A'V' = (P A Pᵀ)(P V)
          = P A (Pᵀ P) V                            （结合律）
          = P A I V                                 （Pᵀ P = I）
          = P (A V)
          = P O
```

**结论:`Attn(PX) = P·Attn(X)`。打乱输入顺序,输出只是被同一个 P 重排,每一行的数值分毫不变——这就是置换等变(equivariant),不是置换不变(invariant)。**

第 4 步的 `Pᵀ P = I` 是整个证明的命门:打分矩阵里被引入的**列置换 Pᵀ**,在与被同样行置换的 V(即 PV)相乘时正好抵消——顺序信息"自我湮灭"。注意力压根没有保留"谁在前谁在后"的任何痕迹。(这一等式可数值验证:取 N=4、随机 X 与随机 W_Q/W_K/W_V,任取置换 perm,`Attn(PX)` 与 `P·Attn(X)` 完全相等。)

推广到整层:逐位置 FFN/MLP、LayerNorm/RMSNorm 都是逐行算的,残差 `X + Sublayer(X)` 两项都等变、和仍等变。所以**不含位置信息时,任意多层 Transformer 编码器对 token 排列都等变**。

**第二部分:带因果 mask 的解码器还满足这个等式吗?**

**不满足任意置换等变。** 因果 mask M(下三角)显式规定"第 i 个位置只能看 ≤ i 的位置",打分变成 `S = QKᵀ + M`(被 mask 处置为 −∞)。把第 2、3 步代入会发现 mask 项 M 不是按 `P(·)Pᵀ` 跟着置换的——M 是一个**固定的、与位置绑定的**矩阵,换行换列后 `P M Pᵀ ≠ M`(除非 P 恰好是保持因果偏序的平凡置换)。所以 `Attn_masked(PX) ≠ P·Attn_masked(X)`:**mask 本身就编码了"谁能看谁"的顺序信息,破坏了等变性。**

**那它为什么仍需要位置编码?**——关键是把 mask 提供的信息和位置编码提供的信息**分开**说:

- mask 提供的是**偏序(partial order)**:它只告诉模型"j 在 i 前面(j ≤ i),i 看不到 j 之后"。
- mask **不提供距离(distance)**:它知道 j 在 i 前面,却**不知道隔了 1 个还是 100 个 token**。对所有满足 j ≤ i 的 j,mask 一视同仁地"放行",分不出远近。

而语言/代码的语义高度依赖**相隔多远**(相邻词 vs 隔一段)。因此即使是因果解码器,**仍需位置编码来补上 mask 缺失的"距离/绝对位置"这一维信息**。GPT 系既有因果 mask 又有位置编码,正是这个道理。**一句话:mask 给"方向/偏序",位置编码给"距离",二者是正交的两类信息,不能互相替代。**(这是本节最常被面试问倒的点:"因果 mask 能否替代位置编码?"——不能。)

### 练习 2:推导 RoPE 的相对性 `⟨R(mθ)q, R(nθ)k⟩ = g(q,k,m−n)`

**第一步:先验证两条旋转矩阵性质**

二维旋转矩阵(逆时针转 α):

```
        ⎡ cos α   −sin α ⎤
R(α) =  ⎢                ⎥
        ⎣ sin α    cos α ⎦
```

**(性质一)正交性 `R(α)ᵀ = R(−α)`。** 转置把非对角元 −sin α 与 sin α 对调位置:

```
        ⎡ cos α    sin α ⎤            ⎡ cos(−α)  −sin(−α) ⎤
R(α)ᵀ = ⎢                ⎥ ，而 R(−α)=⎢                   ⎥
        ⎣ −sin α   cos α ⎦            ⎣ sin(−α)   cos(−α) ⎦
```

用 cos(−α)=cos α、sin(−α)=−sin α,R(−α) 的四个元素正是 (cos α, sin α; −sin α, cos α),与 R(α)ᵀ 逐元素相等。**故 R(α)ᵀ = R(−α)** ✓(等价于 R 正交:R(α)ᵀR(α)=I)。

**(性质二)可加性 `R(α)R(β) = R(α+β)`。** 手算 2×2 矩阵乘法:

```
R(α)R(β) = ⎡ cosα −sinα ⎤⎡ cosβ −sinβ ⎤
           ⎣ sinα  cosα ⎦⎣ sinβ  cosβ ⎦

  (1,1) = cosα cosβ − sinα sinβ = cos(α+β)
  (1,2) = −cosα sinβ − sinα cosβ = −(sinα cosβ + cosα sinβ) = −sin(α+β)
  (2,1) = sinα cosβ + cosα sinβ = sin(α+β)
  (2,2) = −sinα sinβ + cosα cosβ = cos(α+β)
```

四个元素正好拼成 R(α+β)。**故 R(α)R(β) = R(α+β)** ✓(用了和角公式 cos(α+β)=cosαcosβ−sinαsinβ、sin(α+β)=sinαcosβ+cosαsinβ)。

**第二步:二维情形化简内积**

RoPE 把位置 m 处的 query 旋转 mθ:`f(q,m)=R(mθ)q`;位置 n 处的 key 旋转 nθ:`f(k,n)=R(nθ)k`。算内积:

```
⟨ R(mθ)q, R(nθ)k ⟩
 = (R(mθ)q)ᵀ (R(nθ)k)               ← 内积 = 行向量·列向量
 = qᵀ R(mθ)ᵀ R(nθ) k               ← 转置反序 (AB)ᵀ = BᵀAᵀ
 = qᵀ R(−mθ) R(nθ) k               ← 性质一 R(α)ᵀ = R(−α)
 = qᵀ R(−mθ + nθ) k                ← 性质二 R(α)R(β) = R(α+β)
 = qᵀ R((n−m)θ) k
```

**最后一行只含相对位置 (n−m),绝对位置 m、n 各自消失。** 即 `⟨R(mθ)q, R(nθ)k⟩ = g(q,k,n−m)`,目标方程成立。✓

直觉:旋转矩阵"转置=反向旋转"让 m 那次旋转被翻成 −mθ,再"相乘=角度相加"与 nθ 合成,绝对角度相减只剩相对角度 (n−m)θ。**RoPE 把"相对位置"从一个待学的东西,变成了点积运算的代数恒等式。**

**第三步:推广到 d 维(分块对角)**

真实 q、k 是 d 维(d 偶数),两两配对成 d/2 个二维子空间,第 i 对用频率 θᵢ = 10000^(−2i/d)。整体旋转 R_m 是分块对角矩阵(d/2 个 2×2 旋转块,第 i 块为 R(mθᵢ))。**为什么各块的相对性可以独立叠加**:分块对角矩阵乘向量 = 各块各自作用在对应的二维分量上,块与块之间没有耦合(非对角块全 0)。于是整体内积按子空间拆成和:

```
⟨ R_m q, R_n k ⟩ = Σ_{i=0}^{d/2−1} ⟨ R(mθᵢ)q⁽ⁱ⁾, R(nθᵢ)k⁽ⁱ⁾ ⟩
                 = Σ_i  q⁽ⁱ⁾ᵀ R((n−m)θᵢ) k⁽ⁱ⁾          ← 对每块套用二维结论
                 = h(q, k, n−m)
```

(q⁽ⁱ⁾ 是 q 的第 i 个二维分量。)每一块都只依赖 n−m,故整体也只依赖 n−m。**高维 RoPE 的内积仍只依赖相对位置,只是各频率子空间贡献的叠加。**

**第四步:"高频远距离振荡相消、低频携带粗距离"对应的直觉**

第 i 块的贡献含因子随 (n−m)·θᵢ 旋转。频率分布是指数衰减的:

- **低维 i(θᵢ 大,高频、转得快):** 相对距离 n−m 稍微变大,角度 (n−m)θᵢ 就已转过很多圈。远距离时它在单位圆上快速绕圈,正负贡献来回振荡、相互抵消(求和趋近平均),**对远距离几乎不携带可分辨的信息**——但近距离时各高频块还没转开,能精细区分相邻几个位置。这对应"**近处看得细**"。
- **高维 i(θᵢ 小,低频、转得慢,波长可达上万):** 即便 n−m 很大,角度 (n−m)θᵢ 仍只转了一小段,贡献是缓慢单调变化的,**能稳定携带"大致隔多远"的粗距离信息**。这对应"**远处看得粗**"。

**整体直觉:RoPE 让模型天然获得"近处分辨率高、远处分辨率低"的距离感**——这正是语言的合理先验(相邻词关系最关键,远处只需知道个大概),且无需任何额外参数,是构造里自带的。

### 练习 3:正弦 / 可学习绝对 / RoPE / ALiBi 外推对比

**统一分析框架(先建坐标系)**:把四种方案按两个维度归类,外推差异基本由此决定:

```
方案            注入位置        加性/乘性    相对性来源
─────────────────────────────────────────────────────────
正弦绝对         加在输入        加性        涌现(被交叉项稀释)
可学习绝对       加在输入        加性        几乎没有(纯记忆绝对位置)
RoPE            乘在 Q/K        乘性(旋转)   构造保证(代数恒等式)
ALiBi           加在分数        加性偏置     显式(距离的线性函数)
```

**(a) 可学习绝对:为什么是硬上界(不能外推)**

机制:每个位置 pos 配一个可训练向量,存在 `nn.Embedding(max_len, d)` 里。位置 ≥ max_len 时,**表里根本没有对应行,向量不存在**——不是"算出来不准",而是"压根没有"。训练设 max_len=2048,推理给 4096 直接越界报错/被迫截断。**这是设计上的硬上界,不是性能问题**,任何调参都救不了。需要长上下文外推的现代大模型从设计起就不该选它。(额外的脆性:模型可能学到"利用绝对位置作弊"的捷径,如"答案常在第 5 位",分布平移时崩。)

**(b) 正弦:为什么名义可外推但实践有限**

机制:PE(pos) 是个解析函数 sin/cos,**对任意大的 pos 都能算出编码**,所以**名义上(形式上)支持外推**,零参数。但实践有限:训练时模型只见过短序列对应的那段相位,W_Q/W_K 学到的"如何处理这些相位"未必能泛化到**没见过的长波长相位组合**;推到远超训练长度时,低频维度进入训练从未出现的相位区间,注意力往往退化。所以"能算编码"不等于"模型会用",退化拐点强依赖模型/任务(具体数值「待核」)。

**(c) RoPE:为什么是"有结构的有限外推",需要什么扩展**

机制——能外推的原因有二:① **编的是相对量,旋转角 m·θᵢ 对任意大的 m 都有定义**,不像 Embedding 表越界,形式上能处理任意长度,不硬崩;② **注意力主要由小的 n−m(局部)主导,小相对距离对应的旋转角在训练时被充分见过**,局部注意力模式能迁移。

但外推**有上限**,根因是**高频维度的绕圈混叠**:高频块(θᵢ 大)在训练时,最大相对距离 N_train 对应的角度 N_train·θᵢ 可能还没绕满一圈;推到 N_test ≫ N_train 时,这些角度进入训练中**从未出现过的相位区间**(绕了更多圈、相位组合是新的),模型没学过怎么处理,注意力分数分布漂移,性能下降。**所以朴素 RoPE 直接外推到远超训练长度通常退化**。需要的扩展技术:**Position Interpolation(PI)、NTK-aware scaling、YaRN**——它们或缩放位置、或调大频率基数 base(如把 10000 调到 1e6 量级「待核」),把测试时的角度压回训练见过的范围。关键认知:**RoPE 的外推是"有结构因而可被进一步扩展"的,不是开箱即用的无限外推**——把"RoPE 自带无限外推"当结论会出事。

**(d) ALiBi:为什么外推最直接,代价是什么**

机制:`score(i,j) = qᵢ·kⱼ/√d − s_h·|i−j|`,在 softmax 前给分数按相对距离加一个**线性单调惩罚**(s_h 是每头固定斜率,按几何级数设定,如头 h 约 1/2^h「待核」,零训练参数)。外推最直接的原因:**惩罚就是关于距离 |i−j| 的简单单调线性函数,对任意距离都定义良好、行为一致**——没有 RoPE 那种高频绕圈混叠问题,推到任意长度这个函数照常成立。这正是 ALiBi 论文"短训练、长推理"主打的卖点。

代价:这个"越远越弱"是一个**很强且固定的归纳偏置**,它假设注意力应随距离单调衰减。对很多语言任务这个先验对(局部更重要),但它**牺牲表达力**——模型**无法表达"重点关注很远处某个特定 token"**(如长文档回指开头的实体),因为远处一律被线性压制。

**结论:外推由强到弱排序与机制**

```
外推强弱：ALiBi  >  RoPE(+scaling)  >  正弦  >  可学习绝对
          最直接    有结构可扩展      名义可外推    硬上界(不能)
```

- **ALiBi 最强且最直接**:距离的单调线性函数处处定义良好,无混叠;代价是强先验牺牲表达力。
- **RoPE 居中**:相对量+局部一致让它"有结构地"外推,但高频混叠设了上限,需 PI/NTK/YaRN 把角度压回训练范围才能长外推。
- **正弦名义可外推但实践有限**:解析函数能算任意 pos,但没见过的长波长相位组合泛化差。
- **可学习绝对垫底**:max_len 硬上界,位置越界直接没有向量,根本不能外推。

**核心权衡一句话:ALiBi 用更强的距离先验换更好的外推和更简实现;RoPE 用更弱的先验换更强的表达力,外推靠后续 scaling 技术补。** 现代主流(LLaMA/Qwen/Mistral/DeepSeek 等)选了 RoPE。

### 练习 4:用代码对拍 RoPE 的相对性(含反例)

下面给两份等价实现:**numpy 版(CPU 直接可跑,已验算)** 和 **torch 版(对应正文 `apply_rope`,需 `pip install torch`,CPU 即可)**。核心思路:取固定 q、k,把位置对 (m,n) 同时平移 t 到 (m+t, n+t),相对位置 n−m 不变 → 内积应不变;再做两个反例展示相对性被破坏。

**numpy 版(零额外依赖,可直接运行)**

```python
import numpy as np

def build_rope_cache(seq_len, d, base=10000.0):
    """预计算 cos/sin，d 为 head_dim（必须偶数）。"""
    inv_freq = 1.0 / (base ** (np.arange(0, d, 2).astype(float) / d))  # θ_i, [d/2]
    pos = np.arange(seq_len).astype(float)                            # [L]
    freqs = np.outer(pos, inv_freq)                                   # [L, d/2] = pos*θ_i
    emb = np.concatenate([freqs, freqs], axis=-1)                     # [L, d] 前后半复制
    return np.cos(emb), np.sin(emb)

def rotate_half(x):
    """后半取负搬到前面：实现每个二维块 (x0,x1)->(-x1,x0)。"""
    d = x.shape[-1]
    x1, x2 = x[..., :d // 2], x[..., d // 2:]
    return np.concatenate([-x2, x1], axis=-1)

def apply_rope_vec(v, cos_row, sin_row):
    """对单个向量按某位置旋转：v⊙cos + rotate_half(v)⊙sin。"""
    return v * cos_row + rotate_half(v) * sin_row

np.random.seed(0)
L, d = 16, 8                       # 序列长、head_dim（偶数）
cos, sin = build_rope_cache(L, d)
q = np.random.randn(d)
k = np.random.randn(d)

def score(m, n, cq=cos, sq=sin, ck=cos, sk=sin):
    """旋转后 q、k 的内积 ⟨R_m q, R_n k⟩。"""
    qr = apply_rope_vec(q, cq[m], sq[m])
    kr = apply_rope_vec(k, ck[n], sk[n])
    return float(qr @ kr)

# ---- 正例：平移同一个 t，相对位置 n-m 不变 → 内积应相等 ----
m, n, t = 2, 5, 3
s1, s2 = score(m, n), score(m + t, n + t)
print(f"score(2,5)={s1:.6f}  score(5,8)={s2:.6f}  allclose={np.allclose(s1, s2, atol=1e-6)}")
# 不同相对距离应当不同（佐证内积确实随 n-m 变化）
print(f"score(2,6)={score(2,6):.6f}  (相对距离变了，应与上面不同)")

# ---- 反例1：q、k 用不同 θ（不同 base），相对性被破坏 ----
cos_k, sin_k = build_rope_cache(L, d, base=500.0)   # k 用不同频率
b1 = score(2, 5, ck=cos_k, sk=sin_k)
b2 = score(5, 8, ck=cos_k, sk=sin_k)
print(f"[反例:不同θ] score(2,5)={b1:.4f}  score(5,8)={b2:.4f}  allclose={np.allclose(b1, b2, atol=1e-6)}")

# ---- 反例2：把 V 也旋转（这里用 k 充当 value），相对性同样不再纯由 n-m 决定 ----
# 概念演示：RoPE 只该转 Q/K；若对 value 也旋转，⟨R_m q, R_n k⟩ 仍依赖 n-m，
# 但注意力输出 A·(R V) 会把绝对位置注入 value 通路 → 破坏语义。
# 这里直接展示"用不同 θ"已足以打破相对性恒等式。
```

**正例平移后内积完全相等 → 直接证明 `⟨R_m q, R_n k⟩` 只依赖 m−n;反例中 q/k 用不同 θ 后平移不再相等 → 相对性被破坏。** 这就是把第二节目标方程落成代码的最短闭环。

**torch 版(对应正文实现,需 `pip install torch`,CPU 可跑)**

```python
import torch

def build_rope_cache(seq_len, d, base=10000.0, device=None):
    inv_freq = 1.0 / (base ** (torch.arange(0, d, 2, device=device).float() / d))  # [d/2]
    pos = torch.arange(seq_len, device=device).float()                            # [L]
    freqs = torch.outer(pos, inv_freq)            # [L, d/2]
    emb = torch.cat([freqs, freqs], dim=-1)       # [L, d]
    return emb.cos(), emb.sin()

def rotate_half(x):
    x1, x2 = x.chunk(2, dim=-1)
    return torch.cat([-x2, x1], dim=-1)

def apply_rope(q, k, cos, sin):
    """q,k: [B, n_head, L, d]; cos,sin: [L, d] 广播到前两维。"""
    cos = cos[None, None, :, :]
    sin = sin[None, None, :, :]
    return q * cos + rotate_half(q) * sin, k * cos + rotate_half(k) * sin

torch.manual_seed(0)
B, H, L, d = 1, 1, 16, 8
cos, sin = build_rope_cache(L, d)
q = torch.randn(B, H, L, d)
k = torch.randn(B, H, L, d)
qr, kr = apply_rope(q, k, cos, sin)

m, n, t = 2, 5, 3
s1 = (qr[0, 0, m] * kr[0, 0, n]).sum()          # ⟨R_m q, R_n k⟩
s2 = (qr[0, 0, m + t] * kr[0, 0, n + t]).sum()  # 平移 t 后，相对位置不变
print("平移前后相等:", torch.allclose(s1, s2, atol=1e-5))   # -> True

# 反例：对 q、k 用不同 base 的 cos/sin（破坏"同一 θ"前提）
cos_k, sin_k = build_rope_cache(L, d, base=500.0)
qr2 = q * cos[None, None] + rotate_half(q) * sin[None, None]
kr2 = k * cos_k[None, None] + rotate_half(k) * sin_k[None, None]
b1 = (qr2[0, 0, m] * kr2[0, 0, n]).sum()
b2 = (qr2[0, 0, m + t] * kr2[0, 0, n + t]).sum()
print("不同θ时仍相等?:", torch.allclose(b1, b2, atol=1e-5))  # -> False
```

**两份实现结论一致:相对性是 RoPE 的代数恒等式(平移不变),前提是 Q/K 用同一组 θ 旋转、且只动 Q/K。一旦 q/k 用不同 θ(或对 value 也旋转把绝对位置引入值通路),这个恒等式就不再成立。** 关于"旋转 V"的反例,核心要点不在内积本身(⟨R_m q, R_n k⟩ 仍只依赖 n−m),而在于:若把旋转也施加到 value,则注意力输出 `A·(R_n V)` 会把**绝对位置 n** 注入 value 通路,污染语义、并破坏"分数只依赖相对位置"的整体设计——所以 **RoPE 只旋转 Q/K,绝不旋转 V**。
