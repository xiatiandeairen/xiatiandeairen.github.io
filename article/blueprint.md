# Blueprint: skill-experience

v1 唯一骨架。所有 draft 都按此结构填充。

## 五节骨架

```
# {title}                          # 一行标题，含核心反共识或具体场景

## 起源: 遇到的真实问题
- 不是"我想分享…"，而是"我遇到了…"
- 必须有具体场景：什么任务 / 哪一类输入 / 当时怎么卡住
- 1-3 段

## 尝试过的方案
- 至少 1 个被你否决的方案，写明否决理由
- 不是"我直接想到了正确答案"——读者要看到选择过程
- 1-3 段，可分小节

## 踩的坑
- 至少 1 个非显然的坑（即"看起来对、跑起来错"）
- 写出"为什么会以为这样行"——元认知比错误本身更有价值
- 1-3 段

## 最终设计
- 当前用的方案，不是"完美方案"
- 明确说明取舍：放弃了什么、为什么可以接受
- 给出可读者复现的关键代码片段或配置 snippet
- 2-4 段

## 反共识 takeaway
- 必须包含 1 条"和直觉/主流做法相悖"的结论
- 加一条"何时不要这样做"的边界
- 1-2 段，简短有力
```

## AI 扩写 Prompt（draft 时使用）

```
你正在为 article skill 生成一篇"AI skill 经验类"文章草稿。

输入：
- topic: {用户给的主题}
- core opinion: {用户给的核心观点}
- 素材: {inbox/ 中筛出的相关 .md 内容}

要求：
1. 严格按 blueprint.md 五节结构填充。每节标题不得改写。
2. **不编造**：所有具体例子、数字、命令、引用必须来自素材或用户观点；找不到对应素材的段落，写 `<!-- TODO: 需补充具体例子 -->` 占位。
3. **用户观点优先**：core opinion 必须出现在"反共识 takeaway"节并展开。
4. **去 AI 腔**：禁用以下短语——"在当今快速发展的"/"综上所述"/"让我们深入探讨"/"重要的是要"/"无论…还是…"长排比。
5. **具体化**：抽象论述每出现一段，下一段必须给具体例子（命令、代码、对话片段、数字）。
6. **元认知优先**：在"踩的坑"节解释"为什么会以为这样行"，比"做错了什么"更有价值。
7. **长度控制**：总字数 1500-3000 中文字符。每节 200-700 字。
8. **frontmatter 必填**：article-ctl.sh 模板生成 frontmatter 骨架后，你必须用 Edit 工具把以下字段改为实值，**不得留 pending / 0 / 空字符串**：

| 字段 | 规则 |
|------|------|
| `title` | 改为实际标题字符串，≥1 字符 |
| `question.type` | 从 `["方法论", "系统设计", "工程实践", "AI 工程化"]` 中选一个 |
| `question.subType` | 更细类；无则删此行 |
| `quality.overall/coverage/depth/specificity` | 填 1-10 整数（实话自评，不凑高分）|
| `quality.reviewer` | `"ai"`（AI 扩写默认）|
| `analysis.objectivity.{factRatio,inferenceRatio,opinionRatio}` | 三数之和必须 = 1.0（允差 0.01）；经验类典型：`0.6 / 0.25 / 0.15` |
| `analysis.assumptions` / `analysis.limitations` | 字符串数组；空可保持 `[]` |
| `tags` | ≥1 条，形如 `[{name: "github"}, {name: "ci", parent: "engineering"}]`；**对象数组，不是字符串数组** |
| `topics` | ≥1 条，形如 `[{name: "AI 工程化"}, {name: "开发工具"}]` |
| `review.status` | draft 阶段固定 `"draft"` |
| `gate_passed` | 固定 `false`（gate 阶段用户勾选通过后才改 `true`）|

输出：完整的草稿 markdown，含五节内容（不要把这条 prompt 写进去）。
```

## 风格参考

参考站点 `src/content/notes/` 已有文章（如 12-ai-agent-os.md, 13-sprint-pipeline.md）的语气：
- 第一人称，直接说"我"
- 短句，少长定语
- 例子在前，观点在后
- 反共识结论用粗体强调
