# Roadmap

## 版本策略

| 版本 | 目标 | 状态 |
|------|------|------|
| **v1** | 将当前功能打磨至成熟 | 🔄 进行中 |
| v2 | 新功能扩展（待 v1 完成后规划） | ⏳ 待定 |

## v1 — 功能打磨

**完成标准：**
- 所有页面在 375px / 768px / 1440px 三个断点下无布局问题
- 暗色模式下无硬编码颜色残留
- Lighthouse Performance ≥ 90, Accessibility ≥ 90

---

### Phase 1 — 基础可用

> 让每个页面在所有设备上都不 broken。

| # | 任务 | 说明 | 状态 |
|---|------|------|------|
| 1.1 | 全站响应式适配 | 双栏在 ≤768px 改单栏；导航在移动端折叠；首页特稿区改为堆叠布局 | ⬜ |
| 1.2 | 暗色模式完整适配 | 清理 about/archive/index 中 inline style 的硬编码颜色；prose 样式暗色适配；确保所有颜色走 CSS 变量 | ⬜ |
| 1.3 | 关于页真实内容 | 替换占位符为真实个人简介、联系方式、GitHub 链接 | ⬜ 需用户提供 |

### Phase 2 — 阅读体验

> 让来了的读者愿意多读几篇。

| # | 任务 | 说明 | 状态 |
|---|------|------|------|
| 2.1 | 文章目录导航 (TOC) | 从 markdown heading 自动提取；文章详情侧栏显示；滚动时高亮当前章节 | ⬜ |
| 2.2 | 系列文章入口 | 新增 `/series` 页面；AI Skills 7 篇文章有独立入口和阅读顺序；文章详情页显示系列导航 | ⬜ |
| 2.3 | 相关文章推荐 | 文章底部基于标签重合度推荐 2-3 篇；无足够相关文章时不显示 | ⬜ |
| 2.4 | 代码块增强 | 语言标签（右上角）；一键复制按钮；复制成功反馈 | ⬜ |

### Phase 3 — 细节打磨

> 从"能用"到"好用"。

| # | 任务 | 说明 | 状态 |
|---|------|------|------|
| 3.1 | 搜索体验优化 | 结果关键词高亮；空状态引导文案；结果卡片样式对齐设计语言 | ⬜ |
| 3.2 | 标签/主题页样式对齐 | 用新的排版层级（type scale、间距、hover 交互）统一这两个页面 | ⬜ |
| 3.3 | 归档页 URL 参数 | 筛选状态写入 URL query string；支持分享特定分类视图；浏览器后退保留筛选 | ⬜ |
| 3.4 | 清理示例数据 | 删除 01-11 号原始示例笔记，或替换为真实内容 | ⬜ |
| 3.5 | Open Graph 图片 | 自动生成文章分享卡片图（标题 + 站名 + 日期） | ⬜ |
